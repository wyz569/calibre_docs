Alignment Dialog Box

To access: View > Alignment

The Wi2Mi Alignment dialog box allows you to adjust the overlay between mask and wafer inspection data.

Figure 1. Alignment Dialog Box With Automatic Tab

Objects

Table 1. Alignment Dialog Box Contents

Field

Description

Stacking Fields

Specifies how many dies in a field cycle and which die in the wafer data is the base die. See “Wafer Stacking”.

Alignment

Scale Reflection Rotation

As described in “Alignment”.

Manual Tab

Contains fields for specifying an x offset and a y offset. The units are as specified in the UnitOfMeasure node in the dat-ini.xml file.

Semi-Automatic Tab

Specifies the coordinates of a mask defect and a wafer defect, and automatically calculates offset. You can either enter the coordinates in the x and y fields of the MI defect and WI defect fields, or click Select Mi and Wi Defects and then click in the Defect Map window.

Automatic Tab

Specifies the coordinates of a mask defect and a wafer defect, and automatically calculates offset. You can either enter the coordinates in the x and y fields of the MI defect and WI defect fields, or click Select Mi and Wi Defects and then click in the Defect Map window.

Buttons

Align — Aligns mask and wafer data using the values in the rest of the dialog. Does not perform any analysis.

Align and Correlate — Aligns mask and wafer data, and identifies common defects in the Repeatability Analysis display.

Reset — Sets the fields in the Alignment dialog box to the default values. Text on page 583 says “that is the value which was displayed before any modification.”

These are set in the wi2mi-settings.xml file.

Usage Notes

Until data is correlated with the current settings, the Align and Correlate button is green. After you correlate data, the button returns to grey.

Once the mask and wafer data have been aligned, the offset fields reset to 0.
