Specifying Advanced Netlist Options

The advanced controls for the extracted parasitic netlist include options specific to the output netlist format.

Procedure

  1. Click the PEX Options button on the left panel, or choose Setup > PEX Options if the button is not present.
  2. Click the Netlist > Advanced sub-tab and made the desired selections. Table 1 describes the available options.
    Table 1. Advanced Netlist Format Options

    Selection

    Comment

    DSPF

    Eliminate instance section

    As described.

    NOINSTANCESECTION keyword for PEX Netlist

    Suppress top-level subcircuit

    As described.

    NOTOPSUBCKT keyword for PEX Netlist

    SPEF

    Generate name map

    Enable to generate a list that maps instance names to numerical identifiers.

    MAPNAMES keyword for PEX Netlist

    DSPF/SPEF

    Set bus delimiter

    Enter a string to set the BUS_DELIMITER variable in the DSPF or SPEF header. The default is brackets ( [] ).

    “BUSDELIM string” keyword and value for PEX Netlist

    HSPICE

    Redefine pin to subcircuit delimiter

    _ (entry) (enter a character, _ is the default)

    PINNAMESEP keyword for PEX Netlist

    Use short pin names

    Enable to cause the formatter to replace pin names with an integer when creating SPICE net names.

    SHORTPINNAMES keyword for PEX Netlist

    CALIBREVIEW

    Use iCellMap

    Enable to use the iCellMap syntax for the CalibreView netlist.

    ICELLMAP keyword for PEX Netlist

    Cellmap File

    Enable the checkbox and enter a Calibre View cellmap file in order to use it in the Calibre View flow.

    If Calibre Interactive was invoked from Cadence Virtuoso, the cellmap file is entered into the “Cellmap File” field in the “Calibre View Setup Dialog Box” (Calibre > Setup > Calibre View) during the PEX run. However, if the Calibre View Setup dialog box is open at runtime, the setting in the Calibre View Setup dialog box takes precedence and it is not updated with the Calibre Interactive setting.

    General

    Remove X in the instance names

    Enable to cause the formatter to filter X’s in HSPICE, DSPF, CALIBREVIEW, SPEF, and Spectre output.

    NOINSTANCEX keyword for PEX Netlist

    Retain X in flattened instance names

    Enable to cause the formatter to retain X’s in DSPF and CALIBREVIEW output.

    LEADINGX keyword for PEX Netlist

    Remove devices not attached to selected nets

    Enable to remove all intentional devices not attached to the extracted nets.

    PRUNE keyword for PEX Netlist

    Scale parasitic resistor properties

    Enter a value for the scaling factor for parasitic resistor properties.

    RSCALE value keyword and value for PEX Netlist

    Output temperature coefficients

    As described. This option is ignored for CalibreView and SPEF netlist formats.

    TCOEFF keyword for PEX Netlist

    Combine transistor-level SPICE data

    Enable to combine the transistor-level SPICE data, the subcircuit definitions of the parasitic net models for each extracted net, the parasitic resistors, the parasitic capacitors, and the instances of the net models into a single parasitic netlist output file. This option is only valid for HSPICE, ELDO, and Spectre netlist formats.

    SINGLEFILE keyword for PEX Netlist

    Create new instance names for smashed devices

    Enabled — Create new instance names for smashed devices.

    Disabled — Leave smashed device instance names unchanged.

    Default: Enabled.

    PEX Netlist Create Smashed Device Names

    Upper case element names, HSPICE and DSPF keywords

    As described.

    Default: Disabled.

    PEX Netlist Uppercase

    Connect disjoint net model fragments

    Enabled — Connect disjoint net model fragments created by Virtual Connect statements to the main net model using a resistor. This option is typically used for debug purposes.

    Disabled — Leave net model fragments unconnected.

    PEX Netlist Virtual Connect

    Report contact/via area

    The contact/via area that is used for extracting parasitic resistors is reported as a comment. This option applies to DSPF, SPEF, and HSPICE netlists.

    RAREA keyword for PEX Netlist
