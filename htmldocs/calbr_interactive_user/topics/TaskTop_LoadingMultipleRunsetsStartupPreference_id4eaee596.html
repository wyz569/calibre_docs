Loading Multiple Runsets with a Startup Preference

You can load one or more runsets into Calibre Interactive using a startup preference. When multiple runsets are used, they are loaded in order and the settings loaded last take precedence.

Tip

You can also load multiple runsets from the command line or with the environment variable MGC_CALIBRE_***_RUNSET_FILE. See “Loading Runsets on Invocation” for the order of precedence of these different methods. The startup preference has the lowest priority.

You can also load multiple runsets from the GUI. Select File > Load Runset and enter a space-separated list of runset names in the Runset File Path field.

Prerequisites

  • You have a Calibre Interactive runset for your application. Multiple runsets may be specified.

Procedure

  1. Open Calibre Interactive.
  2. Choose Setup > Preferences.
  3. Choose the Startup tab.
  4. Enable “Load runset files.”
  5. Specify a runset in the text field.
  6. (Optional) If specifying multiple runsets, do the following:
    1. Click the down arrow to expand the file entry text box. Figure 1 illustrates this step.
    2. Enter the runset filenames, one per line.
  7. Exit Calibre Interactive.

Results

The “Startup Runset” preference setting is automatically saved with your other startup preferences. The specified runset or set of runsets is loaded the next time you start Calibre Interactive. Multiple runsets are loaded in order and the settings loaded last take precedence.
