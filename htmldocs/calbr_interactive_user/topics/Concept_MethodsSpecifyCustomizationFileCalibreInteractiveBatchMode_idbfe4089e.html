Methods to Specify the Customization File for Calibre Interactive Batch Mode

You can specify the customization file for a batch mode run in the runset or on the command line.

Requirements

  • A runset for the batch mode run.

  • A customization file. See “Customization Language” and “Customization Commands”.

Methods

Choose one of the following methods to specify a customization file for a Calibre Interactive batch mode run:

  • Runset specification (Customization GUI does not open)

  • Command line specification with ‑customgui (Customization GUI opens)

  • Command line specification with ‑custom (Customization GUI does not open)

Runset specification (Customization GUI does not open)

  1. In Calibre Interactive, choose Setup > Preferences from the main menu, then click the Startup tab.

  2. Enable the “Customization File” checkbox and specify the filename of the customization file.

  3. Save the runset with File > Save Runset.

  4. Run Calibre Interactive in batch mode; for example:

    calibre -gui -lvs -runset my_runset ‑batch

    When the runset is used to run Calibre Interactive in batch mode, the initial settings in the customization file are applied to the Calibre run; the Customization GUI does not open. If the runset contains customization override settings, those take precedence over the initial settings in the customization file. If there is a callback procedure specified with CUSTOM::setOKCallback, the callback procedure is executed automatically.

Command line specification with ‑customgui (Customization GUI opens)

  • Use the “‑customgui filename ‑batch” options on the command line. For example:

    calibre -gui -lvs -runset my_runset -customgui my_custom -batch

    The Customization Settings dialog box appears at the beginning of the Calibre run. Once the customization selections are made and the dialog box is closed, the Calibre run continues in batch mode, without opening the Calibre Interactive GUI.

    Also see “Saving Customization GUI Settings in Calibre Interactive Batch Mode Runs”.

Command line specification with ‑custom (Customization GUI does not open)

  • Use the “‑custom filename ‑batch” options on the command line. For example:

    calibre -gui -lvs -runset my_runset -custom my_custom -batch

    The initial settings in the customization file are applied and used in the batch run; the Customization Settings dialog box does not open. If the runset contains customization override settings, those take precedence over the settings in the customization file. If there is a callback procedure specified with CUSTOM::setOKCallback, the callback procedure is executed automatically.
