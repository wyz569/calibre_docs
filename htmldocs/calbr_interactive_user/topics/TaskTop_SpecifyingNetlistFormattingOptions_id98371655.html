Specifying Netlist Formatting Options

The netlist formatting options specify how the netlist is constructed.

Procedure

  1. Click the PEX Options button on the left panel, or choose Setup > PEX Options if the button is not present.
  2. Click the Netlist > Format sub-tab and make selections as desired. The following table describes the available options.
    Table 1. PEX Netlist Formatting Options

    Selection

    (SVRF Command)

    Comment

    Ground node name

    (GROUND keyword in PEX Netlist …)

    Specify the name of the ground node in the PEX Netlist command. All intrinsic capacitors are grounded to this node. The default node name is 0.

    Ground layer name

    (PEX Ground Layer)

    Specifies layers with shapes for ground regions. Only used in C+CC and R+C+CC extraction.

    Enable the “Stray” checkbox to add the STRAY keyword; this eliminates all netlist output of intrinsic capacitance to substrate for the specified ground region.

    Hierarchy separator

    (SEPARATOR keyword in PEX Netlist …)

    Use this option to change the character used to denote hierarchical separation in the extracted parasitic netlist.

    Output intentional device locations

    (LOCATION keyword in PEX Netlist …)

    Device locations appear in the extracted parasitic netlist as comment-coded parameters ($X=x_value $Y=y_value)

    Output values for intentional R,C devices

    (RCNAMED keyword in PEX Netlist)

    R and C values of intentional devices appear in the netlist for HSPICE, DSPF, and Eldo netlists.

    Output multiple net ports from multiple text ports

     (MULTITXTPORT keyword in PEX Netlist)

    Generates multiple ports on the net from multiple text ports with the same name in the layout. Only used with the DSPF netlist format.

    Enclose string properties in quotes in the netlist

    (STRINGQUOTES keyword in PEX Netlist)

    Specifies whether or not to enclose string properties in quotes (“ ”) in the netlist”. By default, string properties are enclosed in quotes.

    Output R

    (adds keywords to PEX Netlist)

    Choose this option to include comments in the extracted parasitic netlist which specify additional parasitic resistor information. Select from:

    locations — keyword RLOCATION

    comment: $x=x_coord $y=y_coord $x2=x2_coord $y2=y2_coord

    widths — keyword RWIDTH

    comment: $w=width

    minimum — keyword MINIMUM

    Specifies to report the minimum width for non-rectangular shapes, otherwise the average width is reported. Must be specified with “widths”; “minimum” and “actual” may not be used at the same time.

    lengths — keyword RLENGTH

    comment: $l=length

    layers — keyword RLAYER

    comment: $layer=layer_name

    thicknesses — keyword RTHICKNESS

    actual — keyword RACTUAL. When selected, actual parasitic resistor widths are reported with the keyword RWIDTH; otherwise the drawn width is reported.

    viacount — keyword RVIACOUNT

    comment: $count=number

    layermap — keyword LAYERMAP, only available with DSPF format. When specified with the “locations” option, the resistor locations are reported as a bounding box with labels $llx, $lly, $urx, and $ury.

     

    If you specify resistor reduction, choosing “layers” prevents combining resistors on different layers.

    Note: Parasitic resistor information is not specified if you choose both “No R/C” and “No Inductance” in the Extraction Type selections on the Outputs pane.

    Output C

    (adds keywords to PEX Netlist)

    Choose this option to include comments in the extracted parasitic netlist which specify additional parasitic capacitor information. Select from:

    locations — keyword CLOCATION

    Extract floating nets

    (PEX Extract Floating Nets)

    Specify how floating nets such as metal fill are handled during extraction. Select from:

    ALL

    REDUCED

    GROUNDED — (default) A text entry field for the scale factor is available; enter a value greater than 0, or use the default value of 1.

    Ignore port text — Ignore port text placed on floating nets during extraction.

    Size multiplier for length units (in meters)

    (scale parameter for output netlist format in PEX Netlist …)

    Value must be a positive floating-point number.

    This specifies the size multiplier (in units of meters) for the L, W, A, P, AS, AD, PS, and PD parameters of MOSFET, diode, and JFET devices. Set the size multiplier to 1e-6 meters to see lengths parameters in microns and areas in square microns. Default is 1.

    Formatter map file:

    (PEX BA Mapfile)

    Specify a backannotation mapfile that contains the device, pin name, and model mappings used to map the extracted intentional devices in a Calibre View, DSPF or HSPICE netlist to their corresponding simulation models.

    Suppress LPE values from CalibreView netlist — As described; adds the CALIBREVIEW keyword.

    Enable corner extraction

    (calibre -xrc -fmt -corner)

    (calibre -xact -fmt -corner)

    Select to output corners in the netlist. Select from:

    All — (default)

    Specify — A text entry box appears; enter the specified corner names in a space or comma-separated list.

     

  3. (Optional) Set options to control formatter message output:
    1. Click the Misc tab on the PEX Options pane.
    2. Enable “Display formatter warnings” to add the ‑fmt_warnings option to the command line for the formatting stage.
    3. Enable “Display formatter information” to add the ‑fmt_info option to the command line for the formatting stage.
