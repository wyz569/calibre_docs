How Calibre Interactive Uses Runset Data

Calibre Interactive treats runset data differently depending on whether you invoke the tool from a design tool or from the command line.

If you open the Calibre Interactive interface from the command line, loading a runset populates the GUI with the data in the runset. Unless you manually change these values, Calibre Interactive uses them when generating the control file for the run.

If you open Calibre Interactive from a design viewer, loading a runset populates the GUI with all the data in the runset, except the names of the system-generated input and output files. For these files, Calibre Interactive generates names based on the templates that are defined through Calibre Interactive preferences settings. Refer to “Templates for File Naming” for more information on name-generation templates. Calibre Interactive combines the generated filenames with the other runset settings when generating the control file for the run.

Calibre Interactive has a certain set of default values. You can create a default runset if you want different default values. Default runsets are loaded only when no other runset is loaded.
