Environment Variables for Calibre Interactive

Several environment variables are available to control the behavior of Calibre Interactive.

The following table lists environment variables used in Calibre Interactive. The Type column indicates whether the environment variable is available in the shell environment (env) or in SKILL scripts.

Table 1. Calibre Interactive Environment Variables

Variable

Type

Description

CALIBRE_CI_REMOTE_COMMAND_USE_BASH

env

When set, use the Bash shell to launch the MTflex master script.

See “Configuring for Distributed (MTflex) Execution” for this and the next two environment variables.

CALIBRE_CI_REMOTE_COMMAND_LSF_USE_BASH

env

When set, use the Bash shell to launch the MTflex master script for Spectrum LSF runs only.

CALIBRE_CI_REMOTE_COMMAND_GRID_USE_BASH

env

When set, use the Bash shell to launch the MTflex master script for Univa Grid Engine runs only.

CALIBRE_HOME

env

Specifies the software tree invoked when Calibre Interactive or Calibre RVE is called from Pyxis (see “Using Calibre Interactive with Pyxis”).

Also sets the location of the software tree for Calibre tools; see “Setting the CALIBRE_HOME Environment Variable” in the Calibre Administrator’s Guide.

CDS_NETLISTING_MODE

env or

SKILL

Sets netlisting export mode in Cadence Virtuoso (“Netlist Export in Cadence Composer”).

MGC_CALGUI_RELEASE_LICENSE_TIME

env

Sets the time after which the Calibre Interactive license is released if there is no activity (“License Timeout in Calibre Interactive”). The default time unit is hours.

MGC_CALIBRE_ALLOW_VIEWER_TRIGGERS

env or SKILL

Specifies access level for triggers running in the design tool. See “Trigger Access to the Design Tool Environment”.

MGC_CALIBRE_CGIDB_DIR

env

Sets location of preference files (“Startup Preferences”).

MGC_CALIBRE_CUSTOMIZATION_FILE

env

Sets location of customization file; see “Customization Files”.

MGC_CALIBRE_DB_READ_OPTIONS

env

Sets database translation options for OpenAccess and LEF/DEF database. This environment variable must be set when using a Calibre Interactive control file to perform a command line Calibre run if the layout database format is OpenAccess or LEF/DEF. (“MGC_CALIBRE_DB_READ_OPTIONS Environment Variable for Database Options”)

MGC_CALIBRE_DFM_RUNSET_FILE

env

Sets the Calibre Interactive DFM runset (“Loading Runsets on Invocation”).

MGC_CALIBRE_DFM_RUNSET_LIST

env

Populates Calibre Interactive DFM runsets in the Load Runset File dialog box and opens the Load Runset dialog at startup (“Loading Runsets on Invocation”).

MGC_CALIBRE_DRC_RUNSET_FILE

env

Sets the Calibre Interactive DRC runset (“Loading Runsets on Invocation”).

MGC_CALIBRE_DRC_RUNSET_LIST

env

Populates Calibre Interactive DRC runsets in the Load Runset File dialog box and opens the Load Runset dialog at startup (“Loading Runsets on Invocation”).

MGC_CALIBRE_EXPORT_LAYOUT_TEMPLATE_FILE

SKILL

Sets layout template file location in Cadence Virtuoso (“Calibre Layout Export Setup Dialog Box in Cadence Virtuoso”).

MGC_CALIBRE_EXPORT_NETLIST_TEMPLATE_FILE

SKILL

Sets netlist template file location in Cadence Virtuoso (“Calibre Netlist Export Setup Dialog Box in Cadence Composer”).

MGC_CALIBRE_LAYOUT_SERVER

env

Specifies a hostname and communications socket port number for Calibre Interactive and Calibre RVE to use for communication with a connected layout viewer (“Setting the Calibre Interactive Socket Port for a Layout Viewer”).

MGC_CALIBRE_LAYOUT_SERVER_NAME

env

This environment variable is set automatically by the tool to the type of layout viewer. The allowed values are the same as those for the cmnLayoutServerType runset variable.

MGC_CALIBRE_LAYOUT_TMP_FILE

env

Sets the name of the intermediate file used in layout editor communication.

MGC_CALIBRE_LVS_RUNSET_FILE

env

Sets the Calibre Interactive LVS runset (“Loading Runsets on Invocation”).

MGC_CALIBRE_LVS_RUNSET_LIST

env

Populates Calibre Interactive LVS runsets in the Load Runset File dialog box and opens the Load Runset dialog at startup (“Loading Runsets on Invocation”).

MGC_CALIBRE_MAP_BUS_NAME

env or SKILL

See Automatically Convert Bus Name Delimiter for Calibre Interactive and Calibre RVE.

Note: this environment variable does not affect netlist export; see Convert Bus Name Delimiter During Schematic Netlist Export.

MGC_CALIBRE_MRU_SIZE

env

Sets the list length of the “most recently used” lists in Calibre Interactive, Calibre Interactive xACT, and Calibre RVE. The environment variable also controls the list length for the Recent Runsets and Recent Text Files menu selections.

The default list length is 20. The maximum length is 50.

MGC_CALIBRE_PERC_RUNSET_FILE

env

Sets the Calibre Interactive PERC runset (“Loading Runsets on Invocation”).

MGC_CALIBRE_PERC_RUNSET_LIST

env

Populates Calibre Interactive PERC runsets in the Load Runset File dialog box and opens the Load Runset dialog at startup (“Loading Runsets on Invocation”).

MGC_CALIBRE_PEX_RUNSET_FILE

env

Sets the Calibre Interactive PEX runset (“Loading Runsets on Invocation”).

MGC_CALIBRE_PEX_RUNSET_LIST

env

Populates Calibre Interactive PEX runsets in the Load Runset File dialog box and opens the Load Runset dialog at startup (“Loading Runsets on Invocation”).

MGC_CALIBRE_SAVE_ALL_RUNSET_VALUES

env

Controls whether you save both default and non-default data to a runset file (“Save All GUI Options to the Runset File”).

MGC_CALIBRE_SCHEMATIC_SERVER

env

Specifies a hostname and communications socket port number Calibre Interactive and Calibre RVE to use for communication with a connected schematic viewer (“Setting the Calibre Interactive Socket Port for a Schematic Viewer”).

MGC_CALIBRE_SCHEMATIC_SERVER_NAME

env

This environment variable is set automatically by the tool to the type of schematic viewer. The allowed values are the same as those for the cmnSchematicServerType runset variable.

MGC_CALIBRE_VERILOG_EXPORT_PARAM_FILE

env

Specifies the path to a file with saveNetlist parameters; see “Importing saveNetlist Parameters for Verilog Export in Cadence Encounter”.

MGC_CALIBRE_VIEWER_MENU_CMDS_SINGLE_TOOL

env

Specifies custom menu items to behave the same way as standard menu items when selected multiple times.

See “Creating Custom Menus in Design Tools”.

MGC_CALIBRE_XACT_RUNSET_FILE

env

Used in Calibre Interactive xACT to pre-populate the Recent Runsets and Recent Text Files lists in the File menu, and the Load Runset Files dialog box.

See “Populating the Load Runset Files Dialog Box and Recent Files Lists”.

MGC_CI_EXPORT_ENABLE_COLORING

env

When set, specifies to export coloring information during streamout from Cadence Virtuoso. The option “Enable coloring” is checked in the Calibre Layout Export Setup dialog box. By default, coloring information is not exported.

MGC_FDI_OA_VERSION

env

Sets the Open Access version for FDI and DBdiff utilities. The allowed values are 22.43 and 22.50. The default is 22.43. The SVRF statement LAYOUT SYSTEM OA overrides this variable.

If you specify the OA version in the fdi2gds or fdi2oasis utilities, the environment variable is ignored.

Also see “Providing OpenAccess Input” and “FDI Environment Variables” in the Calibre Layout Comparison and Translation Guide.

MGC_RVE_INIT_SOCKET_AT_STARTUP

env

Sets socket initiation preference for Cadence Virtuoso (“Table 2”) and for Synopsys IC Compiler and IC Compiler II (“Setting Socket Connections with Synopsys IC Compiler and IC Compiler II”). Used for both Calibre Interactive and Calibre RVE.

MGC_RVE_RELEASE_LICENSE_TIME

env

Sets the time in hours after which the Calibre RVE license is released if there is no activity (License Timeout in Calibre RVE” in the Calibre RVE User’s Manual).

If the environment variable MGC_CALGUI_RELEASE_LICENSE_TIME is not set, then MGC_RVE_RELEASE_LICENSE_TIME also sets the license timeout for Calibre Interactive. (“License Timeout in Calibre Interactive”).

MGCCALIBREMENUVIEWTYPE

SKILL

Specifies window type in which to load a Calibre menu (“Creating an Interface to Cadence Virtuoso”).

OA_HOME1

env

Calibre uses the OpenAccess library at $CALIBRE_HOME/shared/pkgs/icv_o. If additional plug-ins are needed, use the OA_HOME environment variable to specify the path to an OA library that contains the plug-ins. Note, that the library specified by the OA_HOME variable is not used. Calibre always uses the default OpenAccess library included with the Calibre installation. (See “Specifying Third Party Database Options in Calibre Interactive”)

Note: As of the 2013.3 release, the supported OpenAccess version in Calibre is oa22.43p006. Also see footnote.

PEX_FMT_HP_PORT_MAP_MODE

env

Specifies the use of templates in Calibre Interactive PEX. When this environment variable is defined, the templates directory in the SVDB is not deleted before a run. See “Templates” in the Calibre xRC User's Manual.

Parent Topic: Environment Variables
1 If this variable is set, then the specified library must be version oa22.43p006 or later. For Linux platforms, libstdc++.so.6.0.8 must be available from the host (RHEL 4u6 and later have it by default) or through the LD_LIBRARY_PATH environment variable.
