Customization Control Summary

The customization commands are functionally divided into three groups: control file statements, runset control statements, and Customization GUI appearance statements.
Table 1. Customization Command Types

Customization Command Type

Description

Control File Statements

Adds SVRF or TVF statements to the control file created by Calibre Interactive.

Runset Control Statements

Gets and sets the value of the fields in the Calibre Interactive GUI and defines a callback procedure.

Customization GUI Appearance Statements

Define labels, separators, and sets the size of the dialog box.

The syntax conventions for the command descriptions are given in “Syntax Conventions”.

Note:

Braces and square brackets are used in both syntax definitions and Tcl constructs. If a syntax definition includes a Tcl construct with brackets or braces, the Tcl elements are enclosed in single quotes to indicate a literal element. For example, the following syntax indicates the square brackets are literal within a syntax definition.

'[' list a b c ']'

Control File Statements

The control file statements add SVRF or TVF statements to the control file created by Calibre Interactive. See “About Control Files” for more information.

Table 2. Control File Statements in the Customization File

Command

Description

CUSTOM::CHECK

Initializes the check selection status in the Figure 1 for ERC checks in Calibre Interactive nmLVS and places ERC [Un]Select Check statements in the control file.

The Check Selection Recipe is the preferred method for selecting the rule checks that are evaluated in a Calibre Interactive nmDRC or Calibre Interactive DFM run. See “Precedence of CUSTOM::CHECK Controls and Check Selection in Calibre Interactive” for important information if you are using CUSTOM::CHECK controls with Calibre Interactive nmDRC or Calibre Interactive DFM.

CUSTOM::DEFINE

Places a #DEFINE or #UNDEFINE pre-processor directive in the control file for SVRF rule files or defines a Tcl variable in the control file when the ‑tvf argument is specified.

CUSTOM::VARIABLE

Places a Variable statement in the control file for SVRF rule files or defines a Tcl variable in the control file when the ‑tvf argument is specified.

Runset Control Statements

The runset control statements get and set the value of the fields in the Calibre Interactive GUI and define a callback procedure.

Table 3. Runset Control Statements in the Customization File

Command

Description

CUSTOM::getRunsetVarValue

Gets the Calibre Interactive GUI setting for the specified control.

CUSTOM::getCalibreVersionString

Returns the current Calibre version string, for example, v2012.4_0.28.

CUSTOM::runsetVarExists

Returns true or false depending on whether the specified control name corresponds to a valid Calibre Interactive GUI control.

CUSTOM::setRunsetVarValue

Sets the specified Calibre Interactive GUI control to the given value.

CUSTOM::setOKCallback

Specifies a Tcl proc that is executed when you click the OK button in the Customization GUI.

CUSTOM::setOKCallbackError

Specifies a message to display if the OKcallback function returns an error.

Customization GUI Appearance Statements

The Customization GUI Appearance statements affect the appearance of the Customization GUI.

Table 4. Customization GUI Appearance Statements

Command

Description

CUSTOM::LABEL

Inserts a label in the customization dialog box.

CUSTOM::SEPARATOR

Inserts a separator in the customization dialog box.

CUSTOM::SETWINGEOM

Resizes and relocates the customization dialog box.

You can specify a primary-dependent relationship between two CUSTOM commands; this is described in the section “Primary-Dependent Commands”.

The Customizations Settings dialog box includes control buttons such as Load and Reset by default. See the section “Customization Settings Dialog Box” for a description of these control buttons.

The standard Tcl/Tk command options config and cget are available with the CUSTOM commands. These options are useful for troubleshooting. They are described the section “Config and Cget Options”.
