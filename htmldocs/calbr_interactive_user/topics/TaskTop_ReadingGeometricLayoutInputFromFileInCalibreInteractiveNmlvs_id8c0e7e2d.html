Reading Geometric Layout Input from File in Calibre Interactive nmLVS

Layout data for a Calibre Interactive nmLVS run is specified on the Inputs pane on the Layout tab. You can read the layout data file design files saved on disk. Geometric layout data is used in Layout vs Netlist and Netlist Extraction runs. The layout netlist is extracted from the geometric layout.

Prerequisites

  • The Calibre nmLVS analysis type is “Layout vs Netlist” or “Netlist Extraction.” See “Specifying the Calibre nmLVS Run and Analysis Type”. The options available for layout input depend on the run and analysis type.

Procedure

  1. Click the Inputs button on the left panel.
  2. Click the Layout tab.
  3. Uncheck “Export from layout viewer” to specify that the layout data is contained in the specified files.
  4. Select the layout format using the “Format” dropdown button list:

    • GDSII — GDS format

    • OASIS — OASIS format

    • LEFDEF — see “Providing LEF/DEF Input”

    • OPENACCESS — see “Providing OpenAccess Input”

    For a full list of supported layout databases, refer to the “Layout Databases” section of the Calibre Verification User’s Manual. Also see “Third Party Database Options”.

  5. Specify the filename for the layout database in the “File” field.
  6. Specify the top cell name in the “Top Cell” field.
  7. Enter the name for the extracted layout netlist in the “Layout Netlist” field. This option is not used in flat runs. In hierarchical runs, Calibre nmLVS extracts the layout netlist from the geometric database and writes it to this filename.
  8. (Optional) If you have multiple layout inputs for your run, you can specify them using the Database > Library sub-tab on the LVS Options pane. See “Specifying Multiple Layouts for an Calibre nmLVS Run”.
  9. Possible next steps are:

    • “Specifying a Layout Text File in Calibre Interactive”

    • “Source Netlist Input in Calibre Interactive nmLVS”
