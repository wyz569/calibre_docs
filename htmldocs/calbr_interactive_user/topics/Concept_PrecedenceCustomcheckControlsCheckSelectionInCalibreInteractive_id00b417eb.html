Precedence of CUSTOM::CHECK Controls and Check Selection in Calibre Interactive

The CUSTOM::CHECK controls in the Customization GUI interact with the check selection methods for Calibre Interactive nmDRC, Calibre Interactive DFM, and Calibre Interactive nmLVS. It is important to know the precedence of the various check selection methods.

The precedence rules are described in the following two sections:

“Calibre Interactive nmDRC and Calibre Interactive DFM Check Selection”

“Calibre Interactive nmLVS Check Selection”

Calibre Interactive nmDRC and Calibre Interactive DFM Check Selection

A Check Selection Recipe is used in Calibre Interactive nmDRC and Calibre Interactive DFM to select the checks that are run; this can potentially cause a conflict if CUSTOM::CHECK controls in a Customization GUI are also used.

CAUTION:

The Check Selection Recipe is the preferred method for selecting the rule checks that are evaluated in a Calibre Interactive nmDRC or Calibre Interactive DFM run.

If you do use CUSTOM::CHECK controls in a customization file, the default built-in check recipe “Checks selected in the rules file” should be used to ensure that the check selections in the Customization GUI are obeyed.

The following two statements govern the interaction of CUSTOM::CHECK controls in the Customization GUI with the Check Selection Recipe:

  • Settings from CUSTOM::CHECK controls in a customization file only affect the evaluation of the “Checks selected in the rules file” expression in a Check Selection Recipe.

  • Settings in the Check Selection Recipe take precedence over settings made by the Customization GUI if there is a conflict.

If a CUSTOM::CHECK control in the Customization GUI enables or disables a check in the rule file, this is reflected in the Check Selection Recipe Editor when the “Checks selected in the rules file” expression is displayed. All other recipe expressions ignore the CUSTOM::CHECK controls in the Customization GUI. Therefore, unexpected results may occur if a check recipe with expressions other than “Checks selected in the rules file” is used together with CUSTOM::CHECK controls in a Customization GUI.

Calibre Interactive nmLVS Check Selection

Calibre Interactive nmLVS uses the Figure 1 to select the ERC checks that are executed during a run. CUSTOM::CHECK controls in a Customization GUI initialize the selection status of checks in the Select Checks dialog box. Changes made in the Select Checks dialog box override the initial selections made by the CUSTOM::CHECK controls.

Check selections in the Select Checks Dialog Box are not saved to the runset, so selecting ERC checks with a customization file is a useful alternative for Calibre Interactive nmLVS runs. See “Customization File to Select ERC Rule Checks” and “Rule Check Selection in Calibre Interactive Batch Runs” for more information.
