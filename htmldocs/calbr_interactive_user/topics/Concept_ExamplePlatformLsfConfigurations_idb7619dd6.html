Example Spectrum LSF Configurations

Custom options specified in the Resource Options column have priority over options specified in the individual table columns. The following examples of resource option statements illustrate the use of both custom filters and standard column filters.

Example 1: Variables Inherit Column Filters

If you leave the Resource Options table cell completely blank, the following substitution occurs in which each of the values specified in the Min CPUs, Type, and Memory columns are substituted for their respective variable as described in Table 1 in the topic “Using the Spectrum LSF Option”:

-R "select[ncpus>=%c && type==%o && maxmem>=%m]"

Example 2: Literal Values Override Column Filters

If you enter an options statement in the Resource Options field that contains literal (non-variable) values, these literal values take priority over any values specified in the Min CPUs, Type, and Memory columns. In this case, only hosts with at least 4 CPUs and a minimum memory of 1024 MBytes are returned in the Matching Hosts field.

-R "select[ncpus>=4 && maxmem>=1024]"

Example 3: Variable Defines an Additional Filter

If you enter an options statement in the Resource Options field that contains both literal values and variables, each variable takes the value of its corresponding column value if there is one.

In this case, the Min CPUs, Type, and Memory column values are substituted into the variables. The remaining tmp option (Disk Space in /tmp in Megabytes) retains its value of 1024.

-R "select[ncpus>=%c && type==%o && maxmem>=%m && tmp>=1024]"

Example 4: Performance is Optimized for Remote Hosts

You can attempt to gain the best performance by configuring Spectrum LSF to use the -x Submit option; this option requires remote hosts on your distributed run to be exclusively dedicated to your Calibre job. However, with this option, if a host is currently running another job, it is not considered available as a remote host when queried.

An alternate method of optimizing the performance of a distributed run is to use the span option to force LSF to use all available remote hosts as shown in this example.

-R "select[ncpus>=%c && type==%o && maxmem>=%m && hname!=%M] &&
span[ptile=800]"
