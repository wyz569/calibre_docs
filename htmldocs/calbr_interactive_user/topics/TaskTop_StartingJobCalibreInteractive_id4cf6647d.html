Starting a Job with Calibre Interactive

Use these general steps to set up and run a Calibre job for any of the supported applications.

Prerequisites

  • Calibre Interactive System Requirements are met.

Procedure

  1. Start Calibre Interactive with one of the methods described in “Invoking Calibre Interactive Tools”.
  2. Do one of the following to populate the Calibre Interactive GUI:

    • Load a runset with File > Load Runset.

    • Specify all rule file, input, output, and run control information by clicking the appropriate button in the left button panel, and entering information in the data entry panel. Refer to the application-specific chapters for detailed information.

    All setup buttons on the left side of the window must be green before clicking the Run button. A setup button changes from red to green when you have specified all required information associated with that button.

    Be aware that simply because all of the buttons are green does not necessarily mean the information in the GUI is correct. It means that there appears to be sufficient information to start a Calibre job.

  3. (Optional) Specify trigger functions, as described in “Trigger Functions in Calibre Interactive”.
  4. (Optional) Specify a customization file, as described in “Customization Files”.
  5. (Optional) Choose Setup > Preferences to specify preferences; see “Setup Preferences”.

    You can enable “Run Calibre when runset is opened” on the Misc tab to have Calibre Interactive start the run automatically when the runset is loaded.

  6. (Optional) Save a runset with File > Save Runset. This saves all the Calibre Interactive settings except those in the Select Checks dialog box for Calibre Interactive nmLVS and Calibre Interactive 3DSTACK.
  7. Click the Run button when you have verified that the GUI setup is correct. The name of the button reflects the Calibre application you are invoking.
    Tip

    You can generate and view a control file without running Calibre by holding down the Ctrl key and clicking Run. In addition, the calibre command line for the run is written to the Transcript pane.

Results

If an error occurs in Calibre Interactive before the Calibre run is started, the error is displayed in a pop-up window or in the Calibre Quality Agent for Calibre Interactive.

When the Calibre run starts, the view switches to the Transcript pane, and any run output, including errors and warnings, is displayed here. See “Transcript Pane in Calibre Interactive”.

Some applications have options to open the results database in Calibre RVE when the run is complete; see “Getting Started with Calibre RVE” in the Calibre RVE User’s Manual. In addition, the Calibre Interactive GUI may open a report file in a text viewer when the run completes; see “Viewing and Editing Text Files in Calibre Interactive”.
