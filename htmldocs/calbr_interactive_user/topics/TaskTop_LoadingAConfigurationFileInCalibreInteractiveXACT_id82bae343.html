Loading a Configuration File in Calibre Interactive xACT

You can load a configuration file with a command line option, by specifying the configuration file with an environment variable, loading it from the GUI, or loading a runset with a configuration file specified.

Multiple configuration files can be loaded with all methods. Files are loaded in order (left to right as specified) and settings in the most recently loaded (rightmost) file take precedence if there is a conflict.

Configuration files specified on the command line have the highest precedence, while configuration files specified in a runset have the lowest precedence.

If you invoke Calibre Interactive from a design tool, you must specify the configuration file with an environment variable, specify it in the runset, or load it manually from the GUI.

Prerequisites

  • A configuration file. See “Configuration File Format for xACT GUI Customization”.

    These two videos demonstrate how to develop a configuration file.

    Video: Customizing the Calibre Interactive xACT GUI: How to Hide and Lock Options

    Video: How To Create a Custom Page for the Calibre Interactive xACT GUI

Procedure

Do one of the following. Assume you have configuration files named cfgRC.cfg, cfgRCC.cfg, and cfgLM.cfg.
Table 1. Loading a Configuration File

Method

Steps

Command line specification

(highest precedence)

Specify the configuration file with the ‑config command line option. You may enter more than one configuration file on the command line.

For example: 

calibre -gui ‑xact ‑runset xact_runset -config cfgRC.cfg
calibre -gui ‑xact -config cfgRCC.cfg -config cfgLM.cfg

Environment variable specification

  1. Specify the configuration file(s) with the MGC_CALIBRE_XACT_CONFIG_FILE environment variable before invoking the GUI. For example: 

    setenv MGC_CALIBRE_XACT_CONFIG_FILE cfgRC.cfg

    Specify multiple configuration files as a space separated list enclosed in quotes.

  2. Invoke Calibre Interactive.

    See “Invoking the Calibre Interactive xACT GUI”.

GUI and runset specification

(lowest precedence)

You can specify a configuration file on the Preferences page in the GUI.

  1. Click Preferences on the left panel.

  2. Enable the “Configuration” checkbox.

  3. Specify one or more configuration files. The configuration files are loaded in the order specified.

  4. Click Load Configuration Files.

The configuration files are saved to the runset. When the “Configuration” checkbox is checked, the configuration files are loaded the next time you load the runset.

Note:

Configuration files specified on the command line or with an environment variable have higher precedence. If either is specified, configuration files specified in the runset are not loaded and the Load Configuration Files button is not present.

Results

The configuration file(s) are loaded. If a runset is specified on the command line, it is loaded after the configuration file is initialized.
