Defining Hcell and Xcell Input (Gate-Level, Hierarchical Extraction and ADMS Only)

The hcell list specifies the correspondence between layout and source cell names for hierarchical Calibre nmLVS operations. The xcell list is used during parasitic extraction and defines the cells that are preserved during extraction.

Note:

You do not need to perform these steps for Transistor-level extraction.

Prerequisites

  • Calibre Interactive PEX is open; see “Invoking Calibre Interactive Tools”.

Procedure

  1. Click Inputs on the left panel of Calibre Interactive PEX.
  2. Click the H-Cells tab.
  3. (Optional) If you are using source names for the extracted netlist, you can enable “Match cells by name.” This option adds the ‑automatch option to the “calibre ‑lvs” command line.

    This option specifies automatic correspondence by name for cells in Calibre nmLVS‑H comparison, in addition to the cells listed in the hcell file. The ‑automatch option does not apply to netlist extraction.

    Note:

    The “Match cells by name (LVS automatch)” option should only be used if the layout cells contain the same devices as the source subcircuits of the same name.

  4. If using source names for the extracted netlist, enable “Use LVS H-Cells” file and specify the hcell filename. When performing netlist extraction using source names, the “calibre ‑lvs” step for netlist extraction and PHDB creation uses the specified hcell file.

    This option is not required if you specified “Use Names From” as “LAYOUT” on the Outputs pane in the Netlist tab.

  5. Specify the xcell filename in the field “PEX x-Cells file.” The “calibre -xrc” steps use the xcell file.
    Tip

    You can use an hcell file as the xcell file. However, an xcell list can have additional options that are not allowed in an hcell list.
