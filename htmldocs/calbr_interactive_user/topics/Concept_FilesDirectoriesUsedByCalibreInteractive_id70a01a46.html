Files and Directories Used By Calibre Interactive

Calibre Interactive uses several important files and directories.

Files

Runset

A Calibre Interactive generated file that stores the settings you specify in the GUI windows. Refer to “About Calibre Interactive Runsets” for more information on this file.

Control File

A system-generated file that controls the Calibre run. Refer to “About Control Files” for more information on this file.

Rule File

Your primary SVRF rule file for the Calibre run. Certain settings in this file can be overridden by Calibre Interactive, depending on the settings in the interface. Refer to “About Rule Files” for more information on how Calibre Interactive uses rule files.

Preference Files

System-generated files created by Calibre Interactive to save your preferences as defined in the Setup Preferences dialog box. Calibre Interactive creates a different preference file for each type of run. See “Setup Preferences”.

Customization File

An optional Tcl file in which you program custom controls to create a Customization Settings Dialog Box. See “Customization Overview”.

Trigger Script

A trigger script or function can be specified to execute at certain times in the Calibre Interactive execution process. For more information, see “Trigger Functions in Calibre Interactive”.

Note:

The preceding list does not apply to Calibre Interactive xACT. See “Using Calibre Interactive xACT” for information.

Directories

Working Directory

The directory from which you invoke Calibre Interactive.

Run Directory

The directory location from which you run the Calibre application and from which all relative pathnames are resolved. By default, Calibre Interactive sets the run directory to your current directory. Changing the run directory may require that you update the paths to input and output files. The run directory is set from the Rules pane.
