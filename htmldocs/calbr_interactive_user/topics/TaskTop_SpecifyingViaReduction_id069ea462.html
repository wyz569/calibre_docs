Specifying Via Reduction

You can use the Netlist > Vias tab on the PEX Options pane to override the PEX Reduce Via Resistance statements in your rule file or add new statements.

Figure 1. Netlist > Vias Tab in Calibre Interactive PEX

Prerequisites

  • Calibre Interactive PEX is open.

  • A rule file is specified on the Rules pane.

Procedure

  1. (Optional) On the Rules pane, click Load to load the rule file settings into the GUI.

    This loads the PEX Reduce Via Resistance statements in your rule file into the GUI, along with other rule file settings. This step is necessary if you want to change the rule file settings for via reduction.

  2. Click the PEX Options button on the left panel to display the PEX Options pane. If the button is not available, choose Setup > PEX Options.
  3. Click the Netlist tab.
  4. Click the Vias sub-tab.

    If you loaded the rule file in Step 1, the via reduction statements in the rule file are loaded in table.

  5. Select one of the following options:

    • Use settings from rules — Use PEX Reduce Via Resistance statements from the rule file. (default)

    • Turn off via reduction — Turn off via reduction; this adds the PEX Reduce Via Resistance OFF statement to the Calibre Interactive control file.

    • Specify via reduction — Use via reduction statements specified in the table. You can do the following:

      Action

      Step

      Disable or enable specific statements.

      Use the checkboxes in the Include column to select the statements that are included in the run.

      Edit the parameters for a statement.

      Click in the Value column and edit the parameters.

      Add a new statement.

      Click  to add a new statement, then fill in the parameters in the Value column.
  6. (Optional) Save the runset. Any via reduction statements in the table are saved to the runset and loaded into the table when you load the runset.

Results

Statements are added to the Calibre Interactive control file depending on the settings in the GUI.

Note:

Rule file compilation by Calibre Interactive may cause the order of [layer1 layer2] entries in the GUI to be different than that in the rule file.
