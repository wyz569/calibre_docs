Setting Up the Verilog Translator (v2lvs)

You can specify Verilog netlists as input source netlists for your Calibre nmLVS run. The Verilog netlist is translated into an LVS SPICE netlist before Calibre nmLVS is run using the v2lvs translator provided in the Calibre software tree.

The settings made in the Verilog Translator dialog box are saved in the runset.

Procedure

  1. Choose Setup > Verilog Translator.
  2. Click the Libraries tab in the Setup Verilog Translator dialog box. Enter the location of the library files (Verilog or SPICE) in the corresponding “Verilog Library Files” or “SPICE Library Files” fields. If you have included SPICE files, specify the path in the “Include SPICE files” field.
  3. Click the Strings tab in the Setup Verilog Translator dialog box to specify options with string parameters. Table 1 lists the command options for this tab. Enter the values for the options in the corresponding text field.
    Table 1. Verilog String Options

    Dialog Box Option

    Equivalent v2lvs Option

    Description

    Prefix for gate level primitives

    -p prefix

    Adds the prefix to Verilog gate-level primitive cells. This allows construction of specific SPICE subcircuits that correspond to the Verilog gates used.

    Prefix for unnamed pins & gates

    -u unnamed_pin_prefix

    Adds the prefix to unnamed pin connections in module instantiations, and to unnamed primitive and gate instantiations.

    Change array delimiters to

    -a delimiter1 delimiter2

    Changes the array delimiter characters to the ones you specify. The default delimiters are brackets ( [] ). Use this option to change characters used to describe Verilog net selections as SPICE nets.

    Change illegal SPICE characters to

    -c char1 [char2]

    Sets the substitution characters for characters illegal in SPICE. Parameter char1 replaces “$”, “,” and “-”. Parameter char2 replaces “/”. When characters normally illegal in SPICE are encountered in Verilog, the char1 and char2 characters are substituted.

    Connect pins with 0 value to net

    -s0 groundnet

    Specifies that the default global ground is changed to groundnet. The groundnet is declared as a .GLOBAL net.

    Connect pins with 1 value to net

    -s1 powernet

    Specifies that the default global ground is changed to powernet. The powernet is declared as a .GLOBAL net.

    More v2lvs program options

    None

    Allows you to specify any v2lvs program option that you wish to pass to the translator.
  4. Click the Options tab in the Setup Verilog Translator dialog box to specify command options for the v2lvs translator. The options are described in Table 2 along with the equivalent command line options used by the v2lvs translator.
    Table 2. Verilog Translator Options

    Dialog Box Option

    Equivalent v2lvs Option

    Description

    Use aggregate mode for bus bits

    -lsr

    -lsp

    Unrolls bus bits from the input Verilog files. If this option is not selected, bus net names are treated as non-aggregate nets.

    Preserve backslash in escaped identifiers

    -b

    Preserves the backslash character in escaped identifiers.

    Treat warnings as errors

    -w0

    -w1

    -w2

    -w3

    -w4

    Sets warning levels to be treated as errors when encountered:

    Level 0: No warnings.

    Level 1: Issues warnings about skipped modules and multiple declarations of modules.

    Level 2: All Level 1 warnings and calls to undeclared modules and pin array width wider than port.

    Level 3: All level 2 warnings and called port array width mismatches and unsupported compiler directives.

    Level 4: All level 3 warnings and all ignored constructs.

    Don’t connect supply0/supply1 nets to global supply

    -sk

    supply0 and supply1 nets are not connected to the global power/ground.

    Add pins to each subckt

    -addpin

    Adds specified pins to each subcircuit, if they do not already exist. This option is not compatible with the “Use positional pin order” option.

    Use positional pin order (don’t use $PINS)

    -i

    Sets instance pin connections in positional order. If this option is not selected, pins are connected with $PINS.

    Create numbered nets for unconnected pins

    -n

    All unconnected pins get numbered net connections starting from 1000.

    Generate xRC source template file

    -t

    The translator outputs an xRC source template file to the svdb database directory.

    Prefer Calibre-CB license during license search

    -cb

    The translator will attempt to check out a Calibre Cell/Block license. If it fails, the translator will attempt to check out a Calibre nmLVS license.

    Always translate before running Calibre

    None

    Translator is run before every Calibre run. If unchecked, the translator does not run if the Verilog files are older than the translated SPICE file.

     

  5. Click OK to close the dialog box.
