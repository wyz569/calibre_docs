Saving Customization GUI Settings in Calibre Interactive Batch Mode Runs

The Customization GUI is displayed in Calibre Interactive batch mode runs invoked with the ‑customgui command-line option. You can control whether or not changes to the Customization GUI are saved to the runset. The settings on the Misc tab of the Setup Preferences dialog box for saving a runset also apply to Calibre Interactive batch runs.

Prerequisites

  • A runset for the batch mode run.

  • A customization file. See “Customization Language” and “Customization Commands”.

Procedure

  1. Load the runset for the batch mode run in Calibre Interactive.
  2. Choose Setup > Preferences to open the Setup Preferences dialog box.
  3. Click the Misc tab and make selections as follows:

    • Prompt to save a runset if customization settings change:

      Enable “Prompt to save changes before closing runset.”

    • Save the runset without prompting:

      Disable “Prompt to save changes before closing runset.”

      Enable “Save changed settings before closing runset.”

    • Do not save the runset and do not prompt:

      Disable “Prompt to save changes before closing runset.”

      Disable “Save changed settings before closing runset.”

Results

If you make changes to the Customization GUI during a batch mode run, saving the runset is controlled by the settings listed in Step 3. If “Prompt to save changes before closing runset” is enabled and you make changes to the Customization GUI, the prompt to save the runset is displayed immediately after closing the Customization GUI.
