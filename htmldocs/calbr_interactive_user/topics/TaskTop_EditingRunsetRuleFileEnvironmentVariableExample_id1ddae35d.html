Editing a Runset or Rule File Environment Variable Example

You can change the value of an environment variable by editing it in the Runset Value column of the Set Env. tab in the Set Environment Variables dialog box. This applies to both rule file and runset environment variables. If you want to use the new value in the Calibre Interactive session, you must make sure that the Runset column has a green box with a check mark to indicate that the runset value is used.

This example shows how to assign the runset value 8a to the rule file environment variable DRC_process used in the example setup described in, “Setup for Environment Variable Examples”. This allows you to use a different value for the run without redefining the variable in the shell. You can then easily switch between using the shell value and the runset value in your Calibre Interactive session.

Prerequisites

  • The shell environment, GUI, and rule file are set up as described in “Setup for Environment Variable Examples”.

Procedure

  1. Choose Setup > Set Environment to display the Set Environment Variables dialog box.
  2. Click the Set Env tab.
  3. Click in the Runset Value column for the DRC_process environment variable, and enter the value 8a.
  4. Click the icon in the Runset column to change it to a green check mark().

    The green check mark indicates that the runset value is used in the Calibre Interactive session.

    See Figure 1 for an illustration of Steps 3 and 4.

  5. Click Apply to save the changes.

    Now when you run Calibre, the value 8a is used for the environment variable DRC_process.

    Tip

    You can change the runset value for any environment variable listed in the Set Env tab and specify that the runset value be used.
