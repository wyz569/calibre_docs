CUSTOM::getRunsetVarValue

Command type: Customization command for Calibre Interactive.

Returns the setting of the Calibre Interactive GUI control given by the specified runset option name.

Usage

CUSTOM::getRunsetVarValue name

Arguments

name

The name of the GUI control. The name is given by the corresponding runset option variable name. See “Option Descriptions” for a list of runset options.

Return Values

Returns the setting of the specified GUI control.

Description

Returns the setting of the Calibre Interactive GUI control given by name.

The customization file is processed after template settings are applied in Calibre Interactive, so the get commands return the GUI settings after template settings are made.

An error dialog is displayed if name is not a valid GUI control name. If the error occurs during reading of the customization file, the customization file is still processed and displayed. If the error occurs during execution of the CUSTOM::setOKCallback proc, then the setOKCallback proc does not complete and the customization dialog box is not updated or closed; you can use the Cancel button to close the dialog box.

Examples

This example creates a CUSTOM::VARIABLE control initialized to the Calibre Interactive GUI setting for “Top Cell.” The CUSTOM::VARIABLE command returns a Tk widget which is set to the Tcl variable topCellEntry, which is used later to get the setting of the customization control. 

set topCellEntry [CUSTOM::VARIABLE -name "topCell_var" -choices {} \
	-initval [CUSTOM::getRunsetVarValue drcLayoutPrimary]  \
	-boolean 0 -select 1 -prompt "Top Cell (GUI setting)" ]

See CUSTOM::setRunsetVarValue for a complete example which uses a CUSTOM::setOKCallback proc and CUSTOM::setRunsetVarValue to set the “Top Cell” value in the GUI. Also see “Customization File to Read and Set Calibre Interactive GUI Controls”.

When CUSTOM::getRunsetVarValue is used to initialize a control as in this example, CUSTOM::getRunsetVarValue is only evaluated the first time the customization file is read, unless the customization file changes. This means that changing the setting in Calibre Interactive after closing the Customization GUI does not result in a new initial value the next time the Customization GUI is opened.

In addition, ‑initval settings can be overridden by cmnCustomFileOverrideValues settings in the runset, as explained in “Saving Customization Settings”.
