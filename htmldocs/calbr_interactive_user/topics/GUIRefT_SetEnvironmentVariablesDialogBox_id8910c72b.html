Set Environment Variables Dialog Box

To access: Choose Setup > Set Environment to open the Set Environment Variables dialog box.

The Set Environment Variables dialog box allows you to view shell, runset, and rule file environment variables. You can add and edit runset variables, which are variables defined for the Calibre Interactive session and saved in the runset.

Description

The Set Environment Variables dialog box has two tabs for viewing environment variables:

  • Set Env Tab — Displays rule file and runset environment variables. You can also add and edit runset variables, and choose whether to use the shell value, use the runset value, or undefine the variable for the run.

  • Shell Env Tab— Displays the shell environment variables and their values. You can specify a pattern for which environment variables are displayed.

The two tabs of the Set Environment Variables dialog box are shown in Figure 1. The controls for the Set Env tab are described in a table following the figure.

The use of the Set Environment Variables dialog box is explained with the examples in the section “Examples for Creating and Modifying Environment Variables”. The view in Figure 1 shows the setup used for the examples, which is described in “Setup for Environment Variable Examples”.

Objects

Table 1. Set Env Tab for Set Environment Variables Dialog Box

Control or Column

Definition

Add

Adds a new runset environment variable. A new row appears in the table; click in the columns Name and Runset Value to enter values.

Delete

Deletes the selected environment variable. Only runset environment variables can be deleted (those with an in the In Rules column).

Name

The name of the environment variable. Click in the column to enter the name for a new variable or edit the name of a runset variable.

Env.

A green check mark in this column indicates that the value in the Env. Value column is used for the run. Click in the Env. column to select the Env Value for the run.

Env. Value

The value defined in the shell. This value cannot be changed.

Runset

A green check mark in this column indicates that the value in the Runset Value column is used for the run. Click in the Runset column to select the Runset Value for the run.

Runset Value

The value defined in the Calibre Interactive session or runset. Click in the column to enter a value for a new variable or edit an existing value.

Unset

A green check mark in this column indicates that the variable is considered undefined for the run. Click in the Unset column to unset the variable for the run.

In Rules

— Indicates a rule file environment variable. When a rule file is loaded, the referenced environment variables are automatically added to the Set Env. tab. You cannot change the name of a rule file environment variable or delete it.

— Indicates a runset environment variable. Runset environment variables are created and defined in your Calibre Interactive session or saved in a runset. You can change the name and the value of a runset environment variable.

OK

Apply changes and close the dialog box.

Apply

Apply changes.
