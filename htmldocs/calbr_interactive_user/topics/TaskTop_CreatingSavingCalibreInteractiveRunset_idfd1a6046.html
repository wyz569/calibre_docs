Creating and Saving a Calibre Interactive Runset

A Calibre Interactive runset is a text file that stores the Calibre Interactive GUI settings. You can load a runset in a later Calibre Interactive session to restore the GUI settings.

By default, Calibre Interactive only saves non-default settings to the runset. You can define an environment variable to have all settings saved to the runset.

Note:

Runset files should be created using the Calibre Interactive application interface. Editing a runset file manually may lead to unexpected results.

Prerequisites

  • (Optional) To make Calibre Interactive save all GUI settings to a runset, set the environment variable MGC_CALIBRE_SAVE_ALL_RUNSET_VALUES to a non-zero value before starting Calibre Interactive.

    If you do not set this environment variable, only non-default settings are saved to the runset. This is usually more desirable than saving all GUI settings.

Procedure

  1. Start Calibre Interactive.
  2. (Optional) If you have an existing runset that is close to your desired settings, load it with File > Load Runset.
  3. Specify all GUI settings as desired.
  4. (Optional) Choose Setup > Preferences, then the Info tab.

    Provide the Runset Type and Runset Info information. If a runset has been tagged with this information, it is displayed in the Load Runset File dialog box.

  5. Choose File > Save Runset to save the runset.

    By default, Calibre Interactive saves runset files to the run directory.

    Tip

    You can choose File > New Runset to clear current GUI settings and return Calibre Interactive to default settings.

    If you launched Calibre Interactive from a design tool, then the input database, top cell, and other parameters related to the open design are filled in automatically using file naming templates.
