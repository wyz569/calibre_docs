Specifying the Layout Input in Calibre Interactive PEX

A layout database is required for PEX runs. The source of your layout data can vary depending on whether your Calibre Interactive session is connected to a viewer or not.

Tip

You can set a preference for the setting of “Export from layout viewer” that depends on whether your Calibre Interactive session is connected to a layout viewer. By default, this preference is set so that” Export from layout viewer” is enabled if Calibre Interactive is started from a layout viewer, regardless of the saved runset value for the option. See “Export from Viewer Preferences”.

Prerequisites

  • Calibre Interactive PEX is open; see “Invoking Calibre Interactive Tools”.

Procedure

  1. Click Inputs on the left panel.
  2. Click the Layout tab.
  3. Select a layout format using the “Format” button dropdown list:

    • GDSII — GDS format

    • OASIS — OASIS format

    • OPENACCESS — see “Providing OpenAccess Input”

    For a full list of supported layout databases, refer to the “Layout Databases” section of the Calibre Verification Users Manual. Also see “Third Party Database Options”.

  4. Specify the input source for the layout database, and the filename. There are two choices for the input source:

    • A connected layout viewer (GDS and OASIS format only) — Go to sub-step 4.a.

    • Layout database files — Go to sub-step 4.b.

    1. A connected layout viewer supplies the layout (GDS and OASIS format only).

      1. Enable “Export from layout viewer.”

        This specifies the layout viewer as the input source.

        Batch runs: In addition, you must also disable the setting “Ignore layout and source export in batch mode” (Setup > Preferences, on the Misc tab). Also see “Specifying Layout and Schematic Export for a Calibre Interactive Batch Run” for details on the layout viewer connection for a batch run.

      2. Verify the filename for the extracted layout database in the “File” field.

        CAUTION:

        When “Export from layout viewer” is enabled, the file specified in the “File” field is overwritten if it exists already. Do not enter a tapeout database in the “File” field, or use the extracted layout named in the “File” field for tapeout or any other use.

        When exporting a layout database from a connected viewer, the layout is extracted and saved to the file named in the “File” field. This filename is generated automatically based on the setting on the Templates > Inputs sub-tab of the Setup Preferences dialog box (Setup > Preferences).

        If the filename is red, the file does not exist and Calibre Interactive creates the file. If the filename is green, the file exists and will be overwritten. Enter another filename if you want to keep the existing file.

      3. Go to the next step, Step 5.

    2. Layout database files are the source of the layout.

      1. Disable the setting “Export from layout viewer,” if present.

        This specifies a file as the input source, rather than a layout viewer.

      2. Specify the filename for the layout database in the “File” field.

        You can specify the layout filename using any of the following methods:

        • Enter it in the “File,” “Library,” or “Directory” field, depending on the database format.

        • Specify it in the rule file, and load the rule file. Note: This assumes the “Read Inputs/Outputs fields when loading rules file” option is selected in the Misc tab of the Setup Preferences dialog box (Setup > Preferences).

        • Load a saved runset that has the layout file specified.

  5. (Optional) If you have multiple layout inputs for your PEX run, you can specify them using the Database > Library tab on the PEX Options pane. This is valid both for layout data from a file and exported from a layout viewer. See “Specifying Multiple Layouts for a PEX Run”.
  6. Specify the top cell name.

    If your layout is exported from a connected layout viewer, the top cell is determined from the current cell in the viewer.
