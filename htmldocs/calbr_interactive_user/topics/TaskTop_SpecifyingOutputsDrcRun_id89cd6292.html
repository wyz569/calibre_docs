Specifying Outputs for a DRC Run

You specify the results database and format on the Outputs pane. You can also select to have Calibre RVE open automatically and choose side RDBs to view. In addition, you can create a DRC HTML Report and a CTO file.

Be aware that Calibre Interactive creates new directories if you specify new pathnames. If you have the “Read Inputs/Outputs fields when loading rules file” option selected, relevant rule file settings will populate the Outputs pane when you click Load on the Rules pane.

Procedure

  1. Click the Outputs button on the left panel.


  2. Check the following information and edit if necessary:

    • File — Specify the Results Database filename. Filenames that end in .gz are compressed using gzip. Filenames that end in .bz2 are compressed using pbzip2. If either compression utility (gzip or pbzip2) is not in your path, you are prompted to supply the path.

    • Format — Specify the results database format.

      • ASCII — You must choose ASCII if you intend to use Calibre RVE to cross-probe the results.

      • GDSII or OASIS — If you choose GDSII or OASIS output format, you have the option of specifying a prefix and suffix for the cell name.

    • Report results from pseudocells in — Select from “User cells,” “User merged,” “Pseudocells,” “Top cell,” or “Combine.” See the tooltip for descriptions.

  3. Specify whether to open results in Calibre RVE after the run is finished:

    If the main DRC results database is in GDS or OASIS format, only side RDBs are opened.

    Show results in RVE

    Specify whether to automatically open Calibre RVE when the DRC run completes.

    Side RDBs to open

    Click the button to open a dropdown list and select the RDBs you want to open in Calibre RVE. Side RDBs which are empty are not automatically opened.

    Also see these related settings on the Misc tab of the Setup Preferences dialog box: “Start RVE as soon as results are available,” “Start RVE after batch run,” and “Do not open empty main results database in RVE.”

  4. Set other options as desired. The available options depend on the database format (ASCII, GDS, or OASIS), as described in the following two tables.
    Table 1. Output Options for All Results Database Formats (ASCII, GDS, OASIS)

    Option

    Description

    Output cell errors in cell space

    Instruct Calibre to use local cell space coordinates when reporting errors. Errors are output in top cell coordinates if this option is disabled. This option is for Calibre nmDRC-H only.

    This option is on by default in Calibre Interactive. This option is recommended for Calibre Automatic-Waiver.

    Specify the amount of rule check text

    Specify the amount of rule check text that is written to the results database. See “Specifying the Rule Check Text Written to the Results Database”.

    Rule file statement: DRC Check Text {{[COMMENTS | ALL] [RFI]} | NONE}

    Check Text File

    Specify a results database to write all selected rulechecks and check text as defined by the other output options to. Results are not written to this file. See “Specifying the Rule Check Text Written to the Results Database”.

    This option is available if “Specify the amount of rule check text” is enabled.
    Table 2. Output Options for an ASCII DRC Results Database

    Option

    Description

    Create HTML Report

    Create a DRC HTML report after the run completes. This option is only available if the layout format on the Inputs pane is GDSII or OASIS.

    To set report options, choose Setup > Preferences and the HTML Report tab. See “Creating a DRC HTML Report from Calibre Interactive nmDRC”.

    Create CTO file

    Create a DRC RVE Check Text Override File (CTO File) during the run.

    See “Creating a Check Text Override (CTO) File Using Calibre Interactive nmDRC”. To specify a layer map file, choose Setup > Preferences, the CTO File tab, and enter a filename for “Viewer Layer Map File.”

     

    The options for GDS and OASIS output add keywords to the DRC Results Database statement in the control file.

    Table 3. Output Options for a GDS or OASIS Output Database

    Option

    Description

    Cell prefix

    Add the specified string as a prefix to output cell names (the PREFIX keyword).

    Cell suffix

    Add the specified string as a suffix to output cell names (the APPEND keyword).

    CBLOCK

    Embed compressed data in the OASIS results database. (OASIS only)

    STRICT

    Create strict mode output. (OASIS only)

    AUTOMAP

    Output each check to a separate layer. (OASIS only)

    INDEX

    Index the output database. (OASIS only)

    NOVIEW

    Optimize for direct input to Calibre MDP functions, but omit optimizations intended for interactive viewing. (OASIS only)

    Check “Write DRC summary report file” to specify the filename and options for the DRC summary report file. (DRC Summary Report statement)

    If Area DRC is run, the summary report is modified to indicate the error filtering that occurred due to eliminating errors in the halo region. The number of filtered results is always written to the transcript. The summary report includes the following information:

    • Number of errors filtered out and the final result count in the relevant RULECHECK line in the RULECHECK RESULTS STATISTICS section

    • The total FILTERED and FINAL results counts in the final TOTAL DRC Results Generated line

    Check “Reusable Hierarchical Database” to specify to save or restore an RHDB (Reusable Hierarchical Database) file. An RHDB is useful in some flows to save time by avoiding the overhead of repeatedly constructing a hierarchical database.

    • Save — Saves a proprietary image of the hierarchical database, generated from all input files in the rule file, as an RHDB file to the specified file. The RHDB file is saved with an embedded checksum. This option disables CTO file generation, HTML report generation, Area DRC, and automatic RVE invocation.

    • Restore — Restores an RHDB file from the specified file. A checksum is generated from all input files and compared with the checksum in the specified filename. A mismatch between the checksums generates an error.

  5. At this point, you may have specified all the information you need to run Calibre nmDRC. If so, then click Run DRC.
