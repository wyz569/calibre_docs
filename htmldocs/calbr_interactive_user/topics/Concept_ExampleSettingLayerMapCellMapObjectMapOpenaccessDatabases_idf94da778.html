Example: Setting the Layer Map, Cell Map, and Object Map for OpenAccess Databases

This example describes how to specify the layer map, cell map, and object map files in Calibre Interactive when using OpenAccess as the Layout System.

To specify the mapping files, do one of the following, depending on the GUI you are using:

  • Calibre Interactive nmDRC, nmLVS, DFM, PERC, and PEX

    1. Choose Setup > app Options.

    2. Click the Database tab, and OpenAccess/LEFDEF sub-tab.

    3. Choose the Mapping Files category.

  • Calibre Interactive xACT

    1. Click OPENACCESS on the left panel. If OPENACCESS is not present, go to the Inputs page and set the Layout Format to OPENACCESS.

    2. Expand the Mapping Files area.

The following sections have examples for the layer map, cell map, and object map files.

Layermap file

By default, input layers are mapped using the input layer numbers and a datatype record of 0. Assume you want to override this behavior and map specific input layers to a user-specified list of output layers and datatypes. You can do this by writing the following mapping to a text file.

Create a file called layermap.txt, as follows:

# input_layer_name   purpose_name   output_layer  datatype

        met1           drawing           11           1
        met2           drawing           12           1
        met3           drawing           13           1

This file maps the indicated input layers to the output layer numbers and datatypes. To specify the layermap.txt file, enable “Use Layer Map File” and specify the file; this is shown for Calibre Interactive— nmDRC in Figure 1.

Cellmap file

By default, the cell names that appear in the OpenAccess database are used on input. Assume you want to override this behavior and rename the cells. Create a file called cellmap.txt, as follows: 

# library   input_name  view_name  output_name
  DesLib     nand2.1     layout      nand2x1
  DesLib     nand2.2     layout      nand2x2

This file maps the indicated input cell name from the specified design library and view to the output cell name. To specify the cellmap.txt file, enable “Use Cell Map File” and specify the file; this is shown for Calibre Interactive—nmDRC in Figure 1.

Objectmap file

You can use an object map file to specify which objects in the design are output. Create a file called objectmap.txt as follows:

#Type         SubType    LayerName(opt)    gdsLNum    gdsDType
LayerBlockage fill       METAL1            31         121
Boundary      PR                           300        101

This object map indicates that fill LayerBlockage on layer METAL1 and PR Boundary objects are output. See the “object type table” in the Calibre Layout Comparison and Translation Guide for a complete list of object types and subtypes. To specify the objectmap.txt file, enable “Use Object Map File” and specify the file; this is shown for Calibre Interactive—nmDRC in Figure 1.
