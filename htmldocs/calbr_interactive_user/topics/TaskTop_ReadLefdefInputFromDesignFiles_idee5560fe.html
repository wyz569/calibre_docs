Read LEF/DEF Input from Design Files

You can read LEF/DEF input from design files using Calibre Interactive.

Not supported in Calibre Interactive PEX.

Prerequisites

  • Make sure environment variables for database read are set correctly. See “FDI Environment Variables” in the Calibre Layout Comparison and Translation Guide.

Procedure

  1. Click Inputs on the left panel of Calibre Interactive.
  2. Click the Layout tab.
  3. Select a Format of LEFDEF.
  4. Disable “Export from layout viewer.”
  5. Provide DEF Input as follows:

    • Files — One or more DEF files.

    • Directories — DEF directories.

    At least one of the Files or Directories entries must be filled in for DEF input.

    Multiple files or directories should be entered on separate lines; use the arrow button to expand or collapse the text entry box.

  6. Provide LEF Input as follows:

    • Tech Files — One or more LEF technology files.

    • Files — One or more LEF files.

    • Directories — LEF directories.

    At least one of the Tech File, Files, and Directories entries must be filled in for LEF input. The Tech File entry is optional as long as the technology information is one of the LEF file or directory entries. If the Tech File entry is not used, then the LEF technology file should be first in the list of LEF files.

  7. Specify the “Top Cell.”
  8. Set other database conversion options as described in “Third Party Database Options”.
    Tip

    You can define a file naming template for the DEF file. Choose Setup > Preferences, the Templates tab, the Database sub-tab, and specify a template in the “DEF File” field. See “Templates for File Naming” for more information.
