Customization Overview

Customization files allow you to customize several settings that affect Calibre runs. The customization file creates a Customizations Settings dialog box, or Customization GUI, which you display with Calibre Interactive.

You can program controls to do the following:

  • Write Variable statements to the Calibre Interactive control file.

  • Write #DEFINE and #UNDEFINE pre-processor statements to the Calibre Interactive control file.

  • Define Tcl variables in the control file if you are using a TVF rule file.

  • Get and set values in the Calibre Interactive GUI using Runset Control Statements in the customization file.

    The customization file is processed after template settings are applied in Calibre Interactive, so the get commands return the GUI settings after template settings are made.

  • Save customization settings to the runset, as described in “Saving Customization Settings”.

  • Use CUSTOM::CHECK commands to select the ERC checks which are executed in a Calibre Interactive nmLVS run. ERC [Un]Select Check statements are written to the Calibre Interactive control file. The settings in the customization file initialize the settings in the Select Checks dialog box for Calibre Interactive nmLVS (see “Selecting ERC Checks”).

    Note: The Check Selection Recipe is the preferred method for selecting the rule checks that are evaluated in a Calibre Interactive nmDRC or Calibre Interactive DFM run; CUSTOM::CHECK controls are not recommended for these runs.

Customization files are written in Tcl/Tk and use commands in the CUSTOM namespace, as described in the sections “Customization Language” and “Customization Commands”.
