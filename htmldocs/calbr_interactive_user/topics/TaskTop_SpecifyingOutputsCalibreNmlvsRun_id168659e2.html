Specifying Outputs for a Calibre nmLVS Run

The Outputs pane in Calibre Interactive nmLVS specifies the LVS report file, the SVDB (Standard Verification Database), and its options.

Procedure

  1. Click the Outputs button on the left panel to display the Outputs pane.


  2. Click the Report/SVDB tab.
  3. (Optional) Enable “View Extraction Report after LVS finishes.”
  4. (Optional) Enable “View Summary Report after LVS finishes.” If LVS Summary Report is not specified in the rule file, then Calibre Interactive adds the statement to the control file with a report name of lvs.summary.
  5. Specify the LVS report filename.
  6. (Optional) Enable the “View Report after LVS finishes” option to automatically display the LVS report in a text editor window at the completion of your LVS run.
  7. Enable the “Create SVDB Database” option and specify the SVDB directory name, as desired. Creating the SVDB database is recommended, since it is required to run Calibre RVE.
  8. Enable “Start RVE after LVS finishes” to automatically start Calibre RVE at the completion of your Calibre nmLVS run. If you are creating a runset for a batch run, you may want to enable “Start RVE after batch run” on the Misc tab of the Setup Preferences dialog box.

    When Calibre RVE is started from Calibre Interactive nmLVS, the settings on the Run Control pane are used to set the ‑turbo option for Calibre RVE; see the section “Multithreaded Parallel Processing for Calibre RVE Operations (-turbo Option)” in the Calibre RVE User’s Manual.

  9. If you want to output data for parasitic extraction, enable the “Generate data for Calibre-xRC” option.

    Once enabled, the netlist file extracted by Calibre nmLVS is automatically copied to the SVDB directory so that SVDBs generated by Calibre nmLVS can be seamlessly imported into PEX.

  10. If you want to generate ASCII cross-reference (XDB) files in the SVDB database, select the “Generate ASCII cross-reference files” option.

    If the run type is flat the -nxf and -ixf command line options are also used.

  11. If you want to generate Calibre Connectivity Interface (CCI) data in the SVDB database, select the “Generate Calibre Connectivity Interface data” option. This option requires a Calibre Connectivity Interface (CI) product license.

    If desired, select “Suppress pin-location information” to add the NOPINLOC keyword to the Mask SVDB Directory statement. This option is only available when “Generate Calibre Connectivity Interface data” is selected.

    Tip

    All left panel buttons must be green before clicking the Run LVS button. A button changes from red to green when you have specified all required information associated with that button.
