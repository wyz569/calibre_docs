CUSTOM::setRunsetVarValue

Command type: Customization command for Calibre Interactive.

Populates the setting of the Calibre Interactive GUI control given by name with the specified value.

Usage

CUSTOM::setRunsetVarValue name value

Arguments

name

The name of the Calibre Interactive GUI control. The name is given by the corresponding runset option name. See “Option Descriptions” for a list of runset options.

value

The value for the Calibre Interactive GUI control.

Description

Populates the setting of the Calibre Interactive GUI control given by name with the specified value.

An error dialog is displayed if name is not a valid GUI control name. If the error occurs during reading of the customization file, the customization file is still processed and displayed. If the error occurs during execution of the CUSTOM::setOKCallback proc, then the setOKCallback proc does not complete and the customization dialog box is not updated or closed; you can click the Cancel button to close the dialog box.

Examples

The following example can be used in a CUSTOM::setOKCallback proc to set “Top Cell” on the Calibre Interactive Inputs pane to the string “myTopCell”.

CUSTOM::setRunsetVarValue LayoutPrimary "myTopCell"

The following extended example reads the “Top Cell” setting from the Calibre Interactive GUI and creates a CUSTOM::VARIABLE control initialized to the current GUI setting. You can change the setting for “Top Cell” in the Customization GUI, if desired. The CUSTOM::VARIABLE command returns a Tk widget which is set to the Tcl variable topCellEntry.

The callback proc myOKCallback reads the value of topCellEntry and uses CUSTOM::setRunsetVarValue to set the GUI control for “Top Cell” to the value currently set in the Customization GUI control.

# customization file
# get CI value for top cell, and initialize control with value
set topCellEntry [CUSTOM::VARIABLE -name "topCell_var" -choices {""} \
	-initval [CUSTOM::getRunsetVarValue drcLayoutPrimary]  \
	-boolean 0 -select 1 -prompt "Top Cell (GUI setting)" ]

CUSTOM::setOKCallback myOKCallback
proc myOKCallback {} {
	global topCellEntry 
	# get customization gui setting for topCellEntry widget
	set wv_topCell [$topCellEntry cget -cvar]
	# write setting to CI GUI control for top cell
	CUSTOM::setRunsetVarValue drcLayoutPrimary [set $wv_topCell]
}

When CUSTOM::getRunsetVarValue is used to initialize a control as in this example, CUSTOM::getRunsetVarValue is only evaluated the first time the customization file is read, unless the customization file changes. This means the initial value for the customization control does not change even if the Calibre Interactive GUI setting is changed sometime later.

In addition, ‑initval settings can be overridden by cmnCustomFileOverrideValues settings in the runset.
