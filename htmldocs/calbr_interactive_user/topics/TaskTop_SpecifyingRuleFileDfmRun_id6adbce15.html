Specifying a Rule File for a DFM Run

In general, it is best to begin setup for a DFM run in the Rules pane. After you define a rule file, you can load the rule file to populate other GUI fields with rule file settings. Calibre Interactive interprets any relative pathnames as relative to the run directory, which you define on this pane.

Procedure

  1. Click Rules to display the Rules pane.


  2. Specify the name of the rule file. You can use the View button to view and edit the rule file. After you load a rule file, any information you specify in the GUI supersedes the information in your loaded rule file.

    Be aware that Calibre Interactive automatically recognizes a rule file with the literal name “rules” and displays the name in green. Because the name is shown in green, you may assume the file has been loaded but it has not been; the file must be explicitly loaded using the Load button.

  3. (Optional) Click Load to populate the GUI with the settings in the rule file.

    A rule file can be specified in a runset. After loading that runset, the rule file is included in the control file but its setting will not be loaded until you click Load.

    Tip

    The setting “Read Inputs/Outputs fields while loading rules file” (Setup > Preferences, Misc tab) controls whether or not rule file settings populate the GUI options on the Inputs and Outputs panes when Load is clicked. This setting is enabled by default. See “Read Inputs/Outputs fields while loading rules file” in Table 1.

    After you load a rule file, any information you specify in the GUI supersedes the information in your loaded rule file.

  4. Specify the run directory name. If the directory you specify does not exist, Calibre Interactive queries as to whether you want to create the directory.
  5. (Optional) View the layer derivation tree at the bottom of the Rules pane; see “Layer Derivations Tree” for information.
