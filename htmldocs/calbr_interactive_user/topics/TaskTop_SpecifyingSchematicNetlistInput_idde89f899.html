Specifying Schematic Netlist Input

Schematic netlist input is optional for a PEX run. A schematic netlist is needed if you want to use source schematic names instead of layout names in the extracted netlist. Also see the “Use Names From” setting on the Outputs pane.

Netlist data is also needed when using the Blocks tab on the Inputs pane to control cell extraction options for an ADMS run. See “Manipulating Cell Extraction Options Using the Blocks Tab”.

Prerequisites

  • Calibre Interactive PEX is open; see “Invoking Calibre Interactive Tools”.

Procedure

  1. Click the Netlist tab on the Inputs pane.


  2. Select the input source for the schematic netlist with the “Export from schematic viewer” option as follows:

    • Enabled — A connected schematic viewer supplies the schematic source.

      Batch runs: In addition, you must also disable the setting “Ignore layout and source export in batch mode” at the Setup > Preferences on the Misc tab. Also see “Specifying Layout and Schematic Export for a Calibre Interactive Batch Run” for details on the viewer connection for a batch run.

    • Disabled — Schematic netlist files provide the source schematic.

  3. Specify the format of the source netlist using the “Format” button dropdown list.

    If Calibre Interactive is connected to a schematic viewer and “Export from schematic viewer” is enabled, specify the netlist format of the schematic that your viewer outputs.

    Select from these format options:

    • SPICE — The netlist is in SPICE format.

    • VERILOG — The netlist is in Verilog format. If you specify Verilog as the source format, Calibre Interactive calls a utility named v2lvs to translate the Verilog netlist to SPICE. Choose Setup > Verilog Translator to set up the translation parameters. For more information about setting up v2lvs, see the section “Setting Up the Verilog Translator (v2lvs)” in this manual. The SPICE netlist name is taken from the “Files” field.

    • MIXED — Select the “Mixed” option to use a mixed mode of both SPICE and Verilog files. The application displays separate netlist entry fields for the SPICE and Verilog files. This option cannot be specified when “Export from schematic viewer” is enabled.

  4. Specify the netlist filename. The behavior depends on the setting for “Export from schematic viewer”:

    • "Export from schematic viewer" is enabled. Go to sub-step 4.a.

    • "Export from schematic viewer" is disabled. Go to sub-step 4.b.

    1. “Export from schematic viewer” is enabled.

      This specifies the schematic viewer as the input source for the schematic netlist.

      1. Verify the filename for the extracted schematic in the “Files” field. This file will be overwritten if it exists already.

        When exporting a source netlist from a connected schematic viewer, the schematic is extracted and saved to the file named in the “File” field. This filename is generated automatically based on the setting on the Templates > Inputs tab of the Setup Preferences dialog box (Setup > Preferences).

        If the filename is red, the file does not exist and Calibre Interactive creates the file. If the filename is green, the file exists and will be overwritten. Enter a different filename if you want to keep the existing file.

      2. Go to the next step, Step 5.

    2. “Export from schematic viewer” is disabled.

      This specifies that a file is the input source for the schematic, rather than a schematic viewer.

      1. Specify the filename for the source netlist in the “Files” field.

      2. Go to the next step.

  5. Enter the top cell name in the “Top Cell” field. If you are connected to a viewer, this field is filled in automatically.
  6. (Optional) Specify additional SPICE netlist files. This may be necessary to provide INCLUDE files for the source netlist.
    1. Click the PEX Options button or choose Setup > PEX Options.
    2. Click the Database tab and the Library sub-tab.
    3. Use the buttons next to “Additional SPICE Files” to specify the required SPICE files.
      Note:

      Calibre Interactive verifies that the source netlist is complete before starting the run. You can turn off this check by disabling “Check source netlist is complete before running PEX” (Setup > Preferences, on the Misc tab). See “Miscellaneous Preferences” for information on the source netlist check.

Results

The appropriate statements are written to the Calibre Interactive control file at runtime.

If you specified multiple source netlist files, Calibre Interactive creates a file named _source.net_ with .INCLUDE statements for each netlist file; _source.net_ is used as the source netlist for the Source Path statement in the control file. Files entered in the Database > Library tab on the PEX Options pane are listed before files in the Netlist tab on the Inputs pane.
