Providing OpenAccess Input

You provide OpenAccess database input in the Layout tab on the Inputs pane in Calibre Interactive. You specify database conversion options on the Options pane in the Database > OpenAccess/LEFDEF sub-tab.

Prerequisites

  • For Cadence Virtuoso users, make sure CDS_INST_ROOT or CDS_INST_DIR is set. CDS_INST_ROOT takes precedence if both CDS_INST_ROOT and CDS_INST_DIR are defined.

  • Make sure LD_LIBRARY_PATH includes the libstdc++.so.6.0.8 compiler libraries. This is true by default for Linux®1 machines with RHEL 4u6 or later.

    For example with Cadence Virtuoso 6.15.500:

    setenv LD_LIBRARY_PATH ${CDS_INST_DIR}/share/oa/lib/linux_rhel40_gcc44x_64/opt:
${CDS_INST_DIR}/tools/cdsSkillPcell/lib/64bit:
${CDS_INST_DIR}/tools/lib/64bit:${LD_LIBRARY_PATH}

    The actual path depends on the tool version.

  • Make sure environment variables for database read are set correctly. See “FDI Environment Variables” in the Calibre Layout Comparison and Translation Guide.

    In particular, if your design tool requires OpenAccess version 22.50, specify this setting with one of the following options:

    • In the rules file, include the SVRF statement LAYOUT SYSTEM OA “22.50”.

    • Set the environment variable MGC_FDI_OA_VERSION to 22.50.

    Calibre Interactive does this automatically when connected to Cadence Virtuoso versions 6.17 or later. The setting specified by the SVRF statement overrides MGC_FDI_OA_VERSION.

Procedure

  1. Click the Inputs button on the left panel of Calibre Interactive.
  2. Click the Layout tab.
  3. Select a Format of OPENACCESS.
  4. Enter the path to the directory containing the library definitions file in the “Library Path” field. The “Library Path” should be a directory that contains either a cds.lib (if OA_HOME is set) or lib.defs file.

    If the library definitions file is in the current working directory (the directory you invoked Calibre Interactive from), it is not necessary to define “Library Path.”

  5. Enter the name of the design library in the “Library Name” field. You can click the button to open a library browser. The library browser allows you to select the library, cell, and view, then automatically fills in the correct GUI fields.

    Note that the library browser will not work correctly if “Library Path” is not correctly defined.

  6. Specify the Top Cell and View name if these have not been filled in by the library browser.
  7. Choose Setup > app Options from the menu, then click the Database > OpenAccess/LEFDEF sub-tab.
  8. For Cadence Virtuoso users, do the following to automatically set certain Cadence environment variables:
    1. Choose the Other Options category on the OpenAccess/LEFDEF tab.
    2. Enable “Set OA_HOME when CDS_INST_ROOT or CDS_INST_DIR is set.”
    3. Enable “Set CALIBRE_READDB_LD_LIBRARY_PATH when CDS_INST_ROOT or CDS_INST_DIR is set.”
    4. Enable “Set PATH when CDS_INST_ROOT or CDS_INST_DIR is set.”
    5. If you are using Express Pcells, enable “Set CDS_EXP_PCELL_DIR when CDS_INST_ROOT or CDS_INST_DIR is set.”
    6. If you are using Express Pcells, enable “Set CDS_EXP_ALL_PARAMS when CDS_INST_ROOT or CDS_INST_DIR is set.”
    7. If you are using Express Pcells, enable “Set CDS_ENABLE_EXP_PCELL when CDS_INST_ROOT or CDS_INST_DIR is set.”
  9. Set other database conversion options as needed in the Database > OpenAccess/LEFDEF sub tab. For complete details, see “OpenAccess/LEFDEF Options Tab”.

    These options control what is read from the OpenAccess database and passed on as input to Calibre.

    Tip

    You can define a file naming template for the Library and View. Choose Setup > Preferences, the Templates tab, the Database sub-tab, and specify templates in the “Library” and “View” fields. See “Templates for File Naming” for more information.
