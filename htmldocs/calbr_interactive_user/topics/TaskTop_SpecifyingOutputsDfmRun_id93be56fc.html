Specifying Outputs for a DFM Run

You specify the format of the output database on the Outputs pane of Calibre Interactive DFM.

Throughout this section, be aware that Calibre Interactive creates new directories if you specify new pathnames. If you have the “Read Inputs/Outputs fields when loading rules file” option selected on the Misc tab of the Setup Preferences dialog box, relevant rule file settings will populate the Outputs pane.

Procedure

  1. Click the Outputs button on the left panel of Calibre Interactive.
  2. If there are multiple tabs on the Outputs pane, select the Common tab.
  3. Select the format of the output database from the “Results Format” selections:

    • DFM Database

    • ASCII

    • GDSII

    • OASIS

    The fields present in the Outputs pane depend on the “Results Format” you choose. For a view of the Outputs pane with a results format of DFM Database, see the following figure.

    Figure 1. DFM Outputs Pane

  4. Fill in the required information for the results format you chose:

    • DFM Database — enter the directory path.

      The path specified in the Directory field is the parent directory of the DFM Database. The DFM database is created as a subdirectory within that parent directory and is named topcell.dfmdb, where topcell is the top cell name from the Inputs pane.

      See “Specifying Layers and Annotations for DFM Results Database Output” for additional settings.

      Corresponding rule file statement: DFM Database.

      • ASCII, GDSII, and OASIS — Enter the filename for results database. Additional options are available depending on the database type; see the command DRC Results Database in the SVRF Manual for a description of the options.

        • Pseudocells in — Selections: “User cells”, “User merged,” or “Pseudocells,” and “Top cell.” See the tooltip for descriptions.

        • Cell prefix and Cell suffix — Available for GDSII and OASIS formats.

        • Specify the amount of rule check text — Specify the amount of rule check text that is written to the results database. This option only applies to ASCII results databases. See “Specifying the Rule Check Text Written to the Results Database”.

        • CBLOCK — Available for OASIS results format.

        • STRICT — Available for OASIS results format.

        • INDEX — Available for OASIS results format.

        • NOVIEW — Available for OASIS results format if INDEX is enabled.

  5. (Optional) Select “Show Results in RVE” to automatically start Calibre RVE for DFM when the run is done. Only DFM Database and ASCII results formats can be opened in Calibre RVE.

    If you want RVE to open after a batch run, choose Setup > Preferences, then the Misc tab, and enable “Start RVE after batch run.”

    Side RDBs to Open — This selection is available if your results database format is ASCII, GDS, or OASIS. Click the button to open a dropdown list with the auxiliary databases (side RDBs) defined in the rule file; select the RDBs you want to open by checking the box in the “Open” column of the dropdown list. If the main database is in GDS or OASIS format, only the side RDBs will be opened in Calibre RVE. Side RDBs which are empty are not automatically opened.

    Also see the setting “Do not open empty main results database in RVE” on the Misc tab of the Setup Preferences dialog box.

    When Calibre RVE for DFM is started from Calibre Interactive, the settings on the Run Control pane are used to set the ‑turbo option for Calibre RVE; see the section “Multithreaded Parallel Processing for Calibre RVE Operations (-turbo Option)” in the Calibre RVE User’s Manual.

  6. (Optional) Specify a Summary Report File by enabling Write Summary Report File, and entering the filename. Adjust the other options for the summary report if needed.
  7. (Optional) See “Running a YieldServer Script on the DFM Database” to specify a YieldServer post-processing file and arguments.
  8. At this point, you may have specified all the information you need to run DFM. If so, then click Run DFM. There are more options to consider, and these are discussed next.
