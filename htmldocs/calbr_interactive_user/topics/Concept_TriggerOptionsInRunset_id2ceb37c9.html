Trigger Options in the Runset

The way in which Calibre Interactive saves trigger definitions in the runset differs according to the Calibre version.

This process is followed when reading trigger options in the runset:

  • 2011.3 release and earlier

    Trigger definitions are saved in the runset with the *cmnPreTrigger and *cmnPostTrigger runset options; these are the only runset options recognized for defining triggers. The trigger definition applies to all Calibre Interactive applications.

  • 2011.4 release through 2013.2

    When trigger definitions are saved in the runset, the runset option name reflects the Calibre Interactive application; for example: *drcPreTrigger and *drcPostTrigger. The trigger definition only applies to the specified Calibre Interactive application.

  • 2013.3 and later

    Multiple triggers may be defined and the triggers may run in different environments. The runset options now have an “s” at the end of the option name. The runset options are a list of lists of this format:

    *drcPreTriggers: {name type enable} …

    • name — The program or procedure (and optional arguments) to execute before Calibre is run.

    • type — Type may have the values layout, schematic, or process.

    • enable — A Boolean value specifying whether to run the trigger.

    See “Setting Calibre Interactive Internal Triggers” for complete information and the meaning of the different trigger types.

The following table lists the runset options available for defining triggers.

  • Calibre version 2011.4 and later — These trigger definitions are used only if the application-specific trigger definition is not present.

  • Calibre version 2011.3 and earlier — Not recognized.

*drcPreTriggers, *drcPostTriggers

*lvsPreTriggers, *lvsPostTriggers

*dfmPreTriggers, *dfmPostTriggers

  • Calibre version 2013.2 and earlier — Not recognized.

  • Calibre version 2013.3 and later — Trigger definitions are a list of lists. See “Setting Calibre Interactive Internal Triggers” and “Runset Options”.

The following rules are followed when saving a runset in Calibre versions 2011.4 through 2013.2:

  • If *cmnPreTrigger or *cmnPostTrigger runset options are present in the runset, their definitions are not changed.

  • When a runset is saved, trigger definitions are saved as the application-specific runset options.

    However, if a runset with *cmnPreTrigger or *cmnPostTrigger runset options is loaded, you will not be prompted to save the runset unless the triggers or other runset options have changed.

When saving a runset in Calibre version 2013.3 or later, the runset options are automatically updated to the new format.

See “Applying Triggers According to the Calibre Interactive Application” for an example of using application-specific triggers.
