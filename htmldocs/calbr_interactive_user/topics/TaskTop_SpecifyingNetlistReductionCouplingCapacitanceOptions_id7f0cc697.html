Specifying Netlist Reduction and Coupling Capacitance Options

The netlist reduction and coupling capacitance options provide additional control over the extracted parasitic netlist.

Procedure

  1. Click the PEX Options button on the left panel, or choose Setup > PEX Options if the button is not present
  2. Click the Netlist > Reduction and CC sub-tab and make the desired selections. Table 1 described the available options.
    Table 1. Netlist Reduction and CC Options

    Selection

    (SVRF Command)

    Comment

    Enable TICER reduction below

     

    (PEX Reduce TICER)

    Enter the TICER cutoff frequency in Hertz. Scientific notation may be used.

    Performs reduction of distributed RC extraction data in a way that has little impact on circuit characteristics such as delay up to a specified frequency.

    Specify the “Port Merge” and “Max Degree” values to add the PORTMERGE and MAXDEG keywords. See the command reference for a description of these parameters and the required units.

    Enable CC reduction

     

    (PEX Reduce CC)

    Absolute — Enter threshold (> 0). The threshold is scaled by the Unit Capacitance value.

    Ratio — Enter ratio value between 0 and 1.

    Scale — Optional parameter, with a default of 1.

    Controls the reduction of coupled capacitance between nets based on total capacitance. See the description in the SVRF Manual for details.

    Enable analog reduction:

     

    (PEX Reduce Analog)

    Delay — Enter the timing delay threshold in seconds to be used for parasitic reduction.

    Noise — Enter the error threshold used for parasitic reduction; the value is a ratio between 0 and 1.

    Specifies performing reduction of extracted data from analog designs.

    Enable MinCap reduction

     

    (PEX Reduce Mincap)

    Enter a non-negative threshold in units specified by Unit Capacitance. If Unit Capacitance in not specified, picofarads is used.

    Select this to combine or remove parasitic capacitors below a threshold value to reduce netlist size.

    Enable MinRes reduction

     

    (PEX Reduce Minres)

    Select this to combine and short parasitic resistors below a threshold value to reduce netlist size.

    • Select YES in the dropdown box to enable GUI controls.

      Enter a non-negative threshold in units specified by Unit Resistance. These two commands are added to your control file:

      PEX Reduce Minres Combine combine_threshold
      PEX Reduce Minres Short short_threshold

      If you select this option and do not change the threshold value from the default of zero, then the command uses zero for the threshold.

      Select the LAYERWISE checkbox to specify the reduction of parasitic resistors for each layer and not across different layers.

    • Select NO to disable MinRes reduction and override rule file settings.

    Enable MinMutual reduction below

     

    (PEX Reduce Minmutual)

    Select this to remove parasitic mutual inductance below a threshold value to reduce netlist size. Enter a filtering threshold for small inductance coupling coefficient (K) values. The threshold value must be greater than zero and less than one (0 < K < 1).

    Enable Parallel MOS reduction

     

    (PEX Reduce Parallel MOS)

    Controls whether or not mosfets that are wired in parallel, and are the same model type and subtype, are reduced to a single mosfet in the generated netlist.

    Types — Enter ALL (the default) or specify one or more parallel mosfet or user-defined mosfet device type names to combine.

    R <threshold> — The resistance threshold between parallel device pins.

    SAMEWL — Enable to merge only parallel devices of the same width and length.

    Ground all coupling capacitors

    Convert each parasitic coupling capacitor between two nets into a capacitance from each net to ground in the extracted parasitic netlist. The value of each grounded capacitor equals the value of the original coupling capacitor.

    Uses the command: 

    calibre -xrc -fmt -g

    Keep coupling capacitors separate

    Keeps coupling capacitance separate from intrinsic capacitance. Uses the ‑cselect option when specifying nets to include in the extracted netlist. See “Including/Excluding Nets”.

    Keep parasitic net models marked with _noxref from netlists

     

    (PEX Netlist Noxref Net Names)

    Enabled — Nets in the design with “noxref” in the name still appear in the intentional circuit and are treated as ideal nets.

    Disabled — _noxref nets are suppressed.

    Default: Enabled.

    Use current-controlled current sources to model mutual resistance

     

    (PEX Netlist Mutual Resistance)

    As described.

    Default: Enabled.

    Set global net names

     

    (PEX Netlist Global Nets)

    Enable to remove the named nets from the subcircuit and place them in the global name definition. Enter the net name or names in the text field.

    Global nets must be excluded from the netlist, since you cannot netlist parasitics on global nets in hierarchical extraction. See “Including/Excluding Nets”.

    Specify device resistance model

     

    (PEX Netlist Device Resistance Model)

    Select to specify which model to use for parasitic resistance calculation. Select from:

    • SPICE — Use the internal HSPICE model.

    • CALCULATE — Use the width (W) and length (L) from the Device statement in the rule file.

    Default: SPICE
