Specifying Outputs for a Calibre PERC Run

Outputs for a Calibre PERC run include the PERC report file and the SVDB. The output for a Calibre PERC-LDL run is a DFM Database, which is specified in the rule file.

Throughout this section, be aware that Calibre Interactive creates new directories if you specify new pathnames. Also, if the option “Read Inputs/Outputs fields when loading rules file” is selected on the Misc tab of the Setup Preferences dialog box, relevant rule file settings will populate the Outputs pane when you click the Load button on the Rules pane.

Corresponding rule file statements: PERC Report, Mask SVDB Directory

Procedure

  1. Click Outputs to display the Outputs pane, as shown here for a Calibre PERC run.


  2. Specify the PERC Report filename.
  3. (Optional) Enable the “View report after PERC finishes” and “View Summary Report after PERC finishes” options to automatically display the Calibre PERC reports in a text editor window at the completion of your Calibre PERC run.
  4. If the run type is “PERC (Hierarchical)” or “PERC (Flat),” enable the “Create SVDB Database” option and specify the SVDB directory name, as desired. The SVDB database is required if you want to view results in Calibre RVE.

    You do not need to specify an output database if the run type is “PERC-LDL,” since Calibre PERC-LDL requires that the rule file include a DFM Database statement.

  5. If desired, check “Start RVE after PERC finishes” to automatically start Calibre RVE at the completion of your Calibre PERC run. If you are creating a runset for a batch run, you may want to enable “Start RVE after batch run” on the Misc tab of the Setup Preferences dialog box.
