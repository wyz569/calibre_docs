Specifying the Run Type for a DRC Run

The run type for a Calibre nmDRC run determines whether Calibre is run in hierarchical or flat mode, as a FastXOR run for Layout versus Layout comparison, or in Autowaiver Creation mode.

Procedure

  1. Click Inputs on the left panel of Calibre Interactive nmDRC.
  2. In the Run area at the top of the pane, choose from the following selections:

    • DRC (Hierarchical) — Run Calibre nmDRC in hierarchical mode, with the ‑hier command line option.

      When selected, you can also optionally specify to run in Calibre nmDRC Reconnaissance mode, its inverse, or Calibre DRC Analyze:

      • Recon — Runs with the -recon command line option. Automatically selects a subset of DRC rulechecks that identify gross errors early in the design and assembly process.

      • Recon Inverse — Runs with the “-recon inverse” command line option. Runs the subset of checks that Calibre nmDRC Recon mode does not run.

        For more information on Calibre nmDRC Recon, see “Chip-Level Verification” in the Calibre Solutions for Physical Verification manual.

      • Analyze — Runs with the -analyze command line option. Creates a DFM database containing error distribution results data for gross analysis of areas in the design that require attention.

        When “Analyze” is enabled, all options in the Outputs page are disabled aside from “Show results in RVE”. If “Show results in RVE” is enabled, the summary file opens after the DRC Analyze run completes.

        For more information on DRC Analyze, see “Reviewing DRC Analyze Results” in the Calibre Solutions for Physical Verification manual.

    • DRC (Flat) — Run Calibre nmDRC in flat mode, without the ‑hier command line option.

    • Calibre-CB (Flat) — Use the Calibre Cell/Block license package. This license package is only used for flat runs.

    • Fast XOR — Run FastXOR Layout Comparison. See the section “Performing a FastXOR Layout Comparison Run in Calibre Interactive” for instructions.

    • Autowaiver Creation — Generate a waiver shape database for the Calibre Auto-Waivers flow. See “Generating Waiver Shapes with Calibre Interactive”.

  3. The typical next step is specifying layout input data; see “Inputs for a Calibre Interactive nmDRC Run”.
