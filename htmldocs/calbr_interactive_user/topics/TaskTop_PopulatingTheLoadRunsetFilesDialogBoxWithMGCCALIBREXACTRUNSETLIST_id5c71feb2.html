Populating the Load Runset Files Dialog Box and Recent Files Lists

The environment variable MGC_CALIBRE_XACT_RUNSET_LIST populates the Load Runset Files dialog box and the recent files lists for runsets and text files. Used only in Calibre Interactive xACT.

The runset list file specified by MGC_CALIBRE_XACT_RUNSET_LIST uses the following formats. You cannot mix formats within the same file.

Basic Format — Used to specify single runsets:

runset1
runset2
runset3
...

Advanced Format — Used to specify single runsets, runset groups, runset type and info, and text files. 

XACT_RS paths: runset [runset ...] [runsetType: type] [runsetInfo: info]
XACT_RS paths: runset [runset ...] [runsetType: type] [runsetInfo: info]
...
TXT paths: text_file [text_file ...]
TXT paths: text_file [text_file ...]
...

where file entries are defined as follows:

  • XACT_RS paths: runset_file … — Keyword and parameter specifying one or more runset files. If more than one runset file is specified, they are grouped together in the Load Runset Files dialog box.

    Include one line for each runset file or runset group you want to list in the Recent Runsets list.

  • runsetType: type — Optional entry with “XACT_RS paths:” that specifies information to display in the Type column of the Load Runset Files dialog box.

  • runsetInfo: info — Optional entry with “XACT_RS paths:” that specifies information to display in the Info column of the Load Runset Files dialog box.

  • TXT paths: text_file … — Keyword and parameter specifying one or more text files. Text files are listed one per line in the Recent Text Files list.

Both relative and absolute filenames can be used. Relative filenames are evaluated relative to the Run Directory specified on the Rules page. Environment variables can be used in the filenames.

Prerequisites

  • “Requirements for Calibre Interactive xACT”.

  • A text file in the basic or advanced format, described previously.

Procedure

  1. Specify the runset list file with the environment variable MGC_CALIBRE_XACT_RUNSET_LIST. For example (in csh):
    setenv MGC_CALIBRE_XACT_RUNSET_LIST xact_runset_list

    If the file referenced by MGC_CALIBRE_XACT_RUNSET_LIST is a relative pathname, the tool searches for it in the directory from which you invoke Calibre Interactive xACT.

    You can define the environment variable as a list of multiple files:

    setenv MGC_CALIBRE_XACT_RUNSET_LIST "xact_list1 xact_list2"

    Different files can have different formats (advanced or basic), but you cannot mix formats within the same file.

  2. (Optional) Specify the default selected runset with the environment variable MGC_CALIBRE_XACT_RUNSET_FILE. When MGC_CALIBRE_XACT_RUNSET_LIST is defined, the runsets specified by MGC_CALIBRE_XACT_RUNSET_FILE are selected if they are in the list of predefined runsets.
  3. Invoke Calibre Interactive xACT.

    See “Invoking the Calibre Interactive xACT GUI”.

Results

When you choose File > Load Runset, the dialog box is populated with both the listed runsets and recent runsets. The Load Runset Files dialog box includes a dropdown selection to view “Predefined Runsets” (listed runsets) or “Recent Runsets.”

When you choose File > Recent Runsets, the list is pre-populated with the listed runsets.

If you used the advanced format runset list and specified text files, when you choose File > Recent Text Files, the list is pre-populated with the listed text files.

If you open other runsets or text files during the Calibre Interactive session, those files are added to the appropriate recent files list.

Examples

Basic Runset List

For this basic runset list:

runset_rcc
runset_rccl
runset_A
runset_B

The Load Runset Files dialog box is populated with the listed runsets. To view recent runsets, click the header and choose “Recent Runsets.”



When you choose File > Recent Runsets, that list is also pre-populated with the listed runsets.

Advanced Runset List

This is an example of an advanced format runset list file:

XACT_RS paths: runset_rcc runsetInfo: RCC
XACT_RS paths: runset_rccl runsetInfo: RCCL
XACT_RS paths: runset_A runset_B
TXT paths: $PROJECT/README_Project

This example specifies two single runsets, runset_rcc and runset_rccl, with relative filenames; these files must exist in the Run Directory specified on the Rules pane. The optional runsetInfo entry is provided for these two runsets.

The runsets runset_A and runset_B are specified as a group. They appear as a group in the Load Runset Files dialog box, and are both loaded if selected as a group. The two runsets are listed separately in the Recent Runsets list.

The text file, $PROJECT⁄README_Project, is specified with an absolute path and uses an environment variable for the directory path.
