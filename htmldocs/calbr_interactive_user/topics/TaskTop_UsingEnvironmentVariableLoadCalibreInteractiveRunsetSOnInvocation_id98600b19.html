Using an Environment Variable to Load Calibre Interactive Runset(s) on Invocation

Define the environment variable MGC_CALIBRE_DRC_RUNSET_FILE to point the runset(s) you want to load. Replace DRC in the environment variable name with LVS, PERC, PEX, or DFM for those applications. The runset(s) is loaded when you start Calibre Interactive.

Note:

Runsets specified on the command line or with the MGC_CALIBRE_***_RUNSET_LIST environment variable both take precedence over a MGC_CALIBRE_***_RUNSET_FILE environment variable. See “Loading Runsets on Invocation” for the complete order of precedence for automatic loading runsets in Calibre Interactive.

Prerequisites

  • You have a Calibre Interactive runset for your application. Multiple runsets may be specified.

Procedure

  1. Depending on your Calibre Interactive application, define one of the following environment variables with the pathname of the runset(s) to load:

    Application

    Environment Variable

    DRC

    MGC_CALIBRE_DRC_RUNSET_FILE

    LVS

    MGC_CALIBRE_LVS_RUNSET_FILE

    PERC

    MGC_CALIBRE_PERC_RUNSET_FILE

    PEX

    MGC_CALIBRE_PEX_RUNSET_FILE

    DFM

    MGC_CALIBRE_DFM_RUNSET_FILE

    Specify multiple runsets as a space separated list enclosed in double quotes.

  2. Start Calibre Interactive.

Results

The specified runset(s) are loaded when you invoke Calibre Interactive. If multiple runsets are used, they are loaded in order and the settings loaded last take precedence.

If you set one of these environment variables to “ ” (blank), no runset is opened.

Examples

setenv MGC_CALIBRE_DRC_RUNSET_FILE "drc_runset"
setenv MGC_CALIBRE_PEX_RUNSET_FILE "pex_runset1 pex_runset2"
