OpenAccess/LEFDEF Options Tab

Access with: Setup > Options, the Database > OpenAccess/LEFDEF tab. (For PEX, it is the Database > OpenAccess tab.)

You can control database conversion options for third party databases using Calibre Interactive. These options apply to OpenAccess input for all Calibre Interactive applications, and to LEF⁄DEF input for all applications except PEX (parasitic extraction).

Note:

If the environment variable MGC_CALIBRE_DB_READ_OPTIONS is defined, the Database > OpenAccess/LEFDEF sub-tab displays the options set with the environment variable; the GUI controls are not active.

Tip

When using direct read of third-party database formats (input Format set to LEFDEF or OPENACCESS) the database translation options for the fdi2gds utility are reported at the beginning of the control file after the line “// MGC_CALIBRE_DB_READ_OPTIONS”. To review these settings, press the Ctrl key and click the Run app button to open the control file, then make corrections on the Database > OpenAccess/LEFDEF sub-tab if needed.

CAUTION:

If you use the statement SVRF FDI Options to set third-party database read options in the rule file, these options are not used in a run started from Calibre Interactive. When using Calibre Interactive, the FDI options must be set in the GUI or with MGC_CALIBRE_DB_READ_OPTIONS.

Description

The options on the tab are described in the following tables. See “Database Translation Utilities” in the Calibre Layout Comparison and Translation Guide for a more detailed description.

Fields

Table 1. Read Options in the OpenAccess/LEFDEF Tab

Option

(fdi2gds/fdi2oasis option)

Description

Read:

  

    Texts

    (‑noText)

Read text objects.

The default is to read text objects.

    Pins

    (‑noPinShape)

Read pin shapes.

The default is to read pin shapes.

    Blockages

    (‑outputBlockages)

Read blockages for LEF/DEF or OpenAccess databases; see the documentation of the ‑outputBlockages option for important information.

The default is to not read blockages.

    Instance Pins

    (‑instPinAsShape)

Read instance pin shapes.

The default is to not read instance pin shapes.

Read Net Names As

(‑annotateNets)

Specifies to read net names and how to annotate them.

This option is off by default.

Read Pin Names As

(‑annotatePins)

Specifies to read pin names and how to annotate them.

This option is off by default.

Read Net Type As

(‑annotateNetType)

Specifies to read net types and annotate them as a property.

This option is off by default.

Read Instance Name As

(‑annotateInsts)

Specifies to read instance names and annotate them as a property.

This option is off by default.

Abort on Reading Empty PCells

(‑abortOnEmptyPCell)

Causes the Calibre application to exit when an empty PCell appears in an OpenAccess database.

This option is on by default in the GUI.
Table 2. Mapping Files Options in the OpenAccess/LEFDEF Tab

Option

(fdi2gds/fdi2oasis option)

Description

Use Layer Map File

(‑layerMap)

Specifies the pathname of the layer mapping file to use when the database is read. The layer mapping file maps specific input layers to a user-specified list of output layers and datatypes. By default, the application maps objects with their corresponding layer number and a datatype of 0 (zero).

See the “Example: Setting the Layer Map, Cell Map, and Object Map for OpenAccess Databases”.

Use Object Map File

(‑objectMap)

Specifies the pathname of the object mapping file to use when the database is read. The object mapping file determines what objects are output to GDS, and how they are mapped.

See the “object type tables” in the Calibre Layout Comparison and Translation Guide, and the “Example: Setting the Layer Map, Cell Map, and Object Map for OpenAccess Databases”.

Use Net Properties File

(‑netProp)

Specifies the pathname of the net properties file to use when the database is read.

Use Cell Map File

(‑cellMap)

Specifies the pathname of the cell mapping file to use when the database is read. The cell mapping file maps input cell names to user-specified cell names. The contents of the cell mapping file are layout system dependent. By default, input cell names are used throughout the run.

See “Example: Setting the Layer Map, Cell Map, and Object Map for OpenAccess Databases”.

Use Input Exception File

(‑inputException)

Specifies the pathname to a file that gives exception levels for certain tool behaviors.

Description

Use Cell GDS Files

(‑cellGDS)

Table 4. Other Options in the OpenAccess/LEFDEF Tab

Option

(fdi2gds/fdi2oasis option)

Description

OpenAccess View List

(‑viewList)

Specifies the order in which to open views for lower-level cells. By default the view list is empty.

Write Log File

(‑logFile)

Specifies to write a runtime log file for the translation.

Write Map Info File

(‑mapInfo)

Specifies to write a map information output file which reports the mappings applied from the ‑cellMap, ‑layerMap, and ‑propertyMap options.

Write Layer Names File

(‑layerNames)

Specifies to use an output layer naming file.

Set OA_HOME when CDS_INST_ROOT or CDS_INST_DIR is set

Specifies that OA_HOME is set to $CDS_INST_DIR/share/oa or $CDS_INST_ROOT/share/oa if CDS_INST_ROOT or CDS_INST_DIR is defined. CDS_INST_ROOT is used if both variables are set.

Set CALIBRE_READDB_LD_LIBRARY_PATH when CDS_INST_ROOT or CDS_INST_DIR is set

Specifies that $CDS_INST_ROOT/tools/lib and $CDS_INST_ROOT/tools/lib/64bit are added to CALIBRE_READDB_LD_LIBRARY_PATH if CDS_INST_ROOT is defined. The corresponding action is taken if CDS_INST_DIR is defined. CDS_INST_ROOT is used if both variables are set.

Set PATH when CDS_INST_ROOT or CDS_INST_DIR is set

Specifies that $CDS_INST_ROOT/share/oa/bin, $CDS_INST_ROOT/tools/dfII/bin, and $CDS_INST_ROOT/tools/bin are added to the PATH variable if CDS_INST_ROOT is defined. The corresponding action is taken if CDS_INST_DIR is defined. CDS_INST_ROOT is used if both variables are set.

Set CDS_EXP_PCELL_DIR when CDS_INST_ROOT or CDS_INST_DIR is set

Specifies that CDS_EXP_PCELL_DIR is set to .⁄expressPcells if CDS_INST_ROOT or CDS_INST_DIR is defined.

Set CDS_EXP_ALL_PARAMS when CDS_INST_ROOT or CDS_INST_DIR is set

Specifies that CDS_EXP_ALL_PARAMS is set to true if CDS_INST_ROOT or CDS_INST_DIR is defined.

Set CDS_ENABLE_EXP_PCELL when CDS_INST_ROOT or CDS_INST_DIR is set

Specifies that CDS_ENABLE_EXP_PCELL is set to TRUE if CDS_INST_ROOT or CDS_INST_DIR is defined.
