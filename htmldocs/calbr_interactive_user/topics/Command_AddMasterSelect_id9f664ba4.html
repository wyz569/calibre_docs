addMasterSelect

Function call for the custom configuration file for Calibre Interactive xACT

Specifies that the visibility of a control depends on the state of another control (the primary control) defined in the Calibre Interactive xACT configuration file. The dependent control is displayed if the primary control is not selected.

Usage

ctrl_var_dep.addMasterSelect(ctrl_var_master)

Parameters

ctrl_var_dep

The variable name for the dependent control.

ctrl_var_master

The variable name for the primary control.

Description

addMasterSelect designates that the state of a primary control (ctrl_var_master) determines the visibility of the dependent control (ctrl_var_dep). The dependent control is displayed only if the checkbox for the primary control is not checked.

Examples

A dependent control depends on the state of an addDefine control, which does not have a value. 

// primary control 
var dCornerMaster = tool.addDefine("USE_CORNER");
dCornerMaster.value = false;
dCornerMaster.label = "Use corners";
// info label when master USE_CORNER is not selected
var noCornerLb = tool.addLabel("Corners not used")
noCornerLb.addMasterSelect(dCornerMaster);
