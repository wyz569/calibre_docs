Template Definitions in the Calibre Interactive xACT Runset

File-naming templates for the Calibre Interactive xACT GUI are saved to the runset as function calls.

Function Syntax

pex.setNameTemplate("option", "template_value")

Parameters

  • template_value

    The value of the template, which is a string for most template options. The string value can include replaceable parameters given in the following table. The value can also include an environment variable if the relevant GUI option supports environment variables.

    Unlike the other template options, the template_value for export settings (nt_efLayVwr and nt_efSrcVwr) is an integer, not a string. The possible values are described in the preceding table of option values.

    Table 1. Replaceable Parameters in Template Values for Calibre Interactive xACT

    Parameter

    Definition

    %l

    The primary cell in the layout database.

    %s

    The primary cell in the source netlist.

    %L

    The name of the layout library for OpenAccess databases.

    %V

    The name of the layout view for OpenAccess databases.

    %S

    The name of the source library for OpenAccess databases.

    %W

    The name of the source view for OpenAccess databases.

    Replaceable parameters for the output parasitic netlist filename

    %X

    The extraction type. %X is replaced by these values:

    • rcc — R+C+CC extraction

    • rc — R+C extraction

    • r — R only extraction

    • cc — C+CC extraction

    • norc — No R/C extraction

    %x

    The parasitic netlist format. %x is replaced by these values:

    • cv — Calibre View netlist

    • dspf — DSPF format

    • spi — Eldo format

    • sp — HSPICE format

    • scs — SPECTRE format

    • spef — SPEF format

    • spice — SPICE format

    %o

    The netlist format option. %o is replaced by these values:

    • ar — ADiT Rail netlist compatibility

    • pt — PrimeTime netlist compatibility

    • hsim — HSIM netlist compatibility

    • us — ULTRASIM netlist compatibility

    • totem — TOTEM netlist compatibility

    • qc — QUICKCAP netlist compatibility

    • ng — Net Geometry

 

Example 1

pex.setNameTemplate("nt_layFile", "%l.exported.gds")

This sets the template for the layout file. The template is applied when Calibre Interactive xACT is invoked from a design tool. The replaceable parameter %l is replaced with the name of the open cell in the design tool.

If the open cell is opamp, the layout filename on the Inputs page is opamp.exported.gds. When the runset is saved, this is saved for the corresponding runset option: 

pex.layout.layoutFile.value = “opamp.exported.gds”

Example 2

The %X, %x, and %o replaceable parameters can be used to indicate the extraction type and netlist format:

pex.setNameTemplate("nt_netFile", "%l.pex.%X.%x")

For an open cell named opamp, RCC extraction, and a SPEF netlist format, the parasitic netlist file name is opamp.pex.rcc.spef. When the runset is saved, this is saved for the corresponding runset option:

pex.pexNetlist.netlistFile.value = "opamp.pex.rcc.spef"
