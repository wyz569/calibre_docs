Populating the Calibre Interactive GUI

The settings in the Calibre Interactive GUI can come from several sources, including direct user input, a Calibre Interactive runset, template settings, and the Customization GUI.

Calibre Interactive settings come from the following sources:

  • Default values — Default values are programmed into Calibre Interactive. You see the default values when you open Calibre Interactive without a runset and not connected to a design tool. The default settings vary between applications.

  • Runset — A Calibre Interactive runset is a text file that stores the Calibre Interactive GUI settings. You can load a runset in a later Calibre Interactive session to restore the GUI settings from a previous session. See “About Calibre Interactive Runsets” for methods to save and load a runset.

  • Default runset — A default runset allows you to specify your own default values for Calibre Interactive. The default runset is read when Calibre Interactive is invoked without a runset, as described in “Default Runsets in Calibre Interactive”.

  • System-generated parameters based on template settings — When Calibre Interactive is invoked from a layout or schematic viewer, the default behavior is to overwrite certain GUI settings with values derived from the open database. For example, the output results database in Calibre Interactive nmDRC is automatically named top_cell.drc.results, where top_cell is the name of the primary layout cell in the open design. See “Templates for File Naming”.

  • Rule file — Settings from the rule file populate the GUI when you click the Load button on the Rules pane. You can also instruct Calibre Interactive to load the rule file when a runset is loaded, as described in “Automatically Loading a Rule File.”

  • Customization file — The Customization GUI can set values in the Calibre Interactive GUI using runset control statements in the customization file, as described in the “Customization Files” chapter. The Customization GUI also adds statements to the Calibre Interactive control file.

  • User input — Direct user input overrides any preset value or setting in the Calibre Interactive GUI.

The settings present in the Calibre Interactive GUI at the time of the run always take precedence over rule file settings if there is a conflict.
