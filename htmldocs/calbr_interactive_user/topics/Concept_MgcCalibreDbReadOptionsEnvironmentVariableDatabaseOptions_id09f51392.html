MGC_CALIBRE_DB_READ_OPTIONS Environment Variable for Database Options

You can specify database read options for third party databases with the environment variable MGC_CALIBRE_DB_READ_OPTIONS. The options apply to OpenAccess input for all Calibre Interactive applications, and to LEF/DEF input for all applications except PEX.

Define the environment variable MGC_CALIBRE_DB_READ_OPTIONS with the desired options before you start Calibre Interactive. See the Layout System statement and the table MGC_CALIBRE_DB_READ_OPTIONS in the SVRF Manual for a description of the environment variable.

For example, using the C shell:

setenv MGC_CALIBRE_DB_READ_OPTIONS "-layerMap layer_map.txt"

When Calibre Interactive is started, the Database > OpenAccess/LEFDEF tab on the Setup > DRC Options pane displays the options set with the environment variable; the GUI controls are not active.

Note that if MGC_CALIBRE_DB_READ_OPTIONS is specified, then the options displayed on the Database > OpenAccess/LEFDEF tab are not saved when you save a runset unless you also set MGC_CALIBRE_SAVE_ALL_RUNSET_VALUES to a non-zero value.

If your layout database format is OpenAccess or LEF/DEF and you are using a Calibre Interactive control file to perform a command line Calibre run, then you must set MGC_CALIBRE_DB_READ_OPTIONS in order for the database translation options to be set correctly. The required value for the environment variable is included as a comment at the beginning of the control file; an example is shown here:

//  If this rule file is run directly from the command line,
//  to set the environment correctly for database translation,
//  please set the value of the environment variable
//  MGC_CALIBRE_DB_READ_OPTIONS to:
// 
//  -abortOnEmptyPCell
