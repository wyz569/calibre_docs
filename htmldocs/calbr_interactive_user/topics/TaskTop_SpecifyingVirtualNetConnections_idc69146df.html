Specifying Virtual Net Connections in Calibre Interactive

Using the virtual net connections options, you can instruct Calibre to treat multiple, physically-disjoint layout net segments as if they were actually connected.

Corresponding rule file statements: Virtual Connect Colon, Virtual Connect Name, Virtual Connect Report, Virtual Connect Report Maximum

Note:

The Virtual Connect statements are primarily intended for Calibre nmLVS, PERC, and PEX applications.

Virtual connections in Calibre nmDRC and DFM applications can cause unintended results. The statements do not function when DRC Incremental Connect Yes is specified because virtual connections do not obey connectivity zones in a rule file.

Procedure

  1. Click the app Options button on the left panel, where app is one of DRC, LVS, DFM, PERC, or PEX. If the button is not present, choose Setup > app Options from the menu to display it.
  2. Click the Connect tab.
  3. Select the handling of nets with a colon (:).

    • Connect nets with colon (:) — Virtually connect physically disjoint nets whose names agree up to the first colon character found in the names. The name of the new net is the shared name leading up to the colon. For example, virtually connect net clock:1 to net clock:2 as the net clock.

  4. Select the handling of named nets. Net name matching is case-insensitive for all options.

    • Don’t connect nets by name — Treat synonymous disjoint nets as separate nets. This is the default.

    • Connect all nets by name — Virtually connect physically disjoint nets whose names are identical. For example, choosing this option virtually connects net RESET to net Reset and net reset.

    • Connect nets named — Virtually connect physically disjoint nets whose names are in the list. The wildcard character “?” matches zero or more characters.

  5. Specify reporting options:

    • Report connections made by name — Instruct Calibre to document virtual connections in the report.

    • Don’t report connections that are physically connected — Do not report virtual connections by name that are physically connected higher in the hierarchy; only virtual connections that are not physically connected higher in the hierarchy are included in the report. (Virtual Connect Report YES UNSATISFIED)

    • Max. connected messages printed in report — Select “All” or specify a “Count” to control the number of messages included in the report.
