Invoking the Calibre Interactive xACT GUI

You invoke Calibre Interactive xACT from a design tool or from the command line. Calibre DESIGNrev and Cadence Virtuoso are supported.

Prerequisites

  • “Requirements for Calibre Interactive xACT”

  • (Optional) You can specify one or more runsets to load automatically by defining the environment variable MGC_CALIBRE_XACT_RUNSET_FILE. For example: 

    setenv MGC_CALIBRE_XACT_RUNSET_FILE "runset1_path runset2_path"

    The specified runset(s) are loaded when Calibre Interactive xACT is invoked. The ‑runset command line option has precedence over the environment variable.

  • (Optional) You can define the environment variable MGC_CALIBRE_XACT_CONFIG_FILE to specify one or more configuration files. The ‑config command line argument has precedence over the environment variable.

  • (Optional) You can specify the environment variable MGC_CALIBRE_XACT_RUNSET_LIST to pre-populate the Recent Runsets and Recent Text Files lists in the File menu. When this environment variable is defined, the runsets specified by MGC_CALIBRE_XACT_RUNSET_FILE are selected. See “Populating the Load Runset Files Dialog Box and Recent Files Lists”.

Procedure

  1. Use one of the following methods to start the GUI:

    Invoke from …

    Method

    Calibre DESIGNrev

    Verification > Run xACT

    Cadence tools

    Calibre > Run xACT

    Command line

    calibre -gui -xact [-runset runset]… [‑config file]… [-batch]

    Calibre Interactive palette
    calibre -gui

    The -runset and -batch command line options are the same as for other Calibre Interactive applications.

    The ‑config file switch and argument option loads a custom configuration file; see “GUI Customization for Calibre Interactive xACT”. You can specify multiple configuration files.

    Note:

    For consistency with other Calibre Interactive applications, the -runset switch is optional, but it is recommended in all cases. In particular, the ‑runset switch should be used when using “‑config config_file” to specify a custom configuration file, to avoid parsing errors.

    It is also possible to specify multiple runsets as a space-separated list of runsets. However, best practice is to specify the ‑runset switch with each runset.

  2. Proceed to “Setting Up and Starting a Calibre Interactive xACT Run”.

Results

If you invoke the GUI from a layout tool, the open design is automatically used as the layout input and the extracted netlist is named cell.pex.netlist, where cell is replaced with the name of the open design cell.
