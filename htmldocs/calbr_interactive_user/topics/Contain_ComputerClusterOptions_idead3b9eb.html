Computer Cluster Options

Large Calibre jobs may benefit from the performance of a computer cluster (also known as load balancing distributed resources). Calibre Interactive offers an interface for setting up execution of Calibre jobs on IBM® Spectrum™ LSF® and Univa® Grid Engine®, along with a generic interface for other distributed computing solutions.

The most important consideration when configuring a computer cluster for running a Calibre job is to take care not to overload or overrun the primary or remote hosts. You can manage this concern by being familiar with the configuration options of the computer cluster. It is particularly important to consult both the documentation for your distributed solution and your system administrator prior to testing your Calibre Interactive computer cluster configuration.

Initial setup tasks and configurations for each type of cluster vary. In some cases you need to take configuration steps beyond the default setup for a satisfactory distributed run.

Tip

Environment variables set using the instructions in the section “Viewing and Editing Environment Variables” are available to the cluster host in any computer cluster solution.

Note:

You can Ctrl-left-click in any host table to display the shell script that will execute based on the current configuration.
