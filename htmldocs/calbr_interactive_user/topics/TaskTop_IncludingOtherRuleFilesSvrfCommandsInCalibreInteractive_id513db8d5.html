Including Other Rule Files and SVRF Commands in Calibre Interactive

You can specify additional rule files and SVRF commands using the Calibre Interactive GUI. This allows you to make adjustments to the run without editing your rule file.

Procedure

  1. View the Options pane for your Calibre Interactive application. If the Options button is not present in the left side of the Calibre Interactive window, choose Setup > app Options, where app is one of DRC, LVS, DFM, PEX, or PERC.
  2. Click the Include tab.
  3. To include one or more rule files, enter one or more include filenames in the top text area.

    Enable “Include Rule Files and Statements After Main app Rules File” if you want the included rule files to be placed after the rule file specified on the Inputs pane. This is useful if the order of rule file statements matters; this can happen if the main rule file includes conditional compilation directives, such as #DEFINE, which can impact the compilation of statements found in an included file. This option is off by default.

  4. Do the following to include individual rule file statements:
    1. Enable “Include Rule Statements.”
    2. If the rule file is TVF, use the button dropdown list to indicate whether you are including SVRF or TVF statements.

      (Only SVRF statements may be included in an SVRF rule file.)

    3. Enter one or more commands into the text area; one command per line is allowed.

      The rule file statements are included in the Calibre Interactive control file; if SVRF statements are included in a TVF rule file, the SVRF statements are enclosed within a TVF::VERBATIM block.

      Tip

      When you click the Load button on the Rules pane or the Run app button, Calibre compiles all included rule files and rule file statements along with the rule file specified in the Rules pane.
