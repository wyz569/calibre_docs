Customization File to Select ERC Rule Checks

The customization file in this example selects the ERC rule checks that are executed in a Calibre Interactive nmLVS run.

The example defines a Tcl variable _select_by to determine whether all checks are run or if checks are selected by group name. The ‑master option with the _select_by variable is used to determine if the CUSTOM::CHECK controls are displayed; see “Primary-Dependent Commands”.

If you are also using the Select Checks dialog box in Calibre Interactive nmLVS to select ERC checks, then the selections in the Customization GUI initialize settings of the Select Checks dialog box. as described in “Precedence of CUSTOM::CHECK Controls and Check Selection in Calibre Interactive”. See “Selecting ERC Checks” for information on the Select Checks dialog box.

See “Rule Check Selection in Calibre Interactive Batch Runs” for an example using a customization file with a Calibre Interactive batch run.

Example

### Customization file to select ERC checks in LVS

# define var to select all checks, or select individually
# no commands needed for "All checks" since this is the default operation
set _select_by [CUSTOM::VARIABLE -name "select_by" \
	-choices {{"All checks" 0} {"individually" 1} } -initval {0} \
	-prompt "select ERC checks:" -display 1 -tool LVS ]
	
# select checks individually
CUSTOM::LABEL -prompt "Select ERC checks individually " \
	-tool LVS -master [list $_select_by 1]
CUSTOM::CHECK -name "ERC_Path" -prompt "ERC_Path" \
	-tool LVS -master [list $_select_by 1]
CUSTOM::CHECK -name "Floating" -prompt "Floating" \
	-tool LVS  -master [list $_select_by 1]
CUSTOM::CHECK -name "ERC_Check" -prompt "ERC_Check" \
	-tool LVS  -master [list $_select_by 1]

CAUTION:

The Check Selection Recipe is the preferred method for selecting the rule checks that are evaluated in a Calibre Interactive nmDRC or Calibre Interactive DFM run.

If you are using CUSTOM::CHECK controls in a Customization GUI for Calibre Interactive nmDRC or DFM, make sure that the “Checks selected in the rules file” built-in check recipe is selected, as described in “Precedence of CUSTOM::CHECK Controls and Check Selection in Calibre Interactive”.
