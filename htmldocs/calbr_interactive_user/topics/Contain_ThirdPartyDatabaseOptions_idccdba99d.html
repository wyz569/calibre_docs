Third Party Database Options

You can specify the options that Calibre uses when reading in OpenAccess and LEF/DEF databases.

You set third-party database options with one of the methods in the following table.

Table 1. Methods for Setting Third Party Database Options

Method

Comment

With an environment variable

Set the environment variable MGC_CALIBRE_DB_READ_OPTIONS before starting Calibre Interactive. See “MGC_CALIBRE_DB_READ_OPTIONS Environment Variable for Database Options”.

In the GUI

Choose Setup > app Options and set options on the Database > OpenAccess/LEFDEF sub-tab, where app is one of DRC, LVS, DFM, PERC, or PEX. See “OpenAccess/LEFDEF Options Tab”.

Third party database options include several mapping files, as explained in the section “Example: Setting the Layer Map, Cell Map, and Object Map for OpenAccess Databases”.

Note:

If the environment variable MGC_CALIBRE_DB_READ_OPTIONS is defined, the Database > OpenAccess/LEFDEF sub-tab displays the options set with the environment variable; the GUI controls are not active.

CAUTION:

If you use the statement SVRF FDI Options to set third-party database read options in the rule file, these options are not used in a run started from Calibre Interactive. When using Calibre Interactive, the FDI options must be set in the GUI or with MGC_CALIBRE_DB_READ_OPTIONS.
