Specifying a Rule File for a Calibre PERC Run

You need to load a rule file for any Calibre PERC run. You can load a rule file automatically with a runset or you can specify one manually using this procedure.

To automatically generate a rule file for the High Level Checks flow, see “Creating a Calibre PERC Rule File with High Level Checks”.

Prerequisites

  • Calibre Interactive PERC is running; see “Invoking Calibre Interactive Tools”

  • A rule file for Calibre PERC is available

Procedure

  1. Click the Rules button to display the Rules pane.


  2. Specify the rule filename in the PERC Rules File field. You can use the View button to view and edit the rule file.

    Be aware that Calibre Interactive automatically recognizes a rule file with the literal name “rules” and displays the name in green. Because the name is shown in green, you may assume the file has been loaded but it has not been; the file must be explicitly loaded using the Load button.

  3. Click Load to populate the GUI with the settings in the rule file.

    A rule file can also be specified in a runset. After loading that runset, the rule file is included in the control file but its setting will not be loaded until you explicitly click the Load button.

    Tip

    The setting “Read Inputs/Outputs fields while loading rules file” on the Misc tab of the Setup Preferences dialog box controls whether or not rule file settings populate the GUI options on the Inputs and Outputs panes when Load is clicked. This setting is enabled by default.

    After you load a rule file, any information you specify in the GUI supersedes the information in your loaded rule file.

  4. Specify the run directory name in the PERC Run Directory field.
  5. (Optional) View the layer derivation tree is at the bottom of the Rules pane; see “Layer Derivations Tree” for information.
