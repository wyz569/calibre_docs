Customization File to Read and Set Calibre Interactive GUI Controls

You can read and set Calibre Interactive GUI controls using a customization file, as shown in this example.

In particular, this example customization file does the following:

  • Creates text entry fields for the top cell and results database controls and initializes them to the Calibre Interactive GUI settings.

  • Creates checkbox controls for the “Output cell error in cell space” and the “Show results in RVE” controls and initializes them to the Calibre Interactive GUI settings.

  • Defines a callback procedure which does the following:

    1. Reads the setting for the text entry fields for top cell and results database in the Customization GUI.

    2. Reads the state of the checkbox for the “Output cell error in cell space” and the “Show results in RVE” controls in the Customization GUI.

    3. Writes the Customization GUI settings to the corresponding Calibre Interactive GUI controls.

Example

#customization file 
CUSTOM::SETWINGEOM 500x250

# display control for top cell and initialize to current setting
set topCellEntry [CUSTOM::VARIABLE -name "topCell_var" -choices {""} \
	-initval [CUSTOM::getRunsetVarValue drcLayoutPrimary]  \
	-boolean 0 -select 1 -prompt "Top Cell (GUI setting)" ]
# display control for results DB and initialize to current setting
set resultsEntry [CUSTOM::VARIABLE -name "resultsFile_var" \
	-choices {""} -initval [CUSTOM::getRunsetVarValue drcResultsFile]  \
	-boolean 0 -select 1 -prompt "Results File (GUI setting)"  -tool DRC]
	
# display Boolean checkbox for Output in Cell Space and initialize to 
# current value
set cellNameEntry [CUSTOM::DEFINE -name "cellName_var" \
	-select [CUSTOM::getRunsetVarValue drcCellName]  \
	-prompt "Output in cell space (GUI setting)" -tool DRC]
	
#display Boolean checkbox for Start RVE and initialize to current value
set startRVEEntry [CUSTOM::DEFINE -name "startRVE_var" \
	-select [CUSTOM::getRunsetVarValue drcStartRVE]  \
	-prompt "Start RVE (GUI setting)" ]
proc myOKCallback { } {
    global topCellEntry 
    global resultsEntry
    global cellNameEntry
    global startRVEEntry
  
	# get settings from customization GUI control
	# use cget -cvar to get value, cget -bvar to get checkbox setting
    set wv_topCell [$topCellEntry cget -cvar]
    set wv_results [$resultsEntry cget -cvar]
    set wb_cellName [$cellNameEntry cget -bvar]
    set wb_startRVE [$startRVEEntry cget -bvar]
  	# write setting to Calibre Interactive GUI 
	CUSTOM::setRunsetVarValue drcLayoutPrimary [set $wv_topCell]
	CUSTOM::setRunsetVarValue drcResultsFile [set $wv_results]
	CUSTOM::setRunsetVarValue drcStartRVE [set $wb_startRVE]
    
	# test for existence of CI control drcCellName before writing
	if {[CUSTOM::runsetVarExists drcCellName]} {
		CUSTOM::setRunsetVarValue drcCellName [set $wb_cellName]
	} else {
		CUSTOM::setOKCallbackError "no such runset var: drcCellName"
		return -1
	}
}
CUSTOM::setOKCallback myOKCallback

Figure 1 show how the Customization GUI looks and the steps for changing and setting the Top Cell value in the Calibre Interactive GUI.

Figure 1. Customization Example to Set Calibre Interactive GUI Controls

When CUSTOM::getRunsetVarValue is used to initialize a control as in this example, CUSTOM::getRunsetVarValue is only evaluated the first time the customization file is read, unless the customization file changes. This means that changing the setting in Calibre Interactive after closing the Customization GUI does not result in a new initial value the next time the Customization GUI is opened.
