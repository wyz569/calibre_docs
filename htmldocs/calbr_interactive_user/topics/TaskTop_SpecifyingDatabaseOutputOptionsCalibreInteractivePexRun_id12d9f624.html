Specifying Database Output Options for a Calibre Interactive PEX Run

You specify that results database name and options on the Outputs tab of the Outputs pane.

Prerequisites

  • Calibre Interactive PEX is open; see “Invoking Calibre Interactive Tools”.

Procedure

  1. Click Outputs on the left panel.
  2. Click the SVDB tab to specify the standard verification database and its options. These options control the parameters for the Mask SVDB Directory command.

    SVDB directory — Required. Specify the SVDB directory name.

    Start RVE after PEX — Optional. Select this to start Calibre RVE after the PEX run completes. If you are creating a runset for a batch run, you may want to enable “Start RVE after batch run” (Setup > Preferences, Misc tab).

    Generate cross-reference data for RVE — Recommended and on by default. Generates the binary cross-reference database (XDB) for Calibre RVE and the Query Server.

    Generate ASCII cross-reference files — Generate ASCII cross-reference (ixf, nxf, lph, sph) files in the SVDB database.

    Generate Calibre Connectivity Interface data — Generate Calibre Connectivity Interface (CCI) data in the SVD database; requires a Calibre Connectivity Interface (CI) license.

    Suppress pin-location information — Add the NOPINLOC keyword to the Mask SVDB Directory statement. This option is only available when “Generate Calibre Connectivity Interface data” is selected.

  3. Select the PDB generation method.

    By default, the PDB is recreated every time the PDB generation step is run.

    Enable “Generate PDB incrementally” when performing iterative extraction. This is often done when extracting a netlist with mixed parasitic networks. See the sections “Extracting a Netlist with Mixed Parasitic Networks” and “Mixing Parasitics from Calibre Interactive” in the Calibre xRC User’s Manual, and “Controlling PEX Step Execution” in this manual.
