Rule File Generation Tab

Access with: The Rule File Generation tab on the Inputs pane (only available when “Create XOR rules file” is enabled for the “Fast XOR” run type in Calibre Interactive nmDRC)

The Rule File Generation tab for FastXOR allows you to specify output file and DBdiff options when the FastXOR rule file is created by Calibre Interactive.

Objects

Table 1. Format Options for FastXOR Rule File Generation

Option

Description

Result Format

Select any combination of ASCII, GDS, and OASIS for the format of the output results database(s).

Rules File

Specifies the name of the generated rule file. The default is rules.fxor.

This option is not displayed by default. To display the option, select Setup > Preferences, click the Disable tab, clear the checkbox for “Disable Fast XOR generated rules file custom name,” and click OK to close the Setup Preferences dialog box.

Output Files Prefix

Enter a prefix for the created output files. See the “prefix” option in the DBdiff ‑write_xor_rules syntax.

Note: The prefix specified here does not override output filenames specified on the Outputs pane of Calibre Interactive, since GUI filenames take precedence over output filenames specified in the rule file.

Add New and Missing shapes rules

Adds checks in the FastXOR rules file which use the NOT operation to identify new and missing shapes.

Common Only

When enabled, only compare layers that are common to both designs.

Table 2. Mapping Files Options for FastXOR Rule File Generation

Option

Description

Use Layer Map File

Specifies a layer mapping file. See the ‑layermap argument for ‑write_xor_rules option in DBdiff.

Use Base Layers

Specifies layers for the ‑base_layers argument for the ‑write_xor_rules option in DBdiff. This option adds the Layout Base Layer statement with the specified layers to the generated rule file.

Use Include Rules File

Specifies a file containing SVRF statements that are imported directly into the generated FastXOR rule file. See the ‑include_rules argument to DBdiff.

Use Include Layers File

Specifies a file listing layers to compare. See the ‑include_layer argument to DBdiff.

Use Exclude Layers File

Specifies a file listing layers to exclude from comparison. See the ‑exclude_layer argument to DBdiff.
Note:

The Use Layer Map File and Use Base Layers options are mutually exclusive by default. To use both options simultaneously, select Setup > Preferences, click the Disable tab, clear the checkbox “Disable Fast XOR layer map and base layers combined usage,” and click OK to close the Setup Preferences dialog box.
