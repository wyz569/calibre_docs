Specifying Extracted Netlist Options for a Calibre Interactive PEX Run

The Netlist and Nets tabs on the Outputs pane specify the netlist format and which nets to extract parasitics for.

Prerequisites

  • Calibre Interactive PEX is open; see “Invoking Calibre Interactive Tools”.

Procedure

  1. Click Outputs on the left panel.
  2. Click the Netlist tab to specify the extracted netlist format and options.

    The settings on the Netlist tab control the parameters for the PEX Netlist command.

    1. Choose the output format from the “Format” dropdown list; available options are CALIBREVIEW, DSPF, ELDO, HSPICE, SPECTRE, and SPEF. Some formats have options that can be selected in the bottom half of the format dropdown list.

      If a format selection appears in red text, then the selection is not allowed with one of the other output selections, such as the extraction type or the “Use Names From” setting.

      For the CALIBREVIEW format, you can select the iCellMap syntax on the PEX Options pane one the Netlist > Advanced sub-tab.

      Also see “Creating a SPECTRE Netlist from the Calibre View Netlist”.

    2. Choose the source for the net and instance names using the “Use Names From” dropdown list.

      • SCHEMATIC — Use names from the source schematic, but use the layout hierarchy. You must specify an LVS report name on the Reports tab and specify a schematic netlist on the Inputs pane on the Netlist tab. You can specify an optional “Device Info File” defining device parameters. (SOURCENAMES parameter)

      • LAYOUT — Use names from the layout. (LAYOUTNAMES parameter)

      • SOURCEBASED — Use the circuit pin order and cell hierarchy in the source netlist. (SOURCEBASED parameter)

      • SCHEMATICONLY — The parasitic netlist is based on LVS source hierarchy, with intentional device parameters extracted and backannotated from the layout. See the PEX Netlist command for details. You can specify an optional “Device Info File” defining device parameters. (SCHEMATICONLY parameter)

      The “SOURCEBASED” and “SCHEMATICONLY” options are most appropriate for Calibre View creation and other backannotation flows, since the parasitic netlist is based on the source schematic.

    3. Enter the PEX netlist filename.

      If Calibre Interactive is connected to a design tool, you can use a template to name the output parasitic netlist based on the extraction type, format, and format options. See “Naming the Extracted Parasitic Netlist Based on the Extraction Type and Format”.

  3. Click the Nets tab to select the nets to extract parasitics for and specify the parasitic model for specific nets. When the extraction mode is xACT 3D/xRC Hybrid, these controls specify the nets extracted using Calibre xRC model-based extraction.
    1. Select the setting for “Extract parasitics for”:

      • All nets — Parasitic extraction is done for all nets.

      • Specified nets — Specify nets to include or exclude from extraction. For more information, refer to “Including/Excluding Nets”.

    2. Select netlist model for specific nets — Select this setting and enter the net model filename to specify a parasitic model for specific nets. When this option is selected, the SVRF statement PEX Netlist Select File is added to the control file, and the ‑netmodel option is added to the Calibre xRC command line.

      The option “Select netlist model for specific nets” is only available if the extraction type specified is “Transistor Level” or “Gate Level” with R+C+CC, R+C+CC+L, or R+C+CC+L+M. Other restrictions may apply; see the description for the PEX Netlist Select File command.

  4. If the extraction mode is “xACT 3D/xRC Hybrid,” then choose Setup > PEX Options then the xACT3D tab to specify the subset of nets extracted using the xACT 3D fieldsolver.

    Choose a setting for “Extract parasitics using xACT 3D for”:

    • All nets — xACT 3D extraction is done for all nets.

    • Specified nets — Specify nets to include or exclude from xACT 3D extraction.

      For included nets, you can also specify the accuracy, as follows:

      200 — Less accurate, with faster runtime. This is the default.

      600 — More accurate, with slower runtime.

  5. (Optional) If you selected “Specified nets” in Step 3 and “Include”, you can specify to keep coupling capacitance separate from intrinsic capacitance with the following steps:
    1. Click the PEX Options button or choose Setup > PEX Options.
    2. Click the Netlist tab and the Reduction and CC sub-tab.
    3. Enable “Keep coupling capacitors separate.”

    The ‑cselect command line option is used instead of ‑select if “Keep coupling capacitors separate” is enabled. This option may only be used with extraction types of C+CC, R+C+CC, and R+C+CC+L.
