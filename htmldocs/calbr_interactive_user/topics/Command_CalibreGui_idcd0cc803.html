calibre -gui

Command line invocation for Calibre Interactive.

Note:

For Calibre Interactive xACT, see “Invoking the Calibre Interactive xACT GUI”.

Usage

Interactive operation:

calibre -gui [ {-drc | -dfm | -lvs | -pex | ‑perc | ‑3dstack}
   [‑nowait | ‑wait n]
   [runset | -runset runset]
   [-custom filename] ]

Batch operation:

calibre -gui {-drc | -dfm | -lvs | -pex | ‑perc | ‑3dstack}
   [-nowait | -wait n]
   {runset | -runset runset |‑runset_options_display runset}
   [-customgui filename | ‑custom filename]
   -batch

Calibre PERC Rule File Generator GUI:

calibre ‑gui -perc_rule_gen [-prgfile rulegen_setup.prg]

Arguments

-gui

A required switch to open Calibre Interactive. When specified with ‑drc, ‑dfm, ‑lvs, ‑perc, or ‑pex, the respective user interface window opens. When specified alone, the Calibre Interactive Palette opens, as shown in “Invoking Calibre Interactive from the Command Line”. You can open the various application-specific GUIs from the Calibre Interactive Palette.

-drc | -dfm| -lvs | -pex | -perc | ‑3dstack

An optional switch used with -gui to specify the Calibre Interactive application. The interfaces are Calibre nmDRC, DFM, Calibre nmLVS, PEX, Calibre PERC, and Calibre 3DSTACK, respectively.

‑nowait

An optional switch which specifies that Calibre Interactive does not wait for a license if one is not available immediately.

‑wait n

An optional switch and argument which specifies the amount of time in minutes that Calibre Interactive waits for a license.

runset | -runset runset

A switch and argument specifying the pathname of a runset. When specified, the runset is automatically loaded when the user interface opens. A runset is required for batch operation (‑batch); it is optional for interactive operation.

You can load multiple runsets by specifying a space separated list of runsets enclosed in quotes; the runsets are loaded in the order specified and the settings loaded last take precedence.

Also see “About Calibre Interactive Runsets” and “Using Multiple Calibre Interactive Runsets to Combine Different Sets of Options”.

‑runset_options_display runset

A switch and argument that specifies the pathname of a runset and also instructs the tool to display the runset options to the transcript. This option is only valid for batch operation (the ‑batch switch).

You can specify ‑runset_options_display more than once to load multiple runsets, and you can combine the ‑runset_options_display and ‑runset switches. Runsets are loaded in the order given in the command line; options in runsets occurring later in the command line override options in runsets occurring earlier on the command line. Only options in runsets specified by ‑runset_options_display are echoed to the transcript. If an option in a ‑runset_options_display runset is overridden by a later runset specified with the ‑runset option, then the option is not echoed to the transcript.

-custom filename

An optional switch and argument specifying the pathname of a customization file.

In interactive mode, the file is read when Calibre Interactive starts and the Customization button is added to the left panel of Calibre Interactive.

In Calibre Interactive batch mode, the initial settings from customization file are applied to the batch run; the Customization Settings window is not opened. If the runset includes customization override settings, those take precedence over the initial settings in the customization file. See “Customization Files” and “Customization GUI Operation in Batch Mode”.

This switch overrides the MGC_CALIBRE_CUSTOMIZATION_FILE environment variable. This switch overrides a customization file specified in the Setup Preferences dialog box on the Startup tab and saved to the runset.

-customgui filename

An optional switch and argument specifying the pathname of a customization file. This option is only available in batch mode. The Customization Settings window appears at the beginning of the Calibre run. Once the customization selections are made and the window is closed, the Calibre run continues in batch mode, without the user interface window. For more information, see “Customization Files” and “Customization GUI Operation in Batch Mode”.

This switch overrides the MGC_CALIBRE_CUSTOMIZATION_FILE environment variable, if set.

-batch

An optional switch used to run Calibre Interactive in batch mode without opening the user interface. A runset must be specified for batch operation.

Calibre Interactive starts the Calibre job using the runset and the optional customization file (-customgui or ‑custom switch). Layout and netlist databases need to be specified in the rule file that appears in the runset. See “Running Calibre Interactive in Batch Mode” for some examples.

In some cases Calibre Interactive must compile the rule file separately from the rule file compilation done for the Calibre run. Calibre Interactive in batch mode compiles the rule file in these cases:

  • The command line includes a check-related option.

  • You have specified a customization file.

Most runset parameters can be specified in -batch mode as a command line argument. For example:

calibre -gui -drc -runset rset -batch -drcLayoutPaths CHIP.gds ‑drcLayoutPrimary TOP

The ‑drcLayoutPaths and ‑drcLayoutPrimary arguments are runset parameters. See “Calibre Interactive Runset Contents” for an explanation of these parameters.

Note:

The name given for the runset parameter in the command line argument must match the actual runset parameter name exactly. You can examine a runset file for any arguments you want to use in batch mode.

‑gui -perc_rule_gen [-prgfile rulegen_setup.prg]

Argument set used to invoke the standalone Calibre PERC Rule Generator GUI.

  • ‑gui -perc_rule_gen — Required argument that specifies to open the Calibre PERC Rule Generator GUI.

  • -prgfile rulegen_setup.prg — An optional switch and argument specifying the path to a binary Calibre PERC Rule Generator setup file (.prg file). The setup file is used to populate settings in the GUI.

See “Running the Rule File Generator GUI” in the Calibre PERC User’s Manual for more information.

You can also invoke the Calibre PERC Rule Generator GUI from Calibre Interactive PERC; see “Creating a Calibre PERC Rule File with High Level Checks”.
