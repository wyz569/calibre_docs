Specifying the Host Processor and Run Options

Calibre Interactive allows you to run Calibre jobs on local, remote, and distributed resources. You can choose single threaded, multithreaded, or distributed execution with a choice of options such as number of CPUs and hyperscaling.

Note:

Although Calibre Interactive DRC, DFM, LVS, PERC, and PEX offer nearly identical run control options, the Graphical User Interface (GUI) for the Run Control pane varies slightly with each tool.

Prerequisites

  • For runs using remote hosts, configure your environment to allow rsh to execute your Calibre job remotely. This may include creating and editing a .rhosts file in your home directory.

Procedure

  1. Click the Run Control button on the left panel of the Calibre Interactive GUI.
  2. Click the Performance tab.
  3. Choose the host processor or cluster option in the “Run Calibre on” dropdown list from the options in the following table.
    Table 1. Run Calibre On Options in the Run Control Pane

    Selection

    Description

    Local Host

    Calibre runs on the computer currently running Calibre Interactive.

    Remote Host

    Calibre runs on a remote computer.

    1. Type the name of the remote host in the text entry field and press Enter.

      Basic information about the remote host is displayed in the Host Information field.

    2. Configure the remote environment as described in “Configuring the Remote Environment”.

    Platform LSF

    Use the IBM® Spectrum™ LSF® (Load Sharing Facility) distributed computing solution. See “Using the Spectrum LSF Option”.

    Univa Grid Engine

    Use the Univa® Grid Engine® batch-queuing system. See “Using the Univa Grid Engine Option”.

    Other Cluster

    Use a custom computer cluster. See “Using the Other Cluster Option”.

     

  4. Select execution options in the “Run Calibre” section.
    Table 2. Run Calibre Execution Options in the Run Control Pane

    Selection

    Description

    Processing mode (from dropdown list)

    Single-Threaded

    Use single-threaded execution. This is the default.

    Multi-Threaded

    Use multithreaded execution on a single host computer. Select the number of CPUs to use in the entry fields.

    Distributed (MTflex)

    Use your existing computer network as a distributed computing platform with Calibre® MTflex™. Follow the setup procedure in “Configuring for Distributed (MTflex) Execution”.

    Execution options

    The available execution options depend on the choice of processing mode.

    Number of CPUs to use for app

    Specifies the number of processors in multithreaded (MT) mode (the value for the ‑turbo command line option).

    for LITHO

    (Calibre nmDRC and DFM only)

    Specifies the number of processors to use for RET and MDP applications in multithreaded (MT) mode (the value for the ‑turbo_litho command line option).

    Synchronize Min CPUs in Host Select table

    (Multithreaded Platform LSF and Univa Grid Engine runs only)

    Forces the CPU specification in the host selection table to be the same as the number of CPUs specified for “Number of CPUs to use for app.” For DRC and DFM runs, the value is set to the greater of the “Number of CPUs to use for app” and “for LITHO” values.

    Halt <app> if licenses for number of CPUs cannot be obtained

    As described. Adds the ‑turbo_all command line option. Only available for “Multi-Threaded” runs.

    Number of hierarchical LVS licenses for PERC

    (Calibre PERC only)

    Specifies the “‑lvs_supplement num_processors” command line option, which uses Calibre nmLVS-H licenses for the circuit extraction portion of the run.

    This option is only available for multithreaded runs with “Halt PERC if licenses for number of CPUs cannot be obtained” unchecked.

    Hyperscale

    Use the -hyper command line switch. See “Hyperscaling Mode”. Recommended.

    The hyperscale option is available for “Multi-Threaded” and “Distributed (MTflex)” runs. The hyperscale option is not available for PEX runs.

    Hyperscale Compare

    (Calibre nmLVS only)

    Use the “‑hyper cmp” command line switch in Calibre nmLVS‑H. The “Hyperscale” option must also be enabled.

    See “Hyperscale Compare Option in Calibre Interactive nmLVS”.

    Hyperscale PathChk

    (Calibre nmLVS only)

    Use the “-hyper pathchk” command line switch in Calibre nmLVS-H. The “Hyperscale” option must also be enabled.

    See “Hyperscale Pathchk Option”.

    Remote Data Server (RDS)

     

    (Calibre nmDRC, Calibre nmLVS, Calibre DFM, and Calibre PERC runs)

    Use the ‑remotedata command line switch. See “Remote Data Server (RDS) Option” for details.

    “Remote Data Server” cannot be specified with FastXOR runs.

    Backup Data

    Use the -recoverremote command line switch to specify recovery from a remote host failure. This option is valid only when “Remote Data Server” is enabled. See “Backup Data Option”.

    Hyperscale Remote

    Use the “-hyper remote” command line switch and option to improve the performance and scaling of hyperscaling. This option is valid only when “Remote Data Server” is enabled. See “Hyperscale Remote Option”.

    In Calibre nmLVS and Calibre PERC, the “Hyperscale Remote” option cannot be specified with “Hyperscale Compare.”

    Connect to Remote Hosts

    (Calibre nmDRC only)

    Specify when to connect to remote hosts in HDB (hierarchical database) construction. Only available when the processing type is set to Distributed (MTFlex).

    • At later stages of HDB construction — Uses the “‑hdbflex” command line switch to specify that most of the HDB construction is to be performed by the local host, which does not connect to remotes until the later stages of HDB construction.

    • In parallel with HDB construction — Uses the “‑hdbflex_acquire” command line switch to specify to acquire resources in parallel with the HDB construction process.
