Calibre Auto-Waivers in Calibre Interactive

When verifying a design, some rule check violations may be waived by the foundry. A waived result is a violation that is considered acceptable for the design and process. Designers often want to eliminate waived errors from their Calibre results so they can focus on the errors that need to be fixed. You can perform many of the tasks related to Calibre Auto-Waivers using Calibre Interactive.

Calibre provides the followings two methods for managing waivers:

  • Calibre RVE Waived Results

    Results can be waived using the Calibre RVE interface; results can be waived individually, by check, or by cell. Waived results are stored in a drc_db.waived file.

    This flow is simple and easy to use, but hierarchy and cell name changes are difficult to handle. See “Waived Results in Calibre RVE for DRC” in the Calibre RVE User’s Manual for more information.

  • Calibre Auto-Waivers

    Calibre Auto-Waivers manages waivers by generating waiver shapes that are merged with your design so that waivers can be applied automatically during a verification run. These waivers take into account hierarchical polygon interactions, as well as interactions of multiple waivers that may apply to a single result.

    See the Calibre Auto-Waivers User’s and Reference Manual.

See “Calibre Auto-Waivers Versus the .waived File Method” in the Calibre Auto-Waivers User’s and Reference Manual for a more detailed comparison of these two methods.

You can run Calibre Auto-Waivers in Calibre Interactive. You should be familiar with Calibre Auto-Waivers and Calibre Interactive.
