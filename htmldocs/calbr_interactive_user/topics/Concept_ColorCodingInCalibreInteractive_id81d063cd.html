Color Coding in Calibre Interactive

The user interface uses color coding to indicate status.

Status is indicated as follows:

  • Color coding in data fields for file or directory names indicates whether the file exists or not.

    • Red text — Input file that does not exist. You must provide the information.

    • Green text — File exists. If a file of the same name is to be generated by Calibre Interactive, the tool will overwrite the existing file. Files loaded in a viewer connected to Calibre Interactive are not overwritten.

    • Black text — Output file that does not exist. The file will be created by Calibre Interactive.

  • Color coding in the button pane indicates readiness for running the tool.

    • Green Setup buttons — Indicate that you have provided enough information of the type associated with this button, and it appears to be the right type of information.

    • Red Setup buttons — Indicate that you have not provided enough information.

    • Black Action buttons — All action buttons are black, regardless of readiness for running Calibre.
