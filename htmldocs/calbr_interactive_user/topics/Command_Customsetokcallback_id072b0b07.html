CUSTOM::setOKCallback

Command type: Customization command for Calibre Interactive.

Specifies a callback procedure that is executed when you click the OK button in the Customization GUI. If the Customization GUI is not displayed in a Calibre Interactive batch run, the callback procedure is automatically executed.

Usage

CUSTOM::setOKCallback OKproc

CUSTOM::setOKCallback ‘[’ list OKproc arg1 [arg2 …] ‘]’

Arguments

OKproc

The name of the Tcl callback procedure.

arg1 [arg2 …]

One or more optional arguments to the callback procedure. If the callback procedure has arguments, they should be specified with OKproc as a Tcl list, as shown in the second syntax line.

Description

The CUSTOM::setOKCallback command specifies a callback procedure that is executed when you click the OK button in the Customization GUI. The callback procedure is automatically executed if the Customization GUI is not displayed in a Calibre Interactive batch run—this is the case for batch runs without the ‑customgui command-line option. The CUSTOM::setRunsetVarValue command can be used in the callback procedure to set Calibre Interactive GUI controls based on Customization GUI input. The callback procedure should not include commands that create customization controls (the commands in Table 2 and Table 4). See “Customization File to Read and Set Calibre Interactive GUI Controls” for a complete example.

If the callback procedure causes a Tcl error, a warning dialog appears containing the error message.

If the callback procedure OKproc returns -1, a warning dialog is displayed indicating there was a problem. The command CUSTOM::setOKCallbackError specifies a user-supplied error message for this warning dialog.

Examples

See “CUSTOM::setRunsetVarValue” for an extended example.

The following example specifies a callback procedure with no arguments:

proc myOKCallback {} {

# code ...
}

CUSTOM::setOKCallback myOKCallback

This example specifies a callback procedure with one argument:

proc myOKCallback {arg1} {
puts "starting myOKCallback with argument set to $arg1"
# code ...
}

CUSTOM::setOKCallback [list myOKCallback 5]

When you click the OK button in the Customization GUI, myOKCallback executes and prints the following line to stdout:

starting myOKCallback with argument set to 5
