Layers File for Device Signature Creation

Input for: The Signatures tab on the Inputs pane in Calibre Interactive nmLVS.

The Layers File is used to define the device types, device layers, pin names and pin layers for signature creation. The entries in the Layers File populate the fields of the Edit Parameters for Device dialog box in Calibre Interactive. You create the Layers File with a text editor.

Table 1 defines the entries in the Layers File for device signature creation. An example is provided in the “Examples” section.

Format

  • ASCII text file.

  • One entry per line.

  • Comment lines start with a “//”.

  • Layer entries may have an optional abbreviation entry. If present, the layer abbreviation is used in the Edit Parameters for Device Dialog Box.

  • For the CONTEXT_LAYER entry only, a list of layers enclosed by double quotes may be mapped to an abbreviation.

Parameters

Table 1. Layers File for Device Signature Creation Contents

Contents

Definition

*DEVICE_TYPES

<device_type>

…

List the device types you plan to create signatures for. The device types should be those permitted in the SVRF Device command.

*DEVICE_MODELS

<device_model>

…

List the device models you plan to create signatures for. The device models should be those permitted in the SVRF Device command.

*REFERENCE_LAYERS

<reference_layer> [<abbreviation>]

…

List the layers that serve as the reference_layer parameter for signature creation in the Device statement.

*CONTEXT_LAYERS

<context_layer> [<abbreviation>]

…

 

or:

“<layer> <layer> …” [<list_name>]

List the layers that serve as the context_layer parameter for signature creation in the Device statement.

 

A list of context layers enclosed in double quotes may also be mapped to a layer list name. If the context layer list is selected in the Edit Parameters for Device Dialog Box, the list contents are enclosed in braces ({ }) when shown in the device statement; the braces are not included in the device statement written to the control file.

*PIN_LAYERS

<pin_layer> [<abbreviation>]

…

List the layers that serve as the pin_layer parameter in the Device statement.

*PIN_NAMES

<pin_name> [<abbreviation>]

…

List the layers that serve as the pin_name parameter in the Device statement.

*SEED_LAYERS

<seed_layer> [<abbreviation>]

…

List the layers that serve as the seed layer (or device_layer parameter in the Device statement).

*TEXT_MODEL_LAYERS

<model_text_layer> [<abbreviation>]

…

List the layers that serve as the text_layer parameter for the TEXT MODEL LAYER keyword in the Device statement.

*SEED_LABEL_LAYERS

<seed_text_layer> [<abbreviation>]

…

If you are using seed shapes that are texted for the purpose of signature generation, list the layers that have text labels for the seed layer. See Table 1 for an explanation of how this parameter is used with the option “Use only seed layers with attached text label” in the Edit Parameters for Device Dialog Box.
