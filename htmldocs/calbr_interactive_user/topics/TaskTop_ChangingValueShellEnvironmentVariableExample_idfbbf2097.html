Changing the Value of a Shell Environment Variable Example

You can change the value used for a shell environment variable by assigning it a runset value and specifying that the runset value be used during the Calibre Interactive session.

If the shell environment variable is listed in the Set Env tab of the Set Environment Variables dialog box, follow the same procedure described for, “Editing a Runset or Rule File Environment Variable Example”.

This example assigns a runset value to the shell environment variable MGC_DRC_output. As outlined in, “Setup for Environment Variable Examples”, this variable is defined in the shell, and is used in the GUI field for the output DRC database filename. The variable MGC_DRC_output is not listed in the Set Env tab (see Figure 1 in “Set Environment Variables Dialog Box”), and this example shows how to enter it and assign a runset value.

Prerequisites

  • The shell environment, GUI, and rule file are set up as described in “Setup for Environment Variable Examples”.

How To Set Rule File Environment Variables in Calibre Interactive



Procedure

  1. Choose Setup > Set Environment to display the Set Environment Variables dialog box.
  2. Click the Set Env tab.
  3. Click Add at the top right.

    A new row appears in the table of environment variables.

  4. Click in the Name column of the new row and enter the name of the shell environment variable MGC_DRC_output.
  5. Press Enter on the keyboard after entering the variable name.

    The value defined for the variable in the shell is listed in the Env. Value column.

  6. Click in the Runset Value column and enter the new value.
  7. Verify that the Runset column has a green check mark (), indicating that the runset value is used in the Calibre Interactive session.

    New variables are initialized with a green check mark in the Runset column. However, if this is not the case, click the X in the Runset column to change it to a green check mark.

  8. Click Apply to save the changes.
    Tip

    To return to using the shell environment value for the variable, click the X in the Env. column to change it to a green check mark.
