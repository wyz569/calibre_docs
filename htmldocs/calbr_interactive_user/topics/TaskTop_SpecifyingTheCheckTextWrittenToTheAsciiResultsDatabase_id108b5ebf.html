Specifying the Rule Check Text Written to the Results Database

Rule check text consists of rule file comments and rule file information, such as the rule file title and pathname. You can use Calibre Interactive to specify the amount of check text written to the ASCII results database.

If you specify DRC rule check comments for RVE in the rule file, you must output check text to the results database. See “DRC Rule Check Comments for Calibre RVE” in the Calibre RVE User’s Manual.

Rule file statement: DRC Check Text {{[COMMENTS | ALL] [RFI]} | NONE}

Prerequisites

  • You are running Calibre Interactive nmDRC or Calibre Interactive DFM.

Procedure

  1. Click Outputs on the left pane of Calibre Interactive.
  2. Select “Specify the amount of rule check text.”
  3. Specify the amount of check text. The corresponding keywords for the DRC Check Text statement are given in parentheses.
    Comments with RFI

    Output rule check comments (those preceded with an @ symbol) and rule file information. (COMMENTS RFI).

    Comments only

    Output rule check comments (those preceded with an @ symbol). (COMMENTS)

    All Check Text

    Output the entire text of the each rule check. (ALL)

    All Check Text with RFI

    Output the entire text of the each rule check and rule file information. (ALL RFI)

    No Check Text

    Do not output any rule check text or rule file information. (NONE)

  4. (Optional) Select “Check Text File” and specify a file path. When specified, this database contains the rule checks selected for the run and the check text specified by the keyword options. Results are not included in this file.
