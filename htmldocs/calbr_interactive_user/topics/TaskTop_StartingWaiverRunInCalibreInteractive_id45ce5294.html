Starting a Waiver Run in Calibre Interactive

You can use Calibre Interactive to start a Calibre verification run with Calibre Auto-Waivers. The waiver setup parameters are specified on the Waivers tab of the Inputs pane. The waiver run adds the “‑waiver” option to the Calibre command line.

All settings on the Waivers tab are saved to the runset (see “About Calibre Interactive Runsets”). If you have a runset loaded with desired settings, you can skip to the end of the following procedure.

Prerequisites

  • “Requirements for Calibre Auto-Waivers”

  • You are running Calibre Interactive nmDRC, Calibre Interactive nmLVS with ERC, or Calibre Interactive DFM with Critical Feature Analysis (CFA).

  • If you are using waivers from multiple sources, then all waiver shapes must be on the same layer/datatype. This may not be the case if waivers are provided by different IP vendors and a foundry, for example. For additional requirements and instructions for handling this situation, see “Performing a DRC Waiver Run Using Multiple Waiver Criteria Files” in the Calibre Auto-Waivers User’s and Reference Manual.

Procedure

  1. Click Rules on the left panel of Calibre Interactive and specify the rule file and run directory.
  2. Click Inputs on the left panel of Calibre Interactive. Specify information for the layout database on the Layout tab. The layout format must be GDSII or OASIS for Calibre Auto-Waivers.
  3. Click the Waivers tab on the Inputs pane.
  4. Enable “Use Waivers when running DRC/LVS/CFA.”
  5. (Optional) If you have a waiver setup file already created and want to use it for the waiver run, click the Import Waiver Setup File button. This populates the GUI fields with the values in the selected waiver setup file; only GUI settings specified in the imported file are overwritten.

    Use the following steps to confirm that the imported setup parameters are correct. The waiver setup parameter that corresponds to the GUI control is given in parentheses.

  6. Choose the Description Files category; the Description Files pane is shown for Calibre Interactive nmDRC.


    1. Specify the “Waiver Criteria File(s).” It is not necessary to specify the Waiver Criteria file if you specify “Use criteria from the waiver shape files” in the DRC Run category in Step 9.

      Click the button to edit the file in a text editor. See “Waiver Criteria File Format” in the Calibre Auto-Waivers User’s and Reference Manual for complete information. (WAIVER_CRITERIA)

      Note:

      The corresponding parameter in the waiver setup file is given in parentheses.

    2. (Optional) Specify the directory for temporary files in the Temp Files Directory field. The default is the working directory. (TMP_DIR)
  7. Choose the category Waiver Shapes and specify the waiver database as follows, depending on the method of waiver shape creation:

    If …

    Do this …

    All of the waiver shapes are embedded in the main design specified on the Layout tab of the Inputs pane.

    Set Waiver Database to Embedded.

    (MERGE YES)

    At least some waiver shapes exist in separate waiver shape files, and the waivers embedded in the design should be taken into consideration.

    1. Set Waiver Database to Both.

      (MERGE NO)

    2.  Specify the waiver shape file(s).

      (WAIVER_DATABASE)

    The specified waiver shape files are combined at runtime with the design file on the Layout tab of the Inputs pane.

    At least some waiver shapes exist in separate waiver shape files, and the waivers embedded in the design should not be taken into consideration.

    • Set Waiver Database to External.

      (MERGE IGNORE_DESIGN_WAIVERS)

    • Specify the waiver shape file(s).

      (WAIVER_DATABASE)

    Only the external waivers are taken into consideration when applying waivers.
  8. (Optional) Select the setting for “Use rule file precision” (in the Waiver Shapes category):

    • Enabled — (default) Specifies PRECISION_CONVERSION YES in the waiver setup file.

    • Disabled — Specifies PRECISION_CONVERSION NO in the waiver setup file.

    This setting controls how waiver shape precision and magnification is handled if the rule file precision differs from the layout database precision; it has no effect if the precisions are the same. See “Waiver Cell Precision and Magnification Considerations” in the Calibre Auto-Waivers User’s and Reference Manual for details. (PRECISION_CONVERSION).

  9. Choose the category DRC Run and specify run options as desired:

    • Use criteria from the waiver criteria files — As described. Use this setting when all waiver criteria is contained in the specified file(s). (WAIVER_CRITERIA file)

    • Use criteria from the waiver shape files — Extract waiver criteria from the waiver shape file. The specified waiver criteria file is overwritten or created if it does not exist. (WAIVER_CRITERIA EXTRACT file)

      Note:

      If both “Use criteria from the waiver criteria files” and “Use criteria from the waiver shape files” are enabled, then APPEND is used in the WAIVER_CRITERIA specification.

    • Verify waiver cell checksums — Verifies that the contents of the waiver cells match the checksum. (RUN_LAYOUT_CHECKSUM)

    • Verify Calibre version — Verifies the Calibre version in waiver cells. (RUN_CALIBRE_VERSION_CHECKSUM)

    • Retain edge and error result types — Retain unwaived error and edge results from a waiver run instead of converting them to polygons. (RETAIN_RESULT_TYPE)

    • Verify IP_MATCH checksums — As described. (RUN_IP_MATCH)

    • Verify rule checksums — Use rule checksums to verify that the rule that produced the waiver shape has not changed. (RUN_RULE_CHECKSUM)

    • Ignore waiving density operation — Do not apply waivers to rules with density checks. (IGNORE_DENSITY)

    • Enable dynamic results reporting — Enables viewing of waived results in Calibre RVE as soon as they are available. (ENABLE_DYNAMIC_RESULTS_REPORTING)

  10. Choose the Waiver RDB Files category and specify settings as desired to control the output of RDB files. (WAIVER_RDB, UNUSED_WAIVER_RDB, USED_WAIVER_RDB, DENSITY_WAIVER_RDB, GENERATE_COMBINED_RESULTS)

    Select “Enable waiver cell reporting” to add the cell names as an extra property in generated waiver RDB files. This option is disabled by default and corresponds to ENABLE_WAIVER_CELL_REPORTING.

    The option “Ignore waiver statistics in summary file” corresponds to OPTIMIZE_NONE_WAIVER_STATISTICS. The option is available if any of the waived, unused, or waived RDB options are set to NONE.

    Set “Generate combined results” to YES to generate a combined waivers results database with both the used and unused waiver shapes. The entries in the combined database include additional properties to indicate whether the waiver was applied or unused.

  11. Click the Layers and Text category and specify settings as desired.

    These settings control the layer and datatype for the waiver shapes and text annotations. During a waiver run these settings specify the layer and datatype pairs that contain the waiver shapes and text annotations.

    See “Waiver Setup File Format for Waiver Verification” in the Calibre Auto-Waivers User’s and Reference Manual for a description of the options and their defaults; also see “Text Object Annotations” in the same manual.

  12. (Optional) Click the View Waiver Setup File button to view the waiver setup parameters. You can choose File > Save As to save the file.
  13. Make other settings as desired.
    Note:

    In Calibre Interactive nmDRC, the preference setting “Start RVE as soon as results are available” has no effect if Calibre Auto-Waivers is enabled.

  14. (Optional) Choose File > Save Runset to save the runset.
  15. Click Run DRC when you are ready to start your Calibre nmDRC run.
    Note:

    When waiver shapes are saved in a separate file (“Waivers have been merged with existing design” OFF) rather than embedded in the design, it is possible for the precision of the waiver database to be different than the precision of the layout database. Calibre adjusts the magnification automatically if this happens, as described in the section “Waiver Cell Precision and Magnification Considerations” in the Calibre Auto-Waivers User's Manual.

Results

In addition to the output files specified on the Outputs pane, the following files are created or modified during a waiver run:

  • Waiver setup file — A waiver setup file is generated by Calibre Interactive based on the GUI parameters and named _waiver_setup_ by default. To specify a different file name, choose Setup > Preferences, click the Misc tab, enable the option “Specify waiver setup file name,” and specify the file name.

    The parameter WORKING_DIRECTORY is set to the run directory specified on the Rules pane of Calibre Interactive.

  • Waiver criteria file — If “Use criteria from the waiver shape files” is specified on the DRC run category pane, then the waiver criteria file is created or modified based on the waiver criteria included in text objects in the waiver shape file(s).

  • calibre-waiver.summary — Contains runtime messages and results statistics.

The presence of the following RDB files depends on the settings on the Waivers tab in the Waiver RDB Files category:

  • waived.rdb

  • unused_waiver.rdb

  • used_waiver.rdb

  • density_waived.rdb

See these sections in the Calibre Auto-Waivers User's Manual for a complete description of the output from a waiver verification run:
