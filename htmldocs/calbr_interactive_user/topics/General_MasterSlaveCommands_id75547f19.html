Primary-Dependent Commands

You can specify that the visibility of one control (the dependent control) depends on the state of one or more other controls (the primary) in the customization file.

The following set of options are allowed for the customization commands that create GUI controls or elements:

{ {{-master master_spec } [‑master_select {0 | 1}]} | ‑master_expr expr_spec}

  • -master master_spec — An optional argument that specifies this control as a dependent to another primary control in the customization file. In other words, the appearance of this GUI object in the customization dialog box depends on the value of another control.

    The argument master_spec specifies the GUI control object that is the primary control and optional values to match. Suppose $_master references the primary control and value1 and value2 are two possible values for the control. The following syntax options are available, where the brackets and Tcl list command are literal:

    ‑master $_master — Show the dependent control if $_master is selected.

    ‑master [list $_master value1] — Show the dependent control if $_master is equal to value1.

    ‑master [list $_master [list value1 value2 …] — Show the dependent control if $_master is equal to any value in the list value1, value2 ….

    The optional matched value for the control is either from the ‑choices list or from a text entry field associated with the $_master control. Double quotes in the value must be escaped. See “Getting the Tcl control variable” in the “Description” section for an example of how to reference the GUI control object.

  • ‑master_select {0 | 1} — An optional argument specifying when the current (dependent) control is displayed. This option is only used with ‑master.

    0 — Display the dependent control when the primary control checkbox is not selected.

    1 — (default) Display the dependent control when the primary control checkbox is selected.

  • ‑master_expr expr_spec — An optional argument that specifies this control as dependent to one or more primary controls in the customization file, where expr_spec is a Tcl expression that determines if the dependent control is displayed.

    Suppose $_master1 and $_master2 reference primary controls and value1 and value2 are two possible values for $_master1. The following syntax options demonstrate the types of expressions that are available, where the brackets and Tcl list command are literal:

    • ‑master_expr [list $_master1 || $_master2] — Show the dependent control if either $_master1 or $_master2 is selected.

    • ‑master_expr [list $_master1&& $_master2] — Show the dependent control if both $_master1 and $_master2 are selected.

    • ‑master_expr [list [list $_master1 [list value1 value2]] || $_master2] — Show the dependent control if $_master1 is equal to value1 or value2 or if $_master2 is selected.

    • ‑master_expr [list [list $_master1 value1] && $_master2] — Show the dependent control if $_master1 is equal to value1 and $_master2 is selected.

    • ‑master_expr [list ! $_master1] — Show the dependent control if $_master1 is not selected or not visible.

    The above list of expressions is not exhaustive; see “Expression Syntax for ‑master_expr” in the “Description” section for a formal definition of the expression syntax.

Description

The display of the control with the ‑master or -master_expr option is determined by the value of one of more other primary controls in the customization GUI. By default, the dependent control is shown if the checkbox for the primary control is enabled; this can be changed by specifying “‑master_select 0” with the ‑master option to indicate that the dependent control is shown if the primary control checkbox is disabled. Use the ‑master_expr to specify a Tcl expression with multiple primary controls.

A dependent control can be a primary control for another control.

Getting the Tcl control variable

Each CUSTOM command returns a Tcl/Tk widget. You can assign this widget to a Tcl variable using the Tcl set command, as shown in this example: 

set _master [CUSTOM::DEFINE -name "change_defines" \
	-prompt "Change pre-processor defines"]

Here the CUSTOM command is enclosed in brackets to indicate command substitution. The Tcl variable is referenced using the dollar sign ($), as in $_master.

Expression Syntax for ‑master_expr

The Tcl expression expr_spec in the option “‑master_expr expr_spec” is defined as follows:

expr_spec = subexpr [op subexpr]....
subexpr = "(" subexpr ")" | control_expr [op {subexpr | control_expr}].... 
control_expr = ["!"] control [value_list]
control = <value of Tcl var associated with customization dialog control> 
value_list = <list of acceptable values for the given control>
op = "&&" | "||"

where control in the above expression is found as shown in the section “Getting the Tcl control variable”.

Examples

Example 1

In this example, the control for the second CUSTOM::DEFINE command is only displayed if the checkbox for the control associated with the _masterDefines variable is enabled.

# Defines a primary control for the CUSTOM::DEFINE controls
set _masterDefines [CUSTOM::DEFINE -name "change_defines"  -select 1 \
     -prompt "Change pre-processor defines" -display 1 -tool DRC ]

CUSTOM::DEFINE -name "DRC_DENSITY" -boolean 1 -select 1  \
     -prompt "Define DRC_DENSITY" -tool DRC -master $_masterDefines

Example 2

This example demonstrates several of the options for the ‑master option. Commands for three separate dependent controls are included to demonstrate matching a single value, matching a value in a list, and for displaying the dependent control if the primary control is not selected. 

# Add a DEFINE for process type, it is a master for other controls
# process can have three possible values: wmp, 71m, or other
set _master_process [CUSTOM::DEFINE -name process \
     -choices {{"wide metal" wmp} {"process 7" 71m} {"other" other}} \
     -initval wmp -prompt "Add #DEFINE for process type" ]
# dependent control is displayed if process is set to wmp 
# for _master_process control
CUSTOM::DEFINE -name "b_var" -boolean 1 -select 1  \
     -prompt "b_var, show if process wide metal (wmp)"  \
     -master [list $_master_process wmp]
# dependent control is displayed if process is set to wmp or 71m 
# for _master_process control
CUSTOM::DEFINE -name "c_var" -boolean 1 -select 1  \
     -prompt "c_var, show if process wide metal or 7 (wmp or 71m)" \
     -master [list $_master_process [list wmp 71m]]

# dependent control (a label) is displayed if _master_process control 
# is not enabled
CUSTOM::LABEL -prompt "process var not defined" \
 	-master $_master_process -master_select 0

Example 3

This example demonstrates several of the options for the ‑master_expr option. Commands for three separate dependent controls are included to demonstrate matching a single value, matching a value in a list, and for displaying the dependent control if the primary control is not selected. 

# Add a DEFINE for process type as a primary control for other controls
# process can have three possible values: wmp, 71m, or other
set _master_process [CUSTOM::DEFINE -name process \
     -choices {{"wide metal" wmp} {"process 7" 71m} {"other" other}} \
     -initval wmp -prompt "Add #DEFINE for process type" ]
# Add a DEFINE for extra_checks, it is a primary control 
# for other controls
set _master_checks [CUSTOM::DEFINE -name extra_checks \
     -initval wmp -prompt "run extra checks" ]

# dependent control is shown if either _master_process or _master checks
# control is enabled
CUSTOM::DEFINE -name "a_def" -boolean 1 -select 1  \
     -prompt "a_def, show if either process or extra checks defined"  \
     -master_expr [list $_master_process || $_master_checks]
# dependent control is shown if both _master_process and _master checks
# controls are enabled
CUSTOM::DEFINE -name "ab_def" -boolean 1 -select 1  \
     -prompt "ab_def, show if both process and extra checks defined"  \
     -master_expr [list $_master_process && $_master_checks]
# dependent control is shown if process 
# is wmp or 71m (from _master_process)
# or if _master_checks control is enabled
CUSTOM::DEFINE -name "b_def" -boolean 1 -select 1  \
     -prompt "b_def, show if process is wmp or 71m, \
			or if extra checks is defined" \
     -master_expr [list [list $_master_process [list wmp 71m]] || \
			$_master_checks]     
# dependent control is shown if process 
# is wmp (from _master_process control)
# and if _master_checks control is enabled
CUSTOM::DEFINE -name "c_def" -boolean 1 -select 1  \
	-prompt "c_def, show if process is wmp and extra checks is defined" \
	-master_expr [list [list $_master_process wmp] && $_master_checks]     

# dependent control (a label) is shown if _master_checks control \
# is not enabled
CUSTOM::LABEL -prompt "extra checks not defined" \
	-master_expr [ list !$_master_checks]
