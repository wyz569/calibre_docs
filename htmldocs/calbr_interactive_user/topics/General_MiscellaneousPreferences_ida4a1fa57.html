Miscellaneous Preferences

The preferences on the Misc tab of the Setup Preferences dialog box in Calibre Interactive control warnings and prompts, automatic behaviors, and other options.

Choose Setup > Preferences and click the Misc tab to make changes to the miscellaneous preferences. The preferences are saved with the runset.

Table 1. Miscellaneous Preferences

Option

Remarks

Automatically load rules file

(Default: false)

 

Show prompt

(Default: true)

When “Automatically load rules file” is enabled, Calibre Interactive automatically loads the rule file at these times:

  • When you load a runset with a rule file definition.

  • When you start a Calibre run and the rule file has been modified since it was last loaded.

Show prompt — Enable or disable displaying a confirmation dialog before automatically loading the rule file. If “Show prompt” is disabled, the setting for “Automatically load rules file” applies during Calibre Interactive batch operation.

See the section “Automatically Loading a Rule File” for more details.

Also see the section “How Rule Files Relate to Control Files”.

Read Inputs/Outputs fields while loading rules file

(Default: true)

If enabled, input and output statements from the rule file populate the Inputs and Outputs pane when Load is clicked.

Run Calibre when runset opened

(Default: false)

Start a run immediately after a runset is opened.

Warn before overwriting layout database

(Default: true)

Recommended. Display a warning before overwriting the system- generated layout database. This occurs when the layout GDSII or OASIS®1 database is extracted from the layout viewer.

Specify control file name

(Default: false)

Specifies a control filename. This value is stored in the runset.

Write control file suitable for command line run

 

(Default: true)

If enabled, the #IFDEF $MGC_CALIBRE_INTERACTIVE line is omitted before the DRC ICSTATION YES line in the control file, along with the corresponding #ENDIF. A control file saved with this option enabled can be used in a command line Calibre run.

See “Running Batch Calibre with a Calibre Interactive Control File”.

If the layout database format is LEF/DEF or OpenAccess, you also need to set database translation options with an environment variable; this is described in the referenced task.

Warn before overwriting source database

(LVS/PEX only)

 

(Default: true)

Display a warning before overwriting the source SPICE netlist. This occurs when the source SPICE netlist is extracted from the schematic viewer.

Prompt to save changes before closing runset

 

(Default: true)

When enabled, Calibre Interactive displays a prompt dialog when a modified runset is about to be closed. If this setting is not enabled, the prompt dialog is not displayed.

Save changed settings before closing runset

 

(Default: false)

Controls whether or not the runset is saved. This setting is only available if “Prompt to save changes before closing runset” is disabled.

Save runset each time Calibre is run

 

(Default: false)

When enabled, the runset is saved automatically each time Calibre is run.

Save runset to current directory each time Calibre is run

 

(Default: false)

When enabled, each time Calibre is run a copy of the runset is saved to the directory from which Calibre Interactive was invoked.

If you loaded multiple runsets and this setting is enabled, the runset is saved as runset.autosaved.

Close RVE when Calibre Interactive closes

 

(Default: true)

This option relates only to those Calibre RVE sessions invoked from Calibre Interactive.

Close RVE before running …

 

(Default: true)

Close the Calibre RVE session before starting a new Calibre run.

Display text of included rule files in transcript

 

(Default: false)

Displays contents of included rule files in the transcript when Calibre is run.

Note: This setting takes precedence over the environment variable CALIBRE_ECHO_RULE_FILE.

Start RVE as soon as results are available

(DRC and DFM only)

 

(Default: true)

Opens the ASCII results database when the first DRC check is completed instead of waiting until DRC checking completes. This is referred to as “dynamic RVE”, since results are updated as they are made available. The option “Show results in RVE” on the Outputs pane must also be enabled.

This option has no effect in a Calibre Interactive nmDRC run with Calibre Auto-Waivers enabled.

Do not open empty main results database in RVE

(DRC and DFM only)

 

(Default: false)

When enabled, an empty main results database (RDB) is not opened if at least one side (auxiliary) RDB will open automatically. If no side RDBs are specified to open automatically, or if all side RDBs are empty, then an empty main RDB is opened.

Side RDBs can be specified to open automatically with the Calibre Interactive setting “Side RDBs to open” on the Outputs pane or with the Calibre RVE setting “Auto Open” in the “Side RDBs” table. See “Viewing Auxiliary Databases” in the Calibre RVE User’s Manual.

This option is automatically disabled if “Start RVE as soon as results are available” is checked.

Verify checks are selected before running DRC

(DRC only)

 

(Default: true)

When enabled, Calibre Interactive verifies that at least one rule check is selected before starting the run.

Stop PEX if LVS indicates mismatch

(PEX only)

 

(Default: false)

Stops the parasitic extraction run if the LVS step detects a mismatch between the layout and source designs.

Enable editing for text file viewer

 

(Default: false)

Opens text files in edit mode by default. If this setting is not enabled, you must click the red edit box to unlock the text file for editing.

Save TVF Rules to SVRF before running Calibre

 

(Default: false)

Enables saving of TVF rules to an SVRF file.

See “Running Calibre Interactive PEX” for a note on how this option works for PEX runs. Also see “Calibre Interactive Operation with a TVF Rule File”.

Only compile the rules file when Run button is pressed

 

(Default: false)

Prevents Calibre invocation when you click the Run button. Instead, only the rule file compiler is invoked. Any compilation errors are displayed in an error dialog box. If compilation succeeds, a message to that effect is printed in the Transcript. This setting is saved to the runset.

Ignore layout and source export in batch mode

 

(Default: true)

This setting is enabled by default, causing batch runs of Calibre Interactive to ignore the settings for “Export from layout viewer” and “Export from schematic viewer” on the Inputs pane.

Disable this setting if you want to run Calibre Interactive in batch mode and read the layout or source data from a connected viewer. See “Specifying Layout and Schematic Export for a Calibre Interactive Batch Run”.

Check source netlist is complete before running LVS

Check source netlist is complete before running PEX

Check source netlist is complete before running PERC

 

(Default: true)

Verify that the source netlist is complete before starting the run; the source netlist check is case insensitive. If referenced cells are not defined or any .include files are missing, a warning is issued with the choice to continue or cancel the run. Up to eleven levels of subcircuits in the netlist are checked.

The GUI only displays the names of five subcircuits that are used but not defined. Checking stops if eleven subcircuits are used but not defined.

If an internal pre-execution trigger is defined, the check for a complete source netlist is done after the trigger executes. See “Internal Triggers in Calibre Interactive”.

Specify waiver setup file name

(DRC, LVS, and DFM only)

 

(Default: false)

When enabled, the waiver setup file generated by Calibre Interactive during waiver creation or during a waiver run is saved to the specified filename. By default, the generated waiver setup file is named _waiver_setup.

Start RVE after batch run

(Default: false)

When enabled, Calibre RVE is automatically opened after a Calibre Interactive batch run.

Warn when library layout file format differs from primary layout format

(Default: false)

When enabled, Calibre Interactive displays a warning at runtime if additional layout files on the Options > Database > Library tab are not the same format as the layout file on the Inputs pane.
