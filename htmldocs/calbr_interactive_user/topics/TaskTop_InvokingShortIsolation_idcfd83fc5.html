Invoking Short Isolation

Calibre has a short isolation feature to help pinpoint the location of layout shorts between texted nets. You can specify this option in Calibre Interactive.

Procedure

  1. Click LVS Options to display the LVS Options pane. If the LVS Options button is not available, choose Setup > LVS Options.
  2. Click the Shorts tab.
  3. Enable the “Run short isolation” option and specify a type, if desired.

    • Yes — Perform short isolation, subject to specified cell and name filters. Adds “LVS ISOLATE SHORTS YES”.

    • Power Shorts Only — Isolate shorts between power net labels. Adds “&& NAME LVS_POWER_NAME” and disables the name option.

    • Ground Shorts Only — Isolate shorts between ground net labels. Adds “&& NAME LVS_GROUND_NAME” and disables the name option.

    • Power Ground Shorts — Isolate shorts between power net labels, shorts between ground net labels, and shorts between power-ground nets. Adds “&& NAME LVS_POWER_NAME LVS_GROUND_NAME” and disables the name option.

    • IO Shorts — Isolate shorts between signal nets and shorts between signal and power-ground nets. Adds “&& !NAME LVS_POWER_NAME LVS_GROUND_NAME” and disables the name option.

    Note: Enabling short isolation automatically adds the option SI to the Mask SVDB Directory statement.

  4. Select general options:

    Output shorts by layer — Group results by layer. If not selected, polygons from all layers involved in the short are grouped together into one report cluster. This option adds the BY LAYER keyword.

    Run flat short isolation — Select this to choose flat short isolation. Hierarchical isolation is the default.

    Output shorts by cell — Report the cell of origin for every polygon written to the shorts isolation database. This option adds the BY CELL ALSO keyword to the LVS ISOLATE SHORTS command.

    Exclude contact polygons — Remove contact polygons from the output. This option adds the NO CONTACTS keyword.

  5. Choose desired Hierarchical Options:

    These options are not available if the “Run flat short isolation” option is selected.

    1. Specify the depth of short isolation.

      • Isolate shorts in top cell only — Equivalent to the CELL PRIMARY keyword set in LVS Isolate Shorts.

      • Isolate shorts in all cells — Equivalent to the CELL ALL keyword set in LVS Isolate Shorts.

    2. Specify how to combine the depth and name sub-options:

      • AND — A short must satisfy both conditions. For example, Calibre reports the short only if it occurs in the top-level cell and is between nets named VDD and VSS.

      • OR — Any short satisfying either condition is reported. For example, report the short if it is between any top-level nets or if the short is between VSS and VDD at any level in the hierarchy.

      • NOT — Select this setting to exclude the names specified in step c.

    3. Choose the desired name option. This option specifies how the tool processes text names when identifying shorts. Adds the NAME keyword.

      • Between all names — Isolate shorts between all text object names.

      • Between names — Isolate shorts between the net names specified in the text field.

    4. Enable “Identify common paths,” if desired. This option instructs the tool to identify a common path, if it exists, shared by multiple paths between shorted text objects. This option adds the ACCUMULATE keyword.
      Tip

      Invocation of short isolation creates a short isolation database which can be viewed using Calibre RVE for LVS. Calibre RVE for LVS includes tools for isolating the short, such as highlighting by cell, layer, or net. See “Interactive Short Isolation” and “Verifying Short Repairs” in the Calibre RVE User’s Manual.

  6. Enable “Maximum Short Results,” if desired. If set to “Count,” this option limits the maximum number of isolated short paths that are identified and reported to the provided value. When not enabled or set to “All,” all short paths are reported.
