Specifying a Rule File for a Calibre nmLVS Run

You need to specify a rule file for any Calibre nmLVS run. You specify the rule file on the Rules tab of Calibre Interactive. You can click the Load button to load the rule file settings into the GUI fields.

If you want to use device signatures in the device identification process, you can use the Create Device Signature dialog to create the device signatures and add the appropriate statements to your rule file. See “Device Signature Creation with Calibre Interactive nmLVS”.

Procedure

  1. Click the Rules button to display the Rules pane.


  2. Specify the rule filename in the LVS Rules File field. You can use the View button to view and edit the rule file.

    Be aware that Calibre Interactive automatically recognizes a rule file with the literal name “rules” and displays the name in green. Because the name is shown in green, you may assume the file has been loaded but it has not been; the file must be explicitly loaded using the Load button.

  3. (Optional) Click Load to populate the GUI with the settings in the rule file.

    A rule file can also be specified in a runset. After loading that runset, the rule file is included in the control file but its settings will not be loaded into the GUI until you click Load.

    Tip

    The setting “Read Inputs/Outputs fields while loading rules file” (Setup > Preferences, Misc tab) controls whether or not rule file settings populate the GUI options on the Inputs and Outputs panes when Load is clicked. This setting is enabled by default. See “Miscellaneous Preferences”.

    After you load a rule file, any information you specify in the GUI supersedes the information in your loaded rule file.

  4. Specify the run directory name in the LVS Run Directory field.
  5. (Optional) View the layer derivation tree at the bottom of the Rules pane; see “Layer Derivations Tree” for information.
