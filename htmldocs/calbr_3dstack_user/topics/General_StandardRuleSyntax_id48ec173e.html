Standard Rule Syntax

The standard syntax supports the same set of rules as the extended syntax, but the options and usage of the rules is different.

Versions of the documentation prior to 2018.1 include more detailed descriptions of the options for the standard syntax. The following table includes the usage and examples for each command.

Table 1. Standard Syntax Rule Application

Command

Standard Syntax Rule Usage and Examples

centers

 
centers -check_name check_name 
 -placement1 placed_layer1 [-same_die]
[-placement2 placed_layer2] -constraint "constraint"
 [-alignment {octagonal_only | orthogonal_only}]
 [-overlapping][-square] [-comment "comment"] [rve_option …]

Check that pads are evenly spaced by 80 um.

centers -check_name controller_pads \
 -placement1 ${cont}_CON_if -constraint "== 80" \
 -alignment orthogonal_only -comment "Controller pads \
  spaced 80 um in x or y from all pad centers"

connected

 
connected -check_name check_name 
-placement1 placed_c_layer1 [-placement2 placed_c_layer2] 
 [‑net_mismatch {ALL | MULTI_NAME | MISMATCH | MISSING_NAME}]
 [‑no_dangling_ports] [‑no_extra_ports] [-no_missing_ports]
 [-pin_list list_of_pins]
 [‑text_checks ‘{’ALL | NONE | [type …]‘}’] 
 [‑comment "comment"] [rve_option …]

copy

 
copy -check_name check_name -placement placement_name 
 [-comment "comment"] 
copy -check_name enc_check -placement placement1 \
 -comment "EXPORT OPERATION\n Writing placement1 to DFM \  database format."

dangling_ports

 
dangling_ports ‑check_name check_name 
 ‑placements placement_name_list 
 [-comment "comment"] [rve_option …]

Verify that all of the ports on the ram placement are connected to physical geometry:

dangling_ports ‑check_name ram2_pad_text_rule \
	‑placements p_ram_2 \
	-comment "ERROR: Unconnected ports detected!"

dangling_no_text

 
dangling_no_text ‑check_name check_name 
 {-chips chips_list | {‑placement1 name -placement2 name}
 [-comment "comment"] [rve_option …]

Flag orphan ports that are missing text labels:

dangling_no_text ‑check_name orphans \
	‑chips {die1 die2} \
	-comment "ERROR: Unconnected ports detected!"

density

 
density -check_name check_name 
 -placements ‘{’placed_layer …‘}’
 -constraint "constraint_expression" 
 [-expression "density_expression"] 
 [-window {wxy | wx wy} [-step {sxy | sx sy}] ]
 [-window_type {truncate | backup | ignore | wrap}]
 [-inside { extent | placed_layer}]
 [-centers value] [-comment "comment"] [rve_option ...]
 
density -check_name density_tracer_and_interposer \
 -placements { i_tsv m_mtsv_trace } -constraint " < 0.1" \
 -window 50 -window_type "wrap" -centers 25 -step {10 20} \
 -comment "Density check for m-tracer and interposer"

density -check_name density_tracer_and_interposer \
 -placements { i_tsv m_mtsv_trace } -constraint " < 0.1" \
 -expression "(AREA(i_tsv) + AREA(m_mtsv_trace)) / AREA()" \
 -window 20 -window_type "truncate" -centers 10 \
 -comment "density check i_tsv m_mtsv_trace < 0.1"

enclosure

 
enclosure -check_name check_name 
 -placement1 placement1 -placement2 placement2
 -constraint "constraint_value" 
 [-comment "comment"] [rve_option …]

Check for Through Silicon Vias that are not enclosed within metal pads by at least 0.5 um.

enclosure -check_name TSV_enclosure \
	-placement1 pInterp_TSV -placement2 pInterp_bmetal1 \
	-constraint "< 0.5 REGION" -comment "TSV is less than 0.5 microns from     the edge of pad on layer bmetal1"

external

 
external -check_name check_name 
 -placement1 placement1 [-placement2 placement2]
 -constraint "constraint_value" 
 [-comment "comment"] [rve_option …]

Check spacing between two placed layers:

external -check_name external_check 
 -placement1 p_stack_if1_stack_bot_if \
 -placement2 p_base_bot_if -constraint "< 0.4 REGION" \
 -comment "Checking < 0.4 REGION between \
  layer p_stack_if1_stack_bot_if and layer p_base_bot_if"

Check for sufficient spacing between chip placements. Three chips are placed on an interposer using the following assembly operations:

place_chip -placement Controller -chip CONT -x_origin 40 \
 -y_origin 40 
place_chip -placement Mem_1 -chip RAM -x_origin 250 \
 -y_origin 100 
place_chip -placement Mem_2 -chip RAM -x_origin 280 \
 -y_origin 40 -rotate 90

The chip instantiations can be directly referenced in a geometrical check as follows:

external -check_name controller_spacing_check \
 -placement1 Controller -constraint "< 65 REGION" \
 -comment "Chip placement closer than 65 um."

extra_ports

 
extra_ports ‑check_name check_name 
 ‑placements placement_name_list 
 [-comment "comment"] [rve_option …]

Verify that the layout pads match the source netlist for one of the placements of the ram chip.

extra_ports ‑check_name ram2_pad_text_rule \
	‑placements p_ram_2 \
	-comment "ERROR: Extra ports detected!"

floating_texts

 
floating_texts ‑check_name check_name 
 {‑chips chip_name_list | ‑placements placement_name_list } 
 [-comment "comment"] [rve_option …]

Verify that all text is correctly attached to geometry on the controller and interposer die. 

floating_texts ‑check_name cont_int_pad_text_rule \
	‑chips {controller interposer} \
	-comment "ERROR: Text not attached to any geometry!"

Verify floating text for only one of the placements of the ram chip.

floating_texts ‑check_name ram2_pad_text_rule \
	‑placements p_ram_2 \
	-comment "ERROR: Text not attached to any geometry!"

internal

 
internal -check_name check_name 
 -placement1 placed_layer1 [-placement2 placed_layer2]
 -constraint "constraint_value" 
 [-comment "comment"] [rve_option …]
 
internal -check_name internal_check \
 -placement1 p_stack_if1_stack_bot_if \
 -placement2 p_base_bot_if \
 -constraint "< 0.3 REGION" \
 -comment "Checking < 0.3 REGION between layer \    p_stack_if1_stack_bot_if 		    and layer p_base_bot_if"

locations

 
locations ‑check_name check_name ‑placement1 placement_layer1  ‑placement2 placement_layer2 
 [‑constraint "constraint_value"] 
 [‑text_only | ‑overlap_only]  
 [-no_case] [‑comment  "comment"] [rve_option …]
 
locations	 -check_name	 locations_check  \
	-placement1 p_layout_l_pads  \
	-placement2 p_source_s_pads

missing_ports

 
missing_ports ‑check_name check_name 
 ‑placements placement_name_list 
 [-comment "comment"] [rve_option …]

Verify that the layout contains matching pads found in the source netlist for one of the placements of the ram chip.

missing_ports ‑check_name ram2_pad_text_rule \
	‑placements p_ram_2 \
	-comment "ERROR: The layout is missing ports!"

multi_texts

 
multi_texts ‑check_name check_name 
 {‑chips chip_name_list | ‑placements placement_name_list } 
 [-comment "comment"] [rve_option …]

Verify that each pad on the controller and interposer die only contains a single text label. 

multi_texts ‑check_name cont_int_pad_text_rule \
	‑chips {controller interposer} \
	-comment "ERROR: Pads contain multiple text labels!"

Perform the same check on the pad text for only one of the placements of the ram chip.

multi_texts ‑check_name ram2_pad_text_rule \
	‑placements p_ram_2 \
	-comment "ERROR: Pads contain multiple text labels!"

no_texts

 
no_texts ‑check_name check_name 
 {‑chips chip_name_list | ‑placements placement_name_list } 
 [-comment "comment"] [rve_option …]

Verify that all pads are correctly labeled on the controller and interposer die. 

no_texts ‑check_name cont_int_pad_text_rule \
	‑chips {controller interposer} \
	-comment "ERROR: Pads are missing text labels!"

Verify pad text for only one of the placements of the ram chip.

no_texts ‑check_name ram2_pad_text_rule \
	‑placements p_ram_2 \
	-comment "ERROR: Pads are missing text labels!"

offgrid_centers

 
offgrid_centers -check_name check_name 
 -placement placed_layer 
 –resolution {value | {x_value y_value }} 
 [-comment "comment"] [rve_option …]

Check that pads are aligned to a 5 micron square grid:

offgrid_centers -check_name offgrid_memory -placement m_mtsv \
 -resolution 5 -comment "m_mtsv pads align to 5x5 um grid"

Check that pads are aligned to a 5 um by 10 um (x by y) grid:

offgrid_centers -check_name offgrid_memory -placement m_mtsv \
 -resolution "5 10" -comment "m_mtsv pads must align to \
  5x10 um grid"

overlap

 
overlap -check_name check_name -placement1 placed_layer1 
 -placement2 {placed_layer2 [placed_layer2 …]} 
 -constraint "constraint_expression"
 [-merge][-by_area] [-intersection] [-comment "comment"]
 [rve_option …]

Measure the overlap by percentage and output the intersection region.

overlap -check_name overlap_tracer_with_interposer \
	-placement1 m_mtsv_trace -placement2 i_tsv \
	-constraint "<= 80.0" -intersection \
	-comment "TSV and interposer pads must overlap by 80%"

Measure the overlap by area.

overlap -check_name overlap_tracer_with_interposer \
	-placement1 m_mtsv_trace -placement2 i_tsv \
	-constraint "<= 20.0" -by_area \
	-comment "Overlap not at least 20um^2 between tsv and \    interposer pads"

Check that all of the interposer pads have suitable overlap with all of the pads of the chips placed on the interposer.

overlap -check_name interposer_pads \
	-placement1 p_interposer \
	-placement2 {"p_controller" "p_ram_1" "p_ram_2" "p_ram_3"} \
	-constraint "<= 95" \
	-comment "Interposer pads must overlap chip pads by 95%!"

Check for completely missing interposer pads. In this case, you can use an overlap percentage constraint. Any pads that are missing are reported.

overlap -check_name interposer_pads \
	-placement1 p_interposer \
	-placement2 {"p_controller" "p_ram_1" "p_ram_2" "p_ram_3"} \
	-constraint "< 90" \
	-comment "Interposer pads must overlap chip pads"

same_size

 
same_size -check_name check_name
-placement1 placed_layer 
-placement2 placed_layer
[-comment “comment”] [rve_options...]

select_checks

 
select_checks { 
 -check_names ‘{’ name_pattern [name_pattern …] ‘}’
 | -placements ‘{’place_name_pattern [place_name_pattern …]‘}’ } 
[-check_types ‘{’check_type …‘}’]

Execute only the checks that are used in the interposer and any of the ram placements:

select_checks -placements {"interposer" "ram_*"}

unselect_checks

 
unselect_checks {
-check_names ‘{’ name_pattern [name_pattern …] ‘}’
 | -placements‘{’place_name_pattern[place_name_pattern …]‘}’ }
 [-check_types ‘{’check_type …‘}’ ]

Exclude checks that are used in the interposer and ram placements:

unselect_checks -placements {"interposer" "ram_*"}
