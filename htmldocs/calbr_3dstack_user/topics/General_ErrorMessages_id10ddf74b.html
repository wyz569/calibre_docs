Error Messages

Error messages must be corrected to continue a Calibre 3DSTACK run.

Error messages take the following form:

3DSTACK_ERROR_number: error_message
Note:

If you encounter a layout peek error, try running Calibre 3DSTACK on a machine with more memory.

3DSTACK_ERROR_607

Syntax error.

  

3DSTACK_ERROR_608

<error_message>

This error number is used in multiple cases; the message describes the specific case, if possible.

One such case that this error can occur is when an input file is not correctly formatted. For example, if you created a file in Windows, it may sometimes include the wrong line encoding (carriage returns instead of line feeds). This can trigger an error such as this:

3DSTACK_ERROR_608: in file file: on line line: invalid command name "command"

The solution for this error is to run dos2unix on the file.

Another error is the following:

The name primary is already defined for lefdef in die name.

This is caused by more than one LEF/DEF inputs having the same top cell name defined in the rule file.

This error can also be generated from an error condition in Calibre DESIGNrev. If the error is a layout peek error, then try running the design on a machine with more memory.

3DSTACK_ERROR_1400

One of the following:

Missing section DECLARATION in iRCX file ircx_file

Missing object DECLARATION in DECLARATION section of iRCX file ircx_file

Missing SECTION LAYER MAPPING in DECLARATION object of DECLARATION section of iRCX file ircx_file

Missing section ${layerMappingSection} in iRCX file ircx_file

Missing layer ${layerNumber} in the layer mapping section of iRCX file ircx_file

One or more of the following issues with iRCX file specified using the     -layer_info ‑ircx option:

  • There is no DECLARATION section.

  • There is no DECLARATION object in the DECLARATION section.

  • There is no SECTION[LAYER MAPPING] in the DECLARATION object.

  • There is no LAYER MAPPING section in the DECLARATION object.

  • There is a missing layer number and datatype in the layer mapping.
