Rule Check Commands

Rule check commands appear in rule check procs and are used to check the design. Some rule check commands may be used in init procs. These are annotated in the notes to the tables in this section.
Table 1. High-Level Commands

Command

Description

perc::check_data

Calls a user-provided procedure to perform a check. This command can be called at most once in any rule check proc, and must be the only high-level “check” command called in the Tcl proc.

perc::check_device

Checks each primitive device in the design for devices that match the specified conditions. This command can be called at most once in any rule check proc and must be the only high-level “check” command called in the Tcl proc. This command passes an instance iterator to a ‑condition proc.

perc::check_device_and_net

Checks each primitive device and net in the design to find the ones that match all of the specified conditions. This command can be called at most once in any rule check proc and must be the only high-level “check” command called in the Tcl proc. This command passes instance and net iterators to condition procs.

perc::check_net

Checks each net in the design for nets that match the specified conditions. This command can be called at most once in any rule check proc and must be the only high-level “check” command called in the Tcl proc. This command passes a net iterator to a ‑condition proc.

perc::report_base_result

Writes user-defined content to the report file. This command must be called in the context of one of the other high-level commands. It may be called any number of times in a rule check proc. This command defers reporting of net-based results until the complete net has been analyzed.

Commands in Table 1 (except perc::check_data and perc::report_base_result) can create iterators that are passed internally to a -condition procedure.

perc::subtype

perc::trace

perc::type

perc::y_coord

tvf::svrf_var

Command

Description

perc::export_pin_pair

Exports pin pairs on the same net for LDL checks.

perc_ldl::custom_r0

Calculates the common interconnect resistance, R0, between a source and a branch location on an LDL P2P resistance path. By default, this resistance is not calculated separately, but doing so can give a more complete understanding of results.

perc_ldl::design_cd_experiment

Defines current density experiments for a Calibre PERC LDL CD run.

perc_ldl::design_p2p_experiment

Defines point-to-point resistance experiments for a Calibre PERC P2P run.

Executes current density experiments for a Calibre PERC LDL CD run.

perc_ldl::execute_p2p_checks

Calculates interconnect resistances between specified pins in a Calibre PERC LDL P2P run.

perc_ldl::restart

Re-runs LDL CD or P2P starting from a specified point in the flow.

Table 9. Termination Commands

Command

Description

perc::terminate_rule_check

Terminates a a rule check with a runtime error and causes the overall run status to be ABORTED.

Terminates a run with an error upon execution of the command.
1 Iterator commands may be used in an initialization procedure. The perc::get_instances, perc::get_nets, and perc::get_pins commands are among the more useful for this purpose.
