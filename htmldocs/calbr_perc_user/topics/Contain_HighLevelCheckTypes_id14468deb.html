High-Level Check Types

Calibre PERC supports the following high-level LDL checks.
Table 1. High-Level Check Types

Check Name

Description

Checked Design

CELL_BASED_CD

Reports current densities along paths containing top-level IO nets, supply nets, and appropriately-connected ESD protection cells. It also finds unprotected top-level IO and supply nets.

Layout

CELL_BASED_P2P

Reports point-to-point resistances along paths containing top-level IO nets, supply nets, and appropriately-connected ESD and clamp cells. It also finds unprotected top-level IO and supply nets.

Layout

CELL_NAME

Finds names of subcircuits that are invalid.

Layout/

Source

DEVICE_BASED_CD

Reports current densities along paths containing top-level IO nets, supply nets, and appropriately-connected ESD and clamp devices. It also finds unprotected top-level IO and supply nets.

Layout

DEVICE_BASED_P2P

Reports point-to-point resistances along paths containing top-level IO nets, supply nets, and appropriately-connected ESD and clamp devices. It also finds unprotected top-level IO and supply nets.

Layout

DEVICE_COUNT

Counts devices in the design.

Layout/

Source

DEVICES_IN_PATH

Verifies that correct protection structures exist between specified ports or pins.

Layout/

Source

DEVICE_NOT_PERMITTED

Finds forbidden devices in the design.

Layout/

Source

FIND_PATTERN

Finds pattern template topology matches in the design and reports them as errors.

Layout/Source

IO_RING

Verifies spacing and count criteria for top-level pads in a LEF/DEF design. (Available only in the Calibre YieldServer LDL rule generator.)

Layout

PATTERN_IN_PATH

Verifies whether a pattern template match exists between two points in the design and reports an error if the pattern topology is absent.

Layout/Source

VOLTAGE_AWARE_DRC

Finds objects meeting voltage difference criteria and that fail a DRC-style check.

Layout

The CELL_BASED_CD, CELL_BASED_P2P, DEVICE_BASED_CD, and DEVICE_BASED_P2P may each only be specified once. They may not be run together in the same rule file.
