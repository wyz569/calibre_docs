Viewing Net Area Ratio Properties and Output with the ACCUMULATE Option

You can use Calibre RVE for DRC to view RDB output and properties from the Net Area Ratio (NAR) SVRF statement.

This procedure shows how to have the AR value (the accumulated ratio of the individual polygon that fails the rule check) output as an property attached to the failing geometry in an RDB. The following is required in the Net Area Ratio operation in the rule check:

  • “ACCUMULATE alayer” argument set, where the accumulation layer alayer has been derived in a previous operation

  • RDB keyword with the denominator layer specified as an rdb_layer. The denominator layer is the gate layer in the example that follows.

  • (Recommended) The BY LAYER keyword when generating RDB output with the Net Area Ratio operation in order to have results reported by layer.

See the section “Results Database (RDB) Creation” with the Net Area Ratio operation for complete details.

Prerequisites

  • A Calibre nmDRC rule file

Procedure

  1. Create the rule check. The example assumes layers MET1, POLY, and gate have extracted connectivity information.
    1. Create an accumulation layer, such as the gate layer. This layer stores the net area ratio numerical result. For example:
      acc = NET AREA Ratio POLY gate >=0 ACCUMULATE
    2. Create the rule check for the antenna ratio using the ACCUMULATE and RDB arguments. 
      nar_check {
	NET AREA RATIO MET1 gate > 300 ACCUMULATE acc
 		RDB nar_check.rdb MET1 gate BY LAYER
}
  2. Run Calibre nmDRC.

    For the example code shown, the NAR RDB nar_check.rdb is created along with the main results database.

  3. Open the NAR RDB in Calibre RVE for DRC.

    See the section “Viewing Auxiliary Databases” for instructions on opening an auxiliary database.

    Figure 1 shows an RDB created from the rule check given in Step 1. The result view is in Details format with the Result Data Pane displayed. The “AR” and “AR acc” properties are shown in the result view table.

  4. To create a histogram, right-click in the tree view and choose Histogram > AR.

Results

Figure 1. Display of Net Area Ratio RDB

See the section “Results Database (RDB) Creation” with the Net Area Ratio operation for a description of the RDB check names, properties, and check text header.

The calculation summary data is only shown for the “denominator” layer (gate in this example) specified by the d_layerN parameter in the Net Area Ratio statement. The values shown in the calculation summary are defined as follows:

  • Net Value — Value of current NAR ratio value or expression.

  • Prev. Accumulation — The previous accumulated ratio value for the layer, which is equivalent to this expression:

    Prev. Accumulation = Total Result - Net Value

  • Total Result — The accumulated ratio (AR) value.
