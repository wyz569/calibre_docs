RVE Highlight Index

A DRC RVE rule check comment which specifies the highlight layer index used for the rule check.

Usage

RVE Highlight Index: index

Parameters

index

A required argument, where index values up to 32767 can be used. However, most layout viewers limit the number of layers available for highlighting, so the highlight layer index may start over after the maximum index is reached.

A space is required between the colon (:) and index.

Description

This rule check comment specifies the highlight layer index. Highlights for the rule check are always made to that layer index; the highlight layer is labeled “rve” in the attached layout editor. RVE Highlight Index is not used in a Calibre RVE for PERC session.

All layout editors support this command. You can set the color in your layout editor, if desired.

See “Methods to Add DRC RVE Rule Check Comments” for instructions on specifying the rule check comment. You can add the rule check comment to the rule check in the rule file or specify it in a Check Text Override (CTO file.

RVE Highlight Index and RVE Highlight Color rule check comments should not be used together in a rule file or a DRC RVE Check Text Override File (CTO File) Format. If both are found in the DRC results database, the RVE Highlight Index statements take precedence and the RVE Highlight Color rule check comments are ignored. You can override what is specified in the results database by selecting Highlight > Set Check Highlight Layers, enabling “Override Check Defaults” in the Set Highlight Layers Dialog Box, and setting the desired highlight layers.

Examples

This example shows the rule check comment placed within a rule check.

// rule file
// ...
metal1_space_rule {
@ comment
@ RVE Highlight Index: 5
< rules ...>
}

This example shows the rule check comment in a DRC RVE Check Text Override File (CTO File) Format:

# DRC RVE .cto file
metal1_space_rule 
RVE Highlight Index: 5
