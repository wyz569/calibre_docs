Examining LVS Comparison Results

This section walks you through methods for examining LVS comparison errors in the Comparison Results tab of Calibre RVE.

Prerequisites

  • An SVDB containing LVS Comparison Results. See “Rule File Requirements for Examining Results with Calibre RVE for LVS”.

  • The layout database for your LVS run.

  • A layout viewer with an interface to Calibre RVE. See “Connection to a Design Tool”.

Procedure

  1. Open your layout in Calibre DESIGNrev or in another supported layout viewer.
  2. Start Calibre RVE from your layout viewer, as described in the procedure “Invoking Calibre RVE From a Layout or Schematic Viewer” and open your SVDB.
  3. View the Extraction Results tab. Your Calibre nmLVS run should have no connectivity extraction errors or warnings. Extraction errors and warnings may cause LVS comparison errors, and should be corrected.
  4. Click the Comparison Results tab.

    The opening view will be similar to that seen in Figure 1, with the results sorted by cell. Look at the Count column to see how many comparison errors are in each cell.

    If a cell has a listing of Not Compared or Incorrect in the discrepancy count column, click the cell in the tree view to select it, then look at the report in the detailed view to discover the problem.

    A red box icon () indicates comparison failed for a LVS Box cell due to a mismatch in the number of ports.

    Figure 1. LVS Comparison Results Tab

    If desired, you can show the clean cells by enabling View > Tree Options > Show Clean Cells.

  5. (Optional) The detailed view (bottom pane) displays only the first 10 results in a category by default. If there are more than 10 results, you can click to view the next group of results, as shown below.


    You can change the number of results displayed with the setting “Show Detailed Results” (Options tab, LVS/PERC/PEX category).

  6. View discrepancies by error type.

        

    Right-click in the header of the first column in the tree view to get a pop-up menu for sorting discrepancies, and choose View by Error Type.

    The results are now listed by type of error. A cell appears under each error type that it has an error for, so a cell may appear in multiple error categories. When you expand the tree for a particular cell it shows all of the results for that cell.

    In addition to the error categories, cells may be listed under Clean, Not Compared, or Incorrect.

  7. In the detailed view, click an element in blue or red text to display the element in the internal schematic viewer. The design element is also highlighted in your layout viewer. (See Figure 1, Figure 1)

    In the internal schematic viewer, use the zoom to fit navigation icon to resize and center the image if necessary. You may also need to move and resize the windows within the main session window.

        

    You can choose the arrangement of the schematic windows from the Window menu. The schematic viewer windows can be undocked and docked from the main session window with the controls shown at right. The selection Window > Dock All returns all undocked windows to the main display.

  8. Now, discover more information about the error. Right-click a blue or red element in the detailed view to display a context-sensitive menu with options for highlighting the element or showing detailed information about the element in the Info Pane.

    In the schematic view, you can use the navigation methods discussed in “Navigating Tips for the Internal Schematic Viewer” to gain more insight about the error. For example, double-click a pin to draw the nets connected to that pin.

    Once you have fixed an error, you can mark it as fixed and move on to the next error.

  9. Mark the error as fixed by right-clicking the error in the tree view and choosing Discrepancy Fixed in the pop-up menu. Discrepancy Fixed is only available on discrepancies, not on Fix Suggestions.
  10. Task Validation — After you correct as many errors as you can, run Calibre nmLVS again. Then choose File > Reload Database in Calibre RVE for LVS, and review the Comparison Results tab to see that the errors are fixed.
