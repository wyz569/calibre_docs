Tracing Possible Hierarchical Short Chains to Non-Floating Extra Pins

You can use Calibre RVE for LVS to trace possible hierarchical shorts in the layout. The discrepancy types “Instances of cells with non-floating extra pins” and “Possible hierarchical short chains to non-floating extra pins” are relevant for this type of problem. In this context, a hierarchical short is an unwanted path between two nets at different levels of the layout hierarchy.

See “Instances of Cells With Non-Floating Extra Pins” in the Calibre Verification User’s Manual for detailed information about this discrepancy type.

Prerequisites

  • An SVDB from a Calibre nmLVS run which includes LVS comparison. See “Rule File Requirements for Examining Results with Calibre RVE for LVS”.

  • The layout database for your LVS run.

  • A layout viewer with an interface to Calibre RVE. See “Connection to a Design Tool”.

Procedure

  1. Open your layout in Calibre DESIGNrev or in another supported layout viewer.
  2. Start Calibre RVE from your layout viewer, as described in the procedure “Invoking Calibre RVE From a Layout or Schematic Viewer” and open your SVDB.
  3. Disable Highlight > Clear Existing Highlights.
  4. Select the Comparison Results tab.
  5. Select “View by Error Type” in the LVS toolbar, or select View > Tree Options > View by Error Type.
  6. Expand the “Possible hierarchical short chains to non-floating extra pins” error type and select a discrepancy. A typical view is shown in the following figure:


  7. Click the net links to highlight them in the connected layout viewer. Nets are highlighted in the context of the cell they are associated with by default. A useful debug process is to highlight the end points of the possible short chain and then highlight each intermediate net starting with the highest point in the hierarchy. (The discrepancy shown in the previous step consists only of the two endpoints, and has no intermediate nets in the short chain.)

    Clicking on a net that is not one of the endpoints sometimes results in the “Highlight in Lower Cell” pop-up dialog box with the message “Layout cell cell has no shapes on net n. ...” This occurs if Calibre nmLVS added ports to the extracted netlist in order to include the possible hierarchical short chain. Such added ports have no corresponding geometry in the layout; however, Calibre can determine where such a port connects in a lower level cell. Click the Highlight in Lower Cell button to highlight the net in a lower level cell in the layout. The Continue and Cancel buttons both result in no highlight.

    The following image shows the two endpoints of the discrepancy highlighted. In this example the short chain consists of only the two endpoints.

  8. Right-click anywhere in the discrepancy view and select Isolate Path.

    This opens the Finder tab, populates the fields for an “Isolate Path on Layout Net” search, and opens the dialog box to select the path endpoints in the layout viewer.



  9. Select endpoints in the layout—one endpoint on the starting net of the short and one endpoint on the ending net.
    It is important that you zoom into the layout enough to make sure the selected points are on the net. If the dialog box displays a message that the location is not on the net, start over with the previous step, as the Finder dialog no longer has the correct information.

    The path search may take some time if the design is large.

  10. Examine the results.

    The Isolate Path results are listed in the Finder result area and the first path is highlighted, as shown in the following figure.



    To highlight a different path, click the IP () icon next to a path result to highlight that path. You may want to first select Highlight > Use Layer Colors to highlight each layer in the path result in a different color. The following figure highlights a different path than the previous figure and has “Use Layer Colors” on.



    You can use the path information to examine the layout polygons and determine the problem.
