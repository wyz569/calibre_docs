Handling Cell Names with $D1$ and $D2$ from DRC Dual Database Runs

When Calibre nmDRC runs in dual database mode, it writes cells with names that end in “$D1$” and “$D2$” to the results output database. Because such cells probably do not exist in the input design, Calibre RVE strips those trailing four characters from cell names by default before sending them to the layout editor when highlighting results. You can change this behavior with an environment variable or an option setting in Calibre RVE for DRC.

Use one of the following methods set the behavior:

  • MGC_RVE_NO_DUAL_DB environment variable

    To change the default behavior, define the environment variable MGC_RVE_NO_DUAL_DB. This instructs Calibre RVE to not strip the trailing “$D1$” and “$D2$” from cell names when sending highlight information to the layout editor.

  • “Strip dual database cell name suffixes while highlighting” option

    The option “Strip dual database cell name suffixes while highlighting” is on the Options tab in the Highlighting category in the “DRC/DFM Highlighting” expandable section.

If MGC_RVE_NO_DUAL_DB is defined the option “Strip dual database cell name suffixes while highlighting” option is disabled when you open Calibre RVE.

Note:

Calibre RVE saves option settings when you exit the tool and restores them when you start another session. Therefore, if you set MGC_RVE_NO_DUAL_DB in order to disable the option “Strip dual database cell name suffixes while highlighting,” the option setting is saved when you exit Calibre RVE. The setting will still be disabled then next time you invoke Calibre RVE even if MGC_RVE_NO_DUAL_DB is no longer defined.
