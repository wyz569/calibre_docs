DRC/DFM Highlighting Options

Access with: Setup > Options, choose the Highlighting category, then expand the “DRC/DFM Highlighting” section.

The DRC/DFM Highlighting area of the Setup Highlighting Options pane controls highlighting options for Calibre RVE for DRC and Calibre RVE for DFM.

Fields

Table 1. DRC/DFM Highlighting Options

Setting

Description

Highlight DRC results in context

Highlight results in the cell in which they are reported, rather than in the top cell. See “Viewing Results in Cell Space”.

Display lower-level cell highlights in top cell

 

(Calibre DESIGNrev only)

When “Display lower-level cell highlights in top cell” is enabled and you highlight a hierarchical result in Calibre DESIGNrev, each instance of the result is highlighted at the top level. “Highlight DRC results in context” must be enabled for this option to be available.

Note: The view depth in Calibre DESIGNrev must be set to at least one in order to view the errors at the top level. In addition, if “Display lower-level cell highlights in top cell” is enabled, the Highlight > Zoom to Last Highlight operation may not zoom to the correct location.

Clear highlights when DRC results are fixed or waived

Enable this to automatically clear highlights when a result is marked as fixed or waived.

Only show check-dependent layers while highlighting results (hides other layers)

When enabled, Calibre RVE makes the layers encoded in the check comment 

@ RVE Show Layers: <layer> ...

visible when highlighting. This command is only available for the Calibre DESIGNrev and Cadence Virtuoso layout editors.

See the following:

  • “RVE Show Layers”

  • “Calibre RVE with Calibre DESIGNrev and Calibre WORKbench”

  • “Overview of Highlighting in Cadence Virtuoso with Calibre RVE”

Annotate result data to highlights as properties

When enabled, Calibre RVE adds result data (check names, result numbers, and properties) as properties to highlight shapes.

This setting works for the Calibre DESIGNrev and Cadence Virtuoso layout viewers only. Use the layout viewer controls to view the properties for the highlight object.

See “Displaying Result Data as Properties in Calibre DESIGNrev” for instructions on using this feature with Calibre DESIGNrev.

See “Displaying Result Data as Properties in Cadence Virtuoso” for instructions on enabling this feature in Cadence Virtuoso.

Strip dual database cell name suffixes while highlighting

When enabled, Calibre RVE strips the trailing “$D1$” and “$D2$” characters from dual database cell names when highlighting results.

This setting is enabled by default. You can define the environment variable MGC_RVE_NO_DUAL_DB to disable this setting. Also see “Handling Cell Names with $D1$ and $D2$ from DRC Dual Database Runs”.

Override check defaults to set the highlight layer

Select the method for assigning the layers used in the layout viewer for highlight operations.

 

See the section “Set Highlight Layers Dialog Box” for a description of the selections.

Show Check Names

Display the check name in the center of each result shape when highlighting.

Show Result IDs

Display the result number in the center of each result shape when highlighting.

Show Properties

Display properties in the center of each result shape when highlighting. Select one of the following options:

  • All — Display all properties

  • Text box — Display only the properties listed in the text box. Enter a space-separated list of property names. If a desired property contains spaces, then quote the property name. Wildcard characters are permitted.

    Example: DV “DA M1*” DN

Tip

Set the precision for the display of property values with the “Apply precision to display of reals” setting in the Session category on the Options tab.

Override check text properties

This setting is available if “Show Properties” is enabled.

Enabled — Use the “Show Properties” setting to determine which properties are shown when highlighting and ignore the RVE Show Properties check text comment.

Disabled — If the RVE Show Properties check text comment is present for a rule check, use it to determine which properties are shown when highlighting, otherwise, use the “Show Properties” setting.
