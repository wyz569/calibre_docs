Links to Custom Reports in Calibre RVE for PERC

If you have a custom text or HTML-formatted report, you can create a link to the custom report in the Calibre RVE for PERC display.

There are three methods for creating the link:

  • By editing the svdb/topcell.extf file to create a link in the Navigator pane, as explained in “Creating Links to Custom Reports in Calibre RVE for LVS”.

  • By adding the RVE Link rule check comment using a DRC RVE Check Text Override File (CTO File) Format. The custom link appears in the Check Text pane of Calibre RVE.

    An example CTO file is the following:

    rule_2
RVE Link: rule2_debug.htm TEXT Link to rule_2 debug file

    where rule2_debug.htm is the custom report, and “Link to rule_2 debug file” is the link text displayed in the Check Text pane when rule_2 is selected. Only the RVE Link check text comment is supported in Calibre RVE for PERC.

    You load the CTO File with Tools > Import CTO File in Calibre RVE or by specifying the CTO file with the environment variable MGC_RVE_CTO_FILE. When you use a CTO file, the check text comments from the RVE Link statement are appended to any comments specified with the ‑comment switch in the Calibre PERC rule check command.

  • By adding the RVE Link rule check comment using the ‑comment switch in a Calibre PERC rule check command. The custom link appears in the Check Text pane of Calibre RVE.

    An example with the perc::check_net command is shown here:

    perc::check_net -netType {!Power && !Ground} \
	-condition rule_3_cond  \
	-comment "RVE Link: report.txt TEXT rule_3 debug file\n \
	Decoupling cap without CDM clamp" 
	# RVE Link must be on a separate line with no leading spaces

    The ‑comment switch in this example is used to output both a text comment and the RVE Link check text comment. The RVE Link check text comment must be output on a separate line with no leading white space.
