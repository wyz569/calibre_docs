Waiver Reporting

You setup the waiver report using certain DFM HTML reporting key names.

Use the following key names in your DFM HTML reporting configuration setup file to report the results of the automatic waiver flow:

  • Errors Marked to be Waived — Use the WaivedCommentFilter, WaivedByFilter, WaivedDateFilter, and WaivedStateFilter keys. The waiver state of the error appears in the Waived column, which the tool displays only if at least one rule contains errors marked to be waived.

  • Waived Errors — Use the WaiveStatesToInclude key. The UnwaivedErrorResults, WaivedErrorResults, and UnusedWaiverResults can then be reported in separate tables.

    Note:

    The waiver flow is only property preserving and does not preserve annotations. Consequently, the WaiveStatesToInclude key ignores any RulesToInclude or MetricsToInclude keys.
