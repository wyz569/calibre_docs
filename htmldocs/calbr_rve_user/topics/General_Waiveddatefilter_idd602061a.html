WaivedDateFilter

Key Value for DRC HTML Report Configuration File, filter category

Usage: Optional. Valid in all report types, and in Common section.

Inheritance: Filter Inheritance. If the GroupBy key references another section, then the key and value are read from that section and any locally specified value is ignored.

Include only results which match the specified date constraint.

Waived dates in the DRC results database are converted from local time to GMT (Greenwich Mean Time) before comparing to the specified date or date range. In some cases the date may be different in local time and in GMT.

Allowed Values

Range date1 date2

Match dates within the given range (inclusive).

The date is given in either of the following formats:

"MMM DD YYYY"
"DD MMM YYYY"

where:

  • DD — The day of the month; the leading 0 is optional for days below 10.

  • MMM — The month, indicated by the first three characters of the name of the month in English. (Jan, Feb, Mar, Apr, May, Jun, Jul, Aug, Sep, Oct, Nov, Dec) The month is case-insensitive.

  • YYYY — The year, given with four digits.

constraint date

Match the date using the given constraint. The possible constraints are: <     >     ==     <=     >=     !=

Examples

Select results waived on or after January 1, 2010 (GMT):

WaivedDateFilter = >= "jan 01 2010"

Select results waived on or between January 1 and March 30, 2010 (GMT):

WaivedDateFilter = Range "Jan 01 2010" "Mar 30 2010"

To select all results waived on a specific date, it may be necessary to compensate for the fact that the waived date may change in some cases when converted from local time to GMT. The following filter selects results waived on March 5 or 6, 2011 (GMT):

WaivedDateFilter = Range "Mar 05 2011" "Mar 6 2011"
