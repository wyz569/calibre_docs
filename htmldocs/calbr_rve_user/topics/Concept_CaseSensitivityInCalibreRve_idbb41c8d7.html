Case Sensitivity in Calibre RVE

Calibre RVE handles case sensitivity differently depending on the application and the task being performed.

Calibre RVE handles case in the following manner:

  • Highlighting — Highlight operations preserve the case found in the results database.

  • Cross-referencing — Cross-referencing between source and layout in Calibre RVE for LVS, PEX, and PERC preserves the case found in the results and cross-reference databases.

  • Schematic browsing — The Internal Schematic Viewer obeys the case-sensitivity specified by the Layout Case and Source Case statements in the rule file.

    The case-sensitivity of the Calibre RVE SPICE File Viewer is set in the Options tab, Schematic Viewer category, of Calibre RVE. See “Using the Calibre RVE SPICE File Viewer”.

  • Find Path — The case-sensitivity of Find Path is determined by the statements Layout Case and Source Case in the rule file. If Layout Case and Source Case are not included in the rule file, the default behavior (not case-sensitive) applies.

For highlighting and cross-referencing between source and layout to work properly, the case of net and cell names must be handled properly in the results database, cross-reference database, and the extracted netlists. This is especially important when highlighting to design tools which are case-sensitive. For example, if a layout net is named Out2, but listed as OUT2 in the cross-reference database and extracted layout netlist, then cross-probing between source and layout will not work properly with a case-sensitive design tool.

Calibre is case-insensitive by default during netlist parsing, netlist extraction, and LVS comparison. The default behavior causes reports, results databases, and cross-reference databases to treat the various design elements in a case-insensitive manner. The default behavior may not be desirable if net names are case-sensitive or your design tool is case-sensitive.

The following statements are available to control the case-sensitivity of Calibre:

  • Layout Case — Specifies case-sensitive parsing of the extracted layout netlist.

  • Source Case — Specifies case-sensitive parsing of the source netlist.

  • LVS Compare Case — Controls case sensitivity during LVS circuit comparison.

  • Layout Preserve Case — Controls how layout text names are handled during connectivity extraction and how Device properties are written to the extracted netlist and generated result databases.

These statements apply to Calibre nmLVS, Calibre PERC, and parasitic extraction. Layout Preserve Case also has applications in Calibre nmDRC. See the specific commands for details. Incorrect settings for these statements can cause incorrect highlighting and cross-probing with Calibre RVE.
