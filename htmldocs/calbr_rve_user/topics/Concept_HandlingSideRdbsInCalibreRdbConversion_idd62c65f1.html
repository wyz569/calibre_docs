Handling of Side RDBs in Calibre RDB Conversion

By default, results from side or auxiliary RDBs are not included when you convert results from an ASCII format Calibre results database (RDB) to a native Place and Route format. You use the “-side_rdb YES” option to include side RDB results.

The following statements describe how results from side RDBs are processed:

  • If a side RDB cannot be found, a warning is issued and processing continues. The warning has the following format, where <filename> is replaced with the filename of the auxiliary database:

    Warning: side RDB '<filename>' cannot be found.

  • Results in a side RDB are not duplicated if the same result exists in the main RDB. When viewing converted errors in the Error Browser, all results with the same check name are listed together; results from the side RDB are not separately identified.

  • By default, all results in the RDB are included. If the ‑cell option is used, only results from the specified cell are included.

  • Side RDBs from Density and Net Area Ratio (antenna) checks are not converted.

  • The tool searches for side RDBs first in the same directory as the main results database, and if not found, the tool then searches in the working directory.

Example

Suppose the main results database, main.rdb, has two side RDBs named density.rdb and m4_space.rdb, where density.rdb is created with the RDB keyword in a Density rule check. The following command includes both main.rdb and m4_space.rdb results in the conversion, however the results from density.rdb are not included because it is a Density RDB. All results from main.rdb and m4_space.rdb are included by default.

icc_shell> Calibre::convert_to_drc_errors main.rdb -side_rdb YES
