calibre -rve Command Line for Calibre RDB Conversion

The Calibre RVE command includes options to convert results from an ASCII format Calibre results database (RDB) to native Place and Route tool formats. You execute the command in the Linux shell and save the converted output to file. The results can then be imported into your Place and Route tool.

Usage

calibre -rve -drc input_RDB
-convert_to { icc_drc | icc_rgf | edi_drc | edi_blk }
-output_file converted_rdb
[ -check_map_file map_file]
[ -min_feature min_size]
[ -offset x_offset y_offset ]
[ -mag mag_value]
[ -cell cell]
[ -checks { ALL | check_N … }]
[ -side_rdb {YES | NO} ]
[-no_transcript]

Arguments

input_RDB

Required parameter specifying the Calibre results database to converted. See “Requirements for Converting Calibre RDBs to Native Place and Route Format” for the RDB formats that are allowed.

-convert_to { icc_drc | icc_rgf | edi_drc | edi_blk }

Required keyword and parameter specifying the format for the converted results.

  • icc_drc — Synopsys IC Compiler DRC error marker format.

  • icc_rgf — Synopsys IC Compiler route guide format.

  • edi_drc — Cadence Encounter DRC error marker format.

  • edi_blk — Cadence Encounter route blockage format.

-output_file  converted_rdb

Required keyword and parameter specifying the filename for the converted output.

-check_map_file map_file

Keyword and parameter specifying the check map file. The check map file is required when converting to route guides or route blockages and optional when converting to DRC error marker format. See “Check Map File Format for Calibre RDB Conversion” for details.

-min_feature min_size

Optional keyword and parameter specifying the minimum width of the converted route guides or route blockages in user units. This option is only valid when converting to Synopsys IC Compiler route guides (‑convert_to icc_rgf) or Cadence Encounter route blockages (‑convert_to edi_blk). A default of 0.002 user units is used if this option is not specified.

-offset x_offset y_offset

Optional keyword and parameters specifying an x- and y-offset. Values are given in user units. If both -offset and -mag are specified, the magnification is applied first.

-mag mag_value

Optional keyword and parameter specifying the magnification of the converted results. A mag_value of 2 magnifies the results by 2; a mag_value of 0.5 shrinks the results.

Use this option if the precision of the input RDB is not the same as the precision of the layout database. If both -offset and -mag are specified, the magnification is applied first.

-cell cell

Optional keyword and parameter specifying to only convert results from cell. If this keyword is not present all results are converted.

-checks { ALL | check_N … }

Optional keyword and parameter specifying the rule check results to convert.

  • ALL — Results from all rule checks are converted. This is the default if the ‑checks option is not included.

  • check_N … — Only results from the listed rule checks are converted.

    If a list of rule checks is provided, only those rule checks are converted. Results from a rule check listed in the check map file but not in the ‑checks list are not converted.

    The asterisk (*) and question mark (?) wildcards may be used but must be escaped to prevent wildcard expansion in the shell environment. Double quotes around the check list and the backslash escape character (\) before the wildcard are required. For example: ‑checks “M2\*” matches all rule checks starting with M2.

-side_rdb {YES | NO}

Optional keyword and parameter specifying whether to include the results from side (auxiliary) RDBs in the conversion. Side RDB results are not included by default. See “Handling of Side RDBs in Calibre RDB Conversion” for more information.

-no_transcript

Optional keyword that turns off transcript output.

Description

The Calibre RVE command with the ‑drc, ‑convert_to, and ‑output_file options is used to convert result shapes from Calibre rule checks to native Place and Route formats. You must have an existing ASCII format Calibre results database (RDB); see “Requirements for Converting Calibre RDBs to Native Place and Route Format”.

The format of the converted results is specified with the ‑convert_to option. See the equivalent tool command for information about the conversion process. The converted output is saved to the file specified with the ‑output_file option. Use the appropriate import command in the Place and Route tool session to import the converted results. The following table lists the equivalent tool command and the associated import command for each ‑convert_to option.

Table 1. Calibre RDB Conversion with calibre -rve

‑convert_to Option

Equivalent Tool Command

Import Command

icc_drc

Calibre::convert_to_drc_errors

Calibre::import_drc_errors

icc_rgf

Calibre::convert_to_route_guides

Calibre::import_route_guides

edi_drc

Calibre::convert_to_drc_errors

Calibre::import_drc_errors

edi_blk

Calibre::convert_to_route_blockages

Calibre::import_route_blockages

The check map file (‑check_map_file option) is required when converting to route guides or route blockages and optional when converting to DRC error marker format. See “Check Map File Format for Calibre RDB Conversion”.

After successful conversion the following message is written to stdout:

//  RDB written as PR input file <converted_rdb>

A transcript with information such as a the cell list, rule check list, precision, output file, and execution details is included by default. The transcript can be turned off with the ‑no_transcript option; errors, warnings, and notes are still output.

Examples

Example 1 — Synopsys IC Compiler

The following command converts results from the results database drc.results to Synopsys IC Compiler DRC error markers. The converted output is saved to the file top_calibre_drc.rdbout. By default all results are converted.

% calibre -rve -drc drc.results -convert_to icc_drc -output_file top_calibre_drc.rdbout

Use the following command at the icc_shell prompt to import the converted results to the Synopsys IC Compiler session:

icc_shell> Calibre::import_drc_errors top_calibre_drc.rdbout

See “Importing Calibre Results as Synopsys IC Compiler DRC Error Markers” for instructions on viewing the results in the Synopsys IC Compiler Error Browser.

Example 2 — Cadence Encounter

The following command converts results from the results database drc.results to Cadence Encounter DRC error markers. The converted output is saved to the file top_calibre_drc.rdbout. By default all results are converted.

% calibre -rve -drc drc.results -convert_to edi_drc -output_file top_calibre_drc.rdbout

Use the following command at the encounter prompt to import the converted results to the Cadence Encounter session:

encounter > Calibre::import_drc_errors top_calibre_drc.rdbout

See “Importing Calibre Results as Cadence Encounter DRC Error Markers” for instructions on viewing the results in the Cadence Encounter Violation Browser.
