Check Map File Format for Calibre RDB Conversion

Input for: Calibre RDB conversion to native Place and Route formats

The check map file is used when converting a Calibre results database (RDB) to native Place and Route formats. It associates check names in the RDB with layer names in the Place and Route environment. The check map file is required when converting to route guide or route blockage format.

The check map file is a two column list of Calibre check names followed by a layer name or list of layers. Wildcards may be used in the checkname. For example: 

M3_w     M3
M4_*     M4
M_s     “M2 M3 M4”

The check map file is typically created once for a given technology. Use the ‑check_map_file option in the Calibre RDB conversion commands to specify the file.

Use in Conversion to Route Guides or Route Blockages

The check map file is required when converting to route guides or blockages and specifies the target layer(s). The layer parameter is required and may be a list of layers. An error is issued and processing stops if the layer parameter is missing.

The rule check must be listed in the check map file in order for the route guide or blockage to be created. If a rule check is not listed in the check map file but has results in the input RDB a warning is issued and a route guide or blockage is not generated from that rule check. Processing continues for other rule checks in the check map file.

Synopsys IC Compiler Route Guides — Route guides are created with the ‑no_signal_layers option to the create_route_guide command, where the target layer(s) in the check map file provide the layer parameter(s) for the ‑no_signal_layers option.

Cadence Encounter Route Blockages — Route blockages are created with the ‑layer option to the createRouteBlk command, where the target layer(s) in the check map file provide the layer parameter(s) for the ‑layer option.

Use in Conversion to DRC Error Markers

The check map file is optional when converting to error markers and can be used to provide the layer property for the rule check in the Synopsys IC Compiler Error Browser. Only one target layer is used; if a layer list is specified only the first layer is used.

The layer parameter is optional when converting to DRC error markers. If no layer is included for a rule check or if a rule check is not included in the check map file, the error marker is created without a layer property.

Format

  • Text file

  • One check name and layer entry per line

  • Comment lines start with #

Syntax for conversion to route guide and route blockage output

check_name {layer | “layer1 layer2 …” }

Syntax for conversion to DRC error marker output

check_name [ layer | “layer1 layer2 …” ]

Parameters

check_name

Specifies the rule check name. Wildcards are supported for the check name, where the question mark (?) replaces a single character and an asterisk (*) replaces multiple characters.

layer | “layer1 layer2 …”

The layer or list of layers associated with the check_name rule check. A list of layers must be enclosed in double quotes. Wildcards are not allowed.

  • Conversion to route guides or route blockages — A layer or layer list is required and provides the target layers for the route guide or blockage.

  • Conversion to DRC errors markers — A layer is optional. The layer is included as a property for the rule check in the Synopsys IC Compiler Error Browser. In Cadence Encounter, the layer is displayed in the LAYER column in the Violation Browser. If a layer list is specified only the first layer in the list is used and the other layers are ignored.

Examples

Example 1

This check map file is for use when converting to route guides or blockages. The layer parameter is required. The route guides or blockages are created with the specified layer(s) as the target layers.

# <check name> {<layer> | <“layer1 layerN ...”> }
Mi2.S           M2
Mi2.E           M2
Mi3.X          “M1 M2 M3”
XYZ.X          “M1 M2 M3”

Example 2

The check map file in “Example 1” can be written as follows using wildcards:

# <check name> {<layer> | <“layer1 layerN ...”> }
Mi2.?           M2
*.X            “M1 M2 M3”

Example 3

This check map file is for use when converting to DRC error markers. The layer parameter is optional and not included for the Mi2.E rule check. A layer list is specified for the XYZ.X rule check but only the first layer is used when converting to DRC error markers. 

# <check name> [ <layer> | <“layer1 layerN ...”> ]
Mi2.S           M2
Mi2.E
Mi3.X           M3
XYZ.X          “M1 M2 M3”
