Example to Generate Calibre PERC Results with Database Properties

You can add properties to a result during the Calibre PERC run with the ‑property switch for the perc::report_base_result command. Properties in the database can be used to group and sort results for easier analysis.

The following example code uses the ‑property switch for the perc::report_base_result to assign a “Severity” property to each result; the severity is assigned depending on the type of error. This example only attaches one property to the result, but you can add more than one property to allow advanced filtering and sorting on the properties.

See “Grouping Results in Calibre RVE for PERC” and “Result Filters in Calibre RVE for PERC” for information on how to use these properties when viewing results in Calibre RVE for PERC.

The following example code shows how to generate database properties for Calibre PERC results:

SOURCE PATH src.net  
SOURCE PRIMARY "TOP"
SOURCE SYSTEM SPICE

PERC NETLIST SOURCE
PERC REPORT "perc.rep"
MASK SVDB DIRECTORY "svdb" QUERY

LVS POWER  NAME vdd
LVS GROUND NAME vss
LVS RECOGNIZE GATES simple
LVS INJECT LOGIC no

PERC LOAD esd XFORM all INIT esd_init SELECT rule_1
