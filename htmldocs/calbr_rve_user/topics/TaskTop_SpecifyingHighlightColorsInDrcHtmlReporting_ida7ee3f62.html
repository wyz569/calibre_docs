Specifying Highlight Colors in DRC HTML Reporting

You specify highlight colors with the CheckHLLayer key. By default the DRC HTML report uses a the DRC RVE Check Text Override file (CTO file), if present, or check text comments in the results database. If not using a CTO file or check text comments, you can highlight with a different color for each check or use a single color for all checks.

The following values are available for CheckHLLayer; it can be specified in the DRCResults, DRCHighlight, and Common sections.

CheckHLLayer Value

Behavior

RDB

(default)

The highlight color is determined by RVE Highlight Color and RVE Highlight Index statements in the CTO file or check text in the rule file. If such statements are not found, a single highlight layer/color is used.

See the following procedure to use a CTO file for highlight colors.

Colors

A different highlight layer/color is used for each rule check and the highlight layer name is set to the check name. The CTO file or check text is not used.

None

A single highlight layer/color is used for all rule checks and the highlight layer name is set to “rve.” The CTO file or check text is not used.

Procedure

  1. Create the check text override file using the format described in “DRC RVE Check Text Override File (CTO File) Format”. The following DRC RVE rule check comments control highlight color:

    • RVE Highlight Index — Sets the layer index for the highlight layer.

    • RVE Highlight Color — Sets the color for the highlight layer.

      Tip

      Calibre RVE and Calibre Interactive can create a CTO file with RVE Show Layers statements to control layer visibility based on the rule file. You can create a CTO file with one of these methods, then add RVE Highlight Index or RVE Highlight Color statements (you may or may not want to keep the RVE Show Layers statements).

      • “Creating a Check Text Override (CTO) File Using Calibre RVE”
      • “Creating a Check Text Override (CTO) File Using Calibre Interactive nmDRC”
  2. Do one of the following to specify the check text override file:

    • Name the check text override file drc_database.cto, where drc_database is the name of your DRC results database, and the .cto extension indicates a CTO file.

    • Set the environment variable MGC_RVE_CTO_FILE to the pathname to the DRC RVE check text override file.

Examples

Example 1

The following check text override file, named drc_lab.rdb.cto, is used automatically when the results database drc_lab.rdb is processed for a DRC HTML report. 

# CTO file
min_spacing_metal1
RVE Highlight Color: yellow
RVE Show Layers: metal1 metal2

metal1_enclose_cont
RVE Highlight Color: green
RVE Show Layers: metal1 contact

It sets the highlight color and specifies the layers shown for two rule checks (min_spacing_metal1 and meta1_enclose_cont). The key CheckHLLayer is not specified, so the default value of RDB is used.

Example 2

This example specifies CheckHLLayer = Colors in the Common section for use with all highlighting. This is useful if you don’t have a CTO file or if the CTO file does not include RVE Highlight Color or RVE Highlight Index statements. 

; config file for DRC HTML report
[Common]
CheckHLLayer = Colors
