Handling of Cells from Hierarchical Results in Calibre RDB Conversion

By default all results are included when you convert results from an ASCII format Calibre results database (RDB) to a native Place and Route format. You use the ‑cell option to convert only results from a specific cell. The cell must be the working design when doing the conversion within the Place and Route tool environment.

In Synopsys IC Compiler, the cell specified with the ‑cell option is opened automatically if it is not already open. In Cadence Encounter the cell must be the working design or an error occurs.

Note:

If a cell specified with the ‑cell argument does not have results in the results database a message is issued and processing stops.

In hierarchical Calibre nmDRC only one instance of a result is typically included in the results database, so only one instance of a result is converted. See the DRC Results Database statement for options that affect the results database construction. Also see “Hierarchical Calibre nmDRC” and “Calibre nmDRC Results” in the Calibre Verification User’s Manual.

Example 1

The following command converts results from the RDB file drc.results. By default, all results are converted. In Synopsys IC Compiler, the top cell is opened if it is not open already.

icc_shell> Calibre::convert_to_drc_errors drc.results

Example 2

The following command converts results from the RDB file drc.results. Only results from cell block1 are converted.

icc_shell> Calibre::convert_to_drc_errors drc.results -cell block1
