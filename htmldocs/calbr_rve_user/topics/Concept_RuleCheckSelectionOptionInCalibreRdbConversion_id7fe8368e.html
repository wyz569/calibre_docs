Rule Check Selection Option in Calibre RDB Conversion

You can control which rule checks are included when you convert results from an ASCII format Calibre results database (RDB) to a native Place and Route format. By default results from all rule checks are converted. Use the ‑checks option to specify a list of rule checks—only results from the specified rule checks are converted.

The ‑checks option has the following syntax:

-checks { ALL | check_N … }

  • ALL — Results from all rule checks are converted. This is the default if the ‑checks option is not included.

  • check_N … — Only results from the listed rule checks are converted. If a check map file is used and a rule check is listed in the check map file but not in the ‑checks list, results from the rule check are not converted.

    The question mark (?) and asterisk (*) wildcards are allowed.

    When wildcards are used with the “calibre -rve” command they must be escaped to prevent wildcard expansion in the shell environment. Double quotes around the check list and the backslash escape character (\) before the wildcard are required when using wildcards with the “calibre -rve” command at the Linux shell. For example: ‑checks “M2\*” matches all checks starting with M2.

Example 1

In Synopsys IC Compiler, the following command converts only results from rule check M2_width.

icc_shell> Calibre::convert_to_drc_errors drc.results -checks M2_width

Example 2

Suppose the results database drc.results has rule checks M2.1, M2.2, M2.3, M3.1, M4.1, and M5.1

The following command converts results from rule checks M2.1, M2.2, and M2.3:

icc_shell> Calibre::convert_to_drc_errors drc.results -checks M2*

The following command converts results from rule checks M2.1, M3.1, M4.1, and M5.1:

icc_shell> Calibre::convert_to_drc_errors drc.results -checks M?.1

Example 3

Wildcard characters must escaped when using the “calibre -rve” conversion command at the Linux shell. For example, the following command is equivalent to the first example in “Example 2”:

%> calibre -rve -drc drc.results -convert_to icc_drc -output_file cal_drc.rdbout
   -checks “M2\*”

Here the wildcard expression M2* is written as “M2\*”, and the converted output is saved to the file cal_drc.rdbout.
