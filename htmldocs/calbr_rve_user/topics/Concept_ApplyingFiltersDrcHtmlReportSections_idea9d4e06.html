Applying Filters to DRC HTML Report Sections

You can apply filters to select the results that are included in each section of the HTML Report. You can filter by check or cell name, waived and fixed state, and area, for example.

The complete list of filter keys is given in Table 1. The following examples are provided:

  • Example — Filter in a Report Section

  • Example — Default Filter in the Common Section

  • Example —Named Filters

  • Example — Combined Filters with AND and OR Logic

Example — Filter in a Report Section

; ;Show tree for not fixed and not waived results
[Summary - Not Resolved]
ReportType = DRCResultsSummary
GroupBy = Check/Cell
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed

This example adds two filters (WaivedStateFilter and FixedStateFilter) so that only unresolved results are shown in the report.

Example — Default Filter in the Common Section

[Common]
...
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed
...

The filter settings in the Common section are used unless they are overridden with local settings, or settings inherited from another section (see “Section Referencing and Key Value Inheritance”). When section referencing is used, all settings are inherited from the referenced section, not just filter settings.

Example —Named Filters

A named filter allows you to easily reuse a filter setting that is used often.

; ; Named filter section
[Filter NotResolved]
ReportType = DRCFilter
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed

; ;Show tree for not fixed and not waived results
[Summary - Not Resolved]
ReportType = DRCResultsSummary
GroupBy = Check/Cell
; ; use a named filter 
Filter = [Filter NotResolved]

The named filter is defined in a section with ReportType = DRCFilter, then referenced using the Filter keyword.

Example — Combined Filters with AND and OR Logic

Most of the filter keys allow you to specify multiple values separated by commas, and the check and cell filters let you specify AND or OR logic. 

OrCellFilter = bg, inv
OrCheckFilter = *metal1*, *metal2*

This filter selects only results in cells bg or inv, and only results from checks that include “metal1” or “metal2” in the check name. It can be represented as the following:

     cell(bg || inv) && check(*metal* || *metal2*)

You can make use of the And version of the check and cell filters to filter out certain checks or cells, as in this example:

OrCellFilter = bg, inv
AndCheckFilter = ! *dens*, ! *via*

This filter selects results in cells bg or inv and results from checks not equal to *dens* and not equal to *via*. The filter logic can be represented as the following:

     cell(bg || inv) && check( ! *metal* && ! *metal2*)
