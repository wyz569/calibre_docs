SortBy

Key value for Configuration File in DFM HTML Reporting

Supported Report Types/Sections: ChipSummary, TotalSummary

Optional key name that specifies the sort order in which to sort the results.

Syntax

SortBy = sort_key, {Ascending | Descending}

Parameters

sort_key,

A required string that specifies the column for the sort order. The column can be either a metric column or a meta column.

  • Meta Column — Use the following syntax:

    Type, Group, Priority, Rule Name

  • Metric Column — Matches a DFM_METRIC annotation on one or more rules in the database. For example:

    Critical_Area, Single_Via_%, Total_via

    or be one of the following:

    Lambda_ANF, WCA, Yield_Exp, Yield_Poisson
Ascending

A required literal that sorts from low-to-high. This is the default.

Descending

A required literal that sorts from high-to-low.

Description

The priority of sort order which is specified by SortBy key is higher than the order which was specified by Sort key, DFM_METRIC_RANGE, and DFM_SORT_KEY annotations.

Examples

The following configuration file example specifies that the ChipSummary report is sorted by Critical_Area metric column in low-to-high order:

[ChipReports]
ReportType = ChipSummary
SortBy = Critical_Area, Ascending
