Symbol Library File Format

Input to: Calibre RVE Internal Schematic Viewer and Calibre RVE SPICE File Viewer

The Symbol Library files provide custom shapes for symbols. The Symbol Library file is an ASCII file using the NlviewTM® Symlib format from Concept Engineering. Calibre RVE uses the files to display custom symbol shapes in the Internal Schematic Viewer and SPICE File Viewer.

See “Symbol Library Files for Custom Symbol Shapes” for directions on how to load the Symbol Library files and how they are used in the Internal Schematic Viewer.

The custom symbol shape is displayed if the subcircuit name and all port names match between the symbol library file and the netlist. Case-insensitive name matching can be specified in the symbol library file.

Tip

If you are creating Symbol Library files, it is not necessary to create the index entry or update it after a change—Calibre RVE creates the index entry if it is missing and automatically makes corrections. Write permission must be set for the file.

Syntax conventions for the documentation are described in “Syntax Conventions”.

Format

The Symlib file is a Tcl-formatted file, with the following standard Tcl formatting rules:

  • Pound sign (#) character indicates the start of a comment line

  • Backslash (\) is used for line continuation

  • Special characters can be protected within braces ({})

  • Other Tcl formatting rules apply

Content

The file has the following content:

symlib_header
symlib_index
…
symbol_definition
…

Parameters

symlib_header

A required header line with the following format:

symlib version lib_name num_sym name_start ref_length [i]

  • symlib — A required keyword.

  • version — The version. Version 1.3 and earlier versions are supported.

  • lib_name — A required library name. The library name is not used.

  • num_sym — The number of symbols that are defined.

  • name_start — The character location in the symlib_index entries of the start of the symbol name.

  • ref_length — The length in characters of the symlib_index entries.

  • i — An optional keyword that specifies case-insensitive name matching for symbol names, view names, and symbol port names.

symlib_index

One or more index lines, where there is one index line for each symbol definition. The index lines enable fast access to the symbol definitions. The index lines have a fixed length that is specified in the symlib_header line. The index line is padded with trailing whitespace and has the following format:

symref symbol_start_char symbol_name

  • symref — A required keyword.

  • symbol_start_char —The starting character position of the symbol definition within the Symbol Library file.

  • symbol_name — The symbol name. The starting character position of the symbol name in the index entry is fixed and is specified in the symlib_header line.

symbol_definition

One or more symbol definitions with the following format:

symbol symbol_name view DEF element …

  • symbol — A required keyword.

  • symbol_name — The symbol name, which must match a .SUBCKT name. Multiple definitions of the same symbol_name are not supported. For example, the same subcircuit name with different pin definitions is not supported.

  • view — The view name; an asterisk (*) may be used. The Internal Schematic Viewer does not use the view name, however, the entry is required.

  • DEF — A required keyword.

  • element — One or more elements that make up the symbol shape. The following keywords are supported for element: port, permute, attrdsp, pinattrdsp, arc, path, fpath, text, and place.

    See “Symbol Shape Elements” for complete information.

Examples

An example file pllclk.sym is located in the software tree at $CALIBRE_HOME/shared/examples/rve/symlib, along with a netlist, a Calibre RVE configuration file (.rvedb), and shell script.

The following example uses some of the symbol definitions from the example file pllclk.sym.

#       Test comment
#
#
symlib 1.3 noname 4 16 26 
symref      160 DFFRHQS1
symref      603 inv     
symref      887 BUF4    
symref     1128 XOR2S1  
symbol DFFRHQS1 * DEF \
	port Q out -loc 80 -70 60 -70 \
	pinattrdsp @name -cr 58 -70 10 \
	port RN in -loc 30 0 30 -15 \
	pinattrdsp @name -lc 30 -16 10 \
	port CK in -loc -20 -30 0 -30 \
	pinattrdsp @name -cl 7 -30 10 \
	port D in -loc -20 -70 0 -70 \
	pinattrdsp @name -cl 2 -70 10 \
	place latch \
	fpath 0 -85 0 -15 60 -15 60 -85 0 -85 \
	path 0 -35 5 -30 0 -25 \
	attrdsp @cell -uc 30 -54 10 \
	attrdsp @name -ll 35 -90 12
symbol inv * DEF \
	port A in -loc -20 -10 0 -10 \
	pinattrdsp @name -ll -20 -10 10i \
	port Y out -loc 40 -10 28 -10 \
	pinattrdsp @name -lr 40 -10 10i \
	fpath 0 -20 0 0 20 -10 0 -20 \
	fpath 20 -10 28a -10a 20 -10 \
	attrdsp @cell -ll 2 -8 4 \
	attrdsp @name -lc 9 -24 12
symbol BUF4 * DEF \
	port A in -loc -15 0 0 0 \
	pinattrdsp @name -cl -10 -5 8i \
	port Y out -loc 45 0 25 0 \
	pinattrdsp @name -cr 40 -5 8i \
	fpath 0 -12 25 0 0 12 0 -12 \
	attrdsp @name -ll 5 -17 12 \
	attrdsp @cell -cl 5 17 10i
symbol XOR2S1 * DEF \
	port B in -loc -20 10 -1 10 \
	pinattrdsp @name -cl -15 5 8i \
	port Y out -loc 50 0 35 0 \
	pinattrdsp @name -cr 45 -5 8i \
	port A in -loc -20 -10 -1 -10 \
	pinattrdsp @name -cl -15 -15 8i \
	permute all_inputs \
	fpath 35 0 25a -10a 10 -15 0 -15 7a 0a 0 15 10 15 25a 10a 35 0 \
	arc -5 15 2 0 -5 -15 \
	attrdsp @name -ll 0 -22 12 \
	attrdsp @cell -cl 0 22 10i
