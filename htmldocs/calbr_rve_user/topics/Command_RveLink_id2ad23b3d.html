RVE Link

A DRC RVE rule check comment which creates a hyperlink that is displayed in the check text pane of Calibre RVE for the rule check.

Usage

RVE Link: document#anchor_tag TEXT display_text

Parameters

document

A required parameter giving the path to the document to be opened. A space is required between the colon (:) and document.

Some HTML browsers may require “file://” to precede the file path of the document.

#anchor_tag

An optional character (#) and parameter (anchor_tag), where anchor_tag specifies a named destination within document.

TEXT display_text

An optional keyword and parameter specifying the display text for the hyperlink. If this keyword and parameter is not included, the command itself is used for the hyperlink text.

Description

This rule check comment creates a hyperlink which is displayed in the check text pane for the rule check. The specified document is opened when the hyperlink is clicked. If anchor_tag is present, the document is opened at the named destination. By default the document is opened in an HTML browser.

To open PDF files in a PDF viewer instead of an HTML browser, set the environment variable MGC_RVE_PDF_VIEWER to the path of the PDF viewer. See “Viewing PDF Files Using RVE Link” for more information.

Environment variables may be used in the rule check comment. The tool recognizes the end of the environment variable name when it reaches a space, the ‘#’ character, or the path separator character ‘/’.

See “Methods to Add DRC RVE Rule Check Comments” for instructions on specifying the rule check comment. You can add the rule check comment to the rule check in the rule file or specify it in a Check Text Override (CTO) file.

HTML Browser Specification and Compatibility When Using RVE Link

Set the environment variable MGC_RVE_HTML_BROWSER to the path for a browser executable, and that browser is opened when viewing a document with the RVE Link rule check comment. A recent browser version is recommended, as unexpected results may occur in older browsers.

Viewing PDF Files Using RVE Link

The environment variable MGC_RVE_PDF_VIEWER specifies the path to a PDF viewer that RVE Link uses to open PDF files.

If MGC_RVE_PDF_VIEWER is not set, PDF files are opened in an HTML browser and a PDF reader plug-in is required—the settings in your HTML browser control how the PDF document is opened. Some HTML browsers support the anchor tag “#page=num”, where num is a page number in the PDF document, while other browsers do not support this anchor tag and always open the document to the first page. For example, a PDF file rule_descr.pdf can be specified with a page destination as follows in a DRC RVE Check Text Override File (CTO File) Format:

RVE Link: file://$DOC_DIR/rule_descr.pdf#page=10

Troubleshooting — If PDF documents do not open correctly, or do not open to a page destination, check the following:

  • The Adobe Reader plug-in is correctly specified for your HTML browser.

  • Other add-on tools or extensions do not duplicate or conflict with the plug-in specification.

  • HTML browser preferences for PDF documents are set correctly.

  • Shell scripts do not cause conflicts with the Adobe Reader plug-in specification.

Examples

For an example with Calibre RVE for PERC, see “Links to Custom Reports in Calibre RVE for PERC”. The following examples are for Calibre RVE for DRC.

Example 1

The following example shows the RVE Link rule check comment placed in a rule check. The path to the document is preceded by “file://”, as is required by some HTML browser versions. An environment variable DOC_DIR is defined to specify the directory name of the document being referenced.

// rule file
// ...
min_spacing_metal1 {
@ Minimum metal1 spacing = 1.0
@ RVE Link: file://$DOC_DIR/wide_space.html#Procedure TEXT metal rules
ext metal1 < 1.0
}

The named destination can be provided in the HTML document with code similar to this:

<a name="Procedure"></a>

Example 2

The following example shows the RVE Link rule check comment placed in a DRC RVE Check Text Override File (CTO File) Format. An environment variable DOC_DIR is defined to specify the directory name of the document being referenced. (In this example the document path is not preceded by “file://”, but this may be necessary for some HTML browsers.)

# DRC RVE check text override file for XYZ
min_spacing_metal1
RVE HIGHLIGHT COLOR: green
RVE Link: $DOC_DIR/wide_space.html#Procedure TEXT doc:wide metal rules
