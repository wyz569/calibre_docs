Creating a Check Text Override (CTO) File Using Calibre RVE

Select Tools > Create CTO File to have Calibre RVE create a CTO file to control layer visibility for highlight actions. The CTO file is used by Calibre RVE for DRC and DRC HTML reporting. The automatically generated CTO file causes only the layers used in the rule check to be displayed when highlighting a result.

The CTO file includes RVE Show Layers statements based on the rule checks in your DRC rule file. If your design tool uses a layer map file to control the layout export to GDS, the layer map file can be specified so that the layout viewer layer names are used in the CTO file.

Layer and Layer Map statements in the rule file are considered when creating the RVE Show Layers command in the CTO file.

Tip

You can also create a CTO file using the Calibre RVE command line. See “calibre -rve” for the complete syntax.

Restrictions and Limitations

  • Without a layer map file — Layer Map statements in the rule file that have open-ended ranges for the layer or datatype are ignored and a warning is issued. The corresponding Layer statement alone is used to determine the layer number for the RVE Show Layers statement.

  • With a layer map file — See “Layer Map File Usage in Check Text Override (CTO) File Creation”.

The effect of these different restrictions is shown in “Examples” in the topic “Layer Map File Usage in Check Text Override (CTO) File Creation.”

Prerequisites

  • A Calibre nmDRC rule file or both a Calibre PERC LDL CD database and a Calibre PERC rule file.

  • (Optional) A layer map file with one of the formats described in “Layer Map File Usage in Check Text Override (CTO) File Creation”.

    Use the layer map file if you want the RVE Show Layers statements in the CTO file to use layer and purpose pairs or layer names rather than layer numbers.

Procedure

  1. Open Calibre RVE.
  2. (Optional) If using a Calibre PERC LDL CD results database, load the file and select its results tab.
  3. Select Tools > Create CTO File.
  4. Fill in the entries in the Create CTO File dialog box.

    Select Groups or Layers to create a CTO file with those categories. You can select both options.

    The layer map file is optional; see the Prerequisites section.

  5. Click OK.
  6. Examine the resulting CTO file. If you received any warnings, examine the layers and rule checks affected by the warnings and correct the CTO file if necessary.

    If you are using a layer map file and there are Layer statements in your rule file without a corresponding Layer Map statement, all layer:purpose pairs mapped to the corresponding GDS layer in the layer map file are used in the RVE Show Layers statement for that layer.

Results

A CTO file with RVE Show Layers statements for each rule check is created. The format of the layer parameter in the RVE Show Layers statement is as follows:

  • Layer map file is used — The layer is given as layer:purpose.

  • Layer map file not used — The layer is given as layer, where layer is the layer number, or layer.datatype if Layer Map statements in the rule file provide a datatype.

Results with and without a layer map file are shown in the “Examples” section of “Layer Map File Usage in Check Text Override (CTO) File Creation”. The CTO file header contains the rule file path and the layer map path, if provided.

If a results database with the name results_rdb is open in Calibre RVE and the created CTO file is named results_rdb.cto and in the same directory, then the results database is reloaded and the CTO file is applied.
