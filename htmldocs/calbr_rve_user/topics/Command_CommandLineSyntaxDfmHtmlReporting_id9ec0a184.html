Command Line Syntax for DFM HTML Reporting

You create the DFM HTML report from the command line using “calibre -rve -dfm” with the appropriate options.

Usage

calibre -rve -dfm {dfmdb_path | -input db_to_layout_map_file } -report config_file
[-display display] -outputdir output_dir  [‑cmp]

Arguments

-rve -dfm

The switches -rve and -dfm are required.

dfmdb_path

A required value that specifies the path to the DFM database.

The dfmdb_path and -input db_to_layout_map_file are mutually exclusive. You must specify one or the other, but not both.

-input db_to_layout_map_file

A required switch and value set that specifies the path to the DFM HTML Report Database Map File.

The dfmdb_path and -input db_to_layout_map_file are mutually exclusive. You must specify one or the other, but not both.

-report config_file

A required keyword and value set that specifies the path to the configuration file.

-display display

An optional keyword and value set that specifies the display to run batch reporting on. Calibre opens a number of GUI windows when running batch reporting. By default, HTML reporting is run on an Xvfb (virtual framebuffer X) server so that none of these windows are visible. The display argument allows you to disable the Xvfb server and specify an alternate display to run batch reporting on.

-outputdir output_dir

A required keyword and value set that specifies the directory where the output files are written.

[‑cmp]

Optional switch used to indicate that the database was produced by a Calibre CMPAnalyzer run.
