Multithreaded Parallel Processing for Calibre RVE Operations (‑turbo Option)

Multithreaded parallel processing is supported for some operations performed in Calibre RVE, such as Interactive Short Isolation in Calibre RVE for LVS and DRC operations executed from a YieldServer script or the Layer Browser tab in Calibre RVE for DFM.

You specify multithreaded parallel processing with one of the following methods:

  • “‑turbo num_proc” command line option

    Where num_proc is an optional parameter specifying the number of CPUs to use for multithreaded processing.

  • “Run multi-threaded RVE” checkbox

    The checkbox “Run multi-threaded RVE” is available when you start Calibre RVE from a supported design tool and select a database type of DFM or LVS.

Automatic ‑turbo setting — When Calibre RVE is started from Calibre Interactive DFM or Calibre Interactive nmLVS, the settings on the Run Control pane are used to set the ‑turbo option for Calibre RVE processing. This behavior is described in described in Table 1.

Table 1. Automatic Calibre RVE -turbo Option

Run Control Settings in Calibre Interactive nmLVS or Calibre  Interactive DFM

Automatic Calibre RVE ‑turbo Option

Single-Threaded

none

Multi-Threaded

‑turbo num_proc

where num_proc is the number of CPUs.

Distributed (MTflex)

‑turbo

 

Disable automatic ‑turbo setting — You can disable the automatic inclusion of the ‑turbo option in Calibre RVE by setting the environment variable MGC_RVE_DISABLE_TURBO to any value. The environment variable applies to starting Calibre RVE from Calibre Interactive or from a layout editor.
