DFM HTML Reporting and CFA Waiver Flow

DFM HTML reporting can with used with the DFM automatic waiver flow.

For DFM automatic waiver flow results reporting, use the key names to differentiate the following types of errors:

  • Errors marked to be waived

  • Errors which are already waived

You can find a listing of these key words in Table 1.

For example, the following report section can be used to report all errors that are marked to be waived by user some_user:

[MarkedToBeWaivedErrorTable]
ReportType = ByErrorResults
BinsToInclude = 1_IMPACT
MetricsToInclude = Metric
RulesToInclude = PO.EN.1_dfm.a
WaivedStateFilter = Waived
GroupBy = Check

The following example reports errors for rule PO.EN.1_dfm.a that were waived during DFM automatic waiver flow run:

[WaivedErrorTable]
ReportType = ByErrorResults
BinsToInclude = *
MetricsToInclude = *
WaiveStatesToInclude = WaivedErrorResults
RulesToInclude = PO.EN.1_dfm.a
GroupBy = Check
