Shape Output Options in Calibre RDB Conversion (‑offset, ‑min_feature, and ‑mag)

You can control conversion options such as x- and y-offset, magnification, and minimum feature size when you convert results from an ASCII format Calibre results database (RDB) to a native Place and Route format.

The following options controlling shape output are available for the Calibre RDB conversion commands:

Table 1. Shape Output Options for Calibre RDB Conversion

Option

Description

‑offset x_offset y_offset

Specifies an x_offset and y_offset for the converted result, where the offset is given in user units. The offset is applied after a ‑mag value if both are specified.

 

Some physical verification flows apply an offset to the design before verification. Use the -offset option to correct for the offset so that the results are displayed correctly in the original design.

‑mag mag_value

Specifies a magnification value to apply to the converted results. Use this option if the precision of the input RDB is not the same as the precision of the layout database. A mag_value of 2 magnifies the results by 2; a mag_value of 0.5 shrinks the results.

 

If both ‑offset and ‑mag are specified, the magnification is applied first.

‑min_feature min_size

 

(Valid only for conversion to Synopsys IC Compiler route guide and Cadence Encounter route blockage formats)

Specifies the minimum width of the converted result shape in user units. For example, a single edge result is converted to a rectangle of width min_size. A default of 0.002 user units is used if this option is not specified.

 

Use this option to change the minimum feature size when converting to route guides. This is useful if the minimum width for route guides in the design tool is greater than 0.002 user units. For conversion to Cadence Encounter route blockages, set min_size to the grid size or larger to prevent conversion errors.

 

Note: The tool uses a fixed minimum feature size of 0.002 user units when converting to DRC error markers.
