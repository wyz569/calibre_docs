Layer Map File Usage in Check Text Override (CTO) File Creation

When creating a CTO file using Calibre RVE or Calibre Interactive, you can specify a layer map file if you want the RVE Show Layers statements in the CTO file to use layer and purpose pairs or layer names rather than layer numbers. You can use a Calibre DESIGNrev layerprops file or a third-party layer map file. Some restrictions apply when using a layer map file. This topic includes examples of CTO files created with and without a layer map file.

Format

The following formats are supported for the layer map file:

  • Calibre DESIGNrev layerprops file — See “layerprops File Format” in the Calibre DESIGNrev Layout Viewer User’s Manual

  • Third-party layer map file format:

    • One entry per line with the following format: 

      viewer_layer_name viewer_purpose_name GDS_layer_number GDS_datatype

    • ASCII text file.

    • Comment lines start with #.

The Cadence Innovus layer map file (streamOut.map) is not supported.

Restrictions and Limitations

If a layer map file is used, the following must be true in order for the layer to be included in the RVE Show Layers statement in the CTO file:

  • Each layer must be listed in the layer map file.

  • Layer Map statements in the rule file must have closed ranges for the layer and datatype.

If any of the above conditions is not met when using a layer map file, the corresponding layer is not used in the RVE Show Layers statement.

When using a layer map file, if there are Layer statements in your rule file without a corresponding Layer Map statement, all layer:purpose pairs mapped to the corresponding GDS layer in the layer map file are used in the RVE Show Layers statement for that layer.

See “Creating a Check Text Override (CTO) File from the Rule File” for information on creating the CTO file.
