Using Design Fix Suggestions for LVS Comparison Results

Design Fix Suggestions (if enabled) provide easy to understand suggestions for fixing certain types of errors. Many of the fix suggestions also have a Visualize Fix option that displays the fix suggestion in a schematic viewer window.

Prerequisites

  • An SVDB containing LVS Comparison Results and Fix Suggestions. See “Rule File Requirements for Examining Results with Calibre RVE for LVS”. Fix Suggestions is enabled with the following SVRF statement:

    LVS REPORT OPTION FX

  • The layout database for your LVS run.

  • A layout viewer with an interface to Calibre RVE. See “Connection to a Design Tool”.

Procedure

  1. Open your layout in Calibre DESIGNrev or in another supported layout viewer.
  2. Start Calibre RVE from your layout viewer, as described in the procedure “Invoking Calibre RVE From a Layout or Schematic Viewer” and open your SVDB.
  3. View the Extraction Results tab. Your Calibre nmLVS run should have no connectivity extraction errors or warnings. Extraction errors and warnings may cause LVS comparison errors, and should be corrected.
  4. Click the Comparison Results tab.
  5. Choose View > Tree Options > View by Fix Type to sort the comparison results by fix type.
     

    You can also choose View by Fix Type from the LVS Toolbar, as shown in the figure.

    Note: You can also view Fix Suggestions when the tree is sorted by View by Layout Cell.



  6. Choose one of the fix types, expand the tree view until you get to a specific fix suggestion for a cell, and select that fix suggestion. Look at the report in the detailed view to see the error and the fix suggestion.

    In Figure 1 a cell has been expanded in the tree view and an open circuit error is listed. The last line of the detailed result listing has a suggestion for fixing the problem.

    Figure 1. Design Fix Suggestions

  7. View the Fix Suggestion by right-clicking in the result and choosing Visualize Fix, if available. Fix Visualization is available for many fix types; if it is not available, skip to Step 8.

    Fix Visualization opens three schematic windows, as shown in Figure 2 (the background color of the center schematic has been adjusted to improve the contrast).

    Figure 2. Fix Visualization Schematic Windows

    The Fix Visualization view in the center of Figure 2 contains layout instances with the connectivity of the source schematic. You can highlight elements from the Fix Visualization view from a right-click menu, and the view responds to highlight events from other windows. For more details, see “Fix Visualization View”.

  8. If Visualize Fix is not available, highlight the affected design elements. From the detailed view, right-click or left-click the design elements in blue or red text:

    • Left-click — Opens the Internal Schematic Viewer to that element in both the layout and source and highlights the element in your layout viewer. (See Figure 1).

    • Right-click — Displays a context-sensitive menu with options for highlighting the element or showing detailed information about the element in the Info Pane.

    Also see “Cross-Probing in Calibre RVE for LVS”.

  9. Fix the problem in the layout or schematic, and go on to the next Fix Suggestion.

    Note: You cannot mark a Fix Suggestion as Fixed, and Calibre RVE does not maintain a correspondence between Fix Suggestions and Discrepancies.

  10. Task Validation — After you correct as many errors as you can, run Calibre nmLVS again. Then choose File > Reload Database in Calibre RVE and review the Comparison Results tab. The discrepancy count should be reduced if the fixes were successful.
