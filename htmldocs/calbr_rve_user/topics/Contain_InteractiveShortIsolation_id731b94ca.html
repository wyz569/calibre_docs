Interactive Short Isolation

Shorts are often difficult to isolate in a design since they can have many different causes and may cross cell boundaries. Use this method to interactively isolate a short. After you isolate the short, you can use Calibre RVE for LVS to verify the short repair before you edit the design database.

This method applies to shorts on texted nets; for untexted nets, see “Isolating Shorts Between Untexted Nets”.

The short isolation process is broken into the following sub-tasks:

  • “Viewing the Complete Short”

  • “Viewing Details of Short Polygons”

  • “Assigning Net Labels to the Polygons in a Short”

  • “Selecting Polygons to Remove from the Short Path”

  • “Verifying Short Repairs”

  • “Saving and Viewing the Removed Polygons from Interactive Short Isolation”

Requirements

  • A Calibre nmLVS/Calibre nmLVS‑H license in addition to a Calibre RVE license.

  • An SVDB that contains a short isolation database created by the LVS Isolate Shorts YES statement in your rule file. The following command and keywords are recommended:

    LVS Isolate Shorts YES BY CELL BY LAYER

    The BY CELL keyword is required in order for Calibre RVE to virtually remove a polygon from identical cells at the same level in the hierarchy when you mark a polygon for removal in a lower level cell.

  • The SI keyword in the Mask SVDB Directory statement in your rule file, as shown here:

    Mask SVDB Directory svdb <options> SI

    where svdb is the name of your SVDB directory.

    The SI keyword enables interactive short verification (the Verify Short right-click menu selection) and virtual polygon removal (the REMOVE selection for net assignment); see “Verifying Short Repairs” for more information. You can view and highlight a short without the SI keyword being used.

    All figures in this section assume the SI keyword is included in the Mask SVDB Directory statement.

  • The layout database for your short isolation run.

  • A layout viewer with an interface to Calibre RVE. See “Connection to a Design Tool”.

Note:

In the shorts database created by Calibre nmLVS, 2×2 dbu squares are added to the polygonal paths between shorted text objects. The text object base points intersect these squares. This is to ensure text objects in the shorts database are not isolated from short paths. The squares are on the layers that the text objects are attached to, and the squares abut or overlap other shapes in a short path. See “Short Isolation” in the Calibre Verification User’s Manual for other details about short isolation

These 2×2 dbu squares are seen as the first and last segments in the short path displayed by Calibre RVE, but do not correspond to actual layout geometries.
