DRC Rule Check Comments for Calibre RVE

You can use rule check comments to provide additional control over the Calibre RVE display and highlighting actions. The rule check comments are placed in your DRC Results database or a check text override (CTO) file and interpreted by Calibre RVE for DRC.

The following table lists the available rule check comments.

Table 1. DRC Rule Check Comments in Calibre RVE

Rule Check Comment

Description

RVE Highlight Index

Specifies the highlight layer index used for the rule check.

RVE Highlight Color

Specifies the highlight layer color used for the rule check in supported layout viewers.

RVE Highlight Text Size

Specifies the text size to use when adding text objects during highlight operations

RVE Link

Creates a hyperlink that is displayed in the check text pane for the rule check.

RVE Priority

Sets the display priority for a rule check.

RVE Show Layers

Specifies the layers that are shown in supported layout editors when highlighting a result from the rule check.

RVE Show Properties

Specifies the properties to display during a highlight operation.

RDB

Can only be specified in the rule file.

Specifies a side RDB (auxiliary database) to be displayed as a link in the check text pane for a rule check.

Other Statements

RVE Category

The RVE Category statement is used in CTO files to enable filtering by category; see “DRC RVE Check Text Override File (CTO File) Format”.

There are two ways to add DRC RVE rule check comments: in the rule file or in a Check Text Override File (CTO File).
