Fix Visualization View

Access with: Right-click a Fix Suggestion in tree view and choose Visualize Fix. Three schematic windows open: layout, Fix Visualization, and source.

The Fix Visualization View displays a schematic visualization of a Design Fix Suggestion. The visualization can help in fixing some LVS comparison errors.

Description

A Fix Visualization for a cross connect error is shown in Figure 1 (the background color of the center schematic has been adjusted to improve the contrast). The view shows layout instances with the connectivity of the source schematic. You can highlight elements from the Fix Visualization view from a right-click menu, and the view responds to highlight events from other windows. The Fix Visualization View uses the Internal Schematic Viewer.

Figure 1. Fix Visualization View

Fix Suggestions can be generalized into the three categories shown in the following table.

Objects

Table 1. Fix Visualization Categories and Object Naming

Category

Fix Method

Object Naming

Short circuit

Split layout net into two nets.

The two new nets are named source_net_<X> and source_net_<Y>, where <X> and <Y> are the names of the two source nets that should be created from the shorted layout net.

The right-click menu for both of the new nets contains the layout net involved.

Open circuit

Join two layout nets.

The new net is named source_net_<X>, where <X> is the name of the source net that should be emulated. See Figure 2.

The right-click menu contains the two layout nets involved.

Connectivity error

Correct connectivity, but nets are not created or removed.

Nets are named source_net_<X>, where <X> is the name of the source net.

The right-click menu contains the layout net.

Usage Notes

The following limitations exist:

  • Visualize Fix is not available for all Fix Suggestions. In particular, databases that have had logic injected may produce one of the following popup messages:

    • “No Fix Visualizer Available” from right-click in Fix Suggestion

    • “Problem encountered while trying to visualize this fix suggestion” after choosing Visualize Fix

  • Calibre nmLVS sometimes flattens the hierarchy; this can result in the fix being applied to only the lowest level of the hierarchy displayed but not propagated up the hierarchy.
