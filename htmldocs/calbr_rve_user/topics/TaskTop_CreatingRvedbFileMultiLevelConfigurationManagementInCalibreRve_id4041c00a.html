Creating a .rvedb File for Multi-Level Configuration Management in Calibre RVE

You can create a .rvedb configuration file to open Calibre RVE with specific option settings. The location of the .rvedb file is specified with the MGC_RVEDB_DIR environment variable. For example, you can create a .rvedb file with settings for layout magnification and rotation and use that configuration file for projects that require these settings. Another possibility is option settings for waiver export from Calibre RVE for Calibre Auto-Waivers.

The .rvedb file created with the following process can also be used for Multi-Level Configuration Management in Calibre RVE and to set options for the Calibre RVE SPICE File Viewer and the Scripting Interface to the Calibre RVE SPICE File Viewer.

Procedure

  1. Set the MGC_RVEDB_DIR environment variable to point to the directory where you want the .rvedb file to be saved.

    For example, to create the .rvedb file in the $PROJ/rve_opts directory:

    % setenv MGC_RVEDB_DIR $PROJ/rve_opts
  2. Delete or rename any .rvedb file already in the MGC_RVEDB_DIR directory so that default settings are used when you open Calibre RVE.
  3. Open Calibre RVE.
  4. Set options as desired in the Calibre RVE session. Be aware that database-specific options for Calibre RVE for DRC and Calibre RVE for PERC are not saved to the .rvedb file.
  5. Exit Calibre RVE. The .rvedb file is automatically saved to the directory specified by MGC_RVEDB_DIR.
  6. Make the .rvedb file read-only.
  7. Define MGC_RVEDB_DIR as desired. See the “Examples” section for possible uses of the .rvedb file and MGC_RVEDB_DIR. You must include a writable .rvedb file location in the definition of MGC_RVEDB_DIR in order for changed settings to be saved.

Examples

Example 1: Project-Level and Personal Settings

In order to apply project-level settings each time Calibre RVE is opened and allow for personal settings to be saved to your home directory, define MGC_RVEDB_DIR as follows: 

% setenv MGC_RVEDB_DIR {$PROJ/rve_opts}:{$HOME}

Assume the file $PROJ/rve_opts/.rvedb is read-only. When you exit Calibre RVE, settings that were changed during the Calibre RVE session are saved to $HOME/.rvedb. When you open Calibre RVE the next time, the settings from $PROJ/rve_opts/.rvedb and $HOME/.rvedb are applied, with settings in $PROJ/rve_opts/.rvedb taking precedence.

Example 2: Project-Level, Application, and Personal Settings

This example uses .rvedb files in three locations: project-level defaults, settings for waiver export, and the $HOME directory. Settings in the leftmost directory take precedence if there is a conflict. 

% setenv MGC_RVEDB_DIR {$PROJ/rve_opts}:{$CALIBRE/RVEWAIVER}:{$HOME}
