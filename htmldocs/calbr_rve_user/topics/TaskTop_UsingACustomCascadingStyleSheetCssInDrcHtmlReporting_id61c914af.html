Using a Custom Cascading Style Sheet (CSS) in DRC HTML Reporting

You can specify a custom Cascading Style Sheet (CSS) file for the DRC HTML report. The CssFilePath key name specifies the path to the custom CSS file.

Note:

If CssFilePath is not specified, the DRC HTML report tool creates a default CSS file, default.css, in the output directory. The default CSS file contains basic parameters that control the background color and table style in the DRC HTML report.

A custom CSS file is specified in the Common section of the configuration file with the CssFilePath key name set to the full path to the custom CSS file. You can also specify the CSS file when creating a HTML report using Calibre RVE; see “Creating a DRC HTML Report from Calibre RVE for DRC”.

The following procedure describes how to create a custom CSS file based on default.css.

Prerequisites

  • “Requirements for DRC HTML Reporting”

Procedure

  1. Create a DRC HTML report without setting CssFilePath in the configuration file. For examples, see “Automatically Creating the DRC HTML Report” and “Creating a Basic Report in DRC HTML Reporting”.
  2. Find the default CSS file, default.css, in the report directory and copy it to a filename of your choice, for example, custom1.css. You may want to place the custom CSS file in a central location.
  3. Edit your custom CSS file as desired.

    For an example used with an HTML Customization file, see “HTML Customization File for DRC HTML Reports”.

  4. Do one of the following, depending on whether you are creating a custom DRC HTML report configuration file or creating the HTML report using Calibre RVE for DRC.

    • Custom configuration file — Edit your DRC HTML Report Configuration File and add the CssFilePath key to the Common section. Specify the absolute path to the custom CSS file. For example:

      [Common]
WorldView = 1
CustomizationFilePath = custom.htm
CssFilePath = /project/DRC_HTML_setup/custom1.css

      In addition, specify LinkCss = 0 in the Common section if you want the specified CSS file to be copied to the output directory. The default behavior (LinkCss = 1) is to not copy the file. If the HTML report will be viewed on a Windows platform, specify LinkCss=0, because some browsers for Windows are unable to resolve the path to the CSS file when the file is linked to.

    • Calibre RVE for DRC — Choose Tools > Create HTML Report in Calibre RVE, check “Stylization File”, and specify the CSS file.
