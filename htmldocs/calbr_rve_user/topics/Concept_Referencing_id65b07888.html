Referencing

Each optional section in the configuration file has a type specified by the key name.

Each type corresponds to an element in the DFM Report Card, such as a tab, histogram, or color map.

  • ReportType

These elements use a single group, rule, cell, or window as input and display a corresponding set of rules, cells, or windows as output.

The Cell Summary tab in the DFM Report Card displays a table of cells for each rule you select in the dropdown list. Thus, its input is a rule, and its output is a list of cells.



Similarly, the cell Drill Down tab displays a list of rules for a particular cell. Therefore, its input is a cell, and its output is a list of rules.



The DFM HTML reporting tool allows you to generate separate elements (contents of tabs, histograms, or colormaps) for each possible input value. To do this, use the output of one section as the input to another section in the configuration file (note that the output of the referenced section must match the input of the other section). In the previous example, the DFM Report Card can display a list of rules in the Drill Down tab for each cell shown in the Cell Summary tab. The DFM HTML reporting tool can generate a list of Drill Down reports for each cell in a similar fashion by using referencing:



This configuration file generates a separate report containing the contents of the cell Drill Down tab for each input cell. The list of input cells is obtained from the output of the Cell Summary tab. The reports generated from this configuration file are equivalent to the reports you could manually generate by selecting Drill Down for each cell in the Cell Summary tab in the DFM Report Card.

Note that key names can be recursively referenced back to the Common section. The Common section is referenced internally, so it is not necessary to reference it explicitly.

References can be either direct or indirect. Direct references match the output type of the referenced section to the value of the corresponding key name, while indirect references do not. All references in the previous example are direct references.

Sections cannot be referenced in such a way that cyclic dependencies are created.
