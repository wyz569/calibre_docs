Overview of Highlighting in Cadence Virtuoso with Calibre RVE

Calibre RVE allows you to highlight various layout elements reported in Calibre results databases. Calibre RVE packages the highlight data in a file (Setup > Design Tools, “Highlight data file” setting) and sends a message to the SKILL Interface over the socket directing it to read the file. The SKILL Interface interprets the data from the file and performs the highlighting.

If a hierarchical net or device is highlighted, the schematic corresponding to the containing cell is automatically opened and zoomed to the net or device. For example, if you highlight net X0/net1 in cell XOR and cell XOR is open in window 1, Calibre RVE descends into the cell corresponding to X0, highlights net1, and zooms to it. If a subsequent highlight request is for net net2 in cell XOR, Calibre RVE pops back up to the XOR cell in window 1 and highlights and zooms to the net. Calibre RVE displays a message in the CIW if a schematic object cannot be successfully highlighted.

When Calibre RVE requests highlighting in a particular cell, the SKILL Interface checks to see if the cell is already open in a window. If not, it checks the colon-delimited cell libraries path specified in the Setup RVE dialog box for the cell (choose Calibre > Setup > RVE, then see the “Libraries RVE should look in for” entries). If it cannot find the cell in the path, it prompts you with the open cell dialog box. When you choose a cell in that dialog box from a library not in the path, that library is automatically added to the cell libraries path. One consequence of this is that if a cell is in two libraries, the cell from the first library in the path is always picked. If you want to switch designs in the midst of a Calibre RVE session, you may want to clear or reset the Calibre RVE libraries path explicitly to avoid the SKILL Interface picking a cell from the wrong library.

Highlights to Pcell instances are made to the top cell regardless of the Calibre RVE setting for “Highlight in cell context.”

See the following topics for information about controlling the cell and window that Calibre RVE highlights to:

  • “Map Calibre RVE Cells to Different Cadence Library for Highlighting”

If the environment variable or SKILL variable MGC_RVE_RAISE_WINDOW is set to any value, the window the highlight is made in is raised to the front if it is not already at the front. In SKILL, the value must not be nil.
