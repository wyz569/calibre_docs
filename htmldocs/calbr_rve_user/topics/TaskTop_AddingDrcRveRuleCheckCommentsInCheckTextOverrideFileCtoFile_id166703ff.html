Adding DRC RVE Rule Check Comments in a Check Text Override File (CTO File)

You can specify DRC RVE rule check comments in a CTO file. This method allows you to specify rule check comments for a Calibre nmDRC results database without editing the rule file or running Calibre nmDRC again.

Procedure

  1. Create a DRC RVE Check Text Override File (CTO File) Format. The following example file specifies a DRC RVE check text comment for the min_ext_diff_poly rule:
    # DRC RVE check text override file for XYZ
#
min_ext_diff_poly 
RVE Highlight Color: blue

    The RDB rule check comment cannot be specified in a Check Text Override file.

  2. To have the CTO file load automatically when you open the results database, save the check text override file with a name of drc_database.cto, where drc_database is the name of your DRC results database, and place it in the same directory as the DRC results database file.
  3. Open drc_database in Calibre RVE for DRC.

    Calibre RVE automatically loads the file drc_database.cto when opens drc_database.

    You can also specify the check text override file with an environment variable or load it manually as described in “Specifying and Loading the DRC RVE Check Text Override File”. These methods are useful if you do not use the default filename of drc_database.cto.
