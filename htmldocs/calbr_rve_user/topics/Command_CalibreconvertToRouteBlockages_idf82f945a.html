Calibre::convert_to_route_blockages

Converts results from an ASCII format Calibre results database (RDB) to Cadence Encounter route blockages. The command is executed from the Cadence Encounter command prompt.

Usage

Calibre::convert_to_route_blockages input_RDB
-check_map_file map_file
[ -min_feature min_size]
[ -offset x_offset y_offset ]
[ -mag mag_value]
[ -cell cell ]
[ -checks { ALL | check_N … }]
[ -side_rdb {YES | NO} ]
[ -output_file rdb_blk]
[-no_transcript]

Arguments

input_RDB

Required parameter specifying the Calibre results database to be converted. See “Requirements for Converting Calibre RDBs to Native Place and Route Format” for the RDB formats that are allowed.

-check_map_file map_file

Required keyword and parameter specifying the check map file. The check map file maps each Calibre rule check to one or more target layers for the route blockage. See “Check Map File Format for Calibre RDB Conversion” for details.

-min_feature min_size

Optional keyword and parameter specifying the minimum width of the converted route blockage in user units. A default of 0.002 user units is used if this option is not specified.

-offset x_offset y_offset

Optional keyword and parameters specifying an x- and y-offset. Values are given in user units. If both -offset and -mag are specified, the magnification is applied first.

-mag mag_value

Optional keyword and parameter specifying the magnification of the converted results.

Use this option if the precision of the input RDB is not the same as the precision of the layout database. If both -offset and -mag are specified, the magnification is applied first.

-cell cell

Optional keyword and parameter specifying to only convert results from cell. If this keyword is not present all results are converted.

-checks { ALL | check_N … }

Optional keyword and parameter specifying the rule check results to convert.

  • ALL — Results from all rule checks are converted. This is the default if the ‑checks option is not included.

  • check_N … — Only results from the listed rule checks are converted. The question mark (?) and asterisk (*) wildcards are allowed.

    If a list of rule checks is provided, only those rule checks are converted. Results from a rule check listed in the check map file but not in the ‑checks list are not converted.

-side_rdb {YES | NO}

Optional keyword and parameter specifying whether to include the results from side (auxiliary) RDBs in the conversion. Side RDB results are not included by default. See “Handling of Side RDBs in Calibre RDB Conversion” for more information.

-output_file  rdb_blk

Optional keyword and parameter specifying that the converted data is saved to the file rdb_blk. The converted data is not imported to the current Cadence Encounter session. Use the Calibre::import_route_blockages command to import the converted route blockages.

If this keyword is not specified the input_RDB is converted and the route blockage shapes are automatically imported to the Cadence Encounter session.

-no_transcript

Optional keyword that turns off transcript output.

Return Values

Returns 1 for success or 0 for failure.

Description

This command converts result shapes from Calibre rule checks to Cadence Encounter route blockages. You must have an existing ASCII format Calibre results database (RDB); see “Requirements for Converting Calibre RDBs to Native Place and Route Format”. Consult your CAD group for advice on which Calibre rule checks produce result shapes that can be used as route blockages.

The design or cell containing the results you want to convert must be the working design in your Cadence Encounter session or an error occurs.

The required check map file specifies the target layer(s) per rule check for the route blockages; see “Check Map File Format for Calibre RDB Conversion”. Route blockages are created with the ‑layer option to the createRoutBlk command. The target layer(s) in the check map file provide the layer parameter(s) for the ‑layer option.

See the topics in “Configuring Calibre RDB Conversion to Native Place and Route Format” for additional information on the conversion options.

A transcript with information such as a the cell list, rule check list, precision, output file, and execution details is included by default. The transcript can be turned off with the ‑no_transcript option; errors, warnings, and notes are still output.

The route blockages are named with the following format:

mgc_rve_check_<cell>_<check>_<layers>_<result-num>

where

  • If the route blockage is created on multiple layers each layer is separated by an underscore, as follows:

    <layer1_layer2_ ... layerN>

  • The <result-num> result numbers start at zero.

For example, the route blockage in cell top for check m4width created on layer M4 has the following name for the first result:

mgc_rve_check_top_m4width_M4_0

On successful completion a message is written to the transcript:

// RDB written as PR input file <file>

The route blockages are automatically imported into the Cadence Encounter session unless you specify the ‑output_file keyword to save the results to file. Turn on visibility of blockage type “Routing Blkg” in order to view the route blockages. If you use the ‑output_file option, use the Calibre::import_route_blockages command to import the converted route blockages.

Examples

Use the following command to convert results from rule check m3_pattern in cell BLOCK2. The check map file is named map and has a target layer of M3 for check m3_pattern.

encounter > Calibre::convert_to_route_blockages drc.results -check_map_file map
‑cell BLOCK2 -checks m3_pattern

The route blockages are created on layer M3 and automatically imported into the Cadence Encounter session. Turn on visibility of blockage type “Routing Blkg” in order to view the route blockages. You can view blockage properties in the Attribute Editor—select the blockage and right-click for a menu.

See “Importing Calibre Results as Cadence Encounter Route Blockages” for complete instructions on converting Calibre results to route blockages, including an image of a converted route blockage.
