Multi-Level Configuration Management in Calibre RVE

You can define multiple locations for the .rvedb configuration file with the MGC_RVEDB_DIR environment variable. This allows you to apply both global and project level settings, for example, and define the order of precedence for applying options.

MGC_RVEDB_DIR Usage

MGC_RVEDB_DIR is defined as a list of colon-separated directories, with the first (leftmost) directory having precedence. Updated settings are saved to the first writable .rvedb location in the directory list.

The usage for multi-level configuration management is best described with an example. Suppose MGC_RVEDB_DIR is defined as follows:

setenv MGC_RVEDB_DIR /site_dir:/design_dir:$HOME

where /site_dir, /design_dir, and $HOME are directories. The .rvedb file in /site_dir is not writable, while the .rvedb files in /design_dir and $HOME are writable.

Options are read and saved as follows:

  • Applying settings and precedence — The .rvedb files found in the directories listed in MGC_RVEDB_DIR are read when Calibre RVE is invoked, with settings in the leftmost directory taking precedence. If an option is not specified, the default is used.

    In the example, if option X is set in all three directories, the setting in /site_dir is applied. If a setting exists in /design_dir but not in /site_dir, the setting in /design_dir is used.

    Filters in all directories are applied, with the ones from the leftmost directories being appended to those in the rightmost directories. If there is a naming conflict, the filter in the leftmost directory (/site_dir) takes precedence.

  • Saving settings — Options are saved to the first (leftmost) writable .rvedb location in the directory list. Options are saved as follows:

    • Options that are read from the writable .rvedb file are saved.

    • Options that were modified during the Calibre RVE session are saved.

    For example, suppose option Y is defined in $HOME to a non-default value. If option Y is changed to the default value, then the default value is saved to the .rvedb file in ⁄design_dir.

    For another example, suppose option Z is read from /site_dir and not modified during the session. The setting is not saved to the writable .rvedb file.

    Because project-level .rvedb files are not supposed to be writable, project-specific settings may not be saved to the project-level .rvedb. When switching projects, settings from other projects may then be shown.
