Specifying Layer Visibility in DRC HTML Reporting

You can specify which design layers are displayed when a result highlight is shown in the DRC HTML Report. This is done using a DRC RVE Check Text Override File (CTO file) with RVE Show Layers check text comments to specify the layer visibility. You can create the CTO file automatically or create a custom CTO file. You can also use the key HideNonHLLayers to hide all design layers and only show highlight layers; this is useful for overview snapshots.

The following procedure describes how to specify layer visibility with a CTO file and RVE Show Layers check text comments. To hide all non-highlight layers, see the HideNonHLLayers keyword.

Procedure

  1. Create the DRC RVE Check Text Override File (CTO File) Format with RVE Show Layers statements using one of the following methods.
    Table 1. Create CTO File for Layer Visibility in DRC HTML Report

    Method

    Description

    Create the CTO file automatically as part of the DRC HTML report creation.

    Specify CreateCTOFile in the common section of the DRC HTML Report Configuration File Format.

    Example:

       [Common]
   CreateCTOFile = 1

    This creates a CTO file with RVE Show Layers statements based on the rule file; see CreateCTOFile for further information.

    Create the CTO file using Calibre RVE.

    See “Creating a Check Text Override (CTO) File Using Calibre RVE”.

    Create the CTO file using Calibre Interactive.

    See “Creating a Check Text Override (CTO) File Using Calibre Interactive nmDRC”.

    Construct your own CTO file with RVE Show Layers statements.

    See “Methods to Add DRC RVE Rule Check Comments”.

    This method allows you to specify additional RVE check text comments for each rule check in the CTO file.

  2. If you constructed your own CTO file, do one of the following to specify that the check text override file is used for DRC HTML report creation:

    • Name the check text override file drc_database.cto, where drc_database is the name of your DRC results database, and the .cto extension indicates a CTO file.

    • Set the environment variable MGC_RVE_CTO_FILE to the pathname to the DRC RVE check text override file.

    If the CTO file is created automatically or by Calibre RVE or Calibre Interactive, it is automatically named drc_database.cto, where drc_database is the name of your DRC results database.

Examples

The following check text override file, named drc_lab.rdb.cto, is used automatically when the results database drc_lab.rdb is processed for a DRC HTML report:

min_spacing_metal1
RVE Show Layers: metal1 metal2

metal1_enclose_cont
RVE Show Layers: metal1 contact

It specifies the layers displayed for highlight results from two rule checks (min_spacing_metal1 and metal1_enclose_cont).
