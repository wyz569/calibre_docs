Calibre RVE Macro Configuration File

Input for: Calibre RVE

The Calibre RVE macro configuration file is used to add an item to the Macros menu in Calibre RVE. The new menu item calls a user-defined Tcl procedure.

Note:

Calibre RVE unsets TCLLIBPATH at invocation, so TCLLIBPATH and the auto_path variable do not have the same value in the Calibre RVE environment as in the invoking shell. You must explicitly set the auto_path variable within the macro configuration file if you want to use it.

Format

The file is a standard Tcl file.

Contents

  • The Tcl procedures you want to call.

  • At least one rve::addmenu statement.

File Name and Location

Calibre RVE sources the file from the following locations in the order given:

  • Home directory — The file must be named rverc and is sourced on startup before any databases are opened.

  • Current working directory — The file must be named rverc and is sourced on startup before any databases are opened.

  • DFM database directory — If DFM Database DIRECTORY is specified in the rule file, then Calibre RVE looks in the parent DFM database directory for the macro configuration file. The file must be named topcell.rverc, where topcell is the primary cell in the design. For example, if the DFM database is located at dfmdb/TOPCELL.dfmdb, the file must exist at dfmdb/TOPCELL.rverc. This file is sourced when the DFM database is opened.

Up to three separate Calibre RVE macro configuration files may be sourced at startup, and they are sourced in the order specified.
