DRC RVE Rule Check Comment Precedence

A Calibre nmDRC results database can have rule check comments from two sources: the rule file and the Check Text Override file (CTO file). Settings in the DRC RVE check text override file take precedence over settings found in the DRC results database as a result of rule check comments in the rule file, except in the case of the RVE Link statement. By default, RVE Link statements in the check text override file are merged with those found in the results database; merging can be turned off by disabling the option “Append RVE LINK statements to existing check text” on the Calibre RVE Options tab in the DRC Browser category.

If a Check Text Override file uses wildcards in the check name, it is possible for more than one rule check comment section to match a specific rule check. In this case, the latest rule check comment section in the Check Text Override file takes precedence; see “Example 2” in “DRC RVE Check Text Override File (CTO File) Format”. Rule check sections without a wildcard in the check name always have the highest precedence.
