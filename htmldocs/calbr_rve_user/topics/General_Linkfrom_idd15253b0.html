LinkFrom

Key Value for DRC HTML Report Configuration File, report specification category

Usage: Optional. Valid in DRCResults, DRCHighlight, DRCHistogram, and DRCColormap sections. Not valid in the Common section.

Inheritance: Not allowed.

Specifies to place hyperlinks between the current report and the DRCResultsSummary report referenced by the LinkFrom key. Use of LinkFrom enables you to have links between the report sections without inheriting report settings from the DRCResultsSummary section though use of the GroupBy key; in particular it is possible to have different filter settings in the two sections and still have links between the reports.

Without LinkFrom, links between a DRCResultsSummary report and the detailed report are only made if the detailed report references the DRCResultsSummary report with the GroupBy key. In this case the detailed report inherits the filter settings from the DRCResultsSummary, which may not be desired in all cases.

When LinkFrom is used, the section uses the same GroupBy setting as the referenced DRCResultsSummary section; it is not necessary to specify GroupBy. If GroupBy is specified and differs from the setting in the referenced DRCResultsSummary section, a warning is issued in the transcript stating that the GroupBy setting is ignored.

Allowed Values

[section_name]

The name of a section with ReportType = DRCResultsSummary.

Examples

Example 1

This example creates a DRCResultsSummary report with all results listed. Highlight results are only wanted for the not waived and not fixed metal1 results, so a DRCResults report section with the appropriate filters is created. The LinkFrom key specifies to include links between the two report sections.

In the following image links are made for two of the three metal1 checks; the min_spacing_metal1 check is waived, and therefore not processed by the DRCResults report section. 



[Summary - sorted]
ReportType = DRCResultsSummary
ShowCheckText = 0
GroupBy = Check
SortBy = Priority

[highlight metal1]
ReportType = DRCResults
LinkFrom = [Summary - sorted]
Highlight = 1
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed
OrCheckFilter  = *metal1*

If LinkFrom were not used, there would not be links between the summary section and the highlight reports. If GroupBy = [Summary - sorted] were used in the DRCResults report, links would be automatically included, but all checks would be highlighted because the filters in the DRCResults would not be used due to inheritance from the [Summary - sorted] report.

Example 2

This example uses LinkFrom in more than one report type. The example also demonstrates the behavior of the LinkFrom key by comparing a DRCResultsSummary section with and without LinkFrom.

Figure 1. DRC HTML Report with LinkFrom

The following configuration file uses the LinkFrom key in three report sections (DRCHighlight, DRCResults, and DRCColormap) and produces DRCResultsSummary report shown for the “With LinkFrom” report in the preceding figure. Each of the detailed report sections use a different OrCheckFilter setting to control which results are processed; this allows you to write a different report for different checks. Although the DRCResultsSummary section references the named filter [Filter NotResolved], the three report sections cannot do so because the OrCheckFilter setting would then be ignored. The LinkFrom key specifies that a link to the report is placed in the referenced DRCResultsSummary section, which would not otherwise be done because the section does not inherit the GroupBy setting from the DRCResultsSummary. 

[Filter NotResolved]
ReportType = DRCFilter
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed
; ;Show tree for not fixed and not waived results
[Summary - Not Waived and Not Fixed]
ReportType = DRCResultsSummary
ShowCheckText = 0
GroupBy = Check
Filter = [Filter NotResolved]
[highlight - spacing]
ReportType = DRCHighlight
GroupBy = Check
OrCheckFilter  = *spacing*
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed
LinkFrom = [Summary - Not Waived and Not Fixed]
[Results - poly and metal]
ReportType = DRCResults
GroupBy = Check
Highlight = 1
OrCheckFilter  = *poly*, *metal*
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed
LinkFrom = [Summary - Not Waived and Not Fixed]
[colormap]
ReportType = DRCColormap
GroupBy = Check
CheckFilter = *density*
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed
LinkFrom = [Summary - Not Waived and Not Fixed]

The following code produces the DRCResultsSummary report shown for the “Without LinkFrom” image in the previous figure. Each report inherits the GroupBy setting from the DRCResultsSummary report, so each report processes all the checks processed by the DRCResultsSummary section. Links to each report section are automatically placed in the DRCResultsSummary report. 

[Filter NotResolved]
ReportType = DRCFilter
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed
; ;Show tree for not fixed and not waived results
[Summary - Not Waived and Not Fixed]
ReportType = DRCResultsSummary
ShowCheckText = 0
GroupBy = Check
Filter = [Filter NotResolved]
[highlight - spacing]
ReportType = DRCHighlight
GroupBy = [Summary - Not Waived and Not Fixed]
OrCheckFilter  = *spacing*
[Results - poly]
ReportType = DRCResults
GroupBy = [Summary - Not Waived and Not Fixed]
Highlight = 1
OrCheckFilter  = *poly*, *metal*
[colormap]
ReportType = DRCColormap
GroupBy = [Summary - Not Waived and Not Fixed]
CheckFilter = *density*
