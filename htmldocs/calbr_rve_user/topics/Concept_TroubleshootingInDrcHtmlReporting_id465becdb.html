Troubleshooting in DRC HTML Reporting

Some common problems in DRC HTML reporting are settings being ignored or no results in a report section.

This section discusses the following common problems:

  • Problem: Settings are not applied

  • Problem: “No results for specified report criteria” notice in the DRC HTML report

  • Problem: Filters are not applied (WaivedStateFilter, PropertyFilter, and so forth)

Problem: Settings are not applied

Cause: Possible causes are misspellings in the key or value, improper use of the key, or problems with inheritance rules.

Solution: Check for typos, verify that the key applies to the section it is in, and review the inheritance rules described in “Section Referencing and Key Value Inheritance”.

Problem: “No results for specified report criteria” notice in the DRC HTML report

Cause: This message can be caused by the following: 1) No results meet the filters defined in the section. 2) Invalid value for GroupBy key when ReportType = DRCResultsSummary.

Solution: Do one or more of the following depending on the ReportType for the section:

  • All reports — Review your filter criteria if you think you should see some output for the section. Check for valid values and misspellings.

    If you are using the PropertyFilter key, make sure that the property name does not include single quotes.

  • DRCResultsSummary reports— Make sure the GroupBy is a valid value. Also make sure the GroupBy key does not reference another section; GroupBy must be locally specified for a DRCResultsSummary report.

  • DRC Histogram and DRC Colormap — Make sure the property name is spelled correctly and exists in the database. Single quotes should not be used in property names.

Problem: Filters are not applied (WaivedStateFilter, PropertyFilter, and so forth)

Cause: If the section references another section with the GroupBy key, then the local filter settings are ignored. If the referenced section has filter settings, they are used. See “Filter Inheritance”.

Solution: Remove the reference to the other section, or move the filter definitions to the referenced section.
