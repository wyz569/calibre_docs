RVE Waive

A DRC RVE rule check comment which can toggle a rule check as waivable or not, and also prevent this setting from being overwritten. This comment can be applied to a single rule check, a group of rule checks, or a mix of rule checks in a group.

Usage

RVE Waive: { Enable | Disable | Lock}

Parameters

{ Enable | Disable | Lock}

Specify “Disable” to make the check not waivable; waiving is enabled by default.

Specify “Lock” to prevent the check’s waiver setting from being overwritten when loading a new CTO file or resetting the CTO file. Checks are not locked by default.

Description

This rule check comment enables or disables waiving for a given rule check or set of rule checks, or prevents the check’s waived setting from being overwritten when a new CTO file is loaded or the CTO file is reset. When not specified, a rule check is waivable and not locked by default. This comment also affects the ability to edit waive comments.

Examples

The following CTO file disables waiving for a single rule check, then locks it.

# Disable waiving for the rule check min_spacing_m1
#
min_spacing_m1
RVE Waive: Disable
RVE Waive: Lock

The following CTO file disables waiving for a set of rule checks.

# Disable waiving for all rule checks that begin with min_spacing
#
min_spacing*
RVE Waive: Disable
