permute

Shape element for a Symbol Library file

The permute element defines the allowed permutations for the ports in a symbol shape definition.

Usage

permute {all_inputs | permute_def}

Parameters

Parameter

Description

permute

A required keyword.

all_inputs | permute_def

One of the following:

  • all_inputs — All input ports are permutable.

  • permute_def — A permutation definition with the following syntax:

    () — Defines a permutable group, with ports separated by commas.

    [] — Defines a non-permutable group.

    # — The # character at the beginning of a permutation definition indicates that port index numbers are used instead of names. Port index numbers start at zero and follow the port definition order in the symbol definition.

Examples

All input ports can be permuted:

permute all_inputs

Ports A and B are permutable:

permute (A,B)

Ports a and b can be permuted, ports c and d can be permuted, and the two groups can be permuted:

permute ((a,b),(c,d))

Ports a and b cannot be permuted, ports c and d cannot be permuted, but the group a,b can be permuted with c,d:

permute ([a,b],[c,d])

The first four defined ports can be permuted:

permute #(0,1,2,3)
