Output Directory Structure in DFM HTML Reporting

The output from the DFM HTML reporting tool is written to a directory specified from the command line.

This directory contains the following items:

  • An HTML index file (index.htm), called the summary report, that provides an overview of the generated output files in tabular form.

  • HTML, CSV, and GIF files ChipSummary reports are generated for each section in the configuration file as follows:

    section_name-input_name-priority_name.report_type

    where

    • section_name — The name of the section in the configuration file the file is associated with (excludes Common).

    • input_name — The name of a group, rule, cell or window from the input data set to the section.

    • priority_name — The priority name specified with the PrioritiesToInclude key. Valid values are none, all, or any actual priority name.

    • report_type — File extension that can be one of the following: htm, csv or gif.

    Examples:

    CellReports-All-Total.htm
ChipReports-All-All.htm

  • A CSS file (default.css), if the CssFilePath key name is not specified in the configuration file.
