Verifying Short Repairs

You can virtually repair a short in Calibre RVE for LVS by marking polygons for removal and assigning net labels to polygons. Then, still within Calibre RVE, you can run short verification to confirm that a short repair fixes the short and does not cause other shorts to appear. Once the short verification is successful, you make the actual change(s) in the layout database.

Short verification in Calibre RVE for LVS uses the same short isolation algorithm that is used in batch Calibre runs, and uses the Calibre Query Server to make changes to the shorts database in memory.

Note:

Short verification within Calibre RVE for LVS does not alter the actual layout database; it operates on the shorts database in the SVDB.

After using Calibre RVE to verify that a particular design change will repair the short, you must make the corresponding change in the layout database, then run Calibre nmLVS again to confirm the fix.

Prerequisites

  • Requirements described in “Interactive Short Isolation”

Procedure

  1. Identify the cause of the short and the fix needed to repair the short, as summarized in “Interactive Short Isolation”.

    The fix typically involves removing a polygon and possibly assigning net labels to polygons without a label. It is useful to keep track of the polygons you edit within Calibre RVE, because you will need to make the corresponding changes in the layout database once you have verified that a change fixes the short.

  2. If needed, assign net labels to polygons without a label (N/A); this may have already been done in the step “Assigning Net Labels to the Polygons in a Short”.

    You should not assign a net label to a via or contact layer, as this causes an error during the short verification process.

  3. Mark a polygon for removal as described in “Selecting Polygons to Remove from the Short Path”.
  4. Run short verification using one of the following methods:

    • Right-click in the short summary listing and choose Verify Short.

    • Click the icon in the shorts toolbar and choose Verify Short to verify the currently selected short. After you have edited all shorts, you can choose Verify All Shorts, if desired.

    Note:

    The shorts database is saved after each Verify Short, Verify All Shorts, or Reset All Shorts action.

    If you exit Calibre RVE and then re-open the SVDB, you will see the same view in the Layout Shorts tab that was displayed immediately after the verify or reset action. However, the .rvedb File for Calibre RVE Settings must be writable for this to occur.

  5. View the results. Short verification has one of the following results:

    • Failure — A pop-up message reports that the change did not fix the short. The new set of polygons for the short is displayed. In some cases the changes introduce a new short, which is also displayed in Calibre RVE.



      Take one of the following actions before reviewing the short again:

      • Keep working — You can continue working with the new set of short polygons. You can work with the current assignments for net labels and removed polygons or change the assignments.

      • Start over — Choose > Reset All Shorts to reset the shorts view to the results from the last Calibre run.

      Go back to “Interactive Short Isolation” to identify the cause of the short.

    • Success — If the short repair is successful, a check mark is placed next to the short in the summary view. If you marked one or more polygons with REMOVE, they are listed in the short detailed view below the short summary table. If a removed polygon is part of a via array, all vias in the array are listed.



      Tip

      If you think one of the REMOVE polygons does not need to be removed, click in the Net column for the polygon and select N/A to undo the REMOVE designation. Then run short verification again to confirm the fix.

      Go to Step 6 to edit your layout database and complete the short repair process.

  6. Make the appropriate changes in your layout database once you have verified that the changes repair the short without causing any new shorts.

    If you marked one or more polygons with REMOVE, they are listed in the short detailed view below the short summary table. Double-click a polygon to highlight it in your layout viewer, then remove the shape. Highlighting obeys the setting “Highlight polygons in layout shorts in Top Cell/Short Cell” on the Options pane, Highlighting category ( in the toolbar).

    Tip

    To save the REMOVE polygons for later use, select File > Save Removed Polygons. The polygons are saved in an RDB file that can opened in Calibre RVE. See “Saving and Viewing the Removed Polygons from Interactive Short Isolation”.

  7. After you edit the layout database, do the following to confirm the short repair:
    1. Run Calibre nmLVS again.
    2. Choose File > Reload Database in Calibre RVE.
    3. Review the results on the Layout Shorts tab to see that the short is fixed.
