Symbol Library Files for Custom Symbol Shapes

Symbol library (*.sym) files provide custom symbol shapes for the Internal Schematic Viewer and the Calibre RVE SPICE File Viewer.

Symbol Library File Format

The symbol library file is an ASCII file using the Concept Engineering’s Nlview™ Symlib format. See “Symbol Library File Format” for details.

An example file, netlist, Calibre RVE configuration file (.rvedb), and shell script are located at $CALIBRE_HOME/shared/examples/rve/symlib in the software tree.

Creating or Obtaining Symbol Library Files

If you use Concept Engineering’s Nlview or SpiceVision®, you may already have symbol library files. If you are writing symbol library files, see Appendix “Symbol Library File Reference” for the file format and the symbol shape definitions.

Note the following about the implementation in Calibre RVE:

  • The portBus, pinBus, netBundles, and boxcolor elements in the symLib file are not supported.

  • The index entries may be omitted when you are writing or editing symbol library files. If an index entry is missing or incorrect, Calibre RVE attempts to update it. The update is done automatically and is only possible if write permission is set for the file.

The custom symbol shape is displayed if the subcircuit name and all port names match between the symbol library file and the netlist. Case-insensitive name matching can be specified in the symbol library file.

Specifying Symbol Library Files

  1. Choose Setup > Options to open the Options tab.

  2. Choose the Schematic Viewer category.

  3. Enable “Symbol Library files” and enter the files; enter one file per line or use the file browser to select multiple files.

If there is an error loading a symbol library file, Calibre RVE displays a message in the bottom right corner of the window.
