Calibre::convert_to_route_guides

Converts results from an ASCII format Calibre results database (RDB) to Synopsys IC Compiler route guides. The command is executed from the icc_shell prompt.

Usage

Calibre::convert_to_route_guides input_RDB
-check_map_file map_file
[ -min_feature min_size]
[ -offset x_offset y_offset ]
[ -mag mag_value]
[ -cell cell ]
[ -checks { ALL | check_N … }]
[ -side_rdb {YES | NO} ]
[ -output_file rdb_rg]
[-no_transcript]

Arguments

input_RDB

Required parameter specifying the Calibre results database to be converted. See “Requirements for Converting Calibre RDBs to Native Place and Route Format” for the RDB formats that are allowed.

-check_map_file map_file

Required keyword and parameter specifying the check map file. The check map file maps each Calibre rule check to one or more target layers for the route guide. See “Check Map File Format for Calibre RDB Conversion” for details.

-min_feature min_size

Optional keyword and parameter specifying the minimum width of the converted route guide in user units. A default of 0.002 user units is used if this option is not specified.

-offset x_offset y_offset

Optional keyword and parameters specifying an x- and y-offset. Values are given in user units. If both -offset and -mag are specified, the magnification is applied first.

-mag mag_value

Optional keyword and parameter specifying the magnification of the converted results. A mag_value of 2 magnifies the results by 2; a mag_value of 0.5 shrinks the results.

Use this option if the precision of the input RDB is not the same as the precision of the layout database. If both -offset and -mag are specified, the magnification is applied first.

-cell cell

Optional keyword and parameter specifying to only convert results from cell. If this keyword is not present all results are converted.

-checks { ALL | check_N … }

Optional keyword and parameter specifying the rule check results to convert.

  • ALL — Results from all rule checks are converted. This is the default if the ‑checks option is not included.

  • check_N … — Only results from the listed rule checks are converted. The question mark (?) and asterisk (*) wildcards are allowed.

    If a list of rule checks is provided, only those rule checks are converted. Results from a rule check listed in the check map file but not in the ‑checks list are not converted.

-side_rdb {YES | NO}

Optional keyword and parameter specifying whether to include the results from side (auxiliary) RDBs in the conversion. Side RDB results are not included by default. See “Handling of Side RDBs in Calibre RDB Conversion” for more information.

-output_file  rdb_rg

Optional keyword and parameter specifying that the converted data is saved to the file rdb_rg. The converted data is not imported to the current Synopsys IC Compiler session. Use the Calibre::import_route_guides command to import the converted route guides.

If this keyword is not specified the input_RDB is converted and the route guide shapes are automatically imported to the Synopsys IC Compiler session.

-no_transcript

Optional keyword that turns off transcript output.

Return Values

Returns 1 for success or 0 for failure.

Description

This command converts result shapes from Calibre rule checks to Synopsys IC Compiler route guides. You must have an existing ASCII format Calibre results database (RDB); see “Requirements for Converting Calibre RDBs to Native Place and Route Format”. Consult your CAD group for advice on which Calibre rule checks produce result shapes that can be used as route guides.

The cell containing the results you want to convert must be the working design in your Synopsys IC Compiler session; the cell is opened automatically if it is not already open. Route guides are created in the CEL view. The design should be writable, otherwise the generated route guides cannot be saved.

The required check map file specifies the target layer(s) per rule check for the route guides; see “Check Map File Format for Calibre RDB Conversion”. Route guides are created with the ‑no_signal_layers and ‑no_snap options to the create_route_guide command. The target layer(s) in the check map file provide the layer parameter(s) for the ‑no_signal_layers option.

A transcript with information such as a the cell list, rule check list, precision, output file, and execution details is included by default. The transcript can be turned off with the ‑no_transcript option; errors, warnings, and notes are still output.

The route guides are created as rectangular shapes with a default minimum width of 0.002 user units. You can use the “‑min_feature min_size” option to set a different minimum feature size, where min_size is given in user units. In order to produce rectangular route guides from non-rectangular result shapes the tool uses the process given in the following table.

Table 1. Calibre RDB Conversion Process for Route Guides

Calibre Result Shape

Conversion Diagram

Converted Shape

Point

Square with sides of length min_size.

Single edge

Rectangle of width min_size.

Edge pair

Bounding box of result extent.

Rectilinear polygon

Polygon is fractured into multiple rectangles.

See the topics in “Configuring Calibre RDB Conversion to Native Place and Route Format” for additional information on the conversion options.

The route guides are named with the following format:

mgc_rve_check_<cell>_<check>_<layers>_<result-num>_<rectangle-num>

where

  • If the route guide is created on multiple layers each layer is separated by an underscore, as follows:

    <layer1_layer2_ ... layerN>

  • The <result-num> result numbers start at zero.

  • <rectangle-num> is used only if a result shape is fractured into multiple rectangles.

For example, the route guide in cell top for check m4width created on layer M4 has the following name for the first result:

mgc_rve_check_top_m4width_M4_0

On successful completion a message is written to the transcript:

// RDB written as PR input file <file>

The route guides are automatically imported into the Synopsys IC Compiler session unless you specify the ‑output_file keyword to save the results to file. Turn on route guide visibility in order to view the route guides. If you use the ‑output_file option, use the Calibre::import_route_guides command to import the converted route guides.

If you create route guides more than once within the same Synopsys IC Compiler session you must remove the existing route guides before creating new route guides. You can use the “remove_route_guide mgc_rve*” command to remove route guides created by the Calibre RDB conversion process.

Examples

Use the following command to convert results from rule check m3_pattern in cell BLOCK2. The check map file is named map and has a target layer of M3 for check m3_pattern.

icc_shell> Calibre::convert_to_route_guides drc.results -check_map_file map
   -cell BLOCK2 -checks m3_pattern

The route guides are created on layer M3 and automatically imported into the Synopsys IC Compiler session. Turn on route guide visibility to view them.

You can view a list of the resulting route guides with the get_route_guides command. Suppose for this example the check m3_pattern has one result in cell BLOCK2 and that the result shape was fractured into three rectangles. The following route guides are created on layer M3:

icc_shell> get_route_guides *mgc*
{mgc_rve_check_BLOCK2_m3_pattern_M3_0_0
mgc_rve_check_BLOCK2_m3_pattern_M3_0_1
mgc_rve_check_BLOCK2_m3_pattern_M3_0_2}

See “Importing Calibre Results as Synopsys IC Compiler Route Guides” for complete instructions on converting Calibre results to route guides, including an image of a converted route guide.
