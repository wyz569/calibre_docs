Calibre::convert_to_drc_errors

Converts results from an ASCII format Calibre results database (RDB) to native Place and Route DRC error marker format. Used in the Synopsys IC Compiler or Cadence Encounter environment.

Usage

Calibre::convert_to_drc_errors input_RDB
[ -check_map_file map_file]
[ -offset x_offset y_offset]
[ -mag mag_value]
[ -cell cell ]
[ -checks { ALL | check_N … }]
[ -side_rdb {YES | NO} ]
[ -output_file rdb_drcerr]
[-no_transcript]

Arguments

input_RDB

Required parameter specifying the Calibre results database to be converted. See “Requirements for Converting Calibre RDBs to Native Place and Route Format” for the RDB formats that are allowed.

-check_map_file map_file

Optional keyword and parameter specifying the check map file. The check map file maps each Calibre rule check to a target layer property which is displayed in the Synopsys IC Compiler Error Browser. See “Check Map File Format for Calibre RDB Conversion” for details about the check map file.

-offset x_offset y_offset

Optional keyword and parameters specifying an x- and y-offset. Values are given in user units. If both -offset and -mag are specified, the magnification is applied first.

-mag mag_value

Optional keyword and parameter specifying the magnification of the converted results. A mag_value of 2 magnifies the results by 2; a mag_value of 0.5 shrinks the results.

Use this option if the precision of the input RDB is not the same as the precision of the layout database. If both -offset and -mag are specified, the magnification is applied first.

-cell cell

Optional keyword and parameter specifying to only convert results from cell. If this keyword is not present all results are converted.

-checks { ALL | check_N … }

Optional keyword and parameter specifying the rule check results to convert.

  • ALL — Results from all rule checks are converted. This is the default if the ‑checks option is not included.

  • check_N … — Only results from the listed rule checks are converted. The question mark (?) and asterisk (*) wildcards are allowed.

If a list of rule checks is provided, only those rule checks are converted. Results from a rule check listed in the check map file but not in the ‑checks list are not converted.

-side_rdb {YES | NO}

Optional keyword and parameter specifying whether to include the results from side (auxiliary) RDBs in the conversion. Side RDB results are not included by default. See “Handling of Side RDBs in Calibre RDB Conversion” for more information.

-output_file rdb_drcerr

Optional keyword and parameter specifying that the converted data is saved to the file rdb_drcerr. The converted data is not imported to the current Synopsys IC Compiler session. Use the Calibre::import_drc_errors command to import the converted error markers.

If this keyword is not specified the input_RDB is converted and the error markers are automatically imported to the Synopsys IC Compiler session.

-no_transcript

Optional keyword that turns off transcript output.

Return Values

Returns 1 for success or 0 for failure.

Description

This command converts result shapes from Calibre rule checks to Synopsys IC Compiler or Cadence Encounter error markers. You must have an existing ASCII format Calibre results database (RDB); see “Requirements for Converting Calibre RDBs to Native Place and Route Format”.

See the topics in “Configuring Calibre RDB Conversion to Native Place and Route Format” for additional information on the conversion options. If you use the ‑output_file option, you must use the Calibre::import_drc_errors command to import the converted DRC error markers to the current1 design tool session.

A transcript with information such as a the cell list, rule check list, precision, output file, and execution details is included by default. The transcript can be turned off with the ‑no_transcript option; errors, warnings, and notes are still output.

See the following sections for tool-specific behavior:

  • “Cadence Encounter Operation”

  • “Synopsys IC Compiler Operation”

Cadence Encounter Operation

The design or cell containing the results you want to convert must be the working design in your Cadence Encounter session or an error occurs.

Close the Violation Browser using the Close button before issuing the Calibre::convert_to_drc_errors command. For some Cadence Encounter versions the conversion process can hang if the Violation Browser is open during the process or if you close the Violation Browser with the X icon in the top right corner of the window.

The converted DRC error markers have a fixed minimum feature size of 0.002 user units; for example, a single edge error is converted to a rectangle with a width of 0.002 user units. Edge pairs and polygons are converted to rectangles as shown in the following table:

Table 1. RDB Conversion Process to Cadence Encounter Error Markers

Calibre Result Shape

Conversion Diagram

Converted Shape

Point

Square with sides of length 0.002 user units.

Single edge

Rectangle of width 0.002 user units.

Single edge not parallel to x or y axis

The bounding box for the edge, with the sides of the box parallel to the x and y axis.

Edge pair

Bounding box of result extent.

Rectilinear polygon

Polygon is fractured into multiple rectangles.

If a rectilinear polygon is fractured into multiple rectangles, as shown in the preceding table, each component rectangle is converted as a separate error marker. When this occurs, a single Calibre nmDRC result is converted to multiple error markers in the Violation Browser. As a result, the error count in the Violation Browser may not match the Calibre nmDRC result count.

The DRC error markers are automatically imported into the Cadence Encounter session unless you specify the ‑output_file keyword to save the results to file. You can view the errors in the Violation Browser.

Synopsys IC Compiler Operation

The cell containing the results you want to convert must be the working design in your Synopsys IC Compiler session. The cell is opened if it is not open already.

The converted DRC error markers have a fixed minimum feature size of 0.002 user units; for example, a single edge error is converted to a rectangle with a width of 0.002 user units. Edge pairs are converted to polygons.

Table 2. RDB Conversion Process to Synopsys IC Compiler Error Markers

Calibre Result Shape

Conversion Diagram

Converted Shape

Point

Square with sides of length 0.002 user units.

Single edge

Rectangle of width 0.002 user units.

Edge pair

Polygon.

Polygon

Polygon.

The DRC error markers are automatically imported into the Synopsys IC Compiler session unless you specify the ‑output_file keyword to save the results to file. You can view the errors in the Error Browser.

Examples

Example 1 — Synopsys IC Compiler

The following command converts results from the RDB file drc.results. All results are converted by default. The top cell must be the working design in the Synopsys IC Compiler session; the cell is opened automatically if it is not already open.

icc_shell> Calibre::convert_to_drc_errors drc.results

The converted results are automatically imported into the Synopsys IC Compiler session. Choose Verification > Error Browser to view the errors. If the Open Error View dialog box does not open automatically, choose File > Open in the Error Browser to open it, select an error view of type “Thirdparty DRC” and browse to the correct cell and version. The converted Calibre results are shown in the Error Browser.

Example 2— Cadence Encounter

The following command converts results from the RDB file drc.results. All results are converted by default. The design must be open in the Cadence Encounter session.

encounter > Calibre::convert_to_drc_errors drc.results

The converted results are automatically imported into the Cadence Encounter session. View the errors in the Violation Browser.

Example 3— ‑checks and ‑cell arguments

The following command converts results from the RDB file drc.results. Results from check met2_sp in cell block1 are converted.

icc_shell> Calibre::convert_to_drc_errors drc.results -checks met2_sp -cell block1

Example 4— Save to file

The following command converts all results and saves the converted results to the file top_calibre_drc.rdbout.

icc_shell> Calibre::convert_to_drc_errors drc.results -output_file top_calibre_drc.rdbout

The results are not automatically available in the design tool session. Use the Calibre::import_drc_errors command to import the converted results, as follows:

icc_shell> Calibre::import_drc_errors top_calibre_drc.rdbout
