Automatically Process Side (Auxiliary) RDBs

DRC HTML Reporting can automatically open and process the side RDBs created with the main results database. You can specify which side RDBs are opened and specify what types of RDBs are processed by each report section in the configuration file.

By default, Layout Input Exception, ERC Pathchk, and Waiver side RDBs are opened automatically.

Procedure

  1. Use the Open ... DBs keys in the Common section to specify the side RDBs to open. For example: 
    [Common]
; LIE, PATHCHK, and Waiver RDBs are opened by default
; do not open PATHCHK RDBs
OpenPATHCHKDBs = 0
; open Density RDBs
OpenDensityDBs = 1 
; remaining common section specifications
...

    When the side RDBs are opened they are automatically assigned the appropriate RDB type.

  2. Use the ApplyTo key to specify what type of RDB a report section applies to. By default, a report section applies to the RDB type of “General”. If you specify another report type with the ApplyTo keyword, you must explicitly specify the “General” RDB type if you want the report section to apply “General” RDBs also, or specify “ApplyTo = All” to have the report section apply to all report types.

    For example:

    [General and Density report section]
ApplyTo = General, Density
ReportType = DRCResults
GroupBy = Cell/Check
; rest of report section specifications
    [Waiver RDB section]
ApplyTo = Waiver
ReportType = DRCResults
GroupBy = Cell/Check
; rest of report section specifications
