Section Referencing and Key Value Inheritance

In DRC HTML reporting, you can inherit settings from another report section.

You may specify that key values in a report section other than DRCResultsSummary are inherited from a DRCResultsSummary section in the configuration file. This is done by referencing the section to inherit from with the GroupBy key, as shown here:

GroupBy = [DRCResults_Summary_section_name_to_inherit_from]

The GroupBy key is the only key that can reference another section in this way. Sections cannot be referenced in such a way that cyclic dependencies are created, and the GroupBy key in a DRCResultsSummary report sections cannot reference another section.

Key values can also be inherited from the Common section, if they are not defined locally within a section or defined in a referenced section.

The three types of inheritance, standard, filter, and sorting, as defined in the following sections. An example is included.

Standard Inheritance

Standard inheritance is used for all keys except the *Filter, SortBy, and SortOrder keys. Standard inheritance follows these rules of precedence:

  1. Use the key value specified in the section, if present.

  2. If the key value is not specified in the section and the GroupBy key references another section, inherit the value from the referenced section.

  3. If the key value is not specified in the section and the GroupBy key does not reference another section, inherit the value from the Common section.

  4. If the Common section does not specify the key value, use the default.

Filter Inheritance

Filter inheritance is used for the *Filter keys, as described here:

  • If the GroupBy parameter references another section:

    • Use the key value specified in the referenced section.

    • Ignore any locally specified value.

    • Ignore the Filter keyword.

  • If the GroupBy key does not reference another section:

    • If the key value is not specified locally, inherit the value from the Common section.

Sorting Inheritance

Sorting inheritance is used for the SortBy and SortOrder keys, rather than the standard inheritance rules. These keys inherit only from the Common section if the value is not specified in the section; SortBy and SortOrder are not inherited from a referenced section.

Example for Section Referencing

The section definitions below specify that the DRC_Highlight and Colormap section inherits key values from the Summary section. The DRC_Highlight section inherits ZoomLayoutFactor from the Common section, while the Colormap section specifies ZoomLayoutFactor locally and does not inherit the value. The DRC_Highlight section specifies LayoutWidth and LayoutHeight locally, while the Colormap section uses the default values for LayoutWidth and LayoutHeight.

[Common]
ZoomLayoutFactor = 0.5
DrvOptionsFilePath = /mypath/calibredrv_dir
[Summary]
ReportType = DRCResultsSummary
GroupBy = Check
; highlight section
[DRC_Highlight]
ReportType = DRCHighlight
GroupBy = [Summary]/Cell
LayoutWidth = 200
LayoutHeight = 200
; colormap section
[Colormap]
ReportType = DRCColormap
GroupBy = [Summary]
Property = dv
; specify zoom locally, do not inherit
ZoomLayoutFactor = 1.0
