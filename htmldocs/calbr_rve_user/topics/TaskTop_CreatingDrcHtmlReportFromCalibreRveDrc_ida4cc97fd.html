Creating a DRC HTML Report from Calibre RVE for DRC

You can create an HTML formatted report of the results that are displayed in Calibre RVE for DRC. The report generation is completely automatic and is based on the current view in Calibre RVE for DRC. Results highlights can be included if the layout is available in GDS, OASIS, or one of the formats supported by Calibre MDPview. Choose Tools > Create HTML Report to start the report creation. A shell script is created so that you can run the report at a later time from the command line.

Prerequisites

  • A DRC results database is open in Calibre RVE for DRC.

  • (Optional) A layout database in order to create highlight images. The layout database can be GDS, OASIS, or one of the formats supported by Calibre MDPview.

    If the layout database is open in a Calibre DESIGNrev or Calibre MDPview session connected to Calibre RVE, the information for the layout database is automatically entered in the Configure HTML Report dialog box. In addition, the tool creates a layerprops file based on the current settings in the viewer and uses those settings for the layout snapshots.

    If the design is not open in Calibre DESIGNrev or Calibre MDPview, you may specify the layout database information in the Configure HTML Report dialog box. If a layout.layerprops file exists, where layout is the name of the layout database, the layerprops file is automatically used when creating the layout images for the HTML report.

    Note:

    In Calibre DESIGNrev, if using an OASIS database, a CTO file, and an external layerprops file, the layerprops file must be specified with the following steps. First, disable the “Prefer layernames from Oasis file” option in DESIGNrev under Options > Misc Preferences > Layer Palette. Then, load the layerprops file under Layer > Load Layer Properties.

  • (Optional) A DRC RVE Check Text Override File (CTO File) Format. The CTO file is used to set layer visibility, highlight color, and other options for the results database.

    The CTO file is loaded automatically if it is named drc_db.cto, where drc_db is the filename of the DRC results database. The file drc_db.cto should exist in the same directory as the results database. You can also load a CTO file as stated in Step 1 of the following procedure.

    Tip

    See “Creating a Check Text Override (CTO) File Using Calibre RVE” for information on automatically creating a CTO file to control layer visibility.

  • (Optional) An HTML customization file. See “HTML Customization File for DRC HTML Reports”.

  • (Optional) A custom Cascading Style Sheet (CSS) file. See “Using a Custom Cascading Style Sheet (CSS) in DRC HTML Reporting”.

  • (Optional) Set the environment variable MGC_RVE_HTML_BROWSER to the path for a browser executable to specify the HTML browser that is opened when viewing the DRC HTML Report. A recent browser version is recommended, as unexpected results may occur in older browsers. See “Output Format for DRC HTML Reports” for a list of browsers that support MHTML format reports. Your default browser is used if this environment variable is not set.

Procedure

  1. (Optional) If you want to use a CTO file that is not named drc_db.cto, choose Tools > Import CTO File and specify the CTO file.
  2. Set the tree hierarchy as desired with View > Tree Options > Group By. Do not group by statistics; this grouping is not supported when creating the DRC HTML report.
    Tip

    To include only one result per Shape Class (#SC) property per rule check, create a custom grouping hierarchy of Check/#SC. The setting “Number of results to report” in the Configure HTML Report dialog box is automatically set to 1 when the last leaf in the grouping hierarchy is #SC.

  3. Set up the view as desired. For example, you can use a DRC Result Filter, select the columns that are shown, and select various options on the Options tab (Setup > Options).

    If the Check Text pane is displayed in Calibre RVE, the check text is included in the DRC HTML report. Close the Check Text pane if you do not want to include check text.

    The active results database is used for the report; results from auxiliary (side) RDBs are not included. Histogram and colormap output is not supported.

  4. Choose Tools > Create HTML Report and specify the following options in the Configure HTML Report dialog box:
    Table 1. Configure HTML Report Dialog Box Options

    Option

    Description

    Report Format

    The format of the report:

    • HTML — Standard HTML format

    • MHTML — Single file HTML format

    Output Directory

    The directory in which the created report and supporting files are saved. The directory is created if it does not exist.

    Number of results to report

    The number of results to report per node in the tree hierarchy:

    • All — Report all results

    • number — Report the first number results per node.

    This option is automatically set to 1 when the last leaf in the grouping hierarchy is #SC.

    Open report after it is created

    As described.

    Layout Snapshots

    Include a layout image with a highlight for each result.

    Specify the layout format and layout path. The format can be one of the following:

    • MDP — One of the formats supported by Calibre MDPview

    • GDS — GDS format

    • OASIS — OASIS format

    Customization File

    Use an HTML customization file. This option adds the CustomizationFilePath key to the configuration file used for the HTML report.

    Stylization File

    Use a custom CSS file. This option adds the CssFilePath key to the configuration file used for the HTML report.

  5. Click OK.

Results

A report is created for the active results tab and saved to the output directory. The report is automatically opened if “Open report after it is created” is enabled. The report is named index.htm for HTML formatted reports and report.mht for MHTML formatted reports.

The tool also performs the following actions:

  • Creates a report.ini configuration file in the output directory.

  • Creates rdbSpec.txt (DRC HTML Report Database Map File) in the output directory if layout snapshots were included in the report.

  • Creates a layerprops file named layerProperties.txt in the output directory if the layout is open in a Calibre DESIGNrev or Calibre MDPview session connected to Calibre RVE.

  • Creates a shell script in the working directory; you can execute the shell script to recreate the HTML report at any time. The shell script is named createReport.sh if the leaf name of output directory is report (the default) or createReport_rptDir.sh otherwise, where rptDir is the leaf name of the output directory.

    If the “Open report after it is created” option is enabled, you can include the “‑open” switch when running the script to automatically open the report in a web browser after it has been created.

  • Displays the shell transcript in a pop-up window.
