Requirements for Converting Calibre RDBs to Native Place and Route Format

You can convert results from an ASCII format Calibre results database (RDB) to error markers and route guides in your Place and Route tool. Certain requirements for the design tool and Calibre results database must be met.

The requirements are the following:

  • A Calibre RVE license.

  • You are using Synopsys IC Compiler or Cadence Encounter.

  • The standard Calibre interface is loaded for the design tool. See one of the following:

    “Loading the Calibre Interface to Synopsys IC Compiler”

    “Creating an Interface to Cadence Encounter”

  • A Calibre results database in ASCII format produced by any of the following Calibre applications: Calibre nmDRC, Calibre PERC LDL, Calibre Pattern Matching, and Calibre Multi-Patterning.

    The following error types in the RDB are supported: single edge, edge pair, and polygon. However, in the Cadence Encounter environment, single edge results can only be imported as DRC error markers; they cannot be imported as Cadence Encounter route blockages.

    Enhanced simulation result output from Calibre PERC LDL applications is not supported and is ignored during conversion.

    Check text is automatically included when converting to DRC Error format. Properties in the RDB are not included in the converted output.
