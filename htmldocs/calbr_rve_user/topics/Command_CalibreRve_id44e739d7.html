calibre -rve

Command line invocation for Calibre RVE (Results Viewing Environment).

Usage

Interactive Display

calibre[-nowait | -wait n] [-cb] -rve [‑win_title title]
   [ {[-drc] drc_db_file [drc_db_file …]
      | -dfm dfmdb [-cmp] [-rdb results.rdb [results2.rdb …]]
      | {-lvs | -perc | -pex} svdb_directory [layout_primary]
         [-rdb results.rdb [results2.rdb …]]
      | -spice spice_file [-top topcell]}
      [-turbo [num_proc]]

CSV Output for Calibre nmDRC and Calibre PERC Results

calibre-rve
   { {-drc drc_db_file -output_csv csv_out_file
      [‑write_check_text] [-write_top_coords] }
   | {-perc svdb_directory [layout_primary] -output_csv csv_out_file }}

CTO File Creation

calibre ‑rve ‑drc [rdb_file] [rdb_file …] ‑createcto  
[-ctotype CD] ‑rules rule_file
[‑layermap layer_map_file]  
[‑category {group | layer | group,layer}]
[‑output cto_file | ‑outputdir directory]
[‑warnOverwrite]
[-comments]
[-noshowlayers]

Waiver Report Generation

calibre -rve -drc rdb_file -waiver_report [output_file] [-group_by {check | cell}] [-useGMT]

Scripting Interface to Calibre RVE Spice File Viewer

calibre -rve -nodisplay -spice spice_file -script print_schem.tcl

RDB Conversion to Native Place and Route Tool Formats

calibre -rve -drc input_RDB -convert_to { icc_drc | icc_rgf | edi_drc | edi_blk }
‑output_file converted_rdb
[-check_map_file map_file]
[-min_feature min_size]
[-offset x_offset y_offset ]
[-precision { 1000 | 2000 | 4000 | 10000 | 20000 }]
[-cell cell]
[-checks { ALL | check_N … }]
[-side_rdb {YES | NO} ]

Batch Filtering

calibre -rve -drc rdb_input [-split] [-verbose] [-useGMT]
{ -filter       
   [-NOT]
   [-cell "cell1 [cell2 ...]"]
   [-check "check1 [check2  ...]"]
   [-property "prop"]
   [-property "prop { < | <= | == | != | >= | > } value"]
   [-property "minval { <= | < } prop { <= | < } maxval"]
   [-include_area "cellname coords"]
   [-exclude_area "cellname coords"]
   [ -waived | -unwaived ]
   [-waived_by "user1 [user2 ...]"]
   [-waived_comment "comment"]
   [-waived_date "{ < | <= | == | != | >= | > } date"]
   [-noempty_checks]
   [-OR filter_spec]
   [-other]
-output "rdb_output"} ...

Arguments

‑nowait

An optional switch which specifies that Calibre RVE does not wait for a license if one is not available immediately.

-wait n

An optional switch and argument which specifies the amount of time in minutes that Calibre RVE waits for a license.

-cb

Specifies to use the Calibre CB (Cell/Block) license package. This option is discussed in the “Calibre CB” section of the Calibre Verification User’s Manual. The options ‑cb and ‑turbo can not be specified together, since the Calibre CB (Cell/Block) license package does not support multithreaded parallel processing.

-rve

A switch that specifies Calibre RVE.

When you specify this switch without optional arguments, a dialog box displays and prompts you to specify the Calibre RVE application and database or database directory.

-win_title title

Specifies a title for the Calibre RVE window. Spaces and quotes are not allowed in the title string. This option must appear after the ‑rve option.

When the ‑win_title option is specified, the title of the Calibre RVE window is the Calibre version and the title string. The window title can not be changed during the Calibre RVE session. If this option is not specified, the title of the Calibre RVE window is the Calibre version and the name of the results database.

Legal title string: DRC_check1-08/07/2011

Illegal title string: “LVS test run 2b”

[‑drc]drc_db_file [drc_db_file … ]

A switch and database specification for Calibre RVE for DRC.

The parameter drc_db_file is the absolute or relative pathname to an ASCII DRC results database (this includes ERC databases produced by Calibre nmLVS). You can open multiple ASCII DRC results databases.

Compressed files with the .gz (gzip format) or .bz2 (pbzip2 format) extension are supported. If the gzip or pbunzip2 utility is not in your path, you are prompted to supply the path.

-dfm dfmdb [-cmp]

A switch and database specification for Calibre RVE for DFM, where dfmdb is one of the following:

  • Path to a DFM Database directory.

  • Path to an ASCII DFM results database.

The optional ‑cmp switch causes Calibre RVE for DFM to display the CMP Hotspot report rather than the Report Card view. In addition, this switch enables Calibre RVE for DFM to access Calibre YieldServer commands in the cmp::namespace.

{-lvs | -perc | -pex} svdb_directory [layout_primary]

This set of options is used to start Calibre RVE for LVS, PERC, or PEX. The ‑lvs, ‑perc, and ‑pex switches control which report tabs are opened when Calibre RVE starts. All reports present in the SVDB directory are available, even if not opened at startup.

  • -lvs — A switch that specifies Calibre RVE for LVS. If you specify a SVDB directory (svdb_directory) without a ‑lvs, ‑perc, or ‑pex switch when you invoke Calibre RVE, Calibre RVE for LVS is opened by default. However, you must specify one of the ‑lvs, ‑perc, or ‑pex switches if the ‑rdb switch is used.

  • -perc — A switch that specifies Calibre RVE for PERC.

  • -pex — A switch that specifies Calibre RVE for PEX. Calibre RVE is opened to the net parasitics table. Calibre nmLVS extraction and comparison results are still available.

  • svdb_directory — The absolute or relative pathname to the SVDB directory. The SVDB directory contains the results from Calibre nmLVS, Calibre PERC, or PEX execution. For Calibre PERC LDL results, svdb_directory is the path to the DFM database produced by the run.

  • layout_primary — The top cell name as specified by the Layout Primary specification statement in the rule file. If you do not specify this name, Calibre RVE attempts to locate a single primary cell in the SVDB directory. If it finds multiple top cells, a dialog box displays and prompts you with the top cells it found.

-rdb results.rdb [results2.rdb …]

An optional switch and parameter(s) that specify one or more DRC-style results databases to open in addition to the main results database. The ‑rdb switch is allowed with the ‑dfm, ‑lvs, ‑perc, and ‑pex switches.

  • results.rdb — A DRC-style results database, such as an ERC results database or the database produced with the RDB keyword to the Net Area Ratio command.

-spice spice_file [-top topcell]

A switch and parameters to open the Calibre RVE SPICE File Viewer.

  • spice_file — A SPICE netlist file. Compressed files ending in the .gz (gzip format) or .Z file extensions are supported. Files ending in .Z are opened with the gunzip utility.

  • ‑top topcell — Optional switch and parameter that specify the top cell name.

-turbo [num_proc]

This option specifies multithreaded parallel processing. The optional parameter num_proc is a positive integer that specifies the number of CPUs to use for multithreaded processing.

If num_proc is not specified, Calibre RVE runs on the maximum number of CPUs available for which you have licenses. If num_proc is specified but not enough CPUs are available, Calibre RVE runs on the maximum number of CPUs that are available.

Multithreaded parallel processing is available for certain operations within Calibre RVE, such as Interactive Short Isolation in Calibre RVE for LVS, or DRC operations executed from a YieldServer script or the Layer Browser tab in Calibre RVE for DFM. The ‑turbo option may be specified with any Calibre RVE invocation, but the option only applies to supported operations.

If Calibre RVE is started from Calibre Interactive DFM or Calibre Interactive nmLVS, the ‑turbo option is included automatically in certain cases, as described in the section “Multithreaded Parallel Processing for Calibre RVE Operations (‑turbo Option)”.

The options ‑cb and ‑turbo cannot be specified together, since the Calibre CB (Cell/Block) license package does not support multithreaded parallel processing.

CSV Output for Calibre nmDRC and Calibre PERC Results

-drc drc_db_file

A switch and argument specifying the database type and database to process for CSV (Comma-Separated Values) output. drc_db_file specifies a DRC results database in ASCII format.

-perc svdb_directory [layout_primary]

A switch and argument specifying the database type and database to process for CSV output. svdb_directory specifies a database from a topological Calibre PERC run. The layout_primary argument is optional and specifies the top cell.

‑output_csv csv_out_file

A switch and argument that specifies to output the results database in CSV format to the file csv_out_file. The Calibre RVE GUI is not displayed. This argument is only valid with the ‑drc and ‑perc options.

The file suffix .csv is not automatically added to the output file. See the “Description” section for more information.

‑write_check_text

An optional argument used with ‑drc and ‑output_csv that specifies to include check text in the output. Only check text within the DRC results database is processed—any Check Text Override (CTO) file associated with the database is not processed.

-write_top_coords

An optional argument used with ‑drc and ‑output_csv that specifies to include cell transformation details and the transformed results expressed in top level coordinates. The top level coordinates values are produced by applying the transformations to the error results.

For more information, see DRC Cell Name in the Standard Verification Rule Format (SVRF Manual.

CTO File Creation

-drc [rdb_file] [rdb_file …]

A required switch and optional argument specifying one or more ASCII DRC or Calibre PERC LDL CD results databases.

If -output is not specified, the rdb_file filename is used to form the name of the created CTO file. If rdb_file is not specified, the rule_file filename is used instead. One CTO file is created for each rdb_file. By default the CTO file is named rdb_file.cto.

rdb_file is required if the CTO file type is set to CD (-ctotype CD).

‑createcto

A required argument specifying to create a CTO file.

-ctotype CD

An optional argument specifying that rdb_file is an RDB file from a Calibre PERC LDL CD run. The CTO file is generated using information from both rule_file and rdb_file to determine the rule check and layer name association.

‑rules rule_file

A required switch and argument specifying the rule file to use when creating the CTO file. The file path appears in the header of the CTO file.

‑layermap layer_map_file

An optional switch and argument specifying a layer map file. When a layer map file is specified, the RVE Show Layers statements in the CTO file use layer names or layer/purpose pairs from the layer map file rather than layer numbers. The layer map file also affects the layer names used for the Layers category when “‑group layer” is specified. The file path appears in the header of the CTO file.

See “Layer Map File Usage in Check Text Override (CTO) File Creation” for information on the format of the layer map file.

‑category {group | layer | group,layer}

An optional switch and argument specifying to create category definitions in the CTO file. See “Filtering DRC Results by Category”.

Specify a single category or a comma separated list with no spaces:

  • group — Rule checks are categorized by the Group statements in the rule file.

  • layer — Rule checks are categorized by layer, where each layer category contains the rule checks that use that layer.

‑output cto_file

An optional switch and argument specifying the name of the CTO file that is created. The filename must end with the .cto file extension. If cto_file does not end in .cto, it is assumed to be a directory name and the CTO file is named ./cto_file/drc_db_file.cto.

‑outputdir directory

An optional switch and argument specifying a directory location for the generated CTO file.

‑warnOverwrite

An optional switch specifying to print a warning message and halt if a CTO file with the same name as the generated CTO file already exists. Existing CTO files are overwritten if this argument is not specified.

-comments

An optional switch specifying to include rule check comments in the generated CTO file.

-noshowlayers

An optional switch specifying to remove RVE Show Layers rule check comments from the generated CTO file.

Waiver Report Generation

rdb_file

A required argument specifying the Calibre results database (RDB) containing the waiver information.

-waiver_report

A required argument specifying to generate a waiver report.

output_file

An optional argument specifying the name of the generated report file. If not specified, the default name of ${RDBFileName}.report.txt is used.

-group_by {check | cell}

An optional switch and argument that controls the hierarchy of the output.

  • check — Prioritizes check then cell. This is the default option.

  • cell — Prioritizes cell then check.

-useGMT

An optional argument that specifies to give times in GMT. When not specified, times are given in the local timezone.

Scripting Interface to Calibre RVE Spice File Viewer

-nodisplay -spice spice_file -script print_schem.tcl

The switches and arguments to process a SPICE netlist with a Tcl script. See the “Description” section for more information.

RDB Conversion to Native Place and Route Tool Formats

calibre -rve ‑drc input_RDB -convert_to options

The Calibre RVE command includes options to convert results from an ASCII format Calibre results database (RDB) to native Place and Route tool formats. The results can then be imported into your Place and Route tool. See “Calibre RDB Conversion: Working with Calibre Results in Place and Route Tools” for information.

CSV Output for Calibre nmDRC and Calibre PERC Results

The ‑output_csv argument instructs Calibre RVE to save the results database in CSV (comma-separated value) format. Calibre nmDRC and topological Calibre PERC results can be saved to CSV format as follows:

  • Calibre nmDRC results — All results are written to the CSV file, with coordinates written in cell space. The Calibre RVE calculated properties #SC, #EL, #PP, and so on, are not included. Also see “Saving DRC Results Data to CSV Format from the Command Line”.

  • Calibre PERC results — See “Saving Calibre PERC Results Data to CSV Format from the Command Line” for more information on the data that is saved to the CSV file.

CTO File Creation

The Check Text Override (CTO) file enables you to specify DRC RVE rule check comments for a DRC results database. See “DRC RVE Check Text Override File (CTO File) Format”.

You can automatically create a CTO file using the ‑createcto and related arguments in the Calibre RVE command line. By default, the generated CTO file includes RVE Show Layers statements for controlling layer visibility during highlight actions. You can also create Group and Layer category definitions by specifying the ‑category option with the desired arguments.

See “Creating a Check Text Override (CTO) File from the Rule File” for information on the process. Also see “Example 2” in this topic.

Waiver Report Generation

You can generate a text report of DRC results that are waived using Calibre RVE. See “Waived Results in Calibre RVE for DRC” for more information on waived results.

Scripting Interface to Calibre RVE Spice File Viewer

The scripting interface enables you to write a Tcl script to set up and save a schematic view to a postscript or Scalable Vector Graphics (SVG) file. See “Scripting Interface to the Calibre RVE SPICE File Viewer” for complete information.

RDB Conversion to Native Place and Route Tool Formats

You can convert results from an ASCII format Calibre results database (RDB) to native Place and Route tool formats. The results can then be imported into your Place and Route tool. See “Calibre RDB Conversion: Working with Calibre Results in Place and Route Tools” for information.

Examples

Example 1

The following are several invocations of Calibre RVE:

  • Invoke Calibre RVE for LVS:

    % calibre -rve -lvs svdb top_cell

  • Invoke Calibre RVE for LVS and also open an ERC database:

    % calibre -rve -lvs svdb top_cell -rdb erc.results

  • Invoke Calibre RVE for DRC and specify a window title: 

    % calibre -rve ‑win_title test3_results -drc results.db

Example 2

Create a CTO file with only RVE Show Layers statements. The generated CTO file is named drc.db.cto by default. 

% calibre -rve -drc drc.db -createcto -rules rules

Create a CTO file with RVE Show Layers statements and with Group and Layer category definitions using the rule file 28nm.rules. The rule file must have Group statements in order for Group category definitions to be created. Name the generated CTO file 28nm.rules.cto. The specified CTO file name must end with the .cto file extension. (The command is all on one line.)

% calibre -rve -drc drc.db -createcto -rules 28nm.rules 
	-category group,layer -output 28nm.rules.cto
