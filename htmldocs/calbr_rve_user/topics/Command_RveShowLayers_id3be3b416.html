RVE Show Layers

A DRC RVE rule check comment that specifies the layers that are shown in the layout editor when highlighting a result from the rule check.

Note:

This command is only supported by the Calibre DESIGNrev/WORKbench, Cadence Virtuoso, Cadence Encounter, Cadence Innovus, Siemens Pyxis Layout, and Synopsys IC Compiler design tools.

Usage

RVE Show Layers: layer1 layer2 … layerN

Parameters

layer1 layer2 … layerN

A space-separated list of layers. At least one layer must be specified. Layer names or numbers can be specified. A space is required between the colon (:) and layer1. The terminating delimiter for the layers list is a newline. Wildcards (*) may be used in the layer designation with all design tools except Pyxis Layout.

Some specific usage notes apply to the following design tools:

  • Cadence Virtuoso

    The layer parameter can be specified in these ways:

    layer:purpose — You can use names or numbers for the layer and only names for the layer purpose. In addition, when using a layer number, the layer purpose must be omitted. For example, MT1:drawing or 6 are both valid. Wildcards (*) may be used only for the layer name. Using a wildcard for the layer purpose results in no match being found.

    layerName — You can specify the layer name by itself. Wildcards may be used for the layer name. If no wildcard is used in the layer name the purpose layer defaults to the drawing purpose layer. If a wildcard is used in the layer name, all purpose values are matched.

    See “Example 3: Wildcard Matching in Cadence Virtuoso”.

  • Cadence Innovus

    The layer parameter can be specified in these ways:

    layerObjName — Specify the layer object name. The asterisk (*) and question mark (?) may be used as wildcard characters. (If the format layer:purpose is used, the purpose parameter is ignored.)

    layerGroup — Specify a layer group name such as allM1 or allM1Cont.

    Note:

    Calibre makes use of the layer information available from Cadence Innovus when determining which design layers to show for a given layer object name. However, Calibre RVE does not have access to a layer mapping utility, so the mapping of layer object names to layers in the Cadence Innovus database may not be correct in some cases.

    See “Example 4: RVE Show Layers for Cadence Innovus”.

  • Synopsys IC Compiler II

    layerNum — Specify a layer number in the design database.

    layerName — Specify a layer name in the design database. The asterisk (*) and question mark (?) may be used as wildcard characters.

  • Calibre DESIGNrev

    If layer names are used in the rule check comment, you must load a layer properties file in Calibre DESIGNrev to specify the correspondence between layer numbers and layer names.

    You can specify a layer and datatype pair as layer.datatype. Wildcards (*) can be used for both the layer and datatype.

    See “Example 5: Wildcard Matching in Calibre DESIGNrev”.

Description

This rule check comment specifies the layers that are displayed in the layout editor when highlighting a result from the rule check. All layers not specified in this statement are automatically hidden when any results from the check are highlighted.

You must also enable the Calibre RVE option “Only show check-dependent layers while highlighting results (hides other layers)” on theOptions tab in the Highlighting category, “DRC/DFM Highlighting” area. This option is on by default.

This command is only supported by the Calibre DESIGNrev/WORKbench, Synopsys IC Compiler, and Cadence Virtuoso layout editors.

If you use RVE Show Layers, you can use the Restore Default Layers button to toggle between displaying the default set of visible layers and the layers specified with RVE Show Layers. See the section “Restore Default Layers Toolbar”.

See “Methods to Add DRC RVE Rule Check Comments” for instructions on specifying the rule check comment. You can add the rule check comment to the rule check in the rule file or specify it in a Check Text Override (CTO) file.

Examples

Example 1: Rule Check Comment in the Rule File

The following rule check shows an example using GDS layer names for use with the Calibre DESIGNrev layout editor.

// rule file ...
min_ext_oxide_poly {
	@ minimum enclosure of poly by oxide
	@ RVE Show Layers: poly oxide 
	@ RVE Highlight Color: blue
	enc oxide poly < 1.25
}

Example 2: Rule Check Comment in the CTO File

The following example shows a DRC RVE Check Text Override File (CTO File) Format with the RVE Show Layers and RVE Highlight Color rule check comments:

# DRC RVE check text override file for XYZ
#
min_ext_oxide_poly 
RVE Show Layers: poly oxide 
RVE Highlight Color: blue
# ...

Example 3: Wildcard Matching in Cadence Virtuoso

Suppose the following layer:purpose pairs exist in the design: M1:drawing, M12:label, M144:drawing, M2:drawing, M22:label, L11:drawing, L181:drawing, L25:drawing.

The following table shows how wildcard matching is performed with Cadence Virtuoso:

Command

Matched Layers

RVE Show Layers: M1

M1:drawing

RVE Show Layers: M1*

M1:drawing, M12:label, M144:drawing

RVE Show Layers: L1*1:drawing

L11:drawing, L181:drawing

RVE Show Layers: M1*:draw*

No match.

Example 4: RVE Show Layers for Cadence Innovus

This example corresponds to Cadence Innovus 16.22. The exact results may be different depending on the database and the version of Cadence Innovus.

Command

Visible Database Layers

RVE Show Layers: M3

M3Wire&Via

RVE Show Layers: M3*

M3Wire&Via, M3Track, M3Pin, M3Blkage, M3Viol, M3Patch, VIA2Wire&Via, VIA2Pin, VIA2Blkage, VIA2Viol

RVE Show Layers: allM3

M3Wire&Via, M3Track, M3Pin, M3Blkage, M3Viol, M3Patch

RVE Show Layers: allM3Cont

VIA2Wire&Via, VIA2Track, VIA2Pin, VIA2Blkage, VIA2Viol, VIA2Patch

Example 5: Wildcard Matching in Calibre DESIGNrev

Suppose these layers exist in the design: 237, 237.2, 238, 238.2 and 238.3.

The following table shows how wildcard matching is performed with Calibre DESIGNrev:

Command

Matched Layers

RVE Show Layers: 23*.*

237.2, 238.2 and 238.3

RVE Show Layers: 23*

All layers (237, 237.2, 238, 238.2 and 238.3)

RVE Show Layers: *.2

237.2 and 238.2
