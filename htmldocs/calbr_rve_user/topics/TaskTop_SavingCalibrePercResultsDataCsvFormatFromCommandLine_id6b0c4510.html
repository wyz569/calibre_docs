Saving Calibre PERC Results Data to CSV Format from the Command Line

You can save a topological Calibre PERC results database in CSV (comma-separated value) format from the command line, without displaying the Calibre RVE interface. All results are saved to the CSV file.

Prerequisites

  • A Calibre PERC results database.

Procedure

  1. Use the following command line at the shell:
    calibre -rve -perc svdb  -output_csv csv_out.csv

    where svdb is a topological Calibre PERC results database, and csv_out.csv is the name of the output CSV file.

  2. Open csv_out.csv in a spreadsheet program that can read CSV files.

Results

All results are written to the CSV file. The Calibre RVE GUI is not displayed.

The following data columns are included by default but are not relevant for Calibre PERC databases: Flat Placement Count, Priority, Type, Fixed, Vertex Count, and Coordinates.

Tip

Data in the “Result Text” column is enclosed with quotes and often begins with one or more blank lines. Depending on your spreadsheet tool, you may need to adjust the cell format or the viewing options to properly view this data. Toggling the setting to wrap text or increasing the row height may help if you do not initially see any data in the “Result Text” column.

The inclusion of the columns “Placement” and “Placement Count” depends on the setting of “Show only one placement for each hierarchical PERC result” and whether the SVBD does in fact have multiple placements of results. This is the same behavior as seen in the Calibre RVE for PERC display. It is best to keep the same setting for “Show only one placement for each hierarchical PERC result” when comparing results from different runs.
