Creating a Check Text Override (CTO) File from the Rule File

There are several methods to create a CTO file from the rule file. You can use Calibre Interactive nmDRC, Calibre RVE for DRC, or the Calibre RVE command line. The available options differ for the different methods.

All methods for creating a CTO file add RVE Show Layers statements based on the rule checks in the DRC rule file. Layer and Layer Map statements in the rule file are considered when creating the RVE Show Layers statements. If your design tool uses a layer map file to control the layout export to GDS, the layer map file can be specified so that the layout viewer layer names are used in the CTO file. You can also use a Calibre DESIGNrev layerprops file as a layer map file.

Restrictions and Limitations

  • Without a layer map file — Layer Map statements in the rule file that have open-ended ranges for the layer or datatype are ignored and a warning is issued. The corresponding Layer statement alone is used to determine the layer number for the RVE Show Layers statement.

  • With a layer map file — See “Layer Map File Usage in Check Text Override (CTO) File Creation”.

The effect of these restrictions is shown in “Examples” in the topic “Layer Map File Usage in Check Text Override (CTO) File Creation.”
