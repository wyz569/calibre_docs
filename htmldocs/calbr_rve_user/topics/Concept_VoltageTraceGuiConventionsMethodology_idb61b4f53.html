Voltage Trace GUI Conventions and Methodology

The Voltage Trace GUI relies on the presence of voltage data from the perc::create_voltage_path command in the Calibre PERC database. The end points of the voltage trace may be affected by the options to perc::create_voltage_path. The voltage tracing method differs depending on whether you use vectored or vectorless mode.

When a net is traced through multiple levels of the hierarchy, the Trace Voltage GUI uses the net name at the highest level of the net. An example of this is shown in the “Results” section of “Tracing Multiple Voltages with the Voltage Trace GUI”, where a trace for net IN2 is requested, but the Trace Voltage GUI reports the net as N3.

Starting and Ending Points for the Voltage Trace

The trace start point (the voltage sink point) is set when you select a net and voltage using the Trace right-click menu.

The voltage trace ends when one of the conditions listed in the following table is encountered. Voltages are not propagated across the top level.

Table 1. End Point Labels for Voltage Trace GUI

Label

Condition

BREAK

Encountered a break net. Break nets are always labeled BREAK.

TOP

Encountered a top-level net that only has a single voltage. Top-level nets with a single voltage are labeled TOP even if the voltage is assigned with perc::define_net_voltage.

Voltage tracing continues through a top-level non-breaking net if the option “Allow PERC voltage tracing through top-level non-break nets” is enabled on the LVS/PERC/PEX category of the Options tab.

DEFINED VOLTAGE

Encountered a net with one of these conditions:

  • A net that is not at the top level and is defined with the perc::define_net_voltage_by_placement command.

  • A top-level net that is not a break net and has multiple voltages, where the traced voltage is defined with the perc::define_net_voltage or perc::define_net_voltage_by_placement command.

MISSING

One or more of the voltages in the set is missing for a multiple voltage trace.

BIDIR

One or more voltages are being driven from more than one direction in a multiple voltage trace. This condition stops a multiple voltage trace.
Note:

A voltage that is applied directly to a net due to a command in the rule file has the “ud” (user-defined) attribute.

Keyboard Shortcuts

The keyboard shortcuts for panning and zooming in the Internal Schematic Viewer are available in the Voltage Trace GUI; see “Navigating Tips for the Internal Schematic Viewer”.

Voltage Trace Methodology

Calibre PERC uses iterative, connectivity-based voltage propagation and does not consider resistance or current. The trace reported from output to inputs reflects at least one (but possibly more) of all possible sources of the requested output net's voltage. Specifically, the trace is not guaranteed to report all possible paths from output to input nor is it guaranteed to report the shortest possible path from output to input.

Calibre PERC can perform voltage analysis in vectored or vector-less mode. See the Calibre PERC User’s Manual for a complete discussion.

If your PERC Load statement includes the XFORM keyword, then netlist transformation is performed before any initialization or rule check procedures are performed. The schematic view in the Trace Voltage GUI reflects the transformed netlist. For example, suppose resistors R1, R2, and R3 are reduced to resistor R1. If you examine R1 in the Trace Voltage GUI, it has different connectivity than present in the unreduced netlist. Similarly, resistors R2 and R3 are not drawn in the Trace Voltage GUI because they have been reduced.
