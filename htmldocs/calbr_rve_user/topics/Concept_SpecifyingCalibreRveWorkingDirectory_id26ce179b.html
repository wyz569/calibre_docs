Specifying a Calibre RVE Working Directory

By default, the working directory for Calibre RVE is set to the directory Calibre RVE was called from. You can set the Calibre RVE working directory with the environment variable MGC_CALIBRE_DB_DIR. When MGC_CALIBRE_DB_DIR is set, the Calibre RVE file browser defaults to the directory set with MGC_CALIBRE_DB_DIR, not the directory Calibre RVE was called from.

To set the Calibre RVE working directory to rve_working_dir, do one of the following, depending on your shell:

  • In a C shell:

    % setenv MGC_CALIBRE_DB_DIR rve_working_dir

  • In Bourne or Korn shell:

    $ MGC_CALIBRE_DB_DIR=rve_working_dir $
    export MGC_CALIBRE_DB_DIR

Table 1 gives the behavior of Calibre RVE for different invocation methods when MGC_CALIBRE_DB_DIR is set.

Table 1. Calibre RVE Invocation Behavior with MGC_CALIBRE_DB_DIR Set

Invocation Method

Calibre RVE Behavior

Command line

Relative pathnames for the Calibre RVE database in the command line are relative to the working directory set by MGC_CALIBRE_DB_DIR (not relative to the current directory).

From a layout or schematic viewer

File browser opens to the directory set by MGC_CALIBRE_DB_DIR.

From Calibre Interactive

Calibre RVE first attempts to open the database given on the Outputs pane of Calibre Interactive. If the output database or directory on the Outputs pane is a relative pathname, it is evaluated relative to the Calibre Interactive run directory specified on the Rules pane.

 

Once Calibre RVE is started, the working directory is set to the directory specified by MGC_CALIBRE_DB_DIR.
