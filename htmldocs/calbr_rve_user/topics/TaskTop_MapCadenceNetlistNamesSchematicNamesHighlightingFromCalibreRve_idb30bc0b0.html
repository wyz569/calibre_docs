Map Cadence Netlist Names to Schematic Names for Highlighting from Calibre RVE

If you use Virtuoso CDL Out to export the source netlist file for use in a Calibre run, the netlist names may differ between the Cadence schematic and the netlist file seen by Calibre RVE. You can instruct Calibre RVE to use the name mapping information created by the Cadence netlisting process. This allows Calibre RVE to highlight the correct net in the Cadence Composer schematic view. This mode is enabled with the environment variable MGC_CALIBRE_ENABLE_ONM.

The source netlist used by Calibre can be exported from Cadence Composer during a Calibre Interactive run or exported separately from Cadence using File > Export > CDL.

Prerequisites

  • Set the following environment variable to enable name mapping by Calibre RVE:

    setenv MGC_CALIBRE_ENABLE_ONM 1

  • Netlisting is performed by Cadence in analog mode by doing one of the following:

    • In the Cadence Virtuoso CDL Out dialog box, set “Netlisting Mode” to Analog.

    • Set the following environment variable: 

      setenv CDS_Netlisting_Mode Analog

  • The following files and directories created by the Cadence netlist process must be kept:

    ihnl/
map/
si.env
netlist

    where netlist is the output CDL netlist file specified in the Virtuoso CDL Out dialog box or the filename specified in the Calibre Interactive field for the exported source netlist.

  • A Calibre results database from a Calibre nmLVS, parasitic extraction, or Calibre PERC run.

Procedure

  1. Open Cadence Composer and load the schematic database.
  2. In Cadence Composer, choose Calibre > Start RVE to open Calibre RVE. You may also start Calibre RVE from a Calibre Interactive session that is connected to Cadence Composer.

    Calibre RVE is automatically connected to the Cadence Virtuoso session. See “Connection to a Design Tool” for more information.

  3. In the Calibre RVE dialog box, select the correct results database and database type and open the database.
  4. In Calibre RVE, choose Setup > Options, choose the Highlighting category, and expand the “LVS/PERC/PEX Highlighting” section. Make sure the settings for “Strip SPICE instance prefix of” and “Strip SPICE device prefix of length” are correct.
  5. In Calibre RVE, choose View > Schematics > Source to open the Internal Schematic Viewer.
  6. Highlight a net or instance from Calibre RVE.

    The correct instance is highlighted in the Internal Schematic Viewer and in Cadence Composer. In the example below, instance X2 in jnTOPCELL is highlighted from the Finder in Calibre RVE. The instance in highlighted in the Internal Schematic Viewer and in Cadence Composer as I;0 in jnTest2.
