Getting Started with Calibre RVE for DFM

This section provides an overview of the Calibre RVE for DFM functionality. It provides links to additional procedural sequences or usage descriptions in pertinent manuals.

You can configure your Calibre RVE for DFM environment with the options that you want and use that configuration each time you invoke. All changes you make from the default Calibre RVE for DFM settings are automatically saved to the .rvedb File for Calibre RVE Settings. This file is accessed every time you invoke Calibre RVE. To return to a default option setting, you must restore the setting or remove the .rvedb file.

Prerequisites

  • A Calibre RVE license

  • A Calibre YieldServer license — Calibre RVE applications which open a DFM Database require a Calibre YieldServer license

  • (Optional) A Calibre nmDRC/Calibre nmDRC‑H license — required by CMP, CAA, and CFA applications in Calibre RVE for DFM

See the section “Calibre RVE System Requirements” for general information on supported platforms, licensing, and connecting to supported layout and schematic editors.

Procedure

  1. Open a DFM Database in Calibre RVE, as described in “Calibre RVE Invocation Methods”.
  2. In the Calibre RVE session window, choose Setup > Options.

    The Calibre RVE Options tab displays as shown in Figure 1.

  3. Choose the Highlighting category and specify the layout viewer behavior for viewing DFM errors. The default is to not change the cell view after highlighting. For more information on each of the available options, see “Options Tab and Common Calibre RVE Option Panes”.
  4. Enable the “Clear existing highlights before showing new highlights” option if desired. This is equivalent to selecting “Clear Highlights” in the Highlight menu before each new highlight.
  5. Choose the Design Tools category to specify layout viewer options. For more information on each of the available options, see “Setup Design Tools and Connection to RVE Pane”.

    • Your design tool.

    • The hostname the layout viewer is running on. Generally, this is localhost if the layout viewer is running on the same system as Calibre RVE.

    • The socket port the layout viewer listens on; default is 9189. Refer to the section “Setting the Calibre RVE Socket Port for a Layout Viewer” for information on customizing the port.

    • The intermediate file into which you want your query results stored. The default filename is query_results. Confirm that the path to the specified file is accessible to the layout viewer, especially if the layout viewer is running on another host.

    • If the layout viewer is running but you do not have a connection to Calibre RVE, click the connect button () to establish the connection. Make sure the Hostname and Socket Number fields are correct.

  6. To open a window to a Tcl shell, choose View > Terminal from the menu.
  7. Visit the other options in the Options tab and note the default settings. Change any settings as needed.

    For more information on the DFM specific options, see “Setup DFM Browser Options Pane”. For more information on the options common to all Calibre RVE applications, see “Options Tab and Common Calibre RVE Option Panes”.

  8. Click Apply to enable your changes. An asterisk next to a section in the option tree indicates that a change has been made. Your changes are saved in the .rvedb file.
