OrCheckFilter

Key Value for DRC HTML Report Configuration File, filter category

Usage: Optional. Valid in all report types, and in Common section.

Inheritance: Filter Inheritance. If the GroupBy key references another section, then the key and value are read from that section and any locally specified value is ignored.

Include only the specified check(s). OR logic is used if multiple checks are specified with a comma-separated list of values.

Allowed Values

[!] name[, [!] name] …

The name parameter is a check name or a string with wildcards (*). Multiple check names can be specified with a comma-separated list. OR logic is used if multiple check names are specified.

The ! character may be used before a check name to indicate not matched. There must be a space after the ! character.

Examples

OrCheckFilter = metal2_ext
OrCheckFilter = metal*_ext, metal*_int
