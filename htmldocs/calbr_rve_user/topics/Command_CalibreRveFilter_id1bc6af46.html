calibre -rve Command Line for Calibre RVE Batch Filtering

Command line invocation for filtering results in Calibre RVE in batch mode.

You can filter results based on their properties, waived states, locations in their cells, and other criteria, then save them to new results databases. Full usage is given by calibre -rve -filter -help.

Usage

calibre -rve -drc rdb_input [-split] [-verbose] [-useGMT]
{ -filter       
   [-NOT]
   [-cell "cell1 [cell2 ...]"]
   [-check "check1 [check2  ...]"]
   [-property "prop"]
   [-property "prop { < | <= | == | != | >= | > } value"]
   [-property "minval { <= | < } prop { <= | < } maxval"]
   [-include_area "cellname coords"]
   [-exclude_area "cellname coords"]
   [ -waived | -unwaived ]
   [-waived_by "user1 [user2 ...]"]
   [-waived_comment "comment"]
   [-waived_date "{ < | <= | == | != | >= | > } date"]
   [-noempty_checks]
   [-OR filter_spec]
   [-other]
-output "rdb_output"} ...

Arguments

rdb_input

A required argument specifying the results database (RDB) file containing the results to be filtered.

-split

Optional. When specified, each result is only included with the output file of the first matching filter. This argument ensures a result does not appear in multiple output files.

The -split option is always used when -other is specified.

-verbose

Optional. When specified, writes a check-by-check report of the original result count and the filtered result count for each output RDB to standard output.

-useGMT

Optional. When specified, the date provided for the -waived_date argument is interpreted in GMT. If not specified, date is interpreted in the local timezone.

-filter

A required argument that precedes a filter specification. Multiple filter types can be specified based on properties, their waived states, and where they are located in different cells.

Different filter types in the same filter specification (such as using both -cell and -waived) are combined together as an “AND” operation. If the same filter type is used multiple times, they are combined together as an “OR” operation; see Example 1.

You can specify multiple filter specifications in a single command.

-NOT

Optional. When specified, reverses the next filter criteria. For example:

-filter -NOT -cell cell1 -property a1

returns results outside of cell1 and that have the a1 property. To return results outside of cell1 and without that property, specify -NOT for both:

-filter -NOT -cell cell1 -NOT -property a1
-cell "cell1 [cell2 ...]"

Optional. When specified, returns results in the specified cell names. You can specify multiple cells as a space-separated list of cell names. Cell names can be specified with wildcards (? and *). If specifying multiple cells or a cell with a space in its name, enclose the cell name(s) in quotation marks (“” or ‘’).

-check "check1 [check2 ...]"

Optional. When specified, returns results that came from the specified check names. You can specify multiple checks as a space-separated list of check names. Check names can be specified with wildcards (? and *). If specifying multiple checks or a check with a space in its name, enclose the check name(s) in quotation marks (“” or ‘’).

-property "prop"

Optional. When specified, returns results that have any value of that property. Property names can be specified with wildcards (? and *). If the property has a space in its name, enclose the property name in quotation marks (“” or ‘’).

-property "prop { < | <= | == | != | >= | > } value"

Optional. When specified, returns results that fulfill the logic defined by the specified operator and value. Separate the property name, operator, and value with spaces. Enclose the logic statement in quotation marks (“” or ‘’).

-property "minval { <= | < } prop { <= | < } maxval"

Optional. When specified, returns results within the bounds defined by the operators and the provided minvalue and maxvalue. For example:

-property "200 < CD <= 300"

returns results where the value of the property “CD” is greater than 200 and less than or equal to 300.

Enclose the logic statement in quotation marks.

-include_area "cellname coords"

Optional. When specified, filters results that are in cellname and in a region defined by coords. coords can be defined as a rectangle using the four coordinates of the lower-left and upper-right vertices (min-x min-y max-x max-y), or as a polygon with the coordinates representing x and y values of each vertex.

If cellname is the top cell, results may be returned from the top cell or in any sub-cell in the defined region.

Separate cellname and coords with spaces and enclose them in quotation marks (“” or ‘’).

-exclude_area "cellname coords"

Optional. When specified, filters results that are NOT in cellname and in a region defined by coords. coords can be defined as a rectangle using the four coordinates of the lower-left and upper-right vertices (min-x min-y max-x max-y), or as a polygon with the coordinates representing x and y values of each vertex.

-exclude_area is equivalent to using the -NOT argument with -include_area.

Separate cellname and coords with spaces and enclose them in quotation marks (“” or ‘’).

-waived | -unwaived

Optional. When specified, filters results based on their waived state. -unwaived is equivalent to using the -NOT argument with -waived.

-waived_by "user1 [user2 ...]"

Optional. When specified, filters results based on the username who waived them. You can specify multiple usernames. Usernames can be specified with wildcards (? and *). If specifying multiple users or a user with a space in their name, enclose the username(s) in quotation marks (“” or ‘’).

-waived_comment "comment"

Optional. When specified, filters results based on whether they have a designated waiver comment. Waiver comments can be specified with wildcards (? and *). If comment contains a space, enclose it in quotation marks (“” or ‘’).

-noempty_checks

Optional. When specified, empty checks are not written into the corresponding output RDB file when using the -cell, -include_area, -exclude_area, or -property arguments. By default, empty checks are written to the RDB file.

-OR filter_spec

Optional. When specified, allows a set of different filters to be specified for the same output RDB file, essentially combining multiple filters into one. For example:

calibre -rve -drc input.rdb
	-filter -cell “cell1” -property “a1” -OR
	-cell “cell2” -property “b1”
	-output “cell1ORcell2.rdb”

produces an output cell1ORcell2.rdb containing all results in cell1 with the property a1 and all results in cell2 with the property b1.

-other

Optional. This argument is intended to work with -split and, when specified, includes all results not matching any previous filter. See Example 2.

If -other is used, -split is implied and always used.

-output “rdb_output”

A required argument specifying the output RDB for the filter. Each filter must have a corresponding -output argument, unless merged together with the -OR argument. If rdb_output contains a space in its name, enclose it in quotation marks (“” or ‘’).

Description

The Calibre RVE command with the -drc, -filter, and -output options is used to filter Calibre DRC results based on various criteria and save them to new results databases. Filters are specified with -filter and can each contain multiple criteria. Each filter must have one output RDB file specified with the -output option.

When defining a filter, if the same filter type is specified multiple times, the filter criteria is applied using OR logic; see Example 1. When different filter types are specified, such as filtering on a check name and a property, the filter is applied using AND logic unless the -OR option is included; see Example 7.

Examples

Example 1

Match multiple checknames using wildcards:

calibre -rve -drc input.rdb \
-filter -check "OD* PO*" -output rve_filter.rdb

When this command is run, the output file has the results created by checks with names that begin with “OD” or “PO”. You can also run:

calibre -rve -drc input.rdb \
-filter -check OD* -check PO* -output rve_filter.rdb

Example 2

Split into three databases using -other:

calibre -rve -drc drc.results -split \
	-filter -cell cell1 -output cell1.rdb \
	-filter -cell cell2 -output cell2.rdb \
	-filter -other -output other.rdb

Results from cell1 go into cell1.rdb, results from cell2 go to cell2.rdb, and all other results not in those cells go to other.rdb.

Example 3

Area filter by a rectangle:

calibre -rve -drc input.rdb \
	-filter -area "cella 10 10 100 100" -output arearect.rdb

Results are filtered based on whether they are in cella and in the rectangular region defined by the lower-left vertex with coordinates (10, 10) and the upper-right vertex with coordinates (100, 100).

If cella is the top cell, returned results may be from any sub-cell in the defined rectangular area.

Example 4

Area filter by a polygon:

calibre -rve -drc input.rdb \
	-filter -area "cellb 10 10 10 50 50 50 10 10" -output areapoly.rdb

Results are filtered based on whether they are in cellb and in the triangular region defined by the vertices (10, 10), (10, 50), and (50, 50). The region’s boundaries are defined by lines between sequential pairs of coordinates.

Example 5

Filter using -NOT:

calibre -rve -drc input.rdb \
	-filter -cell cell4 -NOT -waived_by jdoe -output notjdoe.rdb

Results are filtered based on if they are in cell4 and were not waived by user jdoe.

Example 6

Filter by existing property and property range:

calibre -rve -drc input.rdb \
	-filter -property a1 -property "200 <= b1 < 300" \
	-output a1b1.rdb

Results are filtered based on whether they possess the property a1 and whether their value for the property b1 is greater than or equal to 200 and less than 300.

Example 7

Filter using -OR:

calibre -rve -drc input.rdb \
	-filter -check default -property "71.72 <= ERROR(%) <= 76.89" -OR \
	-check default -NOT -property "Layer == CONT" \
	-output filtered_by_property.rdb

These results come from a PERC LDL check that includes the properties “Layer” and “ERROR(%)”. The generated RDB file contains the filtered results that came from the check default (-check default) and are either not on the layer CONT (-NOT -property “Layer == CONT”) or have an error percentage within the defined range (-property “71.72 <= ERROR(%) <= 76.89”).
