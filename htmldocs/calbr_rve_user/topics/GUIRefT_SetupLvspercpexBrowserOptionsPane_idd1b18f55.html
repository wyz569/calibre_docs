Setup LVS/PERC/PEX Browser Options Pane

Access with: Setup > Options, then choose the LVS/PERC/PEX category

This option pane controls settings for the results browsers in LVS, PERC, and PEX sessions.

Fields

Table 1. Setup LVS/PERC/PEX Browser Options Pane Contents

Item

Description

Views

View to display on startup

Selects the tabs that are shown on startup:

  • Auto — Automatically select the view; this is the default.

  • LVS/PERC/PEX Results

  • Extraction Results

View Extraction Results by

Selects the sorting method for connectivity extraction results:

  • Cell

  • Warning Type

View Comparison Results by

Selects the sorting method for comparison results:

  • Cell

  • Error Type

  • Fix Type

View PERC Results by

Selects the sorting method for comparison results:

  • Cell

  • Rule

Show Detailed Results

Selects the display preference for results in the Extraction, Comparison, and PERC Results tabs. This option affects how many results are shown at once. For the Extraction and Comparison Results tabs it affects the number of results shown in the detailed view (bottom pane, below tree view); the tree view always displays all results. For the PERC Results tab, it affects the number of results shown in the Result Data pane.

  • All — Show all results.

  • In groups of num — Show only num results at a time.

A message is shown after the last result to display the next group of results.

The “All” setting is not recommended for databases with a very large number of results, as it can result in long load times to populate the detailed views.

Show clean cells on startup

Sets the default behavior for showing clean cells. This setting applies to Comparison Results and PERC Results with the following behavior:

LVS Comparison Results

  • Enabled — Always show clean cells in the tree view when a database is opened, and enable View > Tree Options > Show Clean Cells.

  • Disabled — Use the setting View > Tree Options > Show Clean Cells to determine if clean cells are shown.

Calibre PERC and PERC LDL Results

“Show clean cells on startup” is only used in Calibre RVE for PERC the first time a results database is opened. After the first open of a results database, database-specific options are saved and restored when you reopen the database. See “Saving Calibre RVE Settings and Options” for more information.

Open tree nodes on single-click

Sets the method for opening nodes in the tree view. This option is on by default. A single click in the node name in the tree view expands the node. If this option is disabled, a double-click in the node name expands the node.

Show only one placement for each hierarchical PERC result

Specifies whether only one placement for each hierarchical result is displayed when loading a database in Calibre RVE for PERC. See these topics for important details:

  • “Speeding Up Database Loading by Viewing One Placement of Results in Calibre PERC”

  • “Viewing Separate Placements of a Result in the PERC Results Tab”

Show LVS/PERC runtime errors and warning on startup

Specifies whether to display a pop-up dialog box with information on runtime errors when the database is loaded.

The LVS or PERC Runtime Errors button is added to the display if runtime errors or warnings exist, regardless of the setting for this option. Click the button to view the errors at any time.

See “LVS Runtime Errors and Warnings Dialog Box” and “PERC Runtime Errors and Warnings Dialog Box”.

Specify custom message to be appended at the end of LVS/PERC runtime errors

As described.

Show PERC voltage paths in a new Voltage Trace window

Specifies whether to open a new window for each voltage trace. If disabled, the current window is closed before displaying a new trace.

See “Using the Voltage Trace GUI in Calibre RVE for PERC”

Allow PERC voltage tracing through top-level non-break nets

Specifies voltage tracing behavior when a top-level non-breaking net is encountered.

If this option is checked, tracing continues through a top-level non-breaking net until a break condition or a voltage source is encountered. When this option is unchecked, voltage tracing halts when a top-level port, a break condition, or a voltage source is encountered.

Ask to choose closest coordinate in PEX P2P

This option applies when selecting coordinates in the layout viewer for point-to-point resistance calculations. The option is off by default.

When enabled, a pop-up warning is displayed if coordinate substitution is going to take place because the specified coordinate does not coincide with the end point of an extracted resistor. See “Considerations When Specifying Coordinate Locations for Point-to-point Calculations” for more details.

Use extracted PDB for calculating PEX P2P

Use the extracted Parasitic Database (PDB) when calculating Point-to-point (P2P) values. This is the default.

When this setting is off, the reduced PDB is used.

Calibre RVE automatically reloads open views when this setting is changed.

Use the layout net name in PEX detailed parasitics tab

Use the layout net name as the label for the PEX detailed parasitics tab. This option is on by default. If the option is unchecked, the source names are used for the tab label if source names are available.

Use Viewer Layer Map file

Specify a layer map file in order to display both the drawn layer/purpose pair and the Calibre layer name within the Calibre RVE display. The layer map file has the following format:

layer purpose gds_number gds_datatype

where layer and purpose are the drawn layer name and purpose used in the layout viewer.

This option affects the display of original layer names; see “Viewing Drawn Layer/Purpose Pairs in the Calibre RVE for LVS Display” for specific information. If open, the affected displays must be closed and re-opened in order to see the effect of the option. This option does not affect highlighting.

Show Info Pane

Show the Info Pane.

Show Info Pane in new window

Info for each design element is displayed in a separate undocked window rather than as a new entry in the Info pane.

Show Deviceless cells

When enabled, show deviceless cells while tracing nets inside instances and while browsing devices and ports on nets.

Show Pseudo cells

When enabled, show pseudo cells (named ICV_*) while tracing nets inside instances and while browsing devices and ports on nets.

Show Navigator Pane

Show the Navigator Pane.
