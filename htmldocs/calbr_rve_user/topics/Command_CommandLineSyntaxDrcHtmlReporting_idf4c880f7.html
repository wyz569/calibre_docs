Command Line Syntax for DRC HTML Reporting

You can create a DRC HTML report from the shell. Use the “calibre -rve” command with appropriate options.

Usage

calibre -rve -drc {drc_db_file …| ‑input db_to_layout_map_file }
‑report html_config.ini ‑outputdir output_dir
[‑summaryfile drc_summary_rpt]

Arguments

-rve -drc

The switches ‑rve and ‑drc are required.

drc_db_file

Specifies one or more DRC results databases to be used for the DRC HTML report. You must specify either drc_db_file or the switch and value ‑input db_to_layout_map_file.

A results database specified on the command line is assigned the RDB type of “General” (see the ApplyTo keyword).

‑input db_to_layout_map_file

Switch and value specifying the path to the DRC HTML Report Database Map File.

The database map file specifies one or more DRC results databases to process for the DRC HTML report, with optional mapping to a layout database. You must specify either drc_db_file or the switch and value ‑input db_to_layout_map_file.

‑report html_config.ini

Required switch and value specifying the configuration file used to create the DRC HTML report. See “DRC HTML Report Configuration File Format”.

‑outputdir output_dir

Required switch and value specifying the path to the output directory.

‑summaryfile drc_summary_rpt

Optional switch and value specifying a DRC summary report created by the DRC Summary Report specification statement in the SVRF rule file.

If ‑summaryfile is specified, then a table with summary information taken from drc_summary_rpt is included at the beginning of the DRC HTML report.

Description

DRC HTML reporting creates an HTML formatted report based on views in Calibre RVE for DRC. The report contents and format are controlled by the DRC HTML Report Configuration File Format specified with the ‑report argument.

A DRC results database file is required input; this is specified on the command line after the ‑drc switch or with ‑input db_to_layout_map_file. If both drc_db_file and db_to_layout_map_file are specified, then db_to_layout_map_file takes precedence and drc_db_file is ignored.

Side RDBs that are included in the main results database are processed automatically, as described in “Automatically Process Side (Auxiliary) RDBs”.

If result highlighting or colormaps are specified in the configuration file, then Calibre DESIGNrev is invoked to create the highlight image or colormap. The ‑input switch specifies the DRC HTML Report Database Map File; this file specifies the layout database to be used to create the result highlight or colormap.

DRC HTML reporting creates a report in the specified output directory (output_dir ).You can specify HTML (the default) or MHTML report format with the OutputFormat key. The report is named index.htm for HTML reports and report.mht for MHTML reports. All created files are saved in output_dir . The output directory is also used for temporary files used during the report creation process. You can customize the report as described in the section “Output Format and Customization for DRC HTML Reports”.

DRC HTML reporting reads the DRC RVE Check Text Override File (CTO File) Format, if present. The check text override should be named drc_db_file.cto, where drc_db_file is the DRC results database, or defined with the environment variable MGC_RVE_CTO_FILE. See the section “DRC Rule Check Comments for Calibre RVE” for more information.

If -summaryfile is specified, then the summary report and the rule file specified in the summary report are copied into the output directory (output_dir) if the files are readable. Files specified with Include statements in the rule file are not copied. If the summary report and main rule file are successfully copied, the links to these files in the DRC HTML report point to the files in output_dir, otherwise the links point to the original pathnames.

Database triggers are ignored by DRC HTML reporting.

Options in the .rvedb file (Calibre RVE configuration file) are ignored. If there is not a corresponding setting in the DRC HTML report configuration file, the default Calibre RVE option is used. The key ClassifyShapes is on by default in DRC HTML reporting, although the corresponding setting in Calibre RVE is off.
