RulesToInclude

Key value for Configuration File in DFM HTML Reporting

Supported Report Types/Sections: ByCellColormap, ByCellHistogram, ByErrorColormap, ByErrorHighlight, ByErrorHistogram, ByErrorResults, ByErrorResultsSummary, ByWindowColormap, ByWindowHistogram, CAPlot, CellSummary, MemoryPlot, WindowMemoryPlot, WindowSummary

Required key name that specifies a list of rules to include in the report.

Syntax

RulesToInclude = group_names

Parameters

group_names

A required comma-separated list of group names, or a reference to a section that outputs groups. Can directly reference other sections.

Specifying an asterisk (*) include all rules. The default value is “  ” (empty list).

Multi-character wildcards (*) and single-character wildcards (?) are supported. Wildcards can be used any number of times anywhere within a string. The “*” wildcard matches zero or more characters, and the “?” wildcard matches exactly one character.

The default value is “*” for all rules.

When wildcards are used in a list, matches to the database are done in the order specified in the list. For example, if the database contains items AB, AC, BC, CD, and BE, and the list contains “B*, A*”, the order of items returned is BC, BE, AB, AC.
