Setup Database Filters and Triggers Options Pane

Access with: Setup > Options, then choose the Databases category

Use this pane to set filters for the database name and the unzip directory when you use a compressed database. Trigger functions that execute on certain database actions are also set on the Databases pane, along with options related to loading databases.

Fields

Table 1. Setup Database Filters and Triggers Options Pane Contents

Item

Comment

Database Filters

Enter a filename filter for use with the file browser for the specified database type.

The following expressions are allowed:

? — Match any single character.

* — Match any sequence of zero or more characters.

a  b … — Matches any of the listed strings a, b ….

    DRC Databases

Default: *

    DFM Databases

Default: *dfmdb

    LVS/PERC/PEX DBs

Default: svdb

    Spice Netlist Files

Default: *

Unzip Directory

Specify the target extraction directory for DRC databases and SPICE netlist files compressed with gzip.

When reading compressed SPICE files, the parent directory must be writable.

If the parent directory is not writable when reading a compressed DRC database, the database is still opened but results cannot be specified as waived or fixed.

DRC Database Triggers

Calibre RVE for DRC offers additional options for setting triggers around database access events.

    Pre-Open

Enable a trigger to run immediately before opening a database.

The name of the database file is appended to the list of parameters given to the trigger.

Equivalent environment variable: MGC_RVE_PRE_OPEN_TRIGGER

    Post-Open

Enable a trigger to run after loading a database is complete.

The name of the database file is appended to the list of parameters given to the trigger.

Equivalent environment variable: MGC_RVE_DB_OPEN_TRIGGER

    Post-Close

Enable a trigger to run after a database is closed.

The name of the database file is appended to the list of parameters given to the trigger.

Equivalent environment variable: MGC_RVE_DB_CLOSE_TRIGGER

Database Loading

If DRC/LVS/PERC/PEX database is modified

Select action to take when a database is modified.

  • Reload — Automatically reload the database.

  • Ignore — For DRC databases, continue operation without reloading the database. For non-DRC databases, the “Ignore” option is not recognized and is treated as “Ask me”.

  • Ask me — Prompt to reload the database.

In addition to specifying the behavior for the main database, this setting affects the reload behavior of most individual files in the database, such as the shorts database or netlist.

Load LVS/PERC/PEX DBs

Select when to load layout and cross-reference databases for LVS/PERC/PEX.

Automatically open ERC databases on startup.

As described; this option is disabled by default.
