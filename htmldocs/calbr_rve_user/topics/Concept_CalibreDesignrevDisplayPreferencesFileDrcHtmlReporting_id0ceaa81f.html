Calibre DESIGNrev Display Preferences File for DRC HTML Reporting

The layout images used in DRC HTML reporting are created by Calibre DESIGNrev or Calibre MDPview, both of which use a preferences file to control display options. A preferences file that is optimized for creating DRC HTML reports is used by default. You can also specify your own preferences file, and specify to use settings optimized for faster processing.

See “preferences File Format” in the Calibre DESIGNrev Layout Viewer User’s Manual for complete information.

You can specify preferences files with the methods described in the following table. Options from different files are combined. If settings from different files conflict, the option in the file with the highest precedence is used.

Table 1. preferences File Locations and Precedence for DRC HTML Reporting

Precedence

preferences File Description

1 (highest)

Specify with: PaletteSnapshotPerCheck=1

Description: Tool-created file with optimized settings for faster processing.

Location: Directory DrvPrefsOverride in the output directory for the HTML report.

2

Specify with:DrvOptionsFilePath=path

Description: A custom preferences file.

Location: path

3

Specify with: MGC_CWB_CONFIG_DIRS environment variable

Description: See “Multilevel Configuration” in the Calibre DESIGNrev Layout Viewers User’s Manual.

Location: As specified with MGC_CWB_CONFIG_DIRS

4 (lowest)

Specify with: no specification (the default)

Description: The default preferences file, which is optimized for DRC HTML reports.

Location: $CALIBRE_HOME/shared/pkgs/icv.$(VCO)/examples/rve/templates/batchreporting/

where $VCO is returned by the utility $CALIBRE_HOME/bin/calibre_vco and indicates the vendor, CPU, and operating system.
Note:

The preferences file in $HOME/.calibrewb_workspace is not used unless MGC_CWB_CONFIG_DIRS specifies that directory.

An example is given at the end of this section.

About the Default Preferences File

The default preferences file for DRC HTML reporting sets the following option:

prefs_layerpalette_filterBy visible

This option causes only visible layers to be included in the Calibre DESIGNrev layer legend. When this option is not set, hidden layers are listed with a strikethrough in the layer legend. Setting this option for DRC HTML reports creates a better formatted report, and is especially useful when the RVE Show Layers check text comment is used; see “Specifying Highlight Colors in DRC HTML Reporting”.

To override this setting and include invisible layers in the layer legend, include the following line in your preferences file for DRC HTML reporting: 

prefs_layerpalette_filterBy none

Use DrvOptionsFilePath to specify the preferences file.

Example

This example creates a preferences file which turns off grid lines and changes the background color.

  1. Create a text file with the following lines: 

    prefs_grid_style -1
prefs_view_bg gray25

  2. Save the text file from Step 1 with the filename $HOME/calibre_HTML/preferences.

  3. Specify the key DrvOptionsFilePath in the Common section of the HTML report configuration file:

    [Common]
DrvOptionsFilePath = $HOME/calibre_HTML
