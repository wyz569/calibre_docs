Rule File Requirements for Examining Results with Calibre RVE for LVS

Calibre RVE for LVS requires a standard verification database (SVDB). The SVDB is specified in the rule file and is created as part of your Calibre nmLVS run. Certain rule file statements and options are required in order to properly set up the results database.

The following rule file settings enable the features needed for your Calibre RVE for LVS debug session.

  • Set case sensitivity correctly; see “Case Sensitivity in Calibre RVE” for information.

  • Make sure the following statement is in your rule file:

    Mask SVDB Directory svdb QUERY [SI] ...

    where svdb is the path to your SVDB directory. The QUERY option is recommended; you may use additional options.

    The SI option is required for short verification in Calibre RVE for LVS.

  • (Optional) Include the following statements in order to run interactive short isolation and verification in Calibre RVE for LVS:

    LVS Isolate Shorts YES BY CELL BY LAYER

    To enable short verification, you must also include the SI option in the Mask SVDB Directory statement, as follows:

    Mask SVDB Directory svdb QUERY SI ...

  • (Optional) Calibre RVE for LVS can display Design Fix Suggestions for certain types of common LVS problems. This option is enabled with the following SVRF statement:

    LVS Report Option FX
    Tip

    You can use Calibre Interactive nmLVS to set all of the above options.
