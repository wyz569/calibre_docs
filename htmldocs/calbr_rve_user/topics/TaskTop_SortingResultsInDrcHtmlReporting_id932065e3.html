Sorting Results in DRC HTML Reporting

You use the SortBy key to sort the results in your DRC HTML report. Sorting can be by the available columns or by a property value. You can use the Priority value (set with the RVE Priority check text comment) to specify a custom sort order for rule checks.

Prerequisites

  • A DRC HTML Report Configuration File.

Procedure

  1. Specify SortBy depending on the report type and desired sorting method:

    To do this …

    Do the following …

    Sort by a column value

    Set SortBy to the column name, where the available columns depend on the report type.

    For example, in a DRCResults Report: 

    SortBy = “Waived Comment”

    Sort by a property value in a DRCResults report

    Set SortBy to the property name. The #SC calculated property is included in DRC HTML reports by default.

    Sort alphabetically by the rule check name

    For a DRCResults report:

    SortBy = Check

    In reports other than DRCResults, you can sort by Check if the GroupBy hierarchy is set the Check.

    (Results are sorted in the internal database order by default in a DRCResults report.)

    Specify a custom sort order for rule checks

    Specify the following:

    SortBy = Priority

    Set the desired sort order with the rule check comment RVE Priority in the rule file or a Check Text Override (CTO) file. A Priority value of 1 has the highest sort order. A value of 0 or an unset priority has the lowest sort order. A priority value of 0 is displayed as blank in the HTML report.
  2. If desired, set SortOrder to Descending. The default is Ascending.
