Waiver Export to a Cadence Virtuoso Database for Calibre Auto-Waivers

Calibre RVE for DRC can export waiver cells and shapes to a Cadence Virtuoso database. This is useful if you want to keep the waiver shapes in the native design database format rather than save the waiver shapes to a GDS or OASIS format database.

This section describes the details of the export process, how cell naming conflicts are resolved, and how the streamin template is used to control the export from Calibre RVE to Cadence Virtuoso.

Waiver Export to Cadence Virtuoso

An active connection to a Cadence Virtuoso session is required, and a results database with waived results open in Calibre RVE. Waiver export is controlled with the Export Waived Results dialog box, which is opened by choosing Tools > Export Waivers in Calibre RVE. The basic process for exporting waiver shapes to a Cadence Virtuoso database is given in “Exporting Waiver Shapes to a Cadence Virtuoso Database Using Calibre RVE” in the Calibre Auto-Waivers User’s and Reference Manual.

The following controls are not available in the Export Waived Results dialog box when the “Export format” is “Virtuoso”:

  • “Merge from file”

  • “Allow writing of shapes into existing waiver cells”

Calibre RVE takes the following actions after Export is clicked in the Export Waived Results dialog box:

  1. Calibre RVE creates a GDS file containing the waiver cells and shapes. Waiver cells are named according to the standard Calibre Auto-Waivers cell naming process; see “Waiver Cell Description” in the Calibre Auto-Waivers User’s and Reference Manual.

  2. Calibre RVE generates a streamin template file as follows, depending on the setting of “Use Streamin Template” in the Export Waived Results dialog box:

    • Enabled — Calibre RVE uses all settings in the specified streamin template file except the input file and library name. Calibre RVE fills in the values for the library name and input file.

    • Disabled — Calibre RVE generates a streamin template file. For an OpenAccess database, the streamin template contains only the strmFile and library fields. For a CDBA database, the streamin template contains inFile, libName, scale (0.001), and units (micron).

  3. The GDS file of waivers shapes is streamed into Cadence Virtuoso, using either 'pipo' (CDBA) or 'strmin' (OA), and the streamin template file (see Step 2).

    The waiver cells are created in the “Waiver library” specified in the Export Waived Results dialog box. The waivers are created with the layout view. If the waiver library does not exist, a prompt is displayed with the choice to create the library or abort the waiver export.

    Cell name conflicts — If the streamin process would result in a waiver cell which already exists in the waiver library, then the new waiver cell is created with the suffix _N, where N is an integer. New waiver shapes cannot be merged into an existing waiver cell.

    For example, suppose an existing waiver library contains the waiver cell Waive$wv$_checkB_in_cellA. If new results for checkB in cellA are waived and exported to Cadence Virtuoso, then the new waiver shapes are placed in a new waiver cell named Waive$wv$_checkB_in_cellA_0.

  4. If errors or warnings occur during the streamin process, they are reported in a dialog box. A prompt is displayed with the choice to proceed with placing the waiver cells or abort the waiver export.

  5. Calibre RVE places the waiver cells in the correct cell, view, and library in Cadence Virtuoso. Calibre traverses the cell hierarchy in the open design, starting from the top cell location indicated by “Cell library,” and places the waiver cells at the correct locations in the design.

    If duplicate versions of a cell having waivers are found in the hierarchy, Calibre RVE prompts the user to choose which cell should receive the placement of the waiver cells. This may happen, for example, if the waiver library contains waived results from cellA (a waiver cell named Waive$wv$_checkB_in_cellA, for example) and the hierarchy contains versions of cellA from two different libraries.

    If a cell having waivers is not found in the hierarchy, Calibre RVE attempts to match it with one or more cells found in the hierarchy, and asks the user whether to use one of those cells, or abandon the operation. This may happen when cells of the same name exist in the library and are renamed during streamout to Calibre, or if waiver cells are renamed to resolve name conflicts, as in Step 3. For example, if two versions of cellB exist in the design, the streamout to Calibre may result in a cell named cellB_2. If DRC results for checkB are waived in cellB_2, then the waiver cell is named Waive$wv$_checkB_in_cellB_2. When Calibre RVE attempts to place this waiver cell, it cannot find cellB_2 in the Cadence Virtuoso design, but does find two versions of cellB, and prompts the user to select which version of the cell to place the waiver cell in.
