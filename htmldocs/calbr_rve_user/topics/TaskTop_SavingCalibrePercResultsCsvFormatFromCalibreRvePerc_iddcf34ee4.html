Saving Calibre PERC Results to CSV Format from Calibre RVE for PERC

Calibre RVE for PERC can save results data to a CSV (comma-separated value) formatted file; the file can then be imported into most spreadsheet tools for later analysis. You can filter the data by applying a Calibre RVE result filter; only data that meets the applied filter is saved. In addition, you have the option of saving all displayed results, selected results, or property statistics.

Prerequisites

  • A Calibre PERC results database is open in Calibre RVE for PERC.

Procedure

  1. (Optional) If desired, apply a result filter, as described in the section “Result Filters in Calibre RVE for PERC”.

    You can use this step to filter the results so that only the subset you want to analyze is saved. Only results that satisfy the applied result filter are saved.

  2. (Optional) If you want to manually select specific results to save, use Shift- and Ctrl‑click to select the results to save.
  3. Choose File > Save Results As.
  4. Enter the filename for the saved data, including a file extension, if desired.
  5. Select the results you want to save in the Save dropdown list:

    • Displayed Results — Save all results shown in the Calibre RVE display. If a result filter is applied, only results that satisfy the filter are saved. Clean rule checks and clean cells are not saved.

    • Selected Results — Save only results selected with Shift- and Ctrl-click (Step 2).

      Empty rule checks are saved if no results are selected in the result view and at least one empty check is selected in the tree view; empty checks are saved with a zero in the first column of the CSV file.

Results

Results are saved to the CSV file according to the selection in Step 5.

Tip

Data in the “Result Text” column is enclosed with quotes and often begins with one or more blank lines. Depending on your spreadsheet tool, you may need to adjust the cell format or the viewing options to properly view this data. Toggling the setting to wrap text or increasing the row height may help if you do not initially see any data in the “Result Text” column.

The inclusion of the columns “Placement” and “Placement Count” depends on the setting of “Show only one placement for each hierarchical PERC result” and whether the SVBD does in fact have multiple placements of results. This is the same behavior as seen in the Calibre RVE for PERC display. It is best to keep the same setting for “Show only one placement for each hierarchical PERC result” when comparing results from different runs. For information on this option, see “Viewing Separate Placements of a Result in the PERC Results Tab” and “Speeding Up Database Loading by Viewing One Placement of Results in Calibre PERC”.

The data columns Flat Placement Count and Priority are included by default but are not relevant for Calibre PERC databases.
