Database Requirements for Calibre RVE

Calibre RVE requires the certain database files depending on the Calibre RVE application.

The database requirements are as follows:

  • Calibre RVE for DRC— a results database in ASCII format, which can be any of the following types:

    • DRC Results Database in ASCII format. See DRC Results Database in the SVRF Manual.

    • ERC Results Database in ASCII format. See ERC Results Database in the SVRF Manual.

    • Auxiliary databases (RDBs). Auxiliary databases can be created by the DRC commands Net Area Ratio, Density, Layout Input Exception, and by several DFM commands. See “Viewing Auxiliary Databases”.

  • Calibre RVE for DFM — DFM results output in one of two formats:

    • DFM Database. See DFM Database in the SVRF Manual.

    • ASCII format results database. See DRC Results Database in the SVRF Manual.

  • Calibre RVE for LVS and PEX — A standard verification database (SVDB). See the command Mask SVDB Directory in the SVRF Manual. The SVDB can include various database types, depending on the rule file and the type of Calibre run.

    Various cross-reference databases must be present in the SVDB directory for result viewing with Calibre RVE. You can specify the creation of these databases as follows, depending on how you run Calibre:

    • From the command line — Include the QUERY keyword in the Mask SVDB Directory command in your rule file.

    • From Calibre Interactive — Enable the setting “Generate ASCII cross-reference files” on the Outputs pane in Calibre Interactive.

  • Calibre RVE for PERC — A standard verification database (SVDB) created by the PERC run (see the entry for Calibre RVE for LVS and PEX), or a DFM database created by a PERC LDL run.

  • Calibre RVE SPICE File Viewer — A SPICE netlist.
