Importing Calibre Results as Cadence Encounter Route Blockages

You can convert results from an ASCII format Calibre results database (RDB) to Cadence Encounter route blockages. The conversion process automatically imports the route blockages to the Cadence Encounter session. You can use NanoRoute in Cadence Encounter to repair violations.

Prerequisites

  • A Calibre RVE license.

  • The Calibre interface is loaded; see “Creating an Interface to Cadence Encounter”.

  • A Calibre results database in a supported format; see “Requirements for Converting Calibre RDBs to Native Place and Route Format”. Consult your CAD group for advice on which Calibre rule checks produce result shapes that can be used as route blockages.

  • A check map file associating a target layer to each rule check. See “Check Map File Format for Calibre RDB Conversion”.

Procedure

  1. Open the design containing the Calibre DRC results in Cadence Encounter.
  2. Issue the Calibre::convert_to_route_blockages command at the encounter prompt.

    Suppose your RDB is named drc.results and the check map file is named map. Use the following command to convert all results for check m3_pattern:

    encounter > Calibre::convert_to_route_blockages drc.results
       -check_map_file map -checks m3_pattern

    All results are converted, including results in lower level cells.

    To convert results for rule check m3_pattern in cell block1, make sure block1 is the working design and issue the following command:

    encounter > Calibre::convert_to_route_blockages drc.results
       -check_map_file map -cell block1 -checks m3_pattern

    Refer to “Configuring Calibre RDB Conversion to Native Place and Route Format” for additional information about the available options. You can specify options for side RDBs, x- and y-offset, and other parameters.

    Tip

    Some physical verification flows apply an offset to the design before verification. Use the -offset option to correct for the offset so that the results are displayed correctly in the original design.

    The following message is written to the transcript after successful conversion:

    //  RDB written as PR input file <filename>
  3. View the route blockages in Cadence Encounter. You may need to enable visibility for blockages of type “Routing Blkg” and redraw the display. You can right-click the blockage and open the Attribute Editor to view details on the routing blockage.

    You can run NanoRoute in Cadence Encounter with the new route blockages to attempt to repair the violations.

Results

Route blockages are created on the target layer(s) specified in the check map file. See the Calibre::convert_to_route_blockages command reference for complete details.
