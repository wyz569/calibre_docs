DRC HTML Report Database Map File

Input for: DRC HTML Reporting

The database map file specifies one or more DRC database files as input for DRC HTML reporting. If result highlight images are included in the DRC HTML report, the layout database file and an optional Calibre DESIGNrev layout properties (layerprops) file are also specified in the file.

Format

  • ASCII text file

  • Comma-separated parameters

  • One line per DRC results database

  • Comment lines begin with the ‘;’ character

Syntax

rdbFile, rdbType, layoutFile, layerPropertiesFile, layoutFileType

where all commas are required even if optional fields are blank.

Parameters

rdbFile

A required parameter specifying the DRC results database.

rdbType

An optional parameter specifying the results database type. Valid values are those allowed for the ApplyTo keyword, plus user-defined RDB types. The following are valid values:

  • General — (default) This database type is used if rdbType is not specified and when a database is specified on the command line, without using the database map file. “General” is also the default setting of the ApplyTo keyword.

  • Density — RDBs produced by the Density operation.

  • NetAreaRatio — RDBs produced by the Net Area Ratio operation.

  • DFM — RDBs produced by DFM operations.

  • Waiver — RDBs produced by a Calibre Auto-Waivers run.

  • PATHCHK — Results databases produced by the Pathchk operation and ERC Results Database statement.

  • LayoutInputException — RDBs produced by the Layout Input Exception RDB statement.

  • CheckMap — Results databases specified by the DRC Check Map statement.

  • Other — RDBs not of type Density, NetAreaRatio, DFM, Waiver, PATHCHK, LayoutInputException, or CheckMap.

  • user_type — You can specify a user-defined RDB type and specify it as the RDB type in the ApplyTo keyword.

layoutFile

An optional parameter specifying the layout database, which must match the type specified by layoutFileType. This parameter must be specified in order for layout images to be included in the DRC HTML report. layoutFile may be a directory if layoutFileType is MDP.

layerPropertiesFile

An optional parameter specifying a Calibre DESIGNrev layer properties (layerprops) file to define layer color and fill pattern when creating layout images.

If your Calibre setup uses the environment variable MGC_CWB_CONFIG_DIRS for layout viewer configuration, options in the layerPropertiesFile are combined with options from layerprops files specified with MGC_CWB_CONFIG_DIRS. See “Multilevel Configuration” in the Calibre DESIGNrev Layout Viewers User’s Manual. If there is a conflict, options in the layerPropertiesFile take precedence over options in other layerprops files. Also see “Using Calibre DESIGNrev Configuration Files in DRC HTML Reporting”.

Note:

The layerprops file in $HOME/.calibrewb_workspace is not used unless MGC_CWB_CONFIG_DIRS specifies that directory.

For MDP layouts (layoutFileType MDP), layer properties are read from the job deck by default and the layer properties file is ignored. Set prefs_load_useMDPChipLayerNames to 0 in the preferences file in order to have Calibre MDPview read the layer properties file. See “preferences File Format” in the Calibre DESIGNrev Layout Viewers User’s Manual for complete information.

layoutFileType

An optional parameter specifying the type of layout database.

Valid values: GDS, OASIS, and MDP

Default: GDS

If layoutFileType is MDP, then Calibre MDPview is used to display the layout images; otherwise Calibre DESIGNrev is used. All layout formats supported by Calibre MDPview are supported for HTML reporting.

Examples

The following example specifies two DRC results databases with corresponding layout database files. The optional parameter rdbType is omitted. A layer properties file (layerprops) is specified.

cellA.rdb, , cellA.gds, layerprops,
cellB.rdb, , cellB.gds, layerprops,

See “Process Multiple DRC Results Databases” for an example which specifies a user-defined rdbType.
