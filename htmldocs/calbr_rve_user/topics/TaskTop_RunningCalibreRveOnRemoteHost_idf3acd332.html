Running Calibre RVE on a Remote Host

Calibre RVE runs on the local host by default. You can also specify that Calibre RVE run on a remote host or computer cluster.

All environment variables beginning with MGC or CALIBRE are sent to the remote host or cluster, except for any environment variables that are host specific, such as the following: MGC_HOME, MGC_TMPDIR, MGC_VCO and CALIBRE_HOME, CALIBRE_BASE_PKG, and CALIBRE_ECHO_RULE_FILE.

Prerequisites

  • Platform type — Calibre RVE is launched using rsh with the DISPLAY parameter set to that of the launching host. This restricts the remote host to have the same platform type as the launching host.

  • Access control — localhost must have access control enabled to allow the remote host to make connections to the X server.

  • You are starting Calibre RVE from a supported design tool or from Calibre Interactive.

Procedure

  1. Start Calibre Interactive or your design tool.
  2. Use one of the following methods to start Calibre RVE on a remote host, depending on how you invoke Calibre RVE: from Calibre Interactive or from a viewer:

    • Calibre RVE starts automatically from the Calibre Interactive GUI

      1. Click the Outputs button on the left panel of Calibre Interactive. Enable the option “Show results in RVE” or “Start RVE …”, depending on your application.

      2. Click the Run Control button.

      3. Click the Performance tab.

      4. Disable “Run Calibre-RVE on local host.”

        Calibre RVE uses the host selected for the main Calibre run in the “Run Calibre on” dropdown button list. Calibre RVE is launched using a control script similar to the control script for the main Calibre run. The environment variables specified on the Environment tab on the Run Control pane are passed to the remote host.

    • Calibre RVE invoked from a supported viewer

      1. Invoke Calibre RVE from a supported viewer as described in “Invoking Calibre RVE From a Layout or Schematic Viewer”.

      2. In the “Run RVE on” dropdown button list, choose “Remote Host” or “Cluster.”

        Remote Host — Enter the remote host name in the text field.

        Cluster — Enter the queue command in the text field to run Calibre RVE on a computer cluster.

        These substitutions are available in the queue command:

        %c — Replace with calibre command

        %o — Replace with calibre options

        %C — Replace with calibre command and calibre options (%c and %o)

        %k — Replace with the name of the generated cluster control script file

        It is recommended to use %k to specify a control script, since this ensures that required environment variables are passed to the remote host.

        Example queue commands:

        qsub -l "num_proc=4,arch=lx24-amd64,mem_total=0" %k
