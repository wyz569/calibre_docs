SortBy

Key Value for DRC HTML Report Configuration File, group and sort category

Usage: Optional. Valid for all report types, and in Common section.

Inheritance: Sorting Inheritance — Inheritance only from Common section if the value is not specified locally; key is not inherited from a referenced section.

Sort the results by the specified column. Sorting is done within each leaf in the GroupBy hierarchy.

The allowed values for SortBy depend on the report type.

When sorting by Priority, a value of 1 has the highest sort order. A value of 0 or an unset priority has the lowest sort order. A priority value of 0 is displayed as blank in the HTML report. The priority for a rule check is set with the rule check comment RVE Priority in the rule file or a Check Text Override (CTO) file. For example, you can specify a custom sort order for rule checks by using SortBy = Priority and RVE Priority statements in a CTO file.

Allowed Values DRCResults Report

None

Sorted in internal db order. (default)

column

Sort by column, where column is one of the following:

  • Check

  • Cell

  • Flat — flat count

  • Fixed

  • Priority

  • Waived

  • “Waived Comment”

  • “Waived By”

  • “Waived Date”

property_name

Sort by a property, where property_name is a valid property name. The #SC calculated property is included in DRC HTML reports by default; other calculated properties can be included by setting CalculateProperties=1.

Allowed Values Reports Other than DRCResults Report

<GroupBy value>

Sort by the GroupBy criteria used for the report section. For example, if GroupBy = Check for the section, you can specify SortBy = Check

column

Sort by column, where column is one of the following:

  • Priority

  • Results

  • Flat

  • Fixed

  • Waived
