DRC HTML Report Configuration File Format

Input for: DRC HTML Reporting

The DRC report configuration file contains various parameters that control how the DRC HTML report is generated.

The configuration file follows the generic INI file format. You specify the configuration file on the command line; see “Command Line Syntax for DRC HTML Reporting”.

Sections in the Configuration File

The configuration file contains multiple sections which define the content of the HTML output. Section names are enclosed in brackets, and the sections contain key/value pairs specified one per line:

; comment
[Common]
key1 = value1
key2 = value2; comment on report section

[report section]
key1 = value1
key2 = value2

A typical configuration file contains a optional common section, multiple report sections, and optional filter definition sections.

Format

The following formatting rules apply to the configuration file:

  • Comment lines begin with a semicolon ( ; ).

  • All key/value pairs and section names are case-sensitive.

  • The order in which key names appear in a section is irrelevant.

  • Section names must be unique.

  • For values set to comma-separated lists, include a space before and after each comma.

The following rules apply to section names in the configuration file:

  • Section names are case-sensitive, and must be unique.

  • Slashes (“/” ) are not allowed in section names. The backslash (“\”) is allowed.

  • The order in which sections appear in the configuration file is irrelevant.

  • If a key name is not specified in a section, its value is determined as described in “Section Referencing and Key Value Inheritance”.

Parameters

The key values used in the DRC HTML configuration file are given in the section “Key Values for DRC HTML Reporting”.
