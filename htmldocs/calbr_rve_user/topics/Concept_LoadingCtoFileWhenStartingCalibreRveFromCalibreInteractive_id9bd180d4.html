Loading a CTO File When Starting Calibre RVE from Calibre Interactive

When you start Calibre RVE from a Calibre Interactive nmDRC session, most of the methods for loading a Check Text Override file (CTO file) are the same as when you start Calibre RVE from a design tool or from the command line.

The same methods described in the section “Specifying and Loading the DRC RVE Check Text Override File” apply when you start Calibre RVE for DRC from Calibre Interactive. However, if MGC_RVE_CTO_FILE is not defined and a drc_database.cto file is found, where drc_database is the results database named on the Outputs pane of Calibre Interactive nmDRC, then MGC_RVE_CTO_FILE is set to drc_database.cto. This is done so that Calibre RVE for DRC automatically loads the drc_database.cto file when it opens auxiliary (side) RDBs created during the Calibre nmDRC run.
