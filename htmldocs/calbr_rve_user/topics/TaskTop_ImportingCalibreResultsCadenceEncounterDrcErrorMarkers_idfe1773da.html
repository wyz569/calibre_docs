Importing Calibre Results as Cadence Encounter DRC Error Markers

You can convert results from an ASCII format Calibre results database (RDB) to Cadence Encounter DRC error markers. The conversion process automatically imports the error markers to the Cadence Encounter session, where you can view them in the Violation Browser.

Prerequisites

  • A Calibre RVE license.

  • The Calibre interface is loaded; see “Creating an Interface to Cadence Encounter”.

  • A Calibre results database in a supported format; see “Requirements for Converting Calibre RDBs to Native Place and Route Format”.

Procedure

  1. Open the design or cell containing the results you want to convert in Cadence Encounter.
  2. Close the Violation Browser using the Close button if it is open. For some Cadence Encounter versions, the process of updating the Violation Browser can take a very long time, making it seem as if the conversion process is hung. This may occur if the Violation Browser is open during the conversion process or if you close the Violation Browser with the X icon in the top right corner of the window.
  3. Issue the Calibre::convert_to_drc_errors command at the encounter command prompt.

    For example, to convert all DRC results for the results database drc.results:

    encounter > Calibre::convert_to_drc_errors drc.results

    All results are converted, including results in lower level cells.

    To convert results for rule check M2.1 in cell block1, make sure block1 is the working design and issue the following command:

    encounter > Calibre::convert_to_drc_errors drc.results -cell block1 -checks M2.1

    Refer to the Calibre::convert_to_drc_errors reference page and “Configuring Calibre RDB Conversion to Native Place and Route Format” for details. You can specify options for side RDBs, x- and y-offset, and other parameters.

    Tip

    Some physical verification flows apply an offset to the design before verification. Use the -offset option to correct for the offset so that the results are displayed correctly in the original design.

    At completion the transcript window displays an information message with “// RDB written as PR input file <file>.”

  4. Use the Cadence Encounter Violation Browser to view Calibre DRC errors in your design.

Results

The Calibre DRC results are listed in the Cadence Encounter Violation Browser. All results are converted to rectangular error markers. See the Calibre::convert_to_drc_errors command reference for details about the conversion process.
