Creating a Check Text Override (CTO) File Using Calibre Interactive nmDRC

Calibre Interactive—nmDRC can create a Check Text Override file to control layer visibility for highlight actions and to define categories for filtering by category. The CTO file is used by Calibre RVE for DRC and DRC HTML reporting. The automatically generated CTO file causes only the layers used in the rule check to be displayed when highlighting a result. Category definitions are optional.

The CTO file includes RVE Show Layers statements based on the rule checks in your DRC rule file. If your design tool uses a layer map file to control the layout export to GDS, the layer map file can be specified so that the layout viewer layer names are used in the CTO file.

Layer and Layer Map statements in the rule file are considered when creating the RVE Show Layers command in the CTO file.

Restrictions and Limitations

  • Without a layer map file — Layer Map statements in the rule file that have open-ended ranges for the layer or datatype are ignored and a warning is issued. The corresponding Layer statement alone is used to determine the layer number for the RVE Show Layers statement.

  • With a layer map file — See “Layer Map File Usage in Check Text Override (CTO) File Creation”

The effect of these different restrictions is shown in “Examples” in the topic “Layer Map File Usage in Check Text Override (CTO) File Creation.”

Prerequisites

  • A Calibre nmDRC rule file.

  • (Optional) A layer map file in one of the formats described in “Layer Map File Usage in Check Text Override (CTO) File Creation”.

    Use the layer map file if you want the RVE Show Layers statements in the CTO file to use layer and purpose pairs or layer names rather than layer numbers. The layer map file is also used when generating the Layers category definition; see Step 3.c.

Procedure

  1. Open Calibre Interactive—nmDRC.
  2. Specify the rule file and other settings for the run.
  3. Click the Outputs button.
    1. Select ASCII as the format of the results database.
    2. Enable “Create CTO file.”
    3. (Optional) Enable “Generate CTO Categories from” and select Groups, Layers, and⁄or Recipes to include the selected categories in the CTO file. The category definitions are used to filter by category in Calibre RVE for DRC; see “Filtering DRC Results by Category”.
  4. (Optional) Specify preferences, including a layer map file, using following steps.
    1. Select Setup > Preferences.
    2. Select the CTO File tab.
    3. (Optional) Specify a layer map file in the “Viewer Layer Map File” text area.

      This step is needed if you use a layer map file to control GDS export from a design tool and want your design tool layer and purpose pairs to be used in the RVE Show Layers statements rather than the GDS or OASIS layer numbers.

      See the “Prerequisites” section for information on layer maps.

    4. If desired, enable “Warn before overwriting CTO file.”
    5. Click OK to close the Setup Preferences dialog box.
  5. Click Run DRC.
  6. Examine the resulting CTO file. If you received any warnings, examine the layers and rules checks affected by the warnings and correct the CTO file if necessary.

    If you are using a layer map file and there are Layer statements in your rule file without a corresponding Layer Map statement, all layer:purpose pairs mapped to the corresponding GDS layer in the layer map file are used in the RVE Show Layers statement for that layer.

Results

A CTO file with RVE Show Layers statements for each rule check is created. If you specified categories in Step 3.c, the category definitions are included.

The format of the layer parameter in the RVE Show Layers statement is as follows:

  • Layer map file is used — The layer is given as layer:purpose.

  • Layer map file not used — The layer is given as layer, where layer is the layer number, or layer.datatype if Layer Map statements in the rule file provide a datatype.

Results with and without a layer map file are shown in the “Examples” section of “Layer Map File Usage in Check Text Override (CTO) File Creation”.

If a results database with the name results_rdb is open in Calibre RVE and the created CTO file is named results_rdb.cto and in the same directory, then the results database is reloaded and the CTO file is applied.
