Case Sensitivity in Calibre RVE with Cadence Virtuoso

There are environment variables specific to Cadence Virtuoso for controlling case sensitivity in Calibre RVE. Calibre RVE also has methods for controlling case sensitivity.

See the section “Case Sensitivity in Calibre RVE” for general information on controlling case-sensitivity in Calibre. The following settings control the case-sensitive behavior of Calibre RVE when interfacing with Cadence Virtuoso:

  • Case sensitivity in source — Source cell searching with Calibre RVE can be made case-insensitive by defining the environment or SKILL variable MGC_RVE_SOURCE_CASE_FOLD.

  • Case sensitivity in layout — Case sensitivity in layout export can be set through the SKILL variable streamOutKeys‑>caseSensitivity, oasisOutKeys->case, or the environment or SKILL variable MGC_RVE_LAYOUT_CASE_FOLD. Calibre RVE searches for highlight cells in a case-sensitive or case-insensitive manner depending on the setting.

    Possible values are:

    upper — cell names are converted to upper case during export.

    lower — cell names are converted to lower case during export.

    preserve — the default; cell name case is preserved during export.

    any other value — the variable streamOutKeys‑>caseSensitivity or oasisOutKeys‑>case is set to the default value of “preserve”.

    The values are case-insensitive. For example, the values “UPPER” and “upPER” are equivalent.

    • If streamOutKeys‑>caseSensitivity is not set to a value, you can use MGC_RVE_LAYOUT_CASE_FOLD to set streamOutKeys‑>caseSensitivity.

    • The variables streamOutKeys‑>caseSensitivity and oasisOutKeys->case are not set when the tool is first invoked. If the value is not set as part of the startup process, it is set to the default value of “preserve”.
