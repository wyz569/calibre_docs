OrPropertyFilter

Key Value for DRC HTML Report Configuration File, filter category

Usage: Optional. Valid in all report types, and in Common section.

Inheritance: Filter Inheritance. If the GroupBy key references another section, then the key and value are read from that section and any locally specified value is ignored.

Include only results that meet the specified property expression. OR logic is used if multiple property expressions are specified with a comma-separated list.

See PropertyFilter for a property filter that uses AND logic with multiple property expressions.

Allowed Values

Specify a single expression or a comma-separated list of expressions. An expression can take the forms in the following table, where the parameter property_name is a valid property name.

property_name Exists

Include results for which property_name exists.

property_name Range val1 val2

Include results for which property_name is in the range val1 to val2.

property_name constraint val

Include results for which property_name meets the given constraint. The following constraints are allowed: >   >=   <   <=   ==   !=   

 

The constraint operators must be surrounded by spaces.

You many use any property present in the results database, including calculated properties. The #SC (Shape Class) calculated property is included in DRC HTML reports by default; see the ClassifyShapes key value. Other calculated properties can be included by setting CalculateProperties=1.

The property Priority may be used if it is present in the results database, for example, if the RVE Priority rule check comment is include in the rule check or in a Check Text Override (CTO) file.

Examples

The property #SC must equal one:

OrPropertyFilter = #SC == 1

Specify constraints for two properties, #SC and DV. The filter is evaluated using OR logic: either #SC must equal one, or DV must be in the range 0.26 - 1.8. 

OrPropertyFilter = #SC == 1, DV Range 0.26 1.8
