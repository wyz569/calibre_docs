RVE Priority

A DRC RVE rule check comment which sets the display priority for a rule check. The rule check priority is displayed in the tree view of Calibre RVE for DRC and can be used to sort and filter the rule checks.

Usage

RVE Priority: priority

Parameters

priority

A required non-negative integer which specifies the rule check priority. A space is required between the colon (:) and priority.

Description

The priority value for the check is displayed in the Priority column of the tree view in Calibre RVE for DRC. You can click in the column header to sort by priority value; a second click in the column header reverses the sort order. You can also group the tree view by the priority value; select View > Tree Options > Group By > Custom > Priority or use the GroupBy icon () above the tree view. You can filter on the Priority property with a custom filter; see “Creating and Editing Custom DRC Results Filters”.

A priority of zero is assigned if there is no RVE Priority rule check comment present in the rule check.

See “Methods to Add DRC RVE Rule Check Comments” for instructions on specifying the rule check comment. You can add the rule check comment to the rule check in the rule file or specify it in a Check Text Override (CTO) file.

Examples

This example shows the rule check comment placed within a rule check.

// rule file
// ...
my_rule { @ RVE Priority: 2
INT m1 < .08 ABUT < 90 SINGULAR
}

This example shows the rule check comment in a DRC RVE Check Text Override File (CTO File) Format:

# DRC RVE .cto file
# ...
my_rule 
RVE Priority: 2
