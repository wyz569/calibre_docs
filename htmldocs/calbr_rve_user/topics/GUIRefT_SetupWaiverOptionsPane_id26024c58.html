Setup Waiver Options Pane

Access with: Setup > Options, then choose the Waivers category

This option pane controls settings for waiver display, waiver export, and trigger functions tied to waiver export.

Fields

Table 1. Setup Waivers Options Pane Contents

Setting

Description

On Creating Waivers

Show comment dialog while reviewing (PERC only) or waiving results

Enable the option to show a dialog box for entering waiver comments each time you waive a result.

To force the use of the waiver comment dialog box, set the environment variable MGC_CALIBRE_RVE_ENFORCE_WAIVER_COMMENT to 1 before starting Calibre RVE. See “Force Use of Waiver Comment Dialog Box”.

Specify initial comments

(only displayed if “Show comment dialog …” is checked)

Enable the option and fill in the text box that is displayed to pre-populate the waiver comment dialog box with the specified text. The initial comment text is displayed in the Waived Result Comment dialog box each time you waive a result.

Display waived results timestamps in GMT

Enable the option to show timestamps in GMT rather than local time.

Export PERC waivers with property criteria from database

Add property criteria for the results to the waiver description file. For each property attached to a result, a ‑property or ‑propertyString parameter is added with the setting {propertyName == value}, where value is the corresponding property value in the results database.

Use this option to make waiving of a result conditional on the property criteria.

When including property criteria, the Library and Delta export options in the are not available in the Export PERC Waiver Description File dialog box.

On Exporting DRC Waivers

Show warning dialog when exporting edge results

As described.

Show warning dialog when merging with existing layout file.

As described.

Truncate synthesized check names when exporting waivers.

Check names that in include “::<” are truncated starting at that location when waiver cells are created.

For example, the density check “dens_m1” can produce results with check names such as “dens_m1::<1>”. When exporting waivers with rule checksum text, the rule check names must match between the exported waivers and the rule file—enabling this option causes the checknames for such results to match what is in the rule file.

DRC Waivers Triggers

DRC Pre-Export Waivers

Enable a trigger function to run immediately before exporting DRC waivers. The name of the file is appended to the list of parameters given to the trigger function.

DRC Post-Export Waivers

Enable a trigger function to run immediately after exporting DRC waivers. The name of the file is appended to the list of parameters given to the trigger function.

PERC Waivers Triggers

PERC Pre-Export Waivers

Enable a trigger function to execute immediately before exporting the Calibre PERC Waivers File. The name of the waivers file is appended to the list of parameters for the trigger function.

PERC Post-Export Waivers

Enable a trigger function to execute immediately after exporting the Calibre PERC Waivers File. The name of the waivers file is appended to the list of parameters for the trigger function.

On Importing PERC Delta Waivers

Echo summary of imported waivers to a file

When a delta waivers file is imported, a summary is echoed to the specified log file.

  • Replace — The log file is replaced when the first delta waiver file is imported in the Calibre RVE session. Information from subsequent imports is appended while the database is open, so that the log file includes information for a single session.

  • Append — Information is appended to the log file, so that the log file may include information from multiple sessions.

Usage Notes

Force Use of Waiver Comment Dialog Box

You can force the use of the waiver comment dialog box by setting the environment variable MGC_CALIBRE_RVE_ENFORCE_WAIVER_COMMENT to 1. This applies when waiving results in Calibre RVE for DRC and Calibre RVE for PERC. The following behavior occurs:

  • The option “Show comment dialog while reviewing (PERC only) or waiving result” is enabled and cannot be changed.

  • A non-whitespace comment must be typed in the waiver comment dialog box before it can be dismissed.
