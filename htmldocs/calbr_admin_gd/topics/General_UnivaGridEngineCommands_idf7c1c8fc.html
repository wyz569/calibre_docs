Univa Grid Engine Commands

Some commonly used Univa Grid Engine commands include qhost, qstat, and qsub. For more information on these and other Univa Grid Engine commands, refer to your Univa Grid Engine documentation.

Table 1 provides an alphabetical listing of some of the more commonly used Univa Grid Engine commands.

Table 1. Commonly Used Univa Grid Engine Commands

Univa Grid Engine Command

Description

Example

qconf -sc

Displays the current complex configuration.

Display the current complex configuration:

qconf -sc

qconf -sql

Displays a list of all currently configured cluster queues.

Display a list of configured cluster queues:

qconf -sql

qdel

Cancels a job that is running or waiting to execute.

Deletes job 4321:

qdel 4321

qhost

Displays host information (for example, architecture, number of processors, and load).

Display host information:

qhost

Display host information, including information about specific jobs running on each host:

qhost -j

qmod -sj

Suspends one or more grid jobs.

Suspend job 8765:

qmod -sj 8765

qmod -usj

Resumes one or more grid jobs.

Resume job 8765:

qmod -usj 8765

qstat

Displays information about the jobs in the Univa Grid Engine queues.

Displays a list of all running and waiting jobs:

qstat

Displays summary information on all queues and the queued job list:

qstat -f

qsub

Submits a job to the Univa Grid Engine for batch execution and assigns the job a unique ID. Some of the valid qsub options include the following:

  • -cwd directs Univa Grid Engine to run the job in the same directory from which you submitted it.

  • -v variable passes an environment variable to the job.

  • -o path redirects stdout from the Univa Grid Engine script.

  • -l resources specifies a list of resources required for your job.

Submits a job to the default queue and specifies stdout is output to drc.log. Instructs the Univa Grid Engine to use an 8 CPU Linux host with at least 2 Gb of free memory, and uses the run_calibre.csh script to switch Calibre trees and execute Calibre:

qsub -o drc.log -l 
  "num_proc=8,
  archlx24x86,
  mem_free>2G"
  run_calibre.csh
