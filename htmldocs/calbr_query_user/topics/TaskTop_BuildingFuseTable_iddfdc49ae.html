Building a Fuse Table

This procedure creates a flat netlist containing all R devices with model name FUSE, with schematic instance pathnames and top-level coordinates, representing center points of the fuse devices.

You may put the Query Server Tcl shell commands into a script.

Prerequisites

  • LVS connectivity extraction rule file.

  • The Mask SVDB Directory CCI option is used in the rule file.

  • Calibre nmLVS-H, Calibre Query Server, and Calibre Connectivity Interface licenses.

Procedure

  1. Run hierarchical connectivity extraction as usual, for example: 
    calibre -spice lay.net -turbo -lvs -hier -hcell cells rules
  2. Start the Query Server: 
    calibre -qs -svdb svdb
  3. Issue the following commands: 
    set dfltr [dfm::create_filter -device_names "R(FUSE)"]  
dfm::set_layout_netlist_options -filter $dfltr -hierarchy FLAT \
  -device_location CENTER -names SOURCE INSTANCES
dfm::write_spice_netlist fuse.net
qs::quit

Results

The SPICE netlist fuse.net contains the R(FUSE) devices.
