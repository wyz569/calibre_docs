LVS Rule File Query Commands

The LVS rule file query commands return information about settings in the LVS rules. These commands are helpful in certain third-party tool flows that require information about the contents of the Standard Verification Database (SVDB).

Commands in the dfm:: scope are documented in the Calibre YieldServer Reference Manual.

Table 1. LVS Rule File Query Commands

Command

Description

dfm::get_layout_magnify

Returns the value of the Layout Magnify setting.

dfm::get_layout_system

Returns the Layout System setting.

dfm::get_layout_system2

Returns the Layout System2 setting.

dfm::get_source_system

Returns the Source System setting.

qs::device_templates_used

Returns a report of the Device statements actually used to generate instances in the extracted netlist.

qs::rules_connectivity

Returns rule file connectivity statement information.

qs::rules_file_name

Returns the name of the rule file used to create the SVDB database.

qs::rules_layer_types

Returns a Tcl list of lists of rule file layers and their types.

qs::rules_layout_case

Returns the Layout Case setting.

qs::rules_lvs_compare_case

Returns the LVS Compare Case setting.

qs::rules_lvs_db_layer

Returns the LVS DB Layer arguments.

qs::rules_lvs_db_connectivity_layer

Returns the LVS DB Connectivity Layer arguments.

qs::rules_lvs_downcase_device

Returns the LVS Downcase Device setting.

qs::rules_lvs_ground_name

Returns the LVS Ground Name settings.

qs::rules_lvs_power_name

Returns the LVS Power Name settings.

qs::rules_lvs_spice_replicate_devices

Returns the LVS Spice Replicate Devices setting.

qs::rules_precision

Returns the Precision setting.

qs::rules_source_case

Returns the Source Case setting.

qs::rules_unit_length

Returns the Unit Length setting.

qs::write_lvs_settings_report

Returns a summary report of the LVS rule file settings for rule file query commands.
