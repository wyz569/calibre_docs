Soft Connection Checking Commands

The soft connection checking commands access LVS Softchk data stored or perform soft connection checks on layer data stored in the PHDB.

The qs::softchk and qs::softchks commands are only functional in Calibre nmLVS Reconnaissance. They do not work in the Query Server Tcl shell. They are documented here because they are in the qs:: scope.

Table 1. Soft Connection Checking Commands

Command

Description

qs::softchk

Performs soft connection checks in a Calibre nmLVS Reconnaissance run on specified layers that exist in the PHDB.

qs::softchks

Performs soft connection checks in a Calibre nmLVS Reconnaissance run on the data in the PHDB.

qs::ran_softchk

Returns a 1 if LVS Softchk has been run. Otherwise a 0 is returned.

qs::get_run_summary_data ‑softchk_database

Returns the name of the LVS Softchk results database.
