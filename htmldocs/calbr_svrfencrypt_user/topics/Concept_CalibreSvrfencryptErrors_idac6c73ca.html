Calibre SVRFencrypt Errors

You may see the following errors when encrypting an SVRF rule file.
Table 1. Calibre SVRFencrypt Error Descriptions

Error Text

  • Resolution

Undefined or empty environment variable

  • Ensure that the environment variable is set and resolvable.

ERROR: Error INCL1 on line <line_number> of <full_filename> - problem with access, file type, or file open of this include file: <included_filename>.

  • Ensure that the filename referenced in an INCLUDE statement is accessible.

ERROR: Error OPEN1 - problem with access, file type, or file open of file: <full_filename>.

  • Ensure that the rule file you are attempting to encrypt is accessible.

ERROR: Error INCL2 on line <line_number> of <full_filename> - this file name is included recursively: <included_filename>.

  • Do not specify INCLUDE statements that cause loops. For example: rule file A INCLUDEs rule file B, which INCLUDEs rule file A again.

ERROR: Error ENCRYPT2 on line <line_number> of <full_filename> - Nested #ENCRYPT (missing #ENDCRYPT?)

  • Ensure that you are not nesting #ENCRYPT and #ENDCRYPT directive pairs within each other.

ERROR: Error ENCRYPT3 on line <line_number> of <full_filename> - #ENDCRYPT without #ENCRYPT (missing #ENCRYPT?)

  • Ensure that you have an #ENDCRYPT directive for every #ENCRYPT directive.
