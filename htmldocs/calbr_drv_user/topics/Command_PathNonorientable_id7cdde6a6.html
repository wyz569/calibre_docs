PATH_NONORIENTABLE

Input for: readerprefs file (see readerprefs File Format)

A path cannot be oriented after expansion.

Usage

PATH_NONORIENTABLE {0 | 1 | 2 | 3 | 4}

Note:

In HC mode, the path is silently read into the database.

Arguments

0

Ignore the path.

1

Warn and ignore the path (default).

2

Error.

3

Fix the path using a double-merge technique. This technique generally succeeds in filling non-orientable polygons in an intuitive fashion. The polygon is processed with no message.

4

Fix the path using a double-merge technique. This technique generally succeeds in filling non-orientable polygons in an intuitive fashion. The polygon is processed with a warning message.

Examples
