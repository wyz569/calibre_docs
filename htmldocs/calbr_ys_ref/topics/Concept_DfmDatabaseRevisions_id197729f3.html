DFM Database Revisions

To modify and save DFM databases, Calibre YieldServer supports database revisions.

The following table discusses database state definitions. These states govern whether a database can be revised.

Table 1. DFM Database Revision States

State

Description

Valid Commands

Frozen

A finalized revision. Once a revision is frozen, it can no longer be modified, but can be used as a parent for new revisions.

dfm::open_rev, dfm::close_db, dfm::create_rev

Unfrozen, Saved

A revision that has no unsaved changes but has not been frozen.

dfm::open_rev, dfm::freeze_rev, dfm::close_db

Open, Unsaved

A revision created from a frozen revision that has unsaved changes.

dfm::create_rev, dfm::save_rev, dfm::close_db -force

Unfrozen, Unsaved

A revision that has unsaved changes and has not been frozen.

dfm::save_rev, dfm::close_db -force
