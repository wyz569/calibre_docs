DFM Create Layer Polygons

Layer operation

License: Calibre YieldServer

Creates polygons on a new layer in a DFM database. Only used within the context of the dfm::new_layer Calibre YieldServer command.

Usage

DFM CREATE LAYER POLYGONS [NODAL] [ORDERED] ‘[’
polygon_specification [polygon_specification …]
‘]’

Where the syntax for polygon_specification is this:

   vertex_count x1 y1 x2 y2 [… xn yn ] [NET net_num] [CELL cell_name]

Arguments

NODAL

An optional keyword that assigns node numbers to the polygons on the newly created layer. If NODAL is specified, a net_num argument must be supplied for each polygon.

ORDERED

An optional keyword that assigns ordinal numbers to polygons specified with polygon_specification. This keyword must be specified if DFM properties are added to the new layer.

‘[’  ‘]’

Literal brackets that enclose each polygon_specification.

polygon_specification

A required argument set defining a polygon to be created on the new layer. You must include at least one polygon_specification. The syntax is as follows:

vertex_count x1 y1 x2 y2 [… xn yn] [NET net_num] [CELL cell_name]

  • vertex_count

    A required argument specifying the number of vertices for a polygon.

  • x1 y1 x2 y2 [… xn yn]

    A required argument set defining the vertex coordinates of the polygon. Each value is a floating-point number in user units. The cell coordinate space is top level by default, but this can be changed using the CELL keyword. The vertices are checked to ensure that polygons are well-formed. Unlike Polygon or Layout Polygon, this specification supports non-simple shapes. See “Negative Numbers in Coordinate Lists” for information on handling negative values.

  • NET net_num

    A keyword and argument that assign a net number to the polygon. A net_num is a positive integer. This keyword is required when NODAL is specified and forbidden if it is not specified.

  • CELL cell_name

    An optional keyword and argument that define the cell that is the parent of the polygon. If this keyword is not specified, the polygon is added to the top-level cell.

Description

Creates the polygon defined by the polygon_specification. The various options configure attributes of the generated polygons. This operation is used as an argument to a dfm::new_layer rule file script only. The operation cannot be specified outside this context.

During execution, overlapping or abutting polygons are merged to create the output layer. It is an error if NODAL is specified and the number of merged polygons on the output layer is different from the number of polygons specified in the operation.

The ORDERED option is required when a created layer is assigned DFM properties in Calibre YieldServer. Properties can be added in the dfm::new_layer rule file script or using dfm::add_property. The presence of polygon index numbers as a prerequisite for attaching properties is an internal requirement of the tool. A few YieldServer commands access these index numbers, but aside from those uncommon usages, the indices are not of concern to the user.

Examples

# Creates three polygons, each with 4 vertices on the new
# layer. For each polygon, vertices are specified as
# x1 y1 x2 y2 x3 y3 x4 y4
dfm::new_layer -svrf { x = DFM CREATE LAYER POLYGONS [ 
4 0.0 25.0 20.0 25.0 20.0 50.0 0.0 50.0 
4 0.0 50.0 20.0 50.0 20.0 75.0 0.0 75.0 
4 0.0 75.0 20.0 75.0 20.0 100.0 0.0 100.0 
] }
