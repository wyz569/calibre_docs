PEX Map Required

Parasitic extraction

Specifies a required physical layer name that must be mapped to a calibrated layer with PEX Map. Use only in Calibre xACT and Calibre xACT 3D.

Usage

PEX MAP REQUIRED physical_layer_name

Arguments

physical_layer_name

A required parameter that specifies a single original layer name or derived layer name. This LVS layer must be of the proper type and have appropriate connectivity defined for the rules it inherits.

Description

An optional statement that specifies a required LVS layer name that must be mapped to a calibrated layer using the PEX Map statement. Use this statement to specify the required LVS layers for your extraction run. This statement can be specified more than once.

If a PEX Map statement that contains this specified LVS layer name does not exist, then extraction stops with the error:

ERROR: Error PEX40 - Incompatible statements: PEX MAP REQUIRED: missing mapping or mapping to undefined layer for LVS layer <layer_name>.

Both the physical LVS layer name and the calibrated layer name must be valid. The PEX Exception Severity statement can be used to control the severity of map errors.

Examples

To specify the LVS layers MET1 and MET2 as required, include the following statements in your rule file:

PEX MAP REQUIRED MET1
PEX MAP REQUIRED MET2

If you have specified the following PEX Map statements in your rule file:

PEX MAP METAL1 MET1
PEX MAP METAL2 MET20

The extraction stops with the following error:

ERROR: Error PEX40 - Incompatible statements: PEX MAP REQUIRED: missing mapping or mapping to undefined layer for LVS layer MET2

because a PEX Map statement for MET2 is not found.
