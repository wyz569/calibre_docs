Non-Persistent DFM Properties

Non-persistent properties are properties that are not saved to the output layer created by the DFM Property operation. Non-persistent properties are used to filter the input layer(s) when satisfying the constraint is important, but the exact value is not.

Non-persistent, or temporary, properties are only used in the DFM Property family of operations and are indicated with the property name “-” or a property name starting with a plus sign (+).

  • The property name “-” (that is, the single character “-”) is a non-persistent property that cannot be used in Expression Chains. When an operation contains multiple expressions for the “-” property, each is treated as a separate property. Only those layout objects satisfying the constraints for all the expressions are written to the output layer.

  • Any property name starting with the “+” character is a named non-persistent property that can be used in an expression chain. When an operation contains multiple expressions for the same property name starting with the “+” character, the first successful evaluation of an expression for that property prevents all subsequent expressions for that property from being evaluated. Named non-persistent properties can also be used in the property reference functions (those ending in _REF). A named non-persistent property must be enclosed in quotation marks, for example: “+TEMP_X”.

Example 1

In this example: 

E_LONG = DFM PROPERTY E [- = EC(E)] >= 1

the output layer contains only the error clusters with an EC value greater than or equal to 1 user unit. No properties are associated with the layout objects on the output layer.

Example 2

The following example uses an expression chain and a named non-persistent property. The operation examines a derived error layer and generates a layer with error clusters for which the measurement function EC is undefined (for example, clusters with more than two edges). The property value is not needed, so a named non-persistent property is used. 

EFAIL = DFM PROPERTY E ["+X" = EC(E)] !>=0 ["+X" = -1]

For each edge cluster on layer E, the measurement EC(E) is one of the following:

  • Defined and returns a non-negative value, which does not meet the constraint.

  • Not defined and the first expression fails to evaluate, in which case the second expression sets the property X to the default value -1.

The resulting layer EFAIL contains only the edge clusters for which the EC function failed to evaluate. No property value is saved to layer EFAIL.
