DFM Histogram

Layer operation

License: Calibre YieldAnalyzer

Analyzes the distribution of DFM Property values, computes a histogram of property values, and prints the histogram data to a text file.

Usage

DFM HISTOGRAM file_name layer_name [layer_name …]
PROPERTY property_name
[MIN { min_value | AUTO }] [MAX { max_value | AUTO }]
[BINS num_bins | STEP step_size | AVERAGE shapes_per_bin]
[BY CELL [NOPSEUDO] [NOEMPTY] [ACCUMULATE]]
[COMMENT “comment_string”] …

Arguments

file_name

A required filename in which to write the histogram data. Multiple DFM Histogram commands can write to the same file, in which case, each new histogram is appended to the end of the file.

layer_name [layer_name …]

A required argument specifying the name of one or more derived layers used to compute the distribution of properties. The layer must be a derived layer created using the DFM Property operation. At least one layer must be specified. If multiple layers are specified, the resulting histogram represents the cumulative distribution across all layers.

PROPERTY property_name

A required argument that specifies the name of the property to be represented as a histogram. If the named property is not present on one or more of the input layers, a warning is printed in the transcript and the layers are ignored by the DFM Histogram command. If the specified property is not present on any layers, the command does not produce any output.

MIN {min_value | AUTO}

An optional argument that defines the lower boundary of the range of the property values represented by the histogram. One of two arguments must be specified:

  • min_value — Specifies a minimum value for the property values represented by the histogram.

  • AUTO — Automatically computes the lower limit based on the property values. This limit is then used to determine the centers of the leftmost bin. This is the default.

MAX {max_value | AUTO}

An optional argument that defines the upper boundary of the range of the property values represented by the histogram. One of two arguments must be specified:

  • max_value — Specifies a maximum value for the property values represented by the histogram.

  • AUTO — Automatically computes the upper limit based on the property values. This limit is then used to determine the centers of the rightmost bin. This is the default.

BIN num_bins | STEP step_size | AVERAGE shapes_per_bin

An optional argument specifying the number of bins used for the histogram. The default is 10 bins.

  • BIN num_bins — Specifies the number of bins for the histogram. The value must be an integer.

  • STEP step_size — Specifies a step size between all bins in the histogram. The specified step_size determines the number of bins output for the histogram. Smaller step sizes increase the number of bins; larger step sizes reduce the number of bins. In the following example, DFM Histogram specifies a STEP of 6 between bins (shown in red). The output histogram file, met1_area_step, results in 47 bins (not all shown), each bin separated by a step_size of 6:

    DFM HISTOGRAM "met1_area_step" metal1 PROPERTY "AREA" STEP 6

BY CELL [NOPSEUDO] [NOEMPTY] [ACCUMULATE]

An optional argument that generates a separate histogram for each cell in the design. Each histogram has the same range and bin count. By default, DFM Histogram computes the distribution for the entire chip, flattened. BY CELL provides three methods of handling cell computation output. All methods may be used together.

  • NOPSEUDO — Excludes Calibre tool‑created pseudo cells. The results from these cells are counted in the design cells containing the pseudo cell; the results are flattened up through pseudo cells.

  • NOEMPTY — Excludes cells without results from the output. By default, BY CELL prints zero values for cells without results.

  • ACCUMULATE — Reports the results from all placements of the specified cell. The specified cell is considered flat, allowing results from all lower-level cells referenced within the specified cell to be counted. Without ACCUMULATE, the histogram reports results in the top level of the specified cell itself and, possibly, in pseudo cells contained within the cell.

COMMENT “comment_string”

An optional argument that specifies a comment string to add to the histogram file. COMMENT may be specified multiple times. Each comment adds one line of check text to the histogram file directly following the first two header lines.

The comment_string must be a single string, and comments containing spaces or special characters must be enclosed in quotation marks (“ ”).

Description

DFM Histogram analyzes the distribution of DFM Property values, computes a histogram of property values, and prints the histogram data to a text file. If a property value is coincident with the border between two bins, it is counted in the bin on the right side of the border.

When using the AUTO option with the MIN or MAX argument, it is not possible to center both the leftmost and the rightmost bins on the extreme property values. Therefore, the leftmost bin is centered on the lower limit and the rightmost bin is guaranteed to enclose the rightmost property value. If only one limit is computed automatically, then that limit is used as a center of the corresponding bin, and the other explicitly-specified limit remains unchanged.

For example, assume that the property values are 1, 2, 3, 4, and 5, and the DFM Histogram operation specifies MIN AUTO MAX AUTO STEP 1. Five bins are created with ranges defined as follows: 0.5-1.5, 1.5-2.5, 2.5-3.5, 3.5-4.5, 4.5-5.5.

For details regarding the organization of histograms, refer to “Histogram File” in the Calibre YieldAnalyzer and YieldEnhancer User’s and Reference Manual.

Examples

This code finds locations where two polygons on layers MET1 or MET2 are closely spaced and computes the distribution of lengths of these areas:

MET1S = EXT MET1 < 0.25
MET2S = EXT MET2 < 0.2
MET1SE = DFM PROPERTY MET1S [ EC = EC( MET1S ) ]
MET2SE = DFM PROPERTY MET2S [ EC = EC( MET2S ) ]
CHECK { 
	DFM HISTOGRAM "hist1_2h" MET1SE MET2SE
		PROPERTY "EC"
		BY CELL 		ACCUMULATE
		BINS 5
}
