LVS DB Connectivity Layer

Specification statement

Specifies connectivity layers to be merged and saved with their nodal information in a persistent hierarchical database.

Usage

LVS DB CONNECTIVITY LAYER layer [layer …]

Arguments

layer

A required name of an original or derived polygon layer with connectivity information. More than one layer may be specified.

Description

This statement passes connectivity-bearing layers along with their node numbers through to a persistent hierarchical database (PHDB) for post-processing in the Calibre Connectivity Interface (CCI). Layers specified in this statement are typically processed by the qs::port_table or PORT TABLE WRITE commands in CCI.

This statement functions similarly to LVS DB Layer, with the exception that nodal information is stored in addition to shapes.

Specified layers that are intended to replace Connect or Sconnect (connect set) layers to which port objects are attached should meet these conditions:

  • Each layer covers associated connect set layer polygons. This is not a Calibre circuit extraction requirement, but often is required by downstream parasitic extraction tools.

  • Each layer resides in the same cells as its associated connect set layer. This involves care in the derivation of LVS DB Connectivity layers so that interaction with shapes higher up in the hierarchy does not cause promotion out of cells in which port objects reside.

  • Shapes from exactly one layer intersect port objects. That is, shapes from more than one layer do not overlay a given port.

A Calibre nmLVS Advanced license is required when this statement is used during circuit extraction (calibre ‑spice) in a DFM, LVS, or PEX flow. See the Calibre Administrator’s Guide for license details.

Examples

The following example passes layers m1_a and m1_b to the PHDB, along with their nodal information. The layers can be written to an AGF file in CCI.

CONNECT M1
m1_a = M1 AND A
m1_b = M1 AND B
PORT LAYER TEXT M1
LVS DB CONNECTIVITY LAYER m1_a m1_b // add these to AGF output in CCI

MASK SVDB DIRECTORY svdb CCI
