DFM OR Edge

Layer operation

License: Calibre YieldEnhancer, except in LVS runs, which use a Calibre ADP (Advanced Device Properties) license.

Performs a Boolean OR operation on edges from the input layers. Input layers may be polygon, edge, or error layers, or any combination of these layer types. Edge direction is considered. Used primarily in Calibre YieldEnhancer applications.

Usage

DFM OR EDGE layer [layer ...] [CONFLICTS | BOTH]

Arguments

layer

A required argument that specifies one or more input layers. All layer types are supported: polygon, edge, and error layers. Layers of different types can be specified in the same operation. Derived polygon layers may be merged or unmerged.

Objects on the input layers are converted to edges as described in the “Description” section.

CONFLICTS | BOTH

An optional keyword choice that specifies which edges to output.

By default, an edge segment is not output for coincident input edge segments if any of the overlapping segments face opposite directions.

  • CONFLICTS — Output only coincident input edge segments where at least one input edge faces in the opposite direction of the other overlapping edges. The output edge segments are arbitrarily oriented. This output is the complement of the default output.

  • BOTH — Output both the default output and the CONFLICTS output.

Description

Performs a Boolean OR operation on edges from the input layers. The output is an edge layer. DFM OR Edge is not node-preserving. Unlike OR Edge and DFM Copy, DFM OR Edge accepts more flexible inputs and enables more control over the output.

The input layers are converted to edges as follows:

  • A polygon layer is represented by the bounding edges of polygons. Unmerged derived layers are not merged before processing; however, original layers are merged on input, as usual. (Unmerged polygon layers can be generated by a limited number of operations; see “Unmerged Polygon Layers” in the Calibre Verification User’s Manual.)

  • An edge layer is unmodified.

  • An error layer is represented by the edges that make up the edge clusters.

DFM Or Edge considers edge direction when processing coincident edges (edges that are collinear and overlapping). Coincident edges are broken into subsegments determined by the endpoints of the edges that are collinear. By default, if all coincident subsegments face the same direction, one of the subsegments is output. If all coincident subsegments do not face the same direction, no output is generated from those subsegments by default. This is the same behavior as with OR Edge and DFM Copy.

When the CONFLICTS keyword is specified, output is generated only from the coincident subsegments that include edges facing in opposite directions. The direction of the output edge is arbitrary. When BOTH is specified, the output includes the default output and the CONFLICTS output.

For the default output, all non-coincident edges are output, along with subsegments A and B, because the coincident edges face the same direction. For the CONFLICTS output, only subsegments C and D are output, because the coincident edges face opposite directions.

DFM OR Edge operations run concurrently if the set of input layers is the same.

Examples

Example 1

The following example shows how to use DFM OR Edge to output two edge layers that satisfy these criteria:

  • Output only coincident edges with opposing edge directions.

  • Output all edges except coincident edges with opposing edge directions.

// MET1, MET2, and MET3 are original layers
check {
         // output coincident edges with opposing directions
         M1M2M3_OPPOSITE_EDGE = DFM OR EDGE MET1 MET2 MET3 CONFLICTS
         // do not output coincident edges with opposing directions
         M1M2M3_BOUND_EDGE = DFM OR EDGE MET1 MET2 MET3
         //write generated edges
         DFM RDB M1M2M3_OPPOSITE_EDGE out.rdb
         DFM RDB M1M2M3_BOUND_EDGE out.rdb
}

Example 2

This example uses DFM OR Edge on unmerged polygon layers that are produced by DFM Expand Edge. 

LAYER layer1
edge1_props = DFM PROPERTY SELECT SECONDARY layer1
                    SELECT layer1:EDGE 
                    [ xlen = LENGTHX(layer1:EDGE)/4 ]
                    [ ylen = LENGTHY(layer1:EDGE) ] 

poly_ee = DFM EXPAND EDGE edge1_props 
              OUTSIDE BY "xlen" INSIDE BY "ylen"
              UNMERGED
              
poly_ee_or = DFM OR EDGE poly_ee
