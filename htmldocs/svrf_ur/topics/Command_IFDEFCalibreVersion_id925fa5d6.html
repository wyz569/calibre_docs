#IFDEF CALIBRE_VERSION

The #IFDEF CALIBRE_VERSION pre-processor directive provides conditional compilation based upon the Calibre version, allowing for easier integration of new functionality into existing SVRF files.

Usage

   #IFDEF CALIBRE_VERSION value
      rule_file_text
   [ #ELSE
      rule_file_text ] ...
   #ENDIF

Arguments

value

A valid Calibre version string of the form: vYYYY.Q_D*.D*

For example:v2011.2_37.32

rule_file_text

Rule file statements and operations to be performed, depending on the true or false status of the conditional directive. The rule_file_text string can also contain conditionals, which creates nested pre-processor directives.

Description

This directive triggers compilation if the Calibre software version (displayed at the top of the banner in the Calibre transcript) is greater than or equal to the specified value. “Greater than or equal to” means the Calibre software version must be the same or more advanced than the specified value. Beta and customer‑specific version numbers are truncated prior to any comparison with the specified value. If value is not specified, this directive is ignored.

Examples

In the following example, if the Calibre version is greater than or equal to v2011.2_0.0, then perform the first OR operation. Otherwise, perform the second OR operation.

#IFDEF CALIBRE_VERSION v2011.2_0.0
		X = OR A B C D
	#ELSE
		X = (A OR B) OR (C OR D)
#ENDIF
