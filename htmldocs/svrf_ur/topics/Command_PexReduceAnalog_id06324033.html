PEX Reduce Analog

Parasitic extraction

Specifies performing reduction of extracted data for analog designs.

Usage

Syntax 1, for Calibre xACT and Calibre xACT 3D:

PEX REDUCE ANALOG {NO | YES [DELAY_ERROR delay] [NOISE_ERROR noise]}

Syntax 2, for Calibre xRC:

PEX REDUCE ANALOG {NO | YES [DELAY_ERROR delay] [NOISE_ERROR noise]}

Arguments

NO

Keyword indicating to not apply analog reduction. This is the default behavior for CalibreÂ xRC.

YES

Keyword indicating that analog reduction will be applied during the formatter step of parasitic extraction. This the default behavior for the Calibre xACT 3D field solver and Calibre xACT.

DELAY_ERROR delay

Optional keyword set specifying the timing delay threshold used for parasitic reduction. The delay units are seconds. The default is 0.5e-12 seconds (0.5 picoseconds) for Calibre xRC. The default is 2e-14 seconds (20.0 femtoseconds) for Calibre xACT and Calibre xACT 3D direct netlisting flows.

NOISE_ERROR noise

Optional keyword set specifying the error threshold used for parasitic reduction. The noise error is defined as the amount of change in noise amplitude relative to a full strength signal. The value for noise is a ratio, a unitless floating number. The default is 0.01, or 1% for Calibre xRC, Calibre xACT, and Calibre xACT 3D.

Description

Specifies performing reduction of extracted data from analog designs. This statement combines the effects of the PEX Reduce TICER and PEX Reduce CC SVRF statements. For Calibre xRC, these latter statements run after PEX Reduce Analog if they are all used in the same rule file. For Calibre xACT, PEX Reduce Analog takes precedence over PEX Reduce TICER and PEX Reduce CC statements; this means if PEX Reduce Analog NO is specified, then no reduction is performed. Use the PEX Reduce Digital statement for digital designs. If both PEX Reduce Analog and PEX Reduce Digital are specified in the rule file, Calibre stops with an error.

PEX Reduce Analog

For Calibre xACT and Calibre xACT 3D direct netlisting flows, if neither PEX Reduce Analog nor PEX Reduce Digital is specified, then analog reduction is enabled by default with a TICER frequency of 1e12 for GDS and OASIS designs. Digital reduction is enabled by default for LEF/DEF designs. The default TICER frequency for Calibre xACT 3D PDB flow is 1e13. The default TICER frequency for Calibre xRC is 40e9.

For Calibre xACT and Calibre xACT 3D, when PEX Reduce Analog NO is specified, PEX Reduce CC reduction is turned off as shown in Table 1. If you want to use PEX Reduce CC then you must specify PEX Reduce Analog YES.

If the PEX Reduce CC statement is not specified with the PEX Reduce Analog YES statement, the default settings for PEX Reduce CC statement keyword sets, ABSOLUTE threshold and RATIO percent, are as follows:

  • For Calibre xRC, the default value for threshold is 0 femtofarads.

    For the Calibre xACT LEF/DEF extraction flow, the default value for threshold is 3 femtofarads.

    For the Calibre xACT GDS flow, the default value for threshold is 1 femtofarad.

  • For Calibre xRC, the default value for percent is 0.01.

    For the Calibre xACT LEF/DEF extraction flow, the default value for percent is 0.03.

    For the Calibre xACT GDS flow, the default value for percent is 0.01.

You may specify either or both parameters DELAY_ERROR and NOISE_ERROR. If you do not specify a value for either, the default is used.

Using PEX Netlist RLAYER, RLENGTH, RTHICKNESS, RLOCATION or RWIDTH with PEX Reduce Analog YES generates a warning message and RLAYER, RLENGTH, RTHICKNESS, RLOCATION, or RWIDTH keywords are ignored. For Calibre xACT, if you want to use these keywords, then you must specify PEX Reduce Analog NO.

For example, using PEX Netlist RWIDTH together with PEX Reduce Analog YES generates the following warning message and RWIDTH is ignored:

RULES: PEX NETLIST "netlist.dist" DSPF 1e-6 LAYOUTNAMES GROUND VSS LOCATION RWIDTH
RULES: PEX REDUCE ANALOG YES
WARNING: RWIDTH not supported with PEX REDUCE ANALOG

Examples

Example 1

This statement specifies that analog RC reduction will not take place:

PEX REDUCE ANALOG NO

Example 2

This statement specifies that analog reduction will be done with the default values for DELAY_ERROR (0.5 picosecond) and NOISE ERROR (1%):

PEX REDUCE ANALOG YES

Example 3

This statement specifies a reduction with a delay error of 5 picoseconds and a noise error of 15%: 

PEX REDUCE ANALOG YES DELAY_ERROR 5e-12 NOISE_ERROR 0.15

Example 4

This example shows how one of the optional parameters may be used: 

PEX REDUCE ANALOG YES DELAY_ERROR 5e-12

In this case the default value for NOISE_ERROR (0.01) will be used.
