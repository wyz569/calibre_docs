PEX CMP Mode

Parasitic extraction

Specifies the type of CMP mode used to handle layer thickness variation during parasitic extraction.

Usage

PEX CMP MODE
{VCMP SUFFIX | PRAESAGUS FILENAME | CMPA SUFFIX} string
[DIRECTORY path]
[DEFAULT_DENSITY value]

Arguments

VCMP SUFFIX

Specifies using TSMC’s Virtual Chemical Mechanical Polishing (VCMP) data format when handling thickness variations for capacitance calculations and Rho-based resistance calculations.

PRAESAGUS FILENAME

Specifies using the Praesagus Copper Prediction and Verification database when handling thickness variations for capacitance calculations and Rho-based resistance calculations.

CMPA SUFFIX

Specifies using the Calibre CMPAnalyzer data format when handling thickness variations for capacitance calculations and Rho-based resistance calculations.

string

Specifies a filename for PRAESAGUS mode, or a suffix for VCMP or CMPA modes.

DIRECTORY path

Optional keyword and value set for using CMP thickness parameters where path defines the directory location for the CMP files.

DEFAULT_DENSITY value

Optional keyword and value set used with feature level VCMP models only. If not defined, the default for value is 0.4, or 40% density.

Description

This statement specifies the type of CMP mode used to handle layer thickness variation during parasitic extraction. Metal layer thickness variations are used for capacitance calculations and Rho-based resistance calculations.

In the calibrated rule files, layer thickness is defined as a function of drawn width and drawn spacing or expressed by equations that are a function of drawn width and density. Simulated thickness from CMP is used to override these calculated thickness values. The CMP model that enables the CMP simulation must be provided by the foundry.

The VCMP and CMPA modes use the following convention for naming the data files:

  • VCMP — metal?z2o.txt, where “z2o” indicates metal thickness data and “?” is replaced with the metal number.

  • CMPA — metal?_thk.txt, where “?” is replaced with the metal number.

Examples

Example 1

To use VCMP mode with file suffix z2o.txt and a density of 30%, use the following statement:

PEX CMP MODE VCMP SUFFIX "z2o.txt" DIRECTORY "./myInput/vcmpDir" 
	DEFAULT_DENSITY 0.3

In this case a VCMP file for the metal1 layer would be located in:

./myInput/vcmpDir/metal1z2o.txt

To use the PRAESAGUS mode with a file, you can add one of the following statements:

PEX CMP MODE PRAESAGUS FILENAME "./myInput/psgs.txt"

or

PEX CMP MODE PRAESAGUS FILENAME "psgs.txt" DIRECTORY "./myInput"

In both cases the complete file path is: 

./myInput/psgs.txt

Example 2

Suppose in preparation to use CMPA data for thickness, RHO_T or RSH_T tables and Thickness tables were specified for the metal layers in the calibrated rule files. Also, Calibre CMPAnalyzer was used to generate thickness files for the metal layers 1 through 10, and these files were written in a directory named xRC. 

112% ls xRC
metal10_thk.txt
metal1_thk.txt
metal2_thk.txt
metal3_thk.txt
metal4_thk.txt
metal5_thk.txt
metal6_thk.txt
metal7_thk.txt
metal8_thk.txt
metal9_thk.txt

To use CMPA mode with these files, specify the following statement:

PEX CMP MODE CMPA SUFFIX "_thk.txt" DIRECTORY "./xRC"

During extraction, Calibre xRC uses these CMPA thickness variation files for capacitance calculations and rho-based or rsh-based resistance calculations in place of the models defined in the calibrated rule files.
