DFM Property Singleton

Layer operation

License: Calibre nmDRC-H

Creates a polygon layer with no physical geometries and containing user-defined DFM properties calculated from one or more input layers.

Usage

DFM PROPERTY SINGLETON layer [layer …]
[CORNER]
[INSIDE OF x1 y1 x2 y2] …
[GLOBALXY]
[DBU]
[property_definition …]

where property_definition is

      '[' property_name = expression ']' [ ['!'] constraint]…

Arguments

layer

A required original or derived layer. The input layer may be a layer created by DFM Property Net or another DFM Property Singleton operation.

CORNER

An optional keyword used to control how the operation treats edge clusters produced by Calibre nmDRC measurements between corners of rectangles aligned to the x and y axes. Because Calibre nmDRC measurements operate on edges, measurement between corners produces two errors, that is two pair of edges, rather than one. This can result in faulty error counts because each corner is counted twice.

Using this keyword instructs the operation to retain only one of the edge clusters produced from Calibre nmDRC measurements of corners of rectangles. The operation selects the edge pair to retain that, if converted to a polygon, would have the larger area.

Figure 1. DFM Property CORNER

This keyword is meaningful only when used with edge clusters, and is valid only with a single layer operation. It affects only parallel edge clusters as shown in the preceding figure. Perpendicular edge clusters are unaffected.

INSIDE OF x1 y1 x2 y2

An optional keyword used to restrict the operation to geometries that have at least a partial overlap with the rectangular window specified by the arguments of the INSIDE OF option. This option applies to input layers of any type. An object is said to overlap with the INSIDE OF window if any part of the extent of the object overlaps the window area.

The arguments, x1 y1 x2 y2, must be specified in chip-level coordinates. If a coordinate is negative, it must be enclosed in parentheses ( ).

Multiple INSIDE OF options can be specified, defining multiple selection windows. In this case, only geometries that overlap at least one of the windows are considered.

Note:

When the INSIDE OF option is specified, the operation produces flattened results, with geometries from placements of lower level cells within the window promoted to the top cell. Because of this, this option is intended for use with small windows for detailed analysis of small portions of the chip.

GLOBALXY

An optional keyword that calculates measurements with X- and Y-axes aligned with the top-level coordinate system. The GLOBALXY keyword allows axis-aligned measurements to be meaningful in the global frame of reference. Use of the keyword increases processing time, so it should be used only when needed.

See the GLOBALXY definition with the DFM Property command reference for more detail and examples.

DBU

An optional keyword that causes all measurements to be performed and returned in database units rather than user units. As a side effect when the DBU option is present, the PRECISION function in DFM expressions returns 1.0. The DBU option can be used to avoid rounding effects from math with floating point numbers. See “Example 9” with the DFM Property operation.

property_definition

An optional property definition using the same syntax as that for DFM Property. See “Property Evaluation” in the Description section for evaluation details specific to DFM Property Singleton.

If no property definition is specified, the output layer is generated as usual but no properties are attached to it.

Description

The output layer from this operation is a polygon layer with attached DFM properties, if specified, and no physical geometries. Property values are stored in the top cell of the layout. This type of layer information is useful when only the DFM properties are needed for the output layer.

If all property evaluations are successful and the optional constraints are met, the output layer contains one virtual polygon and the defined properties. An empty layer, without the virtual polygon, is produced if a property expression does not meet its optional constraint or if expression evaluation fails—the presence or absence of an empty layer can be used to control processing in subsequent operations. The COUNT function on an empty layer returns zero. See the following section for more details on the property evaluation.

Property Evaluation

Property expressions are evaluated in the same manner as for the DFM Property operation; however, DFM Property Singleton does not have an explicit primary layer input. DFM Property Singleton creates a single virtual polygon in the top cell of the layout and every geometry on the input layer(s) is clustered with this virtual polygon. The input layers are processed as secondary layers in the property expression. Because the input layers are processed as secondary layers, a function returns the sum of the function values over all secondary layer objects. This behavior can be changed with the use of per-shape functions. See the DFM Property operation and the discussion of secondary layers. Some functions include an ordinal argument that specifies the index of the secondary layer geometry. The ordinal is optional in some functions (PROPERTY, for example), but required in other functions if the layer argument is a secondary layer (NETPROPERTY, for example).

If expression evaluation fails, or any expression does not satisfy its constraint, then the output layer is empty; in other words, the COUNT function returns zero.

Note:

DFM functions such as VECTOR cause promotion to the top layer, and therefore performance degradation.

Use of Singleton Layers as Input to Other DFM Property Operations

Layers produced by DFM Property Singleton (singleton layers) can be used as secondary layers for subsequent DFM Property operations; all clustering modes support singleton layers. All variants of DFM Property can use singleton layers as a secondary layer (DFM Property, DFM Property Merge, DFM Property Net, DFM Property Select Secondary, and DFM Property Singleton).

The virtual geometry on the singleton layer clusters with every primary layer geometry in the DFM Property operation, regardless of the MULTI or NOMULTI keyword in the DFM Property operation. Property values from the singleton layer can be used in any property expression. Property values from the singleton layer are available without promotion in every cell. In particular, use of a singleton layer as a secondary layer does not affect the hierarchical location where expressions are evaluated.

When a singleton layer is used as a secondary layer input to another DFM Property operation, the Measurement Functions that measure geometric attributes are not supported; the COUNT function is supported, however. The property access functions are supported; see “Property Access Function Summary”.

Examples

The output layer from DFM Property Singleton can be used to provide if/then control for a subsequent operation. For example, the following code performs a distance measurement only if the maximum voltage in an input layer exceeds some THRESHOLD value:

// Does layer1 have any polygons with maxvolt > THRESHOLD?
selector = DFM PROPERTY SINGLETON layer1
            [- = MAX(PROPERTY(layer1, maxvolt))] > THRESHOLD

// Efficient operation to copy layer1 to ext_input if maxvolt > THRESHOLD
// for any polygon, and leave ext_input empty otherwise.
ext_input = DFM PROPERTY layer1 selector OVERLAP MULTI
            [- = COUNT(selector)] > 0

// EXT operation is performed on nonempty data 
// only if maxvolt > THRESHOLD.
err = EXT ext_input < DISTANCE
Note:

MULTI is not necessary in the DFM Property operation, because secondary layers generated by DFM Property Singleton always cluster with every primary layer geometry, regardless of the presence of MULTI or NOMULTI.
