Maximum Results and Polygon Segmentation Warnings

This section describes runtime warnings related to maximum results count and polygon segmentation.

These warnings are reported by Calibre nmDRC applications and by the ICrules CHEck DRc command.

Table 1. Maximum Results and Polygon Segmentation Warnings

Explanation

WARNING: <number> polygon result(s) segmented in DRC RuleCheck <name>.

 

A polygon result of a rule check required segmentation prior to instantiation in the DRC results database. This message is displayed once for each Calibre nmDRC rule check with polygon results requiring segmentation. The <name> is the name of the rule check and <number> is the number of polygon results within the rule check which were segmented.

WARNING: Maximum result count of <number> exceeded in DRC RuleCheck <name>.

 

The number of results added to the DRC results database for a rule check during the Calibre nmDRC run would exceed the maximum number allowed. The <number> is the value of the maximum results parameter and <name> is the name of the rule check.
