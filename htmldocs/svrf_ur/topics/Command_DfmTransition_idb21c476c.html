Arguments

layer1

A required original or derived polygon layer that is the interconnect layer below layer3.

layer2

A required original or derived polygon layer that is an interconnect layer other than layer1 and is the interconnect layer above layer3.

layer3

A required original or derived polygon layer that is a via layer.

NODAL

An optional keyword that causes internally-generated nodal information to be written to the RDB. A node number is a property of the improved transition itself, not of the individual geometries in the improved transition. The node number of an improved transition is that of the original layer3 shape in the transition. In the unlikely event that a START constraint other than == 1 is specified, and the layer3 geometry in the transition collectively has more than one node number, then a node number is chosen arbitrarily.

Note that the DFM Transition operation remains a non-node-preserving layer constructor, regardless of the presence of the NODAL keyword. In other words, while its presence causes the operation to write nodal information to the RDB, the output is a derived layer or rule check and does not contain nodal information.

USEVIAS vianame1 [vianame2 …]

A required keyword and argument that define a list of via names in decreasing order of priority. Each via should have a corresponding DFM Spec Via specification with the same name that defines the via parameters.

SPACE1 space1_value

A required keyword and positive real number in user units that specify the minimum spacing between existing layer1 polygons and new layer1 polygons. New polygons generated from layer1 must satisfy this spacing value.

SPACE2 space2_value

A required keyword and positive real number in user units that specify the minimum spacing between existing layer2 polygons and new layer2 polygons. New polygons generated from layer2 must satisfy this spacing value.

METALSPACE w1 prl1 s1 [FSPACE s11 s12] [w2 prl2 s2 [FSPACE s21 s22]] …| HORIZ wi1 prli1 si1 [FSPACE si11 si12] …VERT wj1 prlj1 sj1 [FSPACE sj11 sj12] …

An optional keyword set that specifies spacing constraints for neighboring shapes on a metal layer. This keyword can only be used in conjunction with SPACE1 and SPACE2 keywords.

The list of triples is defined as follows:

  • wi — The width threshold.

  • prli — The Parallel Run Length (PRL) threshold. This can be a negative value.

  • si — The resulting space value.

If the width of one of the neighboring shapes on the metal layer is:

wi < w <= wi+1 (suppose the triples are listed in increasing order of wi-s) and they have prl > prli then the space between them should not be less than si. If width is greater than the max value specified, then the max space value given applies. If width is less than or equal to w1, then space1_value applies.

Alternatively, the width, length, and space values can be specified with either the HORIZ keyword or the VERT keyword, or both using one of the following methods:

  • Non-Directional Spacing Method — Using this method, the list of triples (wi, prli, si) are specified as follows:

    • wi — The width threshold.

    • prli — The PRL threshold.

    • si — The space value.

    If the width of one of the neighboring shapes on the metal layer wi < w <= wi+1 (for example, the triples are listed in increasing order of wi to s) and PRL >= prli, then the space between these shapes must be not less than si.

    If width is greater than or equal to the max value specified, then the max space value is used. If width is less than w1, then Layer1_space value is used.

  • Directional Spacing Method — Using this method, the spacing is defined in the horizontal and the vertical directions as follows (see Figure 1):

    • The HORIZ keywords specification is:

      wi < shape_horiz_width <= wi+1 and prl >= prli, then s >= si

      The shape_horiz_width is the internal distance of edges in the horizontal direction and this value may differ from the real width of the shape.

    • The VERT case works similarly.

Figure 1. Directional Spacing Method

  • FSPACE s11 s12

    An optional keyword set that specifies a range of space values that are forbidden for certain metal layers. The keywords are defined as follows:

    • s11 — The starting value of the forbidden space range. This value must be greater than the minimum design rule space value for the layer.

    • s12 — The ending value of the forbidden space range.

    Specify the FSPACE keyword set after each triple (w, prl, s) of METALSPACE as follows:

    METALSPACE w1 prl1 s1 FSPACE s11 s12

    The forbidden space is defined as follows:

    If the width of the metal layer is > w1, and the PRL with the neighboring metal layer is >= prl1, then the space between the metal layer and the neighboring metal layer must be >= s1, and the space defined by s11 <= distance < s12 is forbidden.

    Multiple sets of METALSPACE triples with the same space value, s, can be specified with different FSPACE values.

    Similarly, the parameter definitions and constraints apply if the FSPACE keyword is specified with the HORIZ and VERT keywords as follows:

    METALSPACE HORIZ wi1 prli1 si1 FSPACE si11 si12

    METALSPACE VERT wj1 prlj1 sj1 FSPACE sj11 sj12

ENDTOEND w prl s

An optional keyword and list of triples defined as follows:

  • w — The width threshold.

  • prl — The parallel run length threshold.

  • s — The resulting space value.

This means that the space between two line ends on the corresponding layer should be at least s if they have parallel run length greater than prl. The line end is width < w edge formed by two consecutive 90-90 degree convex corners. All the edges that are not line ends are lines. This keyword can only be used in conjunction with SPACE1 and SPACE2 keywords.

ENDTOLINE w prl s

An optional keyword and list of triples defined as follows:

  • w — The width threshold.

  • prl — The parallel run length threshold.

  • s — The resulting space value.

This means that the space between a line end and a line on the corresponding layer should be at least s if they have parallel run length greater than prl. The line end is width < w edge formed by two consecutive 90-90 degree convex corners. All the edges that are not line ends are lines. This keyword can only be used in conjunction with SPACE1 and SPACE2 keywords.

DENSELINEEND W Q T S R  Sp

An optional keyword and value set that specify the dense metal line end rule. This keyword is used with the SPACE1 or SPACE2 spacing rules.

Figure 2. Dense Line End Rule

The following figure illustrates this rule. The red line in the figure is the violation.

Figure 3. DENSELINEEND

Referring to the preceding figure, the rule is defined as follows:

  • A line end is an edge that is formed by extending the line by T and has length less than W (the first parameter in DENSELINEEND).

  • Region 1 is formed by taking Q (the second parameter) distance from the line end endpoint(s) along with its adjacent edge and T (third parameter) is extension to the outer side. S2 = S (fourth parameter) - width(shape).

  • Region 2 is formed by extending the line end by T on both sides and enlarging by Sp value.

When using this rule, both regions cannot simultaneously contain any metal. If there is metal in Region 1, then it is called dense metal line end. If there is also metal in Region 2, then the rule is violated.

For small jogs with length < R, Region 1 is not considered. For most cases if there is no small jog, then Region 1 will appear on both sides of metal line end.

[WIDTHENC1 {RANGE wmin wmax {GOOD val1 val2 val3 val4}} [RANGE wmin {GOOD val1 val2 val3 val4}]]

An optional keyword and value set that specify a width dependent enclosure check for layer1 where:

  • wmin — Specifies the minimum width value.

  • wmax — Specifies the maximum width value. If not specified, then this rule applies to all widths such that widths > wmin.

For the given width range wmin, wmax if some via metal enclosure width is wmin < width <= wmax, then that via enclosure must be greater than or equal to the GOOD specifications given.

The rule should contain at least one RANGE, GOOD couple defined. In other words, each RANGE should have at least one GOOD quartet. The RANGE values should be non-negative intervals that may overlap, but only as endpoints. The maximum value for the last RANGE should be omitted, which means that the rule applies to all widths > wmin of that RANGE.

FORBIDDENPITCH1 [SINGLE | DOUBLE] width_value space1 space2

An optional keyword and value set that specify for layer1 a width sensitive zone check where the sum of width and space (w+s+w+s or s+w+s+w) must not be within a range of values.

The width sensitive check is such that if the line width is less than width_value, then the sum across the zone must not be inside [space1, space2], where space1 < space2.

The optional SINGLE and DOUBLE keywords specify checking a single pitch (w+s), or a double pitch (w+s+w+s) respectively. The default is to check the DOUBLE pitch.

SPECWIDTH1 w1 [w2 …] | HORIZ w1 [w2 …] VERT w3 [w4…]

An optional keyword and value set that specify a discrete width rule check for layer1.

On some layers, there is a fixed set of discrete line widths allowed. When a via with enclosure is added, the enclosure must match one of these widths, if specified. The enclosure may be filled to meet this rule. Choose one of the following methods to specify the width:

  • w1 [w2 …] — Specifies a single list of widths.

  • HORIZ w1 [w2 …] VERT w3 [w4…] — Specifies lists of allowed widths in horizontal and/or vertical directions.

This applies only to line widths. The width of a line is generally defined as the internal distance between the two long edges of a rectangle. In case of a complex shape, the shape is decomposed into overlapping rectangles, and the check is performed on these.

When specifying a width list, the last element of the list is applied with >= inequality. In cases where this inequality is not required, specify a 0 (zero) at the end of the list. For example:

SPECWIDTH1 0.040 0.080 0

If the HORIZ and VERT parameters are defined, then the rule is applied for both directions, that is both dimensions of the shape are checked and corrected if needed.

ADJACENTEDGE1 width1_val1 length1_val1 [width1_val2 length1_val2]

If the edge length is less than width_val, then the adjacent, longer edge must be greater than or equal to corresponding length_val as shown in Figure 9. The newly-added metals are expanded in such a way that they do not violate that rule.

You can only specify at most two width-length pairs.

When two width‑length pairs are specified, the tool automatically orders the width‑length pair specifications with the lesser width first as width1_val1 followed by width1_val2.

The following conditions apply when two width‑length pairs are specified:

  • If width_val is less than width1_val1, then length_val must be greater than or equal to length1_val1.

  • If width_val is greater than or equal to width1_val1 and less than width1_val2, then length_val must be greater than or equal to length1_val2.

An optional keyword set (one of three) that specifies a rule that defines the spacing between a via to underlying metal layer shapes of different nets. This rule checks the via-to-metal spacing of a via below the via that is being inserted.

You can specify up to three VIAMSPACE rules in a DFM Transition operation: one for the below via layer, one for the insertion via layer, and one for the above via layer. Refer to Figure 13 for a cross section of the via and metal layers.

Figure 13. VIAMSPACE — Above Via and Below Via Layer Cross Sections

In general, the VIAMSPACE specifications check for three cases of via‑to‑metal layer spacing violations that can be introduced when via enclosures are enlarged due to via insertion.

  • Below Via — In the first case example, a V2 via layer insertion affects metal layers M2 and M3 and can result in a violation on V1 (the below via layer) if an added M2 shape meets the threshold width and encloses a V1 via. This type of violation breaks rules for M1‑V1 spacing and can occur even when the tool is not specifically adding V1.

  • Insertion Via — In the second case example, a violation can happen on the via insertion layer (layer3) when an M3 (layer2) shape meets the threshold width and encloses a via enclosure on M2 (layer1).

  • Above Via — In the third case example, a V2 insertion can result in a violation on V3 (the above via layer) when a shape is added on M3, and the V3 via has an enclosure on M4 that reaches the threshold width.

As seen in these cases, when a via is inserted, spacing conflicts can be introduced to the above and below via layers including the associated metal layers.

Description

DFM Transition is an SVRF rule file operation for improving via or contact transitions between two levels of interconnect. DFM Transition is part of the Calibre YieldEnhancer toolset. The operation is a polygon-directed, non-node-preserving layer constructor.

You can use DFM Transition to insert redundant transition polygons into your layout to improve interconnect integrity. The DFM Transition operation inserts square vias into valid locations, according to conditions you specify. It also adds polygons as needed to the metal layers so that additional vias can be accommodated. Spacing and enclosure constraints for new polygons are part of the operation syntax.

This operation attempts to add only one via at a time, at valid transition locations for each DFM Transition operation. This means multiple runs or multiple operations in a single run might be necessary to obtain the desired number of vias at transitions. If a via cannot be placed at a transition, that transition gets no new polygons.

DFM Transition Legacy Syntax and Examples

Note:

The following sections describe the legacy syntax of the DFM Transition SVRF statement.

When DFM Transition adds vias to the layout, the vias will be placed in one of 16 possible configurations. There are four configuration types and four orientations for each.

Figure 25. Possible Via Configurations (DFM Transition Legacy Syntax)

The operation works as follows:

  1. For each transition, the operation attempts to add one additional square of layer3, with side length size3_value. The operation does not move or replace existing layer3 geometry.

  2. To add the one additional square of layer3, the operation tries one of four locations. These locations are directly north, south, east, or west of one of the layer3 polygons in the transition. The exact location and layer3 polygon that is used is selected as follows:

    1. The operation examines all possible insertion options per single via (16 possible) to identify the Calibre nmDRC clean orientations for each cell type.

    2. It chooses the option(s) that add the fewest jogs. For example, Option A adds fewer jogs than Option B:



    3. From the remaining options, it chooses the option(s) that adds the least total metal. For example, Option A adds less metal than Option B:



    4. If more than one option has the same amount of added metal, then the operation selects the placement based on the following preference:

      Pattern1-SS > Pattern2-SF > Pattern3-FS > Pattern4-FF

      where patterns are shown in Figure 25.

    5. If there is more than one option remaining, it will pick the via cell in order randomly.

  3. The polygons added consist of a layer3 square of specified size, along with potentially new polygons from layer1 or layer2 that enclose the layer3 square by a specified amount. All new polygons on all three layers satisfy minimum spacing and enclosure values defined in the operation, with both new and existing polygons as specified here:

    • SPACE2: New polygons on layer2 satisfy external minimum spacing greater than or equal to space2_value.

    • SPACE3: New polygons on layer3 satisfy external minimum spacing greater than or equal to space3_value.

    • ENC1: New polygons in layer1 enclose new squares on layer3 by greater than or equal to enc1_value1. If enc1_value2 is specified, then layer1 encloses the new squares of layer3 by enc1_value1 on two opposite sides, and enc1_value2 on the other two sides.

    • ENC2: New polygons in layer2 enclose new squares on layer3 by greater than or equal to enc2_value1. If enc2_value2 is specified, then layer2 encloses the new squares of layer3 by enc2_value1 on two opposite sides, and enc2_value2 on the other two sides.

    This part of the algorithm does not require the minimum width of layer1 and layer2. The following is assumed:

    • minimum width in layer1 <= (size3_value + min1)

    where min1 is the lesser of enc1_value1 and enc1_value2. Correspondingly, the following is also assumed:

    • minimum width in layer2 <= (size3_value + min2)

    where min2 is the lesser of enc2_value1 and enc2_value2.

    This assumption is necessary to ensure that new layer1 and layer2 polygons enclosing the added vias, when extended to touch the existing geometry, do not violate minimum width in those layers.

  4. The output of this operation consists of all new polygons created for layer1 if OUTPUT1 is specified (the default), for layer2 if OUTPUT2 is specified, or for layer3 if OUTPUT3 is specified.

    The NODAL keyword is used to generate connectivity information on layer3 for various uses, primarily by DFM visualization and backannotation environments.

RDB Output

RDB output is optional, but helpful in most cases where a detailed analysis of the results is desirable. RDB specification details and syntax are covered in the “Transitions RDB File Format” section of the Calibre YieldAnalyzer and YieldEnhancer User’s and Reference Manual.

Limitations

The actual Calibre nmDRC rules (including antenna rules) involving the input layers may be complex and extensive. This operation cannot take all of these factors into account. It is your responsibility to provide a simple parameter set that maintains DRC cleanliness over a potentially complex set of DRC relationships.

Concurrency

All DFM Transition operations with the same input layers (in the same order), and the same measurement parameters, are executed concurrently.

Any new polygons that are generated by the operation conform to these values.

The OUTPUT1, OUTPUT2, and OUTPUT3 keywords specify which layer the output comes from, and they follow the order of the layers specified in the DFM Transition operation. The layers M1_new, M2_new, and via_new can subsequently be merged with the original layers by using an OR operation.

The placement of the vias is based upon internal algorithms, and is arbitrary. The tool attempts to place new vias north, south, east, or west of the location of the existing vias. The spacing, enclosure, and via width parameters determine where new polygons can be placed, and the dimensions of new polygons. No new vias are placed at transitions where a valid location cannot be found. The following figure shows how polygons might be generated for the previous example.



An additional keyword for the DFM Transition operation is the START keyword. You specify it with a constraint, and it instructs the DFM Transition operation to begin at transitions where the specified number of vias can be found. For instance, specifying START == 2 causes the operation to begin at transitions with two vias.

Continuing the previous example, here is how you could place successive vias at M1/M2 transitions:

M1_new = DFM TRANSITION M1 M2 via 
					SPACE1 0.10 SPACE2 0.12 SPACE3 0.08 ENC1 0.02 ENC2 0.03
					SIZE3 0.05 OUTPUT1
M2_new = DFM TRANSITION M1 M2 via 
					SPACE1 0.10 SPACE2 0.12 SPACE3 0.08 ENC1 0.02 ENC2 0.03
					SIZE3 0.05 OUTPUT2
via_new = DFM TRANSITION M1 M2 via 
					SPACE1 0.10 SPACE2 0.12 SPACE3 0.08 ENC1 0.02 ENC2 0.03
					SIZE3 0.05 OUTPUT3
//add results to the original layers
next1 = M1 OR M1_new
next2 = M2 OR M2_new
next3 = via OR via_new
//add third via, where possible
M1_new2 = DFM TRANSITION next1 next2 next3
					SPACE1 0.10 SPACE2 0.12 SPACE3 0.08 ENC1 0.02 ENC2 0.03
					SIZE3 0.05 START == 2 OUTPUT1
M2_new2 = DFM TRANSITION next1 next2 next3 
					SPACE1 0.10 SPACE2 0.12 SPACE3 0.08 ENC1 0.02 ENC2 0.03
					SIZE3 0.05 START == 2 OUTPUT2
via_new2 = DFM TRANSITION next1 next2 next3 
					SPACE1 0.10 SPACE2 0.12 SPACE3 0.08 ENC1 0.02 ENC2 0.03
					SIZE3 0.05 START == 2 OUTPUT3
//add results to previous
M1_final = next1 OR M1_new2
M2_final = next2 OR M2_new2
via_final = next3 OR via_new2
Note:

The DFM Transition operation considers only spacing and enclosure constraints defined within the operation, and cannot account for all of the DRC conditions that need to be met in your design. You should always verify your modified design by using the standard Calibre nmDRC checks for all required layers. LVS and parasitic extraction verification may also be necessary after modifying interconnect layers.
