Connectivity Extraction Statements

Connectivity extraction statements define the electrical connectivity and net names of a design. They support many varied applications including on-the-fly connectivity maintenance within ICverify, connectivity-based dimensional check operations within the Calibre nmDRC system, parasitic extraction operations, and LVS. This section summarizes the connectivity extraction statements.

The following table lists the connectivity extraction statements. For detailed descriptions, refer to the command reference.

Table 1. Connectivity Extraction Statements

Operation

Description

Attach

Transfers connectivity information from one layer to another. Used primarily for text label attachment.

Connect

Defines electrical connections on input layers.

Disconnect

Deletes the existing connectivity model. Applies only to Calibre nmDRC applications using incremental connectivity.

Label Order

Specifies the order in which connectivity extraction searches for shapes to connect to other shapes.

Layout Text

Specifies layout database text directly in the rule file. Not used in ICverify.

Layout Text File

Specifies a file that contains Layout Text objects. This is used when there are many text objects to specify. Not used in ICverify.

Sconnect

Specifies one-directional connection between objects on specified layers. Used in detection of soft connections.

Stamp

Connectivity-related operation that constructs a layer containing containing polygons from one layer having the connectivity of overlapping polygons from another layer.

Text

Specifies top-level layout database text objects (free-standing text) directly in the rule file. It also allows text objects read from the layout database to be edited.

Text Depth

Specifies the hierarchical depth for reading text objects from the layout database for the connectivity extractor.

Text Layer

Specifies the layers in the database from which connectivity extraction text is read.
