PEX Probe File

Parasitic extraction

Specifies specific points on a net to use for timing verification.

Usage

PEX PROBE FILE filename [PROTECT] [PROBEPORT]

Arguments

filename

Required name of a file containing one or more cell entries in the following format:

CELL cell_name

probe_name local_x local_y layername [SINGULAR][DEVICE][IGNORE]

[probe_name local_x local_y layername [SINGULAR][DEVICE][IGNORE]]...

where:

  • CELL cell_name is a required keyword paired with a valid extracted cell and a layout cell name.

  • probe_name must fall on a net for which parasitics are netlisted. More than one probe name specification entry can be specified for a CELL.

  • a polygon exists on layername, at the specified coordinates (local_x, local_y), where the coordinates are given in user units.

  • SINGULAR is an optional parameter that limits the probe to a single instance.

  • DEVICE is an optional parameter that causes probe connections to be treated as device connections.

  • IGNORE is an optional parameter that specifies to ignore resistivity on resistor bodies where the probe is placed.

Note:

During device extraction, Calibre may promote polygons out of a cell. If you defined a probe point for a polygon, and the polygon got promoted out of the cell, the probe point you defined will not be connected. You can prevent this from happening by putting probe points at the highest level for which a net exists.

PROTECT

An optional keyword that prevents probe points from being removed when a probe point is at the same node as a pin or port. When PROTECT is specified, the Calibre xRC Formatter inserts a 0.01 ohm resistor between the probe point and the port or pin. (When PROTECT is not specified, the probe point is removed and a warning is printed.) The resistor value can be modified using the PEX Netlist Unshort Device Pins statement.

PROBEPORT

An optional keyword that specifies to replace probes on the net represented as subnodes (*|S) with ports (*|P) when extracting a DSPF netlist.

Description

This statement instructs the formatter to insert specific points on each net in the extracted netlist, which can be used to verify timing.

Within the parasitic model, each probe point exists as an internal connection with a unique probe_name. You can access each probe point by net and probe_name, depending on your simulator’s behavior. The probe file parameter set CELL cell_name defines the cell context of the probe_names listed after it and before the next CELL parameter set in the probe file.

Calibre xRC returns warnings when using the PEX Probe File statement under the following conditions:

  • If a duplicate probe point is found on a net, Calibre xRC ignores the duplicate.

  • If a probe point is defined to be on a layer that was promoted out of a cell as a side effect of device extraction, Calibre xRC does not connect the probe point.

  • If a probe point conflicts with a port, a device pin, or an hport and the PROTECT keyword is not specified, Calibre xRC ignores the probe point.

When a probe point is defined for a top level cell on a route inside a specific smaller cell instance in a hierarchical design, the probe is automatically replicated for all instances of this smaller cell. When the probe file parameter SINGULAR is specified, a single probe is applied to one instance of the small cell. The SINGULAR parameter should only be used for probes in top level cells. The cell instance the probe appears in is controlled by the probe location specified in the probe file. If a singular probe is found on a cell that is not a top level cell, then a warning is generated and the cell is ignored.

Examples

Example 1

This statement uses the information specified in testfile to insert specific points on each net in the extracted netlist:

PEX PROBE FILE testfile

Example 2

Given the following probe file, myprobefile:

CELL xrc_test2
//probe name local_x local_y layername
   vss1        50       -23      8
   vss2        150      -23      8

Specifying the following statement when extracting a DSPF netlist:

PEX PROBE FILE myprobefile

produces probes on the net represented as subnodes (*|S) on the net, for example:

*|S (VSS1 50 -23)
*|S (VSS2 150 -23)

To replace the probes on the net represented as subnodes (*|S) with ports (*|P) when extracting the DSPF netlist, use the PROBEPORT keyword:

PEX PROBE FILE myprobefile PROBEPORT

For example:

*|P (VSS1 Z 0 50 -23)
*|P (VSS2 Z 0 150 -23)

Example 3

Given the following probe file, p.file:

CELL top_cell
//probename local_x local_y layername
   P1A        5.59    5.3   metal3    SINGULAR

Specifying the following statement when extracting a DSPF netlist:

PEX PROBE FILE p.file

produces a single probe on the net represented as a subnode (*|S), for example:

*|S (P1A 5.59 5.3)
