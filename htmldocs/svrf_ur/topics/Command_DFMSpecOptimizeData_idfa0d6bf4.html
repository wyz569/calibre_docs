DFM Spec Optimize Data

Specification statement

License: Calibre YieldEnhancer

Defines an optimization expression used with DFM Spec Optimize.

Usage

DFM SPEC OPTIMIZE DATA "optimizer_name"
‘[’ optimization_expression ‘]’
{ { < upper_bound [TOMAX]} | { > lower_bound  < upper_bound} }
[WINDOW {w | wx wy} [STEP {s | sx sy}] ]

Arguments

optimizer_name

A required string giving the name of the optimizer. The string must be in quotes.

optimization_expression

A required argument defining the expression used to optimize the fill layer. The expression must be enclosed by [] brackets. See DFM Spec Fill Optimizer for details on writing the expression.

Typically, the expression is in the form of a density, with the denominator of the expression set to AREA(), which returns the area of the current data capture window. However, other expressions may be used; see the “Description” section for expression requirements.

{ < upper_bound [TOMAX]} | { > lower_bound  < upper_bound }

A required argument choice that defines a numeric optimization constraint. The constraint defines a goal that the algorithm attempts to achieve, in terms of optimization_expression values.

These forms are supported for the optimization constraint:

  • < upper_bound [TOMAX]

    Shapes are removed until the optimization_expression is less than or equal to the upper_bound.

    The optional TOMAX keyword has no effect, but is included for consistency with DFM Spec Fill Optimizer.

  • > lower_bound  < upper_bound

    Shapes are removed until the optimization_expression value is less than or equal to upper_bound, and greater than or equal to lower_bound.

    The lower_bound has a higher priority than the upper_bound. In order to meet the lower_bound requirement of the local optimizer, some windows may exceed the upper_bound for the local or global optimizer.

    If the initial optimization_expression value is less than lower_bound in a window, it is not changed, because DFM Optimize can only removes shapes.

Unlike in DFM Spec Fill Optimizer, the upper_bound and lower_bound may not be expressions and you cannot omit the constraint operators (< and >).

WINDOW {w | wx wy} [STEP {s | sx sy}]

An optional argument set that specifies the size of the data capture window within which the optimization_expression is calculated. The WINDOW keyword is used when defining the local optimizer for DFM Spec Optimize.

  • WINDOW {w | wx wy} — The window is a square with sides w user units long, or a rectangle that is wx user units in the x-direction and wy user units in the y-direction.

  • STEP {s | sx sy} — An optional keyword set with WINDOW that defines an offset for the analysis window. The optimization_expression is evaluated first with the analysis window grid at an offset of (0,0). The grid is then shifted and the analysis is repeated at a series of offsets. See the definition of STEP in DFM Spec Fill Optimizer.

The windows are tiled over the extent of the GUARDED and OPTIMIZABLE layers specified in DFM Spec Optimize.

The WINDOW keyword is not used when defining the global optimizer. In this case the optimization expression is evaluated over the extent of the GUARDED and OPTIMIZABLE layers specified in DFM Spec Optimize.

Description

DFM Spec Optimize Data defines the optimizers that are referenced by DFM Spec Optimize. These optimizers are used by the DFM Optimize operation to optimize fill for variable-size fill shapes.

When writing the optimization expression, layers that are not being optimized are referred to by name and should be listed as GUARDED layers in DFM Spec Optimize. The layers being optimized are referred to by the special layer name _FILL_ in the optimization expression. The layers being optimized are listed as OPTIMIZABLE in DFM Spec Optimizer, but are not referenced by name in the optimization expression.

An example optimization expression and constraint is the following:

[((AREA(m1) + AREA(m2) + AREA(_FILL_))/AREA()] < 0.5

AREA() returns the area of the current data capture window. The layers m1 and m2 are listed as GUARDED in the corresponding DFM Spec Optimize statement. The layers being optimized are listed with the OPTIMIZABLE keyword in DFM Spec Optimize.

The numerator of the optimization expression must be of this form:

c + x*AREA(_FILL_)

Where c may be an expression and the “c +” portion is optional. The scaling factor x is optional and x must be a positive scalar when used; x cannot be an expression.

The denominator of the optimization expression cannot contain AREA(_FILL_) and the denominator should not change as a result of fill optimization.

You can also write an optimization expression that is not in the form of a density, that is, without AREA() in the denominator. See “Example 3:” with DFM Optimize.

When WINDOW is not specified, the optimization expression is evaluated over the extent of the GUARDED and OPTIMIZABLE layers specified in DFM Spec Optimize. When WINDOW is specified, evaluation of the optimization expression starts with the window at the lower left of the extent of all GUARDED and OPTIMIZABLE layers.

See DFM Optimize for a complete example.
