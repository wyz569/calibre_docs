LVS Preserve Floating Top Nets

Specification statement

Controls preservation of floating named top-level nets in the extracted layout SPICE netlist. Used only in Calibre nmLVS-H and Calibre PERC.

Usage

LVS PRESERVE FLOATING TOP NETS {NO | YES}

Arguments

NO

Keyword that does not preserve floating named nets (no device connected) in the top-level cell. This is the default behavior if you do not include this statement in your rule file, and it causes only nets connected to devices to be written to the SPICE netlist.

YES

Keyword that preserves floating named nets in the top cell.

Description

Controls preservation of floating, named, top-level nets in the extracted layout SPICE netlist. If NO is specified, or the statement is not used, only top-level nets connected to devices are written to the SPICE netlist.

If YES is specified, top-level nets that have names but no device attached are preserved in the SPICE netlist. These nets are represented with the *.J SPICE statement (an alias for a *.CONNECT statement) using identical parameters:

*.J <name> <name>

For example:

*.J net1 net1

Note that specifying identical parameters in a *.J (or a *.CONNECT) statement does not follow typical usage for such statements, but is used in this case to preserve the floating net name.

Floating nets preserved by the YES setting are preserved by the LAYOUT NETLIST WRITE command in the Calibre Connectivity Interface (CCI). See the Calibre Query Server Manual for details on the command.

See also LVS Netlist Unnamed Isolated Nets.

Examples

// Preserve floating top nets in the SPICE netlist
// This flags floating top-level nets during LVS comparison
LVS PRESERVE FLOATING TOP NETS YES
