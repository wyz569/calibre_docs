LVS Center Device Location

Specification statement

Identifies device body location coordinates.

Usage

LVS CENTER DEVICE LOCATION {NO | YES}

Arguments

NO

A keyword that specifies arbitrary coordinate points are returned for the device seed shape. This is the default.

YES

A keyword that returns a centered device seed shape location.

Description

This statement controls the reporting of device body location coordinates. These appear in $X and $Y coordinates in an extracted SPICE netlist or in a PDSP file from the Calibre Connectivity Interface (CCI).

Normally, the location of a device is an arbitrary coordinate pair from some point on the device seed shape. When YES is specified, the location is selected as follows:

  • Center of the rectangular extent of the seed shape is chosen if that point actually lies on the seed shape.

  • When the center of the rectangular extent of the seed shape does not lie on the seed shape (say for a donut or C-shaped device), then the resulting polygon is segmented into trapezoidal shapes and the point selected is at the centroid of an arbitrarily selected trapezoid. Note that for a circular donut shape this will place the “center” near an outer edge of the donut, but inside the seed shape boundary.

When YES is specified, the Query Server LAYOUT NETLIST DEVICE LOCATION setting can only be set to CENTER. The VERTEX setting is not available (and is the default). If it is requested, the following error message is issued:

ERROR(184): Rule file setting LVS CENTER DEVICE LOCATION YES disallows this setting.

Similarly, the Tcl Query Server (-qs) and Calibre YieldServer only allow the dfm::set_layout_netlist_options ‑device_location CENTER argument to be used when YES is specified. If the VERTEX argument is used (the default), the following error message is issued:

Error 12: vertex is an invalid argument for command dfm::set_layout_netlist_options with LVS CENTER DEVICE LOCATION YES in rules.

See also LVS Center Device Pins.

Examples

If this rule file setting is used:

LVS CENTER DEVICE LOCATION YES
MASK SVDB DIRECTORY svdb QUERY CCI

Then these QUERY SERVER commands give the centroid coordinates of device seed shapes as their locations:

DEVICE INFO X0/X1/M1
LAYOUT NETLIST DEVICE LOCATION CENTER
LAYOUT NETLIST WRITE cci.spi
