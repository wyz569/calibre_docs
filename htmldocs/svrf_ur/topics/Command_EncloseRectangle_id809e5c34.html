Enclose Rectangle

Layer operation

Selects all polygons that can enclose a rectangle of the specified dimensions.

Usage

ENCLOSE RECTANGLE layer width length [ORTHOGONAL ONLY [FIXED]]

Arguments

layer

A required original or derived polygon layer.

width

A required positive, floating-point number interpreted in user units.

length

A required positive, floating-point number interpreted in user units.

ORTHOGONAL ONLY [FIXED]

An optional keyword that specifies the enclosed rectangle must have sides that are parallel (respectively) to the coordinate axes of the database in order for the layer polygon to be selected.

  • FIXED — Specifies that the width and length are measured along the x-axis and y‑axis, respectively, and cannot be interchanged.

Description

Selects all polygons on the specified layer that can enclose a rectangle of the specified width and length. One or more edges of an enclosed rectangle can be coincident with the enclosing polygon.

By default, the tool considers only enclosed rectangles that have sides parallel (respectively) to the coordinate axes or that have sides at a 45-degree angle to the coordinate axes. If ORTHOGONAL ONLY is specified, the enclosed rectangle must have sides parallel to the coordinate axes.

By default, the tool may align width in any direction: along either the x- or y-axis, or either the plus or minus 45-degree angle. To align width and length with the x-axis and y-axis respectively, specify FIXED with ORTHOGONAL ONLY. By default, the operation behaves exactly the same if width and length are interchanged. For example, the following two operations produce equivalent results:

ENCLOSE RECTANGLE poly 8 2
// this produces the same results: 
ENCLOSE RECTANGLE poly 2 8

Performance is faster with ORTHOGONAL ONLY.

See also Not Enclose Rectangle, Enclose, Not Enclose, Enclosure, Rectangle Enclosure.

Examples

rule { ENCLOSE RECTANGLE metal 2 4 }
// select all polygons on metal that can enclose a 
// 2 x 4 rectangle that is orthogonal to the database axes or
// is oriented at 45 degrees to the database axes.
