tvf_numeric_function

Device TVF data access function.

Provides access to a numeric value returned by a Tcl procedure defined within a TVF Function definition.

Usage

TVF_NUMeric_FUNction::tvf_func::tcl_proc( parameter [, parameter …] )

Arguments

tvf_func

Required name of a TVF Function.

tcl_proc

Required name of a Tcl procedure in the tvf_func block.

parameter

Required name of a parameter that is passed to the tcl_proc. More than one parameter is specified in a comma-delimited list.

Return Values

A numeric value from the tcl_proc.

Description

This function is used in a device annotation or property computation program to access numeric values from a procedure in a TVF Function block.

TVF Numeric Object

A TVF numeric object represents a simple number, numeric variable, or output from a numeric Device property function. The object can be evaluated using [$object] to access the value contained in the object.

The following example accesses the property value for L, passes it to the calc_eff_a function, and evaluates the TVF object L as follows:

DEVICE MN NGATE NGATE SD SD <DIFF>
  [ PROPERTY W, L, SEFFA, SEFFB
  ...
    SEFFA = TVF_NUM_FUN::device_func::calc_eff_a( ENC, W, L )
  ...
  ]

TVF FUNCTION device_func [/*
  proc calc_eff_a { enc W L } {
    ...
    set length [$L] 
    ...
  }
*/]

String Arguments in TVF_NUMeric_FUNction

TVF_NUMeric_FUNction accepts strings and string properties as input arguments. These arguments are available inside the called Tcl proc as TVF string objects.

Strings may be passed in as quoted text, values derived from the Device statement’s text_string function, or Variable statements from the rule file:

VARIABLE vProp "p2=4.3"
DEVICE MN(nfet) ngate ngate(g) nsd(s) nsd(d) psub(b) 
  TEXT PROPERTY LAYER DTEXT [
    PROPERTY p1, p2, p3
    StrProp = text_string( DTEXT, " " )
    p1 = tvf_num_fun::device_func::scan_text( StrProp )
    p2 = tvf_num_fun::device_func::scan_text( vProp )
    p3 = tvf_num_fun::device_func::scan_text( "p3=99" )
]

In this example, the Device statement searches the layer DTEXT for text objects to be used for assigning property values. Within the property computation section, three properties are declared: p1, p2, and p3. The text_string() function assigns the values of text objects on layer DTEXT that intersect ngate polygons to the StrProp variable. If no objects are found on an ngate polygon, then a space character is assigned to StrProp for that polygon.

The three TVF_NUMeric_FUNction calls in this example all call a proc named scan_text. In the first case, the value of the variable StrProp is passed as an argument to scan_text. In the second case, the value of the rule file variable vProp is passed to scan_text. In the third case, the string “p3=99” is passed to scan_text.

The scan_text proc should return a number for each of the properties p1, p2, and p3. If a number is not returned, then an error like this is produced:

*** Error obtaining results from TVF call to scan_text
Incorrect type returned for TVF_NUMeric_FUNction: expected floating-point 
number but got "p2=4.3"

Strings passed as arguments from TVF_NUMeric_FUNction are accessed inside the called Tcl proc as TVF string objects. These TVF objects accept the single subcommand called string to return a Tcl string representing the object that was passed in. The syntax is this:

object string

For example, strings passed to TVF_NUMeric_FUNction as follows:

p1 = tvf_num_fun::device_func::scan_string( "p1=3e-06 p2=2.3", "p1" )

can be accessed as follows:

TVF FUNCTION device_func [/*
# this proc expects key-value pairs linked with "=" and that the pairs are 
# delimited by spaces. the proc also expects a separate key value. 
# scan_string ("p1=3e-06 p2=2.3", "p1") returns "3e-06"
proc scan_string {text_input key_input} {
  set rv 0
  set text_string [ $text_input string ]; # get the key-value pairs
  set key [ $key_input string ];          # get the key
  if {$text_string != "empty"} {
    regexp "$key=(\\S+)"  $text_string junk rv
    # performs a regular expression match for the key-value pair in
    # $text_string. the entire string match is sent to junk. the value
    # from (\\S+) is sent to the variable rv.
  }
  return $rv 
}
*/]

Deprecated Syntax

The original form of TVF_NUMeric_FUNction is shown in this example and is deprecated:

PROP_A = TVF_NUM_FUN( "get_prop", "device_func", ENC, StrVal, W, L )

This is the current syntax:

PROP_A = TVF_NUM_FUN::device_func::get_prop( ENC, StrVal, W, L )
