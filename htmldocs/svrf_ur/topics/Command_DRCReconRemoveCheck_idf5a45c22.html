DRC Recon Remove Check

Specification statement

Specifies rule check statements or rule check groups not to be executed in a Calibre nmDRC Reconnaissance run.

Usage

DRC RECON REMOVE CHECK rule_check [rule_check …]

Arguments

rule_check

A required argument that specifies the name of a rule check statement or rule check Group. You can specify rule_check any number of times in one statement. The rule_check must be a selected rule check after resolution of any DRC Select Check and DRC Unselect Check statements, or their defaults.

The rule_check argument can also be one of the keywords in the following table. These keywords specify a predefined set of rule checks. The Default column indicates whether the set of rule checks is executed by default in a Calibre nmDRC Recon run.

Keyword

Definition

Default?

RECON_DEFAULT

All rule checks that would be executed by default in Calibre nmDRC Recon mode.

Yes

RECON_DENSITY

All selected rule checks that require a Density operation.

Yes

RECON_OFFGRID

All selected rule checks that require an Offgrid Directional or non-simple Offgrid operation. A non-simple Offgrid operation uses any of the following keywords: INSIDE OF LAYER, RELATIVE, ABSOLUTE, EDGE, HINT, or OFFSET.

No

RECON_CONNECT

All selected rule checks that require connectivity, including those with DFM DV operations.

No

RECON_DV

All selected rule checks that require a DFM DV operation.

No

RECON_MP

All selected rule checks that require a Calibre Multi-Patterning operation.

No

RECON_PM

All selected rule checks that require a Calibre Pattern Matching operation.

No

Description

Specifies a set of rule checks to exclude in Calibre nmDRC Recon mode. This statement may appear in the rule file any number of times. It is ignored when -recon is not specified on the command line.

By default, the Calibre nmDRC Recon mode chooses a set of rule checks from the rule file, which may or may not be the complete set of checks. See “Chip-Level Verification” in the Calibre Solutions for Physical Verification manual. You can specify a custom set of checks to execute with this statement and the related statements DRC Recon Add Check, DRC Recon Add Check By Layer, and DRC Recon Remove Check By Layer.

The rule checks that are explicitly named must be selected for execution in a standard Calibre nmDRC run. For example, these two statements result in a compiler error:

DRC UNSELECT CHECK min_spacing_metal1
DRC RECON REMOVE CHECK min_spacing_metal1 
// Compiler error:
ERROR: Error SPC27 - DRC RECON REMOVE CHECK min_spacing_metal1 is not 
a recon RuleCheck.

When using the rule check group keywords, it is not necessary for all rule checks that belong to the group to be selected. For example, this compiles successfully: 

DRC UNSELECT CHECK m1_density  //unselect one density check
DRC RECON REMOVE CHECK RECON_DENSITY
// Result: All default Calibre nmDRC Recon checks are executed, but 
// Density checks are not executed

Check selection for a Calibre nmDRC Recon run is done by this process:

  1. Determine the checks that are selected to run in a standard Calibre nmDRC run.

  2. From the set of checks selected in Step 1, enable checks for the Calibre nmDRC Recon run:

    • If no DRC Recon Add Check or DRC Recon Add Check By Layer statements exist in the rule file, enable the default set of Calibre nmDRC Recon rule checks.

    • If DRC Recon Add Check or DRC Recon Add Check By Layer statements exist in the rule file, enable only those rule checks.

  3. If DRC Recon Remove Check or DRC Recon Remove Check By Layer statements are in the rule file, disable those rule checks.

    This step may remove rule checks added in the previous step.

Examples

Also see “Selecting Checks in Calibre nmDRC Recon” in the Calibre Solutions for Physical Verification manual.

Example 1

Remove density checks from the Calibre nmDRC Recon run:

DRC RECON REMOVE CHECK RECON_DENSITY

Example 2

If you use DRC Recon Add Check By Layer, it adds all checks that require the specified layer, including checks not selected by default in a Calibre nmDRC Recon run. To remove checks not run by default, do the following:

DRC RECON ADD CHECK BY LAYER m1
DRC RECON REMOVE CHECK RECON_CONNECT RECON_DV RECON_MP 
          RECON_PM RECON_OFFGRID

Example 3

Assume this statement is in the rule file and is the only DRC Select Check or DRC Unselect Check statement:

DRC SELECT CHECK met1.1 met1.2 met1.3

The following statement specifies that the met1.1 check is not run in Calibre nmDRC Recon mode, but all other checks that run by default in Calibre nmDRC Recon mode are executed.

DRC RECON REMOVE CHECK met1.1
