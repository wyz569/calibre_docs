LVS Netlist Box Contents

Specification statement

Specifies whether to write internal contents of layout LVS Box cells to the output netlist. Used only in Calibre nmLVS-H, Calibre PERC, and Calibre xRC.

Usage

LVS NETLIST BOX CONTENTS {YES | NO}

Arguments

YES

Keyword that instructs the SPICE netlister to write the internal contents of layout LVS Box cells to the extracted netlist. This is the default behavior if you do not include this statement in the rule file.

This keyword does not apply when LVS Box BLACK or LVS Box GRAY is specified. It applies when LVS Box GRAY DEVICES is specified, however.

NO

Keyword that instructs the SPICE netlister to output layout LVS Box cells only by their respective .SUBCKT and .ENDS statements and by comment lines.

Description

Specifies whether the SPICE netlister includes the internal contents of LVS Box cells in the output netlist. This statement can be specified once.

If you specify YES, or if this statement is omitted, then LVS Box cells appear in the netlist with full contents just like regular cells.

If you specify NO, then layout LVS Box cells are described in the output netlist only by their respective .SUBCKT and .ENDS statements, and by comment lines; SPICE elements or subcircuit calls within the LVS Box subcircuit are not written out. This is indicated with the words CONTENTS DISCARDED in a comment line following the .SUBCKT statement, as shown in this example:

.SUBCKT inv GROUND VCC OUT
** N=4 EP=3 IP=0 FDC=2 CONTENTS DISCARDED
.ENDS

This statement does not affect the subcircuit definitions of layout cells that are not LVS Box cells, even if those cells are placed exclusively within LVS Box cells. Such cells are fully described in the netlist, regardless of this statement.

The LVS Preserve Box Cells YES statement has the same effect as LVS Netlist Box Contents NO and supersedes LVS Netlist Box Contents YES. Additionally, LVS Preserve Box Cells YES disables seed promotion out of box cells, so the devices that originate from box cells are not netlisted. LVS Netlist Box Contents NO does not have this latter effect.

The Calibre Connectivity Interface LAYOUT NETLIST TRIVIAL PINS YES command has precedence over the LVS Netlist Box Contents NO rule file setting when generating CCI netlists. The LVS Netlist Box Contents YES rule file setting has precedence over LAYOUT NETLIST TRIVIAL PINS NO.

Examples

LVS BOX cella cellb

// Netlist the internal circuitry of LVS BOX cells.
LVS NETLIST BOX CONTENTS YES
