PEX Netlist 3DIC

Parasitic extraction

Specifies a replacement subcircuit model for a resistor on a specified layer or ubump LEF cell.

Usage

PEX NETLIST 3DIC {res_layer | CELL cell_name}
FILE file_name [SUBNODE subnode_name] [FLIP]

Arguments

res_layer

A required value that specifies the resistor layer name for a particular shape; normally a TSV, ubump, or pillar layer. For Calibre xACT, the res_layer can be an LVS layer name or a calibrated layer name. For Calibre xRC, the res_layer must be an LVS layer name.

CELL cell_name

A required keyword set that specifies the name of a ubump LEF cell.

FILE file_name

A required keyword set that specifies the file name containing the replacement subcircuit model for the resistor layer.

SUBNODE subnode_name

An optional keyword set that specifies to preserve the name of a node, subnode_name, in the subcircuit.

FLIP

An optional keyword that specifies to reverse the pin order when replacing the resistor pins with the subcircuit pins.

Description

Replaces a resistor on a specified layer or a ubump LEF cell with a subcircuit model in the output netlist. Use this statement for DSPF, SPEF, and HSPICE netlist formats. This statement is optional and may be specified more than once for different resistor layers or ubump LEF cells in your rule file. Use this statement together with the PEX 3DIC Coupling statement to perform TSV-to-TSV coupling extraction.

By default, the temperature coefficient values (TC1 and TC2) are taken from the rule files and applied to the extracted resistance values for the TSV layer resistors. However, if the PEX Netlist keyword TCOEFF is specified, then resistance values for the TSV layer resistors are not adjusted and are followed by TC1 and TC2 parameters and values in the generated netlist.

When generating the TSV coupling to ground in the netlist, “0” is the default ground node name used in the TSV subckt. Use the PEX Netlist statement with the GROUND ground_name keyword set to specify a ground node name other than 0. The ground_name replaces the ground node “0” in the TSV subckt.

This statement can be specified with any of the resistance reduction statements, however the reduction statements do not affect any resistors situated on the specified res_layer. Resistance reduction is done before subcircuit node substitution for these res_layer resistors.

Examples

Example 1

Given the following RCL subckt defined in the file tsv_rcl.subckt:

.subckt tsv top bottom
l1 n3 n2 1111
r1 top n2 2222
rs1 top n1 3333
rs1 n1 n2 4444
l2 n3 n4 1111
r2 n4 bottom 2222
rs2 n4 n5 3333
ls2 n5 bottom 4444

ctsv n3 n6 5555
rsub n6 0 6666
csub n6 0 7777
c12 top bottom 8888
.ends

To replace the resistance on layer tsv with an effective parasitic network supporting resistance, capacitance, and inductance components as defined in the given RCL subckt, include the following statement in your rule file:

PEX NETLIST 3DIC tsv FILE ./tsv_rcl.subckt

Example 2

This example shows how to use the SUBNODE keyword set to properly connect coupling parasitics to the TSV internal nodes.

The following is an RC subcircuit defined in the file tsv_rc.subckt, with a res_layer named tsv:

.subckt tsv top bottom
R1 top n1 0.009
R2 n1 bottom 0.009
C1 n1 ssub 112f
.ends

If the resistor is replaced with subckt, then ssub is treated as an internal node and output as an individual node for each instance in the netlist. This leaves all C1 capacitors unconnected creating dangling nodes.

To connect all the C1 capacitors to a global net called ssub in the output netlist, include the following statement in your rule file:

PEX NETLIST 3DIC tsv FILE ./tsv_rc.subckt SUBNODE ssub

Example 3

Given the connectivity of a tsv layer shown in the following sample from an extracted netlist:

rQ[21]/1 Q[21]:5 Q[21]:6 10 #layer=tsv

and given the following RC subckt definition in the file tsv_rc.subckt:

.subckt tsvsub top bottom
R1 top n2 7777
R2 n2 bottom 8888
C n2 sub 9999
.ends

Include the following statement in your rule file:

PEX NETLIST 3DIC tsv FILE ./tsv_rc.subckt

to replace the resistor with the subcircuit model information and produce an extracted netlist with results such as:

...
rQ[21]/3386 Q[21]:10525 Q[21]:6 8888
rQ[21]/3387 Q[21]:5 Q[21]:10525 7777
...

where node 5 (top) has a value of 7777 and node 6(bottom) has a value of 8888.

To reverse the connections to the tsvsub subcircuit pins top and bottom, include the following statement in your rule file:

PEX NETLIST 3DIC tsv FILE ./tsv_rc.subckt FLIP

Specification of the FLIP keyword produces the following netlist:

...
rQ[21]/3386 Q[21]:10525 Q[21]:5 8888
rQ[21]/3387 Q[21]:6 Q[21]:10525 7777
...

where node 5 (top) has a value of 8888 and node 6 (bottom) has a value of 7777.

Example 4

To replace the ubump cells PAD84_a and PAD34 with an effective parasitic network supporting resistance, capacitance, and inductance components as defined in the replacement subckt model file rc_tsv.subckt, include the following statements in your rule file:

PEX NETLIST 3DIC CELL PAD84_a FILE ./rc_tsv.subckt SUBNODE sub
PEX NETLIST 3DIC CELL PAD34 FILE ./rc_tsv.subckt SUBNODE sub

Example 5

Given the following RC subckt definition in the file tsv_rc.subckt:

.subckt tsvsub top bottom
R1 top d 8888
R2 d bottom 7777
C d 0 66f
.ends

When the following statements are specified in your rule file:

PEX NETLIST "netlist.dspf" DSPF SOURCENAMES
PEX NETLIST 3DIC tsvsub FILE ./tsv_rc.subckt

the temperature coefficient values (TC1 and TC2) are used to adjust the values of the resistors inside the subckt. This is a sample from a generated DSPF netlist: 

...
r21 35 11 9125.08
r22 8 35 11021.3
...

If the TCOEFF keyword is specified in the PEX Netlist statement, for example:

PEX NETLIST "netlist.dspf" DSPF SOURCENAMES TCOEFF
PEX NETLIST 3DIC tsvsub FILE ./tsv_rc.subckt

then the resistance values of the resistors inside the subckt are not affected by the temperature coefficient settings and temperature coefficient parameters are included in the netlist for the resistors. For example:

...
r21 35 11 7777 tc1=0.003437 tc2=-1.1e-06
r22 8 35 8888 tc1=0.003437 tc2=-1.1e-06
...

Example 6

This example shows how to include the couplings (Rsub and Csub) from the TSVs to the substrate ground node name myGND. The SUBNODE keyword set connects coupling parasitics to the TSV internal nodes.

The following is an RC subcircuit defined in the file tsv_rc.subckt, with a res_layer named tsv:

.subckt tsv top bottom
R1 top n1 0.009
R2 n1 bottom 0.009
C1 n1 ssub 112f
Rsub ssub 0 170
Csub ssub 0 3.6e-14
.ends

To use myGND node in the extracted netlist, include the following statements in your rule file:

PEX NETLIST netlist_xrc.dspf DSPF LAYOUTNAMES LOCATION GROUND myGND
PEX NETLIST 3DIC tsv FILE ./tsv_rc.subckt SUBNODE ssub

The resulting netlist contains Rsub and Csub connected to myGND ground node. For example:

...
c8 SO[5]:34 myGND 36.1f
r10 SO[5]:34 myGND 170
...
