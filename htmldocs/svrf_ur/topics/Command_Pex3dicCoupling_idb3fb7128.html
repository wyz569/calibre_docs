PEX 3DIC Coupling

Parasitic extraction

Specifies whether to extract the TSV-to-TSV coupling for a digital or analog circuit. This statement cannot be used without the PEX Netlist 3DIC statement.

Usage

PEX 3DIC COUPLING tsv_layer {ANALOG | DIGITAL  frequency} MAXDISTANCE distance

Arguments

tsv_layer

A required value that specifies the TSV layer name. The tsv_layer can be an LVS layer name or a calibrated layer name.

ANALOG

A required keyword that specifies to insert a parallel RC circuit between the TSVs and use the parasitic values from the TSV capacitance (Csub) and resistance (Rsub) tables in the foundry calibrated rule files. These parasitic values are a function of spacing between the TSVs.

DIGITAL  frequency

A required keyword set that specifies to insert an effective TSV coupling capacitance between the TSVs and use the parasitic values from the effective TSV capacitance (Ceff) tables in the foundry calibrated rule files. These parasitic values are a function of spacing between the TSVs and the operating frequency in hertz.

MAXDISTANCE distance

A required keyword set that specifies the maximum distance allowed between the TSV edges. Parasitics for spacing that is greater than the specified distance are not extracted. distance is a positive floating point value in microns.

Description

An optional statement used together with the PEX Netlist 3DIC statement to specify whether to perform TSV-to-TSV coupling extraction for a digital (LEF/DEF) or analog (GDS) circuit. For coupling parasitics to be netlisted, a subnode must be inserted using the PEX Netlist 3DIC statement with the SUBNODE keyword. If the PEX Netlist 3DIC statement is not specified with the PEX 3DIC Coupling statement, then extraction stops with an error. This statement can be used for DSPF, SPEF, and HSPICE netlist formats.

Analog Designs

Coupling parasitics between TSVs through the substrate in an analog design are calculated based on a resistance table (Rsub) and a capacitance table (Csub). The parasitic values are provided as a function of the spacing between the edges of TSVs from the foundry calibrated rule files.

For TSV extraction in analog designs, use the ANALOG keyword to insert a parallel RC (Rsub and Csub) circuit between the TSVs as shown in Figure 1. The parasitic values extracted for Rsub and Csub are obtained from the TSV_RESISTANCE table and the TSV_CAPACITANCE table for the TSV layer. See the xCalibrate Batch User’s Manual for more information about Table Syntax.

Figure 1. Parallel RC Circuit For ANALOG

Digital Designs

Coupling capacitance between TSVs in a digital design is calculated based on an effective capacitance table (Ceff). The parasitic values are provided as a function of spacing between the TSVs and the operating frequency in tables from the calibrated rule files.

For TSV extraction in digital designs, use the DIGITAL keyword to insert a single effective capacitor (Ceff) subcircuit between the TSVs as shown in Figure 2 and specify the operating frequency. The parasitic values extracted for Ceff are obtained from the TSV_CAPACITANCE table as a function of the spacing and frequency. See the xCalibrate Batch User’s Manual for more information about Table Syntax.

Figure 2. Ceff Circuit for DIGITAL

For more information on TSV extraction, see “TSV Extraction” in the Calibre xACT User’s Manual.

Examples

Example 1

Given the following RC subcircuit defined in the file tsv_rc_analog.subckt, with a res_layer named tsv1a:

.subckt tsv1a top bottom
R1 top n1 0.009
R2 n1 bottom 0.009
C1 n1 ssub 112f
.ends

In an analog design, to extract couplings for TSV layer tsv1a within a distance of 300 microns, include the following statements in your rule file:

PEX NETLIST 3DIC tsv1a FILE ./tsv_rc_analog.subckt SUBNODE ssub PEX 3DIC COUPLING tsv1a ANALOG MAXDISTANCE 300

Example 2

Given the following RC subcircuit defined in the file tsv_rc_dig.subckt, with a res_layer named tsv1d: 

.subckt tsv1d top bottom
R1 top n1 0.008
R2 n1 bottom 0.008
C1 n1 ssub 115f
.ends

In a digital design, to extract couplings for TSV layer tsv1d within a distance of 200 microns and at operating frequency of 3.5e09, include the following statements in your rule file:

PEX NETLIST 3DIC tsv1d FILE ./tsv_rc_dig.subckt SUBNODE ssub PEX 3DIC COUPLING tsv1d DIGITAL 3.5e09 MAXDISTANCE 200
