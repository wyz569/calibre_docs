HIDDEN PROPERTY

Device property computation language function

Defines device properties used in device property annotation programs but are not printed in output netlists or the PDSP report.

Usage

HIDDEN PROPERTY prop_name_1[, prop_name_2 …]

Arguments

prop_name_1

A required name of a property for which to calculate an annotated value. More than one property name is specified in a comma-delimited list.

Return Values

None.

Description

This function behaves essentially the same as the PROPERTY function, with the exception that properties defined by HIDDEN PROPERTY are not written to the extracted netlist, to a Calibre Connectivity Interface (CCI) netlist, or to device pushdown separated properties (PDSP) files.

The primary use for this function is situations when non-LVS properties are needed for downstream tools, such as in the PDSP flow. Computation of such properties is more suitable for device property annotation programs included with Device Layer ANNOTATE operations and with Device statements. Non-netlisted properties can be calculated in the device property computation program. Using HIDDEN PROPERTY, they are internally passed to property annotation programs, where the “hidden” property values are used to calculate annotation properties. Annotation properties are accessed later in CCI.

See “Device Annotation Built-In Language” for complete information about annotation programs. See “Writing PDSP Properties in CCI” in the Calibre Query Server Manual for information about extracting PDSP properties from annotated layers.

Examples

Consider this Device property computation program:

DEVICE MN(NM) ngate poly(G) ndiff(S) ndiff(D) well_vss(B) <nact> [

PROPERTY W, L, AS, AD, PS, PD
W = (perimeter_coincide(ngate, S) + perimeter_coincide(ngate, D)) / 2
L =  perimeter_outside (ngate, ndiff) / 2
PI_S_nact = perimeter_inside(S,nact)
PI_D_nact = perimeter_inside(D,nact)
AS = area(S) * (W / PI_S_nact)
AD = area(D) * (W / PI_D_nact)
PS = (perimeter(S) * (W / PI_S_nact) - W)
PD = (perimeter(D) * (W / PI_D_nact) - W)
]

If the W and L properties are the only ones of interest for LVS, and the rest of the properties are for downstream tools, then the S/D pin-specific properties are better handled by device annotation programs. The area and perimeter of pin shapes can be added as DFM properties to an annotation layer and then accessed in a Device annotation program block, like this:

// assign DFM properties to annotation layer
ndiff_p = DFM PROPERTY ndiff
          [ AREA = AREA(ndiff) ]
          [ PERIM = PERIMETER(ndiff) ]

DEVICE MN(NM) ngate poly(G) ndiff(S) ndiff(D) well_vss(B) <nact> [
 
PROPERTY W, L
HIDDEN PROPERTY P_S_nact, P_D_nact 
 
// LVS properties for netlisting
W = (perimeter_coincide(ngate, S) + perimeter_coincide(ngate, D)) / 2
L =  perimeter_outside (ngate, ndiff) / 2
 
// hidden properties calculated and passed to the annotation program.
// these are not netlisted.
PI_S_nact = perimeter_inside(S,nact)
PI_D_nact = perimeter_inside(D,nact)
 
] <ndiff_p> [
 
// annotation program block for properties accessed in CCI
PROPERTY  AS, AD, PS, PD
AS = DFM_NUM_VAL( ndiff_p, AREA, S ) * (W / PI_S_nact)
AD = DFM_NUM_VAL( ndiff_p, AREA, D ) * (W / PI_D_nact)
PS =(DFM_NUM_VAL( ndiff_p, PERIM, S) * (W / PI_S_nact) - W)
PD =(DFM_NUM_VAL( NDiff_p, PERIM, D) * (W / PI_D_nact) - W)
]

The annotation program block shown in the previous code could also be used in a Device Layer ANNOTATE program. The P_S_nact and P_D_nact properties would be accessible by the Device Layer annotation program.
