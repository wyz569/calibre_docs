PEX Netlist Corner All

Parasitic extraction

Specifies a subset of the corner names used to override the default corners netlisted with the “‑corner all” command line option.

Usage

PEX NETLIST CORNER ALL corner [corner…]

Arguments

corner

A required corner name identifying one or more corners for which the capacitance and resistance rules have been calibrated. Only corner names defined with the PEX Corner and PEX Corner Custom statements may be used.

Description

An optional statement that specifies a subset of the corner names used to override the default corners that are netlisted with the “-corner all” command line option. This statement may be specified more than once. Additional specifications will append to the list of all corners to be netlisted. All corners (custom and non-custom) are netlisted by default when this statement is not specified.

Examples

Given the following corner definitions:

PEX CORNER typical 
PEX CORNER CUSTOM CCbest CCworst REFERENCE typical typical

Include the following statement in your rule file to override the default behavior of the “‑corner all” command line option and only netlist a subset of the corners:

PEX NETLIST CORNER ALL typical CCworst
