Property Computation Language Characteristics

The device property computation language has syntactical and structural details that are important to understand when writing programs. These details are discussed in the following table.
Table 1. Property Computation Built-In Language Specifics

Element

Description

Structure

The built-in language has a required sequence of statements, shown here:

[
DEBUG     // optional, must be first if present
PROPERTY  // required property computations  
          // the properties in the PROPerty statement
          // must be defined in the program
]

There is no WARN statement in this language. Refer to “Structure” in “Common Features in the Built-In Languages” for general notes on the structure of the language.

Macros

Property computations can be written as macros that can be called multiple times in a rule file. See “Macros in User-Defined Device Property Calculations” for details.

Local variables

You can assign intermediate values to local variables within a property specification. Numeric, string, and complex type local variables are supported. Types for these variables are determined by the functions that are used. Once a type is associated with a variable, the type may not change.

Assignment statements

Assignment statements have the following forms:

 

local_variable_or_property_name = numeric_expression

local_variable_or_property_name = string_expression

local_variable_or_property_name = function

The last case is for functions that return complex-type variables.

You can specify only one equals sign (=) operator per assignment statement.

Note that variable types cannot be changed after the initial assignment.

Examples:

A = 5                                //numeric
B = AREA(seed) + A + 7               //numeric
C = "ABC"                            //string
D = TEXT_STRING ( txtlay1, "default1" ) //string
E = DFM_NUMERIC_ARRAY( sd_prop, "NC" )  //array

Units of measurement

For information about which units to use for representing physical quantities such as length and area, refer to “Units of Measurement” in the Calibre Verification User’s Manual.

TVF interface

The TVF Functional Interface for Device Property Calculation is governed by the rules of Tcl code, which are somewhat different from the Device property computation built-in language. When working with TVF Function blocks, ensure that you use valid Tcl code.

Pre-processor directives

Pre-processor directives, or conditionals, are permitted in Device property calculations. These can help you to control which properties get calculated for the type of run you are using. See “Pre-Processor Directives”.
