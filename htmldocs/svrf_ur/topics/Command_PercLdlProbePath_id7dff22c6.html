PERC LDL Probe Path

Specification statement

Specifies the pathname of a probe mapping file, which defines probe locations for an LDL CD or P2P run.

Usage

PERC LDL PROBE PATH filename

Arguments

filename

A required argument that specifies a pathname of a probe mapping file.

Description

Specifies to use a probe mapping file for LDL CD or P2P pins. The file defines source and sink probe locations instead of port or pin names. This statement is primarily useful for debugging LDL check code or testing a small set of pin pairs.

When this statement is specified, all PERC Load statements in the rules are ignored. Hence, this statement replaces perc::export_pin_pair commands for the LDL flows.

A probe mapping file has this basic format:

VERSION 1
PROBE_POINTS
<probe_name1> <x> <y> <layer_name> [<comment>]
<probe_name2> <x> <y> <layer_name> [<comment>]
...
END PROBE_POINTS
PIN_PAIRS
{<probe_name1> <probe_name2> {cd | p2p} <rule_name>
  [-annotations {{<name> <value>}...}]
  [-sink_voltage <voltage>]
  [-source_current <current>]
  [-comment ''<string>'']}
...
END PIN_PAIRS

The brackets ([ ]) indicate optional arguments and are not literal. The braces ({ }) are for argument grouping and are not literal. The ellipses (…) indicate repetition and are not literal. Whitespace in the file is ignored. The definitions of the file parameters are in the following table.

Table 1. Probe Mapping File Elements

Element

Description

VERSION 1

Required literal string that may appear only once. If used, it is either the first line of the file or it follows a <comment> line.

PROBE_POINTS

Required literal string that begins the section of the file where probe points are defined. This string may appear only once. Any number of probe point definitions may appear in the PROBE_POINTS section.

<probe_nameN>

Required user specified name of a probe point definition. At least two probe names are required to define a pin pair. Probe names must be unique and begin with a letter. After the initial letter, the name may consist of letters, numbers, bracket characters ([ ]), underscores (_), or periods (.).

<x> <y>

Required pair of top-level space X and Y coordinates in user units. The coordinates must intersect a resistance layer shape to be useful.

<layer_name>

Required name of a layer that the probe point coordinates intersect.

<comment>

An optional string specifying a comment. Comments begin with the hash symbol (#). Everything after the # symbol on the same line is ignored. Comments may start at the beginning of a line anywhere in the file. Comments may appear as the final argument of a probe point definition.

END PROBE_POINTS

Required literal string that terminates the probe point definitions.

PIN_PAIRS

Required literal string that begins the section of the file where pin pairs are specified. This string may appear only once. Any pin pair specifications may appear in the PIN_PAIRS section.

cd

Literal string that specifies the pin pair is for a current density check. Either cd or p2p must be specified.

p2p

Literal string that specifies the pin pair is for a point-to-point resistance check. Either cd or p2p must be specified.

<rule_name>

Required name of a rule check specified in either a perc_ldl::design_cd_experiment or perc_ldl::design_p2p_experiment command.

-annotations {{<name> <value>}…}

Optional keyword set that specifies a name-value pair to use as an annotation for the pin pair. The <name> is a string that must begin with a letter and may not be quoted. The <value> may be a string or a number. At least one name-value pair must be specified. Annotations appear as user annotations in Calibre RVE for PERC.

-sink_voltage <voltage>

Optional keyword set that specifies the drain voltage at the sink pin (corresponds to the second probe name in the pair). The <voltage> is a positive number in millivolts. If specified, this voltage takes precedence over any voltage specified in a current density rule check experiment.

-source_current <current>

Optional keyword set that specifies the current at the source pin (corresponds to the first probe name in the pair). The <current> is a non-negative number in milliamps. If specified, this current takes precedence over any current specified in a current density rule check experiment.

-comment ''<string>''

Optional keyword set specifying a pin pair comment. The <string> appears as a user annotation in Calibre RVE for PERC. The <string> should be quoted.

END PIN_PAIRS

Required literal string that terminates the probe point definitions.

Examples

Assume the following probe mapping file probe.defs:

PROBE_POINTS
P0 21.0 46.0 metal2 # top port
P1 35.0 62.0 poly   # pmos gate 
END PROBE_POINTS
PIN_PAIRS
P0 P1 p2p rule_1 -annotations probe IN1 -comment "signal port to MP gate"
END PIN_PAIRS

Here is a rule file that references the probe mapping file. Notice that no PERC Load statement is required to perform the pin-pair check.

// LVS circuit extraction and calibrated resistance rules
INCLUDE circuit_extraction.rules
INCLUDE resistance_extraction.rules

PERC NETLIST LAYOUT
PERC REPORT "perc.report"
PERC LDL PROBE PATH "./probe.defs"

TVF FUNCTION p2p_checks [/*
  proc exec_p2p {} {
    set exp1 [perc_ldl::design_p2p_experiment -rulecheck rule_1]
    set exp_list [list $exp1]
    perc_ldl::execute_p2p_checks -p2p_experiment_list [list $exp_list]
    set p2pItr [dfm::get_ldl_results -p2p -all -rulecheck rule_1]
    puts "NETS: [dfm::get_ldl_data $p2pItr -net_name]"
    puts "DATA: [dfm::get_ldl_data -p2p_experiment_data]"
    exit
  }
*/]

DFM YS AUTOSTART p2p_checks exec_p2p
