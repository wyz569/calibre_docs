LVS Netlist Internally Floating Pins

Specification statement

Specifies whether internally floating cell pins should appear in the extracted netlist. In general, the YES option should be avoided unless required by the flow.

Usage

LVS NETLIST INTERNALLY FLOATING PINS {NO | YES}

Arguments

NO

Keyword that instructs the SPICE netlister not to output internally floating cell pins. This is the default.

YES

Keyword that instructs the SPICE netlister to output internally floating cell pins.

Description

Specifies whether internally floating cell pins should appear in the extracted layout netlist (as in calibre -spice). The YES keyword is intended for specialized methodology checks and should generally be avoided as it can degrade performance and increase memory requirements and file sizes. This statement may be specified only once.

Internally floating pins are pins of a cell that are not connected to devices or LVS Box placements within the cell or within the sub-hierarchy of the cell. (Recall that LVS Box cells have their own set of rules governing netlisting of internally floating pins. Those rules take precedence over LVS Netlist Internally Floating Pins. Specifically, texted pins of LVS Box cells are netlisted regardless of LVS Netlist Internally Floating Pins setting.)

When NO is specified, internally floating pins are omitted from .SUBCKT definitions of cells and from subcircuit calls (X statements) referencing the cells.When YES is specified, internally floating pins appear in .SUBCKT definitions and in subcircuit calls, including Calibre Connectivity Interface (CCI) generated netlists.

The CCI LAYOUT NETLIST TRIVIAL PINS YES command has precedence over the LVS Netlist Internally Floating Pins NO rule file setting when generating CCI netlists. The LVS Netlist Internally Floating Pins YES rule file setting has precedence over LAYOUT NETLIST TRIVIAL PINS NO.

Note that, with respect to pins that are both internally-floating and texted, the LVS Netlist All Texted Pins specification statement takes precedence. Specifically, when LVS Netlist All Texted Pins YES is specified, texted, internally-floating pins appear in the netlist regardless of LVS Netlist Internally Floating Pins setting.

Examples

With this rule file setting:

LVS NETLIST INTERNALLY FLOATING PINS YES

Internally floating pins appear in the extracted netlist. The following Query Server command includes those pins in the CCI generated netlist, regardless of the LAYOUT NETLIST TRIVIAL PINS setting in CCI:

LAYOUT NETLIST WRITE cci.netlist
