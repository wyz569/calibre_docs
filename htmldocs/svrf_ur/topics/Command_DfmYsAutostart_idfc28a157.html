DFM YS Autostart

Specification statement

Specifies a Calibre YieldServer script to be executed immediately after the calibre -dfm module completes.

Usage

DFM YS AUTOSTART TVF_function tcl_proc [tcl_proc …]

Arguments

TVF_function

A required argument that specifies the name of a TVF Function defined in the rule file.

tcl_proc [tcl_proc …]

A required argument that specifies the name of one or more Tcl procedures defined inside the TVF_function. The Tcl procedures are executed in the order that they are specified.

Description

Specifies Tcl code referenced in a rule file to be executed automatically after the calibre -dfm module. The ‑ys -autostart options must be specified on the command line in order for the TVF_function block to be executed. A Calibre YieldServer license is required to execute the TVF_function.

It is a good practice to test the YieldServer script first in a calibre -ys run before calling the script from the rule file.

Examples

When included in a rule file, the following code lists the vertices for geometries on layer met1. It can be executed by running “calibre -dfm -hier rule_file -ys -autostart” at the command line. Alternatively, if the DFM database already exists and the code was present in the rules when the database was created, you can execute the code by running “calibre -ys -dfmdb dfmdb ‑autostart” at the command line.

DFM YS AUTOSTART sample_function
    list_vertices
    exit_ys
TVF FUNCTION sample_function [/*
    proc list_vertices {} {
        set layer met1
        set iter [dfm::get_geometries $layer]
        puts "Vertices for geometries on layer $layer:\n"
        while { $iter ne "" } {
            set vertices_list [dfm::get_data $iter -vertices]
            puts "$vertices_list\n"
            dfm::inc iter
        }
    proc exit_ys {} {
        puts "Exiting"
        exit -force
    }
*/]
