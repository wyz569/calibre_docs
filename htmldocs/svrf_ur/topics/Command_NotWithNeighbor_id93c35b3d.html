Not With Neighbor

Layer operation

Selects orthogonal rectangles that do not have the specified number of orthogonal rectangles within the specified distance.

Usage

NOT WITH NEIGHBOR layer1 [layer2] count_constraint SPACE constraint2
[SQUARE] [CENTERS [OCTAGONAL ONLY | ORTHOGONAL ONLY]]
{[INSIDE OF LAYER layer3] | [[NOT] CONNECTED]}

Arguments

layer1

A required original layer or layer set, or a derived polygon layer.

layer2

An optional original layer or layer set, or a derived polygon layer.

count_constraint

A required constraint that places a condition on the number of orthogonal rectangles in proximity to a given orthogonal rectangle. A layer1 rectangle is selected if the constraint is not met. Orthogonal is determined with respect to the database axes.

The constraint is one of those listed in the “Constraint Notation” column of Table 1 in the “Constraints” topic. The constraint must contain non-negative integers.

SPACE constraint2

A required keyword and constraint that specifies the dimensions of the measurement region around a layer1 polygon.

The constraint should contain non-negative, floating-point numbers in user units of length. The constraint may not be of the form “> a”, “>= a”, or “!= a”.

SQUARE

An optional keyword that specifies the measurement region formed by constraint2 uses the SQUARE metric rather than the Euclidean metric.

CENTERS [OCTAGONAL ONLY | ORTHOGONAL ONLY]

An optional keyword set that specifies to measure between the centers of the rectangles, with optional conditions on the alignment of the centers.

  • CENTERS — Specifies that the measurement region formed by constraint2 is measured from the center of a given rectangle rather than from its perimeter. Distances are measured to the centers of other rectangles rather than to their edges. See Figure 3.

These optional keywords specify alignment requirements:

  • OCTAGONAL ONLY — Specifies that the distance between the center points of orthogonal rectangles is defined to satisfy the SPACE constraint only if the center points are aligned at multiples of 45 degrees.

  • ORTHOGONAL ONLY — Specifies that the distance between the center points of orthogonal rectangles is defined to satisfy the SPACE constraint only if the center points are aligned in either the x- or y-direction.

INSIDE OF LAYER layer3

An optional argument that specifies to only consider polygons that are inside of layer3, where layer3 is a polygon layer. Refer to the algorithm description for more information. May not be specified with the CONNECTED keyword.

[NOT] CONNECTED

An optional keyword that modifies the operation to consider only rectangles from layer1 (and layer2, if specified) that are on the same electrical nets (or different electrical nets if NOT is used). If this keyword is specified, connectivity is required on layer1 and layer2, if specified. May not be specified with INSIDE OF LAYER.

Description

Selects orthogonal (with respect to the database axes) rectangles if they do not have the specified number of orthogonal rectangles within the specified distance.

Not With Neighbor is the complement of the With Neighbor operation; it selects polygons that do not satisfy a With Neighbor operation with the same arguments. The arguments specify conditions for the complementary With Neighbor operation. Polygons on layer1 that do not satisfy the conditions as applied to the complementary With Neighbor operation are selected.

With only layer1 specified, the operation works as follows:

  1. For each orthogonal rectangle R on layer1, let C be the number of other orthogonal (with respect to the database axes) rectangles having a distance to R that satisfies constraint2.

  2. If C does not satisfy count_constraint, then R is output.

The distance between orthogonal rectangles R and S is determined using each edge of R and S similar to the External operation. The Euclidean measurement metric is used by default. Measurement regions having a radius or radii determined by constraint2 are constructed around each edge of R, creating a “halo” around R. The distance between R and S is said to satisfy constraint2 if S intersects the halo, and a constraint of the form “< a” or “<= a” is used.

If an interval constraint is used, then the distance between R and S satisfies constraint2 if there is any point of S that satisfies the upper bound of the constraint interval relative to R, but no point of S is closer than the lower bound of the interval.

If R and S intersect at a singularity, then the distance is defined to be zero. (There are additional considerations when layer2 is used that are discussed later in this section.) See the following figure:

Figure 1. Halo Region Using Euclidean Metric

If SQUARE is specified, then the measurement region halo is constructed using the SQUARE metric rather than the Euclidean metric. See the following figure:

Figure 2. Halo Region Using SQUARE Metric

If CENTERS is specified, then the measurement region halo is created about the center point of R. The distance between R and S satisfies constraint2 if the center point of S intersects the halo. See the following figure:

Figure 3. Measurement Region Using CENTERS

The ORTHOGONAL ONLY and OCTAGONAL ONLY keywords are used primarily for advanced pitch analysis and via alignment checks. These options are used with the CENTERS keyword and control how the alignment of centers is checked. ORTHOGONAL ONLY specifies alignment checking in the x- and y- directions. OCTAGONAL ONLY also does this, but also checks alignment diagonally at multiples of 45 degrees.

All With Neighbor and Not With Neighbor operations with the same input layer, constraint2, and SQUARE and CENTERS keyword specifications are executed concurrently.

If layer2 is specified, then this algorithm is used:

  1. For each orthogonal rectangle R on layer1, let C be the number of orthogonal rectangles on layer2 whose distance to R satisfies constraint2.

  2. If C does not satisfy count_constraint, then R is output.

The distance between two rectangles from layer1 and layer2 is defined to be 0 if the rectangles touch at any point, except when CENTERS is specified. If CENTERS is specified, then the distance between rectangles on layer1 and layer2 is measured between their centers, regardless of any overlap.

If INSIDE OF LAYER layer3 is specified, then this algorithm is used:

For each polygon P on layer3,

  1. For each orthogonal rectangle R on layer1 that is inside of P, do one of the following:

    • If only layer1 is specified, let C be the number of other orthogonal rectangles on layer1, which are also inside of P and whose distance to R satisfies constraint2.

    • If layer2 is specified, let C be the number of orthogonal rectangles on layer2, which are also inside of P and whose distance to R satisfies constraint2.

  2. If C does not satisfy count_constraint, then R is output.

If [NOT] CONNECTED is specified, then this algorithm is used:

  1. For each orthogonal rectangle R on layer1, do one of the following:

    • If only layer1 is specified, let C be the number of other orthogonal rectangles on layer1 such that the rectangles are on the same (or different, if NOT is used) electrical net as R, and whose distance to R satisfies constraint2.

    • If layer2 is specified, let C be the number of orthogonal rectangles on layer2 such that the rectangles are on the same (or different, if NOT is used) electrical net as R, and whose distance to R satisfies constraint2.

  2. If C does not satisfy count_constraint, then R is output.
