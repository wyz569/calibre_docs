NIPROPERTY

Property access function for DFM expressions

Accesses net-indexed numeric property values from the input layer. Supported in DFM Property and DFM Property Select Secondary.

Usage

NIPROPERTY(ni_layer, property_name, netID)

Arguments

ni_layer

A required net-indexed layer produced by DFM Property Net.

ni_layer must be a secondary layer in the DFM Property or DFM Property Select Secondary operation.

property_name

A required argument that specifies the name of a numeric property to access.

netID

A required argument that specifies the netID for which to read the property value.

The netID can be determined with the NETID or NETPROPERTY function operating on another input layer in the DFM Property operation. Only NETID and NETPROPERTY are supported; other functions that return netIDs, such as NETPROPERTY_REF and VBYNETMIN, are not supported.

Return Values

Numeric value.

Description

NIPROPERTY accesses net-indexed numeric property values from the input layer.

The net-indexed input layer (ni_layer) must be used as a secondary layer in the DFM Property or DFM Property Select Secondary operation, but because it has no physical geometries, it does not cluster directly with the primary layer. The property values on ni_layer are accessible only through the NIPROPERTY function. When used with DFM Property, OVERLAP or INTERSECTING clustering must be used; NODAL clustering is not allowed.

NIPROPERTY is a more efficient way to access net-indexed properties than the method required when not using NIPROPERTY. See the alternate method in the “Examples” section.

Examples

Method with NIPROPERTY

This example creates the net-indexed layer ni_layer with the attached AREA property, ni_area. The rule check bad_area checks for shape pairs on the error layer pairs with areas that differ by more than a given amount.

LAYER layer1 1
Connect layer1 

match_area{
// attach area to shapes, indexed by net
ni_layer = DFM PROPERTY NET layer1 [ni_area = AREA(layer1)]
pairs = EXTERNAL layer1 < 1.2
edges = EXTERNAL [layer1] < 1.2

bad_area = DFM PROPERTY pairs edges ni_layer OVERLAP MULTI
	[ area_diff = ABS(( NIPROPERTY(ni_layer, ni_area, NETID(edges, 1)) - 
			NIPROPERTY(ni_layer, ni_area, NETID(edges, 2)) )) ] > 0.5

DFM RDB bad_area area.rdb CELL SPACE CHECKNAME "%_l_" 
}

Alternate Method Without NIPROPERTY

The preceding example can be done as follows without the NIPROPERTY function. It requires writing the net-indexed property to an intermediate layer with a DFM Property NODAL operation (highlighted in orange), and is therefore less efficient. 

LAYER layer1 1
Connect layer1 

match_area{
// attach area to shapes, indexed by net
ni_layer = DFM PROPERTY NET layer1 [ni_area = AREA(layer1)]
pairs = EXTERNAL layer1 < 1.2
edges = EXTERNAL [layer1] < 1.2
edges_prop = DFM PROPERTY edges ni_layer NODAL MULTI 
			[ni_area = PROPERTY(ni_layer, ni_area)]

bad_area = DFM PROPERTY pairs edges edges_prop OVERLAP MULTI
	[ area_diff = ABS(( PROPERTY(edges_prop, ni_area, 1) - 
			PROPERTY(edges_prop, ni_area, 2) )) ] > 0.5

DFM RDB bad_area area.rdb CELL SPACE CHECKNAME "%_l_" 
}
