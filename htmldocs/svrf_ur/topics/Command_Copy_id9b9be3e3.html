Copy

Layer operation

Selects a layer and generates a copy of it.

Usage

COPY layer

Arguments

layer

A required original layer or layer set, or a derived polygon or edge layer.

Description

Copies the layer directly to a derived layer. The derived layer is the same type as layer unless layer is an original layer; then, the layer is a derived polygon layer. If the input layer has DFM properties they are copied to the output layer.

As a layer selector, Copy is a node-preserving operation frequently used to create connectivity sets. When building connectivity sets from the same layer, it is better to do this:

copyA1 = COPY layerA
copyA2 = COPY copyA1

CONNECT copyA1 layerB

than this:

copyA1 = COPY layerA
copyA2 = COPY layerA   // copyA1 and copyA2 are the same layer in the
                       // Calibre database.

CONNECT copyA1 layerB  // copyA2 and layerB are now shorted!

The “Node-Preserving Layer Operations” section in the Calibre Verification User’s Manual has important details related to connectivity that pertain to Copy and to layer selector operations generally.

See also Flatten, Merge, and DFM Copy.

Examples

Example 1

The Copy operation is useful in debugging rule check statements and layer derivations. For example, consider this rule check statement:

metal_long_space {
@ Spacing between metal edges longer than 100um must be at 
@ least 4um.
long_metal = LENGTH metal > 100
EXTERNAL long_metal < 4 }

During debugging of this rule check statement, you might desire to view the long_metal layer. One way you can do this is to comment out the External operation and duplicate the Length operation as follows:

metal_long_space {
@ Spacing between metal edges longer than 100um must be at 
@ least 4um.
long_metal = LENGTH metal > 100
LENGTH metal > 100
// EXTERNAL long_metal < 4 }

However, the Copy operation makes this method more straightforward, as follows:

metal_long_space {
@ Spacing between metal edges longer than 100um must be at 
@ least 4um.
long_metal = LENGTH metal > 100
COPY long_metal  	// For debugging.
EXTERNAL long_metal < 4 }

Example 2

You can use the Copy operation to debug global layer derivations. For example:

layer oxide 1
layer poly 2
inter_poly = poly NOT oxide

If you want to check what inter_poly is visually, you can write a rule check:

debug {copy inter_poly}

Example 3

The Copy operation enables you to easily attach multiple names to the same derived layer; for example:

X = INSIDE EDGE poly diff
Y = COPY X

The following works just as well, however:

X = INSIDE EDGE poly diff
Y = INSIDE EDGE poly diff

Example 4

The Copy operation is often used to derive layer copies in order to build up a new connectivity set from existing layers. For example:

CONNECT metal1 poly BY contact //original connectivity set

// derive layer copies for new connectivity set
c_metal1 = COPY metal1
c_contact = COPY contact
c_poly = COPY poly

CONNECT c_metal1 c_poly BY c_contact //special purpose connectivity set

The new connectivity set has different node IDs from the original one.

Also see “Example 2” and “Example 3” in the Connect section for some situations to avoid.
