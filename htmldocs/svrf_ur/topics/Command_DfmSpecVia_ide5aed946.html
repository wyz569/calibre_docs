DFM Spec Via

Specification statement

License: Calibre YieldEnhancer

Defines a via specification that can be used by DFM Transition USEVIAS.

Usage

DFM SPEC VIA via_name VIASHAPE x [y] ENC1 x [y] ENC2 x [y]

Arguments

via_name

A required argument that specifies the name of the via.

This argument is used as the USEVIAS argument to the DFM Transition statement and indicates that this type of via can be used in that DFM Transition step.

VIASHAPE x [y]

A required keyword and argument set that defines the size of the via in the default orientation. Dimensions are in user units.

  • x — A required argument that specifies horizontal size of the via. If x is larger than y, then the default orientation is horizontal. If x is less than y, then the default orientation is vertical.

  • y — An optional argument that specifies the vertical size of the via. The y value can be omitted if it is equal to x.

ENC1 x [y] ENC2 x [y]

A required keyword and argument set that defines the minimum enclosure of the via by below and upper metal layer polygon in the default orientation.

  • x — A required non-negative value that defines horizontal enclosure.

  • y — An optional non-negative value that defines vertical enclosure value in user units. Omit if equal to x.

Description

Defines a via specification that can be used by DFM Transition USEVIAS.

Examples

See “DFM Transition” for an example using the DFM Spec Via specification.
