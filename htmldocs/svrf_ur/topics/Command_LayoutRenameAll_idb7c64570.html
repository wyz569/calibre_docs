Layout Rename All

Specification statement

Specifies to use unique identification in all cells across multiple input files. Used only in Calibre.

Usage

LAYOUT RENAME ALL {NO | YES}

Arguments

NO

Keyword that instructs Calibre not to use unique identification of all cells across multiple input files. This is the default behavior when you do not include this statement in the rule file.

YES

Keyword that instructs Calibre to use unique identification of all cells across multiple input files.

Description

This statement is used to enable unique identification of all cells across multiple files in a Layout Path or Layout Path2 specification statement (but not across the two). The Layout Primary or Layout Primary2 value is not renamed (except if Layout Primary “*” is used).

Note:

Calibre issues an error when Layout Rename All YES is specified and the name argument for the Layout Primary or Layout Primary2 statement contains one or more wildcard characters.

Layout Rename All YES is normally used in conjunction with Layout Allow Duplicate Cell YES (or Layout Input Exception Severity DUPLICATE_CELL). This combination allows an input layout database to span multiple files but concatenates only the top-cell contents. It avoids any risk of the erroneous combination of cell A in one file and cell A in another file, in cases where the cells may actually be different, perhaps due to previous processing by some Calibre run.

In the following discussion “previous file” means a file (in a series of Layout Path or Layout Path2 files) that precedes the current file being processed for cell renaming. If Layout Rename Cell is specified with a filename parameter and there is more than one file specified with Layout Path or Layout Path2, then Layout Rename All YES will not rename cells if they match a name in a previous file and the following conditions are met:

  • The previous file is the first one in a Layout Path or Layout Path2 sequence.

  • The previous file is the product of a Layout Place Cell statement.

  • The name in the previous file is that of a placement, not a cell.

    Note:

    This statement has a very limited application and can result in severe performance degradation if not used carefully.

Assume that Layout Path has multiple file parameters, as shown here:

LAYOUT PATH file1 file2 ... fileN

Also assume that filei (i > 1) contains a cell whose name matches a cell name in filej, where j < i (after any renaming has been applied to filej). When YES is specified, this cell is renamed by appending $CALnumberN number$, where each <number> is an internally-generated positive-length sequence of decimal digits. This insures the new name is unique and does not match a cell name in any filej with j < i.

See also Layout Rename Cell, Layout Rename Clone, and Layout Rename ICV.

Examples

In the following example, assume cellA exists in both layout files, and the cells are not identical.

LAYOUT PATH layout1.gds layout2.gds
LAYOUT ALLOW DUPLICATE CELL YES
LAYOUT RENAME ALL YES

The duplicate cells are allowed and are renamed (internally) so they are handled as unique.
