LVS Spice Port Annotations

Specification statement

Provides access to comment-coded port annotations in a SPICE netlist. Used only in Calibre PERC.

Usage

LVS SPICE PORT ANNOTATIONS {NO | YES}

Arguments

NO

Specifies that comment-coded annotations are not used. This is the default.

YES

Specifies that comment-coded annotations are used.

Description

This statement controls access to comment-coded pin and port annotations in Calibre PERC rule checks.

A comment-coded port annotation in a SPICE netlist has this form:

     *keyword [port_name …]

The asterisk (*) indicates a user comment. The keyword can be one of these.

Table 1. Comment Annotation Keywords

Keyword

Description

INPUT

Input signal

OUTPUT

Output signal

INOUT

Bi-directional signal

GROUND

Ground supply

POWER

Power supply

MUXIN

MUX input signal

MUXOUT

MUX output signal

.PININFO

*.PININFO signal

These keywords are case-insensitive. More than one annotation of a given keyword can be specified in the netlist. If this is done, the port names across all annotations having the same keyword are internally concatenated into a single list.

There can be zero or more port_name parameters, where a non-empty parameter is a port name. Multiple port_name parameters are delimited by whitespace characters as used in SPICE (this includes commas). Port names do not need to appear on the .SUBCIRCUIT line.

Access to these annotations is through the perc::get_comments command. The command returns a list of names. The tool assigns no meaning to the names and does not check the design based upon the names. It is up to the user to process the returned list in a way that is meaningful for the rules. Typically the output of perc::get_comments is used to set up values for the initialization procedure or to produce some form of report.

Examples

Assume the following netlist:

.subckt subckt_1 a b c d e
*input b c
*output a
*inout e
...
.ends

This procedure in the rules accesses the annotation comments when LVS Spice Port Annotations YES is specified:

proc demo {} {
  set result [perc::get_comments -cellName subckt_1 -annotationName input]
  foreach p $result { puts "cell=subckt_1 annotation=input port=$p" }
}

In the run transcript, this would appear:

cell=subckt_1 annotation=input port=b
cell=subckt_1 annotation=input port=c
