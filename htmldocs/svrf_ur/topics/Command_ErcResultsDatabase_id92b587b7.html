ERC Results Database

Specification statement

Specifies the pathname of the ERC results database.

Usage

ERC RESULTS DATABASE filename [ASCII] [PSEUDO | TOP]

Arguments

filename

A required pathname where the tool writes the ERC results database.

The filename parameter can contain environment variables. See “Environment Variables in Pathname Parameters”.

ASCII

An optional keyword that indicates the ERC results database is in ASCII format. This is the default behavior if you do not include ASCII in this rule file statement.

PSEUDO

An optional keyword that specifies to generate a results database that includes pseudocells generated during hierarchical injection. This only applies to hierarchical circuit extraction.

TOP

An optional keyword that instructs the hierarchical database constructor to transform geometry to the top-level cell (without merging) prior to results database output. Use of this keyword is limited, and it should be used with caution. This keyword applies only to hierarchical circuit extraction and may not be specified with PSEUDO.

Description

Specifies the pathname and type of ERC results database generated during circuit extraction. You can specify this statement only once in your rule file and is required for ERC execution. See the “Electrical Rule Checks” chapter in the Calibre Verification User’s Manual for more information on ERC. See the “ERC Results Database” section in the same chapter for related information.

When you specify PSEUDO in this rule file statement, Calibre includes the additional cells that Calibre creates to support its hierarchical algorithms. The cells are named ICV_n, where n is a unique integer.

If you do not specify PSEUDO, Calibre does not include the pseudocells in the ERC results database and flattens the data residing within the pseudocells up to the next level of user hierarchy. For ASCII databases, transformation to the top-level proceeds normally.

DFM RDB may be used in an ERC rule check to perform a similar function as DRC Check Map. This can be especially useful if you want mask results output instead of an ASCII results database.

You can specify ERC options from the LVS Options pane in Calibre Interactive—nmLVS. See “Executing Electrical Rule Check (ERC) Operations in Calibre Interactive nmLVS” in the Calibre Interactive (New GUI) User’s Manual.

See also LVS Execute ERC, ERC Maximum Results, ERC Maximum Vertex, ERC Select Check, and ERC Unselect Check.

Examples

The following statement outputs an ASCII format of the results database to the specified location. You can use Calibre RVE to analyze the results or bring the results into Pyxis Layout for viewing.

ERC RESULTS DATABASE "~/design1/drc_database" ASCII
