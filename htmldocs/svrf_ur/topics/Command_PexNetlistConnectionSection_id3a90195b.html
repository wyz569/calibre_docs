PEX Netlist Connection Section

Parasitic extraction

Specifies to include or omit *|I and *|P from DSPF and SPEF netlists.

Usage

PEX NETLIST CONNECTION SECTION {YES [INST_LOC] | NO}

Arguments

YES

Required keyword that specifies to include nonstandard SPICE statements in the connection section of the netlist. This is the default.

NO

Required keyword that specifies to not include nonstandard SPICE statements in the connection section of the netlist.

INST_LOC

Optional keyword that specifies to include instance location and layer information as comments in the *|I section of the netlist produced for the EM/IR analysis flow. Use only with the DSPF netlist format. Use only with Calibre xACT direct netlisting flow. Do not use with the NO keyword.

Description

This statement specifies whether or not to include the statements beginning with ‘*|I’ or ‘*|P’ in a DSPF or SPEF netlist. If PEX Netlist Connection Section is not included in a rule file, then statements beginning with ‘*|I’ or ‘*|P’ occur in the connection section.

Examples

Example 1

This statement includes netlist code with ‘*|I’ or ‘*|P’ at the beginning of a line:

PEX NETLIST CONNECTION SECTION YES

Example 2

If you are using the Calibre xACT direct netlisting flow, specify this statement in your rule file to include the instance location and layer information as comments in *|I section in the extracted DSPF netlist: 

PEX NETLIST CONNECTION SECTION YES INST_LOC

The following is sample output from a resulting DSPF netlist:

*|NET X1/X2 0.954P
*|I (X1/X1:D X2/X3 D B 0 2.04 5.78 //$x=1.9954 $y=5.788 $x1=2.1982 $y1=5.788 $layer=96
