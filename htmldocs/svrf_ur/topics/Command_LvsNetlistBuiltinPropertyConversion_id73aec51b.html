LVS Netlist Builtin Property Conversion

Specification statement

Specifies how properties for built-in devices are written by the Calibre SPICE and Calibre Connectivity Interface (CCI) netlisters.

Usage

LVS NETLIST BUILTIN PROPERTY CONVERSION {SPICE | QD}

Arguments

SPICE

Keyword that specifies property netlisting occurs according to the tool default behavior. Built-in properties are uppercased, and others are lowercased. Rule file order is not followed.

QD

Keyword that specifies property netlisting for bipolar junction transistors (type Q) and diodes (type D) occurs according to the tool default behavior, but properties for capacitors (type C), resistors (type R), and MOS devices (type MD, ME, MN, MP) are netlisted using alternate criteria suitable for certain downstream tools.

Description

Specifies the criteria for netlisting of built-in properties for devices that are not of type Q or D. The following table shows how properties are netlisted based upon the statement keywords.

Examples

Consider the following rule file excerpt:

DEVICE MP PGATE PGATE PAA PAA NTB_S [
  property F, w, l
  F = 3
  w = perimeter_co(PGATE,PAA) * 0.5
  l = perimeter_outside(PGATE,PAA) * 0.5
]

DEVICE D diode_layer anode(POS) cathode(NEG)[
  property F, a, p
       F = 3
       a = area(diode_layer)
       p = perim(diode_layer)
]

LAYOUT PRESERVE PROPERTY CASE YES
LVS NETLIST BUILTIN PROPERTY CONVERSION SPICE

Devices from these statements netlist as follows:

M0 1 2 3 4 P L=6e-09 W=6e-09 F=3 $X=2030 $Y=9400 $D=0
D0 1 2 a AREA=1e-18 PJ=4e-09 F=3 $X=2030 $Y=9020 $D=1

Notice that the order of the properties and the text case for the devices do not follow the Device statements.

When LVS Netlist Builtin Property Conversion QD is set, the same devices netlist as follows:

M0 1 2 3 4 P F=3 w=6e-09 l=6e-09 $X=2030 $Y=9400 $D=0
D0 1 2 a AREA=1e-18 PJ=4e-09 F=3 $X=2030 $Y=9020 $D=1

The MP device properties now follow the rule file. The D device properties are unaffected (Q devices would behave similarly).
