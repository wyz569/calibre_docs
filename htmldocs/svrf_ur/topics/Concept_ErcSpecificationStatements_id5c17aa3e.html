ERC Specification Statements

This section lists the Electrical Rule Check (ERC) specification statements. These specification statements set the operating parameters for the ERC system. ERC is performed as a part of LVS.
Table 1. Specification Statements: ERC

Statement

Description

ERC Cell Name

Specifies whether to append the cell name associated with each ERC result to its signature line in an ASCII ERC results database. Used only for Calibre nmLVS-H.

ERC Check Text

Specifies the quantity and type of comments to appear in ERC results databases.

ERC Keep Empty

Specifies whether empty rule checks (rule checks with zero results) are retained in ERC results.

ERC Maximum Results

Specifies the maximum number of results for an individual rule check in ERC.

ERC Maximum Vertex

Specifies the maximum vertex count of any ERC result polygon written to the ERC results database.

ERC Path Also

Allows capacitors, diodes, and bipolar transistors to be considered part of a “path” in ERC path checks.

ERC Pathchk

Reports nets in the layout that do or do not have a path to power, ground or labeled nets, or that satisfy a combination of these conditions.

Pathchk is preferred over the ERC Pathchk statement.

ERC Results Database

Specifies the pathname of the ERC results database.

ERC Select Check

Selects rule check statements and rule check groups to be executed by ERC.

ERC Summary Report

Specifies the report filename and writing method for ERC.

ERC TVF

Generates device properties for downstream simulation tools.

ERC Unselect Check

Specifies rule check statements and rule check groups not to be executed by the ERC process.

Group

Names a collection of rule check statements.

LVS Execute ERC

Specifies whether or not ERC is performed by Calibre nmLVS.
