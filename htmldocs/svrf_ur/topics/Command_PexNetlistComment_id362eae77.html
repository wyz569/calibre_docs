PEX Netlist Comment

Parasitic extraction

Appends a user-specified comment string to the end of the netlist header information.

Usage

PEX NETLIST COMMENT { "string"
| { FILE "file1" ... "filen" }
| { "string" FILE "file1" ... "filen"}}

Arguments

string

A required user-specified string. The string must be enclosed in quotation marks (“ ”).

FILE "file1" ... "filen"

A required keyword set that specifies one or more filenames whose contents are to be added as comments to the netlist in the order they are specified. The filenames must be enclosed in quotation marks (“ ”).

Description

An optional statement used to add user-specified comments after the header comments in the netlist. May be used with all netlist formats. This statement may only be specified once.

Examples

Example 1

If you are extracting a DSPF netlist, including the following statement in your rule file:

PEX NETLIST COMMENT “Process 25 - Corner 1”

inserts the following comment lines just after the DSPF netlist header information:

* USER DEFINED COMMENTS
* Process 25 - Corner 1
*

Example 2

Consider the following files, project.txt and test.txt.

The file project.txt contains:

Project: Tango

The file test.txt contains:

Test Netlist: 100

To insert the contents of these files in the DSPF netlist just after the header information include the following statement in your rule file:

PEX NETLIST COMMENT FILE “project.txt” “test.txt”

For example:


...
* USER DEFINED COMMENTS
* Project: Tango
*
* Test Netlist: 100
...

Example 3

Specifying both a string and the FILE keyword set in the same statement is also allowed. For example the following statement combines examples 1 and 2:

PEX NETLIST COMMENT “Process 25 - Corner 1” FILE “project.txt” “test.txt”

and inserts the following user-defined comments in the DSPF netlist:


...
* USER DEFINED COMMENTS
* Process 25 - Corner 1
*
* Project: Tango
*
* Test Netlist: 100
...
