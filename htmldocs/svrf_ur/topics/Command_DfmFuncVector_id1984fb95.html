VECTOR

Vector and per-shape function for DFM expressions

Creates a vector of numeric type properties. Supported only in DFM Property and DFM Property Merge operations.

Usage

VECTOR(arg_expr1[, arg_expr2] …)

VECTOR::NOFAILARG(arg_expr1[, arg_expr2] …)

Arguments

arg_expr1[, arg_expr2]

A required numeric value or an expression that evaluates to a numeric value. Multiple arguments are separated by commas. The number of arguments corresponds to the tuple size; each argument is evaluated to form a tuple element.

Arguments can be an expression involving DFM measurement functions and supported math functions. Measurement functions can reference either the primary or secondary layer. If more than one argument references a layer, the same layer must be referenced. See the Description section for the behavior with primary or secondary layers.

The property reference function PROPERTY_REF cannot be used within VECTOR, however, it can be used in VTUPLE.

The arguments may be omitted in order to create an empty vector; see “Empty Vectors” in the Description section.

VECTOR::NOFAILARG

An alternate syntax form that specifies the VECTOR evaluation does not fail if evaluation fails for any argument expression. By default, VECTOR fails to evaluate if arg_expr fails to evaluate. See “Vector Evaluation Failure and Use of ::NOFAILARG” in the Description section.

Return Values

Vector of numeric values.

Description

VECTOR creates a vector with tuple elements consisting of the arguments to the function. VECTOR can only be used in DFM Property and DFM Property Merge operations. The property definition cannot use a constraint if the VECTOR function is used.

The terminology used for vector notation is described in “Vector DFM Properties”. In brief, {} indicates a vector and <> indicates a tuple within the vector. For the vector {<1 2 3> <4 5 6>}, each tuple has 3 elements (tuple size 3) and there are 2 tuples (tuple count 2). The tuple count is also called the vector length.

The argument expressions (arg_expr) are evaluated to form the tuple elements; the number of arguments determines the tuple size. If the primary layer or no layer is referenced, the vector consists of one tuple (tuple count one). If a secondary layer is referenced, the vector consists of one tuple for each secondary layer object clustered with the primary layer object; in other words, the tuple count corresponds to the number of secondary layer objects in the cluster.

The behavior is demonstrated in the following example, where A and B are polygon layers. 

X = DFM Property A B [ v_123 = VECTOR(1, 2, 3)]
                     [ v_area_perim_A = VECTOR(AREA(A), PERIM(A)) ]
                     [ v_area_perim_B = VECTOR(AREA(B), PERIM(B)) ]

The vectors are constructed as follows for each primary layer object: 

v_123 = {<1 2 3>}
v_area_perim_A = {<AREA(A) PERIM(A)>}
v_area_perim_B = {<AREA(B1) PERIM(B1)> <AREA(B2) PERIM(B2)> ... }

where v_area_perim_B consists of a tuple for each secondary layer B object clustered with the primary layer object.

Accessing Individual Vector Elements

You can access individual vector elements with the function VELEMENT. You can also access vector elements with a user-defined function created by a DFM Function statement; see “Accessing Vector Elements”.

Empty Vectors

An empty vector (tuple count zero) of a specified tuple size is specified by omitting the function arguments and specifying only the appropriate number of commas. For example, VECTOR() specifies a zero-length (empty) vector with tuple size 1; VECTOR(,,) generates an empty vector with tuple size 3.

Examples

See “Example 2” in DFM Property Merge and “Example Device Annotation Programs Using DFM Property”.

Example 1

This example demonstrates how vectors are created. Assume the following geometry:

  • Primary layer A with one geometry A1, where AREA(A1) = 100

  • Secondary layer B with 3 geometries clustered with A1: B1, B2, B3, where AREA(B1)=2, AREA(B2)=3, AREA(B3)=4.

  • Secondary layer C with 2 geometries clustered with A1: C1, C2, where AREA(C1)=20, AREA(C2)=30.

Create properties with the following operation:

X = DFM PROPERTY A B C
     [ V_AREA_B = VECTOR(AREA(B)) ]
     [ V_AREA_C = VECTOR(AREA(C)) ]
     [ SUM_AREA_B = AREA(B) ]
     [ SUM_AREA_A_B = AREA(B)+AREA(C)]
     [ AREA_A = AREA(A)]
     [ V_AREA_A = VECTOR(AREA(A))]

The object A1 is written to the output layer X with the following properties:

Example 2

This example uses an expression chain to provide an alternate vector property if evaluation fails for an expression.

X = DFM PROPERTY A 
  [ V_A = VECTOR(1/(AREA(A)-1), PERIM(A)) ] 
  [ V_A = VECTOR(1/(AREA(A), PERIM(A)) ]

If any object on layer A has an area of 1, evaluation fails due to division by zero in the first property definition, causing the second property definition to be used.
