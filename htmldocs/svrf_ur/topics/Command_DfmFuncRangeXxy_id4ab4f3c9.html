RANGE_XXY

Vector property function for DFM expressions

Computes a vector of tuples, with each tuple containing a coordinate point value and associated measurement values from the input vector. The coordinate values of the output tuples must be within a specified interval.

Usage

RANGE_XXY(vector, interval_vector)

Arguments

vector

A required vector produced by SELECT_MERGE_XXY or SORT_MERGE_XXY.

interval_vector

A required vector argument containing one tuple of size 2, where the first value is the minimum coordinate of an interval, and the second value is the maximum coordinate of the interval.

Return Values

Vector string containing tuples of the form <c m1 m2 … mn>, where c is an interval endpoint coordinate in user units, and each mi is the measurement value in user units selected from the input vector.

Description

This function is designed for use when there are multiple edge spacings to be analyzed for a given edge. It may only be used in a DFM Property operation expression. It is used to filter the output of a SELECT_MERGE_XXY or SORT_MERGE_XXY command. The values RANGE_XXY returns are from the interval_vector range, which is a subset of the entire range spanned by the vector. It is possible to specify a range that is not a subset of the entire vector range, but this is not generally useful.

Assume the following input vector from a SELECT_MERGE_XXY function:

{<0 0.5 0> <0.5 0.5 1.5> <1 0 1.5> <1.5 0 0>}

Each tuple is of the form <c m1 m2>, where c is an interval endpoint coordinate (x or y), and each mi is a measurement value associated with the coordinate. If this interval_vector is used:

{<0.6 1.4>}

then the following vector is returned:

{<0.6 0.5 1.5> <1 0 1.5> <1.4 0 0>}

This vector contains the values of the input vector that lie on the interval [0.6, 1.4). The explanations of the tuples are as follows:

<0.6 0.5 1.5>

Coordinate 0.6 is the start of the interval range. Because this value lies between the coordinates 0.5 and 1 of the input vector, the measurement values 0.5 and 1.5 from the second tuple of the input vector are associated with it.

 
<1 0 1.5>

Coordinate 1 lies in the specified interval [0.6, 1.4), so the third tuple of the input vector lies within the interval range and is output.

 
<1.4 0 0>

Coordinate 1.4 is the end of the interval and values of 0 are associated with it since the interval is half open.

The general algorithm for the command is as follows. Let c be the first element of each tuple of the input vector. Let the interval_vector be represented by [min, max), where min and max are numeric values representing the range of the interval. These rules govern the output of the vector tuples:

  • If all c lie outside the range interval, then output <min 0 …>.

  • If min <= c < max, then output the tuple containing c from the input vector.

  • If ci < min < cj, then output <min m1 …mn>, where the mn values come from the input vector tuple corresponding to ci.

  • Output <max 0 …> as the final tuple.
