LVS Report Pinswapped

Specification statement

Specifies whether swapped pins for device instances generate errors in the LVS report.

Usage

Syntax 1:

LVS REPORT PINSWAPPED component_type[‘(’component_subtype‘)’] {[component_type[‘(’component_subtype‘)’]]…}

Syntax 2:

LVS REPORT PINSWAPPED ORIGINAL SUBTYPE component_type[‘(’component_subtype‘)’] {[component_type[‘(’component_subtype‘)’]]…}

Arguments

component_type

A required device element name. More than one component_type may be specified.

‘(’component_subtype‘)’

An optional device model name in parentheses. Must be specified with a component_type.

ORIGINAL SUBTYPE

A required keyword in Syntax 2 specifying that an original model name, before any LVS Map Device model name mapping is applied, is compared to the component_subtype arguments. If the original model name appears in the list, then the instances with swapped pins are reported.

Description

Specifies that an error is given for any device instance matching the component_type and optional component_subtype when pin swapping occurs in LVS comparison.

Assume that a MOSFET device is specified. If the source pin of the MOSFET in the layout is matched to the drain pin of the corresponding MOSFET in the source, that MOSFET is reported in the information and warnings section of the relevant cell report, with an error in the summary section.

When the ORIGINAL SUBTYPE keywords are used, the original subtype of each instance before any LVS Map Device statement is applied is compared to the component_subtype arguments. Instances matching the statement’s conditions are then reported.

Specifying component_type alone with ORIGINAL SUBTYPE provides no additional functionality over not specifying that keyword.

Consider the following rule file code:

LVS REPORT PINSWAPPED yyy(AAA)
LVS MAP DEVICE  xxx yyy(AAA)
LVS DEVICE TYPE MOS yyy

and a layout SPICE netlist that contains this:

.subckt xxx d g s b
.ends

.subckt top vcc vss
x1 src gate drn blk xxx
.ends

Assume the source has the src and drn pins swapped. This results in the following message in the top cell’s summary:

                               OVERALL COMPARISON RESULTS



                  #   #         #####################  
                   # #          #                   #  
                    #           #     INCORRECT     #  
                   # #          #                   #  
                  #   #         #####################  


  Error:    Pins of instances of specified models were swapped during comparison.

In addition, in each cell containing an instance that required swapping, both the preceding message and a message similar to the following appears:

*************************************************************************
                             PINSWAPPED INSTANCE CONNECTIONS

       LAYOUT NAME                                               SOURCE NAME
*************************************************************************

        (This section contains detailed information about connections of
         matched instances that are crossmatched if called for in
         LVS REPORT PINSWAPPED or LVS REPORT PINSWAPPED ORIGINAL SUBTYPE).

-------------------------------------------------------------------------

       x001  yyy(aaa)                                            x001  yyy(aaa)
         G: gate                                                   G: gate
        *S: drn                                                   *D: drn
        *D: src                                                   *S: src
         B: blk                                                    B: blk

Note the particular pins that were swapped are indicated by the leading asterisk (*) in the report.

In the preceding example, the original instance is of type xxx, which was changed by the LVS Map Device statement to yyy(AAA). The LVS Device Type statement made the instance a MOS type device, allowing the swapping of the S and D pins but without changing the type. The LVS Report Pinswapped statement then caused the reporting of the instance that had its pins swapped.

When the ORIGINAL SUBTYPE keyword is used, a Mask SVDB Directory statement with a CCI, QUERY, or XDB keyword must be specified. Failure to do so results in a RPT1 compiler error.

See “Pin Swapping” in the Calibre Verification User’s Manual for more information on the topic of pin swapping. See also LVS All Capacitor Pins Swappable and LVS Builtin Device Pin Swap.

Examples

//report instances of these devices when pin swapping occurs
LVS REPORT PINSWAPPED MP(p_hv) MN(n_hv)
