LVS Pathchk Maximum Results

Specification statement

Specifies an upper limit on the number of polygons output by ERC Pathchk statements and Pathchk operations.

Usage

LVS PATHCHK MAXIMUM RESULTS {number | ALL}

Arguments

number

A positive integer that specifies an upper limit on the number of polygons output by an ERC Pathchk statement or Pathchk operation.

ALL

Keyword that specifies no limit to polygon output.

Note:

This option can exhaust the memory of a host machine and cause it to crash.

Description

For this discussion, the term Pathchk refers to the ERC Pathchk statement or to the Pathchk layer operation.

LVS Pathchk Maximum Results controls the number of polygons output by Pathchk. By default, the output limit is 12.5 million polygons per statement or operation.

For each multiple of 6 million polygons processed by a Pathchk command, up to the specified limit, this message is written in the ERC module section of the transcript:

WARNING: This PATHCHK Operation is generating excessive results: HGC=<n> LVHEAP=<n>  SHARED=<n>  MALLOC=<n>

When any Pathchk command exceeds the specified limit, this message is given:

WARNING: This PATHCHK Operation exceeded maximum results limit: HGC=<n>.
Operation results are incomplete

If any Pathchk command exceeds the specified limit during a run, this message is written toward the end of the transcript:

WARNING: An ERC Operation exceeded the maximum results limit: HGC=<n>.
Operation results are incomplete. See ERC section of the transcript.

Testing of settings that exceed the default output limit is recommended so that machine storage capabilities are not exceeded in a production environment.

Examples

// set an output maximum of 16 million polygons per Pathchk operation
LVS PATHCHK MAXIMUM RESULTS 16E6
