LVS Report Trivial Ports

Specification statement

Specifies whether trivial ports are reported as warnings during hierarchical netlist extraction. Used only in hierarchical netlist extraction.

Usage

LVS REPORT TRIVIAL PORTS {NO | YES}

Arguments

NO

Keyword specifying that trivial layout port objects do not trigger a warning. This is the default.

YES

Keyword specifying that trivial layout port objects trigger a warning.

Description

Specifies whether trivial layout port objects should trigger a warning during netlist extraction. Port objects are designated with the Port Layer Text and Port Layer Polygon specification statements.

A trivial layout port object is a port object attached to a net that does not serve as a pin in its containing cell in any placement of that cell. A net serves as a pin of a cell if the net interacts with geometries outside the cell, or with top-level port objects, or if the cell is an LVS Box cell and it has a named port object attached to it. Otherwise, the net does not serve as a pin, and any port object on that net is trivial. Trivial ports do not appear in the extracted netlist.

The main reason trivial port objects occur is their respective nets are never connected to upper-level nets in any placement in the design.

When YES is specified, trivial port objects are reported in the circuit extraction transcript and report. The trivial port object warning looks like this:

WARNING: Trivial port "R" at location (8,0) in cell "ff_p"

The YES option can be useful in flows where subcell ports are rigorously texted with port text objects and should always be connected at higher level cells, devices, or to top-level ports. In such flows, trivial port reports can help to identify missing connections in the design.

The YES option is most useful in cells that are specified as hcells. Like shorted net reporting in subcells, trivial port reporting does not occur for cells that are expanded during hierarchical database construction.

Promotion and injection of geometries can change the status of a port object from trivial to non-trivial, so ports that are attached to trivial nets (nets not connected to other cells, devices, or top-level ports) may or may not be listed as trivial ports. When this statement is used in conjunction with LVS Netlist All Texted Pins YES, all port objects will either be pins in the layout netlist, or they will be listed as trivial ports.

Trivial port warnings are not reported for non-hcells when the LVS Report Warnings Hcell Only YES specification statement is present. Trivial port objects are never top-level ports, so the LVS Report Warnings Top Only YES statement is not affected.

See also LVS Report Option.
