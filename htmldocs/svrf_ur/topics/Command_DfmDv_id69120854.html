DFM DV

Layer operation

License: Calibre nmDRC-H

Performs a voltage-dependent External operation on the input layer(s). Edge pairs that satisfy both the external constraint and the voltage constraint are output. Voltages are determined by properties attached to a net-indexed input layer.

Usage

DFM DV layer1 [WITH DEFAULT VOLTAGES  
                       | WITH VOLTAGE PROPERTIES Vmin_layer_prop Vmax_layer_prop]
[layer2 [WITH DEFAULT VOLTAGES  
              | WITH VOLTAGE PROPERTIES Vmin_layer_prop Vmax_layer_prop]]
ext_constraint
DVPARAMS ni_layer
Vmin_prop Vmin_default Vmax_prop Vmax_default
syncID_prop voltage_constraint
[OVERRIDE override_layer]
[UNIDIRECTIONAL]
[ANNOTATE [NETS]]
[NONTRIVIAL ONLY]
[MAXIMUM {count | ALL}]
[external_options]

Arguments

layer1 [layer2]

One or two required layer(s) for an External check, which measures the separation between exterior-facing edges on the input layer(s). The External check can be adjusted by the external_options, which are a subset of the options available for the External operation.

  • layer1 — A required original layer or layer set, or a derived polygon or edge layer.

  • layer2 — An optional original layer or layer set, or a derived polygon or edge layer, which is used in the two-layer form of the External check.

See the External reference page for complete details.

The input layer(s) must have connectivity information unless WITH DEFAULT VOLTAGES is specified for the layer. See “Connectivity Extraction” in the Calibre Verification User’s Manual.

WITH DEFAULT VOLTAGES

An optional keyword that specifies that the default voltages (Vmin_default and Vmax_default ) are always used for shapes on the corresponding layer, regardless of the voltage properties.

The placement of this keyword is order-sensitive. This keyword cannot be specified with OVERRIDE.

WITH VOLTAGE PROPERTIES Vmin_layer_prop Vmax_layer_prop

An optional keyword set that specifies that the voltage values for the corresponding layer are obtained from DFM properties on the layer, not from properties on ni_layer. Vmin_layer_prop and Vmax_layer_prop specify the property names for the minimum and maximum voltages on the associated net.

The placement of this keyword is order-sensitive. This keyword cannot be specified with OVERRIDE.

ext_constraint

A required constraint for the external check, which must be of type “<” or “<=”. The constraint must contain non-negative real numbers and is interpreted in user units. Edge pairs that satisfy both the ext_constraint and the voltage_constraint are output.

DVPARAMS ni_layer

A required keyword and argument, where ni_layer is a net-indexed layer produced by DFM Property Net. The ni_layer includes properties that specify the minimum and maximum voltages associated with the geometries on layer1 and layer2 (if present).

The ni_layer may additionally include the syncID property, which is a netID property used to determine if two nets are switching synchronously. See the “Description” section for details.

Vmin_prop Vmin_default Vmax_prop Vmax_default

A required set of arguments that provide information regarding the minimum and maximum voltages for a net.

  • Vmin_prop — The name of a numeric property on ni_layer, where the property value provides the minimum voltage on the associated net.

  • Vmin_default — A number providing the default minimum voltage to be used for nets that do not have the Vmin_prop property on ni_layer.

  • Vmax_prop — The name of a numeric property on ni_layer, where the property value provides the maximum voltage on the associated net.

  • Vmax_default — A number providing the default maximum voltage to be used for nets that do not have the Vmax_prop property on ni_layer.

syncID_prop

A required name of a netID property on ni_layer, where the syncID property value is used to determine if two nets switch synchronously.

If the syncID property is not used, define a syncID property for all nets and assign the property value NONETID using the function NONETID. In this case all nets are assumed to switch asynchronously (the worst case). See the “Description” section for details.

voltage_constraint

A required constraint on the maximum voltage difference between the two nets associated with each edge pair that satisfies the constraint for the External operation (ext_constraint). The voltage constraint must be of type “>”, “>=”, “==”, or a range (see “Constraints”). Edge pairs that meet the voltage_constraint are output. See the “Description” section for details on how the maximum voltage difference for two nets is calculated.

OVERRIDE override_layer

An optional keyword and layer name that specify to use information from properties on override_layer to determine the maximum voltage difference (DELTA_V) values for specific net ID pairs. The override_layer is generated by a DFM DV Override operation, and is typically used in analog applications.

If OVERRIDE is specified, then neither WITH DEFAULT VOLTAGES nor WITH VOLTAGE PROPERTIES may be specified.

For an example using OVERRIDE, see the example included with the DFM DV Override operation.

UNIDIRECTIONAL

An optional keyword that changes the maximum voltage difference calculation as described in Figure 1.

The type of switching (synchronous or asynchronous) and the syncID property are ignored when UNIDIRECTIONAL is specified.

ANNOTATE [NETS]

An optional keyword that adds DFM properties to the shapes on the output layer. The property values contain the data used to calculate the voltage difference between two nets.

The optional NETS keyword adds the netID property for each edge to the property annotation.

See “ANNOTATE Output” for details.

NONTRIVIAL ONLY

An optional keyword that causes edge pairs with two zero-length edges to be dropped from the output. See “Trivial Edges” in the Calibre Verification User’s Manual for general information on trivial edges. The statement DRC Extend Trivial Edge does not apply to DFM DV.

MAXIMUM {count | ALL}

An optional keyword set that specifies the maximum number of results that are output.

  • count — A positive integer that specifies the maximum result count.

  • ALL — A keyword that sets the maximum result count to 264. This is the default if MAXIMUM is not specified.

Depending on how your rule checks are constructed, however, you may not want to use the MAXIMUM keyword; see “Example 3”.

external_options

An optional argument that specifies options for the External operation that is performed. The following options are supported; refer to the External reference page for the definition of each option.

  • OPPOSITE [EXTENDED value] — See “metric”.

  • MEASURE ALL — See “polygon_containment”. Only valid if layer2 is specified.

  • [NOT] CONNECTED — See “connectivity_filter”.

  • CORNER TO CORNER [corner_constraint] — See “corner_filter”.

  • SINGULAR — See “intersection_filter”.

  • ABUT [abut_constraint] — See “intersection_filter”.

  • INSIDE ALSO — See “reversal”. Only valid if layer2 is specified.

  • PROJECTING — See “projection_filter”.

Description

DFM DV performs a voltage-dependent External operation on the input layer(s). Edge pairs that satisfy both the external constraint and the voltage constraint are output. Some External keywords are supported, but not all.

For edge pairs that satisfy the external constraint ext_constraint, the minimum and maximum voltage on the net associated with each edge is given by the properties Vmin_prop and Vmax_prop on the net-indexed input layer ni_layer. If the Vmin or Vmax property does not exist for a net, the specified default value (Vmin_default or Vmax_default) is used. The maximum voltage difference DELTA_V between the two nets is determined, and edge pairs that satisfy the voltage constraint voltage_constraint are output.

The maximum voltage difference (DELTA_V) between two nets is calculated as shown in the following figure; this is called the net-based DELTA_V. The net-based DELTA_V is affected by whether the voltages on the nets switch synchronously or not, and by the presence of the UNIDIRECTIONAL keyword. The type of switching is not considered if UNIDIRECTIONAL is specified. If OVERRIDE is specified, the DELTA_V value for specific net ID pairs can be overridden by values on the override_layer.

Figure 1. DFM DV: DELTA_V for Synchronous and Asynchronous Switching

The property syncID_prop is used to determine if two nets switch synchronously. The syncID property is of type netID; two nets with equal syncID values are considered to be in sync. The syncID may be the netID of the net, as determined by the GLOBALNETID function, or a netID determined by some other means, as shown in “Example 2”. A property value of NONETID specifies to use asynchronous switching. The syncID property is used as follows to determine the type of switching.

Table 1. DFM DV: Determination of Synchronous Switching

Synchronous Switching

Asynchronous Switching

The syncID property is defined for both nets, the values are equal, and the values are not equal to NONETID.

The syncID values are not equal, either value is NONETID, or the syncID property is not defined for at least one net.

Note:

A warning is issued if the syncID property is missing. If the syncID property is not used or is not known for a particular net, it is preferable to specify asynchronous switching by defining the syncID property with a value of NONETID in order to avoid transcript warnings. See “Example 1”.

The determination of synchronous or asynchronous switching affects the calculation of the net-based DELTA_V as shown in Figure 1.

The UNIDIRECTIONAL keyword is useful in a two-layer check in which results are only desired when FIRST_VMAX is greater than SECOND_VMIN, for example when checking spacing and voltages on anode and cathode layers.

If VMIN > VMAX for any edge in the input, a warning is issued and no results are reported for any such edge. Flat and hierarchical runs may produce different warning messages.

The DFM DV Analyze operation creates an RDB file that can aid in debugging the results from DFM DV. The DFM DV operation must use the ANNOTATE NETS keyword if the output layer is used in DFM DV Analyze.

Concurrency

DFM DV operations are executed concurrently if they have the following:

  • Same input layers (layer1 and layer2), and the layers have the same setting for WITH DEFAULT VOLTAGES and WITH VOLTAGE PROPERTIES

  • Same net-indexed layer (ni_layer)

  • Same property names for Vmin_prop, Vmax_prop, and syncID_prop, and for Vmin_layer_prop and Vmax_layer_prop if WITH VOLTAGE PROPERTIES is used

  • Same values for Vmin_default and Vmax_default

  • Same setting for NONTRIVIAL ONLY

The voltage constraint (voltage_constraint), distance constraint (ext_constraint), ANNOTATE setting, and External options (external_options) may vary.

Hierarchy Considerations

Depending on the design hierarchy and the method used to assign voltage properties to the polygons that make up a net, it is possible for more than one voltage property to be assigned to a net. DFM DV promotes geometries as needed in order to evaluate the voltage_constraint. However, DFM DV with the ANNOTATE keyword may cause more promotion in order to report unique voltages.

ANNOTATE Output

The ANNOTATE keyword attaches DFM properties to the shapes on the output layer. The following properties are reported, containing the data used to calculate the maximum voltage difference (DELTA_V) between two nets (refer to Figure 1).

Table 2. Properties Added by DFM DV ANNOTATE

Property Name

Description

FIRST_VMIN

The minimum voltage value for the first edge in the output edge pair.

FIRST_VMAX

The maximum voltage value for the first edge in the output edge pair.

SECOND_VMIN

The minimum voltage value for the second edge in the output edge pair.

SECOND_VMAX

The maximum voltage value for the second edge in the output edge pair.

SYNC

A value indicating how the DELTA_V value was determined for the edge pair:

0.0 — Asynchronous switching

1.0 — Synchronous switching

2.0 — OVERRIDE input

Synchronous and asynchronous switching is determined as described by Table 1.

If OVERRIDE is specified and the DELTA_V value was provided by information on the override_layer, then the SYNC value is 2.0.

The SYNC property is not attached if UNIDIRECTIONAL is specified and OVERRIDE is not specified.

DELTA_V

The maximum voltage difference calculated as shown in Figure 1 or provided by OVERRIDE input.

If ANNOTATE NETS is specified, the netID properties for each edge are reported in addition to those in the preceding table. However, the netID property for a layer is not attached if WITH DEFAULT VOLTAGES or WITH VOLTAGE PROPERTIES is specified for the layer.

Table 3. Properties Added by DFM DV ANNOTATE NETS

Property Name

Description

FIRST_NET

The netID of the first edge in the output edge pair.

SECOND_NET

The netID of the second edge in the output edge pair.

For a two-layer operation, the properties beginning with FIRST and SECOND correspond to edges on the first and second input layers, respectively. The correspondence is arbitrary for a single-layer operation.

The error clusters returned by DFM DV with the ANNOTATE keyword are fully merged. Two clusters are merged if every edge of each cluster is collinear with, touches, and has the same direction as an edge of the other cluster. However, clusters are not merged if the voltage values in the clusters differ.

Examples

Also see “Example 2” with DFM Property Net for an example that assigns net-indexed voltage values based on text attachments. Logic is included to use the maximum voltage value per net or a default if no text attachment is present for a net.

Example 1

The following example performs a single-layer DFM DV check. The syncID property is not used, so asynchronous switching (the worst case) is specified by attaching the syncID property named “nosync” to each net and assigning the value NONETID.

Voltage properties are assigned to the net-indexed layer net_prop_layer1 only if the area of the layer1 geometry is greater than 2—this means that default voltages are used if the area is less than or equal to 2. 

LAYER layer1 1
CONNECT layer1 
VARIABLE VH_net1 4
VARIABLE VL_net1 1
net_prop_layer1 = DFM PROPERTY NET layer1 
   [ vhigh = VH_net1 ]
   [ vlow = VL_net1 ]
   [ - = AREA(layer1)] > 2
   [ nosync = NONETID() ]

dv_1 {
   DFM DV layer1 < 1.5 DVPARAMS net_prop_layer1
      "vlow" 0 "vhigh" 5 "nosync" >= 4
}

Also see DFM Select By Net for an example of a warning check for use in a DFM DV flow. The warning check finds polygons for which the minimum voltage is greater than the maximum voltage.

Example 2

This example performs two different two-layer external operations—one operation uses the syncID property and the other uses asynchronous switching by setting the “nosync” property to NONETID. The minimum and maximum voltages depend on which layer the net is associated with.

Objects on layer1 or layer2 that interact with the same object on layer3 are considered to be in sync, and the syncID property is assigned the value of the NETID property for the layer3 object.

DELTA_V for the asynchronous case is 8, while DELTA_V for the synchronous case is 4. The voltage constraint is set to 5, so the synchronous nets do not result in an error. 

LAYER layer1 1
LAYER layer2 2
LAYER layer3 3
CONNECT layer1 
CONNECT layer2
CONNECT layer3 

VARIABLE VH_net1 5
VARIABLE VL_net1 1
VARIABLE VH_other 3
VARIABLE VL_other -3
// used for DFM DV check with asynchronous switching
// set nosync property to NONETID()
net_prop_layer = DFM PROPERTY NET layer1 layer2
   [ vhigh = ( (COUNT(layer1) > 0) ? VH_net1 : VH_other )]
   [ vlow = ( (COUNT(layer1) > 0) ? VL_net1 : VL_other )]
   [ nosync = NONETID()]
// assign the syncID property so that layer1 and layer2 shapes that 
// overlap the same layer3 polygon have the same syncID
or_12 = OR layer1 layer2
sync_layer = or_12 INTERACT layer3
CONNECT layer3 sync_layer

sync_one = DFM PROPERTY layer1 layer3 OVERLAP MULTI 
   [ - = COUNT(layer3)] > 0
   [syncID = NETID(layer3, 1)]

sync_two = DFM PROPERTY layer2 layer3 OVERLAP MULTI 
   [ - = COUNT(layer3)] > 0
   [syncID = NETID(layer3, 1)]
net_prop_wsync = DFM PROPERTY NET layer1 layer2 sync_one sync_two
   [ vhigh = ( (COUNT(layer1) > 0) ? VH_net1 : VH_other )]
   [ vlow = ( (COUNT(layer1) > 0) ? VL_net1 : VL_other )]
   [ syncID = (COUNT(sync_one) > 0) ? NETPROPERTY(sync_one, syncID, 1) : 
               (COUNT(sync_two) > 0) ? NETPROPERTY(sync_two, syncID, 1) :
                NONETID()]

// syncID property "nosync" is NONETID, so all nets are asynchronous
dv_12 {
DFM DV layer1 layer2 < 1.5 DVPARAMS net_prop_layer
   "vlow" 0 "vhigh" 5 "nosync" >= 5.0
}
// the syncID property is used, and synchronous nets do not produce 
// output for the given voltage constraint 
dv_12_sync {
DFM DV layer1 layer2 < 1.5 DVPARAMS net_prop_wsync
   "vlow" 0 "vhigh" 5 "syncID" >= 5.0
}

The following figure shows the results. The dv_12 check results, for which the nosync property is NONETID, are shown on the left. There are five results for dv_12.

The dv_12_sync results are shown on the right—there are three results. The dv_12_sync check uses the syncID property, and layer1 and layer2 shapes that overlap the same layer3 polygon have the same syncID. The two uppermost layer1 polygons and the uppermost layer2 polygon all overlap the uppermost layer3 polygon, so the layer1 and layer2 nets are synchronous. This means the DELTA_V value is 4 for those nets and they do not satisfy the voltage constraint.

Example 3

This example uses the MAXIMUM keyword. This rule file has three DFM DV checks. The first two checks are debug checks that may produce many errors, so MAXIMUM 100 is specified. The third check post-processes the DFM DV output with a DFM Property operation, so all results are needed and MAXIMUM is not specified. 

dv_1 { 
   DFM DFM DV layer1 < 1.5 DVPARAMS net_prop_layer1
      "vlow" 0 "vhigh" 5 "nosync" >= 4
      MAXIMUM 100
}

dv_2 {
   DFM DFM DV layer2 < 1.8 DVPARAMS net_prop_layer2
      "vlow" 0 "vhigh" 5 "nosync" >= 4
      MAXIMUM 100
}

dv_12_sync {  
   dv_err_all = DFM DV layer1 layer2 < 1.5 DVPARAMS net_prop_wsync
      "vlow" 0 "vhigh" 5 "syncID" >= 5.0 // do not specify MAXIMUM
      
   // filter output with DFM PROPERTY
   DFM PROPERTY dv_err_all dv_err_waived OVERLAP MULTI 
      [- = COUNT(dv_err_waived)] == 0 
}
