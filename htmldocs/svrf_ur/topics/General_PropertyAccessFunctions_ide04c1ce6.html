Property Access Function Summary

Certain DFM functions access values of properties that have been defined previously. The functions are divided into two classes. The first class accesses properties defined in the scope of some other layer operation and saved to a layer geometry. The second class, all containing the suffix string _REF, accesses properties within the scope of the same layer operation.

The following two examples use PROPERTY and PROPERTY_REF to demonstrate the usage of the two different classes of property access functions:

// annotate SD1 layer objects with NC, count of clustered contact polygons
SD1 = DFM PROPERTY SD CONT [ NC = COUNT(CONT) ] 
// read NC from SD1 layer with PROPERTY 
GATE1 = DFM PROPERTY GATE SD1 OVERLAP ABUT ALSO [ NC = PROPERTY(SD1, NC) ]
// Use property reference function to read property X in same operation.
// Optimizes performance by only calculating property X once.
A_out = DFM PROPERTY A B 
		[X = PERIM(B)/AREA(B)] [Y = PROPERTY_REF(X)*PERIM(A)]

The functions in the two classes of operation are summarized in these sections:

  • Property Access Functions for Layer Objects

  • Property Reference Functions for Properties Defined in the Same Operation

Property Access Functions for Layer Objects

The functions in the following table access properties saved to a layer geometry in a previous layer operation.

Table 1. Property Access Function Summary

Function

Summary

NARAC

Accesses values computed by Net Area Ratio operations with the ACCUMULATE keyword specified.

NETPROPERTY

Accesses DFM properties of type netID from objects on the input layer.

NETVPROPERTY

Accesses DFM properties of type vnetID (vector of net IDs) from objects on the input layer.

NIPROPERTY

Accesses net-indexed numeric property values from the input layer.

PROPERTY

Accesses numeric DFM properties from objects on the input layer.

PSPROPERTY

Accesses a string type property from an object on the input layer. The returned string is prefixed by Xn/Xm/.../ to indicate placement hierarchy.

SPROPERTY

Accesses string type properties from objects on the input layer.

SVPROPERTY

Accesses DFM properties that are vectors of strings from objects on the input layer.

VPROPERTY

Accesses vector type DFM properties from objects on the input layer.

VSTRING

Creates a vector of string type properties. The arguments to VSTRING can be a string value or an expression that evaluates to a string, including the string access functions SPROPERTY and PSPROPERTY.

Property Reference Functions for Properties Defined in the Same Operation

The functions in the following table reference properties already defined in the same operation. These functions are only available with the DFM Property operation. See “Referencing DFM Properties Defined in the Same Operation” for general usage information.

Table 2. Property Reference Function Summary

Function

Summary

NETPROPERTY_REF

References net number properties.

NETVPROPERTY_REF

References vector net number properties.

PROPERTY_REF

References numeric properties.

PSPROPERTY_REF

References string properties. The returned string is prefixed by Xn/Xm/.../ to indicate placement hierarchy.

SPROPERTY_REF

References string properties.

SVPROPERTY_REF

References vector string properties.

VPROPERTY_REF

References vector numeric properties.
