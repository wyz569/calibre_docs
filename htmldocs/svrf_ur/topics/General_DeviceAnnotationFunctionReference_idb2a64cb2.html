Device Annotation Function Reference

A variety of built-in functions, which are based upon DFM properties, provide access to geometric and connectivity information. You can use these functions within numeric, string, or complex expressions for device property computation. Numeric results can be used as arguments to relational operators.

Device annotation functions may be used in support of the device pushdown separated properties (PDSP) flow. They may also be used in the LVS Annotate Devices flow. Both of these flows defer extraction of certain specialized properties that are not typically needed for LVS to post-processing using the Query Server.

Many of the functions in the following table take one or more arguments that are references to either a layer or a pin. When referencing a pin, the compiler uses the name specified in the Device operation. When referencing a layer, the compiler uses the name or number of the layer only if it matches the name or number in the layer list of the Device operation. If the reference could be interpreted as either a pin or a layer, the pin interpretation is chosen.

Table 1. Device Annotation Built-In Functions

Function

Description

DEBUG

Enables debug monitoring during device annotation computation.

PROPERTY

Defines numeric properties to be computed.

SELECT

Defines seed, pin, or auxiliary layers that must be present on a device in order for an annotation to apply.

device_numeric_value

Returns the numeric value associated with a property.

pin_net

Returns the net number of a specified pin. In hierarchical applications, this function returns the node number local to the cell, which is a limitation.

Device TVF Functions

The Tcl Verification Format interface for Device property calculations is described under “TVF Functional Interface for Device Property Calculation”.

device::debug

Returns “1”, or “true”, if the DEVice statement calling the TVF Function block in which device::debug appears uses a DEBUG function and “0”, or “false”, otherwise.

device::dfm_vec_arr_measurements

Converts DFM-style rectilinear measurement array vectors into Device enclosure-vector-style width and length measurements in database units.

device::scaled_dfm_vec_arr_measurements

Converts DFM-style rectilinear measurement array vectors into scaled Device enclosure-vector-style width and length measurements in database units.

device::dfm_vec_measurements

Converts DFM-style rectilinear measurement vectors into Device enclosure-vector-style width and length measurements in database units.

device::scaled_dfm_vec_measurements

Converts DFM-style rectilinear measurement vectors into scaled Device enclosure-vector-style width and length measurements.

device::eval_dfm_func

Enables access to DFM Function TABLE and MATRIX functions. This function is only used within a TVF Function block.

tvf_numeric_function

Provides access to a numeric value returned by a Tcl procedure defined within a TVF Function definition.

tvf_string_function

Provides access to a string value returned by a Tcl procedure defined within a TVF Function definition.

Device TVF Layer Generation Functions

Create layers used with device property computations.

device::blocked_extent

Returns a derived polygon layer representing the extent of the input layout design. This extent is ignored in the context of a specified set of parameterized device cells (pcells).

device::enclosure_measurements

Returns a layer containing vectors of numbers that correspond to edge enclosure measurements. This function must be used in a TVF Function block and has performance advantages over the other enclosure implementations. However, if device orientation is transformed in the hierarchy, the following two functions are used instead.

The following two functions are generally preferable to this one because they handle coordinate transforms appropriately throughout the hierarchy.

device::enclosure_parallel_measurements

Similar to device::enclosure_measurements but measures enclosures parallel to gate edges only. Used instead of device::enclosure_measurements when the device orientation is not preserved throughout the hierarchy.

device::enclosure_perpendicular_measurements

Similar to device::enclosure_measurements but measures enclosures perpendicular to gate edges only. Used instead of device::enclosure_measurements when the device orientation is not preserved throughout the hierarchy.

device::select_bottom_device_edge

Returns a derived edge layer containing marker edges coincident with the bottom edges of device seed shapes.

device::select_left_device_edge

Returns a derived edge layer containing marker edges coincident with the left edges of device seed shapes.

device::select_right_device_edge

Returns a derived edge layer containing marker edges coincident with the right edges of device seed shapes.

device::select_top_device_edge

Returns a derived edge layer containing marker edges coincident with the top edges of device seed shapes.

device::select_left_bottom_device_marker

Returns a derived polygon layer containing marker shapes coincident with the bottom-left corners of device seed shapes.

device::select_left_top_device_marker

Returns a derived polygon layer containing marker shapes coincident with the top-left corners of device seed shapes.

device::select_right_bottom_device_marker

Returns a derived polygon layer containing marker shapes coincident with the bottom-right corners of device seed shapes.

device::select_right_top_device_marker

Returns a derived polygon layer containing marker shapes coincident with the top-right corners of device seed shapes.

DFM Property Data Retrieval Functions

These functions are used in conjunction with the DFM Property operation to retrieve property values for use within Device property computation programs. Refer to “DFM Property Data Retrieval Functions”.

dfm_numeric_value

Provides access to a DFM property from a single polygon on a pin, pin layer, or auxiliary layer. The DFM property is accessible as a regular numeric value.

dfm_string_value

Provides access to a DFM property from a single polygon on a pin, pin layer, or auxiliary layer. The DFM property is accessible as a string value.

dfm_numeric_array

Provides access to numeric DFM property values from a set of polygons on a pin, pin layer, or auxiliary layer. The set of properties is accessible through a TVF object created by this function.

dfm_vector_value

Provides access to a DFM vector property from a single polygon on a pin, pin layer, or auxiliary layer. The set of properties is accessible through a TVF object created by this function.

dfm_vector_array

Provides access to DFM vector properties from a set of polygons on a pin, pin layer, or auxiliary layer. The set of properties is accessible through a TVF object created by this function.

Unit Functions

Return units of measure.

precision

Returns the current value of the Precision statement (1000 by default).

unit_capacitance

Returns the current value of the Unit Capacitance statement (1E-12 farads by default).

unit_length

Returns the current value of the process Unit Length statement (1E-6 meters by default).

unit_resistance

Returns the current value of the Unit Resistance statement (ohms by default).
