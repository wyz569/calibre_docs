PEX Netlist Uppercase

Parasitic extraction

Specifies when to upper case element names, HSPICE, DSPF, and SPECTRE keywords, device model names, and device parameters in the output netlist.

Usage

PEX NETLIST UPPERCASE [KEYWORDS {NO | YES}]
[MODELNAMES {NO | YES}] [PARAMETERS {NO | YES}]

Arguments

At least one of the following keyword sets must be specified:

KEYWORDS {NO | YES}

Specifies whether or not to write element names, and HSPICE, DSPF, and SPECTRE keywords in upper case text in the output netlist. KEYWORDS NO is the default if this keyword set is not specified.

MODELNAMES {NO | YES}

Specifies whether or not to write device model names in upper case text in the output netlist. MODELNAMES NO is the default if this keyword set is not specified.

PARAMETERS {NO | YES}

Specifies whether or not to write device parameters in upper case text in the output netlist. PARAMETERS NO is the default if this keyword set is not specified.

Description

An optional statement that specifies when to upper case keywords, model names, and device parameters in an output netlist. Use this statement only with HSPICE, DSPF, and SPECTRE netlist formats.

Examples

Example 1

Include the following statement in your rule file for uppercase keywords and element names:

PEX NETLIST UPPERCASE KEYWORDS YES

Example 2

Include the following statement in your rule file to write model names in uppercase in the netlist:

PEX NETLIST UPPERCASE MODELNAMES YES

Example 3

Include the following statement in your rule file to write model names and device parameters in uppercase in the netlist:

PEX NETLIST UPPERCASE MODELNAMES YES PARAMETERS YES
