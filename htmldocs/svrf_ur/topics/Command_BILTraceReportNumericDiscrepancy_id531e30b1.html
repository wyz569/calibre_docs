report_numeric_discrepancy

Trace Property computation language function.

Generates a Trace Property numeric discrepancy in the LVS Report with the error value given as a percentage.

Usage

REPort_NUMeric_DISCrepancy(prop_name, numexpr)

Arguments

prop_name

A required name of a property declared in a PROPERTY statement.

numexpr

A required expression that evaluates to a number. Whatever this expression evaluates to is reported as an ERROR percentage, regardless of whether it is a percentage.

Return Values

None.

Examples

Suppose your program contains this:

TRACE PROPERTY MN(N) [
PROPERTY l, w, as, ad, nx
...
avg = (layout + source)/2
report_numeric_discrepancy(nx, avg)

Here is a possible report example:

*************************************************************************
                                PROPERTY ERRORS
DISC#  LAYOUT                                  SOURCE               ERROR
*************************************************************************
  1    M4(13.375,14.000)  MN(N)                M4  MN(N)
       nx: 6                                   nx: 4                  5%

The layout and source nx property values are reported as 6 and 4. The avg value 5 is reported as a percentage, even though it may not be one (as is the case here). You are responsible for ensuring the calculation of a percentage value occurs in the program.
