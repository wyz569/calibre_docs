LVS Clone By LVS Box Cells

Specification statement

Specifies whether to clone subcells of regular LVS Box cells that are also placed outside box cells. This statement is intended for certain Query Server CCI flows involving regular box cells.

Usage

LVS CLONE BY LVS BOX CELLS {NO | YES}

Arguments

NO

A keyword that specifies cloning of regular LVS Box subcells does not occur. This is the default.

YES

A keyword that specifies subcells of regular LVS Box cells are cloned when those subcells also have placements in non-box cells.

Description

Specifies whether subcells of regular LVS Box cells are cloned when such subcells are also placed outside box cells. This statement does not apply to LVS Box BLACK or GRAY cells and may only be specified once.

When YES is specified, subcells of regular box cells that are also placed outside box cells are replaced by copies of those subcells. This process is referred to as cell cloning, and the copies are referred to as clone cells, or simply clones. A cloned cell’s name has the form “cellname$LB$”, where cellname is the name of the cell that is copied. Cloning does not occur for regular box subcells that lie exclusively within box cells. The cloned cells are treated as regular box cells.

The YES option is useful in Query Server CCI flows that include AGF FILTER [IN] BOX LAYERS or dfm::create_filter -box_layers commands. Specifying YES causes the CCI box layer filter to be applied throughout the hierarchies of regular box cells. Without enabling cloning of box subcells, undesired geometry from cells placed outside any regular box cells can appear in the hierarchy of box cells in the AGF file.

The YES option is also useful when LVS Preserve Box Cells YES is used in conjunction with LVS Push Devices SEPARATE PROPERTIES YES (a flow that employs CCI post-processing). In that case, box subcells that also are placed outside of box cells can prevent the pushdown of devices through the hierarchy of a box cell. Enabling cloning of the box subcells allows pushdown to occur.

Examples

Suppose a design has cells TOP, C1, C2, C1_1, and C2_1, with a hierarchy like this, where --> indicates a placement.

TOP
   --> C1
   --> C1_1
   --> C2
         --> C1_1
         --> C2_1

If these statements appear in the rules:

LVS BOX C2
LVS CLONE BY LVS BOX CELLS YES

Then this is the internal representation of the hierarchy:

TOP
   --> C1
   --> C1_1
   --> C2
         --> C1_1$LB$
         --> C2_1

Cell placement C2/C1_1 is cloned because it is a subcell of regular box cell C2, and C1_1 is also placed in TOP. C2_1 is not cloned because it appears only in C2, which is a regular box cell.

Cloning ensures that only the desired layer data is output for the box cell placement of C1_1 when a box layer filter is used in a Query Server CCI flow.
