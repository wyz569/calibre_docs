Not Touch Inside Edge

Layer operation

Selects complete edges that do not coincide with an edge or segment from a polygon on a second layer having the same interior side.

Usage

NOT TOUCH INside EDGE layer1 layer2 [ENDPOINT {ALSO | ONLY}]

Arguments

layer1

A required original layer or layer set, or a derived polygon or edge layer.

layer2

A required original layer or layer set, or a derived polygon or edge layer.

ENDPOINT {ALSO | ONLY}

An optional keyword group that specifies the handling for collinear, same facing edges that interact only at their end points. By default, such edges are output by the operation. (These keywords work the same as in Touch Inside Edge, except that Not Touch Inside Edge returns the edges that are not selected.)

  • ENDPOINT ALSO — Specifies that collinear, same facing edges that interact only at their end points also satisfy Touch Inside Edge. layer1 edges that satisfy this behavior are therefore not output by Not Inside Edge, unlike in the default behavior. See “Example 2”.

  • ENDPOINT ONLY — Specifies that collinear, same facing edges that interact only at their end points are the only edges that satisfy Touch Inside Edge. All layer1 edges that do not satisfy this behavior are output by Not Touch Inside Edge. See “Example 3”.

Description

Selects all complete layer1 edges that are not coincident with layer2 edges, and including any coincident edges where there is no adjacent shared interior polygon area. This is the complement of the Touch Inside Edge operation with the same arguments. If layer2 is empty, the Not Touch Inside Edge operation generates output equivalent to the edge data on layer1.

For input edges oriented at integral multiples of 45 degrees with respect to the coordinate axes, the overlap between the edges must be less than 1 dbu to satisfy the non-touching condition. Input edges not in that set (these are called skew edges) having zero overlap satisfy the non-touching condition.

By default, layer1 edges that are collinear with a same facing edge on layer2 and that interact only at their end points are selected. The ENDPOINT ALSO and ENDPOINT ONLY keywords change this behavior.

See also Not Touch Edge, Not Touch Outside Edge, Not Coincident Inside Edge, and Not Touch.
