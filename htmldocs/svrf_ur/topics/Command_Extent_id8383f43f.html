Extent

Layer operation

Constructs a derived polygon layer that represents the least enclosing rectangle of the database.

Usage

EXTENT [layer]

Arguments

layer

An optional original layer or layer set, or a derived polygon or edge layer.

Description

Generates a derived polygon layer consisting of one rectangle that equals the database extent read in at runtime, including text. If you specify the optional layer parameter, the rectangle represents the minimum bounding box of all objects on layer. In hierarchical mode, Calibre may choose to divide the extent rectangle into polygons that are distributed across the hierarchy to facilitate more efficient processing of subsequent operations.

The Calibre database extent consists of layers that are required for Calibre nmDRC results, connectivity, or device recognition. Layers in the database that are not needed for runtime processing are not read in and do not contribute to the database extent. If you want to ensure that drawn layers not required for the flow contribute to the definition of extent, then see the Extent Drawn operation.

Calibre expands text object basepoints by 2 dbu in all directions. This affects the extent of text layers.

ICverify views the extent as that of the top-level cell template of the appropriate window.

In certain parasitic extraction flows, an extent layer that can be ignored by a set of parameterized device cells in order to avoid recognizing a substrate pin during extraction is desirable. For this application, see device::blocked_extent.

See also Extent Cell and Extents.

Example 2

Figure 2 shows a layout with a derived layer called bulk. Assume a simple rule file as follows:

Layer nwell 1
Layer poly  2
Layer met   3 // not needed in the run

bulk = EXTENT

rule {copy poly copy nwell}

Note that the layer met plays no role in deriving bulk in this rule file, because met is not needed in a rule check.

Figure 2. Bulk = EXTENT

If you want all drawn original layers in the database to contribute to the definition of bulk, then you may want to use this derivation instead:

bulk = EXTENT DRAWN ORIGINAL
