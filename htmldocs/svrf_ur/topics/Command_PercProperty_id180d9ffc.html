PERC Property

Specification statement

Specifies properties made available for Calibre PERC, even if the properties are not otherwise needed for LVS.

Usage

PERC PROPERTY [STRING] component_type [ ‘(’ component_subtype ‘)’ ]
property [property …]
‘[’
    [TOLERANCE {property_name tolerance_number}
                             [{property_name tolerance_number}…]]
‘]’

Arguments

STRING

An optional keyword that specifies that the properties listed in this statement are string properties. If STRING is not present, then numeric properties are expected.

component_type

A required device component type. See Device and “Built-In Device Types” in the Calibre Verification User’s Manual for a discussion of device component types.

The parameter component_type is case-sensitive if the LVS Compare Case specification statement has been specified with the YES or TYPES parameter.

(component_subtype)

An optional parameter enclosed in parentheses that specifies the device component subtype (or model name).

The parameter component_subtype is case-sensitive if the LVS Compare Case specification statement has been specified with the YES or SUBTYPES parameter.

property

A required, valid, SPICE device property that appears in the netlist. You can specify property any number of times in this statement, but each property must be unique. See “Device” Table 1 for a list of built-in devices with default properties. Properties generated by LVS Property Initialize are permitted.

Property names are case-insensitive.

TOLERANCE property_name tolerance_number

An optional keyword set that specifies a tolerance for matching numeric properties in a pattern template or for grouping devices when using perc::report_base_result -collapseList. You can specify more than one pair of property_name tolerance_number parameters but only one TOLERANCE keyword in a PERC Pattern Path statement. The TOLERANCE keyword set must be enclosed in brackets. The parameters are these:

  • property_name — A device property such as A (area), R (resistance), or L (length).

  • tolerance_number — A positive, floating-point number that specifies the allowable percentage variance between values in order to consider them the same. The default tolerance_number is 0.001, which gives 0.00001 as a precision.

This argument set may not be specified with STRING. This argument set does not apply to configurable patterns (those with the perc_config property).

Description

Instructs Calibre PERC to read the given list of properties from the input netlist, regardless of whether the properties are needed for LVS. These properties are then available for use during Calibre PERC rule checking.

By default, Calibre PERC only reads device properties that are actionable by LVS. The complete description of LVS-actionable properties is under LVS Netlist Property. LVS‑actionable properties are automatically available for use during Calibre PERC rule checking (but properties declared with LVS Netlist Property are not).

For each unique combination of component_type, component_subtype, and the keyword STRING, there can be at most one PERC Property statement. If component_subtype is not present, then the statement applies to all instances of the component_type, except for subtypes listed in a separate PERC Property statement.

The TOLERANCE keyword set can be used together with PERC Pattern Path or PERC Load PATTERN to enable property value matching within a range of values. The comparison is made with respect to the property value in the pattern template and is based upon the component type and/or subtype in the pattern template, not in the design netlist. As an example, if tolerance_number is 10 and the property value in the pattern template is 1.0, then a successful property-value match occurs if the corresponding property value in the design netlist is from 0.9 to 1.1, inclusive. The tolerance is applied in whatever device type/subtype in the design netlist that is matched to the pattern template.

Note that the computer representations for most floating-point values are close approximations of, but not precisely equal to, mathematical values. For example, the mathematically precise value 1.1 may be represented internally in binary form as 1.09999999 or 1.10000001 (although the computer might print out the value as 1.1). Hence, given a tolerance_number of 10 and a pattern-template property value of 1.0, a corresponding netlist value 0.9 might not be considered to match. If this occurs, one solution is to revise the tolerance_number to 10.01.

If PERC Load XFORM is used and the property does not exist for an instance of a specified device, this results in a runtime error:

ERROR: Properties missing on instances in design

and the overall status of the run is ABORTED. The affected cells are also shown in the report with an ABORTED status, regardless of whether they are expanded. If PERC Load XFORM is not used in this situation, then there is no error.

Examples

Example 1

// Read width and length properties for all MP devices
PERC PROPERTY mp  w l

Example 2

// Read width and length properties for all MN devices,
// except for MN(na) devices, which need the area properties
PERC PROPERTY mn     w  l
PERC PROPERTY mn(na) as ad

Example 3

// Read two string properties for all resistors
PERC PROPERTY STRING r str1 str2

Example 4

// Allow +/-5 percent tolerance in property R value for resistors.
// The additional 0.01 is to accommodate potential floating-point 
// representation differences.
PERC PROPERTY R [TOLERANCE R 5.01]
