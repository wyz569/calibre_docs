DFM Summary Report

Specification statement

Enables creation of a file containing a summary report for a Calibre DFM run.

Usage

DFM SUMMARY REPORT filename [REPLACE | APPEND] [HIER]

Arguments

filename

A required argument that specifies the filename of the summary report file.

The filename parameter can contain environment variables. For information regarding the use of environment variables in the filename parameter, refer to “Environment Variables in Pathname Parameters”.

REPLACE | APPEND

An optional keyword choice that specifies the behavior if filename exists.

  • REPLACE — Overwrite the previous contents of the summary file each time you run Calibre DFM. This is the default behavior if you do not specify this keyword.

  • APPEND — Append to an existing summary report file.

HIER

An optional keyword that instructs Calibre to create a section in the summary report that lists non-empty rule check statistics by layout database cell.

Description

Enables creation of a summary report file for a calibre -dfm run. The report includes the following information:

  • The date and time of the DFM run.

  • The rule file pathname and title, if specified.

  • The top-level layout cell pathname.

  • The current working directory and user name.

  • A summary of the number of original shapes processed per original layer.

  • A summary of the number of results generated per rule check, as well as a listing of which rule checks from the rule file were not performed.

  • All warnings that were generated.

  • The total run time, result count, number of original shapes processed, and number of rule checks executed.

If you do not specify this statement, no summary file is generated.

Examples

Write the summary report to the file dfm_report. The file is overwritten if it exists. The report includes check statistics by layout cell (the HIER keyword). 

DFM SUMMARY REPORT dfm_report HIER
