Device Property Computation Function Reference

A variety of built-in functions provide access to geometric and connectivity information about an instance. You can use these functions within numeric, string, or complex expressions. Numeric results can be used as an argument to a relational operator.

“Pin and Layer References” describes how references to pin names and layers are used in the device property functions.

The following table lists the data retrieval functions available for the built-in language, along with descriptions of their use.

Table 1. Device Property Computation Built-In Functions

Function

Description

DEBUG

Enables debug monitoring during device property computation.

PROPERTY

Defines properties to be computed.

HIDDEN PROPERTY

Defines device properties used in device property annotation programs but not printed in output netlists or PDSP reports.

Area functions

area

Returns the total area of shapes that are part of the specified pin or layer.

area_common

Returns the total area that is common to shapes on both of the specified pins or layers.

Counting functions

These functions do counts of various quantities.

bends

Returns the total bends in the shapes of the specified pin or layer. The result is expressed in units of right angles.

count

Returns the total number of shapes on the specified pin or layer.

Perimeter functions

perimeter

Returns the total length of the perimeter of the shapes for the specified pin or layer.

perimeter_inside

Returns the total length of the parts of perimeters on the first pin or layer that lie strictly inside shapes of the second pin or layer.

perimeter_outside

Returns the total length of the parts of perimeters on the first pin or layer that lie strictly outside shapes of the second pin or layer.

perimeter_coincide

Returns the total length of the parts of perimeters on the first pin or layer that coincide with the perimeter of the second pin or layer.

perimeter_coincide_inside

Returns the total length of the parts of perimeters on the first pin or layer that coincide with the perimeter of the second pin or layer, and where the coincident edges are adjacent to the interiors of shapes on the first pin or layer.

perimeter_coincide_outside

Returns the total length of the parts of perimeters on the first pin or layer that coincide with the perimeter of the second pin or layer, and where the coincident edges are adjacent to the exteriors of shapes on the first pin or layer.

Instance function

instance

Returns the instance number of the current device instance.

Net functions

Detect pins that are connected to special nets such as power or ground.

named_net

Returns the number of a named net. This function is not implemented hierarchically.

pin_net

Returns the net number of a specified pin. In hierarchical applications, this function returns the node number local to the cell, which is a limitation.

Unit functions

Return units of measure.

precision

Returns the value of the Precision statement (1000 by default).

unit_capacitance

Returns the value of the Unit Capacitance statement (1E‑12 farads by default).

unit_length

Returns the value of the Unit Length statement (1E‑6 meters by default).

unit_resistance

Returns the value of the Unit Resistance statement (ohms by default).

Location functions

x_location

Returns an x coordinate in user units associated with the given pin or layer shape.

y_location

Returns a y coordinate in user units associated with the given pin or layer.

Text functions

text_numeric

Returns numeric values of text objects in the layout that intersect the device seed shape.

text_string

Returns string values of text objects in the layout that intersect the device seed shape.

DFM Property data retrieval functions

Used in conjunction with the DFM Property operation to retrieve property values for use within Device property computation programs. Refer to “DFM Property Data Retrieval Functions”.

Device TVF functions

The Tcl Verification Format (TVF) interface for Device property calculations is described under “TVF Functional Interface for Device Property Calculation”. Also see “Table 1”. These functions are supported in Device property computation blocks.
