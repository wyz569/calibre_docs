DRC Exclude False Notch

Specification statement

Suppresses false notch errors in single-layer External operations. Used only in Calibre nmDRC‑H applications.

Note:

This statement is deprecated.

The EXTernal EXCLUDE FALSE keyword set used on a per-rule-check basis is preferable when false notch measurements are intolerable.

Usage

DRC EXCLUDE FALSE NOTCH {NO | YES}

Arguments

NO

Keyword that instructs Calibre not to suppress false notch errors. This is the default behavior if you do not include this statement in your rule file.

YES

Keyword that instructs Calibre to suppress false notch errors.

Description

Minimizes the possibility of false errors reported on notches for hierarchical single-layer External operations. A false violation can be reported across a bend in a polygon that traverses the hierarchy.

The YES option can cause substantially longer runtimes and should only be used if false notch errors are actually being observed. It is applied to all single-layer External operations regardless of whether they produce results or not. It does not apply to flat DRC checking and is specified at most once. See “False Measurement Reduction” in the Calibre Verification User’s Manual for more information.

Examples

DRC EXCLUDE FALSE NOTCH NO
