Layout Windel Cell

Specification statement

License: The HALO and HIER keywords require a Calibre RealTime Digital license.

Specifies a list of cell names whose placement extents are used to exclude input shapes and text from use in rule checks. Used only in Calibre.

Usage

LAYOUT WINDEL CELL {{cell_name [cell_name …]} | {LIST cell_list_name}} [HIER [PRESERVE layer [layer …]]] {[BY LAYER layer [PRIMARY]] | [ORIGINAL [OCCUPIED]]} [HALO value]

Arguments

{cell_name [cell_name …]} | {LIST cell_list_name}

A required argument specifying the names of cells or a list of cells whose placement extents are used to include input shapes and text for use in rule checks.

  • cell_name [cell_name …] — Specifies the name of a cell. You can specify cell_name any number of times in one statement. Each cell_name may contain one or more asterisk (*) wildcard characters, where the * character matches zero or more characters. When using the * character, be sure to enclose the cell name in quotation marks (“ ”). Otherwise, a compilation error occurs because * is a reserved symbol.

    String variables can also be used for cell names (see Variable).

  • LIST cell_list_name — Specifies the name of a cell list that is defined by the Layout Cell List statement. The cell_list_name is case‑sensitive and may not be a variable.

HIER [PRESERVE layer [layer …]]

An optional keyword that specifies hierarchical windowing that causes the specified cells to be treated as top-level cells. That is, each specified cell is treated as if it were a primary cell, with no effect upon its super-hierarchy. This argument is ignored in flat applications with a warning. This option requires a Calibre RealTime Digital license.

  • PRESERVE layer [layer …] — Specifies the names of original layers that are to be excluded from the window filter and flattened up into the specified cells. Because of the flattening, this argument is discouraged for layers that are dense and have deep hierarchy in the specified cells or their sub‑hierarchy.

[BY LAYER layer [PRIMARY]] | [ORIGINAL [OCCUPIED]]}

An optional keyword used to specify whether a layer or geometries are used to compute the extent of the specified cell.

  • BY_LAYER layer  [PRIMARY] — Specifies the name of a layer that forms the cell extent (not the boundary). The layer is flattened up from the bottom of the cell hierarchy by default. A compilation error occurs if specified without HIER.

    • PRIMARY — Specifies to use only geometry in the top-level of the cell. The layer is always merged at the top-level of the cell prior to any window formation.

  • ORIGINAL [OCCUPIED] — Specifies that all geometries in the input layout database are used to compute the cell extent.

    • OCCUPIED — Specifies that only the cells containing geometry required in the Calibre run (including anywhere in their subhierarchy) have their original extents returned; all other cells are ignored.

HALO value

An optional keyword that specifies the cell extents are undersized by the specified value and used instead. The value is a positive floating numeric dimensioned in user units and may be a numeric expression containing variables. An extent that disappears as a result of being undersized is ignored (treated as empty) with a warning. This option requires a Calibre RealTime Digital license.

Description

Specifies to exclude shapes and text objects that occur in windows formed for each cell_name during a Calibre run. This statement functions essentially like Layout Windel, but rather than providing coordinates, you provide cell names. For each cell C, the extents of all placements of C in the input layout database from the flat viewpoint are scanned. The coordinates of each extent are then used in an implicit Layout Windel statement.

Calibre warns for each cell_name argument that is not located in the input layout database.

If ORIGINAL is specified, then all objects in the input layout database are used to compute the specified cell extent(s), not just those objects actually required by the flow. If OCCUPIED is additionally specified, then all objects in the input layout database are used to compute the specified cell extent(s), if there is at least one object required by the flow within the cell or its subhierarchy; however, if a specified cell does not actually contain objects that are required by the flow (either in the cell or its sub-hierarchy), then the extent returned by this operation for that cell is empty (that is, as if ORIGINAL were not specified).

By default, the cell extents used by these statements are the extents of only those objects (text and geometry) actually required by the run. Calibre applications completely ignore objects in the input layout database that are not actually required by the flow. For example, if the input layout database contains objects on layer 53, but layer 53 is nowhere in a rule check or is not required for connectivity in the current run, then this layer is ignored as the input layout database is processed. Being ignored implies that objects on layer 53 are not used to compute the extents of individual cells.

See also Layout Windel Layer, Layout Window Cell, Layout Window Clip, and Exclude Cell.

Examples

Example 1

All extents of dff_* and memblock instances become exclusion windows; use all objects in input database for computing extents.

Layout Windel Cell "dff_*" memblock ORIGINAL
