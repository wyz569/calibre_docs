DFM Copy

Layer operation

License: Calibre nmDRC-H

Copies input layers to a new layer. The type of input layer can be changed in the output.

Usage

DFM COPY layer [layer …]
[{REGION [UNMERGED]} | EDGE | {CENTERLINE [CLUSTER]}  
    | {TIE [CENTER] [CLUSTER]} | {CLUSTER [LAYERID]} | UNMERGED]
[CELL LIST list_name] [PREMERGE]

Arguments

layer

A required layer name. The layer must be an original or derived polygon layer, a derived edge layer, or a derived error (cluster) layer. If more than one layer is specified, all input layers are copied into a single output layer. If the input layers contain merged data, the output layer is merged. If the input data is unmerged, see “Merging Behavior” for details.

Note:

The secondary keywords REGION, EDGE, CENTERLINE, TIE, CLUSTER, UNMERGED are mutually exclusive.

REGION [UNMERGED]

An optional keyword set that instructs the operation to convert error edge clusters on layer into polygons. This keyword may be used only if layer is a derived error layer. Error clusters are converted as follows:

  • Clusters with two edges are converted by filling the space between the edges, such that the two edges become the sides of a polygon as shown here:



  • Clusters with more than two edges are converted into polygons using the convex hulls of the edges as shown here:



  • Clusters of single or collinear edges are expanded into rectangles of approximately 4 database units in width.

By default, the resulting polygons are merged within each cell, so polygons from different error clusters are merged if they touch or overlap. If this is not the desired behavior, use the UNMERGED keyword.

  • UNMERGED — Specifies to produce an output layer containing unmerged polygons. Unmerged polygon layers can be processed by a limited number of operations; see “Unmerged Polygon Layers” in the Calibre Verification User’s Manual.

EDGE

An optional keyword that instructs the operation to convert error clusters on layer into edges and to output a derived edge layer. This keyword may be used only if layer is a derived error layer. Each edge of a cluster is copied to the output layer as a separate edge. Edge direction is preserved. Note that edges on the output layer are merged within each cell. Edges from different error clusters are merged into one edge if they are collinear and share at least one point. Similar to the OR Edge operation, coincident edges that face in opposite directions are not output.

Note:

Error clusters do not preserve the polygon numbers of their edges, therefore the output of DFM Copy with EDGE does not have polygon numbers attached to its edges. If polygon numbers are desired, for example if an edge layer is to be passed to a measurement operation that consults polygon numbers, edges should be obtained directly from the corresponding polygon layer, for example, with Coincident Edge or positive edge-directed output from a measurement operation.

CENTERLINE [CLUSTER]

An optional keyword set that instructs the operation to convert two-edge error clusters on layer into a single edge that is the centerline of the cluster. The output is a derived edge layer, which is merged by default. The direction the output edge faces (for downstream dimensional check operations) is arbitrary. This keyword may be used only if layer is a derived error layer.



See “CENTERLINE Behavior” for details about the centerline derivation.

  • CLUSTER — An optional keyword that produces a derived error layer with single-edge clusters instead of a derived edge layer. Use this keyword to avoid merging the output edges, for example, if you want to preserve collinear edges.

TIE [CENTER] [CLUSTER]
An optional keyword set that instructs the operation to convert two-edge error clusters on layer into single edges, where the single edges are ties between the original cluster edges. By default, the tie represents the shortest distance between the two cluster edges. If the cluster edges are parallel and project onto each other, the tie is placed in the center of the projection region. The output is a derived edge layer, which is merged by default. The direction the output edge faces (for downstream dimensional check operations) is arbitrary.

  • CENTER — An optional keyword that places the tie at the center points of the two edges in the cluster.

  • CLUSTER — An optional keyword that produces a derived error layer with single-edge clusters instead of a derived edge layer. Use this keyword to avoid merging the output edges, for example, if you want to preserve collinear edges.

CLUSTER [LAYERID]

An optional keyword set that combines one or more input edge layers into a single output derived error layer composed of single-edge clusters. Edges are not merged before the conversion, so the single-edge clusters can be coincident with each other. All input layers must be derived edge layers. DFM properties on the input layers are not preserved when CLUSTER is specified.

  • LAYERID — An optional keyword that instructs the operation to assign a numeric DFM property named LID to each output edge. The property value indicates the input layer that the edge is derived from. For example, a property value of 1 means the edge is derived from the first input layer, 2 means it is derived from the second input layer, and so on.

UNMERGED

An optional keyword that is used in two cases:

  • To specify that an input layer is an unmerged polygon layer

    The UNMERGED keyword is required if any input layer is an unmerged polygon layer. The output layer is also unmerged.

  • To specify unmerged polygon output with merged polygon input.

    For multilayer input, overlapping and abutting shapes on different layers are not merged.

Unmerged derived polygon layers are supported by a limited number of operations; see “Unmerged Polygon Layers” in the Calibre Verification User’s Manual.

CELL LIST list_name

An optional keyword that specifies the cells from which layers are copied. If this keyword is specified, only geometries in the cell list specified by list_name are copied to the output layer. If this keyword is not specified, all geometries from layer are copied to the output layer. This keyword is applied to all input layers.

The list_name parameter is one of the following:

  • name — A cell list name created with the Layout Cell List statement.

  • LVS_BOX_AND_EXCLUSIVE_BOX_SUBCELLS — A string that specifies to use cells that are LVS box cells or exclusive box subcells. An exclusive box subcell is a cell that is a child cell of an LVS box cell only. The cell list is determined internally.

  • NON_LVS_BOX_NOR_EXCLUSIVE_BOX_SUBCELLS — A string that specifies to use cells that are not LVS box cells and not exclusive box subcells. Depending on the sub-hierarchy of the LVS box cells, this list can contain cells that are sometimes child cells of LVS box cells and sometimes not. The cell list is determined internally.

    Also see LVS Clone By LVS Box Cells.

PREMERGE

An optional keyword that causes error clusters in the input layer to be merged before the copy operation. Applies only to derived error layer input. The keyword can be combined with the other keyword sets that apply to error layer input (REGION, TIE, and CENTERLINE).

Two error clusters are merged if every edge of each cluster is collinear with, touches, and has the same direction as an edge of the other cluster. The input error clusters are not merged if DFM properties are attached to the input layer. Clusters are merged only within the same input layer. See “Example 8”.

Description

Copies layers of the same type to a new layer. Node numbers and DFM properties are copied to the output layer when a single input layer is specified and the CLUSTER keyword is not used. Node numbers and DFM properties are not preserved when multiple input layers are specified.

If more than one polygon layer is specified, the output layer is merged and contains copies of all polygons from all input layers. Therefore, this operation:

P = DFM COPY A B C D

is equivalent to:

P = (A OR B) OR (C OR D)

If more than one edge layer is specified, the output layer is merged and contains copies of all edges from all input layers. Therefore, this operation:

E = DFM COPY A B C D

is equivalent to:

E = ( A OR EDGE B ) OR EDGE ( C OR EDGE D )

If more than one error layer is specified, the output layer contains copies of all edge clusters from all input layers. If REGION is specified, all edge clusters are converted to polygons and resulting polygons from all input layers are copied into the output layer and merged, as if by the OR operation. If EDGE or CENTERLINE are specified, all edge clusters are converted to edges and resulting edges from all input layers are copied into the output layer and merged, as if by the OR Edge operation.

CENTERLINE Behavior

The CENTERLINE keyword works as follows. Let PQ and RS be the two edges of a cluster.

  1. Form two pairs of segments as PR and QS, and PS and QR, respectively.

  2. If exactly one of the pairs of segments from the previous step does not intersect, that pair is selected. Otherwise, a pair is chosen arbitrarily.

  3. Connect the midpoints of the segments selected in the previous step with a segment.

    If PQ and RS share a common endpoint, then the centerline bisects the angle formed by the edges.

The constructed segment is the center line of the edge cluster. If a centerline edge is output to GDS or OASIS (such as with DRC Check Map) the centerline object is a path record, not a polygon record such as with the REGION CENTERLINE keyword for Enclosure, External, and Internal.

Merging Behavior

For original layer input, overlapping and abutting shapes on the same layer are always merged within the same cell, but not across different cells.

If multiple merged polygon input layers that abut or overlap are specified, or multiple collinear edges or edge clusters are specified, DFM Copy merges them upon output by default.

The UNMERGED keyword with merged polygon layer input results in an unmerged polygon layer output. For multilayer input, shapes on different layers are not merged.

This operation also accepts unmerged layers as input. The following table describes how unmerged data is processed:

Table 1. DFM Copy Merging Behavior for Unmerged Input

Unmerged Input

Output

Single unmerged edge layers

Not merged within cells and across cells.

Multiple unmerged edge layers

Merged within cells and across cells, except when CLUSTER is specified.

Single and multiple unmerged error layers

Not merged within cells and across cells.

Single and multiple unmerged polygon layers

(Requires the UNMERGED keyword)

Not merged within cells and across cells.

By default, the CENTERLINE and TIE keywords produce a derived edge layer that is merged on output. To avoid merging the output, use the CLUSTER keyword to produce a derived error layer with single-edge clusters.

You can specify PREMERGE with derived error layer input to merge the input error layers before the copy operation. Clusters are merged only within the same input layer.

Edges Facing Opposite Directions

The DFM Copy operation does not merge two edges or edge segments when they face opposite directions. This is similar to the behavior shown in Figure 1 with the OR Edge operation. The output from DFM Copy in these situations can cause unexpected results from layer derivations that use that output. To merge coincident edge segments that face opposite directions, use DFM OR Edge BOTH instead of DFM Copy.

Examples

Example 1

Copies the error layer D to a new layer D1:

D = EXTERNAL MET <= 0.1
D1 = DFM COPY D

Example 2

Copies polygon layers A and B into a single, merged output layer:

A_B = DFM COPY A B

Example 3

Converts an error layer to a polygon layer:

MET_S = EXT MET <= 0.1
MET_SR = DFM COPY MET_S REGION

Example 4

Converts two-edge clusters to single edges. The edges are located on the centerline between the two edges in the cluster and face an arbitrary direction:

MET_S = EXT MET <= 0.1
MET_SCL = DFM COPY MET_S CENTERLINE

Example 5

Copies all geometries in cells that start with “A” from layer MET to layer MET_A:

LAYOUT CELL LIST A "A*”
MET_A = DFM COPY MET CELL LIST A

Example 6

Copies all geometries in cells that do not start with “A” from layer MET to layer MET_NOTA:

LAYOUT CELL LIST NOTA "*" "!A*"
MET_NOTA = DFM COPY MET CELL LIST NOTA

Another way to accomplish the same task using a regular expression:

LAYOUT CELL LIST NOTA "^[^A]"
MET_NOTA = DFM COPY MET CELL LIST NOTA

Example 7

Converts derived edge layers MET1_EDGE and MET2_EDGE into a single derived error layer composed of single-edge clusters:

MET_ERROR = DFM COPY MET1_EDGE MET2_EDGE CLUSTER

Example 8

Use PREMERGE to merge error layer input before the copy operation. 

metal1_ext = EXT METAL1 < 0.1
metal1_err_cp_merge = DFM COPY metal1_ext PREMERGE

PREMERGE merges only within each input layer. The following example merges multiple error input layers. 

combine_err = DFM COPY err_input_1 err_input_2
merged_err = DFM COPY combine_err PREMERGE
