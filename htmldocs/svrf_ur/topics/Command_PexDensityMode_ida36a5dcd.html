PEX Density Mode

Parasitic extraction

Specifies the boundary window handling mode for density computation.

Usage

PEX DENSITY [layer] MODE
{TRUNCATE | BACKUP | WRAP |CONTEXT [CENTER]}

Arguments

layer

A optional LVS layer name.

TRUNCATE

A required keyword that truncates partial windows to include only the extent of the design.

BACKUP

A required keyword that specifies the last window be shifted back to align it right-hand or top edges with the design extent. This means it now overlaps the previous window.

WRAP

A required keyword that specifies the design geometries be repeated beyond the design extent as if several instances of the design were tiled. The boundary window will include some of the geometries from the next instance of the design.

CONTEXT [CENTER]

A required keyword that specifies to use complete tiles beyond the extent of the design in both the x and y directions, starting from the lower left corner of the design extent. If a tile covers at least some portion of the extent, it is used without truncation.

When specified with the optional CENTER keyword, the first tile is placed in the center of the design instead of the lower left corner of the extent. Then the tiles go from the center to the edges of the extent without truncation.

The density used for portions beyond the extent of the design is controlled by the PEX Density Context statement, or 0.5 if the PEX Density Context statement is not specified.

Description

An optional statement that controls the window handling mode for PEX density calculations for a specified layer or the design. This statement may be specified more than once.

You can use this statement to override the mode specified in any PEX Density Window statement defined for the same layer. The other density window parameters such as window size or STEP are not affected.

If a layer is not specified in the PEX Density Mode statement, then the specified mode overrides all PEX Density Window modes for all layers except those specified in another PEX Density Mode statement in the rule file.

Examples

Consider the following statements which are defined in the calibrated rule file:

PEX DENSITY M1 WINDOW 50 50 TRUNCATE
PEX DENSITY M2 WINDOW 50 50 STEP 25 25 TRUNCATE

To change the window handling mode to CONTEXT without affecting the window size setting, specify the following statements in your rule file:

//global setting overrides mode for all layers to be CONTEXT
PEX DENSITY MODE CONTEXT
// specific override for layer M2 to make its mode CONTEXT CENTER
PEX DENSITY M2 MODE CONTEXT CENTER

As a result, M1 uses CONTEXT mode and M2 uses CONTEXT CENTER mode. Layers M1 and M2 maintain their window size and step size settings from the original statements in the calibrated rule file.
