dfm_numeric_value

Device DFM Property data retrieval function.

Provides access to a DFM property from a single polygon on a pin, pin layer, or auxiliary layer. The DFM property is accessible as a regular numeric value.

Usage

DFM_NUMeric_VALue(pin_or_layer, “property_name”[, pin_name])

Arguments

pin_or_layer

A required pin or layer name. A layer is derived from a DFM Property or Device Layer ANNOTATE operation. A pin must have DFM properties associated with it.

In a device property computation program, this parameter is a required pin, pin layer, or auxiliary layer name.

In a device annotation program this parameter is a name of a pin layer or an annotation auxiliary layer in a Device operation.

property_name

A required DFM property name. The property must be defined in the same layer operation used to derive the layer for the pin_or_layer argument.

pin_name

An optional pin name corresponding to a Device pin layer that intersects the pin_or_layer polygon. This parameter only applies to the device property annotation language, not the device property computation language. The pin name allows for selection from multiple annotation geometries based on overlap of the original pin location. (Note that the pin location is represented by an arbitrary point on the original device seed/pin boundary, so the optional pin geometry must unambiguously cover all of the pin area or boundary that established the original device.)

Return Values

A numeric property value. The device is classified as a “BAD DEVICE” in the connectivity extraction report if the selected pin or layer includes more than one polygon.

Description

This function retrieves a single numeric property value by name from one polygon. This function may be used in device annotation and device property computation programs. The commas shown in the syntax are literal.

The pin_name can be used in Device Layer ANNOTATE programs to facilitate easier access to numeric property values. When pin_name is used, the corresponding pin shape must intersect the pin_or_layer shape. The property value on the pin_or_layer that corresponds to the pin_name is then output by the function. To use this capability, the Device element (type and optional subtype) referenced by the Device Layer operation must be unique. See Example 2 with “Example Device Annotation Programs Using DFM Property”.

The dfm_numeric_value function retrieves values from the operation that assigns DFM properties. Measurement values are in user units in this case. However, Device statement measurements are in meters. Depending on the Unit Length setting in the LVS rules, you may need to scale dfm_numeric_value return values in a Device statement program block by multiplying them by the unit_length() function, but setting the rule file Unit Length to 1 makes this unnecessary.

The dfm_numeric_array function allows access to multiple polygons on a layer.

Examples

The following Device computation creates a device property from the DFM property NC that contains the number of polygons on layer cont that interact with the S and D pins of the device:

// Count contacts on SD polygons and assign property "NC" to 
// derived layer sd_prop polygons.
sd_prop = DFM PROPERTY sd cont [ NC = COUNT(cont) ] 
// Produce device properties nc_s and nc_d based on DFM property "NC" 
// for S and D pins.
DEVICE MP pgate poly(G) sd_prop(S) sd_prop(D) [
  property nc_s, nc_d
  nc_s = DFM_NUMeric_VALue( S, "NC" )
  nc_d = DFM_NUMeric_VALue( D, "NC" )
]

Note that when multiple polygons qualify for selection by the DFM_NUMeric_VALue function, the device is classified as bad even if the DFM_NUMeric_VALue function is never executed due to conditional statements. For example:

// Produce device property nc_aux
DEVICE MP pgate poly(G) sd(S) sd(D) <aux1> [
  property nc_aux
  if (COUNT(aux1) == 1 ) { 
  // This conditional protection does not prevent the device from being
  // classified as a bad device in cases where this condition fails.
     nc_aux = DFM_NUMeric_VALue( aux1, "NC" )
  } else {
    nc_aux = 0
  }
]

In cases where there may be more than one polygon associated with the pin or layer, always use the DFM_NUMeric_ARRay function to access the associated DFM properties.

DFM_NUMeric_VALue supports DFM NetID properties. NetID properties act similarly to numeric properties within the Device computation language. For example, the following program produces Net IDs similar to the PIN_NET function for source and drain pins in a MOSFET device:

sd = diff not poly 
sd_prop = DFM PROPERTY sd [ NET = NETID( sd ) ]

device mp pgate poly(G) sd_prop(S) sd_prop(D) well(B)
[
    PROPERTY src_net_prop, drn_net_prop, src_pin_net, drn_pin_net
    src_net_prop = dfm_num_val( s, "NET" )
    drn_net_prop = dfm_num_val( d, "NET" )
    src_pin_net = pin_net( s )
    drn_pin_net = pin_net( d )
]
