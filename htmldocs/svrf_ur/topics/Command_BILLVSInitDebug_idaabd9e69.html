DEBUG

LVS Property Initialize computation language function.

Enables debug monitoring during LVS Property Initialize computation.

Usage

DEBUG range1[, range2 …]

Arguments

range1

A required device instance ID number or a pair of device instance numbers separated by a hyphen (‑). If the hyphenated form is used, the first number must be less than the second number. In that case, all numbers between and including the numbers specified in the range correspond to instances. At least one range must be specified, although it can contain only a single instance. Multiple ranges are separated by a comma (,) followed by whitespace.

Return Values

None.

Description

Debug monitoring of the LVS Property Initialize computation can be useful in finding errors during the development of new property computation code. When present, DEBUG must follow the PROPERTY statement and precede the optional WARN statement. The debugging trace is written to the transcript. The trace can produce much output, so the use of small ranges is recommended.

This statement can be used in conjunction with the WARN statement, which reports all mathematical computation errors.

The instance IDs represent both the layout and source device in the input pair. Flat instance IDs are unique across all devices in the design. Instance IDs are assigned by the system and are not guaranteed to be sequential, merely unique.

Examples

Assume the run gives this warning in the transcript:

***************************************************
EFFECTIVE property user-program WARNING:
Argument for sqrt must be zero or positive (result set to NaN) on line 20
while transforming layout with:
LVS PROPERTY INITIALIZE ... (within PROPERTY program beginning on rule file line 13)
Use instance ID# 18 in DEBUG trace range.
***************************************************

Then you can do something similar to this:

LVS PROPERTY INITIALIZE R [ 
PROPERTY r
DEBUG 18
...

Instance 18 will have a debugging trace run on it with details in the transcript.
