SVRF Statements Mapped to Compile-Time TVF

Most SVRF specification statements, device extraction statements, connectivity extraction operations, and fracture operations are mapped to TVF commands. The initial keyword of the statement is mapped to a command in the tvf:: namespace, with remaining secondary keywords and parameters passed in as arguments. Most layer operations are not mapped to TVF commands but handled instead with the tvf::SETLAYER and tvf::OUTLAYER functions.

For example, the DRC Select Check statement is mapped in TVF as:

tvf::DRC SELECT CHECK width_rule

or

tvf::drc Select Check width_rule

The TVF command name is tvf::DRC or tvf::drc, with the remainder of the SVRF syntax passed as arguments to the command, which the SVRF compiler interprets. The arguments after the TVF command name are case-insensitive.

Layer Operations Mapped to TVF Commands

Most layer operations are not mapped directly to TVF commands. Instead, layer assignment, layer derivation, and rule check output is handled with the tvf::SETLAYER, tvf::OUTLAYER, and tvf::RULECHECK TVF commands and the layer operation is passed in as command arguments.

However, exceptions exist for the Copy operation and the DFM and MDP families of layer operations. For example, you may use “tvf::COPY layer” instead of “tvf::OUTLAYER Copy layer” within a tvf::RULECHECK body for results output. The tvf::SETLAYER command must still be used for layer assignment.

See “Layer Definitions and Assignments” and “Mapping of SVRF Commands to TVF” for additional information.

Example: SVRF to TVF Translation

This example shows how SVRF specification statements are translated into TVF commands.

SVRF: 

Layout System GDSII        // (1) use of case insensitivity 
Layout Path $input_layout  // (2) use of environment variable for pathname
Layout Primary TOP
Source System SPICE        // (3) "source" is a Tcl command
Source Path $input_source
Source Primary TOP

TVF:

tvf::layout System GDSII
# (1) use all upper- or lowercase letters for command names; 
# "System GDSII" are parameters passed to the tvf::layout command and 
# are not case-sensitive
tvf::LAYOUT PATH $env(input_layout)
# (2) Tcl environment variable mechanism
tvf::LAYOUT PRIMARY TOP
tvf::SOURCE System SPICE
# (3) SOURCE must be uppercase to avoid collision with Tcl source command
tvf::SOURCE PATH $env(input_source)
tvf::SOURCE PRIMARY TOP
