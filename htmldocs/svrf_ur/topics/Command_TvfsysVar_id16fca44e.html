tvf::sys_var

Application: runtime only.

Accesses the value of a system variable in runtime TVF.

Usage

tvf::sys_var variable

Arguments

variable

A required argument that specifies the name of a system variable. The available system variables are listed in the following table:

Table 1. System Variables for tvf::sys_var

Variable Name

Description

PRECISION

Resolves to the rule file precision value (which may or may not have been explicitly defined by a Precision specification statement).

PRECISION_N

Resolves to the numerator of the rule file Precision value if it is expressed as a ratio; otherwise, resolves to the rule file Precision.

PRECISION_D

Resolves to the denominator of the rule file Precision value if it is expressed as a ratio; otherwise, resolves to 1.

RESOLUTION

Resolves to the value defined with the Resolution specification statement when a single argument is specified, or two equal arguments are specified.

RESOLUTION_X

Resolves to the first value defined with the Resolution specification statement when two arguments are specified.

RESOLUTION_Y

Resolves to the second value defined with the Resolution specification statement when two arguments are specified.

LAYOUT_PATH

Resolves to the pathnames specified with the Layout Path specification statement.

LAYOUT_PRIMARY

Resolves to the top-level cell or subcircuit name specified with the Layout Primary specification statement.

The PRECISION, PRECISION_N, and PRECISION_D variable names are also System Variables and may be used outside of a TVF Function block. See the Variable statement for more information.

Return Values

The value of the system variable. If the system variable does not exist, tvf::sys_var returns an empty string (“”).

Examples

// accesses the value of PRECISION
set precision [tvf::sys_var PRECISION]
