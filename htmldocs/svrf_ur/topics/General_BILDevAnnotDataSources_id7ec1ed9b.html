Data Sources

Data used in device property annotation programs can come from numerous sources.

Data sources include the following:

  • Constants — You can use numeric, floating-point constants, and string literals directly within the property specification.

  • Numeric Process Variables — These values are accessed by using a rule file variable statement just as any other variable within the property specification. Rule file variables may never appear as the left-hand side of an assignment. Doing so results in a compiler error.

  • String Process Variables — Rule file variables may be assigned to property variables (names declared in the PROPERTY statement) or assigned to local variables. No other use of string rule file variables is supported.

    Properties starting with the string ICV_ are reserved for internal use by the tool.

  • Instance Data — Geometric, connectivity, and text data associated with an instance are accessed by the provided built-in functions. Properties generated on specific shapes using a DFM Property operation are available to commands that support them. A summary table of the device annotation language functions and their definitions is given under “Device Annotation Function Reference”.

DFM Property Data Flow

A distinguishing feature of device annotation is the use of DFM properties. This is an outline of how these properties are created and used.

  • Get edges of interest using DFM Space or Device TVF Layer Generation Functions.

  • Collect measurements from the previous step and assign them to layers of interest using DFM Property.

  • Compute device annotation properties using Device TVF Functions in a TVF Function block.

  • Assign properties in a Device or Device Layer operation’s annotation program using DFM Property Data Retrieval Functions and tvf_numeric_function.
