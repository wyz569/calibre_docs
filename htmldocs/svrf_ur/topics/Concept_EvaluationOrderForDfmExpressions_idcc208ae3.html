Evaluation Order for DFM Expressions

A DFM expression can have multiple elements, such as built-in and user defined functions, variables, and math operators. Some operations can have multiple DFM expressions. It is important to understand the order of expression evaluation.

DFM expressions are described in “DFM Expressions”. The evaluation order is the following:

  1. Measurement functions that require iteration over layer geometries are evaluated before expression evaluation begins.

    For example, although the default expression for a per-shape function is not evaluated unless needed, any measurement functions within the default expression are always evaluated before expression evaluation begins.

  2. If multiple DFM expressions occur in an operation, they are evaluated in the order specified. Optional constraints apply to the expression that immediately precedes the expression. Constraints do not affect the order of evaluation.

    Also see “Expression Chains” for rules governing their use.

    When using the *PROPERTY_REF functions, the referenced property must be defined in a previous expression in the same operation. See “Referencing DFM Properties Defined in the Same Operation”.

  3. Parentheses are evaluated next within any set of expression brackets ([]).

  4. Functions, such as built-in math functions, are evaluated next within any set of expression brackets. In nested functions, evaluation begins with the innermost function and progresses to the outermost. Otherwise, functions are evaluated left to right within the expression brackets.

  5. Within any set of expression brackets, unary operators are evaluated next, followed by binary operators, followed by Conditional Expressions. See “Math Operators” for the evaluation order of the binary operators.
