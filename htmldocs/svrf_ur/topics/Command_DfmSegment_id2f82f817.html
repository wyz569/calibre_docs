DFM Segment

Layer operation

License: Calibre YieldEnhancer

Divides an edge layer into segments. In Syntax 1, segmentation is done by count of segments. In the Syntax 2, the edge is divided into alternating primary and gap segments with specified lengths.

Note:

The syntax was changed in the 2020.3 release.

Usage

Syntax 1: Segment by Count

DFM SEGMENT {edge_layer | ‘(’edge_layer‘)’} count

Syntax 2: Segment by Length

DFM SEGMENT {edge_layer | ‘(’edge_layer‘)’} primary_length gap_length
[DISTRIBUTE mode]

Arguments

edge_layer | (edge_layer)

A required edge layer. Alternate segments are output depending on whether the layer name is enclosed in parentheses. See the Description section for details.

count

A required non-negative integer used in Syntax 1. Each edge is divided into count segments. If count is a floating-point number, it is rounded to the nearest integer. See “Segment by Count”.

primary_length gap_length

A required argument pair in Syntax 2. Each edge is divided into alternating primary and gap segments, with lengths primary_length and gap_length, respectively. The lengths are specified as non-negative numbers in user units.

The DISTRIBUTE mode setting specifies how the segments are placed. Depending on the mode setting, the gap_length may be adjusted to achieve uniform placement of the primary segments.

Primary segments are output when edge_layer is specified. Gap segments are output when (edge_layer) is specified.

If gap_length is zero, the output from the primary length segments is a copy of the input edge, and there is no output when (edge_layer) is specified.

DISTRIBUTE mode

An optional keyword set in Syntax 2 that specifies how the alternating primary and gap segments are placed. Primary segments always have length primary_length. If there is remaining length on the input edge after placing primary and gap segments of the specified length, the remaining length is added to gap segments using a method determined by the mode setting. If the input edge length is smaller than primary_length, a gap segment is placed for that edge.

  • 0 — This is the default. Segments are placed starting at the lowest or leftmost point of the edge, starting with a primary segment and alternating with gap segments of length gap_length. If the last segment is a gap segment, it is placed, even if it is shorter than gap_length. If the last segment would be a primary segment shorter than primary_length, it is not placed and the previous gap segment is increased in length until the end of the input edge is reached.

  • 1 — Not used.

  • 2 — Alternating primary and gap segments are centered on the input edge, with the gap segments having a length of gap_length. Any extra length is distributed as equally as possible in the gap segments at each end of the input edge.

  • 3 — Similar to mode 2, but the segments are placed first at each end of the input edge, with extra length added to gap segments in the center of the input edge.

    If only one primary segment can be placed, it is placed in the center of the input edge and extra length is added to the surrounding gap segments. Otherwise, primary segments are placed at each end of the input edge. If there is an odd number of primary segments (implying that the center segment is a primary segment), any extra length is added to the gap segments on either side of the center segment. If there is an even number of primary segments, any extra length is added to the center gap segment.

  • 4 — The specified gap_length is considered a minimum spacing. A fixed spacing g >= gap_length is found for each edge which results in the maximum number of primary segments for that edge. Alternating segments are placed along the edge, with gap segments having a length of g. Any extra length is distributed as evenly as possible in the gap segments at each end of the input edge (as with mode 2).

  • 5 — Similar to mode 4, but segments are placed first at each end of the input edge, as with mode 3. Extra length is handled as with mode 3 and added to gap segments in the center of the edge.

  • 6 — The specified gap_length is considered a minimum spacing. The primary segments are distributed as uniformly as possible along the input edge.

    Two primary segments are first placed at each end of the input edge, then more primary segments are spaced evenly between the two end segments, such that the gaps between primary segments are at least gap_length and the maximum possible number of primary segments is placed. The gaps between primary segments differ by at most 1 dbu, and the longer gap segments are placed in the center of the edge. If only one primary segment can fit, it is centered in the edge.

Description

Divides an edge layer into segments using one of two methods:

  • Segment by Count (Syntax 1)

    Segments the edges on edge_layer into count segments and outputs alternating segments as an edge layer. See “Segment by Count” for details.

  • Segment by Length (Syntax 2)

    Segments the edges on edge_layer into alternating primary and gap segments, each with a specified length. The DISTRIBUTE mode setting determines how the segments are placed and how leftover edge length is handled. See “Segment by Length”.

Operations that differ only in the presence of parentheses around edge_layer  run concurrently.

DFM Segment only processes edges that are orthogonal to the database axes. Non-orthogonal edges are not included in the output. DFM Segment does not preserve connectivity.

Segment by Count

Syntax 1 divides each input edge into count segments. Segmentation starts at the lowest or leftmost point of the edge, and ends at the top or rightmost point. If the edge cannot be divided into count equal segments, the edge is divided as equally as possible, with the top or rightmost segment having a slightly different length than the other segments.

The first segment and every other segment is output when edge_layer is specified; the second segment and every other segment is output when (edge_layer) is specified. 

M_A_N6 = DFM SEGMENT  M_edge  6 // First and alternating segments
M_B_N6 = DFM SEGMENT (M_edge) 6 // Second and alternating segments

See “Example 1”.

Segment by Length

Syntax 2 divides the input edge into alternating primary and gap segments, each with a specified length. The DISTRIBUTE mode setting determines how the segments are placed and how any leftover edge length is handled. For all modes, the primary segments have a length of primary_length. The effect of the mode on the gap between primary segments is summarized in the following table:

DISTRIBUTE mode

Summary

0, 2, and 3

gap = gap_length

The gap segment length is fixed at gap_length, except for adjustments made to gap segments at the right or top end (mode 0), at each end (mode 2), or in the middle (mode 3) to include leftover length.

4 and 5

gap >= gap_length, fixed for each edge

The gap for each edge is chosen to provide the maximum number of primary segments for that edge. Adjustments are made to gap segments at each end (mode 4), or in the middle (mode 5), to include leftover length.

6

gap >= gap_length, adjustable

The primary segments are distributed as uniformly as possible along each input edge. The resulting gap segment length may vary slightly along each edge, but is approximately uniform.

Modes 0, 2, and 3 are useful when both the primary and gap segments should have the specified length. Modes 4, 5, and 6 are useful when uniform spacing of the primary segments is needed with a minimum separation. See the argument description for details about the segment placement.

Examples

Example 1

This example does segmentation by count. Edges of length greater than 2 user units are selected for processing.

Figure 1. DFM SEGMENT by Count

M_edge_long = LENGTH metal1 > 2

// Segment by count
M_A_N6 = DFM SEGMENT M_edge_long 6
M_B_N6 = DFM SEGMENT (M_edge_long) 6
out_M_A_N6 { COPY M_A_N6}
out_M_B_N6 { COPY M_B_N6}

Example 2

This example outputs primary segments of length 1.0 user units, with a minimum spacing of 0.1 between the primary segments. DISTRIBUTE mode 6 is used, meaning the gap between primary segments is adjusted to provide uniform spacing. Only the primary segments are output. 

M_edge_long = LENGTH metal1 > 3

M_P_P1_G01 = DFM SEGMENT M_edge_long 1.0 0.1 DISTRIBUTE 6
