VMAX

Vector property function for DFM expressions

Determines the maximum value in a specified slice of a vector tuple. Additionally can determine the tuple index of a maximum value of a specified slice and use that index to return a corresponding value in another slice, possibly in another vector. Supported only in the DFM Property operation.

Usage

VMAX(vec1,slice1[,slice2])

VMAX(vec1,slice1,vec2,slice2)

Arguments

vec1

A required numeric vector.

slice1

A required non-negative integer that specifies the slice (column) index in vec1 to evaluate. The value of slice1 must be less than the tuple size of vec1.

slice2

A non-negative integer that specifies a slice index from which to return a value corresponding to the maximum value found in slice1. The definition of value correspondence is in the “Description” section. When specified without vec2, slice2 is optional and refers to a slice of vec1. When specified with vec2, slice2 is required and refers to a slice of vec2. The value of slice2 must be less than the tuple size of the vector to which it applies.

vec2

A required numeric vector when using the second syntactical form of the function. When vec2 is specified, a value from slice2 in vec2 that corresponds to the maximum value in slice1 is returned. The definition of value correspondence is in the Description section. The tuple count of vec2 must match vec1.

Return Values

Number.

Description

When used in its simplest form, this function returns the maximum value in slice1 of vec1.

When the slice2 argument is provided in either syntax, the maximum value is first determined in slice1. The index of the vector tuple in vec1 containing that maximum value is noted. (If the maximum value is not unique, then the index of the first tuple in which a value could serve as a maximum is noted.) A value in slice2 of the tuple having the noted index is taken either from vec1 (if vec2 is not used) or vec2. This value in slice2 is then said to correspond to the maximum value from slice1. The value from slice2 is returned by the function.

Therefore, if slice2 is specified without vec2, then the function returns the corresponding value (using the preceding correspondence definition) to the maximum value in slice1.

If slice2 is specified with vec2, then the behavior is similar to the preceding paragraph except that slice2 refers to vec2 rather than vec1.

Examples

Assume vectors V1 and V2, each having two tuples of size 3:

tuple index     0        1
         V1 {<0 1 2> <3  4  5>} 
         V2 {<6 7 8> <9 10 11>}
slice index   0 1 2   0  1  2

The following function evaluations apply:

VMAX(V1,2) = 5 (maximum element of slice 2 in V1; occurs in tuple 1)

VMAX(V1,2,0) = 3 (slice 0 element of tuple 1 in V1; corresponds to “5” of slice 2)

VMAX(V1,2,V2,0) = 9 (slice 0 element of tuple 1 in V2; corresponds to “5” of slice 2 in V1)

These are some corresponding DFM expressions to produce the results:

[ V1 = CONCAT(VECTOR(0,1,2),VECTOR(3,4,5)) ]
[ V2 = CONCAT(VECTOR(6,7,8),VECTOR(9,10,11)) ]
[ "vmax_V1_2"      = VMAX(VPROPERTY_REF(V1),2) ]
[ "vmax_V1_2_0"    = VMAX(VPROPERTY_REF(V1),2,0) ]
[ "vmax_V1_2_V2_0" = VMAX(VPROPERTY_REF(V1),2,VPROPERTY_REF(V2),0) ]
