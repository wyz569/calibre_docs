Conditional Expressions

A conditional expression is a way to specify an if/then expression within a DFM expression—it returns one result if the specified condition is true and another result if the condition is false. You can use conditional expressions in the DFM expressions within DFM Analyze, DFM Spec Fill Optimizer, DFM Property and its related operations, and DFM Function.

Usage

(condition) ? if-expression : else-expression

Arguments

  • (condition)

    A required logical expression that must include a relational operator. The supported relational operators are <, <=, ==, >=, >, and !=. The condition must be enclosed in parentheses.

  • if-expression

    A required argument that must be a numeric constant or a valid DFM expression. The expression may be a conditional expression. The if-expression is evaluated and is the result of the conditional expression if the condition is true.

  • else-expression

    A required argument that must be a numeric constant or a valid DFM expression. The expression may be a conditional expression. The else-expression is evaluated and is the result of the conditional expression if the condition is false.

Description

The following rules apply to conditional expressions:

  • Spaces must separate the ? and : symbols from the if-expression and else-expression unless the expressions are enclosed in parentheses.

  • Within the condition, the logical “and” (&&) and “or” (||) operators can be used to build a logical expression from several relations. See “Math Operators”.

  • Within the condition, the unary negation operator (!) can be used to negate a logical expression.

  • The precedence of the ? : operator is lower than that of the arithmetic operators. For example, this expression:

    (x > 1) ? x - 1 : x + 1

    evaluates to either (x - 1) or (x + 1).

  • Nested conditional expressions are allowed, and the inner conditional expression is evaluated first. For example, this expression:

    (x > 0) ? (y > 0) ? x + y : x - y : (y > 0) ? y - x : -y - x

    evaluates to the sum of absolute values of x and y.

    While the preceding expression is allowed, it is better practice to include parentheses to insure the proper grouping of conditional expressions.

  • Both the if-expression and else-expression can be any valid DFM expression and can include calls to math functions, SUM and PROD functions, measurement functions such as AREA(layer), and user-defined functions specified with DFM Function.

    Note:

    When an expression (if-expression or else-expression) includes a per-shape function such as SUM or PROD, the argument to the per-shape function is evaluated for every geometry, regardless of whether or not the result is needed to evaluate the conditional expression.

Conditional expressions are useful in performing comparisons of non-numeric properties, such as netIDs, strings, and vectors, because the property constraint cannot be used for non-numeric properties. See “Comparison of DFM Properties”.

Example 1

Layer A is a polygon layer. The property big is attached to each output polygon, where big has the value 1 if the area is greater than one, and the value 0 otherwise. 

A_area = DFM Property A 
				[ big = ( AREA(A) > 1) ? 1 : 0 ]

Example 2

Suppose you have this DFM Function:

DFM FUNCTION [ F(NUMBER X) TABLE 
    { 0.0 1.00 0.1 0.90 0.2 0.82 0.3 0.74 0.4 0.67 0.5 0.61 
      0.6 0.55 0.7 0.50 0.8 0.45 0.9 0.41 1.0 0.38 } ]

While the function accurately interpolates the table for all values of X in the range between 0 and 1, outside of this range it extrapolates the function and is unlikely to give the desired results.

Suppose that for x greater than 1 the expression exp(-x) provides a sufficiently accurate approximation to the desired function, while for negative x the function should return 0. This function can be expressed using a conditional expression:

DFM FUNCTION [ G(NUMBER X) (X < 0) ? 0 : ( (X > 1) ? EXP(-X) : F(X) ) ]

Example 3

This DFM Function defines a function F(X,L) where X is a numeric argument and L is a layer name. The value of the function F(X,L) is the result of the measurement function AREA(L) if X equals 1, and is the result of the measurement function LENGTH(L) for all other values of X. 

DFM FUNCTION [ F(NUMBER X, LAYER L) (X==1) ? AREA(L) : LENGTH(L) ]

Note that the result of the measurement functions (and the conditional expression itself) depends on the context in which the expression is used. For example, if used in a DFM Analyze expression, the function F(X,L) returns the sum of the area or length for all geometries in each DFM Analyze window. However, if F(X,L) is used in a DFM Property operation where L is the primary layer, the function returns the area or length for each geometry.
