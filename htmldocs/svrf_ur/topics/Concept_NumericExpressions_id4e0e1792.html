Numeric Expressions

A numeric expression can specify any numeric parameter in any layer operation. Numeric expressions can also be used to define variables in a Variable specification statement. Numeric expressions may be used as numeric parameters in many specification statements and in functions used by various built-in languages used within SVRF statements.

A numeric expression is a combination of numeric constants, numeric variables, and the operators shown in order of precedence from highest to lowest in the following table.

Table 1. Numeric Operators in Order of Precedence

Operator

Operator Type

Definition

( )

grouping

evaluates the numeric value of the enclosed expression

+ - ! ~

unary

positive, negative, negation, logical complement

* / ^ %

binary

multiplication, division, power, modulus

+ -

binary

addition, subtraction

Operators of equal precedence occurring within corresponding parentheses are evaluated in the order they appear from left to right. Nested pairs of parentheses are evaluated from the innermost pair and working outward.

In this example, “3 * (GRID + 6)” is a numeric expression:

rule { EXT metal <  3 * ( GRID + 6 ) } // GRID is a numeric variable

The sub-expression inside the parentheses is evaluated before the multiplication in this case.

SVRF numeric expressions may contain the functions shown in the following table.

Table 2. Math Functions Supported in SVRF Numeric Expressions

Function

Definition

CEIL(x)

Returns the nearest integer not less than the given value.

FLOOR(x)

Returns the nearest integer not greater than x.

TRUNC(x)

Returns the nearest integer not greater in magnitude than x.

SQRT(x)

Computes the square root of x.

ABS(x)

Computes the absolute value of x.

EXP(x)

Returns e raised to the power of x.

LOG(x)

Computes the natural logarithm of x (to base e).

SIN(x)

Computes the sine of x radians.

COS(x)

Computes the cosine of x radians.

TAN(x)

Computes the tangent of x radians.

MIN(expression1,expression2)

Returns the lesser value of the two expressions.

MAX(expression1,expression2)

Returns the maximum value of the two expressions.

These operators and functions are supported as a part of the SVRF language, but the entire set may not be supported in all tool-specific languages referenced by SVRF statements.
