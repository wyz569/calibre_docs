PEX Netlist Auto_LPE

Parasitic extraction

Controls whether or not to include the LPE properties in the netlist.

Usage

PEX NETLIST AUTO_LPE {ON | OFF}

Arguments

ON

Required keyword that causes the formatter to include the LPE properties in the netlist. This is the default behavior if you do not specify this statement and the netlist format is CalibreView.

OFF

Required keyword that suppresses the LPE properties from the netlist. This is the default behavior if you do not specify this statement and the netlist format is HSPICE, DSPF, SPEF, SPECTRE, ELDO, or SPICE.

Description

This optional statement controls whether or not to include the LPE properties in the netlist. This statement may be specified once in your rule file.

The default LPE property value depends on the specified extraction mode as shown in the following table.

Table 1. Default LPE Values

  Extraction Mode

Default LPE Value

              C

               1

              R

               2

             RC

               3

            RCC

               3

              L

               0

          Simple

               0

Extraction modes without R or C have a default LPE value of 0.

Examples

Example 1

If you are extracting a CalibreView netlist, specify the following statement in your rule file if you do not want LPE properties in your netlist: 

PEX NETLIST AUTO_LPE OFF

Example 2

If you are extracting an HSPICE netlist, specify the following statement in your rule file to include LPE properties in your netlist: 

PEX NETLIST AUTO_LPE ON
