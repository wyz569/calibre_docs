PROPERTY

Device annotation language function.

Defines numeric properties to be computed.

Usage

PROPerty prop_name_1[, prop_name_2 …]

Arguments

prop_name_1

Required name of a property for which to calculate an annotated value. More than one property name is specified in a comma-delimited list.

Return Values

None.

Description

This statement is required and, in the absence of the DEBUG or SELECT statements, must be the first statement in a property computation. More than one PROPERTY statement may be used in a property annotation program. If more than one is used, they must all be at the start of the program.

Property names are treated as variables within computational statements. The final value assigned to the property name is the value of the DFM property associated with the instance.

Property names must begin with a letter character followed by any permitted SPICE characters. See “SPICE Syntax Conventions” in the Calibre Verification User’s Manual for more details. The rule file compiler will allow you to specify property names that are not SPICE-compliant, so care should be taken in choosing names. The Calibre nmLVS SPICE reader will complain about disallowed property names that appear in a netlist generated from bad property names in the rule file.

Examples

See “Example Device Annotation Programs Using DFM Property”.

This shows a program with conditional compilation. All the PROPERTY commands must appear first in the program.

DEVICE MN(nfet) NGATE NGATE NSD NSD BULK
[
    PROPERTY W, L
#ifdef ADD_ASAD
    PROPERTY AS, AD
#endif
#ifdef ADD_PSPD
    PROPERTY PS, PD
#endif
    W = perimeter_co(NGATE,NSD) * 0.5
    L = perimeter_outside(NGATE,NSD) * 0.5
#ifdef ADD_ASAD
    AS = AREA(S)
    AD = AREA(D)
#endif
#ifdef ADD_PSPD
    PS = PERIMETER(S)
    PD = PERIMETER(D)
#endif
]
<gate_prop>
[
#ifdef CCI_FLOW_1
    PROPERTY p1
#endif
#ifdef CCI_FLOW_2
    PROPERTY p2
#endif
#ifdef CCI_FLOW_1
    p1 = DFM_NUM_VAL( gate_prop, p1_val )
#endif
#ifdef CCI_FLOW_2
    p2 = DFM_NUM_VAL( gate_prop, p2_val )
#endif
]

Any of the conditionals ADD_ASAD, ADD_PSPD, CCI_FLOW_1, and CCI_FLOW_2 may be activated. Even if none of them are, the (empty) annotation program is still valid.
