NETVPROPERTY

Property access function for DFM expressions

Accesses DFM properties of type vnetID (vector of net IDs) from objects on the input layer. Supported only in DFM Property and DFM Function.

Usage

NETVPROPERTY(layer, property_name[, ordinal])

Arguments

layer

A required layer containing the layout objects to which the property is attached.

If layer is a secondary layer in a DFM Property operation and ordinal is not specified, the returned property is a concatenation of the vnetID properties on all shapes clustered with the primary layer objects.

property_name

A required argument specifying the name of the property of interest. The property must be a vnetID property.

ordinal

An optional argument used when layer is a secondary layer in a multilayer DFM Property operation. The ordinal specifies the index of the secondary layer geometry whose property value is to be returned. The ordinal must be a positive integer value and can itself be an expression. See the “Description” section for details on behavior and restrictions. If ordinal is not specified when layer is a secondary layer, the vector properties from all clustered secondary layer objects are concatenated. (This is the standard behavior for DFM Property with vector properties on secondary layer objects.)

Note:

When it is known that only one secondary layer object clusters with the primary layer object, it is more efficient to specify NETVPROPERTY(sec_layer, prop_name, 1) than NETVPROPERTY(sec_layer, prop_name). The two functions have the same effect, but the function without the ordinal copies data, which uses more memory and more processor time doing the copy.

Return Values

vnetID

Description

Reads DFM properties of type vnetID from objects on the input layer. vnetID properties are created with the VNETID function.

The vector of net IDs and the individual net IDs returned by NETVPROPERTY can be compared within the context of the cell in which the property evaluation takes place. When a vnetID property is written to an object during a DFM Property operation, Calibre also keeps track of the cell context. If NETVPROPERTY is used to read the vnetID property as part of a DFM Property operation in a different cell context, Calibre makes the appropriate translation for the netID values so that they are meaningful in the current cell context. Also see “Cell-Context-Dependent Property Functions Results in Hierarchical Batch Runs”.

If the BY CELL keyword is used in the DFM Property operation, the vnetID property for the cell is a concatenation of the vnetID properties for all geometries in the cell. Similarly, if the BY NET keyword is used, the vnetID property for the net is a concatenation of the vnetID properties for all geometries on the net.

Restrictions

The following restrictions apply to the DFM Property operation in which the NETVPROPERTY function is used:

  • The INSIDE OF keyword may not be used.

  • The OVERLAP or NODAL keyword must be specified for multilayer DFM Property operations; in other words, the default clustering method of INTERSECTING is not allowed.

  • A constraint cannot be specified for a property of type vnetID. The equality (==) and inequality (!=) operators can be used within a conditional expression to compare two properties of type vnetID. See the “Examples” section and “Comparison of DFM Properties”.

Behavior with Ordinal

The ordinal argument is used to read a property value from a specific object on a secondary layer in a multilayer DFM Property operation. The clustering method must be OVERLAP or NODAL.

Layout objects on secondary layers are numbered beginning with 1. The order in which the layout objects are processed is arbitrary, however, each secondary layer object is assigned a unique ordinal that is consistent within the same DFM Property operation.

  • For ordinal = 1 through the total number of clustered geometries on the secondary layer, the function call returns the value of the specified property on the specified secondary layer geometry clustered with the primary layer geometry.

  • Multiple function calls in the same DFM Property operation that reference the same layer have a consistent order for the secondary layer objects. For example, PROPERTY(S, “X”, 1) and VPROPERTY(S, “Y”, 1) return property values from the same geometry when used in the same DFM Property operation.

  • If the ordinal value is out of range (either greater than the number of clustered geometries or less than 1), the function fails to evaluate.

Examples

LAYER layer1 1
LAYER layer2 2
LAYER layer3 3
LAYER layer4 4
CONNECT layer1 layer2
CONNECT layer3 layer4

// write globalnetids for primary layer
gnetid_layer = DFM PROPERTY layer1 
					[gnet_v = VNETID(GLOBALNETID(layer1))] 

test_v_eq  = DFM PROPERTY layer3 gnetid_layer OVERLAP  
  			[netvprop_eq = ( NETVPROPERTY(gnetid_layer, gnet_v, 1) == 
	 				NETVPROPERTY(gnetid_layer, gnet_v, 2)) ? 1 : 0 ]  == 1
