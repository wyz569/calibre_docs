Layout Preserve Case

Specification Statement

Specifies whether matching net or device property names that differ only by case are treated as identical or different during circuit extraction. Used only by Calibre nmLVS/Calibre nmLVS-H, Calibre PERC, and certain Calibre YieldAnalyzer applications.

Usage

LAYOUT PRESERVE [NET | PROPERTY] CASE {NO | YES}

Arguments

NO

Keyword that preserves the case of layout net names but treats names that differ only by case as identical during circuit extraction. This is the default behavior if this statement is not specified in the rule file. This keyword also causes non-built-in Device property names to be written in lowercase regardless of any PROPERTY specification in another instance of this statement.

YES

Keyword that specifies layout net names must match exactly, including case, to be considered identical when NET is also specified, or if NET is accepted as the default. If PROPERTY is specified, then the case of non-built-in Device property names is preserved when they are written.

NET

Optional keyword that specifies the statement applies to layout net names. This is the default behavior. If specified, this keyword must appear as shown in the Usage line. This keyword can also be used by certain Calibre YieldAnalyzer applications. See “Performing Net-Based CAA” in the Calibre YieldAnalyzer and YieldEnhancer User's and Reference Manual.

PROPERTY

Optional keyword that specifies the statement applies only to non-built-in Device property names. If specified, this keyword must appear as shown in the Usage line.

Description

This statement controls how layout text names are handled during circuit extraction. It also controls how non-built-in Device properties are written to the extracted layout netlist, the SVDB, all results generated by the Query Server, and so forth.

Each NET or PROPERTY keyword may appear in at most one Layout Preserve Case statement.

Remember that all LVS comparison and matching of non-string property names, such as property names in Trace Property statements, are always case-insensitive because Layout Preserve Case only affects how property names are written, not how they are matched. For example, consider the following rule file statements:

DEVICE R RES MET1 MET1 [ PROPERTY Val = 1 ] 
LAYOUT PRESERVE PROPERTY CASE YES
TRACE PROPERTY RES val VAL 0

The extracted netlist has a property named “Val” on all resistors, for example:

R1 1 2 Val=1 $X=300 $Y=600 $D=1

The Trace Property statement causes LVS to consider this layout property “Val” equivalent with a source property “val”, because matching of non-STRING property names is always case-insensitive. Tracing of string properties can be made case-sensitive.

The Net and Not Net layer operations are case-sensitive when Layout Preserve [Net] Case YES is specified.

For a complete discussion of the selection of extracted layout netlist names, see “Selection of Extracted Netlist Names” in the Calibre Verification User’s Manual.

For case-sensitive netlisting of user-defined devices that have the same number of pins, see Layout Case.

Examples

Example 1

In the following example, the same net is named “abc” in one location and “ABC” in another location. This is not considered a short circuit and the net name in the extracted SPICE netlist is either “abc” or “ABC,” chosen arbitrarily. Alternatively, the names “abc” and “ABC” appearing on different nets are treated as an open circuit, and one of the names is removed.

LAYOUT PRESERVE CASE NO

Example 2

In the following example, the same net is named “xyz” in one location and “XYZ” in another location, and is considered a short circuit. These same names appearing on two different nets are not considered an open circuit and both names appear in the extracted netlist.

LAYOUT PRESERVE CASE YES

Example 3

In this example, net names are treated in a case-insensitive manner during circuit extraction, but the case of Device property names is preserved when they are written:

// Layout Preserve [NET] Case NO is in effect by default.
LAYOUT PRESERVE PROPERTY CASE YES

This is the same as specifying either:

LAYOUT PRESERVE CASE NO // NET is assumed by default
LAYOUT PRESERVE PROPERTY CASE YES

or:

LAYOUT PRESERVE NET CASE NO
LAYOUT PRESERVE PROPERTY CASE YES
