DFM Database

Specification statement

Specifies the path to the DFM database directory or the parent directory of a set of DFM databases.

Usage

DFM DATABASE [DIRECTORY] directory [OVERWRITE [REVISIONS]]
‘[’{{{[CONNECT] [DEVICES] [PINLOC]} | [NOCONNECT]} [ORIGINAL] [ANALYZE] [MEASURE] [PROPERTY] [DENSITY] [ALL]}‘]’
[ANNOTATE ‘[’ “name” = “value” ‘]’] …

Arguments

directory

A required argument that specifies the path to the output directory. By default, the directory is the DFM database. If DIRECTORY is specified, then the directory is the parent directory of the DFM database subdirectory.

DIRECTORY

An optional keyword that specifies the path to the parent directory of the DFM database. The DFM database is created as a subdirectory within that parent directory and is named “topcell.dfmdb”. If Layout Primary “*” is specified in the rule file, the name of the matched top cell is used for topcell. If Layout Primary “name” is specified, the primary cell name is used. If the directory does not exist, it is created. If the directory already exists, it is not modified except to create or overwrite the DFM database subdirectory.

OVERWRITE [REVISIONS]

An optional keyword set that specifies to overwrite the DFM database directory if it already exists. If the given directory already exists and the OVERWRITE argument is not provided, an error is issued.

  • REVISIONS — An optional keyword that specifies to overwrite a DFM database directory with revisions. By default, OVERWRITE does not overwrite the DFM database directory if it contains revisions other than the primary revision.

‘[’ {{[CONNECT] [DEVICES] [PINLOC]} | [NOCONNECT]}  [ORIGINAL] [ANALYZE] [MEASURE] [PROPERTY] [DENSITY] [ALL] ‘]’

An optional set of keywords that specify what information to save to the DFM database. Literal brackets ([ ]) must enclose the keyword set.

 

Keyword

Definition

CONNECT

Saves all Connect layers to the DFM database. This is the default behavior when the rule file does not contain a DRC Incremental Connect YES statement. If the DRC Incremental Connect YES statement is specified, then CONNECT must be explicitly specified in order for connectivity layers to be output. Specifying CONNECT in this case can result in a large number of database revisions.

DEVICES

Writes device seed layers to the DFM database. Including this argument causes device extraction to be performed. Device statements must be present in the rule file for this keyword to have any effect.

PINLOC

Writes device pin location information to the DFM database. This keyword implies DEVICES behavior as well.

NOCONNECT

Specifies not to save all Connect layers to the DFM database. This is the default behavior when the rule file contains a DRC Incremental Connect YES statement. Connect layers can still be saved by other means, such as with DFM RDB or Copy statements, or as input layers to operations that appear as layer selectors in the DFM Database statement. NOCONNECT cannot be specified with CONNECT but can be specified with the ALL keyword.

The NOCONNECT keyword is suggested to save disk space.

ORIGINAL

Writes original (merged) layers to the DFM database. Only those original layers that are used for output to the DFM database are saved. ORIGINAL does not cause layers to be output if they are not required for the run.

ANALYZE

Saves the input layers to all DFM Analyze operations

MEASURE

Saves the input layers to all DFM Measure operations.

PROPERTY

Saves the input layers to all DFM Property operation

DENSITY

Saves the input layers to all Density operations

ALL

Equivalent to specifying DEVICES, ORIGINAL, ANALYZE, MEASURE, PROPERTY, DENSITY, and CONNECT (although not when NOCONNECT is specified)
ANNOTATE ‘[’ name = “value” ‘]’

An optional keyword set that annotates the DFM database with metadata. The brackets ([ ]) are required.

  • name — A string, which can be quoted or unquoted and need not be unique.

  • “value” — A quoted string.

You can specify this keyword set multiple times. Each occurrence adds another annotation (name and “value” pair) to the database.

Description

Specifies the location and contents of the DFM database. The output layers of selected rule checks are always saved to the DFM database. Connect layers are usually saved by default when connectivity extraction is performed (DRC Incremental Connect YES introduces an exception to the default). Layers that are not required for the run are not output.

Maximum result counts and maximum vertex counts are ignored when saving layers to the DFM database.

The DIRECTORY keyword is useful when you want to store multiple DFM databases in a common directory, where the databases are differentiated by primary cell name.

A DFM database is frequently used for post-processing in Calibre YieldServer. The arguments of the DFM Database statement are chosen to facilitate such post-processing. Refer to “Working with DFM Databases” in the Calibre YieldServer Reference Manual for more information on DFM databases.

Annotations assigned with the ANNOTATE keyword are accessible in Calibre YieldServer through annotation commands such as dfm::list_annotated_layers and dfm::list_annotation_names.

Examples

Example 1

// Specifies the path to the DFM database as /tmp/dfmdb,
// and overwrites the database if it already exists, even if
// it contains revisions other than the primary revision.
DFM DATABASE "/tmp/dfmdb" OVERWRITE REVISIONS

Example 2

// Specifies the path to the DFM database as ./full.dfmdb,
// and annotates it with two name/value pairs.
DFM DATABASE "full.dfmdb" OVERWRITE
ANNOTATE [ Purpose = "Complete run with all layers" ]
ANNOTATE [ Phase = "1" ]

Example 3

// Specifies the path to the DFM database parent directory
// as "/tmp/sample_design". The DFM database is created as a
// subdirectory within the parent directory with the naming
// convention "<topcell>.dfmdb".
DFM DATABASE DIRECTORY "/tmp/sample_design" OVERWRITE 
    [CONNECT DEVICES ORIGINAL]
