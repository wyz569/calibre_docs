PERC LDL Enable Cell Based Flow

Specification statement

Specifies whether to enable the GDS cell-based flow (likewise, OASIS) in Calibre PERC LDL runs.

Usage

PERC LDL ENABLE CELL BASED FLOW {NO | YES}

Arguments

NO

A keyword that specifies the GDS cell-based flow is disabled in LDL runs. This is the default.

YES

A keyword that specifies the GDS cell-based flow is enabled in LDL runs.

Description

Specifies whether to use the GDS (likewise, OASIS) cell-based flow in LDL runs. This is similar in certain respects to the LEF/DEF cell-based flow discussed under “Cell-Based Checks” in the Calibre PERC User’s Manual. The -soc command line option is not used with this statement.

Using a cell-based flow forgoes device extraction. Therefore, a clean LVS with devices present is not a prerequisite for running LDL on cell placements. Also, since the device modules are skipped during circuit extraction, this can result in a performance benefit.

In order to use the GDS cell-based flow, you must ensure that the ports of the cells of interest are labeled and that the Port Layer Text statement specifies the port text layers. The checked cells must be specified as hcells or referenced through the Layout Preserve Cell List statement. LVS Ground Name and LVS Power Name must also be specified, which is typical of LDL flows generally.

When YES is specified, device extraction is disabled, PERC LDL Preserve Trivial Pins ALL is enabled, and all empty subcircuits are preserved in the extracted layout netlist for topological analysis. These conditions facilitate checking of connectivity to any desired cell port (where the cell acts as an hcell).

The automatic extraction of trivial pins can cause unexpected outputs when the rule file code anticipates connections between cell instance pins and ports without actually verifying in some way that such connections exist. This verification can be performed by selecting pins with an appropriate net or path type, or by guaranteeing a pin has the name of a net known to connect to a port.

Examples

Following is a P2P rule setup for using the GDS cell-based flow. The DEBUG flag in the esd function can be enabled if there is uncertainty about port labeling. A similar flag is used in the execute_ldl_checks function to sanity check net resistances.

LAYOUT SYSTEM GDS // not LEFDEF for this flow
LAYOUT BASE LAYER cont diff poly
PERC LDL LAYOUT REDUCE TOP LAYERS m4 m5 m6 m7 m8 m9

// required for LDL GDS cell-based flow 
// ------------------------------------
PORT LAYER TEXT metal2 metal1 "port"

LVS GROUND NAME "gnd"
LVS POWER NAME "pwr?"

HCELL CellA CellA
HCELL nand  nand

PERC LDL ENABLE CELL BASED FLOW YES
// ------------------------------------
PERC LOAD esd INIT init SELECT rule_1

TVF FUNCTION esd [/*
package require CalibreLVS_PERC

# all top level ports are in the PORTS net type
  proc init { } {
    perc::define_net_type PORTS { lvsTopPorts }
# control debug comments to transcript
    set ::DEBUG 0
  }

  proc rule_1 { } {
    perc::check_device -type {CellA nand} -instanceOnly \
      -condition export_pins
  }

# Exports top-level ports and cell pins. 0 ohms is the minimum
# reported value.
# Set DEBUG to 1 in the global scope to enable transcript messages.
  proc export_pins {inst} {
    puts "-----DEBUG setting: \"$::DEBUG\""
    if {$::DEBUG eq 1} {
      puts "-----EXPORT FROM: [perc::name $inst -fromTop]\
                             ([perc::type $inst])"
      puts "            PINS:"
    }
    set inst_pins [perc::get_pins $inst]
    while {$inst_pins ne ""} {
      set pin_name [perc::name $inst_pins]
  # if the pin is not connected to a port, skip it
      if {[perc::is_pin_of_net_type $inst $pin_name "!PORTS"] } {
        perc::inc inst_pins
        continue
      }
  # if unsure about pin labels, turn DEBUG on
      if {$::DEBUG eq 1} {
        if {[string match \[a-zA-Z\]* $pin_name]} {
          puts "                 $pin_name"     
          perc::export_pin_pair [list lvsTopPort port $inst $pin_name ] \
            -p2p "0"
        } else {
          puts "                 Skipping unnamed pin: $pin_name"
        }
      } else {
        if { [ catch { perc::export_pin_pair \
                       [list lvsTopPort port $inst $pin_name] \
                       -p2p "0" } errMsg ] } {
           puts "-----Problem with pin ${pin_name}: $errMsg"
        }
      }
      perc::inc inst_pins
    }   
    return 0
  }*/]
