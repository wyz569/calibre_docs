Comments

Comments are permitted in the rule file to annotate the source code. Two slash characters (//) start a comment that terminates at the end of the line on which they occur. Multiline C-style comments (/* … */) are also permitted.

A rule check comment is preceded by the “at” symbol (@). This symbol starts a comment that terminates at a new line and appears within a rule check block. These comments are used as debugging aids and appear as comments in Calibre RVE and Pyxis Layout when you view rule check results in these interfaces. Variables can be resolved within a rule check comment by placing a caret character (^) in front of the variable name; see “Variables in Rule Check Comments” with the Variable command reference.

Calibre RVE for DRC also recognizes certain rule check comments as configuration commands for highlighting and display preferences; see “DRC Rule Check Comments for Calibre RVE” in the Calibre RVE User’s Manual.

See “Pre-Processor Directives” for information on compilation order of comments.
