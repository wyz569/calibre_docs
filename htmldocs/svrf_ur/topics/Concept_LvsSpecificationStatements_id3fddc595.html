LVS Specification Statements

This section lists the LVS specification statements. These are specification statements used in layout versus schematic checking.
Table 1. Specification Statements: LVS

Statement

Description

Device

Defines devices to be recognized and used in LVS comparison. Not a specification statement per-se, but used by LVS processes.

ERC TVF

Generates device properties for downstream simulation tools.

Hcell

Specifies a pair of subcircuits—one from layout and one from source—that correspond.

Layout Dummy Layer

Specifies dummy fill layers for optimized handling during hierarchical database construction.

Layout Preserve Case

Specifies whether matching net or device property names that differ only by case are treated as identical or different during circuit extraction.

LVS Abort On ERC Error

Specifies whether LVS terminates when ABORT LVS keywords are specified in ERC Select Check statements. Used only in Calibre nmLVS/Calibre nmLVS-H and Calibre PERC applications.

LVS Abort On Softchk

Specifies whether to abort processing if a Softchk violation is found.

LVS Abort On Supply Error

Specifies whether to abort processing when connectivity extraction detects a problem with the power or ground nets. Not used in ICverify.

LVS Abort On Supply Short

Specifies whether to abort processing when connectivity extraction detects a supply net short.

LVS All Capacitor Pins Swappable

Specifies whether capacitor pins should be considered swappable during LVS comparison. In ICverify, sets the value of application variable lvs_all_capacitor_pins_swappable.

LVS Annotate Devices

Specifies to annotate extracted device shapes with properties stored on an annotation layer.

LVS Apply Unused Filter

Specifies to apply an unused device filter to a set of cells specified in a list.

LVS Auto Expand Hcells

Specifies whether to expand hcells automatically during hierarchical circuit extraction in order to optimize performance.

LVS Black Box Port

Defines port objects for the LVS Box BLACK statement.

LVS Black Box Port Depth

Defines the hierarchical levels from which ports are recognized in LVS Box BLACK cells.

LVS Box

Specifies cells with contents that are not to be compared.

LVS Box Cut Extent

Specifies how layers derived using Extent-based operations are handled in LVS Box BLACK and GRAY flows.

LVS Box Peek Layer

Specifies original layers in LVS Box BLACK and GRAY cells that can interact with and participate in layer derivations with layers in their super-hierarchies.

LVS Builtin Device Pin Swap

Specifies whether LVS should apply default pin swappability rules in built-in M and R devices.

LVS Builtin MOS NRD_NRS

Specifies whether NRD/NRS properties are treated as built-in during parallel MOS reduction.

LVS Cell List

Specifies a list of cells for overrides of LVS rules. Used only in Calibre nmLVS-H and Calibre PERC.

LVS Cell Supply

Controls propagation of supply net names from lower-level nets. Used only in Calibre nmLVS‑H and Calibre PERC.

LVS Center Device Location

Identifies centered device location coordinates.

LVS Center Device Pins

Identifies centered pin location coordinates for Calibre CCI.

LVS Check Port Names

Specifies whether to check the names of matched ports.

LVS Clone By LVS Box Cells

Specifies whether to clone subcells of regular LVS Box cells that are also placed outside box cells.

LVS Compare Case

Controls case sensitivity in the comparison of names.

LVS Compare Case Strict

Controls enforcement of SPICE netlist case-sensitivity settings in the rule file.

LVS Component Subtype Property

Specifies the property name to be used in determining component subtypes for source instances. Sets the value of the ICtrace lvs_component_subtype_property variable.

LVS Component Type Property

Specifies property names to be used in determining component types for source instances. Sets the value of the ICtrace lvs_component_type_property variable.

LVS Configure Unused Filter

Specifies a cell-specific unused device filter.

LVS Cpoint

Specifies a correspondence point between a layout net and a source net.

LVS DB Layer

Specifies layer names to be merged and saved in a persistent hierarchical database (PHDB).

LVS DB Connectivity Layer

Specifies connectivity layers to be merged and saved with their nodal information in a persistent hierarchical database.

LVS Device Type

Specifies instances having non-built-in device names to be treated as instances of built-in devices in ERC and LVS comparison. Devices are not renamed by this statement.

LVS Discard Pins By Device

Specifies whether rule file Device operations should be examined in order to discard extra pins from input devices during circuit comparison.

LVS Downcase Device

Specifies to use lowercase representations of Device element names, model names, and pin names.

LVS EDDM Process M

Specifies that ICtrace apply M properties to other properties in an EDDM database.

LVS Exact Subtypes

Specifies whether circuit comparison uses both device type and subtype in the initial match.

LVS Exclude Hcell

Specifies hcells to ignore in all circumstances. These hcells are also ignored by the Query Server.

LVS Execute ERC

Specifies whether ERC operations are executed during the circuit extraction phase.

LVS Expand Ambiguously High Shorted Hcells

Specifies whether to expand high-shorted hcells that are matched in a many-to-one configuration and that cause false LVS comparison discrepancies.

LVS Expand On Error

Specifies whether to examine LVS comparison discrepancies and, in certain cases, to automatically perform additional comparison runs with fewer hcells.

LVS Expand Seed Promotions

Specifies whether to expand hcells in LVS comparison that were subject to seed promotion during hierarchical device recognition.

LVS Expand Unbalanced Cells

Specifies whether to expand hcells that are instantiated a different number of times in the source and layout.

LVS Filter

Filters out source and layout instances having properties with the specified values.

LVS Filter Unused Bipolar

Specifies whether to filter unused bipolar transistors.

LVS Filter Unused Capacitors

Specifies whether to filter unused capacitors.

LVS Filter Unused Diodes

Specifies whether to filter unused diodes.

LVS Filter Unused MOS

Specifies whether to filter unused MOS transistors.

LVS Filter Unused Option

Controls the filtering of unused devices.

LVS Filter Unused Resistors

Specifies whether to filter unused resistors.

LVS Flatten Inside Cell

Specifies whether to flatten the internal hierarchy of cells.

LVS Global Layout Name

Specifies to report inconsistent connections that involve global layout signals.

LVS Globals Are Ports

Specifies whether global nets are treated as ports of the top-level cell.

LVS Gray Box Not Port

Specifies Connect or Sconnect layers that cannot form pins in LVS Box GRAY cells.

LVS Ground Name

Specifies a list of ground net names.

LVS Hcell Report

Specifies that Calibre nmLVS-H generate a report giving details about hcell correspondences used during the run.

LVS Ignore Device Pin

Specifies whether LVS comparison should ignore certain pins of a specified device.

LVS Ignore Ports

Specifies whether the LVS comparison algorithm should ignore ports.

LVS Ignore Trivial Named Ports

Controls the handling of ports to lower-level cells that are not connected to any devices and that have corresponding names elsewhere in the design.

LVS Inject Logic

Specifies whether to perform logic injection.

LVS Isolate Shorts

Specifies whether to enable text short isolation.

LVS Layout Disable High Shorts Cell List

Disables high-shorting of cell pins in specified cells.

LVS Map Device

Specifies to map original device component types and subtypes to different ones.

LVS Maximum Short Results

Specifies the maximum number of isolated short paths output to the shorts database.

LVS MOS Swappable Properties

Specifies swappable, user-defined source and drain pin properties.

LVS Netlist All Texted Pins

Specifies whether to netlist internally-floating, texted pins.

LVS Netlist Allow Inconsistent Model

Specifies to allow certain MOS devices to use model names that are inconsistent with Device statement conventions. Use of YES is discouraged.

LVS Netlist Box Contents

Specifies whether to write internal contents of layout LVS Box cells to the output netlist.

LVS Netlist Builtin Property Conversion

Specifies how properties for built-in devices are written by the Calibre SPICE and Calibre Connectivity Interface netlisters.

LVS Netlist Comment Coded Properties

Specifies whether the circuit netlister should use the comment-coded parameters $A and $P (for capacitors) or $W and $L (for bipolar transistors).

LVS Netlist Comment Coded Substrate

Specifies whether the circuit netlister should use the comment-coded parameter $SUB to represent substrate pins in three-pin diodes, capacitors, and resistors, and four-pin bipolar devices.

LVS Netlist Connect Port Names

Specifies whether the circuit extractor should connect port top-level names to nets in the top-level cell when the port names (from Port Layer Text) differ from any net name or port name in the cell.

LVS Netlist Internally Floating Pins

Specifies whether internally floating cell pins are written to the extracted layout netlist.

LVS Netlist Property

Specifies Device properties that behave as LVS-actionable properties during netlist extraction.

LVS Netlist Unnamed Box Pins

Specifies whether the circuit netlister includes certain unnamed pins of LVS Box cells in the output netlist.

LVS Netlist Unnamed Isolated Nets

Controls preservation of unnamed, isolated, layout nets in the extracted layout SPICE netlist.

LVS NL Pin Locations

Controls the flat layout netlist created by the -nl command line option. Instructs flat applications to output pin locations and related information for user defined devices in the -nl netlist.

LVS Non User Name

Specifies instance, net, or port names (except power or ground names) that should not be treated as user-given names, even if they otherwise fit the criteria for user-given names.

LVS Out Of Range Exclude Zero

Specifies how Trace Property … ABSOLUTE tolerance warnings are handled when property values are zero.

LVS Pathchk Maximum Results

Specifies an upper limit on the number of polygons output by ERC Pathchk statements and Pathchk operations.

LVS Pin Name Property

Specifies prioritized list of zero or more ICtrace property names to be used in determining pin names for source instance pins.

LVS Power Name

Specifies a list of power net names.

LVS Precise Interaction

Specifies whether to compute precise interaction regions during hierarchical device recognition.

LVS Prefer Nets

Specifies names of nets that are preferred when assigning names to nets that are shorted during LVS transformations due to LVS Filter SHORT processing.

LVS Prefer Port Nets

Specifies the precedence of port nets as the surviving nets after circuit transformation.

LVS Preserve Box Cells

Specifies how LVS Box cell contents are written to an extracted layout netlist.

LVS Preserve Box Ports

Specifies how LVS Box cell ports are handled during high-short resolution.

LVS Preserve Floating Top Nets

Controls preservation of floating named top-level nets in the extracted layout SPICE netlist.

LVS Preserve Parameterized Cells

Prevents flattening of parameterized subcircuits.

LVS Property Initialize

Instructs the tool to add numeric properties simultaneously to instances in the layout and source databases in memory and initialize those properties to specified values.

LVS Property Map

Instructs the tool to re-map property names in the specified input database.

LVS Property Resolution Maximum

Specifies that Calibre nmLVS use property values to resolve all groups of ambiguous elements that contain no more than a maximum number of elements.

LVS Push Devices

Specifies whether to push device seed polygons down the hierarchy after seed promotion.

LVS Recognize Gates

Specifies whether to recognize logic gates from transistor-level data.

LVS Recognize Gates Tolerance

Specifies whether to limit gate recognition using device property tolerance checking.

LVS Reduce

Specifies generic device reduction instructions.

LVS Reduce Parallel Bipolar

Specifies whether to reduce bipolar transistors connected in parallel into single transistors.

LVS Reduce Parallel Capacitors

Specifies whether to reduce capacitor devices connected in parallel into single capacitors.

LVS Reduce Parallel Diodes

Specifies whether to reduce diodes connected in parallel into single diodes.

LVS Reduce Parallel MOS

Specifies whether to reduce MOS transistors connected in parallel into single transistors.

LVS Reduce Parallel Resistors

Specifies whether to reduce resistor devices connected in parallel into single resistors.

LVS Reduce Semi Series MOS

Specifies whether to reduce MOS devices connected in series including those with bypass nets.

LVS Reduce Series Capacitors

Specifies whether to reduce capacitor devices connected in series into single capacitors.

LVS Reduce Series MOS

Specifies whether to reduce MOS devices connected in series.

LVS Reduce Series Resistors

Specifies whether to reduce resistor devices connected in series into single resistors.

LVS Reduce Split Gates

Specifies whether to reduce MOS split gates formed as serial-up or serial-down structures into single gate structures.

LVS Reduction Priority

Specifies which type of device reduction to perform when both series and parallel reduction are possible on the same group of devices.

LVS Report

Specifies the LVS report pathname. This statement is required in LVS comparison.

LVS Report Maximum

Specifies an upper limit on the number of items that LVS prints out in various sections of its report.

LVS Report Option

Controls the level of detail and contents of an LVS Report.

LVS Report Pinswapped

Specifies whether swapped pins for device instances generate errors in the LVS Report.

LVS Report Trivial Ports

Specifies whether trivial ports are reported as warnings during hierarchical netlist extraction.

LVS Report Units

Specifies whether units should be reported with built-in device property values.

LVS Report Warnings Hcell Only

Specifies whether connectivity extraction warnings should be reported for cells not specified in an hcell file or the Hcell statement.

LVS Report Warnings Top Only

Specifies whether to limit circuit extraction warnings to the top level.

LVS Reverse WL

Specifies to reverse the default L, W order in MOS devices in SPICE netlists and within SPICE-like properties in EDDM and Direct Mode ICverify designs.

LVS Select Device By Seed Layer

Constrains device recognition to use Device statements having specified seed layers.

LVS Short Equivalent Nodes

Specifies whether to short together equivalent nodes in MOSFET split gate structures.

LVS Show Seed Promotions

Specifies whether instances in the design where devices are formed by interaction of polygons from different levels of hierarchy are found.

LVS Show Seed Promotions Maximum

Specifies the maximum number of polygons that appear in any check performed by the LVS Show Seed Promotions statement.

LVS SI Select Connects

Selects Connect and Sconnect operations to participate in Calibre nmLVS Reconnaissance Short Isolation.

LVS Signature Maximum

Specifies that the signature algorithm compute signatures for neighborhoods no bigger than a maximum size.

LVS Soft Substrate Pins

Specifies whether substrate and bulk pins have a standard effect on how circuit elements are matched.

LVS Softchk

Selects polygons involved in conflicting connections from all Sconnect operations.

LVS Softchk Maximum Results

Specifies the number of polygons reported in the softchk results database.

LVS Spice Allow Duplicate Subcircuit Names

Specifies whether the SPICE parser allows duplicate subcircuit definitions with a warning or disallows them.

LVS Spice Allow Floating Pins

Specifies whether to allow floating pins in subcircuit calls in SPICE netlists.

LVS Spice Allow Inline Parameters

Specifies whether to allow inline parameter references for subcircuit calls (X calls) in SPICE netlists.

LVS Spice Allow Unquoted Strings

Specifies whether the SPICE parser should allow unquoted string values in parameter assignments.

LVS Spice Conditional LDD

Specifies how the SPICE reader processes $LDD designators in M elements.

LVS Spice Cull Primitive Subcircuits

Specifies whether to ignore primitive subcircuits having names that do not appear in the rule file.

LVS Spice Exclude Cell

Specifies to ignore calls to a subcircuit in a SPICE netlist.

LVS Spice Implied MOS Area

Specifies whether to calculate implied device areas for “M” elements in SPICE netlists.

LVS Spice Multiplier Name

Specifies parameter names that serve as multiplier factors in SPICE netlists instead of the standard SPICE parameter name M.

LVS Spice Option

Specifies warning and duplicate subcircuit handling behaviors of the SPICE parser.

LVS Spice Override Globals

Specifies whether pin assignments override global signals in SPICE netlists.

LVS Spice Prefer Pins

Specifies whether to prefer subcircuit pins over global nets when resolving net names in SPICE netlists.

LVS Spice Redefine Param

Specifies whether .PARAM values may be redefined in a SPICE netlist.

LVS Spice Rename Parameter

Specifies that SPICE parameters are renamed.

LVS Spice Replicate Devices

Specifies whether the SPICE parser should replicate device elements that have the M parameter.

LVS Spice Scale X Parameters

Specifies whether .OPTIONS SCALE factors in SPICE netlists apply to built-in devices instantiated as X elements.

LVS Spice Slash Is Space

Specifies whether the slash character should be treated as white space within subcircuit definitions and calls in SPICE netlists.

LVS Spice Strict WL

Controls the parsing of width and length parameters for MOSFETs in SPICE netlists.

LVS Split Gate Ratio

Specifies to check property ratios in split gate structures.

LVS Strict Subtypes

Controls the processing of component subtypes based on whether unspecified component subtypes are matched to instances with specified subtypes.

LVS Summary Report

Specifies that Calibre nmLVS-H generate a summary report file after a circuit comparison run.

LVS Use Careful AND

Causes circuit extraction (as in calibre -spice) to use a different hierarchical algorithm for AND layer operations. The alternate algorithm preserves the original hierarchy as much as possible at the cost of approximately twice the normal run time.

LVS Unselect Device By Seed Layer

Causes device recognition not to use Device statements having specified seed layers.

LVS Write Injected Layout Netlist

Specifies to output a netlist report from the layout, including injected logic.

LVS Write Injected Source Netlist

Specifies to output a netlist report from the source, including injected logic.

LVS Write Layout Netlist

Specifies to output a transformed netlist report from the layout.

LVS Write Source Netlist

Specifies to output a transformed netlist report from the source.

Mask Results Database

Specifies the pathname for the ICtrace mask results database and the type of information to be excluded from that database.

Mask SVDB Directory

Creates a Standard Verification Database Directory (SVDB) in Calibre.

PEX Preserve Cell List

Specifies a list of cells to receive special protection from hierarchical manipulation during circuit extraction in parasitic extraction flows.

Port Depth

Specifies the hierarchical depth for reading of port objects for use in the top-level cell. Not used in ICverify.

Port Layer Merged Polygon

For input layout databases, causes shapes on the specified layer(s) in the top-level cell to be merged on a per-cell, per-layer basis, read, and treated as geometric ports. Not used in ICverify.

Port Layer Polygon

For input layout databases, causes shapes on the specified layer(s) in the top-level cell to be read and treated as geometric ports. Not used in ICverify.

Port Layer Text

For input layout databases, causes text objects on the specified layer(s) in the top-level cell to be read and treated as text ports. Not used in ICverify.

Source Case

Specifies whether the source should be processed in a case-sensitive manner.

Source Path

Specifies the source database pathname for Calibre nmLVS/Calibre nmLVS-H, Calibre PERC, and Calibre xRC.

Source Primary

Specifies a design filename, subcircuit, or cell name for SPICE or CNET source systems. Used only in Calibre nmLVS/Calibre nmLVS-H, Calibre PERC, and Calibre xRC.

Source System

Specifies the layout database type for Calibre nmLVS/Calibre nmLVS-H, Calibre PERC, and Calibre xRC.

Trace Property

Controls which device properties are compared during LVS and how the comparison is performed.

Virtual Connect Specification Statements

Control specification of virtual connections. These statements apply during circuit extraction.
