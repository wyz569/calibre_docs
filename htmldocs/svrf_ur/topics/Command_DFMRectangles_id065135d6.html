DFM Rectangles

Layer operation

License: Calibre YieldEnhancer

Generates an output layer consisting of rectangles with the specified dimensions and spacing, where these parameters can be provided by explicit numeric values or by DFM properties attached to the input layer. The rectangles are generated within the polygons on an input layer.

Usage

Array Mode

DFM RECTANGLES layer x_length y_length
{spacing | x_spacing y_spacing }
[OFFSET {offset | x_offset  y_offset }]
[MAINTAIN SPACING [REGION]]

MAX Mode

DFM RECTANGLES layer MAX

Arguments

layer

A required polygon layer. The rectangles are generated within the polygons on this layer.

Each polygon on the layer may optionally have attached DFM properties that provide the length, spacing, and offset values. In this case the corresponding argument is the property name enclosed in quotes.

Array Mode Arguments

x_length y_length

A required argument pair that specifies the x and y dimensions of the rectangles in array mode. The values are positive floating-point numbers in user units or property names enclosed in quotes.

In hierarchical applications, the x and y directions are determined in the local cell frame of reference, after possible promotion. See “Hierarchical Applications” with the Rectangles command reference.

spacing | x_spacing y_spacing

A required argument set that specifies the spacing between rectangles in array mode. The values are positive floating-point numbers in user units, or property names enclosed in quotes.

  • spacing — The spacing in both the x and y directions.

  • x_spacing y_spacing — The spacings in the x and y directions, respectively.

OFFSET {offset | x_offset y_offset }

An optional keyword and argument set the specifies the horizontal and vertical offsets between adjacent rectangles in array mode. The values are non-negative floating-point numbers in user units or property names enclosed in quotes.

  • offset — The offset in both the x and y directions.

  • x_offset y_offset  — The offsets in the x and y directions, respectively.

If an offset value is greater than the corresponding spacing value, the minimum spacing between rectangles is controlled by the corresponding spacing value. More discussion of the OFFSET keyword is provided in the Rectangles command reference.

MAINTAIN SPACING [REGION]

An optional keyword set that affects the spacing between rectangles in array mode.

When MAINTAIN SPACING is specified, a halo area is constructed around each rectangle using the spacing or x_spacing y_spacing values. No other rectangle may fall within this halo region. Without the REGION keyword, the spacing check is only performed for rectangles within the same layer polygon.

  • REGION — An optional keyword used only for hierarchical runs. When specified, the MAINTAIN SPACING check is also performed between rectangles within different layer polygons.

    If spacing values are provided with DFM properties, the spacing check between regions uses the largest spacing value.

MAX Mode Arguments

MAX

A required keyword in MAX mode. For each Manhattan polygon on layer, the largest possible Manhattan polygon that is completely within the polygon is output. Non-Manhattan polygons on layer are ignored.

Description

Generates an output layer consisting of rectangles with the specified dimensions and spacing. This operation is similar to Rectangles with “INSIDE OF LAYER layer”, except that the rectangle parameters can be specified by DFM properties on the input layer. The MAX and REGION keywords are unique to DFM Rectangles.

See the Rectangles command reference for details about the algorithm used to generate the rectangles in array mode.

Examples

Example 1

This example defines the rectangle dimensions and spacing as properties based on the shortest edge length of the polygon that is being filled: 

prop_layer1 = DFM PROPERTY layer1 
                  [ xy_len = MIN(LENGTH(layer1:EDGE))/10 ]
                  [ xy_sp = PROPERTY_REF(xy_len)/5 ]

rect_B = DFM RECTANGLES prop_layer1 "xy_len" "xy_len" "xy_sp"

Example 2

This example uses the MAX keyword to create the largest possible Manhattan rectangle within each polygon that is being filled. Non-Manhattan shapes on layer1 are ignored. 

rect_max = DFM RECTANGLES layer1 MAX
Figure 1. DFM RECTANGLES With MAX Keyword

Example 3

This example defines the rectangle dimensions and spacing as properties based on the largest x and y dimensions of the polygon that is being filled. MAINTAIN SPACING REGION is specified so that spacing between rectangles that fill different polygons is checked. The region to region spacing check is done using the largest spacing value for the regions being checked. 

prop_layer1 = DFM PROPERTY layer1 
                  [ x_sp = MAX(LENGTHX(layer1:EDGE))/16 ]
                  [ x_len = PROPERTY_REF(x_sp)*3 ]
                  [ y_sp = MAX(LENGTHY(layer1:EDGE))/16 ]
                  [ y_len = PROPERTY_REF(y_sp)*3 ]

rect_C_mr = DFM RECTANGLES prop_layer1 "x_len" "y_len" "x_sp" "y_sp" 
                MAINTAIN SPACING REGION
Figure 2. DFM RECTANGLES With MAINTAIN SPACING REGION
