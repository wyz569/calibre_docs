DFM OR

Layer operation

License: Calibre nmDRC-H

Performs an OR operation on the input polygon layers and includes DFM properties from the input layers on the output layer. Properties are merged if conflicting property values exist on overlapping polygons from different layers. Optional keywords specify the method for merging properties.

Usage

DFM OR layer [layer …] [ SUM | REPRESENT | VECTOR ]

Arguments

layer [layer …]

One or more required polygon layers. These layers are typically the result of a DFM Property or related operation and have attached DFM properties.

SUM | REPRESENT | VECTOR

An optional keyword choice that specifies how to handle conflicts in property values. A conflict occurs when polygons from different input layers overlap and have different values for the same property name and type. The property value for the merged output polygon P is determined with one of the following methods:

  • SUM — SUM is the default behavior. Numeric property values are summed and vector values are concatenated. For non-summable properties, such as netIDs and strings, the value is chosen as with the REPRESENT method.

  • REPRESENT — A representative value is chosen from one of the input polygons that contribute to the output polygon, with polygons prioritized according to the order of the input layers to the operation. The representative value for a polygon P is taken from the first-occurring input layer that overlaps P and has a property of the correct name and type. If more than one polygon from the chosen layer overlaps P, the property value is selected arbitrarily from the overlapping polygons of that layer.

  • VECTOR — All output properties are vector-type. The value for a polygon P is a vector containing all the values for all the input polygons that overlap P. If the input property is not a vector type (number, netID, or string), then the output property is VECTOR, VNETID, or VSTRING with tuple size 1. If the input property is a vector-type property, then the output value concatenates the input values, as with the default behavior.

See “Examples for Special Cases”.

Description

Performs an OR operation on the input polygon layers and attaches DFM properties from the input layer shapes to the output shapes. Each shape on the output layer includes the complete set of property names that exist on the input layers.

A common application of DFM OR is dividing a polygon layer into component layers according to some criteria, deriving properties separately for the component layers, then using DFM OR to combine the component layers. This process is useful when the property derivations are different for the component layers. This example divides the input layer by polygon area and computes a param property, using a derivation that differs depending on the polygon area: 

layer_big = DFM Property layer [ area = [AREA(layer)] > 1
               [ param = [ AREA(layer)/PERIM(layer) ]
layer_small = DFM Property layer [ area = [AREA(layer)] <= 1
               [ param = [ AREA(layer)*0.5 ]
layer_all = DFM OR layer_big layer_small

The output layer_all contains all shapes from the input layers, with attached properties area and param. It is possible to perform the above derivation with one DFM Property operation that uses conditional expressions, but the method with DFM OR is simpler, especially when many properties are involved and the conditions are more complex.

The primary usage of DFM OR is with two or more input layers. One-layer operations with SUM or REPRESENT create a copy of the input layer. If VECTOR is specified, the DFM properties are converted to vectors, as described with the VECTOR keyword.

The simplest application of DFM OR is when the input polygon layers have the same set of properties and none of the shapes overlap, as with the preceding example. Special cases occur when the input layers do not have the same set of properties, or there is a conflict in property values for input layer shapes that overlap.

The following sections provide more detail, with examples following the discussion.

Disjoint Shapes With Identical Property Sets

None of the shapes in the input layers overlap, and all the input layers have the same set of properties. Each shape on the output layer has all properties copied from the corresponding shape on the input layer.

Differing Property Sets

Differing property sets occur when a property is not present on all input layers, or if the same property name is present on more than one layer, but the property type is not the same. The output layer always has the complete set of property names that exist on the input layers. A default value is assigned when a property is missing. When the same property name has different types on different layers, the property type found on the first layer in the list of input layers is used.

  • Missing property — For example, the property A does not exist on all input layers. If an output layer shape does not have a contributing value for property A, a default value is assigned, depending on the property type:

    Property Type

    Default Value

    Numeric

    0

    netID

    NONETID()

    String

    EMPTYSTRING()

    Vector

    Empty vector of the corresponding type (VECTOR, VNETID, or VSTRING)

  • Different property types — For example, input layer1 has a numeric property A and input layer2 has a string property A. The output property type is taken from the earliest occurring layer in the list of input layers. For vector properties, the tuple size is determined from the earliest layer. Input layer properties that do not match the output layer property type (including tuple size for vectors) are discarded.

Property Value Conflicts From Overlapping Shapes

When polygons from different input layers overlap, then a single polygon in the output of DFM OR may have multiple input property values mapped to it. With SUM (the default), the output property is the sum of the input property values for the shape. The REPRESENT and VECTOR keywords change the behavior; see the keyword descriptions for details.

Examples

Divides the input layer into three component layers and attaches a DFM property size_prop that is later used by the DFM Size operation. DFM OR is used to merge the three component layers. 

// increase size so that shorter edge has length 1
metal1_small = DFM PROPERTY metal1 
            [ min_len = MIN(LENGTH(metal1:EDGE)) ] < 1
            [ size_prop = (1-PROPERTY_REF(min_len))*0.5]

// do not change size
metal1_medium = DFM PROPERTY metal1 
            [ min_len = MIN(LENGTH(metal1:EDGE)) ] >= 1
            [ - = MAX(LENGTH(metal1:EDGE)) ] <= 3
            [ size_prop = 0]

// decrease size, but shorter edge must be at least length 1
metal1_big = DFM PROPERTY metal1 
            [ min_len = MIN(LENGTH(metal1:EDGE)) ] >= 1
            [ - = MAX(LENGTH(metal1:EDGE)) ] > 3
            [ size_prop = -1.0*FMIN(0.25*PROPERTY_REF(min_len), 
                                    0.5*(PROPERTY_REF(min_len)-1))]

metal1_sizeprops = DFM OR metal1_small metal1_medium metal1_big

metal1_resize = DFM SIZE metal1_sizeprops BY "size_prop"
