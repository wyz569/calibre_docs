Resolution

Specification statement

Defines the layout grid step-size (the number of database units on which original geometry is required to be aligned).

Usage

RESOLUTION {s | x y}

Arguments

s | x y

A required positive integer, or pair of positive integers, that specifies the layout grid (also called the user grid) step size in database units.

  • s — A positive integer that specifies the grid step size in both the x- and y-directions.

  • x y — Two positive integers that specify the grid step size in the x- and y-directions, respectively.

Description

Defines the layout grid step size. This is the multiple of database units on which any original geometry is to be aligned. This statement can appear once in a rule file. It is independent of any other settings.

The Resolution statement is optional. If it is not included in the rule file, the default behavior sets s equal to one database unit.

The primary use of the Resolution statement is to allow off-grid original polygons to be flagged. Note that this statement has no effect upon the Snap or Offgrid layer operations.

The resolution affects generated fill shapes if the default resolution of 1 is not used. See “Resolution and Precision Setting and Effect on Fill Shapes” with the DFM Spec Fill statement.

See also Drawn Offgrid, Flag Offgrid, Snap Offgrid, Layer Resolution, and Precision.

Examples

The following grid step size requires original shapes to align at 250, 500, 750, or 1000 database units. This is assuming the layout’s physical precision matches the rule file Precision statement.

PRECISION 1000						// 1000 database units per user-unit.
RESOLUTION 250
// Alignment points of original polygons must be every 0.25 
// user units.
