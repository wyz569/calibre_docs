PEX Reduce Parallel

Parasitic extraction

Controls whether or not resistors in parallel are combined in the netlist during the formatting stage. Used only in Calibre xRC and Calibre xACT 3D PDB flow.

Usage

PEX REDUCE PARALLEL {NO | YES}

Arguments

NO

Keyword that instructs the tool not to combine parallel resistors during the formatting stage. This is the default behavior if you do not include this statement in your rule file.

YES

Keyword that instructs the tool to combine parallel resistors during the formatting stage.

Description

Specifies whether or not resistors in parallel are combined in the netlist during the formatting stage. Two or more resistors are in parallel when they are connected between the same pair of nodes.

When PEX Reduce Parallel is set to YES, parallel resistors are replaced by a single equivalent resistor with resistance R, where

1/R = 1/R1 + 1/R2 + ... + 1/Rn

and R1, R2, …, Rn are the resistances of the conductors in parallel.

Using PEX Netlist RLAYER, RLENGTH, RTHICKNESS, RLOCATION or RWIDTH with PEX Reduce Parallel YES will generate a warning message. RLAYER, RLENGTH, RTHICKNESS, RLOCATION, and RWIDTH properties are invalid and will not be netlisted for resistors that result from parallel combination.

The reduction statements PEX Reduce ROnly, PEX Reduce TICER, PEX Reduce Analog, and PEX Reduce Digital perform parallel reduction even if PEX Reduce Parallel NO is specified.

Examples

To instruct the formatter to combine resistors in parallel in the output netlist, include the following statement in your rule file:

PEX REDUCE PARALLEL YES
