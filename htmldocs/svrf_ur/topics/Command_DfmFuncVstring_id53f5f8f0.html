VSTRING

String vector function for DFM expressions

Creates a vector of string type properties. Used only in DFM Property and DFM Function.

Usage

VSTRING(arg_expr1[, arg_expr2] …)

Arguments

arg_expr1[, arg_expr2]

A required string value or an expression that evaluates to a string. Multiple arguments are separated by commas. Each argument is evaluated to form a tuple element.

Arguments can be strings, expressions involving the SPROPERTY function, or functions that return strings. Expression functions can reference either the primary or a secondary layer. If more than one argument references a layer, the same layer must be referenced in all cases. See the Description section for the behavior with primary or secondary layers.

The property reference function SPROPERTY_REF cannot be used within VSTRING, however, it can be used in SVTUPLE.

The arguments may be empty strings in order to create an empty vector.

Return Values

String-type property vector.

Description

This function creates a property vector of string-type elements.

For each primary layer geometry, function evaluation proceeds as follows:

  • If the tuple arguments reference a secondary layer, then for each secondary-layer geometry clustered with the primary-layer geometry, the tuple arguments are evaluated and the results appended to the vector. The order in which they are appended is arbitrary. If there are no secondary-layer geometries in the cluster, the result is a zero-length (empty) vector with a tuple size of one more than the number of commas in the expression.

  • If the tuple arguments reference the primary layer, then the tuple arguments are evaluated exactly once (for the current primary-layer geometry) and the results appended to the vector.

    The same is true if the function’s expression makes no reference to a layer. For example, the function call VSTRING(“one”, “two”) results in a vector with one tuple containing strings “one” and “two”. In this case, the primary layer is implicitly assumed as the referenced layer.

Empty vectors, such as VSTRING(,,,), are permitted. The tuple size is equal to the number of commas plus 1.

Any expression that does not evaluate successfully for any tuple argument in the VSTRING function call results in the entire VSTRING function failing to evaluate successfully. When this happens, the following apply:

  • If the property is defined multiple times to form an expression chain, the operation attempts to evaluate the expression for the next definition. If there is no next definition, the geometry on the primary layer is not written to the output layer.

  • If the property is defined once (that is, not chained), the geometry on the primary layer is not written to the output layer.

“Expressions that Do Not Evaluate Successfully” has additional details.

String vectors may be compared lexicographically using ==, !=, >, >=, <, <= operators; such comparisons can be done using Conditional Expressions. Two string vectors are equal if they have the same tuple count, tuple size, and the same element values in the same order. Otherwise, the two vectors are unequal. For example, these comparisons are both true:

(vstring(a,b,c) == vstring(a,b,c)
(vstring(a,b,c) <  vstring(a,b,d)

If two string vectors are of unequal tuple size or count, the vector having fewer elements is considered to be strictly less than the other vector.

It is not permitted to use a constraint in an expression with vector properties, unless the expression is conditional. See the Examples section to see this usage.

You can access individual vector elements with the function SVELEMENT. You can also access vector elements with a user-defined function created by a DFM Function statement; see “Accessing Vector Elements”.

Examples

This example demonstrates making vector string properties from text object values, placing those properties on polygons, and then testing which polygons have the vector string property <A> <A>.

LAYER zero 0
LAYER txt  1 // text layer

// create a layer of string properties whose values are derived 
// from the txt layer.
txt_str_p = DFM TEXT txt PROPERTY STRING s

// merge the txt_str_p properties with layer zero shapes.
// there can be more than one txt_str_p object that interacts with
// a layer zero polygon.
// create a vector property txt_str_vec to store the string property
// values.
merged_txt_vec_p = DFM PROPERTY MERGE zero txt_str_p
                   [txt_str_vec = VSTRING(SPROPERTY(txt_str_p,s))]

// select layer zero polygons overlapping merged_txt_vec_p polygons
// where the txt_str_vec property equals <A> <A>.
zero_AA_p = DFM PROPERTY zero merged_txt_vec_p OVERLAP
            [ - = (SVPROPERTY(merged_txt_vec_p,txt_str_vec) ==
                   CONCAT(VSTRING("A"),VSTRING("A"))) ? 1 : 0 ] == 1
