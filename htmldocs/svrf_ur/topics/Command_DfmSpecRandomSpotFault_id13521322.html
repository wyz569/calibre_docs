DFM Spec Random Spot Fault

Specification statement

Specifies a random spot defect definition that is referenced by a DFM CAF or DFM GCA operation.

Usage

DFM SPEC RANDOM SPOT FAULT spec_name
PRIMITIVE FAULTS
   {primitive_fault_name1
      layer1 [short_layer2 …]
      {SHORT [{EXCLUDE FAULTS BETWEEN layer1 '&' short_layer2} …]
      | OPEN} ';'
   } ...
END PRIMITIVE FAULTS

[COMPLEX FAULTS
   {complex_fault_name
      primitive_fault_name1 EXCLUDING primitive_fault_name2 ';'
   } ...
END COMPLEX FAULTS]

Arguments

spec_name

A required argument that specifies a name for the random spot fault specification that is referenced in DFM CAF and DFM GCA operations.

PRIMITIVE FAULTS ... END PRIMITIVE FAULTS

A required keyword set that contains a primitive fault list. Any number of unique primitive fault definitions may be specified between the PRIMITIVE FAULTS and END PRIMITIVE FAULTS keywords, both of which are required. Each definition must be terminated by a space and semicolon.

  • primitive_fault_name1

    A required argument that specifies the name of the primitive fault.

  • layer1 [short_layer2 …]

    A required argument that specifies the original or derived polygonal input layer. Multiple input layers are allowed only for a short defect type.

  • SHORT

    A keyword that specifies the short defect type. Either OPEN or SHORT commands must be specified.

    • EXCLUDE FAULTS BETWEEN layer1 '&' layer2

      An optional argument, used only by SHORT, that excludes faults between the specified layers. The layers must appear in the list declared within the primitive fault defining the SHORT.

  • OPEN

    A keyword that specifies the open defect type. Either OPEN or SHORT must be specified.

COMPLEX FAULTS ... END COMPLEX FAULTS

An optional keyword set that contains a list of complex and primitive fault definitions. Any number of unique complex fault definitions may be specified between the COMPLEX FAULTS and END COMPLEX FAULTS keywords. Each definition must be terminated with a space and a semicolon.

  • complex_fault_name

    An argument that specifies the complex fault name.

  • primitive_fault_name1 EXCLUDING primitive_fault_name2

    An argument set that specifies the primitive faults previously specified within the PRIMITIVE FAULTS definition. A primitive fault defect type can be specified as either a SHORT or OPEN. The input_layer of a primitive fault is the input layer that was declared in the primitive fault definition. For example, where the primitive fault named M1_SHORT specifies an input layer of M1, and the primitive fault M1_SHORT defect type is SHORT, the command is written:

    DFM SPEC RANDOM SPOT FAULT spec_name
	PRIMITIVE FAULTS
 		M1_SHORT
			M1
			SHORT ;
	END PRIMITIVE FAULTS
...

    Given the above, the primitive fault can be specified as:

    • defect_type1 — The defect type for primitive_fault_name1

    • defect_type2 — The defect type for primitive_fault_name2

    Consequently, the following code calculates defect_type1 on the input layer of primitive_fault_name1. This is used in conjunction with primitive_fault_name2 EXCLUDING primitive_fault_name1 to subtract the region of defect_type1 and defect_type2 overlap from the total critical area.

    COMPLEX FAULTS
	com_fault1
		primitive_fault_name1 EXCLUDING primitive_fault_name2 ;
	com_fault2
		primitive_fault_name2 EXCLUDING primitive_fault_name1 ;
END COMPLEX FAULTS

Description

Defines a random spot defect specification that can be referenced by a DFM CAF and a DFM GCA operation.

Examples

Example 1

A simple, single-layer short definition is illustrated here:

DFM SPEC RANDOM SPOT FAULT M1_SHORT_spec
	PRIMITIVE FAULTS
		M1_SHORT
			M1
			SHORT ;
	END PRIMITIVE FAULTS

After implementing the DFM Spec Random Spot Fault definition, the following two DFM CAF commands are identical. In the first command, the DFM Spec Random Spot Fault name, M1_SHORT_spec, and the PRIMITIVE FAULTS name, M1_SHORT, is referenced by a DFM CAF operation (no input_layer may be specified with DFM CAF SPEC):

DFM CAF SPEC M1_SHORT_spec M1_SHORT R 50 INSIDE OF LAYER M1_EXTENT

The second DFM CAF command is equivalent, but does not use the SPEC keyword:

DFM CAF M1 SHORT R 50 INSIDE OF LAYER M1_EXTENT

Example 2

As the color coding in this example illustrates, the name of the DFM Spec Random Spot Fault command is propagated to the DFM CAF SPEC commands, and the PRIMITIVE FAULT names are propagated to the COMPLEX FAULT definitions.

DFM SPEC RANDOM SPOT FAULT diff_shopen
	PRIMITIVE FAULTS
		diff.short
			DIFF1 GATE_DIFF
			SHORT
				EXCLUDE FAULTS BETWEEN GATE_DIFF & GATE_DIFF ;
		diff.open
			DIFF_NOT_POLY
			OPEN ;
	END PRIMITIVE FAULTS
	COMPLEX FAULTS
		diff.shopen.short
			diff.short EXCLUDING diff.open ;
		diff.shopen.open
			diff.open EXCLUDING diff.short ;
	END COMPLEX FAULTS
DIFF.SHORT = DFM CAF SPEC diff_shopen diff.shopen.short R 0.04
	INSIDE OF LAYER DIFF.SHORT_extent
DIFF.OPEN = DFM CAF SPEC diff_shopen diff.shopen.open R 0.0289
	INSIDE OF LAYER DIFF.OPEN_extent

Example 3

In this example, the DFM Spec Random Spot Fault statement provides concurrent execution of the following two DFM CAF operations. Critical area extraction is performed concurrently for short and open defect types.

DFM SPEC RANDOM SPOT FAULT M1_SHOPEN_spec
	PRIMITIVE FAULTS
		M1_SHORT M1
			SHORT ;
		M1_OPEN M1
			OPEN ;
	END PRIMITIVE FAULTS
	COMPLEX FAULTS
		M1_SHOPEN_SHORT
			M1_SHORT EXCLUDING M1_OPEN ;
		M1_SHOPEN_OPEN
			M1_OPEN EXCLUDING M1_SHORT ;
	END COMPLEX FAULTS

DFM CAF SPEC M1_SHOPEN_spec M1_SHOPEN_SHORT R 50 INSIDE OF LAYER M1_EXTENT
DFM CAF SPEC M1_SHOPEN_spec M1_SHOPEN_OPEN R 50 INSIDE OF LAYER M1_EXTENT

The preceding code is equivalent to these two DFM CAF commands:

DFM CAF M1 SHORT EXCLUDING OPEN R 50 INSIDE OF LAYER M1_EXTENT
DFM CAF M1 OPEN EXCLUDING SHORT R 50 INSIDE OF LAYER M1_EXTENT
