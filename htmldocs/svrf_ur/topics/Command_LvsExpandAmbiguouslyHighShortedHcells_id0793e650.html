LVS Expand Ambiguously High Shorted Hcells

Specification statement

Specifies whether to expand high-shorted hcells that are matched in a many-to-one configuration and that cause false LVS comparison discrepancies.

Usage

LVS EXPAND AMBIGUOUSLY HIGH SHORTED HCELLS {NO | YES}

Arguments

NO

Keyword that specifies not to expand high-shorted hcells. This is the default behavior if you do not specify the statement in the rule file.

YES

Keyword that specifies to expand ambiguously high-shorted hcells.

Description

Controls the expansion of hcells with high-shorted pins and that cause false LVS comparison discrepancies.

High shorts are created when an hcell’s pins are consistently shorted in the cell’s super-hierarchy in all instances of the cell. In certain situations (particularly in designs using “colored” metal layers), hcells that are matched in a many-to-one configuration can have high-shorted pins that cause false LVS comparison discrepancies. Due to connection ambiguities in such configurations, the best way to resolve the issue is to allow the affected hcells to be expanded.

For LVS comparison, affected hcells meet the following criteria:

  • Many-to-one cell correspondence.

  • The “one” cell in the many-to-one correspondence is not consistently high-shorted in all placements.

As as result, there is an inconsistent number of ports in the corresponding cells, which causes an LVS discrepancy.

The problem is particularly likely to occur during circuit extraction under these conditions:

  • One of the cells on the “many” side of the many-to-one correspondence is consistently high-shorted.

  • The “one” cell in the many-to-one correspondence is not consistently high-shorted in all placements.

  • The pins in question are not texted or are not similarly texted.

These conditions cause the consistently high-shorted hcell in the “many” set to lose one of the ports that would otherwise allow it to compare successfully.

When YES is specified, the ambiguously high-shorted hcells are expanded and do not participate as hcells. All expanded instances are promoted into any containing hcells, and the ambiguous high-shorts are resolved correctly.

See “High-Short Resolution” and “Hcell Correspondence” in the Calibre Verification User’s Manual for related information.

See also LVS Auto Expand Hcells and LVS Hcell Report.

Examples

// a many-to-one hcell correspondence cell has high-shorted pins 
// resulting in a false LVS error
LVS EXPAND AMBIGUOUSLY HIGH SHORTED HCELLS YES
