Deangle

Layer operation

Constructs a derived polygon layer by replacing skew edges with sequences of orthogonal or 45-degree edges. Used only in Calibre applications.

Usage

DEANGLE layer maximum_deviation
[SKEW | SKEW ORTHOGONAL ONLY | ALL]

Arguments

layer

A required original layer or layer set, or a derived polygon layer.

maximum_deviation

A positive, floating-point numeric in user units that must be an integer when multiplied by the process precision.

SKEW | SKEW ORTHOGONAL ONLY | ALL

An optional keyword choice that specifies which edges to process and how to replace them.

  • SKEW — Processes skew edges only and replaces them with combinations of orthogonal and 45-degree edges. This is the default behavior if no keyword from this set is specified.

  • SKEW ORTHOGONAL ONLY — Processes skew edges only and replaces them with orthogonal edges only.

  • ALL — Processes all angled edges regardless of their angle with respect to the database axes and replaces them with orthogonal edges only.

Description

By default, generates a derived polygon layer by replacing skew (not orthogonal or at a 45-degree angle with respect to the database axes) edges from layer with sequences of orthogonal or 45-degree edges. For the remainder of this discussion, an orthogonal edge is one that is orthogonal to the database axes. Optional keywords change the edges that are processed and the edges that are output. The operation is a non-node-preserving layer constructor. If layer is empty, the output layer is empty.

If either SKEW or no optional keyword is specified, the operation proceeds as follows. If layer has no skew edges, then the output layer is a copy of layer. Otherwise, skew edges of polygons on the input layer are replaced with one or more 45-degree or orthogonal edges. Edge replacement can occur on both the inside and outside of the original skew edge. The maximum_deviation parameter describes the maximum perpendicular distance from the original skew edge to any point on any new 45-degree or orthogonal edge. See the following figure for illustrations of what occurs for segments of skew edges.

Figure 1. Deangle Output

Skew edges forming an angle A with the horizontal such that 22.5 < A < 67.5 degrees are approximated using 45-degree edges, unless the maximum_deviation value and the geometry of the shape do not permit it. Skew edges forming an angle B with the horizontal such that 0 < B <= 22.5 degrees, or 67.5 <= B < 90 degrees, are approximated using orthogonal edges.

If SKEW ORTHOGONAL ONLY is specified, then only orthogonal edges are used in the approximation, regardless of the angle of the skew edge with the horizontal.

If ALL is specified, then the operation proceeds as follows. If the input layer has no angled edges (see Drawn Angled for the definition), then the output layer is simply a copy of the input layer. Otherwise, angled edges of polygons on the input layer are replaced with one or more orthogonal edges. Edge replacement occurs both on the inside and outside of the original angled edge. The maximum_deviation parameter describes the maximum perpendicular distance from the original edge to any point on any newly-constructed orthogonal edge.

Examples

// approximate skew edges on layer1 with orthogonal edges using a
// maximum deviation of 0.1
ortho = DEANGLE layer1 0.1 SKEW ORTHOGONAL ONLY
