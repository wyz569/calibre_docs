device::debug

Device property computation function for the TVF functional interface

Returns “1”, or “true”, if the DEVice statement calling the TVF Function block in which device::debug appears uses a DEBUG function and “0”, or “false”, otherwise.

Usage

device::debug

Arguments

None.

Return Values

None.

Description

This function is used in conjunction with the TVF Functional Interface for Device Property Calculation. It must appear in a TVF Function block.

Examples

Consider the following DEVice statement and TVF Function block:

DEVICE MP (pch) pgate gate psd psd nw
[
DEBUG 3-5 
PROPERTY W,L,AG
W = perimeter_coincide(pgate,psd_s) / 2
L = perimeter_outside(pgate,psd_s) / 2
AG = tvf_num_fun::device_functions::effective_a( W, L )
]

TVF FUNCTION device_functions [/*
  proc effective_a { W L } {
    if { [device::debug] } {
      puts "TVF Debug info: [$W] [$L]"
    }
    return [ expr { [$W] * [$L] } ]
  }
*/]

If a DEBUG function is active in the DEVice statement that calls device_functions, then device::debug returns 1 and a “TVF Debug info:” line is printed to the run transcript along with W and L values. In the preceding code, the DEVICE MP statement would trigger that line to be printed because of the DEBUG 3-5 function.
