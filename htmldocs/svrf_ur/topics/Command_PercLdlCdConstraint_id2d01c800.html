PERC LDL CD Constraint

Specification statement

Specifies the LDL CD current density constraint reporting limit.

Usage

Default syntax:

PERC LDL CD CONSTRAINT VALUE number

Per-layer syntax:

PERC LDL CD layer CONSTRAINT {VALUE number | ‘[’user_program‘]’}

Arguments

VALUE number

A keyword set that specifies a fixed lower-limit current density reporting constraint. The number is a non-negative floating-point value in milliamperes/micron for interconnect layers, and milliamperes/square micron for contact and via layers.

In the default syntax, this keyword set is required when the statement is specified (number defaults to 0 if it is not). In the per-layer syntax, either this keyword set or the user_program must be specified.

layer

A required connectivity layer in the per-layer syntax. The layer must have parasitic resistance values specified in the rule file.

user_program

A required user program in the per-layer syntax enclosed in brackets ([ ]) that calculates a variable lower-limit current density reporting constraint. This program typically calls a Tcl proc that performs the constraint value assignment for parasitic resistors that meet program criteria. The Tcl proc must return a non-negative floating-point numeric value. The value is in milliamperes/micron for interconnect layers and milliamperes/square micron for contact and via layers. In the per-layer syntax, either this program or the VALUE keyword set must be specified. A CMACRO call to a DMACRO definition containing the user program block may be used. See the “User Program Details” and the “Examples” subsections.

Description

Specifies the lower-limit current density reporting value for Calibre PERC LDL CD checks. Current density values above the one provided by this statement appear in the LDL CD report in the CurrentDensityLimit column.

Note:

The forms of this statement using the VALUE keyword supersede the perc_ldl::execute_cd_checks ‑constraint_hash argument.

This statement uses two syntactical forms. The default syntax specifies a number for any layer that does not have a lower-limit CD constraint defined by other means. If a default syntax statement is not specified in the rule file, the default number is 0, which means all parasitic resistors are reported for any layer to which the default applies. The per-layer syntax specifies a constraint for a given layer, either as a constant number or through a user_program.

The order for determining the CD constraint for a layer is determined by the following criteria:

  1. Per-layer syntax specification.

  2. Constraint taken from a PEX calibrated layer that is mapped to an original or derived layer.

    For example:

    LAYER m1 10       // original layer
PEX MAP metal1 m1 // metal1 is a calibrated layer
PERC LDL CD CONSTRAINT metal1 VALUE 10

    Layer m1 gets its CD constraint from the calibrated layer metal1 due to criterion 2, unless m1 is explicitly specified in some other PERC LDL CD Constraint statement, in which case criterion 1 applies.

  3. Default syntax specification (0 if not otherwise defined).

Parasitic resistance must be specified for the layer. This is usually provided in a calibrated rule file. The PEX Map, Resistance Connection, and Resistance Sheet statements can also be used.

Negative numbers specified for the number or computed by the user_program are interpreted as 0 because negative current density is not meaningful.

User Program Details

The user_program controls how the current density constraint value is assigned in the per-layer syntax. The program defines criteria for parasitic resistors on the layer. The parasitic resistors that meet the program’s criteria have the appropriate current density constraint value assigned to them.

The basic form of the program is as follows:

PERC LDL CD M1 CONSTRAINT [
  CONSTRAINT <value>
  <variable_assignments>
  <value> = tvf_numeric_function::<tvf_function>::<tcl_proc>\
(<var1>[, <var2> ...])
]

It must have one value argument, which is used for assigning the CD constraint value. Variable assignments use the built-in user_program functions in Table 1.

The variables are used in a TVF_NUMeric_FUNction call to calculate the constraint value. If more than one variable name gets passed, the list of variables is comma-delimited.

The tvf_function is the name of a TVF Function block that does the CD value calculations. The block has this form:

TVF FUNCTION <tvf_function> [/*
  proc <tcl_proc> {<var1> [<var2> ...]} {
// Tcl commands involving [$<varN>]
...
  return <number> // proc must return a constraint value
*/]

The tcl_proc is a Tcl procedure in the TVF Function block. The varN variables are assigned in the user_program. At least one variable must be passed to the tcl_proc. The tcl_proc must return a numeric value to the user_program that calls it. Any functions in the Tcl package may be used in the proc.

Here is an example:

PERC LDL CD m1 CONSTRAINT [
  CONSTRAINT value
  RW = get_resistor_width() // resistor width
  value = tvf_num_fun::my_func::my_proc(RW)
]
TVF FUNCTION my_func [/*
  proc my_proc {RW} {
    if { [$RW] > 0 && [$RW] <= 0.1 } { 
      return 1.0
    } elseif { [$RW] > 0.1 && [$RW] < 0.2 } { 
      return 0.7
    }
    return 0 
  }
*/]

In this example, “value” is the name of the current density constraint. The RW variable is the width of a parasitic resistor. It is passed to my_proc in the my_func block.

Notice the use of the [ ] operators around the variable $RW in the my_proc body. The [ ] operators get the resistor width value from each parasitic resistor instance.

The my_proc procedure returns a value to the TVF_NUMeric_FUNction call, depending on the width of the resistor. That value is the current density constraint for parasitic resistors that meet the conditions in the program.

This table shows the built-in functions that may be used for variable assignment in the user_program:

Table 1. CD Constraint Program Built-In Functions

Function

Description

get_current()

Returns the current through a parasitic resistor in milliamperes.

get_em_length()

Returns the electromigration (EM) length of the longest path of contiguous parasitic resistors from a given layer on a given net. The perc_ldl::execute_cd_checks -em option must be specified when using this function.

get_em_width()

Returns the electromigration width of the widest parasitic resistor on the path from which the EM length is taken. The perc_ldl::execute_cd_checks -em option must be specified when using this function.

get_layer_name()

Returns the name of the layer of a parasitic resistor. The text case of the Layer statement name is used in the returned string if the layer is original. The text case of the assigned name is used if the layer is derived.

get_resistor_area()

Returns the parasitic area of contact or conducting layer polygons in um2. For contact layers, the returned value is the area computed by the PEX engine. For conducting layers, the returned value is the product of the length and width of a polygon as computed by the PEX engine.

get_resistor_length()

Returns the length of a parasitic resistor in um.

get_resistor_width()

Returns the width of a parasitic resistor in um. For contact layers, the returned value is NaN.

get_resistor_x()

Returns the x-coordinate of the midpoint of parasitic resistor. The coordinates are in um in top-level space.

get_resistor_y()

Returns the y-coordinate of the midpoint of parasitic resistor. The coordinates are in um in top-level space.

get_via_current_direction()

Returns the direction of current flow through vias. The perc_ldl::execute_cd_checks -em option must be specified when using this function. Additionally, layer heights must be specified in a calibrated rule file or the direction cannot be determined. Possible property values include these:

UP — current flows up the interconnect layer stack.

DOWN — current flows down the interconnect layer stack.

Unknown — insufficient information in the rules.

N/A — layer is a conductive layer.

The LDL CD results database contains these values in Via_I_Direction properties.

get_voltage_drop()

Returns the voltage drop across a parasitic resistor in volts. The return value is positive. However, values sufficiently close to 0 get truncated to 0.

No other functions may be used in the user_program block.

See “Current Density Checks” in the Calibre PERC User’s Manual.

Examples

Example 1

This shows both syntactical forms of the statement. The first statement sets a default value for layers other than metal1. Current densities above 0.5 appear in the LDL CD report for metal1 parasitic resistors. For other layers, the lower limit is 1.0.

PERC LDL CD CONSTRAINT VALUE 1.0
PERC LDL CD metal1 CONSTRAINT VALUE 0.5

Example 2

This example shows a constraint specification with a user program for interconnect layers:

PERC LDL CD poly CONSTRAINT CMACRO cd_macro
PERC LDL CD m1   CONSTRAINT CMACRO cd_macro
PERC LDL CD m2   CONSTRAINT CMACRO cd_macro
DMACRO cd_macro {[
CONSTRAINT val
  drop = get_voltage_drop() // voltage drop
  L = get_layer_name() 
  val = tvf_num_fun::cd_lib::calc_cd(drop, L)
]}
// assign constraints for metal and poly layers
TVF FUNCTION cd_lib [/*
  proc calc_cd {drop L} {
    if {[regexp -nocase {m[1-9]} [$L]]} { 
      if {[$drop] < 0.5} {
        return 0.2
      } else {
        return 0.5
      }
    }
    if {[string equal -nocase [$L] poly]} { 
      if {[$drop] < 0.2} {
        return 0.1
      } else {
        return 0.3
      }
    }
    return 0
  }
*/]

Example 3

The following example effectively waives any CD violation if the electromigration (EM) length is below 10 um (due to the return value of 1000000). It also effectively waives any CD violation if the EM length is >= 10 um, but the width is >= 5 um. For the remaining case, the current density limit is 0.5. Each layer should have a similar PERC LDL CD Constraint statement. 

PERC LDL CD layer1 CONSTRAINT CMACRO em_macro
DMACRO em_macro {[ 
  CONSTRAINT VALUE 
  EM_L = get_em_length() 
  EM_W = get_em_width() 
  VALUE = TVF_NUM_FUN::ldl_lib::em_constraint(EM_L,EM_W) 
]}
TVF FUNCTION ldl_lib [/* 
  proc em_constraint {EM_L EM_W} { 
    if {([$EM_L] >= 10) && ([$EM_W] < 5)} {return 0.5} 
    return 1000000 
  } 
*/]

In order to have EM data calculated, the perc_ldl::execute_cd_checks -em option must be used in the LDL CD rules. This program can be used in conjunction with the perc::export_pin_pair command’s ‑sink_voltage and -source_current options to perform EM calculations.
