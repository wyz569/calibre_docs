VNETIDMAX

Net-based vector property function for DFM expressions

Determines the tuple index of a maximum value of a specified slice in one vector and uses that index to return a corresponding element (a net ID) in another slice of another vector. Supported only in the DFM Property operation.

Usage

VNETIDMAX(vec1, slice1, vec2, slice2)

Arguments

Argument semantics are the same as for the four-argument VMAX function syntax with the exception that vec2 is a vector of net IDs rather than numeric values.

Return Values

Property of type netID.

Description

The function’s behavior is essentially the same as for VMAX(vec1, slice1, vec2, slice2). The differences are that in VNETIDMAX, vec2 is a vector of net IDs rather than numbers, and the function returns a net ID rather than a number.

Examples

In this example, the first operation generates a “net_ids” vector of net IDs for secondary layer polygons overlapping primary layer polygons. The net_ids vector has one slice. A “measures” vector of corresponding areas and perimeters of the secondary layer polygons is also generated. The measures vector has two slices. The second operation selects net IDs from the net_ids vector based upon the maximum perimeter values in the measures vector.

// put all interacting net ids and associated areas and perimeters of 
// overlapping secondary polygons onto a primary polygon

pri_vec_props = DFM PROPERTY pri sec1 sec2 OVERLAP
     [net_ids  = CONCAT(VNETID(NETID(sec1)),VNETID(NETID(sec2)))]
     [measures = CONCAT(VECTOR(AREA(sec1),PERIM(sec1)),
                        VECTOR(AREA(sec2),PERIM(sec2)))]

// select net ids based upon corresponding max perimeters

netid_by_perim = DFM PROPERTY pri pri_vec_props OVERLAP
      [max_netid_1_0 = VNETIDMAX(VPROPERTY(pri_vec_props,measures),1,
                                 NETVPROPERTY(pri_vec_props,net_ids),0)]
