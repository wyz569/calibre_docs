tvf::SETLAYER

Application: runtime and compile-time.

Used for layer derivation in both compile-time and runtime TVF. The command executes a layer operation and stores the output in a specified layer. Options are available in compile-time TVF to return a list of layers that were derived using the SETLAYER command and to return the layer operation expression for a given layer.

Usage

Syntax 1: Layer derivation, compile-time and runtime

tvf::SETLAYER layer_name = arguments

Syntax 2: Get layer names, compile-time only

tvf::SETLAYER ‑getnames {‑global | ‑local}

Syntax 3: Get layer expression, compile-time only

tvf::SETLAYER layer_name ‑getexpr

Arguments

layer_name

A required argument that specifies a layer name.

arguments

A required argument in Syntax 1 that forms a valid layer operation using the SVRF syntax. Some restrictions apply when used in runtime TVF, as noted in “Layer Operations in Runtime TVF”.

The arguments may be enclosed in double quotes or braces. Both of these grouping elements may appear in the same group of arguments. Double quotes (“ ”) allow for variable substitution in a list of arguments, whereas braces ({ }) pass their contents as a list with no variable substitution.

‑getnames {‑global | ‑local}

A required argument in Syntax 2 that causes the SETLAYER command to return the list of layers derived with SETLAYER commands. This option is used in Syntax 2 and is only available in compile-time TVF.

  • ‑global — Return global layers. Global layers are derived with SETLAYER commands occurring outside of tvf::RULECHECK commands.

  • ‑local — Return local layers. Local layers are derived with SETLAYER commands occurring within a tvf::RULECHECK command.

‑getexpr

A required argument in Syntax 3 that causes the SETLAYER command to return the layer operation used to derive layer_name. This option is used in Syntax 3 and is only available in compile-time TVF.

Return Values

The following values are returned:

Table 1. Returned Values from tvf::SETLAYER

Syntax

Command Option

Returned Value

Syntax 1, layer derivation

layer_name = arguments

layer name

Syntax 2, get layer names

‑getnames

layer list

Syntax 3, get layer expression

layer_name ‑getexpr

a string with the layer operation used to derive the layer

Description

Syntax 1, Layer Derivation

Executes the layer operations given by arguments and stores the output in layer_name. Used for layer derivation in both compile-time and runtime.

See the section “Layer Operations in Runtime TVF” for restrictions on the layer operations and syntax in runtime operation.

The layer operations specified to SETLAYER use the same syntax as corresponding SVRF layer operations, with some restrictions for runtime operation. Layer operations are passed as arguments to the SETLAYER command, and are then passed to the SVRF compiler.

Syntax 2, Get Layer Names

The SETLAYER command keeps track of layers defined both globally (outside a tvf::RULECHECK command) and locally (within a tvf::RULECHECK command.) When the ‑getnames option is used, the SETLAYER command returns a list of all the layer names that appear as the first argument of a SETLAYER command. This syntax is only available in compile-time TVF.

Syntax 3, Get Layer Expression

When used with the ‑getexpr option, the SETLAYER command returns the layer expression used to derive the specified layer. This syntax is only available in compile-time TVF.

Examples

Example 1, Layer Derivation

tvf::SETLAYER lout = ((L1 AND L2) OR "($L3 AND $L4)") AND {(L5 OR L6)}

Example 2, Layer Definitions and Assignments

The following layer definitions and assignments are global in scope in a compile-time TVF script: 

tvf::LAYER POLY    1
tvf::LAYER NPOLY   31 
tvf::LAYER PPOLY   32 
tvf::SETLAYER ALLPLY = {( POLY OR NPOLY ) OR PPOLY } 
# equivalent to ALLPLY = (POLY OR NPOLY) OR PPOLY

set GR1 3.4 
# usual tcl variable syntax

tvf::SETLAYER LARGEPLY = "SIZE ALLPLY BY $GR1" 
# double quotes allow variable substitution

Example 3, Get Layer Names

The following statement assigns to variable layerList a list of layer names that appear as the first argument of the SETLAYER commands occurring outside of RULECHECK commands.

set layerList [tvf::SETLAYER -getnames -global]

Example 4, Get Layer Expression

Suppose the layer ALLPLY is defined with this command:

tvf::SETLAYER ALLPLY = {POLY2 OR NPOLY2}

The following statement assigns the string “POLY2 OR NPOLY2” to the variable ALLPLY_expr.

set ALLPLY_expr [tvf::SETLAYER ALLPLY -getexpr]
