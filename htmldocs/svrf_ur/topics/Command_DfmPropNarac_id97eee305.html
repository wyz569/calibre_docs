NARAC

Function for DFM expressions

Accesses values on a Net Area Ratio Accumulate (NARAC) layer. Supported only in the DFM Property operation.

Usage

NARAC(layer)

Arguments

layer

A required Net Area Ratio Accumulate (NARAC) layer. NARAC layers can be derived with a Net Area Ratio operation with the ACCUMULATE keyword, a Net Area Ratio Accumulate operation, a Net Area Ratio Coupled operation, or a DFM NARAC operation.

Return Values

None.

Description

NARAC can be used in DFM Property expressions to access the accumulated values on net area ratio accumulation (NARAC) layers. This function is valid for primary and secondary layers, as well as in a per-shape context as an argument to SUM, VECTOR, and other per-shape functions. The NARAC function can be used only on NARAC layers; an attempt to use the NARAC function on any other layer results in a compilation error:

“layer must be the output of a NET AREA RATIO … ACCUMULATE operation”.

Examples

The following example accesses the accumulated values on the A_NARAC layer and shows how various per-shape functions can be used when A_NARAC is a secondary layer to control how the values are calculated:

CONNECT A
A_NARAC = NET AREA RATIO A >= 0 [ AREA(A) ] ACCUMULATE
A_PROP = DFM PROPERTY A_NARAC [ V = NARAC(A_NARAC) ]
B_PROP = DFM PROPERTY B A_NARAC OVERLAP 
         [ V_SUM = NARAC(A_NARAC) ]  // Implicit sum
         [ V_VEC = VECTOR(NARAC(A_NARAC)) ]
