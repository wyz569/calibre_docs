PEX Reduce CC

Parasitic extraction

Performs reduction of coupled capacitance on a net basis.

Usage

PEX REDUCE CC {ABSOLUTE threshold | RATIO percent |
{ABSOLUTE threshold [AND | OR] RATIO percent}} [SCALE scale]

Arguments

ABSOLUTE threshold

Keyword and threshold value. If the total coupling capacitance between a pair of nets is less than threshold, that coupling is applied as grounded capacitances to both nets. The threshold parameter is a positive floating point number scaled by the Unit Capacitance value.

For the Calibre xACT LEF/DEF extraction flow, the default value for threshold is 3 femtofarads. For the Calibre xACT GDS flow, the default value for threshold is 1 femtofarad. Note that setting the PEX Reduce Analog statement can also affect the default value for threshold. For Calibre xRC, the default value for threshold is 0 when PEX Reduce Analog, PEX Reduce Digital, or PEX Reduce CC statements are not specified.

RATIO percent

Keyword and percent value specified as a decimal number between 0 and 1. The total coupled capacitance between two nets is compared to the total capacitance for each net individually. If for both nets the coupled capacitance accounts for less than percent (expressed as a decimal) of the total net capacitance, that coupling is applied as grounded capacitances to both nets.

For the Calibre xACT LEF/DEF extraction flow, the default value for percent is 0.03. For the Calibre xACT GDS flow, the default value for percent is 0.01. Note that the PEX Reduce Analog statement can also affect the default value for percent. For Calibre xRC the default value for percent is 0 when PEX Reduce Analog, PEX Reduce Digital, or PEX Reduce CC statements are not specified.

AND | OR

Optional keywords that specify how to apply reduction based on the conditions defined by the ABSOLUTE and RATIO keywords. The default is AND.

SCALE scale

Optional keyword and value that applies the scale factor to the reduced capacitances before grounding. Only the coupled capacitances that are converted to grounded capacitance are scaled; the other capacitances are not modified. By default, scale is 1.0.

Description

This optional reduction statement controls reduction of coupled capacitance between nets based on total capacitance. The reduction occurs during the formatting stage before the PEX Reduce Digital TICER reduction. You can specify this statement once in your rule file.

You can combine the ABSOLUTE and RATIO options using the AND and OR keywords. Both conditions must be in the same statement; it is an error to specify the statement once for each. When both ABSOLUTE and RATIO options are specified and separated using the OR keyword, the coupled capacitance is grounded if either of the conditions are true. When both ABSOLUTE and RATIO options are specified using the AND keyword or no keyword, both conditions must be true before the coupled capacitance is grounded.

Examples

Example 1

The RATIO condition is met when the total coupling capacitance between two nets is less than the specified percent on both. For example, net A and net B are coupled by 26 fF. The total capacitance (intrinsic and coupled) on net A is 250 fF and on net B is 300 fF.

PEX REDUCE CC RATIO 0.1      //Ground cc < 10% of net

The A-B coupling will not be grounded because although 26 fF is less than 10% of net B’s 300 fF, it is greater than 10% of net A’s 250 fF.

Example 3

The PEX Reduce CC statement can be specified with both ABSOLUTE and RATIO keywords using the AND keyword:

PEX REDUCE CC ABSOLUTE 5 AND RATIO 0.1 // AND explicitly specified

or white space:

PEX REDUCE CC ABSOLUTE 5 RATIO 0.1 // AND keyword is the default

Both ABSOLUTE and RATIO conditions must be true before the coupled capacitance is grounded. Note, the PEX Reduce CC statement can only be specified once in your rule file.

Example 4

When both ABSOLUTE and RATIO keywords are specified and separated using the OR keyword, the coupled capacitance is grounded if either of the conditions are true:

PEX REDUCE CC ABSOLUTE 5 OR RATIO 0.1

Example 5

Specifying the PEX Reduce CC statement more than once in your rule file generates an error:

PEX REDUCE CC ABSOLUTE 5 
PEX REDUCE CC RATIO 0.1
