DFM Spec Via Redundancy

Specification statement

Defines a via redundancy analysis specification that can be referenced in a DFM Redundant Vias operation.

Usage

DFM SPEC VIA REDUNDANCY vra_spec_name
CONNECTIVITY
   layer1 layer2 [layer3 …]‘;’
   …
END CONNECTIVITY
[TREAT AS REDUNDANT [via_layer WITH AREA > area_value ‘;’] …
   [via_layer WITH ASPECT > ratio_value ‘;’] … END]

Arguments

vra_spec_name

A required argument that specifies a name for a via redundancy analysis specification that can be referenced in a DFM Redundant Vias operation.

CONNECTIVITY

A required keyword that specifies the beginning of the layer connectivity definitions.

layer1 layer2 [layer3 …]‘;’

A required argument set that specifies a layer connectivity definition, which indicates that an electrical connection exists between overlapping shapes on the specified layers. Each layer in the definition must be a Connect layer if the definition is used with DFM Redundant Vias in HIERARCHICAL mode. At least two layers must be specified; the layers must be listed in the order that they occur in the layer stack, that is from bottom to top or from top to bottom.

Any number of unique connectivity definitions may be specified between the CONNECTIVITY and END CONNECTIVITY directives. Each definition must be terminated with a semicolon, which must be separated from the last layer defined with a space. In cases where the connectivity definitions are a subset of the complete Connect sequence, performance can be improved by using a reduced Connect sequence that includes only the layers specified for the connectivity definitions.

Layer definitions that have a common layer in the first position of one definition and in the last position of another definition can be joined together. For example, the following definitions:

CONNECTIVITY
	M1 VIA1 ;
	VIA1 M2 ;
	M2 VIA2 ;
	VIA2 M3 ;
END CONNECTIVITY

are equivalent to:

CONNECTIVITY
	M1 VIA1 M2 VIA2 M3 ;
END CONNECTIVITY

Layers that are concurrent in the stack, such as PO and OD, must be specified using separate definitions. In the following example, the three sets of definitions are equivalent.

CONNECTIVITY
PO CO M1 ;
OD CO M1 ;
M1 VIA1 M2 VIA2 M3 ;
END CONNECTIVITY
// or
CONNECTIVITY
PO CO M1 ;
OD CO M1 VIA1 M2 VIA2 M3 ;
END CONNECTIVITY
// or
CONNECTIVITY
PO CO M1 VIA1 M2 VIA2 M3 ;
OD CO M1 ;
END CONNECTIVITY
END CONNECTIVITY

A required keyword that specifies the end of the layer connectivity definitions.

TREAT AS REDUNDANT

An optional keyword that specifies the beginning of via categorization specifications. This keyword can be used to identify vias that would otherwise be treated as single vias to be categorized as redundant vias which are less prone to failure. There are two types of via categorization: by area and by shape aspect.

  • via_layer WITH AREA > area_value ‘;’

    An optional keyword and argument specification for categorization by via area. The rule specifies that if the area of a via is greater than the specified area_value, then the via is not categorized as a single via (treated as redundant). You can make multiple specifications as needed with each occurrence adding another via categorization by area. Each specification must be terminated with a semicolon (;) which must be separated from the area_value with a space.

    The specified arguments for categorization by via area are defined as follows:

    • via_layer — Specifies the via layer.

    • area_value — Specifies the via area in user units.

  • via_layer WITH ASPECT > ratio_value ‘;’

    An optional keyword and argument specification for categorization by via shape aspect which is defined by the ratio of the longer side to the shorter side. The rule specifies that if the aspect ratio of the via shape is greater than the specified ratio_value, then the via is not categorized as a single via (treated as redundant). You can make multiple specifications as needed with each occurrence adding another via categorization by shape aspect. Each specification must be terminated with a semicolon (;) which must be separated from the ratio_value with a space.

    The specified arguments for categorization by via shape aspect are defined as follows:

    • via_layer — Specifies the via layer.

    • ratio_value — Specifies the shape aspect ratio of the via.

  • END

    A keyword that must be specified at the end of the via categorization specifications. It is used in conjunction with the TREAT AS REDUNDANT keyword.

Description

Defines a via redundancy analysis specification that can be referenced in a DFM Redundant Vias operation.

Examples

Example 1

In this example, the via redundancy analysis specification “m1m2_connectivity” is referenced in the following DFM Redundant Vias operation.

CONNECT M1 M2 BY VIA1
DFM SPEC VIA REDUNDANCY m1m2_connectivity
CONNECTIVITY
	M1 VIA1 M2 ;
END CONNECTIVITY
m1m2_redundant_via1 = DFM REDUNDANT VIAS VIA1 m1m2_connectivity

Example 2

In this example, vias on the V1 layer with area greater than 0.02 (user units) or aspect ratio greater than 1 are not output by the single_V1 rule. They are treated as redundant and output by the redundant_V1 rule.

DFM SPEC VIA REDUNDANCY V1_Spec
CONNECTIVITY
    M1 V1 M2 ;
END CONNECTIVITY
TREAT AS REDUNDANT
  V1 WITH ASPECT > 1 ;
  V1 WITH AREA > 0.02 ;
END
single_V1 {
      DFM NON REDUNDANT VIAS V1 V1_Spec HIERARCHICAL
}
redundant_V1 {
      DFM REDUNDANT VIAS V1 V1_Spec HIERARCHICAL
}
