LVS Gray Box Not Port

Specification statement

Specifies Connect or Sconnect layers that cannot form pins in LVS Box GRAY cells.

Usage

LVS GRAY BOX NOT PORT layer [layer ...]

Arguments

layer

A required Connect or Sconnect layer. More than one layer may be specified.

Description

Specifies connectivity layers in gray box cells that cannot form pins in those cells.

In some LVS Box GRAY flows, it may be necessary to disallow certain connectivity layers within gray box cells from forming connections with layers outside those cells. Excluding such layers prevents polygons from these layers from forming pins in gray box cells.

Consider the following rule file excerpt:

LVS BOX GRAY gb
// gb_pin is a pin layer; met is an interconnect layer outside cell gb
CONNECT gb_pin met

Ordinarily, any polygons in a gray box cell on layer gb_pin form pins when the polygons touch polygons on layer met, such as in this figure:



If this is specified in the rules:

LVS GRAY BOX NOT PORT gb_pin

then no pin is formed in the preceding layout.

Examples

LAYER m2 12
LAYER v2 13
LAYER m3 14

TEXT LAYER m2 m3 // text_layers

CONNECT m2 m3 BY v2

LVS BOX GRAY ram 
// do not form pins from layer m2 in gray box cells
LVS GRAY BOX NOT PORT m2
