PEX LEF Extract Cell Obstructions

Parasitic extraction

Controls whether obstruction geometries are treated as layout objects and included as ground when calculating parasitic capacitance.

Usage

PEX LEF EXTRACT CELL OBSTRUCTIONS {NO | YES}

Arguments

NO

Specifies not to include obstruction geometries during extraction. This is the default behavior.

YES

Specifies to include obstruction geometries.

Description

Controls whether obstruction geometries are treated as layout objects and included as ground when calculating parasitic capacitance. Obstruction geometries are used in LEF files to block routing. By default, Calibre xRC does not include obstructions in extraction.

If you turn on short and/or open detection in Calibre xACT with PEX Report Opens and PEX Report Shorts, and you have specified PEX LEF Extract Cell Obstructions YES, Calibre xACT also detects shorts, opens, and minspacing violations based on LEF obstruction shapes.

Examples

Example 1

To include obstruction geometries during extraction, put the following statement in the rule file:

PEX LEF EXTRACT CELL OBSTRUCTIONS YES

Example 2

To include obstruction geometries, and to detect and report shorts during extraction with Calibre xACT, include the following statements in the rule file:

PEX LEF EXTRACT CELL OBSTRUCTIONS YES
PEX REPORT SHORTS my_shorts.report
