Layout Clone By Polygon

Specifies to clone cells during the hierarchical database construction phase based upon interaction with a polygon in the specified clone set.

Usage

LAYOUT CLONE BY POLYGON x1 y1 x2 y2 [… xN yN] clone_set

Arguments

x1 y1 x2 y2

A required set of floating-point coordinates in top-level space that represent the vertices of a polygon in user-units. At least two pair of non-coincident coordinates are required. Exactly two pair of coordinates are interpreted as the lower-left and upper-right corners (they must be specified in that order) of a rectangle having sides parallel, respectively, to the coordinate axes. If more than two pair of coordinates are specified, and the final pair do not represent the same vertex, then they are considered consecutive vertices of the polygon. These coordinates may be numeric variables, as specified in a Variable statement.

The polygon must be oriented either clockwise or counter-clockwise when more than one pair of vertices are specified. The polygon must be simple; that is, it may not self-intersect.

clone_set

A required non-negative integer constant that defines a layer on which all of the specified polygons are placed and merged. This layer number is internal to the set of Layout Clone By Polygon statements and has no dependency on the Layer statement.

Description

Specifies that cells are cloned during the internal hierarchical database construction phase of Calibre applications based upon interaction with a polygon in the specified clone_set. The statement is primarily useful in post-tapeout EUV flows.

Each distinct clone_set in the totality of Layout Clone By Polygon statements is treated as a layer with all of the specified polygons placed on the layer and merged.

This statement works as follows for each of the polygons, in the order specified, and in the same clone_set.

  1. All geometry in the clone_set is temporarily flattened into the top-level cell.

  2. From the top-level down recursively, all placements that cut (share a portion of their area) flattened geometry on the clone_set are expanded. After this step, all cell placements from the flat (top-level) point of view are either completely inside or outside of the clone_set geometry.

  3. Cells which, from the flat point of view, have placements both inside of and outside of geometry in the clone_set are then cloned into two distinct cells: one cell has all placements inside of the clone_set geometry, and the other cell has all placements outside of the clone_set geometry. Each cloned cell name is the original cell name with a string of the form $Pnumber$ appended to the name.

The process is repeated for the next distinct clone set, ordered from lowest to highest.

Examples

Example 1

In this example there are four different clone sets that are processed in order from lowest to highest.

LAYOUT CLONE BY POLYGON 67 6 1500 90 1
LAYOUT CLONE BY POLYGON 1500 6 3000 90 2
LAYOUT CLONE BY POLYGON 3000 6 5000 90 3
LAYOUT CLONE BY POLYGON 5000 6 6000 90 4

Example 2

In this example clone set 10 is comprised of two polygons that are placed on the same layer and merged.

LAYOUT CLONE BY POLYGON 50 50 2000 100 200 0 400 300  10
