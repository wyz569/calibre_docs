#UNDEFINE

The #UNDEFINE pre-processor directive undefines variables in the SVRF rule file. The variables are used to control conditional execution of portions of the rule file.

Usage

#UNDEFINE var_name

Arguments

var_name

A string giving the variable name. You should enclose var_name within quotes if var_name includes spaces or characters other than letters and numbers.

If the #UNDEFINE statement is used to undefine a variable defined in the shell, the var_name should have an initial dollar sign ($).

Description

A variable specified by the #DEFINE or #UNDEFINE statements can be used in an #IFDEF or #IFNDEF statement to control conditional execution of rule file statements. See “About Conditional Directives for Compilation Control” for complete information on using the conditional directives, including the use of variables defined in the shell and information on compilation order.

#UNDEFINE statements only affect the rule file pre-processor; they do not affect the shell environment. Do not comment out a #DEFINE statement to change the state of a conditional statement; use the #UNDEFINE statement.

Examples

This example undefines a variable defined with a #DEFINE statement in the rule file:

#UNDEFINE process  //undefines the variable defined in the rule file

The following example undefines a variable defined in the shell, for example with “setenv process”:

#UNDEFINE $process //undefines the variable defined in the shell

The two preceding examples do not undefine the same names.
