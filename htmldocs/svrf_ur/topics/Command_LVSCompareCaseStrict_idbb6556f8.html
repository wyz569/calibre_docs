LVS Compare Case Strict

Specification statement

Controls enforcement of SPICE netlist case sensitivity settings in the rule file. Used in Calibre nmLVS/Calibre nmLVS-H, Calibre PERC, and DFM-type runs.

Usage

LVS COMPARE CASE STRICT {NO | YES}

Arguments

NO

A keyword that specifies no validation of netlist case sensitivity settings is performed. This is the default.

YES

A keyword that specifies if LVS Compare Case YES is set, then Layout Case and Source Case must also be set to YES, or the run aborts.

Description

Controls the enforcement of rule file statements affecting the case-sensitive handling of SPICE netlists.

The LVS Compare Case YES setting by itself does not cause netlists to be treated in a case-sensitive manner, but it does have certain other side-effects that can be desirable on their own, such as case-sensitive treatment of hcell lists.

In certain flows, ensuring that netlists are parsed in a case-sensitive manner is desirable whenever LVS Compare Case YES is set. For example, in Calibre PERC, setting LVS Compare Case Strict YES is useful to prevent checks from behaving unexpectedly due to case sensitivity mismatch issues.

For additional information, see “Case-Sensitive Handling of Names” in the Calibre Verification User’s Manual or “Case Sensitivity for Names” in the Calibre PERC User’s Manual.

Examples

Example 1

// LAYOUT CASE and SOURCE CASE unspecified (both NO by default).
// Netlists are NOT case-sensitive with this setting alone:
LVS COMPARE CASE YES

Example 2

// Force complete case-sensitive treatment of netlists.
LVS COMPARE CASE YES
LVS COMPARE CASE STRICT YES

// Run aborts with these settings:
SOURCE CASE NO // or unspecified
LAYOUT CASE NO // or unspecified
