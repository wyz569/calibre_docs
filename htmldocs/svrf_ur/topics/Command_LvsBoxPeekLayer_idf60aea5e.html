LVS Box Peek Layer

Specification statement

Specifies original layers in LVS Box BLACK and GRAY cells that can interact with layers and participate in layer derivations in the cells’ super-hierarchies.

Usage

LVS BOX PEEK LAYER layer [layer …]

Arguments

layer

A required original layer name that appears in an LVS Box BLACK or GRAY cell. More than one layer may be specified.

Description

Specifies layers within LVS Box BLACK or GRAY cell hierarchies that are permitted to participate in forming derived layers outside the box cells. Typically well or substrate layers are used in LVS Box Peek Layer. The statement resolves some issues with black or gray box flows that demonstrate extra text shorts that are not present in regular LVS Box or non-box flows.

Recall that black and gray box cells are processed independently of their placement context, so the layer shapes that are inside the box cells are not used for layer derivations that happen outside the box cells. For example, consider this rule file excerpt:

LAYER C 1
LAYER D 2
E = D NOT C // in a black or gray box cell, layer C is not present here,
            // so E becomes essentially a copy of D
CONNECT E

together with this layout:



The shape on layer C, which resides anywhere inside the gray box cell, does not contribute to the derivation for layer E. So layer E is essentially a copy of layer D at the top level. As a result, the net labels “A” and “B” on layer E will be shorted at the top level.

By adding a LVS Box Peek Layer statement for layer C to the preceding example:

LVS BOX PEEK LAYER C // C is processed in a black or gray box cell
LAYER C 1
LAYER D 2
E = D NOT C         // layer C is used, so layer E is split
CONNECT E

This is the outcome:



The peeked layer C in the gray box cell is now processed in the parent cell, and layer C affects the derivation of layer E at the top level. Net labels “A” and “B” are no longer shorted.

See also LVS Box Cut Extent.

Examples

Assume that the substrate layer needs to be permitted in layer derivations in order to avoid shorts in layers in the super-hierarchy of ramcell:

LAYER substrate 1

LVS BOX GRAY ramcell
LVS BOX PEEK LAYER substrate // allow this layer to interact outside
                             // the box cell
