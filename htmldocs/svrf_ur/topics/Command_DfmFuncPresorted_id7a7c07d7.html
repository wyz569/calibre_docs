PRESORTED

Net-based vector property function for DFM expressions

Given an input property vector of sorted netID values, returns a vector that is sorted at the hierarchy level in which the expression evaluation takes place.

Usage

PRESORTED( NETVPROPERTY(layer, property_name, ordinal) )

Arguments

NETVPROPERTY(layer, property_name, ordinal)

A required argument that must be a NETVPROPERTY function call that accesses a sorted vector of netIDs. The ordinal argument is required. If layer is the primary layer in a DFM Property operation, specify an ordinal of 1.

Vectors can be sorted with the SORT and SORT_UNIQUE functions.

Return Values

A vector of netIDs.

Description

Given an input property vector of sorted netID values, returns a vector that is sorted at the hierarchy level in which the expression evaluation takes place. The sort process is done in place, meaning that subsequent accesses of the netID property vector return sorted values.

For the FIND_SORTED function, which requires a sorted input vector, the PRESORTED function provides an efficient way to insure that a sorted netID property vector is still sorted when promoted up the hierarchy. Promotion can change the netID values, so that a sorted netID property vector may no longer be sorted after promotion. PRESORTED should be used whenever FIND_SORTED searches a netID vector property in order to prevent search failures due to non-sorted input vectors. For flat hierarchies, PRESORTED is not necessary, but can still be used. PRESORTED causes a negligible performance impact when the input vector is not promoted.

Examples

This example demonstrates the use of PRESORTED when using FIND_SORTED on a netID vector property. 

DRC RESULTS DATABASE "drc_results"
DRC SUMMARY REPORT "drc_summary"
LAYOUT PRIMARY TOPCELL
LAYER base 0
LAYER top_layer 1  // shapes at top level
LAYER layer2 2
// Assign node numbers to base polygons
CONNECT base

// Create a sorted vector of base netIDs that overlap layer2 shapes.
second = DFM PROPERTY layer2 base OVERLAP MULTI
							 [ V = SORT( VNETID( NETID(base) ) ) ]
// possible layer operations that promote the layer second
// Use  FIND_SORTED() with PRESORTED(). 
// For the first base shape that overlaps the second layer, find
// the ordinal of the netID in the netID vector property V.
// Finding the COUNT(top_layer) causes second layer to be promoted  
find_ordinal = 
	DFM PROPERTY second base top_layer OVERLAP MULTI ABUT ALSO
		[ - = COUNT(top_shapes) ] > 0
		[ netid_1 = NETID(base, 1) ]
		[ find_index_1 = FIND_SORTED( NETID(base, 1),
										PRESORTED( NETVPROPERTY(second, V, 1) ) )]
  // Because PRESORTED() performs in-place sorting for promoted 
  // netID property vectors, the following property is in sorted order.
		[ V_read = NETVPROPERTY(second, V, 1) ]

Layer operations that promote the layer holding the netID vector (second in this example) can cause the netID vector to no longer be sorted. If this happens, the return value of the FIND_SORTED function is not defined. The use of PRESORTED prevents this problem.
