PEX Extract Cgate

Parasitic extraction

Specifies how to netlist the gate coupling capacitance for transistors. Use only with the Calibre xACT or Calibre xACT 3D direct netlisting flow.

Usage

PEX EXTRACT CGATE {ALL | gatelayername [gatelayername …]}
COUPLING {CENTER | GATEENDS}

Arguments

ALL

A required parameter that specifies to apply the gate coupling capacitance calculation settings to all transistor gate layers. This is the default.

gatelayername

A required parameter that specifies the transistor gate layer. You can specify more than one gate layer name.

COUPLING {CENTER | GATEENDS}

An optional keyword set that controls how coupling capacitance is netlisted for transistors. The CENTER keyword specifies to netlist coupling capacitance from the center of the gate. The GATEENDS keywords specifies to netlist coupling capacitance from both ends of the gate. The default is COUPLING CENTER.

Description

An optional statement that specifies whether coupling capacitance is netlisted from the center of the gate or from the ends of the gate. If this statement is not specified, the coupling capacitance is netlisted from the center of the gate for all layers. This statement can be specified more than once, but only once per gate layer. This statement can be used with PEX Extract Rgate statement.

Examples

Example 1

To netlist coupling capacitance from the both ends of geometries on layer gate1 include the following statement in your rule file:

PEX EXTRACT CGATE gate1 COUPLING GATEENDS

The remaining gates are netlisted from the center, which is the default.

Example 2

To netlist coupling capacitance from the center of gate1 and the gate ends of all remaining gate layers, include the following statements in your rule file:

PEX EXTRACT CGATE ALL COUPLING GATEENDS
PEX EXTRACT CGATE gate1 COUPLING CENTER

Example 3

To netlist coupling capacitance from the both ends of gate1 and gate2, include the following statement in your rule file:

PEX EXTRACT CGATE gate1 COUPLING GATEENDS
PEX EXTRACT CGATE gate2 COUPLING GATEENDS

The following statement syntax produces equivalent results:

PEX EXTRACT CGATE gate1 gate2 COUPLING GATEENDS

Example 4

In this example, extraction stops with an error when gate1 is specified in more than one PEX Extract Cgate statement. For example: 

PEX EXTRACT CGATE gate1 gate2 COUPLING GATEENDS
PEX EXTRACT CGATE gate1 COUPLING CENTER

Example 5

This example demonstrates the effects of the PEX Extract Cgate statement on the output netlist. The following statement represents the default behavior:

PEX EXTRACT CGATE ALL COUPLING CENTER

Here is a sample taken from an extracted netlist with reduction off (PEX Reduce Analog NO):

...
cc_22 M0:s M0:g 0.068f // layer1=nsrcdrn layer2=ngate
cc_72 M0:d M0:g 0.068f // layer1=nsrcdrn layer2=ngate
r9 M0:g InvIn:12 41 $layer=ngate
r10 M0:g InvIn:13 41 $layer=ngate
...

To netlist the coupling capacitance from both ends of all the gates instead of the center, specify the following statement:

PEX EXTRACT CGATE ALL COUPLING GATEENDS

The resulting sample output is:
