DEBUG

Device property computation language function

Enables debug monitoring during device property computation.

Usage

DEBUG [CELL=name] range1[, range2 …]

Arguments

range1

A required device instance ID number or a pair of device instance numbers separated by a hyphen (‑). If the hyphenated form is used, the first number must be less than the second number. In that case, all numbers between and including the numbers specified in the range correspond to instances. At least one range must be specified, although it can contain only a single instance. Multiple ranges are separated by a comma (,) followed by whitespace.

CELL=name

An optional keyword and cell name that causes the function to apply only to the specified cell.

Return Values

None.

Description

Facilitates property computation debugging. This function can be useful in finding errors during the development of new code. If used, DEBUG must be the first statement in the program. The CELL option makes the DEBUG function cell-specific, and specification of the function more than once can then be useful.

The debugging trace information is written to the transcript, such as this:

************************************************
ERROR: Numeric error during property initiation.
   Rule file: rules
   Device: mn  (line 84 of rule file)
   Division by zero (result set to zero).
************************************************
Note:

The debug trace can produce much output, so the use of small ranges is recommended.

Unique instance ID numbers can be determined from the LVS Report in a flat run, such as this:

1    0(5.375,14.000)  mn 
     l: 0.625 u 
     w: 30 u

Here the instance ID number is 0.

The instance function can also be used to write the instance ID numbers to the extracted netlist as a property, like this:

DEVICE ... [
  PROPERTY I, L, W ...
  I = INST()
...

In a hierarchical run, the LVS Report does not show ID numbers, so the only option is to use the instance() function in the relevant Device statement’s property computation. The instance IDs generated by the instance() function are not unique but are generated per-cell. Hence, debugging an instance ID hierarchically will result in trace output on a per-cell basis for all instances having the specified ID number.

Examples

Consider the following DEVice statement and TVF Function block:

DEVICE MP (pch) pgate pc_i psd_s psd_s nw
[
DEBUG CELL=mgc_inv8 3-5 
DEBUG CELL=mgc_oai21 2
PROPERTY W,L,AG
W = perimeter_coincide(pgate,psd_s) / 2
L = perimeter_outside(pgate,psd_s) / 2
AG = tvf_numeric_function::device_functions::effective_a( W, L )
]

TVF FUNCTION device_functions [/*
  proc effective_a { W L } {
    if { [device::debug] } {
      puts "TVF Debug info: [$W] [$L]"
    }
    return [ expr { [$W] * [$L] } ]
  }
*/]

Debug output is produced in the cell mgc_inv8 for device indices 3-5 and for cell mgc_oai21 for device index 2. In addition, the TVF Function prints the string “TVF Debug info:” followed by the values of the W and L parameters for all reported devices.
