DFM Size

Layer operation

License: Calibre YieldEnhancer, except in LVS runs, which use a Calibre ADP (Advanced Device Properties) license.

Oversizes or undersizes the polygons on the input layer by the amount specified. The size value is specified by a DFM property attached to the input layer polygons.

Usage

DFM SIZE layer1 BY size_property
[ OVERLAP ONLY | INSIDE OF layer2 | OUTSIDE OF layer2 ]
[ STEP step_property ]
[ BEVEL value] [TRUNCATE distance_multiplier ]

Arguments

layer1

A required polygon layer that must have an attached DFM property size_property or step_property, or both. Such layers are typically created with a DFM Property or DFM Property Select Secondary operation.

BY size_property

A required keyword set that specifies the value in user units by which to resize each polygon on the input layer. size_property is the name of the DFM property on layer1 that contains the size value for each polygon. Alternatively, you may specify size_property as a constant numeric value or numeric variable defined by a Variable statement.

Positive values expand each polygon, negative values shrink each polygon, and zero values leave each polygon unchanged.

OVERLAP ONLY | INSIDE OF layer2 | OUTSIDE OF layer2

An optional argument choice that restricts the output.

  • OVERLAP ONLY — Specifies that the output consists only of regions where the oversized polygons overlap, not the oversized polygons themselves. This is overlap of distinct oversized polygons.

  • INSIDE OF layer2 — Restricts polygons on layer1 from expanding outside the boundary of polygons on layer2, where layer2 is an original layer or layer set, or a derived polygon layer. See “INSIDE OF Behavior”.

    The following is true for both oversizing and undersizing with INSIDE OF:

    • Polygons on layer1 that are outside all layer2 polygons are not resized and are excluded from the output layer.

    • Polygons on layer1 that are not outside layer2 polygons are clipped at the boundary of the layer2 polygons before resizing.

    • The output polygons are always constrained to be within the initial enclosing layer2 polygon.

  • OUTSIDE OF layer2 — Restricts polygons on layer1 from intruding inside the boundary of polygons on layer2, where layer2 is an original layer or layer set, or a derived polygon layer. See “OUTSIDE OF Behavior”.

    The following is true when OUTSIDE OF is specified:

    • Polygons on layer1 that are inside layer2 polygons are not resized and are not included in the output layer.

    • Polygons on layer1 that are not inside a layer2 polygon are clipped at the boundary of any interacting layer2 polygon, and only the portion outside the layer2 polygon is resized.

    • Resized polygons are constrained to remain outside all layer2 polygons.

STEP step_property

An optional keyword set that specifies to perform the resizing in multiple steps. step_property is the name of the DFM property on layer1 that contains the step value, which must be non-negative for both under and oversizing. Alternatively, you may specify a constant numeric value or numeric variable for step_property. Resizing is performed incrementally in steps of size step_property.

If this keyword is not specified, or the value of step_property for a particular polygon is zero, the polygon is resized by the entire amount specified by the value of size_property in a single step. If this keyword is used with one of the bounding keywords (INSIDE OF or OUTSIDE OF) the bounding layer constraint is applied after each resizing step.

BEVEL value

An optional keyword set that causes orthogonal corners (two edges with an interior angle of 90 degrees) on layer1 to be approximated by segmented arcs on the output layer. The value argument is a positive integer that specifies the number of segments in the arc. The BEVEL value is ignored for polygons with a negative size value. BEVEL may not be specified with OVERLAP ONLY.

Refer to Size for a description of BEVEL.

TRUNCATE distance_multiplier

An optional keyword set that controls truncation of spikes. The distance_multiplier argument is a dimensionless, non-negative floating-point value. The truncation distance is the absolute value of size_property multiplied by distance_multiplier. The default value is 1/cos(67.5), which is approximately 2.61.

Refer to Size for a full description of truncation.

Description

Expands or shrinks polygons on layer1 by specified amounts. Positive expansion values expand polygon edges outward by that amount. Negative expansion values shrink polygon edges inward by the absolute value of that amount. This operation is similar to Size, however, DFM Size can take size and step values from DFM properties attached to the polygons on layer1, which means the values can be different for each polygon. At least one of the parameters size_property and step_property must be specified as a DFM property.

INSIDE OF Behavior

When INSIDE OF is specified, the layer1 input polygons are clipped to the inside of the layer2 boundary layer before resizing, then clipped again after resizing. The sized output is always constrained to be within the initial enclosing layer2 polygon. If STEP is specified, the clipping takes place at each resizing step. This behavior differs from that of the Size operation, which does not clip the input polygons before the initial resizing, and allows an oversized polygon result to expand into a nearby layer2 polygon.

When using Size, a small STEP value can be used to prevent expansion into nearby layer2 polygons—this is not necessary when using DFM Size. Also, unlike the Size operation, negative values of size_property are allowed when using INSIDE OF and OUTSIDE OF.

OUTSIDE OF Behavior

When OUTSIDE OF is specified, the layer1 input polygons are clipped to the outside of the layer2 boundary layer before resizing, then clipped again after resizing. The result is always constrained to be outside all layer2 polygons. If STEP is specified, the clipping takes place at each resizing step. This behavior differs from that of the Size operation.

Examples

This example demonstrates how to create an input layer with DFM Property and use it in a DFM Size operation. The minimum edge length of the input polygon is used as the size value. 

// expand each polygon on layer m1 by its width in one step
m1_prop = DFM PROPERTY m1 [ min_len = MIN(LENGTH(m1:EDGE)) ]
m1_expand = DFM SIZE m1_prop BY min_len
