Circuit Extraction Messages

This section describes runtime messages from circuit extraction such as occurs with calibre ‑spice.
Table 1. Circuit Extraction Messages

Explanation

ERROR: The unconditional must-connect condition between the following instance pins has not been satisfied.

 

Pins of an instance that belong to a single unconditional must-connect group in the instance's template are not connected in the cell that contains the instance and cannot be connected at any higher level. The pin names are reported in the form: instance_name.pin_name.

ERROR: The conditional must-connect condition between the following instance pins has not been satisfied.

 

Pins of an instance that belong to a single conditional must-connect group in the instance's template are not connected in the cell that contains the instance, and cannot be connected at any higher level. This error is reported only if a connection was made to one of the pins. The pin names are reported in the form: instance_name.pin_name.

ERROR: A must-connect condition exists on instance pins but is not defined for the corresponding ports.

 

Pins of an instance that belong to a single must-connect group in the instance's template and that are not connected in the containing cell, are connected instead to ports of the cell, but the ports do not belong to a single must-connect group in the cell. The instance pin names in the form instance_name.pin_name and the corresponding port names are reported.

ERROR: Can not determine MASK SVDB directory from rule file name for extracted netlist output. Specify MASK SVDB directory or use “-spice filename” command line option.

 

The Layout System is geometric and calibre -lvs -flatten or -hier has been specified on the command line. In order to extract the layout SPICE netlist, either the rule file must have a Mask SVDB Directory statement in it, or the ‑spice command line option must be used.

ERROR: Correspondence “cell” “cell” leads to a many-many correspondence.

 

Specification of hcells in the rule file or -hcell list has caused an ambiguous many-to-many mapping of cell names.

ERROR: -genhcells requires SOURCE SYSTEM SPICE and a valid SOURCE PATH in the rulefile.

 

The -genhcells command line option is used but Source System SPICE or Source Path is unspecified in the rules.

ERROR: Hierarchical extraction with flat comparison is not supported (use ‑hier switch).

 

The -spice option may not be specified with -lvs but without -hier (or -flatten).

Error: LVS GENERATE HCELLS -genhcells failed.

 

The -genhcells option is specified but automatic generation of hcells failed. See the transcript for details.

ERROR: Numeric error during property evaluation.

 

Rule file: rules

Device Layer: type (line N of rule file)

Cell: cell

error type

A Device Layer property annotation program has performed an impossible calculation. The error type describes the discrepancy. The remaining information identifies the layout device type and Device Layer statement in the rule file that generated the error.

ERROR: Numeric error during property initiation.

 

Rule file: rules

Device: type (line N of rule file)

error type

A Device property computation has performed an impossible calculation. The error type describes the discrepancy. The remaining information identifies the layout device type and Device statement in the rule file that generated the error.

ERROR: Power or ground short detected. ABORT ON SUPPLY SHORT is specified - aborting.

 

LVS Abort On Supply Short YES is specified and a short has been detected involving an LVS Ground/Power Name net.

Error: The following products could not be licensed sufficiently:

Error: - LVS Device DFM Properties

 

The DFM Property operation is used to calculate Device properties. This requires a Calibre ADP license.

Error: The following products could not be licensed sufficiently:

Error: - LVS Device TVF

 

There is a TVF FUNCTION block that requires a Calibre ADP license.

Error: Property “prop” not found on device instance of type in cell cell when grouping total property.

 

The -total option is specified with erc::setup_device_parameters and a device instance is found that lacks a grouping property. The instance is skipped.

ERROR: Results generated by one or more ERC checks with ABORT LVS - aborting.

 

An ERC Select Check statement uses the ABORT LVS keyword set, and an ERC check has produced results. LVS comparison aborts. Check the ERC Results Database and ERC Summary Report for information.

ERROR: SCONNECT conflicts detected. ABORT ON SOFTCHK is specified - aborting.

 

LVS Abort On Softchk YES is specified and Sconnect conflicts have been detected. See the circuit extraction report for details.

ERROR: Supply error detected. ABORT ON SUPPLY ERROR is specified ‑ aborting.

 

LVS Abort On Supply Error YES is specified and a supply error has been detected. See the circuit extraction report for details.

ERROR: Short isolation failed - no short found. (Virtual connections were present.)

 

LVS Isolate Shorts YES is specified but shorts cannot be traced because virtual connections are specified.

ERROR: Unable to access file: filename No such file or directory

 

The -genhcells option is specified, but the generated hcell file cannot be found.

WARNING: BAD DEVICE on layer layer at location (x,y) in cell name

 

A device has been detected that nearly matches a Device statement, but is somehow defective. The device seed layer, location, and cell name are reported. This is followed by a listing of pin layer interactions with the seed shape and possible device element names that the bad instance might represent.

A related set of warnings appears in the circuit extraction report file and also in the respective subcircuit in the extracted SPICE netlist, where it is preceded with “**”.

WARNING: Black/grey box flows require CONNECT and DEVICE statements.

 

The BLACK or GRAY options for LVS Box require Connect and Device statements in the rules.

WARNING: BPF file for DEVICE seed layer layer contains NET IDs, not DEVICE IDs.

 

A Binary Polygon Format layout input is used and a Device seed layer is also a connectivity layer. The net ID is preserved but the device ID is discarded.

WARNING: Call for value of property “property” on layer layer will return 0.0 -- property not found.

 

A function call has been made in a device annotation block to a property that does not exist on the reported layer. The value 0.0 is returned by the function.

WARNING: Cannot enable LVS PUSH DEVICES because pin location information was requested.

 

LVS Push Devices YES is specified while the Mask SVDB Directory PINLOC option is set (the CCI option enables this by default).

WARNING: Cannot establish connectivity of this layer in the MASK connectivity set: layer.

 

A layer argument is specified that requires the layer to have connectivity, but the connectivity does not exist. The layer needs to appear in a Connect, Sconnect, or Stamp operation, or it needs to be derived from a node-preserving layer operation that references a connectivity layer as its first argument.

WARNING: Error reporting limit exceeded.

n missed connections not reported.

 

A Stamp operation’s limit of 100 reported errors is exceeded. The number of additional missed connections that are unreported is given in the message.

WARNING: DFM PROPERTY access failed for DEVICE type, seed layer:layer SEPARATE PROPERTY ANNOTATION.

 

An auxiliary layer shape is missing properties requested by a function in a device property annotation block.

WARNING: HIERARCHICAL SHORT ISOLATION is not performed inside box cell name

 

LVS Isolate Shorts YES and LVS Box have been specified in a hierarchical run, but box cell contents are unavailable for short isolation.

WARNING: LAYOUT CELL <name> conflicts with a DEVICE with the same name and was renamed to __<name> in the layout netlist.

 

A Device statement defines a non-standard SPICE device whose name conflicts with a cell name. The cell name is modified in the extracted netlist to avoid duplicate subcircuit names.

WARNING: LVS BOX cell “name” not located or not allowed.

 

An LVS Box statement specifies a cell name that cannot be identified in the design.

WARNING: LVS PUSH DEVICES SEPARATE PROPERTIES cannot be enabled because LVS PUSH DEVICES NO was specified.

 

Change LVS Push Devices NO to YES when using the SEPARATE PROPERTIES YES option.

WARNING: Port contains unconnected shapes. It can contribute false feedthroughs on instances of the cell.

 

A port with multiple shapes or paths is not physically connected inside the cell. The port name is reported.

WARNING: Port name “name” on net name in cell “primary” conflicts with net: name. Port connection skipped for this port.

 

A top-level Port Layer Text object has the same name as a net. This message occurs when LVS Netlist Connect Port Names YES is specified.

WARNING: Direct connection between different ports.

 

A direct connection was made between distinct ports of the cell. The port names are reported. You can disable this warning by including the LVS Report Option P specification statement in your rule file.

WARNING: Short circuit — Different names on one net.

 

Several different text object names have been found on a single net. One name is chosen and the other names are ignored. Conflicts are resolved in favor of, in order of precedence: (1) power names, if specified, in rule file order; (2) ground names, if specified, in rule file order; and (3) the alphabetically least name. Power and ground names are specified with optional LVS Power Name and LVS Ground Name statements in the rule file. All names found on the net are reported, along with their locations and layers. The name actually assigned to the net is indicated. In mask mode extraction, the net ID is also reported.

WARNING: Stamping conflict in SCONNECT - Multiple source nets stamp one target net.

 

Indicates that several different nets in an Sconnect operation tried to form a connection to a single, lower-layer target polygon (similar to what the Stamp operation does). The net selected for stamping is reported; this is the net to which the target polygon gets connected. The rejected nets are reported as well. In hierarchical operation, all net names and net numbers are in the context of the cell being discussed. In hierarchical operation, the net name or net number of the target polygon may also be reported. This is done when the target polygon retains its own distinct net number within the cell and the connection is made instead at a higher level of hierarchy. For example: 


WARNING: Stamping conflict in SCONNECT - Multiple source nets stamp one target net.
                    Target net: 10.
                    Net VSSQ is selected for stamping.
        Rejected nets: 13 14

This example is from a hierarchical run. Net VSSQ was selected for stamping. Nets 13 and 14 were rejected. The target polygon retained its own net number — net 10 in the cell — but nets 10 and VSSQ were connected at a higher level of hierarchy. This type of warning is issued when LVS Report Option S is specified.

WARNING: Open circuit — Same name on different nets.

 

An attempt was made to assign the same name to two or more different nets. One of the nets is arbitrarily chosen and the other nets are left unnamed. The name is reported, along with all locations and layers on which it was found. In mask mode extraction, the report also includes the net IDs on which the name was found, and the net ID to which the name was actually assigned.

WARNING: Ambiguous label attachment — One label intersecting different nets.

 

A text label intersects the regions of two or more different nets and an attempt was made to assign the label to those nets by means of Attach or Label Order operations in the rule file. One of the nets is arbitrarily chosen. The label name and its location and layer are reported. In mask mode extraction, the report also includes the IDs of all nets involved in the conflict, and the ID of the net to which the label was actually assigned.

WARNING: An ERC Operation exceeded the maximum results limit: HGC=1n. Operation results are incomplete. See ERC section of the transcript.

 

An ERC Pathchk statement or Pathchk operation has exceeded the LVS Pathchk Maximum Results limit. Details are in the ERC module section of the transcript.

WARNING: Conflicting global names.

 

The LVS Global Layout Name specification statement is used and there are naming conflicts for global net names.

WARNING: Net name on net id in cell cell not valid for netlisting; net id used instead.

 

A net name is poorly formed and has been ignored. The internally assigned node number is used.

WARNING: Unattached label.

 

A text label cannot be assigned to any net. The label name, location, and layer are reported.

WARNING: Unattached feedthrough or overflow pads; feedthrough or overflow ignored.

 

This warning is issued in Pyxis Layout when a feedthrough (instance pin with multiple shapes) or overflow cannot be used to establish connectivity. This happens when an Attach operation in the rule file cannot be applied to the feedthrough or overflow because the target layer of the Attach is not present at the location of the feedthrough or overflow. The location and layer are reported.

WARNING: Unattached must-connect pad; must-connect condition ignored.

 

A must-connect condition on an instance pin cannot be used. This happens when an Attach operation in the rule file cannot be applied to the must-connect because the target layer of the Attach is not present at the location of the must-connect. The must-connect number, pin name, layer and location are reported.

WARNING: Unattached port pads; port ignored.

 

A port cannot be associated with any net. This occurs when the port layer does not appear in Connect, Attach, or Label Order specification statements, or when there is no geometry at the port location that the port can be attached to. The port name, layer and location are reported.

Calibre nmLVS-H warns about unattached ports, but does not send them to the comparison phase. Flat LVS (no -flatten option) reads unattached ports into the comparison phase. They are then filtered out, unless they serve as initial correspondence points. (A port serves as initial correspondence point if the source has a port with the same name). The LVS end result (CORRECT or INCORRECT) is usually the same either hierarchical or flat, except in some rare cases; for example, the end result may be different when the layout has both a good, attached port text “A” and a false, unattached port text “A”.

WARNING: Hcells “cellname” and “cellname” have incompatible port swap sets - correspondence ignored.

 

One or more pairs of component types are specified as corresponding in the hcell list but have different pin swappability. The hcell correspondence is ignored.

WARNING: Hcells “cellname” and “cellname” are of different kinds (“LVS box cell or empty subcircuit”) and (“Subcircuit”) - correspondence ignored.

 

Hcells are of dissimilar types. For example, the hcell list contains the pair “AAA BBB” but AAA is a cell and BBB is a primitive device; or, AAA is defined to be a CMOS N transistor and BBB is defined to be a CMOS P transistor. This warning is also given if one hcell of a corresponding pair is an empty subckt or an LVS Box cell. Detailed information appears in the transcript. The correspondence in such cases is ignored.

WARNING: Hcells “cellname” and “cellname” have different property rules - correspondence ignored.

 

One or more pairs of component types are specified as corresponding in the hcell list but have different Trace Property rules specified. The hcell correspondence is ignored.

WARNING: ENCLOSURE_PERPENDICULAR/ENCLOSURE_PARALLEL may require excessive computations for this design -- consider using DFM PROPERTY

 

Device statement property computations use the ENCLOSURE_PERPENDICULAR or ENCLOSURE_PARALLEL functions and they possibly required excess runtime to perform their computations. In this case, the DFM Property operation can be useful in reducing runtime.

WARNING: Invalid PATHCHK request net_type: no net_type nets present, operation aborted.

 

A Pathchk operation specifies a net type or name that is not present in the input design.

WARNING: POWER, GROUND, or LABELED nets required by ERC operations do not exist. See ERC section of the transcript shows specific warnings for the missing net types

 

A Pathchk operation specifies net type conditions for which no nets can be found. Read the ERC section of the transcript for detailed warnings. These are also contained in the RUNTIME WARNINGS section of the ERC and circuit extraction reports.

WARNING: Results generated by an ERC check with ABORT LVS (check name) - aborting.

 

An ERC Select Check statement uses the ABORT LVS keyword set, and an ERC check has produced results. The LVS comparison run is aborted.

WARNING: This DEVICE template was not selected for extraction: type $D=n

 

An LVS [Un]Select Device By Seed Layer statement exists in the rules and has caused a Device statement to be disabled during device recognition. The indicated device element is not extracted or netlisted.

WARNING: This PATHCHK Operation exceeded maximum results limit: HGC=n. Operation results are incomplete

 

An ERC Pathchk statement or Pathchk operation has exceeded the LVS Pathchk Maximum Results limit. The output results are truncated at the limit.

WARNING: This PATHCHK Operation is generating excessive results: HGC=n LVHEAP=n SHARED=n MALLOC=n

 

An ERC Pathchk statement or Pathchk operation is processing a multiple of 6 million polygons. This message is repeated each time such a multiple is reached, up to the LVS Pathchk Maximum Results limit.

WARNING: Top level port name “name” on net id at location (x,y) already used on net id; ignored.

 

A top-level port naming conflict was found by the hierarchical SPICE netlister. This occurs when two or more nets are connected to top-level-cell ports with identical names. The warning appears only when the port names would have been otherwise used for netlisting. Specifically, the two nets in question must be unnamed or must have names that are not valid for netlisting, and the port name must be valid for netlisting. The hierarchical SPICE netlister uses the port name to identify one of the nets and its port, and uses the internal number of the other net to identify that net and its port. The net that receives the port name is chosen arbitrarily. This message occurs when LVS Netlist Connect Port Names YES is specified.

WARNING: Top level port name “name” at location (x, y) on net net‑number not valid for netlisting; net id used instead.

 

Indicates a top-level port name not valid for netlisting. The name is poorly formed.

WARNING: Virtually connected Label “label” in “cell”

 

Virtual Connect Report YES UNSATISFIED is specified and two labels are not physically connected.

NOTE: <n> devices that matched LVS FILTER...OPEN statements were removed from the extracted -spice netlist by LVS NETLIST FILTER DEVICES YES.

  LVS Netlist Filter Devices YES was specified and devices have accordingly been omitted from the extracted SPICE netlist.

NOTE: Virtually connected…

 

A name-based virtual connection exists. This situation represents a physically open circuit that is ignored, for example, from the Virtual Connect Name or Virtual Connect Colon specification statements. For each pair of labels that cause a virtual connection, the label name and both label locations are indicated. This reporting is enabled with the LVS Report Option V specification statement.
