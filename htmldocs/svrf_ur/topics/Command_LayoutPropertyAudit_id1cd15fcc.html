Layout Property Audit

Specification statement

Specifies property names and values to be inserted, appended, or modified in an OASIS DRC results database. Used only in Calibre nmDRC-H.

Usage

LAYOUT PROPERTY AUDIT prop_name prop_value [ prop_value …]
[ REPLACE | APPEND ]

Arguments

prop_name

A case-sensitive name or string constant that is a property name. If the parameter is a name preceded by the $ character, then this represents an environment variable that is expected to resolve in the environment. If not, a compiler error results.

prop_value

A case-sensitive name or string constant that is a property value. If the parameter is a name preceded by the $ character, then this represents an environment variable that is expected to resolve in the environment. If not, a compiler error results. More than one prop_value may be specified per prop_name.

REPLACE | APPEND

An optional keyword choice that determines how property values are written.

  • REPLACE — Specifies that prop_value data replaces the property values for the given prop_name argument from the database that is read in. This is the default behavior.

  • APPEND — Specifies that prop_value data is appended to the property values for the given prop_name argument from the database that is read in.

Description

This statement allows property replacement, property insertion, or property modification of file-level properties in OASIS-type results databases in Calibre nmDRC (GDSII-type results databases are not modified).

The statement may be specified any number of times but only once for any given prop_name.

The Layout Property Audit specification statement works as follows:

  1. When the OASIS-type input layout database is read by Calibre nmDRC-H, all file-level properties P1, …, Pn are saved. A file-level property is defined as a property that is attached to the START record of the (first) OASIS input layout database.

    If the Layout System is GDSII or dual-database mode is in effect, then the set of properties Pi is defined to be empty and the algorithm proceeds.

  2. Then for each OASIS-type results database D written by Calibre nmDRC-H and for each Layout Property Audit specification statement L in the rule file, the following three steps are performed:

    1. If REPLACE is specified in L either explicitly or by default, then the property in L is attached to the START record of D. The property’s name is prop_name and its values are the set of prop_value parameters written as b-strings.

    2. Otherwise, if none of the property names of the input properties Pi from step 1 coincide with the prop_value of L, then the property in L is written as in step 2a.

    3. Otherwise, for each input property Pi whose property name matches the prop_name of L, a new property is written to D as in step 1, except that the property values are the set of prop_value parameters of L appended in order to the existing property value list of Pi. prop_value is written as a b-string.

For details regarding OASIS results databases, see “OASIS DRC Results Database Format” in the Calibre Verification User’s Manual.

See also DRC Results Database and DRC Check Map.

Examples

The following statement inserts property “ver” with the string “1.0” into the OASIS results database:

DRC RESULTS DATABASE results.oas OASIS
DRC MAXIMUM RESULTS ALL
DRC KEEP EMPTY YES

LAYOUT PROPERTY AUDIT ver "1.0"
