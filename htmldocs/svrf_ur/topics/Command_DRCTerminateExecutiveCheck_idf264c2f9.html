DRC Terminate Executive Check

Specification statement

Specifies the name of a rule check causing the processing of subsequent rule check operations to be expedited by producing no output.

Usage

DRC TERMINATE EXECUTIVE CHECK [EMPTY] name [name …]

Arguments

EMPTY

Optional argument specifying that the rule check is empty.

name

Specifies the name of a rule check statement or rule check group in the rule file.

Description

This statement is used to expedite remaining rule checks when the specified rule check is non-empty, or empty if EMPTY is specified, by causing subsequent operations to produce no output.

If a rule check or rule check group is not selected, then the statement has no effect for that rule check or rule check group. This statement may be specified any number of times. It is only supported with Calibre nmDRC and Calibre nmDRC-H. The use of this statement does not change the order of rule check execution prior to reaching the specified check.

Hyperscaling (‑hyper) can perform rule checks in a different order than appears in the rule file. Non‑hyperscaling runs follow the rule file order.

Examples

DRC TERMINATE EXECUTE CHECK min_oxide_spacing

The transcript includes the following runtime warning if min_oxide_spacing has results:

--- RUNTIME WARNINGS
---
Early executive termination triggered by DRC RuleCheck min_oxide_spacing.
