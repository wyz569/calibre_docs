Not Touch

Layer operation

Selects polygons that do not share an outside coincident edge or segment as the only interaction with polygons on a second layer. Optional arguments modify the selection criteria.

Usage

NOT TOUCH layer1 layer2 [constraint [BY NET] [EVEN | ODD]]

Arguments

layer1

A required argument specifying an original layer or layer set, or a derived polygon layer. The operation selects polygons from this layer that meet the conditions of the operation.

layer2

A required argument specifying an original layer or layer set, or a derived polygon layer.

constraint [BY NET] [EVEN | ODD]

An optional argument that places conditions on the number of layer2 polygons that touch a layer1 polygon. If the number of layer2 polygons or nets that touch a layer1 polygon does not satisfy the constraint, the layer1 polygon is selected. A layer1 polygon with zero touching layer2 polygons is always selected.

The constraint is one of those listed in the “Constraint Notation” column of Table 1 in the “Constraints” topic. The constraint must contain non-negative integers. Not Touch with a constraint of == 0 selects all layer1 polygons; use Touch instead.

  • BY NET — Specifies that the constraint applies to the number of distinct nets in the set of layer2 polygons that touch a layer1 polygon. Connectivity is required on layer2.

  • EVEN | ODD — Specifies that the number of layer2 polygons or nets that touch the layer1 polygon is even or odd, respectively. The layer1 polygon is selected if the constraint is not met. For example, if a constraint of “> 1 EVEN” is specified, a layer1 polygon that touches three layer2 polygons is selected, while a layer1 polygon that touches four layer2 polygons is not selected.

Description

Selects layer1 polygons that do not share an outside coincident edge or segment as the only interaction with a layer2 polygon. This is the complement of the Touch operation. If layer2 is empty, the Not Touch operation generates output equivalent to the polygon data on layer1.

Not Touch selects polygons that do not satisfy a Touch operation with the same arguments. If a constraint is specified, the constraint and optional keywords specify conditions for the complementary Touch operation. Polygons on layer1 that do not satisfy the conditions as applied to the complementary Touch operation are selected.

See also Not Interact and Not Touch Edge.

Examples

Example 1

The following example uses the Not Touch operation to check for gates where polysilicon does not completely cross diffusion. The bad_gates check finds gates that do not have exactly two touching source/drain polygons. The gates_not_even check finds gates that do not have an even number of touching source/drain polygons.

bad_gates { 
@ "Gates" where poly does not completely cross diffusion.
   gate = poly AND diff       // Gate regions. 
   srcdrn = diff NOT poly     // Source/drain regions.
   gate NOT TOUCH srcdrn == 2 // Must touch two source/drain regions.
}
gates_not_even { 
   NOT TOUCH gate srcdrn > 1 EVEN // Must touch an even number of 
                                  // source/drain regions.
}
