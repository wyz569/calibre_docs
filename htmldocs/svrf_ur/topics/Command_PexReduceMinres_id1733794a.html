PEX Reduce Minres

Parasitic extraction

Specifies to combine or short parasitic resistors below a threshold value to reduce netlist size.

Usage

PEX REDUCE MINRES COMBINE threshold [LAYERWISE]

PEX REDUCE MINRES SHORT threshold

Arguments

COMBINE

A required keyword that causes resistors with values below threshold to be added together and reported as a single parasitic resistor.

SHORT

A required keyword that causes resistors with values below threshold to be shorted.

threshold

A non-negative, floating point value that specifies a resistor threshold for COMBINE or SHORT. The Unit Resistance SVRF statement sets the units for this value.

LAYERWISE

An optional keyword that specifies the reduction of parasitic resistors for each layer, and not across different layers. May not be used with the SHORT keyword.

Description

Combines (in series) or shorts (removes) extracted resistors less than a user-defined threshold value. Use this statement to reduce the size of the netlist by decreasing the number of parasitic resistors. The reduction is done in the formatting step before PEX Reduce TICER. Specifying PEX REDUCE MINRES COMBINE preserves total network resistances, whereas PEX REDUCE MINRES SHORT does not. If both PEX REDUCE MINRES SHORT and PEX REDUCE MINRES COMBINE statements are specified, then the resistors are combined first then shorted.

If the netlist includes parasitic self or mutual inductances extracted using the broadband model, PEX Inductance MAXIMUM Frequency, resistors associated with these inductors are not combined or shorted, even if their values are less than the specified threshold. This reduction technique can be used when the netlist includes parasitic self or mutual inductance as extracted by Calibre xL or the Calibre xACT inductance flow. For reduction of networks with inductance, you can also use PEX Reduce Analog or PEX Reduce TICER.

In some cases resistors smaller than the specified threshold are not removed. Specifically, small resistors between two pins, between a pin and a junction, or between two junctions are not shorted. A junction is a node with more than two incident resistors. Shorting a resistor means removing the resistor from the network, then merging the two nodes it was connected to. Note that ports, hports, and probes have the same effect as pins in preventing short removal.

Note:

PEX Reduce Minres replaces the deprecated PEX Tolerance Distributed R statement. If both statements are used, PEX Reduce Minres overrides PEX Tolerance Distributed R.

PEX Reduce Minres reduces resistive networks without complex analysis or error control. Because of the simple way in which it handles capacitors, it does not necessarily preserve interconnect (Elmore) delay. This command should not be used when a high degree of netlist accuracy is required.

Using PEX Netlist RLAYER, RLENGTH, RTHICKNESS, RLOCATION or RWIDTH with PEX Reduce Minres COMBINE generates a warning message and RLAYER, RLENGTH, RTHICKNESS, RLOCATION, or RWIDTH will be ignored.

For example, using PEX Netlist RLOCATION together with PEX Reduce Minres COMBINE will output the following warning message and RLOCATION will be ignored:

RULES: PEX NETLIST "netlist.dist" DSPF 1e-6 LAYOUTNAMES GROUND VSS LOCATION RLOCATION
RULES: PEX REDUCE MINRES COMBINE 10
WARNING: For asic mode, turning on PEX REDUCE DIGITAL by default
WARNING: RLOCATION not supported with PEX REDUCE DIGITAL
WARNING: RLOCATION not supported with PEX REDUCE MINRES COMBINE

Use the LAYERWISE keyword to reduce parasitic resistance on a per layer basis for all layers in the design either when combining resistors in series or when filtering them out from the design based on a certain threshold. The layer-based reduced netlist can be used for downstream flows and analysis, but should not be used for simulation.

The analyzer disables the following reduction statements when PEX Reduce Minres LAYERWISE is specified:

  • PEX Reduce TICER

  • PEX Reduce Analog

  • PEX Reduce Digital

  • PEX Reduce ROnly

and a warning message similar to the following example is generated by the formatter:

WARNING: PEX REDUCE MINRES LAYERWISE disabling reduction mode PEX REDUCE ANALOG

When used with Calibre xACT where PEX Reduce Analog is on by default, or in LEF/DEF or ASIC mode extraction where PEX Reduce Digital is on by default, these reduction statements are also disabled and the warning is generated.

PEX Reduce Parallel YES can be used with PEX Reduce Minres SHORT, but PEX Reduce Parallel YES will only apply to same-layer vias.

Examples

Example 1

To combine extracted resistors falling below 0.5 ohms, include the following in your rule file:

PEX REDUCE MINRES COMBINE 0.5

Example 2

To short any resistor with resistance less than 0.6 ohms, include the following in your rule file:

PEX REDUCE MINRES SHORT 0.6

Example 3

Include the following statement in your rule file to combine extracted resistors that fall below 0.6 ohms for each layer in the design:

PEX REDUCE MINRES COMBINE 0.6 LAYERWISE

Example 4

To combine extracted resistors falling below 0.4 ohms and then short any resistors with a resistance less than 0.6 ohms, include the following statements in your rule file:

PEX REDUCE MINRES COMBINE 0.4
PEX REDUCE MINRES SHORT 0.6
