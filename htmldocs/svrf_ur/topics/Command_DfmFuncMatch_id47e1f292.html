MATCH

String function for DFM expressions

Determines whether or not a string matches a pattern and returns true or false (1 or 0). Valid only in the DFM Property operation.

Usage

MATCH(“string”, “pattern”)

Arguments

string

A required string to match. Strings containing space characters must be enclosed in quotation marks (“ ”).

pattern

A required regular expression for matching the string. The expression should be enclosed in quotation marks.

Return Values

Integer.

Description

The MATCH function uses the GNU implementation of extended regular expressions. A return value of 1 indicates a successful match with 0 being the complement. Matching is case sensitive, and substring matches are considered successful. If a full match is required, the pattern should begin with “^” and end with “$”, which forces the pattern to match the beginning and end of the string, respectively.
