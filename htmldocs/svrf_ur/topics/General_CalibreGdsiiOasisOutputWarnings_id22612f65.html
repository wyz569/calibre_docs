Calibre GDSII and OASIS Output Warnings

This section describes runtime warnings related to GDSII and OASIS output.
Table 1. Calibre GDSII and OASIS Output Warnings

Explanation

The following warnings are reported by Calibre nmDRC applications:

WARNING: RuleCheck output to GDSII results database <name> with maximum polygon vertex count of <number> reset to 8190.

A DRC rule check outputs to a GDSII-type results database with a maximum polygon vertex count greater than 8190. This warning is printed for each DRC results database generated, and includes the filename of the database and the actual maximum vertex value.

WARNING: RuleCheck output to (GDSII or OASIS) results database <name> with maximum result count of <number> (less than ALL).

A DRC rule check outputs to a GDSII or OASIS-type results database with a maximum result count less than ALL. This warning is printed for each generated results database. The file name of the database and the actual maximum result value are placed into the message. The intent is to warn of inadvertent use of the default value 1000 when writing mask data versus DRC errors. This warning is printed in the CALIBRE DRC results database initialization module.

WARNING: RuleCheck <name> output to (GDSII or OASIS) results database <name> with default layer 0, datatype 0.

A DRC rule check outputs to a GDSII or OASIS type DRC results database with a default layer number (namely 0). This may occur because a layer number is not specified in the DRC Check Map specification statement for the rule check or because no DRC Check Map is specified for the rule check. This warning is printed for each DRC rule check which outputs a default layer number to a GDSII or OASIS type DRC results database. The names of the rule check and the results database are placed into the message. This warning is printed in the CALIBRE DRC results database initialization module.

WARNING: Structure names longer than 32 characters have been written to GDSII DRC results database <name>.

One or more structure names written to an output GDSII-type DRC results database has a length exceeding 32 characters. This length limit is a GDSII anachronism and is rarely enforced by current stream readers; hence, this warning is simply meant to be informative. The file name of the affected DRC results database appears in the message.

The following are reported by Calibre nmDRC-H:

WARNING: DRC CHECK MAP AREF cell name <name> duplicates layout database cell name—AREF output suppressed.

An AREF cell name in a DRC Check Map specification statement coincides with a placed cell name in the input layout database. AREF output is not performed for that particular AREF specification. The name of the cell appears in the message.

WARNING: DRC MAXIMUM CELL NAME LENGTH truncation of output cell name <name> to <number> characters is not possible.

A structure name written to an output GDSII- or OASIS-type DRC results database has a length exceeding that specified in the DRC Maximum Cell Name Length specification statement. This is due to the fact that the name could not be truncated to any positive length and avoid duplication with any other output cell name. The name of the cell and the DRC Maximum Cell Name Length parameter appears in the message.
