DRC Summary Report

Specification statement

Specifies the DRC summary report filename and method in which it is written.

Usage

DRC SUMMARY REPORT filename [REPLACE | APPEND] [HIER] [EXECUTED]

Arguments

filename

A required argument specifying the name of a report file that is either created or appended to during execution.

The filename parameter can contain environment variables. See “Environment Variables in Pathname Parameters”.

REPLACE | APPEND

An optional keyword that specifies whether to overwrite or append results to the specified summary file.

  • REPLACE — Overwrites the previous contents of the summary file, if any, each time you run Calibre nmDRC. This is the default behavior if you do not specify this keyword.

  • APPEND — Appends the results to an existing summary report file. This parameter is useful in creating a log of runs in a single file.

HIER

An optional keyword that instructs Calibre nmDRC-H to create a section in the summary report that lists non-empty rule check statistics by layout database cell.

The HIER option is not commonly used for post-tapeout flows because it can lead to excessively large files when DRC Check Map AREF is used. The AREF option can create millions of array placements that must be written out.

EXECUTED

An optional keyword used to suppress any summary report for unselected DRC rule checks.

Description

Specifies the Calibre nmDRC summary report filename and method in which it is written. Enables creation of a file containing a summary report of the run, which includes the following information:

  • The date and time of the Calibre nmDRC run.

  • The rule file pathname and title, if specified.

  • The top-level layout cell pathname.

  • The current working directory and user name.

  • A summary of the number of original shapes processed per original layer.

  • A summary of the number of results generated per rule check, as well as a listing of which rule checks from the rule file were not performed. The EXECUTED keyword suppresses output of any rule checks that were not performed.

    The number of results is the number sent to the global DRC Results Database. It does not include results sent to DRC Check Map or DFM RDB databases related to the rule check.

  • All warnings generated.

  • The total run time, result count, number of original shapes processed, and number of rule checks executed.

If you do not specify this statement, no summary file is generated.

You can specify the DRC Summary Report in Calibre Interactive—nmDRC. See “Specifying Outputs for a DRC Run” in the Calibre Interactive User’s Manual and “Outputs Page in Calibre Interactive nmDRC” in the Calibre Interactive (New GUI) User’s Manual.

See the “DRC Summary Report” section of the Calibre Verification User’s Manual for related information.

In Pyxis Layout, sets the initial filename summary file parameter and default setting for the summarymode switch in the $check_drc() function.

Examples

DRC SUMMARY REPORT drc_report APPEND HIER
// write to drc_report by appending it, include hierarchical 
// statistics by cell
