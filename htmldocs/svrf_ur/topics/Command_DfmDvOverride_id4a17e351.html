DFM DV Override

Layer operation

License: Calibre nmDRC-H

Collects DFM property data from input layers and generates an output layer with net ID and voltage information. The output layer is used by DFM DV with the OVERRIDE keyword to override certain voltage values, typically for analog applications.

Usage

DFM DV OVERRIDE dvprop_layer [ [‘>’] dvprop_layer ]…  
[ DVPARAMS ni_layer Vmin_prop Vmin_default Vmax_prop Vmax_default syncID_prop ]
[ RDB conflict_rdb ]
[{SELECTED | INVALID} dvprop_layer ]

Arguments

dvprop_layer [ [‘>’] dvprop_layer]…

  • dvprop_layer — One or more required layers, which must be polygon layers or derived edge layers. Each layer must have these DFM properties attached to its geometries: NET1, NET2, and DELTA_V. The DELTA_V value provides the override voltage for the net pair. See “dvprop_layer”.

  • > — An optional character that separates the layers into equal priority groups, with the groups to left of the > having higher priority.

Example: L1 L2 > L3 > L4 L5

Layers L1 and L2 have the highest and equal priority. L3 has the next highest priority. L4 and L5 have the lowest and equal priority.

See “Default Behavior” for further information.

DVPARAMS ni_layer Vmin_prop Vmin_default Vmax_prop Vmax_default syncID_prop

An optional argument set that provides per-net minimum and maximum voltage values, similar to the DVPARAMS argument in DFM DV. If DVPARAMS is specified, then the property values on ni_layer are used to find the net-based DELTA_V for a net pair. If the override DELTA_V value for a net pair on a dvprop_layer is larger than the corresponding net-based DELTA_V value calculated using the ni_layer, the dvprop_layer value is ignored.

  • ni_layer — A net-indexed layer produced by DFM Property Net. The ni_layer includes properties that specify the minimum and maximum voltages associated with the geometries on the input layers.

    The ni_layer may additionally include the syncID property, which is a netID property used to determine if two nets are switching synchronously.

  • Vmin_prop — The name of a numeric property on ni_layer, where the property value provides the minimum voltage on the associated net.

  • Vmin_default — A numeric value providing the default minimum voltage to be used for nets that do not have the Vmin_prop property on ni_layer.

  • Vmax_prop — The name of a numeric property on ni_layer, where the property value provides the maximum voltage on the associated net.

  • Vmax_default — A numeric value providing the default maximum voltage to be used for nets that do not have the Vmax_prop property on ni_layer.

  • syncID_prop — A required name of a netID property on ni_layer, where the syncID property value is used to determine if two nets switch synchronously

    If the syncID property is not used, define a syncID property for all nets and assign the property value NONETID using the function NONETID. In this case all nets are assumed to switch asynchronously (the worst case).

See Figure 1 with the DFM DV operation for information on how the net-based DELTA_V value is calculated. See “Behavior With DVPARAMS” for information on how the override DELTA_V is determined when DVPARAMS is specified.

RDB conflict_rdb

An optional keyword and filename. If the property values on the input layers result in conflicting values of DELTA_V for a given NET1/NET2 pair, the conflicts are written to the RDB named conflict_rdb. See “Conflict RDB”.

{SELECTED | INVALID} dvprop_layer

An optional argument set that specifies to output a subset of the specified dvprop_layer rather than the standard output layer.

  • SELECTED — Output geometries from the specified dvprop_layer that result in valid override voltages. See “SELECTED Output”.

  • INVALID — Output geometries from the specified dvprop_layer that do not result in valid override voltages. Such geometries have an error condition, a voltage conflict, or are unused due to the layer priority. See “INVALID Output”.

Description

DFM DV Override generates an output layer that is used by the DFM DV operation to provide alternate maximum voltage difference (DELTA_V) values for specific net ID pairs. The layer is specified by the OVERRIDE keyword in DFM DV. This is useful in analog applications, to provide a more accurate value of DELTA_V for nets that are switching asynchronously, but not completely out of phase, as shown in the following figure. The DELTA_V value in the analog case is less than that of the digital case.



The DFM DV Override operation calculates an override DELTA_V for each net pair provided on the input dvprop_layer(s). The calculation uses the property values on the dvprop_layer(s) and, if DVPARAMS is specified, the properties on ni_layer. This DELTA_V overrides the DELTA_V value otherwise determined by the DFM DV operation for the net ID pair. However, if the override DELTA_V value is greater than the net-based value of DELTA_V calculated by the DFM DV operation, the override value is not used.

The output layer depends on the keywords that are specified:

  • Standard Output—Without SELECTED or INVALID

    The output layer is a polygon layer with one polygon per cell and attached DFM properties. The polygon coordinates are meaningless. This layer can be specified as the override layer in DFM DV with the OVERRIDE keyword. The DFM properties are used internally by the DFM DV operation.

  • Debug Output—With SELECTED or INVALID

    The output layer consists of a subset of the specified dvprop_layer with attached DFM properties. See “SELECTED Output” and “INVALID Output”.

    You can examine the INVALID layer for voltage errors, conflicts, and unused geometries.

    The SELECTED layer can be specified in a DFM DV Analyze operation with the OVERRIDE keyword. The geometries for which the override DELTA_V value is used to calculate a DFM DV result are output to an RDB.

    The union of SELECTED and INVALID outputs for a dvprop_layer layer should be geometrically equivalent to the dvprop_layer, possibly with some polygons promoted to distinguish between selected and invalid placements.

dvprop_layer

The DFM properties NET1, NET2, and DELTA_V must be attached to the shapes on the dvprop_layer(s). NET1 and NET2 are netID properties. DELTA_V is a numeric property that gives the maximum voltage difference between geometries on NET1 and NET2.

Default Behavior

The default behavior is used when DVPARAMS is not specified. The DELTA_V values on the dvprop_layer(s) are used as the override DELTA_V for each NET1/NET2 pair.

If the properties on the dvprop_layer(s) result in multiple different DELTA_V values for a particular net pair, the DELTA_V value from the highest priority layer is used, where the priority is determined by the > character. For layers with equal priority, the maximum DELTA_V value is used.

The override DELTA_V value should be no greater than the net-based maximum voltage difference calculated for the net pair. See the calculation of DELTA_V in Figure 1 with the DFM DV command reference. The DFM DV operation uses an internal optimization to determine which geometries to measure, based on the voltages associated with the corresponding nets. Because of this optimization, it is possible for measurements to be missed if the override DELTA_V value is greater than the net-based maximum voltage difference calculated by DFM DV for the net pair. For edge pairs that are measured by DFM DV, a warning is issued if the override DELTA_V value is greater than the calculated net-based value for the net pair, and the calculated net-based value of DELTA_V is used.

Example: DFM DV OVERRIDE L1 L2 > L3 > L4 L5

For any net pair that occurs in layer L1 or L2, the override DELTA_V is the maximum DELTA_V found in L1 or L2, and the other layers are ignored. For any net pair that occurs in layer L3 but not in L1 or L2, the override DELTA_V is taken from L3; L4 and L5 are ignored. For any net pair that occurs only in L4 or L5, the override DELTA_V is the maximum DELTA_V found in L4 or L5.

When the DFM DV operation processes the override layer, if any override DELTA_V is greater than the corresponding net-based DELTA_V calculated by the DFM DV operation, the override DELTA_V is ignored and the net-based DELTA_V is used.

Behavior With DVPARAMS

If DVPARAMS is specified, then the property values on ni_layer are used to find the net-based DELTA_V for a net pair (see the calculation of DELTA_V in Figure 1 with the DFM DV command reference). If a DELTA_V value for a net pair on a dvprop_layer is larger than the corresponding net-based DELTA_V value calculated using properties on the ni_layer, the dvprop_layer value is ignored. When the dvprop_layer properties results in multiple DELTA_V values for a particular net pair, the result can be different than what occurs with the default behavior. This is demonstrated in the following example.

Example: DFM DV OVERRIDE L1 > L2 DVPARAMS ni “vmin” 0.2 “vmax” 2.5 “sync”

The value 2.5 on layer L1 is ignored because it is greater than the net-based DELTA_V calculated using the ni layer. The value of 1.5 from layer L2 is used as the override DELTA_V and this value is used by the DFM DV operation.

In contrast, the default behavior results in an override DELTA_V of 2.5. When processed by the DFM DV operation, the override DELTA_V is found to be larger than the net-based DELTA_V of 2.0, and the override value is ignored. The DFM DV operation uses the net-based DELTA_V of 2.0.

SELECTED Output

When SELECTED is specified the operation outputs geometries from the specified dvprop_layer that result in valid override voltages. The properties NET1, NET2, and DELTA_V are copied to the output layer. Unlike the standard output, this layer cannot be used as override layer input to DFM DV.

The output layer produced by the SELECTED keyword can be input to DFM DV Analyze with the OVERRIDE keyword. Because the SELECTED layer output only includes geometries that result in valid override voltages, the SELECTED layer output may be more practical than the original dvprop_layer, since geometries that do not contribute to DFM DV results are eliminated from the DFM DV Analyze reporting.

Promotion: If a geometry in a cell C is selected for some placements of C and invalid for others, then the geometry is promoted to the place in the hierarchy where it is selected or invalid, but not both.

INVALID Output

When INVALID is specified the operation outputs geometries from the specified dvprop_layer that do not result in valid override voltages. Unlike the standard output, this layer cannot be used as override layer input to DFM DV. Geometries are categorized and output as follows:

  • Out of range errors — If DVPARAMS is specified and the DELTA_V value for some geometry G on dvprop_layer exceeds the net-based DELTA_V calculated using the ni_layer properties for the net pair on G, then G is categorized as “out of range” and written to the output layer.

  • Voltage conflicts — If two or more geometries from equal priority input layers supply different DELTA_V values for the same net pair, and no higher priority input layer supplies a DELTA_V value for that net pair, then the geometries from dvprop_layer that are less than the maximum DELTA_V for the equal priority layer group are considered in conflict. Geometries from this group that are not in the “out of range” category and not output by the SELECTED keyword for any placement of the geometry are categorized as “conflict” and written to the output layer.

  • Unused geometries — Geometries on dvprop_layer that are not in the “out of range” or “conflict” category and not output by the SELECTED keyword are categorized as “unused” and written to the output layer. This occurs when the DELTA_V value of the geometry is unused because the priority of dvprop_layer is lower than that of the selected geometry for the same net pair.

Conflict RDB

It is possible for the properties on the input layers to result in multiple different DELTA_V values for a particular NET1/NET2 pair. These conflicts are reported in the conflict_rdb. All dvprop_layer geometries with conflicts are reported in the RDB, unlike the output layer produced with the INVALID keyword, which contains only geometries from the specified dvprop_layer.

Conflicts are detected as low in the hierarchy as possible. Each detected conflict is assigned a unique identifying value (a UID), and the input geometries that contribute to the conflict for that UID are reported in the RDB file. For each input layer that contributes to a particular conflict, the contributing geometries are written as a check whose name has the following format:

  uid=<UID> deltaV = [<vlow>, <vhigh>] <layer>

Here <UID> is the unique identifier, vlow and vhigh are the minimum and maximum voltages associated with the conflict, and layer is the name of the contributing dvprop_layer layer. The geometries are written with their NET1, NET2, and DELTA_V properties, so the source of the conflict can be seen. (Unlike the INVALID conflict output, all geometries contributing to the conflict are reported, including the geometry with the selected override voltage.)

Out of range errors are reported in the conflict RDB in the same manner as with the INVALID output, and may be promoted if necessary. The conflict RDB does not include unused geometries (unlike the INVALID output).

Concurrency

DFM DV Override operations execute concurrently if these conditions are met:

  • All input layers are the same and have the same ‘>’ priority separators

  • If DVPARAMS is specified, all DVPARAMS arguments are the same

  • If RDB is specified, it must be specified for all operations, with the same RDB filename.

Examples

Tip

If you are writing several concurrent operations with different specifications for SELECTED and INVALID, you can define a DMACRO to provide the arguments that are the same in all operations. See “Example 2”.

Example 1

This example is similar to “Example 2” with the DFM DV operation, for the case that uses the syncID property to determine if synchronous switching is present. Shapes on layer4 are used to determine if an override DELTA_V value should be used. The override DELTA_V is provided by the variable DV_OVERRIDE. DFM DV Analyze is used to create a debug RDB. Two separate DFM DV Analyze operations are necessary—one with the input layer to DFM DV Override and one with all other relevant layers. 

LAYOUT PATH "layout.oas" 
LAYOUT SYSTEM OASIS   
LAYOUT PRIMARY TOPCELL  
DRC RESULTS DATABASE drc.db 
DFM DATABASE dfmdb OVERWRITE 
DRC CELL NAME YES CELL SPACE XFORM 
LAYER layer1 1
LAYER layer2 2
LAYER layer4 4
LAYER layer3 3
CONNECT layer1 
CONNECT layer2
CONNECT layer3 

VARIABLE VH_net1 5
VARIABLE VL_net1 1
VARIABLE VH_other 3
VARIABLE VL_other -3

VARIABLE DV_OVERRIDE 6
// calculate syncID values and the net-indexed input layer for DFM DV
or_12 = OR layer1 layer2
sync_layer = or_12 INTERACT layer3
CONNECT layer3 sync_layer
sync_one = DFM PROPERTY layer1 layer3 OVERLAP MULTI 
   [ - = COUNT(layer3)] > 0
   [syncID = NETID(layer3, 1)]
sync_two = DFM PROPERTY layer2 layer3 OVERLAP MULTI 
   [ - = COUNT(layer3)] > 0
   [syncID = NETID(layer3, 1)]
net_prop_wsync = DFM PROPERTY NET layer1 layer2 sync_one sync_two
   [ vhigh = ( (COUNT(layer1) > 0) ? VH_net1 : VH_other )]
   [ vlow = ( (COUNT(layer1) > 0) ? VL_net1 : VL_other )]
   [ syncID = (COUNT(sync_one) > 0) ? NETPROPERTY(sync_one, syncID, 1) : 
         (COUNT(sync_two) > 0) ? NETPROPERTY(sync_two, syncID, 1) :
          NONETID()]
// use override if both layer1 and layer2 geometry overlap a layer4 shape 
ov_input = DFM PROPERTY layer4 layer2 layer1 net_prop_wsync OVERLAP MULTI 
            [ - = COUNT(layer1) ] >= 1 
            [ - = COUNT(layer2) ] >= 1
            [ NET1 = NETID(layer1, 1) ]
            [ NET2 = NETID(layer2, 1) ]
            [ DELTA_V = DV_OVERRIDE ]

dv_ov_layer = DFM DV OVERRIDE ov_input RDB "override.rdb"

dv_12_ov = DFM DV layer1 layer2 < 1.5 
            DVPARAMS net_prop_wsync
            "vlow" 0 "vhigh" 5 "syncID" >= 5.0
            OVERRIDE dv_ov_layer
            ANNOTATE NETS

dv_12_ov { dfm copy dv_12_ov }
dv_dbg = DFM DV ANALYZE layer1 layer2 net_prop_wsync dv_12_ov RDB dbg.rdb
dv_dbg_ov = DFM DV ANALYZE ov_input RDB dbg.rdb OVERRIDE

dv_dbg_check { 
   copy dv_dbg
   copy dv_dbg_ov
}

Example 2

This example uses two input layers for the DFM DV Override operation, with priority specified using the ‘>’ character. The DVPARAMS keyword is used so that the net-based DELTA_V values can be used.

Two concurrent DFM DV Override operations with the SELECTED keyword generate input layers for DFM DV Analyze. A DMACRO is defined to provide the input layers and DVPARAMS argument set. This makes it easy to insure the same input layers are provided to the operations. 

// Define the DMACRO for the DFM DV OVERRIDE inputs
DMACRO DFMDV_OV_INPUTS { 
ov_in1 > ov_in2 
DVPARAMS net_prop_wsync "vlow" 0 "vhigh" 5 "syncID"
}
// generate an override layer for DFM DV
dv_ov_layer = DFM DV OVERRIDE CMACRO DFMDV_OV_INPUTS 

// generate input layers for DFM DV Analyze
ov_in1_selected = DFM DV OVERRIDE CMACRO DFMDV_OV_INPUTS
                    SELECTED ov_in1
ov_in2_selected = DFM DV OVERRIDE CMACRO DFMDV_OV_INPUTS
                    SELECTED ov_in2
dv_12_ov = DFM DV layer1 layer2 < 1.5 
            DVPARAMS net_prop_wsync "vlow" 0 "vhigh" 5 "syncID" >= 5.0
            OVERRIDE dv_ov_layer
            ANNOTATE NETS

dv_12_ov { dfm copy dv_12_ov }
dv_dbg = DFM DV ANALYZE layer1 layer2 net_prop_wsync dv_12_ov RDB dbg.rdb
dv_dbg_ov = DFM DV ANALYZE ov_in1_selected ov_in2_selected 
                RDB dbg.rdb OVERRIDE

dv_dbg_check { 
   copy dv_dbg
   copy dv_dbg_ov
}
