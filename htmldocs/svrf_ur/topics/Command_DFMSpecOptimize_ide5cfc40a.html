DFM Spec Optimize

Specification statement

License: Calibre YieldEnhancer

Defines a set of criteria to be used when optimizing fill with DFM Optimize.

Usage

DFM SPEC OPTIMIZE "spec_name"
GUARDED guard_layer1 [guard_layer2 …]
OPTIMIZABLE opt_layer1 [opt_layer2 …]
OPTIMIZER "optimizer_local" "optimizer_global"
[INSIDE OF LAYER analyze_layer]
[RDB rdb_name]

Arguments

spec_name

A required string giving the name of the specification. The string must be in quotes. This name is referenced by DFM Optimize.

GUARDED guard_layer1 [guard_layer2 …]

A required keyword set listing the layers that cannot be changed by the optimizer. The GUARDED layers also contribute to the extent calculation for the analysis region. Original drawn layers should be listed here.

OPTIMIZABLE opt_layer1 [opt_layer2 …]

A required keyword set listing the fill layers to be optimized. Fill shapes should be rectangular, single layer fill shapes of a single color. Fill shapes on different layers should not overlap, as this causes incorrect evaluation of the optimization expression.

OPTIMIZER "optimizer_local" "optimizer_global"

A required keyword set that gives the names of the optimizers that are used. Optimizers are defined with DFM Spec Optimize Data.

  • optimizer_local — The local optimizer, which uses the WINDOW keyword in DFM Spec Optimize Data.

  • optimizer_global — The global optimizer, which does not use the WINDOW keyword.

INSIDE OF LAYER analyze_layer

An optional keyword set that specifies to optimize fill within the overall extent of analyze_layer, which must be a polygon layer. When not specified, the default analysis region is the extent of all input layers to DFM Spec Optimize. See “Example 4” with DFM Optimize.

RDB rdb_name

An optional keyword set that causes the operation to generate a results database file named rdb_name that contains statistics about the optimization criteria.

The results are organized as check results for the local and global optimizer. These statistics are reported as property values for each optimizer:

Property

Definition

WI

Window index

SDV

Starting density value

FDV

Final density value

The RDB format is similar that produced by the RDB keyword in DFM Spec Fill; see “Fill RDB File Format” in the Calibre YieldAnalyzer and YieldEnhancer User’s and Reference Manual.

Description

DFM Spec Optimize defines the layers used in the optimization expression, the layers to optimize, and the optimizers that are used when optimizing fill with DFM Optimize. A local and a global optimizer are used—the local optimizer runs first.

All layers referenced by name in the optimization expression defined by DFM Spec Optimize Data should be listed as GUARDED layers in DFM Spec Optimize. These layers are not changed by the optimizer.

The OPTIMIZABLE layers are referenced by the special layer name _FILL_ in the optimization expression in DFM Spec Optimize Data. If more than one OPTIMIZABLE layer is listed, the layers are merged internally before applying the optimization criteria. However, the DFM Optimize operation can output the individual optimized fill layers; see “Single Layer Optimized Output” with the DFM Optimize reference.

By default, the extent of the GUARDED and OPTIMIZABLE layers defines the analysis region. The optional “INSIDE OF LAYER analyze_layer” keyword set specifies that the analysis region is extent of all shapes on analyze_layer.

See DFM Optimize for a complete examples.
