Calibre Layout Input Exceptions and Warnings

This section describes runtime exceptions and warnings related to layout input.
Table 1. Calibre Layout Input Exceptions and Warnings

Comment

Explanation

Warnings preceded by “†” can have their severities controlled by Layout Input Exception Severity. Warnings preceded by “*” are fatal if Layout Error On Input YES is specified.

The following warnings are reported by Calibre nmDRC applications:
 

WARNING: Cannot open database input file <filename> for read access.

The layout database format is binary or ASCII and the specified input polygon file could not be opened for read access. The run continues as if the file were empty.
 

WARNING: DRC CHECK MAP AREF cell name <name> is unplaced but exists in the original layout input database.

Calibre nmDRC-H does not allow an AREF cell name to duplicate any placed structure name in the input layout database, so AREF output is suppressed for the DRC CHECK MAP AREF component.
 

WARNING: EXPAND CELL <name> not located or not allowed.

A cell in an EXPAND CELL specification statement is not located in the input database or is the top-level cell.

The following exceptions are reported by all Calibre applications:

†

EXCEPTION: Layer <number> contains unmapped objects and is the source layer of LAYER MAP <source-layer> DATATYPE <source-datatype>.

A required simple layer number is the source layer of a datatype map but contains objects which themselves are not mapped. The number of the source layer and a representative datatype map are placed into the message.

Exception Name: DATATYPE_MAP_SOURCE

†

EXCEPTION: Layer <number> contains unmapped objects and is the target layer of LAYER MAP <source-layer> DATATYPE <source-datatype>.

A required simple layer number is the target layer of a datatype map but contains objects which themselves are not mapped. The number of the target layer and a representative datatype map are placed into the message.

Exception Name: DATATYPE_MAP_TARGET

†

EXCEPTION: Unused geometric data is present in the input layout database on layer <number>, datatype <number>.

Geometry which is not required in the Calibre run exists in the input layout database.

Exception Name: UNUSED_DATA

†

EXCEPTION: A simple layer required in the Calibre run has no geometric data.

A simple layer is any layer number, after application of pertinent datatype maps.

Exception Name: EMPTY_LAYER

†

EXCEPTION: Empty simple layers which are LAYOUT BASE LAYERs are present in the input layout database.

A simple layer is any layer number, after application of pertinent datatype maps.

Exception Name: EMPTY_BASE_LAYER

†

EXCEPTION: Data is present in the input layout database on layer <layer>, datatype <datatype> and this layer is >= to the LAYOUT BUMP2 value.

In a dual database application, there is data in the first input layout database whose layer number is greater than or equal to the value of the Layout Bump2 specification statement. These objects are not processed.

Exception Name: LAYER_OVER_BUMP

†

EXCEPTION: Non-closed boundary at location <point> in cell <name> on layer <layer> datatype <datatype>.

A boundary with more than two vertices and unequal first and last vertices is encountered in a GDSII input layout database. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: POLYGON_NOT_CLOSED

†*

EXCEPTION: Degenerate (vertex count = <number>) polygon at location <point> in cell <name> on layer <layer> datatype <datatype>.

A polygon has 0 or 1 vertices. Reporting is once per cell with the point reported in cell coordinates if the input layout database format is GDSII, OASIS, LEF/DEF, or OPENACCESS; otherwise, reporting is flat with the point reported in world coordinates. The cell name is omitted if the layout database format is Binary or ASCII.

Exception Name: POLYGON_DEGENERATE

†*

EXCEPTION: Non-orientable polygon at location <point> in cell <name> on layer <layer> datatype <datatype>.

A polygon is non-orientable. Reporting is once per cell with the point reported in cell coordinates if the input layout database format is GDSII, OASIS, LEF/DEF, or OPENACCESS; otherwise, reporting is flat with the point reported in world coordinates. The cell name is omitted if the layout database format is Binary or ASCII.

Exception Name: POLYGON_NONORIENTABLE

†

EXCEPTION: Non-simple polygon at location <point> in cell <name> on layer <layer> datatype <datatype>.

A polygon or boundary is non-simple but orientable. Reporting is once per cell with the point reported in cell coordinates if the input layout database format is GDSII or OASIS; otherwise, reporting is flat with the point reported in world coordinates. The cell name is omitted if the layout database format is Binary or ASCII.

Exception Name: POLYGON_NONSIMPLE

†

EXCEPTION: Acute polygon at location <point> in cell <name> on layer <layer> datatype <datatype>.

A GDSII boundary or OASIS polygon has an acute angle. Reporting is once per cell with the point reported in cell coordinates.

The default severity is 0. Severity may be explicitly set with the specification statement Layout Input Exception Severity. Layout Error On Input does not affect this exception.

Exception Name: POLYGON_ACUTE

†

EXCEPTION: Zero-area polygon at location <point> in cell <name> on layer <layer> datatype <datatype>.

A GDSII boundary or OASIS polygon has zero area. Reporting is once per cell with the point reported in cell coordinates.

Severity may be explicitly set with the specification statement Layout Input Exception Severity. Layout Error On Input does not affect this exception.

Exception Name: POLYGON_AREA_ZERO

†

EXCEPTION: Two-point boundary at location <point> in cell <name> on layer <layer> datatype <datatype>.

A two-point boundary is encountered in a GDSII input layout database.

Reporting is once per cell with the point reported in cell coordinates.

Exception Name: POLYGON_IS_RECTANGLE

†

EXCEPTION: (Rectangle | 2-point boundary | box) of zero length or width at location <point> in cell <name> on layer <layer> datatype <datatype>.

A rectangle has zero length or width. Reporting is once per cell with the point reported in cell coordinates. Note that 2-point boundaries in input layout databases are interpreted by Calibre as orthogonal rectangles.

Exception Name: RECTANGLE_SIDE_ZERO

†*

EXCEPTION: Degenerate (vertex count = <number>) path at location <point> in cell <name> on layer <layer> datatype <datatype>.

A layout path has 0 or 1 vertices. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_DEGENERATE

†*

EXCEPTION: Non-orientable path at location <point> in cell <name> on layer <layer> datatype <datatype>.

A layout path is non-orientable. A non-orientable path is a path that produces a non-orientable polygon when expanded. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_NONORIENTABLE

†

EXCEPTION: Non-simple path at location <point> in cell <name> on layer <layer> datatype <datatype>.

A layout path is non-simple but orientable. A non-simple path is one whose expansion is a non-simple polygon. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_NONSIMPLE

†

EXCEPTION: Path of odd width (<number> dbu) at location <coordinates> in cell <name> on layer <layer> datatype <datatype>.

A GDSII path has a width that is not divisible by 2. Reporting is hierarchical and the width of the path is placed into the message along with the location, cell name, layer, and datatype.

Exception Name: PATH_WIDTH_ODD

†

EXCEPTION: Path of zero width at location <point> in cell <name> on layer <layer> datatype <datatype>.

A layout path has zero width and has been ignored. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_WIDTH_ZERO

†*

EXCEPTION: Spike in path at location <point> in cell <name> on layer <layer> datatype <datatype>.

A layout path has a spike (the path doubles back upon itself). Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_SPIKE

†*

EXCEPTION: Path of absolute width at location <point> in cell <name> on layer <layer> not yet supported.

A layout path with absolute width is encountered. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_WIDTH_ABSOLUTE

†*

EXCEPTION: Problem extending (type 4 | variable end) path at location <point> in cell <name> on layer <layer> datatype <datatype> by <number>.

The system failed to expand a type-4 GDSII or variable-end OASIS path. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_EXTENSION

†

EXCEPTION: Type 4 GDSII Variable end (OASIS) path with negative start (end) extension at location <point> in cell <name> on layer <layer> datatype <datatype>.

A type 4 GDSII path or variable end OASIS path has a negative start or end extension. Reporting is hierarchical and <location> is either the first or last point in the path, depending on whether the negative extension is at the start or end, respectively.

Exception Name: PATH_NEGATIVE

†

EXCEPTION: Path width <width-value> / path width multiple <path-width-multiple-value> is not an even integer. Path at location <point> in cell <name> on layer <layer> datatype <datatype>.

The full path width, as specified in the input layout database in database units, divided by the Layout Path Width Multiple value is not an even integer. The message includes the path width and specification statement value. Reporting is hierarchical.

Exception Name: PATH_WIDTH_MULTIPLE

†

EXCEPTION: Path endsegment length <number> at location <point> in cell <name> on layer <layer> datatype <datatype> is less than 1/2 of path width <number>.

The endsegment length of a (multiple-segment) type-0 GDSII path or flush-ended OASIS path is less than 1/2 of the path width and expansion of the path by the given width would result in a notch at the short end. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_ENDSEGMENT_SHORT

†

EXCEPTION: Angled path at location <point> in cell <name> on layer <layer> datatype <datatype>.

A layout path whose centerline contains one or more angled segments has been encountered. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_ANGLED

†

EXCEPTION: Path with acute angle at location <point> in cell <name> on layer <layer> datatype <datatype>.

A layout path whose centerline contains one or more acute angles has been encountered. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_ACUTE

†

EXCEPTION: Path with coincident endpoints at location <point> in cell <name> on layer <layer> datatype <datatype>.

A layout path of zero length has been encountered. This situation is reported once.

Exception Name: PATH_COINCIDENT

†

EXCEPTION: Path of type 1 at location <point> in cell <name> on layer <layer> datatype <datatype>.

A type-1 (circular endcap) GDSII path has been encountered. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: PATH_CIRCULAR

†*

EXCEPTION: Zero-vertex text object <name> in cell <name> on layer <number>.

A text object has more than one vertex. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: TEXT_DEGENERATE

†*

EXCEPTION: Multi-vertex text object <name> at location <point> in cell <name> on layer <number>.

A text object has more than one vertex. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: TEXT_BIG

†

EXCEPTION: Cell name <name> for LAYOUT TEXT object <name> is not defined.

The cell name of the rule file LAYOUT TEXT object having the given name is not located in the input layout database.

Exception Name: LAYOUT_TEXT

†*

EXCEPTION: Invalid call (rotation is (0,0)) of symbol <number> in cell <name>.

A symbol call with rotation (0,0) is encountered.

Exception Name: CALL_ROTATION

†

EXCEPTION: Circle at location <point> in cell <name> on layer <layer> datatype <datatype>.

An OASIS circle has been encountered. Reporting is once per cell with the (center) point reported in cell coordinates.

Exception Name: CIRCLE

†

EXCEPTION: Circle of radius zero at location <point> in cell <name> on layer <layer> datatype <datatype>.

An OASIS circle has zero diameter. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: CIRCLE_RADIUS_ZERO

†

EXCEPTION: Unsupported XGEOMETRY at location <point> in cell <name> on layer <layer> datatype <datatype>.

An OASIS XGEOMETRY record (not supported) has been encountered. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: XGEOMETRY_UNSUPPORTED

†

EXCEPTION: Degenerate TRAPEZOID at location <point> in cell <name> on layer <layer> datatype <datatype>.

An OASIS TRAPEZOID record is degenerate (that is, cannot be formed into a polygon as specified). Reporting is once per cell with the point reported in cell coordinates.

Exception Name: TRAPEZOID_DEGENERATE

†

EXCEPTION: Degenerate CTRAPEZOID at location <point> in cell <name> on layer <layer>.

An OASIS CTRAPEZOID record is degenerate that is, cannot be formed into a polygon as specified). Reporting is once per cell with the point reported in cell coordinates.

Exception Name: CTRAPEZOID_DEGENERATE

†

EXCEPTION: TRAPEZOID of zero area at location <point> in cell <name> on layer <layer> datatype <datatype>.

An OASIS TRAPEZOID has zero area. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: TRAPEZOID_AREA_ZERO

†

EXCEPTION: CTRAPEZOID of zero area at location <point> in cell <name> on layer <layer> datatype <datatype>.

An OASIS CTRAPEZOID has zero area. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: CTRAPEZOID_AREA_ZERO

†

EXCEPTION: BOX record at location <point> in cell <name> on layer <layer> datatype <datatype>.

A GDSII BOX record has been encountered. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: BOX_RECORD

†

EXCEPTION: NODE record at location <point> in cell <name> on layer <layer> datatype <datatype>.

A GDSII NODE record has been encountered. Reporting is once per cell with the point reported in cell coordinates.

Exception Name: NODE_RECORD

†

EXCEPTION: XNAME record in input file <name> at record offset <number> not supported.

An OASIS XNAME record (unsupported) has been encountered. The file name and record offset of the XNAME record is placed into the message.

Exception Name: XNAME_UNSUPPORTED

†

EXCEPTION: XELEMENT record in input file <name> at record offset <number> not supported.

An OASIS XELEMENT record (unsupported) has been encountered. The file name and record offset of the XELEMENT record is placed into the message.

Exception Name: XELEMENT_UNSUPPORTED

†

EXCEPTION: Non n-string for CELLNAME record in input file <name> at record offset <offset>.

The string in a CELLNAME, CELL, or PLACEMENT record is not an OASIS n-string. The file name and record offset are placed into the message.

Exception Name: CELLNAME_NSTRING

†

EXCEPTION: Non a-string for TEXTSTRING record in input file <name> at record offset <offset>.

The string in a TEXTSTRING or TEXT record is not an OASIS a-string. The file name and record offset are placed into the message. If available, the cell name, text object location, and text object layer are also inserted into the message.

Exception Name: TEXTSTRING_ASTRING

†

EXCEPTION: Non n-string for PROPNAME record in input file <name> at record offset <offset>.

The string in a PROPNAME or PROPERTY record is not an OASIS n-string. The file name and record offset are placed into the message.

Exception Name: PROPNAME_NSTRING
