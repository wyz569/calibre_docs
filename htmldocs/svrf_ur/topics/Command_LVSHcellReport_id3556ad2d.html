LVS Hcell Report

Specification statement

Specifies that Calibre nmLVS-H generate a report giving details about hcell correspondences used during the run. Detailed data is provided in the report to help improve the hcell list. Used only in Calibre nmLVS-H.

Usage

LVS HCELL REPORT filename [BY REJECTION | BY HCELL] [WITH EXPANDED threshold]

Arguments

filename

A required filename of the report.

The filename parameter can contain environment variables. See “Environment Variables in Pathname Parameters”.

BY REJECTION

An optional keyword that specifies correspondence details are grouped by cause for those correspondences that were considered but rejected. The correspondences that were established are listed last. This format facilitates look-up of all correspondences rejected for a particular reason. This is the default behavior.

BY HCELL

An optional keyword that specifies correspondence details are grouped by the hcell specification that led to the correspondence’s consideration. (Each correspondence that was considered was either established or rejected.) This format facilitates look-up of what correspondences were established or rejected in response to each specified hcell. Correspondences considered due to -automatch are shown in their own section. Hcell specifications that did not lead to the consideration of any correspondences are listed as empty sections.

WITH EXPANDED threshold

An optional keyword and number specifying that an UNBALANCED HCELLS EXPANDED section is included in the report when there is at least one hcell that meets the threshold.

The threshold is a positive integer corresponding to a placement count of a cell in the source or layout design after non-hcells have been expanded (ignoring the effects of unbalanced hcell expansion and any LVS Flatten Inside Cell specification).

The additional section of the report lists unbalanced expanded hcells that meet or exceed the threshold. Choosing a threshold of 1 lists all unbalanced expanded hcells.

Description

Specifies the writing of a report describing hcell usage during a hierarchical LVS comparison run. Hcells that are specified by any means are evaluated and reported. Potential hcells that are rejected are listed, along with the reasons. Cells that may act as good hcells are also reported.

The BY REJECTION and BY HCELL keywords control the reporting format of correspondence details only. Other data in the report is unaffected by these keywords.

If the circuit extraction step is performed separately from comparison, then physical layout information germane to hcell reporting is only available in a subsequent comparison run if LVS Hcell Report is specified for the circuit extraction run.

Details of the report are given under “Calibre LVS Hcell Report Format” in the Calibre Verification User’s Manual. Other related information appears in the “Hcells” section of the Calibre Verification User’s Manual and the “Hcell and Hierarchical Analysis” chapter of the Calibre Query Server Manual.

See also Hcell, LVS Exclude Hcell, LVS Auto Expand Hcells, LVS Expand Ambiguously High Shorted Hcells, and LVS Expand On Error.

Examples

For this specification:

LVS HCELL REPORT "lvs.report.hcells"

                              POTENTIAL HCELL ADDITIONS
                              =========================

HIC IMPROVEMENT - LAYOUT

These layout cells have the potential to reduce the layout Hierarchical 
Instance Count (HIC) by roughly the stated percentage. Reducing the HIC
tends to reduce both the LVS memory requirement and the  LVS run time.

  Cell name           HIC reduction
  ---------           -------------
  INVX2                          5%


NOTE: The HIC reduction factors depend on the set of hcells to which the 
given one is added. Therefore, adding any of these as a new hcell will
invalidate the metrics for the others. The hcell analyzer in the Calibre
Query Server overcomes this limitation by updating the metrics as each new
addition is considered. It is the  preferred tool for finding potential
hcell additions.
