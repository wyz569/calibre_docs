PEX Ignore Resistance Marker

Parasitic extraction

Used to ignore parasitic resistance on one or more specified layers during extraction in the regions defined by a marker layer.

Usage

PEX IGNORE RESISTANCE MARKER [layer [layer…]]
[VIA layer1 layer2 [VIANAME vianame] [VIA layer1 layer2 [VIANAME vianame]]…] marker_layer

Arguments

marker_layer

A required parameter that specifies an original or derived polygon layer used to mark a region of interest on the conductor layer(s). This layer must be passed through the PHDB using the LVS DB Layer statement.

At least one of the following keyword sets must be specified:

layer

One or more original or derived conductor layers. At least one layer must be specified.

VIA layer1 layer2 [VIANAME vianame]

A keyword set that specifies to ignore via resistance on vias connecting layer1 and layer2 within the specified marker region.

VIANAME vianame is an optional keyword set that specifies to ignore resistance for a specifically named via, vianame, within the specified marker region. May only be specified once with each VIA layer1 layer2 keyword set.

More than one via connection may be specified.

Description

An optional statement used to ignore any resistivity on one or more specified conductor and/or via layers within the drawn regions of the marker layer during extraction. Connectivity is unchanged but the region covered by the marker layer contributes no resistance to the parasitic extraction results. When two or more conductor layers are specified, the contact or vias connecting those layers are also ignored. This statement can be specified more than once. The same original or derived conductor layer cannot be specified for more than one marker layer.

Examples

Example 1

To ignore resistance for the area of metal1_net that is overlapped by marker layer MARKER1, include the following statements in your rule file:

LAYER MARKER1 555
LVS DB LAYER MARKER1
PEX IGNORE RESISTANCE MARKER metal1_net MARKER1

Example 2

The diagram shown in Figure 1 is used to illustrate how resistance is ignored for vias under a marker layer.

Figure 1. Resistance ignores for Marker Layers

The statement 

PEX IGNORE RESISTANCE MARKER M0 M1 VIA M0 M1 res_marker

specifies to ignore the resistance on layers M0 and M1, and on any of the vias connecting layer M0 and M1 that are under the marker layer named res_marker. In this case, the resistance for layers M0, VIA0, and M1 under the res_marker layer is ignored.

The statement

PEX IGNORE RESISTANCE MARKER VIA M0 M1 VIANAME VIA0 VIA M1 M2 VIANAME VIA1 res_marker

specifies to ignore the resistance on via layers VIA0 and VIA1 under the marker layer res_marker. However, in Figure 1, VIA1 is not under the res_marker marker layer, so resistance for VIA1 is not ignored.

The statement

PEX IGNORE RESISTANCE MARKER VIA M0 M1 VIA M1 M2 res_marker

also specifies to ignore the resistance on via layers VIA0 and VIA1 under the marker layer res_marker, however, since VIA1 is not under the res_marker layer, the resistance is not ignored.

Example 3

In Figure 2, more than one via connects M0 and M1.

Figure 2. Resistance Ignores for Two Vias

Use the statement:

PEX IGNORE RESISTANCE MARKER VIA M0 M1 VIANAME VIA0_b res_marker

to ignore the resistance on via layer VIA0_b under the marker layer res_marker.
