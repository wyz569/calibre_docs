DFM Critical Area

Arguments

input_layer

A required original or derived polygon layer. This is the interconnect layer on which to perform analysis.

Application Keywords for Outputting Short and Open Defect Areas

These keywords specify the type of failure for which defect areas are output. Either SHORT, OPENWIRE, OPENUWIRE, or OPENVIA must be specified for the DFM Critical Area command.

SHORT [EXCLUDE EDGE exclude_layer] [POLYGONAL]

A keyword, with exclusive options, that detects shorts between two different nets on input_layer. Connectivity is required on the input_layer.

  • EXCLUDE EDGE exclude_layer

    An optional keyword used for SHORT keyword calculations. EXCLUDE EDGE excludes edges from input_layer in the short calculation when input_layer edges are coincident with edges specified by exclude_layer. EXCLUDE EDGE suppresses false critical area results on gate regions when extracting short critical area on the active layer.

  • POLYGONAL

    An optional keyword that enables hyperscaling for short calculations. If POLYGONAL is not specified in a -hyper run, DFM Critical Area runs on HDB0 only and produces identical results to a non-hyperscaling run.

    By default, DFM Critical Area is a nodal operation, using hierarchical connectivity information present on the input_layer. The POLYGONAL keyword allows short calculations to be made without hierarchical connectivity information.

    Defect results using POLYGONAL may differ from runs without POLYGONAL. Defects may be output between polygons that are connected in various places through the hierarchy. However, U shapes, notches, or other shapes from single polygons within a cell never result in critical area output.

OPENWIRE

Detects open circuits and defects covering line-end portions on the input_layer.

OPENUWIRE

Detects open circuits on the input_layer. This type of analysis recognizes the line-end portions of the interconnect based on edge and corner conditions. It does not report defects that cover line-ends that do not block current flow.

OPENVIA [enclosure_layer]

Detects open via connections to or from the input_layer. OPENVIA optionally provides an enclosure_layer argument, a derived polygon layer enclosing vias that comprise an array, treated as a single point of connection between two layers. OPENVIA can also be used in conjunction with a separate OPENUWIRE operation to catch defects blocking metal regions above or below vias, in addition to the vias themselves.

In the following code, the derivation of the enclosure_layer , via1_enc, typically establishes a conjunction of the top and bottom interconnect layers, using M1 AND M2, and then enclosing via shapes by the ENCLOSE via1 operation.

via1_enc = (M1 AND M2) ENCLOSE via1 > 1

If the enclosure_layer is not specified with OPENVIA, all vias on the input_layer are assumed to be individual connections (non-redundant in the case of arrays).

General Keywords

S defect_radius [OUTPUT] [S defect_radius] ...

A required keyword that specifies the radius of a defect area. The defect_radius must be a non-negative real number expressed in user units specified as a numeric value or a variable. By default, the actual defect is modeled by a square with a width equal to twice the defect_radius (2*defect_radius). This keyword may be specified any number of times. The defect radii specified must be ordered from smallest to largest.

  • OUTPUT

    An optional keyword that specifies which defect_radius parameter to use for determining the critical area output. This keyword is positional. It must immediately follow the defect_radius parameter of interest.

    The OUTPUT keyword may only be specified once. It is only needed if more than one S keyword appears in the rule. If this keyword is omitted, the largest defect_radius in the rule is output, which includes all radii that are less than or equal to the largest radius.

D depth

An optional keyword that specifies a distance between a defect boundary and the edge of the input_layer polygon. The D keyword may only appear once per DFM Critical Area command. Separate commands for each D value must be specified. The depth must be a non-negative real number in user units specified as a numeric value or a variable. The default defect depth is 0.

DFM Critical Area interprets depth differently depending on type of failure:

  • For open circuits, the depth models the remaining minimum amount of material that must remain for functional current flow. It defines the distance between the exterior-facing edge of a defect and the interior-facing edge of a wire polygon. As the depth increases for a given S defect_radius, the critical area increases.

    Figure 1. Open Circuit Defect Area

  • For short circuits, the depth models the minimum amount of overlap that causes a short. The depth is the distance between the inside-facing edge of a defect and the inside edge of a wire polygon. The wire must be larger than or equal to the depth to conduct current. As depth decreases for a given S defect_radius, the critical area increases.

    Figure 2. Short Circuit Defect Area

METRIC SQUARE

An optional keyword that enables the square metric when determining edges within the defect radius and a square defect model for critical area region generation. The Euclidean metric is used for the defect radius by default. METRIC SQUARE enables the generation of a critical area using a true representation of a square defect. METRIC SQUARE cannot be used with OPENVIA.

REGION OCTAGONAL

An optional keyword that enables use of equilateral octagons instead of squares for critical area region generation. REGIONAL_OCTAGONAL uses the default Euclidean metric when determining edges within the defect radius.

Description

DFM Critical Area calculates and outputs critical areas associated with open and short circuits for manufacturing defect analysis. It finds critical areas in a layout where high probabilities exist that defects of the given radii, modeling random particles, would cause an open or a short circuit.

The SHORT keyword addresses short circuit applications, and the OPENWIRE, OPENUWIRE, and OPENVIA keywords address open circuit applications.

No additional keywords are required for enabling hyperscaling for DFM Critical Area with the OPENWIRE, OPENUWIRE, and OPENVIA keywords. The POLYGONAL option is required to enable hyperscaling when SHORT is specified.

The shape of the defect has an effect on the size of the critical area. A circular defect area is generally the most accurate model of irregularly shaped defects practically encountered. Because of runtime constraints, the DFM Critical Area operation uses a default square defect of

width = 2*defect_radius

This results in a slightly larger defect area than a circular defect of defect_radius. The calculation of the critical area for a square defect is more efficient, because the result for orthogonal input is an orthogonal defect area that can be stored and analyzed more easily. Calibre can also calculate defect area using octagonal defect shapes.

The METRIC SQUARE and REGION OCTAGONAL keywords allow you to change the measurement method Calibre uses to determine edges within the defect radius and the shape of the defect model used. These options can be used to change defect area accuracy as seen in the following table and figure.

Concurrency

The way DFM Critical Area commands are written has a significant run-time impact. All DFM Critical Area commands specifying the same input layer, failure type, D parameter, and S parameters are performed concurrently. Defects within critical areas of the layout for each size range should be reported on different layers for output. For example, these are three defect ranges are grouped by size, and the minimum defect overlap causing a short between metal lines is 0.005:

  • 0 to 0.01

  • 0.01 to 0.015

  • 0.015 to 0.02

This solution using separately processed DFM Critical Area commands works, but is relatively slow:

X = DFM CRITICAL AREA m1 SHORT S 0.01 OUTPUT D 0.005
Y = DFM CRITICAL AREA m1 SHORT S 0.015 OUTPUT D 0.005
Z = DFM CRITICAL AREA m1 SHORT S 0.02 OUTPUT D 0.005

This solution using concurrently processed DFM Critical Area commands reduces run-time by two thirds:

X = DFM CRITICAL AREA m1 SHORT S 0.01 OUTPUT S 0.015 S 0.02 D 0.005
Y = DFM CRITICAL AREA m1 SHORT S 0.01 S 0.015 OUTPUT S 0.02 D 0.005
Z = DFM CRITICAL AREA m1 SHORT S 0.01 S 0.015 S 0.02 OUTPUT D 0.005

The location of the OUTPUT keyword specifies which defect radius is used for the output data.

Running concurrent operations calculates critical areas for each size listed, regardless of whether results are reported. In this example, the third operation is commented; calculations of the critical area for all three sizes are performed, but only the first two are output:

X = DFM CRITICAL AREA m1 SHORT S 0.01 OUTPUT S 0.015 S 0.02 D 0.005
Y = DFM CRITICAL AREA m1 SHORT S 0.01 S 0.015 OUTPUT S 0.02 D 0.005
//Z = DFM CRITICAL AREA m1 SHORT S 0.01 S 0.015 S 0.02 OUTPUT D 0.005

Optimal performance is delivered when concurrent operations only calculate the sizes of interest. For example, if shorts induced by particles in the 0 to 0.01 and 0.01 to 0.015 ranges are of interest only, the best performance is found by excluding the defect_radius specification of S 0.02:

X = DFM CRITICAL AREA m1 SHORT S 0.01 OUTPUT S 0.015 D 0.005
Y = DFM CRITICAL AREA m1 SHORT S 0.01 S 0.015 OUTPUT D 0.005

Examples

Example 1

These steps are used to locate open defects caused by short circuits:

  • Identify the critical areas on the input_layer, m1.

  • Focus on shorts between wires on different nets.

  • Specify three specific particle sizes: 0.5, 1.0, and 2.0.

  • Output the critical area for each particle size on a different output layer.

  • Ensure particles must overlap polygons by at least 0.25 microns to cause a problem.

The approach for performing a critical area check with these short circuit constraints are shown here:

m1_short1 = DFM CRITICAL AREA m1 SHORT S 0.5 OUTPUT S 1.0 S 2.0 D 0.25
m1_short2 = DFM CRITICAL AREA m1 SHORT S 0.5 S 1.0 OUTPUT S 2.0 D 0.25
m1_short3 = DFM CRITICAL AREA m1 SHORT S 0.5 S 1.0 S 2.0 OUTPUT D 0.25

The following figure demonstrates identification of critical areas for wire regions on a single layer.

Figure 4. Single-Layer Short Critical Area Model

In the preceding figure, the dimension, d, is the distance between an interior-facing edge of a wire and the interior-facing edge of a defect. This distance is the minimum overlap of a wire polygon by a defect, and this overlap is referred to as depth. If the edges of the interconnect polygons satisfy the expression:

EXTERNAL layer < 2 * (r - d)

then they lie within a critical area region. The critical area region is extended by the defect_radius length from each end of the wire.

Example 2

These steps are used to locate open defects caused by compromised vias:

  • Critical areas on the input_layer, via1, must be identified.

  • Focus on defects covering vias in such a way as to inhibit current flow.

  • Specify three specific particle sizes: 0.5, 1.0, and 2.0.

  • Output each particle size saved to a different output layer.

  • Ensure particles must overlap a via by at least 0.25 microns to cause a problem.

The following code demonstrates performing a critical area check for compromised vias:

via1_open1 = DFM CRITICAL AREA via1 OPENVIA
	S 0.5 OUTPUT S 1.0 S 2.0 D 0.25
via1_open2 = DFM CRITICAL AREA via1 OPENVIA
	S 0.5 S 1.0 OUTPUT S 2.0 D 0.25
via1_open3 = DFM CRITICAL AREA via1 OPENVIA
	S 0.5 S 1.0 S 2.0 OUTPUT D 0.25
Figure 5. Using OPENVIA for Isolated Via Critical Area Analysis

The preceding figure illustrates the dimension d is the distance between an interior-facing edge of an isolated via and the exterior-facing edge of a defect. This distance is the maximum width of a via left remaining by a defect.

The following figure shows the enclosure_layer enclosing all vias of the array. Where vias are enclosed by the enclosure_layer, they must be completely overlapped by the defect area to be blocked. The vias and the defect area determine the size and location of the critical area.

Figure 6. Using OPENVIA for Arrayed Via Critical Area Analysis

The following code illustrates deriving the enclosure_layer and then invoking DFM Critical Area to process via arrays:

array_via1 = (M1 and M2) ENCLOSE via1 > 1
via1_open1 = DFM CRITICAL AREA via1 OPENVIA array_via1 
	S 0.5 OUTPUT S 1.0 S 2.0 D 0.25
via1_open2 = DFM CRITICAL AREA via1 OPENVIA array_via1 
	S 0.5 S 1.0 OUTPUT S 2.0 D 0.25 
via1_open3 = DFM CRITICAL AREA via1 OPENVIA array_via1
	S 0.5 S 1.0 S 2.0 OUTPUT D 0.25

Example 3

This example explores two methods to locate single-layer wire open defects: using Method A: Using OPENWIRE, or using Method B: Using OPENUWIRE with OPENVIA. Both methods use steps to locate open defects caused by open circuits:

  • Critical areas on the input_layer, m1, must be identified.

  • Focus on defects that break wires on any net.

  • Specify three specific particle sizes: 0.5, 1.0, and 2.0.

  • Output the critical areas for each particle size as a different output layer.

  • Consideration of particles within a 0.25 micron proximity from the edge of a wire may cause an open circuit.

Method A: Using OPENWIRE

DFM Critical Area uses a different heuristic for general interconnect layer regions. With this method, OPENWIRE finds the open circuits in the input_layer. The following code provides a derivation solution using the OPENWIRE method:

m1_open1 = DFM CRITICAL AREA m1 OPENWIRE S 0.5 OUTPUT S 1.0 S 2.0 D 0.25
m1_open2 = DFM CRITICAL AREA m1 OPENWIRE S 0.5 S 1.0 OUTPUT S 2.0 D 0.25
m1_open3 = DFM CRITICAL AREA m1 OPENWIRE S 0.5 S 1.0 S 2.0 OUTPUT D 0.25

A portion of the width of the original polygon can remain intact, but this is still modeled as an open. The following figure shows how this is modeled for wire regions.

Figure 7. Single-Layer Open Critical Area Model

The preceding figure illustrates that dimension d is the distance between an interior-facing edge of a wire and the exterior-facing edge of a defect. This distance is the maximum remaining width of any polygon interacting with a defect area.

Method B: Using OPENUWIRE with OPENVIA

With this method, the OPENUWIRE keyword is used instead of OPENWIRE to enable handling general interconnect polygons, and OPENVIA is used to handle the wire regions containing vias.

The following code demonstrates what is required for the conjunctive OPENUWIRE and OPENVIA method:

m1_open1 = DFM CRITICAL AREA m1 OPENUWIRE S 0.5 OUTPUT S 1.0 S 2.0 D 0.25
m1_open2 = DFM CRITICAL AREA m1 OPENUWIRE S 0.5 S 1.0 OUTPUT S 2.0 D 0.25
m1_open3 = DFM CRITICAL AREA m1 OPENUWIRE S 0.5 S 1.0 S 2.0 OUTPUT D 0.25
via1_open1 = DFM CRITICAL AREA via1 OPENVIA array_via1 
	S 0.5 OUTPUT S 1.0 S 2.0 D 0.25
via1_open2 = DFM CRITICAL AREA via1 OPENVIA array_via1 
	S 0.5 S 1.0 OUTPUT S 2.0 D 0.25
via1_open3 = DFM CRITICAL AREA via1 OPENVIA array_via1 
	S 0.5 S 1.0 S 2.0 OUTPUT D 0.25
m1_crit_area1 = m1_open1 OR via1_open1 
m1_crit_area2 = m1_open2 OR via1_open2
m1_crit_area3 = m1_open3 OR via1_open3

The critical area region generated by the OPENWIRE method will generally be larger than for OPENUWIRE. The following figure compares the two method behaviors.

Figure 8. OPENUWIRE and OPENWIRE

Example 4

This example visually compares the difference between OPENWIRE, OPENUWIRE, OPENVIA and the logical combination of OPENUWIRE and OPENVIA. These steps are used to locate these defects:

  • A square defect model is established with a radius (r) of 0.2.

  • A depth value (D) of zero is established. This is only for conceptual simplicity. In practice, the depth is normally set to a value other than zero.

  • The critical area is generated from potential opens on a metal layer (M1) and a via layer (VIA1).

The code shown here provides a solution:

M1_OW_040 = DFM CRITICAL AREA M1 OPENWIRE S 0.20 D 0
M1_OU_040 = DFM CRITICAL AREA M1 OPENUWIRE S 0.20 D 0
M1_OV_040 = DFM CRITICAL AREA VIA1 OPENVIA S 0.20 D 0
M1_OUOV_040 = OR M1_OU_040 M1_OV_040

The following figure shows the critical area that is generated from these checks for a sample design.

Figure 9. Open Examples

For the OPENWIRE case, critical area is generated whenever the defect particle covers a line‑end or causes an open on M1. For the OPENUWIRE case, critical area is generated only if the defect causes an open on M1. The OPENUWIRE and OPENVIA case is a logical OR of their independent results.

Example 5

This example checks for shorts on the active layer, specifically between gates. The gate shown in the following figure provides the example layout.

Figure 10. Generic Two-Input Gate

Performing critical area analysis on the entire active layer may report results of an excessively large critical area, depicted in the following figure.

Figure 11. Large Defect Area

Additionally, if a critical area analysis is performed on the source-drain regions (where sd = active NOT poly), the operation returns an even larger area that includes shorts between false edges on the sd layer as seen in the following figure.

Figure 12. False Edges

The correct way to calculate critical area in the gate region of the active layer is by checking the sd layer, excluding the false edges. A gate layer is generated that identifies false edges, then using the EXCLUDE EDGE keyword to exclude all edges coincident with the gate layer. The following code provides a solution, and Figure 13 demonstrates this solution topologically.

gate = poly AND active
sd = active NOT poly
CONNECT M1 sd BY contact
short_layer = DFM CRITICAL AREA SHORT sd
	EXCLUDE EDGE gate S 0.080 OUTPUT
Figure 13. Finding the Correct Critical Area

Example 6

This example demonstrates the use of octagonal shapes to eliminate artifacts in critical area regions. In designs containing edges that are not vertical or horizontal, the square defect approximation can generate triangular artifacts in the critical area region. For example, the following rule uses a square with edge length 0.220 to model a defect particle in a design containing 45-degree edges:

SHORT_M1_SQ = DFM CRITICAL AREA M1 SHORT S 0.110

which produces the critical area shown in the following figure.

Figure 14. Critical Area Artifacts

You can eliminate triangular artifacts for 45-degree edges by specifying the REGION OCTAGONAL keyword with the DFM Critical Area operation. This keyword uses regular octagons instead of squares to approximate defects when calculating critical area shown in the following figure.

Figure 15. Critical Area Calculation

In Figure 15, the rule is modified as follows:

SHORT_M1_OCT = DFM CRITICAL AREA M1 SHORT REGION OCTAGONAL S 0.110

which produces following critical area in the following figure.

Figure 16. Final Critical Area

The elimination of artifacts is achieved at the expense of an increase in runtime. Also, using the octagonal approximation only eliminates artifacts for 45-degree edges. Edges that are not at angles of 45-degree multiples still produce artifacts.
