LVS Abort On Supply Error

Specification statement

Specifies whether to abort processing when connectivity extraction detects a problem with the power or ground nets. Observed in calibre ‑dfm, ‑lvs, and ‑perc runs. Not used in ICverify.

Usage

LVS ABORT ON SUPPLY ERROR {YES | NO}

Arguments

YES

A keyword that instructs the tool to abort processing after connectivity extraction detects a problem with any of the power or ground nets. This is the default behavior for flat Calibre nmLVS.

NO

A keyword that instructs the tool not to abort processing when connectivity extraction detects a problem with any of the power or ground nets. This is the default behavior for Calibre nmLVS-H.

Description

Specifies whether to abort processing when connectivity extraction detects a problem with any of the nets appearing in the LVS Power Name and LVS Ground Name specification statements.

This transcript error is issued when YES is specified and a supply problem is detected:

ERROR: Supply error detected. ABORT ON SUPPLY ERROR is specified ‑ aborting.

Problems that cause processing to abort include these:

  • Connectivity extraction did not assign a power or ground name to a net because the name conflicted with another name that was found on the net (a text short).

  • Connectivity extraction found a single power or ground name on two disjoint nets (an open).

  • Connectivity extraction found an unattached power or ground name.

  • Connectivity extraction found an ambiguously attached top-level text.

  • Connectivity extraction found an error in a Stamp operation involving a power or ground net.

If a supply label is involved with a text short to a non-supply label and the supply label prevails, this does not cause the tool to abort when YES is specified.

In hierarchical runs, connectivity problems must affect the top-level cell in order for the run to abort.

When YES is specified and a supply problem is detected, LVS aborts after circuit extraction is complete but before the comparison stage. If hyperscaling is used (‑hyper cmp command line option), then LVS does not terminate until after the source netlist is read.

When circuit extraction aborts, the extracted layout netlist contains a comment that causes any future LVS comparison to abort when that netlist is used.

In a calibre -dfm run, the DFM Database DEVICES keyword must be specified for the YES keyword to have any effect.

A Calibre PERC run aborts before the PERC module if a supply error is detected and the YES keyword is specified.

The NO option is most frequently specified when debugging issues involving power and ground nets. LVS Abort On Supply Short YES can be used to stop processing only when a supply short is detected rather than other supply errors.

You can specify power supply management options in Calibre Interactive—nmLVS. See “Setting Power Supply Options” in the Calibre Interactive User’s Manual.

Examples

These rule file statements cause a combined LVS netlist extraction and comparison run to abort if there is a supply problem:

LVS POWER NAME pwr
LVS GROUND NAME gnd
// Abort LVS before the comparison phase upon detection of a supply error.
LVS ABORT ON SUPPLY ERROR YES

When supply errors occur, check the circuit extraction report.
