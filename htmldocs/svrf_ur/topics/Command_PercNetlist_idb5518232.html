PERC Netlist

Specification statement

Specifies the input netlist for Calibre PERC runs.

Usage

PERC NETLIST {LAYOUT | SOURCE}

Arguments

LAYOUT

A required keyword that specifies Calibre PERC operates on the layout netlist. This is the default if this statement is not included in the rule file.

SOURCE

A required keyword that specifies Calibre PERC operates on the source netlist.

Description

Specifies the input design netlist for Calibre PERC. This statement may appear at most once.

If your Layout System is geometric (not SPICE) and the LAYOUT keyword is in effect, then circuit extraction is performed automatically when Mask SVDB Directory is specified. The extracted layout netlist is stored under that directory. If Mask SVDB Directory is unspecified, then the -spice command line option is needed to extract a netlist.

When the -layout command line option is specified, the netlist referenced by that option is used during the run regardless of what appears in the rule file.

You can cause Calibre PERC to output its internal view of the connectivity by specifying LVS Write Layout Netlist or LVS Write Source Netlist. The type of output netlist should match either the LAYOUT or the SOURCE keywords, otherwise the output netlist is empty.

See also PERC Load.

Examples

Example 1

This example shows the configuration for a Calibre PERC run on a source netlist. Typically, you will want to check your source netlist before creating the layout design.

SOURCE SYSTEM SPICE
SOURCE PATH "/home/designs/spice/top.spi"
SOURCE PRIMARY top

PERC NETLIST SOURCE
PERC REPORT "perc.source.report"
PERC LOAD perc_check INIT name_nets SELECT check1 check2
