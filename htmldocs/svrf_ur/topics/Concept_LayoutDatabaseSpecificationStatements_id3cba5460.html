Layout Database Specification Statements

This section lists the layout database specification statements. These are specification statements used to describe contents of layout databases, pathnames, manipulation of layout objects, global flagging of error conditions in the layout, hierarchy management of cells and layers, and layout input control.

These statements are used in all Calibre and ICverify applications except where noted.

Table 1. Specification Statements: Layout Database

Statement

Description

Exclude Acute

Specifies layers to be excluded from Flag Acute and Drawn Acute checking. Not used in ICverify.

Exclude Angled

Specifies layers to be excluded from Flag Angled and Drawn Angled checking. Not used in ICverify.

Exclude Cell

Specifies layout cells that are not to be processed.

Exclude Offgrid

Specifies layers to be excluded from Flag Offgrid and Drawn Offgrid checking. Not used in ICverify.

Exclude Skew

Specifies layers to be excluded from Flag Skew and Drawn Skew checking. Not used in ICverify.

Expand Cell

Specifies cells having instances to be flattened one level into the space of parent cells in the hierarchy. Used in hierarchical Calibre applications.

Expand Cell Text

Specifies that the text from placements of a source cell be used higher up in the layout hierarchy. Used in hierarchical Calibre applications.

Flag Acute

Specifies to output acute angle information to the run transcript. Only layers actually needed for the run are read.

Flag Angled

Specifies to output information about edges that are not orthogonal to the database axes to the run transcript. Only layers actually needed for the run are read. Not used in ICverify.

Flag Nonsimple [Polygon]

Specifies to output information about non-simple polygons to the run transcript. Only layers actually needed for the run are read. Not used in ICverify.

Flag Nonsimple Path

Specifies to output information about non-simple paths to the run transcript. Only layers actually needed for the run are read. Not used in ICverify.

Flag Offgrid

Specifies to output information about offgrid vertices to the run transcript. Only layers actually needed for the run are read.

Flag Skew

Specifies to output information about skew edges to the run transcript. Skew edges do not include 45-degree edges. Only layers actually needed for the run are read.

Flatten Cell

Specifies the cells having instances to be expanded into the space of parent cells. Used only in hierarchical Calibre applications.

Flatten Inside Cell

Specifies that the internal hierarchy of each cell in the parameter list is to be flattened during the hierarchical database construction phase. Used only in hierarchical Calibre applications.

Layer

Specifies input layer names and numbers to be used in the rule file.

Layer Directory

Specifies a directory for disk-based layers. Not used in Calibre nmLVS or Calibre xRC applications.

Layer Map

Specifies datatype or texttype maps from GDSII or OASIS input to Calibre layer numbers. Not used in ICverify.

Layer Resolution

Overrides the default Resolution specification statement parameters for any number of original layers during off-grid vertex checking for Flag Offgrid, Drawn Offgrid, or the ICrules CHEck DRc -FLAGOFFGRID option.

Layout Allow Duplicate Cell[s]

Specifies whether multiple records for the same layout cell are allowed. Not used in ICverify.

Layout Base Cell

Specifies base cell names for gate array designs. Used only in hierarchical Calibre applications.

Layout Base Layer

Specifies layers that are considered when determining design style. Used only in hierarchical Calibre applications. Counterpart to Layout Top Layer.

Layout Bump2

Specifies that the layer numbers in a database are increased by a specified value. For use with dual database capability. Not used in ICverify.

Layout Case

Specifies whether the layout should be processed in a case-sensitive manner. Not used in DRC applications.

Layout Cell List

Specifies a cell list for statements that use cell-specific processing. Used only in hierarchical Calibre applications.

Layout Clone By Layer

Specifies to clone cells during the hierarchical database (HDB) construction phase if they are marked by a layer.

Layout Clone Rotated Placements

Specifies whether to clone cells that are rotated by 90 or 270 degrees during HDB construction.

Layout Clone Transformed Placements

Specifies whether to use cell cloning for all rotated or transformed cells during HDB construction.

Layout Depth

Specifies the hierarchical depth for reading shapes from the layout database. Not used in ICverify.

Layout Dummy Layer

Specifies dummy fill layers for optimized handling during hierarchical database construction.

Layout Error On Input

Specifies whether certain warnings reported when reading the layout database are to become fatal errors. Not used in ICverify.

Layout Ignore Layer

Specifies to treat one or more specified layers as geometrically empty in a GDSII or OASIS layout input database.

Layout Ignore Text

Specifies to discard layout database text for cells in a given list. Used only in hierarchical Calibre applications.

Layout Input Exception RDB

Specifies a results database for storing exception objects from specified Layout Input Exception Severity statements.

Layout Input Exception Severity

Specifies how layout input exceptions are handled. Not used in ICverify.

Layout Magnify

Specifies that the input layout database is magnified by the given value as it is being read into the Calibre application.

Layout Merge On Input

Directs the layout reader to merge polygons on a per-cell, per-layer basis as it reads the stream into memory. Not used in ICverify.

Layout Net List

Specifies net names for DFM CAF operations.

Layout Path

Specifies the layout database pathname(s) for the database type specified by the Layout System statement. Not used in ICverify

Layout Path Extension Multiple

Specifies a value that is used to detect whether a path extension is an odd multiple of the new database unit after grid conversion.

Layout Path Width Multiple

Specifies a multiple value that is used to detect whether the full path width is an odd multiple of the new database units after grid conversion.

Layout Path2

Specifies the layout database pathname(s) for the database type in the Layout System2 statement. For use with dual database comparison. Not used in ICverify.

Layout Place Cell

Specifies to construct a GDSII or OASIS database from parameters given in this statement.

Layout Placement Multiple

Specifies a value that is used to detect whether placements are offgrid.

Layout Polygon

Allows original database shapes to be specified directly in the rule file. Used only in Calibre.

Layout Precision

Allows specification of an alternate precision value for checking the precision of input layout databases in Calibre.

Layout Preserve Cell List

Specifies to protect certain layout cells from most hierarchy modifications, including automatic expansion, during circuit extraction in Calibre nmLVS-H.

Layout Primary

Specifies the layout database topcell name for Calibre. Not used in ICverify.

Layout Primary2

Specifies the layout database topcell name for a second database in Calibre. For use with dual-database comparison. Not used in ICverify.

Layout Process Box Record

Specifies whether BOX and BOXTYPE records in GDSII input layout databases are processed. Not used in ICverify.

Layout Process Node Record

Specifies whether NODE and NODETYPE records in GDSII input layout databases are processed. Not used in ICverify.

Layout Property Audit

Specifies property names and values to be inserted, appended, or modified in an OASIS Calibre nmDRC results database.

Layout Property Text

Specifies to treat properties attached to shapes as text objects with PROPVALUE as the name of the text object. Used only in Calibre.

Layout Property Text OASIS

Specifies to treat OASIS properties as input layout database text. Used only in Calibre.

Layout Rename All

Specifies to use unique identification in all cells across multiple input files. Used only in Calibre.

Layout Rename Cell

Specifies cells to be renamed in the geometric input layout database. Not used in ICverify.

Layout Rename ICV

Specifies to use unique identification of clone cells across multiple input files. Used only in Calibre.

Layout Rename Clone

Specifies to use unique identification of ICV cells across multiple files. Used only in Calibre.

Layout Rename Text

Specifies text values to be edited or replaced. The statement edits the text value as it is being read. Not used in ICverify.

Layout System

Specifies the layout database format. Not used in ICverify.

Layout System2

Specifies the format of a second layout database. For use with dual-database comparison. Not used in ICverify.

Layout Text

Specifies layout database text directly in the rule file. Not used in ICverify.

Layout Text File

Specifies a file that contains Layout Text objects. This is used when there are many text objects to specify. Not used in ICverify.

Layout Top Layer

Specifies layers that are ignored when distinguishing the design style. Used in hierarchical Calibre applications. Counterpart to Layout Base Layer.

Layout Use Database Precision

Specifies to use the input database precision rather than the rule file precision.

Layout Windel

Specifies a region that defines the exclusion of input shapes and text from use in rule checks. Not used in ICverify.

Layout Windel Cell

Specifies a list of cell names whose placement extents are used to exclude input shapes and text from use in rule checks. Not used in ICverify.

Layout Windel Layer

Specifies a list of layers whose shapes are used to exclude input shapes and text from use in rule checks. Not used in ICverify.

Layout Window

Specifies a region that defines the inclusion of input shapes and text for use in rule checks. Not used in ICverify.

Layout Window Cell

Specifies a list of cell names whose placement extents are used to include input shapes and text for use in rule checks. Not used in ICverify.

Layout Window Clip

Specifies whether area-based filtering is confined to what lies strictly within window regions. Not used in ICverify.

Layout Window Layer

Specifies a list of layers whose shapes are used to include input shapes and text for use in rule checks. Not used in ICverify.

Polygon

Specifies original database shapes directly in the rule file.

Port Depth

Specifies the hierarchical depth for reading of port objects for use in the top-level cell. Not used in ICverify or Calibre nmDRC applications.

Port Layer Polygon

Specifies shapes to treated as geometric ports. Not used in ICverify or by Calibre nmDRC applications.

Port Layer Text

Specifies text objects to be treated as text ports. Not used in ICverify or by Calibre nmDRC applications.

Precision

Defines the ratio of database units to user units.

Push Cell

Specifies cell names whose placements are to be completely flattened, then pushed to the lowest possible level by hierarchical Calibre applications.

Resolution

Defines the layout grid step-size (the number of database units on which original geometry is required to be aligned).

Snap Offgrid

Specifies that original layer shapes have their vertices placed on the grid defined by Resolution or Layer Resolution statements.

SVRF FDI Options

Specifies FDI options for controlling the read-in of third-party databases by Calibre tools that support them.

Text

Specifies top-level layout database text objects (free-standing text) directly in the rule file. It also allows text objects read from the layout database to be edited.

Text Depth

Specifies the hierarchical depth for reading text objects from the layout database for the connectivity extractor.

Text Layer

Specifies the layers in the database from which connectivity extraction text is read.
