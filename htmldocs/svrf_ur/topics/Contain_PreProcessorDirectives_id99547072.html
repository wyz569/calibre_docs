Pre-Processor Directives

Pre-processor directives, also known as conditionals, include rule file compiler directives for technology process control, version control, and environment variables.

Pre-processor directives have these special considerations:

  • The pre-processor directives used for Calibre and ICverify applications are not as general as most C-language pre-processors.This results in testing only the existence of variables, not expressions.

  • You cannot put multiple pre-processor directives on one line.

  • Each pre-processor directive must be the first entry on a line, except for spaces.

  • Conditional compilation directives that have dependencies on Include statements or C-style comments (⁄* … *⁄ ) are compiled after Include statements and before C-style comments.

    In general, you should not comment out pre-processor directives, but instead use the correct pre-processor directive to get the required logic. For example, use an #UNDEFINE statement to cancel a #DEFINE statement.

Summary
About Conditional Directives for Compilation ControlThe portions of the rule file that Calibre or ICverify compiles can be controlled with the #DEFINE, #UNDEFINE, #IFDEF, #IFNDEF directives within your rule file. These directives define and undefine variables and provide conditional execution based on the variable definition. Variables for compilation control can also be defined as environment variables in the shell. Usage information and examples are provided.
#DEFINEThe #DEFINE pre-processor directive defines variables in the SVRF rule file. The variables are used to control conditional execution of portions of the rule file.
#UNDEFINEThe #UNDEFINE pre-processor directive undefines variables in the SVRF rule file. The variables are used to control conditional execution of portions of the rule file.
#IFDEF/#IFNDEFThe #IFDEF (if defined) and #IFNDEF (if not defined) pre-processor directives provide conditional compilation of the SVRF rule file based on variable definitions.
#IFDEF CALIBRE_VERSIONThe #IFDEF CALIBRE_VERSION pre-processor directive provides conditional compilation based upon the Calibre version, allowing for easier integration of new functionality into existing SVRF files.
#PRAGMA ENVThe #PRAGMA ENV pre-processor directive defines the default value for an environment variable used in SVRF statements.
