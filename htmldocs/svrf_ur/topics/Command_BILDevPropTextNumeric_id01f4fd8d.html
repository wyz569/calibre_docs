text_numeric

Device property computation language function

Returns numeric values of text objects in the layout that intersect the device seed shape.

Usage

TEXT_NUMeric(layer, default)

Arguments

layer

A required name or number of an original layer or layer set, which is specified in a TEXT PROPERTY LAYER parameter in a Device operation.

default

A required non-negative floating-point value that is assigned by the function if no other value can be assigned.

Return Values

When successful, the text_numeric function returns a floating-point numeric value that may be assigned to numeric-valued variables or properties. The conversion process is described in “Conversion From Text to Numeric Floating-Point Value.”

The following table lists the possible return values.

Table 1. Return Values of TEXT_NUMeric Function

Outcome

Return Value

Found one text object on layer that intersects the seed polygon.

Floating point value.

More than one text object on layer intersects the seed polygon.

The device is rejected and classified as BAD.

No text objects on layer intersect the seed polygon.

The default value.

A text object exists on layer intersecting the seed polygon of the device, but the text object is empty or zero length.

The default value.

A text object exists on layer intersecting the seed polygon of the device, but it cannot be evaluated as a valid, floating-point, numeric value.

The default value.

Description

The device recognition module looks for text objects on layer such that the text location intersects the seed polygon of the device. If a text object is found, then its numeric value is returned by the text_numeric function. If more than one text object on layer intersects the seed polygon, then the device is rejected and classified as BAD.

Conversion From Text to Numeric Floating-Point Value

This section is adapted from the strtod(3D) man page.

When a text object exists on layer intersecting the seed polygon of the device, the text_numeric function converts the initial portion of the text to a numeric floating-point value as follows.

Initially, it decomposes the text into three parts:

  • An initial sequence of whitespace characters, possibly empty

  • A subject sequence interpreted as a floating-point value

  • A final sequence of one or more unrecognized characters

Next, it attempts to convert the subject sequence into a floating-point value and returns the result.

The expected form of the subject sequence is an optional plus or minus sign (+ or - ), then a non-empty sequence of digits optionally containing the radix character (which is restricted to the period character), then an optional exponent part. An exponent part consists of the character “e” or “E”, followed by an optional sign, followed by one or more decimal digits.

The subject sequence is defined as the longest initial sub-sequence of the input text, starting with the first non-whitespace character, that is of the expected form. The subject sequence is empty if the text is empty or consists entirely of whitespace characters, or if the first character that is not whitespace is other than a sign, a digit, or the radix character.

If the subject sequence has the expected form, then the sequence starting with the first digit or the radix character (whichever occurs first) is interpreted as a numeric, floating-point value, except that if neither an exponent part nor a radix character appears, then a radix character is assumed to follow the last digit in the sequence of characters. If the subject sequence begins with a minus sign (-), the value resulting from the conversion is negative.

Examples

Here is a complete example in the context of a Device operation:

DEVICE MP Pseed poly(G) PSD(S) PSD(D)
TEXT PROPERTY LAYER tpl1 tpl2
[
    PROPERTY NumProp1, NumProp2, AreaSeed
    NumProp1 = text_num( tpl1, 0.0 )
    NumProp2 = text_num( tpl2, 1.1 )
    AreaSeed = area( Pseed )
]

Each device instance recognized by the preceding device operation has a property named AreaSeed with a numeric value representing the area of the device seed. Each device instance also has a numeric-valued property named NumProp1 with either the value 0.0 in cases where no valid numerically-valued text objects on layer tpl1 intersect the device seed, or the numeric value of a text object on layer tpl1. Similarly, each device instance also has a numeric-valued property named NumProp2 with either the value 1.1 or the numeric value of a text object on layer tpl2, which intersects the seed.
