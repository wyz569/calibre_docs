PEX Reduce Parallel MOS

Parasitic extraction

Controls whether or not mosfets that are wired in parallel, and are the same model type and subtype, are reduced to a single mosfet in the generated netlist. Used only in Calibre xRC and Calibre xACT 3D PDB flow.

Usage

PEX REDUCE PARALLEL MOS [ALL | mos_type [mos_type …]] {NO | YES}
[SAMEWL] [R threshold]

Arguments

ALL

An optional keyword that specifies to combine all parallel mosfets and user-defined mosfet devices during the formatting stage. This is the default behavior if you do not specify mosfet device type by name.

mos_type [mos_type….]

An optional argument that specifies one or more parallel mosfet or user-defined mosfet device type names to combine.

NO

A keyword that instructs the tool not to combine parallel mosfets and user-defined mosfet devices during the formatting stage. Note that this keyword disables the merging of all parallel mosfets and ignores any other specified parameters. This is the default behavior if you do not include this statement in your rule file.

YES

A keyword that instructs the tool to combine parallel mosfets and user-defined mosfet devices during the formatting stage.

SAMEWL

An optional keyword that specifies to merge only parallel devices of the same width and length.

R threshold

An optional keyword set that specifies the resistance threshold between parallel device pins. The threshold value is a positive floating point value that specifies the maximum resistance value between mosfet device pins.

Description

Specifies whether or not mosfets that are wired in parallel and are the same model type and subtype are reduced to a single mosfet with an “M” property in the generated netlist. This reduction of parallel devices affects LAYOUTNAMES and SOURCENAMES netlists. SOURCEBASED and SCHEMATICONLY netlists are not affected by this statement.

Two parallel mosfet devices are merged if the resistance between all of the source, drain, gate, and bulk parallel device pins is less than or equal to the specified threshold value. If the resistance between any of the parallel device pins is greater than the threshold, the two parallel devices are not merged in the output netlist.

Mosfet device type names, mos_type, come from the Device statement definition of the device. If a netlist model is defined for the device, mos_type is the netlist model name. If a netlist model is not defined, mos_type is the device model_name. If one or more mosfet devices are specified by name, then only the parallel devices of those types are reduced to a single device; parallel mosfet devices other than those specified are not merged.

If the resistance threshold is not specified, then all parallel mosfets and user-defined devices are merged regardless of the resistance value between the device pins.

Examples

Example 1

To instruct the formatter to merge parallel mosfet devices in the output netlist when the resistance value between the parallel device pins is less than or equal to 0.01, include the following statement in your rule file:

PEX REDUCE PARALLEL MOS YES R 0.01

Example 2

Given the following Device statement which specifies a netlist model:

DEVICE MP(P) pgate poly(G) tpdiff(S) tpdiff(D) nxwell(B) <pthin> \
	<OD> <pgate_aux> netlist model pch

To merge only the parallel mosfet devices for this device type in the output netlist, include the following in your rule file:

PEX REDUCE PARALLEL MOS pch YES

Example 3

Given the following Device statement which does not specify a netlist model:

DEVICE MP(P) pgate poly(G) tpdiff(S) tpdiff(D) nxwell(B) <pthin> \
	<OD> <pgate_aux>

To merge only the parallel mosfet devices for this device type in the output netlist, include the following in your rule file:

PEX REDUCE PARALLEL MOS P YES

Example 4

To instruct the formatter to only merge parallel mosfet devices pch and nch in the output netlist, include the following statement in your rule file:

PEX REDUCE PARALLEL MOS pch nch YES

Example 5

To instruct the formatter to only merge parallel mosfet devices pch and nch that are the same width and length in the output netlist, include the following statement in your rule file:

PEX REDUCE PARALLEL MOS pch nch YES SAMEWL

Example 6

This statement disables the merging of all parallel mosfet devices, not just nch:

PEX REDUCE PARALLEL MOS nch NO SAMEWL

The parameters SAMEWL and nch are ignored.
