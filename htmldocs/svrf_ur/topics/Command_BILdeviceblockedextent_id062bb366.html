device::blocked_extent

Device TVF layer generation function. Used only in a compile-time TVF rule file.

Returns a derived polygon layer representing the extent of the input layout design. This extent is ignored in the context of a specified set of parameterized device cells (pcells).

This function is only used in parasitic extraction flows that use pre-characterized device models for cells that require a specific layer (typically substrate) not to be present. This function is always called in the context of a tvf::SETLAYER command and is part of the CalibreDFM_Device Tcl package. This function is incompatible with the LVS Box GRAY keyword.

Usage

device::blocked_extent {{-block_cells ‘{’cell_list‘}’} | {-block_cells_list layout_cell_list}} [‑size_by value]

Arguments

-block_cells cell_list

An option specifying a Tcl list of at least one pcell name. The * wildcard may be used, which matches zero or more characters. May not be specified with ‑block_cells_list.

-block_cells_list layout_cell_list

An option specifying a Layout Cell List name. The referenced cell list contains pcell names. May not be specified with ‑block_cells.

‑size_by value

An option that causes cell extents to be expanded by the value. The value is a positive floating-point number in user units of length. This option behaves like the Size layer operation.

Return Values

Derived polygon layer.

Description

Creates layers that are normally generated using the Extent operation, but it is desired to shield certain primitive device cells from that Extent derived layer. Either -block_cells or ‑block_cells_list must be specified.

Layers like p-substrate are frequently defined like this:

bulk = EXTENT
psub = bulk NOT NWELL

In certain parasitic extraction flows, this can create a problem for parameterized device cells that implement devices lacking a psub terminal, for example. Since the Extent layer is defined to exist everywhere, it has no way for the user to exclude it from certain cells. device::blocked_extent provides an extent layer that blocks out specified pcells.

For example, to derive a layer like psub so it is not present in the cell Dev2T, change the psub definition from this:

tvf::setlayer psub = BULK NOT NWELL

to this:

package require CalibreDFM_DEVICE

tvf::setlayer extent_blocked = [device::blocked_extent \
              -block_cells { Dev2T } ]
tvf::setlayer psub = extent_blocked NOT NWELL

This could also be done using -block_cells_list together with Layout Cell List.

Note:

Only small parameterized cells should be specified. Using this function for larger cells or blocks results in severe performance degradation.

Unblocked cells that overlap blocked cells (they mutually share area from a flat viewpoint) are treated as blocked. For example, assume that Dev2T is specified as a blocked cell but Dev3T is not, and assume they overlap. In this case, Dev3T is promoted above Dev2T in the Calibre database, and Dev3T is processed as blocked.

Using a layer generated by device::blocked_extent requires a Calibre nmLVS Advanced license.

Examples

The following creates a blocked cell extent substrate layer called psub. Blocked extents of cells in the pcell_list are expanded by 0.002 user units.

#! tvf    
package require CalibreDFM_DEVICE

tvf::setlayer blocked = [device::blocked_extent \
  -block_cells_list pcell_list -size_by 0.002]

tvf::verbatim {
  LAYOUT CELL LIST pcell_list "p_*"

  psub = blocked NOT NWELL
}
