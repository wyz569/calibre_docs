DFM Text

Layer operation

License: Calibre nmDRC-H

Converts text objects on original layers to DFM properties.

Usage

DFM TEXT layer [PRIMARY ONLY]
PROPERTY { STRING | {[INVALID] NUMBER}}
property_name

Arguments

layer

A required original layer name.

PRIMARY ONLY

An optional keyword that specifies to use only text from the top-level cell. This keyword set must appear after layer and before PROPERTY.

PROPERTY { STRING | {[INVALID] NUMBER} }

A required argument set that indicates the type of DFM property that is created from the text objects.

  • STRING — Specifies string-type properties are created.

  • INVALID — An optional keyword that precedes NUMBER, which specifies a layer containing non-numeric property values is output. The output polygons are at locations where the text object’s value could not be converted to a number.

  • NUMBER — Specifies numeric-type properties are created. If this conversion fails, a warning is printed in the transcript, and no geometry is created.

property_name

A required argument that specifies the (case-sensitive) name of the DFM property that is created. This argument must appear last in the syntax.

Description

Converts text objects to DFM properties. Flat Calibre nmLVS runs are not supported with DFM Text; use the ‑hier or ‑flatten command line option.

This operation can only be used to generate a derived, unmerged, polygon layer. The DFM Property Merge operation is used to merge the unmerged layer. DFM RDB can write unmerged layers, and DFM Copy can copy these layers. When merging layers created by the DFM Text command, you must specify how to treat the overlapping text markers—refer to Example 1.

There are no limitations on how close together two text objects can be in the layout; consequently, the resulting marker polygons can overlap.

Examples

Example 1

This example shows how to use DFM Text to create a string-type property, then access the property using the SPROPERTY() function.

LAYER txt 1
txt_prop = DFM TEXT txt PROPERTY STRING s
merged = DFM PROPERTY MERGE txt_prop [ s = SPROPERTY(txt_prop,s,1) ]

These derivations show how to detect locations where multiple DFM Text marker shapes overlap. The merged_unique layer contains shapes where there were no overlaps on the txt_prop layer. The conflicts layer contains shapes where there were overlaps.

merged_unique = DFM PROPERTY MERGE txt_prop 
                  [s = SPROPERTY(txt_prop,s,1)]
                  [- = COUNT(txt_prop)] == 1
conflicts = DFM PROPERTY MERGE txt_prop 
                  [- = COUNT(txt_prop)] > 1

Example 2

This example shows how to collect all properties on the set of overlapping markers in a vector property.

text_prop_string = DFM TEXT txt1 PROPERTY STRING s
text_prop_number = DFM TEXT txt2 PROPERTY NUMBER s
merged_string = DFM PROPERTY MERGE text_prop_string
                  [text = VSTRING(SPROPERTY(text_prop_string,s))]
merged_number = DFM PROPERTY MERGE text_prop_number
                  [text = VECTOR(PROPERTY(text_prop_number,s))]
