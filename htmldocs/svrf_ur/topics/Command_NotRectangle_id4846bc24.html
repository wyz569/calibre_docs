Not Rectangle

Layer operation

Selects polygons that are not rectangles, or are rectangles that do not meet specified criteria.

Usage

NOT RECTANGLE layer [constraint1 [BY constraint2] [MEASURE EXTENTS]]
[ASPECT constraint3] [ORTHOGONAL ONLY [FIXED]]

Arguments

layer

A required argument that specifies an original layer or layer set, or a derived polygon layer.

constraint1 [BY constraint2] [MEASURE EXTENTS]

An optional argument that specifies conditions for the edge lengths of the rectangles. A rectangle is selected if constraint1 is not satisfied by any edge. (Non-rectangular polygons are always selected, unless MEASURE EXTENTS is specified.)

The form of constraint1 and constraint2 is listed in the “Constraint Notation” column of Table 1 in the “Constraints” topic. A constraint must contain non-negative real numbers and is interpreted in user units.

  • BY constraint2 — Specifies that a rectangle is selected if its edges do not meet either constraint1 or constraint2. In other words, if one pair of edges satisfies constraint1 and the other pair of edges satisfies constraint2 the rectangle is not selected.

  • MEASURE EXTENTS — Specifies to measure the individual rectangular extents of the polygons on layer. The extent measurement is used when applying the constraint(s). May not be specified with ORTHOGONAL ONLY.

    See “Example 3” for an example that finds polygons of any shape that do not fit within a rectangular extent of a specified size.

ASPECT constraint3

An optional keyword and constraint that specifies a condition for the ratio of the longer side to the shorter side of a rectangle. A rectangle is selected if it does not meet the constraint for the aspect ratio. The following figure shows the aspect concept:

The following operation selects all rectangles that do not have an aspect ratio of 1, in addition to selecting polygons that are not rectangles. 

NOT RECTANGLE layer1 ASPECT == 1
ORTHOGONAL ONLY [FIXED]

An optional keyword that specifies to select rectangles that are not orthogonal to the database axes. (Non-rectangular polygons are always selected, unless MEASURE EXTENTS is specified.)

If a constraint is specified, orthogonal rectangles that do not meet the constraint(s) are also selected.

  • FIXED — Specifies that the first constraint (constraint1) is measured along the x-axis and the second constraint (constraint2) is measured along the y-axis. Both constraints must be specified if FIXED is used.

ORTHOGONAL ONLY may not be specified with MEASURE EXTENTS.

Description

Selects all non-rectangular polygons on layer by default. This is the complement of the Rectangle operation. The operation can also select layer rectangles with edge lengths that do not meet the given constraints. The MEASURE EXTENTS keyword can be used to select polygons of any shape that do not fit within a specified rectangular extent.

Not Rectangle selects polygons that do not satisfy a Rectangle operation with the same arguments. If optional arguments are specified, they specify conditions and behavior for the complementary Rectangle operation. Polygons on layer that do not satisfy the conditions as applied to the complementary Rectangle operation are selected.

In the event that non-orthogonal (with respect to the coordinate axes) rectangles exist on layer, you should be careful when specifying constraints in the Not Rectangle operation. The problem arises when you make the assumption about the exact size of rectangles. For example, avoid using constraints such as “== 5” when layer contains non-orthogonal rectangles. A better choice of constraint might be “> 4.995 < 5.005”. This allows for a tolerance of ten database units, if your Precision is 1000.

Examples

Example 1

Basic contact check.

// Check that all contacts are rectangular with exact 
// dimensions of 0.3 user units by 0.4 units.

contact_size_error { not rectangle contact == 0.3 by == 0.4 }

Example 2

Using ORTHOGONAL ONLY.

// Contacts must be 0.27 by 0.27 user units with edges 
// parallel to the coordinate axes.

contact_rule { 
    not rectangle contact == .27 by == .27 orthogonal only 
}

Example 3

Using MEASURE EXTENTS.

// Find all vias having extents that do not fit inside a 
// 0.1 by 0.1 box. The vias can be of any shape. 

bad_via { 
   not rectangle via <= 0.1 by <= 0.1 measure extents 
}

Example 4

Finding non-rectangular polygons.

// Generate a derived layer containing all contacts that are 
// not rectangular.

not_rectangular_contacts = not rectangle contact

Example 5

Using ASPECT.

/* Select contact rectangles that are not square. */

rule { NOT RECTANGLE contact ASPECT == 1 }

Example 6

Figure 2 shows three Not Rectangle operations. The operation on the top right selects layer1 polygons that are not rectangles. The operation on the bottom left selects layer1 polygons that are not rectangles or are rectangles and do not have a side greater than two user units. The operation on the bottom right selects layer1 polygons that are not rectangles or that are rectangles and do not have both sides greater than two user units.

Figure 2. Not Rectangle

Example 7

Using ORTHOGONAL ONLY FIXED.

/* Select contact polygons that are not rectangles and not orthogonal to 
  the database axes. Also select rectangles whose dimension along the
  x-axis is greater than 0.1 user unit or whose dimension along the 
  y-axis is greater than 0.25 user units. */

rule {NOT RECTANGLE contact <= 0.1 BY <= 0.25 ORTHOGONAL ONLY FIXED}
