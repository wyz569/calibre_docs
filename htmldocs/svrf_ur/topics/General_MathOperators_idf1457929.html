Math Operators

Built-in math operators are available for use in DFM expressions. These are operators familiar in other programming contexts.

DFM Expressions permit the following types of math operators:

  • Unary Operators — Take one argument.

  • Binary Operators — Take two arguments.

  • Ternary Conditional Operator — Takes three arguments and specifies an if/then, or conditional, expression. This operator is not allowed in all DFM expressions. See “Conditional Expressions”.

Unary Operators

The unary operators are of equal precedence and all are of higher precedence than the binary operators. These operators are permitted in conditional and unconditional expressions.

Table 1. Unary Operators

Operator

Definition

+

indicates a positive number

-

indicates a negative number

!

returns 1 (true) if its argument is 0; returns 0 (false) otherwise

~

returns 1 (true) if its argument is non-positive; returns 0 (false) otherwise

For example, in a DFM Analyze operation, the expression !AREA(poly) returns the following values:

  • 1 — If there is no poly in the data capture window.

  • 0 — If there is poly present in the capture window.

Binary Operators

Binary operators take two numeric input arguments and produce a numeric value.

Without any grouping by parentheses, operators are executed in a predefined order according to precedence. Precedence is from 1 to 4, with 1 being highest, or executed first, and 4 being last. Parentheses in an expression can override the usual precedence in the same way as occurs in mathematics.

Operators can appear in the context of either a conditional or unconditional expression. A conditional expression involves evaluating an if-then structure; see “Conditional Expressions”.

Ternary Conditional Operator (Conditional Expression)

The following is an example of the ternary conditional operator: 

(x>0) ? 1: 0

This evaluates to 1 if x is greater than 0; it evaluates to 0 otherwise. This operator has lower precedence than the unary and binary operators. This operator is not allowed in all DFM expressions. See “Conditional Expressions”.
