Sample Device Annotation Program

The following is a basic example of a device annotation program using Device Layer ANNOTATE. Only one annotation program is allowed per Device Layer operation. Annotation specifications cannot be shared by multiple Device Layer operations.

Suppose you want to annotate a property value from a property named P1 to device seed shapes classified by the following DEVice operation:

DEVICE mp(p) pgate1 pgate1(g) sd(s) sd(d) bulk1(b) <dfm_1> <dfm_2>

Using the built-in language, the Device Layer operation could look like this:

   1   anno_layer = DEVICE LAYER mp(p) ANNOTATE dfm_1 dfm_2
   2   [
   3       SELECT dfm_1, dfm_2
   4       PROPERTY P1, P2
   5       P1 = DFM_NUM_VAL( dfm_1, "P1" )
   6       P2 = DFM_NUM_VAL( dfm_2, "P2" )
   7   ]

This example illustrates many of the features of the language. The line numbers on the left are not part of the language. The language is case-insensitive.

Lines 1 through 7 contain the complete Device Layer operation with an annotation program. The dfm_N parameters are layers on which DFM properties are found and intersect the device seed shapes.

Line 3 contains the optional SELECT statement that declares layers that must be present in addition to ANNOTATE layers in order for a device seed shape to be selected by this Device Layer operation. The SELECT layers must be seed, pin, or auxiliary layers. In this case, they are auxiliary layers. The SELECT function acts as a filter on the device seed shapes returned by the Device Layer operation.

Line 4 contains a required PROPERTY function. This function defines which properties are annotated to the selected devices.

Line 5 assigns the value of property P1 from dfm_1 to the device shapes. The DFM_NUM_VAL function selects the numeric value of DFM property P1 from dfm_1 and assigns it to property P1, which is then annotated to all selected device shapes as a DFM property.

Line 6 assigns the value of property P2 similarly to property P1 in line 5. In this case, dfm_2 is used as the annotation layer.

The LVS Annotate Devices specification statement is also required to attach annotation properties to extracted device shapes when Device Layer ANNOTATE is used.

The annotation program can also appear in a Device statement device_annotation_program block.
