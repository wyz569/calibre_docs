tvf::OUTLAYER

Application: runtime and compile-time.

Specifies a layer operation in compile-time and runtime TVF and outputs the results. The output method depends on whether the command is called in runtime or compile-time TVF.

Usage

tvf::OUTLAYER layer_op

Arguments

layer_op

A required argument that forms a valid SVRF layer operation.

The OUTLAYER command arguments may be enclosed in double quotes or braces. Both of these grouping elements may appear in the same group of arguments. Quotation marks (“ ”) allow for variable substitution in a list of arguments, whereas braces ({ }) pass their contents as a list with no variable substitution.

Return Values

None.

Description

OUTLAYER provides a means of doing layer operations in both runtime and compile-time TVF.

Compile-time operation

Sends the results of layer_op to the DRC results database. OUTLAYER is used within the tvf::RULECHECK command. See “Rule Checks in Compile-Time TVF” for several examples.

Runtime operation

Executes layer_op and returns the layer to the TVF operation that called the TVF Function. The corresponding TVF Function statement must have exactly one tvf::OUTLAYER invocation, and the output layer type must match the output type specified for the TVF Function. The default output type for TVF Function is a derived polygon layer, See “Simple Example TVF Function and TVF Layer Operation Call”.

See “Layer Operations in Runtime TVF” for restrictions on the layer operations and syntax in runtime operation.
