PEX Report Opens

Parasitic extraction

Generates a report file containing the location of any open circuits or weak connections in the design. Used only in the Calibre xACT digital flow.

Usage

PEX REPORT OPENS filename
[WEAKCONNECTIONS {NO | YES}]
[CREATESHORTS {NO | YES}]

Arguments

filename

A required filename for the report.

The filename parameter can contain environment variables. For information regarding the use of environment variables in the filename parameter, refer to “Environment Variables in Pathname Parameters”.

If the filename ends in either the .Z or .gz suffix, then the file is automatically compressed using the compress or gzip commands, respectively. This assumes these commands are available in your $PATH.

WEAKCONNECTIONS {NO | YES}

An optional keyword set that specifies whether or not to detect and report cases of through connections in the report. The default is NO if this keyword set is not specified.

CREATESHORTS {NO | YES}

An optional keyword set that specifies whether or not to short detected opens in the extracted netlist. The CREATESHORTS NO setting does not modify the extracted netlist. The default is NO if this keyword set is not specified.

Description

An optional statement that creates a report file containing the location of any opens in the design. This statement also detects and reports segments of nets that are completely disjoint from the remainder of the net but were defined as connected in the design. For example: 

Detected unconnected segment on net ’openb’ on layer ’M8’ @ (250 1200)

Use the WEAKCONNECTIONS YES keyword set to detect cases of through connections and output them in the report. A through connection is made connecting to two different points on a cell port. Routing tools may consider all points on a port as being electrically equivalent, but using through connections this way may significantly affect parasitic resistance and capacitance, depending on the properties of the layer on which the port is located.

Use the CREATESHORTS YES keyword set to report opens and deliberately short these connections in the extracted netlist. The generated report lists any unconnected nets or opens that were detected and the shorts that are inserted in the extracted netlist.

Pins disconnected in the DEF files are also included in the generated netlist when CREATESHORTS YES keyword set is specified. If this keyword set is not specified, then the disconnected pins are not included in the extracted netlist.

The CREATESHORTS YES keyword set is the only option that modifies the extracted netlist; by default, PEX Report Opens does not modify the extracted netlist.

Examples

Example 1

To report opens detected in the Calibre xACT digital flow, include the following statement in your rule file:

PEX REPORT OPENS myOpens.report

The following is an example of output generated in the report by this statement:

...
Detected unconnected segment on net 'openb' on layer 'M1' @ (250 1200)
Detected unconnected segment on net 'openb' on layer 'M1' @ (500 1200)
Detected unconnected segment on net 'openb' on layer 'M1' @ (750 1200)
Detected open on net 'openb' for pin 'pin_1br:A'
Detected open on net 'N0' for pin 'B0:P'
Detected open on net 'N6' for pin 'B0:P'
Detected open on net 'N15' for pin 'B0:P'
Detected open on net 'N22' for pin 'B0:P'
...

Example 2

To report weak or through connections detected in the Calibre xACT digital flow, include the following statement in your rule file:

PEX REPORT OPENS myWeakConns.report WEAKCONNECTIONS YES

The following is an example of output generated in the report by this statement:

...
Detected weak connection on net 'instr\[4\]' for port ’instr\[4\]'
Detected weak connection on net 'N10' for pin 'OUT:1' 
...

Example 3

To report opens and to short these connections in the extracted netlist, include the following statement in your rule file:

PEX REPORT OPENS myCreateShorts.report CREATESHORTS YES

The following is an example of output generated in the report by this statement:

...
Detected open on net 'openb' for pin 'pin_1br:A' 
Created short on net 'openb' between pin 'pin_1bl:A and pin 'pin_1br:A'
...
