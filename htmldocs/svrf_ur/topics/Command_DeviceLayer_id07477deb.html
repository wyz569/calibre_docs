Device Layer

Layer operation

Generates a layer that contains seed shapes from all device instances with the specified type and model name. Seed shapes are those from device_layer arguments in Device operations. Used only in Calibre nmLVS/Calibre nmLVS-H, Calibre YieldAnalyzer, and ICtrace.

Usage

DEVICE LAYER {{element_name [‘(’model_name‘)’]} | {TEMPLATE template_name}}
[ANNOTATE layer [layer …] [‘[’device_annotation_program‘]’]]
[EXCLUDE UNUSED] [PROPERTY]

Arguments

element_name

A built-in device component type. Certain predefined element names specify built-in (or hard-coded) devices. See Table 1 with the Device statement. All other types are user-defined (generic) devices. Either this parameter or the TEMPLATE keyword must be specified.

(model_name)

An optional string, enclosed in parentheses, corresponding to the device model. It is also known as the subtype of the device. Specified with element_name.

TEMPLATE template_name

A keyword set where the template_name is a string that must conform to these criteria:

  • If a Device statement uses a TEMPLATE keyword set, then that template_name may be used.

  • If a Device statement has a unique element_name(model_name) specification (the model_name is optional), and that Device statement does not use the TEMPLATE keyword set, then that element_name(model_name) may be used as the template_name.

Otherwise, TEMPLATE may not be specified. Either TEMPLATE or element_name must be specified by the operation.

ANNOTATE layer

An optional keyword and layer argument that instruct the tool to output a layer that is annotated with DFM properties present on the specified layer. More than one annotation layer parameter may be specified. The layer must be the output of a DFM Property operation.

ANNOTATE is useful for attaching complex properties to devices without affecting LVS comparison. The output layer from the operation is then specified in an LVS Annotate Devices specification statement.

‘[’device_annotation_program‘]’

An optional user-defined program that specifies how to annotate devices on the ANNOTATE layer. The program appears between literal brackets. The program itself may be blank, as may occur in the processing of conditional (#IFDEF) statements. See “Device Annotation Built-In Language” for complete details. Properties calculated with this program are not extracted for use during LVS comparison. Such properties may be extracted in the Calibre Connectivity Interface (CCI).

EXCLUDE UNUSED

An optional keyword that instructs the tool to exclude unused devices from the result layer according to the specified unused device filters. Unused devices are specified with LVS Filter Unused Option or equivalent LVS Filter Unused statements. All options for LVS Filter Unused Option are supported except the INV, P, and Q options.

PROPERTY

An optional keyword that specifies to annotate device properties as DFM properties to the result output. For information about DFM properties, see “DFM Properties”.

Description

Creates a layer containing seed shapes from all device instances with the specified element_name and optional model_name, or with the specified template_name. Seed shapes are those from the corresponding Device operations.

If element_name is specified without model_name, then all devices with the specified element_name (with any model name or no model name) are processed.

If more than one Device operation matches a Device Layer operation’s parameters, then the results are merged together by an OR operation.

The TEMPLATE keyword set is used to differentiate between the seed shapes identified by Device statements having the same element_name and optional model_name. See Example 5.

The Device Layer operation does not preserve node numbers from input shapes. The result layer of a Device Layer operation cannot be used in Connect, Sconnect, or Device operations, and cannot be used to derive layers for those operations. The Device Layer operation is often used as part of ERC.

In Calibre nmDRC/Calibre nmDRC-H and ICrules, the Device Layer operation produces an empty result layer and generates a warning.

ANNOTATE Keyword

The ANNOTATE keyword is useful for attaching properties computed using the DFM Property operation to output layers. If more than one annotation layer is specified, then properties can be read from each layer.

ANNOTATE is used in conjunction with the LVS Annotate Devices specification statement for device property annotation. The output layer generated using the ANNOTATE keyword may be as an auxiliary_annotation_layer specified in a Device statement.

The ANNOTATE keyword supports the optional device_annotation_program. This is a user-defined program that defines how annotations are made. This capability is useful for adding property measurements like stress effects that have the potential to disrupt the hierarchy of more exotic devices. See “Sample Device Annotation Program” for a discussion of these programs.

“Example Device Annotation Programs Using DFM Property” contains complete examples.

PROPERTY Keyword

Without the PROPERTY keyword, a Device Layer operation can refer to multiple Device elements and the result layer contains seed polygons of all matching devices. When the PROPERTY keyword is used, the Device Layer operation must refer to exactly one Device element, otherwise a compiler error is output. Both string and numeric properties are output as DFM properties when this keyword is specified.

An element_name may be specified without a model_name if all models of the device generate identical properties in the same order. When it is used this way, the output layer contains shapes that represent all of the recognized devices regardless of model name. All output shapes are annotated with device properties.

In the following example, the Device Layer operation is used to find all MP devices connected to VCC through the PSD pin layer. The seed polygons of these devices are written to a DFM RDB together with device properties.

DEVICE MP PGATE PGATE PSD PSD PWELL 
  [ PROPERTY L, W, AS, AD 
    W = PERIMETER_COINCIDE(PGATE, PSD)/2
    L = AREA(PGATE)/W
    AS = AREA(S)
    AD = AREA(D)
  ]  

MP_VCC = ( DEVICE LAYER MP PROPERTY ) INTERACT ( NET PSD "VCC" ) 
ERC SELECT CHECK MP_CHECK
MP_CHECK {
  DFM RDB MP_VCC "mp_vcc.rdb" ALL CELLS CHECKNAME "MP - Properties"
}

Examples

Example 1

The following example finds all MP devices connected to VCC through the PSD pin layer. 

ECONN50 { ( DEVICE LAYER MP ) INTERACT ( NET PSD "VCC" ) }
ERC SELECT CHECK ECONN50

Using Not Interact would find MP devices disconnected from PSD polygons on net VCC.

Example 2

The following example finds all nets that do not connect to MP devices through the PSD pin layer. Outputs all PSD shapes on those nets. 

ELDISC50 { 
  NET AREA RATIO PSD ( PSD INTERACT ( DEVICE LAYER MP ) ) == 0
}
ERC SELECT CHECK ELDISC50

Example 3

The following example finds all nets that connect to MP devices through the PSD pin layer. Outputs all PSD shapes on those nets. 

ELCONN50 { 
  NET AREA RATIO PSD ( PSD INTERACT ( DEVICE LAYER MP ) ) > 0
}
ERC SELECT CHECK ELCONN50

Example 4

See “Example Device Annotation Programs Using DFM Property” for an example showing the ANNOTATE keyword.

Example 5

The following example causes only the MN(n1) seed shapes identified by the second Device statement to appear in the aux_1_seeds layer:

# both of these extract MN(n1) devices
DEVICE MN(n1) ngate ngate sd sd bulk
DEVICE MN(n1) ngate ngate sd sd bulk <aux_1> TEMPLATE mn_aux_1

# get only the MN(n1) device shapes associated with the aux_1 layer
aux_1_seeds = DEVICE LAYER TEMPLATE mn_aux_1
