PEX Report Point2Point

Parasitic extraction

Generates a report for point-to-point resistance on specified nets and writes it to a specified output file, or to the screen. Extraction is required for generating the report. This statement can be specified multiple times.

Usage

PEX REPORT POINT2POINT [UNIT_LENGTH | DBU] in_file [out_file] [LAYOUTNAMES | SOURCENAMES [CALIBREVIEW]]
[SAMEPORT {OPEN | SHORT}]

Arguments

UNIT_LENGTH

An optional keyword that sets the units for COORD location parameter in in_file to a user-specified unit. UNIT_LENGTH uses the value set by the Unit Length SVRF statement. If Unit Length is not specified, microns is the default.

DBU

An optional keyword that sets the units for COORD location parameter in in_file to the layout database units.

in_file

A required parameter that specifies the name of a file containing entries in the following format for a point-to-point resistance calculation: 

RESISTANCE net_name location net_name location

where:

  • RESISTANCE is a required keyword

  • net_name is the name of a net

  • location can take one of the following forms: 

PIN device_name pin_name

Name of a device pin

 
PORT | PROBE p_name

Name of port or probe

 
COORD x y layer

Coordinates and layer name to be used to find the closest node on the desired net. The coordinates are in database units (DBU) or user specified (UNIT_LENGTH).

 
ALL

Keyword that specifies to include resistance measurements from the specified location to all pins, ports, and probes. This keyword can only be specified for the second location.
Note:

COORD x y layer location syntax is not supported in Calibre xACT. This also means that UNIT_LENGTH and DBU keywords will have no effect in Calibre xACT.

The file may contain comments. Comments begin with // and run to the end of the line.

Note that PIN, PORT, and PROBE names are from the layout, not from the source. Using source-based names in a PEX Report Point2Point statement is not supported.

out_file

An optional parameter naming the output file. The default is standard out. The format for out_file is the same as in_file with the addition of the calculated resistance at the end of the line. The value of the reported resistance is in ohms.

LAYOUTNAMES

An optional keyword that specifies to use the net names from the layout in the report. This is the default behavior.

SOURCENAMES [CALIBREVIEW]

An optional keyword that specifies to use the net names from the source netlist in the report.

Use the optional CALIBREVIEW keyword with SOURCENAMES to instruct the formatter to include a reference to the generated point2point report file in the extracted CalibreView netlist. This enables the display of resistance information in the Map Schematic Nets to CalibreView Nets dialog box. The CALIBREVIEW keyword can only be used if the netlist format is CalibreView.

SAMEPORT {OPEN | SHORT}

An optional keyword set that specifies whether or not to short ports with the same net name, and controls how resistance for these ports is reported.

When OPEN is specified, ports with the same net name are not shorted and the least resistive path is reported.

When SHORT is specified, ports with the same net name are shorted and the equivalent total resistance between both points is reported. This is the default behavior when this keyword set is not specified.

Description

Generates a report for point-to-point resistance on specified nets and writes it to a specified output file, or to the screen. Extraction is required for generating the report. The point-to-point calculations take temperature effect into consideration when generating the report or looking at values in Calibre RVE.

You can include multiple PEX Report Point2Point statements in the same rule file. The Calibre xRC formatter generates a report for each statement.

The SOURCENAMES option must be used if any of the specified net names are from the source netlist.

It is possible to mix ports, probes, pins, and coordinates in the same statement. That is, you can specify the starting net and location from a pin and the ending net and location from a coordinate.

When using the COORD option, the terminals of the closest extracted resistor are used. If no extracted resistor exists at one of the coordinates, then the reported value is calculated based on a fraction of an existing parasitic resistor. When this occurs, the Calibre xRC tool issues a warning statement. Figure 1 shows an example of this case. A resistance report statement for NET1 is specified as:

RESISTANCE NET1 COORD 10 10 M1 NET1 COORD 30 15 M1

where the coordinates (10, 10) and (30, 15) are in microns.

Figure 1. COORD Usage Example

COORD 1 is aligned with the start of SEGMENT1, which has an associated parasitic resistance. COORD 2 is not aligned with the start or end of SEGMENT2. PEX Report Point2Point will report the resistance from COORD 1 to COORD 2 by estimating how much of the SEGMENT2 resistance is used. In this case, COORD 2 is at about 1/3 of SEGMENT2. The warning issued by the Calibre xRC tool is as follows:

WARNING: For COORD (30, 15) an exact node was not found.
WARNING: A new node between node 3 and node 4, at 33% from node 3 will be inserted.

If the specified locations are not electrically connected, then a large resistance value, 1e38 Ohms, is reported. If the locations do not exist or there are other problems with how the locations are specified, the output from PEX Report Point2Point will contain an error message as follows:

RESISTANCE net_name location net_name location: ‘ERROR: …’

Examples

Example 1

This example shows a point-to-point resistance calculation between the drain pin of a MOSFET to the gate pin of another MOSFET. Since COORD is not specified in the input file, it is not necessary to specify DBU or UNIT_LENGTH.

// SVRF Rule File Statements
PEX REPORT POINT2POINT myInput.res myOutput.rep
// Point-to-Point Input File
// <type> <net_name> <location> <net_name> <location>
RESISTANCE top_only PIN M200 d top_only PIN M300 g

where:

  • top_only is the net name

  • PIN M200 d defines the drain pin on MOSFET M200

  • PIN M300 g defines the gate pin on MOSFET M300

The output for this statement is:

// POINT-TO-POINT REPORT
RESISTANCE top_only PIN M200 d top_only PIN M300 g 9.8822204e+00

Example 2

This example shows a point-to-point resistance calculation between a probe point and a coordinate on the M1 layer using the specified unit length. 

// SVRF Rule File Statements
UNIT LENGTH u							//microns
PEX REPORT POINT2POINT UNIT_LENGTH myInput.res myOutput.rep
// Point-to-Point Input File
// <type> <net_name> <location> <net_name> <location>
RESISTANCE NET1 PROBE probe1 NET1 COORD 511 10 M1

where:

  • NET1 is the net name

  • PROBE defines a user specified probe point on the NET1 net, probe1

  • COORD defines a location in the layout on the M1 layer for the NET1 net

The output for this statement is:

// Point-to-Point REPORT
// For the COORD option: 1 database unit = 1000 * 1 user unit
RESISTANCE NET1 PROBE probe1 NET1 COORD 511 10 M1 1.9117458e-01

Since the COORD option is specified, a statement in the output file shows how the unit length is applied.

Example 3

To report the least resistive path between two points, specify the following statements in your rule file:

PEX REPORT POINT2POINT myInput.res myOutput.rep SAMEPORT OPEN

Example 4

If you have a multi-finger device, specify the following statements in your rule file to short ports with the same net name and report the total equivalent resistance between two points:

PEX REPORT POINT2POINT myInput.res myOutput.rep SAMEPORT SHORT

Example 5

This example shows how to report a point-to-point resistance calculation between the drain pin of a MOSFET and all pins, ports, and probes.

// SVRF Rule File Statements
PEX REPORT POINT2POINT myInput.res myOutput.rep
// Point-to-Point Input File
// <type> <net_name> <location1> <net_name> <location2>
RESISTANCE top_only PIN M200 d top_only ALL

where:

  • top_only is the net name

  • PIN M200 d defines the drain pin on MOSFET M200

  • ALL indicates to all pins, ports, and probes.

The output for this statement is:

// POINT-TO-POINT REPORT
RESISTANCE top_only PIN M200 d top_only PIN M200 g 9.8822204e+00
RESISTANCE top_only PIN M200 d top_only PIN M200 b 7.8822204e+00

The report includes resistance measurements from pin “M200 d” to all pins, ports, and probes.

Example 6

To include a reference to the point-to-point output report in your extracted CalibreView netlist, include the following statements in your rule file:

// SVRF Rule File Statements
PEX NETLIST "netlist.cv" CALIBREVIEW SOURCENAMES 
PEX REPORT POINT2POINT myInput.res myOutput.rep SOURCENAMES CALIBREVIEW
