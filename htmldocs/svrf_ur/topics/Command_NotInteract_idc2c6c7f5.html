Not Interact

Layer operation

Selects polygons that do not share any area, edges, or edge segments with polygons on a second layer. Optional arguments modify the selection criteria.

Usage

NOT INTERACT layer1 layer2 [constraint [BY NET] [EVEN | ODD]]
[SINGULAR ALSO | SINGULAR ONLY]

Arguments

layer1

A required original layer or layer set, or a derived polygon layer. The operation selects polygons from this layer that meet the conditions of the operation.

layer2

A required original layer or layer set, or a derived polygon layer.

constraint [BY NET] [EVEN | ODD]

An optional argument that places conditions on the number of layer2 polygons that interact with a layer1 polygon. If the number of layer2 polygons or nets that interact with a layer1 polygon does not satisfy the constraint, the layer1 polygon is selected. The SINGULAR ALSO and SINGULAR ONLY keywords affect which layer2 polygons are counted. A layer1 polygon that interacts with zero layer2 polygons is always selected.

The constraint is one of those listed in the “Constraint Notation” column of Table 1 in the “Constraints” topic. The constraint must contain non-negative integers. The constraint == 0 selects all layer1 polygons; use Interact instead.

  • BY NET — Specifies that the constraint applies to the number of distinct nets in the set of layer2 polygons that interact with a layer1 polygon. Connectivity is required on layer2.

  • EVEN | ODD — Specifies that the number of layer2 polygons or nets that interact with the layer1 polygon is even or odd, respectively. The layer1 polygon is selected if the constraint is not met. For example, if a constraint of “> 1 EVEN” is specified, a layer1 polygon that interacts with three layer2 polygons is selected, while a layer1 polygon that interacts with four layer2 polygons is not selected.

SINGULAR ALSO | SINGULAR ONLY

An optional keyword choice that indicates how to handle outside point-to-point and point-to-edge interactions (singular interactions). If neither of these keywords is used, outside point-to-point and point-to-edge intersections are not considered as interacting, and are therefore selected by Not Interact.

  • SINGULAR ALSO — Specifies that layer1 and layer2 polygons with singular interactions are also considered as interacting, along with the default definition for the Interact operation. A layer1 polygon that intersects a layer2 polygon in any manner, including singular interactions, is not selected.

    If a constraint is specified, layer2 polygons that intersect the layer1 polygon in any manner are counted. If the number of layer2 polygons does not satisfy the constraint, the layer1 polygon is selected.

  • SINGULAR ONLY — Specifies that only layer1 and layer2 polygons with at least one singular interaction are considered as interacting. A layer1 polygon that intersects a layer2 polygon in at least one singular interaction is not selected; the presence or absence of other interaction types is not considered.

    If a constraint is specified, only layer2 polygons that intersect the layer1 polygon in a singular interaction are counted. If the number of layer2 polygons does not satisfy the constraint, the layer1 polygon is selected.

See “Example 3”.

These keywords may not be specified with BY NET.

Description

Selects all layer1 polygons that do not share any area, edges, or edge segments with a layer2 polygon. This is the complement of the Interact operation. If layer2 is empty, the Not Interact operation generates output equivalent to the polygon data on layer1. The optional SINGULAR ALSO and SINGULAR ONLY keywords specify how to handle outside point-to-point and point-to-edge interactions.

Not Interact selects polygons that do not satisfy an Interact operation with the same arguments. If optional arguments are specified, the constraint and optional keywords specify conditions for the complementary Interact operation. Polygons on layer1 that do not satisfy the conditions as applied to the complementary Interact operation are selected.

See also Not Cut, Not Enclose, Not Inside, Not Touch, Outside, and Net Interact.
