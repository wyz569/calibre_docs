Encrypted Rule File Errors

This section describes errors generated when running Calibre with an encrypted SVRF rule file.
Table 1. Encrypted Rule File Errors

Error #

Explanation

DECRYPT1

Decryption system is not enabled. The version of Calibre you are using does not support encrypted rule files. Obtain the current release of Calibre.

DECRYPT2

Unable to initialize the decryption system. The decrypt tag following the #DECRYPT directive is corrupt or it is an unsupported encryption version. Obtain a new version of the encrypted rule file.

DECRYPT3

Unable to decrypt line. A line between the #DECRYPT and #ENDCRYPT directives is corrupt. Obtain a new version of the rule file.

DECRYPT4

Error on encrypted line # of filename. Calibre cannot compile the encrypted rule file. This is not a decryption error. Possible reasons for the error include duplicate layer names, duplicate rule names, incorrectly set environment variables, incorrect Calibre version, or mismatches in the number of arguments between a CMACRO and DMACRO. Contact the supplier of the encrypted rule file for support.

DECRYPT5

Error on encrypted line # of filename. The SVRF rule deck requires a specific environment variable to be set. Contact the foundry for assistance.
