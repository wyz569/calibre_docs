LVS Softchk Maximum Results

Specification statement

Specifies the number of polygons reported in the softchk results database.

Usage

LVS SOFTCHK MAXIMUM RESULTS {ALL | number}

Arguments

ALL

Keyword that specifies the maximum results count is 264. This is the default value.

number

Positive integer that specifies the maximum results count for an individual soft connection check.

Description

Specifies the maximum number of polygons reported for an LVS Softchk statement in the soft connection results database.

You can specify this statement once in your rule file.

The results counts per LVS Softchk statement and the total number of SOFTCHK module results are reported in the transcript.

When the maximum results number is exceeded for a soft connection, the tool issues this transcript warning, and no further results are added to the softchk results database:

WARNING: This SOFTCHK Operation exceeded maximum results limit: number . Operation results are incomplete.

This message appears at the end of the transcript and in the LVS report in a comparison run:

WARNING: A SOFTCHK Operation exceeded maximum results limit: number. Operation results are incomplete.

Examples

Example 1

// Send all results to the soft connection RDB for each check.
LVS SOFTCHK MAXIMUM RESULTS ALL

Example 2

// Send first 50 results to the soft connection RDB for each check.
LVS SOFTCHK MAXIMUM RESULTS 50
