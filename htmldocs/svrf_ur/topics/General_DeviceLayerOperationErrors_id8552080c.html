Device Layer Operation Errors

This section describes compilation errors from the Device Layer operation.
Table 1. Device Layer Operation Errors

Error #

Explanation

DVL1

A Device Layer operation is missing its element name parameter or has an invalid element name parameter.

DVL2

A model name within a Device Layer operation is not of the form “(” <name> “)”. The device element name appears in the message.

DVL3

A Device Layer operation has no corresponding Device definition in the rule file. The device element name appears in the message.

DVL4

More than one Device definition matches a Device Layer operation with the PROPERTY keyword. The device element name appears in the message.

DVL5

A pin is missing or is inconsistent for instances this device.

DVL6

The specified layer is not used for any devices.

DVL7

The Device Layer operation corresponding to the property being accessed in a DEVice_NUMeric_VALue function references devices that do not have the same seed layer. Either specify a model name in the Device Layer operation or use the SELECT statement in the device annotation program.

DVL8

The property being accessed in a DEVice_NUMeric_VALue function does not have the same LVS-actionable status in all devices selected by the Device Layer operation. Either specify a model name in the Device Layer operation, use a SELECT statement in the device annotation program, or ensure the property has the same LVS-actionable status for all related devices (this is often done by using parallel Trace Property configurations for the devices).

DVL11

A Device Layer ANNOTATE program uses the SELECT keyword, but no corresponding layers exist in the Device statement.

DVL12

A Device Layer ANNOTATE program references a pin, which requires the operation to correspond to a unique Device statement. But there is more than one Device statement that satisfies the program’s criteria.

DVL13

A Device Layer ANNOTATE program references a pin that does not correspond to the device.
