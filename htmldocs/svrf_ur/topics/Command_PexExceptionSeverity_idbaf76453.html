PEX Exception Severity

Parasitic extraction

Specifies an exception type and how the exception is handled when encountered during extraction through a user-selected severity value.

Usage

PEX EXCEPTION SEVERITY exception_name severity_value

Arguments

exception_name

A required case-insensitive argument that specifies the name of the exception. Supported exceptions are listed in Table 1.

severity_value

Non-negative integer that allows you to individually control the severity of selected exceptions. Severity may include complete quiet, cautionary notes, warnings, or fatal errors.

Description

Specifies an exception type and how the exception is handled when encountered during extraction through a user-selected severity_value. This statement may be specified once per exception_name. It is a compilation error for the same exception_name to appear in multiple PEX Exception Severity statements. It is also a compilation error for a severity_value to be outside the range of severities for the particular exception.

Table 1. Exceptions for PEX Exception Severity

Cells

Exception Name: XCELL_NOT_FOUND

Description: Controls the reporting of missing specified xcells.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.(default)

2: Fatal error.

Layers

Exception Name: CHECK_ALL_UNMAPPED_LAYER

Description: Checks all layers that are not mapped to a calibrated layer used in a PEX Map statement, even if the layer is not part of the design. By default, a warning is generated for all unmapped layers except gate layers. If the unmapped layer is a gate layer, then a fatal error is generated.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.

2: Fatal error.

Exception Name: CONTACT_LAYER_CONNECTIVITY

Description: Contact layer defined in a Connect statement without or before the BY keyword.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.(default)

2: Fatal error.

Exception Name: DATA_ON_LAYER_MAPPED_TO_NON_CALIBRATED

Description: A PEX Map statement is specified with an undefined calibrated layer name and a physical layer name that contains shapes but has not been unmapped with a PEX Unmap statement.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.

2: Fatal error.(default)

Exception Name: DRC_PEX_UNMAP_NON_EMPTY_LAYER

Description: Layer exists but is intentionally unmapped. This is a foundry predefined exception. Use only with Calibre xACT.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.(default)

2: Fatal error.

Exception Name: IGNORE_EXPLICIT_UNMAPPED_LAYER

Description: Layer is intentionally not mapped to a calibrated layer used in a PEX Map statement.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.(default if IGNORE_UNMAPPED_LAYER is not specified)

2: Fatal error.

Exception Name: IGNORE_UNDEFINED_LAYER

Description: Undefined layer specified in a PEX Ignore statement.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.(default)

2: Fatal error.

Exception Name: IGNORE_UNMAPPED_LAYER

Description: Layer not mapped to a calibrated layer used in a PEX Map statement. By default, a warning is generated for all unmapped layers except gate layers. If the unmapped layer is a gate layer, then a fatal error is generated.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.

2: Fatal error.

Exception Name: LVS_REQUIRED_MODEL_LAYER

Description: The LVS layer specified in a PEX Map Required statement does not appear in a valid PEX Map statement.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.

2: Fatal error. (default)

Exception Name: MAP_NOT_CALIBRATED_LAYER

Description: Undefined xCalibrate layer specified in a PEX Map statement.

Severities:

0: Quietly ignore. (default)

1: Warn and ignore the condition.

2: Fatal error.

Exception Name: MAP_UNDEFINED_LAYER

Description: Undefined physical (LVS) or device layer listed in a PEX Map statement.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.(default)

2: Fatal error.

Exception Name: PEX_DEF_MAP_CHECK

Description: Non-original layer or unmerged layer specified in a PEX DEF Map statement.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.

2: Fatal error. (default)

Exception Name: UNMAP_UNDEFINED_LAYER

Description: Undefined layer listed in a PEX Unmap statement.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.(default)

2: Fatal error.

Exception Name: VIA_MAPPED_TO_CONDUCTOR

Description: A conductor layer is mapped to a via/contact layer, or a via/contact layer is mapped to a conductor layer in a PEX Map statement. Use only with Calibre xACT.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.

2: Fatal error. (default)

Nets

Exception Name: PEX_DETECT_SHORTS

Description: Short circuit detected between any two nodes. Applies to LEF/DEF layouts only.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition.(default)

2: Fatal error.

Formatter

Exception Name: BAMAPFILE_SYNTAX_ERROR

Description: Syntax errors encountered when parsing the mapfile used by the PEX BA Mapfile statement. Use only with Calibre xRC or Calibre xACT 3D PDB flow.

Severities:

0: Quietly ignore.

1: Generate syntax error and ignore the condition. Continue formatter run. (default)

2: Fatal error and stop the formatter run.

Exception Name: NO_GROUND_NETNAME

Description: GROUND net name not specified in the PEX Netlist statement. Use only with Calibre xRC or Calibre xACT 3D PDB flow.

Severities:

0: Quietly ignore.

1: Warn and ignore the condition. Continue formatter run. (default)

2: Fatal error and stop the formatter run.

Deprecation Messages

Exception Name: ALIAS_DEPRECATION_MESSAGE

Description: Deprecated statement PEX Alias specified in rule file.

Severities:

0: Warn and ignore the condition.(default)

1: Quietly ignore.

2: Fatal error.

 

Examples

Example 1

The following statement specifies to quietly ignore the MAP_UNDEFINED_LAYER exception when an undefined layer is listed in a PEX Map statement: 

PEX EXCEPTION SEVERITY MAP_UNDEFINED_LAYER 0

Example 2

The following statement specifies to treat a MAP_UNDEFINED_LAYER exception as a fatal error: 

PEX EXCEPTION SEVERITY MAP_UNDEFINED_LAYER 2

Example 3

The following statement specifies to treat a UNMAP_UNDEFINED_LAYER exception as a fatal error: 

PEX EXCEPTION SEVERITY UNMAP_UNDEFINED_LAYER 2

Example 4

The following statement specifies to quietly ignore the ALIAS_DEPRECATION_MESSAGE exception when a PEX Alias statement is processed in the rule file: 

PEX EXCEPTION SEVERITY ALIAS_DEPRECATION_MESSAGE 1

Example 5

The following statement specifies to quietly ignore the XCELL_NOT_FOUND exceptions generated when xcells specified in the xcell file are missing from the design: 

PEX EXCEPTION SEVERITY XCELL_NOT_FOUND 0

Example 6

If a short circuit is detected between any two nodes in the LEFDEF layout a PEX_DETECT_SHORTS warning is generated. For example: 

WARNING: Layout shorts detected in cell: “cell_name” on layer “layer_name” between nets “netname1” and “netname2”. Assigning net “netname2” to net “netname1”.

Specify the following statement to treat a PEX_DETECT_SHORTS warning as a fatal error: 

PEX EXCEPTION SEVERITY PEX_DETECT_SHORTS 2

Example 7

A layer defined as a contact and a conductor layer in the same rule file generates a warning during extraction. For example:

WARNING: Contact layer ntap cannot appear as one of the connected layers on CONNECT statement. CONNECT statement ignored.

To treat this CONTACT_LAYER_CONNECTIVITY warning as a fatal error include the following statement in your rule file: 

PEX EXCEPTION SEVERITY CONTACT_LAYER_CONNECTIVITY 2

Example 8

In this example, an unmapped layer, layer_UM, has not been specified with the PEX Unmap statement, and an unmapped layer, layer_EUM, has explicitly been unmapped using the PEX Unmap statement.

...
CONNECT layer_UM
CONNECT layer_EUM
...
PEX MAP L1 L2       // No PEX MAP for layers layer_UM and layer_EUM
...
PEX UNMAP layer_EUM
...

To suppress warnings for layer layer_EUM, that has intentionally been defined as unmapped, specify the following statement:

PEX EXCEPTION SEVERITY IGNORE_EXPLICIT_UNMAPPED_LAYER 0

Since layer_EUM was explicitly unmapped, no warnings about this unmapped layer are generated during extraction. Only warnings about layer_UM being unmapped are generated.

Example 9

Suppose your mapfile definition, used by the PEX BA Mapfile statement, does not contain the required PARM section. By default, the following errors are generated by the formatter, and the netlist is still generated:

ERROR: PEX BA MAPFILE(11) - PARM section is required for ADD/DEL sub-sections.
ERROR: PEX BA MAPFILE is turned off due to errors in parsing the mapfile.

To quietly ignore the error and still produce the netlist, specify the following statement in your rule file: 

PEX EXCEPTION SEVERITY BAMAPFILE_SYNTAX_ERROR 0

To treat the syntax errors as a fatal error, specify the following statement in your rule file:

PEX EXCEPTION SEVERITY BAMAPFILE_SYNTAX_ERROR 2

In this case the formatter stops with the following errors:

ERROR: PEX BA MAPFILE(11) - PARM section is required for ADD/DEL sub-sections.
FATAL ERROR: PEX BA MAPFILE is turned off due to errors in parsing the mapfile.

Example 10

If a ground name is not specified with the GROUND keyword in your PEX Netlist statement, the following warning is generated by the formatter:

WARNING: No ground net name defined in PEX NETLIST statement and "0" will be used in the netlist.

To treat this condition as a fatal error, specify the following statement in your rule file: 

PEX EXCEPTION SEVERITY NO_GROUND_NETNAME 2

In this case, the formatter stops with the following fatal error:

FATAL ERROR: No ground net name defined in PEX NETLIST statement.

Example 11

If a conductor layer specified with a Connect statement is mapped to a contact or via layer using the PEX Map statement, then errors are generated and extraction stops. For example:

ERROR: Layer ’ptap’ is present as conductor name in a connect statement but mapped to a contact layer ’XCALIBRATE_ptap’.
FATAL ERROR: Connectivity errors encountered - exiting...

To treat this VIA_MAPPED_TO_CONDUCTOR error as a warning and continue extraction, include the following statement in your rule file: 

PEX EXCEPTION SEVERITY VIA_MAPPED_TO_CONDUCTOR 1

To ignore this condition as a fatal error and continue extraction, specify the following statement in your rule file: 

PEX EXCEPTION SEVERITY VIA_MAPPED_TO_CONDUCTOR 0

Example 12

If a MOS gate layer specified with a Device statement is not mapped to a calibrated layer using the PEX Map statement, then an error is generated and extraction stops. For example:

FATAL ERROR: Gate Layer "ngate_1" is not mapped to a calibrate layer.

To treat this condition as a warning and continue extraction, include the following statement in your rule file: 

PEX EXCEPTION SEVERITY IGNORE_UNMAPPED_LAYER 1

To ignore this condition without generating an error or warning and continue extraction, specify the following statement in your rule file: 

PEX EXCEPTION SEVERITY IGNORE_UNMAPPED_LAYER 0

Example 13

The following error occurs when the LVS layer name specified in a PEX Map Required statement has not been specified in a PEX Map statement:

ERROR: Error PEX40 - Incompatible statements: PEX MAP REQUIRED: missing mapping or mapping to undefined layer for LVS layer MET2

To treat this condition as a warning and continue extraction, include the following statement in your rule file: 

PEX EXCEPTION SEVERITY LVS_REQUIRED_MODEL_LAYER 1

To ignore this condition without generating an error or warning and continue extraction, specify the following statement in your rule file: 

PEX EXCEPTION SEVERITY LVS_REQUIRED_MODEL_LAYER 0

Example 14

The following message is generated when the layer name specified as the first parameter in a PEX Map statement is not a calibrated layer:

PEX MAP layer is not a calibrated layer.

To treat this condition as a warning and continue extraction, include the following statement in your rule file: 

PEX EXCEPTION SEVERITY MAP_NOT_CALIBRATED_LAYER 1

To treat this condition as an error and stop extraction, specify the following statement in your rule file: 

PEX EXCEPTION SEVERITY MAP_NOT_CALIBRATED_LAYER 2

Example 15

In this example, Layer B, which contains shapes, is mapped to a calibrated layer A using a PEX Map statement. If layer A does not exist in the calibrated rule files, and layer B has not been unmapped with a PEX Unmap statement, then by default extraction stops with an error. For example:

ERROR: Layer M11 contains shapes, appears in PEX MAP with undefined primary layer

To treat this condition as a warning and continue extraction, include the following statement in your rule file: 

PEX EXCEPTION SEVERITY DATA_ON_LAYER_MAPPED_TO_NON_CALIBRATED 1

To ignore this condition without generating an error or warning and continue extraction, specify the following statement in your rule file: 

PEX EXCEPTION SEVERITY DATA_ON_LAYER_MAPPED_TO_NON_CALIBRATED 0
