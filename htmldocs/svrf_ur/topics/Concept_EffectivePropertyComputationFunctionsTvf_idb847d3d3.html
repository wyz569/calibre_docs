Effective Property Computation Functions for TVF

TVF effective property programs use the foreach_instance and set_effective_property functions. These functions are used in a Tcl procedure called by TVF_NUMeric_FUNction. They control iteration over device instances and assignment of effective property values, respectively. These functions may only be used in a TVF Function block.

Integral Numeric Values in TVF Programs

Certain operators in the Tcl expr command require integral numbers. This requirement can be satisfied through the int function. For example, the following Tcl proc returns the bitwise OR of a passed in property b: 

proc bitwise_or { b } { 
  set bset 0 
  foreach_instance { 
    set bset [ expr { $bset | int([$b]) } ] 
  } 
  return $bset 
}

See the Tcl expr command documentation for additional information.

The numeric values used in the effective property computation language are double-precision floating-point numbers. When TVF is used in effective property computation, the values returned by property evaluation in the Tcl code are Tcl double-precision floating-point numbers. It is possible to use these for integral numbers as long as the integer fits within the range that can be represented precisely by a double-precision floating-point number, or approximately 253.

However, once the effective property computation program terminates, the properties are stored as floating-point numbers. Comparisons between floating-point numbers in Trace Property calculations have a minimum 0.001 percent tolerance factor, so the maximum useful integral value is +/-100000, assuming you want to detect a variation of +/-1 in the property value.
