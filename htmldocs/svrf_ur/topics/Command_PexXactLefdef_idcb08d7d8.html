PEX XACT LEFDEF

Parasitic extraction

Specifies whether to ignore or distribute the parasitic capacitance for one or more specified nets during extraction in the Calibre xACT digital flow.

Usage

PEX XACT LEFDEF {IGNORE | DISTRIBUTE} netname [netname …]

Arguments

IGNORE

A required keyword that causes all geometries associated with the specified net(s) to be completely ignored during extraction. May not be specified with the DISTRIBUTE keyword.

DISTRIBUTE

A required keyword that causes all geometries associated with the specified net(s) to lose their association with that net and to merge the specified net(s) with whatever other nets they are touching during extraction. May not be specified with the IGNORE keyword.

netname

A required parameter that specifies one or more net names. You can specify netname any number of times in one statement. The netname can contain one or more question mark (?) characters. The ? is a wildcard character that matches zero or more characters. Each netname must be enclosed in quotation marks (“”).

Description

An optional statement specified together with the PEX Extract Exclude statement that causes all geometries associated with one or more specified nets to either lose their association with the nets or to completely ignore the nets during extraction no matter which layer the net is on. If PEX Extract Exclude is not also specified for the same nets, then the nets may still appear in the netlist with very small capacitance values. This statement may be specified more than once.

Use this statement to correctly process TRIMMETAL and SADP shapes in a LEF/DEF design database. To prevent TRIMMETAL shapes from causing shorts use the IGNORE keyword. To correctly model line-end extensions as designated by SADP shapes use the DISTRIBUTE keyword.

If trim layers, LEF layers with a TRIM attribute, are not mapped to the rule layers, the trim layers are ignored. If trim layers are not mapped to calibrated layers in they rule file, the trim layers are ignored.

When using the DISTRIBUTE keyword, if segment(s) of the specified net cause shorts between other nets, then the connected geometries are merged and the new net is arbitrarily name after one of the original nets.

If the same netname is specified using both the IGNORE and DISTRIBUTE keywords:

PEX XACT LEFDEF IGNORE NET_A
PEX XACT LEFDEF DISTRIBUTE NET_A

then Calibre xACT stops with the following error:

ERROR: NET_A nets can either be ignored or distributed, not both.

Examples

Example 1

Consider the following net geometries:

Figure 1. PEX XACT LEFDEF IGNORE TRIMMETAL_FILL

To completely ignore the TRIMMETAL_FILL net during capacitance extraction, specify the following rules:

PEX EXTRACT EXCLUDE TRIMMETAL_FILL
PEX XACT LEFDEF IGNORE TRIMMETAL_FILL

As a result, the total capacitance of NETA1 will match NETA2 and capacitance of NETB1 will match NETB2.

Example 2

Consider the following net geometries:

Figure 2. PEX XACT LEFDEF DISTRIBUTE SADP_FILL

To distribute the capacitance for the exclude net SADP_FILL, specify the following rules:

PEX EXTRACT EXCLUDE SADP_FILL
PEX XACT LEFDEF DISTRIBUTE SADP_FILL

As a result, the total capacitance of net C1 is greater than the capacitance of net C2.
