Net Area Ratio Coupled

Layer operation

Connectivity-related operation that allows computation of a coupled area between input layers. This can be useful when doing antenna checks involving coupled gates.

Usage

NET AREA RATIO COUPLED layer1 layer2 constraint ‘[’expression‘]’ [UNIQUE]

Arguments

layer1

A required original layer, layer set, or a derived polygon layer. Connectivity must be established on this layer.

layer2

A required original layer, layer set, or a derived polygon layer. Connectivity must be established on this layer.

‘[’expression‘]’

A required numeric expression within brackets that allows customizable control over the operation computations. The expression defines calculations to be made using the input layers.

The expression may involve numbers (including numeric variables), operators, parentheses ( ), and the functions given in the following table:

Table 1. Net Area Ratio Coupled Math Functions

Function

Definition

AREA()

Returns the coupled area of layer1 in user units of length squared. Note this function takes no arguments.

SQRT(x)

square root of x

EXP(x)

exponential (base e) of x

LOG(x)

natural logarithm of x

SIN(x)

sine of x radians

COS(x)

cosine of x radians

TAN(x)

tangent of x radians

MIN(expression1,expression2)

Returns the lesser value of the two expressions. The expressions must be of the same form as the primary Net Area Ratio expression, without the brackets.

MAX(expression1,expression2)

Returns the maximum value of the two expressions. The expressions must be of the same form as the primary Net Area Ratio expression, without the brackets.

In Table 1, x is a numeric argument (including numeric variables and other numeric-valued expressions). There is no compile-time or runtime exception checking on the arguments of these functions, and unreasonable or undefined values may result from certain arguments.

The expression must not result in strictly negative values because such values cannot be checked by a constraint. Division by zero does not satisfy any constraint.

Parentheses have the highest precedence for the calculation of numeric values and may be used for grouping of terms.

Operators — Unary operators (+,  -, !, ~) require only one numerical argument and all such operators have the same precedence. The + and - operators are the usual positive and negative signs. The other unary operators do the following:

  • ! operator — Returns 0 (or false) if its argument is non-zero and 1 (or true) if its argument is 0.

  • ~ operator — Returns 0 (or false) if the argument is positive and 1 (or true) if the argument is non-positive.

The binary operators ^, *, /, +, - require two numerical arguments. The ^ operator is the same as the C language pow( ) function, where x ^ y means x raised to the y power. The *,  ⁄, +, and - operators are multiplication, division, addition, and subtraction, respectively, and have the customary precedence. The ^ operator has the same precedence as * and /.

The binary operators || (OR) and && (AND) are the same as the C language operators of the same type, except that they have the same precedence as binary + and -. They expect numeric-valued inputs.

constraint

A required string that is one of the constraints listed in the “Constraint Notation” or “Alternate Notation” column of Table 1 in the “Constraints” topic. It is interpreted as the constraining value of the operation.

UNIQUE

An optional keyword that limits the number of output polygons to one for a given pair of input polygons from a given pair of interacting nodes.

Description

Computes the coupled area common to the input layers. This operation can be useful when doing antenna checks involving coupled gates. This operation is not a variant of Net Area Ratio but has some similarities to it.

This operation selects all polygons from layer1 that, after computation of the expression, satisfy the constraint. The computed value from the expression is attached to the output layer. The output is a Net Area Ratio accumulation (NARAC) layer, which makes it suitable input for operations that use such layers. For hierarchical applications, all NARAC layers are treated hierarchically. If either of the input layers are empty, then the output is empty.

Consider this operation:

Z = NET AREA RATIO COUPLED X Y constraint [expression]

The algorithm used to compute the output of the coupled area is as follows:

  1. Insure geometry equivalency between X and Y:

    1. Set X1 = X AND Y. Connectivity is passed from X to X1.

    2. Set Y1 = Y AND X. Connectivity is passed from Y to Y1.

  2. Create a list of the net interactions between X1 and Y1. Each element of the list has an attached interaction area.

    For each coincident polygon pair consisting of PX from X1 and PY from Y1 do the following:

    1. Let NX = net number of PX.

    2. Let NY = net number of PY.

    3. Do nothing if NX = NY. That is, self-coupling is not counted.

    4. Otherwise let R = min(NX, NY ) and S = max( NX, NY ). That is, for any interaction pair (R, S), it does not matter from which layer either R or S originated.

    5. Let the interaction area A of the nets corresponding to (R, S) be the area of PX (or equivalently PY).

    6. If (R, S) is already in the list of interactions, then add A to the area for this interaction pair. Otherwise, add (R, S) to the list and initialize its interaction area to A.

  3. Compute the output.

    For each polygon PX in X1:

    1. Let NX be its node number and let NY be the node number of the corresponding object on Y1.

    2. Let R = min( NX, NY ) and S = max( NX, NY). The coupled area A of PX is the interaction area of the nets corresponding to (R, S) recorded in Step 2.

    3. Evaluate the expression substituting A for AREA().

    4. If the value of the expression satisfies the constraint, then PX is output with the value of the expression attached. If UNIQUE is specified, then PX is not output if a polygon corresponding to (R, S) was previously output.

The following example calculates the coupled area (Z) that is common to the first input layer (X) and the second input layer (Y).

CONNECT X
CONNECT Y
// find portions of X polygons that share coupled area with Y
Z = NET AREA RATIO COUPLED X Y > 0 [AREA()]


The following are notes on two of the operators:

  • Negation

    ! operator — Returns “0” (or false) if its argument is non-zero and “1” (or true) if its argument is zero. You can use successive ! operators, for example: !!AREA(). If AREA() is not 0, !AREA() returns “0”. Therefore, !!AREA() would return a “1”. In effect !!AREA() acts as a detector of whether or not layer1 has coupled area.

  • Complement

    ~ operator — Returns “0” (or false) if the argument is positive and “1” (or true) if the argument is non-positive. This is different from the ! operator. Recall, the ! operator returns “0” if the argument is positive or negative and returns “1” if the argument is zero. The ~ operator is particularly useful where the sign of the argument is of concern.
