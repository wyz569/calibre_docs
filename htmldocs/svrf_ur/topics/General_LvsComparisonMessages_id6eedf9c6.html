LVS Comparison Messages

This section describes runtime messages from LVS comparison.

Table 1. LVS Comparison Messages

Explanation

The following errors and warnings are reported by Calibre applications:

EFFECTIVE property user-program WARNING:

<discrepancy type> on line <N>

while transforming layout with:

LVS REDUCE … (within EFFECTIVE program beginning on rule file line <N>)

Use instance ID# <N> in DEBUG trace range.

 

An LVS device reduction effective property computation attempted an impossible calculation described by the <discrepancy type>. Use the DEBUG and WARN commands in the effective property computation to further diagnose the issue.

EFFECTIVE property user-program WARNING:

<discrepancy type> on line <N>

while transforming layout with:

LVS PROPERTY INITIALIZE … (within PROPERTY program beginning on rule file line <N>)

Use instance ID# <N> in DEBUG trace range.

 

An LVS Property Initialize computation attempted an impossible calculation described by the <discrepancy type>. Use the DEBUG and WARN commands in the effective property computation to further diagnose the issue.

ERROR: Can not determine the SOURCE PATH from rule file <filename>.

 

The Source Path statement is required for the specified run type.

ERROR: Correspondence “<cell>” “<cell>” leads to a many-many correspondence.

 

Specification of hcells in the rule file or -hcell list has caused an ambiguous many-to-many mapping of cell names.

ERROR: Duplicate cell name “<name>” in cell lists “<list>” and “<list>” after wildcard substitution.

 

Wildcards are used in an LVS Cell List statement, resulting in a cell being matched by two different lists. The conflict must be resolved.

ERROR: Hcell “<layout_pattern>” “<source_pattern>” specifies incompatible patterns (different numbers of wildcard characters).

 

The number of wildcards does not match between layout and source cell name patterns in an hcell list specification.

ERROR: Parameter “<name>” ambiguously renamed in file “<netlist>” at line <line>

 

LVS Spice Rename Parameter is specified and a parameter name is ambiguously renamed.

ERROR: Supply error detected. ABORT ON SUPPLY ERROR is specified ‑ aborting.

 

LVS Abort On Supply Error is specified and a supply error has been detected. Circuit comparison is aborted.

ERROR: Property <prop> was declared string in TRACE PROPERTY user program, but (layout) input contains numeric data.

 

A user-defined Trace Property program declares a property as a string, but the property is numeric in the layout.

Error: Source passthrough ports missing in layout (see INFORMATION AND

WARNINGS).

 

LVS Report Option LPE is specified and passthrough ports in the source are missing in the layout. See “Passthrough Nets” in the Calibre Verification User’s Manual for more information.

Error: Layout passthrough ports missing in source (see INFORMATION AND

WARNINGS).

 

LVS Report Option SPE is specified and passthrough ports in the layout are missing in the source. See “Passthrough Nets” in the Calibre Verification User’s Manual for more information.

TRACE PROPERTY user-program WARNING:

<discrepancy type>

while tracing with:

TRACE PROPERTY … (within PROPERTY program beginning on rule file line <N>)

Use source instance ID# <N> in DEBUG trace range.

 

A Trace Property computation attempted an impossible calculation described by the <discrepancy type>. Use the DEBUG and WARN commands in the effective property computation to further diagnose the issue.

WARNING: Cell <name> cannot be made to correspond with cell <name> due to a hierarchical conflict.

 

Two cells cannot be used as hcells because they would form a cycle (recursion). The hcell list should be checked and corrected as necessary.

WARNING: Cells “<name>” …expanded due to seed promotion - correspondence ignored.

 

LVS Expand Seed Promotions YES is specified and cells have been expanded accordingly.

WARNING: Duplicate subckt definition “<subcircuit>” at line <N> in file “<file>”

 

LVS Spice Allow Duplicate Subcircuit Names YES is specified and a duplicate name is found while parsing a netlist.

WARNING: Duplicate subckt name “<name>” at line <N> in file “<file>”

 

LVS Spice Option S is specified and a duplicate subcircuit name is detected.

WARNING: Floating pins in file “<file>” at line <N> (<N> pins)

 

LVS Spice Option F is specified and floating pins are detected.

WARNING: LVS BOX cell “<cell>” not found in input.

 

An LVS Box cell cannot be identified in an input design.

Warning: Hierarchical connect involving call to excluded subcircuit “<cell>” ignored in file “<name>” at line <N>.

 

A *.CONNECT or *.J control statement is ignored because an LVS SPICE Exclude Cell statement has excluded a subcircuit containing a net that is referenced by the control statement.

WARNING: lvs cpoint name <name> not found in <design>.

 

LVS Cpoint is specified but a name cannot be found in one of the designs.

WARNING: LVS property resolution maximum exceeded.

 

The LVS Property Resolution Maximum value has been exceeded. Default is 32. Not all circuit ambiguities were processed by ambiguity resolution. False property errors may result.

WARNING: Undefined parameter “<param>” ignored in file “<file>” at line <N>

 

An undefined parameter has been found in a SPICE netlist. Possibly using LVS Spice Allow Inline Parameters YES can resolve the problem if it is an inline parameter. See “Element Statements” in the Calibre Verification User’s Manual for a description of correct SPICE syntax for built-in elements.
