PERC Group

Specification statement

Specifies a group name for a set of rule check procedures. The group name can be referenced in a PERC Load statement to execute the checks specified in the group.

Usage

PERC GROUP name check_proc [check_proc …]

Arguments

name

A required name of a rule check group. The name must be unique among group names and may not match any check_proc name. Additionally, to avoid syntactical collisions with other statements in the PERC family, the name may not be any of these keywords:

all

parallel

selecttype

info

pattern

unwaivable

init

reduction

xform

injection

select

 
check_proc

A required name of a rule check proc. At least one check_proc must be specified. The name must be literal (no wildcard) and may not match any group name.

Description

This statement performs a similar role to the Group statement in DRC-type runs. The name may be specified in a PERC Load statement in place of a rule check proc name. All proc names in the group are processed by the containing PERC Load statement as if the names had been specified literally (including order) in that statement.

There is no compile-time checking that any check_proc name exists as the name of an active Tcl proc in the TVF Function block. If a check_proc fails to match a valid rule check proc name, then the run stops with an error when the check_proc is called for execution.

Examples

Example 1

// specify rule check groups
PERC GROUP G1 R1 R2
PERC GROUP G2 R1 R2 R3

// specify groups in a PERC LOAD statement
PERC LOAD ... SELECT G1 R4 G2 SELECTTYPE INFO G1

The preceding code is equivalent to this:

PERC LOAD ... SELECT R1 R2 R4 R3 SELECTTYPE INFO R1 R2

Example 2

Assume this rule file configuration:

PERC GROUP G1 rule_1 rule_4
PERC LOAD perc_lib SELECT PARALLEL (G1) rule_2 rule_3

In a -turbo run, rule checks rule_1 and rule_4 are run sequentially in the same thread. Rule checks rule_2 and rule_3 are run in parallel threads with the G1 group.
