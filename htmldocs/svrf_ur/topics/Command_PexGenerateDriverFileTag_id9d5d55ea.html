PEX Generate Driver File Tag

Parasitic extraction

Instructs the Calibre xL tool to generate a driver and receiver file. Used only in Calibre xL.

Usage

PEX GENERATE DRIVER FILE TAG tags

Arguments

tags

A required list of one or more tags that specifies the endpoint of a net as the driver based on the type of device pin, h-port, or text port that is attached to the endpoint of that net. At least one of the following tags must be specified. The possible choices are:

  • d[rain]

  • s[ource]

  • t[ext port] (identifies text ports on any net as a driver)

  • c[ollector]

  • e[mitter]

  • neg

  • pos

Description

Instructs the Calibre xL tool to generate a single file that contains the driver and receiver information used to identify the paths to evaluate for self and mutual impedance extraction. The default name of the file generated is driver.xl. Use the PEX Inductance Driver File statement to specify a different filename and/or location. The Calibre xL tool calculates self and mutual impedance for all paths that are not filtered.

Examples

Example 1

The following statement instructs the Calibre xL tool to generate the driver and receiver file, driver.xl, using the collector, drain, emitter, and source of nets in the layout as drivers.

PEX GENERATE DRIVER FILE TAG c d e s

Example 2

The following statements generate a driver and receiver file with the name, test_driver.xl, located in directory, Shared/Design/data/. This file specifies all pos and neg pins that are connected to nets are drivers.

PEX INDUCTANCE DRIVER FILE "./Shared/Design/data/test_driver.xl"
PEX GENERATE DRIVER FILE TAG pos neg

Note that the./Shared/Deign/data/ directory must already exist. If the directory does not exist, then Calibre xL stops with an error.
