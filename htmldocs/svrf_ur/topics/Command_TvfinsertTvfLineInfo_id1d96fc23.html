tvf::insert_tvf_line_info

Application: compile-time only.

Inserts a comment into the generated SVRF file with the line number of the corresponding command in the compile-time TVF file.

Usage

tvf::insert_tvf_line_info { 1 | 0 }

Arguments

1 | 0

A required argument choice that specifies whether to enable (1) or disable (0) line insertion in the generated SVRF file.

Return Values

None.

Description

When tvf::insert_tvf_line_info is enabled, a comment line is inserted into the generated SVRF file with the TVF rule file name and the line number in the TVF rule file corresponding to the SVRF statement. This information can make it easier to correlate the generated SVRF code to the TVF file.

This command must be used with the ‑tvf_instrument and ‑E command line options.

If the TVF command generates multiple lines of SVRF code, the comment line is inserted once before the first of the SVRF statements.

You can enable and disable line insertion so that comment lines are inserted only in desired sections of the code.

Examples

Suppose the TVF rule file tvf.rules has the following code:

#!tvf
tvf::insert_tvf_line_info 1

# ... basic rule file setup and layer derivations

tvf::RULECHECK tvf_contact_width {
   tvf::@ Minimum contact width = 1.75 
   tvf::OUTLAYER {int contact< 1.75 }
}

Run Calibre with the following command line call:

calibre -drc -hier -E svrf_save.rul -tvf_instrument tvf.rules | tee log

The file svrf_save.rul has the following text:

// tvf.rules:182
tvf_contact_width {
    @ Minimum contact width = 1.75
    int contact< 1.75 
}
