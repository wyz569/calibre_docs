DFM Remove Edge

Layer operation

License: Calibre YieldEnhancer

Removes edges shorter than a specified value from the input polygons and outputs the resulting polygons.

Usage

DFM REMOVE EDGE layer {length | x_length y_length} [NONOTCH] [NOHOLE]

Arguments

layer

A required original or derived polygon layer. Derived layers must be merged.

length | x_length y_length

A required argument that specifies the minimum edge length that is kept. The value(s) are non-negative numbers in user units. Edges with length less than length are removed.

If two values are specified, they give separate constraints for edge lengths in the x and y directions, respectively. If one of the values is 0, it is ignored and edges in that direction are not removed. At least one value must be positive.

NONOTCH

An optional keyword that specifies to ignore notch violations.

NOHOLE

An optional keyword that specifies to ignore holes. By default, edges that make up holes are processed.

Description

Removes edges shorter than the specified value from the input polygons and outputs the resulting polygons. Non-orthogonal polygons are not processed and are not included in the output layer.

This removal process is used:

  • Polygons are processed one at a time. Processing starts at the lower left vertex of each polygon and proceeds counter-clockwise. The outer edges are processed first, followed by interior edges forming holes (if present). Holes are ignored if NOHOLE is specified.

  • Edges are removed in two phases: 1. Remove bumps, 2. Remove other edge violations.

  • For each of the two phases, when an edge violation is found, it is fixed and processing starts again at the lower left vertex, until no more violations are found or the polygon is removed. This process is used so that new violations introduced by edge removal are fixed.

  • If an edge removal splits the polygon into multiple polygons, processing continues on the resulting polygon with the greatest area. All smaller polygons are discarded.

Figure 1. DFM REMOVE EDGE

In most cases the output polygon will not contain area that is outside the original polygon. This is not true for narrow holes, which are filled, as described in the next section.

Narrow Holes

A narrow hole has at least one edge that is shorter than the minimum allowed length, where removal of the short edge causes the hole to be filled. This is shown is the following figure.



Narrow holes are not considered notches, and are filled even when NONOTCH is specified. You can specify NOHOLE to ignore holes.

Examples

Remove edges shorter than 0.008, but do not process holes. 

layer1_smooth = DFM REMOVE EDGE layer1 0.008 NOHOLE
