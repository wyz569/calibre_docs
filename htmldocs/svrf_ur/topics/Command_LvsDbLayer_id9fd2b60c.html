LVS DB Layer

Specification statement

Specifies layers to be merged and saved in a persistent hierarchical database.

Usage

LVS DB LAYER layer [layer …]

Arguments

layer

A required name of an original or derived layer. More than one layer may be specified.

Description

Specifies layers to be merged and passed through to a persistent hierarchical database (PHDB). These are the conditions for saving specified layers:

  • The CCI, XACT, or XRC option is used in the Mask SVDB Directory statement in a calibre -lvs run. By default, only Device and connectivity layers (without connectivity information) are stored in the PHDB.

  • The DEVICES option is used in the DFM Database command in a calibre -dfm run. By default, only layers required for the run are saved.

  • The statement is used in a calibre -perc -ldl run.

To save connectivity layers with their nodal information, use the LVS DB Connectivity Layer statement instead.

In the Calibre Connectivity Interface (CCI) flow, layers may be specified in this statement for inclusion in an annotated geometry file (AGF) written with qs::write_agf or AGF WRITE.

Examples

The following example passes layer border to the PHDB, and the layer can be written to an AGF file in CCI.

LVS DB LAYER border // add this to AGF output in CCI

MASK SVDB DIRECTORY svdb CCI
