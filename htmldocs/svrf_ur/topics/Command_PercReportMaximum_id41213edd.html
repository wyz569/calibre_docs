PERC Report Maximum

Specification statement

Specifies the maximum number of reported items in the Calibre PERC report file.

Usage

PERC REPORT MAXIMUM {number | ALL}

Arguments

number

A positive integer that specifies the number of results to report for the Calibre PERC run. If this statement is not specified, the number is taken from the LVS Report Maximum statement in the rule file. The default for this latter statement is 50.

ALL

A keyword that specifies all generated results for the run appear in the PERC Report.

Description

Specifies the maximum number of results that can appear in various sections of the PERC Report. This statement may be specified once. If this statement is explicitly specified, then it overrides the LVS Report Maximum setting in a Calibre PERC run.

The ALL keyword must be specified when generating results to be exported as Calibre PERC waivers.

The limit of the number of results the ALL keyword reports can be adjusted using the perc::set_parameters -resultMaxCount option in the rule file.

See “Calibre PERC Reports” in the Calibre PERC User’s Manual for details on the PERC Report.

See also PERC Load.

Examples

In this example, the PERC Report Maximum statement overrides the LVS Report Maximum setting.

LVS REPORT MAXIMUM 50

PERC REPORT "perc.rep"
PERC REPORT MAXIMUM 25
PERC LOAD perc.func INIT init_1 SELECT rule.1 rule.2
