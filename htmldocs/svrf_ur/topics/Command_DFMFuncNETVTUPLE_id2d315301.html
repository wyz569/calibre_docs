NETVTUPLE

Vector net property function for DFM expressions

Creates a vector of netID values. It is most useful in DFM Property Select Secondary, where it creates one tuple for each cluster that is grouped with the select geometry.

Usage

NETVTUPLE(arg_expr1[, arg_expr2] …)

Arguments

arg_expr1[, arg_expr2]

A required argument that specifies one or more expressions. The argument expressions (arg_expr1, arg_expr2 and so on) must be functions that return a netID property, such as GLOBALNETID, NETID, NETPROPERTY, or NETPROPERTY_REF. Multiple arguments are separated by commas. The number of arguments corresponds to the tuple size; each argument is evaluated to form a tuple element.

If a function such as NETID or NETPROPERTY references a secondary layer, the ordinal must be specified.

Return Values

Vector of netID values.

Description

NETVTUPLE creates a vector of netID values with tuple elements consisting of the arguments to the function. This function is only supported in the DFM Property family of operations, and is most useful in DFM Property Select Secondary. One tuple is created per cluster associated with the shape on the output layer.

The terminology used for vector notation is described in “Vector DFM Properties”. In brief, {} indicates a vector and <> indicates a tuple within the vector. For the vector {<1 2 3> <4 5 6>}, each tuple has 3 elements (tuple size 3) and there are 2 tuples (tuple count 2). The tuple count is also called the vector length.

The argument expressions (arg_expr) are evaluated to form the tuple elements; the number of arguments determines the tuple size. There is one tuple for each cluster associated with the geometry on the output layer. For DFM Property Select Secondary there can be multiple clusters associated with each select layer geometry written to the output layer. For the other DFM Property operations there is one cluster associated with each primary layer object written to the output layer.

Any expression that does not evaluate successfully for any tuple argument in the NETVTUPLE function call results in the entire NETVTUPLE function failing to evaluate successfully. When this happens, the following apply:

  • If the property is defined multiple times to form an expression chain, the operation attempts to evaluate the expression for the next definition. If there is no next definition, the geometry is not written to the output layer.

  • If the property is defined once (that is, not chained), the geometry is not written to the output layer.

“Expressions that Do Not Evaluate Successfully” has additional details. Also see “Expression Chains”.

Examples

net_vtup = DFM PROPERTY SELECT SECONDARY m1 m2 OVERLAP ABUT ALSO MULTI
		[ m1_nid_vtup = NETVTUPLE ( NETID(m1))]
	SELECT m2

net_vtup { DFM RDB net_vtup vtup.rdb }
