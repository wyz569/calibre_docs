SVTUPLE

String vector function for DFM expressions

Creates a vector of string type properties. It is most useful in DFM Property Select Secondary, where it creates one tuple for each cluster that is grouped with the select geometry.

Usage

SVTUPLE(arg_expr1[, arg_expr2] …)

Arguments

arg_expr1[, arg_expr2]

A required string value or an expression that evaluates to a string. Multiple arguments are separated by commas. The number of arguments corresponds to the tuple size; each argument is evaluated to form a tuple element.

Arguments can be strings, expressions involving the SPROPERTY function, or functions that return strings. Functions can reference either the primary or a secondary layer. If a function such as SPROPERTY references a secondary layer, the ordinal argument must be provided. See the Description section for the behavior with primary or secondary layers. The property reference function SPROPERTY_REF is supported.

Return Values

String-type property vector.

Description

SVTUPLE creates a vector of strings with tuple elements consisting of the arguments to the function. This function is only supported in the DFM Property family of operations, and is most useful in DFM Property Select Secondary. One tuple is created per cluster associated with the shape on the output layer.

The terminology used for vector notation is described in “Vector DFM Properties”. In brief, {} indicates a vector and <> indicates a tuple within the vector. For the vector {<1 2 3> <4 5 6>}, each tuple has 3 elements (tuple size 3) and there are 2 tuples (tuple count 2). The tuple count is also called the vector length.

The argument expressions (arg_expr) are evaluated to form the tuple elements; the number of arguments determines the tuple size. There is one tuple for each cluster associated with the geometry on the output layer. For DFM Property Select Secondary there can be multiple clusters associated with each select layer geometry written to the output layer. For the other DFM Property operations there is one cluster associated with each primary layer object written to the output layer.

Any expression that does not evaluate successfully for any tuple argument in the SVTUPLE function call results in the entire SVTUPLE function failing to evaluate successfully. When this happens, the following apply:

  • If the property is defined multiple times to form an expression chain, the operation attempts to evaluate the expression for the next definition. If there is no next definition, the geometry is not written to the output layer.

  • If the property is defined once (that is, not chained), the geometry is not written to the output layer.

“Expressions that Do Not Evaluate Successfully” has additional details. Also see “Expression Chains”.

String vectors may be compared lexicographically using ==, !=, >, >=, <, <= operators; such comparisons can be done using Conditional Expressions. See VSTRING for information and an example of using conditional expressions to perform string vector comparison.

Examples

m2_string = DFM PROPERTY m2 m1 OVERLAP ABUT ALSO MULTI
		[ m1_m2_str_prop = ( count(m1) == 1) ? "one" : 
									((count(m1) == 2) ? "two" : "other") ]

string_vtup = DFM PROPERTY SELECT SECONDARY m1_string m2_string m3 
						OVERLAP ABUT ALSO MULTI
	[ str_m1_m2_vtup = SVTUPLE( SPROPERTY(m2_string, m1_m2_str_prop, 1) ) ]
	SELECT m2_string

str_vtup { DFM RDB string_vtup vtup.rdb }
