tvf::RULECHECK

Application: compile-time only

Denotes a Calibre nmDRC rule check in compile-time TVF.

Usage

tvf::RULECHECK name ‘{’
rulecheck
‘}’

Arguments

name

A required argument that specifies the name of the rule check.

rulecheck

A required argument that specifies the body of the rule check. The rule check body is enclosed in braces as shown. See the “Description” section for the commands that are allowed within the rule check body.

Return Values

None.

Description

An SVRF rule check is coded with the tvf::RULECHECK construct in compile-time TVF.

These commands are allowed within the rule check body:

tvf::SETLAYER

tvf::@

tvf::OUTLAYER

tvf::COMMENT

tvf::COPY

 

The SETLAYER command handles layer derivations and the OUTLAYER command sends output to the DRC results database. Rule check comments are handled with the tvf::@ or TVF::COMMENT command.

Examples

tvf:// TVF rule check
tvf::RULECHECK my_rule {
   tvf::@ my rule check comment
   tvf::SETLAYER layer1 = {layer2 OR layer3}
   tvf::OUTLAYER {layer4 NOT layer1}
}
