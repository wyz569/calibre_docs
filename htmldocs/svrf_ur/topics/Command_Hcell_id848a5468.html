Hcell

Specification statement

Specifies a pair of subcircuits — one from layout and one from source — that correspond. Used only in hierarchical applications.

Usage

HCELL layout_name source_name

Arguments

layout_name

A required cell name in the layout that corresponds to a cell name in the source design. A layout_name can contain one or more asterisk (*) wildcard characters that match zero or more characters. Names that use wildcards must be enclosed in quotation marks because the * is a reserved symbol.

source_name

A required cell name in the source that corresponds to a cell name in the layout. A source_name can contain one or more asterisk (*) wildcard characters that match zero or more characters. Names that use wildcards must be enclosed in quotation marks because the * is a reserved symbol.

Description

Specifies the names of a pair of cells that correspond in layout and source. Such cells are otherwise known as hcells. This statement may appear any number of times and information is accumulated.

In hierarchical DFM, Calibre nmDRC, Calibre PERC, and Calibre nmLVS-H connectivity extraction, hcells are preserved (not expanded automatically) and are exempt from certain layout transformations. In Calibre nmLVS‑H, hcells are also used as correspondence cells for circuit comparison.

Note:

The Calibre xRC tool ignores this statement — you must use an xcell list instead. Refer to the Calibre xRC User’s Manual.

The effect of this statement in Calibre nmLVS-H is the same as specifying a pair of cell names in an hcell list with the -hcell command line option or generating hcell pairs automatically with the -genhcells option. For more information, see “Calibre nmLVS and Calibre nmLVS-H Command Line” and the “Hcells” section in the Calibre Verification User’s Manual.

The layout_name and source_name may be the same or different. You can specify a 1-to-n relation by placing a layout name in several Hcell statements with different source names. Similarly, you can specify a n-to-1 relation by placing a source name in several Hcell statements with different layout names. However, n-to-m relations are not allowed. The latter restriction is enforced by the hierarchical circuit comparison module (calibre ‑lvs -hier) but is not enforced by the rule file compiler, the circuit extraction module, or by Calibre nmDRC-H.

While the statement is checked for syntactical correctness at compilation time, cell names are not matched to the input designs during rule file compilation. Failure to match specified cell names results in runtime warnings. The source_name parameter is not read if the run does not require a source netlist.

In the Calibre nmLVS-H comparison module (calibre -lvs -hier), hcells established with the ‑hcell and -genhcells or ‑automatch command line options are combined with hcells from Hcell specification statements. All hcells specified by any of these methods are processed as a single concatenated hcell list. The corresponding cells in the layout and source netlists can then be compared hierarchically. For detailed information on generating candidate hcell lists for circuit comparison, see LVS Hcell Report, “Hcell and Hierarchical Analysis” in the Calibre Query Server Manual, or “Performing Hcell Analysis in Calibre Interactive” in the Calibre Interactive (New GUI) User’s Manual.

In the Calibre nmLVS-H circuit extraction module (calibre -spice), specifying hcells ensures that all hcells are preserved as subcircuits in the extracted layout netlist and are available for use as hcells in the comparison phase. Note that hcell specification may slow down circuit extraction in cases where it would be beneficial to expand the hcells (for example due to dense overlap removal). If you want to control hcells separately during connectivity extraction, use Layout Preserve Cell List. You can control how the tool expands hcells during connectivity extraction through the LVS Auto Expand Hcells statement.

Case sensitivity of hcell names in Calibre nmLVS-H is controlled by the LVS Compare Case specification statement. Specifically, hcell names are case-insensitive by default; hcell names are case-sensitive if you specify LVS Compare Case YES or LVS Compare Case TYPES. This applies in Calibre nmLVS-H to circuit extraction as well as to circuit comparison.

You can specify hcells in Calibre Interactive—nmLVS. See “Supplying Hcell Data” in the Calibre Interactive (New GUI) User’s Manual.

In order to guarantee that devices are properly formed, seed promotion can occur during device recognition. This is true even of hcells. See “Seed Promotion and Hcells” in the Calibre Verification User’s Manual for a discussion of this behavior. In many cases, the seed promotion is mitigated when LVS Push Devices YES is specified, which is the default.

Using Wildcards to Specify Hcells

Wildcards may be used in either layout or source name parameters. These rules apply:

  • Names with wildcards must be quoted in the Hcell statement, but the same does not hold in a ‑hcell list where quotes are treated as literal.

  • If wildcards are used for only a layout or source name (not both), then the wildcards match any strings in that design’s cell names according to the pattern specified for the name. If more than one cell name matches the pattern, this establishes a many-to-one hcell correspondence.

  • If wildcards are used in both layout and source names in a single Hcell statement, then the following apply:

    • The number of wildcards used in the layout pattern must match the number of wildcards used in the source pattern. If there is a mismatch in the number of wildcards, and the layout pattern matches at least one cell name, then a runtime error results, and the overall status of an LVS comparison run is NOT COMPARED. Any strings that match the layout wildcards are substituted for the wildcards in the source pattern, one-for-one, and in the order the wildcards occur.

    • The wildcards on the source side are placeholders for the strings that the layout wildcards match and are replaced by any matched strings to form candidate source cell names. For any candidate source cell name that exists in the source design, that cell is used for hcell correspondence.

    • Each layout pattern wildcard matches the longest possible string of characters in each layout cell name. If there is more than one wildcard in a pattern, matching starts with the left-most wildcard and proceeds to the right, with each successive wildcard matching the longest possible string and leaving the rest of the name (possibly null) to be matched by the next wildcard in the pattern.

See “Wildcards in Hcell Names” in the Calibre Verification User’s Manual for additional details.

Calibre nmDRC-H Use of Hcell Statements

Note:

For Calibre nmDRC-H, the Hcell statement is only for the flows described here. Otherwise, Hcell statements should not appear in your Calibre nmDRC rule file. Calibre nmDRC‑H generates a warning if Hcells are present to let you know that performance may be severely impacted.

Use of the Hcell specification statement in a Calibre nmDRC-H rule file is applicable to specialized applications (for example, sizing and Litho operations used for manufacturability) where a Calibre nmLVS-H run on an output mask layout database is planned. Calibre nmDRC‑H observes the Hcell specification statements in the rule file and does not allow expansion of hcells on the layout side. See “Calibre nmDRC‑H Use of Hcells” in the Calibre Verification User’s Manual.

The Layout Preserve Cell List statement has similar behavior as the Hcell statement in Calibre nmDRC.

Clone Cell Handling of Hcells

If an hcell is cloned due to Layout Clone Rotated Placements or Layout Clone Transformed Placements YES, then all clones of that hcell are treated as hcells.

The presence of hcells that match more than one source cell to one layout cell can result in a many-to-many hcell match once cloning is completed. This situation is not allowed and aborts Calibre with a fatal error.

For example, the following Hcell statements:

HCELL esd_dio esd_dio_1
HCELL esd_dio esd_dio_2

could cause Calibre to print following message:

ERROR: Correspondence “esd_dio$C0$” “esd_dio_2” leads to a many-many correspondence.

In this message, “esd_dio$C0$” is one of the clone cell names created from the original layout cell esd_dio. If such errors occur, you need to adjust the hcell list to mitigate them.

See also LVS Exclude Hcell, LVS Expand Ambiguously High Shorted Hcells, LVS Cell List, and LVS Report Warnings Hcell Only.

Examples

//    layout    source

// one-to-one correspondence
HCELL alu       alu
HCELL buf       buffer
// one-to-two correspondence
HCELL cellA     cellB
HCELL cellA     cellC
// two-to-one correspondence
HCELL cellD     cellF
HCELL cellE     cellF
// possible many-to-one correspondences
HCELL "cellG*" cellJ
HCELL cellH    "cellI*"
