#IFDEF/#IFNDEF

The #IFDEF (if defined) and #IFNDEF (if not defined) pre-processor directives provide conditional compilation of the SVRF rule file based on variable definitions.

Usage

   {#IFDEF var_name [var_value] | #IFNDEF var_name [var_value]}
      rule_file_text
   [ #ELSE
      rule_file_text ] ...
   #ENDIF

Arguments

var_name

A string giving the variable name. You should enclose var_name within quotes if var_name includes spaces or characters other than letters and numbers.

var_value

A string giving an optional value for the variable.

rule_file_text

Rule file statements and operations to be performed, depending on the conditional directive. The rule_file_text string can also contain conditionals, which creates nested pre-processor directives.

Description

Conditional directives provide conditional compilation for sections of the rule file. #IFDEF and #IFNDEF test whether or not a variable is defined or undefined, respectively, and optionally compare the variable’s value to a specified value (var_value). An #IFDEF or #IFNDEF statement can include an optional #ELSE statement, which is executed if the preceding #IFDEF or #IFNDEF conditional evaluates to false.

The variable (var_name) can be defined in the shell environment or in the rule file with a #DEFINE or #UNDEFINE pre-processor directive. See “About Conditional Directives for Compilation Control” for a complete discussion, including extended examples and a precise definition of the conditional compilation logic.

The SVRF Error statement is useful for inserting compiler error messages within conditional constructs.
