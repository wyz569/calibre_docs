DRC_EQ

Function for DFM expressions

Performs an equal to (==) comparison with a built-in tolerance of 10e‑8.

Usage

DRC_EQ(a, b)

Arguments

a

A required numeric value, or an expression that evaluates to a numeric value. The expression can include DFM functions.

b

A required numeric value, or an expression that evaluates to a numeric value. The expression can include DFM functions.

Return Values

Floating-point value.

Description

Performs an equal to (==) comparison with a built-in tolerance of 10e‑8. The comparison is suitable for use when comparing geometric measurements that may be affected by grid-snapping.

DRC_EQ returns 1.0 if a and b are equal within an absolute or relative tolerance of 10e‑8, and returns 0.0 otherwise.

See “DRC Constraint Comparison Functions” for a general discussion of the DRC comparison functions, including a discussion of relative and absolute comparison. DRC_EQ provides the same comparison behavior within a property definition as the == constraint in a layer operation.

See “Equality Tests and Comparisons for Floating-Point Values” for relevant discussions of floating-point arithmetic.

Examples

Example 1

Output geometries with attached property Area_A_B if AreaA equals AreaB. The areas are compared with a tolerance of 10e‑8. 

area_out = DFM PROPERTY A B OVERLAP MULTI ABUT ALSO
	[ - = DRC_EQ(AREA(A), AREA(B))] > 0
	[Area_A_B = AREA(A) ]

Example 2

Example 1 can also be done this way:

area_out = DFM PROPERTY A B OVERLAP MULTI ABUT ALSO
	[ Area_A_B = ( DRC_EQ(AREA(A), AREA(B)) > 0 ) ? AREA(A) : 0 ] > 0

This shows the correct practice when using DRC_EQ() as part of the condition for a conditional expression. DRC_EQ returns a floating-point value so it cannot be used directly as a condition—it must be compared to a value.
