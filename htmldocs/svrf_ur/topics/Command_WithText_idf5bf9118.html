With Text

Layer operation

Selects polygons that intersect the position of the specified text object.

Usage

WITH TEXT layer name [text_layer] [PRIMARY ONLY] [CASE SENSITIVE]

Arguments

layer

A required original layer or layer set, or a derived polygon layer.

name

A required name of a free-standing text object, which can contain one or more question mark (?) characters. The ? is a wildcard character that matches zero or more characters. The name can be a string variable (see Variable).

text_layer

An optional original layer or layer set where text objects are found.

PRIMARY ONLY

An optional keyword that specifies only top-cell text is to be used in the operation. Text objects in a placed cell of the Layout Place Cell statement with one placed cell parameter are treated as top-level text objects.

CASE SENSITIVE

An optional keyword that specifies name parameters must match by case.

Description

Selects all layer polygons that intersect the positions of text objects having the specified name. The specified parameter order must be observed to avoid ambiguity.

If text_layer is specified, only text objects with the specified name on text_layer are considered. The name parameter is case-insensitive by default.

Layout Text and Layout Text File text objects participate in With Text operations. The Text, Text Depth, and Text Layer statements have no effect on With Text operations.

As a performance optimization, the Calibre database constructor only reads in text objects that are actually needed for the run. If you require all text objects interacting with a given layer to be read for applications like Calibre PERC LDL that perform DFM Database post-processing, then using “?” for the name parameter is useful. Be aware that this can increase memory consumption. See “Example 2.”

See also Not With Text.

Examples

Example 1

// Select all pads larger than 5000 square microns that have 
// test-pads as a text label on layer 14:
large_pads = pads AREA > 5000
test_pads = large_pads WITH TEXT "test-pads" 14

Example 2

The Calibre database constructor only reads in text objects that are actually needed for the run. In order to read in all text objects from a given layer, including those that are detached from polygons, you can do this:

READ_TEXT {
@ force reading of text objects for these layers
  CMACRO READ_TEXT L1
  CMACRO READ_TEXT L2
  ...
}
// read in text objects on the input layer but leave results empty
DMACRO READ_TEXT lay {
  x = (EXTENT) WITH TEXT "?" lay
  AREA x == 0
}
