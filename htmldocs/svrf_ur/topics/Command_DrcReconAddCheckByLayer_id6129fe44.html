DRC Recon Add Check By Layer

Specification statement

Selects rule checks to be executed in a Calibre nmDRC Reconnaissance run based on original layers. A rule check is executed if it requires any of the specified layers in the layer derivation chain in order to produce the output.

Usage

DRC RECON ADD CHECK BY LAYER layer_name [layer_name …]

Arguments

layer_name

A required original layer name (not a number). You can specify layer_name any number of times in one statement. Each specified layer can contain one or more wildcard (*) characters, which match zero or more characters. Layer names containing a wildcard must be enclosed in quotes; otherwise, a compilation error is generated.

Description

This statement identifies rule checks to run in Calibre nmDRC Recon mode based upon the occurrence of the specified layer names anywhere in the rule checks’ layer derivation trees, including connectivity derivations. If DRC Recon Add Check By Layer is included in the rule file, only the selected rule checks are executed, plus any checks selected by DRC Recon Add Check statements. This statement may appear in the rule file any number of times and is ignored when ‑recon is not specified on the command line.

By default, the Calibre nmDRC Recon mode chooses a set of rule checks from the rule file, which may or may not be the complete set of checks. See “Chip-Level Verification” in the Calibre Solutions for Physical Verification manual. You can specify a custom set of checks to execute with this statement and the related statements DRC Recon Add Check, DRC Recon Remove Check, and DRC Recon Remove Check By Layer.

The DRC Recon Add Check By Layer statement works as follows: For each rule check C in the rule file, let L be the set of all original layers that are required to execute C. That is, the layers of L are 1) in the derivation tree of all output operations contained in C, or 2) in the derivation tree of required connectivity operations, if connectivity extraction is required to execute C. If any member of L is specified as a layer_name, then a “DRC Recon Add Check C” statement is implicitly generated.

Each rule check C that is enabled by the preceding process must be selected for execution in a standard Calibre nmDRC run, either by default or by use of DRC Select Check and DRC Unselect Check statements. A compiler error is issued if this is not true. See example “Example 2”.

Overall check selection for a Calibre nmDRC Recon run is done by this process:

  1. Determine the checks that are selected to run in a standard Calibre nmDRC run.

  2. From the set of checks selected in Step 1, enable checks for the Calibre nmDRC Recon run:

    • If no DRC Recon Add Check or DRC Recon Add Check By Layer statements exist in the rule file, enable the default set of Calibre nmDRC Recon rule checks.

    • If DRC Recon Add Check or DRC Recon Add Check By Layer statements exist in the rule file, enable only those rule checks.

  3. If DRC Recon Remove Check or DRC Recon Remove Check By Layer statements are in the rule file, disable those rule checks.

    This step may remove rule checks added in the previous step.

Examples

Also see “Selecting Checks in Calibre nmDRC Recon” in the Calibre Solutions for Physical Verification manual.

Example 1

Assume the following rule file with five rule checks POLY.1, POLY.2, POLY.3, NWELL.2, and M1.1:

LAYER DIFF 1
LAYER NWELL 3
LAYER POLY 9
LAYER CONT  10
LAYER M1    11

rawpoly   = COPY POLY
gate      = rawpoly AND DIFF
fieldpoly = rawpoly NOT gate
sd = DIFF NOT gate
m1_i = M1 ENCLOSE CONT

CONNECT m1_i sd BY CONT
ntap = sd AND NWELL
SCONNECT m1_i NWELL BY ntap
CONNECT gate fieldpoly
POLY.1 {
@ Min width of POLY = 0.05
  INTERNAL POLY < 0.05 ABUT < 90 SINGULAR
}
POLY.2 {
@ Min spacing of gate = 0.14
  EXTERNAL gate < 0.14 ABUT < 90 SINGULAR
}
POLY.3 { 
@ Max area of fieldpoly per net is 4 sq um
  NET AREA fieldpoly > 4
}
NWELL.2 {
@ min nwell spacing for wells of different potentials is 0.35
  EXT NWELL < 0.35 ABUT < 90 SINGULAR NOT CONNECTED
}

M1.1 {
 @ Minimum M1 width = 1.0 
INTERNAL M1 < 1.0
}

When no specification statements in the DRC Recon family are active, calibre -recon -drc -hier executes the POLY.1, POLY.2, and M1.1 checks. POLY.3 and NWELL.2 are connectivity checks, so they are not executed by default.

Several different uses of DRC RECON ADD CHECK BY LAYER are demonstrated:

Case A:

This statement causes only the POLY checks and NWELL.2 to be executed::

DRC RECON ADD CHECK BY LAYER POLY

POLY.1 explicitly uses the POLY layer. POLY.1 and POLY.2 use POLY in the derivation of layers gate and fieldpoly, respectively. NWELL.2 requires POLY in the connectivity derivation for the NWELL layer. The M1.1 check is not executed because it does not use the POLY layer.

Case B:

To also include all rule checks that are selected by default in a Calibre nmDRC Recon run, do this: 

DRC RECON ADD CHECK BY LAYER POLY
DRC RECON ADD CHECK RECON_DEFAULT

This set of statements also includes the M1.1 check, in addition to the checks selected in Case A.

Case C: 

DRC RECON ADD CHECK BY LAYER M1

This statement includes checks M1.1 and NWELL.2 only. The M1.1 check explicitly uses the M1 layer. The NWELL.2 check requires M1 in the connectivity derivation for the NWELL layer.

Example 2

This example produces a compiler error because the DRC Recon Select Check By Layer statement attempts to enable a check that is not selected.

DRC UNSELECT CHECK m1_dens // density check for layer m1
DRC RECON ADD CHECK BY LAYER m1 // compiler error

ERROR: Error SPC26 - DRC RECON ADD CHECK m1_dens is not a recon RuleCheck.

To enable only rules checks that use layer m1, but exclude the m1_dens check, do the following:

DRC RECON ADD CHECK BY LAYER m1 // enable all rule checks using layer m1
DRC RECON REMOVE CHECK m1_dens  // disable m1_dens
