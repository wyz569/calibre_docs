VMIN

Vector property function for DFM expressions

Determines the minimum value in a specified slice of a vector tuple. Additionally can determine the tuple index of a minimum value of a specified slice and use that index to return a corresponding value in another slice, possibly in another vector. Supported only in the DFM Property operation.

Usage

VMIN(vec1,slice1[,slice2])

VMIN(vec1,slice1,vec2,slice2)

Arguments

vec1

A required numeric vector.

slice1

A required non-negative integer that specifies the slice (column) index in vec1 to evaluate. The value of slice1 must be less than the tuple size of vec1.

slice2

A non-negative integer that specifies a slice index from which to return a value corresponding to the minimum value found in slice1. The definition of value correspondence is in the “Description” section. When specified without vec2, slice2 is optional and refers to a slice of vec1. When specified with vec2, slice2 is required and refers to a slice of vec2. The value of slice2 must be less than the tuple size of the vector to which it applies.

vec2

A required numeric vector when using the second syntactical form of the function. When vec2 is specified, a value from slice2 in vec2 that corresponds to the minimum value in slice1 is returned. The definition of value correspondence is in the Description section. The tuple count of vec2 must match vec1.

Return Values

Number.

Description

When used in its simplest form, this function returns the minimum value in slice1 of vec1.

When the slice2 argument is provided in either syntax, the minimum value is first determined in slice1. The index of the vector tuple in vec1 containing that minimum value is noted. (If the minimum value is not unique, then the index of the first tuple in which a value could serve as a minimum is noted.) A value in slice2 of the tuple having the noted index is taken either from vec1 (if vec2 is not used) or vec2. This value in slice2 is said to correspond to the minimum value from slice1. The value from slice2 is returned by the function.

Therefore, if slice2 is specified without vec2, then the function returns the corresponding value (using the preceding correspondence definition) to the minimum value in slice1.

If slice2 is specified with vec2, then the behavior is similar to the preceding paragraph except that slice2 refers to vec2 rather than vec1.

Examples

Assume vectors V1 and V2, each having two tuples of size 3:

tuple index     0        1
         V1 {<0 1 2> <3  4  5>} 
         V2 {<6 7 8> <9 10 11>}
slice index   0 1 2   0  1  2

The following function evaluations apply:

VMIN(V1,2) = 2 (minimum element of slice 2 in V1; it occurs in tuple 0)

VMIN(V1,2,0) = 0 (slice 0 element of tuple 0 in V1; corresponds to “2” of slice 2)

VMIN(V1,2,V2,0) = 6 (slice 0 element of tuple 0 in V2; corresponds to “2” of slice 2 in V1)

These are some corresponding DFM expressions to produce the results:

[ V1 = CONCAT(VECTOR(0,1,2),VECTOR(3,4,5)) ]
[ V2 = CONCAT(VECTOR(6,7,8),VECTOR(9,10,11)) ]
[ "vmin_V1_2"      = VMIN(VPROPERTY_REF(V1),2) ]
[ "vmin_V1_2_0"    = VMIN(VPROPERTY_REF(V1),2,0) ]
[ "vmin_V1_2_V2_0" = VMIN(VPROPERTY_REF(V1),2,VPROPERTY_REF(V2),0) ]
