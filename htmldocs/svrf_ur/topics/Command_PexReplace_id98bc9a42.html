PEX Replace

Parasitic extraction

Specifies the replacement of LVS layer geometries with overlapping non-LVS layer geometries before extraction. This statement can be specified multiple times.

Usage

PEX REPLACE [INSIDE] lvs_layer layer

Arguments

lvs_layer

A required LVS layer name. The layer may be a conductor or via layer, and must exist in a Connect statement.

layer

A required non-LVS layer name. The non-LVS layer cannot exist in a Connect statement and it must be defined in an LVS DB Connectivity Layer statement.

INSIDE

An optional keyword that specifies to only replace LVS layer geometries that are completely enclosed within the non-LVS layer geometries.

Description

An optional statement that specifies to replace LVS layer geometries with overlapping non-LVS layer geometries and update the connectivity before extraction. Only geometries covered by the non-LVS layer are replaced. LVS layer geometries that are not covered by a specified non-LVS layer geometry are not replaced. If the LVS layer is a via layer, then the length and width of the LVS via layer is replaced by the length and width of the overlapping non-LVS layer polygons. This statement should only be used for coplanar layers.

This statement can be specified more than once. When multiple statements are specified for an LVS layer with different non-LVS layers, geometries in those non-LVS layers must not share any common areas (overlap).

Example 3

When PEX Replace is specified for via layers, the LVS via shapes that are overlapped by the non-LVS marker layer are completely removed and replaced by the non-LVS marker shape during extraction. Given the LVS via layer vlayer1 and non-LVS polygon vlayer2:

Figure 5. Via Layer Replacement

To replace the LVS via layer, include the following statement in your rule file:

PEX REPLACE vlayer1 vlayer2

The entire smaller rectangular via vlayer1 shape is replaced with a larger square via shape vlayer2.
