SELECT_MERGE_XXY

Vector property function for DFM expressions

Computes the nth smallest measurement values associated with coordinate intervals contained in a set of input vectors. Outputs the selected measurement values with associated interval coordinates from each of the input vectors in vector form.

Usage

SELECT_MERGE_XXY(ordinal1, vector[, ordinal2, vector …])

Arguments

ordinal1

A required positive integer that represents the ordinal-smallest measurement values to be selected across all interval endpoint coordinates from all input vector arguments. For example, 1 corresponds to the smallest measurement value, 2 corresponds to the second smallest, and so forth. An ordinalN value must be specified with each vector argument, separated by a comma. A comma must separate each “ordinalN, vector” argument pair.

vector

A required vector argument containing at least one tuple of the form <c1 c2 m> where c1, c2, and m are floating-point numbers, as follows:

  • c1 — The starting x- or y-coordinate in user units of a linear interval.

  • c2 — The ending x- or y-coordinate in user units of a linear interval.

  • m — A measurement value in user units associated with the interval.

A vector argument may be a comma-delimited list of vector tuples of the proper form. Conceptually, if “myvec = {<a b c> <d e f>}”, where “a” through “f” are numeric values, then myvec can be properly formed as a vector argument to the function.

Return Values

Vector string containing tuples of the form <c m1 m2 … mn>, where c is an interval endpoint coordinate in user units, and each mi is the ordinalN measurement value in user units selected from an input vector.

Description

This function is designed for use when multiple edge spacings are to be analyzed for a given edge. It may only be used in a DFM Property operation expression.

This function is similar to SORT_MERGE_XXY, but rather than computing the minimum measurements across all interval coordinates, SELECT_MERGE_XXY computes the nth smallest measurement values.

SELECT_MERGE_XXY analyzes input vector measurement values over all the intervals defined by the input vectors. Each vector argument tuple specifies a half-open interval [c1, c2) with an associated measurement value m. Overlapping intervals in the input arguments are permitted. For each interval, the nth smallest measurement value across all input vectors is associated with an interval endpoint coordinate.

For example, assume these input vectors:

vec1 = {<0 1 0.5> <0 1 1>}
vec2 = {<0.5 1.5 1.5> <0.5 1.5 2>}

These can be represented graphically as y-intervals with associated measurement values as shown here.

Figure 1. Vector Input Intervals to SELECT_MERGE_XXY

For illustration, these vectors can be represented in a layer operation as follows. The VECTOR function numeric values are shown as constants simply for clarity. Typically they are calculated by DFM expression functions like ECMIN, ECMAX, EW, and VECTOR.

sel_prop = DFM PROPERTY ngate_side_orient 
             ["+vec1" = CONCAT(VECTOR(0,1,0.5),VECTOR(0,1,1))]
             ["+vec2" = CONCAT(VECTOR(0.5,1.5,1.5),VECTOR(0.5,1.5,2))]
             [select_vector:1 = SELECT_MERGE_XXY(
                                1,VPROPERTY_REF("+vec1"),
                                1,VPROPERTY_REF("+vec2")
                                )
             ]

In this example, the ordinalN values are 1, so the minimum measurement values across all interval endpoint coordinates and across all vector tuples are returned.

The select_vector:1 property in a results database is this:

select_vector:1 <0 0.5 0> <0.5 0.5 1.5> <1 0 1.5> <1.5 0 0>

Recall the input vectors are these:

vec1 = {<0 1 0.5> <0 1 1>}
vec2 = {<0.5 1.5 1.5> <0.5 1.5 2>}

The explanations of the select_vector:1 tuples are as follows. Let m1 be the minimum measurement value from vec1 and m2 be the minimum measurement value from vec2.

<0 0.5 0>

At y=0, m1 is 0.5, and m2 is 0 because y=0 is not in any vec2 interval.

 
<0.5 0.5 1.5>

At y=0.5, m1 remains unchanged, and m2 is 1.5.

 
<1 0 1.5>

At y=1, vec1 is not defined (intervals are half-open), so m1 is 0. m2 is unchanged.

 
<1.5 0 0>

At y=1.5, all measurements are 0 because it is the final coordinate in any interval, and since all intervals are half-open, no interval is defined it this coordinate.

If this expression is added to the layer operation:

[select_vector:2 = SELECT_MERGE_XXY(
                   2,VPROPERTY_REF("+vec1"),
                   2,VPROPERTY_REF("+vec2")
                   )
]

the ordinalN values are 2, so the second-smallest measurement values across all interval endpoint coordinates and across all vector tuples are returned. In this case, the returned vector would be as follows, with select_vector:1 also shown (the spacing intervals are adjusted for ease of comparison):

select_vector:2 <0 1   0> <0.5 1   2>   <1 0 2>   <1.5 0 0>
select_vector:1 <0 0.5 0> <0.5 0.5 1.5> <1 0 1.5> <1.5 0 0>

Where interval coordinates are defined in the input vectors, the second-smallest measurement values are selected (shown in bold). Measurements remain 0 for coordinates that are undefined for a given vector’s intervals.

The return value is a vector of this form:

<c1 m1 ... mn> <c2 m1 ... mn> ... <ci m1 ... mn> ... <cn 0 ... 0>

where each ci is a coordinate of an interval endpoint, and each mi is the nth smallest measurement value associated with the coordinate across all input vectors. The ci values are sorted in ascending order. If no measurement value is defined for some ci, then mi is 0. An mi of 0 is always returned in the final tuple of the returned vector. The final cn value is always the maximum interval endpoint coordinate across all input vectors.

This function’s output is often used as input to RANGE_XXY to obtain the desired set of vector tuples corresponding to a defined range of coordinate values.
