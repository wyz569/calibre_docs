Expressions that Do Not Evaluate Successfully

A DFM expression that does not evaluate successfully has no value and does not satisfy any constraint. When any portion of an expression cannot be evaluated, the overall expression cannot be evaluated. For DFM Property, the primary layer object is not written to the output layer; for DFM Analyze, the capture window is not output.

There are several reasons an expression might not evaluate successfully, including the following:

  • Division by zero.

  • Evaluation failure of a per-shape function (SUM, PROD, MIN, or MAX). This can happen if the input layer contains no valid geometries or evaluation fails for all geometries, unless a default is provided. See “Per-Shape Evaluation for DFM Measurement Functions”.

  • A call to VECTOR(expression, …) when any of its argument expressions do not evaluate successfully, unless ::NOFAILARG is used.

When the DFM Property or DFM Analyze operations encounter an expression that they cannot evaluate successfully for one of these reasons, a warning is printed in the transcript, which includes a count of the number of times each failure mode was encountered.

When either DFM Property or DFM Analyze cannot successfully evaluate a property expression for a particular geometry or capture window, nothing is written to the output layer. When the property is a multilayer property in a DFM Property expression, and DFM Property cannot evaluate the expression for a single geometry on a secondary layer, the layout object on the primary layer that this geometry is clustered with is not written to the operation's results layer.

Expression Chains can be used with the DFM Property family of operations to protect against evaluation failure by providing a default value. When expression chains are used, evaluation warnings are reported only for the last expression in the chain if it does not evaluate successfully. For example, consider the following command:

R = DFM PROPERTY P S1 S2 [A = AREA(S1)/AREA(S2)] [A = AREA(P)]

If AREA(S2) evaluates to zero, causing division by zero, as long as the next expression A=AREA(P) evaluates successfully, a warning is not reported.
