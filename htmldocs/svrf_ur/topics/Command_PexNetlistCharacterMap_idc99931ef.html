PEX Netlist Character Map

Parasitic Extraction

Specifies to replace and/or remove characters in net names.

Usage

PEX NETLIST CHARACTER MAP {NONE | {“oldchar  newchar [oldchar  newchar]…” [REMOVE “char1 [char2 ...] ” ] }}

Arguments

NONE

Required keyword if character mappings are not specified. NONE disables character replacement. This is the default behavior if the statement is not in the rule file.

oldchar newchar

Required parameters that specify to substitute newchar for oldchar in net names. You can specify any number of additional pairs. Embedded spaces and unpaired characters are ignored. The entire string of pairs must be enclosed in double quotes (“”). Character mappings are not required if the REMOVE keyword set is specified.

REMOVE “char1 [char2 ...] ”

Optional parameter set that specifies to remove one or more characters from net names. Embedded spaces are ignored. The entire string of characters to be removed must be enclosed in double quotes (“”). A removed character is replaced with a space. This keyword set can be specified without character mappings.

Description

Replaces characters that are part of net names, including parasitic models. It does not affect other types of information.

Text Changed:

Text Not Changed:

Net names

Property specifications

Instance names

Separators indicating hierarchy in net names

Model names

Netlist keywords
Note:

Separator characters are not replaced. If the design has been flattened into an xcell then the name for a net that is part of the flattened hierarchy reflects the former hierarchy for that net. If you need to change the hierarchical indicator, use the SEPARATOR keyword in the PEX Netlist statement to define the character as shown in “Example 2.”

If this statement is specified along with PEX Netlist Smashed_Device Delimiter and PEX Netlist Replicated_Device Delimiter, it does not affect the device delimiters. If those statements are not specified, it does.

If this statement is specified when the PEX Netlist statement is specified with the FLATTENXCELLS keyword, then a warning is generated and the PEX Netlist Character Map statement is ignored.

Examples

Example 1

The following example replaces equal signs (=) with underscores (_):

PEX NETLIST CHARACTER MAP "=_"

Note that property specifications, such as w=1e-06, are not changed.

Example 2

The following example replaces equal signs (=) and non-hierarchical slashes (/) with underscores (_):

PEX NETLIST CHARACTER MAP "=_ /_"

The space between the first underscore and the slash is not required, but makes it easier to read.

This character map acts on the net name portions of a flattened hierarchy, but not the separator character. For example, if a net named W/R is part of a flattened hierarchy, such that its full net name is cell1/cell2/cell3/W/R, then the slashes between cells will not be replaced but the slash in W/R will. This produces cell1/cell2/cell3/W_R as the final name.

Example 3

This example turns off all character replacement:

PEX NETLIST CHARACTER MAP NONE

Example 4

The following statement removes parentheses from net names:

PEX NETLIST CHARACTER MAP REMOVE “()”

Example 5

The following statement maps “<” to “_”, and removes “>”: 

PEX NETLIST CHARACTER MAP "<_" REMOVE “>”

So for example, net names of the form bus<1> are changed to bus_1.
