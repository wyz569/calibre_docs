PEX Bias Preferred Direction

Parasitic extraction

Specifies whether the preferred or nonpreferred table is used when scaling the design in the horizontal or vertical direction during extraction.

Usage

PEX BIAS PREFERRED DIRECTION
{HORIZONTAL | VERTICAL | REFERENCE ref_layername} layername

Arguments

HORIZONTAL

Required keyword that specifies the preferred table applies to the horizontal direction.

VERTICAL

Required keyword that specifies the preferred table applies to the vertical direction.

REFERENCE ref_layername

Required keyword set that specifies the layer used to determine the preferred direction to apply. If the reference layer, ref_layername, does not have a direction specified with a PEX Bias Preferred Direction statement, then the direction is set based on the prevailing direction of the polygons on the reference layer.

layername

A required parameter that specifies the name of the layer to which the scaling is applied.

Description

Use this statement to control whether the preferred or nonpreferred table is used when scaling the design in the horizontal or vertical direction. A separate statement is required for every layer in your .mipt file that has the bias_type table parameter defined. If more than one of these statements is defined for the same layer, then extraction stops with an error message.

If your MIPT file contains both preferred and nonpreferred tables for property WIDTH, but you have not specified PEX Bias Preferred Direction in your rule file, extraction stops with the following error:

ERROR: Layer layername contains preferred and nonpreferred tables for property WIDTH, but the PEX BIAS PREFERRED DIRECTION statement is missing from the rule file.

If there are no preferred and nonpreferred tables for the given layer, but you have specified PEX Bias Preferred Direction in your rule file for that layer, extraction stops with the following error:

ERROR: PEX BIAS PREFERRED DIRECTION is set for layer layername, but there are no preferred and nonpreferred tables for this layer.

When the PEX Bias Preferred Direction statement is specified in the rule file, its setting takes precedence over the DIRECTION statement specified in the technology LEF LAYER definition.

Examples

Example 1

To set the bias direction to horizontal for layer M1, include the following SVRF statement in your rule file: 

PEX BIAS PREFERRED DIRECTION HORIZONTAL M1

Example 2

To use the same bias direction for layer M3 that was specified for layer M1 in Example 1, include the following SVRF statement in your rule file: 

PEX BIAS PREFERRED DIRECTION REFERENCE M1 M3

Example 3

Suppose the majority of the polygons on layer M2 are tall, thin polygons and rectangles, and layer M2 does not have a preferred direction specified with a PEX Bias Preferred Direction statement in the rule file. If you include the following SVRF statement in your rule file: 

PEX BIAS PREFERRED DIRECTION REFERENCE M2 M4

then the preferred direction for layer M4 is vertical.
