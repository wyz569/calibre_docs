PROPERTY

Property access function for DFM expressions.

Accesses numeric DFM properties from objects on the input layer.

Usage

PROPERTY(layer, property_name[, ordinal])

Arguments

layer

A required layer containing the layout objects to which the property is attached.

If layer is a secondary layer in a DFM Property operation and ordinal is not specified, the returned property value is summed over the clustered secondary layer objects.

property_name

A required argument specifying the name of the property of interest. The property must be a scalar (numeric) property.

ordinal

An optional argument used when layer is a secondary layer in a multilayer DFM Property operation. The ordinal specifies the index of the secondary layer geometry whose property value is to be returned. It must be a positive integer value and can itself be an expression. See the Description section for details on behavior and restrictions.

Return Values

Floating-point number.

Description

Reads numeric DFM properties from layer objects, where the properties were previously created by a DFM Property or DFM Text operation, or with Calibre YieldServer.

If the property property_name does not exist on layer, the PROPERTY function returns zero and a warning is issued in the transcript.

Restrictions

The following restrictions apply to the operation in which the PROPERTY function is used:

  • The INSIDE OF keyword may not be used with DFM Property.

  • For multilayer DFM Property operations, if the PROPERTY function references a secondary layer, the OVERLAP or NODAL keyword must be specified; in other words, the default clustering method of INTERSECTING is not allowed when reading properties from a secondary layer.

Behavior with Ordinal

The ordinal argument is used to read a property value from a specific object on a secondary layer in a multilayer DFM Property operation. The clustering method must be OVERLAP or NODAL.

Layout objects on secondary layers are numbered beginning with 1. The order in which the layout objects are processed is arbitrary, however, each secondary layer object is assigned a unique ordinal which is consistent within the same DFM Property operation.

  • For ordinal = 1 through the total number of clustered geometries on the secondary layer, the function call returns the value of the specified property on the specified secondary layer geometry clustered with the primary layer geometry.

  • Multiple function calls in the same DFM Property operation that reference the same layer have a consistent order for the secondary layer objects. For example, PROPERTY(S, “X”, 1) and VPROPERTY(S, “Y”, 1) return property values from the same geometry when used in the same DFM Property operation.

  • If the ordinal value is out of range (either greater than the number of clustered geometries or less than 1), the function fails to evaluate.

Also see “Property Access Function Summary”.

Examples

Example 1

The following example shows how to annotate gate polygons of a MOS device with the number of contact polygons in the source/drain areas of the device:

LAYER POLY 1
LAYER DIFF 2
LAYER CONT 3

SD = DIFF NOT POLY
GATE = POLY AND DIFF
SD1 = DFM PROPERTY SD CONT [ NC = COUNT(CONT) ] 
GATE1 = DFM PROPERTY GATE SD1 OVERLAP ABUT ALSO [ NC = PROPERTY(SD1, NC) ]

Example 2

This example reads a property from a secondary layer geometry and uses an expression to determine the ordinal value. If a geometry on the primary layer P overlaps two geometries on the secondary layer S, the property x is the property called “WIDTH” from the secondary geometry that has the greater value of the property called “LENGTH”.

DFM PROPERTY P S OVERLAP
    [x = PROPERTY(S,"WIDTH",  
       ( PROPERTY(S,"LENGTH",1) > PROPERTY(S,"LENGTH",2) ) ? 1 : 2 )]
