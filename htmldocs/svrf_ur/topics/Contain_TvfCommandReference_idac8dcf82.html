TVF Command Reference

Compile-time and runtime TVF commands are described in this section. A table lists the commands available in both compile-time and runtime TVF, along with a short description.
Table 1. TVF Command Summary

Command Name

Description

tvf::// comment

Specifies a single-line user-supplied comment that does not affect code execution.

Application: compile-time only.

tvf::@ comment

Specifies a single-line user-supplied comment within a rule check (a rule check comment.)

Application: compile-time only.

tvf::block_transcript

Disables the echoing of the rule file selected with the ‑E command line option and disables the transcript output produced by SVRF commands.

Application: compile-time only.

tvf::COMMENT comment

Specifies a single-line user-supplied comment within a rule check (a rule check comment). Identical to tvf::@ comment.

Application: compile-time only.

tvf::COPY layer

Identical to the SVRF Copy operation. This is the only TVF command mapped to an SVRF layer operation.

Application: compile-time only.

tvf::DELETE_LAYER $layer

Deletes the layer specified by $layer.

Application: runtime only.

tvf::do_echo_svrf string

Writes a string to the output location specified for the tvf::echo_svrf command.

tvf::echo_svrf { 0 | 1 } [file_id]

Controls echoing of the transcript SVRF created from translating TVF.

Application: compile-time and runtime.

tvf::echo_to_svrf_file string

Writes a string directly to the output SVRF file.

Application: compile-time only, pre-processor mode only.

tvf::epilog

An internal procedure that automatically cleans up temporary and deleted layers.

Application: runtime only.

tvf::exists_global_variable variable_name [(index1[, index2]…)]

Checks for the existence of a global variable.

Application: compile-time only.

tvf::get_calibre_version

Returns the Calibre version you are running.

tvf::get_global_variable variable_name

[(index1[, index2]…)]

Retrieves the value of a global TVF variable.

Application: compile-time only

tvf::get_tvf_arg

Returns the argument specified with the -tvfarg command line option.

Application: compile-time only.

tvf::GET_LAYER_ARGS layer1 [layerN …] [args]

Imports the layers passed to a TVF Function by a TVF layer operation, and sets up the layer handles. This command must be used once in a TVF Function statement that is called by a TVF layer operation.

Application: runtime only.

tvf::GET_LAYER_EXTENT $layer

Returns the bounding box of the extent of the layer as a list in the format {x1 y1 x2 y2}.

Application: runtime only.

tvf::GET_LAYER_NAME $layer

Returns the layer name for the layer handle $layer. See “Runtime TVF Example” for usage.

Application: runtime only.

tvf::insert_tvf_line_info { 1 | 0 }

Inserts a comment into the generated SVRF file with the line number of the corresponding command in the compile-time TVF file.

tvf::IS_LAYER_EMPTY $layer

Used to test for an empty layer. Returns true (1) if $layer is empty.

Application: runtime only.

tvf::is_transcript_blocked

Checks if the calibre transcript output is blocked.

Application: compile-time only.

tvf::OUTLAYER arguments

OUTLAYER provides a means of returning the results of a layer operation. The parameter arguments is a valid layer operation.

Application: runtime and compile-time.

tvf::output_to_svrf string

Writes a string to output. The string is written to the output location specified for tvf::echo_svrf and to the file specified with the -E command line option, if present.

tvf::print_call_stack { 0 | 1 }

Prints a trace of all TVF layer operation calls during runtime.

Application: runtime only.

tvf::RULECHECK { rulecheck }

Denotes a Calibre nmDRC rule check.

Application: compile-time only.

tvf::run_as_encrypted

Specifies a block of TVF to run as encrypted with the specified encryption id string.

Application: compile-time only.

tvf::set_disclaimer disclaimer

Writes a formatted header to the top of the SVRF output file and replaces the default header. This command, if used, must be the first command specified in your compile-time TVF file.

Application: compile-time, pre-processor mode only.

tvf::set_global_variable variable_name [(index1[, index2]…)] value

Sets the value of a TVF global variable given its name and optional index values.

Application: compile-time only.

tvf::set_rule_check_indentation indentation

Controls indentation of each rule check printed in the output SVRF file (the default is a tab).

Application: compile-time, pre-processor mode only.

tvf::set_traceback_line line_number

Used to trace SVRF syntax errors at line_number to line numbers in your TVF rule file.

Application: compile-time and runtime.

tvf::SETLAYER layer_name = arguments

Executes the arguments (which are layer operations) and stores the output in the layer_name handle. Used for layer derivation in both compile-time and runtime.

Application: runtime and compile-time.

Also see “Layer Definitions and Assignments”.

tvf::svrf_var variable_name

Gets the value of a variable defined with the Variable statement in the SVRF rule file. This statement gives the runtime TVF Function block access to variables defined with the Variable statement.

Application: runtime only.

tvf::sys_var variable

Accesses the value of a system variable in runtime TVF.

Application: runtime only.

tvf::unblock_transcript

Enables echoing of the rule file selected with the -E command line option and to enables the transcript output produced by SVRF commands.

Application: compile-time only.

tvf::unset_global_variable variable_name [(index1[, index2]…)]

Clears a global variable.

Application: compile-time only.

tvf::VERBATIM { verbatim_SVRF_ code }

Passes a block of SVRF commands directly to the SVRF compiler. You should use this command as much as possible in your compile-time TVF scripts to pass blocks of SVRF code that do not need to be changed to TVF syntax.

Application: compile-time only.
