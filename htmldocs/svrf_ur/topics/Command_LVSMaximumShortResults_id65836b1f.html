LVS Maximum Short Results

Specification statement

Specifies the maximum number of isolated short paths output to the shorts database.

Usage

LVS MAXIMUM SHORT RESULTS {number | ALL}

Arguments

number

Positive integer that specifies the maximum number of output short paths. The default is 500.

ALL

Keyword that specifies to output all short paths. This is the default.

Description

Specifies the maximum number of isolated short paths that are written to the shorts database. This applies to LVS Isolate Shorts YES or Mask SVDB Directory with the SI option.

You can specify this statement only once in your rule file.

Let “short” mean there is a pair of labels with different names on the same net. Let “path” mean a route that can be traced between two label locations on a net. Let “isolate” mean to trace a path for a short. Let “isolated short” mean a short for which at least one path has been traced. If the maximum path count is exceeded, then a warning like this is given:

Warning: <i> short paths from <j> out of <k> shorts were selected [in primary cell <name>] for short isolation.

where, i, j, and k are integers. If the warning applies to the primary cell, then the primary cell name is given.

A single short can have multiple paths between any number of text label pairs. In the warning message shown previously, k is a total number of detected shorts, j is a number of isolated shorts, and i is the number of independent paths that exist for the j shorts. This statement limits the total value i can take, which can implicitly limit j also.

Examples

Example 1

// Request .shorts database.
LVS ISOLATE SHORTS YES BY LAYER BY CELL
// Limit the output to 100 short paths.
LVS MAXIMUM SHORT RESULTS 100

Example 2

// Use interactive short isolation in the Query Server.
MASK SVDB DIRECTORY svdb SI
// No short path limit.
LVS MAXIMUM SHORT RESULTS ALL
