device::enclosure_perpendicular_measurements

Device TVF layer generation function. Used only in a compile-time TVF rule file.

Returns a layer containing a DFM property vector of numbers that correspond to edge enclosure measurements perpendicular to device seed shape edges. The output layer is an edge layer with DFM vector properties used to derive an auxiliary layer argument in a Device statement. This function may only be called in the context of a tvf::SETLAYER command.

Usage

device::enclosure_perpendicular_measurements ‑base layer ‑measurement layer ‑orient layer [‑max_per value] [‑help | ?]

Arguments

-base layer

A required argument set that defines the layer from which to measure the enclosure. Usually this is the gate layer of a MOS transistor. Base shapes must be rectangles with edges parallel to the database axes.

-measurement layer

A required argument set that defines the layer that encloses the ‑base layer. Usually the ‑measurement layer is a well layer.

-orient layer

A required argument that defines an orientation layer. Usually this is a source/drain layer.

-max_per value

An optional argument set that defines the maximum perpendicular-oriented search distance in user units from a -base layer edge to an enclosing -measurement layer edge. The value must evaluate to a number. This option overrides any global specification of this search distance. The default value is -1.0, which indicates the option is not set locally.

-help | ?

Prints the usage message for this function in a Tcl shell.

Return Values

Layer.

Description

Note:

The device::select_* functions are preferable to this one as they offer more flexibility in assigning DFM properties in a consistent way to various sides of a device while also giving better performance.

The device::enclosure_perpendicular_measurements function is analogous to device::enclosure_parallel_measurements and has essentially the same characteristics except for the direction in which enclosure measurements are taken and the property name. The details described under device::enclosure_parallel_measurements are generally applicable to both commands, which are typically used together. This section discusses only the main differences from device::enclosure_parallel_measurements.

This function takes similar measurements to enclosure_perpendicular.

The -max_per option may be specified in the function. Otherwise, this parameter must be specified in the global scope of the rules. To set the search distances (in user units) in the global scope, specify the following variable using your own search distance:

set device::max_per <value>

This setting applies to all instances of device::enclosure_perpendicular_measurements unless overridden by ‑max_per. The distance parameter must evaluate to a number.

The search distance must be specified explicitly in the rule file or a compile-time error results. There are no default values contained in the TVF package. You may set the search distance to 0 if the process allows it.

Edges on the device::enclosure_perpendicular_measurements output layer include DFM vector objects containing coordinate-value/enclosure-measurement pairs for the sides of a base layer polygon:

  • PER — Coordinate values taken from the gate edges coincident with a source/drain shape together with enclosure measurements taken perpendicular to the edges.

See device::enclosure_measurements for related information and diagrams regarding perpendicular enclosure properties.
