dfm_string_value

Device DFM Property data retrieval function.

Provides access to a DFM string property from a single polygon on a pin, pin layer, or auxiliary layer.

Usage

DFM_STRing_VALue(pin_or_layer, “property_name”[, pin_name])

Arguments

pin_or_layer

A required pin or layer name. A layer is derived from a DFM Property or Device Layer ANNOTATE operation. A pin must have DFM properties associated with it.

In a device property computation program, this parameter is a required pin, pin layer, or auxiliary layer name.

In a device annotation program this parameter is a name of a pin layer or an annotation auxiliary layer in a Device operation.

property_name

A required DFM property name. The property must be defined in the same layer operation used to derive the layer for the pin_or_layer argument.

pin_name

An optional pin name corresponding to a Device pin layer that intersects the pin_or_layer. This parameter only applies to the device property annotation language, not the device property computation language. The pin allows for selection from among multiple annotation geometries based on overlap of the original pin location. (Note that the pin location is represented by an arbitrary point on the original device seed/pin boundary, so the optional pin geometry must unambiguously cover all of the pin area or boundary that established the original device.)

Return Values

A string property value. The device is classified as a “BAD DEVICE” in the connectivity extraction report if the selected pin or layer includes more than one polygon.

Description

This function retrieves a single string property value by name from one polygon. This function may be used in device annotation and device property computation programs. The behavior is similar to the text_string function but accesses DFM properties rather than text object values. The commas shown in the syntax are literal.

The pin_name can be used in Device Layer ANNOTATE programs to facilitate easier access to string property values. When pin_name is used, the corresponding pin shape must intersect the pin_or_layer shape. The property value on the pin_or_layer that corresponds to the pin_name is then output by the function. To use this capability, the Device element (type and optional subtype) referenced by the Device Layer operation must be unique.

Examples

In this example, the DFM Text operation turns text objects into unmerged geometry layers having DFM properties assigned to them. Those layers are then merged and passed to a DEVice property computation program.

// input layer that contains text objects with string property values:
LAYER tlayer 126
// convert text objects into geometry with string properties
tlayer_prop = DFM TEXT tlayer PROPERTY STRING "P"
// merge the text properties onto a regular DFM PROPERTY layer
tlayer_prop_merged = DFM PROPERTY MERGE tlayer_prop
                     [ P = SPROPERTY( tlayer_prop, "P", 1 ) ]
// annotate a pin layer with the P property
pin_p = DFM PROPERTY "pin" tlayer_prop_merged OVERLAP ABUT ALSO MULTI
        [ P = SPROPERTY( tlayer_prop_merged, "P", 1 ) ]
DEV C(NA) CSEED pin_p(pos) pin_p(neg)
[
    PROPERTY pos_prop, neg_prop
    pos_prop = DFM_STRing_VALue( pos, P )
    neg_prop = DFM_STRing_VALue( neg, P )
]

An alternative to assigning the properties in a device property computation is to use the device annotation interface:

// Define a device property annotation layer
annot_layer = Device Layer C(NA) ANNOTATE pin_p
  [ 
    PROPERTY pos_prop, neg_prop
    pos_prop = DFM_STRing_VALue( pos, P )
    neg_prop = DFM_STRing_VALue( neg, P )
]
// use the Calibre connectivity interface to get the properties
Mask SVDB Directory svdb QUERY CCI
// use the device annotation layer in the Device Layer operation
LVS Annotate Devices annot_layer
