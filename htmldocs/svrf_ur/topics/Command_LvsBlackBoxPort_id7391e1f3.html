LVS Black Box Port

Specification statement

Defines port objects for the LVS Box BLACK statement.

Usage

LVS BLACK BOX PORT original_layer text_layer interconnect_layer

Arguments

original_layer

A required original layer or layer set that forms a port for a black box cell. By default, these polygons are at the primary level of the box cell.

text_layer

A required name of an original layer or layer set containing text objects that name ports for a black box cell. The text objects that name ports must intersect original_layer objects and be at the same hierarchical level as the original_layer objects in order to form box cell ports. By default, the text objects are at the primary level of the box cell. This layer must be a Text Layer argument in order for black box cell ports to be in the netlist.

interconnect_layer

A required original layer or layer set, or a derived polygon layer, containing objects that connect to the original_layer outside of the black box cell. Connectivity must be established on this layer from a Connect or Sconnect statement.

Description

Specifies that original_layer polygons belonging to an LVS Box BLACK cell, and intersecting with text_layer objects at the same hierarchical level, form cell ports. This statement may be specified any number of times. It must be specified when LVS Box BLACK is used.

Polygons of the original_layer of the black box cell are merged prior to processing. By default, polygons and text objects from the primary level of a black box cell participate in port formation. Ports originating in the sub-hierarchy of a black box cell can be recognized by specifying the appropriate LVS Black Box Port Depth statement.

Ports receive their names directly from respective text objects. Each port polygon forms a distinct net in the box cell, and the net name is the same as the port name. Polygons that comprise a port are on the same net even if they are physically disjoint. Conversely, physically disjoint polygons can be made a part of the same port by texting the polygons with the same name.

Port polygons in placements of black box cells form connections to interconnect_layer polygons outside those cells. Connections are made by polygon abutment or intersection. Connection to a port polygon of a black box cell placement establishes a connection to the underlying net within the placement. In addition, port polygons conjoin distinct polygons on the interconnect_layer that abut or intersect them in the super-hierarchy. Furthermore, port polygons form direct connections to abutting or intersecting port polygons on the same original_layer that belong to other black box cell placements.

Remember when using the Layer Map statement that the DATATYPE keyword applies to geometry layers (original_layer parameters) and the TEXTTYPE keyword applies to text layers (text_layer parameters). If original_layer and text_layer are the same and are target layers of a layer map, then the geometry and text must be mapped in separate Layer Map statements. Here is an example:

LAYER MAP 10 DATATYPE >= 0 100 // geometry is mapped separately from text
LAYER MAP 10 TEXTTYPE >= 0 100 // this is required to get text on 100
LAYER "port" 100
TEXT LAYER "port"

LVS BLACK BOX PORT "port" "port" mlayer

Examples

Example 1

This shows rule file code for setting up black box port recognition.

LAYER m2     12 // original_layer used for black box ports
...
LAYER cap    18
LAYER res    19
...
LAYER txt    50 // text_layer
TEXT LAYER txt 

m2wire = m2 NOT (cap OR res)  // interconnect_layer

CONNECT m2wire m3wire BY via2 // connectivity on interconnect_layer
ATTACH txt m2wire             // label attachment to interconnect_layer

LVS BOX BLACK ram // requires LVS BLACK BOX PORT statement
LVS BLACK BOX PORT m2 txt m2wire

MASK SVDB DIRECTORY svdb QUERY

Example 2

When black box ports are not recognized, the following things appear in the transcript:

PRE-PROCESSING FOR BLACK BOXES ...
CELL nand (BLACK BOX)
     nand (PC=0)

Notice the PC=0 line. This is the port count, and 0 means no ports were recognized for the box cell. In addition, this section of the transcript will show no listed ports:

-------------------------------------------------------------
-----        PORTS FOR BOX CELL: nand
-------------------------------------------------------------

You may also want to check the LVS Black Box Port Depth setting.
