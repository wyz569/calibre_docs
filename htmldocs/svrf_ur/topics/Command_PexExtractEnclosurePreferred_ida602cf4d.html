PEX Extract Enclosure

Parasitic extraction

Specifies the orientation of the overlap width and length of asymmetric via layer pads. The orientation controls which via_resistance or via_resistance2 table values are used during extraction.

Usage

PEX EXTRACT ENCLOSURE {DEFAULT | DXDY | PREFERRED}

Arguments

DEFAULT

Required keyword that specifies to use the shorter side of the via enclosure rectangle as the width and the longer side of the via enclosure rectangle as the length.

DXDY

Required keyword that specifies to use the x-axis as the width and the y-axis as the length.

PREFERRED

Required keyword that specifies to use the direction defined by the PEX Bias Preferred Direction statement as the length and the direction perpendicular to this as the width. If you specify this keyword and the PEX Bias Preferred Direction statement has not been specified in your rule file, then the orientation of the via’s bottom layer is used.

Description

Use this statement to control which resistance values are used from the via_resistance or via_resistance2 tables for an asymmetric via during extraction. This is a foundry predefined statement.

Examples

Example 1

To specify the shorter side of the via enclosure rectangle as the width and the longer side of the via enclosure rectangle as length (Figure 1), include the following statement in your rule file:

PEX EXTRACT ENCLOSURE DEFAULT
Figure 1. PEX Extract Enclosure DEFAULT

Example 2

To specify the via enclosure rectangle using the x-axis for width and y-axis for length (Figure 2), include the following statement in your rule file:

PEX EXTRACT ENCLOSURE DXDY
Figure 2. PEX Extract Enclosure DXDY

Example 3

If you have specified the following PEX Bias Preferred Direction statement: 

PEX BIAS PREFERRED DIRECTION HORIZONTAL M1

To specify the via enclosure rectangle using horizontal for length (Figure 3), include the following statement in your rule file:

PEX EXTRACT ENCLOSURE PREFERRED
Figure 3. PEX Extract Enclosure PREFERRED
