DFM Partition

Layer operation

License: Calibre YieldEnhancer

Creates a polygon layer that represents the input layer broken into rectangles, triangles, and trapezoids. The output shapes meet specified criteria for minimum size and separation. Jogs are removed from the input layer shapes before partitioning.

Usage

DFM PARTITION layer JOG jog_val
{GAP gap_val  | GAPX gapx_val  | GAPY gapy_val | GAPX gapx_val GAPY gapy_val}
MINSIZE minsize_val
[ORTHOGONAL ONLY | ANGLED ONLY] [CENTERED]

Arguments

layer

A required original or derived polygon layer.

JOG jog_val

A required keyword set that specifies the maximum value of a jog width to be removed from the shapes on the input layer. The jog_val is a non-negative numeric value in user units. Jogs less than or equal to jog_val are removed from the input layer shapes prior to partitioning.

A jog is defined as a polygon edge orthogonal to a database axis and having one convex endpoint and one concave endpoint. For a definition of convex and concave endpoints, see Convex Edge.

The following figure shows the effect of jog removal on the input shape. A series of jogs results in another series of jogs that are slightly inside the original polygon.



See “Example 4”.

GAP gap_val | GAPX gapx_val | GAPY gapy_val | GAPX gapx_val GAPY gapy_val

A required argument choice that specifies the minimum separation between output polygons. Gap values are positive numeric values in user units. The minimum separation of partitioned shapes is 1 dbu. You must specify one of the following gap definitions:

  • GAP gap_val — Applies gap_val in the x and y directions.

  • GAPX gapx_val — Applies gapx_val  in the x direction. Gaps are not introduced in the y direction.

  • GAPY gapy_val — Applies gapy_val  in the y direction. Gaps are not introduced in the x direction.

  • GAPX gapx_val GAPY gapy_val — Applies the specified gap values in the x and y directions, respectively.

The x and y directions are determined in the global frame of reference.

Specify only GAPX or GAPY to make cuts in only the vertical or horizontal directions, respectively. This is shown in the following figure, where a case with GAP is included for comparison.



See “Partitioning Process” for additional details.

MINSIZE minsize_val

A required keyword set that specifies the minimum width of each output polygon, where the width is defined as the minimum internal distance between edges. The minsize_val is a non-negative numeric value in user units.

ORTHOGONAL ONLY | ANGLED ONLY

An optional keyword choice limits the output based on edge orientation.

  • ORTHOGONAL ONLY — Return only rectangles with edges orthogonal to the database axes.

  • ANGLED ONLY — Return only polygons having at least one edge that is not orthogonal to a database axis.

CENTERED

An optional keyword that specifies to center the gaps between output polygons. An example is shown in the following figure. The CENTERED keyword is only applied to rectangular output shapes.

Description

Creates a derived polygon layer that represents the input layer broken into rectangles, triangles, and trapezoids. The output polygons are separated from each other as specified by the GAP, GAPX, or GAPY values, with a minimum separation of 1 dbu. The minimum size of the output polygons is determined by the MINSIZE value. Jogs in the input layer that are larger than jog_val are removed before the partitioning takes place. If node numbers are included in the input layer, they are also included in the output layer.

Shapes on the input layer are fully merged before partitioning and promoted as necessary. Partitioned shapes are returned at the same level as the merged shapes.

Partitioning Process

The partitioning process attempts to remove as little area as possible from the input polygon. The following steps summarize the process:

  1. Make cuts in the polygon to partition it into rectangles, triangles, and trapezoids, with preference given to shorter cuts. If only one of GAPX or GAPY is specified, cuts are made only in the vertical or horizontal direction, respectively, as determined in the global frame of reference.

    1. If horizontal and vertical cuts starting from the same vertex have equal lengths, use the vertical cut as determined in the local frame of reference.

  2. If the cuts chosen by the preceding rule intersect, then the polygon is cut into rectangles that can be merged into larger rectangles. In this case, the merge first creates maximum width horizontal rectangles, and then merges those results vertically.

  3. If only GAPX or GAPY is specified, and angled edges are present that force generation of triangle or trapezoid shapes with a gap in the direction orthogonal to that specified, use a separation of 1 dbu in the orthogonal direction. See “Example 3”.

  4. If any output polygons are closer together than the GAP, GAPX, or GAPY specification, make one or more output polygons smaller to obtain the required separation. This process removes the smallest area possible.

  5. Remove any output polygons that do not meet the MINSIZE requirement.

The following figure illustrates some of the concepts of the partitioning process.

Figure 2. Partitioning Process for DFM Partition

The cuts at the lettered locations illustrate these points:

  • A — A vertical and horizontal cut starting at the same vertex are the same length, so the vertical cut is chosen.

  • B — The shorter cut is chosen.

  • C — The dashed lines show intersecting cuts from the initial partitioning. The initial rectangles are merged to create larger rectangles, and therefore remove less area.

Tip

These tips can help generate the desired partitioning outcome:

  • To have cuts in only the vertical direction, specify GAPX without GAPY. To have cuts in only the horizontal direction, specify GAPY without GAPX.
  • Use the CENTERED keyword to center the cut on the polygon vertices. Without CENTERED, cuts are usually collinear with a polygon edge in order to remove the minimum possible area. See Figure 1 with the CENTERED keyword.
