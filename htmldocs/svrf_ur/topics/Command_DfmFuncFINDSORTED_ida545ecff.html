FIND_SORTED

Vector function for DFM expressions

Given a sorted numeric, string, or netID vector, finds the first match to the specified tuple search argument and returns the tuple index. Tuple indexes start at 0. Supported in the DFM Property family of operations.

Usage

FIND_SORTED(tuple_key, vector)

Arguments

tuple_key

A required argument that specifies the tuple value to find. The tuple_key is a vector with one tuple. The tuple size must match that of the input vector. (The tuple_key may be created with VTUPLE, NETVTUPLE, or SVTUPLE.)

If vector has a tuple size of 1, tuple_key may be a number, netID, or string, where the type matches that of the input vector.

vector

A required vector argument, which may be a numeric, netID, or string vector. The vector must be sorted; see SORT and SORT_UNIQUE.

If vector is a netID vector property (such as returned by NETVPROPERTY), it should be inside a PRESORTED function call to insure that the vector is sorted after possible promotion and changes in the connectivity model. See the example in the PRESORTED reference.

Return Values

Integer. Returns ‑1 if tuple_key is not found.

Note:

The return value is not defined if the input vector is not sorted. There is no runtime check to determine if the input vector is sorted.

Examples

Example 1

This example creates a vector of areas, with a tuple size of one. After applying SORT_UNIQUE to the vector, FIND_SORTED gets the index of the tuple that is equal to the first tuple in the original vector. 

layer metal1  0     
layer metal2  1   

m2_geom = DFM PROPERTY metal2 
	[ m2area = AREA(metal2)]
   
exampleD1 = DFM PROPERTY metal1 m2_geom OVERLAP
				[ areavec = VECTOR(AREA(m2_geom))]
				[ areasort_uniq = SORT_UNIQUE(VPROPERTY_REF(areavec))]
				[ area1 = PROPERTY(m2_geom, m2area, 1)] 
				[ find_area1_i = FIND_SORTED( PROPERTY_REF(area1) , 
													   VPROPERTY_REF(areasort_uniq) )]

dfm_out {
   DFM RDB exampleD1 "find.rdb"  ALL CELLS MAXIMUM ALL checkname "%_l_"
}

The following properties are attached to layer exampleD1:

areavec: <0.04> <0.02> <0.03> <0.04> <0.02>
areasort_uniq: <0.02> <0.03> <0.04>
area1: 0.04
find_area1_i: 2

Example 2

This example is similar to Example 1, but creates a vector with a tuple size of two. The tuple elements are the perimeter and area. The function VTUPLE is used to create the tuple search key. 

layer METAL1  0     
layer METAL2  1   

m2_geom = DFM PROPERTY metal2 
							[ m2area = AREA(metal2)]
							[ m2perim = PERIM(metal2)]

exampleD2 = DFM PROPERTY metal1 m2_geom OVERLAP
					[ perim_area_vec = VECTOR(PERIM(m2_geom), AREA(m2_geom))]
					[ perim_area_sort = SORT(VPROPERTY_REF(perim_area_vec))]
					[ key = VTUPLE( PROPERTY(m2_geom, m2perim, 1), 
										 PROPERTY(m2_geom, m2area, 1) )]
					[ find_perim_area_i = FIND_SORTED( VPROPERTY_REF(key), 
													 VPROPERTY_REF(perim_area_sort) )]

dfm_out {
   DFM RDB exampleD2 "find.rdb"  ALL CELLS MAXIMUM ALL checkname "%_l_"
}

The following properties are attached to layer exampleD2:

perim_area_vec: <0.8 0.04> <0.6 0.02> <0.8 0.03> <0.8 0.04> <0.6 0.02>
perim_area_sort: <0.6 0.02> <0.6 0.02> <0.8 0.03> <0.8 0.04> <0.8 0.04>
key: <0.8 0.04>
find_perim_area_i: 3
