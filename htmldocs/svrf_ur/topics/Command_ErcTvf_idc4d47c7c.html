ERC TVF

Specification statement

Generates device properties for downstream simulation tools. This statement is used in support of the pushdown separated properties (PDSP) flow in the Calibre Connectivity Interface (CCI). Requires a Calibre ADP license.

Usage

ERC TVF check_name tvf_function proc_name

Arguments

check_name

A required name for the check. This is used internally by the tool to categorize the procedures producing the output.

tvf_function

A required name of a TVF Function block.

proc_name

A required name of a Tcl procedure in the scope of the tvf_function.

Description

This statement supports generating device properties for the Calibre Connectivity Interface (CCI) PDSP flow. Commands used in ERC TVF statement Tcl procedures are documented under “ERC TVF Commands” in the Calibre Query Server Manual.

Note:

Properties generated using this statement do not appear in an extracted layout netlist such as one produced by calibre -spice. Hence, the properties are not LVS-actionable, and are only accessible through CCI.

Using this statement requires that this command appear in the scope of the tvf_function:

package require CalibreLVS_ERC_TVF_device

In the run transcript, you will see the CALIBRE::PERC module executes the check. This does not require a Calibre PERC license, however.

The erc::setup_device_parameters command, which is used in the tvf_function, has a -total option that calculates the total number of devices connected in parallel, and this count is written as a device property in CCI. There are also -globalx and -globaly options that calculate instance location properties.

See “Writing PDSP Properties in CCI” in the Calibre Query Server Manual for PDSP configuration details. See “Property Definitions and Netlist Behavior for CCI Flows” for related information about device property netlisting.

Examples

The following rule file excerpt causes properties of the name “mos_par_total” to appear in CCI output files for the PDSP flow. These properties show the numbers of MN and MP devices, respectively, having L and W parameters and that are connected in parallel.

ERC TVF erc1 erc_lib mos_params

TVF FUNCTION erc_lib [/*
  package require CalibreLVS_ERC_TVF_device

  proc mos_params {} {
    puts "    EXECUTING mos_params"
    ::erc::setup_device_parameters -devices {MN MP} \
      -total { mos_par_total { L W } } 
  }
*/]
