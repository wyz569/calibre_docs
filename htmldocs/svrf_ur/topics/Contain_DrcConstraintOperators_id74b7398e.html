DRC Constraint Comparison Functions

The DRC constraint comparison functions are DRC_EQ, DRC_GE, DRC_GT, DRC_LE, DRC_LT, and DRC_NE. These functions have a built-in tolerance of 10e‑8, making them more suitable for use when comparing geometric measurements than the exact double precision comparison operators (==, >=, >, <=, <, and !=).
Table 1. DRC Comparison Functions for DFM Expressions

Function

Description

DRC_EQ

Performs an equality (==) comparison with a built-in tolerance of 10e‑8.

DRC_GT

Performs a greater than (>) comparison with a built-in tolerance of 10e‑8.

DRC_GE

Performs a greater than or equal to (>=) comparison with a built-in tolerance of 10e‑8.

DRC_LT

Performs a less than (<) comparison with a built-in tolerance of 10e‑8.

DRC_LE

Performs a less than or equal to (<=) comparison with a built-in tolerance of 10e‑8.

DRC_NE

Performs a not equal to (!=) comparison with a built-in tolerance of 10e‑8.

In DFM property expressions, all mathematical operations and comparisons within the brackets ([]) enclosing the property definition are carried out at double precision. However, the constraints in layer operations perform an absolute and relative comparison with a tolerance of 10e‑8 in order to manage certain layout coordinate issues, such as skew edge handling and grid snapping. See “Equality Tests and Comparisons for Floating-Point Values”.

In some cases a comparison tolerance of 10e‑8 is desired within the property definition, especially when comparing geometric measurements. (See the topic referenced in the previous paragraph.) The DRC comparison functions provide a way to get the same comparison behavior in DFM expressions as that used in layer operations and in the constraints applied to the expression value.

Consider this rule check, which outputs polygons with an area less than 0.24999999999995.

// also outputs an area of 0.25
VARIABLE area_val 0.24999999999995
small_drc  {
	AREA layer4 <= area_val
}

Due to the tolerance used in layer operation constraints, the preceding rule check also outputs shapes with an area of 0.25. The is because constraints for layer operations do comparisons with an absolute or relative tolerance of 10e‑8.

However, a similar comparison performed within a property expression does not output shapes with an area of 0.25. This is because exact double precision comparison is used within the brackets defining the property definition.

// does not output an area of 0.25
VARIABLE area_val 0.24999999999995
lt_exact { 
	lt_ex = DFM PROPERTY layer4 
				[ area = ( AREA(layer4) <= area_val) ? AREA(layer4) : 0] > 0
	COPY lt_ex
}

The DRC_LE function provides a less than or equal comparison with the same behavior as the <= constraint in the preceding small_drc rule check. 

// also outputs an area of 0.25, same as small_drc check
VARIABLE area_val 0.24999999999995
lt_drc { 
	lt_drc = DFM PROPERTY layer4 
				[ area = ( DRC_LE(AREA(layer4), area_val) > 0 ) 
																? AREA(layer4) : 0] > 0
	copy lt_drc
}
Note:

The DRC comparison functions return a floating point value of 1.0 or 0.0, which cannot be used directly as the condition portion of a conditional expression. When used to form the condition for a conditional expression, the output of the DRC comparison function must be compared to another value, as in the preceding example.

The DRC comparison functions perform both an absolute and relative comparison. For the DRC_EQ(A, B) equality function, these two comparisons are made:

// absolute comparison
ABS( A - B) <= tolerance

// relative comparison
ABS(A - B)/ABS(MAX(A, B)) <= tolerance

where ABS returns the absolute value of the argument, and MAX returns the maximum value of the two arguments. If either of the comparisons is true, the DRC_EQ function returns 1.0, otherwise it returns 0.0.
