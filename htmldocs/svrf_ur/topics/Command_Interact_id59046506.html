Interact

Layer operation

Selects all polygons that share points with polygons from a second layer.

Usage

INTERACT layer1 layer2 [constraint [BY NET] [EVEN | ODD]]
[SINGULAR {ALSO | ONLY}]

Arguments

layer1

A required argument specifying an original layer or layer set, or a derived polygon layer. The operation selects polygons from this layer that meet the conditions of the operation.

layer2

A required argument specifying an original layer or layer set, or a derived polygon layer.

constraint [BY NET] [EVEN | ODD]

An optional argument, where the constraint specifies the number of layer2 polygons or nets that interact with a layer1 polygon. The constraint must be met in order for the layer1 polygon to be selected.

The constraint is one of those listed in the “Constraint Notation” column of Table 1 in the “Constraints” topic. The constraint must contain non-negative integers. The constraint == 0 does not produce any output; use Not Interact instead.

  • BY NET — Specifies that constraint applies to the number of distinct nets in the set of layer2 polygons that interact with a layer1 polygon. Connectivity is required on layer2.

  • EVEN | ODD — Specifies that a layer1 polygon is selected if the number of layer2 polygons that meet the constraint is an even or odd number, respectively. For example, the following operation selects polygons from layer1 that interact with four, six, or eight polygons from layer2:

    layer1 INTERACT layer2 >3 <9 EVEN

    It is not useful to specify these keywords if the specified constraint does not allow for the number of polygons to meet the even or odd criterion, such as with == constraints.

SINGULAR {ALSO | ONLY}

An optional keyword choice that indicates how to handle outside point-to-point and point-to-edge interactions (singular interactions). If neither of these keyword sets are used, singular interactions are ignored. May not be specified with BY NET.

  • SINGULAR ALSO — Specifies to select polygons from layer1 that intersect polygons on layer2 in any manner, including singular interactions.

    If a constraint is specified, then the number of layer2 polygons that intersect the layer1 polygon in any manner is counted.

  • SINGULAR ONLY — Specifies to select only polygons from layer1 that intersect polygons on layer2 with at least one singular interaction, regardless of any other interactions. A selected layer1 polygon may or may not have non-singular interactions, but always has at least one singular interaction.

    If a constraint is specified, then the number of layer2 polygons that intersect the layer1 polygon in singular interactions is counted; other types of interactions are not counted.

Description

Selects layer1 polygons that either share some or all of their area with a layer2 polygon, or that are outside all layer2 polygons and have a coincident outside edge or edge segment with a layer2 polygon. If either input layer is empty, there is no output.

The optional SINGULAR ALSO and SINGULAR ONLY keywords specify how to handle outside point-to-point and point-to-edge interactions, which are ignored by default. The optional constraint limits the selection of layer1 polygons according to the number of layer2 polygons with which the interaction occurs.

See also: Cut, Inside, Net Interact, Not Outside, Touch, and Coincident Outside Edge.

Example 2

This example shows uses of Interact BY NET:

layer met1 4
layer poly 5
layer cont 6
connect met1 poly by cont
// Displays met1 shapes that INTERACT with more than 1 net on poly. 
// This flags configurations I, III, and IV.
rule1 { met1 interact poly > 1  by net }
// Displays met1 polygons that INTERACT with exactly 1 net on poly. 
// This flags configuration II.
rule2 { met1 interact poly == 1 by net }
// Displays met1 shapes that INTERACT with exactly 2 nets on poly. 
// This flags configurations I and IV.
rule3 { met1 interact poly == 2 by net }
// NOTE: Using BY NET means you are counting the number of 
// nets on poly--NOT the number of polygons.
