Virtual Connect Name

Specification statement

Specifies identical net names to be virtually connected.

Usage

VIRTUAL CONNECT NAME net_name [net_name …] [CELL cell_name [cell_name …]] [CELL LIST list_name]

Arguments

net_name

A required net name. You can specify net_name any number of times in one statement. The net_name can be a string variable. The question mark (?) is a wildcard that identifies identical patterns of zero or more characters.

CELL cell_name

An optional keyword set that specifies the name of a cell to which the statement applies. At least one cell_name must be specified. By default, this statement applies to the primary cell.

A cell_name parameter may contain one or more asterisk (*) characters; the * character is a wildcard that matches zero or more characters. A string that contains a * character must be enclosed in quotation marks because the * is a reserved symbol.

CELL LIST list_name

An optional keyword set that specifies the name of a Layout Cell List. The Virtual Connect Name statement then applies to the cells in the specified list. By default, Virtual Connect Name applies to the primary cell.

Description

Specifies virtual connections for identical net names (matching is case insensitive by default). Any set of disjoint nets that share the same name become a single net, if that name appears in a Virtual Connect Name statement. This statement can be specified any number of times.

Several different net_name parameters can be listed in the same statement, but different net names are never connected together. For example:

VIRTUAL CONNECT NAME "VDD" "GND"

This statement causes virtual connections between physically disjoint nets named VDD. It also virtually connects physically disjoint nets named GND. However, it does not cause virtual connections between nets named VDD and GND.

The ? wildcard is useful for virtually connecting many identically-named nets using a single net_name parameter in a manner similar to the preceding paragraph. For example, “VDD?” virtually connects physically disjoint nets VDD to VDD and VDD1 to VDD1, but it does not virtually connect VDD to VDD1.

Name matching can be made case-sensitive with the Layout Preserve Case YES statement.

By default, virtual connections are made only in the primary cell. The hierarchical depth to which virtual connections are made is controlled by the Virtual Connect Depth statement. The CELL or CELL LIST keywords cause virtual connections to be applied only in specified cells. Virtual Connect Depth ALL must be specified in order for virtual connections to be properly made in the specified cells when these keywords are used. If both CELL and CELL LIST are specified, then the aggregate of all specified cell names is used.

If Virtual Connect Colon is also specified, then Virtual Connect Name operates on names after all colon suffixes have been removed.

When allowing virtual connections, it is recommended to specify Virtual Connect Report YES.

You can specify virtual connect behavior in Calibre Interactive; see “Specifying Virtual Net Connections in Calibre Interactive” in the Calibre Interactive User’s Manual or “Options Page in Calibre Interactive: Common Options” in the Calibre Interactive (New GUI) User’s Manual for details.

See also Virtual Connect Box Name and Virtual Connect Incremental.

Examples

Example 1

The following example specifies that any set of disjoint nets with the name VDD will become a single net. Also, any set of disjoint nets that share the name VSS will become a single net.

VIRTUAL CONNECT NAME "VDD" "VSS"
// report virtual connections
VIRTUAL CONNECT REPORT YES

Example 2

In the following example, suppose you have four disjoint net segments labeled VSS, VSS, VSS:, and VSS:X. The Virtual Connect Colon statement strips off the colon suffixes. The four nets are then virtually connected as one net named VSS.

VIRTUAL CONNECT COLON YES
VIRTUAL CONNECT NAME "VSS"
// report virtual connections
VIRTUAL CONNECT REPORT YES

Example 3

In the following example, suppose you have four disjoint net segments labeled VSS, VSS, VSS:, and VSS:X. The two net segments labeled VSS are virtually connected, but the nets labeled VSS: and VSS:X remain separate nets.

VIRTUAL CONNECT COLON NO
VIRTUAL CONNECT NAME "VSS"
// report virtual connections
VIRTUAL CONNECT REPORT YES

Example 4

The following statement connects any two nodes having labels with the same name and beginning with the characters VEE or VSS. It connects VEE1 to VEE1, VEE2 to VEE2, VSSXX to VSSXX, and so forth. Note that it will not connect different names; for example it will not connect VEE1 to VEE2.

VIRTUAL CONNECT NAME "VEE?" "VSS?"
// report virtual connections
VIRTUAL CONNECT REPORT YES

Example 5

The following statement connects together any two labels that have the same name:

VIRTUAL CONNECT NAME "?"
// report virtual connections
VIRTUAL CONNECT REPORT YES

Example 6

These statements virtually connect gnd nets only within the specified cells at their primary levels.

VIRTUAL CONNECT NAME gnd CELL logic1 memory1 power1
VIRTUAL CONNECT DEPTH ALL

This is the equivalent using LVS Cell List:

VIRTUAL CONNECT NAME gnd CELL LIST ip
LAYOUT CELL LIST ip logic1 memory1 power1
VIRTUAL CONNECT DEPTH ALL
