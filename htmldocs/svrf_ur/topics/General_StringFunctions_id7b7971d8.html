String Function Summary

String functions create or access string properties in DFM expressions.

The following table summarizes the string functions. See “Property Access Function Summary” for general information on the property access functions ending in PROPERTY and PROPERTY_REF.

Table 1. DFM Property String Functions

Function

Summary

EMPTYSTRING

Returns an empty string (“”).

MATCH

Determines whether or not a string matches a pattern and returns true or false (1 or 0).

NETNAME

Returns a string containing the net name for the specified netID.

PSNETID

Generates string properties corresponding to node IDs on input layer shapes. When an input shape is promoted, the output string contains the prefix Xm/Xn/.../<ID>, where n, m, and so forth are placement numbers of the instances in the hierarchy where the associated net shape originates.

PSPROPERTY

Returns a string type property from an object on the input layer. The string is prefixed by Xn/Xm/.../ to indicate placement hierarchy.

PSPROPERTY_REF

References string properties from a property definition in the current operation. The returned string is prefixed by Xn/Xm/.../ to indicate placement hierarchy.

SPROPERTY

Accesses string type properties from objects on the input layer.

SPROPERTY_REF

References string properties from a property definition in the current operation.

SVPROPERTY

Accesses DFM properties that are vectors of strings from objects on the input layer.

SVPROPERTY_REF

References vector string properties from a property definition in the current operation.

SVTUPLE

Creates a vector of string type properties. It is most useful in DFM Property Select Secondary, where it creates one tuple for each cluster that is grouped with the select geometry.

VSTRING

Creates a vector of string type properties.
