SELECT

Device annotation language function.

Defines seed, pin, or auxiliary layers that must be present on a device in order for an annotation to apply.

Usage

SELECT layer[, layer …]

Arguments

layer

A required name of a layer associated with a device. The layer may contain seed, pin, or auxiliary shapes. More than one layer is specified in a comma-delimited list.

Return Values

None.

Description

This statement filters devices based upon the specified layers. Devices that meet the selection layer criteria have device annotation properties assigned to them. When used, this statement must follow a DEBUG statement (if used) and precede a PROPERTY statement. Multiple SELECT statements are allowed, and they are applied collectively as a set.

Examples

Consider the following set of device statements:

DEVICE MP(p) PGATE1 PGATE1(g) SD(s) SD(d) BULK1(b) <AUX1> <AUX2>  // $D0
DEVICE MP(p) PGATE2 PGATE2(g) SD(s) SD(d) BULK1(b) <AUX1>         // $D1
DEVICE MP(p) PGATE1 PGATE1(g) SD(s) SD(d) BULK1(b)                // $D2
DEVICE MP(p) PGATE2 PGATE2(g) SD(s) SD(d) BULK1(b)                // $D3

This Device Layer operation returns shapes from all $D templates that interact with dfm_layer and transfers the DFM property P1 from dfm_layer onto the resulting shapes:

X = DEVICE LAYER mp(p) ANNOTATE dfm_layer [
    PROPERTY p1
    p1 = DFM_NUM_VAL( dfm_layer, "P1" )
]

The SELECT statement can be used to choose specific device templates that contribute to the result layer:

X = DEVICE LAYER mp(p) ANNOTATE dfm_layer [
    SELECT aux1, aux2
    PROPERTY p1
    p1 = DFM_NUM_VAL( dfm_layer, "P1" )
]

The statement “SELECT aux1, aux2” adds any device that has both layers aux1 and aux2 as a required seed, pin, or auxiliary layer. In this example, it chooses just the first ($D0) device statement because only devices classified by that statement have both aux1 and aux2 layers.

Multiple SELECT statements each contribute additional devices to the result layer:

X = DEVICE LAYER mp(p) ANNOTATE dfm_layer [
    SELECT aux1, aux2
    SELECT pgate2
    PROPERTY p1
    p1 = DFM_NUM_VAL( DFM_LAYER, "P1" )
]

The statement SELECT pgate2 adds to the original selection of $D0. It contributes seed shapes from both $D1 and $D3 because both of these templates have pgate2 in them.
