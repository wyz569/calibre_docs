PEX Xcell Extract Mode

Parasitic extraction

Specifies how to handle geometries inside xcells when calculating parasitics.

Usage

Syntax 1, for Calibre xACT and Calibre xACT 3D:

PEX XCELL EXTRACT MODE {LIGHT_GRAY | GRAY | BLACK}

Syntax 2, for the Calibre xACT digital flow:

PEX XCELL EXTRACT MODE {LIGHT_GRAY | GRAY | BLACK}

Syntax 3, for Calibre xRC:

PEX XCELL EXTRACT MODE {GRAY | BLACK}

Arguments

LIGHT_GRAY

Required keyword that specifies using the xcell pins of cells defined in the xcell file during extraction. This is the default behavior for Calibre xACT and Calibre xACT 3D.

GRAY

Required keyword that specifies using the geometries inside cells defined in the xcell file during extraction. This is the default behavior for Calibre xRC and the Calibre xACT digital flow.

BLACK

Required keyword that specifies ignoring geometries inside cells that are listed in the xcell file.

Description

This statement specifies how to account for the geometries inside xcells when calculating parasitics. Used in Calibre xRC for gate-level or LEF/DEF extraction. Used by the Calibre xACT digital flow for LEF/DEF extraction. Used by Calibre xACT and Calibre xACT 3D for high-accuracy capacitance extraction.

Use the LIGHT_GRAY keyword if you want to include the coupling capacitance between the xcell pins and outside nets in the capacitance calculations done by Calibre xACT, Calibre xACT 3D, or the Calibre xACT digital flow. The GRAY keyword grounds the capacitance to xcell pins and the BLACK keyword eliminates xcell geometries from Calibre xACT or Calibre xACT 3D extraction.

GRAY is the default setting for Calibre xRC and the Calibre xACT digital flow. It indicates that cells contain complete layout information. Use the GRAY setting if you want to include obstruction data for parasitic capacitance calculations in LEF/DEF. For more information on how to handle obstructions, see PEX LEF Extract Cell Obstructions.

Use BLACK when using the abstracted version of standard cells. Standard cell abstracts include pin locations and may also include routing obstructions. When PEX Xcell Extract Mode BLACK is used, only the top cell routing is used for parasitic extraction and internal contents of the cells in the xcell list are ignored.

When using GDS layout data in GRAY mode or using flat extraction, the resulting parasitics include the effects of the standard cell contents.

Figure 1 shows how the extraction is affected when PEX XCell Extract Mode LIGHT_GRAY is used. This is how the contents of a standard cell are seen during flat transistor-level extraction. CC1 and CC2 are parasitic coupling capacitors between metal in the CELL_X and top-level interconnect NET_A. CC1 and CC2 are extracted and netlisted.

PCDEF Extraction and XCELL Extract Mode

When an xcell is designated as a pcell with the -PCDEF option, the capacitance between the cell’s internal shapes is not extracted. In fieldsolver-based extraction, the extraction of xcells depends on the PEX XCELL Extract Mode and how much of the resistor body connects with the structures inside the cell.

In Figure 4, the resistor body NET_B only partially enters the cell and connects with a polygon. In this case more than 50% of the resistor body is located outside the cell, so the wire is considered to be outside the PCDEF cell during extraction.

Figure 4. Wire Outside PCDEF Cell

In Figure 5, most of the resistor body NET_B is located inside the cell. In this case, the wire is considered to be inside the PCDEF cell during extraction.

Figure 5. Wire Inside PCDEF Cell

Figure 6 illustrates how capacitance is extracted when -PCDEF is specified for CELL_X.

Figure 6. Cell Extraction With -PCDEF

In Figure 6 (a), when PEX Xcell Extract Mode LIGHT_GRAY is specified, NET_B is treated as if it was outside the PCDEF cell, so coupling (CC) between NET_B and the internal shape, and intrinsic capacitance (Ci) is extracted.

In Figure 6 (b), when PEX Xcell Extract Mode LIGHT_GRAY is specified and more than 50% of NET_B is inside the PCDEF cell, NET_B is treated as if it was inside the PCDEF cell and coupling (CC) is not extracted. Only intrinsic capacitance (Ci) is extracted for NET_B.

For PCDEF cells when PEX Xcell Extract Mode GRAY is specified (Figure 6 (c) and (d)), NET_B is treated as if it was entirely inside the PCDEF cell and coupling is not extracted. Only intrinsic capacitance (Ci) is extracted for NET_B.

Figure 7 illustrates how capacitance is extracted when -PCDEF -INTRINSIC is specified for CELL_X. The -INTRINSIC option can be used with -PCDEF to ignore intrinsic capacitance.

Figure 7. Cell Extraction With -PCDEF and -INTRINSIC

In Figure 7 (e), when PEX Xcell Extract Mode LIGHT_GRAY is specified, and both ‑PCDEF and -INTRINSIC options are specified for CELL_X, NET_B is treated as if it was outside the PCDEF cell, so coupling (CC) between NET_B and the internal shape, and intrinsic capacitance (Ci) is extracted.

In Figure 7 (f), when PEX Xcell Extract Mode LIGHT_GRAY is specified, and both ‑PCDEF and -INTRINSIC options are specified for CELL_X, and more than 50% of NET_B is inside the PCDEF cell, NET_B is treated as if it was inside the PCDEF cell and neither coupling (CC) or intrinsic capacitance (Ci) is extracted for NET_B.

When PEX Xcell Extract Mode GRAY is specified (Figure 7 (g) and (h)) and both -PCDEF and -INTRINSIC options are specified for CELL_X, NET_B is treated as if it was entirely inside the PCDEF cell and neither coupling or intrinsic capacitance is extracted for NET_B.

Examples

To use the BLACK setting with ASIC extraction include the following line in the rule file:

PEX XCELL EXTRACT MODE BLACK
