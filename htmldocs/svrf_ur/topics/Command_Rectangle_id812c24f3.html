Rectangle

Layer operation

Selects polygons that are rectangles on the specified layer, with optional constraints.

Usage

RECTANGLE layer [constraint1 [BY constraint2] [MEASURE EXTENTS]]
[ASPECT constraint3]
[ORTHOGONAL ONLY [FIXED]]

Arguments

layer

A required argument that specifies an original layer or layer set, or a derived polygon layer.

constraint1 [BY constraint2] [MEASURE EXTENTS]

An optional argument that specifies conditions for the edge lengths of the selected rectangle. At least one pair of edges must satisfy constraint1.

The form of constraint1 and constraint2 is listed in the “Constraint Notation” column of Table 1 in the “Constraints” topic. A constraint must contain non-negative real numbers and is interpreted in user units.

  • BY constraint2 — Specifies that one pair of edges must satisfy constraint1 and the other pair of edges must satisfy constraint2.

  • MEASURE EXTENTS — Specifies to measure the individual rectangular extents of the polygons on layer. A polygon is selected if its rectangular extent meets the constraint(s). May not be specified with ORTHOGONAL ONLY.

    See “Example 5” for an example to find polygons of any shape that fit within a specified rectangular extent.

ASPECT constraint3

An optional keyword and constraint that specifies a condition for the ratio of the longer side to the shorter side of a rectangle. A rectangle is selected if the constraint on the aspect ratio is met. The following figure shows the aspect concept:

Figure 1. Rectangle: ASPECT Keyword

ORTHOGONAL ONLY [FIXED]

An optional keyword that limits selection to rectangles with sides that are parallel (respectively) to the coordinate axes of the database.

  • FIXED — Specifies that the first constraint (constraint1) is measured along the x-axis and the second constraint (constraint2) is measured along the y-axis. Both constraints must be specified if FIXED is used.

ORTHOGONAL ONLY may not be specified with MEASURE EXTENTS.

Description

Selects all rectangles on layer if no constraint is specified. The operation can also select all layer rectangles with edge lengths conforming to the constraints. If you specify only constraint1, the selected rectangles have at least one pair of edges that satisfy the constraint. If you specify constraint1 BY constraint2, the selected rectangles have one pair of edges that meets the first constraint and the other pair of edges that meets the second constraint.

If non-orthogonal (with respect to the coordinate axes) rectangles exist on layer, you should be careful when specifying constraints in the Rectangle operation. A potential problem arises when you make assumptions about the exact size of polygons. For example, you might want to avoid using constraints such as == 5 when layer contains non-orthogonal rectangles. A better choice of constraint is > 4.995 < 5.005. This allows a tolerance of ten database units, if your Precision is 1000.

See also Not Rectangle and Enclose Rectangle.

Examples

Example 1

// Generate a layer containing all contacts that are rectangular.
rectangular_contacts = RECTANGLE contact

Example 2

Using ASPECT.

/* Select contact rectangles having a side longer than 2 and an aspect
ratio greater than or equal to 3. */
rule { RECTANGLE contact > 2 ASPECT >= 3 }

Example 3

This example shows three Rectangle operations. The operation shown at top right selects all layer1 rectangles. The operation shown at bottom left selects only the layer1 rectangles that have a side longer than 2 user units. The operation shown at bottom right selects only the layer1 rectangles that have both sides longer than 2 user units.

Figure 2. Rectangle

Example 4

Using ORTHOGONAL ONLY FIXED.

/* Select orthogonal rectangles whose dimension along the x-axis is less 
  than three user units and whose dimension along the y-axis is exactly 
  five user units. */

rule {RECTANGLE metal1 < 3 BY == 5 ORTHOGONAL ONLY FIXED}

Example 5

Use MEASURE EXTENTS to find polygons of any shape that fit within a rectangular extent of 0.1 by 0.25 user units. The long dimension of the extent may be in the x direction or the y direction.

rect_extents {RECTANGLE layer1 <= 0.1 BY <=.025 measure extents}
