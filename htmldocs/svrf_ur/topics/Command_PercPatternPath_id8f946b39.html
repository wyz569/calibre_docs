PERC Pattern Path

Specification statement

Specifies a Calibre PERC pattern file.

Usage

PERC PATTERN PATH filename

Arguments

filename

A required pathname of a SPICE pattern file.

The filename parameter can contain environment variables. See “Environment Variables in Pathname Parameters”.

Description

Specifies a Calibre PERC pattern file. Each SPICE subcircuit in the pattern file is a template that can be used for pattern matching during rule checking. The perc::get_one_pattern command accesses pattern templates from the filename.

This statement may appear at most once in the rule file. If specified, then that pattern file is available globally to all rule checks. Alternatively, you can specify PERC Load with the PATTERN keyword set for a pattern file that overrides the global file for the checks in the PERC Load statement itself. If you do not include any pattern file specification, then pattern matching is disabled.

If both PERC Pattern Path and PERC Load PATTERN are specified, then subcircuits in their respective pattern files may not call subcircuits in any other statement’s pattern file. Each pattern file is treated as self-contained.

If you use this statement with PERC Load XFORM ALL or XFORM INJECTION, then you should also set LVS Inject Logic NO and LVS Recognize Gates NONE. This is to preserve the topology of the original netlist when searching for patterns.

The pattern template names are case-sensitive if LVS Compare Case YES or TYPES is specified. The pattern device and net names are case-sensitive if LVS Compare Case YES or NAMES is specified. Layout Case YES and Source Case YES must also be specified.

Tolerances for matching netlist property values to pattern template values can be defined with the PERC Property TOLERANCE keyword.

Tip

Avoid using overly simplistic patterns. Using more specific patterns reduces the topological structures the tool must attempt to match. Whenever possible, define pattern templates where the ports have paths to supply nets.

A complete discussion of pattern matching is found under “Pattern-Based Checks” in the Calibre PERC User’s Manual.

Examples

Example 1

This shows a specification of a pattern file.

PERC PATTERN PATH "pattern.sp"

File pattern.sp contains two pattern templates containing only primitive devices. The top subcircuit is needed to instantiate the patterns.

.SUBCKT pat1 a vss
M0 a b c vss N 
M1 d b e vss N 
r0 a vss
.ENDS

.SUBCKT pat2 a b c vss
M0 a b c vss N 
M1 d b e vss N 
C0 a vss
.ENDS

.SUBCKT top        $ name must be "top"
x1 1 2 pat1        $ instantiate pattern
x2 1 2 3 4 pat2    $ instantiate pattern
.ENDS

Example 2

This pattern file shows nested subcircuit calls.

PERC PATTERN PATH "pattern.sp"

File pattern.sp (contains two pattern templates):

.SUBCKT inv in out vss vdd
M0 out in vss vss N
M1 out in vdd vdd P
.ENDS

.SUBCKT two_inv_in_series in out vss vdd
x1 in 1   vss vdd inv
x2 1  out vss vdd inv
.ENDS

.SUBCKT three_inv_in_series in out vss vdd
x1 in 1   vss vdd two_inv_in_series
x2 1  out vss vdd inv
.ENDS

.SUBCKT top                     $ name must be "top"
x1 1 2 3 4 two_inv_in_series    $ instantiate template
x2 1 2 3 4 three_inv_in_series  $ instantiate template
.ENDS

Since the subcircuit inv is not placed within top, it is not a pattern template and is only used by the other subcircuits.

Assuming MN and MP are primitive subcircuits, the pattern template two_inv_in_series is equivalent to the following flat subcircuit:

.SUBCKT two_inv_in_series in out vss vdd
x1/M0 1 in vss vss  MN
x1/M1 1 in vdd vdd  MP
x2/M0 out 1 vss vss MN
x2/M1 out 1 vdd vdd MP
.ENDS

Similarly, the pattern template three_inv_in_series is equivalent to the following flat subcircuit:

.SUBCKT three_inv_in_series in out vss vdd
x1/x1/M0 x1/1 in vss vss MN
x1/x1/M1 x1/1 in vdd vdd MP
x1/x2/M0 1 x1/1 vss vss  MN
x1/x2/M1 1 x1/1 vdd vdd  MP
x2/M0 out 1 vss vss      MN
x2/M1 out 1 vdd vdd      MP
.ENDS

Any calls to lower-level devices by perc::get_one_pattern -patternNode, for example, should contain the flattened path to the device from the top level down.

Example 3

This example compares a strict template with configurable ones. See “perc_config Device Property Format” in the Calibre PERC User’s Manual.

Strict:

** Can only match inverters of two devices in the design

.SUBCKT inv1 in out vdd vss
M0 vdd in out vdd p
M1 out in vss vss n
.ENDS

.SUBCKT top
x0 1 2 3 4 inv1
.ENDS

Configurable:

** Can match inverters in the design with a PMOS stack
.SUBCKT inv2 in out vdd vss
M0 vdd in out vdd p perc_config="series"
M1 out in vss vss n
.ENDS

.SUBCKT top 
x0 1 2 3 4 inv2
.ENDS

Configurable:

** Can match inverters in design with a PMOS stack and an NMOS stack

.SUBCKT inv3 in out vdd vss
M0 vdd in out vdd p perc_config="series"
M1 out in vss vss n perc_config="series"
.ENDS

.SUBCKT top 
x0 1 2 3 4 inv3
.ENDS
