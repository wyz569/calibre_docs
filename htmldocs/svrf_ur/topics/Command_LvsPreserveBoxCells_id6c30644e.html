LVS Preserve Box Cells

Specification statement

Specifies how LVS Box cell contents are written to an extracted layout netlist. Used only in Calibre nmLVS-H and Calibre PERC.

Usage

LVS PRESERVE BOX CELLS {NO | YES}

Arguments

NO

Keyword that instructs the tool to allow seed promotion out of LVS Box cells. Any devices that originate from seed promotion out of box cells are written to the extracted netlist. This is the default behavior. This keyword does not apply when LVS Box BLACK or LVS Box GRAY is specified. It applies when LVS Box GRAY DEVICES is specified, however.

YES

Keyword that instructs the tool to disable device formation due to seed promotion out of LVS Box cells. Such devices are not written to the extracted netlist. This keyword also causes LVS Box cells to be written as empty subcircuits in the netlist.

Description

Controls the writing of LVS Box cell contents during layout netlist extraction. This statement may be specified only once.

If NO is specified (the default), then devices can be formed by seed promotion out of box cells. Such devices are written in the extracted netlist.

If YES is specified, the content of box cells is not written to the layout netlist. Instead, box cells are written as empty subcircuits. In addition, seed promotion from box cells is disabled, so the devices which originate from box cells are not netlisted. (However, the YES keyword does not prevent certain layer derivations from promoting seed shapes from box cells. In particular, seed-to-seed interactions can still cause promotion. This is usually the result of using layer operations that do not preserve hierarchy well. See “Hierarchy Preservation by Layer Operations” in the Calibre Verification User’s Manual for additional information.)

The first effect of the YES keyword can be achieved with the LVS Netlist Box Contents NO statement as well. However, LVS Netlist Box Contents NO does not prevent netlisting of devices which originate in box cells but are recognized in higher cells due to seed promotion.

The YES keyword can be used in combination with LVS Preserve Box Ports YES to prevent Calibre nmLVS-H from modifying box cells to the greatest possible degree.

When YES is used, box cells in the netlist are annotated with several comments: CONTENTS DISCARDED and DEVICE PROMOTION DISABLED appear in the first comment line after the net and pin count. In addition, *.PBOX comment appears in every subcircuit representing a preserved box cell.

Here is an example of a box cell whose contents were discarded by this statement:

.SUBCKT inv GROUND VCC OUT
** N=4 EP=3 IP=0 FDC=2 CONTENTS DISCARDED DEVICE PROMOTION DISABLED
*.PBOX
.ENDS

LVS Preserve Box Cells YES overrides the default LVS Netlist Box Contents YES.

If LVS Preserve Box Cells YES is specified in conjunction with LVS Push Devices SEPARATE PROPERTIES YES, then LVS Clone By LVS Box Cells YES should also be specified.

Examples

This example shows how to preserve LVS Box cells in their original forms to the maximum degree possible.

// Preserve LVS BOX cells in their original forms as much as possible
LVS PRESERVE BOX CELLS YES
LVS PRESERVE BOX PORTS YES
