Not Inside Cell

Layer operation

Selects polygons from the specified layer that are not inside the specified cells. Used primarily in hierarchical Calibre applications.

Usage

NOT INSIDE CELL layer1 {{cell_name [cell_name …]} | {LIST cell_list_name}}
[PRIMARY ONLY]
[{{WITH LAYER [HIER]} | NOT WITH LAYER} layer2]
[WITH MATCH [GOLDEN]]
[CIRCLE | ANNULUS
   | {STADIUM [TOLERANCE_R value] [TOLERANCE_45 value]}]
[[NOT] WITH PROPERTY property_name [property_name …]]
[[NOT] INSIDE PLACEMENT subcell_name [subcell_name …]]
[[NOT] WITH PLACEMENT PROPERTY property_name [property_name …]]

Used primarily in hierarchical Calibre applications.

Arguments

layer1

A required argument specifying an original layer.

{cell_name [cell_name …]} | {LIST cell_list_name}

A required argument choice specifying one or more cell names or a cell name list. Polygons on layer1 that are not inside the specified cells are selected.

  • cell_name [cell_name …] — Specifies the name of one or more cells. The specified cell_name can be a string variable (see Variable) and can contain one or more asterisk (*) wildcard characters, where the * character matches zero or more characters. When using *, enclose the cell name in quotes; otherwise, a compilation error occurs because the asterisk is a reserved symbol.

    Each cell_name may use a regular expression (or regex). A leading slash (/) or exclamation point with a slash (!/) identifies a regex pattern. A regex pattern must be enclosed in quotes since slash (/) is a reserved symbol. The expression should not include a trailing slash (/) and regex flags after a trailing slash (/) are not supported. Matching is case-sensitive. Regular expressions for cell names are not supported if WITH MATCH or [NOT] INSIDE PLACEMENT is specified.

    The string CALIBRE_TOP_CELL (case-insensitive) is understood to be the top-level cell in the database.

  • LIST cell_list_name — Specifies the name of a cell list that is defined by the Layout Cell List statement. The cell_list_name is case‑sensitive and may not be a variable.

By default, a warning is issued when a cell cannot be found in the layout, but the run proceeds. You can control the severity for this situation using the Layout Input Exception Severity INSIDE_CELL exception.

PRIMARY ONLY

An optional keyword that instructs the tool to include only shapes that exist outside the top levels of cells specified by the cell_name or LIST argument.

This keyword may not be specified with [NOT] INSIDE PLACEMENT.

{{WITH LAYER [HIER]} | NOT WITH LAYER} layer2

An optional keyword choice that limits geometry selection depending on whether the specified cells have any geometry on layer2. These keywords are applied as with Inside Cell, then layer1 geometries that are not selected are used by Not Inside Cell.

  • WITH LAYER [HIER] layer2— Limits the initial selection of geometries to the specified cells that have geometry on layer2 in the immediate hierarchy (the primary level of the cell). Not Inside Cell uses the geometries that are not selected.

    The HIER option extends the initial selection of geometries to cells that contain layer2 in their immediate hierarchy or their sub-hierarchy. HIER is not supported in flat Calibre applications and is ignored.

  • NOT WITH LAYER layer2 — Limits the initial selection of geometries to the specified cells that do not have any geometry on layer2 in their immediate hierarchy (the primary level of the cell). Not Inside Cell uses the geometries that are not selected.

These keywords may not be specified with [NOT] INSIDE PLACEMENT.

WITH MATCH [GOLDEN}

An optional keyword that causes a placed cell in the input layout database hierarchy to be treated as input to the operation if and only if the placed cell geometrically matches an unplaced cell specified by the cell_name argument or in the LIST of specified cell names. The following conditions apply when using this keyword:

  • May not be specified with the [NOT] INSIDE PLACEMENT keyword.

  • May only be used in hierarchical Calibre applications.

  • May not be used in dual‑database mode.

When WITH MATCH is used, specify only the unplaced cells that you want to match. It is not necessary to specify placed cells; a placed cell is only considered if it is a geometric match to an unplaced cell that is specified. See “WITH MATCH Keyword” for more information.

  • GOLDEN — Specifies that the operation should match only cells that appear in designs specified with Layout Path or Layout Path2 statements that also use the GOLDEN keyword.

    Note:

    The GOLDEN keyword is not recommended. Siemens EDA recommends using the Calibre Pattern Matching tool or the Calibre Auto-Waivers solution.

CIRCLE | ANNULUS | {STADIUM [TOLERANCE_R value] [TOLERANCE_45 value]}

An optional keyword choice used to limit the selection of geometry by controlling the types of shapes that are selected. These keywords are applied as a filter after the Not Inside Cell operation has selected geometry based upon the other parameters.

These keywords can be particularly useful in certain photonics, through-silicon via, and packaging checks. “Finding Circular Polygons” in the Calibre Solutions for Physical Verification manual includes an example related to CIRCLE, and also demonstrates how to find approximately circular shapes with the DFM Circle Analyze operation.

  • CIRCLE — Limits the selection of geometry to polygons that are approximations of circles. A polygon that approximates a circle must meet the following criteria:

    • The number of polygon vertices is greater than 4.

    • The distance between the center of the extent of the polygon and every vertex varies by less than 1%.

    • The interior angle at each vertex varies by less than 1%.

    • The area of the polygon is greater than 75% of the area of a circle whose radius is equal to the distance between the center of the extent of the polygon and one of its vertices after the initial criteria for a circle is met.

  • ANNULUS — Limits the selection of geometry to polygons that are approximations of annuli. A polygon that approximates an annulus must meet the following criteria:

    • The polygon has exactly one hole.

    • The vertices of the polygon must satisfy the CIRCLE criteria for each of the inner and outer boundaries.

    • The inner and outer boundaries are more than 99 percent concentric. That is, there is less than a 1 percent difference between the minimum and maximum thicknesses of the annulus, where thickness is the difference between the inner and outer radii of the shape.

  • STADIUM [TOLERANCE_R value] [TOLERANCE_45 value] — Limits the selection of geometry to polygons that are approximations of a stadium (or racetrack) shape. A polygon that approximates a stadium must meet the following criteria:

    • The polygon contains two parallel edges that define an orthogonal or 45-degree rectangle. By default, a 45-degree rectangle is a rectangle that has been rotated 45 degrees with respect to the coordinate axes with a tolerance of 0.45 degrees. The TOLERANCE_45 keyword may be used to override the default.

    • The polygon contains vertices on two opposite sides of the rectangle that define approximations of acceptable semicircles. This is determined by translating the semicircle approximations such that their diameters abut, and the resulting shape is checked against the CIRCLE criteria.

    • When one semicircle approximation is translated by the rectangle delta(s) and merged with the other semicircle approximation, by default, the polygon that they form is an approximate circle, as defined by the CIRCLE argument. The TOLERANCE_R keyword may be used to override the default definition of radii variation for an approximate circle.

    The default selection criteria can be modified using these keyword sets:

    • TOLERANCE_R value — Specifies a positive number dimensioned in user units of length that modifies the circle criteria so that the allowed variation between radii must be less than or equal to value instead of the default used by the CIRCLE keyword.

    • TOLERANCE_45 value — Specifies a floating-point numeric between 0 and 1 inclusive that defines the number of degrees that the 45 degree sides may differ by and still be considered a 45 degree rectangle.

[NOT] WITH PROPERTY property_name

An optional keyword set used to select (or ignore) unmerged original geometry based on an attached OASIS property. The property_name specifies a property name in an SVRF Property statement. This keyword set may not be specified with WITH MATCH and is ignored with a warning in flat runs. This keyword set is applied as a filter after the Not Inside Cell operation has selected geometry based upon the other arguments.

WITH PROPERTY causes geometries having the property_name, and that meet the rest of the Not Inside Cell criteria, to be selected. If multiple property_name values are specified in a given keyword set, then only geometries with attached properties matching all values are selected. If more than one WITH PROPERTY keyword set is specified, then the selected geometries must have all the property values associated with exactly one of the WITH PROPERTY keyword sets.

The NOT keyword negates the behavior of WITH PROPERTY and filters out geometry having the attached property_name. If multiple property_name values are specified, then any geometry with an attached property matching at least one of the values is not selected, regardless of anything else in the rule file. The NOT WITH PROPERTY keyword set may be specified only once.

For more information, refer to “Property-Based Geometry Selection.”

[NOT] INSIDE PLACEMENT subcell_name

An optional keyword set used to specify unmerged original geometry selection by placement membership. The subcell_name is the name of a cell that occurs as a placement in the cell_name. This keyword set may be specified any number of times. It may not be specified with the WITH MATCH, [NOT] WITH LAYER, or PRIMARY ONLY keywords and is ignored with a warning in flat runs. This keyword set is applied as a filter after the Not Inside Cell operation has selected geometry based upon the other parameters.

The subcell_name parameters can contain one or more asterisk (*) wildcard characters, where the * character matches zero or more characters. When using *, enclose the cell name in quotes; otherwise, a compilation error occurs because the asterisk is a reserved symbol.

INSIDE PLACEMENT causes geometries in placements of the subcell_name, and that meet the rest of the Not Inside Cell criteria, to be promoted to the cell_name and selected. If multiple subcell_name values are specified in a given keyword set, then geometries in all placements corresponding to the subcell_name list are promoted and selected.

The NOT keyword negates the behavior of INSIDE PLACEMENT and filters out geometries in placements corresponding to all subcell_name parameters. The NOT INSIDE PLACEMENT keyword set may only be specified once.

For more information, refer to “Placement-Based Geometry Selection.”

[NOT] WITH PLACEMENT PROPERTY property_name

An optional keyword set used to select original unmerged geometry based on placement membership and an attached OASIS property. This keyword set is applied as a filter after the Not Inside Cell operation has selected geometry based upon the other parameters.

This keyword set may only be specified if [NOT] INSIDE PLACEMENT is specified.

The [NOT] WITH PLACEMENT PROPERTY keyword set has the same semantics as the [NOT] WITH PROPERTY keyword set discussed previously. The only difference is, [NOT] WITH PLACEMENT PROPERTY applies to the subcell_name cells of the [NOT] INSIDE PLACEMENT keyword set.

For more information, refer to “Placement-Based Geometry Selection.”

Description

Selects all polygons from layer1 that are not instantiated inside the cells you specify. This operation is the complement of Inside Cell, with the exception of the CIRCLE, ANNULUS, and STADIUM keywords, and the property and placement selection keyword sets, as discussed later in this section. The order of the layer1 argument and the cell_name argument or LIST keyword must be observed to avoid ambiguity.

The Not Inside Cell operation functions as follows:

  1. Finds the geometries that are selected by an Inside Cell operation with the specified layer1, cell_name or LIST, PRIMARY ONLY, [NOT] WITH LAYER, and WITH MATCH arguments.

  2. Selects the geometries that are not returned by Step 1.

  3. Applies the CIRCLE, ANNULUS, or STADIUM keyword set, if specified.

  4. Applies the [NOT] WITH PROPERTY, [NOT] INSIDE PLACEMENT, and [NOT] WITH PLACEMENT keywords, if specified.

This statement is used primarily for hierarchical Calibre applications. It can be used in flat Calibre, but its usefulness is limited.

In ICverify applications, this operation copies the input layer to the output layer and issues a warning.

WITH MATCH Keyword

The WITH MATCH keyword causes placed cells to be included as part of the Not Inside Cell calculation based upon being geometrically equal to an unplaced cell that is specified using the cell_name or LIST argument. This feature should only be used in a hierarchical run with validated cells as the specified cells.

Two cells A and B are said to be geometrically equal under the following conditions:

  1. Flatten and merge layer1 in A and its sub-hierarchy into the coordinate space of A.

  2. Flatten and merge layer1 in B and its sub-hierarchy into the coordinate space of B.

  3. Perform a Boolean XOR of the results of Steps 1 and 2. If there are no results from the XOR, then A and B are geometrically equal.

This definition does not include text objects.

The output of WITH MATCH is as follows.

NOT INSIDE CELL layer A1 A2 ... Am WITH MATCH ...

where m >= 1, is made equivalent to

NOT INSIDE CELL layer B1 B2 ... Bn

where n >= 0. Each Bj is the name of a placed cell (the name may or may not be known) that is geometrically equal to some unplaced cell A whose name matches some Ai with (possibly) wildcards. The geometry on layer1 that does not appear in all such cells Bj is output, unless other keywords filter the output.

Property-Based Geometry Selection

The [NOT] WITH PROPERTY keyword set is applied after the Not Inside Cell operation has selected geometry based upon the other parameters. That is, [NOT] WITH PROPERTY criteria are not considered initially in the Not Inside Cell operation, and are applied to the operation’s output in the same way as for Inside Cell.

The [NOT] WITH PROPERTY keyword set enables geometry selection and filtering based upon properties specified with the SVRF Property statement. Consider this example:

SVRF PROPERTY xyz SUB Key 2 "SIG=POWER"

m3_prop_top = NOT INSIDE CELL M3 "TCIP*" WITH PROPERTY xyz

The layer m3_prop_top contains all original unmerged geometry on layer M3 outside of cells matching the pattern TCIP* such that the geometry has an attached OASIS property named “Key” with a string value “SIG=POWER”. The “xyz” is a property name used only in the rule file to reference the OASIS property.

If a NOT WITH PROPERTY specification overlaps with a WITH PROPERTY specification for a given set of property_name values, the NOT WITH PROPERTY specification takes precedence.

The [NOT] WITH PROPERTY keyword set causes OASIS properties to be read automatically from the layout if the Not Inside Cell operation that contains the keywords is executed.

Placement-Based Geometry Selection

The [NOT] INSIDE PLACEMENT keyword set is applied after the Not Inside Cell operation has selected geometry based upon the other parameters. That is, [NOT] INSIDE PLACEMENT criteria are not considered initially in the Not Inside Cell operation, and are applied to the operation’s output in the same way as for Inside Cell.

The [NOT] INSIDE PLACEMENT keyword set specifies geometry selection by placement membership. Be aware that it essentially turns the Not Inside Cell operation from a fully-hierarchical operation to a one-level operation.

This keyword set works as follows:

  1. Let { C1, …, Cn } be the set of all cell_name cells in the design that are matched by the operation. Recall that the cell_name parameters may contain the asterisk “*” wildcard.

  2. For each cell C in { C1, …, Cn }, do the remainder of the algorithm.

  3. Let { P1, …, Pn } be the set of all subcell_name placements in C that are matched by the operation. Recall that the subcell_name parameters may contain the asterisk “*” wildcard.

  4. For each placement P in { P1, …, Pn } do the remainder of the algorithm.

  5. For each geometry G in P’s cell that is on the layer1, transform G up to cell C space and add G to the output of the operation (as an element in cell C). However, if any [NOT] WITH PROPERTY keyword set is specified, G’s OASIS property configuration must additionally satisfy all such keyword sets, as described under “Property-Based Geometry Selection,” to be output.

A [NOT] WITH PLACEMENT PROPERTY keyword set modifies Step 4 in the preceding algorithm as follows:

4. For each placement P in { P1, …, Pn } such that P’s OASIS property configuration satisfies the [NOT] WITH PLACEMENT PROPERTY set in exactly the same way that a geometry’s OASIS property configuration satisfies the [NOT] WITH PROPERTY set, do the remainder of the algorithm.

As with NOT WITH PROPERTY, NOT WITH PLACEMENT PROPERTY may only be specified once.

The [NOT] WITH PLACEMENT PROPERTY keyword set causes OASIS properties to be read automatically from the layout if the Inside Cell operation that contains the keywords is executed.

Examples

// select all metal polygons not inside ramcell or mem_blk
x = NOT INSIDE CELL metal ramcell mem_blk
