SVPROPERTY

Property access function for DFM expressions.

Accesses DFM properties that are vectors of strings from objects on the input layer. Supported only for DFM Property and DFM Function.

Usage

SVPROPERTY(layer, property_name[, ordinal])

Arguments

layer

A required layer containing the layout objects to which the property is attached.

property_name

A required argument defining the name of the property of interest, which must be a vector of strings property.

ordinal

An optional argument used when layer is a secondary layer in a multilayer DFM Property operation The ordinal specifies the index of the secondary layer geometry whose property value is to be returned. It must be a positive integer value and can itself be an expression. See the Description section for details on behavior and restrictions. If ordinal is not specified when layer is a secondary layer, the vector properties from all clustered secondary layer objects are concatenated. (This is the standard behavior for DFM Property with vector properties on secondary layer objects.)

Note:

When it is known that only one secondary layer object clusters with the primary layer object, it is more efficient to specify SVPROPERTY(sec_layer, prop_name, 1) than SVPROPERTY(sec_layer, prop_name). The two functions have the same effect, but the function without the ordinal copies data, which uses more memory and more processor time doing the copy.

Return Values

Vector of strings.

Description

Reads a vector of string type DFM properties from layer objects, where the property was saved to the layer object in a previous operation, such as with DFM Property or Calibre YieldServer.

If layer is a secondary layer in a DFM Property operation and ordinal is not specified, the returned vector is a concatenation of the vectors on the clustered secondary layer objects. If there are no secondary layer objects clustered with the primary layer object, an empty vector is returned.

Restrictions

The following restrictions apply to the DFM Property operation in which the SVPROPERTY function is used:

  • The INSIDE OF keyword may not be used.

  • For multilayer DFM Property operations, if the SVPROPERTY function references a secondary layer, the OVERLAP or NODAL keyword must be specified; in other words, the default clustering method of INTERSECTING is not allowed when reading properties from a secondary layer.

Behavior with Ordinal

The ordinal argument is used to read a property value from a specific object on a secondary layer in a multilayer DFM Property operation. The clustering method must be OVERLAP or NODAL.

Layout objects on secondary layers are numbered beginning with 1. The order in which the layout objects are processed is arbitrary; however, each secondary layer object is assigned a unique ordinal that is consistent within the same DFM Property operation.

  • For ordinal = 1 through the total number of clustered geometries on the secondary layer, the function call returns the value of the specified property on the specified secondary layer geometry clustered with the primary layer geometry.

  • Multiple function calls in the same DFM Property operation that reference the same layer have a consistent order for the secondary layer objects. For example, PROPERTY(S, “X”, 1) and VPROPERTY(S, “Y”, 1), when used in the same DFM Property operation, return property values from the same geometry.

  • If the ordinal value is out of range (either greater than the number of clustered geometries or less than 1), the function fails to evaluate. This includes the case in which an ordinal is specified and there are no clustered secondary layer geometries.
