tvf::run_as_encrypted

Application: compile-time

The included TVF code is run in encrypted mode with the specified encryption id string.

Usage

tvf::run_as_encrypted ‘{’
      TVF_code
      ‘}’ -id id_string

Arguments

{ TVF_code }

A required argument that specifies a block of TVF code enclosed in braces. All TVF and Tcl code in the file must be within the braces.

-id id_string

A required argument that specifies the encryption id string. If id_string contains spaces it must be enclosed in quotes.

Return Values

None.

Description

The TVF code block is run in encrypted mode without the need to encrypt the file with caltvfencrypt. Only one tvf::run_as_encrypted command can be used within a file.

This command is useful during development of an encrypted flow—unencrypted files can be used during development and debugging but still run in the encrypted mode.

See “caltvfencrypt” in the Calibre TVFencrypt User’s Manual for information on encryption and the encryption id string.

Examples

The main rule file drc_rules.tvf sources the files set_var.tvf and set_layout.tvf, both of which include the tvf::run_as_encrypted command. The id_string in set_var.tvf and set_layout.tvf must be the same or the run will fail.

drc_rules.tvf file:

#!tvf
source "./set_var.tvf"
source "./set_layout.tvf"
tvf::VERBATIM {
   DRC RESULTS DATABASE "./drc.db"
   DRC SUMMARY REPORT drc_report
   DRC CHECK TEXT ALL
   CONNECT metal1 metal2 by via MASK DIRECT

   layer metal1            0
   layer metal2            1
   layer via               237

   min_metal1_width {
       @ Minimum metal1 width = ^ext_dist
       int metal2 < ext_dist
   }

   min_spacing_metal1 {
      @ Minimum metal1 width = 2.0
      ext metal1 < 2.00 not connected 
   }
}

set_var.tvf file:

tvf::run_as_encrypted {

set LAYOUT_VAR "./layout.gds"
tvf::VARIABLE ext_dist 2.0

} -id "my_id"

set_layout.tvf file:

tvf::run_as_encrypted {

tvf::LAYOUT PATH \"$LAYOUT_VAR\"
tvf::VERBATIM {
   LAYOUT SYSTEM GDSII
   LAYOUT PRIMARY A1
}
} -id "my_id"
