PERC Report Option

Specification statement

Specifies the level of detail in the PERC Report.

Usage

PERC REPORT OPTION option [option …]

Arguments

option

A required keyword that specifies the verbosity for a particular section of the PERC Report. You may specify option any number of times in one statement, separated by spaces. The keyword names are case-insensitive. The list of keyword names is given in the following table.

Table 1. Keywords for PERC Report Option

Option

Description

ALL_NET_TYPE

Reports all net types or type sets carried by a net. The default is to only report a net’s net types and type sets specified in the rule check.

See “Reporting of Net Types by Rule Checks” in the Calibre PERC User’s Manual for more details.

BX

Lists LVS Box cells in the setup section of the report. Only the box cells that pertain to the PERC Netlist setting (LAYOUT by default) are output. Box cells from the rule file are listed regardless of whether they are in the design.

MIN_MAX_VOLTAGE_ONLY

Reports only the minimum and maximum path voltages for results on a net. For example:

1    Net  net057 [ ] [ ] [ ] [ 0 5.0 ]

The default is to report all path voltages, like this:

1    Net  net057 [ ] [ ] [ ] [ 0 1.8 3.3 5.0 ]

Printing of symbolic voltages is not affected by this option. This option is overridden by NO_NET_VOLTAGE.

NO_DEVICE_PIN

Causes rule check results to report no pin information for device and cell instances.

NO_NET_TYPE

Causes rule check results not to report net types or type sets carried by a net. This keyword has priority over ALL_NET_TYPE if both are specified.

See “Reporting of Net Types by Rule Checks” in the Calibre PERC User’s Manual for more details.

NO_NET_VOLTAGE

Causes rule check results not to report voltages carried by nets. The default is voltages are reported as needed by the rule check. Empty brackets in the report indicate a voltage was present but output was suppressed by this option:

s: VDDIO [ ]

This option takes precedence over MIN_MAX_VOLTAGE_ONLY.

PRINT_TVF_WARNING

Reports detailed runtime warning messages in the TVF FUNCTION WARNINGS section of the PERC Report. By default, only two status warnings are issued to the report and the rest of the warnings are written to the transcript.

See “Warning Messages” in the Calibre PERC User’s Manual for more information.

SKIP_PASSED_CELL

Reports only failed cells from a hierarchical run in the detailed section of the report. By default, all hcells are reported. If this option is specified, the heading of the detailed report section for failed cells changes from the standard CELL VERIFICATION RESULTS to CELL VERIFICATION RESULTS ( FAILED CELLS ONLY ).

SKIP_RESULT_PROPERTY

Causes properties specified with the perc::report_base_result -property option not to appear in the PERC Report.

SKIP_WAIVED_RESULT

Causes the RULECHECK WAIVED RESULTS section not to appear in the PERC Report for a run that uses waivers.

TRACK_RESULT_SUBTITLE

Causes tracking of rule check subtitles specified by the perc::report_base_result -title option. For each rule check, Calibre PERC computes the hierarchical and flat result count (waived result count, if applicable) for each of its subtitles, including empty ones. These counts are reported in the RULECHECK SUMMARY section, in the alphabetical order of the subtitles, with the empty subtitle always being the last. In the detailed result section of each cell, Calibre PERC lists the results in the alphabetical subtitle order for each rule check.

Description

Specifies the level of certain details shown in the PERC Report. This statement may be specified any number of times. Options are accumulated across all statements.

When PRINT_TVF_WARNING is used, only the first 50 warnings are written per initialization or rule check procedure. If there are more warnings for a given procedure, the current list in the PERC Report is truncated with this message:

<number> other warnings are skipped, see the transcript for the full list

The complete list of runtime warnings is always in the transcript.

If conflicting options appear in the rule file, a warning is issued of this form:

WARNING: On line <n> of file "<rule_name_file>" - Only 1 of <conflicting_options> may be specified.

If this warning is encountered, you should fix the conflict in the rule file.

See “Calibre PERC Reports” in the Calibre PERC User’s Manual for details on the PERC Report.

See also PERC Load, PERC Report Maximum, PERC Report Placement List Maximum, and PERC Warning Option.

Examples

Example 1

This example shows how to report only cells that have discrepancies.

PERC LOAD perc.func INIT init_1 SELECT rule.1 rule.2
PERC REPORT "perc.rep"
PERC REPORT OPTION SKIP_PASSED_CELL // only report the bad cells

Example 2

This example shows how PERC Report results are organized when the TRACK_RESULT_SUBTITLE option is used. The subtitles are specified using the perc::report_base_result -title option.

   Subtitle   0 (0)         2 (3)           check condition 3
   Subtitle   5 (8)         0 (0)           (empty)
  COMPLETED   0 (0)         0 (0)         check_2
