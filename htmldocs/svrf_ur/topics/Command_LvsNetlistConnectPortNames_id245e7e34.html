LVS Netlist Connect Port Names

Specification statement

Specifies whether the circuit extractor should connect port top-level names to nets in the top-level cell when the port names differ from any net name or port name in the cell. Used only in Calibre nmLVS-H and Calibre PERC.

Usage

LVS NETLIST CONNECT PORT NAMES {NO | YES}

Arguments

NO

Keyword that instructs the circuit extractor not to connect top-level port names to net names in the top-level cell when the port names differ from any net name in the cell. This is the default if this statement is not specified.

YES

Keyword that instructs the circuit extractor to connect top-level port names to net names in the top-level cell when the port names differ from any net name in the cell.

Description

Specifies whether the circuit extractor (as in calibre ‑spice) should connect top-level port names to top-level net names when the port names (from Port Layer Text) differ from any net name in the cell. For example, consider the following layout:

Figure 1. Net with Multiple Port Texts

When NO is specified, the layout in Figure 1 has the following extracted netlist:

.SUBCKT TOP NETA
R0 NETA NETB
.ENDS

When YES is specified, this is the extracted netlist:

.SUBCKT TOP P1 P2
*.J P1 NETA
*.J P2 NETA
R0 NETA NETB
.ENDS

The *.J statement is equivalent to *.CONNECT and connects port names P1 and P2 to NETA. This assumes P1 and P2 are different from all other net names in cell TOP.

If NETA in Figure 1 is unnamed, the port connection is handled as discussed previously, but the net retains its node ID rather than receiving a name:

.SUBCKT TOP P1 P2
*.J P1 1
*.J P2 1
R0 1 NETB
.ENDS

If a port name is the same as a top-level net name, the port is ignored and a warning is issued:

WARNING: Top level port name “<name>” on net <name> at location (x,y) already used on net <name>; ignored.

If the same port name is attached to different nets, then a warning is issued:

WARNING: Port name “<name>” on net <name> in cell “<primary>” conflicts with net: <name>. Port connection skipped for this port.

In either case, the port is not written to the netlist.

This statement may appear once in your rule file.

Examples

Assume the following layout:

Figure 2. LVS Netlist Connect Port Names YES

This is the extracted layout netlist when LVS Netlist Connect Port Names YES is specified:

.SUBCKT TOP P1
*.J P1 NETA
R0 NETA NETB
.ENDS

Port object P1 is connected to net NETA.

Port text objects P2 and NETA are ignored (with warnings) because P2 occurs on two different nets and port object NETA matches a net name in cell TOP.
