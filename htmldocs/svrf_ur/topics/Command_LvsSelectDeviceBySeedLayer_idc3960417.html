LVS Select Device By Seed Layer

Specification statement

Constrains device recognition to use Device statements having specified seed layers.

Usage

LVS SELECT DEVICE BY SEED LAYER layer [layer …]

Arguments

layer

A required name of a Device seed shape layer. More than one layer may be specified.

Description

Configures device recognition such that only Device statements having the specified seed layers are enabled for device extraction and netlisting. By default, all Device statements are enabled. More specifically, the selection process is as follows:

  • Device recognition uses all Device statements in the rules if there are no LVS Select Device By Seed Layer statements.

  • If there are LVS Select Device By Seed Layer statements, then only those Device statements having the specified seed layers are enabled.

  • If there are LVS Unselect Device By Seed Layer statements, then Device statements corresponding to the seed layers specified in LVS Unselect Device By Seed Layer statements are disabled.

When Device statements are disabled, the associated seed and pin layers may not be read or derived during circuit extraction. This can affect the hierarchical database constructor and the hierarchy of the Calibre representation of the layout.

In flat runs, this statement is recognized when -flatten is used, but not when using historical flat mode with just the -lvs option.

When LVS [Un]Select Device By Seed Layer statements are present, the run transcript shows entries like this:

SELECTED AND UNSELECTED DEVICES:
  UNSELECTED DEVICE: mn $D=0
    SELECTED DEVICE: mp $D=1

The circuit extraction report shows messages like this:

WARNING: This DEVICE template was not selected for extraction: mn $D=0

The extracted netlist (and any Calibre Connectivity Interface output netlist using the same rules) shows comments like this:

*.CALIBRE WARNING UNSELECTED DEVICE: mn $D=0

Other mechanisms that list Device statements present in the rules, like the Query Server DEVICE TABLE command and the Query Server Tcl shell or YieldServer device statement iterators, process all Device statements in the rules as usual.

LVS comparison is unaffected by the disabling of Device statements during circuit extraction, except that disabled Device statements cause certain devices not to appear in the layout netlist, which has comparison implications.

Examples

In this example, the first two Device statements are enabled for recognition, while the third is disabled:

DEVICE M(ngen) gate  gate(g) sd(s) sd(d) <ngen>
DEVICE M       gate ngate(g) sd(s) sd(d) pwell(b)
DEVICE MN     ngate ngate(g) sd(s) sd(d) pwell(b)

LVS SELECT DEVICE BY SEED LAYER gate
