Connect

Connectivity extraction

Defines electrical connections on input layers.

Usage

Syntax 1:

CONNECT layer1 [layerN …] [DIRECT | MASK | DIRECT MASK]

Syntax 2:

CONNECT layer1 layer2 [layerN …] BY layerC [DIRECT | MASK | DIRECT MASK]

Arguments

layer1

A required original layer or layer set, or a derived polygon layer. You can have a maximum of 32 layers in the Connect operation. If this layer is a derived layer, it must be defined in the global scope of the rule file.

layer2

An original layer or layer set, or a derived polygon layer. This parameter is required in Syntax 2. You can have a maximum of 32 layers in the Connect BY operation. If this layer is a derived layer, it must be defined in the global scope of the rule file.

BY layerC

A required keyword and layer for Syntax 2, specifying an original layer or layer set, or a derived polygon layer. This specifies a contact, cut, or via layer. The secondary keyword BY must precede the name of the contact layer when using this syntax. Only one BY keyword may be specified per Connect operation. If layerC is a derived layer, it must be defined in the global scope of the rule file. If additional layers are specified after layerC, they are treated as layerN parameters.

DIRECT

An optional keyword that places the operation in the Direct verification set. This keyword applies only to ICtrace.

MASK

An optional keyword that places the operation in the Mask verification set.

Note:

The combination DIRECT MASK is used by default if neither keyword is specified.

Description

A Connect operation using Syntax 1 specifies electrical connection among layer1 through layerN objects (typically shapes or paths) that abut or overlap, regardless of any other objects present on non-specified layers. For example, the layer1 and layer2 shapes shown in the following figure are all electrically connected.

Figure 1. Connect

There are no restrictions on the ordering of layer1 through layerN in a Connect operation using Syntax 1. The resulting connectivity will be the same regardless of the order chosen for layers in the list. For example, the following Connect operation is equivalent to the Connect operation shown in Figure 1.

CONNECT layer2 layer1

A Syntax 2 (Connect BY) operation specifies electrical connections between objects on layer1 and layerC, and the first available layer2 through layerN object, in that order. The layers specified as layer1 and layerC are connected when they have mutual area overlap with one of layer2 through layerN. The first object found from among layer2 through layerN at a given location participates in the connection; no other layers from that set become connected.

For example, the layer1 shape shown in the following figure is connected to the layer2 shape by the layerC shape. However, layer3 is specified after layer2, so layer3 is not connected in this scheme due to shielding by layer2.

Figure 2. Connect Operation Containing a Contact Layer

If shielding occurs, hierarchical connectivity extraction must search the entire hierarchy to determine whether any layer2 objects are present.

Objects whose connectivity cannot be unambiguously determined are promoted out of a cell and up to a level of hierarchy where connectivity can be determined. This can cause unexpected results such as the breaking of connectivity in hcells. There is also a performance penalty for using Connect BY operations that have shielding. For these reasons, you should generally avoid using Connect BY operations with more than layer1 and layer2. For further discussion of this topic, refer to the “Shielding” section of the Calibre Verification User’s Manual.

To connect all three layers in Figure 2, you can specify a Connect BY operation for each layer pair, for example:

CONNECT layer1 layer2 BY layerC
CONNECT layer1 layer3 BY layerC

The connectivity model is built from all Connect and Sconnect operations in the rule file after all the derived layers required for the Connect and Sconnect operations have been generated. The connectivity model is constructed in a single step before any operation that requires or passes connectivity. (The only exception is when DRC Incremental Connect YES is specified.) The “Node-Preserving Layer Operations” section in the Calibre Verification User’s Manual has important details about layer operations that preserve connectivity. For example:

CONNECT layer1 
layer1 = layer2 AND layer3 // this layer is generated before connectivity

Here, the AND operation is performed before connectivity is established on layer1. If a node-preserving operation derives a layer that appears in a Connect or Sconnect operation, that operation does not pass connectivity. This behavior can have unexpected side-effects if connectivity is not managed carefully. For example:

CONNECT layer1
CONNECT layer2 
layer1 = layer2 AND layer3 // nodal information not passed to layer 1

The AND operation is evaluated before connectivity is established. Node IDs are present on layer1 because of the CONNECT operation, but are not passed by the AND operation because there is no connectivity when this operation is performed (otherwise layer1 and layer2 would be shorted together). This can cause unexpected side-effects such as unattached label warnings involving connectivity layers and net names not being established in certain cases.

A good practice is to use Connect operations for a relatively small set of layers. Then you can use node-preserving operations to pass connectivity to derived layers as needed. Careful management of derived layers appearing in connectivity operations is advised.

The Connect operation also causes net names to be assigned implicitly if the text label is on the same layer as a connectivity layer. For a complete discussion of assigning net labels, see “Net Name Specification” and “Net Label Attachment” in the Calibre Verification User’s Manual.

Calibre nmDRC reports ports on layers that do not appear in a Connect operation as unattached.

Sconnect and Stamp are related connectivity operations. Sconnect provides unidirectional (one-way) transfer of connectivity information between layers. It is most often used in the detection of soft connections. Stamp is a layer operation that generates a derived layer that inherits the connectivity information of an input layer. Stamp also passes connectivity unidirectionally and is useful in situations where bidirectional passage of connectivity (such as with a Connect or Connect BY operation) is undesirable.

Requirements and Restrictions — These restrictions apply when you use original layer sets as input layers to the connectivity extraction operations:

  • You cannot use a member of an original layer set as an input layer to a Connect operation and as an input layer to another connectivity extraction operation. For example, if pwr_met is a member of the layer set all_met, and all_met appears in an Connect operation, pwr_met cannot appear in a separate Connect or connectivity extraction operation.

  • Any two, distinct, original layer sets used in one or more connectivity extraction operations in the same verification set cannot contain common layers. Which means the same layer member cannot appear in both distinct original layer sets.

Examples

Example 1

This shows a typical Connect sequence:

psd = p_diff NOT gate
nsd = n_diff NOT gate
...
/* Establish circuit connectivity to the extent required by the design rule checks.*/
CONNECT metal1 poly BY contact 
CONNECT metal1 psd BY contact
CONNECT metal1 nsd BY contact
CONNECT metal1 metal2 BY via 
...
PSD210 { EXT psd nsd < 4 NOT CONNECTED }

Example 2

This example shows a common problem with using Connect and node-preserving operations. Assume you have the following:

CONNECT layer1
CONNECT layer2 
layer1 = layer2 AND layer3 // AND is just an example. Any node-preserving
                           // layer operation suffices.

In this case, layer1 does not have the connectivity of layer2 because the AND operation has to be performed before the CONNECT operations. If the intent is to have layer1 and layer2 connected, then the CONNECT operations should be combined into one:

CONNECT layer1 layer2

Example 3

This example shows how duplicate layer definitions have the same connectivity. Assume you have the following:

m1A = COPY m1
m1B = COPY m1     // duplicate layer definition 
CONNECT m1A poly
CONNECT m1B well // poly and well are now connected!

Because Calibre is efficient about storing layers, m1A and m1B are aliases for the same layer. When you place them into Connect operations, you are connecting the same layer to other layers (poly and well in this case). This can cause unintended shorting of nets.

This is one solution:

m1A = COPY m1
m1B = COPY m1A // CONNECT applied to m1A and m1B not shorted through m1

While the Copy operation is shown here, the same holds true for any node-preserving operation. See “Node Preservation and Duplicate Layer Definitions” in the Calibre Verification User’s Manual.
