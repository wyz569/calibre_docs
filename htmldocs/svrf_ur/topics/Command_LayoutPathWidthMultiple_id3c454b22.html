Layout Path Width Multiple

Specification statement

Specifies a multiple value that is used to detect whether the full path width is an odd multiple of the new database unit after grid conversion.

Usage

LAYOUT PATH WIDTH MULTIPLE number

Arguments

number

A required positive integer in database units.

Description

Specifies a value used to detect whether the full path width is an odd multiple of the number.

Path records in the layout database are converted to polygons in the Calibre database. Then the polygons are checked to see if they are on grid. If the rule file Precision differs from the layout precision, polygons converted from paths will have their vertices on grid, but the centerline will not be if the full path width is an odd multiple of the new database unit. This situation can only be detected by checking whether the path width is an even multiple of the new database unit prior to polygon conversion. This statement causes that check to be performed.

For example, assume W is the full path width in dbu as specified in the input layout database, and D is the number parameter in the Layout Path Width Multiple statement. When the result of W/D is not an even integer, the Layout Input Exception Severity PATH_WIDTH_MULTIPLE setting determines how the condition is handled.

This statement may be specified only once and is used in conjunction with Layout Input Exception Severity PATH_WIDTH_MULTIPLE to determine how to handle a path width problem when it is detected. It is a compiler error to specify Layout Input Exception Severity PATH_WIDTH_MULTIPLE and not specify the Layout Path Width Multiple specification statement. Checking is performed on a cell-by-cell basis as the data is read from the input database. The PATH_WIDTH_MULTIPLE exception is supported by the Layout Input Exception RDB statement.

Examples

LAYOUT PRECISION 10000
PRECISION 2000 // precision is reduced
// check that path widths are not odd. error out if they are.
LAYOUT PATH WIDTH MULTIPLE 2
LAYOUT INPUT EXCEPTION SEVERITY PATH_WIDTH_MULTIPLE 4
