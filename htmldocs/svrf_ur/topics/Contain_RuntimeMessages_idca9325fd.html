Runtime Messages

This section covers runtime error and warning messages produced by Calibre nmDRC, Calibre DFM, Calibre FRACTURE, Calibre MDP, Calibre OPC, Calibre nmLVS, Calibre xACT, Calibre xRC, and ICverify applications. All rule file compilation errors are discussed earlier in this chapter.

Errors are always fatal. Exceptions can be fatal or non-fatal. Warnings are non-fatal. Notes are informational.

Some warnings generated by Calibre can create a possible data integrity issue since objects causing these warnings may be excluded from processing. The Layout Input Exception Severity statement allows you to configure the handling of exceptions. Such handling can range from quietly ignoring an exception, to specifying it as a fatal error. Layout Error On Input (which is superseded by Layout Input Exception severity) allows you to convert some exceptions into fatal errors.

Calibre nmDRC/Calibre nmDRC-H return status 1 if a fatal error occurred, else status 0.

Exceptions encountered during the loading of placement data for the CHEck DRc command are handled by IC Station, not ICrules.
