DRC Recon Remove Check By Layer

Specification statement

Selects rule checks to be omitted in a Calibre nmDRC Reconnaissance run based on original layers. A rule check is omitted if it requires any of the specified layers in the layer derivation chain in order to produce the output.

Usage

DRC RECON REMOVE CHECK BY LAYER layer_name [layer_name …]

Arguments

layer_name

A required original layer name (not a number). You can specify layer_name any number of times in one statement. Each specified layer can contain one or more wildcard (*) characters, which match zero or more characters. Layer names containing a wildcard must be enclosed in quotes; otherwise, a compilation error is generated.

Description

This statement identifies rule checks to omit in Calibre nmDRC Recon mode based upon the occurrence of the specified layer names anywhere in the rule checks’ layer derivation trees, including connectivity derivations. This statement may appear in the rule file any number of times and is ignored when ‑recon is not specified on the command line.

By default, the Calibre nmDRC Recon mode chooses a set of rule checks from the rule file, which may or may not be the complete set of checks. See “Chip-Level Verification” in the Calibre Solutions for Physical Verification manual. You can specify a custom set of checks to execute with this statement and the related statements DRC Recon Add Check, DRC Recon Remove Check, and DRC Recon Add Check By Layer.

The DRC Recon Remove Check By Layer statement works as follows: For each rule check C in the rule file, let L be the set of all original layers that are required to execute C. That is, the layers of L are 1) in the derivation tree of all output operations contained in C, or 2) in the derivation tree of required connectivity operations, if connectivity extraction is required to execute C. If any member of L is specified as a layer_name, then a “DRC Recon Remove Check C” statement is implicitly generated.

Each rule check C that is selected by the preceding process must be selected for execution in a standard Calibre nmDRC run, either by default or by use of DRC Select Check and DRC Unselect Check statements. A compiler error is issued if this is not true.

Overall check selection for a Calibre nmDRC Recon run is done by this process:

  1. Determine the checks that are selected to run in a standard Calibre nmDRC run.

  2. From the set of checks selected in Step 1, enable checks for the Calibre nmDRC Recon run:

    • If no DRC Recon Add Check or DRC Recon Add Check By Layer statements exist in the rule file, enable the default set of Calibre nmDRC Recon rule checks.

    • If DRC Recon Add Check or DRC Recon Add Check By Layer statements exist in the rule file, enable only those rule checks.

  3. If DRC Recon Remove Check or DRC Recon Remove Check By Layer statements are in the rule file, disable those rule checks.

    This step may remove rule checks added in the previous step.

Examples

Also see “Selecting Checks in Calibre nmDRC Recon” in the Calibre Solutions for Physical Verification manual.

Example 1

Assume the following rule file excerpt:

LAYER DIFF 1
LAYER POLY 9

rawpoly   = COPY POLY
gate      = rawpoly AND DIFF

POLY.1 {
@ Min width of POLY = 0.05
  INTERNAL POLY < 0.05 ABUT < 90 SINGULAR
}

POLY.2 {
@ Min spacing of gate = 0.14
  EXTERNAL gate < 0.14 ABUT < 90 SINGULAR
}

When no specification statements in the DRC Recon family are active, calibre -recon -drc -hier executes both checks. If this statement is added:

 DRC RECON REMOVE CHECK BY LAYER POLY

then neither check is executed because POLY is in the derivation tree of the layers in both checks.
