Layout System2

Specification statement

Specifies the format of a second layout database. For use only in dual-database layout versus layout (LVL) comparison. Used only in Calibre.

Usage

LAYOUT SYSTEM2 { {GDS | GDS2 | GDSII} | OASIS | LEFDEF |
{{OPENACCESS | OA} [“version”]} | MFG}

Arguments

GDS | GDS2 | GDSII

Keyword that specifies that the layout is a GDSII stream file.

OASIS

Keyword that specifies that the layout is OASIS format.

LEFDEF

Keyword that specifies that the layout is LEF/DEF format.

{OPENACCESS | OA} [“version”]

Keyword that specifies the layout is OpenAccess format. The version argument optionally specifies the version of the OA database and accepts either “22.43”, “22.60”, or “22.50”. The version number must be quoted. By default, Calibre supports version oa22.43p006. For example, to use OA version 22.50, set the version option to “22.50”.

You can also set the MGC_FDI_OA_VERSION environment variable to specify the OA version. The statement option takes precedence over the environment variable.

MFG

Keyword that specifies the layout is a MEBES or VSB12 file or job deck. This is used only by Calibre FullScale.

Description

Specifies the second layout database type for dual-database Calibre applications. If you specify Layout System2, then Layout Path2 and Layout Primary2 specification statements are also required. This statement may be specified once in your rule file.

Comparison of differing layout formats is supported. Hence the Layout System may be different from Layout System2. A Calibre nmDRC-H license is required for heterogeneous comparison runs. The supported database version designators are the same as for Layout System.

See “GDSII Layout Format” in the Calibre Verification User’s Manual for details about how Calibre treats GDSII format. See Calibre for the Open Artwork System Interchange Standard for more information about OASIS use in the Calibre toolset.

If LEFDEF or OPENACCESS is specified, and if the SVRF FDI Options statement, MGC_CALIBRE_LAYERMAP_FILE variable, or the MGC_CALIBRE_DB_READ_OPTIONS variable is set, those statement and variable settings apply to the Layout Path2 design also. See “Treatment of Third-Party Layout Databases” for details.

Dual database comparison is described under “Standard Dual Database XOR Comparison” in the Calibre Layout Comparison and Translation Guide.

See also Layout Bump2.

Examples

Example 1

//SPECIFY FIRST DATABASE
LAYOUT PATH "/tmp/ring_new.gds"
LAYOUT PRIMARY "TOP"
LAYOUT SYSTEM GDSII

//SPECIFY SECOND DATABASE
LAYOUT PATH2 "/tmp/ring.gds"
LAYOUT PRIMARY2 "TOP"
LAYOUT SYSTEM2 GDSII
LAYOUT BUMP2 200 // add 200 to layer numbers for this database

...

compare_1 { 1 XOR 201 }

compare_2 { 2 XOR 201 }

Example 2

Assume you have this in the LVL rule file:

//SPECIFY FIRST DATABASE
LAYOUT PATH "/tmp/design1.gds"
LAYOUT PRIMARY "TOP"
LAYOUT SYSTEM OPENACCESS

//SPECIFY SECOND DATABASE
LAYOUT PATH2 "/tmp/design2.gds"
LAYOUT PRIMARY2 "TOP"
LAYOUT SYSTEM2 OPENACCESS
LAYOUT BUMP2 200 // add 200 to layer numbers for this database

And suppose you have the MGC_CALIBRE_LAYERMAP_FILE set to a file that has the following contents:

metal1 drawing 11 0
m1     drawing  11 0

Then the specified layer mapping applies to both input designs because they are both OpenAccess.
