LVS Abort On Softchk

Specification statement

Specifies whether to abort processing if an LVS Softchk violation is found.

Usage

LVS ABORT ON SOFTCHK {NO | YES}

Arguments

NO

A keyword that instructs the tool not to abort processing when a soft connection violation is detected. This is the default behavior when you do not include this statement in the rule file.

YES

A keyword that instructs the tool to abort processing when a soft connection violation is detected.

Description

Specifies whether to abort LVS comparison when the tool finds a soft connection conflict. LVS comparison aborts if Sconnect conflicts are present, regardless of whether you specify LVS Softchk statements.

When YES is specified and an Sconnect conflict occurs in circuit extraction, this transcript error is issued:

ERROR: SCONNECT conflicts detected. ABORT ON SOFTCHK is specified - aborting.

The extracted layout netlist contains a comment that causes any future LVS comparison to abort when the netlist is used. Comments also appear in subcircuits where Sconnect conflicts occur.

If hyperscaling is used (-hyper cmp command line option), then the tool does not terminate until after the source netlist is read.

YES is most frequently specified when it is desirable that shorts through well regions cause LVS to abort.

You can specify soft connection management options in Calibre Interactive—nmLVS. See “Aborting on Sconnect Errors” in the Calibre Interactive User’s Manual.

See also LVS Report Option EC and S.

Examples

These statements cause LVS comparison to abort upon detection of a soft connection in a combined circuit extraction and comparison run:

SCONNECT metal1 nwell BY ntap
// Abort LVS on detection of soft connection.
LVS ABORT ON SOFTCHK YES

When SCONNECT errors occur, check the circuit extraction report.
