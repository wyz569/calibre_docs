device::scaled_dfm_vec_measurements

Device DFM Property data retrieval function.

Converts DFM-style rectilinear measurement vectors into scaled Device enclosure-vector-style width and length measurements.

Usage

device::scaled_dfm_vec_measurements object scale_factor

Arguments

object

A required dfm_vector_value object.

scale_factor

A required floating-point number that is multiplied by an input database unit value. The UNIT_LENGTH() function is frequently used as the scale factor.

Return Values

Vector of numbers.

Description

The device::scaled_dfm_vec_measurements function is used in conjunction with the TVF Functional Interface for Device Property Calculation. It must be used in a TVF Function block.

device:: Measurement Function Family

The device::scaled_dfm_vec_measurements, device::dfm_vec_measurements, device::dfm_vec_arr_measurements, and device::scaled_dfm_vec_arr_measurements functions form a family of related functions. All of their behaviors are discussed in this section for easy comparison. This family of functions provides a convenient access method for DFM Property vectors constructed using the SORT_MERGE_XXY and SELECT_MERGE_XXY functions.

The device::dfm_vec_measurements and device::scaled_dfm_vec_measurements functions take a DFM Property vector object returned from the DFM_VECtor_VALue function, and are used when there is only one polygon of interest interacting with a seed shape.

The device::dfm_vec_arr_measurements and device::scaled_dfm_vec_arr_measurements functions take a DFM Property array vector object returned from the DFM_VECtor_ARRay function, and are used when there is more than one polygon of interest interacting with a seed shape.

All of the functions in this family return a list of measurements that are similar to those provided by the historical Device enclosure operations such as enclosure_parallel and enclosure_perpendicular.

Note the DFM Property function outputs are in database units while the historical Device enclosure_* operations are in user units. Depending on which outputs you use, you may need to use scaling, which accounts for the difference between device::dfm_* and device::dfm_scaled_* functions.

Functional Details

The DFM Property SORT_MERGE_XXY and SELECT_MERGE_XXY functions return a set of coordinate vectors that outline a group of segmented measurement regions where coordinates trace the outlines of measured areas as shown in this figure.



The SORT_MERGE_XXY function provides a rectilinear measurement vector that describes this region as follows:

{ ( x1,y1 ), (x2,y2), (x3,0) }

However, the Device enclosure functions provide measurements that are a sequence of width and length measurements, such as these:

w1 = x2 - x1

w2 = x3 - x2

These measurements are then arranged in a vector that describes this region as follows:

{ (w1,y1), (w2,y2) }

Both methods are valid for describing the region, and the method you choose is simply a matter of preference.

These functions are available in a Tcl package called CalibreLVS_DEVICE_DFM. In order to access these functions, you must have the following command in your TVF FUNCTION block:

package require CalibreLVS_DEVICE_DFM
