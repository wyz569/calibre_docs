Layout Window Cell

Specification statement

Specifies a list of cell names whose placement extents are used to include input shapes and text for use in rule checks. Used only in Calibre.

Usage

LAYOUT WINDOW CELL {{cell_name [cell_name …]} | {LIST cell_list_name}}
[HALO [OVER] value]
{[ORIGINAL [OCCUPIED]] | [BY LAYER layer_name [PRIMARY]]}

Arguments

{cell_name [cell_name …]} | {LIST cell_list_name}

A required argument specifying the names of cells or a list of cells whose placement extents are used to include input shapes and text for use in rule checks.

  • cell_name [cell_name …] — Specifies the name of a cell. You can specify cell_name any number of times in one statement. The cell_name can be a string variable (see Variable).

    Each cell_name may contain one or more asterisk (*) wildcard characters, where the * character matches zero or more characters. When using the * character, be sure to enclose the cell name in quotation marks (“ ”). Otherwise, a compilation error occurs because * is a reserved symbol.

  • LIST cell_list_name — Specifies the name of a cell list that is defined by the Layout Cell List statement. The cell_list_name is case‑sensitive and may not be a variable.

HALO [OVER] value

An optional keyword that modifies the computed extent similar to how the Size operation functions. The value is a positive floating-point numeric value dimensioned in user units of length.

  • OVER — Expands the computed extent by the specified value. The default (when this secondary argument is not specified) is to shrink the computed extent by value. An extent that disappears as a result of being shrunk is ignored (treated as empty) with a warning.

{ORIGINAL [OCCUPIED]} | {BY LAYER layer_name [PRIMARY]}

An optional keyword that specifies whether to use geometries from the input layout or geometries from a specified layer to compute the cell extents.

  • ORIGINAL [OCCUPIED] — Specifies to use the geometries from the input layout to compute the extents of the specified cells.

    • OCCUPIED — Specifies that only the specified cells containing geometry (including anywhere in their subhierarchy) have their original extents returned. All other cells are ignored. The default (when this secondary argument is not specified) is to use all geometries to compute cell extents.

  • BY LAYER layer_name [PRIMARY] — Specifies to use the shapes from layer_name that are inside the specified cell(s) to form the window boundaries. The layer_name is the name of a layer (not a number). By default, the layer is flattened up from the bottom of the cell hierarchy and into the top-level cell. If the specified layer_name is empty in a given cell, then a warning is issued, and no window is applied to that cell.

    • PRIMARY — Specifies that only the layer shapes in the primary level of a cell are used to compute window boundaries.

Description

Specifies to include shapes and text objects that occur in windows formed for each cell_name during a Calibre run. This statement functions essentially like Layout Window, but rather than providing coordinates, you provide cell names. For each cell C, the extents of all placements of C in the input layout database from the flat viewpoint are scanned. The coordinates of each extent are then used in an implicit Layout Window statement.

Calibre warns for each cell_name argument that is not located in the input layout database.

By default, the cell extents used by these statements are the extents of only those objects (text and geometry) actually required by the run. Calibre applications completely ignore objects in the input layout database that are not actually required by the flow. For example, if the input layout database contains objects on layer 53, but layer 53 is nowhere in a rule check or is not required for connectivity in the current run, then this layer is ignored as the input layout database is processed. Being ignored implies that objects on layer 53 are not used to compute the extents of individual cells.

If ORIGINAL is specified, then all objects in the input layout database are used to compute the specified cell extent(s), not just those objects actually required by the flow. If OCCUPIED is additionally specified, then all objects in the input layout database are used to compute the specified cell extent(s), if there is at least one object required by the flow in the cell or its subhierarchy; however, if a specified cell does not actually contain objects that are required by the flow (either in the cell or its sub-hierarchy), then the extent returned by this operation for that cell is empty (that is, as if ORIGINAL was not specified).

The HALO option is used to adjust the sizes of the window boundaries. The BY LAYER keyword basically functions like Layout Window Layer but on a per-cell basis.

See also Layout Windel Cell and Layout Window Clip.

Using the ORIGINAL keyword:

use all objects in input database for computing extents */
