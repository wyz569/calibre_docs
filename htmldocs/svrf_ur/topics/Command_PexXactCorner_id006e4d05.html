PEX XACT Corner

Parasitic extraction

Specifies the name of a process corner for multi-corner processing.

Usage

PEX XACT CORNER corner_name
[DP dp_corner_name1 [dp_corner_name2] [dp_corner_name3]]
[TEMPERATURE temperature [temperature...]]

Arguments

corner_name

A required parameter that specifies the name of a process corner.

DP dp_corner_name1 [dp_corner_name2] [dp_corner_name3]

An optional keyword and parameter set used to specify which double patterning process corners are netlisted. You can specify up to three double patterning corner names.

TEMPERATURE temperature [temperature...]

An optional keyword and parameter set used to define specific process corner - temperature combinations. You can specify one or more temperature values separated by spaces. The default value for temperature is 25 degrees Celsius if neither this keyword or PEX Extract Temperature statement are specified.

Description

An optional statement that specifies the name of a process corner. Use this statement with the TEMPERATURE keyword or with PEX Extract Temperature for multi-corner extraction.

If the TEMPERATURE keyword is not specified, PEX Extract Temperature statement may be used to provide the default temperature values. If PEX Extract Temperature is not specified, then the default nominal temperature value 25 degrees Celsius is used.

If the DP keyword is not specified, then all DP corners in the rule deck are netlisted. If more than three dp_corner_names are specified, then Calibre xACT truncates the list of DP corner names and generates a warning. If a dp_corner_name is specified that is not also specified by PEX Netlist Corner All, then Calibre xACT generates an error and stops.

If PEX Netlist … SPEF MULTIVALUES is specified, then for each specified process-temperature combination the values for each DP corner are separated by colons and written to a single netlist. If the DP keyword is specified for fewer than three DP corners, then the values for the last dp_corner_name specified are repeated in the generated netlist so that the SPEF netlist always uses three values.

For more information on multi-corner multi-temperature extraction see “Netlisting Multiple Corners and Multiple Temperatures” in the Calibre xACT User’s Manual.

Examples

Example 1

Include the following statements to extract a combination of multiple process and temperature corners: 

PEX EXTRACT TEMPERATURE 27 120
PEX XACT CORNER typical
PEX XACT CORNER min
PEX XACT CORNER max
#IFDEF $XACT_SOC_CORNER
	INCLUDE "rules/$XACT_SOC_CORNER/rules.C"
	INCLUDE "rules/$XACT_SOC_CORNER/rules.R"
#ELSE
	INCLUDE "rules/typical/rules.C"
	INCLUDE "rules/typical/rules.R"
#ENDIF

These statements create six netlists, one for each combination of the three process corners with the two PEX Extract Temperature corners. Note that the variable $XACT_SOC_CORNER is set internally by Calibre and in each case carries the value of one of the process corners. The Include statements following the #ELSE statement are required to make this SVRF code example compile in Calibre.

Example 2

Including the following statements: 

PEX XACT CORNER typical TEMPERATURE 140 20
PEX XACT CORNER rcworst TEMPERATURE 0

creates the following three netlists:

  • Process corner “typical” at temperature 140 degrees Celsius

  • Process corner “typical” at temperature 20 degrees Celsius

  • Process corner “rcworst” at temperature 0 degrees Celsius

Example 3

Temperatures specified with the TEMPERATURE keyword override those specified by the PEX Extract Temperature statement. Specifying both of the following statements: 

PEX EXTRACT TEMPERATURE 140 40
PEX XACT CORNER typical TEMPERATURE 20

creates one netlist using the process corner “typical” at temperature 20.

Example 4

Including the following statement: 

PEX XACT CORNER typical DP CCbest typical TEMPERATURE 20 85

creates a netlist for the process corner typical with DP corners CCbest and typical. A netlist is created for temperatures 20 and 85.
