DFM RDB ASCII

Layer operation

License: Calibre YieldEnhancer

Writes original or derived layer data to a custom-formatted text file.

Usage

DFM RDB ASCII OUTPUT FILE filename
[HIGHPREC] [TOP]
FORMAT SPECS
output_spec ‘ ;’
[{output_spec ‘ ;’} …]
END FORMAT SPECS

where output_spec is:

      [PRIORITY {{cells layers} | {layers cells}}]
[DBU]
{{FORMAT FILE "format_file"} |
   {FORMAT ‘[’
                         format_spec
                        ‘]’ }}
[layer1 [layer2 …]]

Arguments

OUTPUT FILE filename

A required keyword set, where filename specifies the name of the output file.

HIGHPREC

An optional keyword that causes numeric properties to print with twelve digits of precision. By default, six digits of precision are printed.

TOP

An optional keyword that causes coordinates to be reported at the top level.

Coordinates are reported in cell space by default.

FORMAT SPECS

A required keyword that is placed before the first output_spec.

output_spec ‘ ;’

A required output specification. Multiple output specifications may be supplied. A semicolon ( ; ) must follow each output_spec, and a space is required before the semicolon.

END FORMAT SPECS

A required keyword that is placed after the last of the output_spec.

Arguments for output_spec

PRIORITY {cells layers | layers cells}

An optional keyword set that specifies the order in which to process hierarchical input.

  • cells layers — Process cells first, then layers. This is the default if PRIORITY is not specified.

  • layers cells — Process layers first, then cells.

DBU

An optional keyword that specifies to report coordinates in database units (DBU). The default unit is user units.

{FORMAT FILE “format_file”} | {FORMAT ‘[’ format_spec ‘]’}

A required argument choice that provides the formatting specifications.

See Table 1 for the format strings that are supported in the format_spec formatting specification.

You must provide one of the following argument sets:

  • FORMAT FILE “format_file”

    Specifies that the text file format_file contains the formatting specifications for the output. The format_file has this syntax:

    [OUTPUT FILE out_file]
FORMAT ‘[’
         format_spec 
        ‘]’}

    If OUTPUT FILE is included in the format_file, the output from format_spec  is written to out_file rather than filename. If FORMAT FILE is specified multiple times, each format_file can write to the same out_file, or to a different file.

    See “Example 2”.

  • FORMAT ‘[’ format_spec ‘]’

    Specifies an inline formatting specification for the output. As shown in the full usage line, the opening bracket ( [ ) must be the last character in the line that is not whitespace or comment. The closing bracket ( ] ) must be the first character in the line that is not whitespace.

    See “Example 1”.

layer1 [layer2 …]

One or more layers to process, where the layers must be original or derived polygon layers. At least one output_spec must specify a layer to process.

Description

DFM RDB ASCII produces custom-formatted output. You can create files in a format that can be used by other programs or analysis tools. For example, you can generate a fill layer with Calibre and write the layer data in a format that can be read by a place and route tool. The operation must be within a rule check and the Calibre run must be hierarchical (the ‑hier command line argument is used).

The operation processes all shapes on the specified layers and prints output to a file based on the formatting specification. Formatting strings generate output based on layer data. For example, %1 is replaced by the cell name, and %4 is replaced by a space separated list of polygon coordinates. This is a simple output_spec with an inline format specification: 

PRIORITY cells layers  // process cells, then layers (the default)
FORMAT [               // only comments may be after the opening bracket
cell: %1 shape: %4     // format specification, leading spaces are printed
]                      // end bracket must be the first character on line
poly ;                 // process poly layer  
                       // must have a space before the semicolon (;)

For a poly layer with two shapes in cell ABC, the following output is generated:

cell: ABC shape: 15 1 16 1 16 2 15 2
cell: ABC shape: -2 -1 -1 -1 -1 5 -2 5

The formatting specification can be placed in a file rather than provided inline. This makes it easy to reuse a format specification. A format file can also specify that its output is written to a different output file. The format file is specified with “FORMAT FILE format_file”. See the “Examples” section.

If the rule file contains multiple DFM RDB ASCII operations, each operation must write to a different output file.

The output of DFM RDB ASCII is affected by the DRC Magnify Results and DRC Results Database Precision statements. Coordinates are reported in cell space unless the TOP keyword is specified.

Examples

Example 1

The layout has two layers. The DFM Property operation is used to calculate the area of shapes on layer1. Three format specifications are included, one of which includes the property output. The final format specification prints a comment but does not process any layer data.

Most format specifications are left-justified, as leading white spaces are included in the output. 

// custom rdb
LAYOUT PATH "layout.oas"
LAYOUT SYSTEM OASIS 
LAYOUT PRIMARY TOPCELL

DRC RESULTS DATABASE drc.db
DRC SUMMARY REPORT "./drc.summary"
LAYER layer1 1
LAYER layer2 2
area_1 = DFM PROPERTY layer1 
      [area_prop = AREA(layer1)] 
A_out {
DFM RDB ASCII OUTPUT FILE "example1_out.txt"
FORMAT SPECS
      
PRIORITY cells layers                           // first output_spec
FORMAT [
#                                               
shape %5                                        // print area_prop 
   on layer %2 in cell %1 has area %(area_prop) // property values 
]
area_1 ;
PRIORITY cells layers                              // second output_spec
FORMAT [
# 
layer %2 in cell %1: (%e1) (%e2) (%e3) (%e4)
] layer2 ;

FORMAT [                                          // third output_spec
# Example end 
] ;   

END FORMAT SPECS
}

The following output is produced for a layout with a total of four shapes:

# 
shape {0 3} {1 3} {1 9} {0 9} 
   on layer area_1 in cell TOPCELL has area 6
# 
shape {8.6 5.2} {10 5.2} {10 3} {11 3} {11 9} {10 9} {10 7.7} {8.6 7.7} 
   on layer area_1 in cell TOPCELL has area 9.5
# 
layer layer2 in cell TOPCELL: (2 1) (13 1) (13 4) (2 4)
# 
layer layer2 in cell CellA: (1.8 -1) (2.9 -1) (2.9 5) (1.8 5)
# Example end

Example 2

To use a format file for the first format specification in the preceding example, the following format file named format1 is created:

FORMAT [
# 
shape %5 
   on layer %2 in cell %1 has area %(area_prop)
]

The format file is then referenced as follows:

PRIORITY cells layers
FORMAT FILE "format1" 
area_1 ;
