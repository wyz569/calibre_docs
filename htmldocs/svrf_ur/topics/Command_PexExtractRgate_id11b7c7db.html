PEX Extract Rgate

Parasitic extraction

Specifies how to calculate the gate resistance of a transistor.

Usage

Syntax 1, for Calibre xACT and Calibre xACT 3D:

PEX EXTRACT RGATE {ALL | gatelayername [gatelayername …]}
{[DELTA] [NEGATIVE | POSITIVE] [SINGLE [factor] [scale]]
[DOUBLE [factor] [scale]]} {[VGDELTA [NEGATIVE | POSITIVE | NO]]}

Syntax 2, for Calibre xRC:

PEX EXTRACT RGATE gatelayername [gatelayername …]
{[DELTA] [NEGATIVE | POSITIVE] [SINGLE [factor] [scale]]
[DOUBLE [factor] [scale]]}

Arguments

ALL

Required parameter that specifies to apply the gate resistance calculation settings to all transistor gate layers. Use only with Calibre xACT or Calibre xACT 3D as shown in Syntax 1.

gatelayername

Required parameter that specifies the transistor gate layer. You can specify more than one gate layer name.

DELTA

Optional parameter that specifies the formation of a delta resistor network.

When DELTA is specified with SINGLE or DOUBLE, a negative resistor is added to the gate resistance using the value defined by scale, forming a delta resistor network.

The negative resistive network for both single and double connected gates is calculated using the following equations:

R1 = Rg * factor

R2 = Rg * factor

Rdelta = Rg * (factor * scale)

This is the default behavior.

The positive resistive network for both SINGLE and DOUBLE connected gates is calculated using the following equations:

R1 = Rg * (factor * scale) - (Rg *factor)

R2 = Rg * (factor * scale) + (Rg * factor)

VGDELTA

Optional parameter that specifies the formation of a via-on-gate delta resistor network. Use only with Calibre xACT or Calibre xACT 3D as shown in Syntax 1.

NEGATIVE | POSITIVE | NO

Optionally specify either the NEGATIVE or POSITIVE keyword to define the resistive network.

NEGATIVE is an optional keyword used to create a negative resistive network. Use the NEGATIVE keyword only for simulators that can accept negative resistance values.

POSITIVE is an optional keyword used to create a positive resistive network.

NO is an optional keyword used only together with VGDELTA to turn off via-on-gate resistance extraction.

NEGATIVE is the default for DELTA if neither NEGATIVE or POSITIVE keyword is specified.

POSITIVE is the default for VGDELTA if the NEGATIVE, POSITIVE, or NO keywords are not specified.

SINGLE [factor]

Optional parameter set that specifies the resistance adjustment factor for a transistor gate connected on one side; factor is optional. The default is 0.3333 if a value for factor is not specified. When DELTA is specified with SINGLE and a value for factor is not specified, then the default for factor is 0.1667.

Cannot be specified with VGDELTA keyword.

DOUBLE [factor]

Optional parameter set that specifies the resistance adjustment factor for a transistor gate connected on both sides; factor is optional. If a value for factor is not specified, then the default is 0.1667.

Cannot be specified with VGDELTA.

scale

An optional positive floating-point multiplier specified with either SINGLE or DOUBLE used to control the value of the delta resistance network. The default is -3 when a scale value is not specified.

Cannot be specified with VGDELTA.

Description

Specifies how to calculate the gate resistance and via-on-gate resistance of a transistor. This statement is useful for differentiating between the resistance of gate and field polysilicon layers. Overlap of gate layer and interconnect polysilicon is not taken into account for the gate resistance calculation.

If you turn off gate resistance calculation in the simulation model for a MOSFET, PEX Extract Rgate can be used to extract this resistance. This statement is only effective for a rectangular-shaped gate area. Do not use the PEX Extract Rgate statement for non-standard shapes, such as a bent gate.

The PEX Extract Rgate statement may be specified more than once in a rule file. Do not use PEX Extract Rgate with Resistance DEVICE_SEED. This can result in a rule compilation error. If the same gatelayername is specified by multiple PEX Extract Rgate statements, then the last statement in the rule file is applied to the layer overriding any previously specified PEX Extract Rgate statements for this layer.

If you are using reduction statements, the gate resistors may be factored out of the netlist. To avoid any gate resistor reduction, include the PEX Reduce Rgate NO statement.

Using the DELTA Parameter

The optional DELTA parameter may be used with either the SINGLE or DOUBLE parameter. If both SINGLE and DOUBLE are specified together with DELTA, then the delta model for all single and double gate connections is applied.

A single-sided gate connection is a transistor gate connected with a metal contact on one side. A double-sided gate connection is a transistor gate connected at both ends with metal contacts to form a loop.

When DELTA is specified with the SINGLE parameter, the resulting network for a single-sided gate connection includes a negative resistor, as shown in the following figure. This is the default behavior.

When specified with the DOUBLE parameter, the resulting network for a double-sided gate connection includes a negative resistor, as shown in the following figure. This is the default behavior.

Examples

Example 1

This example shows how PEX Extract Rgate is used to calculate gate resistances for transistors connected with different gate connections. All the gates are the same size and the gate resistance is 3 Ohms.

The following figure shows three transistors, M1, M2, and M3, with gate connections as single-sided, double-sided without loop, and double-sided with loop respectively. To simplify the netlist, parasitic resistors for the metal 1 and field poly interconnect, poly contacts, and parasitic capacitances are not included in the output netlist.

Figure 9. Gate Resistance Calculation Example

The rule file contains the following SVRF statements:

// gate layer definition 
gate = POLY and DIFF 
// poly interconnect definition
poly_interconnect = POLY not gate
// to keep poly connectivity, connect gate area to poly interconnect
CONNECT gate poly_interconnect
// physical layers gate and poly_interconnect inherit 
// the properties of the calibrated poly layer
PEX MAP POLY gate poly_interconnect
// gate resistance extraction; uses default values
PEX EXTRACT RGATE gate

The netlist, which contains the calculated gate resistors, is as follows:

// M1: Single-sided gate connection
R1 M1:g 1 1
// M2: Double-sided gate connection to different nodes
R2 2 M2:g 1.5
R3 M2:g 3 1.5
// M3: Double-sided gate connection to same node
R4 4 M3:g 0.5
R5 M3:g 5 0.5

Example 2

The following statements specify to netlist resistors on different layers with different resistance adjustment factors for transistor gates connected on one side: 

PEX EXTRACT RGATE ngate_max SINGLE 0.5
PEX EXTRACT RGATE pgate_max SINGLE 0.75

Example 3

To specify a negative delta network of single contact gate: 

PEX EXTRACT RGATE sgate SINGLE 0.5 DELTA NEGATIVE -1
or
PEX EXTRACT RGATE sgate DELTA NEGATIVE SINGLE 0.5 -1

Example 4

To specify a positive delta network of single contact gate: 

PEX EXTRACT RGATE sgate SINGLE 0.5 DELTA POSITIVE -1 
or
PEX EXTRACT RGATE sgate DELTA POSITIVE SINGLE 0.5 -1

Example 5

To specify a negative delta network of double contact gate: 

PEX EXTRACT RGATE dgate DOUBLE 0.75 DELTA NEGATIVE -3
or
PEX EXTRACT RGATE dgate DELTA NEGATIVE DOUBLE 0.75 -3

Example 6

To specify a positive delta network of double contact gate: 

PEX EXTRACT RGATE dgate DOUBLE 0.75 DELTA POSITIVE -3
or
PEX EXTRACT RGATE dgate DELTA POSITIVE DOUBLE 0.75 -3

Example 7

To specify a delta network for all single and double gates:

PEX EXTRACT RGATE mos_gate SINGLE 0.5 DELTA DOUBLE 0.5 DELTA
or 
PEX EXTRACT RGATE mos_gate DELTA SINGLE 0.5 DOUBLE 0.5

Example 8

Given a single Rgate resistance value of 360:

rDIN/0 DIN:27 M5:g 360 $w=2.4e-08 $layer=sgpgate $X=1.456 $Y=-1.367 $X2=1.456 $Y2=-1.319

applying the following statement

PEX EXTRACT RGATE sgpgate SINGLE

generates the following result:

rDIN/0 DIN:27 M5:g 240 $w=2.4e-08 $layer=sgpgate $X=1.456 $Y=-1.367 $X2=1.456 $Y2=-1.319

Example 9

To turn off via-on-gate resistance extraction for all gate layers specify the following statement: 

PEX EXTRACT RGATE ALL VGDELTA NO

Example 10

To extract a negative delta network for all single and double gates, and negative via-on-gate network resistance for all gate layers, specify the following statement: 

PEX EXTRACT RGATE ALL DELTA SINGLE 0.33 DOUBLE 0.16 VGDELTA NEGATIVE
