DFM Select By Net

Layer operation

License: Calibre nmDRC-H

Selects polygons from the first input layer according to node number, where the node numbers must match those provided by the second input layer. The second input layer may be a net-indexed selector layer produced by DFM Property Net or a layer with net ID properties.

Usage

DFM SELECT BY NET layer1 selector_layer [PROPERTY net_prop_name]…

Arguments

layer1

A required original layer or derived polygon layer with connectivity information.

selector_layer

A required layer that is used as a selector layer. The allowed layer types depend on the presence of the PROPERTY keyword set.

PROPERTY

Allowed selector_layer

Not specified

A net-indexed layer that was created by:

  • A DFM Property Net operation

  • A DFM Property operation with a layer created by DFM Property Net as the primary layer

Specified

A layer of any type (polygon, edge, or error layer) that includes the DFM properties specified by the PROPERTY keyword. Connectivity is not required.

PROPERTY net_prop_name

An optional keyword set that specifies DFM properties on the selector_layer are used to select geometries on layer1. The keyword set may be specified multiple times.

net_prop_name is the name of a netID or vector netID property on selector_layer. A warning is issued if selector_layer does not include a specified property.

Description

This operation provides an efficient method for selecting polygons by net that meet criteria determined by the selector_layer.

The selection process depends on the presence of the PROPERTY keyword:

  • Without PROPERTY

    Selects polygons on layer1 that are on the same net as virtual polygons on the selector_layer, which must be a net-indexed layer created by DFM Property Net.

  • With PROPERTY

    Selects polygons on layer1 that have the same netID as a netID value of one of the specified net_prop_name properties.

    If selector_layer is a net-indexed layer, only the property values are used for selection, not the net IDs of the virtual polygons on selector_layer.

Examples

Example 1

This example is for use with a DFM DV operation. It creates a net-indexed selector layer from the net-indexed DVPARAMS layer for the DFM DV operation. The check looks for polygons for which the minimum voltage is greater than the maximum voltage. The following code does not produce any errors, but the general method is useful for DFM DV checks that use more complicated logic to specify the voltages on a net. 

LAYER layer1 1
CONNECT layer1 

VARIABLE VH_net1 4
VARIABLE VL_net1 1

ni_layer1 = DFM PROPERTY NET layer1 
   [ vhi = VH_net1 ]
   [ vlo = VL_net1 ]
   [ - = AREA(layer1)] > 2
   [ nosync = NONETID() ]
// warning check for voltage values
voltage_warn { 
   net_warn = DFM PROPERTY NET ni_layer1 
      [ - = (PROPERTY(ni_layer1, "vlo") - PROPERTY(ni_layer1, "vhi"))] > 0 
   DFM SELECT BY NET layer1 net_warn
}
// the DFM DV check
dv_1 {
   DFM DV layer1 < 1.5 DVPARAMS ni_layer1 "vlo" 0 "vhi" 5 "nosync" >= 4
}

Example 2

This example uses the PROPERTY keyword. The selector layer includes M1 shapes that overlap via shapes with a total area greater than 1.0. The DFM Select By Net operation selects M2 shapes that are on the same net as the overlapped via shapes. 

M1_big_via_prop = DFM PROPERTY M1 via OVERLAP MULTI
               [ big_via_net_id = VNETID(GLOBALNETID(via))]
               [ - = AREA(via)] > 1.0
M2_big_via {
   DFM SELECT BY NET M2 M1_big_via_prop PROPERTY big_via_net_id
}
