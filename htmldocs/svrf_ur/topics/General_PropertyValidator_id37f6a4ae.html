Property Validator

The ‑validprop command line argument invokes the “property validator”, which checks a rule file for missing and unused DFM properties. The ‑validprop argument is only valid with the ‑drc, ‑dfm, ‑lvs, and ‑spice arguments.

Usage

calibre {-drc |‑dfm | ‑lvs | ‑spice} [run_args ...]
			‑validprop 
			[{[missing] [unused] [outputonly] [automatic]} | all] [trace]
			rule_file

Arguments

  • -drc | ‑dfm | ‑lvs | ‑spice

    A required argument specifying the Calibre run mode. Only the listed run modes are supported.

  • run_args

    One or more optional arguments specifying command line arguments needed for Calibre execution.

  • ‑validprop

    Required argument to invoke the property validator.

  • {[missing] [unused] [outputonly] [automatic]} | all

    Optional arguments that control what is reported by the property validator

    • missing — Report properties that are used but not defined.

    • unused — Report properties that are not used.

    • outputonly — Report properties that are used only in output.

      For example, properties are included in output from a DFM RDB operation or a DRC Check Map statement with the PROPERTIES keyword.

    • automatic — Report properties that are not used or output, but are always generated by an operation regardless of any user-specified options. May be abbreviated as auto.

      For example, the Device Layer operation automatically generates DFM properties.

    • all — Report missing, unused, outputonly, and automatic properties. This is the default.

  • trace

    Optional argument that includes a trace of the property analysis in the output.

  • rule_file

    A required argument specifying the Calibre rule file.

Description

The property validator checks a rule file for DFM properties that are missing, unused, used only in generated output, or unused but generated automatically.

The property validator processes command-line arguments, conditional directives, environment variables, and statements such as DRC Select Check in the same way that a Calibre run without ‑validprop does. It only reports on rule checks, layer operations, and specification statements that would be executed during the Calibre run. Registered and Unregistered Layers — The property validator registers layers that are produced by the operations listed in Table 1; other layers are considered unregistered. The property validator only examines the usage of DFM properties on registered layers.

Output Format

The property validation section of the log file has the following format. An explanation of layer name reporting follows the file format.

Header Section

-----              PROPERTY USE VALIDATION                       -----

Results Section

Layer "[<check>::]<derived_layer>" (line <line_number>) uses undefined 
(missing) property "<property>" on [UNREGISTERED]layer "<property_layer>"
 (line  <line_number>).
Layer "[<check>::]<layer_name>" (line <line_number>) has property 
 "<property>" that is unused.
Layer "[<check>::]<layer_name>" (line <line_number>) has property 
 "<property>", used only for output.
Layer "[<check>::]<layer_name>" (line <line_number>) has automatic 
  property "<property>" that is unused.
Layer "[<check>::]<layer_name>" (line <line_number>) has property "<property>" that is only referenced.

Summary Section

Number of missing properties = <number>
Number of usages of missing properties = <number>
Number of usages of missing properties on unregistered layers = <number>
Number of created but unused properties = <number>
Number of automatically created but unused properties = <number>
Number of referenced but unused properties = <number>
Number of properties used only for output = <number>

Footer Section

--- PROPERTY USE VALIDATION COMPLETED. ---

Layer name and line number reporting in the Results Section has these conventions:

  • Locally derived layers (those within a rule check) are reported with the format <check>::<layer_name>.

  • Global layers are reported as <layer_name>.

  • Unregistered layers are noted with UNREGISTERED.

  • If a result occurs in an included rule file, the rule file name is given before the <line_number>.

Referenced properties are those that are used as the argument to one of the property reference functions, such as PROPERTY_REF. If the property is only used in a property reference function, it is counted as unused and reported. For a list of property reference functions, see Table 1 with the “Referencing DFM Properties Defined in the Same Operation” section.

Examples

Example 1

To report all property validation results in a Calibre nmDRC rule file:

calibre -drc -validprop rules >& validprop.log

Example 2

To report missing and unused properties, but not outputonly or automatic properties, in a Calibre nmDRC rule file:

calibre -drc -validprop missing unused rules >& validprop.log

Example 3

To report missing properties only in a rule file used for netlist extraction with property derivations:

calibre -lvs -spice layout.sp -validprop missing rules >& validprop.log
