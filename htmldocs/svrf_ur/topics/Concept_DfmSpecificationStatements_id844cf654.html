DFM Specification Statements

This section lists the specification statements that apply to Calibre DFM applications (Calibre YieldAnalyzer and Calibre YieldEnhancer).
Table 1. Specification Statements: DFM

Statement

Description

DFM Assert

Asserts requirements on various SVRF objects and generates a compiler error if the requirements are not met. This statement may be used inside rule checks, which is not generally true of specification statements.

DFM Database

Specifies the path to the DFM database directory to be created.

Used only with calibre -dfm.

DFM Defaults

Specifies default options for subsequent DFM RDB statements.

DFM Function

Specifies a user-defined function that can be referenced within expressions for DFM Analyze, DFM Property, or other DFM Functions.

DFM LEFDEF Warning Limit

Specifies the maximum number of warning messages generated for a single message type. Used only with LEF/DEF input in flows that generate a DFM database (DFMDB).

DFM Select Check

Specifies the rule checks to perform and write to the DFM database.

Used only with calibre -dfm.

DFM Spec Contour

Defines a contour specification that is used by DFM Contour to generate a contour map of density values. Contour maps of the values of a DFM property can also be generated. Requires a Calibre YieldEnhancer or Calibre YieldAnalyzer license.

DFM Spec Fill

Creates a fill specification that can be used with DFM Fill to generate fill layers.

DFM Spec Fill Optimizer

Defines a set of optimization criteria to be used in finding the optimal fill.

DFM Spec Fill Shape

Defines a fill shape and assigns it a name by which it can be referenced from within a DFM Spec Fill specification.

DFM Spec Optimize

Defines a set of criteria to be used when optimizing fill with DFM Optimize. Uses a Calibre YieldEnhancer license.

DFM Spec Optimize Data

Defines an optimization expression used with DFM Spec Optimize. Uses a Calibre YieldEnhancer license.

DFM Spec Random Spot Fault

Defines a random spot defect specification that can be referenced in a DFM CAF and a DFM GCA operation.

DFM Spec Via

Defines a via specification that can be used by DFM Transition USEVIAS.

DFM Spec Via Redundancy

Defines a via redundancy analysis specification that can be referenced in a DFM Redundant Vias operation.

DFM Spec Via Shift

Specifies via shifting values and constraints to improve via location and metal coverage. Referenced by DFM Via Shift.

DFM Summary Report

Enables creation of a file containing a summary report for a calibre ‑dfm run.

DFM Unselect Check

Specifies DFM rule checks that are not executed when Calibre is run in -dfm mode.

DFM YS Autostart

Allows Tcl code within a rule file to be executed automatically after a calibre -dfm run, or prior to a YieldServer run.
