LVS Center Device Pins

Specification statement

Identifies pin location coordinates for the Calibre Connectivity Interface.

Usage

LVS CENTER DEVICE PINS {NO | YES}

Arguments

NO

A keyword that specifies arbitrary coordinate points are returned for the intersection of a pin layer and device seed layer. This is the default.

YES

A keyword that returns centered pin location coordinates for the intersection of a pin layer and a device seed layer if a centered location can be identified.

Description

This statement controls reporting of device pin locations ($PIN_XY netlist comments) for the Calibre Connectivity Interface (CCI) when using LAYOUT NETLIST PIN LOCATIONS YES (or dfm::set_layout_netlist_options -pin_locations). There is no other circumstance where this command has an effect.

Pin locations available in the LVS CCI flow represent an arbitrary point on the intersection of a pin layer and the device seed layer. For some extraction tools, greater accuracy is achieved if the pin is represented at the center of the intersection of the pin shape and the device seed shape.

Specifying YES indicates that one of the following cases is used, dependent upon the geometry.

  • If the pin and seed shapes overlap one another, then the pin location is defined as the center of the rectangular intersection as seen in the following figure.

    Figure 1. Rectangular Overlap $PIN_XY Location

  • If the pin and seed shape share no common area, then the pin location is the midpoint of the longest coincident edge segment shared by the polygons. The following figure shows the case when there is one such coincident segment.

    Figure 2. Coincident Edge $PIN_XY Location

  • If the center of the rectangular extent of pin and seed shapes does not lie on both the seed and pin shapes, then the resulting overlap area is partitioned into trapezoidal shapes, and the pin location is the centroid of an arbitrarily chosen trapezoidal region. Toroidal, C‑shaped, and racetrack-shaped polygons are common examples of shapes that apply to this situation. The following figure shows an example.

    Figure 3. Irregular Shape $PIN_XY Location

    For toroidal and oval shapes, the selected location will tend to be near the outer edge of the shape, but inside both the pin and seed shape boundaries.

See also LVS Center Device Location.

Examples

Assuming the rule file has the following:

// Use standard pin location reporting in CCI.
// If PEX tool requires intersection centers, specify YES.
LVS CENTER DEVICE PINS NO

Then these CCI commands write device pin locations at arbitrary intersection points of the pins and device seed shapes in the output netlist:

LAYOUT NETLIST PIN LOCATIONS YES
LAYOUT NETLIST WRITE cci.spi
