enclosure_perpendicular

Device property computation language function.

Returns a vector of numbers that correspond to trapezoidal dimensions of average enclosure distances and widths.

Note:

For better performance, use the Device Annotation Built-In Language.

Usage

ENClosure_PERpendicular(base, measurement, orientation, constraint)

ENClosure_PERpendicular_MULtifinger(base, measurement, orientation, constraint)

Arguments

base

A required argument that must reference any pin name, device layer name or number, or auxiliary layer name or number present in the Device statement attached to the property computation program. This argument represents the enclosed shape. For the multifinger function, base cannot be the seed layer since there can be only one seed shape per device.

measurement

A required argument that must reference any pin name, device layer name or number, or auxiliary layer name or number present in the Device statement. This argument represents an enclosing shape.

orientation

A required argument that must reference any pin or layer name present in the Device statement, as long as the respective polygon is coincident with the base polygon base on exactly one edge.

constraint

A required floating-point number in user units that represents the maximum enclosure distance between the base and measurement shapes.

Return Values

Vector.

Description

The semantics of this function are nearly identical to enclosure_parallel. The complete details can be found in the enclosure_parallel section and are not repeated here.

The difference between enclosure_parallel and enclosure_perpendicular is the orientation of the measurements.

Figure 1. ENClosure_PERpendicular and ENClosure_PERpendicular_MULtifinger

Compare this to Figure 1 with enclosure_parallel. ENClosure_PERpendicular and ENClosure_PERpendicular_MULtifinger measure enclosures in the direction perpendicular to the common edges of B and O.

For the non-multifinger form of the function, individual property names are constructed by appending the declared property name with the letter a, b, or w, and a trapezoid number. For example, given the following Device statement definition:

device mp seed poly psd psd well [
    property ex
    ex = enc_perp( seed, well, S, 3 )
]

the device might appear in the extracted layout netlist as follows:

M0 1 4 3 9 p exw1=6e-07 exa1=7e-07 exb1=8e-07 exw2=5e-07 exa2=7e-07 exb2=1e-06

In this example, S is the name of the source pin. Here, the device seed layer is used as B. Layers other than the seed can be used also.
