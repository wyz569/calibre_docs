DRC Extend Trivial Edge

Specification statement

Specifies how trivial edge measurement results are extended in dimensional check operations.

Usage

DRC EXTEND TRIVIAL EDGE {NO | YES}

Arguments

NO

Trivial edges constructed in point-to-point measurements are extended by 2 dbu when the dimensional check operation is error-directed or polygon directed, OPPOSITE or OPPOSITE SYMMETRIC is unspecified, and the measurement constraint is of the form “< m”. This is the default behavior if you do not include this statement in your rule file.

YES

Trivial edges constructed in dimensional check measurement are extended by 2 dbu as in the default mode, but also when additional criteria are met, as defined in the Description section.

Description

This statement controls trivial edge results extension by dimensional check operations. The “Point-to-Point Measurement Output” section in the Calibre Verification User’s Manual discusses the behavior in detail.

The YES keyword primarily supports error-directed input into multi-patterning operations. When YES is specified, trivial edges constructed by a dimensional check operation are extended by 2 dbu as in the default mode, but also when these criteria are met:

  1. The dimensional check operation is not “output-only”; that is, the operation does not only feed the DRC results database or only feed a Copy operation which itself only feeds the DRC results database.

  2. The dimensional check operation is error-directed.

  3. The trivial edge is part of a two-edge cluster. See “Edge Cluster Generation” in the Calibre Verification User’s Manual for details on edge clusters.

  4. It is a true trivial edge, that is generated as part of the actual measurement process and not created as the result of a non-commutative measurement. See “Special Considerations for the Opposite Metric” in the Calibre Verification User’s Manual for a discussion of commutative measurement.

Examples

DRC EXTEND TRIVIAL EDGE NO
