DFM NARAC

Layer operation

License: Calibre nmDRC-H

Converts a layer containing DFM properties into a Net Area Ratio Accumulate (NARAC) layer.

Usage

DFM NARAC layer BY property

Arguments

layer

A required argument that specifies a derived polygon layer with numeric DFM properties.

BY property

A required keyword and argument that specifies the name of a numeric-type property on the input layer. The value of this property defines the accumulated NAR values on the output layer.

Description

Converts a layer containing DFM properties into a Net Area Ratio Accumulate (NARAC) layer that can be used by Net Area Ratio Accumulate. Geometries on the output layer are an exact copy of those on the input layer. The value of the specified property defines the accumulated Net Area Ratio values on the output layer. Any other DFM properties that are associated with the input layer are ignored and are not copied to the output layer.

When the output layer is used as a NARAC layer but the specified property does not exist or is not a numeric-type DFM property, a warning is written to the transcript and an empty output layer is created. If the output layer is not used as a NARAC layer, the existence of the specified property is not checked.

The DFM function NARAC can be used in DFM Property operations to access the values on a NARAC layer.

Examples

Consider the following rule file fragment, which is the example given with Net Area Ratio Accumulate:

DRC INCREMENTAL CONNECT YES

CONNECT poly gate
// find ratio of poly to gate areas and attach to gates
X = NET AREA RATIO poly gate >= 0 
      [(AREA(poly)-AREA(gate))/AREA(gate)] ACCUMULATE

CONNECT metal1 poly BY contact

// find ratio of metal1 to gate areas and attach to gates
Y = NET AREA RATIO metal1 gate >= 0 
      [AREA(metal1)/AREA(gate)] ACCUMULATE

// if the combined accumulation values on X and Y exceed 20, output X
// with the new value on layer Z.
Z = NET AREA RATIO ACCUMULATE X Y > 20 [VALUE(X)+VALUE(Y)]

A combination of DFM Property Net, DFM Property with the NODAL keyword, and DFM NARAC can be used instead when the required computation cannot be defined with Net Area Ratio. The following code uses a conditional expression to apply a multiplication factor to the metal1 to gate area ratio if the metal1 area is greater than 0.1. 

DRC INCREMENTAL CONNECT YES
CONNECT poly gate

// create net-indexed layer with AX property  
X2_A = DFM PROPERTY NET gate poly 
			[AX = (AREA(poly)-AREA(gate))/AREA(gate)]
// for each gate polygon, find the total AX value summed over its net
X2_A_total = DFM PROPERTY gate X2_A NODAL MULTI 
			[AX = PROPERTY(X2_A, AX)]
// create the NARAC layer
X2 = DFM NARAC X2_A_total BY "AX"

CONNECT metal1 poly BY contact

// use a conditional expression in the computation of the ratio of 
// metal1 area to gate area
Y2_A = DFM PROPERTY NET gate metal1 
			[AY = (AREA(metal1) > 0.1) ?  1.2*AREA(metal1)/AREA(gate) 
													: AREA(metal1)/AREA(gate)]
Y2_A_total = DFM PROPERTY gate Y2_A NODAL MULTI 
			[AY = PROPERTY(Y2_A, AY)]
Y2 = DFM NARAC Y2_A_total BY "AY"

Z2 = NET AREA RATIO ACCUMULATE X2 Y2 > 20 [VALUE(X2)+VALUE(Y2)]
