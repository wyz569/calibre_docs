PSPROPERTY_REF

Property access function for DFM expressions

References the value of a string property that was previously defined in the same operation; the returned string is prefixed by Xn/Xm/.../ to indicate placement hierarchy. Used only in DFM Property.

Usage

PSPROPERTY_REF(property_name)

Arguments

property_name

A required name of a string property previously defined in the current operation.

References to temporary or Non-Persistent DFM Properties named “-” are not allowed. References to named temporary properties (properties with names that begin with “+”) are allowed.

Return Values

String.

Description

References the value of a string property computed by a previous property expression within the same operation; the returned string is prefixed by Xn/Xm/.../ to indicate placement hierarchy. The reference function is useful if the value of a property is used in other property definitions. Referencing the property improves performance by reducing the number of times the property value is calculated and can make property expressions simpler to write.

See “Referencing DFM Properties Defined in the Same Operation” for usage rules and examples.

See “Property Access Function Summary” for a list of all property access functions, including both the reference functions and functions that access a property value saved to a layer object in a different operation.
