Net Number DFM Properties

Net number properties store the net ID for a geometry. You can create a vector of net IDs. Net number properties have some special restrictions in DFM expressions.

Two data types are used to store net numbers:

  • netID — Contains a net number property.

  • vnetID — Contains a vector of netID values.

See “Net Based Function Summary” for a summary of the DFM functions that handle netID and vnetID properties.

Net number properties cannot be manipulated in the same way as numeric properties. They cannot be used with math functions, math operators, and measurement functions. For example, [NETID(layer) + 1] causes a compiler error. In addition, it is invalid to specify a constraint for a DFM expression that evaluates to a netID or vnetID property. Net number properties can be compared for equality or inequality, however; see “Comparison of DFM Properties”.

When computing Cell or Net properties with DFM Property (the BY CELL and BY NET keywords), netID properties are silently ignored. vnetID properties are considered, however. The vnetID property for a net is a vector generated by concatenating corresponding vnetID properties for all geometries on the net on a given layer. Similarly, the vnetID property for a cell is a concatenated vector of corresponding vnetID properties for all geometries in the cell on a given layer.

When DRC Incremental Connect YES is specified, net number properties are not maintained across connect zones. See “Incremental Connectivity and Antenna Checks” in the Calibre Verification User’s Manual.
