Mask Data Preparation Statements

This section summarizes the statements that pertain to mask data preparation, including database fracturing and mask verification.
Table 1. Mask Data Preparation Statements

Statement

Description

Fracture Hitachi

Converts layout input data into Hitachi form and outputs to a file for use in third-party mask preparation tools. Requires a Calibre FRACTUREh product license.

Fracture JEOL

Converts layout input data into JEOL form and outputs to a file for use in third-party mask preparation tools. Requires a Calibre FRACTUREj product license.

Fracture MEBES

Converts layout input data into MEBES form and outputs to a file for use in third-party mask preparation tools. Requires a Calibre FRACTUREm product license.

Fracture MICRONIC

Converts layout input data into MICRONIC form and outputs to a file for use in third-party mask preparation tools. Requires a Calibre FRACTUREc product license.

Fracture NUFLARE

Converts layout input data into VSB11/VSB12 form and outputs to a file for use in third-party mask preparation tools. Requires a Calibre FRACTUREt product license.

Fracture NUFLARE_MBF

Converts layout input data into the format for Nuflare multi-beam machines. Requires a Calibre FRACTUREn product license.

Fracture OASIS_MAPPER

Converts layout input data into the OASIS.MAPPER format. Requires a Calibre FRACTUREp product license.

Fracture OASIS_MASK

Converts layout input data into OASIS_MASK form (conforming to the SEMI P44 standard) and outputs to a file for use in third-party mask preparation tools. As OASIS_MASK is a restriction of the OASIS®1 format and a superset of the VB:OASIS format, it requires a Calibre FRACTUREv product license.

Fracture OASIS_MBW

Converts layout input data into the OASIS.MBW format for IMS machines. Requires a Calibre FRACTUREi product license.

Litho CLMPC

Provides modeling, simulation, and correction for the distortions that occur during mask manufacturing for curvilinear and skew-edge mask layouts.

Litho File

Used to specify an in-line file.

Litho Maskopt

Improves fracture quality.

LITHO MPC

Provides modeling, simulation, and correction for the distortions that occur during mask manufacturing.

Litho MPCverify

Simulates the mask process under various process conditions and checks the simulation results.

MDP Checkmap

Outputs single or multiple layers and datatypes with bin injection for large cells and is used primarily for OASIS bin injection.

MDP Embed

Provides section-based processing for some SVRF commands without requiring a FRACTURE operation.

MDP Mapsize

Allows you to variably resize elements of your layout based on their geographical locations.

MDP Oasis_Extent

Computes the extent of an OASIS layout.

MDPmerge

Optimizes the throughput of the mask writing machine, the parameter control like registration and CD, and enables the proximity correction for adjacent placements of individual chips based on various criteria

MDPstat

Analyzes vector beam data such as Hitachi, JEOL, NUFLARE, and OASIS.MASK formatted data, allowing you to assess the quality of the fracture output.

MDPverify

Verifies fractured data by performing Boolean XOR comparison between databases of various types. Requires a Calibre FRACTURE product license.
