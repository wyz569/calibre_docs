PEX Netlist Groundnet Global

Parasitic extraction

Specifies to include or comment out the ground net name statement in the extracted DSPF netlist. Use only with Calibre xACT 3D PDB flow and Calibre xRC.

Usage

PEX NETLIST GROUNDNET GLOBAL {YES | NO}

Arguments

YES

Required keyword that specifies to include the ground net name statement in the extracted DSPF netlist. This is the default.

NO

Required keyword that specifies to not include the ground net name statement in the extracted DSPF netlist. A commented GROUND_NET statement is generated instead.

Description

This statement specifies whether or not to include the GROUND_NET statement as a comment in the extracted DSPF netlist. If the PEX Netlist Groundnet Global statement is not included in a rule file, then a GROUND_NET statement beginning with “*| ” is included in the DSPF netlist.

Examples

Example 1

The statement

PEX NETLIST GROUNDNET GLOBAL YES

includes a ground net name statement in the extracted DSPF netlist. For example:

*|GROUND_NET gnd

Example 2

The statement

PEX NETLIST GROUNDNET GLOBAL NO

includes a commented statement in the extracted DSPF netlist. For example:

* GROUND_NET gnd
