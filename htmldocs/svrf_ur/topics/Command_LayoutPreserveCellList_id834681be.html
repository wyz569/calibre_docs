Layout Preserve Cell List

Specification statement

Specifies to protect certain layout cells from most hierarchy modifications, including automatic expansion, during circuit extraction in Calibre nmLVS-H. Used only in hierarchical Calibre applications.

Usage

LAYOUT PRESERVE CELL LIST list_name

Arguments

list_name

A required name of a list of cells specified in a Layout Cell List specification statement.

Description

Preserves the layout cells in the specified list from expansion during hierarchical runs. Cells identified using this statement are treated in the same manner as hcells in Calibre nmDRC-H and in Calibre nmLVS-H circuit extraction. This statement may be specified only once.

The list_name is a list specified in a Layout Cell List statement. Layout Cell List supports wildcard and regular expression matches of cell names in addition to specifying multiple cell lists through list matching.

Cells declared using the Layout Preserve Cell List statement have no relation to cells specified in the Hcell specification statement or using an hcell list. This statement may be preferable to the Hcell statement in Calibre nmDRC-H applications because of the flexibility of the Layout Cell List matching capabilities. However, if cells are preserved when the Calibre hierarchical engine would prefer to expand them, there can be a performance penalty.

The cells in list_name are not protected from user-specified hierarchy transformations which explicitly mention cell names, such as Expand Cell or Flatten Cell.

Calibre nmLVS-H considerations — Hcells apply during both circuit extraction and netlist comparison, while the cells in Layout Preserve Cell List apply during circuit extraction, but not during comparison. The Layout Preserve Cell List statement should be used only when it is absolutely essential that certain layout cells be preserved during circuit extraction. Calibre nmLVS-H performs various hierarchy transformations to optimize run time and memory use. Exempting cells unnecessarily from these transformations is likely to reduce the effectiveness of these optimizations and to lead to an increase in run time and memory consumption. Hcells specified in Layout Preserve Cell List statements are not subject to expansion by the LVS Auto Expand Hcells statement.

In order to guarantee that devices are properly formed, seed promotion can occur out of preserved cells during device recognition. See “Hcells and Circuit Comparison” in the Calibre Verification User’s Manual for a discussion of this behavior. In many cases, the seed promotion is mitigated when LVS Push Devices YES is specified, which is the default.

See also “Hierarchy Modification Suppression in LVS Connectivity Extraction” in the Calibre Verification User’s Manual.

Examples

These two statements declare all cells having names starting with nm$$ or pm$$ to be preserved cells. The preserved cells are protected from most optimizations of layout hierarchy normally performed during connectivity extraction.

LAYOUT CELL LIST pcells "nm$$*" "pm$$*"
LAYOUT PRESERVE CELL LIST pcells
