PEX Fieldsolver Cell_array

Parasitic extraction

Specifies the boundary conditions used by the capacitance solver and grounds all the conductors in the image. Used only in Calibre xACT 3D.

Usage

Syntax 1, for horizontal orientations:

PEX FIELDSOLVER CELL_ARRAY X {
      {{[POSITIVE | NEGATIVE] REFLECTIVE |
      [POSITIVE | NEGATIVE] PERIODIC}
      [BY_EXTENT min max | BY_ENCLOSURE min max] [OPEN_BOUNDARY]} |
      {{POSITIVE | NEGATIVE} GROUND value
      [{NEGATIVE | POSITIVE} GROUND value]}
}...

Syntax 2, for vertical orientations:

PEX FIELDSOLVER CELL_ARRAY Y {
      {{[POSITIVE | NEGATIVE] REFLECTIVE |
      [POSITIVE | NEGATIVE] PERIODIC}
      [BY_EXTENT min max | BY_ENCLOSURE min max] [OPEN_BOUNDARY]} |
      {{POSITIVE | NEGATIVE} GROUND value
      [{NEGATIVE | POSITIVE} GROUND value]}
}...

Syntax 3, for depth orientations:

PEX FIELDSOLVER CELL_ARRAY Z
      {POSITIVE | NEGATIVE} GROUND value
      [{NEGATIVE | POSITIVE} GROUND value]

Arguments

X

A required parameter that specifies the boundary of the cell is for horizontal orientations.

Y

A required parameter that specifies the boundary of the cell is for vertical orientations.

Z

A required parameter that specifies the boundary of the cell is for the Z direction. Use only with the GROUND keyword as shown in Syntax 3. May not be specified with REFLECTIVE, PERIODIC, BY_EXTENT, BY_ENCLOSURE, or OPEN_BOUNDARY keywords.

REFLECTIVE

A required parameter that specifies the placement of a mirror image of the enclosed geometries on the other side of the boundary, and grounds all conductors in the image. May not be specified with PERIODIC without a POSITIVE or NEGATIVE keyword.

PERIODIC

A required parameter that specifies the replication of the enclosed geometries on the other side of the boundary, and grounds all replicated conductors in the image. A periodic cell_array is different from a reflective cell_array in that the geometries are copies rather than mirror images. May not be specified with REFLECTIVE without a POSITIVE or NEGATIVE keyword.

POSITIVE | NEGATIVE

A parameter that specifies the direction of the PERIODIC and REFLECTIVE image placement. POSITIVE refers to image cells placed to the right or above the center cell, NEGATIVE refers to image cells placed to the left or below the center cell. May optionally be specified with the PERIODIC and REFLECTIVE keywords. Must be specified with the GROUND keyword.

BY_EXTENT min max

An optional parameter set that specifies the absolute location of the boundary corresponding to the specified orientation, X or Y. The floating point values min max represent the minimum and maximum coordinate along the orientation in microns.

If neither BY_EXTENT or BY_ENCLOSURE are specified, the location of the boundary defaults to BY_ENCLOSURE 0 0.

BY_ENCLOSURE min max

An optional parameter set that specifies the distance of the boundary from the edge of the bounding box of metals in the cell along the specified orientation, X or Y. The floating point values min and max are in microns.

If neither BY_EXTENT or BY_ENCLOSURE are specified, the location of the boundary defaults to BY_ENCLOSURE 0 0.

OPEN_BOUNDARY

An optional parameter that specifies not to modify metal geometries that are partially or completely outside the boundaries of the cell.

{POSITIVE | NEGATIVE} GROUND value [{NEGATIVE | POSITIVE} GROUND value]

A parameter set that specifies the location of the ground plane relative to the cell in the positive or negative x, y, or z direction. This parameter set must be specified with the POSITIVE or NEGATIVE keywords. You can specify the GROUND value with both POSITIVE and NEGATIVE keywords in the same statement; one GROUND value keyword set must be specified with POSITIVE and one set must be specified with NEGATIVE. The coordinate value is a floating point number in microns. The value specified with the NEGATIVE keyword must be less than the minimum extent of the design. The value specified with the POSITIVE keyword must be greater than the maximum extent of the design. An error is generated and extraction stops if the specified value is equal to or within the design extent.

Description

This optional statement specifies the boundary conditions used by the capacitance solver. All the conductors in the image are grounded. These boundary conditions also apply to resistance extraction.

Boundary conditions enable the extraction of an isolated cell by simulating the parasitic effects of neighboring conductors. Use this statement to improve extraction accuracy of memory cells or similar models. For example, this is useful for memory bit cell design when simulating cell placements and their effect on the memory cell geometries. Use this statement to improve extraction accuracy of memory cells or similar models. In the absence of this statement, the surrounding space is treated as air.

Specify this statement separately once for the horizontal (X) and vertical (Y) orientations. This statement can not be specified with PEX Fieldsolver Boundary for the same orientation. For example, if your rule file contains a PEX Fieldsolver Boundary X statement, you can not specify a PEX Fieldsolver Cell_array X statement.

The boundary encloses the bounding box of the conductors in the cell. The size of the boundary is specified in terms of extent in each direction, Xmin and Xmax coordinates or Ymin and Ymax coordinates, or in terms of enclosure in the X or Y directions. The default boundary location for BY_ENCLOSURE 0 0 is the same as the bounding box of metals as shown in the following figure.

Use the POSITIVE and NEGATIVE keywords together with either the REFLECTIVE or PERIODIC keyword to specify the position of image cells relative to the center cell. POSITIVE refers to image cells placed to the right or above the center cell, NEGATIVE refers to image cells placed to the left or below the center cell.

The GROUND keyword is used to define a ground plane. Ground plane placement can be specified individually in the positive or negative x, y, or z directions relative to the cell. Use the GROUND keyword together with the POSITIVE or NEGATIVE keywords to specify the direction of the ground plane.

Examples

Example 1

Include the following statements to specify the cell_array type as reflective for X and periodic for Y, and ground all conductors in the image: 

PEX FIELDSOLVER CELL_ARRAY X REFLECTIVE
PEX FIELDSOLVER CELL_ARRAY Y PERIODIC

In this example, all conductors in the x-direction mirror image are grounded and all the copied conductors in the y-direction are grounded. The boundary location is the same as the bounding box since neither BY_EXTENT or BY_ENCLOSURE were specified.

Example 2

Include the following statements to specify the boundary condition along the x-axis as a reflective cell and the y-axis as a periodic cell: 

PEX FIELDSOLVER CELL_ARRAY X REFLECTIVE BY_ENCLOSURE 2 2
PEX FIELDSOLVER CELL_ARRAY Y PERIODIC BY_ENCLOSURE 4 4

In this example, the boundary location in the x-direction is defined by an enclosure of 2 microns, and the reflective images of the conductors are grounded. The boundary location in the y-direction is 4 microns, and the periodic images of the conductors are grounded. For a top view of the reflective and periodic boundary settings, see the following figure.

Figure 5. Cell_array Boundaries Using BY_ENCLOSURE

Example 3

Use the following statements to specify the boundary condition along the x-axis as a reflective cell and the y-axis as a periodic cell: 

PEX FIELDSOLVER CELL_ARRAY X REFLECTIVE BY_EXTENT 1 10
PEX FIELDSOLVER CELL_ARRAY Y PERIODIC BY_ENCLOSURE 4 5

In this example the boundary location in the x-direction is defined by absolute locations, Xmin = 1 and Xmax = 10, along the x-axis. The boundary location in the y-direction is 4 microns from the lower edge of the bounding box and 5 microns from the upper edge. All reflective and periodic images of the conductors are grounded. For a top view of the effect of these reflective and periodic boundary settings, see the following figure.

Use the following statements to specify the boundary condition along the x-axis as a reflective cell and the y-axis as a periodic cell: 

PEX FIELDSOLVER CELL_ARRAY X REFLECTIVE BY_EXTENT 1 10
PEX FIELDSOLVER CELL_ARRAY Y PERIODIC BY_EXTENT 1 15

In this example the boundary location in the x-direction is defined by absolute locations, Xmin = 1 and Xmax = 10, along the x-axis. The boundary location in the y-direction is defined by absolute locations, Ymin = 1 and Ymax = 15, along the y-axis. All reflective and periodic images of the conductors are grounded. For a top view of the effect of these reflective and periodic boundary settings, see the following figure.

Figure 7. Cell_array Boundaries Using BY_EXTENT

Example 5

Use the following statements to specify the boundary condition along the x-axis as a periodic cell and the y-axis as a reflective cell: 

PEX FIELDSOLVER CELL_ARRAY X PERIODIC BY_ENCLOSURE 0 0
PEX FIELDSOLVER CELL_ARRAY Y REFLECTIVE BY_ENCLOSURE 0 0

In this example the boundary location in the x-direction is 0 microns from the left edge of the bounding box and 0 microns from the right edge. The boundary location in the y-direction is 0 microns from the lower edge of the bounding box and 0 microns from the upper edge. All reflective and periodic images of the conductors that do not touch the enclosure are grounded. For a top view of the effect of these reflective and periodic boundary settings, see the following figure.

Example 6

Use the following statements to specify the boundary condition along the x-axis as a periodic cell and the y-axis as a reflective cell: 

PEX FIELDSOLVER CELL_ARRAY X PERIODIC BY_ENCLOSURE 0 0 OPEN_BOUNDARY
PEX FIELDSOLVER CELL_ARRAY Y REFLECTIVE BY_ENCLOSURE 0 0

In this example the boundary location in the x-direction is 0 microns from the left edge of the bounding box and 0 microns from the right edge. The boundary location in the y-direction is 0 microns from the lower edge of the bounding box and 0 microns from the upper edge.

By applying the OPEN_BOUNDARY option in the x-direction, metal segments found outside the boundary enclosure of the cell are not cut. The cell shown in the following figure contains a segment drawn outside the boundary. All reflective and periodic images of the conductors that do not touch the enclosure are grounded. For this example, all conductors are touching the boundary, so none of the images are grounded.

Include the following statement to specify the boundary condition along the positive x-axis as a reflective cell and the negative x-axis as a periodic cell:

Example 9

If the value specified for the ground plane boundary is equal to or within the extent of the design, then an error is generated and extraction stops. For example, given the following statements used in example 8 to specify a three dimensional ground plane boundary condition along the x,y,z-axes to form a cube around the cell: 

PEX FIELDSOLVER CELL_ARRAY X NEGATIVE GROUND 0 POSITIVE GROUND 2.5
PEX FIELDSOLVER CELL_ARRAY Y NEGATIVE GROUND 0 POSITIVE GROUND 4.8
PEX FIELDSOLVER CELL_ARRAY Z NEGATIVE GROUND -0.2 POSITIVE GROUND 3.3

If the x negative ground and y negative ground values are specified as 1 rather than 0:

PEX FIELDSOLVER CELL_ARRAY X NEGATIVE GROUND 1 POSITIVE GROUND 2.5
PEX FIELDSOLVER CELL_ARRAY Y NEGATIVE GROUND 1 POSITIVE GROUND 4.8
PEX FIELDSOLVER CELL_ARRAY Z NEGATIVE GROUND -0.2 POSITIVE GROUND 3.3

the following errors are generated and extraction stops:

ERROR: User defined negative X ground plane (1.000000) is within the design extent (0.000000 2.250000)
ERROR: User defined negative Y ground plane (1.000000) is within the design extent (0.000000 5.0000)
FATAL ERROR:Ground plane definition errors detected - exiting...
