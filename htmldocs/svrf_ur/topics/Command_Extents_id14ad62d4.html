Extents

Layer operation

Constructs a derived polygon layer consisting of the merged minimum bounding boxes of each polygon on the specified input layer. The default output can be changed with optional keywords.

Usage

EXTENTS layer1 [CENTERS [size] | SQUARES | INSIDE OF LAYER layer2]

Arguments

layer1

A required argument specifying an original layer or layer set or a derived polygon layer.

CENTERS [size] | SQUARES | INSIDE OF LAYER layer2

An optional keyword choice that changes the output of the operation. All output is merged.

  • CENTERS [size] — Generates marker squares at the centers of the unmerged bounding boxes of each polygon instead of the bounding boxes themselves.

    The optional size argument specifies the size of the marker squares. The marker square has the dimension of size × size user units. The default value is 1. If the specified size is less than 2 dbu, then the size is adjusted internally to be 2 dbu. See “Example 3”.

  • SQUARES — Generates squares rather than minimum bounding boxes. The squares have the same center points as the unmerged minimum bounding boxes of each polygon. Each square has an edge length equal to the longest edge of the minimum bounding box for the polygon.

  • INSIDE OF LAYER layer2 — For each polygon on layer2, generates the minimum bounding box of all layer1 polygons that are inside the layer2 polygon. The layer2 argument must be an original or derived polygon layer. See the “Description” section for a precise definition and “Example 4”.

Description

Generates a derived polygon layer consisting of the minimum bounding boxes of each polygon on layer1 by default. The CENTERS and SQUARES keywords change the shape and size of the output polygons. All output is merged.

Specifying INSIDE OF LAYER causes the operation to do the following:

z = EXTENTS x INSIDE OF LAYER y

  1. For each polygon Pi ( i = 1 … n) on layer y, let zPi = EXTENT( x INSIDE Pi )

    That is, for each polygon on layer y, find the Extent of layer x shapes Inside the layer y polygon.

  2. z = zP1, zP2, … zPn combined and merged

Calibre expands text basepoints by 2 dbu in all directions. This affects the extent of text layers.

The DFM Property operation is useful when measurement of unmerged extents is needed; see “:EDGE, :EXTENT, and :EXTENT_EDGE Layer Derivation Indicators”. DFM Property Select Secondary can also output unmerged extent shapes; see “Example 7”.

See also Extent and Extent Cell.

Example 2

// Center-to-center pad distance must be 300 microns:
cp = EXTENTS pad CENTERS
rule { EXT cp < 299 }      // Use 299 because centers are 1 x 1

Example 3

Use the CENTERS keyword with the size argument. A marker size of 0.1 user units is specified to reduce the likelihood that the marker shapes overlap and merge. 

extents_centers = EXTENTS metal1 CENTERS 0.1
