VNETID

Vector net property function for DFM expressions

Creates a vector of netID values. Supported only for the DFM Property operation.

Usage

VNETID(GLOBALNETID(layer))

VNETID(NETID(layer))

Arguments

layer

A required layer with connectivity information; may not be an error layer.

Return Values

Vector of net ID values.

Description

Creates a vector of netID type properties, where the net ID value is calculated with either NETID or GLOBALNETID.

This function can only be called in DFM Property expressions. The following table describes the behavior depending on whether layer is the primary layer or a secondary layer in the DFM Property operation.

Input layer

Result

Primary layer

Vector with one net ID element.

Example: {<2>}

Secondary layer

Vector with one tuple for each secondary layer object clustered with the primary layer object. Each tuple contains one net ID element corresponding to the secondary layer object.

Example: {<4> <5> <6>}

You can access individual vector elements with the function NETVELEMENT. You can also access vector elements with a user-defined function created by a DFM Function statement; see “Accessing Vector Elements”

Examples

This example creates a vector of net IDs, where the vector has one net ID element for each layer1 object clustered with the primary layer object (layer3). 

CONNECT layer1 layer2
gnet_out = DFM PROPERTY layer3 layer1 OVERLAP MULTI
	[net_vector = VNetID(GLOBALNETID(layer1))]
