PEX Report Shorts

Parasitic extraction

Generates a report file containing the location of any shorts in the design. Used only in the Calibre xACT digital flow.

Usage

PEX REPORT SHORTS filename [MINSPACING {NO | YES}]

Arguments

filename

A required filename for the report.

The filename parameter can contain environment variables. For information regarding the use of environment variables in the filename parameter, refer to “Environment Variables in Pathname Parameters”.

If the filename ends in either the .Z or .gz suffix, then the file is automatically compressed using the compress or gzip commands, respectively. This assumes these commands are available in your $PATH.

MINSPACING {NO | YES}

An optional keyword set that enables minspacing checks during short detection. The default is NO if this keyword set is not specified.

Description

An optional statement that creates a report file containing the location of any shorts in the design.

Examples

Example 1

To report shorts detected in the Calibre xACT digital flow, include the following statement in your rule file:

PEX REPORT SHORTS myShorts.report

The following is an example of output generated with this statement:

...
Detected short between net 'n4' and net 'n3' @ (5440 4010 5600 5930)
Detected short between net 'n1’ and net 'n5' @ (5440 3190 5830 4010)
Detected short (FILL) on net 'n122' @ (5280 1230 5600 2130)
...

Example 2

To report minimum spacing violations detected in the Calibre xACT digital flow, include the following statement in your rule file:

PEX REPORT SHORTS myMinSpacing.report MINSPACING YES

The following is an example of output generated with MINSPACING YES:

...
Detected violation of minimum spacing between net 'pc\[5\]' and net 'n57' @ (5440 4010 5600 5930)
Detected violation of minimum spacing between net 'pc\[7\]’ and net 'n55' @ (5440 3190 5830 4010)
...
