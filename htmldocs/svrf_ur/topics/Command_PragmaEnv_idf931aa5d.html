#PRAGMA ENV

The #PRAGMA ENV pre-processor directive defines the default value for an environment variable used in SVRF statements.

Usage

#PRAGMA ENV variable_name value

Arguments

variable_name

A string giving the variable name. You should enclose variable_name within quotes if variable_name includes spaces or characters other than letters and numbers.

value

A required string giving the value for the variable. May not be a variable.

Description

Any SVRF operation or specification statement that is specified after the #PRAGMA ENV statement and refers to the environment variable variable_name uses the value of the actual environment variable if one is defined; otherwise, the value from the #PRAGMA ENV statement is used.

The #PRAGMA ENV statement must be specified prior to the first use of the environment variable variable_name in order to function.

The #PRAGMA ENV statement applies only to the variables used in SVRF statements. It has no effect on the environment variables used by Tcl expressions in TVF code, including compile-time and runtime TVF.

Examples

Example 1

For example, in normal operation, if the environment variable INCLUDE_DIR is not defined, this statement does not compile:

INCLUDE $INCLUDE_DIR/rules.inc

If the environment variable INCLUDE_DIR is defined as “./inc” with #PRAGMA ENV, the statement compiles and the Include statement reads the file ./inc/rules.inc.

#PRAGMA ENV INCLUDE_DIR "./inc"
INCLUDE $INCLUDE_DIR/rules.inc

Example 2

Assume that the environment variable INCLUDE_DIR is not defined.

#PRAGMA ENV INCLUDE_DIR "."
INCLUDE $INCLUDE_DIR/rules.inc

This statement compiles and the Include statement reads the file ./rules.inc.

Example 3

Assume that the environment variable INCLUDE_DIR is defined as “./inc”. 

#PRAGMA ENV INCLUDE_DIR "."
INCLUDE $INCLUDE_DIR/rules.inc

This statement compiles and the Include statement reads the file ./inc/rules.inc.
