DFM Select Check

Specification statement

Specifies the rule checks to perform when Calibre is run in DFM mode.

Usage

DFM SELECT CHECK [NODAL] rule_check [rule_check …]

Arguments

NODAL

An optional argument that causes node numbers to be preserved when processing the operations in the specified check.

rule_check

A required argument specifying the name of a rule check or rule check group. You must specify at least one rule_check.

Description

Specifies the rule checks that are executed in a calibre -dfm run. By default, all rule checks are executed. This statement also has an effect on calibre -drc runs.

If DFM Select Check is specified, then a calibre -dfm run executes rule checks from all such statements and excludes any checks specified by DFM Unselect Check. A check specified by both statements is excluded.

If the same check is selected by both DFM Select Check and DFM Select Check NODAL, the check is treated as nodal. All layers produced in rule checks selected with DFM Select Check NODAL must be connectivity layers; otherwise, the application reports the following error:

Cannot establish MASK connectivity of the output layer in DFM NODAL RuleCheck.

For example, the Flatten operation is not node-preserving and produces this error if it appears in a rule check selected with DFM Select Check NODAL. (This behavior applies to both calibre ‑drc and ‑dfm modes.) In -dfm mode, DFM Select Check NODAL causes layer output from a rule check to be saved to the DFM database with node numbers automatically. However, NODAL does not modify the behavior of DFM RDB. The DFM RDB NODAL option must be used to save nodal information to an RDB.

If DFM Select Check is not specified, then a calibre -dfm run executes rule checks specified with DRC Select Check, which includes all checks by default. Any checks specified by DFM Unselect Check, DRC Unselect Check, or ERC Unselect Check are excluded.

If a rule check is specified with DFM Select Check or ERC Select Check, it does not execute when Calibre is run in -drc mode unless it is also explicitly specified with DRC Select Check.

Examples

In the following examples, assume a calibre -dfm run with a rule file that contains four checks: check1, check2, check3, and check4. Each example is a separate case with no interdependencies.

Example 1

// executes check1 and check3 in -dfm mode; check2 and check4 in -drc mode
DFM SELECT CHECK check1 check3

Example 2

// executes check3 and check4 in -dfm mode; all checks in -drc mode
DFM UNSELECT CHECK check1 check2

Example 3

// executes check2 in -drc and -dfm modes
DRC SELECT CHECK check2

Example 4

// executes check4 in -dfm mode; check3 and check4 in -drc mode
DRC UNSELECT CHECK check2
DFM UNSELECT CHECK check3
ERC SELECT CHECK check1

Example 5

// executes check1 in -dfm mode; check2 and check3 in -drc mode
DFM SELECT CHECK check1
DRC SELECT CHECK check2 check3
DRC UNSELECT CHECK check1
