Unit Length

Specification statement

Provides the length scale factor for device calculations. This statement has no effect on the rule file precision or upon layer operation measurement constraints. Used only by Calibre nmLVS.

Usage

UNIT LENGTH {number | factor}

Arguments

number

A positive real number that specifies the preferred user unit of length.

factor

A dimension of length. Possible choices are these:

u

Equivalent to 1e-6 m, a micron. This is the default behavior for Calibre applications if you do not include this statement in the rule file.

mil

Equivalent to 25.4e-6 m, 1e-3 inch.

mm

Equivalent to 1e-3 m, a millimeter.

cm

Equivalent to 1e-2 m, a centimeter.

inch

Equivalent to 25.4e-3 m.

m

Equivalent to 1 m, a meter.

Description

Provides the user unit for length calculations. This statement is optional and, if specified, can appear only once within a rule file. In Calibre, the default unit of length is 1 um (1e-6 m).

This statement controls the output of the UNIT_LENgth() function in built-in languages for LVS.

For geometric databases, if the Unit Length setting is inconsistent with the database precision, a warning is issued and the condition is ignored by default. You can alter this behavior by changing the Layout Input Exception Severity METRIC_RULE_FILE setting.

The Calibre nmDRC applications do not use the Unit Length statement for any constraint measurements.

In Pyxis Layout, the factor parameter and the value of the $unit_length Process variable do not have to match. If you do not include the Unit Length statement within a rule file, the default unit of length is equal to the value of the $unit_length Process Variable.

Examples

Example 1

Include the following statement in your rule file to specify the units of user specified distances for LVS device calculations in microns (um):

UNIT LENGTH u

Example 2

The following uses the scientific notation method of writing the previous Unit Length statement.

UNIT LENGTH 1e-6

Example 3

The Calibre SPICE netlisters are sensitive to Unit Length. Assume the following are netlisted measurements of the same instance:

UNIT LENGTH u: L=3.898e-06 W=7.201e-06

UNIT LENGTH 1: L=3.898        W=7.201

When using a value of 1 (which is the same as using M), the measurement values can be interpreted in absolute user units, as opposed to fractions of a meter, for special netlisting purposes. Be aware that SPICE calls for lengths to have dimensions of meters, so using 1 (or M) to output absolute user units is a deviation from normal SPICE.
