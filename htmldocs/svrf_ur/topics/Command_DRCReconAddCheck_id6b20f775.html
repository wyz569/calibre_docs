DRC Recon Add Check

Specification statement

Specifies rule check statements or rule check groups to be executed in a Calibre nmDRC Reconnaissance run.

Usage

DRC RECON ADD CHECK rule_check [rule_check …]

Arguments

rule_check

A required argument that specifies the name of a rule check statement or rule check Group. You can specify rule_check any number of times in one statement. The rule_check must be a selected rule check after resolution of any DRC Select Check and DRC Unselect Check statements, or their defaults.

The rule_check argument can also be one of the keywords in the following table. These keywords select a predefined set of rule checks. The Default column indicates whether the set of rule checks is executed in a default Calibre nmDRC Recon run.

Keyword

Definition

Default?

RECON_DEFAULT

All rule checks that would be executed by default in Calibre nmDRC Recon mode.

Yes

RECON_DENSITY

All selected rule checks that require a Density operation.

Yes

RECON_OFFGRID

All selected rule checks that require an Offgrid Directional or non-simple Offgrid operation. A non-simple Offgrid operation uses any of the following keywords: INSIDE OF LAYER, RELATIVE, ABSOLUTE, EDGE, HINT, or OFFSET.

No

RECON_CONNECT

All selected rule checks that require connectivity, including those with DFM DV operations.

No

RECON_DV

All selected rule checks that require a DFM DV operation.

No

RECON_MP

All selected rule checks that require a Calibre Multi-Patterning operation.

No

RECON_PM

All selected rule checks that require a Calibre Pattern Matching operation.

No

Description

Specifies a set of rule checks to run in Calibre nmDRC Recon mode. If DRC Recon Add Check is included in the rule file, only the specified rule checks are executed, plus any checks selected by DRC Recon Add Check By Layer statements. This statement may appear in the rule file any number of times. It is ignored when -recon is not specified on the command line.

By default, the Calibre nmDRC Recon mode chooses a set of rule checks from the rule file, which may or may not be the complete set of checks. See “Chip-Level Verification” in the Calibre Solutions for Physical Verification manual. You can specify a custom set of checks to execute with this statement and the related statements DRC Recon Remove Check, DRC Recon Add Check By Layer, and DRC Recon Remove Check By Layer.

The rule checks that are explicitly named must be selected for execution in a standard Calibre nmDRC run. For example, these two statements result in a compiler error:

DRC UNSELECT CHECK min_spacing_metal1
DRC RECON ADD CHECK min_spacing_metal1 RECON_DEFAULT
// Compiler error:
ERROR: Error SPC26 - DRC RECON ADD CHECK min_spacing_metal1 is not a recon RuleCheck

When using the rule check group keywords, it is not necessary for all rule checks that belong to the group to be selected. For example, this compiles successfully: 

DRC UNSELECT CHECK m1_netarea  //unselect one connectivity check
DRC RECON ADD CHECK RECON_DEFAULT RECON_CONNECT
// Result: All default Calibre nmDRC Recon checks are executed, plus all 
// connectivity checks except m1_netarea

Check selection for a Calibre nmDRC Recon run is done by this process:

  1. Determine the checks that are selected to run in a standard Calibre nmDRC run.

  2. From the set of checks selected in Step 1, enable checks for the Calibre nmDRC Recon run:

    • If no DRC Recon Add Check or DRC Recon Add Check By Layer statements exist in the rule file, enable the default set of Calibre nmDRC Recon rule checks.

    • If DRC Recon Add Check or DRC Recon Add Check By Layer statements exist in the rule file, enable only those rule checks.

  3. If DRC Recon Remove Check or DRC Recon Remove Check By Layer statements are in the rule file, disable those rule checks.

    This step may remove rule checks added in the previous step.

Examples

Also see “Selecting Checks in Calibre nmDRC Recon” in the Calibre Solutions for Physical Verification manual.

Example 1

Execute all checks selected by default by a Calibre nmDRC Recon run, plus connectivity checks:

DRC RECON ADD CHECK RECON_DEFAULT RECON_CONNECT

Example 2

Execute all checks selected by default by a Calibre nmDRC Recon run, plus the connectivity check m1_netarea, which would not otherwise be run. 

DRC RECON ADD CHECK m1_netarea
DRC RECON ADD CHECK RECON_DEFAULT

Example 3

Assume this statement is in the rule file and is the only such statement:

DRC SELECT CHECK met1.1 met1.2 met1.3

This statement causes the met1.1 check to be the only check run in Calibre nmDRC Recon mode:

DRC RECON ADD CHECK met1.1

The following statement results in a compiler error because met1.4 is not in the set of selected checks:

DRC RECON ADD CHECK met1.4
