DFM Property Net

Layer operation

License: Calibre nmDRC-H

Creates an empty polygon layer containing DFM properties calculated from one or more input layers having connectivity information and possibly DFM properties. Properties are reported per net in the output layer.

Usage

DFM PROPERTY NET layer [layer …]
[property_definition …]

where property_definition is:
'[' property_name = expression ']' [ ['!'] constraint] …

Arguments

layer

A required layer with connectivity node IDs. This layer may be the output of another DFM Property Net operation. Any number of connectivity layer arguments is allowed.

All node IDs to be indexed should be represented in the layer inputs.

property_definition

An optional argument in the form of an expression as used by other DFM Property operations. Refer to DFM Property for a detailed description of property_definition. Typically, the property_definition is used to define properties to be attached to the output layer.

If no property_definition is specified, the output layer is generated as usual but no properties are attached to it.

Description

Generates an empty polygon layer with attached DFM properties, if specified, where the properties are indexed by the node IDs present on the input layers. This type of layer information is useful when only DFM properties are needed on the output layer for downstream processing.

These are some aspects of the DFM Property Net operation:

  • The operation supports expression evaluation as with the DFM Property operation.

  • Property values are computed and indexed per connectivity node for objects on the input layers.

  • The output layer contains one empty, or virtual, polygon for each node ID that exists on the input layers. Net-indexed properties are attached to these virtual polygons if a property_definition is included. No physical geometries are included in the output layer.

  • Property values are stored in the cell containing each net’s root, where a net root is defined as the highest level in the hierarchy for the connectivity of a particular net.

The DFM Property Net operation is similar to the DFM Property operation with the NODAL keyword, although the concept of a primary layer is not used and no physical geometries are saved to the output layer.

The transcript output reports layer statistics as in the following example, where count_net is the output layer from the DFM Property Net operation:

NET-INDEXED LAYER  count_net:  VIRTUAL HGC=3 FGC=3
count_net (HIER-FMF TYP=1 CFG=3 HGC=0 FGC=0 HEC=0 FEC=0 IGC=0 VHC=T VPC=X)

In this example, there are three virtual geometries (VIRTUAL HGC=3). The second line indicates there are no physical geometries (HGC=0) on the layer count_net.

Using the Output Layer from DFM Property Net

You can do the following to access the net-indexed properties in the output layer created by DFM Property Net:

  • Use DFM Property with the NODAL keyword.

  • Use DFM Property Net.

  • Use the NIPROPERTY function in a DFM property computation.

The output layer created by a DFM Property Net operation can be used as an input layer to the COUNT function. COUNT returns 1 for each virtual geometry.

The output layer from DFM Property Net cannot be used in the measurement functions that calculate a geometric attribute from layer input.

The output layer from a DFM Property Net operation can be used as a layer argument to DFM Property or DFM Property Net.

Examples

Example 1

The following example assigns a property for connectivity layers “blue” and “green” using DFM Property Net and performs a subsequent DFM Property operation.

ni_prop = DFM PROPERTY NET blue green
            [count_green = COUNT(green)] 
            [count_blue = COUNT(blue)]
// ni_prop contains counts of geometries on each net of the blue and 
// green layers.
ni_select = DFM PROPERTY NET blue green
              [count_green = COUNT(green)]
              [count_blue = COUNT(blue)]
              [product = COUNT(green) * COUNT(blue)] > 8
// ni_select can be used for selection filtering.
nm_blue_ni = DFM PROPERTY blue ni_prop NODAL MULTI
               [count_ni = COUNT(ni_prop)]
               [count_green = PROPERTY(ni_prop, count_green)]
               [count_blue = PROPERTY(ni_prop, count_blue)]
// nm_blue_ni contains copies of blue layer shapes with three properties.
// The properties corresponding to the node ID of the primary layer shape 
// are read from the ni_prop layer.

Example 2

This example creates a net-indexed layer with a voltage property assigned per net. Text attachments are converted to voltage properties that are attached to layer shapes. The maximum voltage value per net is used, or a default value if no text attachment is present for a net. Similar derivations are used to construct the net-indexed layer that is used as input to the DFM DV operation. 

LAYOUT SYSTEM GDSII
LAYOUT PRIMARY TOP
LAYOUT PATH "example.gds"
DRC RESULTS DATABASE drc.results

LAYER M1 10 LAYER MAP 1 DATATYPE 0 10
LAYER V1 20 LAYER MAP 2 DATATYPE 0 20
LAYER M2 30 LAYER MAP 3 DATATYPE 0 30
// text layers with voltages
LAYER M1V 11 LAYER MAP 1 TEXTTYPE 1 11
LAYER M2V 31 LAYER MAP 3 TEXTTYPE 1 31

M1T = DFM TEXT M1V PROPERTY NUMBER v
M2T = DFM TEXT M2V PROPERTY NUMBER v
// Create a polygon layer with property v attached to M1 and M2 shapes
// if a shape has more than one text attachment, use the max value.
M1M = DFM PROPERTY MERGE M1 M1T
   [-=COUNT(M1T)] >0
   [v=MAX(PROPERTY(M1T,v))]
M2M = DFM PROPERTY MERGE M2 M2T
   [-=COUNT(M2T)] >0
   [v=MAX(PROPERTY(M2T,v))]
// M1M and M2M only include shapes with a text attachment.
CONNECT M1 M2 BY V1
CONNECT M1 M1M
CONNECT M2 M2M
// Create a net-indexed layer MP with max voltage per net.
// Use a default value of -100 for MAX to handle cases in which 
// a net has the property v for only one of M1M or M2M, but not both. 
MP = DFM PROPERTY NET M1M M2M
         [v=FMAX( MAX(PROPERTY(M1M,v),-100), MAX(PROPERTY(M2M,v),-100) )]
// Create a net indexed layer with voltage values per net.
// Include layer MP with voltage property v from text attachments.
// Include layers M1 and M2 so that nets without text attachments 
//        are indexed and assigned a default voltage value. 
// Use voltage from layer MP, or the value 4.0 if no voltage is assigned.
MN = DFM PROPERTY NET M1 M2 MP
         [v=(COUNT(MP)>0) ? PROPERTY(MP,v) : 4.0]
debug_all {
   out_all  = DFM PROPERTY M1 MN NODAL MULTI
      [v=PROPERTY(MN,v)] [Node = GLOBALNETID(M1)]
   DFM RDB out_all dfm.rdb CHECKNAME "%_l_"
}

If the derivation of MN is this instead, without the M1 and M2 layers:

MN = DFM PROPERTY NET MP
         [v=(COUNT(MP)>0) ? PROPERTY(MP,v) : 4.0]

then the top left shape on the M1 layer is not assigned a voltage property because the layer MP does not include the node ID of that shape.

Example 3

This example shows a way to cluster output by net and get only one result per net. DFM Property Net is used to report properties per net. DFM Property with NODAL and NOMULTI is used to output one geometry per net.

The M2_area and M1_area properties contain the total areas per net for layers M2 and M1. The DFM Property operation outputs one M2 geometry per net with attached properties. 

CONNECT M1 M2 BY V1
check2 { 
area_per_net = DFM PROPERTY NET M2 M1  
    [M2_area = AREA(M2)]
    [M1_area = AREA(M1)]
 
M2_w_areas = DFM Property M2 area_per_net NODAL NOMULTI
                [M2_area = PROPERTY(area_per_net, M2_area)] != 0 
                [M1_area = PROPERTY(area_per_net, M1_area)]
 
DFM RDB M2_w_areas area_out.rdb CHECKNAME "%_l_"
}
Note:

The string SELECT_ONE_POLYGON_PER_NET for the Net operation is another way to select one polygon per net. This can be useful in some situations, but would require an additional layer derivation in this example.
