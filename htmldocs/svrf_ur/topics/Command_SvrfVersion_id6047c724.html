SVRF Version

Specification statement

Defines the earliest Calibre version that is permitted to compile the rule file.

Usage

SVRF VERSION “version_id”

Arguments

“version_id”

A quoted string that identifies a Calibre version. The string is of this form:

vDDDD.D_D*.D*

where D is any decimal digit, D* is one or more digits, and the other characters are literals.

Description

Specifies that the Calibre version used for the run matches the version_id, or is a later version. Attempting to use a version of Calibre that is earlier than the version_id results in a compiler error as shown here:

CALIBRE version v2014.1_82.69 predates SVRF VERSION v2015.1_15.11

ERROR: Error VER2 on line 25 of rules - SVRF VERSION error.

This statement may be specified multiple times, in which case the latest version specified is the earliest version that can be used.

Beta and customer‑specific version numbers are truncated prior to any comparison with the specified “version_id”.

Specifying Calibre versions earlier than 2008.3_16.12 is not useful because this statement did not exist prior to this version, and it causes a compiler error in earlier versions.

Examples

// use this Calibre version, or later
SVRF VERSION "v2008.3_16.12"
