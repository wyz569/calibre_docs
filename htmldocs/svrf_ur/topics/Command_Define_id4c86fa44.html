#DEFINE

The #DEFINE pre-processor directive defines variables in the SVRF rule file. The variables are used to control conditional execution of portions of the rule file.

Usage

#DEFINE var_name [var_value]

Arguments

var_name

A string giving the variable name. You should enclose var_name within quotes if var_name includes spaces or characters other than letters and numbers.

var_value

A string giving an optional value for the variable.

Description

A variable defined with a #DEFINE statement can be used in an #IFDEF or #IFNDEF statement to control conditional execution of rule file statements. See “About Conditional Directives for Compilation Control” for complete information on using the conditional directives, including the handling of variables defined in the shell and information on compilation order.

Do not comment out a #DEFINE statement to change the state of a conditional statement; use the #UNDEFINE statement.

Examples

The following construct establishes “process” as defined, with no assigned value:

#DEFINE process

The following construct defines the variable name “process” and assigns it the value “7lm”: 

#DEFINE process 7lm
