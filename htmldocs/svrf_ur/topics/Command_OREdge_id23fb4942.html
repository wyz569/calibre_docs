OR Edge

Layer operation

Boolean operation that constructs the union of two edge layers.

Usage

OR EDGE layer1 layer2

Arguments

layer1

A required derived edge layer.

layer2

A required derived edge layer.

Description

Performs a Boolean OR on edges from the input layers. This operation is a non-node-preserving layer constructor.

The definition of OR Edge is as follows. This operation:

z = OR EDGE x y

is functionally equivalent to this:

a = x NOT COINCIDENT EDGE y
b = y NOT COINCIDENT OUTSIDE EDGE x
z = a U b // the union of a and b

The OR Edge operation does not merge two edges or edge segments when they face opposite directions, as shown in the following figure. Edges “a” and “b” are produced by a Touch Edge operation, where edge “a” faces left and edge “b” faces right. (This behavior is not specific to Touch Edge, as any layer operation that produces derived edges assigns directions that the edges face.)

Figure 1. OR Edge with Oppositely Facing Edges

The output from OR Edge in these situations can cause unexpected results from layer derivations that use that output. To merge coincident edge segments that face opposite directions, use DFM OR Edge BOTH instead of OR Edge.

See also OR, Coincident Edge, Not Coincident Edge, Not Coincident Outside Edge.

Examples

The following example shows a layer derivation of ngate and pgate edges using OR Edge. 

ndiff = diff AND nplus
pdiff = diff AND pplus
gate = poly and diff

ngate_edge = gate COIN EDGE ndiff
pgate_edge = gate COIN EDGE pdiff

gate_edge = ngate_edge OR EDGE pgate_edge

Figure 2 shows the derived edge layer gate_edge from the above example. For clarity, the original layers diff, nplus, and pplus are not shown in the figure.
