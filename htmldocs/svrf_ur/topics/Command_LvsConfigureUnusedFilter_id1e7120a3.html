LVS Configure Unused Filter

Specification statement

Specifies a cell-specific unused device filter.

Usage

LVS CONFIGURE UNUSED FILTER filter_name [filter_option]
[LAYOUT | SOURCE | LAYOUT SOURCE]
[{DEVICES component_type[‘(’component_subtype‘)’]} …]

Arguments

filter_name

A required name of an unused device filter.

filter_option

An optional filter parameter taken from the LVS Filter Unused Option specification statement. If this parameter is not specified, it means no device filtering is applied to cells referenced in a corresponding LVS Apply Unused Filter statement.

LAYOUT

An optional keyword that specifies the statement applies to the layout design.

SOURCE

An optional keyword that specifies the statement applies to the source design.

Note:

The combination LAYOUT SOURCE is used by default if neither keyword is specified.

DEVICES component_type[‘(’component_subtype‘)’]

An optional keyword set that specifies filtering is performed only on the component_type and component_subtype. At least one component_type is required when DEVICES is specified. The component_type corresponds with a device element name and the component_subtype with a model name. The component_type and component_subtype parameters may be specified more than once. The parentheses are required as shown.

Description

Defines a device filter used during LVS comparison. This statement is used in conjunction with LVS Apply Unused Filter and LVS Cell List to perform cell-specific unused device filtering. The LVS Configure Unused Filter statement sets up the filter. The LVS Apply Unused Filter statement defines the cell list to which the filter applies. The LVS Cell List statement defines the cells in the cell list. The filtering (aside from being cell-specific) is applied similarly to the LVS Filter Unused Option specification statement.

If no filter_option is specified, then the corresponding filter_name acts as an empty filter. This means any cells that have the filter_name applied to them receive no unused device filtering by any applicable LVS Filter Unused statement.

This statement can be specified any number of times. Options and DEVICES specifications are accumulated from all statements. There is no default.

Examples

Example 1

This example shows a basic setup of cell-specific device filtering.

HCELL cellA cellA
HCELL cellB cellB
HCELL cellC cellC

LVS CONFIGURE UNUSED FILTER my_filter RC
LVS APPLY UNUSED FILTER my_filter CELL LIST list1
LVS CELL LIST list1 cellA cellB cellC

This set of statements filters all resistors having pos and neg pins tied together and inside cellA, cellB, and cellC in both layout and source.

Example 2

This example shows how to set up an empty filter for specific cells.

LVS FILTER UNUSED OPTION RC           // global filter.

HCELL cellA cellA
HCELL cellB cellB
HCELL cellC cellC

LVS CONFIGURE UNUSED FILTER my_filter // no filter option provided! 
                                      // any cells using my_filter
                                      // receive no device filtering.
LVS APPLY UNUSED FILTER my_filter CELL LIST list1
LVS CELL LIST list1 cellA cellB cellC

This series of statements causes the RC filter option not to be applied in cellA, cellB, or cellC in both layout and source, but option RC is applied in other cells.
