Flatten

Layer operation

For flat applications, this operation is equivalent to a Copy operation. For hierarchical applications, this operation constructs a flat copy of the input layer.

Usage

FLATTEN layer

Arguments

layer

An original layer or layer set, or a derived polygon or edge layer.

Description

For Calibre nmDRC-H, the resulting layer is a flat copy of the input layer. For flat Calibre nmDRC and ICrules, this is equivalent to a Copy operation.

This operation has limited use. Flat input to any layer operation produces flat output, which can have an adverse impact on computation time and memory usage. Flat input is also problematic in a two-layer operation where exactly one of the input layers is flat, because the tool must internally flatten the other input layer so that the operation itself can be done flat. This situation is indicated by the following warning for the non-flat layer:

     WARNING: Creating a FLAT COPY of layer name.
Note:

No flattened temporary layers may be used in any of the DRC functions called from runtime TVF. See TVF Function and the TVF layer function.

If you want to modify a DRC results layer so that results appear in the same hierarchical context, use Merge instead of Flatten.

See also Push, Flatten Cell, and Flatten Inside Cell.

Examples

This example gets around the natural error suppression in the hierarchical application:

rule6.A { @ Poly spacing to diff = 1u. Show ALL flat errors.
    X = EXT poly diff < 1  ABUT < 90  SINGULAR  REGION
    FLATTEN X
}

Ordinarily, only the lowest-leftmost placement containing layer X would be output. Flattening X causes the results to appear in all placements.

Note that polygon-directed output (REGION keyword) is required in the External operation because error-directed output cannot be saved in a derived layer.
