PEX Reduce Via Resistance

Parasitic extraction

Specifies how to handle clusters of identically-connected vias when calculating resistance. Used only in Calibre xRC and Calibre xACT 3D PDB flow.

Usage

PEX REDUCE VIA RESISTANCE [layer1 layer2] {
{STANDARD [COUNT max] [DISTANCE value]}
| {FLEXIBLE [DISTANCE value] [MINSTEP value]}
| OFF
}
[POWERNETS] [GROUNDNETS] [BY NET toplevelNetName1 [toplevelNetName2 ... ]]

Arguments

layer1 layer2

An optional keyword pair specifying a particular set of vias to which the statement applies. The layers must be connected to the vias and have connectivity and resistance. If layer1 layer2 are not present, the rule applies to all vias in the design.

STANDARD [COUNT max] [DISTANCE value]

A required keyword that specifies standard via reduction. A cluster of vias is reduced to one via.

The optional keyword set COUNT max limits the number of vias that are reduced to a single via. The default is no limit.

The optional keyword set DISTANCE value specifies the maximum distance in microns between vias used to form a cluster. The default is 5 microns.

FLEXIBLE [DISTANCE value] [MINSTEP value]

A required keyword that specifies flexible via reduction. FLEXIBLE reduction takes the original via cluster, breaks it up into rectangular arrays, and then reduces all of the vias to four vias, located on the top, left, bottom, and right of each rectangular section. FLEXIBLE reduction is equivalent to FLEXIBLE MINSTEP 0.

The optional keyword set DISTANCE value specifies the maximum distance in microns between vias used to form a cluster. The default is 5 microns.

The optional keyword set MINSTEP value breaks the via array into square arrays (n x n), where n is the original width of the via array. It breaks it into as many squares as possible. Any remaining shapes are rectangular. Within each of these regions, all vias are reduced to one via that is geometrically as close to the center of the region as possible. The value specifies the minimum number of vias that are reduced to a single via. If the FLEXIBLE keyword is specified without the MINSTEP keyword, the default value for MINSTEP is 0.

If PEX Reduce Via Resistance statement is not specified in the rule file, then PEX Reduce Via Resistance FLEXIBLE MINSTEP 3 is enabled by default.

OFF

A required keyword which turns off via reduction. Turning via reduction off will create a netlist that contains resistors representing all vias in the layout. Using this keyword may cause a substantial increase in parasitic database and netlist size. To prevent diffusion resistors from appearing in the output netlist, set PEX Netlist Unshort Device Pins NO. It is recommended to use this keyword when specifying layers. This keyword is not recommended for use on designs with large via clusters.

POWERNETS

An optional keyword that specifies to reduce vias on all power nets specified with LVS Power Name. If this keyword is specified and the LVS Power Name statement is not specified, Calibre xRC stops with an error. Use only with Calibre xRC.

GROUNDNETS

An optional keyword that specifies to reduce vias on all ground nets specified with LVS Ground Name. If this keyword is specified and the LVS Ground Name statement is not specified, Calibre xRC stops with an error. Use only with Calibre xRC.

BY NET toplevelNetName1 [toplevelNetName2 ... ]

An optional keyword set that specifies how via reduction is handled for one or more distinct top level nets. If the net name is not found in the top level, then a warning is generated and the reduction mode specified for the net is ignored. Use only with Calibre xRC.

Description

Specifies how to handle clusters of identically-connected vias when calculating resistance. (It does not affect capacitance extraction.) Use this operation without layers to indicate a default reduction for all vias, and also override it for particular via layers with additional statements indicating connecting layer pairs. This statement can be specified more than once; see Example 5.

If PEX Reduce Via Resistance statement is not specified, then the default reduction mode is FLEXIBLE MINSTEP 3 for all connect layers on all nets.

During PDB generation, the extraction software clusters adjacent vias, potentially subdivides the cluster into groups, and reduces each group to a single resistor in standard via reduction or four edge-located resistors in flexible via reduction as shown in Figure 1.

The optional DISTANCE value keyword set identifies clusters; vias that are within the limit distance are considered to be part of the cluster, and vias further away are not. For example, if value had been set too large in Figure 1, the 2x4 via block would have been treated as part of the larger cluster. For most applications, a typical reduction limit is five times the maximum via spacing.

This statement supports three distinct modes of reduction that are enabled with the STANDARD, FLEXIBLE, and OFF keyword sets. When defining very specific rule, it is recommended that you use the OFF keyword to turn off reduction then define one or more new reduction rules to tune your reduction settings and override defaults. The succession of statements allows more reduction to occur given more specific reduction criteria. If the more general rule (parent rule) and the more specific rule (child rule) have the same or similar options, then Calibre xRC generates a warning and ignores the more specific rule.

For FLEXIBLE via reduction, groups are determined by geometry. As illustrated in Figure 1, each group is a rectangle, square, or line. Vias that could be included in more than one rectangle form their own group. Each group is then reduced to four vias, which are placed around the outer edge of the group. (For lines, only two points are used.) The optional keyword set MINSTEP value controls how many vias in the cluster are kept. A value of 1 is the most conservative and generates a larger netlist with more vias. A larger value is less conservative and generates a smaller netlist as the number of vias are reduced. The default MINSTEP value is 0.

For STANDARD via reduction, grouping and reduction is controlled with the optional keyword set COUNT max. The location of the via halfway through max is used for the reduced vias. This location may be at an edge of the overall cluster or in the middle. The default is to reduce any cluster to one node, in the approximate center. A cluster is defined by default as a group of vias with a gap of at least 5 microns to the next group of vias. (The cluster itself may extend considerably beyond 5 microns.)

When two vias are leading up and down from the metal layer and are overlapping by at least half their width, the resistors representing those vias connect to the same node inside the metal polygon.

Examples

Example 1

To make the default behavior explicit in your rule file, add the following line:

PEX REDUCE VIA RESISTANCE FLEXIBLE MINSTEP 3

Example 2

To use flexible via reduction by default, but minimize via reduction for contacts in order to analyze sensitive connections, the rule file might contain the following lines:

//Derive layers using seed shapes on layer "crit_area". 
//Connectivity and resistance inherited from ml and poly respectively.
top_preserve = m1 AND crit_area
bot_preserve = poly AND crit_area
PEX REDUCE VIA RESISTANCE FLEXIBLE
PEX REDUCE VIA RESISTANCE top_preserve bot_preserve OFF

Example 3

To turn off via reduction for all layers add the following line to your rule file:

PEX REDUCE VIA RESISTANCE OFF

Example 4

To turn off via reduction between layers metal1 and metal2, add the following line to your rule file:

PEX REDUCE VIA RESISTANCE metal1 metal2 OFF

Example 5

This example demonstrates the precedence of this rule when specified more than once in your rule file. The statement with the most specific settings has the highest precedence. Consider the following set of PEX Reduce Via Resistance rules:

PEX REDUCE VIA RESISTANCE STANDARD COUNT 2
PEX REDUCE VIA RESISTANCE M1 M2 OFF
PEX REDUCE VIA RESISTANCE M1 M2 FLEXIBLE MINSTEP 3 BY NET "VDD" "VSS"

All connect layers between M1 and M2 are reduced with MINSTEP 3 for VDD and VSS nets only. Via reduction is turned off for connect layers between M1 and M2 for the remaining nets. All other nets on all other layers use STANDARD via reduction.

Example 6

To apply via reduction to all power nets specified with LVS Power Name specify the following statement:

LVS POWER NAME VDD VCC
PEX REDUCE VIA RESISTANCE OFF POWERNETS

Example 7

To apply via reduction to all ground nets specified with LVS Ground Name statement specify the following statement: 

LVS GROUND NAME VSS VSS1
PEX REDUCE VIA RESISTANCE OFF GROUNDNETS

Example 8

If only the following detailed form of the statement is specified in your rule file:

PEX REDUCE VIA RESISTANCE FLEXIBLE MINSTEP 3 BY NET VDD1

Calibre xRC compares the reduction mode and options in this statement to the default mode of FLEXIBLE MINSTEP 3, and determines that since the modes are similar and OFF has not been specified, the default mode takes precedence, a warning is generated, and this detailed statement is ignored.
