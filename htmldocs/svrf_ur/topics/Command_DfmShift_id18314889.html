DFM Shift

Layer operation

License: Calibre YieldEnhancer

Shifts the input layer based on DFM properties attached to the input layer or explicit numeric values.

Usage

DFM SHIFT layer BY x_prop_or_value y_prop_or_value

Arguments

layer

A required original layer, derived polygon layer, derived edge layer, or derived error layer.

BY x_prop_or_value y_prop_or_value

A required keyword set, where the strings x_prop_or_value and y_prop_or_value are either the names of numeric properties on layer or explicit numeric values. The values of the shift parameters may be positive, negative, or zero. Negative numbers must be enclosed in parentheses ().

Description

The DFM Shift operation translates the input layer based on DFM properties attached to the input layer or explicit numeric values. The DFM Shift operation is similar to the Shift operation except that numeric DFM properties can be used to provide the shift values. See “DFM Properties”.

The translation of each object is in the global frame of reference; for example, a positive value for x_prop_or_value means translation to the right in the top cell.

Note:

The shifted polygon will always be the exact size of the original polygon. That is, there is not any round-off error if the grid is not exact.

The DFM Shift operation supports multithreaded MT, Calibre MTflex, and Hyperscaling—see the Calibre Administrator’s Guide for complete information.
