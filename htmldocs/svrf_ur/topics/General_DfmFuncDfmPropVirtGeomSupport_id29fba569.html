DFM Property Operations with No Geometric Output: Function and Expression Support

The built-in function and DFM expression support for the DFM Property operations with no geometric output is summarized here.

The DFM Property operations with no geometric output are the following:

  • DFM Property Net

  • DFM Property Singleton

Use of Output Layers with No Geometric Data in DFM Functions

There are some restrictions on the use of layers created by DFM Property Net and DFM Property Singleton as layer arguments to DFM functions because the output layer of these operations contains only virtual geometries, and no real geometries. In general, the functions that operate on geometric layer data cannot be used on input layers that were generated by DFM Property Net and DFM Property Singleton.

Table 1. Support for Virtual Geometry Layers as Input to DFM Functions

Function or Function Group

Comment

Measurement functions

Only the COUNT function can be used on derived layers with no geometric shapes.

ECBEGIN, ECEND, ECMIN, ECMAX

Not supported.

Per-shape functions

The per-shape functions are not useful on the layers created by DFM Property Singleton because there is at most one virtual geometry on such layers. Similarly, when a layer created by DFM Property Net is used with NODAL clustering, there is at most one virtual geometry per cluster. However, the use of per-shape functions on these layers does not generate a compiler error.
