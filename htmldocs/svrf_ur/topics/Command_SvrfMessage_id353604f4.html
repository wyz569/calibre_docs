SVRF Message

Specification statement

Issues a message in the run transcript when this statement is encountered during rule file compilation.

Usage

SVRF MESSAGE “message”

Arguments

“message”

A quoted character string that is printed in the run transcript at compilation time. Variable resolution is supported if a character sequence within the string constant is preceded by the “^” character.

Description

This specification statement prints the given message in the run transcript. This statement may be specified any number of times. The message printed has the following form:

SVRF MESSAGE on line <line-number> of <file-name> - <message text>

where <message text> is the “message” parameter of the SVRF Message statement. For example:

SVRF MESSAGE on line 5 of rule_file - "Calibre nmDRC is fun to use."

It appears in the STANDARD VERIFICATION RULE FILE COMPILATION MODULE portion of the run transcript.

See also SVRF Error for cases in which you want the run to halt with a compilation error.

Variable Resolution

Any non-empty sequence of characters within the string constant preceded by the caret (^) character and terminated by a space, caret (^), backslash (\), or the end of the string is interpreted as the name parameter in a Variable statement. The variable is resolved and its value is substituted in the message for its variable name.

If a variable is defined as an array of values, such as this:

VARIABLE "scale" "um" "x1000 dbu"

only the first value of the array is accessed by ^.

The caret (^) character may be escaped (treated as literal) by preceding it with two backslashes (\\).

The following conditions report a compiler error:

  • The variable is not resolvable and reports compiler error RES8.

  • The variable is not resolvable as a numeric or string type and reports compiler error RES7.

  • The variable is not resolvable as a string array and reports compiler error RES8.

Examples

// echo this message during the run transcript
SVRF MESSAGE "Starting metal width checks"
