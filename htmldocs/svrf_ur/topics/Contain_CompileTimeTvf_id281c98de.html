Compile-Time TVF

Compile-time TVF appears in a special rule file that has a script of compile-time TVF commands. This script is interpreted and translated directly into SVRF code, one line at a time, upon execution of the Calibre command. Compile-time TVF does not access data during the Calibre run.

The TVF script filename is passed to Calibre just like any other rule file name. If you have an SVRF rule file, you can use the Include statement to include a compile-time TVF file; see the Include command for additional details. You can specify a compile-time TVF file as the rule file in Calibre Interactive.

Compile-time TVF commands allow you to do two particularly useful things:

  • Write Tcl procs that can be repeatedly called.

  • Write loops that generate multiple SVRF rule checks in a few lines of code.

The main advantage of compile-time TVF over SVRF is that you can write your rule file in fewer lines of code using TVF constructs such as loops.
