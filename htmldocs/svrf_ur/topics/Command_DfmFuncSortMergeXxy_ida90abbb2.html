SORT_MERGE_XXY

Vector property function for DFM expressions

Computes minimum measurement values associated with coordinate intervals contained in a set of input vectors. Outputs the minimum measurement values with associated interval coordinates in vector form.

Usage

SORT_MERGE_XXY(vector[, vector …])

Arguments

vector

A required vector argument containing at least one tuple of the form <c1 c2 m> where c1, c2, and m are floating-point numbers, as follows:

  • c1 — The starting x- or y-coordinate in user units of a linear interval.

  • c2 — The ending x- or y-coordinate in user units of a linear interval.

  • m — A measurement value in user units associated with the interval.

If more than one vector argument is specified, each vector must be separated by a comma (,). A vector argument may be a comma-delimited list of tuples of the proper form. Conceptually, if “myvec = {<a b c> <d e f>}”, where “a” through “f” are numeric values, then myvec can be properly formed as a vector argument to the function.

Return Values

Vector string containing two-element tuples of the form <c m>, where c is an interval endpoint coordinate in user units, and m is an associated measurement value in user units.

Description

This function is designed for use when there are multiple edge spacings to be analyzed for a given edge. This function may only be used in a DFM Property operation expression.

An intended use is in device property computations where enclosure spacings between edges on one layer (like gate) and edges on another layer (like well) vary. Here is an example showing vertical edge intervals:



The spacing measurements (sN values) and the coordinate intervals (based on yN coordinates) over which those spacings exist are stored in vectors that are inputs to SORT_MERGE_XXY. Input vectors are of the form <c1 c2 m> when c1 and c2 define interval endpoint coordinates (yN values in the figure) and m is a measurement value (sN values in the figure). The input vectors are typically constructed by other DFM expression functions like ECMIN, ECMAX, EW, and VECTOR.

SORT_MERGE_XXY analyzes input vector measurement values over all the intervals defined by the input vectors. Each vector argument tuple specifies a half-open interval [c1, c2) with an associated measurement value m. Overlapping intervals in the input arguments are permitted. For each interval, the minimum measurement value across all input vectors is associated with an interval endpoint coordinate.

For example, assume these input vectors:

vec1 = {<0 1 1.5> <2 3 0.5>}
vec2 = {<0.5 1.5 1>}

These vectors can be represented graphically as y-intervals with associated measurement values as shown here.

Figure 1. Vector Input Intervals to SORT_MERGE_XXY

For illustration, these vectors can be represented in a layer operation as follows. (The VECTOR function numeric values are shown as constants simply for clarity. Typically they are calculated by DFM expression functions.)

prop_L = DFM PROPERTY L
         ["+vec1" = CONCAT(VECTOR(0,1,1.5),VECTOR(2,3,0.5))] 
         ["+vec2" = VECTOR(0.5,1.5,1)]
         [input_vector = CONCAT(
                           VPROPERTY_REF("+vec1"),VPROPERTY_REF("+vec2")
                         )
         ]  
         [sort_vector = SORT_MERGE_XXY( CONCAT(
                          VPROPERTY_REF("+vec1"),VPROPERTY_REF("+vec2")
                        ))
         ]

The input_vector and sort_vector properties in a results database are these:

input_vector <0 1 1.5> <2 3 0.5> <0.5 1.5 1>
sort_vector <0 1.5> <0.5 1> <1.5 0> <2 0.5> <3 0>

Recall that <0 1 1.5> <2 3 0.5> is vec1 and <0.5 1.5 1> is vec2. The explanations of the sort_vector tuples are as follows:

<0 1.5>

At y=0, the only measurement value to consider is 1.5.

 
<0.5 1>

At y=0.5, the minimum measurement of 1 comes from vec2.

no tuple

At y=1 (from vec1) there is no change in the minimum measurement, so there is no vector output for that coordinate.

 
<1.5 0>

At y=1.5 (from vec2), the measurement value is not defined (the interval is open at that point), so a value of 0 is assigned.

 
<2 0.5>

At y=2 (from vec1), the only measurement value is 0.5.

 
<3 0>

At y=3 (from vec1) the final interval coordinate is reached and a measurement value of 0 is always assigned.

The return value is a vector of this form:

<c1 m1> <c2 m2> ... <ci mi> ... <cn 0>

where each ci is a coordinate of an interval endpoint, and each mi is the minimum measurement value associated with the coordinate across all input vectors. The ci values are sorted in ascending order. If no measurement value is defined for some ci, then mi is 0. If the minimum measurement value at some ci does not change from a preceding ci, then that coordinate is skipped in the output. An mi of 0 is always returned in the final tuple of the returned vector. The final cn value is always the maximum interval endpoint coordinate across all input vectors.

This function’s output is often used as input to RANGE_XXY to obtain the desired set of vector tuples corresponding to a defined range of coordinate values.
