WARN

Effective property computation language function.

Issues a warning to the transcript each time the effective property computation program attempts an impossible calculation.

Usage

WARN

Arguments

None.

Return Values

None.

Description

By default, only the first impossible mathematical computation is reported by the tool as discussed under DEBUG. The WARN statement reports all impossible cases.

When any impossible mathematical computations are performed (such as division by zero), the result of the computation is set to the unknown value. If such a value is reported as a discrepancy, it appears as “?” in the flat LVS Report file.

Note that attempting an impossible mathematical computation is not considered an error that stops LVS comparison.

If you specify the WARN statement, then it must appear in the effective property computation procedure after the EFFECTIVE, EFFECTIVE STRING, and optional DEBUG statements.

Examples

See the example under “DEBUG”.
