LVS SI Select Connects

Specification statement

Selects Connect and Sconnect operations to participate in stand-alone short isolation. Used only in Calibre nmLVS Reconnaissance Short Isolation (Calibre nmLVS Recon SI).

Usage

LVS SI SELECT CONNECTS {ALL | {BY LAYER layer [layer …]} |
BY {layer_pair … | layer_triple … | {layer_pair … layer_triple …}}}

Arguments

ALL

Keyword that specifies all Connect and Sconnect operations in the rule file are selected. May not be specified with BY or BY LAYER.

BY LAYER layer [layer …]

Keyword set that specifies a layer name that appears in a Connect or Sconnect operation in the rule file. At least one layer parameter must be specified. The “*” wildcard may be specified in a layer name, and the wildcard matches zero or more characters. Connect and Sconnect operations containing any of the specified layers are selected. May not be specified with ALL or BY.

BY

Keyword and at least one layer_pair or layer_triple argument set as follows:

  • layer_pair

    Argument set that specifies a pair of layers that appears in a Connect or Sconnect operation of the two-layer form (no BY keyword with a contact layer) in the rule file. More than one layer_pair may be specified. A layer_pair has this form:

    ‘[’ layer1 layer2 ‘]’

    The brackets ([ ]) are literal. Connect and Sconnect operations containing any of the specified layer pairs are selected.

    This argument set may be specified with layer_triple.

  • layer_triple

    Argument set that specifies a triplet of layers that appears in a Connect BY or Sconnect BY operation in the rule file. More than one layer_triple may be specified. A layer_triple has this form:

    ‘[’ layer1 layer2 BY layerC ‘]’

    The brackets ([ ]) are literal. Connect BY and Sconnect BY operations containing any of the specified layer_triple arguments are selected.

    This argument set may be specified with layer_pair.

This argument set may not be specified with ALL or BY LAYER.

Description

Selects Connect or Sconnect operations to participate in stand-alone short isolation. This statement only applies in the Calibre nmLVS Recon SI flow. See the -recon -si option definition in the Calibre Verification User’s Manual for related details.

By default, layers that connect to device pins are processed in the Calibre nmLVS Recon SI flow as connectivity layers on which shorts can be detected. The LVS SI Select Connects statement changes this behavior so that the Connect and Sconnect operations matching the layers specified by the statement are processed. The BY and BY LAYER keywords are generally more restrictive than the default behavior. The ALL keyword is completely permissive.

This statement may be specified more than once. This behavior applies:

  1. If a specified layer does not appear in the rules, it is a compiler error.

  2. The set of all LVS SI Select Connect statements is aggregated and the settings evaluated against the set of active Connect and Sconnect statements in the rule file.

  3. An EXT16 compiler error is given if a Connect or Sconnect operation cannot be matched to the layers in the BY keyword set. An EXT3 compiler error is given for a similar situation with the BY LAYER keyword set.

  4. The run proceeds with the selected mappings.

Running short isolation with a more limited set of Connect statements can improve runtime, but this can limit the number of layers upon which shorts are detected.

This statement is analogous to the short_db::select_connects command in the Query Server Tcl shell.

Using the LVS SI Select Connects statement can limit the connectivity layers stored in the PHDB. This, in turn, limits the number of connectivity layers accessible by Query Server Tcl shell commands. When this occurs, the following warning appears in the transcript:

Warning: LVS SI SELECT CONNECTS is specified in batch rules.

In this case, be aware that if -si=tcl_script is specified on the command line, short_db::select_connects can only select from connectivity layers (BY LAYER) and original Connect and Sconnect operations (BY with a list) available in the PHDB, which may be a subset of the complete rule file connect set. The complete set of original LVS SI Select Connects statements maps to which layers and operations are available to the Query Server.

Examples

Assume the following operations in the rules:

CONNECT poly gate
CONNECT poly m1 BY cont
CONNECT m1 m2 BY via1

If this statement is specified:

LVS SI SELECT CONNECTS BY [m1 m2]

then an EXT16 compiler error is given because the statement matches no two-layer Connect operation. But if this statement is specified:

LVS SI SELECT CONNECTS BY [m1 m2 BY via1]

then the third Connect operation is selected and processed by short isolation.

The following statement selects the final two Connect operations:

LVS SI SELECT CONNECTS BY LAYER m1

The following statement does the same thing as the previous one (and could match more layers):

LVS SI SELECT CONNECTS BY LAYER m*
