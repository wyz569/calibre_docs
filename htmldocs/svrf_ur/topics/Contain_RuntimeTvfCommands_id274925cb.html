Runtime TVF Commands

Runtime TVF commands consist of layer functions, supported layer operations, variable functions, and application specific functions.

General runtime TVF command syntax is similar to compile-time syntax, as shown in “Compile-Time TVF File Format and Syntax”. TVF commands are case-sensitive, and both upper- and lowercase command names are available.

The following types of commands are allowed within a TVF Function definition:

  • Layer functions — The section “Layer Functions in Runtime TVF” gives the layer functions that are allowed within a TVF Function definition. These functions are allowed in Calibre nmDRC when using a TVF Layer Operation.

  • Layer operations — The section “Layer Operations in Runtime TVF” gives the SVRF layer operations that are allowed within a TVF Function definition. These operations are allowed in Calibre nmDRC when using the TVF Layer Operation.

  • Variable functions — Two functions are defined for accessing rule file variables and system variables in runtime TVF. See these sections:

    “Rule File Variables in Runtime TVF”

    “System Variables in Runtime TVF”

  • Calibre PERC functions — Calibre PERC uses the TVF Function statement to define rule check procedures; see the Calibre PERC User’s Manual for the supported functions.

  • Device property computation functions — The Device property computation built-in language supports calls to Tcl procedures defined in TVF Function statements. See “Device Property Computation Function Reference” for a list of supported functions.

Other operations and statements are not allowed within a TVF Function statement.
