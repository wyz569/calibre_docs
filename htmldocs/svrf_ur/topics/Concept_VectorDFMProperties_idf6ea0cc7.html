Vector DFM Properties

A vector is a two-dimensional array (or matrix) of numbers composed of tuples. Some DFM properties may be vectors.

A vector tuple can be thought of as a “row,” and the tuple size can be thought of as the number of “columns” in a row. The number of tuples in a vector is arbitrary, as is the tuple size. The tuple size is constant across all tuples in a vector.

Within this documentation, the following conventions are followed with respect to vectors:

  • { } — Notation used to indicate a vector.

  • < > — Notation used to indicate a tuple within a vector. Arguments within a tuple are separated by a single space. Tuples are separated by a single space when there is more than one tuple in a vector.

  • Argument — A single value within a tuple, sometimes called an element.

  • Tuple — A set of arguments enclosed in brackets (< >). Corresponds to a row in an array. Tuples are indexed from 0, so tuple 1 corresponds to the second tuple.

  • Tuple Size — The number of arguments in each tuple in the vector. This corresponds to the number of columns in the array.

  • Slice — Used interchangeably with the word column. A slice is a particular column of a tuple or set of tuples in a vector. Slices are indexed from 0, so slice 1 corresponds to the second column.

In this example, the number of tuples is 2 and the tuple size 3. The set {2 5} corresponds to slice 1 of the vector.

{<1 2 3> <4 5 6>}

See “Vector Function Summary” for a list of the functions that handle vector input and output.

A DFM expression that evaluates to a vector cannot be compared to a constraint. Vector properties can be compared using Conditional Expressions, however; see “Comparison of DFM Properties”.

Accessing Vector Elements

Three functions are available to access individual vector elements in numeric, string, and netID vectors: VELEMENT, SVELEMENT, and NETVELEMENT.

You can also access individual vector elements using a user-defined function created by a DFM Function statement. The functions VELEMENT, SVELEMENT, and NETVELEMENT are sufficient in most cases, however, you can write your own function for special cases. The user-defined function can only be used within DFM Property and DFM Analyze operations. Depending on the type of return value from the function, use either the TVF_NUMber_FUNction or TVF_STRING_FUNction argument to DFM Function to specify a Tcl procedure that is called from the user-defined function. See “TVF_NUMber_FUNction and TVF_STRING_FUNction in DFM Function”.

The vector TVF object passed to the Tcl procedure supports these subcommands:

  • object tuple_count — The number of tuples.

  • object tuple_size — The tuple size.

  • object value i j — The j-th value in the i-th tuple. Indices are zero-based. i and j can also be thought of as the row and column, respectively, in a 2-D matrix.

The following code defines a function to return the i j value of a numeric vector. It performs the same function as VELEMENT, but could be adapted for a different purpose. Error messages are printed if the i or j input values are out of bounds for the vector being processed. Following the function definitions is a code segment that demonstrates how to use the vector access function.

The tuple count and size values are converted to integers with the Tcl int function, as is the netID value. 

DFM FUNCTION [ GetVectorElement( VECTOR V, Number I, Number J) 
						TVF_NUM_FUN( GetVectorElementProc, mylib ) ]
TVF FUNCTION mylib [/*
# get I J element of the input numeric vector
proc GetVectorElementProc { v_in_  I_in_  J_in_} {
set body { 
	set ret -9999
	set count [expr int([ $v_in_ tuple_count])]
	set size  [expr int([ $v_in_ tuple_size])]
	set tuple_i [expr int([$I_in_])]
	set tuple_j [expr int([$J_in_])]
	if { $count > 0 && $tuple_j < $size && $tuple_i < $count} {  
		set elementValue [$v_in_ value $tuple_i $tuple_j]
		if {$elementValue != "" } {
			set ret $elementValue
		}
	}  else {
		puts "[lindex [info level 0] 0]: error with vector indices"  
		puts "  tuple count is $count, size is $size, i,j are \
				$tuple_i,$tuple_j"
	}
}
if [catch {eval $body} msg] {
	puts "ERROR: [lindex [info level 0] 0]: $msg" 
}
return $ret
}
