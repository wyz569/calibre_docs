Variables

An SVRF rule file supports two types of variables: those declared in a Variable statement and environment variables.

For detailed information on declaring variables in the rule file, refer to the Variable specification statement.

Environment variables can be referenced using the Variable specification statement. They can also be referenced in pathnames by preceding them with the dollar ($) character as discussed under “Environment Variables in Pathname Parameters”. Default values for environment variables can be defined using the #PRAGMA ENV pre-processor directive.

Note:

TVF allows greater freedom in the declaration of variables in a rule file. TVF is discussed in detail in the chapter “Tcl Verification Format”.

In ICverify, variables can also be resolved in the Pyxis Layout process file. If a variable is defined both in the process file and in the rule file with a Variable specification statement, then the definitions must match. That is, in ICverify, Variable specification statements cannot redefine variable names defined in the Pyxis Layout process file. An Pyxis Layout process variable can also be used to define a variable in a Variable specification statement.

When using ICverify, be aware that Pyxis Layout variables looked up in the process file during compilation of the rule file are resolved to their values at compilation time. Therefore, if these values become re-associated while the rule file is loaded, then the loaded rule file does not represent the true state of the Pyxis Layout system and you should reload it.
