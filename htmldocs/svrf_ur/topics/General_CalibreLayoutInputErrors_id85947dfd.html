Calibre Layout Input Errors

This section describes runtime errors related to layout input.
Table 1. Calibre Layout Input Errors

Explanation

The following fatal errors are reported by all Calibre applications:

ERROR: At least one lef file and exactly one def file are required in LAYOUT PATH when invoked with -fx.

When the Layout System is LEF/DEF, the tool requires at least one lef file and exactly one def file.

ERROR: At least one lef file and exactly one def file are required in LAYOUT PATH2 when invoked with -fx.

When the Layout System is LEF/DEF, the tool requires at least one lef file and exactly one def file.

ERROR: Specified primary cell <name> not located within the input layout database.

The Layout Primary or Layout Primary2 specification statement parameter is not a cell in the input layout database.

ERROR: Specified primary cell <name> is an excluded cell.

The Layout Primary or Layout Primary2 specification statement parameter is an Exclude Cell specification statement parameter.

ERROR: Recursive database reference near cell <name>.

The structure/symbol name near where the recursive reference was encountered appears in the message.

ERROR: Non-finite or non-positive PLACEMENT magnification <value> in input file <name> at record offset <offset>.

The layout database format is OASIS and a PLACEMENT record has an NaN, infinite, or non-positive magnification. The invalid magnification value, file name, and PLACEMENT record offset appears in the message.

ERROR: Non-finite PLACEMENT angle <value> in input file <name> at record offset <offset>.

The layout database format is OASIS and a PLACEMENT record has an NaN or infinite rotation. The invalid rotation value, file name, and PLACEMENT record offset appears in the message.

ERROR: Zero COLUMN COUNT value in AREF placement of cell <name> within cell <name>.

The layout database format is GDSII and an AREF placement has column count zero.

ERROR: Zero ROW COUNT value in AREF placement of cell <name> within cell <name>.

The layout database format is GDSII and an AREF placement has row count zero.

EXCEPTION: Non-orthogonal rotation (<number>) in AREF placement of cell <name> within cell <name> at location <coordinates>.

A GDSII AREF placement has a non-orthogonal rotation. The rotation in degrees, the cell name of the placement, the containing cell name, and the coordinates of the placement are in the message.

Exception name: AREF_PLACEMENT

Severity is controlled by Layout Input Exception Severity.

EXCEPTION: AREF placement of cell <name> within cell <name>.

A GDSII AREF placement has been encountered. The cell name of the placement and the containing cell name are placed into the message. This exception is intended for uses that explicitly prohibit GDSII AREFs.

Exception name: AREF_PLACEMENT

Severity is controlled by Layout Input Exception Severity.

EXCEPTION: Zero xpitch (ypitch) value in AREF placement of cell <name> within cell <name> at location <point>.

A GDSII AREF placement has its x-dimension (y-dimension) > 1 but the spacing in the x-direction (y-direction) is zero. The cell name of the placement, location of the placement, and the containing cell name are placed into the message. Exception Name: ARRAY_PITCH_ZERO

Severity is controlled by Layout Input Exception Severity.

EXCEPTION: Zero xspace (yspace) value in placement repetition within cell <name> in input file <name> at location <point> at record offset <offset>.

An OASIS placement record has a type 1 or 2 repetition and the x-direction spacing is zero. Or it has a type 1 or 3 repetition and the y-direction spacing is zero. The location of the placement and the containing cell name are placed into the message.

Exception Name: ARRAY_PITCH_ZERO

Severity is controlled by Layout Input Exception Severity.

ERROR: Non-positive <value> START record in input file <name> at record offset <offset>.

The layout database format is OASIS and a START record has an NaN, infinite, or non-positive unit value. The invalid unit value, file name, and START record offset appears in the message.

ERROR: Invalid delta count <count> for type <type> POLYGON point list in input file <file> at file offset <offset>.

The delta count for a type 0 or 1 OASIS POLYGON point list is less than two or is odd. The invalid delta count, point list type, file name, and approximate file offset of the delta count appear in the message.

ERROR: Invalid implicit closure for type <type> POLYGON point list in input file <name> at file offset <offset>.

An OASIS POLYGON point list of type 2 has non-orthogonal closure or an OASIS POLYGON point list of type 3 has non-octagonal closure. The point list type, file name, and approximate file offset of the end of the point list.

EXCEPTION: Cell name <name> for LAYOUT POLYGON is not defined.

The cell name of a rule file Layout Polygon object is not located in the input layout database.

Exception name: LAYOUT_POLYGON

Severity is controlled by Layout Input Exception Severity.

ERROR: Invalid magic byte string in input file <name>.

The magic string at the beginning of an OASIS file is not “%SEMI-OASIS<CR><NL>”. The name of the file is placed into the message.

ERROR: Another START record encountered in input file <name> at record offset <offset>.

A second OASIS START record is encountered in the same OASIS file. The file name and record offset of the second START record are placed into the message.

ERROR: Record ID <id> encountered outside of CELL record in input file <name> at record offset <offset>.

An OASIS PLACEMENT record, geometric record, TEXT record, XELEMENT record, XYRELATIVE record, or XYABSOLUTE record has been encountered outside of a CELL definition. The record ID, file name, and record offset are placed into the message.

ERROR: Unable to assign modal <string> in input file <name> at record offset <offset>.

The value of an OASIS modal variable is undefined. The file name and record offset where the modal value is required to be resolved is placed into the message. <string> identifies the modal value as follows:

repetition to element

point list to POLYGON

point list to PATH

layer number to geometry

layer number to TEXT

datatype number to geometry

texttype number to TEXT

half-width value to PATH

start-extension value to PATH

end-extension value to PATH

width value to RECTANGLE

height value to RECTANGLE

width value to TRAPEZOID

height value to TRAPEZOID

type value to CTRAPEZOID

width value to CTRAPEZOID

height value to CTRAPEZOID

radius value to CIRCLE

name to PLACEMENT

string to TEXT

name to PROPERTY

value list to PROPERTY

ERROR: Invalid type for real number in input file <name> at file offset <offset>.

A type outside of the range [0,7] for an OASIS real number is present. The file name and offset where the type is encountered is placed into the message.

ERROR: Invalid type for property value in input file <name> at file offset <offset>.

A type outside of the range [0,15] for an OASIS property value is present. The file name and offset where the type is encountered is placed into the message.

ERROR: Invalid type for repetition in input file <name> at file offset <offset>.

A type outside of the range [0,11] for an OASIS repetition is present. The file name and offset where the type is encountered is placed into the message.

ERROR: Invalid type for point list in input file <name> at file offset <offset>.

A type outside of the range [0,5] for an OASIS point list is present. The file name and offset where the type is encountered is placed into the message.

ERROR: Invalid type for CTRAPEZOID in input file <name> at file offset <offset>.

A type outside of the range [0,25] for an OASIS CTRAPEZOID is present. The file name and offset where the type is encountered is placed into the message.

ERROR: No CELLNAME record corresponding to cell name reference number <number> in input file <name>.

An OASIS file contains a cell name reference number in a CELL or PLACEMENT record for which there is no corresponding CELLNAME record in the same file. The invalid reference number and file name are placed into the message.

ERROR: No TEXTSTRING record corresponding to text string reference number <number> in input file <name> at record offset <offset>.

An OASIS file contains a TEXT record with a text string reference number for which there is no corresponding TEXTSTRING record in the same file. The invalid reference number, file name, and TEXT record offset are placed into the message.

ERROR: No PROPNAME record corresponding to property name reference number <number> in input file <name> at record offset <offset>.

An OASIS file contains a PROPERTY record with a property name reference number for which there is no corresponding PROPNAME record in the same file. The invalid reference number, file name, and PROPERTY record offset are placed into the message.

ERROR: No PROPSTRING record corresponding to property string reference number <number> in input file <name> at record offset <offset>.

An OASIS file contains a PROPERTY record with a property string reference number for which there is no corresponding PROPSTRING record in the same file. The invalid reference number, file name, and PROPERTY record offset are placed into the message.

ERROR: CELLNAME records 3 and 4 both present in input file <name> at record offset <offset>.

An OASIS CELLNAME record with ID 3(4) has been encountered when another CELLNAME record of type 4(3) has been previously encountered in the same file. The file name and CELLNAME record offset are placed into the message.

ERROR: TEXTSTRING records 5 and 6 both present in input file <name> at record offset <offset>.

An OASIS TEXTSTRING record with ID 5(6) has been encountered when another TEXTSTRING record of type 6(5) has been previously encountered in the same file. The file name and TEXTSTRING record offset are placed into the message.

ERROR: PROPNAME records 7 and 8 both present in input file <name> at record offset <offset>.

An OASIS PROPNAME record with ID 7(8) has been encountered when another PROPNAME record of type 8(7) has been previously encountered in the same file. The file name and PROPNAME record offset are placed into the message.

ERROR: PROPSTRING records 9 and 10 both present in input file <name> at record offset <offset>.

An OASIS PROPSTRING record with ID 9(10) has been encountered when another PROPSTRING record of type 10(9) has been previously encountered in the same file. The file name and PROPSTRING record offset are placed into the message.

ERROR: CELLNAME conflict: (name,number) coordinate (<name1>, <number1>) previously resolved as (<name2>, <number2>).

Two CELLNAME records in the same OASIS file have the same name but different reference numbers or vice-versa. The conflicting names and numbers are placed into the message.

ERROR: TEXTSTRING conflict: (name,number) coordinate (<name1>, <number1>) previously resolved as (<name2>, <number2>).

Two TEXTSTRING records in the same OASIS file have the same name but different reference numbers or vice-versa. The conflicting names and numbers are placed into the message.

ERROR: PROPNAME conflict: (name,number) coordinate (<name1>, <number1>) previously resolved as (<name2>, <number2>).

Two PROPNAME records in the same OASIS file have the same name but different reference numbers or vice-versa. The conflicting names and numbers are placed into the message.

ERROR: PROPSTRING conflict: (name,number) coordinate (<name1>, <number1>) previously resolved as (<name2>, <number2>).

Two PROPSTRING records in the same OASIS file have the same name but different reference numbers or vice-versa. The conflicting names and numbers are placed into the message.

ERROR: CELLNAME record may not post-associate (name, number) coordinate (<name>, <number>).

A cell name seen in a CELL or PLACEMENT record and a reference number seen in another CELL or PLACEMENT record may not both appear (that is, become “post-associated”) in a CELLNAME record later in the same OASIS file. The cell name and reference number are placed into the message. If the OASIS file is strict-mode, then this exception does not normally occur due to pre-scanning by Calibre.

ERROR: Both H and S bits 1 in info byte for RECTANGLE in input file <name> at record offset <offset>.

An OASIS RECTANGLE record has both H and S bits simultaneously set in the info byte. The file name and record offset of the RECTANGLE record are placed into the message.

ERROR: H bit is 1 with type <type> in info byte for CTRAPEZOID in input file <name> at record offset <offset>.

An OASIS CTRAPEZOID record of type 16, 17, 18, 19, 22, 23, or 25 has the H bit set in the info byte. The CTRAPEZOID type, file name, and record offset of the CTRAPEZOID record are placed into the message.

ERROR: W bit is 1 with type <type> in info byte for CTRAPEZOID in input file <name> at record offset <offset>.

An OASIS CTRAPEZOID record of type 20 or 21 has the W bit set in the info byte. The CTRAPEZOID type, file name, and record offset of the CTRAPEZOID record are placed into the message.

ERROR: INTEGER overflow in input file <name> at file offset <offset>.

The value of an OASIS unsigned integer exceeds 264 - 1 or the absolute value of an OASIS signed integer exceeds 263 - 1. The file name and approximate file offset of the integer are placed into the message.

ERROR: REAL with zero denominator in input file <name> at file offset <offset>.

An OASIS real number expressed in rational form has a zero denominator. The file name and approximate file offset of the real number are placed into the message.

ERROR: Non b-string of zero length in input file <name> at file offset <offset>.

An OASIS a-string or n-string has zero length. The file name and approximate file offset of the string are placed into the message.

ERROR: String of length <length> in input file <name> at file offset <offset> exceeds maximum supported length of 65535.

An OASIS CELLNAME, CELL, or PLACEMENT string has length exceeding 65535, which is not supported in Calibre. The length, file name and approximate file offset of the string are placed into the message.

ERROR: Invalid character (<character>) in a-string in input file <name> at file offset <offset>.

The OASIS version string (in the START record) or the string in a property value of type 10 has a non-a-string character. The character, file name and approximate file offset of the character are placed into the message.

ERROR: Component overflow (<name>) in OASIS repetition in input file <name> at file offset <offset>.

An OASIS repetition component exceeds the tool’s magnitude limits. The name of the component (for example, "x-dimension"), file name, and approximate file offset of the component are placed into the message. This error has the following causes.

Define INT_MAX as 2147483647. LONG_MAX is 18446744073709551614.

The x-dimension of a type 1, 2, 4, or 5 repetition exceeds INT_MAX - 2.

The y-dimension of a type 1, 3, 6, or 7 repetition exceeds INT_MAX - 2.

The x-space of a type 1 or 2 repetition exceeds LONG_MAX.

The y-space of a type 1 or 3 repetition exceeds LONG_MAX.

The x-space of a type 4 or 5 repetition (after grid adjustment) exceeds INT_MAX.

The y-space of a type 6 or 7 repetition (after grid adjustment) exceeds INT_MAX.

The n-dimension or m-dimension of a type 8 repetition exceeds INT_MAX - 2.

Any displacement component of a type 8 repetition is outside of the interval [-INT_MAX,+INT_MAX].

The dimension of a type 9, 10, or 11 repetition exceeds INT_MAX - 2.

Any displacement component of a type 9, 10, or 11 repetition (after grid adjustment) is outside of the interval [-INT_MAX,+INT_MAX].

ERROR: Invalid character (<character>) in n-string in input file <name> at file offset <offset>.

An OASIS layer name or the string in a property value of type 12 has a non-n-string character. The character, file name and approximate file offset of the character are placed into the message.

ERROR: Invalid implied a-string reference in property value in input file <name> at file offset <offset>.

The string referenced by OASIS property value type 13 has a non-a-string character. The character, file name and approximate file offset of the property value are placed into the message.

ERROR: Invalid implied n-string reference in property value in input file <name> at file offset <offset>.

The string referenced by OASIS property value type 15 has a non-n-string character. The character, file name and approximate file offset of the property value are placed into the message.

ERROR: Invalid property value format for standard property S_GDS_PROPERTY in input file <name> at record offset <offset>.

A (standard) OASIS PROPERTY with name S_GDS_PROPERTY is incorrectly formatted. The file name and record offset of the PROPERTY record are placed into the message.

ERROR: Comp-type for CBLOCK record in input file <name> at file offset <number> not supported.

The comp-type field in a CBLOCK record has a value other than 0; these are not currently defined in OASIS. The file name and byte offset of the exception is placed into the message.

ERROR: CBLOCK compress format error in input file <name> at file offset <number>.

An improper compression format has been located within a CBLOCK record. The file name and byte offset of the exception is placed into the message.

ERROR: Invalid comp-byte-count for CBLOCK record in input file <name> ending at file offset <number>.

The number of bytes of compressed data in a CBLOCK record (comp-byte-count) does not match the number when the internal format is actually decompressed. The file name and byte offset terminating the compressed data are placed into the message.

ERROR: Invalid uncomp-byte-count for CBLOCK record in input file <name> ending at file offset <number>.

The number of bytes of uncompressed data in a CBLOCK record (uncomp-byte-count) does not match the number when the internal format is actually decompressed. The file name and byte offset terminating the compressed data are placed into the message.

ERROR: START record within CBLOCK record in input file <name> at file offset <number>.

An OASIS START record is illegally inside of a CBLOCK record. The file name and byte offset of the exception is placed into the message.

ERROR: END record within CBLOCK record in input file <name> at file offset <number>.

An OASIS END record is illegally inside of a CBLOCK record. The file name and byte offset of the exception is placed into the message.

ERROR: CELL record within CBLOCK record in input file <name> at file offset <number>.

An OASIS CELL record is illegally inside of a CBLOCK record. The file name and byte offset of the exception is placed into the message.

ERROR: CBLOCK record within CBLOCK record in input file <name> at file offset <number>.

An OASIS CBLOCK record is illegally inside of a CBLOCK record. The file name and byte offset of the exception is placed into the message.
