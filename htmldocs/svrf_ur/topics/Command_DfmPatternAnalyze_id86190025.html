DFM Pattern Analyze

Layer operation

License: Calibre Pattern Matching

Analyzes the symmetry of target layer shapes at marked locations. Keywords define the content of the output layer.

Usage

DFM PATTERN ANALYZE symmetry_output target_layer    
capture_layer‘(’{MARKER capture_size | MASK [oversize]}‘)’

Arguments

symmetry_output

A required argument that specifies what is written to the output layer. The output layer is a merged polygon layer. The output can be patterns with the specified symmetry, the difference between the original pattern and one transformed by the specified symmetry operation, or all patterns.

The symmetry_output argument can be one of the keywords in the following table.

Pattern Output

Difference Output

PATTERN_ALL_180

DIFF_ALL_180

PATTERN_ALL_90

DIFF_ALL_90

PATTERN_ALL_X

DIFF_ALL_X

PATTERN_ALL_Y

DIFF_ALL_Y

PATTERN_ALL

 

See Table 1 and Table 2 for complete descriptions of these keywords.

target_layer

A required original or derived polygon layer containing the shapes to analyze.

capture_layer‘(’{MARKER capture_size | MASK [oversize]}‘)’

A required argument set that defines the capture region for shapes on the target_layer. See Figure 1 for a description of the capture region and the use of the MARKER and MASK keywords.

  • capture_layer — An original or derived polygon layer containing shapes that define the capture locations.

  • (MARKER capture_size) — Specifies that the shapes on capture_layer are marker shapes that define capture locations. The capture regions are squares centered at the center of each marker shape. The square capture region has sides of length capture_size in user units. The parentheses are required.

    Note: The capture_size argument defines the width of the capture region, in contrast to the case with DFM Pattern Capture and DFM Pattern Classify, where the halo argument specifies the half-width of the capture region.

  • (MASK [oversize]) — Specifies that the shapes on capture_layer define the capture regions. The capture regions are the rectangular extents of each polygon on capture_layer. If oversize is specified, the polygon extent is sized up by oversize, as with the Size operation, where oversize is a positive value in user units. The parentheses are required.

Description

Analyzes the symmetry of shapes on the target_layer and writes shapes to the output layer as determined by the symmetry_output keyword. The operation groups shapes on the target_layer into patterns that are within a specified capture region. The capture region is determined by shapes on the capture_layer and by the MASK or MARKER keyword set.

The following figure shows how the capture region is calculated depending on whether the MASK or MARKER keyword is specified.

Figure 1. DFM Pattern Analyze Capture Regions

The operation identifies patterns that are symmetric with respect to 180 and 90 degree rotations or reflections across the X- or Y-axis. You can output pattern layer shapes or difference shapes. The possible symmetry_output keywords are described in the next two tables.

Table 1. Pattern Output in DFM Pattern Analyze

symmetry_output

Output Layer

PATTERN_ALL

All target_layer polygons within a capture region. (The Boolean AND of the target_layer and the capture regions.)

PATTERN_ALL_180

All target_layer polygons that belong to any pattern with an intrinsic symmetry with respect to 180 degree rotations about the center of the capture region.

PATTERN_ALL_90

All target_layer polygons that belong to any pattern with an intrinsic symmetry with respect to 90 degree rotations about the center of the capture region.

PATTERN_ALL_X

All target_layer polygons that belong to any pattern with an intrinsic vertical symmetry (reflection across the X-axis). The direction of X-axis is global, determined at the top-level, and the resulting layer hierarchy may be degraded.

PATTERN_ALL_Y

All target_layer polygons that belong to any pattern with an intrinsic horizontal symmetry (reflection across the Y-axis). The direction of the Y-axis is global, determined at the top-level, and the resulting layer hierarchy may be degraded.
Table 2. Difference Output in DFM Pattern Analyze

symmetry_output

Output Layer

DIFF_ALL_180

The difference between the original pattern geometries and the geometries rotated by 180 degrees around the center of the pattern.

DIFF_ALL_90

The difference between the original pattern geometries and the geometries rotated by 90 degrees around the center of the pattern.

DIFF_ALL_X

The difference between the original pattern geometries and the geometries reflected across the X-axis. The direction of the X-axis is global, determined at the top-level, and the resulting layer hierarchy may be degraded.

DIFF_ALL_Y

The difference between the original pattern geometries and the geometries reflected across the Y-axis. The direction of the Y-axis is global, determined at the top-level, and the resulting layer hierarchy may be degraded.

Concurrency

DFM Pattern Analyze operations are executed concurrently when all arguments other than symmetry_output are the same.

Examples

Example 1

This example checks for symmetry across the x-axis for shapes on the M1 layer. Shapes on the mask_marker layer define the capture regions. The pattern difference is output. 

LAYER M1 2
LAYER mask_marker 3

diff_m1_x { 
    DFM PATTERN ANALYZE DIFF_ALL_X M1 mask_marker(MASK)
}


Example 2

This example checks for 180 degree symmetry of M1 layer polygons in the region of hotspot markers on layer hotspot. The capture size is set with the variable pat_size. Results are output from four checks.

LAYOUT PRIMARY "TOPCELL"
LAYOUT SYSTEM GDSII
LAYOUT PATH "layout.gds"

DRC MAXIMUM RESULTS all
DRC RESULTS DATABASE "drc_results" ASCII PSEUDO

LAYER hotspot 0
LAYER M1 2
VARIABLE pat_size 1.3
m1_pat = DFM PATTERN ANALYZE PATTERN_ALL M1 hotspot(MARKER pat_size)
m1_180 = DFM PATTERN ANALYZE PATTERN_ALL_180 M1 hotspot(MARKER pat_size)
m1_pat { // all M1 pattern shapes: those within a square capture region  
			// of size pat_size centered on the hotspot shapes
	copy m1_pat
}

pat_180 { // patterns with 180 degree symmetry
	copy m1_180
}
not_180 { // patterns without 180 degree symmetry
	m1_pat NOT m1_180
}

diff_180 { // for each pattern, the difference between the 
			  // original pattern and the pattern rotated by 180 degrees
	DFM PATTERN ANALYZE DIFF_ALL_180 M1 hotspot(MARKER pat_size)
}
