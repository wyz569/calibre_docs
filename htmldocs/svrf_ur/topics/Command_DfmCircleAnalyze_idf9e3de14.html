DFM Circle Analyze

Layer operation

License: Calibre nmDRC-H

Characterizes how close to circular the polygons on the input layer are. For each input polygon, the operation finds two concentric circles: one is the minimum enclosing circle and the other is the largest circle that is contained within the polygon. The radii of these two circles are reported as DFM properties attached to the output layer. Annular shapes are also identified. By default the output layer is a copy of the input layer, with properties attached to each polygon.

Usage

DFM CIRCLE ANALYZE layer [CENTER | RING]

Arguments

layer

A required polygon layer.

CENTER | RING

An optional keyword choice that specifies what to write to the output layer. By default the output layer is a copy of the input layer. See “Output Layer” for more information.

  • CENTER — Specifies to write a 2×2 square marker polygon to the output layer for each input polygon. The center of each marker polygon is the grid point closest to the center of the concentric circles that characterize the input polygon. The output is unmerged.

  • RING — Specifies to output polygons that approximate the annular region between the two concentric circles that characterize each input polygon. The output is unmerged. See “RING Output” for more information.

Description

For each polygon on the input layer, the operation finds the minimum enclosing circle C-out and calculates radius properties that characterize the polygon.

The operation also identifies annular regions, which are polygons with exactly one hole and the center of the circle C-out lies in that hole.

The properties defined in the following table are attached to each shape on the output layer. See Figure 1 for examples.

Table 1. Properties Added by DFM Circle Analyze

Property

Definition

OUTRADIUS

The radius of circle C-out, the minimum enclosing circle of the polygon.

INRADIUS

For non-annular regions, the radius of circle C-in, the largest circle whose disk is contained within the polygon and concentric with C-out.

If the center of circle C-out is outside the input polygon and the shape is not an annulus, there is no circle C-in and INRADIUS is zero.

For annular regions, the radius of the circle C-in, the largest circle that is concentric with C-out without going outside the outer polygon boundary.

ANNULUS_OUTRADIUS

For an annulus, the radius of the circle that is concentric with C-out and is the minimum enclosing circle of the inner boundary of the polygon (circle CA-out in Figure 1).

This property is nonzero only if the polygon is an annulus.

ANNULUS_INRADIUS

For an annulus, the radius of the largest circle concentric with the outer circle and fully contained within the hole of the annulus (circle CA-in in Figure 1).

This property is nonzero only if the polygon is an annulus.

For annular regions that are nearly circular and sufficiently thick, INRADIUS > ANNULUS_OUTRADIUS. However, this may not be true for thin annular regions that are not very circular, such as a square with a very large square hole.

DFM property expressions that use INRADIUS, ANNULUS_INRADIUS, or ANNULUS_OUTRADIUS should be written so that the expression evaluates successfully when the property value is zero.

The shape on the top right shows a case where the center of the enclosing circle C-out is outside the input polygon, resulting in INRADIUS = 0. The figure also shows an annulus and a shape that is not an annulus even though it has a hole.

Output Layer

By default the output layer is a copy of the input layer. The RING and CENTER keywords change the shape that is written to the output layer. It is possible for unmerged output, which is produced by the RING and CENTER keywords, to overlap each other or extend outside the cell boundary.

RING Output

When RING is specified, the output polygons approximate the annular region between the two concentric circles that characterize each input polygon.

Property Values

RING Output

INRADIUS nonzero

ANNULUS_INRADIUS = 0

Region between C-out and C-in.

INRADIUS nonzero

ANNULUS_INRADIUS nonzero

Region between C-out and CA-in.

INRADIUS = 0

ANNULUS_INRADIUS = 0

An approximately circular region corresponding to circle C-out.

The separation between the inner and outer boundaries of the output ring shape is on the same order as the separation between the concentric circles that define the ring, within limits. For the case of a nonzero INRADIUS and zero ANNULUS_INRADIUS, the closer an input polygon is to circular, the more vertices will be used to represent its annulus, with an upper bound of 2048 vertices. See “Example 2”.

Concurrency

Operations with the same layer input run concurrently.

Also see the CIRCLE keyword for Inside Cell and “Finding Circular Polygons” in the Calibre Solutions for Physical Verification manual.

Examples

Example 1

This example finds shapes that are nearly circular and outputs them to a new layer named near_circle. Shapes are output if the ratio INRADIUS/OUTRADIUS is greater than 0.90. 

LAYER layer1 0

circle_props = DFM CIRCLE ANALYZE layer1
near_circle = DFM PROPERTY circle_props 
         [ - = PROPERTY(circle_props, INRADIUS)/
               PROPERTY(circle_props, OUTRADIUS)] > 0.90

Example 2

This example uses the RING keyword to output the approximate annular region between the inner and outer circles that characterize each input polygon. The output layer is written out with DFM RDB so that properties can be viewed. The figure shows the results for a hexagon, an octagon, and an annular region. 

LAYER layer1 0
ring_out = DFM CIRCLE ANALYZE layer1 RING
ring_out { DFM RDB ring_out "dfm.rdb"  ALL CELLS CHECKNAME "%_l_" }


Example 3

This example finds shapes that are identified as an annulus. 

LAYER layer1 0

circle_props = DFM CIRCLE ANALYZE layer1
annular_shape = DFM PROPERTY circle_props
         [ - = PROPERTY(circle_props, ANNULUS_INRADIUS)] > 0
