Cell-Context-Dependent Property Functions Results in Hierarchical Batch Runs

All measurement functions that return coordinates, or values based upon orientation with respect to a coordinate system, return them in the context of the cell in which the property expression is evaluated. This may not be the same cell in which the geometries originated in cases where promotion occurs. For these functions, the terms x-axis, y-axis, horizontal, and vertical refer to the coordinate system of the evaluation cell, not to the global coordinate system. Cell-context-dependent property functions do not provide predictable results in hierarchical batch runs and should be treated with care.

For multilayer DFM Property operations, expression evaluation takes place in the clustering location. For single-layer DFM Property operations evaluation takes place either in the cell containing the geometry or at a higher level cell if promotion occurs.

For cell placements that are rotated or reflected the coordinate axes at the cell level is rotated or reflected with respect to the top level coordinate axes. This means there can be inconsistent definitions of horizontal and vertical, or positive and negative directions, at the cell placement level versus the top level. When using the DFM Property operation, the GLOBALXY keyword can be used to provide axis-aligned measurements that are meaningful in the global frame of reference. See “GLOBALXY” for details.

Similarly, net IDs are sensitive to hierarchy. The NETID function returns a net ID at the cell level and does not take into account the cell’s placement in the hierarchy and does not consider high shorts. The GLOBALNETID function returns a net ID that can safely be compared in the context of the cell the DFM property expression is evaluated in. GLOBALNETID considers the cell’s placement in the hierarchy and takes high shorts into account. However, GLOBALNETID does not return unique net IDs across the whole design—the net IDs can only be compared within the context of a cell.
