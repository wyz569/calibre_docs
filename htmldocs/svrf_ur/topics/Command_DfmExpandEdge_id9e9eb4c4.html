DFM Expand Edge

Layer operation

License: Calibre YieldEnhancer, except in LVS runs, which use a Calibre ADP (Advanced Device Properties) license.

Expands edges into polygons based on DFM properties attached to the input layer.

Usage

DFM EXPAND EDGE layer1
{OUTSIDE BY “outside_property”  
  | INSIDE BY “inside_property”  
  | OUTSIDE BY “outside_property” INSIDE BY “inside_property”}
{{[EXTEND BY “extend_property”] [OFFSET “offset_property”]} | [CORNER FILL]}
[UNMERGED]

Arguments

layer1

A required derived edge layer or derived error layer (edge cluster). If layer1 is an error layer, each edge in the cluster is expanded by the same amount.

The names of DFM Properties attached to layer1 can be specified as the arguments outside_property, inside_property, extend_property, and offset_property. Such layers are typically created with a DFM Property or DFM Property Select Secondary operation. Alternatively, any of these arguments may be specified as constant numeric values or as numeric variables defined by a Variable statement.

If layer1 is an unmerged edge or error layer that does not contain DFM properties, it is merged before expansion occurs. If an unmerged layer1 does contain DFM properties, it is not merged before expansion occurs.

OUTSIDE BY “outside_property”

A required argument set that specifies that each edge is expanded in the direction perpendicular to the edge and outside of the original polygon from which it was created. The name of the property that defines the expansion distance is specified by outside_property. The property name must be enclosed in quotes. Alternatively, you may use a constant value or numeric variable for outside_property. The value of outside_property must be non-negative.



INSIDE BY “inside_property”

A required argument set that specifies that each edge is expanded in the direction perpendicular to the edge and inside of the original polygon from which it was created. The name of the property that defines the expansion distance is specified by inside_property. The property name must be enclosed in quotes. Alternatively, you may use a constant value or numeric variable for inside_property. The value of inside_property must be non-negative



EXTEND BY “extend_property”

An optional argument set that specifies that each edge is extended by extend_property in both directions along the edge before expansions are applied. The name of the property that defines the expansion distance is specified by extend_property. It must be enclosed in quotes. Alternatively, you may use a constant value or numeric variable for extend_property.

The value of extend_property can be positive or negative. A positive value lengthens the edge at each end by the extend_property value. A negative value shrinks the edge at each end. The following image shows a positive value for the “ext” property value.



OFFSET “offset_property”

An optional argument set that causes the expanded polygon to be shifted from the original edge. The name of the property that defines the offset value is specified by offset_property. It must be enclosed in quotes. Alternatively, you may use a constant value or numeric variable for offset_property. The value of offset_property should be a non-negative number. Negative values produce no output.

The offset is applied in the direction indicated by the OUTSIDE BY or INSIDE BY keyword. When OFFSET is specified, you can specify only one of OUTSIDE BY or INSIDE BY.

The following figure demonstrates the OFFSET keyword. When used with OUTSIDE BY, the expansion and offset are to the outside. When used with INSIDE BY, the expansion and offset are to the inside.



CORNER FILL

An optional keyword that specifies to fill gaps between adjacent rectangles formed at corners of the input layer. Corners are only filled between generated edges forming a right angle. The input layer edges must either be orthogonal to the database axes or rotated 45 degrees with respect to the axes. Corners are filled in the direction of the expansion. That is, for OUTSIDE BY expansions, corner filling extends outside of polygons on the input layer, and for INSIDE BY expansions, corner filling extends inside of polygons on the input layer. CORNER FILL cannot be specified with EXTEND BY or OFFSET.



UNMERGED

An optional keyword that causes the operation to produce an output layer that contains unmerged polygons (that is, individual polygons created from each edge are preserved, even if they overlap.)

When UNMERGED is specified, the output layer is annotated with any DFM properties from a derived edge input layer. Each output polygon contains the property values from the input edge used to construct the polygon. For derived error input layers (TYP=3 in the transcript), property annotation occurs only if all clusters are single-edge clusters.

Unmerged polygon layers can be processed by a limited number of operations; see “Unmerged Polygon Layers” in the Calibre Verification User’s Manual.

Description

Expands edges into polygons. This operation is similar to Expand Edge. The primary differences between this operation and Expand Edge are as follows:

  • DFM Expand Edge can take expansion values from DFM properties attached to layer1 instead of from values specified with keywords in the operation.

  • DFM Expand Edge does not have a FACTOR keyword.

  • DFM Expand Edge accepts derived error layers as input.

At least one of the options OUTSIDE BY and INSIDE BY must be specified. At least one of the OUTSIDE BY and INSIDE BY values must be positive in order to produce output.

The result of this operation is a derived polygon layer by default. As with all derived polygon layers, overlapping or touching polygons are merged, so it is possible that polygons created for unrelated edges will be merged together. Specify UNMERGED to produce an unmerged polygon layer in which the individual polygons created from each edge remain distinct.

This operation does not require or preserve connectivity. The resulting derived polygon layer does not contain net numbers, even if layer1 contains connectivity.

Examples

Example 1

This example shows how to use a layer created with DFM Property in a DFM Expand Edge operation.

met_edge = m1 LENGTH < 1
met_edge_p = DFM PROPERTY met_edge 
                  [ OUT = LENGTH(met_edge)*.01 ]
                  [ IN  = LENGTH(met_edge)*.02 ]
met_expand = DFM EXPAND EDGE met_edge_p OUTSIDE BY "OUT" INSIDE BY "IN"

Example 2

In this example, layer e is an edge layer. A DFM Property operation is used to create a derived edge layer e_p with two attached DFM properties. One of these properties is used to expand the edge layer. The UNMERGED keyword is used to output unmerged polygons that include both properties from the input edge layer e_p. The unmerged layer is then processed by the DFM Property Merge operation, which computes properties for the merged polygons from their unmerged component polygons according to the specified expression. Specifically, the property A on the output layer exm contains the area of the largest polygon on layer p overlapping the edge from which the output polygon was created.

e_p = DFM PROPERTY e p OVERLAP MULTI
         [ e_out = 0.10*LENGTH(e) ] [ A = AREA(p) ]
ex  = DFM EXPAND EDGE e_p OUTSIDE BY "e_out" UNMERGED
exm = DFM PROPERTY MERGE ex [ A = MAX( PROPERTY(ex,"A") ) ]
