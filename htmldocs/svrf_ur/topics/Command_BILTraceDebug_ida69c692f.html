DEBUG

Trace Property computation language function.

Enables debug monitoring during Trace Property computation.

Usage

DEBUG range1[, range2 …]

Arguments

range1

A required device instance ID number or a pair of device instance numbers separated by a hyphen (‑). If the hyphenated form is used, the first number must be less than the second number. In that case, all numbers between and including the numbers specified in the range correspond to instances. At least one range must be specified, although it can contain only a single instance. Multiple ranges are separated by a comma (,) followed by whitespace.

Return Values

None.

Description

Debug monitoring of the Trace Property computation can be useful in finding errors during the development of new property computation code. When present, DEBUG must follow the PROPERTY and PROPERTY STRING statements, and precede the optional WARN statement. The debugging trace is written to the transcript. The trace can produce much output, so the use of small ranges is recommended.

This statement is used in conjunction with the WARN statement, which reports mathematical computation errors.

The instance IDs represent both the layout and source device in the input pair. Instance IDs are unique across all devices in the design. Instance IDs are assigned by the system and are not guaranteed to be sequential, merely unique.

Examples

Assume the WARN command is used and the run gives this warning in the transcript:

***************************************************
TRACE PROPERTY user-program WARNING:
Division by zero (result set to UNKNOWN).
while tracing with:
TRACE PROPERTY ... (within PROPERTY program beginning on rule file line 27)
Use source instance ID# 4 in DEBUG trace range.
***************************************************

Then you can do this:

TRACE PROPERTY ... [
PROPERTY r
DEBUG 4
WARN
...

Instance 4 will have a debugging trace run on it with details in the transcript.

If you have a problem with string properties not being present, the WARN message would appear similar to this:

TRACE PROPERTY user-program WARNING:
Unknown values were observed in the layout and/or source;
while tracing with:
TRACE PROPERTY ... (within PROPERTY program beginning on rule file line 22)
the user program was NOT executed;
discrepancies were recorded to the report.
