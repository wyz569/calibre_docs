DFM Pattern Match Report

Specification statement

License: Calibre Pattern Matching

Generates a comma-separated values (CSV) file that summarizes the matching statistics from a pattern matching run.

Usage

DFM PATTERN MATCH REPORT count_marker_layer filename
[FIXED_PROP_VALUES]

Arguments

count_marker_layer

A required argument that specifies the count marker output layer from a pattern matching run.

See “Count Markers” in the Calibre Pattern Matching User’s Manual for information on count markers and how to include them in a compiled DMACRO pattern library.

filename

A required filename for the generated output.

FIXED_PROP_VALUES

An optional keyword that specifies to include property names and values associated with each pattern in the report.

Computed properties that are evaluated for each matched instance are not included. The properties match_orient and match_rotation are examples of such properties.

Description

Generates a comma-separated values (CSV) file that summarizes the statistics for the matched patterns in a pattern matching run. The statement must be placed within a rule check. If you include multiple pattern matching CMACRO calls in the pattern matching run, you can specify a corresponding DFM Pattern Match Report statement for each CMACRO call, where each DFM Pattern Match Report statement has a unique count_marker_layer and filename.

The report includes a header and one line of statistics for each matched pattern. Patterns are listed in a descending order of matched count. The statistics line has the following format without FIXED_PROP_VALUES specified:

pat_name,tot_hier(tot_flat),exact_hier(exact_flat),fuzzy_hier(fuzzy_flat)

The line includes the pattern name and pairs of numbers for the total, exact, and fuzzy match counts. The hierarchical match count is given first and the flat count is in parentheses. BCM pattern statistics only include the flat match count. Statistics for the match type (exact or fuzzy) are not reported for BCM patterns with constraints. Unavailable statistics are entered as a “-” (dash). In a flat Calibre run, two match counts are still reported for TEM patterns, but the two counts are the same.

If FIXED_PROP_VALUES is specified, property names and values are appended to the statistics line. For example: prop_name1, value1, prop_name2, value2 …

An example report follows, without FIXED_PROP_VALUES specified:

=========================================================================
--- CALIBRE::PATTERN MATCHING SUMMARY REPORT
---
--- Report Generation Date/Time:  Thursday Thu Sep  7 16:27:32 2017
--- Calibre Version:              v2017.4
--- Match Report File:            rep.txt
--- Layout Path(s):               ./layout1.gds
--- Layout Primary Cell:          *
--- Total Match Count:            -(85)
-------------------------------------------------------------------------
---
--- PATTERN MATCH COUNT
---
Pattern,Total,Exact,Fuzzy
pat1,-(56),-(-),-(-) 
pat2,-(28),-(28),-(0) 
pat3,1(1),1(1),0(0)

Patterns pat1 and pat2 are BCM patterns and report only the flat matched count. Pattern pat1 is a BCM pattern with constraints and only reports the total count, not the match type.

For details on how to use a compiled DMACRO, see “Using a DMACRO in a Rule File for a Pattern Matching Run” in the Calibre Pattern Matching User’s Manual.

Examples

Example 1

To generate a report of pattern counts, include DFM Pattern Match Report in a rule file as follows:

// Include the DMACRO file. The library has one target layer (poly) 
// and two output marker layers (match and count)
INCLUDE myPatternLib.svrf

//set up the CMACRO 
CMACRO pm:myPatternLib poly " " match count 
count {COPY count} 
DRC CHECK MAP count GDSII 200 pm_results.gds MAXIMUM RESULTS ALL 

CHECK {
   DFM RDB count "./pat_match.rdb" ALL CELLS CHECKNAME count 
   DFM PATTERN MATCH REPORT count pm_report.txt 
}

Example 2

This example shows how to generate reports for multiple libraries within the same rule file.

// specify 1st pattern library 
INCLUDE "myPatternLib1.svrf"

//set up the CMACRO for 1st library
CMACRO pm:myPatternLib1  met1 met2 met3 " " match1 count1
match1 {COPY match1}
DRC CHECK MAP match1 GDSII 100 pm_results.gds MAXIMUM RESULTS ALL
count1 {COPY count1}
DRC CHECK MAP count1 GDSII 200 pm_results.gds MAXIMUM RESULTS ALL

// specify 2nd pattern library 
INCLUDE "myPatternLib2.svrf"

//set up the CMACRO for 2nd library
CMACRO pm:input2  met1 met2 met3 " " match2 count2
match2 {COPY match2}
DRC CHECK MAP match2 GDSII 100 pm_results.gds MAXIMUM RESULTS ALL
count2 {COPY count2}
DRC CHECK MAP count2 GDSII 200 pm_results.gds MAXIMUM RESULTS ALL

CHECK {
   DFM RDB match1 "./pat_match1.rdb" ALL CELLS CHECKNAME match1
   DFM RDB count1 "./pat_match1.rdb" ALL CELLS CHECKNAME count1
   DFM PATTERN MATCH REPORT count1 rep1.txt

   DFM RDB match2 "./pat_match2.rdb" ALL CELLS CHECKNAME match2
   DFM RDB count2 "./pat_match3.rdb" ALL CELLS CHECKNAME count2
   DFM PATTERN MATCH REPORT count2 rep2.txt
}
