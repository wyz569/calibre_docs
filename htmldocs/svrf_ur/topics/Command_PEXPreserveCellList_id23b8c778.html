PEX Preserve Cell List

LVS circuit extraction

Specifies a list of cells to receive special protection from hierarchical manipulation during circuit extraction performed in conjunction with PEX flows.

Usage

PEX PRESERVE CELL LIST list_name

Arguments

list_name

Required name of a Layout Cell List.

Description

This statement enables hierarchical LVS circuit extraction to support a greater level of hierarchy preservation for specified cells when parasitic extraction is performed downstream in the flow.

This statement is used when there are layers with large polygons that get pushed into lower-level cells. Often, these large-shape layers are used for things like well and substrate shapes that usually are present at the lowest levels of the design, but, they are only drawn at the top level. These shapes can be pushed to lower levels of the design during circuit extraction. In certain pcell flows, certain higher-level layers, like metal layers, are likewise drawn using a single large polygon at the top level. These shapes can be similarly pushed to lower levels in the design. This pushdown makes gathering accurate parasitics problematic.

To mitigate these problems, PEX Preserve Cell List specifies a list of cells to be preserved during circuit extraction runs. This statement functions similarly to Layout Preserve Cell List, and is not subject to expansion by LVS Auto Expand Hcells. In addition to the usual hierarchical preservation (like the prevention of promoting shapes out of cells) that Layout Preserve Cell List provides, the shape pushdown mentioned previously is inhibited in the specified cells when Mask SVDB Directory uses the XRC, XACT, or CCI options, and lacks the NOPEXPCELLS option.

This type of cell preservation is also enforced with PEX Xcell and the -xcell command line option.

Note:

Layers derived using the Extent and Extent Cell operations are unaffected by xcell preservation; that is, a layer derived from these operations is still pushed down into the lower-level cells when possible.

For certain design layers, pushing of shapes into protected xcells might be preferred. In that case, these alternatives are available:

  • Revert to the pre-2019.1 behavior globally by using Mask SVDB Directory with the NOPEXPCELLS keyword along with the CCI, XACT, or XRC keyword.

  • If a particular original layer is needed inside of the protected cells, it can be drawn inside of those cells in addition to being drawn at the top level.

  • Use a layer derivation like this:

    LAYER DNW 999 // big N-well at top level

// this layer can be pushed into xcells
DNW_push = (EXTENT) AND DNW LVSCAREFUL

Examples

Specify a PEX cell preservation list:

PEX PRESERVE CELL LIST p_cells
LAYOUT CELL LIST p_cells "p_*"
