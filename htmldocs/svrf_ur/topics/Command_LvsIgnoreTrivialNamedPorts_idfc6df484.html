LVS Ignore Trivial Named Ports

Specification statement

Controls the handling of ports to lower-level cells that are not connected to any devices and that have corresponding names elsewhere in the design. Not used in Calibre PERC.

Usage

LVS IGNORE TRIVIAL NAMED PORTS {NO | YES}

Arguments

NO

Keyword that instructs the tool to retain named ports of lower-level cells that do not connect to any devices within the cells (trivial named ports) for comparison. This is the default behavior if you do not include this statement in your rule file.

YES

Keyword that instructs the tool to ignore trivial named ports of lower-level cells.

Description

Controls the handling of lower-level cell ports that are not connected to any devices within the cells where the ports appear. These are referred to as trivial ports. Additionally, such ports have corresponding names in both the source and layout. These are referred to as trivial named ports.

This statement does not apply to ports of the top-level cell.

When NO is specified, trivial named ports are not ignored in LVS comparison and become initial correspondence points. For example, consider the following design:

-- Layout Netlist

.SUBCKT TOP
C0 1 1
X0 1 2 A  $ net 2 is floating
X1 1 3 A  $ net 3 is floating
.ENDS

.SUBCKT A P1 P2   $ P2 is trivial
C0 P1 P1  $ P1 is attached to the capacitor
.ENDS
-- Source Netlist

.SUBCKT TOP
C0 1 1
X0 1 2 A  $ net 2 is floating
X1 1 3 A  $ net 3 is floating
.ENDS

.SUBCKT A P2 P1  $ P1 is trivial
C0 P2 P2  $ P2 is attached to the capacitor
.ENDS

When the netlists are compared, if cell A is an hcell, ports P1 and P2 are identified as initial correspondence points. Since P1 is connected to the capacitor C0 on the layout side for A and P2 is connected to C0 on the source side, a discrepancy results.

When YES is specified, trivial named ports are ignored even if there are similarly named ports in the corresponding source or layout netlist. Assuming YES is specified, in the preceding example the trivial ports (P2 in the layout and P1 in the source) are discarded and the source port P2 is matched with the layout port P1. Thus, the comparison result is CORRECT.

Note that pins of LVS Box cells are never considered floating. Also, since trivial port removal occurs before filtering of unused devices, ports that are rendered trivial by unused device filtering remain in the design during comparison.

Trivial named ports may appear in the layout netlist if LVS Netlist All Texted Pins YES is specified.

See also LVS Ignore Ports and LVS Check Port Names.

Examples

Assume the following is specified:

// Ignore lower-level ports that do not connect to any devices in a cell
LVS IGNORE TRIVIAL NAMED PORTS YES

Then the port P2 in the subcircuit below is ignored:

.SUBCKT A P1 P2 $$ P2 is not connected within the subcircuit.
C0 P1 P1
.ENDS

X0 1 2 A
