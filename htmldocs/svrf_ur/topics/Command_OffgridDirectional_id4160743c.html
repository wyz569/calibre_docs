Offgrid Directional

Layer operation

Checks input edges for their locations based upon an enclosing layer, a direction of measurement, a specified grid, and, optionally, the direction edges face.

Usage

OFFGRID DIRECTIONAL layer1 INSIDE OF LAYER layer2
{{{LEFT | RIGHT | TOP | BOTTOM} initial_distance step} [FACE | NOFACE]} ...
[EDGE]

Arguments

layer1

A required name of an original or derived polygon layer, or a derived edge layer.

INSIDE OF LAYER layer2

A required keyword and name of an original or derived polygon layer that encloses layer1 edges. Enclosure measurements are taken between edges (possibly extended by polygonal extents) on this layer and edges on layer1.

LEFT | RIGHT | TOP | BOTTOM

A required keyword that specifies a horizontal or vertical edge orientation and a direction of measurement. (These are referred to as directional keywords.)

LEFT applies to vertical edges and measurement in the negative x direction. RIGHT applies to vertical edges and measurement in the positive x direction. TOP applies to horizontal edges and measurement in the positive y direction. BOTTOM applies to horizontal edges and measurement in the negative y direction.

By default, input edges are checked without respect to the directions they face. When FACE or NOFACE is specified, then the direction edges face is also considered when performing measurements. The associated directional keyword determines whether an edge is facing in the direction defined by that keyword.

Directional keywords are specified with their own sets of initial_distance and step arguments. At least one keyword and argument set must be specified. Each of the directional keywords may only be specified once in the operation with FACE, NOFACE, or neither keyword, respectively. For example, the combination of RIGHT with FACE may only appear once in an operation. The same holds true for the NOFACE or neither keyword cases.

initial_distance

A required positive floating-point number or numeric expression in user units. This is the initial enclosure distance interval measured from an input edge. This argument multiplied by the database precision must produce an integer. This argument must be specified with each directional keyword.

step

A required non-negative floating-point number or numeric expression in user units. This is an enclosure distance value that is added in integral multiples to an initial_distance value to define the measurement checking grid. The step multiplied by the database precision must produce an integer.

Edges on layer1 with proper orientation and “facing” direction that are initial_distance + (step) × N units from layer2 edges are considered on grid. N takes on the values 0, 1, 2, 3, and so forth, applied internally by the operation.

FACE | NOFACE

An optional keyword that specifies to measure from an input layer edge based upon the direction the edge faces. By default, the direction an edge faces is not considered. These keywords may not be specified together within the same measurement specification of a given directional keyword.

  • FACE — Edges whose outward-facing side faces in the direction of measurement are processed.

  • NOFACE — Edges whose outward-facing side faces away from the direction of measurement are processed.

EDGE

An optional keyword that specifies the output is a derived edge layer. By default, the output is a derived error layer.

Description

Checks grid locations of input layer edges within an enclosing layer. The direction in which measurements are taken, orientation of the checked edges, and possibly the direction the edges face, are considered when making measurements.

Measurements are taken from layer1 edges to edges from enclosing layer2 polygons. Edges that are not parallel to either of the coordinate axes (the edges are angled) are not checked. The measurement grid is in top-level coordinate space. Unlike the Offgrid operation, the measurement values in Offgrid Directional are in user units.

By default, this is an error-directed layer operation that shows the edges involved and the range of the measurement (see Example 1), but this can be changed using the EDGE keyword.

Edges in the Calibre database have references back to the polygons from which the edges originate. Each edge has two sides with respect to the polygon from which the edge originates, an exterior-facing side and an interior-facing side. This is important in the context of the FACE and NOFACE keywords. FACE means that an edge is checked whose exterior-facing side faces the direction of measurement. NOFACE means that an edge is checked whose interior-facing side faces the direction of measurement. Figure 1 depicts what the edge facing and directional keywords refer to.

Figure 1. Directional and Facing Keywords for Offgrid Directional

The set of values given by the expression initial_distance + (step) ×N, where N >= 0, defines the measurement grid, which is associated with the direction defined by a directional keyword. Figure 2 depicts a measurement grid associated with the LEFT keyword.

Figure 2. Offgrid Directional Measurement Grid for LEFT Keyword

Let the set of allowed measurement grid distances be defined as G. The operation proceeds as follows.

  1. If layer1 is a polygon layer, then the polygons are decomposed into their constituent edges. If layer1 is a derived edge layer, no decomposition is required.

  2. Determine which of the input edges are in the interior of layer2 shapes or coincident with the perimeter of layer2 shapes. Only these edges are processed.

  3. For each polygon P on layer2, let (x1,y1) and (x2,y2) be the lower-left and upper-right corners of the extent of P, respectively (see Figure 2).

  4. For each edge E from Step 2 coincident with polygon P, proceed with the remaining steps.

  5. If E is a vertical edge and LEFT is specified, then the horizontal distance from x1 to E must be a member of G. Edges coincident inside with x1 are not checked. If FACE is specified, then E is checked if its exterior side faces to the left. If NOFACE is specified, then E is checked if its exterior side does not face to the left. If neither FACE nor NOFACE is specified, then E is checked.

  6. If E is a vertical edge and RIGHT is specified, then the horizontal distance from x2 to E must be a member of G. Edges coincident inside with x2 are not checked. If FACE is specified, then E is checked if its exterior side faces to the right. If NOFACE is specified, then E is checked if its exterior side does not face to the right. If neither FACE nor NOFACE is specified, then E is checked.

  7. If E is a horizontal edge and TOP is specified, then the vertical distance from y2 to E must be a member of G. Edges coincident inside with y2 are not checked. If FACE is specified, then E is checked if its exterior side faces upward. If NOFACE is specified, then E is checked if its exterior side does not face upward. If neither FACE nor NOFACE is specified, then E is checked.

  8. If E is a horizontal edge and BOTTOM is specified, then the vertical distance from y1 to E must be a member of G. Edges coincident inside with y1 are not checked. If FACE is specified, then E is checked if its exterior side faces downward. If NOFACE is specified, then E is checked if its exterior side does not face downward. If neither FACE nor NOFACE is specified, then E is checked.

  9. If edge E fails the check and the default output applies (derived error layer), then E, with its projection onto the offending side of P’s extent, along with a connection between the midpoints of the two, is output as a three-edge cluster. If EDGE is specified, then E itself is output as part of a derived edge layer.

The compiler restricts the number of directional keyword specifications in order to avoid ambiguity. For example, each of these situations is ambiguous and produces an error:

  • There are more than two RIGHT specifications.

  • There are two RIGHT specifications where neither FACE nor NOFACE is specified.

  • There are two RIGHT specifications, and both use FACE or both use NOFACE.

The same applies for LEFT, TOP, and BOTTOM.

Examples

Example 1

For the following rule, edges on the “check” layer that are inside “enclose” layer polygons are checked for leftward enclosure on a grid with distances 0.2, 0.5, 0.8 … 0.2 + (0.3)N.

left_grid { 
  OFFGRID DIRECTIONAL "check" INSIDE OF LAYER "enclose" LEFT 0.2 0.3 
}

Two results are shown in the figure (other results produced by this rule are omitted for clarity; Example 2 shows the complete set of off-grid edges).



Because neither FACE nor NOFACE has been specified, edges have been processed without consideration of which directions they face.

The top result is from an outward-facing edge on the “check” layer. The off-grid edge is 0.3 um from the extents boundary of the “enclose” layer. (Recall that the extents of layer2 polygons are used for the measurements.) If the NOFACE keyword had been used, this result would not be output.

The bottom result is from an inward-facing edge on the “check” layer. The off-grid edge is 0.3 um from an edge on the “enclose” layer (which also is coincident with the extents of the “enclose” layer.). If the FACE keyword had been specified, this result would not be output.

Example 2

Assume everything in Example 1 remains essentially the same except that the EDGE keyword is added to the operation. This causes the output to be a derived edge layer rather than a derived error layer.

left_grid2 { 
  OFFGRID DIRECTIONAL "check" INSIDE OF LAYER "enclose" LEFT 0.2 0.3 EDGE
}

Example 3

This design rule:

Vertical enclosure of diffusion by grid boundary layer with initial spacing interval of 0.05 um and additional intervals of 0.025 um. Facing edges only.

can be written as follows:

offgrid_diff {
  OFFGRID DIRECTIONAL diff INSIDE OF LAYER GB
    TOP    0.05 0.025 FACE
    BOTTOM 0.05 0.025 FACE
}
