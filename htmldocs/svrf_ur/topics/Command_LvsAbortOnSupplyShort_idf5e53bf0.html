LVS Abort On Supply Short

Specification statement

Specifies whether to abort processing when connectivity extraction detects a supply net short. Not used in ICverify.

Usage

LVS ABORT ON SUPPLY SHORT {NO | YES}

Arguments

NO

A keyword that instructs the tool not to abort processing when connectivity extraction detects a short circuit with any of the power or ground nets. This is the default behavior.

YES

A keyword that instructs the tool to abort processing after connectivity extraction detects a short circuit with any of the power or ground nets that causes a supply net name not to be applied.

Description

Specifies whether to abort processing when connectivity extraction detects a supply short with any of the nets appearing in the LVS Power Name and LVS Ground Name specification statements. This statement is similar to LVS Abort On Supply Error but can only cause the run to abort if connectivity extraction did not assign a power or ground name to a net because the name conflicted with another name that was found on the net (a text short).

If YES is specified, then circuit extraction occurs as usual, and any warnings are written in the run transcript. If a supply short is detected, the run terminates with this transcript error:

ERROR: Power or ground short detected. ABORT ON SUPPLY SHORT is specified - aborting.

When the error is given, the extracted layout netlist contains a *.CALIBRE ABORT_INFO SUPPLY_SHORT comment that causes any future LVS comparison to abort when the netlist is used and YES is specified.

If hyperscaling is used in LVS (‑hyper cmp command line option), then LVS does not terminate until after the source netlist is read.

In hierarchical runs, connectivity problems must affect the top-level cell in order for the run to abort.

In a calibre -dfm run, the DFM Database DEVICES keyword must be specified for the YES keyword to have any effect. A Calibre PERC run aborts before the PERC module if a supply error is detected and the YES keyword is specified.

The NO option (default) is useful when debugging power and ground shorts.

Examples

These rule file statements cause a combined LVS netlist extraction and comparison run to abort if there is a supply short circuit that causes a supply name not to be assigned:

LVS POWER NAME pwr
LVS GROUND NAME gnd
// Abort LVS comparison upon detection of a supply short only.
LVS ABORT ON SUPPLY ERROR NO
LVS ABORT ON SUPPLY SHORT YES

Check the circuit extraction report for short circuit information.
