DFM Classify

Layer operation

License: Calibre nmDRC-H

Groups geometries on a layer by one or more specified DFM properties and outputs a representative geometry from each group. Each output geometry is annotated with a numeric property named “Class” that contains the group number.

Usage

DFM CLASSIFY layer_name
[NOHIER] [ALL | {UNIQUE [KEEPPROPS]} ] [FLOATVAL] [DUPCOUNT]
PROPERTY property_name1 [{ABSOLUTE | RELATIVE} tolerance1]
[property_name2 [{ABSOLUTE | RELATIVE} tolerance2] ]…

Arguments

layer_name

A required argument that specifies the name of a polygon, edge, or error layer.

NOHIER

An optional keyword that causes the representative geometry for each group to be output in the top cell. The NOHIER keyword has no effect in flat runs. NOHIER is ignored when ALL is specified.

Placement information for each geometry is written to a property named “dfm_transform”. However, the “dfm_transform” property is not included if KEEPPROPS is specified.

Output is in per-cell context when NOHIER is not specified. See “Hierarchical Considerations” for more information, including information on the “dfm_transform” property.

ALL | {UNIQUE [KEEPPROPS]}

An optional keyword choice that specifies what to write to the output layer. By default, the output layer contains an arbitrary geometry from each group.

  • ALL — Outputs all geometries on the input layer_name such that the output layer is an exact copy of the input layer. NOHIER is ignored when ALL is specified.

    When ALL is specified, all DFM properties on layer_name are included on the output layer.

  • UNIQUE [KEEPPROPS] — Causes the representative geometry that is output for each for each group to be the lowest of the leftmost polygons with respect to the top cell.

    The optional KEEPPROPS keyword causes all DFM properties on layer_name to be included on the output layer. All properties, including “Class”, can be accessed by the PROPERTY function in a DFM expression. However, the property “dfm_transform” is not included when KEEPPROPS is specified. (See the NOHIER keyword for a description of the “dfm_transform” property.)

FLOATVAL

An optional keyword that causes the “Class” property to be output in a format that can be accessed by the PROPERTY function in a DFM expression. FLOATVAL is not necessary if KEEPPROPS is specified.

Without this keyword the “Class” property value can only be accessed using Calibre YieldServer commands or viewed in an RDB created of the output layer using DFM RDB.

DUPCOUNT

An optional keyword that adds the duplicate count property “DupCnt” to the output layer. The “DupCnt” property contains the total flat count of geometries in each class group.

PROPERTY property_name1 [{ABSOLUTE | RELATIVE} tolerance1]

A required argument set that specifies a DFM property by which to group the input geometries. The argument set can be specified multiple times in order to group geometries by more than one property.

  • property_name1 — Specifies the name of a DFM property attached to the input layer_name. The property must be numeric. There is no support for other types of properties, such as vector properties.

  • {ABSOLUTE | RELATIVE} tolerance1 — Specifies an optional tolerance for the property property_name1. If a tolerance is not specified, it is assumed to be 0 by default and any difference in property values causes geometries to be members of different groups.

    • toleranceN — A non-negative floating-point number. You must specify either ABSOLUTE or RELATIVE before each toleranceN argument.

    • ABSOLUTE — If two geometries have property values A and B, then the geometries are in the same group only if: |A - B| <= toleranceN.

    • RELATIVE — If two geometries have property values A and B, then the geometries are in the same group only if: |A - B| <= |toleranceN * min(A,B)|, where min(A,B) is the minimum of A and B.

Description

Groups geometries on a layer by one or more specified DFM properties Within each group, for each specified property name, the respective property values of all geometries are considered equal within the relevant tolerance range, which is 0 by default.

By default the output layer contains an arbitrary geometry from each group. All geometries on the input layer are output when the ALL keyword is specified.

Output is in per-cell context unless NOHIER is specified. However, NOHIER is ignored when ALL is specified.

Each output geometry is annotated with a numeric property named “Class” that contains the group number. Numbering starts at 1. The “Class” property can be accessed with the PROPERTY function in DFM expressions only when the FLOATVAL or KEEPPROPS keyword is specified. When ALL or KEEPPROPS is specified, all DFM properties on the input layer are copied to the output layer.

Property Value Comparisons

Because floating-point representations of numbers do not necessarily correspond to the mathematical ones, the use of non-zero toleranceN values may be needed to get the expected results. For example, 1e-05 does not necessarily correspond to how this number is represented as a binary floating-point number. Hence, the comparisons of floating-point numeric representations may not be the same as their purely mathematical counterparts. Value comparisons performed by DFM Classify account for round-off effects inherent in operations on floating-point numbers.

When values are compared within tolerances, it is possible that values A and B can be considered equal within a tolerance, and values B and C are considered equal, but values A and C are not considered equal. In other words, the comparison is not transitive. This can result in ambiguous grouping conditions. For example, if the property values are 1.0, 1.1, 1.5, and 1.6, and the absolute tolerance is 0.1, non-transitive comparisons do not occur, and the grouping of values is unambiguous. However, if the property values are 1.0, 1.1, 1.2, and 1.3, and the absolute tolerance is 0.1, the grouping is ambiguous because the value 1.1 can be grouped with 1.0 or with 1.2.

The group classification has the following characteristics:

  • Within each group, all property values for each property are equal within tolerance, whether or not any ambiguity exists in grouping of one or more properties.

  • If the grouping is unambiguous for all properties, then between different groups, no two values are equal within tolerance.

  • If the grouping by at least one property is ambiguous, then certain geometries in different groups may have property values that are equal within tolerance, and the input layer is not guaranteed to be partitioned into the smallest possible number of groups.

Concurrency and Consistency of Group Numbers

DFM Classify operations that differ only by the specification of the ALL option run concurrently and produce consistent group numbers. That is, if a geometry on the output layer of an operation without ALL specified has group number N, this geometry is a representative from the geometries annotated with the same group number N by the corresponding DFM Classify operation with ALL specified.

Hierarchical Considerations

When hierarchical Calibre executes the DFM Classify operation, geometries with equivalent property values are grouped together regardless of whether the geometries belong to the same cell. The representative geometry for each group can be chosen from any cell in which the group has at least one geometry. By default (without the NOHIER keyword), the selected representative geometry remains in the cell in which it exists on the input layer. Representative geometries can be in any cell, and if the cell is placed several times in the hierarchy, then the representative geometry will have several placements when considered flat.

When NOHIER is specified, one representative geometry for each group is reported in the top-level cell. Each representative geometry contains an additional property named “dfm_transform” that contains transform information. The UNIQUE keyword affects the placement of the representative geometry. However, the “dfm_transform” property is not attached if KEEPPROPS is specified with UNIQUE. The “dfm_transform” property is for internal use by the tool and cannot be read with DFM property access functions in a DFM expression.

Examples

Example 1

Classify polygons on layer POLY by their area and select one representative polygon from each group. Areas are considered equal if they are within one percent of each other.

POLY_A = DFM PROPERTY POLY [ A = AREA(POLY) ]
POLY_REP = DFM CLASSIFY POLY_A PROPERTY "A" RELATIVE 0.01 FLOATVAL

Example 2

Classify polygons on layer GATE by the number of contacts in the regions on layer SD which touch each gate polygon, and by the area of the gate, with a tolerance of one percent. Select one top-level placement for each representative gate polygon:

SD_N = DFM PROPERTY SD CONT [ N = COUNT(CONT) ]
GATE_NA = DFM PROPERTY GATE SD_N OVERLAP ABUT ALSO MULTI 
         [ N = PROPERTY(SD_N,"N") ] [ A = AREA(GATE) ]
GATE_REP = DFM CLASSIFY GATE_NA PROPERTY "N" "A" RELATIVE 0.01 
               NOHIER FLOATVAL
