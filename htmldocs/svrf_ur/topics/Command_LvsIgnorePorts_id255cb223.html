LVS Ignore Ports

Specification statement

Specifies whether the LVS comparison algorithm should ignore ports. Not used in Calibre PERC.

Usage

LVS IGNORE PORTS {NO | YES}

Arguments

NO

Keyword that instructs LVS to consider ports during comparison. This is the default behavior if you do not include this statement in your rule file.

YES

Keyword that instructs the tool to ignore ports.

Description

Specifies whether the LVS comparison module should ignore layout and source ports at the top level of the design. This statement controls whether LVS uses such ports as initial correspondence points, and whether it reports any discrepancies involving these ports. Note that this statement does not affect connectivity extraction.

This statement does not check port names, just the existence of ports. To compare port names, use LVS Check Port Names YES.

You can specify port behavior in Calibre Interactive nmLVS. See “Ignoring Ports” in the Calibre Interactive User’s Manual.

This statement can be specified once in your rule file. In Pyxis Layout, this statement sets the value of application variable lvs_ignore_ports.

See also LVS Non User Name, LVS Ignore Trivial Named Ports, and LVS Globals Are Ports.

Examples

// LVS should compare the existence of top-level ports.
LVS IGNORE PORTS NO
