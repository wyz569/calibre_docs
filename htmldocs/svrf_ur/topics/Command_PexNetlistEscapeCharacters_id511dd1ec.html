PEX Netlist Escape Characters

Parasitic extraction

Specifies escape character strings used in the netlist output file.

Usage

PEX NETLIST ESCAPE CHARACTERS {“string” | OFF}

Arguments

string

Specifies a list of one or more special characters enclosed in required quotation marks that are escaped in the extracted netlist. By default, netlist formats other than SPEF and SPECTRE do not automatically escape any characters.

For the SPEF netlist format, the default string is !\"#$%&'()*+,-.:;<=>?@\\^`{|}~ if this statement is not specified in your rule file.

For the SPECTRE netlist format, the default string is \"#$%&'()*+,-./:;<=>?@\\^`{|}~[] if this statement is not specified in your rule file.

OFF

A keyword that turns off escaping of special characters in the output file.

Description

Specifies escape characters that can be used in the output file. You must enclose the string within quotation marks.

This statement is useful for modifying how escape characters are handled in different netlist formats.

Examples

Example 1

To specify the set of default escape characters, include the following statement in your rule file:

PEX NETLIST ESCAPE CHARACTERS "!@#$%^&*(){}|\:;‘’<>?/.,-="

Example 2

To turn off the escaping of braces ( { } ) and slash ( / ) in netlists generated by the Calibre xACT digital flow, include the following statement in your rule file:

PEX NETLIST ESCAPE CHARACTERS OFF
