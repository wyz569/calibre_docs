DRC Connectivity Extraction and Stamp Warnings

This section describes runtime warnings related to DRC connectivity and the Stamp operation.
Table 1. DRC Connectivity Extraction and Stamp Warnings

Explanation

The following warnings are reported by all Calibre nmDRC applications and by the ICrules CHEck DRc command:

WARNING: There is no data for layout net name <name>.

The net name parameter of a Net or Not Net operation required by the check set is not the name of a net formed by the connectivity extractor. (Execution will continue with an empty layer generated for Net and the input layer generated for Not Net).

WARNING: Missing connections STAMPing layer <layer1> by layer <layer2>.

A Stamp <layer1> By <layer2> operation has been performed in a Calibre nmDRC run and some data on <layer1> was not output by the operation due to a missing connection on <layer2>. This message is only issued once per Stamp operation. The names of the layer parameters to the operation are placed in the message.

WARNING: Conflicting connections STAMPing layer <layer1> by layer <layer2>.

During a Stamp <layer1> By <layer2> operation, some data on <layer1> was not output by the operation due to a multiple connection on <layer2>. This message is only issued once per Stamp operation. The names of the layer parameters to the operation are placed in the message.

WARNING: Short circuit [in cell <name>]. Label <name> at location <point> used. Label <name> at location <point> ignored.

Connectivity extraction for Calibre nmDRC detected a redundant net name. This warning is suppressed in an area-based check in ICrules. Reporting is flat with the points reported in world coordinates. The cell name is reported in Calibre nmDRC-H (with points reported in cell coordinates) if the DRC Cell Text YES statement was specified.

WARNING: Open circuit [in cell <name>]. Label <name> used at location <point> and ignored at location <point>.

Connectivity extraction for Calibre nmDRC detected an attempt to assign the same name to two different nets. This warning is suppressed in an area-based check in ICrules. Reporting is flat with the points reported in world coordinates. The cell name is reported in Calibre nmDRC-H (with points reported in cell coordinates) if the DRC Cell Text YES statement was specified.

WARNING: Label <name> at location <point> [in cell <name>] intersects multiple nets; one net chosen arbitrarily.

Connectivity extraction for Calibre nmDRC detected a label intersecting and attached to two different nets. This warning is suppressed in an area-based check in ICrules. Reporting is flat with the points reported in world coordinates. The cell name is reported in Calibre nmDRC-H (with points reported in cell coordinates) if the DRC Cell Text YES statement was specified.

WARNING: Label <name> at location <point> [in cell <name>] is unattached and has been ignored.

Connectivity extraction for Calibre nmDRC detected an unassigned label. This warning is suppressed in an area-based check in ICrules. Reporting is flat with the points reported in world coordinates. The cell name is reported in Calibre nmDRC-H (with points reported in cell coordinates) if the DRC Cell Text YES statement was specified. The total number of these warnings in the transcript are user-controllable with the specification statement DRC Maximum Unattached Label Warnings <value>, where <value> is a non-negative integer.

WARNING: Maximum unattached label warning count of <number> exceeded. <number> unattached label warnings suppressed.

The rule file is limiting the number of unattached label warnings with the specification statement DRC Maximum Unattached Label Warnings. The limit has been exceeded and warnings beyond the specified limit have been suppressed. The number of unreported unattached labels is placed into the message.

WARNING: Virtually connected label <name> at location <point> and location <point>.

Reported whenever connectivity extraction for DRC forms a virtual connection. This warning is suppressed in an area-based check in ICrules. Reporting is flat with the points reported in the world coordinates. This warning is only reported if the Virtual Connect Report YES statement was specified.

WARNING: Sconnect conflict. Multiple stamping occurred.

An Sconnect operation, executed during connectivity extraction for Calibre nmDRC, encountered a conflicting unidirectional connection and chose a single node.

The following warnings are reported by the ICrules CHEck DRc command:

WARNING: Port <name> at location <point> is unattached and has been ignored.

Connectivity extraction for Calibre nmDRC detected a port which cannot be used to establish connectivity. This warning is suppressed in an area-based check. Reporting is flat with the point reported in world coordinates.

WARNING: Feedthrough at location <point> is unattached and has been ignored.

Connectivity extraction for Calibre nmDRC detected an explicit connect which cannot be used to establish connectivity. This warning is suppressed in an area-based check. Reporting is flat with the point reported in world coordinates.

WARNING: DISCONNECT operations are present without DRC INCREMENTAL CONNECT YES specified.

The rule file has one or more Disconnect operations but DRC Incremental Connect YES has not been specified. Disconnect operations are ignored in this case and all Connect operations are executed together.
