Historical Device Enclosure Function Reference

The commands in this section have been historically used to calculate complex device enclosure properties. This information is provided to document functionality you may find in a foundry rule file but is not the currently recommended methodology for calculating well enclosure properties.

Note:

The currently preferred interface for BSIM4 model properties is the Device Annotation Built-In Language.

The following table describes the functions available in the built-in language for well enclosure calculations.

Table 1. Device Enclosure Built-In Functions

Function

Description

Enclosure functions

These are specialized functions for calculating device enclosures.

enclosure_parallel

enclosure_parallel_multifinger

Returns a vector of numbers that correspond to trapezoidal dimensions of average enclosure distances and widths. The Device Annotation Built-In Language gives much better performance for well proximity measurements and is recommended for that application.

enclosure_perpendicular

enclosure_perpendicular_multifinger

Returns a vector of numbers that correspond to trapezoidal dimensions of average enclosure distances and widths. The device annotation built-in language gives much better performance for well proximity measurements and is recommended for that application.

enclosure_vector

Returns a vector of numbers that correspond to trapezoidal dimensions for polygons. This function has performance disadvantages compared to the other enclosure functions but is useful in simpler cases in smaller designs. The device annotation built-in language gives much better performance for well proximity measurements and is recommended for that application.

Vector function

sum

Returns the sum of vector elements. This function manipulates output from operations that produce lists of values.

TVF interface

The TVF Functional Interface for Device Property Calculation is governed by the rules of Tcl code, which are different from the Device property computation built-in language. When working with TVF Function blocks, ensure that you use valid Tcl code.

This interface is shared with the Device Annotation Built-In Language and is documented under that section.
