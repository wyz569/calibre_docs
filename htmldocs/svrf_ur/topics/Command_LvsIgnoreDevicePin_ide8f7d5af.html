LVS Ignore Device Pin

Specification statement

Specifies whether LVS comparison should ignore certain pins of a specified device. Not used in Calibre PERC.

Usage

LVS IGNORE DEVICE PIN component_type [‘(’component_subtype‘)’] pin_name [pin_name …]

Arguments

component_type

A required LVS component type. This includes names of built-in SPICE device elements, user-defined devices, primitive subcircuits, and LVS Box cells. Regular subcircuits may not be specified.

‘(’component_subtype‘)’

An optional component subtype of the component_type parameter. If specified, the component_subtype must appear within parentheses.

If component_subtype is not used, then the statement applies to all instances of the specified component_type regardless of their subtype.

pin_name

A required device pin name. More than one pin name may be specified.

Description

Specifies whether the LVS comparison module should ignore certain pins for instances of the specified component_type and optional component_subtype. This statement does not affect circuit extraction.

When this statement is used, pin swapping occurs according to how the final device, after ignoring pins, may be matched to Device statements and their pin swapping conditions. Default pin swapping during comparison occurs according to the description in the “Pin Swapping” section of the Calibre Verification User’s Manual.

This statement is similar to LVS Discard Pins By Device but does not depend on pins in Device statements.

See also LVS Report Option NP.

Examples

Assume the following netlists:

*** layout
.subckt typ 1 2 
.ends

.SUBCKT cell
x0 in1 out2 typ $[abc]
.ENDS

*** source
.subckt typ 1 2 sub
.ends

.SUBCKT cell
x0 in1 out2 sub typ $[abc]
.ENDS

Comparison of these netlists will result in a “Components with non-identical signal pins” error for the sub pin in the source.

With this specified in the rules:

// Ignore the SUB pin during comparison
LVS IGNORE DEVICE PIN typ(abc) sub

the comparison is clean.
