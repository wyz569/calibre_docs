DRC Maximum Results

Specification statement

Specifies the Calibre nmDRC maximum result count for any given rule check.

Usage

DRC MAXIMUM RESULTS {maxresults | ALL}

Arguments

maxresults

Non-negative integer that specifies the maximum result count for an individual rule check in Calibre nmDRC execution. When you do not include this statement in your rule file, the default value for maxresults is 1000.

ALL

Keyword that specifies the maximum result count is 264.

Description

Specifies the Calibre nmDRC maximum result count for any given rule check. When the maximum results are generated for a rule check, a warning is issued, and no further results are added to the DRC Results Database for that rule check. DRC checking then continues for the next rule check. You can specify this statement only once in a rule file.

Calibre applications print a warning if any rule check outputs to a GDSII- or OASIS-type results database with a maximum result count less than ALL. The warning is printed for each results database so generated. The intent is to warn of inadvertent use of the default value 1000 when writing mask data versus Calibre nmDRC errors. Setting the environment variable CALIBRE_EXIT_MAXIMUM_RESULTS to a non-null value causes Calibre applications to exit immediately after any such warnings are printed.

You can specify the maximum results limit in Calibre Interactive - nmDRC. See “Limiting the DRC Results Count” in the Calibre Interactive (New Gui) User’s Manual.

In Pyxis Layout, sets the initial default value for the maxresults switch in the $check_drc() function.

See also DRC Check Map.

Examples

Example 1

Limit the result count to 50 results per rule check.

DRC MAXIMUM RESULTS 50

Example 2

Specify no limit to the results count. Use this setting when the output is mask data.

DRC MAXIMUM RESULTS ALL
