NETPROPERTY

Property access function for DFM expressions

Accesses DFM properties of type netID from objects on the input layer. Supported only in DFM Property and DFM Function.

Usage

NETPROPERTY(layer, property_name[, ordinal])

Arguments

layer

A required layer containing the layout objects to which the property is attached.

If layer is a secondary layer in a DFM Property operation, ordinal must be specified

property_name

A required argument defining the name of the property of interest. This must be a netID property.

ordinal

A required argument if layer is a secondary layer in a multilayer DFM Property operation The ordinal specifies the index of the secondary layer geometry whose property value is to be returned. It must be a positive integer value and can itself be an expression. See the “Description” section for details on behavior and restrictions.

Return Values

netID

Description

Reads netID properties from objects on the input layer.

The net IDs returned by NETPROPERTY can be meaningfully compared within the context of the cell in which the property evaluation takes place. When a netID property is written to an object during a DFM Property operation, Calibre also keeps track of the cell context. If NETPROPERTY is used to read the netID property as part of a DFM Property operation in a different cell context, Calibre makes the appropriate translation for the netID values so that they are meaningful in the current cell context. Also see “Cell-Context-Dependent Property Functions Results in Hierarchical Batch Runs”.

If the BY CELL or BY NET keyword is used in the DFM Property operation, the netID properties are ignored when computing the cell and net properties.

Restrictions

The following restrictions apply to the DFM Property operation in which the NETPROPERTY function is used:

  • The INSIDE OF keyword may not be used.

  • The OVERLAP or NODAL keyword must be specified for multilayer DFM Property operations; in other words, the default clustering method of INTERSECTING is not allowed.

  • A constraint cannot be specified for a property of type netID. The equality (==) and inequality (!=) operators can be used within a conditional expression to compare two properties of type netID. See the “Examples” section and “Comparison of DFM Properties”.

Behavior with Ordinal

The ordinal argument is used to read a property value from a specific object on a secondary layer in a multilayer DFM Property operation. The clustering method must be OVERLAP or NODAL.

Layout objects on secondary layers are numbered beginning with 1. The order in which the layout objects are processed is arbitrary, however, each secondary layer object is assigned a unique ordinal which is consistent within the same DFM Property operation.

  • The ordinal is required if layer is a secondary layer.

  • For ordinal = 1 through the total number of clustered geometries on the secondary layer, the function call returns the value of the specified property on the specified secondary layer geometry clustered with the primary layer geometry.

  • Multiple function calls in the same DFM Property operation that reference the same layer have a consistent order for the secondary layer objects. For example, PROPERTY(S, “X”, 1) and VPROPERTY(S, “Y”, 1) return property values from the same geometry when used in the same DFM Property operation.

  • If the ordinal value is out of range (either greater than the number of clustered geometries or less than 1), the function fails to evaluate.

Examples

LAYER layer1 1
LAYER layer2 2
LAYER layer3 3
LAYER layer4 4
CONNECT layer1 layer2
CONNECT layer3 layer4

// write globalnetids for primary layer
gnetid_layer = DFM PROPERTY layer1 
		[gnet_num = globalnetid(layer1)] 

test_eq  = DFM PROPERTY layer3 gnetid_layer OVERLAP  
  			[netprop_eq = ( NETPROPERTY(gnetid_layer, gnet_num, 1) == 
	 			NETPROPERTY(gnetid_layer, gnet_num, 2)) ? 1 : 0 ]  == 1
