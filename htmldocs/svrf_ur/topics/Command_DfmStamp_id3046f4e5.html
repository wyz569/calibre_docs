DFM Stamp

Layer operation

License: Calibre nmDRC-H

Produces an output layer that is a copy of the input layer, with node numbers assigned based on a specified DFM property on the input layer. DFM properties on the input layer are copied to the output layer.

Usage

DFM STAMP input_layer BY "netID_property"

Arguments

input_layer

A required argument that specifies a polygon layer or a derived edge layer.

BY “netID_property”

A required keyword and argument that specify the name of a netID property on the input layer. The property name must be enclosed in quotes (“ ”).

Description

Produces an output layer that is a copy of the input layer. Node numbers on the output layer are set by the values of the netID_property. The output layer includes all DFM properties attached to the input layer.

If node numbers on the output layer are not required for rule file processing, only geometries and DFM properties are copied from the input layer.

Examples

Example 1

The node numbers on layer L1_s are assigned using the net ID properties from metal2 shapes that are overlapped by layer L1. If an L1 shape overlaps more than one metal2 shape, the net ID of the first metal2 shape is used. Layer L1_s is then written to a DFM RDB file using the DFM RDB statement:

CONNECT metal2

L1_M2_net = DFM PROPERTY L1 metal2 [netNum = NETID(metal2, 1)]
L1_s = DFM STAMP L1_M2_net BY "netNum"

write_db { 
   DFM RDB L1_s stamp.rdb NODAL
}

Example 2

This example is similar to the previous example, but uses VBYNETMAX to determine the net ID.

Suppose that each WELL polygon encloses several CONT polygons, and not all of the CONT polygons have the same node number. The DFM Property operation annotates each WELL_NET polygon with the property netNum, whose value is the net ID that corresponds to the most CONT shapes enclosed by the WELL polygon. For example, if a well polygon encloses three contacts with net ID 5 and one contact with net ID 2, the value of the property netNum is 5.

For each polygon on layer WELL_NET, the DFM Stamp operation assigns the netNum value as the node number for the shape on the output layer WELL_S.

The DFM Property operation also calculates the property A_CONT, the total area of the CONT shapes enclosed by the WELL polygon. The property A_CONT is copied to the output layer WELL_S. 

CONNECT CONT

WELL_NET = DFM PROPERTY WELL CONT OVERLAP 
            [ netNum = VBYNETMAX( VNETID(NETID(CONT)) ) ]
            [ A_CONT = AREA(CONT)]

WELL_S = DFM STAMP WELL_NET BY "netNum"

write_db { 
   DFM RDB WELL_S stamp.rdb NODAL
}
