PEX Netlist Short Toplevel Ports

Parasitic extraction

Specifies whether to create multiple top-level ports in the extracted DSPF, SPEF, HSPICE, or CalibreView netlist.

Usage

PEX NETLIST SHORT TOPLEVEL PORTS {YES | NO}

Arguments

YES

Required keyword that specifies not to create multiple top-level ports in the output netlist. This is the default.

NO

Required keyword that specifies to create multiple top-level ports in the output netlist.

Description

This statement specifies whether to create multiple top-level ports in the DSPF, SPEF, HSPICE, or CalibreView netlist. By default, multiple ports are not uniquely identified or netlisted but are shorted through a common port. If PEX NETLIST SHORT TOPLEVEL PORTS NO is specified when the netlist format is DSPF or HSPICE, the replicated ports are also added to the “.subckt” definitions.

Examples

To create multiple top-level ports in the extracted netlist, include the following statement in your rule file:

PEX NETLIST SHORT TOPLEVEL PORTS NO
