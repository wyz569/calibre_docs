DRC Check Text

Specification statement

Specifies the amount of rule check text to appear in Calibre nmDRC results databases.

Usage

DRC CHECK TEXT [{[COMMENTS | ALL] [RFI]} | NONE] [filename]

Arguments

At least one keyword or the filename parameter must be used when the statement is explicitly specified:

COMMENTS

An optional keyword that instructs the tool to output rule check comments (those preceded with an “@” symbol) to the DRC Results Database. These comments appear in Calibre RVE. COMMENTS RFI is the default.

ALL

An optional keyword that instructs the tool to output the entire text of each rule check to the DRC results database. May not be used with COMMENTS.

RFI

An optional keyword that instructs the tool to output the rule file title and pathname to the DRC results database. COMMENTS RFI is the default.

NONE

An optional keyword that instructs the tool to not output any rule check text or rule file information to the DRC results database. May not be used with any other keywords.

filename

An optional argument that specifies the name of an output file for creating an ASCII DRC results database in which to output all selected rule checks and check text defined by the DRC Check Text statement. This file does not contain any rule check results.

Description

Specifies the amount of rule check text to appear in Calibre nmDRC results databases.

Text consists of @-sign comments, rule file information, and Calibre nmDRC rule check text. With this statement you can include all or a subset of the text in the Calibre nmDRC results database.

You can only specify this statement once.

In ICverify, sets the initial default value for the textmode switch in the $check_drc() function.

Virtuoso users can specify DRC Check Text COMMENTS to view the check text while stepping through Calibre nmDRC or LVS results database errors.

Examples

Example 1

DRC CHECK TEXT COMMENTS RFI 
// place comments, title, and rule file pathname in DRC DB

Example 2

// Output a DRC results database-like file with the COMMENTS and 
// RFI information included and but no polygon data.
DRC CHECK TEXT check_text.db
