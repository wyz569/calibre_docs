Mapping of SVRF Commands to TVF

SVRF specification statements and layer operations are handled differently in TVF commands due to the different way in which the commands can appear in the SVRF language.

Most SVRF specification statements, device extraction statements, connectivity extraction operations, and fracture operations are easy to translate into compile-time TVF commands. Specification statements typically occur in families, where each family has a predictable initial keyword. Each initial keyword is mapped to a TVF command using the tvf:: namespace prefix. The remainder of the words and parameters in the SVRF statements are passed to the TVF command as arguments.

Layer operations do not necessarily have this feature. Layer operations must appear either in a layer derivation (like x = y AND z) or in a rule check block that begins with an arbitrary name. In such cases, there is no way to predict the first token of these constructs. Hence, there is no way to map the layer operation name as a command in compile-time TVF. Instead, TVF uses the tvf::SETLAYER and tvf::OUTLAYER commands to handle layer assignment, derivation, and results output. The layer operation is passed to these command as command arguments and the layer operation itself is not mapped to the tvf:: namespace.

For example, a DRC Select Check statement is coded like this in TVF:

tvf::DRC SELECT CHECK size_rule

where tvf::DRC is the TVF command name and the remaining elements are arguments.

A rule check, layer derivation, and results output are coded like this:

tvf::RULECHECK my_rule {
tvf::SETLAYER layer1 = {layer2 OR layer3}
tvf::OUTLAYER {layer4 NOT layer1}
}

where layer operations involving the OR and NOT operations are passed as arguments to the tvf::SETLAYER and tvf::OUTLAYER commands.
