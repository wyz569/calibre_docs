PEX Reduce Via Capacitance

Parasitic extraction

Specifies how to handle clusters of identically-connected vias when calculating capacitance. Used only in Calibre xACT 3D.

Usage

PEX REDUCE VIA CAPACITANCE [layer1 layer2]
{{STANDARD [DISTANCE value]} | OFF}

Arguments

layer1 layer2

An optional argument pair specifying a particular set of vias to which the statement applies. The layers must be connected to the vias and have connectivity and capacitance. If layer1 layer2 are not present, the rule applies to all vias in the design.

STANDARD [DISTANCE value]

A keyword that specifies standard via reduction. A cluster of vias is reduced to one via.

The optional keyword set DISTANCE value specifies the maximum side-to-side distance in microns between vias used to define a cluster. Vias within the distance <= value are considered members of the same cluster. Vias at a distance > value are not part of the cluster. The default distance value is the top conductor min_width in microns.

OFF

A keyword which turns off via reduction. Turning via reduction off creates a netlist that contains capacitors representing all vias in the layout. Using this keyword may cause a substantial increase in parasitic database and netlist size. It is recommended to use this keyword when specifying layers. This keyword is not recommended for use on designs with large via clusters.

Description

An optional statement that specifies how to handle clusters of identically-connected vias when calculating capacitance. (It does not affect resistance extraction.) Via clusters are identified and only the vias around the perimeter of each cluster are used when extracting capacitance. Use this operation without layers to indicate a default reduction for all vias, or for particular via layers indicating connecting layer pairs. This statement can be specified more than once. If this statement is not specified, then via reduction does not happen and all vias are processed.

The optional DISTANCE value keyword set identifies clusters; vias that are within the distance value are considered to be part of the cluster, and vias further away are not.

Examples

Example 1

To turn off via reduction for all layers add the following line to your rule file:

PEX REDUCE VIA CAPACITANCE OFF

Example 2

To turn off capacitance via reduction between layers metal1 and metal2, add the following line to your rule file:

PEX REDUCE VIA CAPACITANCE metal1 metal2 OFF

Example 3

To perform standard capacitance via reduction on vias within 5 microns of each other for all layers, add the following line to your rule file:

PEX REDUCE VIA CAPACITANCE STANDARD DISTANCE 5

In this example, vias within 5 microns form the clusters and only the perimeter vias are considered for capacitance extraction. Vias within the clusters are ignored as shown in Figure 1.
