VTUPLE

Vector function for DFM expressions

Creates a vector of numeric type properties. It is most useful in DFM Property Select Secondary, where it creates one tuple for each cluster that is grouped with the select geometry.

Usage

VTUPLE(arg_expr1[, arg_expr2] …)

Arguments

arg_expr1[, arg_expr2]

A required numeric value or an expression that evaluates to a numeric value. Multiple arguments are separated by commas. The number of arguments corresponds to the tuple size; each argument is evaluated to form a tuple element.

Each argument can be an expression involving DFM measurement functions and supported math functions. Measurement functions can reference either the primary or secondary layer(s). See the Description section for the behavior with primary or secondary layers. The property reference functions PROPERTY_REF is supported.

Return Values

Vector of numeric values.

Description

VTUPLE creates a vector with tuple elements consisting of the arguments to the function. This function is only supported in the DFM Property family of operations, and is most useful in DFM Property Select Secondary. One tuple is created per cluster.

The terminology used for vector notation is described in “Vector DFM Properties”. In brief, {} indicates a vector and <> indicates a tuple within the vector. For the vector {<1 2 3> <4 5 6>}, each tuple has 3 elements (tuple size 3) and there are 2 tuples (tuple count 2). The tuple count is also called the vector length.

The argument expressions (arg_expr) are evaluated to form the tuple elements; the number of arguments determines the tuple size. There is one tuple for each cluster associated with the geometry on the output layer. For DFM Property Select Secondary there can be multiple clusters associated with each select layer geometry written to the output layer. For the other DFM Property operations there is one cluster associated with each primary layer object written to the output layer.

The evaluation of arg_expr depends on what layer (if any) the argument expression references and whether the VTUPLE function is used in the Per-Cluster Property Definition or Per-Select Geometry Property Definition of the DFM Property Select Secondary operation or within one of the other DFM Property operations.

  • VTUPLE in Per-Cluster Property Definition of DFM Property Select Secondary

    If arg_expr  references a secondary layer (including the select layer), each function in the expression is summed over the cluster. The tuple count is equal to the number of clusters.

  • VTUPLE in Per-Select Layer Property Definition of DFM Property Select Secondary

    The tuple count is equal to the number of clusters.

    • arg_expr references the select layer — Each function in the expression is evaluated for the select layer object. If the select layer object belongs to more than one cluster, the same tuple value is included for each cluster. In other words, no summing takes place.

    • arg_expr references a secondary layer that is not the select layer — Each function in the expression is summed over the cluster.

  • VTUPLE in a DFM Property operation other than DFM Property Select Secondary

    If arg_expr references a secondary layer, each function in the expression is summed over the cluster. The tuple count is one.

The examples illustrate the behavior of VTUPLE in DFM Property Select Secondary with a simple layout.

If the arg_expr evaluation fails for a tuple, that tuple is not included in the resulting vector. The VTUPLE vector is still created as long as the expression evaluation succeeds for at least one cluster. If the argument evaluation fails for all tuples for a particular select layer geometry, that geometry is not included in the output layer. Expression Chains can be used to protect against evaluation failure; see “Example 2”.

Examples

Example 1

This example demonstrates the behavior of VTUPLE with a simple layout. 

LAYER m1 2
LAYER m2 1
LAYER m3 3

vtup_ex = DFM PROPERTY SELECT SECONDARY m1 m2 m3 OVERLAP ABUT ALSO MULTI
		[ vtup_area_primary = VTUPLE( AREA(m1))]
		[ cl_m2_vec = VECTOR( AREA(m2) )]
		[ cl_m2_vtup = VTUPLE( AREA(m2) )]
		SELECT m2
		[ geom_m2_vtup = VTUPLE( AREA(m2) ) ]
		[ geom_m3_vtup = VTUPLE( COUNT(m3), AREA(m3) ) ]

vtup_out { DFM RDB vtup_ex vtup.rdb }

The per-cluster property definition includes the property cl_m2_vec calculated using VECTOR and the property cl_m2_vtup calculated using VTUPLE so that the resulting vectors can be compared.





The following table explains the results. For the properties that result from a VTUPLE calculation, Results 1 and 2 belong to only one cluster, so the vector has one tuple. Results 3 and 4 belong to two different clusters, so the vector has two tuples.

Property

function call

Context

Comment

vtup_area_primary

VTUPLE(AREA(m1))

cluster

The tuple value is the area of the primary layer object.

cl_m2_vec

VECTOR(AREA(m2))

cluster

Returns a vector of the areas of the m2 layer objects that are part of the cluster. Results 3 and 4 belong to two clusters, so the vectors are concatenated.

cl_m2_vtup

VTUPLE(AREA(m2))

cluster

The tuple value is the sum of the m2 area for the cluster.

geom_m2_vtup

VTUPLE(AREA(m2))

select geometry

The referenced layer is the select layer, so the tuple value is the area for each select layer object (no summing).

geom_m3_vtup

VTUPLE(COUNT(m3), AREA(m3))

select geometry

There are two arguments to the VTUPLE function, so each tuple has two elements. The referenced layer is not the select layer, so the tuple elements are the total count of m3 objects and total m3 area for the cluster.

Example 2

This example uses the same layout as Example 1 in order to demonstrate the behavior with evaluation failure for one cluster and how to use expression chains.

This operation calculates two properties:

vtup_err = DFM PROPERTY SELECT SECONDARY m1 m2 m3 OVERLAP ABUT ALSO MULTI
	[ primary_area = VTUPLE( AREA(m1) )]
	[ geom_m3_ratio = VTUPLE( COUNT(m3), perim(m3)/AREA(m3) ) ]
	SELECT m2

It results in a divide by zero in the geom_m3_ratio property for the Result 4 shape when clustered with the rightmost layer1 shape, which has no m3 shapes in the cluster. The output for Result 4 does not include the corresponding tuple in either property.

The following operation uses an expression chain to provide a value in case of evaluation failure:

vtup_err = DFM PROPERTY SELECT SECONDARY m1 m2 m3 OVERLAP ABUT ALSO MULTI
		[ geom_m3_ratio = VTUPLE( COUNT(m3), perim(m3)/AREA(m3) ) ]
			[geom_m3_ratio = VTUPLE( COUNT(m3), 99)] // chained expression
		[ primary_area = VTUPLE( AREA(m1) )]
	SELECT m2
