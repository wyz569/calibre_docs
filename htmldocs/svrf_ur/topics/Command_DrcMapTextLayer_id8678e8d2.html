DRC Map Text Layer

Specification statement

Specifies layers from which to map text objects to the mask DRC results database. Used only in Calibre nmDRC-H.

Usage

DRC MAP TEXT LAYER { use_layer [use_layer …] |
    [use_layer [use_layer …]] IGNORE ignore_layer [ignore_layer …] }

Arguments

use_layer

A name or number of an original layer on which text objects reside in the input layout database. Text objects on layers specified using this argument are mapped to the output. When the IGNORE keyword is not used, this argument is required. More than one use_layer may be specified.

IGNORE ignore_layer

A keyword set that instructs the tool to ignore the specified layers when mapping text objects to the output. The ignore_layer is the name or number of an original layer on which text objects reside in the input layout database. More than one ignore_layer may be specified. If no use_layer is specified, then the IGNORE keyword set must be specified.

Description

Specifies which text object layers are output when DRC Map Text YES is specified. By default, text objects from all layers are candidates for output with DRC Map Text YES. DRC Map Text Layer may be specified any number of times.

If a simple layer name is used for the use_layer or ignore_layer arguments, then the semantics are equivalent to the use of its layer number in the statement. If a layer set name is used, then the semantics are equivalent to using all of its constituent simple layer numbers in the statement.

The statement works as follows. Let T be a text object on layer L, for which T is a candidate for output due to specification of DRC Map Text YES. If there are no DRC Map Text Layer specification statements, then T is output. Otherwise T is output if one of the following is true:

  • There are no use_layer arguments specified and L is not an ignore_layer.

  • There is at least one use_layer specified, L is on a use_layer, and L is not on an ignore_layer.

The use_layer argument may be a target_layer argument in a Layer Map statement. In such cases, any text objects on the use_layer are output according to the corresponding Layer Map target_layer specification.

See also DRC Map Text Depth.

Examples

Example 1

Basic usage:

LAYER txt 20
LAYER port_txt 21
DRC RESULTS DATABASE cell.gds GDSII
DRC MAXIMUM RESULTS ALL
DRC MAP TEXT YES
DRC MAP TEXT DEPTH ALL
// output text objects from these layers only
DRC MAP TEXT LAYER txt port_txt

Example 2

This example shows the use of text layer mapping. The layout has the following data: polygons on layer 28 datatype 0; text objects on layer 28 texttype 28. The texttype of 28 is maintained for the mapped text objects throughout the flow.

This shows an appropriate method for mapping the layout data to a different layer number in the flow.

DRC RESULTS DATABASE cell.gds GDSII
DRC MAXIMUM RESULTS ALL
DRC MAP TEXT YES // send text objects to the output
DRC MAP TEXT DEPTH ALL
DRC MAP TEXT LAYER 1000 // map only text objects 
                        // from this layer for output

LAYER MAP 28 DATATYPE == 0 1000
LAYER MAP 28 TEXTTYPE == 28 1000 // texttype 28 is maintained

LAYER mapped_28 1000

Text objects from the target layer 1000 datatype 28 are mapped for output.

Example 3

If you want to output text objects using a different texttype than they intrinsically have, the Layer Map statement supports this. In the preceding example, the texttype of 28 would be preserved upon output of the text:

LAYER MAP 28 TEXTTYPE == 28 1000 // texttype 28 is maintained

This statement would change the texttype to 0:

LAYER MAP 28 TEXTTYPE == 28 1000 0 // texttype 0 is used
