DRC Cell Name

Specification statement

Specifies whether to append the cell name to its signature line in the hierarchical results. Used only in Calibre nmDRC-H.

Usage

DRC CELL NAME NO

DRC CELL NAME YES [CELL SPACE] [XFORM] [ALL]

Arguments

NO

Keyword that specifies not to append the cell instance name associated with each Calibre nmDRC result to its signature line in the Calibre nmDRC results database. This is the default behavior when you do not include this statement in the rule file.

YES [CELL SPACE] [XFORM] [ALL]

Keyword set that specifies to append the cell name associated with each Calibre nmDRC result to its signature line in the results database. When you specify YES, you can also specify any or all of the following optional keywords:

  • CELL SPACE — Specifies to output the coordinates of the result in the coordinate space of the cell in which it is instantiated. Top-level coordinate space is the default if you specify YES.

  • XFORM — Specifies to append the Calibre nmDRC result’s transformation information to its signature line in the results database. This keyword is useful if you want to view results in cell-level coordinates in Calibre RVE.

  • ALL — Specifies to list the cell name and transformation information after every Calibre nmDRC result in the database. By default the tool does not append this information to results that are identical to previous results.

Description

Specifies whether to append the cell name associated with each DRC result to its signature line in a hierarchical DRC Results Database or DRC Check Map results. This statement applies to ASCII DRC results databases only and can be specified at most once.

If you specify YES, then Calibre nmDRC-H places the cell name associated with each Calibre nmDRC result on a line beginning with “CN”. The CN line also includes a total flat placement count of the cell referenced in the property. In the following example, the flat count is 6.

TOP 1000
rule_A
15918 15918 1 May 12 08:32:00 2003              
DENSITY M1M PGM >= 0.25 WINDOW 100
p 1 4
CN NAND034 6
-8936900 -4262700
-8935900 -4262700
-8935900 -4261700
-8936900 -4261700
...

If the cell is a clone produced by the Layout Clone Rotated Placements YES or Layout Clone Transformed Placements YES statements, then wherever the “CN” property is attached an additional property of “CL” is also attached:

...
e 16 2
CN NAND034$C2$ c 0 1 -1 0 4562 1348 6
CL NAND034 1 0 0 -1
-2345 -34125 35467 986
...

Note the cell name “$C#$” in the CN property is a Calibre convention used for names of cloned cells. The first parameter of the CL property is the name of the original cell from which the clone is created. The numerics in the CL property form the upper-left 2 × 2 sub-matrix of the transform matrix which was applied to the original cell to produce the clone, similar to the transform in the CN property. The translational components of this transform are always zero and, hence, are not output.

This information, as well as what is described next, enlarges the format of the ASCII DRC results database.

By default, Calibre nmDRC-H reports each DRC result in top-level cell coordinates. If you specify YES with the CELL SPACE keyword, the DRC results are reported in the coordinate space of the cells in which they are instantiated. A c character is written after the cell name in the signature line. This allows you to determine which coordinate space a DRC result was output in. For example:

CN NAND034 c 6

If you specify the XFORM keyword, Calibre nmDRC-H adds the transformation information for each result to the signature line in the results database. If CELL SPACE is also specified, the transformation information contains six integers that are part of a 3 × 3 matrix representing the object’s transformation from cell space to top-cell space coordinates. Without CELL SPACE specified, the matrix is the inverse transformation to cell-space coordinates from top-cell space coordinates.

Cell location coordinates are vectors (x, y, 1), where x, y is a location of a point in database units. Cell location vectors are multiplied by the 3 × 3 transformation matrix to map the location coordinates into the appropriate space.

This is an example for CELL SPACE XFORM:

CN NAND034 c 0 1 -1 0 456547 377748 6

It gives Calibre nmDRC results where the first six numbers after the c character represent the transformation matrix information and the last integer is the flat placement count.

In this example, the row vectors of the matrix would be (000, 101, 002), (-110, 011, 012), (45654720, 37774821, 122). The bold numbers correspond to the integers after the c character. Each cell has a unique set of transformation numbers that represents the transformation of exactly one of its placements in the hierarchy. Generally, this is the hierarchically lowest, left-most placement of the cell. The transformation numbers are used consistently for all DRC results from that cell.

Matrix elements given by the subscripts 00, 01, 10, and 11 control both reflection and rotation, and can have the values -1, 0, or 1. There are eight possible combinations for these four elements, which correspond to the possible unique combinations of reflection (0 or 1 for no or yes) and rotation (0, 90, 180, or 270 degrees) of a cell. Elements with subscripts 20 and 21 are in database units. Elements with subscripts 02 and 12 always have values of 0. Element 22 is always 1.

Floating-point transformation numbers are not supported. They are automatically removed from the input layout database by flattening or cloning. Calibre nmDRC-H does not list the cell’s transformation data after multiple outputs of the same cell name.

The XFORM keyword is generally used by other tools such as Calibre RVE. Calibre RVE uses the transformation data to allow error highlighting in both the cell and top-level contexts. Refer to the section “Viewing Results in Cell Space” of the Calibre RVE User’s Manual for information on how DRC Cell Name affects Calibre RVE behavior.

By default, Calibre nmDRC-H minimizes the ASCII DRC results database size by omitting this information for any Calibre nmDRC result that is identical to that of the previous result (with recurrence). The information is provided for the first result in each Calibre nmDRC rule check. The ALL keyword overrides this default behavior.

You can specify for Calibre to use cell-level coordinates for results reporting in Calibre Interactive—nmDRC. See “Reporting DRC Results in Local Cell Space or Top Cell Space” in the Calibre Interactive (New Gui) User’s Manual.

Examples

DRC CELL NAME YES CELL SPACE XFORM 
// output results in cell-space coordinates and provide 
// transform matrix elements.
// results are shown in Calibre RVE in cell space.
