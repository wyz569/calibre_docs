device::select_left_bottom_device_marker

Device TVF layer generation function. Used only in a compile-time TVF rule file.

Returns a derived polygon layer containing marker shapes coincident with the bottom-left corners of device seed shapes (where “bottom” and “left” can change due to cell placement transformations, but is always consistent with a certain corner). Input layers must be rectangular with edges parallel to the database axes. Pin shapes must be coincident with edges of seed shapes on each side of a seed shape. Devices that do not conform to these criteria cause the function to have empty output. This function is always called in the context of a tvf::SETLAYER command.

Usage

device::select_left_bottom_device_marker seed_layer pin_layer marker_width [‑help | ?]

Arguments

seed_layer

A required argument that specifies a device seed layer. Usually this is the gate layer of a MOS transistor.

pin_layer

A required argument set that defines a pin layer having shapes that touch the seed_layer. Usually this is a source/drain layer of a MOS transistor.

marker_width

A required argument that specifies a floating-point number in user units of length. This is the output width of marker polygons. This value must be less than one half the length of the minimum seed shape edge or no output occurs.

-help | ?

Prints the usage message for this function in a Tcl shell.

Return Values

Derived polygon layer.

Examples

In this example, four enclosure properties are calculated for NMOS devices. The property values are the measurements from the corners of ngate polygons to the enclosing corners of pwell polygons. Four ngate corner marker layers are derived, and then DFM property layers for the enclosure measurements are generated using the marker layers. Then the device annotation layer “n_annot_layer” is derived from the DFM property layers and used for attaching properties to ngate devices.

The following figure shows an ngate polygon with associated corner marker polygons. The “encTL” measurement is one of four such measurements that are taken, and these are used as device properties.

The following rule file calculates the device properties using the LVS Annotate Devices interface. This rule file can be sourced from other circuit extraction rule files.

#! tvf

package require CalibreDFM_DEVICE 
namespace import tvf::// tvf::setlayer tvf::outlayer tvf::verbatim \tvf::rulecheck

verbatim {
// include layer definitions and connectivity rules.

// required for the LVS Annotate Devices flow
MASK SVDB DIRECTORY svdb QUERY CCI
LVS ANNOTATE DEVICES n_annot_layer

// well enclosure proximity distance
VARIABLE wencpd 5

// device orientation marker width
VARIABLE mw 0.010
}
// ngate corner markers
setlayer ngate_BL = [device::select_left_bottom_device_marker  ngate nsd \
                     mw]
setlayer ngate_TL = [device::select_left_top_device_marker     ngate nsd \
                     mw]
setlayer ngate_BR = [device::select_right_bottom_device_marker ngate nsd \
                     mw]
setlayer ngate_TR = [device::select_right_top_device_marker    ngate nsd \
                     mw]
verbatim {
// macro that derives corner edges, gets horizontal and vertical enclosure 
// measurements, and then gets the distance between a seed shape corner
// and the enclosing corner of a well polygon.
// it is assumed there is only one horizontal and vertical enclosure
// distance from a seed shape corner.
DMACRO corner_edges marker seed well "OUTPUT" {
  corner_edges = marker COIN INSIDE EDGE seed
  corner_enc   = ENC corner_edges well < wencpd OPPOSITE
  corner_enc_p = DFM PROPERTY marker corner_enc OVERLAP MULTI
                   [ "+enc_x" = MAX(EWX(corner_enc)) ]
                   [ "+enc_y" = MAX(EWY(corner_enc)) ]
 [ enc_d = SQRT((PROPERTY_REF("+enc_x")^2) + (PROPERTY_REF("+enc_y")^2)) ]
  OUTPUT = DFM COPY corner_enc_p
}
// derive device DFM property layers
CMACRO corner_edges ngate_BL ngate pwell ngate_BL_p
CMACRO corner_edges ngate_TL ngate pwell ngate_TL_p
CMACRO corner_edges ngate_BR ngate pwell ngate_BR_p
CMACRO corner_edges ngate_TR ngate pwell ngate_TR_p

// macro for device annotation properties
DMACRO mos_prop bottom_left top_left bottom_right top_right {
  [ 
    PROPERTY encBL, encTL, encBR, encTR
    W = DEVICE_NUMERIC_VALUE( "W" )
    L = DEVICE_NUMERIC_VALUE( "L" )
 // get the four enclosure property values.
    encBL = dfm_numeric_value(bottom_left,"enc_d")
    encTL = dfm_numeric_value(top_left,"enc_d")
    encBR = dfm_numeric_value(bottom_right,"enc_d")
    encTR = dfm_numeric_value(top_right,"enc_d")
  ]
}

// derive MN device annotation layer
n_annot_layer = DEVICE LAYER MN 
                ANNOTATE ngate_BL_p ngate_TL_p ngate_BR_p ngate_TR_p
               CMACRO mos_prop ngate_BL_p ngate_TL_p ngate_BR_p ngate_TR_p

DEVICE MN ngate ngate(G) nsd(S) nsd(D) pwell(B)

Derivations for PMOS devices would follow similar code. The corner_edges and mos_prop macros can apply to PMOS devices without modification.

qs::write_agf annotated_MN.agds
