PEX Magnify

Parasitic extraction

Specifies to scale the layout before parasitic extraction. Supports optical shrink for in-die variation processes.

Usage

PEX MAGNIFY shrink_value [PROPERTY]

Arguments

shrink_value

A required, positive, floating-point number that specifies the amount by which to scale the layout.

PROPERTY

An optional keyword that indicates to scale the device parameters.

Description

This statement specifies to scale the layout before parasitic extraction. This occurs before in-die variation mapping. The shrink_value parameter is applied to a polygon’s width, length, and spacing properties. This statement can only be specified once. If this statement is specified more than once with different scaling values, extraction stops with an error.

For Calibre xACT and Calibre xACT 3D, your MIPT file must contain the global parameter half_node_scale_factor in order to ensure that scaling is properly handled in the calibrated rule files and the PEX Magnify statement is output by the xCalibrate rule file generator. For more information on MIPT files, refer to the xCalibrate Batch User’s Manual.

Examples

You have a 130 nm design that you want to use with a 110 nm process. In the 130 nm process, the minimum width of metal1 is 0.130 microns. In the 110 nm process, the minimum width of metal1 is 0.110 microns (eighty-five percent of 130 nm), so you need to use a shrink value of 0.85 to shrink your 130 nm design 85 percent. 

PEX MAGNIFY 0.85

With a shrink value of 85 percent, the 0.130 micron line is scaled to the drawn value of 0.110 microns (0.130 × 0.85 = 0.110).
