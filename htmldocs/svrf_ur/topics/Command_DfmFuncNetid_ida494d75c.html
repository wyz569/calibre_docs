NETID

Net based function for DFM expressions

Returns the net ID associated with a geometry on the input layer as type netID. Supported only for DFM Property and DFM Property Merge operations. Connectivity is considered for the cell the property expression is evaluated in and that cell’s descendants; connectivity at a higher level in the hierarchy is not considered.

Usage

NETID(layer[,ordinal])

Arguments

layer

A required layer with connectivity information; may not be an error layer.

ordinal

An optional argument that refers to the index of the secondary layer geometry. The ordinal is required if layer is a secondary layer unless NETID(layer) is an argument to VNETID.

The numbering sequence of all secondary layer geometries clustered with a given primary layer is arbitrary, but is consistent for all expressions within a DFM Property operation. The ordinal numbers start with 1. If the ordinal number exceeds the number of geometries in the cluster, the expression evaluation fails; evaluation failure also occurs if the primary layer object has no secondary layer objects clustered with it.

If ordinal is specified and layer is the primary layer, the value of ordinal must be 1.

Return Values

Net IDs of type netID.

Description

NETID returns the net ID associated with a geometry on the layer argument. NETID is supported only in DFM Property and DFM Property Merge operations. The INSIDE OF keyword set for DFM Property cannot be used if the NETID function is used.

The NETID function returns netIDs that are local to the cell in which the property expression is evaluated. Connectivity is considered for that cell and its descendants, unlike GLOBALNETID, which considers connectivity at all levels in the hierarchy. If two values produced by NETID are different, all that can be concluded is that the corresponding polygons are on different nets in the connectivity model determined by the geometries within the current cell and its descendants. NETID does not find high shorts.

The function behaves as follows, depending on the layer argument and whether ordinal is used:

  • NETID(primary_layer) — Returns the net ID of the primary layer object.

  • NETID(secondary_layer) — If a secondary layer is referenced without an ordinal, the function must be used as an argument to VNETID: VNETID(NETID(secondary_layer)). VNETID creates a vector of net ID values with the number of tuples equal to the number of secondary layer objects clustered with the primary layer object, where the tuple value is the net ID for the secondary layer object.

  • NETID(secondary_layer, ordinal) — Returns the net ID associated with a particular secondary layer geometry clustered with the primary layer, where ordinal gives the index of the secondary layer geometry.

The following case illustrate some the of usage and concepts. The layout has two placements of cellA, where cellA has three layer1 polygons. One placement of cellA has a layer2 polygon in the topcell shorting two layer1 polygons in cellA. The following operation is used:

CONNECT layer1 layer2
net_out = DFM PROPERTY layer1 [net_num = NETID(layer1 )]

Examples

The NETID(layer, ordinal) form of the function can be used to store a netID property on the source/drain pins of a MOSFET device. 

gate_p = DFM PROPERTY gate sd OVERLAP ABUT ALSO MULTI
           [N1 = NETID(sd, 1)]
           [N2 = NETID(sd, 2)]

The following operation determines if the source/drain pins are shorted. Connectivity is considered only for the cell in which the evaluation takes place and its descendants. If the source/drain pins are shorted the gate shape is written to the output layer. 

gate_sd_short = DFM PROPERTY gate sd OVERLAP ABUT ALSO MULTI
            [SD_NETS_EQ = (NETID(sd, 1) == NETID(sd, 2)) ? 1 : 0] == 1
