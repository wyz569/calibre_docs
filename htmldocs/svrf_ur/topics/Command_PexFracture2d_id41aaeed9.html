PEX Fracture_2D

Specifies to apply 2-D fracturing to a specified marker region when performing point-to-point resistance extraction.

Usage

PEX FRACTURE_2D conductor_layer marker_layer

Arguments

conductor_layer

A required original or derived polygon layer identifying the conductor layer.

marker_layer

A required original or derived polygon layer used to mark a region of interest on the conductor layer.

Description

An optional statement used to apply 2-D fracturing to a specified marker region over a conductor when performing point-to-point resistance extraction. Use this statement to accurately extract point-to-point resistance in a particular region. This statement may be specified more than once.

The conductor_layer must be defined as a conductor layer. Only one marker layer can be used to fracture a conductor. If a PEX Fracture_2D statement is specified for a conductor with multiple markers, a warning is generated and the last specified statement for that conductor is applied.

The same marker_layer can be specified for different conductor layers. The marker_layer should be defined using Layer, LVS DB Layer, and Connect statements. For an example, refer to the Examples section of this statement.

Care should be taken when applying 2D fracturing to a conductor. Incorrect position or size of the marker layer could result in incorrect point-to-point results. This statement should only be specified in R-only extraction flows.

Examples

Consider a conductor layer M1 and a marker layer defined on layer 5.

First define the marker layer in your rule file with the following statements:

LAYER MY_MARKER_LAYER 5
LVS DB LAYER MY_MARKER_LAYER
CONNECT MY_MARKER_LAYER

To perform high accuracy point-to-point resistance only extraction in the conductor region covered by the marker layer, include the following statement in your rule file:

PEX FRACTURE_2D M1 MY_MARKER_LAYER
