LVS Prefer Port Nets

Specification statement

Specifies the precedence of port nets as the surviving nets after circuit transformation.

Usage

LVS PREFER PORT NETS {NO | YES}

Arguments

NO

Keyword that specifies global nets are preferred over nets connected to a cell’s ports. Non-port nets with user names are preferred over port nets without user names. This is the default.

YES

Keyword that specifies port nets are preferred over global nets, and port nets without user names are preferred over non-port nets with user names.

Description

Specifies the precedence of port nets in determining which nets survive circuit transformations. The layout or source circuit design can undergo transformations due to device filtering, logic injection, equivalent node shorting, and so forth. Nets that survive transformation replace nets that do not. This statement influences which nets survive.

This statement applies to both layout and source during LVS netlist comparison or in Calibre PERC netlist analysis and can be specified only once.

The process of determining surviving nets is complex. The effects of this statement are strictly those defined in the Parameters section. No other side-effects are implied.

LVS Prefer Nets may be used to specify certain nets by name that should be preferred in specific scenarios. Such preferences, if specified, take precedence over LVS Prefer Port Nets YES. For example, suppose the following statements are in the rules:

LVS PREFER NETS FILTER "pwr?"
LVS PREFER PORT NETS YES

During device filtering, a non-port net named “pwr_1” would be preferred over a port net named “xyz” as a potential surviving net.

Remember that a subcircuit port and a global net are generally distinct even if they have the same name. For example, consider the following SPICE fragment:

.global abc

.subckt bar a abc b
R1 a abc 100
...
.ends

The global net “abc” is a different net from the port net abc. Which one is referenced by the R1 element statement depends on the LVS Spice Prefer Pins statement. If LVS Spice Prefer Pins YES is specified, the second pin of R1 is connected to the port abc. Otherwise, that pin is connected to the global net abc. But if LVS Spice Override Globals YES is present, then the pin abc represents the overridden global net abc, in which case the port and global nets are no longer distinct, and R1 is connected to the overridden global net regardless of any LVS Spice Prefer Pins statement.

For this statement, an overridden global net is not considered a port net. Hence, LVS Prefer Port Nets YES does not cause such a net to be preferred over an original global net. Both simply are treated as global nets.

Examples

Assume the following rule file statements:

LVS FILTER C z SHORT
LVS PREFER PORT NETS YES

and the following subcircuit, where net y is an internal net:

.subckt cell 1 2
R0 1 y 0.1 
C0 y 2 1e-12 z="hv"
R1 2 3 0.1
...
.ends

After device filtering, nets y and 2 are to be shorted together. Because LVS Prefer Port Nets YES is specified, the surviving net is 2 because it is a port net. The transformed netlist has this:

.SUBCKT cell
RR0 1 2  r=0.1
RR1 2 3  r=0.1
.ENDS
