Net Based Function Summary

Depending on the function, the net based functions access node IDs either from connectivity layers or from derived layers containing properties of type netID. Some functions determine maxima or minima from numeric vectors associated with netID vectors, possibly with summation of tuple values. Net based functions rely on connectivity information being present in the referenced layer. The functions are used in DFM expressions.

The net-based functions are summarized in the following table. See “Property Access Function Summary” for general information on the property access functions ending in PROPERTY and PROPERTY_REF.

Table 1. Connectivity Measurement Function Summary

Function

Summary

GLOBALNETID

Returns the net ID associated with a geometry on the input layer as type netID. The net ID returned is based upon the cell placement and provides a net ID that can be compared to other net IDs in the context of a cell. Unlike the NETID function, GLOBALNETID takes high shorts into account.

NETID

Returns the net ID associated with a geometry on the input layer as type netID. Connectivity is considered for the cell the property expression is evaluated in and that cell’s descendants; connectivity at a higher level in the hierarchy is not considered.

NETNAME

Returns a string containing the net name for the specified net ID.

NETPROPERTY

Reads DFM properties of type netID from objects on the input layer.

NETPROPERTY_REF

References net number properties from a property definition in the current operation.

NETVPROPERTY

Reads DFM properties of type vnetID (vector of net IDs) from objects on the input layer.

NETVPROPERTY_REF

References vector net number properties from a property definition in the current operation.

NETVTUPLE

Creates a vector of net ID values. It is most useful in DFM Property Select Secondary, where it creates one tuple for each cluster that is grouped with the select geometry.

NIPROPERTY

Accesses net-indexed numeric property values from the input layer.

NONETID

Returns the net ID value NONETID.

PRESORTED

Given an input property vector of sorted net ID values, returns a vector that is sorted at the hierarchy level in which the expression evaluation takes place.

PSNETID

Generates string properties corresponding to node IDs on input layer shapes.

VBYNETMAX

Returns the net ID occurring most often in a vector of net IDs. Additionally can determine the maximum numeric value in one vector and return the corresponding net ID from another vector. The third variant computes sums of numeric values in one vector corresponding to net IDs in another vector. The net ID corresponding to the greatest sum is returned.

VBYNETMIN

Returns the net ID occurring least often in a vector of net IDs. Additionally can determine the minimum numeric value in one vector and return the corresponding net ID from another vector. The third variant computes sums of numeric values in one vector corresponding to net IDs in another vector. The net ID corresponding to the least sum is returned.

VNETID

Creates a vector of net ID values.

VNETIDMAX

Determines the tuple index of a maximum value of a specified slice in one vector and uses that index to return a corresponding element (a net ID) in another slice of another vector.

VNETIDMIN

Determines the tuple index of a minimum value of a specified slice in one vector and uses that index to return a corresponding element (a net ID) in another slice of another vector.

VNETSUM

Returns the sum of numeric values in a slice (column) of one vector corresponding to net IDs in a slice of another vector, where the net IDs match a specified net ID. The simpler form of the function is a shortened version of the longer one and assumes slice index values of 0.
