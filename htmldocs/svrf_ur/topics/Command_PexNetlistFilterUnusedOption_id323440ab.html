PEX Netlist Filter Unused Option

Parasitic extraction

Filters out unused device instances during the formatter stage based upon component type in either layout or source.

Usage

PEX NETLIST FILTER UNUSED OPTION option [option …]
[SOURCE | LAYOUT | SOURCE LAYOUT]
[DEVICES component_type  [‘(’component_subtype‘)’]]

Arguments

option

A required, case-insensitive keyword that specifies various rules to follow for the filtering of unused devices. You may specify option any number of times in one statement, separated by spaces. The list of keywords is as follows:

AG

Filters MOS devices with all pins tied together, including bulk and optional pins.

RC

Filters resistors with POS and NEG pins tied together.

RE

Filters capacitors with POS and NEG pins tied together.

RG

Filters diodes with POS and NEG pins tied together.
SOURCE

An optional keyword that specifies the statement applies to the source design. This option is used by default if neither SOURCE or LAYOUT keywords are specified.

LAYOUT

An optional keyword that specifies the statement applies to the layout design. This option is used by default if neither SOURCE or LAYOUT keywords are specified.

DEVICES component_type [‘(’component_subtype‘)’]

Optional keyword set that specifies that the option filtering is performed only on the component_type and component_subtype. The filter option must be compatible with the component_type and component_subtype. At least one component_type is required when DEVICES is specified. The component_type corresponds with a device element name and the component_subtype with a model name. The component_type and component_subtype parameters may be specified more than once. The parentheses are required as shown.

Description

Filters out unused device instances during the formatter stage based upon component type in either layout or source. This statement is typically used to filter SPICE device elements. This statement is compatible with all netlist formats. The filtering applies only during netlist formatting.

This statement can be specified any number of times and options can be repeated. Options and DEVICES specifications are accumulated from all statements. Specifying an option without any devices filters all relevant devices and makes any other statements with the same option (with or without devices irrelevant. There is no default.

If only SOURCE is specified, then devices meeting the option criteria in the source design are filtered, but devices in the layout design are not. If only LAYOUT is specified, devices meeting the option criteria in the layout design are filtered, but devices in the source design are not. When neither LAYOUT or SOURCE are specified or if both LAYOUT and SOURCE are specified, devices meeting the option criteria in both the source and layout designs are filtered.

For device filtering to be performed on devices within hierarchical cells, the pins must be tied together at the local level. Even if the pins are tied together externally from the cell, the device are not filtered.

Examples

Example 1

Given a source spice netlist with the following devices: 

.SUBCKT alpha drain gate source bulk
MM9 drain gate source bulk nch l=200n w=4u
ENDS

.SUBCKT et_n45_testcase6 A1 AVSS B1 C1
*.PININFO A1:I AVSS:I B1:I C1:I
MMSP1 AVSS AVSS AVSS AVSS nch l=200n w=4u m=25
MM3 AVSS AVSS AVSS AVSS nch l=200n w=4u m=13
MM7 C1 A1 AVSS AVSS nch l=200n w=4u m=68
MM4 A1 A1 AVSS AVSS nch l=200n w=4u m=68
MM2 AVSS AVSS AVSS AVSS nch l=200n w=4u m=13
MM5 B1 A1 AVSS AVSS nch l=200n w=4u m=8
MM6 B1 A1 AVSS AVSS nch l=200n w=4u m=8
MM8 C1 A1 AVSS AVSS nch l=200n w=4u m=68

xalpha AVSS AVSS AVSS AVSS alpha m=13
* xalpha replaces MM9 with an m-factored instance containing a single MM9
*MM9 AVSS AVSS AVSS AVSS nch l=200n w=4u m=13

MM10 AVSS AVSS AVSS AVSS nch l=200n w=4u m=17
MM11 AVSS AVSS AVSS AVSS nch l=200n w=4u m=17
R1 AVSS AVSS 1.0 RX1
R2 AVSS AVSS 1.5 RX2
C2 AVSS AVSS 2.0
D3 AVSS AVSS diode

.ENDS

Note that all the pins of MOSFETs MMSP1, MM2, MM3, MM10, and MM11 are tied together to net AVSS. The pins for MOSFET MM9 within subckt “alpha” are tied together externally to the subckt. There are also two resistors of different types, a capacitor, and a diode with their pins tied together as well.

A source-based netlist without any device filtering looks similar to the following:

.subckt et_n45_testcase6  A1 AVSS B1 C1
*
* A1 A1
* C1 C1
* B1 B1
* AVSS  AVSS
MMSP1 N_AVSS_MMSP1_d N_AVSS_MMSP1_g N_AVSS_MMSP1_s N_AVSS_xalpha/MM9_b nch
    + L=2e-07 W=4e-06 M=25
MM3 N_AVSS_MM3_d N_AVSS_MM3_g N_AVSS_MM3_s N_AVSS_xalpha/MM9_b nch L=2e-07
    + W=4e-06 M=13
MM7 N_C1_MM7_d N_A1_MM7_g N_AVSS_MM7_s N_AVSS_xalpha/MM9_b nch L=2e-07 
    + W=4e-06 M=68
MM4 N_A1_MM4_d N_A1_MM4_g N_AVSS_MM4_s N_AVSS_xalpha/MM9_b nch L=2e-07 
    + W=4e-06 M=68
MM2 N_AVSS_MM2_d N_AVSS_MM2_g N_AVSS_MM2_s N_AVSS_xalpha/MM9_b nch L=2e-07
    + W=4e-06 M=13
MM5 N_B1_MM5_d N_A1_MM5_g N_AVSS_MM5_s N_AVSS_xalpha/MM9_b nch L=2e-07 
    + W=4e-06 M=8
MM6 N_B1_MM6_d N_A1_MM6_g N_AVSS_MM6_s N_AVSS_xalpha/MM9_b nch L=2e-07 
    + W=4e-06 M=8
MM8 N_C1_MM8_d N_A1_MM8_g N_AVSS_MM8_s N_AVSS_xalpha/MM9_b nch L=2e-07 
    + W=4e-06 M=68
mxalpha/MM9 N_AVSS_xalpha/MM9_d N_AVSS_xalpha/MM9_g N_AVSS_xalpha/MM9_s
    + N_AVSS_xalpha/MM9_b nch L=2e-07 W=4e-06 M=13
MM10 N_AVSS_MM10_d N_AVSS_MM10_g N_AVSS_MM10_s N_AVSS_xalpha/MM9_b nch 
    + L=2e-07 W=4e-06 M=17
MM11 N_AVSS_MM11_d N_AVSS_MM11_g N_AVSS_MM11_s N_AVSS_xalpha/MM9_b nch 
    + L=2e-07 W=4e-06 M=17
R1 AVSS AVSS RX1 1
R2 AVSS AVSS RX2 1.5
C2 AVSS AVSS 2
D3 AVSS AVSS diode
.ends

Include the following statements in the rule file to remove the devices whose pins are shorted:

PEX NETLIST FILTER UNUSED OPTIONS RC SOURCE LAYOUT DEVICES R(RX1)
PEX NETLIST FILTER UNUSED OPTIONS AG RE RG SOURCE LAYOUT

The first statement filters all resistors of type “RX1” with their pins shorted. The second statement filters all types of MOSFETs, capacitors, and diodes whose pins are shorted.

Generating a source-based netlist with filtering enabled results in the following:

.subckt et_n45_testcase6  A1 AVSS B1 C1
*
* A1 A1
* C1 C1
* B1 B1
* AVSS  AVSS
MM7 N_C1_MM7_d N_A1_MM7_g N_AVSS_MM7_s N_AVSS_MM6_b nch L=2e-07 W=4e-06 
	+ M=68
MM4 N_A1_MM4_d N_A1_MM4_g N_AVSS_MM4_s N_AVSS_MM6_b nch L=2e-07 W=4e-06 
	+ M=68
MM5 N_B1_MM5_d N_A1_MM5_g N_AVSS_MM5_s N_AVSS_MM6_b nch L=2e-07 W=4e-06 
	+ M=8
MM6 N_B1_MM6_d N_A1_MM6_g N_AVSS_MM6_s N_AVSS_MM6_b nch L=2e-07 W=4e-06 
	+ M=8
MM8 N_C1_MM8_d N_A1_MM8_g N_AVSS_MM8_s N_AVSS_MM6_b nch L=2e-07 W=4e-06 
	+ M=68
mxalpha/MM9 AVSS AVSS AVSS AVSS nch L=2e-07 W=4e-06 M=13
R2 AVSS AVSS RX2 1.5
.ends

The MOSFETs MMSP1, MM2, MM3, MM10, and MM11 are filtered from the netlist because their pins were shorted and the AG option was specified. The MOSFET mxalpha/MM9 remains unfiltered because a device within a subckt is not filtered if its pins are shorted externally to the subckt. The remaining MOSFETs are not filtered because all of their pins are not shorted.

The capacitor and diode are filtered because their pins are shorted, and because the RE and RG options were specified.

Only one of the resistors is filtered. This is because the RC option is specified with the DEVICE keyword, restricting the filtering to the RX1 subtype. Resistors of type RX2 are not filtered from the netlist.
