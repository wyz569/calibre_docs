tvf::VERBATIM

Application: compile-time only.

Passes in verbatim SVRF code in a compile-time TVF script. The contents of the VERBATIM code block are passed directly to the SVRF compiler, without going through the Tcl interpreter.

Usage

tvf::VERBATIM ‘{’
verbatim_SVRF_code
‘}’

Arguments

verbatim_SVRF_ code

A required argument that specifies a block of SVRF code. The code block is enclosed in braces as shown.

Return Values

None.

Description

Whatever appears between the braces is passed directly to the SVRF compiler and must adhere to SVRF syntactical conventions. Any errors in the SVRF syntax generate a Calibre compiler error, as usual. In addition, unmatched braces ( {} ) anywhere within the VERBATIM code block cause a compiler error, since TVF is processed by the Tcl interpreter. In particular, unmatched braces within strings or SVRF-style comments in a VERBATIM block cause a compiler error.

You should use this command as much as possible in your compile-time TVF scripts to pass blocks of SVRF code that do not need to be changed to TVF syntax. This includes runtime TVF commands within a compile-time TVF script.

Examples

The following example shows how a typical SVRF rule check can be placed in a compile-time TVF script using the VERBATIM command:

tvf::VERBATIM {
   //Minimum m1 width is 0.10
   m1_width {
   INT m1 < .10 ABUT < 90 SINGULAR
   }
}
