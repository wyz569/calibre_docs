DFM Spec Contour

Specification statement

License: Calibre YieldEnhancer. Substitute: Calibre YieldAnalyzer

Defines a contour specification that is used by DFM Contour to generate a contour map of density values. Contour maps of the values of a DFM property can also be generated.

Usage

Mode 1: Polygon Layer Input

DFM SPEC CONTOUR “spec_name” layer [layer …]  
{OUTPUT “output_name” constraint [GRADIENT]} …
[INSIDE OF LAYER layer_region]      
[PIX RES {AUTO | step}]  
[APERTURE {AUTO | half_width}]      
[HALO density_value]                            

Mode 2: DFM Property Layer Input

DFM SPEC CONTOUR “spec_name” PROPERTY prop_layer “prop_name”
{OUTPUT “output_name” constraint} …                  

Arguments

“spec_name”

A required string parameter giving the name of the contour specification. The name is used in DFM Contour.

layer

One or more required polygon layers. The combined density of these layers is calculated in order to determine the contour map. Used with the polygon layer input mode.

The density is calculated as the ratio of the area of the input layer(s) in a data capture window to the area of the data capture window itself. For one layer input, the density calculation returns values between 0 and 1.

PROPERTY prop_layer “prop_name”

A required keyword set used in the property layer input mode.

  • prop_layer — A layer with attached DFM properties containing the value used to generate the contour map. The prop_layer should be the result of a DFM Analyze or DFM Density operation using equal STEP sizes. In addition, DFM Analyze must use the CENTERS keyword.

  • prop_name — The name of the DFM property on prop_layer. The property name is DV for both DFM Analyze and DFM Density derived layers.

See “Mode 2: Property Layer Input” for more information.

OUTPUT “output_name” constraint [GRADIENT]

A required keyword set providing the output name and constraint for the contour output. The keyword set can be repeated multiple times with different constraints in order to generate multiple contour outputs.

  • output_name — A required string giving the name of the output. The output name is used by DFM Contour.

  • constraint — A required constraint. The constraint < is treated as <=, and > is treated as >=.

  • GRADIENT — An optional keyword that specifies to contour the density gradient. The magnitude of the 2D vector gradient is contoured. The gradient is calculated using a Fast Fourier Transform (FFT). GRADIENT is only supported in Mode 1.

INSIDE OF LAYER layer_region

An optional keyword set that specifies to perform the analysis inside the extents of the polygons on layer_region. Used with the polygon layer input mode.

By default, the analysis is performed inside the extent of the input layout.

PIX RES {AUTO | step}

An optional keyword set that defines the resolution of the pixel grid, which is an array of points on a square grid. The density is measured at each point on the pixel grid. Used with the polygon layer input mode.

  • AUTO — Use the default pixel separation of 1.0 um. AUTO is used if PIX RES is not specified.

  • step — The pixel separation in microns. The step should be a positive number less than or equal to 1000000.

APERTURE {AUTO | half_width}

An optional keyword set that defines the size of the measurement region around each pixel. Used with the polygon layer input mode.

  • AUTO — Use the default half_width of 10 um, which creates a 20 um square measurement region centered on each pixel. AUTO is used if APERTURE is not specified.

  • half_width — The measurement region is a square of width 2*half_width um centered on each pixel. For best results, the half_width should be larger than the typical polygon width and spacing.

    The half_width should be a positive number less than or equal to 1000000, and 2*half_width should be an integer multiple of step.

HALO density_value

An optional keyword set that causes contouring to extend to the boundaries of the analysis extent, rather than stop within half_width of the extent. Used with the polygon layer input mode.

Internally, the analyzed extent is expanded by a “halo layer” with a density equal to density_value, which must be a non-negative real number. When analyzing a full chip, density_value should typically be equal to zero. See “Mode 1: Polygon Layer Input” for more details, especially concerning usage with INSIDE OF LAYER.

Description

Defines a contour specification used by DFM Contour to generate a contour map of a property values. Linear interpolation is used between data points to generate the contour map. The contour map can be used to visualize the property across the whole design without the need to calculate the property value at a high resolution.

There are two modes of operation:

Table 1. DFM Spec Contour Modes of Operation

Mode 1: Polygon Layer Input

Mode 2: DFM Property Layer Input

The density of the input layer(s) is mapped. The density is measured at each point on the pixel grid, and linear interpolation is used to determine the density between grid points.

The pixel separation and measurement window are determined by the PIX RES and APERTURE keyword values.

The DFM property on the input layer is mapped. Each shape on the input property layer corresponds to a point on the pixel grid. The property value is read at each point on the pixel grid, and linear interpolation is used to determine the value between grid points.

The pixel separation and measurement window are determined by the DFM Analyze or DFM Density operation that generates the input property layer.

The OUTPUT specifications determine the density or property value range for each contour. The following code uses polygon layer input (Mode 1) to map the metal1 density. Three contours are generated. Figure 1 shows the metal1 layer in a design, and the density map.

DFM SPEC CONTOUR "specDef" metal1
   OUTPUT "high" > 0.50 
   OUTPUT "med"  > 0.25 < 0.50 
   OUTPUT "low"  < 0.25 
   // default PIX RES 1 and APERTURE 10
  
outDef_high = DFM CONTOUR "specDef" high
outDef_med = DFM CONTOUR "specDef" med
outDef_low = DFM CONTOUR "specDef" low
Figure 1. DFM Contour Default Density Contour

More details about the different modes are provided in the following sections.

Mode 1: Polygon Layer Input

The density of the input layers is mapped in this mode. The density is calculated as the ratio of the area of the input layer(s) in a data capture window to the area of the data capture window itself. For single layer input, the default density calculation returns values between 0 and 1.

The DFM Density operation is used internally to calculate the density, using the default density calculation. The following table shows the correspondence of DFM Spec Contour and DFM Density keywords when using polygon layer input:

DFM Spec Contour Keyword

DFM Density Keyword

INSIDE OF LAYER layer_region

INSIDE OF LAYER layer_region

PIX RES step

STEP step

APERTURE half_width

WINDOW 2*half_width

By default the contour map is generated inside the extent of the input layout. This can be changed with the INSIDE OF LAYER keyword.

The PIX RES and APERTURE settings are used to control resolution of the contour map. The effect of different settings in shown in the following figure.

Figure 2. DFM Contour PIX RES and APERTURE

Without the HALO keyword, the contour map does not cover the full extent of the design, as shown in the leftmost image of the following figure. This is because the pixel grid stops half_width inside the extent, and interpolation of density values is only done between pixel grid points. When “HALO density_value” is specified, the analyzed extent is expanded by an internally-generated halo layer around the design extent. The halo layer has a density of density_value. This enables the pixel grid to cover the full design extent.

Figure 3. DFM Contour With HALO

When using HALO, the density_value should typically be zero when analyzing a full chip. When used with INSIDE OF LAYER to analyze a portion of the design, you may want to set the density_value at a reasonable estimate of the density surrounding the region being analyzed.

When HALO is used with INSIDE OF LAYER, the internal halo layer is created around each polygon extent. In some situations this can cause merging of the polygon extents, which affects the pixel grid.

Mode 2: Property Layer Input

The property layer input mode is useful when you want to map the value of a user-defined property. The property layer must be created by a DFM Analyze or DFM Density operation using the following keywords:

DFM Analyze or DFM Density Keyword

Description

[expression [>=0]]

The property value to map. The expression should evaluate to a non-negative number. If expression is not provided, the default expression for the operation is used. See the DFM Analyze and DFM Density documentation for details on the functions supported in the expression.

The constraint “>=0” is the default for DFM Analyze and is required in DFM Density. DFM Contour requires uniform spacing in the polygons on the input property layer, so only a constraint of “>=0” should be used.

STEP step

Points on the pixel grid are separated by step.

WINDOW width

The expression is evaluated in a square window of width width centered on each point on the pixel grid.

CENTERS

Required when using DFM Analyze. This keyword causes the operation to output marker polygons with the value of expression attached as a DFM property.

(CENTERS is the default behavior in DFM Density.)

Linear interpolation is used to estimate the property value between points on the pixel grid. Because interpolation is done only between pixel points, regions within 0.5*width of the data capture extent are not contoured. The data capture extent can be adjusted with the INSIDE OF keyword for DFM Density or DFM Analyze.

Examples

For an example with default parameters, see Figure 1 and the code preceding the figure.

Example 1: Polygon Layer Input

This example specifies PIX RES and APERTURE. 

DFM SPEC CONTOUR "specA" metal1
   OUTPUT "high" > 0.50 
   OUTPUT "med"  > 0.25 < 0.50 
   OUTPUT "low"  < 0.25 
   PIX RES 1 
   APERTURE 5
  
outA_high = DFM CONTOUR "specA" high
outA_med = DFM CONTOUR "specA" med
outA_low = DFM CONTOUR "specA" low

Example 2: Property Layer Input From DFM Analyze

This example uses DFM Analyze to calculate the distribution of metal1 shapes as the product of edge length and area. DFM Contour is used to generate a contour map. 

chip = EXTENT DRAWN metal1
metal1_e = metal1 INSIDE EDGE chip

dfmdist = DFM Analyze metal1 metal1_e 
      [LENGTH(metal1_e) * AREA(metal1)] >= 0
      WINDOW 20 STEP 1 CENTERS .1
DFM SPEC CONTOUR "specdistA" PROPERTY dfmdist "DV"
   OUTPUT "out50"    > 50    < 5000
   OUTPUT "out5000"  > 5000  < 10000
   OUTPUT "out10000" > 10000 < 50000
   OUTPUT "out30000" > 30000 < 500000
   OUTPUT "out50000" > 50000 < 100000
   OUTPUT "out100000" > 100000
out_distA_50   =   DFM CONTOUR "specdistA" out50
out_distA_5000 =   DFM CONTOUR "specdistA" out5000
out_distA_10000 =  DFM CONTOUR "specdistA" out10000
out_distA_30000 =  DFM CONTOUR "specdistA" out30000
out_distA_50000 =  DFM CONTOUR "specdistA" out50000
out_distA_100000 = DFM CONTOUR "specdistA" out100000
out_oas {
  DFM RDB OASIS out.oas out_distA_50     31
  DFM RDB OASIS out.oas out_distA_5000   32
  DFM RDB OASIS out.oas out_distA_10000  33
  DFM RDB OASIS out.oas out_distA_30000  34
  DFM RDB OASIS out.oas out_distA_50000  35
  DFM RDB OASIS out.oas out_distA_100000 36
}

Example 3: Property Layer Input From DFM Density

This example produces the same contour map as Example 1, but uses DFM Density to generate a property layer. The WINDOW value of 10 corresponds to an APERTURE of 5. 

dfmdens3 = DFM Density metal1  
               [AREA(metal1)/AREA()] >= 0
               WINDOW 10 STEP 1 

DFM SPEC CONTOUR "spec_dens3" PROPERTY dfmdens3 "DV"
               OUTPUT "high" > 0.5
               OUTPUT "med"  > 0.25 < 0.5
               OUTPUT "low"  < 0.25

dens3_high = DFM CONTOUR "spec_dens3" high
dens3_med =  DFM CONTOUR "spec_dens3" med
dens3_low =  DFM CONTOUR "spec_dens3" low
