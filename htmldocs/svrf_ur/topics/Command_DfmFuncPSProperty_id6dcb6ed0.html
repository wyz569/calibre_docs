PSPROPERTY

Property access function for DFM expressions

Accesses a string type property from an object on the input layer. The returned string is prefixed by Xn/Xm/.../ to indicate placement hierarchy. Supported only for DFM Property and DFM Function.

Usage

PSPROPERTY(layer, property_name[, ordinal])

Arguments

layer

A required layer containing the layout objects to which the property is attached.

If layer is a secondary layer in the DFM Property operation and ordinal is not specified, the PSPROPERTY function must be within the VSTRING function; the string properties on each secondary layer object clustered with the primary layer object are concatenated and a vector of strings is returned.

property_name

A required argument defining the name of the property of interest, which must be a string type property.

ordinal

An optional argument used when layer is a secondary layer in a multilayer DFM Property operation. The ordinal specifies the index of the secondary layer geometry whose property value is to be returned. It must be a positive integer value and can itself be an expression. See the “Description” section for details on behavior and restrictions.

Return Values

String or vector of strings.

Description

Returns a string type property prefixed by Xn/Xm/.../, where n, m, and so on are placement numbers of the placement hierarchy through which the given geometry was promoted.

String type properties can be created using DFM Property, DFM Text, Calibre YieldServer, or a Device Layer operation. The PSPROPERTY function can be used within the VSTRING function.

For example, suppose that a layer with string properties is created by the Device Layer operation. The polygon corresponding to the first MOSFET device in a cell has a string property with name ICV_DEVICE_NAME and value M0. A DFM Property operation with this layer as one of its input layers may promote this polygon through the hierarchy in order to cluster it with overlapping polygons on other input layers. The SPROPERTY function for this polygon always returns M0. The PSPROPERTY function returns different values for each promoted instance of this polygon. For example, if a particular instance came from the cell placement X3, the property value returned for that instance is X3/M0.

When string properties are written out by a DFM RDB operation, original property values are written even if the geometries are promoted (for example, if NOPSEUDO is specified).

Restrictions

The following restrictions apply to the DFM Property operation in which the PSPROPERTY function is used:

  • The INSIDE OF, BY CELL, and BY NET keywords may not be used.

  • For multilayer DFM Property operations, if the PSPROPERTY function references a secondary layer, the OVERLAP or NODAL keyword must be specified; in other words, the default clustering method of INTERSECTING is not allowed when reading properties from a secondary layer.

Behavior with Ordinal

The ordinal argument is used to read a property value from a specific object on a secondary layer in a multilayer DFM Property operation. The clustering method must be OVERLAP or NODAL.

Layout objects on secondary layers are numbered beginning with 1. The order in which the layout objects are processed is arbitrary, however, each secondary layer object is assigned a unique ordinal which is consistent within the same DFM Property operation.

  • For ordinal = 1 through the total number of clustered geometries on the secondary layer, the function call returns the value of the specified property on the specified secondary layer geometry clustered with the primary layer geometry.

  • Multiple function calls in the same DFM Property operation that reference the same layer have a consistent order for the secondary layer objects. For example, PROPERTY(S, “X”, 1) and VPROPERTY(S, “Y”, 1), when used in the same DFM Property operation, return property values from the same geometry.

  • If the ordinal value is out of range (either greater than the number of clustered geometries or less than 1), the function fails to evaluate.
