DFM Property Merge

Layer operation

License: Calibre nmDRC-H

Manipulates unmerged polygon layers. Unmerged polygon layers can be created with DFM Text, DFM Copy REGION UNMERGED, and other DFM layer operations.

Usage

DFM PROPERTY MERGE primary_layer [secondary_layer …]
[ABUT ALSO] [MULTI | NOMULTI] [EXACT]
['[' property_name = expression ']' [['!'] constraint ] …]

Arguments

primary_layer

A required argument that specifies a polygon layer. An implicit merged layer is created from the primary_layer, and all clustering of geometries from all input layers occurs with respect to that implicit merged layer. The output contains the geometry of this implicit merged layer along with any associated properties.

If this layer is merged, then an unmerged secondary_layer must be specified.

secondary_layer

An optional argument that specifies an additional polygon layer (possibly unmerged) for consideration when calculating and assigning properties to layout objects on the implicit merged layer created from the primary_layer. Any layer other than the primary_layer that is referenced in any expression for the operation must be specified as a secondary_layer. Any number of secondary_layer arguments is allowed.

ABUT ALSO

An optional keyword that instructs the operation to include secondary_layer layout objects that abut a primary_layer layout object in addition to those that overlap it.

MULTI | NOMULTI

An optional keyword choice that controls how the operation treats an input layout object that overlaps or abuts more than one object on the implicit merged layer. This is called the clustering mode. By default, when an input polygon can be grouped into several clusters, the operation assigns the layout object to one of these clusters arbitrarily and a warning is issued in the transcript. Specifying either MULTI or NOMULTI overrides the default behavior as follows:

  • MULTI — The operation assigns an input layout object to the cluster for each implicit merged layer polygon it overlaps or abuts.

  • NOMULTI — The operation acts like the default mode as to clustering behavior but no warning is issued.

EXACT

An optional keyword that causes only identical geometries to be clustered together. Identical geometries must be pointwise equivalent—edges must match, have the same orientation, and be in the same location.

All input layers must be the same type if EXACT is specified. If the output layer from a DFM Property Singleton operation is used as a secondary_layer, the primary_layer must be a polygon layer.

'[' property_name = expression ']' [['!'] constraint ] …

An optional argument set that defines the properties to be associated with geometries on the implicit merged layer. This argument set follows the identical format and semantics as DFM Property expressions. If this argument set is omitted, then the output has no properties.

Description

Used for manipulating unmerged polygon layers. At least one unmerged layer must be an input to the operation. Unmerged polygon layers can be created with certain DFM layer operations and some other operations; see “Unmerged Polygon Layers” in the Calibre Verification User’s Manual.

The operation begins by creating an implicit merged layer from the primary_layer. (If the primary_layer is already merged, the implicit merged layer is essentially that layer.) The operation then creates associations between the layout objects on all input layers and those on the implicit merged layer by grouping them together into clusters—refer to “Clustering Methods in DFM Property”. Each cluster contains exactly one layout object on the implicit merged layer and any number of input layout objects as controlled by the clustering mode (default, MULTI, or NOMULTI).

When an unmerged primary_layer is referenced in a function in the property expression, it is treated as if it were a secondary layer consisting of the unmerged shapes that make up the merged output geometry. By default, the function values are summed over the unmerged shapes, as with secondary layers in a DFM Property operation; see “Primary and Secondary Layers and Measurement Function Evaluation”. If a function requires an ordinal argument when used with a secondary layer (such as SPROPERTY), the ordinal should be provided; 1 is typically used. See Example 1.

The output layer of DFM Property Merge contains geometries from the implicit merged layer. Properties on the output layer are computed from the input geometries using the specified property expressions. Clustering affects which properties are assigned to output polygons in a similar way to DFM Property. Therefore, conceptually, the following operation:

DFM PROPERTY MERGE layer [expressions_referring_to_layer]

is equivalent to the following DFM Property operation:

DFM PROPERTY merged_layer layer OVERLAP [expressions_referring_to_layer]

Note that this equivalence is for illustration purposes only — the DFM Property operation cannot be used to transfer properties from an unmerged layer.

Examples

Example 1

The following DFM Property Merge operation merges polygons on the layer p_um and constructs new layer, p_m, which contains four properties. For each polygon on layer p_m, consider the values of property “X” on the unmerged polygons which are combined into this polygon. Three of the output properties contain the smallest, the largest, and the total value of these values of the property “X”. The output property STR contains a string value read from the first unmerged shape. 

p_m = DFM PROPERTY MERGE p_um 
  [ XMIN = MIN( PROPERTY(p_um, "X") ) ]
  [ XMAX = MAX( PROPERTY(p_um, "X") ) ]
  [ XTOT = PROPERTY(p_um, "X") ]
  [ STR  = SPROPERTY(p_um, "S", 1) ]

Example 2

In the following example, a two-layer DFM Property Merge operation is used to attach polygon properties that are read from the layout to the output polygons as DFM properties.

LAYER metal1 1
prop1 = DFM READ metal1 PROPERTY NUMBER x
metal1_prop = DFM PROPERTY MERGE metal1 prop1
                [ v = VECTOR(PROPERTY(prop1,x)) ]
