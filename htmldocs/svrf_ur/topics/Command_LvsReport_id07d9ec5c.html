LVS Report

Specification statement

Specifies the LVS report pathname. This statement is required in LVS comparison. Used by Calibre PERC for writing a comparison report in the LDL DRC source-based flow.

Usage

LVS REPORT filename

Arguments

filename

A required filename of the LVS report.

The filename parameter can contain environment variables. See “Environment Variables in Pathname Parameters”.

Description

Specifies the LVS report filename. This statement must be specified once in your rule file for an LVS comparison run. The contents of the report are discussed under “LVS Report” in the Calibre Verification User’s Manual.

When neither the -spice command line option nor the Mask SVDB Directory statement is used during circuit extraction, then the extracted layout netlist is written to filename.sp in the current working directory.

When circuit extraction is run, the circuit extraction report is written to filename.ext if LVS Report is specified; otherwise, the default extraction report name is lvs.rep.ext. In Calibre PERC LDL runs, the circuit extraction report is only written in the former case.

LVS Isolate Shorts YES produces the short isolation results filename filename.shorts if LVS Results is specified. Otherwise, the default filename for short isolation results is lvs.layout_cell.rep.shorts.

You can specify the LVS Report in Calibre Interactive—nmLVS. See “Setting Up a Calibre Interactive nmLVS Run” in the Calibre Interactive (New GUI) User’s Manual.

In Pyxis Layout, this statement sets the value of application variable lvs_default_report_name.

See also LVS Report Maximum, LVS Report Option, and LVS Summary Report.

Examples

// Specify LVS Report filename.
LVS REPORT "lvs.report"
