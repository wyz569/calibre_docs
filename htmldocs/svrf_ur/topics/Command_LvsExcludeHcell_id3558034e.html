LVS Exclude Hcell

Specification statement

Specifies to ignore specified hcells in all circumstances. Used only in Calibre nmLVS-H and Calibre PERC. Observed by Calibre Query Server.

Usage

LVS EXCLUDE HCELL cell_name [cell_name …]

Arguments

cell_name

A required cell name that instructs LVS-H, Calibre PERC, and the Query Server to prevent that cell from serving as an hcell in either layout or source. A list of cell names can be used. A cell parameter may contain one or more asterisk (*) characters; the * character is a wildcard that matches zero or more characters. A string that contains a * character must be enclosed in quotation marks because the * is a reserved symbol.

Description

This statement prevents the specified cells from serving as hcells. Any cell specified in an LVS Exclude Hcell statement does not serve as an hcell under any circumstances for the tools that observe this statement.

See “Calibre nmLVS and Calibre nmLVS-H Command Line” in the Calibre Verification User’s Manual for a discussion of Calibre nmLVS command line options ‑automatch, -hcell, and ‑genhcells. All of these options can have their hcells overridden by LVS Exclude Hcell.

Hcell names excluded by this statement are never used in Query Server hcell analysis commands. See “Hcell Analysis Commands” in the Calibre Query Server Manual.

See also Hcell, LVS Auto Expand Hcells, LVS Expand On Error, LVS Box, LVS Filter, LVS Hcell Report, and LVS Spice Exclude Cell.

Examples

Example 1

This example prevents cell “aaa” from serving as an hcell in the layout and source. 

LVS EXCLUDE HCELL "aaa"

Hcell pairs containing aaa are excluded. For example, all of these hcell pairs are excluded:

aaa aaa
aaa bbb
ccc aaa

Example 2

This example prevents all cells whose names start with the characters “bar” from serving as hcells in layout and source, along with cells that correspond to such names in the hcell list:

LVS EXCLUDE HCELL "*bar*"
