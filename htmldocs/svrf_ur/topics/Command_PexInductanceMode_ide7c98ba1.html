PEX Inductance Mode

Parasitic extraction

Specifies the inductance extraction mode used by Calibre xL and the Calibre xACT inductance extraction flow.

Usage

PEX INDUCTANCE MODE {PEEC | LOOP}

Arguments

PEEC

A required keyword that specifies the extraction of partial self and mutual inductances. This keyword can only be used with select net extraction.

LOOP

A required keyword that specifies the extraction of loop self and mutual inductances for automatically identified signal-return path current loops in the layout. This is the default behavior if this statement is not specified.

Description

An optional statement used to set the inductance extraction mode used by the Calibre xL or the Calibre xACT inductance flow. By default, these tools run in loop inductance extraction mode if this statement is not specified.

When the LOOP keyword is specified, a loop inductance extraction is performed, which is consistent with the physical principle of associating inductance as a property of current-carrying loops. These loops are identified among the interconnects and formed by signal nets and nets specified with the PEX Inductance Returnpath statement, which are typically power or ground nets. RLK parasitics are extracted for the complete signal-return path loops. The calculated loop RLK values will then only appear under the corresponding signal net parasitics. There will be no RLK netlisted for the return paths. Use this extraction mode for a full-chip RCCLK extraction when there is a concern that the layout may contain nets with sufficiently long paths that have significant parasitic inductance, such as nets in a clock distribution network.

Setting the inductance mode to PEEC results in partial RLK extraction which is based on the Partial Element Equivalent Circuit method. The PEEC mode can only be applied on a set of selected nets in the layout. If the mode is PEEC and nets have not been selected for inductance extraction, and the Calibre xACT or Calibre xRC command line is run with a -r(cc)l switch, extraction stops with an error:

PEX INDUCTANCE MODE is set to PEEC. Please specify nets for inductance extraction using the PEX XACT CONTROL or PEX EXTRACT INCLUDE statements.

Partial inductance is applied on all the nets specified through the net selection mechanisms of PEX XACT Control or PEX Extract Include. The power and ground nets for which the partial inductances should be extracted should explicitly be included in the net selection.

In most cases, net selection is done with PEX Extract Include statement; For more information on select net extraction, see “Extracting Particular Nets with the Calibre xACT 3D PDB flow” or “Extracting Particular Nets”. In the Calibre xACT inductance flow, to select a particular set of nets among all the extracted nets that will undergo inductance extraction, use the PEX XACT Control statement.

Unlike LOOP mode, in PEEC mode the mutual inductance is extracted and netlisted even if it has not explicitly been specified. You cannot select between self inductance extraction only (-RL), and self and mutual inductance extraction (-RLM). All significant mutual inductance is extracted by construction in order to assure a physically sound solution.

Examples

Example 1: Using Calibre xRC and Calibre xL

Consider that the frequency of extraction for parasitic inductances is 5 GHz, and that you want power net VDD and ground net VSS to be used as return current paths for the inductance calculations.

To perform full-chip RCC extraction and also extract the loop self-inductance of all signal nets exceeding 50 microns between a driver-receiver pair, include the following statements in your rule file:

PEX INDUCTANCE MINLENGTH 50
PEX INDUCTANCE FREQUENCY 5e9
PEX POWER VDD
PEX GROUND VSS
PEX INDUCTANCE RETURNPATH GANDP
PEX INDUCTANCE MODE LOOP // statement specification is optional for LOOP

And invoke PDB extraction as follows:

calibre -xrc -pdb -rccl rules

LOOP inductance extraction mode runs whether or not PEX Inductance Mode statement is specified. Calibre xL identifies all the nets in the layout with path lengths that exceed 50 microns, and extracts the frequency-dependent loop-RL parasitics for these nets using VDD and VSS as current-return paths.

Example 2: Using Calibre xRC and Calibre xL

Consider a layout that contains signal nets S1 and S2, and a ground net GND. To perform a full-chip RCC extraction and an incremental, selective partial inductance extraction for these nets, include the following statements in your rule file:

PEX INDUCTANCE MODE PEEC
PEX EXTRACT INCLUDE LAYOUTNAMES “S1” “S2” “GND”

Extract full-chip RCC with Calibre xRC:

calibre -xrc -pdb -rcc -turbo rules

Add the selected net partial RLK parasitics to the PDB with the following command line:

calibre -xrc -pdb -rl -select rules

Generate the extracted netlist:

calibre -xrc -fmt -all rules

Example 3: Using Calibre xRC and Calibre xL

To perform the incremental extraction described in Example 2, but with loop self and mutual induction extraction and netlisting, include the following statements in your rule file:

PEX GROUND GND
PEX INDUCTANCE RETURNPATH GROUND
PEX INDUCTANCE VICTIM S1
PEX EXTRACT INCLUDE LAYOUTNAMES "S1" "S2" "GND"
PEX INDUCTANCE MODE LOOP

Specifying nets with the PEX Inductance Victim statement is only required for the LOOP inductance mode when you want to also extract mutual inductance for these nets.

Extract full-chip RCC with Calibre xRC:

calibre -xrc -pdb -rcc -turbo rules

Add the selected net loop RLK parasitics to the PDB:

calibre -xrc -pdb -rlm -select rules

Generate the extracted netlist:

calibre -xrc -fmt -all rules

LOOP inductance extraction mode runs whether or not PEX Inductance Mode statement is specified. A graphical representation of the resulting distributed parasitic netlist for nets S1, S2, and GND is show in Figure 2. For LOOP mode, the extracted RLK parasitics are netlisted only for the signal nets. The contribution of the return current net, GND, is already incorporated in the loop RLK parasitic values and net GND does not appear in the extracted netlist.

Figure 2. Parasitic Netlist Schematic Using LOOP Mode

Example 4: Using Calibre xACT Direct Netlisting

Consider a layout that contains signal nets S1 and S2, and a ground net GND. To perform selective inductance extraction using PEEC mode for nets S1, S2, and GND, include the following statements in your rule file:

PEX INDUCTANCE MODE PEEC
PEX XACT CONTROL FILE my_control_file

where my_control_file contains the following:

EXTRACT_CONTROL {
	NET: S1 S2 GND LAYOUTNAMES
	MODEL: RCCL
}

Perform full-chip RCC model extraction using Calibre xACT direct netlisting:

calibre -xact -rcc -turbo rules

Calibre xACT produces a netlist that contains RCCLK model parasitics for nets S1, S2, and GND, and RCC model parasitics for the remaining nets.
