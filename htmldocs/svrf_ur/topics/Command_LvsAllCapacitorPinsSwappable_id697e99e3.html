LVS All Capacitor Pins Swappable

Specification statement

Specifies whether all capacitor pins should be considered swappable during LVS comparison.

Usage

LVS ALL CAPACITOR PINS SWAPPABLE {NO | YES}

Arguments

NO

A keyword that specifies capacitor pins should not be considered swappable during LVS comparison. This is the default behavior when you do not include this statement in the rule file.

YES

A keyword that specifies that capacitor pins are considered swappable during LVS comparison.

Description

Specifies whether capacitor pins on different layers should be considered swappable during LVS comparison. This statement can be specified once in your rule file.

The POS and NEG pin layers of built-in capacitors are declared in Device statements. When these layers differ, the Device statements are active in the rules, and NO is specified, then the pins are not swappable. The NO setting is overridden if the pin names appear in a Device statement swap list that is active during the run. Similarly, if the pins are on the same layer, they are considered swappable regardless of the NO setting.

See “Pin Swapping” in the Calibre Verification User’s Manual for a complete discussion of pin swapping in LVS comparison.

In Pyxis Layout, this statement sets the value of application variable lvs_all_capacitor_pins_swappable.

See also LVS Report Pinswapped.

Examples

// Do not allow cap pins to be swappable.
LVS ALL CAPACITOR PINS SWAPPABLE NO
