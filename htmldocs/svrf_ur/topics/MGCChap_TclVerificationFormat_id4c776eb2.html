Tcl Verification Format

Tcl Verification Format (TVF) is a programmable extension to the Standard Verification Rule Format (SVRF) language for Calibre. TVF is employed in two distinct environments: compile-time and runtime. TVF uses all standard Tcl constructs, which provides a uniform programming structure.

Starting with Calibre version 2019.2, Calibre uses Tcl 8.6. See “Tcl Library in Calibre” in the Calibre Administrator’s Guide for other general information.

There are similarities and differences between compile-time and runtime TVF, as summarized in the following table.

Table 1. Comparison of Compile-Time and Runtime TVF

Compile-Time TVF

Runtime TVF

Compile-time TVF code appears in a compile-time script beginning with “#!tvf” as the opening line. It cannot appear in a standard SVRF rule file.

Runtime TVF appears within an SVRF element called TVF Function. Runtime TVF can appear either in a standard SVRF rule file, or within a compile-time TVF script.

Compile-time TVF is Tcl code that is interpreted at compilation time. It is translated into SVRF commands at the beginning of a run, and the SVRF commands are then fed to the SVRF compiler.

Runtime TVF is Tcl code that is interpreted at run time. Runtime TVF appears within the TVF Function block, and each TVF function operates in its own environment during the run.

Calibre PERC, the Device property computation built-in language, the DFM Function statement, and the TVF layer operation are among the applications and commands that use procedures defined within a TVF Function block.

Compile-time TVF contains no elements that interact with runtime data.

Runtime TVF allows you to access data generated during a Calibre run.
