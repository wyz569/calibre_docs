PEX Netlist Via Device

Parasitic extraction

Specifies to represent vias as devices in the extracted netlist. Used only in the Calibre xACT direct netlisting flow.

Usage

PEX NETLIST VIA DEVICE layer device [layer device…]

Arguments

layer device

A required parameter set that specifies an LVS layer name or number with a desired via device name. More than one layer device pair can be specified.

Description

An optional statement used to output the affected parasitic resistors on the specified via layers as user devices. Can only be used with the DSPF, HSPICE, SPECTRE, and ELDO netlist formats. This statement can be specified more than once.

Examples

Given the following sample output from a DSPF netlist:

...
R1 VSS:1 VSS:2 36.7 $a=0.01 $layer=V0
R2 VSS:3 VSS:4 45.2 $a=0.02 $layer=V1
R2 VSS:5 VSS:6 55.2 $a=0.03 $layer=V2
...

To output the certain parasitic resistors on the specified via layers V0, V1, and V2 as user devices, specify the following statement:

PEX NETLIST VIA DEVICE V0 V0_dev V1 V1_dev V2 V2_dev

The resulting sample output is:

...
XR1 VSS:1 VSS:2 V0_dev R=36.7 $a=0.01 $layer=V0
XR2 VSS:3 VSS:4 V1_dev R=45.2 $a=0.02 $layer=V1
XR2 VSS:5 VSS:6 V2_dev R=55.2 $a=0.03 $layer=V2
...
