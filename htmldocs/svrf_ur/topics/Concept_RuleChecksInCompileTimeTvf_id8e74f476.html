Rule Checks in Compile-Time TVF

The SVRF rule check construct is handled with the tvf::RULECHECK command in compile-time TVF. Rule checks in SVRF have user-given names, so the SVRF syntax cannot be used in TVF.

A typical rule check has this format:

tvf::RULECHECK name ‘{’ 
     tvf::@ a rule check comment
     tvf::SETLAYER layer_name = arguments 
     tvf::OUTLAYER arguments 
‘}’

The commands shown can be used inside the body of tvf::RULECHECK, as well as tvf::COPY, tvf::COMMENT, and standard Tcl constructs. The arguments parameter to tvf::SETLAYER and tvf::OUTLAYER forms a valid SVRF layer operation.

The SETLAYER command is use for layer derivations. You can use as many SETLAYER commands as needed. In this context, SETLAYER creates a layer that is local in scope to the RULECHECK command in which it appears.

The tvf::OUTLAYER command sends the results of its arguments (which form a layer operation) to the Calibre nmDRC results database.

The OUTLAYER and SETLAYER command arguments may be enclosed in double quotes or braces, where double quotes allow variable substitution and braces do not. Both of these grouping elements may appear in the same group of arguments.

For example, the following SVRF rule check:

my_rule {
	layer1 = layer2 OR layer3
	layer4 NOT layer1
}

is coded like this in TVF:

tvf::RULECHECK my_rule {
	tvf::SETLAYER layer1 = {layer2 OR layer3}
	tvf::OUTLAYER {layer4 NOT layer1}
}

The tvf::COPY command may be used for results output without embedding it in a tvf::OUTLAYER command.

Rule check comments can be specified with the tvf::COMMENT or tvf::@ command.
