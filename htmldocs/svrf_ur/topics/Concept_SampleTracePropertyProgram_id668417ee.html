Sample Trace Property Program

This example shows some primary features of the language. Declaration of PROPERTY or PROPERTY STRING is required. The remainder of the program shows variable assignments, conditional blocks, and function calls the language supports. 
   1   TRACE PROPERTY R [
   2       PROPERTY r
   3       PROPERTY STRING str
   4   
   5   // in this program, tolerance is handled as a percentage
   6       tolerance = 0.1 // "constant" stored as a local variable
   7       discrepancy = 0 
   8   
   9       lay_r = layout_numeric_value(r) // retrieve r value from layout
  10       src_r = source_numeric_value(r) // retrieve r value from source
  11   
  12       if ( src_r != 0 ) { // prefer src value as denominator
  13           diff = ABS( lay_r - src_r )
  14           discrepancy = 100 * diff / src_r
  15       } else if ( lay_r != 0 ) { // lay value as denominator
  16           diff = ABS( lay_r - src_r )
  17           discrepancy = 100 * diff / lay_r
  18       }
  19   // if discrepancy > tolerance, report it as a percentage.
  20       if ( discrepancy > tolerance ) {
  21           report_numeric_discrepancy(r, discrepancy)
  22       }
  23   // compare string property values
  24       if (string_compare(layout_string_value(str),
     source_string_value(str))!=0 ) {
  25           report_string_discrepancy(str)
  26           report_message("string discrepancy")
  27       }
  28   ]

The line numbers on the left are an aid for discussion and are not part of the language. The language is case-insensitive.

In the following discussion, input pair means a layout instance and its single, corresponding, source instance. Property values are available based upon this input pair.

Line 1 shows the standard Trace Property statement with its property computation section. The literal brackets are required to indicate the presence of the Trace Property computation section.

Line 2 contains the PROPerty statement. This statement declares the names of all numeric properties for which layout and source values are made available throughout the rest of the procedure. Each property listed here may be used in data retrieval functions shown later (see lines 9 and 10). The names here represent both layout values (properties found on the layout instance) and source values (properties found on the corresponding source instance).

The LVS Property Map specification statement applies to names in the PROPerty declaration statement. LVS Property Map is used to derive layout-specific and source-specific property names based on the name used by the Trace Property computation program.

Line 3 contains the PROPerty STRing statement. This declares string properties to check, as is shown starting on line 24. Either this statement or the PROPerty statement must be used. They can be used together.

Lines 5, 6, 9, and 10 show uses of the comment character //.

Lines 6 and 7 declare local variables used later in the program. Variable initialization is not necessary, but is sometimes convenient.

Line 9 retrieves the value of a property named “r” from the layout instance, while line 10 retrieves r’s value from the corresponding source instance.

Lines 12-18 show a method for avoiding use of zero values from the input pair. If you provided the statement:

 
REPort_NUMeric_DISCrepancy(r,100*ABS((lay_r-src_r)/src_r))

and the “r” value on the source instance was zero, a discrepancy would be reported as the Unknown value (represented by “?” in the LVS Report file). The Unknown value is assigned because division by zero is attempted and is undefined.

Line 13 stores the absolute difference between lay_r and src_r.

Line 14 calculates the discrepancy value using the non-zero src_r as the denominator.

Lines 15-17 show a similar method for when lay_r is non-zero.

Lines 12-18 also show how braces ({ }) can be used to group multiple statements into a single logical statement. As in C or C++, this grouping is useful for indicating the extent of a dependent clause for a conditional statement.

Note that you could replace lines 13-14 with the single statement:

 
discrepancy = ABS( (lay_r-src_r)/src_r )

Line 20 checks whether the discrepancy exceeds the tolerance value. If so, line 21 reports a numeric discrepancy for property r and the input pair of instances. The source value and layout values of r appear in the LVS Report file, along with the “discrepancy” value calculated in the preceding procedure.

Line 24 checks if there is a discrepancy in the string property str. If so, line 25 reports the discrepancy and line 26 gives a report message.
