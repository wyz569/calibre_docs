PERC LDL Preserve Trivial Pins

Specification statement

Specifies that trivial pins are preserved in Calibre PERC runs.

Usage

PERC LDL PRESERVE TRIVIAL PINS {ALL | SEED PROMOTED CELLS}

Arguments

ALL

A keyword that specifies all trivial pins are preserved.

SEED PROMOTED CELLS

A keyword that specifies trivial pins in cells in which seed promotion has occurred (indicated by *.SEEDPROM in the extracted netlist) are preserved.

Description

Specifies that trivial pins (those not connected to devices) should be preserved during any Calibre PERC run. By default, trivial pins are not preserved. This statement may be specified once.

Trivial pins are written to the extracted netlist, as indicated by the ALL or SEED PROMOTED CELLS settings. Trivial pins and nets are noted in the PERC Report when this statement is used.

Trivial pins are discussed in the “Netlisting of Ports and Pins” section of the Calibre Verification User’s Manual. Seed promotion is discussed under “Hierarchical Device Recognition” in the same manual.

Using this statement may reduce performance due to the extra overhead of tracking trivial pins.

Calibre YieldServer has similar functionality in the dfm::set_layout_netlist_options ‑trivial_pins and -seed_promoted_trivial pins options.

Examples

Consider a cell in which seed promotion has occurred:

.SUBCKT cell 1 2 3 
** N=4 EP=3 IP=0 FDC=1 
*.SEEDPROM
.ENDS

Pins 1, 2, and 3 are trivial and ordinarily would be removed from the netlist.

If PERC LDL Preserve Trivial Pins is specified, then the pins are preserved. If the SEED PROMOTED CELLS keyword is used, then trivial pins are preserved only in cells that contain the *.SEEDPROM comment, such as the one shown in this example.
