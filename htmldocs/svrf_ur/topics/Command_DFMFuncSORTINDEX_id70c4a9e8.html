SORT_INDEX

Vector function for DFM expressions

Returns a one dimensional vector of index values. The index values map the results of the function SORT() on the input vector back to the input vector indices. Supported for the DFM Property family of operations.

Usage

SORT_INDEX(vector)

Arguments

vector

A required vector argument, which may be a numeric, netID, or string vector.

Return Values

A numeric vector with tuple size 1.

Description

SORT_INDEX maps the results of sorting the input vector back to the original vector indices. The result of SORT_INDEX(vector) is a vector with tuple size 1 and the same tuple count as the result of SORT(vector). Each value in the SORT_INDEX output is a zero-based index.

Assume these vectors are created from an input vector v_input:

v_sort = SORT(v_input)
v_sort_index = SORT_INDEX(v_input)

This relationship holds between the vectors, where [i] denotes the ith tuple in a vector:

v_sort[i] = v_input[ v_sort_index[i] ]

This demonstrates the behavior with example vectors:

v_input      = {<25> <14> <32> <28>}
v_sort       = {<14> <25> <28> <32>} // output of SORT(v_input)
v_sort_index = {<1>  <0>  <3>  <2> } // output of SORT_INDEX(v_input)

SORT_INDEX is useful when you have multiple vectors with corresponding values, and want to maintain the correspondence after sorting one of the vectors. This concept is demonstrated with the following vectors:

v_num_in     = {<5>      <2>     <4>}
v_strings    = {<"Five"> <"Two"> <"Four">}
v_sort_num   = {<2>      <4>     <5>} // output of SORT(v_num_in)
v_sort_index = {<1>      <2>     <0>} // output of SORT_INDEX(v_num_in)

You can retrieve each of the strings corresponding to the elements in the sorted numeric vector v_sort_num as follows:

for v_sort_num[0]=2: v_strings[v_sort_index[0]] = v_strings[1] = "Two"
for v_sort_num[1]=4: v_strings[v_sort_index[1]] = v_strings[2] = "Four"
for v_sort_num[2]=5: v_strings[v_sort_index[2]] = v_strings[0] = "Five"

Use the VELEMENT and SVELEMENT functions in a DFM property expression to retrieve individual vector elements. The example demonstrates how to use these functions in a DFM Property expression.

Performance note — Vectors are copied during DFM expression evaluation. Use of many sorted vector functions with large vectors can result in slow performance.

Examples

This example implements part of the preceding demonstration of SORT_INDEX. It also makes use of FIND_SORTED to get the index value for a vector value. 

unsorted_vector_layer = DFM PROPERTY test 
		[ v_num_in = CONCAT(VTUPLE(5), VTUPLE(2), VTUPLE(4))]  
		[ v_strings = CONCAT(SVTUPLE("Five"), SVTUPLE("Two"),
								 SVTUPLE("Four"))]  

sort_output = DFM PROPERTY  unsorted_vector_layer
		[ "+v_num_in" = VPROPERTY(unsorted_vector_layer, v_num_in)]
		[ "+v_strings" = SVPROPERTY(unsorted_vector_layer, v_strings)]
		[ v_sort_num = SORT(VPROPERTY_REF("+v_num_in"))]
		[ v_sort_index = SORT_INDEX(VPROPERTY_REF("+v_num_in"))]
	// get string corresponding to the first entry in sorted numeric vector
		[ first_string_index = VELEMENT(VPROPERTY_REF(v_sort_index), 0)]
		[ first_string = SVELEMENT(SVPROPERTY_REF("+v_strings"), 
											PROPERTY_REF(first_string_index))]
	// get string for number 5 in sorted vector,
		[ five_sorted_index = FIND_SORTED(5, VPROPERTY_REF(v_sort_num))]
		[ five_string_index = VELEMENT(VPROPERTY_REF(v_sort_index), 
											PROPERTY_REF(five_sorted_index))]
		[ five_string = SVELEMENT(SVPROPERTY_REF("+v_strings"), 
											PROPERTY_REF(five_string_index))]

These vectors are attached to layer unsorted_vector_layer:

v_num_in     = {<5>      <2>     <4>}
v_strings    = {<"Five"> <"Two"> <"Four">}

These properties are attached to the layer unsorted_vector_layer:

v_sort_num   =       {<2> <4> <5>}
v_sort_index =       {<1> <2> <0>}
first_string_index = 1
first_string =       "Two"
five_sorted_index =  2
five_string_index =  0
five_string =        "Five"
