LVS Netlist All Texted Pins

Specification statement

Specifies whether to netlist internally-floating, texted pins. Used only in Calibre nmLVS-H and Calibre PERC.

Usage

LVS NETLIST ALL TEXTED PINS {NO | YES}

Arguments

NO

Keyword that specifies that internally-floating pins are filtered out of the extracted layout netlist, whether they are texted or not. This is the default behavior.

YES

Keyword that specifies that all texted cell pins should appear in the netlist; specifically, internally-floating pins should not be filtered out if they are texted.

Description

Controls filtering of texted cell pins in the Calibre nmLVS-H netlist extractor. This statement may be specified at most once.

A net can serve as a pin of a cell if the net interacts with geometries outside the cell, or with top-level port objects, or if the cell is an LVS Box cell and it has a named port object attached to it. Otherwise, the net does not serve as a pin and does not appear in the extracted netlist by default. In this context, a cell pin is texted if the respective net is texted in the cell, or if the respective net is connected to texted port objects at the top level of the cell, excluding the sub-hierarchy of the cell. (Texted port objects can be created with Port Layer Text specification statements. To qualify, the port names must be non-null. Unnamed port objects, such as those created by Port Layer Polygon, do not qualify.)

Internally-floating pins are pins of a cell that are not connected to devices or to LVS Box cell placements within the cell or within the sub-hierarchy of the cell. When NO is specified (the default), such pins are omitted from the .SUBCKT definition of the cell and from subcircuit calls (X statements) referencing the cell, whether they are texted or not. Specifying YES causes internally-floating, texted pins to appear in the extracted netlist.

When YES is used in conjunction with LVS Report Trivial Ports YES, all port objects will either be texted pins (possibly internally floating) in the layout netlist, or they will be listed as trivial ports. However, note that LVS Netlist All Texted Pins YES does not write floating pins as defined under LVS Spice Allow Floating Pins in the extracted netlist.

Floating nets caused by the YES setting are preserved by the LAYOUT NETLIST WRITE command in the Calibre Connectivity Interface (CCI).

In cases where a net is both internally floating and texted, the LVS Netlist All Texted Pins statement takes precedence over the LVS Netlist Internally Floating Pins statement.

Examples

Assume cellA has three texted port objects: in, out, and float. Assume float is an internally-floating pin that is connected to a top-level port object called pad. This situation occurs in the extracted netlist when LVS Netlist All Texted Pins YES is specified (the netlist is simplified for illustration purposes):

.SUBCKT cellA out in float $$ float is an internally-floating pin
C0 in out 0
.ENDS
***************************************
.SUBCKT top out in pad     $$ pad is connected to the float pin
X0 out in pad cellA
.ENDS

When NO is specified, the float pin does not appear in the netlist:

.SUBCKT CellA out in
*.CALIBRE ISOLATED NETS: float
C0 out in 0
.ENDS
***************************************
.SUBCKT TOPCELL out in pad
X0 out in cellA
.ENDS
