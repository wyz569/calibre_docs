EC

DFM measurement function

Returns the mutual edge projection length (Edge Concurrency) of two error edges, or the length of a single edge.

Usage

EC(layer)

Arguments

layer

A required error layer with one or two edges per cluster.

Return Values

Floating-point number in user units.

Description

Returns the projection length of a second edge onto a first edge, or in the case of a single edge, returns its length. The resulting EC segment is also the basis for ECXP, ECYP, ECMIN, ECMAX, ECBEGIN, and ECEND.

Input Edge Orientation

EC Calculation

One edge horizontal or vertical

Returns the projection onto that edge.

Parallel edges

Returns the projection of either edge onto the other.

Non-Manhattan, non-parallel edges

Returns the projection onto the edge closest to horizontal or vertical.

Perpendicular edges

Returns 0.

Single edge input

Returns the edge length.

Illustrations for each of these cases are shown in the following sections.

One Edge Horizontal or Vertical

If at least one edge is horizontal or vertical, the projection is onto that edge.



Parallel Edges

If the edges are parallel to each other, the projection length is the same regardless of the edge chosen.



If the EC calculation is made as part of determining endpoint coordinates, EC segment rotation must also be considered; refer to the next case.

Two Angled Non-Parallel Edges

If both edges are non-Manhattan and not parallel to each other, the projection is onto the edge closest to being horizontal or vertical. The resulting projection is the one that requires the least rotation about the origin (0,0) to make it parallel to an axis.



To elaborate, for the edge pair in the preceding figure, there are two possible EC projection segments. The projection onto the lower edge needs clockwise rotation by 27° to align it to the x-axis, the projection onto the upper edge needs anti-clockwise rotation by 45° to align it to the y-axis. Given the smaller rotation degree required, the EC projection onto the lower edge is chosen. Details are in the following figure.



The rotation direction matters because the EC segment length depends on the edge chosen to be projected upon. This also factors into endpoint coordinates returned by commands such as ECBEGIN. For more information, refer to “EC Group: Projection Endpoints”.



Perpendicular Edges

For perpendicular edges, EC returns 0.



Single Edges

For an input cluster of one edge, EC returns the length of the edge.
