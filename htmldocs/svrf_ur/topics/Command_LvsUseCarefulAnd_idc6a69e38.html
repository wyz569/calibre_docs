LVS Use Careful AND

Specification statement.

Causes circuit extraction (as in calibre -spice) to use a different hierarchical algorithm for AND layer operations. The alternate algorithm preserves the original hierarchy as much as possible at the cost of approximately twice the normal run time. Using YES can reduce layer promotion that causes device pins to form higher up in the hierarchy than desired.

Usage

LVS USE CAREFUL AND {NO | YES}

Arguments

NO

A keyword that causes the hierarchy preserving AND operation not to be used. This is the default.

YES

A keyword that causes the hierarchy preserving AND operation to be used everywhere in the rule file.

Description

Allows use of a different hierarchical algorithm for the AND operation globally in the rule file. This algorithm preserves the original hierarchy of first AND input layers as much as possible. By default, the usual AND operation is used (unless its LVSCAREFUL keyword is specified).

The YES option can solve some issues involving seed promotion that result from gate layers being combined with marker or well layers that are at multiple levels of hierarchy, for example. In the standard hierarchical AND operation, output of device-forming layers can end up at higher levels of hierarchy, causing devices to form at a higher level than intended.

There are some potential side effects from using the YES option:

  • Run time is about twice that of the standard AND algorithm.

  • In cases where a higher level marker layer is used as the first layer in the AND operation, the output will show more promotion than the standard hierarchical AND algorithm.

As an example of this second side-effect, if a high level marker layer called MARKER_LAYER is combined with a GATE layer to produce a specialized MARKED_GATE layer as follows:

MARKED_GATE = MARKER_LAYER AND GATE

the output of MARKED_GATE occurs at the level of MARKER_LAYER, possibly causing undesirable promotion. This ordering of the operation is non-standard since the connectivity for MARKED_GATE would come from MARKER_LAYER rather than GATE. (If the MARKED_GATE layer is part of a Connect operation, this might still work from a connectivity standpoint.)

This situation can be fixed by reversing the order of the AND operation to the traditional ordering:

MARKED_GATE = GATE AND MARKER_LAYER

which produces output at the original level of GATE where possible.

The AND LVSCAREFUL keyword allows the behavior of the YES keyword to be applied to specific layer derivations rather than globally.

Use of the YES keyword can mitigate seed promotion during device recognition. See “Seed Promotion” in the Calibre Verification User’s Manual for more information.

Examples

Suppose this layer operation is responsible for promoting the gate layer up the hierarchy so the hvgate device pin is formed in undesirable locations:

hvgate = gate AND hv

Then this statement may be useful in preventing the problem:

LVS USE CAREFUL AND YES
