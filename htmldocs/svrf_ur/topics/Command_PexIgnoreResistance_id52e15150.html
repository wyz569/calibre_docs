PEX Ignore Resistance

Parasitic extraction

Used to ignore parasitic resistance for the specified layer and via connections during extraction.

Usage

PEX IGNORE RESISTANCE [GLOBAL] layer [layer…]

PEX IGNORE RESISTANCE [GLOBAL] INDIE layer [layer…]

PEX IGNORE RESISTANCE [GLOBAL]
{VIA layer1 layer2 [VIANAME vialayer_name [vialayer_name …]]
[VIA layer1 layer2 [VIANAME vialayer_name [vialayer_name …]]]…}

PEX IGNORE RESISTANCE VIANAME vialayer_name [vialayer_name …]

PEX IGNORE RESISTANCE DEVICE [GLOBAL] layer [layer …] [VIA layer1 layer2 …] CELL cell_name [cell_name …]

PEX IGNORE RESISTANCE DEVICE pin_layer [pin_layer …] marker_layer

Arguments

GLOBAL

An optional keyword used to apply the specified ignore to all physical layers mapped to the same logical layer. This option is only valid with layers that have been specified in a PEX Map statement.

layer

One or more original or derived layers. At least one layer must be specified.

INDIE

Specifies the fill layers that should not have in-die variation applied when calculating resistance.

VIA layer1 layer2

Required keyword set that specifies to ignore via resistance on vias connecting layer1 and layer2. More than one via connection may be specified.

VIANAME vialayer_name

Required keyword set that specifies to ignore resistance for one or more vias, specified by LVS via layer name, vialayer_name. May optionally be specified with the VIA layer1 layer2 keyword set to filter vias by LVS via layer name.

DEVICE

Required keyword that specifies to ignore the named cell resistance for the specified device layer and the named cell via resistance when the optional VIA keyword is specified.

CELL cell_name

Required keyword set that specifies the cell_name for the specified device. More than one cell may be specified.

pin_layer

One or more pin layers of the devices recognized by the seed layer. At least one pin layer must be specified.

marker_layer

Required parameter that specifies the seed layer for the device to be ignored.

Description

Ignores any resistivity for the specified layer or connections during extraction. Connectivity is unchanged but the section or cell contributes no resistance to the parasitic extraction results.

If the rule file specifies PEX Ignore Resistance VIA for two layers, the resistances of any vias connecting the two layers are ignored, regardless of the layers on which the vias exist. Include the optional VIANAME vialayer_name keyword set to additionally ignore the resistances of vias that exist on the specified LVS via layer.

If the rule file specifies PEX Ignore Resistance VIANAME, the resistances of all vias that exist on the specified LVS via layer are ignored, regardless of the layers they connect.

If the rule file specifies PEX Ignore Resistance DEVICE VIA … CELL, the resistance of any vias inside the named cells are automatically ignored.

If the rule file specifies PEX Ignore Resistance INDIE, then ignore in-die variation when calculating resistance for the specified layers. Resistance calculations for the specified layers are based on drawn values.

If the rule file specifies PEX Ignore Resistance DEVICE pin_layer marker_layer, then ignore the resistance of the pin layers of the devices recognized by the seed layer.

Examples

The diagram shown in Figure 1 is used to illustrate Examples 1 through 10. It contains the following:

  • Metal layers: M1, M1_a, M2, M2_b.

  • Via layer, VIA_a, which contains a via (A1) connecting M1 to M2, a via (A2) connecting M1 to M2_b, and a via (A3) connecting M1_b to M2_b.

  • Via layer, VIA_b, which contains a via (B1) connecting M1 to M2 and a via (B2) connecting M1_b to M2_b.

  • Via layer, VIA_c, which contains a via (C1) connecting M1 to M2_b and a via (C2) connecting M1_b to M2_b

    Figure 1. Resistance Ignores for Vias and Via Layers

Example 1

The statement

PEX IGNORE RESISTANCE VIA M1 M2

Ignores resistance for all vias connecting layer M1 to layer M2, regardless of the layers on which the vias exist. In Figure 1, vias A1 and B1 are ignored.

Example 2

The statement

PEX IGNORE RESISTANCE VIA M1 M2 VIANAME VIA_a

Ignores resistance for all vias connecting layer M1 to layer M2 that exist on VIA_a. In Figure 1, via A1 is ignored.

Example 3

The statement

PEX IGNORE RESISTANCE VIANAME VIA_a

Ignores resistance for all vias that exist on layer VIA_a, regardless of the layers they connect. In Figure 1, vias A1, A2, and A3 are ignored.

Example 4

The statement

PEX IGNORE RESISTANCE VIANAME VIA_a VIA M1 M2_b

Ignores resistance for all vias that exist on layer VIA_a, regardless of the layers they connect, and all vias connecting layer M1 to layer M2_b. In Figure 1, vias A1, A2, A3, and C1 are ignored.

Example 5

Specify the statement

PEX IGNORE RESISTANCE VIANAME VIA_a VIA_b VIA_c

to ignore resistance for all vias shown in Figure 1.

Example 6

Specify the statement

PEX IGNORE RESISTANCE VIA M1 M2 VIA M1 M2_b VIA M1_b M2_b

to ignore resistance for all vias shown in Figure 1.

Example 7

The statement

PEX IGNORE RESISTANCE VIA M1 M2 VIA M1 M2_b VIANAME VIA_a VIA M1_b M2_b

ignores resistance for all vias except C1 in the example shown in Figure 1.

Example 8

The statement

PEX IGNORE RESISTANCE VIA M1 M2 VIANAME VIA_a VIANAME VIA_c

ignores resistance for vias A1, C1, and C2 in the example shown in Figure 1.

Example 9

The statement

PEX IGNORE RESISTANCE VIA M1 M2 VIANAME VIA_a VIA_b

ignores resistance for vias A1 and B1 in the example shown in Figure 1.

Example 10

The statement

PEX IGNORE RESISTANCE VIA M1 M2 VIANAME VIA_c

does not ignore any resistance for any vias in the example shown in Figure 1. This is because there are no vias on layer VIA_c that connect layer M1 to layer M2.

Example 11

To ignore resistance from metal1 and connection resistances between metal1 and diffusion layers in the nand1 and nor1 cells, specify the following statement in the rule file:

PEX IGNORE RESISTANCE DEVICE metal1 VIA tndiff metal1 VIA tpdiff metal1 CELL nand1 nor1
