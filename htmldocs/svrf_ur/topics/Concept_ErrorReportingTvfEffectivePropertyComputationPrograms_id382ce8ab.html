Error Reporting for TVF Effective Property Computation Programs

When the rule file is compiled, TVF Function blocks called by the effective property programming language are compiled by the interpreter. If these fail to compile, a Tcl error is printed to the transcript.

The error has this form:

** Error in TVF FUNCTION at line <line>

invalid command name “string”

    while executing

“code”

A TVF3 error is issued from the SVRF compiler. The line number issued is the starting line of the TVF Function block:

ERROR: Error TVF3 on line <line> of rules - This Tcl Proc failed to compile: <proc_name>.

After the TVF Function block is compiled, a call is made to the specific proc that failed to compile. If the call fails, a Tcl error is printed:

*** Error in call to TVF FUNCTION <proc_name>

A TVF4 error is issued from the SVRF compiler. The line number issued is the location of the call:

ERROR: Error TVF4 on line <line> of rules - A call to a Tcl Proc failed to compile.

The proc is called with reasonable default values (usually 0) for passed-in variables during this preliminary execution.

Note:

Runtime Tcl errors are printed only to the transcript. Ensure that any runtime Tcl errors are fixed during rule file qualification because the only place they might appear is the transcript. As discussed under “Missing and Unknown Properties in TVF”, passed-in parameters can take on the UNKNOWN value.
