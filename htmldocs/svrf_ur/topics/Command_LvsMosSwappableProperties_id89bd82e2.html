LVS MOS Swappable Properties

Specification statement

Specifies swappable, user-defined source and drain pin properties. Used only in Calibre nmLVS-H and Calibre PERC.

Usage

LVS MOS SWAPPABLE PROPERTIES source_property drain_property

Arguments

source_property

A required name of a MOS Device element source pin user-defined property.

drain_property

A required name of a MOS Device element drain pin user-defined property.

Description

Specifies the names of user-defined MOS Device properties for source and drain pins that can be swapped during device pushdown. MOS equivalent devices defined by LVS Device Type are included. This statement applies to both circuit extraction and LVS comparison.

The specified properties must be derived from the pins they are associated with. So the source_property must be derived from the source pin and similarly for the drain_property and drain pin.

This statement is used in conjunction with LVS Push Devices YES. For MOS devices that have the specified properties, the pins can be swapped to allow for device pushdown. Ordinarily, user-defined property values must match the Device statement for pushdown to occur. The swapping of user-defined properties is the same as the default swapping of AS/AD, PS/PD, and NRS/NRD properties in the same situations.

When such properties are not LVS-actionable and are separated from the extracted netlist due to the LVS Push Devices SEPARATE PROPERTIES YES statement, the properties are swapped in the separated properties file generated by the Calibre Connectivity Interface (CCI).

When LVS-actionable properties are not separated and remain in the extracted netlist, devices that are tentatively promoted are pushed back down the hierarchy if, at the site of tentative promotion, the properties in question are swapped at that level. Then, the devices may be pushed if connectivity shows pin swapping further down the hierarchy. The property values are not swapped in the netlist.

During LVS comparison, when source and drain pins are swapped, the associated properties specified in this statement are swapped as well. The properties may be numbers or strings, and the properties must match according to these types in order to be swapped.

This statement may be specified more than once, and each statement’s property pair must be unique.

Examples

This example shows a typical usage:

DEVICE MP gate gate(G) sd(S) sd(D) nw(B) [

   PROPERTY L, W, AS, AD, PS, PD, SCC, DCC
// SCC is derived from the source pin
// DCC is derived from the drain pin
...
]

LVS PUSH DEVICES YES

// AS, AD, PS, PD are swappable by default
LVS MOS SWAPPABLE PROPERTIES SCC DCC // allow these to be swapped as well
