layer

LITHO FILE Statement and Commands

An input layer called by setlayer for generating SRAFs. Used by RET SBAR and RET MBSRAF.

Usage

layer layer_name [hidden [transmission] | visible transmission]

Arguments

layer_name [hidden [transmission] | visible transmission]

A required argument specifying the name of the layer. Calibre nmSRAF uses this name throughout the LITHO FILE statement to refer to this layer.

Layer names must consist of alpha-numeric characters, and form strings less than 32 characters in length. Any number of layer commands may be specified within a LITHO FILE statement. The layer type and transmission arguments are only used by RET MBSRAF.

  • hidden [transmission]

    • The hidden layer type is applied to all layers that are part of the layout, but not simulated. The transmission argument is optional for the hidden type.

  • visible transmission

    • The visible layer type is applied to all layers that are simulated. The transmission argument is required for the visible type.

Description

A required command that names an input layer. Every layer to be used in either setlayer cnsraf or setlayer mbsraf commands must be declared in the LITHO FILE statement must be declared using the layer command. Layer applications may include target layers, keepout areas (offsetlayer), marker layers (taglayer), preexisting SRAF layers (sblayer), or alternate simulation layers (for example, correctionlayer used by mbSRAF only). Only manhattan- and 45-degree input layer geometries are supported. No skew‑edged geometries are valid target shapes for SRAFs.

Usage for setlayer mbSRAF

The target layer is a required layer for mbSRAF because the shapes provided on the target layer define the evaluation points for the gradient calculation. If a portion of the original target layer is being processed in mbSRAF (typically using mbSRAF as SRAF fill after cnsraf rules-based insertion), the selected region of the target layer must be large enough to provide an empiric topological environment for any subset of target shapes in order for the gradient to be representationally accurate. If a shape is artificially isolated, the gradient can be erroneous due to inadequate sampling of the empiric target environment.

The sblayer and correctionlayer are layers containing shapes that are simulated to provide the image from which the gradient is calculated. If neither of these layers are available, the target layer is used for the image simulation.

When using mbSRAF as fill or an SRAF safety net after rules-based insertion, all layers processed must contain a large enough region centered on the selected shape needing further lithographic validation through an optical diameter, but the generated SRAFs produced by mbSRAF must only be constrained to a smaller halo around the center target (using a value equal to or smaller than maxoffset).

Note:

The layer command must appear before the first setlayer command. The order of layer commands in the LITHO FILE statement represents the order the declared layers will be used in RET MBSRAF and RET SBAR calls.

For more information regarding layer, see layer in the Calibre OPCVerify User’s and Reference Manual.

Examples

The layer command is specified within the LITHO FILE statement:

LITHO FILE LF [
	#layer name    type     transmission 
	layer M1 visible 0 0
	layer offset_layer hidden 
	layer p180 visible -1 0 
	layer sb_layer
	...
]
