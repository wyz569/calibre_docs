intersectionnegative

Style Keywords

Enables valid negative SRAFs to be merged at various junctions. Used by RET SBAR and RET MBSRAF.

Usage

intersectionnegative {J | L | N | P | T} [minjogwidth value]

Arguments

J

Enables valid, generated SRAFs to be merged at jog point intersections. The J option can be used with either positive or negative SRAFs. When using intersectionnegative J with lineendmerge, jogs are merged only when SRAF widths match. This is the default behavior with cleanupversion 1.

Note:

Siemens EDA recommends using intersectionnegative J with the lineendmerge keyword set to a value greater than minoffset. This provides more continuous SRAF results. If no lineendmerge parameter is set, its value is automatically set. This value is reported to the transcript during processing. During Calibre nmOPC and SRAF Print Avoidance (SPA) after SRAF generation, fragments near concave corners cannot be aligned and eliminated. SPA cannot resolve printing at or near jogs because of the inability to align fragments near concave corners.

L

Enables valid, generated SRAFs to be merged at 90-degree junctions, irrespective of orientation angle. The L option can be used with either positive or negative SRAFs. intersectionnegative L is incompatible with cleanupversion 0.

N

Disables SRAF merging. The N option can be used with either positive or negative SRAF generation. The N option is default.

P

Enables valid, generated SRAFs to be merged at orthogonal, non-45-degree junctions (forming four-way intersections or multiple T-junctions). The P option can be used with either positive or negative SRAFs. intersectionnegative P is incompatible with cleanupversion 0.

T

Enables valid, generated SRAFs to be merged where lineend and long edges of SRAFs meet (forming T-junctions). The T option can be used with either positive or negative SRAFs. intersectionnegative T is incompatible with cleanupversion 0.

maxjogwidth value

An optional keyword and argument specified as a positive, floating-point number in microns. Sets the maximum overlap of a jog. It can be set globally or by priority. The maxjogwidth parameter does not apply when smaller than the overlap between line-ends. The default for maxjogwidth is the maximum of the width parameters for all templates at the applied priority.

minjogwidth value

An optional keyword and argument specified as a positive, floating-point number in microns. This keyword overrides the default jog width (equal to the template-based width keyword). The default for minjogwidth is minwidth (if specified) or the minimum width parameter of any template of any priority. This default ensures that no MRC violations can occur when SRAFs of different priorities are jogged.

minstickout value

An optional keyword and argument that defines the minimum length a leg can extend from an intersection formed by intersectionnegative L, T, or P. The minstickout keyword controls both existing overlapping SRAFs and subsequently generated intersections. Existing overlapping SRAFs not meeting the minimum value specified by minstickout are cleaned. The default for minstickout is minwidth (if specified) or the minimum width parameter of any template of any priority. The minstickout value must be specified as a positive, floating-point number in microns.

minwidthfree value

An optional keyword and argument specified as a positive, floating-point number in microns. Specifies the minimum non-overlapping portions of jog intersections. See Figure 3 for more information.

minwidthoverlap value

An optional keyword and argument specified as a positive, floating-point number in microns. Specifies the minimum overlap of line widths forming jog intersections. See Figure 3 for more information.

Description

An optional keyword that enables valid, generated negative SRAFs to be joined at jog point intersections. The intersectionnegative keyword is used only for negative SRAF generation and should only be set globally. The default jog width (established by the template width keyword) can be overridden with minjogwidth. Although describing positive SRAF, see the intersection keyword for more examples of L, P, and T SRAF merging interaction.

Note:

Siemens EDA recommends using intersection J by priority to control jogs with both positive and negative SRAFs within a single run. See the negative keyword for details regarding this procedure. During a Calibre nmOPC run with SRAF Print Avoidance (SPA) after SRAF generation, fragments near concave corners cannot be aligned and eliminated. SPA cannot resolve printing at or near jogs because of the inability to align fragments near concave corners.
