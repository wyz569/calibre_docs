srafring

Templates

Places square SRAFs radially around a target shape. Used by RET SBAR only.

Usage

srafring
[tag name ...]
circumoffset value
pitch value
width value
[ondiagonal {no | yes}]
[prior value]

Arguments

tag name ...

An optional keyword specifying one or more tag names used in the template. The tag name identifies a group of edges or shapes to be used with the template. If specified, tags must appear as the first arguments in the template.

circumoffset value

A required keyword that specifies the center-to-center distance between square SRAFs. This distance is measured along the arc of the ring, not the straight line distance between square SRAFs.

pitch value

A required keyword that specifies the distance from the center of the target shape to the center of the square SRAFs.

width value

A required keyword that specifies the width of the SRAFs placed around the square target shape.

ondiagonal no | yes

An optional keyword that places the initial SRAFs on 45‑degree angles. By default, the initial SRAF is placed on the Y‑axis.

prior value

An optional keyword that specifies a numeric identifier for placed SRAFs. Identifiers are applied sequentially, so a value of 1 is applied first, then 2 is applied, and so on. The default priority is 1. See “priority” for more information.

Description

An optional template and argument set that places square SRAFs radially around a target shape. The number of square SRAFs is determined by the circumoffset distance. The center of the SRAFs are placed the specified distance (pitch) from the center of the target shape. This template cannot generate negative SRAFs.

Examples

Example 1

The srafring template places SRAFs by pitch starting at 0 degrees and then computes the circumoffset for all remaining SRAFs.

Example 2

The ondiagonal option places one SRAF on each 45-degree angle from the target and then computes the circumoffset for all remaining SRAFs.
