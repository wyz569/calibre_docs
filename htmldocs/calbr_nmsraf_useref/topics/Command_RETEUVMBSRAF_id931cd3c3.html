RET [EUV] MBSRAF

SVRF Commands

Invokes Calibre mbSRAF.

Usage

output_layer = RET [EUV] MBSRAF
target_layer
[correction_layer]
[INSIDE OF LAYER enclosing_layer]
[offset_layer]
[sb_layer [sb_layer2]]
[tag_layer]
FILE {filename | name}
MAP name

Arguments

output_layer

A required output layer name. The results of the RET MBSRAF command are written to this layer.

EUV

An optional keyword necessary for application of EUV (extreme ultraviolet) lithographic modeling. If EUV models (those with a 13.5 nm wavelength) are referenced, and the EUV keyword is not specified, a compile error results.

target_layer

A required input layer consisting of target shapes. The target_layer must be specified as the first argument to RET MBSRAF and only one target_layer may be specified.

correction_layer

An optional polygon-based layer acting as a corrected target layer and used only to calculate the gradient. Siemens EDA recommends using the correction_layer for best SRAF performance. Only one correction_layer may be specified.

INSIDE OF LAYER enclosing_layer

An optional keyword specifying a layer of shapes that enclose target shapes for which SRAFs are generated. Generated SRAF shapes intersecting the enclosing_layer are truncated.

offset_layer

An optional polygon layer containing preexisting polygons that define areas where scattering bars cannot be created. The target layer should never be used as the offset_layer. Only one offset_layer may be specified.

sb_layer [sb_layer2]

An optional polygon layer containing preexisting scattering bars to be factored into calculations when generating additional scattering bars. There is no default for this layer setting. If the sb_layer name is specified once, it may contain both positive and negative SRAFs. Alternatively, the sb_layer may be specified twice with no order dependency, one layer for positive SRAF and one layer for negative SRAF.

tag_layer

An optional polygon layer specifying a tag name representing imported layers or edges from SVRF. Any number of tag_layers may be specified.

FILE {filename | name}

A required keyword with one of the following arguments:

  • filename — Indicates the path to the LITHO FILE statement contained within an external file. The filename parameter can contain environment variables. For information regarding the use of environment variables in the filename parameter, refer to “Environment Variables in Pathname Parameters” in the Standard Verification Rule Format (SVRF) Manual.

  • name — The name of a LITHO FILE statement.

MAP name

A required keyword and layer pair specifying the name of a layer that is mapped to the RET MBSRAF output layer. The layer name must be generated by the setlayer command within the LITHO FILE statement that is called by RET MBSRAF.

Description

The RET MBSRAF command inputs target layer shapes and the model file specified in the FILE section to output a new layer containing SRAFs. This command can be specified any number of times in the rule file.

RET EUV MBSRAF also supports black border models for simulation.

Note:

RET EUV MBSRAF uses the Calibre pxOPC algorithm internally to generate its gradient. As of the 2018.1 release, this capability has been extended to RET MBSRAF for DUV. Known issues with the Calibre pxOPC interface in combination with Fullscale processing mode support are undergoing correction, but do not affect RET EUV MBSRAF integration with Fullscale.

Note:

If both dimensional- and model-based templates are used for a given design, Siemens EDA recommends running the dimensional templates with RET SBAR first, and subsequently import the SRAF results to the sb_layer used by RET MBSRAF.

Examples

Example 1

In the following example, the RET MBSRAF command is compiled, and the SVRF rule file is searched for the LITHO FILE statement named mbsraf_LF. The statement name is case-insensitive. The mbsraf_LF statement is used if found within the SVRF file. If not found, mbsraf_LF is searched for as an external file on disk.

The layer order specified within the LITHO FILE statement must match the order as specified in the RET MBSRAF command, and the layer saved by setlayer must appear as the MAP layer as demonstrated in this example:

LITHO FILE mbsraf_LF [ 
	# mbsraf keywords ... 
	layer ca_target visible
	layer sb_layer visible
	... 
	sraf_options mbsraf_opts {
		algorithm 1
		width 0.025 ...
	}

	setlayer mbsb = mbsraf ca_target OPTIONS mbsraf_opts ...
] 

mbSRAF = RET MBSRAF ca_target sb_layer FILE mbsraf_LF MAP mbsb

Example 2

In the following example, the RET EUV MBSRAF command calls a LITHO FILE block named mbsraf_EUV. This LITHO FILE block contains standard and EUV-specific statements:

LITHO FILE mbsraf_EUV [ 
	# mbsraf keywords ... 
	layer ca_target visible
	layer sb_layer visible
	flare_longrange euv_litho_modelconstant 0.06 png flare_file
	euv_slit_x_center 0.0054
	shadow_bias_model_load sh_mod euv_shadow_14
	... 
	sraf_options mbsraf_opts {
		algorithm 1
		width 0.025 ...
	}

	setlayer mbsb = mbsraf ca_target OPTIONS mbsraf_opts ...
] 

mbSRAF_EUV = RET EUV MBSRAF ca_target sb_layer FILE mbsraf_EUV MAP mbsb
