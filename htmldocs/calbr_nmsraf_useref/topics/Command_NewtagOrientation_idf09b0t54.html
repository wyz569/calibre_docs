newtag orientation

Tag Keywords

Specifies a tag name representing orientations for edges, corners and angles. Used by RET SBAR and RET MBSRAF.

Usage

newtag name ‘(’orientation‘)’

Arguments

name

Specifies the name of the tag.

orientation

Specifies the orientation keyword which must be enclosed in parenthesis when accompanied by constraints:

  • angled — Selects all edges not orthogonal to the X- or Y-axes.

  • eastedge — Selects all edges to the right side of shapes.

  • horizontal — Selects all edges parallel to the X-axis.

  • northeastcorner — Selects and outputs 5dbu of the two edges forming the corresponding corner of the upper-right corner of target shapes.

  • northedge — Selects all edges to the upper side of shapes.

  • northwestcorner — Selects and outputs 5dbu of the two edges forming the corresponding corner of the upper-left corner of target shapes.

  • southeastcorner — Selects and outputs 5dbu of the two edges forming the corresponding corner of the lower-right corner of target shapes.

  • southedge — Selects all edges to the lower side of shapes.

  • southwestcorner — Selects and outputs 5dbu of the two edges forming the corresponding corner of the lower-left corner of target shapes.

  • vertical — Selects all edges parallel to the Y-axis.

  • westedge — Selects all edges to the left side of shapes.

Description

The orientation tag selects target edges of the specified orientation or a certain side in an axis-aligned polygon. Excepting the angled orientation, target polygons containing angled edges are not selected. All of the orientations are mutually exclusive. Use boolean tags or specify multiple tags in the templates to combine different orientations.

Only manhattan- and 45-degree input layer geometries are supported. No skew-edged geometries are valid target shapes for SRAFs.

Note:

All corner-based tags are generated with two adjacent edges of 5 dbu in length. These two adjacent edges are imported when specified using the tag argument by the rectedge, rect45corner, and rectcorner templates.

Examples

Example 1 — Basic Newtag Orientation Usage

newtag tag1 vertical
newtag tag2 angled
newtag tag3 northedge

Example 2 — Dimensional Usage

newtag AllShortEdges length <  0.1
newtag AllLongEdges length >= 0.1

Example 3 — Combined Boolean and Dimensional Usage

Note:

In-place assignment by orientation keywords requires their parenthetic enclosure. Two general forms of boolean syntax apply for tags, however, only the latter form is valid for boolean operations with keywords:

  • <operator> <tag> <tag>
  • <tag> <operator> <tag>
newtag AllBottomEdge southedge
newtag ShortBottomEdge (southedge) length <  0.1
newtag LongBottomEdge (southedge) length >= 0.1
newtag AllHorizontalEdge (northedge) OR AllBottomEdge 
newtag AllHorizontalShortEdge AllHorizontalEdge length < 0.1
newtag bottomEdge southedge 
newtag bottomLeftCorner southwestcorner 
newtag LL_or OR bottomEdge bottomLeftCorner
newtag LL_and AND bottomEdge bottomLeftCorner
