maxconflict

Style Keywords

Specifies the maximum size of a conflict region. Used by RET SBAR only.

Usage

maxconflict value

Arguments

value

A required positive floating point number that specifies the maximum size of a conflict region to process. Siemens EDA recommends setting this value less than 0.5 um. If a larger value is needed, create an additional template instead.

Description

A required keyword that sets the limit on the size of a conflict region to process. The maxconflict keyword is critical for promotion during hierarchical processing. MRC conflicts typically result with rule-based SRAF generation in complex topologies. For this reason, maxconflict requires a value for SRAF placement, likely targeting a significant portion of your design.

Conflict regions larger than maxconflict are eliminated. If maxconflict is set to 0, no conflict regions are calculated, and no secondary SRAFs are generated. Setting maxconflict greater than 0 keeps qualifying regions and processes them according to the rules specified by centerthresh. If centerthresh is not specified, one square SRAF is placed in the center of the conflict region. For additional details regarding the size of the SRAF placed in the conflict region, see the sraf and oversizesraf keywords.

This keyword can be set globally and by priority for all cnsraf templates. If no global or priority specification of maxconflict is provided, a compile-time error is issued.

Examples

Example 1

The dimensions of the conflict region (0.220, 0.205) are greater than maxconflict (0.200) so the conflict region is removed and all secondary SRAF discarded.

Figure 1. Conflict Region Discarded by maxconflict

Example 2

The X,Y dimensions of the conflict region (0.110, 0.110) are less than maxconflict (0.200) so the conflict region is processed according to the rules specified by centerthresh. If centerthresh is not specified, a secondary square SRAF, as specified by the cnsraf template, is placed in the center of the conflict region. If the sraf keyword is specified, a secondary square SRAF is generated with the sraf value you specify, not exceeding the boundary of the conflict region. To override the conflict region extent during secondary SRAF generation, use oversizesraf.
