squarewidth

Constraint Keywords

Specifies the width of square SRAFs. Used by RET MBSRAF only.

Usage

squarewidth value

Arguments

value

A positive floating point number specified in microns.

Description

An optional keyword that specifies the width of square SRAFs. The squarewidth keyword may be specified globally or within the localmaxgradient template.

SRAFs generated by squarewidth are placed near corners and have no projection to any target edge. The placement style of square SRAFs at target corners is controlled by cornerpref. The squarewidth keyword may also be specified in the localmaxgradient template.

If the squarewidth parameter is greater than the width parameter, then a square SRAF with a width equaling the squarewidth value is placed around the local maximum pixels.

The squarewidth and width parameters can be simultaneously specified, generating two sizes of square SRAFs. Where width-based square SRAFs are placed in projection with a target edge, they are potentially converted to MRC-compliant rectangular SRAFs. SRAFs generated by squarewidth are never converted to rectangles.

Although minarea applies to all SRAFs, if minsquarearea or minsquarelength are specified, all square SRAFs are checked against these two parameters and not minarea. If squarewidth2 is less than minarea, no square SRAFs are generated, unless either minsquarearea or minsquarelength are specified.

You can specify squarewidth globally or by priority.

Note:

In Calibre 2014.4 and later, squarewidth is no longer automatically calculated. Instead, square SRAFs of an area equal to width2 are generated, given they do not violate either minsquarelength or minsquarearea parameters.
