algorithm

Style Keywords

Modifies geometric consistency of generated SRAF candidates. Used by RET MBSRAF only.

Usage

algorithm 0 | 1

Arguments

0

Forces model-based SRAF to adhere to the original (pre-2016.3) algorithm.

1

Invokes model-based SRAF with an updated algorithm.

Description

A required keyword that modifies the geometric consistency of generated SRAF candidates. This is a global setting and cannot be specified per template or per priority.

Note:

Regarding algorithm 1 requirements and interactions:

  • The algorithm keyword is required as of Calibre release 2018.1. Siemens EDA recommends always using algorithm 1 for new mbSRAF recipes to increase geometric consistency.
  • The algorithm 1 operation modifies several internal parameters. The internally-set keywords peaktype, and modes nearestno45 and collectno45, negate performance of algorithm 1 and must not be specified within the same recipe.
  • Prior to release 2018.1, the default for maxmove where algorithm 1 was specified, was 0nm. The maxmove default as of Calibre release 2018.1 is 1nm. Therefore, existing recipes using algorithm 1 must explicitly specify maxmove 0 to obtain the previously generated SRAF results.

Setting algorithm 1 requires further explicit specification of two keywords:

  • minoffset

  • minfeaturespace

The algorithm keyword also sets the defaults for three keywords:

  • peaktype (set internally and cannot be modified)

  • mode (set internally and cannot be modified)

  • cornerpref (set to no, but may be changed depending on the type of SRAFs required for given models)

Examples

You must specify algorithm as a global keyword.

Example 1 — algorithm 1

sraf_options mbopt1 {
	...
	algorithm 1
	minoffset 0.025
	minfeaturespace 0.040
	...
}
