svrf_var_import

LITHO FILE Statement and Commands

Imports variables from the SVRF into the LITHO FILE command. Used by RET SBAR and RET MBSRAF.

Usage

svrf_var_import var1 ... varN

Arguments

var1 ... varN

A required argument that specifies SVRF variables to be imported into the LITHO FILE statement.

Description

Imports each of the listed variables from SVRF (defined using the VARIABLE command), which creates Tcl variables of the same name in the global scope of the setup command block. If a listed variable name is not defined in SVRF, it results in an error. If a listed variable name already exists in the setup, this overrides the variable value.

Examples

Variable substitution is demonstrated using Tcl inside an SVRF file, within a LITHO FILE block, and by declaring environmental variables prior to running Calibre cnSRAF. The use of uppercase is not required but can be used to differentiate between user variables and keywords.

Example 1. Using Variables With TCL
VARIABLE MINOFFSET 0.020 
... 
LITHO FILE cn_setup [ 
	... 
	svrf_var_import MINOFFSET 
 
	sraf_options cnsraf_opts {
		minoffset {use $MINOFFSET} ...
	}
	setlayer cnsb = cnsraf ref_contact_layer OPTIONS cnsraf_opts \ 
		 ... 
	... 
]
Example 2. Using Variables Within a LITHO FILE Statement
LITHO FILE mb_setup [
	...
	set width 0.035
	...
	sraf_options mbsraf_opts {
		width $width ...
	}
	setlayer mbsb = mbsraf m1_target OPTIONS mbsraf_opts ...
]
Example 3. Environmental Variable Method

In a csh shell, execute:

setenv WIDTH 0.025

After the variable is assigned in the shell, you pass it to the LITHO FILE statement:

LITHO FILE mb_sraf [
	...
	sraf_options mbsraf_opts {
		width $env(WIDTH) ...
	}
	setlayer mbsb = mbsraf m1_target OPTIONS mbsraf_opts \
	...
]
