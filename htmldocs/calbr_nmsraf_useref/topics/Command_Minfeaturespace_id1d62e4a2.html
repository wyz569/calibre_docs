minfeaturespace

Constraint Keywords

Specifies the minimum distance from target shapes to an SRAF. Used by RET SBAR and RET MBSRAF.

Usage

minfeaturespace value
[lineend value
   [lemaxadjacent value]
   [lemaxlength value]
   [leminadjacent value]
   [leminlength value]
   [leprojectionin value]
   [leprojectionside value]
]
[projecting value]
[lineend2long value]
[long2lineend value]
[corner2sqcorner value]
[lineend2sqedge value]
[long2sqedge value]
[sqcorner2corner value]
[sqcorner2sqcorner value]
[sqedge2lineend value]
[sqedge2long value]
[sqedge2sqedge value]

Arguments

value

A required positive floating point number specifying the basic minimum distance from target shapes to SRAFs. If not specified, minfeaturespace defaults to minoffset. The minfeaturespace command enforces a distance based on a fraction of the optical pitch if the specified value is smaller than the calculated value. For recipes using algorithm 1, no internally calculated value is used; only explicitly specified values are enforced.

lineend value

An optional keyword and positive floating point number that specifies the minimum distance from the line-end of a target shape to any SRAF. The lineend distance is measured from the edge declared as the line-end of a rectangular target shape. Square target shapes are not considered by the lineend keyword. The lineend keyword forms a shape that acts as a keepout area for subsequent OPC generation. The lineend value must be greater than or equal to the minfeaturespace value. The default value for lineend is equal to the value specified for minfeaturespace.

Additional line-end keywords are available to refine the qualifying criteria of line-ends and line-end keepout areas. To use these keywords you must first specify lineend:

  • leminlength value — Specifies the lower limit of line-end length. Defaults to 0.

  • lemaxlength value — Specifies the upper limit of line-end length. Defaults to 150 nm.

  • leminadjacent value — Qualifies line-ends by the shortest leg sharing a convex corner of the line-end. This is the lower limit of the leg length. Defaults to lemaxlength.

  • lemaxadjacent value — Qualifies line-ends by the longest leg sharing a convex corner of the line-end. This is the upper limit of the leg length. There is no default.

  • leprojectionin value — Extends the keepout area into the line-end by the value specified. Defaults to 0.

  • leprojectionside value — Extends the keepout area from the sides of the line-end by the value specified. Defaults to 0.

projecting value

An optional keyword and positive floating point number that specifies the minimum side projection distance measured along the long edge of the target shape. In cases of square target shapes, projection is measured on all edges. In cases of angled target edges, projection is measured asymmetrically, from target shapes toward SRAFs, and symmetrically between SRAFs. The value of projecting must be greater than or equal to the value of minfeaturespace, and the projecting value may be greater than or equal to the specified lineend value. The projection default value is equal to the value specified for minfeaturespace.

lineend2long value

An optional keyword and positive floating point number that specifies the minimum distance from the long side of the target shape to the line-end of an SRAF. The measurement is performed during SRAF cleanup. The lineend2long default value is equal to the value specified for projecting. The behavior is depicted in Figure 2.

long2lineend value

An optional keyword and positive floating point number that specifies the minimum distance from the target shape line-end to the long side of the an SRAF. The measurement is performed during SRAF cleanup. The long2lineend default value is equal to the value specified for lineend. The behavior is depicted in Figure 3.

corner2sqcorner value

An optional keyword and positive floating point number that specifies the minimum distance from the corner of an SRAF to the corner of a square target shape (non-projecting cases only). All corner-to-corner parameters must be a smaller value than any edge-related parameter. The measurement is performed during SRAF cleanup. The default is the specified minoffset value. The behavior is depicted in Figure 9.

lineend2sqedge value

An optional keyword and positive floating point number that specifies the minimum distance from the line-end of an SRAF to the edge of a square target shape. The measurement is performed during SRAF cleanup. The default is the specified projecting value. The behavior is depicted in Figure 9.

long2sqedge value

An optional keyword and positive floating point number that specifies the minimum distance from the longer edge of an SRAF to the edge of a square target shape. The measurement is performed during SRAF cleanup. The default is the specified projecting value. The behavior is depicted in Figure 9.

sqcorner2corner value

An optional keyword and positive floating point number that specifies the minimum distance from the corner of a square SRAF to the corner of a target shape (non-projecting cases only). All corner-to-corner parameters must be a smaller value than any edge-related parameter. The measurement is performed during SRAF cleanup. The default is the specified minoffset value. The behavior is depicted in Figure 10.

sqcorner2sqcorner value

An optional keyword and positive floating point number that specifies the minimum distance from the corner of a square SRAF to the corner of a square target shape (non-projecting cases only). All corner-to-corner parameters must be a smaller value than any edge-related parameter. The measurement is performed during SRAF cleanup. The default is the specified minoffset value. The behavior is depicted in Figure 9.

sqedge2lineend value

An optional keyword and positive floating point number that specifies the minimum distance from the edge of a square SRAF to the line-end of a target shape. The measurement is performed during SRAF cleanup. The default is the specified projecting value. The behavior is depicted in Figure 10.

sqedge2long value

An optional keyword and positive floating point number that specifies the minimum distance from the edge of a square SRAF to the longer edge of a target shape. The measurement is performed during SRAF cleanup. The default is the specified projecting value. The behavior is depicted in Figure 10.

sqedge2sqedge value

An optional keyword and positive floating point number that specifies the minimum distance from the edge of a square SRAF to the edge of a square target shape. The measurement is performed during SRAF cleanup. The default is the specified minoffset value. The behavior is depicted in Figure 9.

Note:

MRC parameters such as lineend2long, long2lineend, square2long, etc., must be specified by priority for minfeaturespace. Do not define these conditional parameters globally. Although parsed as valid, global specification of these parameters degrades performance.

Description

An optional keyword that specifies the minimum distance SRAFs are placed from target shapes. You can specify minfeaturespace by priority or globally. All global keywords must be specified first.

Line-End Descriptions

Line-end criteria are specified by the minfeaturespace lineend keyword. Line-ends can be varied depending on their user definition. The line-end length, line-end leg length and adjacent vertices all become criteria for the selection of line-ends for SRAF generation.

The following criteria applies to the various line-end forms depicted in Figure 1, where green line-ends are valid and red line-ends are invalid. The default dimension is minoffset. The line-end includes specific criteria:

Convex Corners

An edge series having only two convex corners. The edges labeled 1 do not have two convex corners and are invalid line-ends.

Line-End Length

The line-end must have a length between the constraints specified by the leminlength and lemaxlength keywords. The edge labeled 2 exceeds lemaxlength and is an invalid line-end.

Line-End Legs

The line-end must have two adjacent perpendicular edges (legs), with both lengths between the constraints specified by the leminadjacent and lemaxadjacent keywords. Other perpendicular edges within the distance of the specified leg length invalidates the line-end. The edges labeled 3 do not exceed leminadjacent for both legs and are invalid line-ends.

Serpentine Shapes

External edges that are part of any serpentine or U-shaped target, or those of rectangles with holes are invalid line-ends. The edge labeled 4 exceeds lemaxlength and is an invalid line-end.

Conditional MRC

Conditional cleanup enforces MRCs that act conditionally for minfeaturespace. The conditional enforcement of MRCs are applied to a subset of tagged target edges and target shapes defined by specified values for a given template. The conditional values override global minfeaturespace values, normally enforced for all target shapes. The condition applies to the target shapes that are found near local SRAFs. During cleanup, the MRC values are applied to a pair of node and contender SRAFs. Where contending SRAFs interact with the tag or meet the specified constraint, the conditional values in the priority statement are enforced.

This code snippet shows the same priority using two separate conditional methods, one using tags of wide width shapes, shown in red, and the other using tags of standard width shapes, shown in green:

newtag wide internal > 0.1
newtag std internal <= 0.1
minfeaturespace 0.030 projecting 0.050
minoffset 0.002 projecting .040
bmaxconflict 0.01 centerthresh 0.15 sbwidth 0.03 sraf 0.05
priority 1 tag wide
	minfeaturespace 0.008 projecting 0.08 lineend 0.07 lineend2long 0.04
priority 1 tag std
	minfeaturespace 0.01 projecting 0.08 lineend 0.07 lineend2long 0.04

Examples

Example 1 - lineend2long

For lineend2long, the parameters define the protection at the line-end of the SRAF and the long edge of the contender during SRAF cleanup. This code:

priority 2 minoffset 0.01 lineend2long 0.035

implies these defaults:

minoffset 0.01 projecting 0.01 lineend2long 0.035 lineend 0.01 long2lineend 0.01
