offsetlayer

Layer Keywords

Specifies a layer name representing areas within which to prohibit positive and negative SRAF generation. Used by RET SBAR and RET MBSRAF.

Usage

offsetlayer name

Arguments

name

Specifies the name of the layer.

Description

An optional keyword and layer name that contains preexisting polygons or edges that define areas where positive and negative scattering bars cannot be generated. SRAFs can abut but not overlap offsetlayer shapes.

The offsetlayer keyword should only be used as an alternative when the lineend parameters provided by minfeaturespace do not provide sufficient keepout areas. To avoid processing redundancies, the target layer and taglayer should never be used as the offsetlayer. Instead, the offsetlayer can always be derived with Calibre SVRF commands and then input to setlayer cnsraf. Only one offsetlayer may be specified. Only manhattan- and 45-degree input layer geometries are supported. Additionally, no skew-edged geometries are valid offsetlayer shapes for SRAFs.

If offsetlayer is specified, it must appear in either the sraf_options statement or the setlayer command to avoid duplication. If the offsetlayer is required in other areas of the recipe, you must derive it again under a different name prior to processing it.

The minoffsetlayerspace keyword may be specified per priority or globally to check the offsetlayer.

Note:

As of release 2013.4 and later, SRAFs may abut the offsetlayer shape. In past releases, all SRAFs were kept at a spacing equal to minfeaturespace from the offsetlayer. To model previous SRAF-offsetlayer spacing behavior, use minoffsetlayerspace to enforce this space.

Examples

The code shown here demonstrates manually-derived keepout areas that prohibit placement of SRAFs. After derivation of the line-end keepout area, the layer is then input to the cnsraf and RET SBAR commands. When the offsetlayer keyword and the lineend debug keyword in the cnsraf debug command are specified, the offsetlayer output is merged into the lineend output layer. See “setlayer cnsraf debug” for more information.

offset_edge = CONVEX EDGE target 
	ANGLE1 == 90 LENGTH1 > 0.100 //leg1 gt value ( leminadjacent) 
	ANGLE2 == 90 LENGTH2 > 0.100 //leg2 gt value (leminadjacent) 
	WITH LENGTH >= 0.090 < 0.700 //le range (leminlength, lemaxlength) 
offset_layer = EXPAND EDGE offset_edge 
	EXTEND BY 0.030 //le side extension (leprojectingside) 
	OUTSIDE BY 0.2  //le outward extension (lineend) 
	INSIDE BY 0.1   //le inward projection (leprojectingin) 
LITHO FILE cn_setup_1 [ 
	layer target 
	layer offset_layer 
	setlayer cnsb1 = cnsraf target offsetlayer offset_layer ... \ 
] 
cnsb1 = RET SBAR target offset_layer FILE cn_setup_1 MAP cnsb1
