flare_longrange

LITHO FILE Statement and Commands

Associates a precomputed long range flare convolution with a flare model contained inside a litho model. Used by RET EUV MBSRAF only.

Usage

flare_longrange litho_model_dir
{none | constant value | png file_name constant {average | minimum | maximum}}

Description

The flare_longrange command specifies how intra- and inter-field flare effects are handled. This version of flare_longrange is a reduced-feature set of the command and is tailored for RET EUV MBSRAF.

Arguments

litho_model_dir

An argument that specifies the name of a litho model directory that contains a flare model with long range flare. The model name must either be found in the modelpath or be a fully qualified path.

none

An argument that specifies that no pre-computed long range convolution is used when performing simulations with the specified model.

constant value | png file_name constant {average | minimum | maximum}

An argument set that specifies where the result of a pre-computed convolution of a fractal kernel (long range part) present in the loaded model and the currently processed layout is stored.

  • constant value — The long range flare is a constant percentage of clear field transmission given by value, where a value of 1 is equal to 100%. The intensity is calculated as:

    I_constant_flare = kf * Iclear * value

    where kf is the coefficient from the flare model, and Iclear is the clear field value taking into account both optics and ddm mask transmission.

  • png file_name constant {average | minimum | maximum} — Make sure your Calibre session has access to the .png file. This file is generated using RET FLARE CONVOLVE. The file may be prefaced by a full or relative path, or found in the modelpath.

    If the constant parameter is specified, the long range flare is represented as a constant value that is a property of the long range flare map, and the constant flare value is used for the entire simulated region, independent of the original flare map extent. Currently supported properties are:

    • average — uses the average value in the flare map

    • minimum — uses the smallest value anywhere in the flare map

    • maximum — uses the largest value anywhere in the flare map

    For more information, see flare_longrange in the Calibre OPCVerify User’s and Reference Manual.
