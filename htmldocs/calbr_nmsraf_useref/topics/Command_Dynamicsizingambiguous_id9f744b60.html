dynamicsizing_ambiguous

Cleanup Keywords

Manages the sizing of SRAFs near tile boundaries during generation. Used by RET SBAR and RET MBSRAF.

Usage

dynamicsizing_ambiguous 0 | 1

Arguments

0 | 1

A positive floating point number specifying the minimum length of the trimmed SRAF. The default setting is 0.

Description

An optional keyword used globally or by priority to resize SRAFs according to potentially uncertain local tile boundary conditions and the minimum SRAF length you specify by its value (you cannot specify dynamicsizing_ambiguous within templates). With cnsraf, candidate SRAFs are sized with respect to other SRAFs until they are conflict-free or are removed. If you use dynamicsizing_ambiguous globally, all applicable templates perform dynamicsizing as specified. When you use dynamicsizing_ambiguous by priority, all applicable templates must specify that priority to perform dynamicsizing as specified.

Square target shapes are sized down uniformly. Rectangular target shapes are reduced in size along the long axis from either end as required. Refer to cleanmode postsizing for more information regarding MRC-clean generation of SRAFs when using dynamicsizing.

Note:

To use dynamicsizing_ambiguous with cleanupversion 1, dynamicsizing_ambiguous must be set to 1.

Note:

The dynamicsizing_ambiguous and conflictonly keywords cannot be used in the same priority.
