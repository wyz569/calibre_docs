cleanmode

Cleanup Keywords

Specifies the method of SRAF removal. Used by RET SBAR only.

Usage

cleanmode [immediate | postsizing]

Arguments

immediate

Removes all SRAFs that violate MRC with input layers. For recipes specifying cleanupversion 0, this is the default behavior.

postsizing

For SRAFs generated using dynamicsizing, postsizing allows SRAFs that violate MRC constraints to be resized prior to cleanup instead of being removed. For recipes specifying cleanupversion 1, this is the default behavior.

Description

An optional keyword that specifies the method in which SRAFs violating MRC are removed. This keyword can be set globally for all setlayer cnsraf templates, and it can be assigned on a priority basis using the priority keyword. Setting cleanmode on a priority basis overrides the global assignment.

Tip

Siemens EDA recommends specifying cleanupversion 1 for all new recipes, which invokes cleanmode postsizing by default, also recommended by Siemens EDA. This mode allows the SRAFs violating MRC rules to be dynamically sized instead of immediately removed.

Examples

The illustration shows the output for cleanmode postsizing and also the output for cleanmode immediate. The immediate option removes any SRAFs that violate MRC.
