rectedge

Templates

Places rectangular SRAFs offset from each edge of the target shape. Used by RET SBAR only.

Usage

rectedge
[tag name ...]
{length value | extension value [extension45 value [width45 value]]}
{center [value] [spacelimit value] |
    offset value |
   {offsetx value offsety value}}
width value
[angledalso]
[conflictonly]
[strictcenter {0 | 1}]
[prior value]

Arguments

tag name ...

An optional keyword specifying one or more tag names used in the template. The tag name identifies a group of edges or shapes to be used with the template. If specified, tags must appear as the first arguments in the template.

length value

One of two required keywords (length or extension) that must specify either the length or the extension of rectangular SRAFs, respectively.

The length keyword determines the length of rectangular SRAFs.

The length keyword is mutually exclusive with the center keyword.

extension value [extension45 value [width45 value]]

One of two required keywords (length or extension) that must specify either the length or the extension of rectangular SRAFs, respectively.

  • value

    The extension keyword and value pair adds to, or subtracts from, the length of rectangular SRAF ends based on the original target edge. Negative extensions that reduce SRAF length are valid if the specified extension is less than half of the target edge length and the resulting SRAF does not violate MRC constraints.

  • extension45 value

    The optional extension45 keyword and value pair controls the extension of SRAFs that are 45 degrees relative to target shapes. To avoid SRAF removal, the minimum value of extension45 must be greater than or equal to width45/sqrt(2). For example, where width45 equals 20 nm:

    extension45 length = 20/sqrt(2) = 14.14 nm

    For additional control of 45-degree SRAF extensions, use the minareaextension and minlengthextension keywords.

  • width45 value

    The optional width45 keyword and value pair controls the widths of those SRAFs that are 45 degrees relative to target shapes for extension45. The width45 option defaults to width. The width45 keyword adheres to upper and lower bounding values, between width/sqrt(2) and width*sqrt(2). For example, where width equals 18 nm:

    width45 range = [12.73, 25.4]

Note:

Regarding extension45 keyword and cleanupversion interactions:

  • Siemens EDA does not yet recommend use of intersection J or lineendmerge with the extension45 option. Unexpected results may occur.
  • The rectedge extension45 keyword is incompatible with cleanupversion 0 and results in a compile-time error.
center [value] [spacelimit value]

One of three mutually exclusive, required keyword or keyword pairs (center, or offset, or offsetx and offsety) that must be specified for the distance SRAFs are placed from the edge of a target shape.

The center keyword forces the output SRAFs to be placed equidistant from both qualifying edges provided by tags or the target shape edges. See Figure 3. If the tags input to rectedge do not provide a pair of qualifying edges, no SRAFs are output when specifying the center keyword.

The center and length keywords are mutually exclusive.

  • value

    A center value may be optionally specified as a positive floating-point number, specifying the spacing from the centerline between facing target edges to the inner edges of the centered pair of parallel SRAFs that are generated. For more information, see Figure 4.

  • spacelimit value

    An optional keyword specified to establish the upper limit for center SRAF generation between one tagged edge and one untagged edge. The spacelimit keyword is compatible only with the euclidean metric.

    Note:

    Excessively large spacelimit values trigger larger edge search areas and significantly increase run time.

CAUTION:

Regarding center keyword and cleanupversion interactions:

  • The rectedge center keyword and its options are incompatible with cleanupversion 0 and result in a compile-time error.
  • The spacelimit option is compatible only with the euclidean metric. If other metrics are required, use external to derive tags.
  • If cleanupversion 1 is specified, any external or internal newtag upper boundary specified overrides spacelimit, irrespective of the values specified.
  • An upper boundary must be specified for newtag internal when implementing it for rectedge center applications.

For cleanupversion 1 shown in the example, the upper limits, 0.5 and 0.7, both override spacelimit which is 0.8, and the largest upper limit, 0.7, overrides all other tag upper limits in the rectedge template. The spacelimit specification never overrides any newtag upper limit:

cleanupversion 1
newtag tag1 external <= 0.5
newtag tag2 external <= 0.7
rectedge tag tag1 tag tag2 center ... spacelimit 0.8

For cleanupversion 1, the spacelimit specification of 0.8 remains in effect for any newtag lower limits, which are 0.15 and 0.09:

cleanupversion 1
newtag tag1 external > 0.15
newtag tag2 external > 0.09
rectedge tag tag1 tag tag2 center ... spacelimit 0.8

Positive SRAF generation using the center option requires specifying newtag of external type for selection of the outside edges of target shapes for correct SRAF placement. Negative SRAF generation using the center option requires specifying newtag of internal type for selection of the inside edges of target shapes for correct SRAF placement.

offset value

One of three mutually exclusive, required keyword or keyword pairs (center, or offset, or offsetx and offsety) that must be specified for the distance SRAFs are placed from the edge of a target shape.

The offset keyword specifies the minimum orthogonal distance from both horizontal and vertical edges of a target shape. All values specified must be positive floating point values.

offsetx value offsety value

One of three mutually exclusive, required keyword or keyword pairs (center, or offset, or offsetx and offsety) that must be specified for the distance SRAFs are placed from the edge of a target shape.

The offsetx and offsety keywords specify the minimum orthogonal distance from the horizontal and vertical edges of a target shape, respectively, from the midpoint of the target edge to the midpoint of the nearest SRAF edge. All values specified must be positive floating point values.

width value

A required keyword that specifies the width of the SRAF.

angledalso

An optional keyword that additionally generates SRAFs at 45-degree target edges for the rectedge template.

conflictonly

An optional keyword that outputs SRAFs within conflict regions only. No other SRAFs of this template are generated. Conflict regions are formed where neighboring SRAFs generated by this template violate MRC rules you specify in the template. The conflictonly argument and dynamicsizing keyword cannot be used within the same priority. Siemens EDA recommends using the conflictonly argument only when conflict resolution shapes are relatively small and not for larger shapes such as line layer designs.

strictcenter {0 | 1}

An optional keyword controlling center SRAF placement between tagged edges. Specifying strictcenter 1 allows SRAFs to be generated only between tagged edges. Specifying strictcenter 0 ignores tags and allows SRAFs to be generated between all target edges, and also permits center SRAFs to be placed with reference to a single tagged edge and a facing target edge.

prior value

An optional keyword that specifies a numeric identifier for placed SRAFs. Identifiers are applied sequentially, so a value of 1 is applied first, then 2 is applied, and so on. The default priority is 1. See “priority” for more information.

Description

An optional template and argument set that places rectangular SRAFs at an offset from each edge of the target or tagged edges.

Note:

Regarding cleanupversion interactions:

  • Siemens EDA recommends always specifying cleanupversion 1 for new recipes.
  • The rectedge center and extension45 keywords are incompatible with cleanupversion 0.
  • Using the rectedge template with cleanupversion 0 may result in SRAF output differences from recipes using cleanupversion 1. Using cleanupversion 1 implements enhanced algorithms performing more robust classification and identification of tagged edges.
