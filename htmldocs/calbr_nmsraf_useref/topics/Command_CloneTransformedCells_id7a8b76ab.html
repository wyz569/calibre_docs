clone_transformed_cells

LITHO FILE Statement and Commands

Copies cells with nonzero transforms so that simulation of asymmetric optical models is correct. Used by RET MBSRAF only.

Usage

clone_transformed_cells {decide | no | yes}

Arguments

decide

The optical model is checked to decide if cell cloning is activated. More specifically, this option causes the Calibre engine to check if any asymmetric sources are present in the optical model. If so, the Calibre database constructor is forced to clone cells that have rotated or mirrored placements, so that hierarchical simulation of the rotated structures is correct. This is the default setting.

Note:

Siemens EDA recommends setting clone_transformed_cells to decide.

no

Override source shape and force it to be considered symmetric (do not clone).

yes

Override source shape and force it to be considered asymmetric (force cloning).

Description

When using an asymmetrical source, cell orientation can have a significant impact on results. You can control how the simulations handle cell orientation when calculating the aerial image for a hierarchical design. The application can:

  • Calculate the aerial image for the original cell, then adjust the orientation of the image to match each cell.

  • Calculate a separate aerial image for each orientation in the cell. To do this, the application creates one clone of the cell for each orientation represented in the design and transforms the clone as necessary to match the design, and calculates a separate aerial image for each clone.

Examples

This example enables optical model checking for cell cloning:

clone_transformed_cells decide
