lineendmerge

Cleanup Keywords

Merges SRAF line-ends during SRAF cleanup. Used by RET SBAR and RET MBSRAF.

Usage

lineendmerge {value | no | 0}

Arguments

value

A positive floating point number establishing a limit for line-end merging.

no

Disables all line-end merging. Existing overlapping SRAFs are separated. Use of lineendmerge no is incompatible with the interpriorityalign, intrapriorityalign, and intersection keywords. The no option is incompatible with cleanupversion 0 and results in a compile-time error.

0

Allows line-end merging only of field (preliminary) SRAFs that align or overlap. Use of lineendmerge 0 is incompatible with the interpriorityalign, intrapriorityalign, and intersection keywords. The 0 option is incompatible with cleanupversion 0 and results in a compile-time error.

Description

An optional keyword used to merge SRAFs at line-ends. The keyword can be specified globally or by priority. Conditions and behaviors include:

  • SRAFs must be exactly aligned, so that no jogs result after merging.

  • Distances between two SRAFs are less than or equal to the specified value. There is no default value for lineendmerge.

  • If the recipe specifies cleanupversion 0, dynamicsizing may need to be used to achieve intended results of lineendmerge in some cases.

Note:

When lineendmerge is specified globally, line-ends of different priorities may be merged. Therefore, Siemens EDA recommends only specifying lineendmerge by priority.

The intrapriorityalign and interpriorityalign keywords cannot be used with lineendmerge 0 or lineendmerge no. The lineendmerge keyword requires a floating-point value to be used with intrapriorityalign and interpriorityalign. If selective SRAF merging is required using tags, Siemens EDA recommends using unique tags and templates for those cases and setting lineendmerge by priority to continue using intrapriorityalign and interpriorityalign keywords globally.
