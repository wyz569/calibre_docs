priority

Priority Keywords

Overrides global parameters by priority as specified by each template. Used by RET SBAR and RET MBSRAF.

Usage

For RET MBSRAF:

priority number [to number] [with number [to number]]
[conditional_cleanup_sets]
[dynamicsizing value]
[interpriorityalign value]
[intersection J [minjogwidth value]]
[intersectionnegative {J | N} [minjogwidth value]]
[intrapriorityalign value]
[lineendmerge value]
[minarea value]
[minfeaturespace value [lineend value] [projecting value]]
[minlength value]
[minoffset value [lineend value] [projecting value]]
[minoffsetlayerspace value]
[minsquarearea value]
[minsquarelength value]
[negative]
[shielding {no | seedpixel | distance}]

For RET SBAR:

priority number [to number] [with number [to number]]
[conditional_cleanup_sets]
[centermerge value]
[centerthresh value [sbwidth value]]
[cleanmode {immediate | postsizing}]
[conflictmode {merge | nomerge | shift}]
[dynamicsizing value]
[interpriorityalign value]
[intersection J [minjogwidth value]]
[intersectionnegative {J | N} [minjogwidth value]]
[intrapriorityalign value]
[lineendmerge value]
[maxconflict value]
[minarea value]
[minfeaturespace value [lineend value] [projecting value]]
[minlength value]
[minoffset value [lineend value] [projecting value]]
[minoffsetlayerspace value]
[minsquarearea value]
[minsquarelength value]
[negative]
[oversizesraf {no | yes}]
[sbwidth value]
[sraf value]

Arguments

number [to number]

The first number argument specifies the priority of SRAFs to which all subsequent keywords are applied, and is required. An optional second number following the ‘to’ keyword is a lower priority limit establishing a range of priorities to which all subsequent keywords are applied. The priorities are always applied sequentially: a priority of 1 is applied first, then a priority of 2 is applied. The default priority is 1.

with number [to number]

Optional keywords and respective values specifying a single priority or a range of priorities for conflicts between SRAFs. The first number argument specifies the priority of SRAFs to which all subsequent keywords are applied, and is required. An optional second number following the ‘to’ keyword is a lower priority limit establishing a range of priorities to which all subsequent keywords are applied. The specified keywords following ‘with’ are applied only to SRAFs of the specified priority. The priority specified for ‘with’ must be a lower number than those specified for ‘to’. SRAF cleanup is performed from the lowest priority number sequentially through the highest priority number. When cleanup has been performed for any priority P, those MRC-clean SRAFs for priority 1 through P-1 remain unmodified. SRAFs of priorities P+1 and upwards are not considered during cleanup for priority P.

conditional_cleanup_sets

One of eight optional keyword sets specifying a constraint and range or name, and the MRC keyword and value pair with optional modifier keywords and values. The sets must be specified with ordered keywords:

constraint argument MRC value modifier value

  • constraint argument

    One of eight optional keyword and argument pairs:

    • area range

      A constraint measuring the area of the contending SRAF. The range may be bounded or unbounded.

    • aspect range

      A constraint measuring the aspect ratio of the contending SRAF. The range may be bounded or unbounded.

    • commonrunlength range

      A constraint measuring the run length of a continuous edge of the contending SRAF. The range may be bounded or unbounded.

    • length range

      A constraint measuring the length of the contending SRAF. The range may be bounded or unbounded.

    • squarearea range

      A constraint measuring the area of any square contending SRAF. The range may be bounded or unbounded.

    • squarelength range

      A constraint measuring the length of an edge of any square contending SRAF. The range may be bounded or unbounded.

    • tag name

      A constraint specifying the name of tagged edges of the contending SRAF.

    • width range

      A constraint measuring the width of the contending SRAF. The range may be bounded or unbounded.

  • MRC value

    One of four optional MRC keywords and respective values:

    • minfeaturespace value

      An optional MRC determining the minimum space between SRAF and target shapes. The SRAF shape is always the contending shape.

    • minfeaturespace value

      An optional MRC determining the minimum space between SRAF and target shapes. The SRAF shape is always the contending shape.

    • minoffset value

      An optional MRC determining the minimum space between SRAFs shapes.

    • minoffsetlayerspace value

      An optional MRC determining the space between SRAFs and any shape designated by offsetlayer. The SRAF shape is always the contending shape.

    • seedoffset value

      An optional MRC determining the minimum space between SRAF seeds and target shapes. The SRAF seed is always the contending shape. Used only by mbSRAF.

  • modifier value

    An optional modifier and its respective value specified after the MRC keyword. See minoffset for comprehensive definitions of all modifiers:

    • corner2sqcorner value

    • lineend value

    • lineend2long value

    • lineend2sqedge value

    • long2lineend value

    • long2sqedge value

    • projecting value

    • sqcorner2corner value

    • sqcorner2sqcorner value

    • sqedge2lineend value

    • sqedge2long value

    • sqedge2sqedge value

Conditional cleanup sets specified within the priority keyword provide great flexibility for SRAF cleanup. Combinations of constraints for a single priority as seen here provide complex cleanup methods:

priority 10 tag cover minfeaturespace 0.010
priority 10 tag cover minoffset 0.060
priority 11 width > 0.1 tag cover minoffset 0.060
priority 12 width > 0.1 minoffset 0.040
priority 13 tag cover width > 0.1 minoffset 0.040 length < 0.2
	tag cover minfeaturespace 0.050
centermerge value

An optional keyword that specifies the maximum distance within which to merge parallel SRAFs. For more information, see centermerge in the sraf_options statement.

centerthresh value [sbwidth value]

An optional keyword that specifies the minimum length of a conflict region where an SRAF is added. For more information, see centerthresh in the sraf_options statement.

cleanmode [immediate | postsizing]

An optional keyword that specifies the method by which SRAFs violating MRC are removed. For more information, see cleanmode in the sraf_options statement.

conflictmode [merge | nomerge | shift]

An optional keyword that specifies the method in which conflict regions are processed. For more information, see conflictmode in the sraf_options statement.

dynamicsizing value

An optional keyword used globally or by priority to resize SRAFs according to local conditions and the minimum SRAF length specified by the value. You may not specify dynamicsizing within a template. The dynamicsizing keyword and conflictonly argument cannot be used within the same priority. For more information, see dynamicsizing in the sraf_options statement.

interpriorityalign value

An optional keyword specified only by priority to align and optionally merge SRAFs of different priorities during SRAF cleanup. To simultaneously permit merging, you must specify the lineendmerge option. When you specify the interpriorityalign keyword for a given priority, all SRAFs of that priority are aligned with SRAFs of higher priorities only. For more information, see interpriorityalign in the sraf_options statement.

intersection J [minjogwidth value]

An optional keyword that enables valid, generated SRAFs to be joined at jog point intersections. The intersection J keyword can be used with either positive or negative SRAFs. The default jog width (established by the template width keyword) can be overridden with minjogwidth. For more information, see intersection in the sraf_options statement.

intersectionnegative {J | N} [minjogwidth value]

An optional keyword that enables valid, generated negative SRAFs to be joined at jog point intersections. The intersectionnegative keyword is used only for negative SRAF generation and should only be set globally. The default jog width (established by the template width keyword) can be overridden with minjogwidth. For more information, see intersectionnegative in the sraf_options statement.

intrapriorityalign value

An optional keyword specified only by priority to align and optionally merge SRAFs of same priorities during SRAF cleanup. To simultaneously permit merging, you must specify the lineendmerge option. When you specify the intrapriorityalign keyword for a given priority, all SRAFs of that priority are aligned with SRAFs of the same priorities only. For more information, see intrapriorityalign in the sraf_options statement.

lineendmerge value

An optional keyword used to merge two SRAFs at line-ends that are exactly aligned and are less than the lineendmerge value specified. For more information, see lineendmerge in the sraf_options statement.

maxconflict value

An optional keyword that specifies the limit on the size of a conflict region to process. For more information, see maxconflict in the sraf_options statement.

minarea value

An optional keyword that specifies the minimum area allowed for any SRAF. SRAFs with an area smaller than minarea are eliminated. For more information, see minarea in the sraf_options statement.

minfeaturespace value [lineend value] [projecting value]

An optional keyword that specifies the minimum distance an SRAF is placed from an input layer target shape. For more information, see minfeaturespace in the sraf_options statement.

minlength value

An optional keyword that specifies the minimum length allowed for rectangular SRAFs. SRAFs with any side of a length smaller than minlength are eliminated during cleanup. The minlength keyword does not apply to square SRAFs. For more information, see minlength in the sraf_options statement.

minoffset value [lineend value] [projecting value]

An optional keyword that specifies the minimum distance allowed between SRAFs. Any SRAFs that fall within this distance are treated as conflicts and are processed according to keywords listed in “Cleanup Keywords”. For more information, see minoffset in the sraf_options statement.

minoffsetlayerspace value

An optional keyword that specifies the minimum distance from any SRAF shape to the offsetlayer. You derive the offsetlayer for additional line-end keepout areas. For more information, see minoffsetlayerspace in the sraf_options statement.

minsquarearea value

An optional keyword that specifies the minimum area allowed for any square SRAF in square microns. Square SRAFs with an area smaller than minsquarearea are eliminated. If minsquarearea is not specified, the minimum area allowed for square SRAFs defaults to minarea. For more information, see minsquarearea in the sraf_options statement.

minsquarelength value

An optional keyword that specifies the minimum length allowed for any square SRAF. Square SRAFs with a length smaller than minsquarelength are deleted. The minsquarelength keyword does not apply to non-square SRAFs. For more information, see minsquarelength in the sraf_options statement.

negative

An optional keyword that specifies the generation of negative SRAFs or slots. Using the negative keyword as a priority allows cnsraf to generate positive and negative SRAFs in a single run. For more information, see negative in the sraf_options statement.

oversizesraf [no | yes]

An optional keyword that allows SRAFs in conflict regions to be larger in size than the conflict region itself. By default, this is set to no, cropping SRAFs to the size of the conflict region. You can use oversizesraf in conjunction with the sraf keyword, or you can use oversizesraf separately by priority. For more information, see oversizesraf in the sraf_options statement.

sbwidth value

An optional keyword that specifies the width of the secondary scattering bars. You can use sbwidth in conjunction with centerthresh, or you can use sbwidth separately by priority. For more information, see centerthresh in the sraf_options statement.

sraf value

An optional keyword that specifies the square size of the SRAFs added in the conflict region. For more information, see sraf in the sraf_options statement.

shielding {distance | no | seedpixel}

An optional keyword that specifies shielding of tagged edges to alter SRAF generation relative only to the tagged edges. Shielding may be specified globally or by priority. Shielding is compatible with positive and negative SRAF generation. For more information, see shielding in the sraf_options statement.

Description

An optional keyword used to override conflict resolution and MRC parameters specified globally. The priority keyword must be followed by number and any additional keywords.

If the prior keyword is not specified in the template, the global keyword and assigned parameter value is used. For complete information regarding these keywords, see their description in the sraf_options statement.

Note:

When using priority-based keywords, global keywords and their respective options must be specified first, otherwise Calibre nmSRAF issues warnings and, possibly, compile-time error messages. Calibre assumes any keyword specified after the priority keyword to be priority based. For any parameter to be considered global, it must be specified before the priority keyword.

Examples

The following examples demonstrate priority usage in context.

Establishes negative SRAF generation for priority 34:

rectEdge tag intEdges extension 0.08 offset 0.17 width 0.07  prior 34
priority 34 negative

Applies oversized SRAFs and a minimum space from the offset layer to SRAFs generated by priorities 3 through 5:

rectCorner offsetx 0.16 offsety 0.16 length 0.07 width 0.02 prior 3
rectCorner offsetx 0.18 offsety 0.18 length 0.07 width 0.02 prior 4
rectCorner offsetx 0.20 offsety 0.20 length 0.07 width 0.02 prior 5
priority 3 to 5 oversizesraf yes minoffsetlayerspace 0.057
