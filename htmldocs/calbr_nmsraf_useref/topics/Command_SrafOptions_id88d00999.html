sraf_options

LITHO FILE Statement and Commands

Allows Calibre nmSRAF keywords to be written in free format. The sraf_options statement is called by the setlayer command for SRAF generation. Used by RET SBAR and RET MBSRAF.

Arguments

statement_name

A required argument that is a unique name for the sraf_options statement. This statement is passed by name to the setlayer cnsraf and setlayer mbsraf commands for processing. Currently, there is a 15-character limit on the statement name. For example:

sraf_options template {
	...
}
sraf_options priority {
	...
}
setlayer cn_sb = cnsraf target OPTIONS template OPTIONS priority \
	...
‘{’ sraf_options_keywords ‘}’

Any number of required keywords placed between braces ({ }). The keywords may share the same line or span multiple lines. You may format the keywords within the sraf_options statement with white space. Some keywords are exclusive to either cnsraf or mbsraf. The keyword descriptions within their respective group listings state their application.

Table 1. sraf_options Keyword and Template Groups

Group

Function

Layer Keywords

Calibre nmSRAF requires input layers for SRAF generation.

Tag Keywords

Tags direct templates where to generate SRAFs. The newtag family of keywords select edges and refine locations of SRAF placement.

Constraint Keywords

Specifies dimensional limits of generated SRAFs.

Templates

Specifies the template for SRAF generation.

Style Keywords

Provides management of SRAF output style.

Conflict Keywords

Provides commands for managing conflicts formed during SRAF generation.

Cleanup Keywords

Invokes cleanup after SRAF generation.

Priority Keywords

Specifies keywords by priority.

Description

The sraf_options statement is used to provide keywords to the setlayer cnsraf and setlayer mbsraf commands for SRAF generation. The setlayer commands do not permit use of white space or comments. The sraf_options statement, however, allows a textual free format for organizing, reading, documenting, and maintaining setlayer cnsraf and mbsraf keywords.

You specify the sraf_options statement inside the LITHO FILE statement and a setlayer command with an OPTIONS argument to call the sraf_options statement. The LITHO FILE statement is read by the RET SBAR or RET MBSRAF commands with the FILE argument:

LITHO FILE cn_LF [
	sraf_options cn_statement {
		cleanupversion 1	 ...
	} 
	setlayer cn_out = cnsraf M1 OPTIONS cn_statement
]
sb_out = RET SBAR M1 FILE cn_LF MAP cn_out

The keywords within the sraf_options statement are parsed once to evaluate all variables before runtime. The sraf_options statement supports recursion. This permits you to reference an OPTIONS statement from within the sraf_options statement. Transcripts contain two iterations of the sraf_options keywords: one during parsing, and another with variable substitution completed.

Note:

If the specified values of any keyword are incompatible with each other, Calibre nmSRAF issues a warning message and no values are adjusted. Additionally, a minarea value is not assumed from the product of minwidth and minlength if no minarea specification is made.

Examples

Example 1 — setlayer cnsraf

The following example demonstrates an abbreviated sraf_options statement implementation. The target layer is defined in the LITHO FILE statement and then called by setlayer cnsraf.

Example 1. Simple sraf_options Statement for setlayer cnsraf
LITHO FILE cn_setup_1 [ 
	layer M1 
	tilemicrons 25 
 
	# sraf_options is named cnsraf.opt inside the LITHO FILE statement 
	# The statement is passed to setlayer cnsraf 
 
	sraf_options cnsraf.opt {
		cleanupversion 1
		newtag All_Tag length < 0.170 
		rectedge tag All_Tag extension 0.03 width 0.04 offset 0.06
		maxconflict 0 sraf 0.030 
		centerthresh 0 sbwidth 0.030 
	} 
 
	setlayer cnsb1 = cnsraf M1 OPTIONS cnsraf.opt 
 
	setlayer 1_pre = cnsraf debug cnsb1 preclean 1
]

Example 2 — setlayer cnsraf

This example demonstrates the use of multiple sraf_options statements organized by function and subsequently called by setlayer cnsraf. The sraf_options statements use white space and comments. To prevent confusion between cleanup keywords and templates always order priorities after templates.

Note:

The order of sraf_options statements are arbitrary. The OPTIONS arguments that call the sraf_options statements determine their order.

Example 2. Organizational sraf_options Statement for setlayer cnsraf
LITHO FILE cn_setup_1 [ 
	layer target 

	sraf_options tags { 
		newTag tag1 length > 0.01 
	} 
 	sraf_options general { 
		cleanupversion 1
		minoffset 0.020 
		maxconflict 0.150 
	} 
	sraf_options templates { 
		rectedge tag tag1 extension 0 width 0.03 offset 0.04 prior 1 
		rectedge tag tag1 extension 0.050 width 0.04 offset 0.14 prior 2 
	}
	sraf_options priorities { 
		priority 1 
			# SRAF edge movement 
			dynamicsizing 0.1 
			# SRAF lineend keepouts 
			minfeaturespace 0.040 
				projecting 0.040 
				lineend 0.070 
		priority 2 
			lineendmerge 0.040 
			minfeaturespace 0.040 
				projecting 0.040 
				lineend 0.140 
	}
	setlayer cnsb1 = cnsraf target OPTIONS tags OPTIONS general \
		OPTIONS templates OPTIONS priorities 
]

Example 3 — setlayer mbsraf

The following example demonstrates an abbreviated sraf_options statement setup and implementation. The name assigned to the sraf_options statement, and the name called by setlayer mbsraf are green. The target layer is red, and is defined in the LITHO FILE statement and called by setlayer mbsraf.

Example 3. sraf_options Statement for setlayer mbsraf
LITHO FILE v0sraf.in [
	modelpath /user/demo/IC/20nm/models/exp02
	optical_model_load opt1 e1_f+0.0000 resist_model_load res exp02.mod \
		background 0.0215 -0.0085
	layer sraf_target visible 0.9785 0.0085
	layer cntctopen visible 0.9785 0.0085
	layer sraf_ref_layer visible 0.9785 0.0085
	...
	tilemicrons 25
# sraf_options is named mbsraf.opt inside the LITHO FILE statement
# The statement is passed to setlayer mbsraf

	sraf_options mbsraf.opt {
		correctionlayer cntctopen 
		sblayer sraf_ref_layer 
		optical opt1 dose 1.0 
		resist res
		algorithm 0
		peaktype 4dir 
		localmaxgradient width 0.028
	}
	setlayer srafout = mbsraf sraf_target OPTIONS mbsraf.opt
]
