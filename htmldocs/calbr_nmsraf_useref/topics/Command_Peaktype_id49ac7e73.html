peaktype

Style Keywords

Specifies the method of SRAF propagation. Used by RET MBSRAF only.

Usage

peaktype {1dir | 2dir | 3dir | 4dir}

Arguments

1dir

Propagates SRAFs if the localmaxgradient is found in at least one direction from the seed pixel. The direction can be N-S, E-W, NE-SW, or NW-SE.

2dir

Propagates SRAFs if the localmaxgradient is found in at least two directions from the seed pixel.

3dir

Propagates SRAFs if the localmaxgradient is found in at least three directions from the seed pixel. If algorithm 0 is set, 3dir is the default setting.

4dir

Propagates SRAFs if the localmaxgradient is found in all four directions from the seed pixel.

Note:

Regarding algorithm interactions:

  • If algorithm 0 is set, 3dir is the default peaktype setting.
  • If algorithm 1 is set, peaktype is set internally and must not be set explicitly.

Description

An optional keyword that specifies the method with which SRAFs are propagated.

Qualification of Seed Pixels

The qualification of seed pixels is considered in four directions: vertical (available for display through debug layer seedv), horizontal (seedh), northwest-southeast (seednwse), and northeast-southwest (seednesw).

For arguments 1dir, 2dir, 3dir and 4dir, the four directions are used to determine whether the seed pixel under analysis contains a gradient value peak in one or more directions. If the pixel demonstrates the desired number of directions that locate peaks, that pixel is selected as a seed pixel (available for display through seedpixels debug layer).
