localmaxgradient

Templates

Creates SRAFs around the local maxima of the gradient. Used by RET MBSRAF only.

Usage

localmaxgradient
[tag name ...]
width value
[centerwidth value]
[extension value]
[minwidth value]
[prior value]
[squarewidth value]

Description

A required template and argument set that creates SRAFs around the local maxima of the gradient. All possible SRAF placements are computed. SRAFs are accepted in the order of decreasing integral values, unless an SRAF overlaps an already accepted SRAF with a larger integral value.

Multiple specification of localmaxgradient templates permit generation of increasingly larger SRAFs outwardly through varied width and squarewidth parameters. Larger SRAFs placed at greater distances from the target benefit the resulting image and reduce SRAF printing.

Note:

For recipes requiring placement of varying SRAF widths by multiple localmaxgradient templates, Siemens EDA recommends against using the maxoffset keyword. For more information, see Example 3 — Use of Multiple localmaxgradient Templates and “Calibre cnSRAF Template Execution and Cleanup”.

Arguments

tag name ...

An optional keyword specifying one or more tag names used in the template. The tag name identifies a group of edges or shapes to be used with the template. If specified, tags must appear as the first arguments in the template.

width value

A required keyword (specified in the localmaxgradient template or globally) that specifies the width of rectangle SRAFs that are generated by the localmaxgradient template. If width is not specified globally or in the template, a parser error is output. All localmaxgradient templates of the same priority must specify the same width value if centerwidth is also specified.

centerwidth value

An optional keyword which generates SRAFs equidistant from input target shapes. The centerwidth value must be greater than or equal to the width value. All localmaxgradient templates of the same priority must specify the same centerwidth value.

extension value

An optional keyword that specifies the length of extension added to both ends of the original SRAF.

The extension keyword supports negative extensions as long as the extension is less than half of the SRAF length and the resulting SRAF does not violate MRC constraints.

minwidth value

An optional keyword that specifies the minimum width allowed for a square SRAF generated by the localmaxgradient template.

prior value

An optional keyword that specifies the priority of the SRAFs placed by localmaxgradient.

The priorities are applied sequentially: a priority of 1 is applied first, then a priority of 2 is applied, and so on. The default priority is 1.

squarewidth value

An optional keyword that specifies the width of square SRAFs generated by the localmaxgradient template. The squarewidth keyword may also be specified globally.

If squarewidth and width are specified together, and squarewidth is greater than or equal to width, then a square SRAF with edges of squarewidth is placed around the local maximum pixels in conformance to the cornerpref setting. All localmaxgradient templates of the same priority must specify the same squarewidth value.

As of Calibre release 2014.4, squarewidth is no longer automatically computed.

Examples

Example 1 — Contextual Use of localmaxgradient

LITHO FILE mb_setup [
	...

	sraf_options s_opt {
		localmaxgradient width 0.035 squarewidth 0.040 prior 3
		...
	}
	
	setlayer sb = mbsraf target OPTIONS s_opt optical f0 resist res

]

Example 2 — Use of a Single localmaxgradient Template With maxoffset

# Assignment of a maximum SRAF offset for a single template
# generating multiple SRAF rings.

maxoffset 0.250
localmaxgradient width 0.016 squarewidth 0.023

Example 3 — Use of Multiple localmaxgradient Templates

# Assignment of priority for templates with different widths.

localmaxgradient width 0.016 squarewidth 0.023 prior 1
localmaxgradient width 0.020 squarewidth 0.025 prior 2
