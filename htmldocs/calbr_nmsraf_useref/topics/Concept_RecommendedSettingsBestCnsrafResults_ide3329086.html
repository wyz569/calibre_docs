Recommended Settings for Best cnsraf Results

Recommended settings improve results produced by the cnsraf command.
Table 1. Recommended Settings for Best cnsraf Results

Option

Recommendation/Notes

minareaextension minlengthextension rectedge

For additional control of 45-degree SRAF extensions with the rectedge template, use minareaextension and minlengthextension keywords.

minoffset minoffsetlayerspace minfeaturespace

Conditional MRC parameters such as lineend2long, long2lineend, square2long, etc., must be specified by priority only for these keywords. Do not define conditional parameters globally. Although parsed as valid, global specification of these parameters degrades performance.

minoffset minoffsetlayerspace minfeaturespace

Siemens EDA recommends always setting cleanupversion 1 when using the square-oriented keyword options (long2sqedge, corner2sqcorner, etc.) for minoffset, minoffsetlayerspace, and minfeaturespace, or square2square keywords.

interpriorityalign intrapriorityalign

Siemens EDA recommends always setting interpriorityalign and intrapriorityalign per priority. If these keywords are set globally, a warning is issued to the transcript.

lineendmerge

Siemens EDA recommends using the lineendmerge keyword in conjunction with the intersection and intersectionnegative keywords to provide more continuous SRAF output. If no lineendmerge parameter is explicitly set, then lineendmerge is automatically set and its resulting value reported to the transcript. When lineendmerge is specified globally, line-ends of different priorities may be merged. Therefore, Siemens EDA recommends only specifying lineendmerge by priority.

algorithm

Siemens EDA recommends setting algorithm to 1 for new recipe development.

cleanupversion

Siemens EDA recommends setting cleanupversion 1 for new recipe development only, and always used in conjunction with cleanmode postsizing to avoid excessive field SRAF and consequent runtime impact during cleaning. Using cleanupversion 1 provides best results for line and space layers, and 2D structures. For existing recipes using Calibre versions prior to 2016.3, if cleanupversion 1 is specified, also specify cleanmode immediate globally. This setting provides more precise MRC enforcement, invokes promotion by default, and improves SRAF coverage for line layers and 2D structures.

minwidth

Siemens EDA recommends using the minwidth keyword to avoid the possibility of outputting SRAFs of a size less than the minimum MRC constraint.

cleanmode

Siemens EDA recommends setting cleanupversion 1 and using cleanmode postsizing when specifying dynamicsizing. This allows SRAFs that are violating MRC constraints to be sized down instead of being immediately removed.

maxconflict

Setting maxconflict too large (greater than 0.5 um) leads to large interaction distances during hierarchical processing. A large maxconflict value also reduces control over the SRAF generated through conflict resolution and rarely provides the desired match to Calibre pxOPC. Siemens EDA recommends keeping the maxconflict setting in the template specification and create additional templates if necessary. Use the priority override feature to set maxconflict values for subsequent templates.

maxlength

Use this keyword as advised. Results at tile boundaries may vary, but this is a cost-effective way to reduce SRAF printing. If printing is of minor concern, Siemens EDA recommends not using this keyword.

minfeaturespace

Set this sufficiently large so that no primary or secondary SRAFs are created too close to the contacts. The primary minfeaturespace should be smaller than the secondary minfeaturespace to help enforce their respective placement with neighboring target shapes.

newtag external and internal, and newtag recta and rectb

Use the external keyword to isolate edges of specific spaces when using the rectedge template. For recta and rectb, use smaller values to avoid flattening data and producing inconsistent results.

rectedge

If you are using the rectedge template that generates long assist features, specify dynamicsizing within the template’s priority to improve coverage. The rectedge template can generate SRAFs that are longer than the interaction distance (the SRAFs can also be longer than the tilesize).

If dynamicsizing is not used, then a conflict at one end of the bar can cause the entire assist feature to be removed. Use dynamicsizing to allow SRAFs to be resized so they can be preserved.

For more efficient rectedge template application, use the external newtag keyword to specify dimensional conditions for scattering bar generation.

tilemicrons

The selection of tile size is critical for optimum processing. If the tile size is too large or too small, primary or remote memory consumption may go into swap and slow performance. The optimum tile size settings found by Siemens EDA correlate to the technology node and balance the size of data per tile. It is recommended that you set tilemicrons to the values based on the node listed in Table 1.
