cornerprefnegative

Style Keywords

Specifies the negative SRAF generation method at target shape corners. Used by RET MBSRAF only.

Usage

cornerprefnegative {all | horizontal | no | square | squareonly | squareonlyscore | vertical}

Arguments

all

Generates rectangular SRAFs inside target shapes and aligned to all target edge axes. Rectangular SRAFs take priority over square SRAFs with this argument.

horizontal

Generates rectangular SRAFs inside target shapes and aligned to horizontal target edge axes.

no

This method inhibits SRAF generation inside target shapes at corners. If algorithm 1 is set, no is the default setting.

square

Generates a square SRAF at seed pixels inside target shapes near their corners. As the localmaxgradient template determines, square SRAFs may be changed to rectangular SRAFs, given necessary alignment criteria. If algorithm 0 is set, square is the default setting.

squareonly

If localmaxgradient keyword squarewidth is defined, squareonly generates square SRAFs at seed pixels inside target shapes near their corners or jogs. Unlike SRAFs generated by argument square, no SRAFs output by squareonly can be changed to rectangles.

squareonlyscore

The squareonlyscore option performs SRAF generation in the same way as squareonly, but performs cleanup in the same way as square. Like squareonly, if localmaxgradient keyword squarewidth is defined, squareonlyscore generates square SRAFs at seed pixels inside target shapes near corners with an area of squarewidth2. Like SRAFs generated by the argument squareonly, no SRAFs generated by squareonlyscore can be changed to rectangles. Cleanup of SRAFs generated by squareonlyscore is score-based, the same as square.

vertical

Generates rectangular SRAFs inside target shapes and aligned to vertical target edge axes.

Note:

Regarding algorithm interactions:

  • If algorithm 0 is set, square is the default cornerprefnegative setting.
  • If algorithm 1 is set, no is the default cornerprefnegative setting.

Description

An optional keyword that specifies the negative SRAF generation method at target shape corners. The cornerprefnegative keyword is applicable only to the negative keyword and may be specified globally, by priority, or within the localmaxgradient template.
