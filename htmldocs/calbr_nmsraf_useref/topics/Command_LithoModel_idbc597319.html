litho_model

LITHO FILE Statement and Commands

Specifies the directory name that contains files for the litho model format. Used by RET MBSRAF only.

Usage

litho_model dir_name

Arguments

dir_name

A required argument that specifies the litho model directory name.

Description

A required keyword that specifies the directory containing the Lithomodel file for mbsraf. The directory you specify must be located within the directory specified by the modelpath keyword. The litho_model keyword is written within an image_options block.

The directory referenced by the litho_model keyword contains a Lithomodel file, optical model directory and resist model file. For more information regarding best practice use of litho model format see “Lithomodel Format Usage for Calibre mbSRAF Execution”.

Note:

Only the litho model format is supported for EUV, including models and all related keywords.

Examples

Example 1

The litho_model keyword and its directory name are shown inside an image_options block within a LITHO FILE statement. The model is declared and called by the image_options keyword:

LITHO FILE mbsraf_setup [
	modelpath .
	image_options M1_LMO {
		litho_model ./litho_model_dir
		layer M1 visible mask_layer 0
		layer correction_layer visible mask_layer 0
	}
	layer logicsraf_target
	layer correction_layer
	tilemicrons 25
	sraf_options mb_OPT {
		correctionlayer correction_layer
		image_options M1_LMO
		...
		localmaxgradient width 0.026  prior 1
		priority 1 lineendmerge 0.06
	}
	setlayer mb_sraf = mbsraf logicsraf_target OPTIONS mb_OPT
	setlayer spx = mbsraf debug logicmb_sraf seedpixels 1
]
mbsraf = RET MBSRAF M1 correction_layer MAP mb_sraf FILE mbsraf_setup
seed_1 = RET MBSRAF M1 correction_layer MAP spx FILE mbsraf_setup

Example 2

For an inline litho model application, all model information is written locally within the LITHO FILE statement. The litho_model keyword and its symbolic name are followed by all data normally kept within the Lithofile file. The litho model is then referenced from the image_options block with its symbolic name and then subsequently called by the sraf_options block for mbsraf:

#ENCRYPT
LITHO FILE inline_LM [/*
	layer target
	layer opc
	modelpath ./LITHO_MODEL
	optical_model_load o1 optical.mod
	ddm_model_load d1 ddm.mod
	resist_model_load r1 {
		version 5
		modelType CM1
		...		
	}
	litho_model LITHO_MODEL {
		version 1
		resist r1
		mask 0 {
			optical o1
			background clear
			mask_layer 0 TRANS atten 0.06 DDM d1
		}
	}
	image_options A {
		layer opc visible
		litho_model LITHO_MODEL
	}
	sraf_options sraf_opts {
		image_options A
		correctionlayer opc
		...
		localmaxgradient width 0.03 squarewidth 0.040 prior 1
	}
	setlayer c1 = mbsraf target OPTIONS sraf_opts
*/]
#ENDCRYPT
mbsraf = RET MBSRAF target correction_layer MAP mb_sraf FILE inline_LM
