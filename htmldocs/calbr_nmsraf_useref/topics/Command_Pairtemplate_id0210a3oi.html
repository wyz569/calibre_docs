pairtemplate

Templates

Places SRAFs at specified angles and offsets from target shape center. Used by RET SBAR only.

Usage

pairtemplate
tag name [tag name ...]
{angle value | offset value | {angle value offset value}}
width value
[length value]
[orient {horizontal | ortho | vertical}]
[pitchfilter value]
[prior value]

Arguments

tag name ...

A required keyword specifying one or more tag names generated only by the pairtag keyword of the newtag command. The tag name identifies a group of connectors to the centers of shapes to be input to the template. Tags must appear as the first arguments in the template.

angle value | offset value | {angle value offset value}

One of two required keyword or keyword pairs specifying the angle and/or the offset SRAFs are placed from target shapes. The connectors used in the measurements must be generated by the pairtag keyword.

The angle keyword specifies the minimum allowed angle between the connector and axis of the proposed SRAF from the end point of the connector. You must specify the value in degrees, between 0 and 90 degrees, inclusive.

Figure 1. Angle influence on SRAF placement

The offset keyword specifies the minimum distance between the center of the proposed SRAF to the end point of the connector.You must specify a positive value in microns. When the offset value is less than half the connector length, SRAFs are placed on the connector at the offset distance from the connector end, as depicted in Figure 2.

Figure 2. Offset less than half connector length

Figure 3 depicts use of the offset keyword where the offset value exceeds half the length of the connector. These SRAFs are placed on either side of the connector.

Figure 3. Offset greater than half connector length

You may specify the angle and offset keywords together as required as depicted in Figure 4.

Figure 4. The offset and angle keywords together

width value

A required keyword that specifies the width of SRAFs.

length value

An optional keyword that specifies the length of rectangular SRAFs to be generated. If not specified, length defaults to width, generating square SRAFs.

orient {horizontal | ortho | vertical}

An optional keyword that orients the generated SRAFs relative to the cell. The horizontal and vertical keywords generate SRAFs parallel to the X-axis, and parallel to the Y-axis, respectively. The ortho keyword generates SRAFs with horizontal orientation for connectors inclined more than 45 degrees, or vertical orientation for connectors inclined 45 degrees or less. The default mode is ortho. Figure 5 depicts a vertical orientation.

Figure 5. SRAFs oriented in vertical direction

pitchfilter value

An optional keyword and parameter pair that allows SRAF generation if no target shapes are found closer to the proposed SRAFs than the specified value. The distance is measured on center-to-center between target shapes and SRAFs. Shape centers at connector end-points are not considered. If the specified value is -1, then the distance is defined between the center of the proposed SRAF and the end point of the connector. The value must be a positive number in microns. It specifies the minimum distance between target shapes and SRAFs for generation of SRAFs to be performed. The distance is measured between centers of the bounding boxes of target shapes and centers of SRAFs. The target shape of origin (target shape with its center at the connector end point) is not considered.If the value is negative (-1), the distance is then defined as between the center of the SRAF and end point of the pairtag connector which generates the SRAF.

Pitchfilter filters out the field SRAFs generated by pairtemplate based on their center-to-center distance between the SRAF and other target shapes (not those associated with pairtag) that did not create these SRAF fields.

Filtering is disabled when pitchfilter is not specified or the specified value is 0. This is the default behavior.

prior value

An optional keyword that specifies a numeric identifier for placed SRAFs. Identifiers are applied sequentially, so a value of 1 is applied first, then 2 is applied, and so on. The default priority is 1. See “priority” for more information.

Description

An optional template and argument set that places rectangular SRAFs at the specified angles and offsets from each target center. The pairtemplate template must be specified with a tag generated by the pairtag keyword of the newtag command.

Examples

Example 1 — Basic Usage

# Generates square SRAFs located 150 nm from connector ends
# (pTag shape centers) with an angle of 30 degrees from the connector.
pairtemplate TAG pTag WIDTH 0.02 offset 0.15 angle 30 prior 1

Example 2 — Comprehensive Usage

# Generates oblong SRAFs with two priorities located at 85 nm and 120 nm 
# from connector ends (pt1 and pt2, respectively) with  angles of 20
# and 40 degrees from connectors.
newtag pt1 pairtag distance >=0.1 <0.2 projection >= 0.02
newtag pt2 pairtag distance >=0.1 <0.3 projection >= 0.04
pairtemplate tag pt1 width 0.020 length 0.070 offset 0.085
	angle 20 prior 10
pairtemplate tag pt2 width 0.030 length 0.090 offset 0.120
	angle 40 prior 20
