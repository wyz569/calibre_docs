Tag Keywords

Tags direct templates where to generate SRAFs. The newtag family of keywords select edges and refine locations of SRAF placement.

Although tags are optional, they provide significant capabilities for SRAF generation for each template. The templates reference tags by a tag keyword, followed by a tag name. Multiple tags can be specified for a template. If a template uses multiple tags, the tags will be combined through a logical OR operation. All templates of the same priority must use the same tags.

In-place Tags

If the name of a tag is defined only for the use of another tag, the first tag name explicitly defined can be replaced by the body of the tag enclosed in a pair of parenthesis. The two lines of the newtag commands can be merged into one. For example:

newtag tag1 recta > 0.01
newtag tag2 tag1 length > 0.02

can be described as: 

newtag tag2 ( recta > 0.01 ) length > 0.02

An automatically generated tag name is assigned to each parenthesized in-place tag. This tag name is used to manage the tag internally and is not accessible outside Calibre.

The components in a boolean tag, as well as the predefined tag specified for some types of tags may be a fully parenthesized boolean expression. All components of these expressions must be previously defined tags. The allowed boolean operators are OR, AND, NOT and XOR.

Calibre nmSRAF does not define precedence rules between the operators. Therefore, you must use parentheses to explicitly define the order of logic. The nested boolean expression must be enclosed in an outermost pair of parentheses. For example:

newtag tag1 length < 0.5
newtag tag2 recta > 0.01
newtag tag3 external > 0.2
newtag tagfinal tag1 OR (tag2 AND tag3)

can be described as:

newtag tagfinal (length < 0.5) OR ((recta > 0.01) AND (external > 0.2))

Calibre cnSRAF Template Behavior

Tags usually select some parts of target edges. The rectedge and rectcorner templates generate their candidates for these edges.

The rectcorner and rectcorner45 templates generate SRAFs for corners of the target layer if the corner is part of a tagged partial edge. They do not generate candidates for end points of tagged partial target edges that are not tagged.

All Calibre cnSRAF templates other than rectedge, rectcorner, and rect45corner are pitch based. They generate candidates only for tagged target rectangles. A target rectangle is referred to as tagged if any part of the rectangle boundary coincides with a tagged edge.

How Design Hierarchy or Tiling Partitions Can Affect Tagging Behavior

Flat designs have no hierarchy-based effects on polygon tagging, but are still susceptible to effects of tile boundaries. Tagging shapes in hierarchical designs is susceptible to hierarchy- and tile boundary-effects.
