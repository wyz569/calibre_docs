processing_mode

LITHO FILE Statement and Commands

Enables various processing modes. Used by RET SBAR and RET MBSRAF.

Usage

processing_mode {flat | hierarchical | hybrid}

Arguments

flat | hierarchical | hybrid

Toggles the processing mode.

Description

An optional command that enables three processing modes:

  • Flat mode enables litho-flat processing by splitting the design into tiles which are processed separately on remote machines.

  • Hierarchical mode processes the design as cells and tiles on remote machines.

  • Hybrid mode examines the layout hierarchy to determine on a per-cell basis whether default hierarchical or litho-flat processing would provide the best runtime performance.

For the RET SBAR command, LAYOUT ULTRAflex is invoked automatically when processing_mode flat is specified and only flat processing_mode flat may use LAYOUT ULTRAflex. See the Calibre Post-Tapeout Flow User’s Manual for more information.

Note:

If ULTRAflex mode is already enabled in your SVRF file, Siemens EDA recommends you also set processing_mode in the LITHO FILE statement, for either RET SBAR or RET MBSRAF, to achieve consistent SRAF results.

Examples

LITHO FILE cn_setup_1 [ 
	processing_mode flat 
	... 
	sraf_options OPTS { 
		cleanupversion 1
		minoffset 0.040 lineend 0.050 maxconflict 0.150  
		rectedge tag All_Wire extension 0.020 width 0.03 offset 0.04 prior 1 
	... 
	} 
	setlayer cnsb1 = cnsraf M1 OPTIONS OPTS 
	... 
]
