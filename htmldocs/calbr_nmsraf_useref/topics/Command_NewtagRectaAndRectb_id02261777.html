newtag recta and rectb

Tag Keywords

Specifies a tag name representing all edges of rectangular target shapes matching the specified constraint. Used by RET SBAR and RET MBSRAF.

Usage

newtag name recta constraint [rectb constraint]

Arguments

name

Specifies the name of the tag.

recta constraint

A keyword to select rectangular target shapes with at least one edge conforming to the specified constraint. Edge-based templates are applied to the recta edge only.

rectb constraint

An optional keyword name identifying orthogonal edges adjacent to edges selected with recta. The rectb keyword used in conjunction with recta to qualify edges of interest in rectangular target shapes. Only the edges selected by recta are applied to the edge-based template.

Note:

When applying the recta and rectb keywords to any pitch-based template, you must provide an upper bound to their constraints to enable limits during edge collection.

Description

The recta and rectb tags select the edge of interest, and any adjacent orthogonal edges, respectively. The selected edges of rectangular target shapes must satisfy the constraints of the specified edge lengths. No consideration of orientation of the rectangle is made. The constraints may define a lower and an upper bound on the length of the long and short edges of the rectangles. The two sets of constraints for the long and short edges may overlap.

Only manhattan- and 45-degree input layer geometries are supported. No skew-edged geometries are valid target shapes for SRAFs.

When using tagging, positive SRAFs may be generated parallel and perpendicular (given template constraints) relative to tagged edges and negative SRAFs are generated only parallel to tagged edges. See the example Figure 2 for more information.

CAUTION:

Using constraint values that exceed cell or tile boundaries may cause flattening and increase run times. Siemens EDA recommends using the recta and rectb newtag types on relatively shorter geometries such as contacts, vias and local interconnect. In addition, if geometries extend past tile boundaries, recta and rectb cannot determine the rectilinearity of those portions of the geometry extending beyond the tile boundary.

Examples

Example 1 — recta Usage

newtag recta_edge recta > 0.1 <= 0.4

Example 2 — recta and rectb Usage

In this figure, the edge named recta_edge is selected by recta, but is first filtered to meet qualification of the edges named orthogonal_edge, selected by rectb.

Figure 1. Edge Selection With recta and rectb

newtag recta_edge recta > 0.1 <= 0.4 rectb > 1 < 2
