conflictmode

Conflict Keywords

Specifies the means of conflict region processing. Used by RET SBAR only.

Usage

conflictmode [bbox | merge | nomerge | shift]

Arguments

bbox

Generates a bounding box around conflict regions. For each pair of SRAFs in conflict, a bounding box is generated. These interacting bounding boxes are merged into polygons. The bounding box of the resulting polygons is used as the conflict region to be processed as specified by the maxconflict and centerthresh rules. These resulting conflict regions are always rectangular. The conflictbbox debug layer helps you visualize this bounding box.

This mode is only supported in cleanupversion 1. Use of this mode in cleanupversion 0 results in warning messages.

merge

Merges all rectangular conflict regions only. For each pair of SRAFs in conflict, a bounding box is generated. These interacting bounding boxes are merged into polygons. Merging may result in the generation of non-rectangular polygons. Resulting polygons are checked with the maxconflict and centerthresh controls and processed. Non-rectangular polygons and rectangles with at least one side exceeding maxconflict are discarded. The hardconflict debug layer helps you visualize these polygons.

This is the default mode for cleanupversion 0 and 1.

nomerge

Does not merge the conflict regions, but processes each one individually. Inserted SRAFs are checked for MRC violations.

shift

Does not merge the conflict regions, but processes each one individually. This method attempts to shift common SRAFs (including 45-degree SRAFs) to the center of the conflict region. If the orientations are not matched, the conflict region is considered for regular conflict resolution.

Tip

Siemens EDA recommends using conflictmode bbox for best results.

Description

An optional keyword that specifies the method by which conflict regions are constructed. This keyword can be set globally for all cnsraf templates and it can be assigned on a priority basis using the priority keyword. Setting conflictmode on a priority basis overrides the global assignment. These options outputs conflict regions that can be displayed by various debug keywords. See “setlayer cnsraf debug” for more information.

Examples

Example 1

The merge option merges all rectangular conflicts into one conflict region. Because the conflict region is MRC clean, it is processed, and a new SRAF is added. The nomerge option processes each conflict region individually. Any new SRAFs violating MRC are removed.

Example 2

The shift option processes each conflict region individually and shifts common SRAFs to the center of the conflict region. If the SRAFs are not aligned, then shift is not applied, and nomerge is used instead.

Example 3

The bbox option merges each conflict region that may overlap and applies a bounding box around the periphery of the merged regions. This bounding box permits a final SRAF to be saved, because the bounding box is always rectangular, whereas the merged conflict regions output by merge mode may not afford the required space. Any resulting non-rectangular conflict regions are discarded with their field SRAFs.
