cleanupversion

Cleanup Keywords

Specifies the version of SRAF cleanup. Used by RET SBAR only.

Usage

cleanupversion 0 | 1

Arguments

0

Forces cleanup to adhere to the original (pre-2016.3) algorithm.

1

Invokes cleanup with an updated algorithm.

Description

A required keyword that specifies the algorithmic version in which SRAFs are removed. This keyword can be set globally for all setlayer cnsraf templates.

Note:

The cleanupversion 0 setting has been deprecated and is to be obsoleted in a future Calibre release.

Note:

The cleanupversion keyword is required as of Calibre release 2018.1. Siemens EDA recommends always setting cleanupversion 1 for new cnSRAF recipes of line-layer and 2D topologies.

Siemens EDA also recommends always setting cleanmode postsizing with cleanupversion 1 to avoid excessive field SRAF. For existing recipes using Calibre versions prior to 2016.3, where cleanupversion 1 is specified, Siemens EDA also recommends specifying cleanmode immediate globally.

The cleanupversion 1 keyword provides a number of algorithmic improvements for SRAF generation:

  • Promotion is automatically enabled.

  • The dynamicsizing algorithm is automatically invoked, which aids with more intelligent conflict resolution.

  • The number of templates required to achieve equivalent SRAF generation coverage is significantly reduced, which requires less tuning of cleanup parameters.

Note:

Various SRAF output differences may be apparent depending on the value of cleanupversion you select. Many recipes produce significantly different results when migrating from cleanupversion 0 to 1 and require cleanup tuning. Use cleanupversion 1 when developing new recipes to obtain equivalent or improved SRAF generation coverage. Other particulars regarding cleanupversion include:

  • When cnsraf cleanupversion 1 is enabled, preclean debug output may not be useful, because the preclean cleanup step is not performed for cleanupversion 1.
  • Promotion is enabled by default for both modes of cleanupversion as of Calibre release 2016.2.
  • Using cleanupversion 1, SRAFs can be trimmed by width only if minwidth is specified. When minwidth is specified, the determination of whether an SRAF width or length is trimmed is determined by the greatest resulting SRAF area.
  • For any cleanupversion, all SRAFs retain their original priority throughout all cleanup phases. SRAFs of each successive priority are cleaned with regard to the SRAFs of previously cleaned priorities.

Examples

Example 1 — Template Requirements Using the Old Cleanup Method

In the first example, templates using conflictonly and dynamicsizing by priority are separate. The conflictonly argument is used in within the template, and dynamicsizing is specified by priority in the second example, eliminating the need to exercise careful tuning of cleanup parameters.

rectedge tag med_end extension 0.035 width 0.019 offset 0.075
	conflictonly prior 1  
rectedge tag med_end extension 0.035 width 0.0175 offset 0.075
	prior 2  
rectedge tag med_end extension 0.035 width 0.0175 offset 0.075
	prior 3 
rectedge tag med_end extension -0.035 width 0.0175 offset 0.075
	prior 4
priority 1 minoffset 0.030 minfeaturespace 0.040 maxconflict 0.250
	centerthresh 0.080  sbwidth 0.018 sraf 0.038 conflictmode nomerge 
priority 2 minoffset 0.030 minfeaturespace 0.040 maxconflict 0.250
	centerthresh 0.080  sbwidth 0.0175 sraf 0.038 conflictmode nomerge
	cleanmode postsizing dynamicsizing 0.030
priority 3 minoffset 0.030 minfeaturespace 0.040 maxconflict 0.250
	centerthresh 0.080 sbwidth 0.0175 sraf 0.038 conflictmode nomerge
	cleanmode postsizing 
priority 4 minoffset 0.030 minfeaturespace 0.040 maxconflict 0.250
	centerthresh 0.080 sbwidth 0.0175 sraf 0.038 conflictmode nomerge
	cleanmode postsizing

Example 2 — Equivalent Code to Achieve Coverage Using the New Cleanup Method

Output differences are shown in Figure 1 between two independent SRAF generation runs, using half the number of templates when using cleanupversion 1. The red SRAFs were generated using cleanupversion 0. The green SRAFs, requiring half the templates and reduced cleanup tuning parameters, were generated using cleanupversion 1.
