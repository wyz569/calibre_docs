newtag external and internal

Tag Keywords

Specifies a tag name representing the inner or outer edges of layers for which to generate SRAFs. Used by RET SBAR and RET MBSRAF.

Usage

newtag name {[predefined_tag] | [from_tag to_tag]}
{[external constraint] [internal constraint]} [fulledge] [metric] [projecting]

Arguments

name

Specifies the name of the tag.

predefined_tag

Previously derived tags generated by taglayer or newtag commands. Mutually exclusive with from_tag and to_tag.

from_tag

A previously derived tag from which to measure to the to_tag. Mutually exclusive with predefined_tag.

to_tag

A previously derived tag from which to measure to the from_tag. Mutually exclusive with predefined_tag.

external constraint

A keyword applied to the outside edges of target shapes used for spacing. The constraint takes several forms:

  • <= value

  • <= value1 >= value2

  • <= >= value

internal constraint

A keyword applied to the inside edges of target shapes used for widths. The internal constraints use the same forms as external. As a default, the promotion distance is used for the upper bound.

CAUTION:

Upper bounds for the internal keyword are critical:

  • When using the internal keyword to generate tags used in the context of rectedge center strictcenter template, you must provide an upper bound to the internal constraint to enable limits during edge collection.
  • When applying the internal keyword to any pitch-based template, you must provide an upper bound to the constraint to enable limits during edge collection.
fulledge

An optional keyword forcing the entire length of a tagged edge to be used even when a shorter portion of the edge is originally tagged. This option is compatible with all constraint types.

Note:

Partial edge tagging with Calibre mbSRAF may lead to unexpected results because of differences between intended SRAF and MRC enforcement. Siemens EDA recommends supplying the entire edge to mbsraf when using tagging.

metric

An optional metric used to qualify the constraint. The euclidean, opposite and opposite_extended metrics are mutually exclusive. Any of these three metric constraints may be used with projecting.

  • euclidean — The external and internal constraint is measured radially from edges. This is the default metric.

  • opposite — The external and internal constraint is measured only between mutually facing edges.

  • opposite_extended value — The external and internal constraint is measured between mutually facing edges, plus a positive extension value added to each end of the edge pair.

projecting

An optional keyword using only those edges that face each other for an edge length that satisfies the constraint. The projecting keyword is compatible with all metrics.

Note:

Partial edge tagging with Calibre mbSRAF may lead to unexpected results because of differences between intended SRAF and MRC enforcement. Siemens EDA recommends supplying the entire edge to mbsraf when using tagging.

Description

An optional keyword used to input tags by name to Calibre nmSRAF templates. Multiple tags can be referenced in a single template. The newtag keyword uses external and internal to measure distance between tags and target for selective edge pair isolation. Alternatively, newtag can be used to measure and isolate edges between two tags. See the examples for further information.

Only manhattan- and 45-degree input layer geometries are supported. No skew-edged geometries are valid target shapes for SRAFs.

When using tagging, positive SRAFs may be generated parallel and perpendicular (given template constraints) relative to tagged edges and negative SRAFs are generated only parallel to tagged edges. See the example in Figure 2 for more information.

Shielding is implicit for all internal and external newtag operations and is equivalent to EXCLUDE_SHIELDED 4 for the SVRF INTERNAL/EXTERNAL commands.

CAUTION:

Using constraint values that exceed cell or tile boundaries may cause flattening and increase run times. Siemens EDA recommends using the recta and rectb newtag types on relatively shorter geometries such as contacts, vias and local interconnect.

Examples

Example 1 — Specifying Tags as a Combination of Space and Width

The first method demonstrates proper use of internal and external keywords setting width and space constraints, respectively, with two newtag commands, the first newtag tag (mapped in red) passed to the second newtag command:

newtag tag0 internal > 0.1 < 0.25
newtag tag1 tag0 external > 0.04 < 0.080

The second tag demonstrates valid usage, but when combining the two keywords in a single newtag command, the internal filter is ignored and the output of tag_50_100 is equal to all of the target edges that satisfy only the external constraint:

newtag tag_50_100 internal >= 0.1 < 0.25 external > 0.05

Example 2 — Applying Metrics to Tags

newtag ltag external >= 0.1 opposite
newtag ltag external >= 0.1 opposite_extended 0.040 projecting

Example 3 — Extracting Projecting Edges by Run Length

newtag external <= 0.070 opposite length > 0.10

Example 4 — Compounding length Tags

newTag tag1 l1 length > 0.10
newTag tag2 l2 length < 0.05
newTag tag3 external <0.13
newTag final tag3 and (tag1 or tag2)

Example 5 — Newtag external Keyword

Figure 1 and the code snippet shown illustrate newtag near applied to external edges with spaces greater than 0.060, but less than or equal to 0.090. Only the red SRAFs of priority 1 are output between these edges. Newtag far is applied to external edges with spacing greater than 0.090. Green SRAFs of priority 2 are output on these edges.

newtag near external > 0.060 <= 0.090
newtag far external > 0.090
localmaxgradient tag near width 0.024 prior 1
localmaxgradient tag far width 0.024 prior 2
Figure 1. mbsraf newtag external Usage

Example 6 — Newtag fulledge Keyword

In the first illustration of Figure 2 without fulledge, newtag external is applied to edges farther apart than 0.15nm. Only portions of the target edges are selected, and the generated SRAFs adhere to these tags. In the second illustration with fulledge, when external is used in conjunction with fulledge, the entire edge of the target shapes is selected, changing the SRAF output. 

newTag m1_t fulledge external <= 0.15 
localmaxgradient tag m1_t width 0.018 extension 0.015
Figure 2. newtag fulledge Usage



Example 6 — Saving Tags for Debugging

These code examples output tags for cnsraf and mbsraf debugging purposes. The tag name is shown in red. The debug keyword required to output tags is green. The rulecheck and database layer names must be identical. They are shown in magenta.

LITHO FILE cnsraf_setup [
	...
	newtag tag_name external < 0.20
	...
	setlayer cn_sraf = cnsraf target ...
	setlayer tag_name = cnsraf debug cn_sraf tags 1
]
tag_name = RET SBAR target MAP tag_name FILE cnsraf_setup
tag_out {copy tag_name} drc check map tag_out 10
LITHO FILE mbsraf_setup [
	...
	newtag tag_name external < 0.20
	...
	setlayer mb_sraf = mbsraf target ...
	setlayer tag_name = mbsraf debug mb_sraf tags 1
]
tag_name = RET mbsraf target MAP tag_name FILE mbsraf_setup
tag_out {copy tag_name} drc check map tag_out 10

Example 7 — Implicit Boolean AND Equivalence

This code example demonstrates the equivalence of the implicit AND during tag construction. The output of both v_internal and v_internal_and are identical.

sraf_options tag {
	newtag v (vertical)
	newtag v_internal v internal >= 0.230 < 0.315
	newtag v_internal_and v and (internal >= 0.230 < 0.315)
}

Example 8 — Measurements Between from_tag and to_tag

In this example, SRAFs must be placed between the rectangular and square target shapes, but not between the two square target shapes. The measurement is performed between two tags that were previously generated, rect_edge and sq_edge, from the rectangular target, and square target shapes, respectively.

Figure 3. newtag From and To Tag Usage

# The rectangle edge is tagged (this is positioned as the from_tag):
newtag rect_edge recta < 0.085 rectb > 0.085
# The square edge is tagged (this is positioned as the to_tag):
newtag sq_edge square < 0.085
# An external measurement is performed between the from_tag and
# to_tag, rect_edge and sq_edge, respectively, and named sq2rect:
newtag sq2rect rect_edge sq_edge external < 0.160
# Finally, tag sq2rect is specified in a rectedge template:
rectedge tag sq2rect extension 0.02 width 0.025 offset 0.05 prior 2
