calibre

Linux®command

Executes SVRF commands.

Usage

$CALIBRE_HOME/bin/calibre
-drc [-hier] svrf_filename
[-turbo [number_of_processors]] [-turbo_litho [number_of_processors]]
[-remotefile filename]

Arguments

-drc [-hier]

Selects hierarchical Calibre nmDRC checking.

svrf_filename

Specifies the SVRF filename.

-turbo [number_of_processors]

Directs Calibre nmDRC-H to use multi-threaded parallel processing for all stages except LITHO operations.

-turbo_litho [number_of_processors]

Directs Calibre nmDRC-H to use multi-threaded parallel processing when performing Litho operations. The optional number_of_processors argument is a positive integer that specifies the number of CPUs to use in the processing. If you do not specify a number, Calibre nmDRC-H runs on the maximum number of CPUs available.

You can specify the -turbo and the -turbo_litho options concurrently in a single command line, and the respective number_of_processors strings can vary between the two options.

-remotefile filename

Enables multi-threaded operations on remote hosts of a distributed network. This option is part of the Calibre MTflex multi-threaded, parallel processing architecture. The remote hosts do not have to be of the same platform type when using this option.

The -remotefile option must be specified in conjunction with the ‑turbo or ‑turbo_litho option. Any of these commands may specify the number of processors you are using, including those on the remote hosts. The filename specifies the pathname of a configuration file containing information for the primary and remote hosts. You must have the required number of licenses for your job.

The -remotefile option only applies to hierarchical applications. For more details, see the Calibre Administrator’s Guide.
