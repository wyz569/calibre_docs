dynamicsizing

Cleanup Keywords

Sizes SRAFs during generation. Used by RET SBAR and RET MBSRAF.

Usage

dynamicsizing value

Arguments

value

A positive floating point number specifying the minimum length of the trimmed SRAF.

Description

An optional keyword used globally or by priority to resize SRAFs according to local conditions and the minimum SRAF length you specify by its value (you cannot specify dynamicsizing within templates). With cnsraf, candidate SRAFs are sized with respect to other SRAFs until they are conflict-free or are removed. If you use dynamicsizing globally, all applicable templates perform dynamicsizing as specified. When you use dynamicsizing by priority, all applicable templates must specify that priority to perform dynamicsizing as specified.

Square target shapes are sized down uniformly. Rectangular target shapes are reduced in size along the long axis from either end as required. Refer to cleanmode postsizing for more information regarding MRC-clean generation of SRAFs when using dynamicsizing.

Note:

Regarding dynamicsizing requirements and interactions:

  • The dynamicsizing and conflictonly keywords cannot be used in the same priority.
  • Siemens EDA recommends outputting intermediate debug layers for each priority when using dynamicsizing during migration processes from any earlier release to 2013.2 and later releases.
