maxmove

Cleanup Keywords

Restricts SRAF movement during cleanup. Used by RET MBSRAF only.

Usage

maxmove value

Arguments

value

A positive floating point number.

Description

An optional keyword used to restrict SRAF movement during cleanup. For example, maxmove is used to limit the distance an SRAF can be moved from its initially generated position during cleanup to comply to MRC constraints. Where maxmove is set to 0nm, no movement of SRAFs is permitted during cleanup. For recipes where algorithm 0 is specified, the default value is the specified width value divided by 2. Where algorithm 1 is specified, the default value is 1nm.

Note:

Prior to release 2018.1, the default for maxmove where algorithm 1 was specified, was 0nm. The maxmove default as of Calibre release 2018.1 is 1nm. Therefore, existing recipes using algorithm 1 must explicitly specify maxmove 0 to obtain the previously generated SRAF results.
