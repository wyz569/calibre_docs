sblayer

Layer Keywords

Specifies a layer name containing preexisting SRAFs. Used by RET SBAR and RET MBSRAF.

Usage

sblayer name [sblayer name]

Arguments

name

The name of a polygon layer containing SRAFs. If the sblayer name is specified once, it may contain both positive and negative SRAFs. Alternatively, the sblayer may be specified twice with no order dependency, one layer for positive SRAF and one layer for negative SRAF.

Description

An optional keyword used to import previously generated SRAF layers. All cnsraf templates can reference these layers by name. No SRAFs are placed within a distance of minoffset to the shapes imported on the sblayer.

If sblayer is specified twice, to differentiate between positive and negative SRAFs:

  • Positive SRAFs are completely outside of any target shape (no logical interaction including butting edges).

  • Negative SRAFs are completely inside target shapes (logical enclosure and no coincident edges).

The MRC keywords used for checking the sblayer may be specified per priority or globally:

  • lineend

  • minoffset

  • projecting

If sblayer is specified, it must appear in either the options block or the setlayer command, but never within both. If sblayer is required in other areas of the recipe, you must derive it again under a different name prior to processing it.

Only manhattan- and 45-degree input layer geometries are supported. No SRAFs with skew-edges are valid as sblayer names.

The sblayer and correctionlayer are the layers containing shapes that are simulated to provide the image from which the gradient is calculated. If neither of these layers are available, the target layer is used for the image simulation.
