centermerge

Cleanup Keywords

Merges two parallel SRAFs within the specified distance. Used by RET SBAR only.

Usage

centermerge max_distance

Arguments

max_distance

A positive real number indicating the maximum distance between which to merge two SRAFs. A negative value turns off centermerge capability.

Description

The centermerge keyword controls merging of two parallel SRAFs within the specified distance. Specifically, two initial SRAFs are replaced, for their common runlength, by a single SRAF centered between them. The default distance is the value specified by the minoffset projecting keyword, and where no projecting value is specified, by the minoffset value. The centermerge keyword may be specified globally or by priority. The centermerge behavior can be modified by the dynamicsizingrunlength keyword.

The centermerge capability can be performed before or after dynamicsizing during the cleanup process. By default, centermerge is performed after dynamicsizing. Siemens EDA recommends specifying dynamicsizingrunlength 0. This forces centermerge to be performed before dynamicsizing, cleaning up SRAFs having a common runlength of more than one third of their length.

Global and Priority Behavior Interactions

There are value-based interactions between global and priority specifications with centermerge. When centermerge is specified globally with a value of less than zero (< 0), centermerge specified per priority is only enabled when specified with a value of greater than zero (> 0). When centermerge is specified globally with a value of greater than zero (> 0), centermerge for all specified priorities is enabled except when any centermerge priority is specified with a value of less than to zero (< 0).

Examples

Example 1 — Application of centermerge Keyword

The code shown in Example 1 generates SRAFs initially reserved as field shapes, as shown in Figure 1, pending further cleanup processing by centermerge. The max_distance specified by centermerge must exceed the distance between initial SRAFs, in this case, 0.030 and 0.0222, respectively.

Example 1. centermerge Code
sraf_options OPT {
	dynamicSizing 0.032 cleanmode postSizing centermerge 0.030
	minOffset 0.020 cleanupversion 1 conflictmode merge maxconflict 0.200
	minlength 0.032 minWidth 0.010 minfeaturespace 0.010
	rectEdge tag tagext extension 0 width 0.01 offset 0.010 prior 3
}
setlayer cnsraf = cnsraf target OPTIONS OPT
