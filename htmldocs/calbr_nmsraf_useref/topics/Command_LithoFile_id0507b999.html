LITHO FILE

LITHO FILE Statement and Commands

Specifies Calibre nmSRAF commands within the SVRF rule file. Used by RET SBAR and RET MBSRAF.

Usage

LITHO FILE name ‘[’
   commands
‘]’

Arguments

name

A required name allowing commands to be referenced by the RET SBAR or RET [EUV] MBSRAF commands. Each LITHO FILE statement must have a unique name.

‘[’ commands ‘]’

A required group of commands placed between brackets ([ ]). Any term on the same line as the brackets is ignored.

Each command within the LITHO FILE statement must reside on its own line. You may span multiple lines with a line termination (backslash \), which must not be followed by any characters or whitespace.

Note:

Siemens EDA recommends always using the sraf_options block to format and organize keywords within the LITHO FILE statement.

Description

The LITHO FILE statement contains cnsraf and mbsraf commands. The LITHO FILE statement is called by the FILE argument of the RET SBAR and RET MBSRAF commands. The LITHO FILE statement contents can be contained within the SVRF rule file or referenced as an external file, and in either case, called by a symbolic name for multiple instantiations.

You can use variables for any names or numbers in the SVRF or LITHO FILE statement, as described by the command “svrf_var_import”.

Note:

Siemens EDA recommends using the litho model format for all model-based SRAF recipes. These non-litho model commands are unsupported and are obsoleted as of release 2019.1:

  • optical
  • optical_model_load
  • plane
  • resist
  • resist_model_load
  • threshold

Examples

Example 1

The following code demonstrates the general format of the Calibre cnSRAF LITHO FILE statement using the sraf_options statement.

LITHO FILE cnsraf_1 [ 
	layer ref_layer 
	sraf_options cnsraf_cmds {
		cleanupversion 1
		rectedge center length 0.025 width 0.015 ... prior 1
		priority 1 minoffset 0.025 minwidth 0.010 ...
	}
	setlayer out_layer = cnsraf ref_layer ... OPTIONS cnsraf_cmds \ 
		maxconflict size 
	... 
]

Example 2

The following code demonstrates a Calibre mbSRAF LITHO FILE statement for the litho model format using the sraf_options statement.

LITHO FILE mb_setup_1 [  
	modelpath ./models 
	layer target 
	image_options IOS { 
      litho_model lithomodel.dir 
      layer target visible 
	} 
	tilemicrons 40 adjust 
	sraf_options mbsraf_cmds {
		algorithm 1
		image_options IOS
		width 0.025 localmaxgradient ...
	}
	setlayer mbsb = mbsraf target OPTIONS mbsraf_cmds ... 
]

Example 3

The following code demonstrates a Calibre mbSRAF discrete model statement. This format is deprecated. Siemens EDA recommends the litho model format used in Example 2.

LITHO FILE setup_name [  
	modelpath models 
	optical_model_load op_mod 
	resist_model_load res_model 

	layer name type trans 
	... 
	sraf_options mbsraf_opts {
		algorithm 1
		width 0.025 localmaxgradient ...
	}
	setlayer out_layer = mbsraf target_layer optical model_name \
		OPTIONS mbsraf_opts
]

Example 4

The following code demonstrates the use of mode nearest to generate SRAFs that are parallel to target shape edges and oriented 45 degrees at target shape corners. 

LITHO FILE sample_lithofile [   
	modelpath /user/opt/models/model1:/user/opt/models/mbsraf_model 
	optical_model_load opticsfile e1_f+0.0000 
	resist_model_load resistfile model1.mod 
	background 0.0215 -0.0085 
	layer CA visible 0.9785 0.0085 
	layer dummy_opc visible 0.9785 0.0085 
	layer sraf_ref_layer visible 0.9785 0.0085 25
	sraf_options mbOPT_statement {
		correctionlayer dummy_opc 
		optical opticsfile 
		algorithm 1
		resist resistfile 
		minfeaturespace 0.035 
		localmaxgradient width 0.030 prior 1 
		priority 1 minoffset 0.040

	setlayer sb_sample = mbsraf CA dummy_opc OPTIONS mbOPT_statement
] 
mbsraf_logic = RET MBSRAF CA dummy_opc FILE sample_lithofile MAP sb_sample


Example 5

The following code demonstrates the implementation of a multi-patterning application.

LITHO FILE mp_mb_setup [  
	modelpath ./models 
	layer target_0
	layer target_1
	layer target_2
	image_options IOS { 
      litho_model mp_litho_dir
      layer target_0 visible mask_layer 0
   		layer target_1 visible mask_layer 1
   		layer target_2 visible mask_layer 2
	} 
	tilemicrons 40 adjust 
	sraf_options mbsraf_cmds {
		algorithm 1
		image_options IOS
		width 0.025 localmaxgradient ...
	}
	setlayer mbsb = mbsraf target OPTIONS mbsraf_cmds ... 
]
mp_mb_output = RET MBSRAF target_0 target_1 target_2
	FILE mp_mb_setup MAP layer1
