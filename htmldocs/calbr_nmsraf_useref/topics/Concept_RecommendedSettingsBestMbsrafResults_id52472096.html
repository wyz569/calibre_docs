Recommended Settings for Best mbsraf Results

Recommended settings improve the results produced by the mbsraf command.
Table 1. Recommended Settings for Best mbsraf Results

Option

Recommendation/Notes

litho_model

Only the litho model format is supported for EUV, including models and all related keywords.

maxmove

As of 2018.1, the previous default setting of maxmove 0 has changed, and can now be explicitly specified. However, recipes previously using algorithm 1 must now explicitly specify maxmove 0 to obtain identical SRAF results prior to 2018.1.

algorithm

Siemens EDA recommends always setting algorithm 1 for improved SRAF consistency of generated SRAFs.

interpriorityalign intrapriorityalign

Siemens EDA recommends always setting interpriorityalign and intrapriorityalign per priority. If these keywords are set globally, a warning is issued to the transcript.

gradientfactornegative

Siemens EDA recommends setting gradientfactornegative to -1 for single exposure simulations.

lineendmerge

Siemens EDA recommends using the lineendmerge keyword in conjunction with the intersection and intersectionnegative keywords to provide more continuous SRAF output. If no lineendmerge parameter is explicitly set, it is automatically set and its resulting value reported to the transcript. When lineendmerge is specified globally, line-ends of different priorities may be merged. Therefore, Siemens EDA recommends only specifying lineendmerge by priority.

minwidth

Siemens EDA recommends using the minwidth keyword to avoid the possibility of outputting SRAFs of a size less than the minimum MRC constraint.

correctionlayer

The correction layer can be a modified target layer (a layer that has been resized by a few iterations of Calibre nmOPC, for example). The modification provides enlarged shapes that lead to an improved gradient. The correctionlayer keyword is used during simulation to define the initial image that forms the basis of the gradient calculation. Siemens EDA recommends using correctionlayer so it leads to a simulated contour within plus or minus 20 percent of the target area.

tilemicrons

The selection of tile size is critical for optimum processing. If the tile size is too large or too small, primary or remote memory consumption may go into swap and slow performance. The optimum tile size settings found by Siemens EDA correlate to the technology node and balance the size of data per tile. It is recommended that you set tilemicrons to the values based on the node listed in Table 1.

clone_transformed_cells

Siemens EDA recommends you set clone_transformed_cells to the decide option. The optical model is checked if cell cloning is activated. The decide option causes the Calibre engine to check if any asymmetric sources are present in the optical model. If so, Calibre clones cells that have rotated or mirrored placements so that hierarchical simulation of the rotated structures is correct.
