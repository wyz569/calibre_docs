rectcorner

Templates

Places SRAFs with four methods relative to corners of target shapes. Used by RET SBAR only.

Usage

center Method

rectcorner
[tag name ...]
{center spacelimit value
   {length value | width value}
   {angle degree | offset distance}}
[corner {both | center | pair}]
[strictcenter]
[prior priority]

center multicorner Method

rectcorner
[tag name ...]
{center multicorner spacelimit value
   {length value | width value}
   {angle degree | offset distance [offsetmultiplier factor]}}
[strictcenter]
[prior priority]

Offset Method

rectcorner
[tag name ...]
{offsetx value | offsety value}
{length value | width value}
[breaksym]
[orient {parallel | perpendicular}]
[prior priority]

scalefactor Method

rectcorner
[tag name ...]
{scalefactor index offset distance
   {length value | width value}}
[edgediff 0 | 1]
[{ext1 value ext2 value [off1 value] off2 value}]
[orient 45]
[protectionradius radius]
[prior priority]

Arguments

Optional Keywords Shared by Methods

Optional keywords used with various methods described in this section.

tag name ...

An optional keyword specifying one or more tag names used in the template. The tag name identifies a group of edges or shapes to be used with the template. If specified, tags must appear as the first arguments in the template. All rectcorner usages accept tag names.

prior priority

An optional keyword, applicable to all three usages, that specifies a numeric priority for placed SRAFs. The priority keyword must be specified after the tag within the template. The default priority is 1. See “priority” for more information.

strictcenter

An optional keyword controlling center SRAF placement between tagged corners. Specifying strictcenter restricts SRAFs to be generated only between two tagged corners. By default, where tags are specified within the rectcorner template using the center method, center SRAFs are placed between one or two tagged corners. If no tags are specified within the rectcorner template, center SRAFs are placed between any qualifying corner, and strictcenter has no effect. This keyword is used only by center and center multicorner methods. See Figure 18 for more information.

center Method

center spacelimit value {length value | width value} {angle degree | offset distance} [corner {both | center | pair}]

The center method places SRAFs equidistant from qualifying corners provided by tagged edges. At least one tag must be specified for center. The center keyword requires cleanupversion 1. For detailed illustrations, see “Example 8 — center offset and angle Effects”. The center usage is mutually exclusive with offset and scalefactor usages.

The center keyword boundary of corner and edge collection is derived from spacelimit which, in the following example, is 0.8. However, spacelimit never overrides any specified newtag external values. In the following case, the boundary for edge collection is established by tag2 (0.9) because its value is greater than spacelimit (0.8):

cleanupversion 1
newtag tag1 external <= 0.5
newtag tag2 external <= 0.9
rectcorner tag tag1 tag tag2 center spacelimit 0.8

Positive SRAF generation using the center option requires specifying newtag of external type for selection of the outside edges of target shapes for correct SRAF placement. Negative SRAF generation using the center option requires specifying newtag internal for selection of the inside edges of target shapes for correct SRAF placement.

  • spacelimit value

    The spacelimit keyword and value define the maximum distance between corners that can generate corner SRAFs. The spacelimit keyword establishes the boundary by euclidean measurement.

    Note:

    Excessively large spacelimit values trigger larger corner search areas and significantly increase run time.

  • length value | width value

    A required keyword argument to specify longer and shorter edges of rectangular SRAFs. Either length or width must be specified.

  • angle degree | offset distance

    One of two required, mutually exclusive keywords that must be specified with the center keyword. The angle keyword specifies the position of generated SRAFs by the angle extended from the corners of target shapes. For two-corner position, only positive values may be specified for degree. SRAF placement defaults to angle 45. See Figure 15 for more information.

    The offset keyword specifies the offset position of generated SRAFs by an orthogonal distance from the center of the line connecting corners of target shapes. Only non-negative values may be specified for distance. There is no default for offset. See Figure 16 for more information.

  • corner {both | center | pair}

    An optional keyword that controls output of corner SRAFs. The corner keyword is used only by the center method.

    • both

      Outputs center and pair SRAFs.

    • center

      Outputs a center SRAF. This is the default.

    • pair

      Outputs a pair of SRAFs.

Note:

The center method only works with cleanupversion 1; parse errors are issued if used with cleanupversion 0.

center multicorner Method

center multicorner spacelimit value length value width value {angle degree | offset distance [offsetmultiplier factor]}

The center multicorner method places SRAFs equidistant from qualifying corners provided by tagged edges. At least one tag must be specified for center. The center keyword requires cleanupversion 1. For detailed illustrations, see “Example 8 — center offset and angle Effects”. The center multicorner method is mutually exclusive with offset and scalefactor methods.

  • spacelimit value

    The spacelimit keyword and value define the maximum distance between corners that can generate corner SRAFs. The spacelimit keyword establishes the boundary by euclidean measurement.

    Note:

    Excessively large spacelimit values trigger larger corner search areas and significantly increase run time.

  • length value width value

    Two required keywords that specify the longer and shorter edges of rectangular SRAFs, respectively. The length and width values can be identical.

  • angle degree | offset distance [offsetmultiplier factor]

    One of two mutually exclusive keywords that must be specified with the center keyword. The angle keyword specifies the position of generated SRAFs by the angle extended from the corners of target shapes. For two-corner position, only positive values may be specified for degree. SRAF placement defaults to angle 45. See Figure 15 for more information.

    The offset keyword specifies the offset position of generated SRAFs by an orthogonal distance from the center of the line connecting corners of target shapes. Only non-negative values may be specified for distance. There is no default for offset. See Figure 16 for more information.

  • offsetmultiplier factor

    The offsetmultiplier keyword specifies a positive integer as the offset multiplier for SRAFs generated with the offset keyword. Multiple SRAFs may be generated with their centers offset as the specified multiple of the specified offset distance. The default factor is 1. The offsetmultiplier keyword is only used by the center multicorner method. See Figure 16 for more information.

Note:

The center multicorner method only works with cleanupversion 1; parse errors are issued if used with cleanupversion 0.

Offset Method

offsetx value offsety value length value width value [breaksym] [orient {parallel | perpendicular}]

The offset method places SRAFs from qualifying corners at the specified offset values. For detailed illustrations, see “Example 1 — Identical offset, width, and length Values”. The offset usage is mutually exclusive with center and scalefactor usages.

  • offsetx value offsety value

    Two required keywords that specify the orthogonal distances from the target corner, in the X- and Y-axes, respectively, to the closest corner of generated SRAFs.

  • length value width value

    Two required keywords that specify the longer and shorter edges of rectangular SRAFs, respectively. The length and width values can be identical.

  • breaksym

    An optional keyword, applied only to the offset usage, that outputs SRAF with a designated length parallel to the Y-axis (vertically-oriented SRAF), that breaks the symmetric SRAF output.

  • orient {parallel | perpendicular}

    An optional keyword that filters SRAFs oriented either parallel or perpendicular to the tags input to rectcorner. For square target shapes, orient perpendicular places SRAFs with edges designated by length perpendicular to the X-axis; orient parallel places SRAFs with edges designated by length parallel to the X-axis. For non-square target shapes, orient parallel places SRAFs with edges designated by length parallel to the longest edge of the target shape; orient perpendicular places SRAFs with edges designated by width perpendicular to the longest edge of the target shape. Application of orient is consistent irrespective of the target shape orientation. All parallel and perpendicular SRAFs are output by default.

scalefactor Method

scalefactor index offset distance {length value | width value} [breaksym] [edgediff 0 | 1] [{ext1 value ext2 value [off1 value] off2 value}] [orient 45] [protectionradius radius]

The scalefactor method generates one‑quarter circles of square SRAFs around target corners. Every generated SRAF placed on the arc is square and MRC-clean. For detailed illustrations, see “Example 7 — scalefactor Indexes, offset Effects, and orient 45”. The scalefactor usage is mutually exclusive with center and offset usages.

  • scalefactor index

    A scalefactor index must be specified within the range of 0.7 through 1.4. Indexes less than 1 pull the SRAFs toward the target; indexes greater than 1 push the SRAFs away from the target.

  • offset distance

    A required argument that specifies the distance from target that SRAFs are placed. This value is the distance along the edge at the corner.

  • length value | width value

    Required keywords that specify length or width of generated SRAFs. When applied to the scalefactor keyword, only one of width or length may be specified. Both keywords render identical results because scalefactor generates square SRAFs.

  • edgediff 0 | 1

    An optional keyword that enables placement of SRAFs around non-orthogonal corners. If edgediff is set to 0, no SRAFs are placed around non-orthogonal corners. The edgediff keyword can be used with ext1, ext2, off1, and off2 keywords. The default is 0.

  • ext1 value

    An optional keyword that determines the orthogonal extension from off1, the offset of the longer of the two edges of the corner. There is no default.

  • ext2 value

    An optional keyword that determines the orthogonal extension from off2, the offset of the shorter of the two edges of the corner. There is no default.

  • off1 value

    An optional keyword that determines the orthogonal offset from the longer of the two edges of the corner. The default for off1 is the value set for offset of the rectcorner template.

  • off2 value

    An optional keyword that determines the orthogonal offset from the shorter of the two edges of the corner. There is no default.

  • orient 45

    An optional keyword that places SRAFs at a 45 degree angle to the target corner. Application of orient 45 is consistent irrespective of the target shape orientation.

  • protectionradius radius

    An optional keyword that maintains the specified radius between SRAFs placed by scalefactor and neighboring SRAFs generated by other templates. The protectionradius keyword only works with the scalefactor method of the rectcorner template. The default is 1/2 of minoffset.

Description

An optional template and argument set that, by default, places eight rectangular SRAFs symmetrically, and orthogonally by X- and Y-offsets, from each corner of a target shape or from a tagged edge.

The corners of each SRAF are placed symmetrically by the coordinate system as shown in Figure 1.
