Calibre cnSRAF Template Execution and Cleanup

A template is a specified arrangement of SRAFs around a contact feature based upon a pitch or distance from edges and corners.

Siemens EDA recommends you define Calibre cnSRAF templates by using Calibre pxOPC for:

  • Model-based template generation with standard Thin Mask Approximation (TMA) models

  • 3D Mask models such as Domain Decomposition Method (DDM)

  • Hybrid Hopkins Abbe (HHA) models.

A model-based approach relies on SRAFs generated by a model-based solution. Defining templates requires model-based solutions on a number of selected contact configurations that represent the best cross section of possible configurations. It is important to take asymmetries in the optical configuration into account. Templates are defined such that they result in the best match of SRAFs to the model results.

For either thin mask or 3D mask models, Calibre pxOPC can be used to generate reference SRAFs for Calibre nmSRAF template generation. For more information see the Calibre pxOPC User’s and Reference Manual.

Alternatively, Calibre cnSRAF templates can be extracted from an optimization procedure. In this methodology, the rules generating template SRAFs are changed algorithmically to minimize the process variation metric.

Calibre cnSRAF Template Execution

Calibre cnSRAF sequentially applies SRAF templates according to assigned priorities. Templates are applied to the layout in order of assigned priority. Each template has a different priority that controls how SRAFs are kept, cleaned, or modified during application. A priority of 1 has the highest priority (and the greatest influence) and is applied first, then a priority of 2 is applied, and so on. As SRAFs generated by subsequent templates are cleaned, only currently processed SRAFs that fit with the previously cleaned SRAFs remain.

After each template application, SRAFs that form conflicts or fail to fit are removed. Applying successive templates can fill the empty conflict regions. Defining the behavior of conflict regions is important to control the transition from one template to the next.

Exclusion Zones

Exclusion zones apply between SRAFs and target shapes, and between SRAFs with different priorities.

Typically, the templates must be defined for various densities. As the spacing between target shapes decreases, the placement of SRAFs may violate spacing rules with another shape. For example, the SRAF might be placed on a shape. In this scenario, the SRAF is considered to be within an exclusion zone defined by the geometry of the target and the manufacturing rule constraints (MRC). To ensure that the SRAFs are clean and do not violate MRC rules, they are automatically excluded from these areas.

Calibre mbSRAF Template Execution

Calibre mbSRAF applies SRAF templates with the same prioritization as Calibre cnSRAF.

Score-Based Cleanup

Calibre mbSRAF cleans up SRAFs by their score. The score is calculated as the integral of the SRAF area over the underlying gradient, and the best match has the highest score. Every SRAF candidate receives a score and the highest score is placed with highest priority. The placement and cleanup of all subsequent SRAF candidates are performed in descending order, until all candidates have been either placed or discarded. All MRC restrictions are maintained while operating on each SRAF candidate. Some candidates are eliminated because of their failure to comply to MRC constraints.

Template Methodology Implementations

Two general methods of template implementation are currently supported:

Multiple localmaxgradient templates

This method requires each localmaxgradient template to specify its own width keyword, without using the maxoffset keyword. This methodology is especially recommended for line layers:

localmaxgradient width 0.016 squarewidth 0.023 prior 1
localmaxgradient width 0.020 squarewidth 0.025 prior 2
...
Single localmaxgradient template

This method requires one localmaxgradient template to specify a width keyword along with the maxoffset keyword:

localmaxgradient width 0.018 squarewidth 0.024
...
maxoffset 0.120

Multiple Template Interaction With maxoffset

Using multiple localmaxgradient templates, each template is specified with one maximum width for its generated SRAF (the localmaxgradient width parameter). Multiple localmaxgradient templates allow generation of multiple SRAF widths. The SRAFs are usually generated in a pattern of smaller widths placed closer to the target. SRAFs of increasingly larger widths are placed farther from the target. This pattern minimizes the risk of SRAF printing.

During multiple template use, each template places its SRAFs on the next available seed pixel radiating outward from the target shape. The maxoffset keyword specifies a radial limit for SRAF placement. All templates attempt to place SRAFs within the specified maxoffset value.

If multiple templates are used with maxoffset, the first template places SRAFs at the highest priority over all existing seed pixels outward until maxoffset is met, then the template with the next highest priority performs its placement, continuing until placement is completed by the lowest priority template. In most cases, the first template covers greater than 90 percent of all seed pixels. Subsequent attempts to place SRAFs of differing widths by lower priority templates fail.

Note:

Although multiple templates may work jointly with maxoffset with specific recipes, Siemens EDA recommends against using the maxoffset keyword with multiple localmaxgradient templates requiring placement of varying SRAF widths.

Specify maxoffset when using a single localmaxgradient template. The maxoffset keyword determines the maximum distance from the target shape that SRAFs are generated. The number of SRAF rows to be generated is unknown at compile time.

Example 1. Use maxoffset With a Single Template
sraf_options {
	...
	algorithm 1
	maxoffset 0.250
	localmaxgradient width 0.045 prior 1
}

You should not specify maxoffset when using multiple localmaxgradient templates. The templates determine the widths and distances from the target shape from which SRAF rows are generated.

Example 2. Do Not Use maxoffset With Multiple Templates
sraf_options {
	...
	algorithm 1
	localmaxgradient width 0.050 prior 1
	localmaxgradient width 0.045 prior 2
	localmaxgradient width 0.040 prior 3
}
