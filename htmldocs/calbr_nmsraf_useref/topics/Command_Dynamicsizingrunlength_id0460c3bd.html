dynamicsizingrunlength

Cleanup Keywords

Disables centermerge emphasis on long SRAFs. Used by RET SBAR only.

Usage

dynamicsizingrunlength value

Arguments

1 | 0

Either 1 or 0, disabling or enabling dynamicsizingrunlength, respectively.

Description

The dynamicsizingrunlength keyword controls centermerge emphasis over long SRAFs. A long run length is determined by the common length of SRAFs in conflict that exceed one-third the length of each.

By default, dynamicsizingrunlength is set to 1, and centermerge emphasis is disabled, consistent with SRAF generation behavior prior to Calibre release 2017.3. Setting dynamicsizingrunlength 1 more often results in two unmerged long SRAFs normally processed by dynamicsizing during cleanup. Setting dynamicsizingrunlength 0 enables emphasis of centermerge during cleanup of long SRAFs, typically resulting in a single long SRAF output in the area of conflict. This keyword may only be specified globally.

Note:

Siemens EDA recommends using dynamicsizingrunlength 0 for line layers when invoking cleanupversion 1.
