newtag pairtag

Tag Keywords

Specifies a tag name, exclusively for pairtemplate, that represents layers or edges for which to generate SRAFs. Used by RET SBAR only.

Usage

newtag name pairtag
distance range
[anchorlayer name]
[angle range]
[projection range]
[shieldinglayer name]
[shielding {line | none}]

Arguments

name

A required name specifying the name of the tag.

pairtag

A required keyword invoking the pairtag tag. This tag may only be used for pairtemplate.

distance range

A required keyword specifying the length of generated connectors. A connector is defined as the distance between the centers of shapes on the named anchor or target layer. The range must contain an upper bound. You specify the range in microns. In Figure 1, the target shapes fall within the range specified as greater than 30nm, but less than 80 nm.

Figure 1. The distance Keyword

newtag sraf pairtag distance >0.030 <0.080
anchorlayer name

An optional keyword specifying the name of the anchor layer used to generate connectors. By default, the anchor layer shapes are formed from bounding boxes derived from polygons on the target layer. The anchorlayer must contain rectangle polygons only. If anchorlayer is specified, it is the only layer used during pairtag generation.

You must declare the anchorlayer within the sraf_options block prior to specifying it in newtag pairtag. In Figure 2, the target shapes have been oversized and declared as an anchorlayer. In this example, the edges fall within the range specified by the projection keyword.

Figure 2. An anchorlayer Application

newtag sraf pairtag distance 0.080
	anchorlayer os_target projection > 0.010 <= 0.040
angle range

Specifies a range of values selecting the minimum angle between the connector and X-axis in degrees (irrespective of the angle’s sign). You must specify a range between 0 and 90 degrees. These figures represent typical SRAF generation possible by pairtemplate seeded by the connectors generated by pairtag.

Figure 3. The angle Keyword

projection range

Specifies a range of mutual projection of rectangular bounding boxes on the anchorlayer. The connector is not generated if mutual projection of the corresponding rectangles is outside the specified range. You use projection less than 0 to generate connectors between rectangles of the anchorlayer which are not projected on each other. In Figure 4, the target layer has been oversized to provide for the anchorlayer. The anchorlayer edges provide an acceptable projection range.

Figure 4. The projection Keyword

shieldinglayer name

An optional name that specifies the name of the shielding layer. The shieldinglayer contains shapes used to hide potential paths of connectors. By default, the target layer is used as the shielding layer.

shielding {line | none}

An optional keyword that specifies the shielding mode. The line option allows connectors to be generated only if its segment does not intersect any shape on the shielding layer. This is the default behavior. None indicates that no shielding is performed.

Description

An optional keyword used to input tags by name to the pairtemplate template. The pairtag keyword generates connectors that extend from the centers of two target shapes or shapes on the anchorlayer. Connectors are qualified by their lengths, degree of angle from the X-axis, and local shielding. Figure 5 shows a connector, fixing the centers of neighboring shapes (within the specified constraint), ready to be passed to pairtemplate for SRAF processing. The connector has an associated slope and length.

Figure 5. Connector

Only manhattan- and 45-degree input layer geometries are supported by pairtag. No skew-edged geometries are valid target shapes for SRAFs.

CAUTION:

Using constraint values that exceed cell or tile boundaries may cause flattening and increase run times.

Examples

Example 1 — Usage with pairtemplate

newtag pt1 pairtag distance >=0.1 <0.3 projection >= 0.04
newtag pt2 pairtag distance >=0.1 <0.2 projection >= 0.02
pairtemplate tag pt1 width 0.040 length 0.040 offset 0.120
	angle 45 orient ortho prior 10
pairtemplate tag pt2 width 0.020 length 0.070 offset 0.085
	angle 0 orient ortho prior 20

Example 2 — Pairtag using angle.

#Generates connectors between shapes on the anchorlayer with distances
	between 100 and 150nm and an angle of less than 45 degrees

newtag c2ctag pairtag anchorlayer anchor_l distance >0.1 <0.15 angle <45

Example 3 — Comprehensive Flow using shieldinglayer and anchorlayer.

pair_CN = RET SBAR target anchor_lay shield_lay FILE cn_setup MAP pair_CNLITHO FILE cn_setup [
	layer target
	layer anchor_lay
	layer shield_lay
	tilemicrons 10
	sraf_options sraf_setup {
		cleanupversion 1
   		anchorlayer anchor_lay
		shieldinglayer shield_lay
		maxconflict 0.03
		minoffset 0.01
		minfeaturespace 0.01
		centerthresh 0.03
		dynamicsizing 0.01
		newtag  short_pt pairtag distance > 0.01 <=0.300
			anchorlayer anchor_lay
			shieldinglayer shield_lay shielding Line
		pairtemplate tag short_pt width 0.017 length 0.050
			angle 0 orient normal prior 1
		priority 1 minfeaturespace 0.040 sbwidth 0.0175
			sraf 0.038 cleanmode postsizing
	}	setlayer pair_CN = cnsraf target OPTIONS sraf_setup
	setlayer 1_tags = cnsraf debug pair_CN tags 1
	setlayer 1_pre = cnsraf debug pair_CN preclean 1
	setlayer 1_conflict = cnsraf debug pair_CN conflict 1
	setlayer 1_processed = cnsraf debug pair_CN resolution 1
]
