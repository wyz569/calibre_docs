cornerpref

Style Keywords

Specifies the SRAF generation method at target shape corners. Used by RET MBSRAF only.

Usage

cornerpref {all | horizontal | no | square | squareonly | squareonlyscore | vertical}

Arguments

all

Generates rectangular SRAFs aligned to all target edge axes. Rectangular SRAFs take priority over square SRAFs with this argument.

horizontal

Generates rectangular SRAFs aligned to horizontal target edge axes.

no

This method inhibits SRAF generation at corners. If algorithm 1 is set, no is the default setting.

square

Generates a square SRAF at seed pixels near the corner of a target shape by the area product of either localmaxgradient keywords width2 or squarewidth2. As the localmaxgradient template determines, square SRAFs may be changed to rectangular SRAFs, given necessary alignment criteria. SRAF candidates generated by square are allowed to project from corners into target edges. Cleanup of SRAFs generated by square is score-based (see “Calibre cnSRAF Template Execution” for more information regarding score-based cleanup). If algorithm 0 is set, square is the default setting.

squareonly

If localmaxgradient keyword squarewidth is defined, squareonly generates square SRAFs at seed pixels near corners or jogs with an area of squarewidth2. The squareonly option does not allow extension of square SRAF candidates to project into target edge space. Unlike SRAFs generated by the argument square, no SRAFs generated by squareonly can be changed to rectangles. Cleanup of SRAFs generated by squareonly is performed first on square SRAF candidates.

squareonlyscore

The squareonlyscore option performs SRAF generation in the same way as squareonly, but performs cleanup in the same way as square. Like squareonly, if localmaxgradient keyword squarewidth is defined, squareonlyscore generates square SRAFs at seed pixels near corners or jogs with an area of squarewidth2. The squareonlyscore option does not allow extension of square SRAF candidates to project into target edge space. Like SRAFs generated by the argument squareonly, no SRAFs generated by squareonlyscore can be changed to rectangles. Cleanup of SRAFs generated by squareonlyscore is score-based, the same as square.

vertical

Generates rectangular SRAFs aligned to vertical target edge axes.

Note:

Regarding algorithm interactions:

  • If algorithm 0 is set, square is the default cornerpref setting.
  • If algorithm 1 is set, no is the default cornerpref setting.

Description

An optional keyword that specifies a single SRAF generation method at target shape corners. This keyword may be specified globally, by priority, or within the localmaxgradient template.
