Gauge Data Objects Defined

Gauge data objects are the internal representation of gauges for a layout.

A gauge data object (purple lines in Figure 1) measures the critical distance with one or more markers that span the critical distance. Gauge data measurements include the exact width of the markers.

Figure 1. Gauges in a Layout

Gauge data objects can also be converted to gauge measurement objects that contain information on the gauges in relation to the test structures.

Gauge data objects can also be used in conjunction with dense printimage operations.

Gauge objects differ from spreadsheet simulation sites in the following ways:

  • A gauge object is not necessarily centered on the nearest edges.

  • A gauge object’s location is determined by the coordinates of its start (x1, y1) and end (x2, y2) point.

  • Gauge objects can be used for modeling variable etch bias.

Methods for Creation of Gauge Data Objects

Gauge data objects can be created using multiple methods.

  • Spreadsheet Files as a Gauge Data Source (ss2gauge) — You must create a spreadsheet for use with an associated test pattern file.

  • Gauge Files as a Gauge Data Source (gaugeRead and gaugeWrite) — You can re-use gauge data files between sessions.

  • Layout File as a Gauge Data Source (layout2gauge) — You can also manually draw paths to represent gauges.

Spreadsheet Files as a Gauge Data Source (ss2gauge)

One of the ways to create a gauge data object is using the ss2gauge command, which takes as inputs a spreadsheet file and the corresponding test pattern.

The source spreadsheet file can be the standard output of the Calibre WORKbench test pattern generator, or one you create using some other tool. It must be in the following format (as defined in the section “Sample Spreadsheet File Format”): 

structure_name row column x y loc drawn other meas weight

The input spreadsheet object is converted to a gauge object. The following gauge-specific arguments to the ss2gauge command (stdev, count, and 95% ci) are appended by Calibre WORKbench to the spreadsheet data format to form the gauge data object format.

Note:

If you use the Test Pattern Generator in Calibre WORKbench, you must fill in the sample spreadsheet meas column with measured sample data before you use the ss2gauge command.

  • measured_etch_CD — Support for empirical etch data, used in Variable Etch Bias modeling mode.

  • simulated_etch_CD — Support for simulated etch data, used in Variable Etch Bias modeling mode.

  • mode — (pre-2009.1, replaced by stdev) Sets treatment for sub-gauges, if present: 0=normal, 1=average, 2=min, 3=max.

  • stdev — Sets the standard deviation in nanometers for characterizing measurement statistics. This input parameter is used along with count to calculate the 95 percent confidence interval (95% ci field). Set this value to a positive value to use it; set it to -1 to deactivate it.

  • count — Definition has changed starting with 2009.1. This field represents the number of samples used to obtain the stdev value.

    Previous to 2009.1, this field was used to indicate the number of sub-gauge markers to place at each location left and right of the main gauge (2 * count +1 markers total). This functionality is no longer supported.

  • 95%ci — Starting with 2009.1, this output field contains the calculated confidence interval based on the following equation:

    95%ci = t-value * stdev / sqrt(count)

    where t-value is a two-tailed Student’s probability of 0.05 with count-1 degrees of freedom, and stdev and count are the input values supplied above.

  • dist — Previous to 2009.1, this field specified the distance between subgauge markers in dbus. It was discontinued in 2009.1 and replaced by the 95% ci field. No replacement functionality exists.

  • group — Specifies a group name consisting of one word, which can be no longer than 256 alpha-numerical, +, -, or underscore characters. The blank space is not allowed. Specifying a value NA, which is uses as the default, or -1 means that the group name is not specified. If all group names of enabled gauges are not NA or -1, this column is used when performing error analysis in modelflow_v2 and sgd_get_error_stat Tcl commands. The data in this column is also used when computing modelflow_v2 group-based objectives, in which case using the default names NA and -1 is not allowed.

  • sempicture — Specifies a regular expression that specifies the path to SEM pictures used for determining measured CD.

Gauge Files as a Gauge Data Source (gaugeRead and gaugeWrite)

Gauge data objects can be saved to a gauge data file using the gaugeWrite command. The gauge data file can be used as input in later sessions using the gaugeRead command.

A gauge data file uses the following format: 

flag structure_name row col x1 y1 x2 y2 loc drawn other meas_CD \ 
simulated_CD meas_etch_CD simulated_etch_CD precision weight stdev \ 
count 95%ci group sempicture [# comment]

The structure_name, row, col(umn), loc(ation), drawn, other, meas(ured CD) and weight values are standard Sample Spreadsheet File Format. The following values are added or modified by Calibre WORKbench:

  • flag — Specifies the enable (1) or disable (0) flag. Only enabled gauges are considered during calibration.

  • x1 y1 x2 y2 — Specifies the coordinates of the beginning and end of the gauge point in database units.

  • measured_CD — Specifies the measured CD space in nanometers (nm).

  • simulated_CD — Specifies the simulated CD space in nanometers (nm) for resist models. The contents of this column are generated as output from modelflow_v2, or are set to -1 if no simulation data was generated.

  • meas_etch_CD — Specifies the measured CD space in nanometers (nm) for etch model information.

  • simulated_etch_CD — Specifies the simulated CD space for etch model information. The contents of this column are generated as output from a modelflow_v2 command. or are set to -1 if no simulation data was generated.

  • precision — Specifies the CD tolerance or absolute error in nanometers (nm) with which the empirical measurement (meas_CD) was performed. During model calibration this parameter is used to calculate the number of gauges that satisfy the inequality:

    |meas_CD - simulated_CD| <= precision

    The number of gauges that are within the precision tolerance range are output to the RUN SUMMARY section of the log file.

  • weight — Specifies a dimensionless non-negative quantity for the weight to be used for the gauge and associated CD space data. This weight is used as a cost function during model calibration.

  • stdev — Contains the standard deviation in nm for measurement statistics. Used with count to calculate the 95% confidence interval. This column is used for error analysis.

  • count — Contains the number of samples used to obtain stdev.

  • 95%ci — Contains the 95% confidence interval computed based on stdev, count, and the T-value of a two-tailed Student’s Probability of 0.05 with count-1 degrees of freedom: 

    95%ci = t-value * stdev / sqrt(count)

  • group — Specifies a DDM group.

  • sempicture — Specifies an absolute path to a SEM image file. Used primarily with the createsrafgauges command.

  • # — Specifies a comment. Anything after a hash (#) symbol on a line is considered a comment. Comments cannot contain commas.

In addition, the stdev, count, and 95%ci elements created specifically during gauge data object generation are added to the gauge file.

Layout File as a Gauge Data Source (layout2gauge)

Gauge data objects can be produced from a layout file containing gauges using the layout2gauge command.

The gauges must be wires (paths) drawn using the Path tool on a separate layer. The wires you draw must obey the following conditions:

  • Cross at least two edges in your design (either cross a feature or a gap between two features)

  • Be perpendicular to the underlying design polygon

  • Be long enough to cross the simulated contours (reference threshold)

    Note:

    The layer containing gauges should not be specified in the setup file.

If you use gauges from a loaded layout, you must ensure that you preserve the paths on the layout when you load it.

If your gauge has subgauges, you must specify the -ssg option in order to search for them.

Tip

If you get the message “The current loaded LAYOUT/LAYERS/GAUGES is not valid to be used in a simulation”, verify that the gauge (1) has the correct polarity for the feature of interest and (2) that the center is inside the polygon you want to measure.
