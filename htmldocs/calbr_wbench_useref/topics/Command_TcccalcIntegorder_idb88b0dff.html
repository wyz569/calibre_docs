TCCcalc_integorder

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Specifies the number of integration points per maximum source sigma to use in the numerical integration routines of TCCcalc.

Usage

TCCcalc_integorder num

Arguments

num

A required argument specifying the number of integration points per maximum source sigma to use in the numerical integration routines of TCCcalc. This number should not be smaller than eight or larger than 64. The default gives good accuracy for most practical optical modeling situations. Higher values than 16 are recommended in cases where high order aberrations are present or the illumination is specified through a file (MAP) and contains very rapid intensity variations. Default is 16.

Description

An optional miscellaneous parameter specifying the number of integration points per maximum source sigma to use in the numerical integration routines of TCCcalc.

For rapidTCC users, using a default value other than the default for this option has a very minor effect on the optical model. You may notice slight differences between aerial image contours created with TCCcalc and rapidTCC when used with a CM1 model that was originally tuned for a TCCcalc optical model. These differences may be more noticeable with certain kinds of CM1 models, such as in process window underexposure conditions.

  • If you switch from TCCcalc to rapidTCC as your engine, it is advised that you remove your TCCcalc_integorder setting and recalibrate the resist model with the new optical model.

  • If you want to perform a comparison between TCCcalc and rapidTCC, use the same setting for TCCcalc_integorder in both optical models.
