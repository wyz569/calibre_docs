modelflow_v2: Input Section Arguments

Parent topic: modelflow_v2

This section specifies input files and settings for the modelflow_v2 command.

Usage

modelflow_v2
{-m gds | -l layout}
      {-gf gaugefile | -sgf supergaugefile}
      [-heightsgf file [-heightweight value] [-heightznum value]]
      [-jogtolerance integer]
      [-f setup | -mlfile filename | -mlstring string]
      [{-cli clifile | -clistring clistring} [–ccalweight value]]
      [-turbo | –remotefile file | –remotestring string]
      [-foptions parallel]
      [-ftolerance normtol]
      [-cachenumber]
      [-cache {ignore | use | clear}]
      [-cacheload dir]
      [-frontlogfile filename]
      [-anchor {gauge_name |gauge_number} [gauge_set]]
      [-noNyquist]
      [-objective setting ]
      [-inputextraclip {value | auto}]
      [-srafcalbyweight]
      [-sraferrweight value]
      [-srafgaugeweight value]
      [-srafbotplane value]

Arguments

One of -m or -l is required, and -m and -l are mutually exclusive.

-m gdsfile

Specifies the input test pattern.

-l layout

Specifies the input layout handle with the test pattern.

One of -gf or -sgf is required, and -gf and -sgf are mutually exclusive.

-gf gaugefile

Specifies the input gauge file.

-sgf supergaugefile

Specifies the input super gauge file.

-heightsgf file

Specifies an input supergauge file for resist height measurement data. This option is only used for 3D resist model calibration (see “3D Resist Model Calibration Flow”), and is not an alternative to the –gf or –sgf options. The height data super gauge data file uses the same format as a super gauge data file, but with CD numbers replaced by height numbers.

-heightweight value

Specifies the weight used for the height calibration data as a floating point value between 0 and 1 relative to the weight of the CD gauges. A value of 1 means that regular CD weights are ignored; a value of 0 means that gauge height information is ignored. (Default is 0.5, which is equal weights height CDs and standard CDs.) This option can only be used when the -heightsgf option is also specified, and is only used for 3D resist model calibration.

-heightznum value

Specifies the number of z-planes simulated through the resist film for resist height calculation as an integer between 2 and 200. (Default is 21.) This option is only used for 3D resist model calibration.

-jogtolerance integer

Specifies the input jog tolerance in dbu. The jog tolerance is used when determining gauge dimensionality; a gauge is considered to be a 1D gauge as long as there is no edge paralllel to the gauge whose length is greater than the specified value within the gauge’s interaction radius. integer must be a non-negative value. If not specified, the default is 5.

–f setup

Specifies an input setup file.

-mlfile filename

Specifies an input model-layer file (see the section “Model-Layer File Format” for more information).

-mlstring string

Specifies an input model-layer string. If you are using flare models, they should be defined using this argument.

-cli clifile | -clistring clistring

Specifies an optional input contour layer info .cli file or file as a string. If this option is set, then the contour layers specified will be used to compute error RMS. The format of the contour layer info file is outlined in the “Importing Contour Calibration Files”. This file specifies the total number of contour layers (contour data sets) available, and for each contour data set it specifies the contour layer number, clip layer number, dose, defocus, and weight values of the process condition associated with this contour layer, as well as the weight of each contour point. If the process condition (dose and defocus) associated with a given contour layer is not present in the input super gauge file, this contour data will be neglected. (Default = no contour layer info)

–ccalweight value

Specifies the input weight of the contour layer data. (Default=0; can only be used when -cli or -clistring are also specified.) The following formula is used to relate this (ccal) weight to the weight of gauge data for a given process condition: 

Wccal + Wgauge = 1
–simpixsz value

This option has been disabled in the current version.

–turbo

If set, then multi-threading (parallel execution on multiple CPUS) will be used for all stages. (Default = no multi-threading)

Using the -turbo option does not require additional licenses. If set, it limits full searches and the exploration stages of the newton and gradient searches to 32 or fewer CPUs. During the newton and gradient search phases, it uses a maximum of 2 CPUs.

The number of threads to be used is determined by the number of CPUs available on your machine and the MT scalability of the search method chosen. By default, this number cannot be greater than 32. Multi-threading is also limited by the optional Calibre WORKbench invocation argument -threads.

For example, specifying the command-line invocation:

calibrewb -threads 2 mfv2_batch.tcl

on a 4 CPU machine will limit the number of simultaneously-running threads to 2.

–remotefile file

Specifies an optional input remote configuration file. If this option is set, then parallel execution on the machines specified in the remote file will be used to tune models during calibration. The total number of CPUs used cannot be greater than 128. The use of the ‑remotefile option does not require additional licenses.

This option cannot be used if the –turbo or –remotestring options are set. (Default = no distributed processing.)

Tip

This option should not be used if stage2 (CM1 optimization) is active; use -turbo instead.

–remotestring string

Specifies an input remote configuration string. If this option is set, then parallel execution on the machines named in the specified string will be used to tune models calibration. The total number of CPUs used cannot be greater than 128. The use of the -remotestring option does not require additional licenses.

This option cannot be used if the –turbo or –remotefile options are set. (Default = no distributed processing.)

-foptions parallel

Use this option to modify the behavior of the front algorithm when the -turbo, -remotefile, or -remotestring options are also set. It improves the scalability of the search by adding more front candidates. The scalability is most noticeable after 5*N iterations, where N is the number of CPUs or remotes available. The number of jobs run is forced to be a multiple of N.

Note:

Use of -foptions parallel may give slightly different models depending on how many remote CPUs are used. This is expected behavior.

-ftolerance normtol

Use this option to specify the norm tolerance, which is a non-negative number (0 <= normtol <=1) that controls the accuracy of the front optimizer (convergence criterion). Setting this parameter to a value greater than FLT_EPSILON (the default) prevents the optimizer from sticking at the lowest local minimum and forces it to better explore other local minima.

-cachenumber

Use this optional flag to return to the Tcl shell a string that contains the number of aerial image grids available in the cache in the format -cachenumber value.

–cache {use | ignore | clear}

Use this flag to enable (use), disable (ignore), or clean up (clear) the optical grid cache. (Default = use.)

The –cache option is designed to speed up simulations when you make multiple calls to the modelflow_v2 command. Each call generates a set of aerial image grids centered at the gauges involved. If the option use is specified, this will allow you to repeatedly use the previously-generated grids provided that the optical model setup and/or the maximal value of diffusion lengths have not been changed between the calls.

–cacheload dir

Use this flag to load previously generated aerial image grids from disk to cache. (Default = none.) Ignored if stage1 is present or an error occurs when reading grids from the specified folder.

Note:

For input gauge data, only entries with the enable/disable flag set to 1 will be used in simulations (for the gauge data format, see “Gauge Object Reference”).

-frontlogfile filename

Use this option to load a previously generated modelflow log and resume a search operation starting with the last recorded iteration in the log. This option is only accepted if the log file meets all of the following conditions:

  • one stage of optimization only is set (stage1 or stage2 or stage 3)

  • the front search method was used

  • optimization parameters, dependencies and intervals exactly match the settings in the log file

-anchor gauge_name | gauge_number [gauge_set]

Use this option to specify an anchor gauge by either its gauge_name or its gauge_number index, and (for super gauge data) which gauge_set it is in. An anchor gauge is a feature whose simulated CD is kept equal to its measured CD during calibration and model building. Both gauge_number and gauge_set must be positive integers. The default for this option is to calibrate the model without enforcing restrictions on simulated CDs.

This option requires stage1 or stage2 to be active. This option can be used if either the -gf or -sgf gauge input options are also specified. If a gauge_set value is not specified for super gauge data, gauge_set 1 is used.

This option is not compatible with the stage1 option threshold {fixed_threshold or optim_set}.

Tip

Use of this option is a recommended best practice for optical (stage1) and CM1 (stage2) calibration, when you have a known anchorable feature available.

In CM1 Center, use the Input field in the Misc tab to add the -anchor option, specifying the gauge number of its row from the Gauges tab as a required argument. If the anchor gauge is not on the G1 subtab in the Gauges tab, add the integer for the gauge subtab. (For example, an anchor gauge on row 17 in tab G3 would be specified as -anchor 17 3.)

-noNyquist

This option is documented for backwards compatibility and use only with pre-2009.4 modelflow_v2 runs; its behavior is the default starting with the 2009.4 release.

-noNyquist turns off the intensity interpolation and computes aerial image profiles directly on fine grids (instead of coarse grids). Its use is recommended for any pre-2009.4 modelflow_v2 runs. Note, however, that the use of this option can result in longer calibration times during stage 2 on calibration data with a very large number (> 500) of 2D features.

In typical cases (30% 2D gauges, 70% of line-space gauges), the optical model calibration (stage 1) is expected to be 10-20% slower, and stage 2 is expected to be 50-90% slower.

Siemens EDA considers the accuracy and consistency gains to be worth the expected slowdown; for pre-2009.4 releases, -noNyquist should be specified.

Tip

In the Calibre WORKbench CM1 Center GUI, you specify this option on the Misc tab, Input field for pre-2009.4 releases. You do not need to specify it starting with version 2009.4.

  • range — Uses a range objective function defined as the difference between the upper and lower limit of the deviation between the simulated and measured CDs.

  • abserr — Minimizes the absolute error, defined as the maximal difference between simulated and measured gauge CDs.

    Note:

    While calibrating a resist model, its linear parameters are optimized by minimizing the RMS error irrespective of whether -objective is set to rms, range, or abserr.

  • errrate — Minimizes the error rate. It instructs the tool to calibrate models by minimizing the predicted error rate for SRAF printing. By default, the overall error rate is calculated according to the following formula:

    overall error rate = (# of false positive errors + # of false negative errors) / ( total # of SRAF gauges)

    A false positive error happens when the wafer measurement shows no printing but the SRAF model indicates there is printing. A false negative error happens when the wafer measurement shows printing, but the SRAF model indicates there is no printing.

    If errrate is selected and stage2off is also specified, a calibrated CTR resist model is written to the output litho model.

    Note:

    Specifying the -sraferrweight argument (described following) modifies the behavior of this argument. If the -sraferrweight value option is also specified, the weighted error rate is used.

  • fixgroup | floatgroup

    Used for fix-group and float-group based objectives, respectively. VEB linear coefficients are determined by minimizing the RMS error.

    • fixgroup — Limits the modeling errors using the fixed bounds defined by the tolerance and shift values in the group file, for a fixed range defined by the tolerance on either side of the center plus the shift. Any errors that lie outside the boundaries are penalized.

    • floatgroup — Limits the modeling error spread by using a floating bounds range that is inside the tolerance range and near the shift value defined in the group file. The tolerance range in this case is explored around the shift value, but the left bound is always negative and the right bound is always positive.

    Cannot be set if –cli or –clistring is present. The fixgroup and floatgroup options requires that none of the group names of any enabled gauges be specified as “NA” or “‑1”.

    Requires that maxlinear be set.

    Group options must be followed by one of the two options for reading group specifications from a file (described in “Group File Format”) or string, respectively gsfile file or gsstring string

  • pwflat — Forces the PW RMS error as a function of defocus to be as flat as possible. This option can be set only if the super gauge data involves more than one defocus value.

    This option is incompatible with the -cli and -clistring options.

  • pwsymmetric — Forces the PW RMS error as a function of defocus to be as symmetric as possible. This option can be set only if the super gauge data are symmetric with respect to defocus d = 0. Symmetric is defined as both of the following being true:

    • Each process condition uses a different defocus value, and there are at least two conditions.

    • For each gauge set with positive defocus d there is also a gauge set with negative defocus -d.

    This option is incompatible with the -cli and -clistring options.

-inputextraclip {value | auto}

This optional argument adds the specified length in um to gauge coordinates when Calibre WORKbench performs layout clipping. The default length is fixed at 5 um; previous to the 2016.2 release, this value was computed internally. To keep the old behavior, specify “auto” for this option.

-srafcalbyweight

This optional argument is used only for SRAF calibration flows. If set, the gauge weight for each gauge data object is used in the SRAF printing prediction error rate calculation.

-sraferrweight value

This optional argument is only used when -objective errrate is also specified. By default, the overall error rate is used. When -sraferrweight value option is specified, the weighted error rate is used, according to the following formula:

weighted error rate = (1-value) * false positive error rate + value * false negative error rate

where

  • false positive error rate = # of false positive errors / # of non-printing gauges

  • false negative error rate = # of false negative errors / # of printing gauges

-srafgaugeweight value

This optional argument is only used when -objective errrate is also specified. It specifies the optional input weight between CD gauges and SRAF gauges for SRAF printing prediction and model calibration, where value is between 0 and 1 and uses the following formula:

weighted objective = (1-value) * CD errRMS + value * SRAF objective

-srafbotplane value

Optionally specifies the input metrology plane for SRAF gauges near the resist bottom for the SRAF printing prediction model, where 0 < = value < resist height.

This option can only be used with Super Gauge Data File Format files of version 2 or higher.

Description

Definitions

  • CTR — Constant Threshold Resist Model of type CM1.

  • Nominal optical, resist, and etch models are the initial models specified in the setup file.

  • optim_set — Defined as one of the optimization expressions as described in Using the Independent, Constrained, and Dependent Optimization Parameters of “modelflow_v2: Stage 1 Optimization Run Modes”.
