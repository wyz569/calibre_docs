kernel

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Defines a function to replace or convolve with the default SOCS kernel.

Usage

kernel N [ TYPE type] [ SCALE scale] [ SQUARE yes | no ] [ USESOCS yes | no ]

Arguments

N

A required argument specifying which kernel to apply changes to.

Must be an integer value. Allowed values are between 0 and M-1, where M is the number of optical model kernels as specified in the Number of Kernels field in the Optical Model Tool.

TYPE type

An optional argument set specifying the function replacing or convolved with the default SOCS kernels.

When TYPE is omitted, the default SOCS kernel is used. The string comprising type defines the function which will either replace or be convolved with the default SOCS kernel. It can be either:

  • tophat out_radius in_radius

    • out_radius — Outer radius of circular tophat in microns.

    • in_radius — Inner radius of circular tophat in microns.

  • gauss sigma

    • sigma — Standard deviation of the Gaussian in microns. (This is the definition for this particular instance of sigma only.).

SCALE scale

An optional argument set specifying he scale on the output of a kernel in the SOCS system. The default behavior is to use the eigenvalue computed by the SOCS decomposition.

SQUARE yes | no

An optional argument set specifying whether to square the output of a kernel.

USESOCS yes | no

An optional argument set specifying whether or not the SOCS kernel is used. Allowed values are yes and no.

  • yes — The SOCS kernel is used, convolved with the user-defined kernel, which will smooth it. The default is yes.

  • no — The SOCS kernel is replaced by the Gaussian kernel, using the extent defined by the hoodpix parameter.

Description

An optional SOCS/Kernel parameter that modifies the default SOCS kernel.

This parameter allows a function to be either convolved with, or to replace, the default SOCS kernel using the specified parameters. There is an option for not squaring the kernel output value and choosing the scaling value of the kernel. All kernel types work with both sparse and dense imaging unless otherwise specified. The default is no kernel modifications, which uses unmodified SOCS kernels. Figure 1 provides a graphical explanation of the functionality provided by this keyword.
