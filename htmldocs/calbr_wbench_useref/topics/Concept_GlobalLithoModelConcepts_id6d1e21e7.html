Global Litho Model Concepts

An EUV through-slit litho model (referred to as a global litho model) is a set of multiple litho models in subdirectories collected under a parent litho model. Each litho model subdirectory contains a variation of a DDM and an optical model, specified at different x-coordinates in the EUV field.

Calibre nmModelflow has the following support for global litho models:

  • Create global litho models in one of the following ways:

    • To create a global litho model manually, see “Creating a Global Litho Model Manually”.

    • To create a global litho model using the GUI, see “Creating a Global Litho Model in Calibre nmModelflow”

  • Load global litho models in the Database Browser

    Use the Database Browser, Litho Model section, Import button to load an existing global litho model.

  • View global litho models in the Database Browser

  • Calibrate litho models as an optical model stage

    No special variables need to be set. All optical models inside a global litho model are set to the same tuned parameters for each calibration test. While Calibre nmModelflow supports calibration with through-slit data using global litho models, it is recommended to do all EUV model calibration with a center slit model and use a global litho model to verify the calibrated resist model with through-slit data.

Global Litho Model Organization

Each litho model in a global litho model contains an optical model and a DDM model.

  • The DDM library is one of the differentiators between litho models. Each individual DDM library has a unique phi value for its chief_ray setting.

  • The optical model is another possible differentiator. If the optical model contains objective pupil or source pupil information, those can be varied across litho models.

While each litho model requires a resist model and a flare model, these must be constant across the entire global litho model. For best results, use symbolic links to link each litho model’s resist and flare models to a common resist and flare model in the parent directory.

Best Practices for Litho Model Selection in Global Litho Models

For each calibration measurement, the litho model is based on the center of an individual tile being tested. The litho model used is the one closest to the center of the tile (“dynamic_generation none” setting), or as an interpolation of the two nearest slit locations (“dynamic_generation ddm” setting, which is the default).

  • Use nine DDM libraries for the best balance between accuracy and speed. DDM library locations should be evenly spaced through the slit if possible. If even spacing is not possible, then the spacing should be denser at the slit edges, because the effects change faster at the edges than the center.

    Given an azimuthal angle range of (-109, -71), the recommended phi values are ‑107, ‑102, ‑98, -90, -82, -78, and -73 degrees for seven DDM libraries, plus two DDM libraries outside the phi range of angles at -110 and -70 degrees (Xs in the figure above). When specifying values for phi in a list in the chief ray parameter, each value must be listed in increasing order.

  • For better performance, avoid using more than nine DDM libraries.

    • Runtime for masksim generation increases linearly when you increase the number of DDM libraries.

    • Memory usage on remote cores increases at a rate of 26 MB for each additional DDM library, and 11 MB more for each additional optical model.

    • Calibre nmOPC, Calibre OPCverify and Calibre nmModelflow runtimes do not increase with multiple DDM models. However, a runtime increase is expected for multiple optical models with dynamic model generation, but no runtime increase is expected with multiple optical models without dynamic model generation.

  • The maximum expected slit location is +/- 15000um.
