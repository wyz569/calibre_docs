Optical Parameters File Format

Optical Model File Formats

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Generated by: Calibre WORKbench Optical Model Tool

The Optical Parameters file format defines the parameters for an optical model. The format can be included inline in a setup file

Format

This file must conform to the following syntax restrictions:

  • Only one keyword per line.

  • All keyword values must appear on the same line as the keyword name.

  • Keyword names must begin in the first column of the line.

  • The comment character ‘#’ indicates that a row is a comment, in which case it can be unformatted text.

Optical Model Compatibility

Starting with the 2017.3 release, a generated optical model file has a comment at the top of the file containing its optical model compatibility (OMC) number, similar to the following:

#Built with v2018.4_0.5; OMC number 2

Whenever a change to the optical model format might have an effect on simulation results, the OMC number is incremented in the regenerated optical model file. Optical models from different Calibre WORKbench releases but having the same OMC number are considered functionally equivalent and compatible. Optical models with different OMC numbers may be incompatible when used together.

Runtime Monitor

To perform a runtime analysis of the optical model calculation, set the environment variable PRINT_OPTICAL_MODEL_RUNTIME_INFO to 1. The runtime statistics are printed to the stdout (screen) when executing optical model generation commands (opticsgen and modelflow). The total time is printed along with the time for each optical model step (TCC computation when applicable, SOCS decomposition and lookup table calculation). The output of the monitor is similar to the following: 

OPTICAL MODEL RuNTIME: total=30.0 s - TCC computation=20.0s (66.8%)
SOCS-decomposition=9.5s (31.8%) - lookup tables=0.4s (1.4%)
