createsrafgauges

CM1 and VEB Commands

Generates a gauge or super gauge data file with SRAF image data from a SEM image.

Usage

createsrafgauges -l layout_file -i gauge_file -o output_gauge_file -s setup_file.in

Arguments

-l layout_file

A required argument specifying a layout file, which this command uses to identify SRAF regions in the SEM image. Only OASIS format layout files are accepted. SRAF geometries must be on their own layer.

-i gauge_file

A required argument specifying an input SRAF gauge or super gauge data file containing SEM image information at each gauge location.

  • Gauges must be centered on the SRAFs.

  • The SEM image name must be present in the sempicture column (the second to last column, right before the Comment column) of the gauge or super gauge data file for every SRAF gauge entry.

-o output_gauge_file

A required argument specifying a gauge or super gauge data filename; the output of this command is written to the file. The “meas” column of the output file is set to either LT:0 (not printing) or GT:0 (printing). The gauge coordinates are set based on SRAF printing and the dimensionality of the main and SRAF patterns.

  • If both the main and SRAF patterns are 1D, the gauge is placed on the center of the SRAF polygon.

  • If both the main and SRAF patterns are 2D, the gauge is placed on the center of the SRAF polygon.

  • If the main pattern is 2D and the SRAF pattern is 1D, the gauge is placed on the SRAF polygon at the location of the largest printing blob.

  • If the main pattern is 1D and the SRAF pattern is 2D, the gauge is disabled because it is unclear which SRAF causes the extra printing.

If the SRAF polygon is not printing, the gauge is placed on the center of the SRAF polygon for all dimensionality cases mentioned above.

If the setup file’s setting for the gauge_center keyword is “main”, createsrafgauges generates an additional file using the super gauge data version 2 format. In this case, the output_gauge_file.sgd file created contains only new SRAF gauges, and the output_gauge_file_v2.sgd file contains gauges on the main pattern structures as well as the new gauges applied to the SRAF patterns.

-s setup_file.in

A required argument specifying a setup file containing configuration information for the createsrafgauges operation. (See Table 1.)

Description

Note:

The gauge output from this command is intended for use with Calibre nmModelflow; it cannot be used with CM1 Center.

The createsrafgauges command runs the Automatic SRAF Gauge File Preparation Tool, converting gauge or super gauge files that contain SRAF image data into gauge files that have printing or not printing information in them in the “meas” column. It reports statistics when the command completes successfully.

  • number of gauges not processed (meas = -1) / total input gauges

    In addition, the Automatic SRAF Gauge File Preparation Tool sets the “enabled” flag to 0 for output gauges whose meas field is -1, including for main pattern features that print poorly.

  • number of printing SRAFs (meas = GT:0) / total processed gauges

  • number of not printing SRAFS (meas = LT:0)/ total processed gauges

The setup file uses the following parameters:

Table 1. createsrafgauges Setup Parameters

Parameter

Description

version

A required parameter that sets the version number. Must be 1.0.

sem_images_base_path

A required parameter that specifies the absolute base path prefix to the SEM image files in the gauge or super gauge data file.

Note that this base path is used as the prefix for all filename arguments, so if the SEM image files include a partial path, the base path should not include the partial path in the SEM filename argument.

Note:

Only JPEG and TIFF SEM file formats are supported. If a SEM image is provided in another format, the “meas” column for that gauge is set to -1.

sem_pixel_size

A required parameter that specifies the pixel size of the SEM image in nm.

main_pattern_layer

A required parameter that specifies the layer number(s) in the OASIS file that contain only main pattern geometries.

Multiple layers can be specified in a comma-separated list. Layer numbers can be integers or integer.datatype format. For example, 1506.319 indicates layer 1506, datatype 319.

assist_feature_layer layernum[,layernum....] {SRAF | SRIF}

A required parameter that specifies the layer numbers in the OASIS file that contain only SRAF or SRIF geometries.

Multiple layers can be specified in a comma-separated list. Layer numbers can be integers or integer.datatype format. For example, 1506.319 indicates layer 1506, datatype 319.

max_alignment_error_x

An optional parameter that specifies the maximum possible misalignment between the center of the SEM image and the gauge location in a horizontal direction, in nm. The default value is 200.

Excessively large values (over 3000 nm) for this parameter (and max_alignment_error_y) should be avoided due to performance and memory requirements.

max_alignment_error_y

An optional parameter that specifies the maximum possible misalignment between the center of the SEM image and the gauge location in a vertical direction, in nm. The default value is 200.

sensitivity_threshold

An optional parameter that specifies the sensitivity of the algorithm that detects printing SRAFs. A lower value corresponds to lesser sensitivity of the algorithm and captures only strong signals. A higher value corresponds to higher sensitivity and captures weak signals; in other words, using a high value detects more printing, but may capture more noise in the process.

Note:

This option is only usable with segmentation_method 1.

The range of the sensitivity threshold is between 0.0 and 1.0, and is set to 0.7 by default.

sraf_dilation_factor

An optional parameter specifying a value in nm that is used to dilate the SRAF patterns in the assist_feature_layer list. The dilation is applied before the calculation of the percentage of printing area for the SRAF “blob”. Small values around 10nm may help detection. This parameter is set to 0 (off) by default.

This parameter is useful in situations where you need some margin around an SRAF pattern, so that the tool only captures the region affected by the SRAF.

area_ratio_threshold

An optional parameter that specifies a threshold value for deciding between printing and non-printing SRAFs. It is the ratio of printing SRAF in the SEM images to the area of SRAF in the layout file.

If the area ratio is larger than the threshold, it is considered printing; if it is less than the threshold, it is considered not printing. By default, the threshold value is 0.00001.

segmentation_method {1 | 2}

An optional parameter that sets the image segmentation method.

Specifying a value of 1 reverts to the original (canny edge) detection method.

Starting with the 2017.3 release, the default is 2, which causes createsrafgauges to use a watershed segmentation detection method.

sraf_type (deprecated)
Note:

Previous to 2016.4, this optional parameter was used to explicitly specify the SRAF type for gauges in the input file as 1D or 2D, because the 1D and 2D types needed to be separate.

Starting with the 2016.4 version, 1D and 2D gauges can be in the same input file. The 1D versus 2D determination is controlled by the optical_radius and jog_tolerance parameters.

The command treats 1D and 2D patterns as follows:

  • 1D patterns and non-symmetric 2D patterns — Each SRAF blob is analyzed independently, and the determination of printing or not printing is not influenced by other blobs.

  • 2D symmetric patterns — If the symmetry_checking parameter is set to true, each 2D pattern is tested for symmetry first. When symmetric SRAF patterns are found, the behavior of nearby SRAF pattern blobs for printing or not printing is taken into consideration for this blob.

optical_radius

An optional parameter that specifies a maximum distance (default value is 800 nm) from the center of the gauge to check for edges of different orientation. A 2D gauge has edges at different orientations; a 1D gauge does not.

jog_tolerance

An optional parameter that specifies the minimum size for an edge to be used to qualify a 2D gauge. The default value is 10 nm.

When determining 1D versus 2D gauges, the command only considers edges greater than jog_tolerance away from the edge with the gauge placed on it.

symmetry_checking

An optional parameter that when set to true activates automatic symmetry checking for the command. The default value is false, which causes the command to consider all 2D patterns to be non-symmetric around the gauge.

When activated, the command separates the 2D patterns from the 1D patterns (using the optical_radius and jog_tolerance parameters) and then checks them for symmetry.

stats_file

An optional parameter specifying the name of a file to write the output to a CSV-format file. The statistics, metrics, and properties can be helpful in determining a threshold that best differentiates between printing and non-printing SRAFs.

By default, statistics are saved to the file

~/.calibrewb_workspace/sraf_print/stats.csv.

consistency_check

An optional parameter that specifies whether a consistency check should be done or not. By default, this parameter is set to “true”, which checks each gauge through multiple doses for the following conditions:

  • An inconsistent pattern through doses (for example, printing at dose 1, non-printing at dose 2, but printing at dose 3)

  • If the gauge is disabled or not present for a dose condition

  • If a SEM image is not present for the gauge in any dose condition

Gauges that fail this consistency check are disabled.

A column is added to the stats output file “yes” or “no”, indicating whether or not the gauge is consistent through multiple doses.

Note:

This parameter requires at least five dose conditions per gauge. If there are not enough dose conditions for an individual gauge, the consistency check is skipped for that gauge.

dump_processed_image

An optional parameter that if set to “true” generates two sets of processed SEM images:

  • Images with a printing SRAF contour highlighted.

  • Images with the layout masked.

For the images where the printing SRAF contour is highlighted, the command adds an SRAFPrintingReview column to the output gauge data or super gauge data file, with a value of 1 if a gauge appears to require further review based on the quality of the printing threshold.

If SRAFPrintingReview returns a value of 1, dump_processed_image also intentionally omits reporting a GT:0 or LT:0 result in the “Meas” column.

For the images where the layout is masked, the command also adds an SEMLayoutMasked column to the output gauge data or super gauge data file with the path to the image.

By default, this parameter is false, and no images are generated.

gauge_center {sraf | main}

An optional parameter that when set to “main” allows createsrafgauges to accept input gauge files centered on the main feature instead of on an SRAF.

When the value specified is “main”, the tool runs as if all gauges were centered on the main pattern. It populates the meas value based on the area ratio calculated for the nearest SRAF pattern from the main pattern. If the two SRAF patterns are at the same distance from the gauge, the tool uses the one that has the larger area ratio.

The default value is “sraf”.

add_new_gauges {false | {true [value]}}

An optional parameter that specifies whether or not new SRAF gauges corresponding to any extra SRAF printing blobs in the SEM image should be added to the output gauge file. By default, this parameter is set to “false”.

Setting this value to “true” creates new gauge entries for the extra printing SRAFs, with the center of each new gauge location being the centroid of a printing SRAF blob. By default, three new gauges are added for each input gauge (value=3). Additional gauges are added starting with the largest area ratio value first.

loc_output {0 | 1}

This option is automatically set on output depending on the gauge location:

If the gauge center is on an SRAF, the loc column is set to 0.

If the gauge center is on an SRIF, the loc column is set to 1.

populate_area_ratio {true | false}

An optional parameter that when set to “true” outputs the calculated area ratio for SRAF patterns from SEM images to the output gauge file. The calculated area ratio value is written to the Area_Ratio column, and is used for plotting the area ratio versus gauge in the Calibre nmModelflow tool. Using these values, outlier gauges (for example, noisy images) can be identified and disabled. The default value is “false”. The area ratio is set to 1.0 for heavily printing SRAFs.

Tip

If on output the Area_Ratio column is set to -1.0 and the Meas column is set to -1, the gauge requires review. This special warning flag combination usually happens for gauges that have a misalignment issue.

populate_blob_features {true | false}

An optional parameter that when set to “true” outputs the calculated blob area, blob contrast, and blob mean for SRAF patterns to the blob_area, blob_contrast, and blob_mean columns, respectively. If the gauge does not have a SEM image, the values are set to -1. If the gauge does have a SEM image but no SRAF pattern is detected, the values are set to 0. These values are plotted versus gauge in the Calibre nmModelflow tool. Using these values, outlier gauges (for example, heavy SRAF printing) can be identified and disabled. The default value is “false”.

Note:

This parameter is only used for segmentation_method 2.

populate_sem_main2background {true | false}

An optional parameter that when set to “true” outputs the ratio of average pixel intensities of the main pattern and non-main pattern regions. The calculated ratio is written to the SEMMain2Background column, and plotted versus gauge in the Calibre nmModelflow tool. Using these values, outlier gauges (for example, noisy images) can be identified and disabled. The default value is “false”.

processed_image_path filepath

An optional parameter used with dump_processed_image. Set this parameter to a directory name to dump generated processed images to. Processed images are grouped into subdirectories using the syntax gaugeSet_dose_focus.

For example, for a gauge with an ID of 3, a gaugeSet number of 1, a dose of 0.99, and a focus of 0, the subdirectory name will be gaugeSet1_dose0.99_focus0; the file processedSEM_gid3 is placed in that directory.

layout_masked_image_path filepath

An optional parameter used with dump_processed_image to save images with the layout masked. Set this parameter to a directory name to save images with the layout masked. The images are grouped into subdirectories using the syntax gaugeSet_dose_focus.

For example, for a gauge with an ID of 4, a gaugeSet number of 1, a dose of 0.98, and a focus of 0, the subdirectory name will be gaugeSet1_dose0.98_focus0; the file layoutMaskedSEM_gid4 is placed in that directory.

input_line {off | on}

An optional parameter that when activated (input_line on) draws a blue line for the actual gauge location in the SEM image. The default value is off.

output_line {on | off}

An optional parameter that when deactivated (output_line off) suppresses the default printing of a red line to indicate the output gauge location in the SEM image. The default value is on.

use_all_srafs_blob {true | false}

An optional parameter that specifies if the command should analyze all SRAFs present in the SEM image. By default, the tool analyzes only the SRAF that corresponds to the gauge (false).

If this parameter is set to true, the tool analyzes all SRAF blobs present in the SEM image, and uses the maximum of the area ratios for all SRAF patterns to decide whether the meas value should be GT:0 or LT:0.

The command also moves the output gauge to the centroid of the SRAF blob where it finds the largest area ratio.

sraf_blob_contrast_threshold value

An optional parameter related to the sensitivity of the tool. An SRAF blob is defined as all pixels with a grey level within value of the intensity mean. Increasing the value decreases the sensitivity of the printing blob. The default value is 11.

Note:

This parameter is only used for segmentation_method 1.

sraf_blob_area_threshold value

An optional parameter related to the sensitivity of the tool. An SRAF blob with an area of connected pixels (as derived from the sraf_blob_contrast_threshold) greater than the specified value is considered as printing. The default value is 8.

Note:

This parameter is only used for segmentation_method 1.

no_of_pixels_to_truncate value

An optional parameter that specifies how many rows should be ignored during processing. Use this parameter to avoid operating on the red scan line in a typical SEM image. By default, this parameter is set to 0. A setting of 0 causes createsrafgauges to automatically analyze the color channels to determine how many top rows should be ignored.

max_sraf_distance value

An optional parameter that specifies the maximum distance that a pixel can be from an SRAF layout polygon and still be considered as a printing SRAF blob in the SEM image. The default value is 50 nm.

use_rof_filter {true | false}

An optional parameter that specifies whether a Rudin-Osher-Fatemi (ROF) image denoising algorithm is applied on the input SEM image before further processing. If this parameter is set to true, the ROF algorithm is applied to the input SEM image. The default value is false and the ROF algorithm is not applied.

Note:

This parameter is only used for segmentation_method 2.

#comments

All comments in the setup file must be prefixed with pound signs (#).

Examples

An example setup file for createsrafgauges follows.

version 1.0
sem_images_base_path /home/caltest/semImages
sem_pixel_size 1.303
main_pattern_layer 3.0, 2.0
assist_feature_layer 4.0, 1.0 SRAF
assist_feature_layer 5.0, 2.0 SRIF
max_alignment_error_x 200
max_alignment_error_y 200
stats_file stats.csv
dump_processed_image true
processed_image_path /home/caltest/Out_images
layout_masked_image_path /home/caltest/Masked_images
populate_area_ratio true
gauge_center sraf
use_all_srafs_blob true
