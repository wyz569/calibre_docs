veb_compose

CM1 and VEB Commands

Related Formats: Variable Etch Bias (VEB) Model File Format

Creates a VEB template object. The VEB object can be optionally output to a file, a Tcl variable for later use, or both.

Note:

The veb_compose command is similar to the cm1compose command in that both of them create a starter model object using templates.

Usage

veb_compose {-m modelform_number | -veb vebobj} [-o filename] [-optimvar varname]

Arguments

-m modelform_number | -veb vebobj

A required argument that specifies which modelform is used as the template (-m option, using a modelform as listed in the table following) or a VEB object containing a nominal model (-veb option).

Table 1. Modelforms Used for veb_compose Command

Modelform

Number and Type of Kernels

0

constant bias model

1

one Gaussian kernel

12

one Gaussian kernel, non-linear terms up to second order

13

one Gaussian kernel, non-linear terms up to third order

20

two Gaussian kernels

21

one Gaussian kernel and one visible kernel looking outwards

22

one Gaussian kernel and one visible kernel looking inwards

23

one Gaussian kernel and one flux kernel looking outwards

24

one Gaussian kernel and one flux kernel looking inwards

25

one Gaussian kernel and one flux kernel looking both inwards and outwards

26

two Gaussian kernels, non-linear terms up to second order

27

two Gaussian kernels, non-linear terms up to third order

30

three Gaussian kernels

31

two Gaussian kernels plus one visible looking outwards

32

two Gaussian kernels plus one visible looking inwards

33

three Gaussian kernels, non-linear terms up to second order

34

three Gaussian kernels, non-linear terms up to third order

40

four Gaussian kernels

41

three Gaussian kernels plus one visible looking outwards

42

three Gaussian kernels plus one visible looking inwards

43

four Gaussian kernels, non-linear terms up to second order

44

four Gaussian kernels, non-linear terms up to third order
-o filename

An optional argument that specifies an output file to save the VEB template to.

-optimvar varname

An optional argument that specifies a Tcl variable to save the template to as a VEB object.

Examples

Creates a VEB object using modelform 10:

set vebobj1 [veb_compose m 10]

Creates a VEB object using modelform 41, dumps the template to a file, and assigns it to the Tcl variable par41:

set vebobj2 [veb_compose m 41 -o “veb.mod” -optimvar par41]
