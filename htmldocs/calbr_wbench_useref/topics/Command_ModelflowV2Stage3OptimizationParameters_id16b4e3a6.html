modelflow_v2: Stage 3 Optimization Parameters

Parent topic: modelflow_v2

This section specifies etch model parameters and ranges for optimization by the modelflow_v2 command.

         (or)
            [un optim_set]

Note:

The “stage3” parameter starts the stage3 block and is followed by the stage3 arguments.

Arguments

Two forms of specifying VEB optimization are accepted for stage 3 of modelflow_v2:

Implicit mode

Implicit mode was implemented in the 2013.3 release. It allows you to specify a set of parameters with shorter commands similar to choosing a modelform.

Explicit mode

Explicit mode is the detailed, customizable format. Individual terms can be specified. This is the mode that is used in releases previous to 2013.3 and it remains available.

Implicit Mode

vebmf {integer | nominal}

Specifies the VEB model form to be calibrated. Accepted model form numbers are the same as in the veb_compose command. Also, if any of the esinterval or uinterval parameters are not specified, the search intervals for es and u parameters coincide with those generated by veb_compose Tcl command. These options cannot be used with explicit optimization settings. If nominal is set, the optimization settings are generated for the nominal model.

esinterval {auto | value} {auto | value}

Specifies the lower and upper bounds of search intervals for the es parameter, which is the diffusion length in nm of n-th resist density. The keyword “auto” tells the tool to use the default (Default is to use search intervals determined by the veb_compose command). This option is not compatible with explicit optimization settings, and it can be set only if the vebmf option is also present.

uinterval {auto | value} {auto | value}

Specifies the lower and upper bounds of search intervals for the u parameter, which is the shift in nm of n-th resist density. The keyword “auto” tells the tool to use the default. (Default is to use search intervals determined by the veb_compose command) This option is not compatible with explicit optimization settings, and it can be set only if the vebmf option is also present. For example, using an implicit optimization setting of the form:

		vebmf 1 \

is equivalent to explicitly setting

		es1 35 600 5 \
		u1 -50 50 5 \

Similarly, using

		vebmf 1 \
		esinterval 10 200 \
		uinterval 0 auto \

is equivalent to setting

		es1 35 200 5 \
		u1 0 50 5 \

Note that the lower limit for es1 remains 35 even though in esinterval it is set to 10. This is because the diffusion length of kernels of gauss, top_gauss, and exponential type cannot be smaller than 35 nm.

Explicit Mode

esn optim_set

diffusion length in nm of n-th resist density, 0 esn, n = 1, 2, 3, or 4. (Default = use nominal model setting.)

un optim_set

shift in nm of n-th resist density (Dn term in the etch model), n = 1, 2, 3, or 4. (Default = use nominal model setting.)

Similar to stage 2, the stage3 option can also be used without specifying the optimization method and parameters. If no options are specified, only linear coefficients of the input etch model are optimized, while the rest of the model parameters (shifts and diffusion lengths) are kept fixed at their nominal values.

Description

Definitions

  • CTR — Constant Threshold Resist Model of type CM1.

  • Nominal optical, resist, and etch models are the initial models specified in the setup file.

  • optim_set — Defined as one of the optimization expressions as described in Using the Independent, Constrained, and Dependent Optimization Parameters of “modelflow_v2: Stage 1 Optimization Run Modes”.
