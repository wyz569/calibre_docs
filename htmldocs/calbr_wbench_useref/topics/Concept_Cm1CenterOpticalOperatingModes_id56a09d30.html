CM1 Center Optical Operating Modes

CM1 Center’s Optical model-based action buttons (Optimize, Try, and eXclude, shown in CM1 Center as O, T, and X buttons) execute the modelflow_v2 command in different modes.


All simulations are done on the currently loaded layout.

Table 1. Optimize, Try, and eXclude Modes in CM1 Center Optical Modes

Mode

Definition

O(ptimize)

Searches for the best values of any optical parameters selected in the Optimization tab, Optics sub-tab for the active optical model.

T(ry)

Uses the active Optical model settings in CM1 Center without optimization. You use this mode as a fitness check on the currently-active optical model in CM1 Center.

(e)X(clude)

Mode is off. Use only for Direct VEB modeling (do not use for the procedures in this chapter).
Note:

Previous to 2009.4, a L(ink) button was also included in the CM1 Center mode bar. This functionality has been moved to a radio button choice in the Optics tab, as described in “Optimizing the Optical Model”.

Tip

The Time button can be used to take a reading on how long one frame of simulation will take to run. This is mainly used to test for the run time of different modelforms in CM1 and VEB models.
