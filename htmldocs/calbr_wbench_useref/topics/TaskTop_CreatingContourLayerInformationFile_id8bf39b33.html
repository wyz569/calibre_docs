Creating a Contour Layer Information File

The contour layer information (.cli) file is the translation file used by Calibre WORKbench to understand the layers and information presented in the contour design file. This contour layer information file is used as input in the resist modeling tools.

A contour layer information file can contain multiple contour sets. Each set is known as a contour block.

Prerequisites

  • Contour design file.

    See “Preparing Contour Design Files” or “Alternate Method to Create Contour Design Files”.

  • List of contour layer numbers in the file.

Procedure

  1. In a text editor, fill out the header information for the contour layer information file:

    • version

    • contourSetCount

    • Any optional header items.

    See “Contour Layer Info (CLI) File Format” for syntax.

  2. For each contour set, define a block that contains the following items:

    • contourSet contour_set_number_x

    • An opening brace ({)

    • A set of information keywords about that contour layer

    • A closing brace (})

  3. Save the file with an extension of .cli.

Results

You have a file that can be used in conjunction with the contour design file to improve process models. See “Model Calibration With Contour Calibration Files”.
