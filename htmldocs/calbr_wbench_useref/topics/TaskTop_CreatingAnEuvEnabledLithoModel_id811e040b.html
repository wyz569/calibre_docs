Creating an EUV-Enabled Litho Model

Litho models simplify access to a collection of process models. The Calibre nmModelflow tool requires litho models for simulation and calibration runs.

Prerequisites

  • Calibre WORKbench invoked and Calibre nmModelflow open

  • At least one optical model, a resist model, a flare model, and its accompanying flare map file

Procedure

  1. In Calibre nmModelflow, select File > Create Lithomodel to open the MDF Configuration Creation Wizard for Calibre nmModelflow. The wizard lists each step that requires input (Output Directory, Lithomodel Components, Exposure, Finish).
  2. For the Output Directory page, enter or navigate to an empty directory to contain the completed litho model. (You can create a new directory from the file chooser dialog.)
  3. For the Lithomodel Components page, select the EUV Model checkbox to show the EUV options. Fill out the fields to match your configuration and click Next.



    Table 1. MDF Configuration Creation Wizard, Lithomodel Components

    Field

    Notes

    No. of Exposures

    Required. Controls the number of exposure masks. Increasing the number of exposures adds more items to the list when you click Next.

    Image Threshold

    Sets the image threshold for the litho model. By default, 0.35 is used.

    Resist Model

    Required. Specifies a resist model.

    • Use Resist Model to specify the path to an existing CM1 resist model file to add to the litho model.

    • Use Resist Model Form to select a preset CM1 modelform template.

    Etch Model

    Optional.

    Topo Model

    Optional.

    Black Border Model

    Optional. Specify the path to a black border model.

    Flare Model

    Optional. Specify the path to a flare model to be used with the litho model and the type of flare map to use (none, related flare map PNG file or a constant flare value).

    Stochastic Model

    Optional. Used only if you are performing stochastic model measurements with your EUV optical model.

    Shadow Bias Model

    This group is used when your EUV modeling process includes a shadow bias model.

    If you specify a shadow bias model, you must also designate the correct value for the Euv Slit X Centre value.
  4. For each exposure, you must fill out three pages:
    1. Nominal Optical Model — Specify the nominal EUV optical model (dose 1.0, defocus 0) in this page.
      Note:

      By default, specify the path to your existing previously-generated EUV optical model. However, the GUI allows you to generate one if you only have an Optical file, and also allows you to invoke the Calibre WORKbench Optical Model Tool if you do not have an Optical file.

      You have a choice of adding additional defocus models.

      • Check “Add Defocus Model using existing model on disk” if you already have the defocus models generated.

      • Check “Add Defocus level to Generate new Optical Models” if you only have the nominal optical model. The wizard will prompt you to enter the defocus range when you click Next.

      Leaving both of the checkboxes unset uses only the nominal optical model.

    2. Background and Dose — Specify the background transmission type and initial dose value if it is different than 1.0 for the nominal optical model.
    3. Mask Layer Index — Specify characteristics for one or more mask layers. All items except Mask Layer Transmission are optional.

      Click Add New Mask Layer if you need to define multiple mask layers. Otherwise, click Next.

      Figure 1. MDF Configuration Creation Wizard, Mask Layer Screen

      Table 2. MDF Configuration Creation Wizard, Mask Layer Index Fields

      Field

      Description

      Mask Layer Transmission

      Specifies a required argument that specifies the transmission type of the mask_layer template.

      The field allows the following types: dark, clear, atten x, phase90, phase270, phase180, or a real imaginary pair.

      Category

      Specifies a notes field that you can set. Does not have any effect on the calculations.

      CX (um)

      Specifies a convex cornerchop value in microns. Corner chop removes a right triangle polygon with isosceles legs equal to the CX value from convex corners. Only right-angle corners are chopped.

      CC (um)

      Specifies concave cornerchop value in microns. Corner chop adds a right triangle polygon with legs equal to the CC value to concave corners. Only right-angle corners are filled.

      X Shift (um)

      Shifts all polygons in the X direction by the specified amount.

      Y Shift (um)

      Shifts all polygons in the Y direction by the specified amount.

      DDM Model

      Specifies a DDM model to associate with the mask layer. Use this option if you have a DDM model to associate with the EUV model.

      This option cannot be used if the All Angles option is selected.

      DDM Group

      Specifies a DDM group configuration. Selecting this radio button opens options to create a DDM group from multiple DDM models or use a previously created DDM group file.

      All Angles

      Specifies that corner chopping should chop all-angle corners, not just right-angled ones.

      This option is only permitted if CX or CC is also specified.

      Bias (um)

      Expands or contracts all edges by the specified amount in microns.

      Bias values are applied to both horizontal (X) and vertical (Y) directions.

      X Bias (um)

      Expands or contracts only horizontal edges.

      Y Bias (um)

      Expands or contracts only vertical edges.
  5. The Finish screen shows a summary of the commands that the MDF Configuration Creation Wizard executes. Select a node type, layer type, and set a description for the litho model in the fields at the bottom of the wizard screen, then click Finish.
