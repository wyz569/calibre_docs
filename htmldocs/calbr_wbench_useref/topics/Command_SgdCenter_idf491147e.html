sgd_center

CM1 and VEB Commands

Centers gauges at the midpoints of associated drawn geometries.

Usage

sgd_center {–obj object | –sgf file} {–l wblayout | –m layoutfile} –layer layerno
[–out file] [–type {drawn | meas | meas_etch | max} value]

Arguments

–obj object | –sgf file

A required argument that specifies an input gauge or super gauge data object (–obj) or input gauge file or super gauge data filename (–sgf).

–l wblayout | –m layoutfile

A required argument specifying an input layout handle (–l) or input layout file (-m) in one of the supported formats (such as GDS or OASIS).

–layer layerno

A required argument that specifies an input layer with drawn geometries where the measurements are to be performed from.

–out file

An optional argument that specifies an output file name for the centered gauge data.

–type {drawn | meas | meas_etch | max} value

An optional argument that tells the tool to resize centered gauges using the input scaling factor value and the specified type (drawn, meas, meas_etch, or maximum) of the three CDs. The input scaling factor value cannot be less than 1.5.

Description

The sgd_center command centers gauges at the midpoints of the associated drawn geometries and optionally resizes them using the input scaling factor and drawn, meas, meas_etch, or maximum of these three CDs. This is done for both enabled and disabled gauges. If the same gauge is present at different process conditions, it is used for rescaling the maximal drawn | meas | meas_etch | maximum of the three CDs computed for this group of gauge replicas. Enabled empty gauges (those that cross less than 2 edges) are disabled, while height gauges and disabled empty gauges remain unchanged.

Examples

set sgd [sgd_center  -sgf  gauge.sgd -m tp.oas -type max 2]
