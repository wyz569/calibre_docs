Step 3b - Setting the Optical Diameter and Kernel Grid Size

The optical diameter defines the size of the square of spatial influence of the optical system. Interactions between proximate edges are possible when the edges are separated by a distance up to the optical diameter.

In practice, choosing the optical diameter is a trade-off between accuracy and speed. The accuracy improves with larger optical diameters, since optical effects between edges at longer distances are accounted for. The larger diameters require larger context area to be calculated for a given area of interest and hence increase run time.

Figure 1. Understanding Optical Diameter

Note:

A larger optical diameter can cause more flattening of the original hierarchy, as the interaction between edges has a larger range. It can also result in longer run times, necessary to handle the increased interactions.

The kernel grid (also known as the optical grid) determines the granularity of the kernel calculations.

  1. In the Optical Model Tool, Kernels/Image tab, check the Sigma Max setting using the dropdown (down arrow) from the Recommend button. Modify if necessary.

    Note:

    Sigma max is the approximate maximum radius where the illumination profile has significant intensity. In cases where the source has a maximum intensity near the radius of 1, you can use a value of 1 for sigma_max.

  2. To set the Optical Diameter and Kernel Grid, click the Recommend button. This sets them to a value compatible with wavelength, NA, and Sigma Max.

    In batch mode, you specify the optical diameter by using the hoodpix keyword. In the Optical model file, you define the Kernel Grid field with the kernel parameter.

  3. Go to the next task, “Step 4 - Entering Film Stack Information”.

Usage Notes

  • Estimating the optical diameter mathematically.

    In general, the size of the minimum pitch that can be imaged with an optical system equals to (λ/NA)/(1+sigmamax), where sigmamax is the maximum normalized radius of illumination source with nonzero intensity. The size of the optical point spread function corresponds to the size of the Airy disc 1.22*λ/NA, or about twice the size of the minimum pitch. The optical diameter of an OPC model should extend over a region that is about an order of magnitude larger than the minimum feature that can be imaged.

    Hence accounting for partial coherence effects, an estimate of the optical diameter can be taken as order of magnitude larger than the characteristic optical spot size of λ/NA: 

    		Optical Diameter = 20 (λ/NA)/(1+sigmamax)

    With sigmamax = 1, NA = 1.35, and λ = 0.193 microns, this formula gives an optical diameter of 1.43 microns. Note that the image intensity error made at an arbitrary layout as a result of the finite size of the optical diameter can only be computed by simulation.

    Note:

    An optical diameter that is significantly greater than the recommended optical diameter could result in increased run times.

  • Calculating the optical diameter from measured CD data.

    One technique for determining the optical diameter based on wafer data is to examine the typical pitch curves for the process being modeled to determine the spacing at which the proximity effects become negligible. Double of this distance can be taken as the optical diameter.

    When using this method, it may seem that a very large optical diameter is required owing to long-range etch effects. Because the CM1 and VT5 models are not intended to model long range etch effects, using a very large optical diameter is not advised.

  • Using Recommend for kernel grid.

    It is recommended that the proper optical grid size be determined from the properties of the exposure tool optical system, as detailed in the next paragraphs. For modelflow_v2, there is an additional requirement that the kernel grid must be less than or equal to 1.5 times the simulation pixel size, where the pixel size is determined based on the optical settings. To prevent errors, it is recommended that you use the Recommend button to automatically select the best values.

    It has been empirically determined that optical kernel grid size should be smaller than about 1/6th and greater than 1/12th of the optical Nyquist rate. This corresponds to:

    (λ/NA)/(1+sigmamax)/24)) <= kerngrid <= (λ/NA)/(1+sigmamax)/12))

    With sigmamax = 1, NA = 1.35, and λ = 0.193 microns, this formula suggests a kerngrid setting of 3 to 6 nm. The largest value in this range is recommended, since it produces the best runtime performance, and the smallest model size with comparable accuracy.

    For Calibre nmOPC applications, kerngrid must be defined, even though it is not used. If the user enters an optical diameter / kernel grid combination that is not an integer multiplier, opticsgen resets one or the other so that the optical diameter is an integer multiple of the kernel grid.

    For Calibre OPCpro simulations, the optical diameter must be an integer multiple of the kernel grid. When the integer multiple is 2N, where N is a positive integer, the optical model computation is faster. Hence it is suggested that optical diameter and kernel grid be related as follows: 

    optical diameter = 2N * kernel grid

    When you use the Recommend button, it sets Kernel Grid to values in the upper range.

  • Forcing kernel symmetry.

    If “Force Kernel Symmetry” on the Kernels/Image tab of the Optical Model Tool is selected, the wizard increments the kernel count until it finds an asymmetric setting where TCC symmetries are preserved.
