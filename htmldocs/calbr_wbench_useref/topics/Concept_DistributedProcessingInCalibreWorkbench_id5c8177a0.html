Distributed Processing in Calibre WORKbench

Distributed processing supports parallel execution on different computers for Calibre WORKbench modeling purposes. It is similar in function to Calibre MTflex.

Why Would I Use Distributed Processing?

Running the modeling commands in distributed processing mode should complete most modeling jobs faster due to the ability to use multiple processors for calculations.

Key Definitions, Acronyms, and Abbreviations

  • primary — The machine running Calibre WORKbench is considered the primary machine; it controls the communication between machines running distributed processing jobs.

  • remote — Any machine running processing jobs for the primary machine is referred to as a remote machine.

  • job queue — Used by the primary machine to coordinate sending processes to the remotes to run.

What are the Distributed Processing Requirements?

  • A list of available host names to use as remote machines, with information on their hardware configuration
