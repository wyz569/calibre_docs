Creating a Distributed Processing Configuration File

Calibre WORKbench uses the configuration file you create in this procedure to identify machines to be used as remotes.

Prerequisites

  • Information on the remote machines.

Procedure

  1. In an editor, open an empty text file.
  2. Create one or more REMOTE statements, one per line and all on one line, using the syntax shown in Table 1:
    REMOTE {HOST {hostname | localhost} cpu_count} 
[MGC_HOME directory_path] [DIR directory_path] 
[EXEC executable_path] [A64] [LAUNCHNAME launchname ] 
[RSH rsh_exec_path ]
  3. After you have added all your machines, save the file.
    Table 1. Remote Statement Arguments

    Argument

    Description

    Required?

    HOST{ hostname | LOCALHOST } cpu_count

    Specifies the hostname (a string with a 256 character maximum) of a remote machine and the number of CPUs that Calibre WORKbench can use to run processes on the machine.

    To specify jobs running locally (on the primary), use the keyword LOCALHOST as the host name.

    cpu_count must be a positive integer between 1 and 128.

    Additional cpu_count recommendations based on which search method you are using are as follows:

    • The full search algorithm can ideally simultaneously use as many CPUs as combinations of optimization parameters need to be explored. In practice, however, the potential number of parameter combinations can be huge. The full search therefore uses a maximum of 32 CPUs in mtfull mode (-turbo option) and 128 CPUs in distributed mode (‑remotefile or -remotestring option)

    • The number of simultaneously used CPUs varies between 4 and 50 during a FRONT search algorithm. The average number of simultaneous CPUs in use depends on the length of the run (explore setting) and the problem at hand; it can be as high as 20 CPUs. The actual FRONT search scalability observed by internal testing is maximal for stage 1 (optical model) calibration (average of 20 CPUs) and minimal in stage2 (CM1 resist) calibration (average of 4 CPUs) with stage3 (VEB etch) being somewhere in between.

      The practical limitation for FRONT search algorithms is a maximum of 32 CPUs in ‑turbo mode and 128 CPUs in -remote mode.

    Yes

    MGC_HOME directory_path

    Specifies the MGC_HOME environment variable as used by a remote machine.

    No

    DIR work_directory

    Specifies a working directory (that must be visible on the primary from the remote) that the remote process switches to for its source files. If not specified, the user’s home directory is used.

    This argument is required only if an optical source map file or the sgd_opticsgen command is used.

    Conditional

    EXEC executable_path

    Specifies the pathname to the executable used by the remote machine to call Calibre WORKbench.

    • If this argument is specified it overrides the default value ($CALIBRE_HOME/bin/calibrewb).

    • If not specified, calibrewb is used.

    No

    A64

    Specifies that this machine is a 64-bit architecture machine.

    No

    LAUNCHNAME launchname

    Specifies the primary name as seen by the remote. If not set, Calibre WORKbench scans for the hostname automatically.

    No

    RSH rsh_exec_path

    Specifies a remote shell command to use for launch. The default is the standard remote shell command.

    No

    • Lines that start with // or # are considered comments and ignored.

    • The REMOTE command and arguments are case insensitive.

    • Calibre WORKbench attempts to run as many processes on the remote machine as are specified in the cpu_count argument.

    • Priority is handled in a top-down fashion; a job is sent to the first waiting machine available starting from the top of the list.

    • You must have permission to run system commands and the calibrewb executable on all remote machines specified in the configuration file. (To test for permissions, try using a remote shell command such as “rsh machinename ls” on Linux® operating systems.)

    • If rsh commands fail, a simple workaround is to add the file .rhosts to your home directory. This file should contain all the hosts you want to connect to in the file, or a plus symbol (+) to connect to any machine.

    • Use only absolute pathnames; do not use relative paths.

    • If your job crashes, check your remotes (rsh remote_name ps -elf | grep slave) and clean up leftover processes if any are found (rsh remote_name kill -9 calibrewb_ID).

Examples

// These are comments
// remote host green 4 EXEC /rp1/CALIBRE_HOME/bin/calibrewb 

REMOTE HOST proton 16 DIR /tmp CALIBRE_HOME /net/calibre1/2007.2/ixf/Mgc_home
REMOTE HOST pickle 32 exec cwb A64 CALIBRE_HOME /home/latest/ixl/Mgc_home
