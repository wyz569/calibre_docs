Configuring the optical system is performed on the Main Tab of the Optical Model Tool, accessed using the Litho menu.

Usage Notes

  • Action is required when using the Optical Model Tool.

    You need to specify your own optical model settings. The defaults in the Optical Model Tool do not represent a meaningful model in most practical situations.

  • Choosing an optical engine.

    When choosing an optical engine, consider the following:

    • rapidTCC is the most recently-developed fast optical engine. It is significantly faster than TCCcalc for optical diameters greater than 1.2 microns. Upsample factor and pupil sampling options are available with the rapidTCC engine. In the Optical parameters file, these correspond to the rapidTCC_uf and pupilSampling options. The recommended default setting for pupil sampling is 0.01.

      An alternative to setting the rapidTCC_uf option is to specify a convergence_tolerance setting instead. This option automatically sets the sampling of the optical system transform function; the recommended value is 0.008. It can be set in the Illumination tab of the Optical Model Tool or added to the optical model file using a text editor.

      Tip

      convergence_tolerance is available starting with the 2009.4 release and is recommended instead of rapidTCC_uf.

    • rapidTCC and TCCcalc can also be used for SMO sources (see “Calibrating Optical Models for SMO Sources”).

    • TCCcalc is slower than rapidTCC for large optical diameters. TCCcalc is recommended for optical diameters less than 1.2 microns. TCCcalc can be used with or without an approximate speed-up option, designated by pupilLookup in the Optical parameter file. The pupilLookup option allows the user to specify the size of the internally generated pupil grid used for speeding the calculation. The number of points for pupil lookup must be an odd value and is recommended to be set to 201.

    • Vector diffraction modeling is recommended.

    The components of the electromagnetic field inside the resist can be added as either scalar or vector quantities. For numerical apertures above 0.8 (NA>0.8), only the vector formulation for the optical system diffraction model is accurate. Practically, the penalty in model generation time when using vector thin-film models is very small compared with scalar thin-film models. Also, there is no performance penalty during OPC when using vector thin-film models. As far as the optical model is concerned, the only parameters relevant to OPC performance are the number and grid size of the SOCS kernels and the optical diameter.

  • Selecting the most recent version is recommended.

    The primary use of the version keyword in optical models is to help maintain backward compatibility. Features introduced in an optical model are not available in versions of Calibre WORKbench than are older than the version that introduced the feature. Features that existed in earlier optical model versions are compatible with more recent versions of Calibre WORKbench. Using the highest model version available is recommended.
