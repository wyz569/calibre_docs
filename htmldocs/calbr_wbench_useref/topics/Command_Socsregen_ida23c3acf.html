socsregen

Optical Model Commands

This command is used for recalculating the optical model from an existing optical model.

Usage

socsregen -in inputModel -out outputModel -numkerns N [-nolookuptables] [-writetccs]

Arguments

-in inputModel

A required argument specifying a previously computed optical model.

-out outputModel

A required argument specifying the name of the output model.

-numkerns N

A required argument specifying the new number of SOCS kernels to be used in the output model. This number can be smaller or greater than the available number of kernels in the input model.

  • If it is smaller (or equal), then the command produces the output model instantaneously, by copying the required kernels and other data to the new model.

  • If it is greater, then the command attempts to read the TCC matrix from the input optical model. If the TCC matrix is not available, then the command will return with an appropriate error message. If it is available, then the TCC matrix is decomposed via SVD/EVD to its SOCS kernels and the output optical model is generated.

-nolookuptables

An optional keyword to signify that the output model will be created containing only the SOCS kernels. The lookup tables and the TCC matrix are not written to the output model. This form of an optical model saves disk space, and is suitable for “dense” (CM1) simulations, but it cannot be used for “sparse” (VT5) simulations.

-writetccs

An optional flag that instructs the command to copy the TCC matrix from the input model to the output model. Note that if a TCC matrix is not available in the input model, the command returns with an appropriate error message.

Description

If the number of kernels in the output model is less than or equal to the input model, then this command creates the output model by copying the necessary data from the input model. If the number of kernels in the output model is greater than the input model, then this command attempts to read the TCC matrix from the input model and decomposes it into kernels.

Note that the output of this command is identical to using opticsgen and having modified the optical file of the input model to have a new number of SOCS kernels.
