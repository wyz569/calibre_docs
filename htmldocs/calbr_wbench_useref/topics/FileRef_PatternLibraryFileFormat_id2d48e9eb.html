Pattern Library File Format

Calibre Pattern Generator File Formats

A text format for generating test patterns using the Calibre Pattern Generator (CPG) tool.

The common file extension for pattern library files is *.xml.

Format

Input for:

  • Calibre Pattern Creator (PC)

  • Calibre Layout Generator (LG)

Generated by:

  • Calibre Pattern Creator (PC)

The following criteria apply:

  • The syntax must conform to the XML specification maintained here:

    https://www.w3.org/TR/REC-xml/

  • The first line must be the following XML declaration:

    <?xml version=“1.0” encoding=“UTF-8”?>

  • The file is case-sensitive. The parameter values themselves may not be treated as such when used within the Calibre Pattern Generator depending on the value and the command that references the value.

  • Elements begin with a start-tag of the form <Name> and end with an end-tag of the form </Name>. Each “Name” is pre-defined as part of the accepted syntax. User-defined elements are unsupported.

  • The root element is <Patterns>.

  • The two main sections are embedded in <Version> and <Pattern_template> elements.

  • Attribute values must be in quotes. Example:

    <element attribute=”value”>

  • Attribute names are pre-defined as part of the accepted syntax. User-defined attribute names are unsupported.

The following is the basic schema:

<?xml version="1.0" encoding="UTF-8"?>
<Patterns>
	<Version>1.0</Version>
	<Pattern_template class="pattern_name">
		<Properties>
			<Property name="property1" default="default_val_1"/>
		</Properties>
		<Layers>
			<Layer LayerNumber="layer_num" Name="layer_name">
			<Property name="Layer_Property1">layer_prop1_val</Property>
			...
		</Layers>
		<CenterGauge name="ctr_gauge_name">
			<ShapeProperties>
				<Property name="Ctr_gauge_prop1_name">Ctr_gauge_prop1_val
				</Property>
				...
			</ShapeProperties>
		</CenterGauge>
		...
		<Gauge name="gauge0_name">
			<ShapeProperties>
				<Property name="gauge0_prop1_name">gauge0_prop1_val
				</Property>
				...
			</ShapeProperties>
		</Gauge>
		...
		<Matrix name="matrix0_name" type="Line">
			<ShapeProperties>
				<Property name="matrix0_prop1_name">matrix0_prop1_val
				</Property>
				...
			</ShapeProperties>
			<Line name="line_name">
				<Property name="line_prop1_name">line_prop1_val
				</Property>
				...
			</Line>
		</Matrix>
		...
		<List name="list0_name" type="Line">
			<ShapeProperties>
				<Property name="list0_prop1_name">list0_prop1_val
				</Property>
				...
			</ShapeProperties>
			<Line name="line_name">
				<Property name="line_prop1_name">line_prop1_val
				</Property>
				...
			</Line>
		</List>
		...
		<Line name="line0_name">
			<ShapeProperties>
				<Property name="line0_prop1_name">list0_prop1_val
				</Property>
				...
			</ShapeProperties>
		</Line>
		...
	</Pattern_template>
	...
</Patterns>

Parameters

<Patterns>

A required element enclosing the pattern library settings.

<Version>

A required element specifying the pattern library version. The default version is 1.0.

<Pattern_template class=“pattern_name”>

A required element specifying a unique pattern in the pattern library.

The <Pattern_template> element must contain <Properties> and <Layers> elements. Gauge elements are optional. At least one shape must be specified.

The <Properties> and <Layers> specify the following:

<Properties>

Specifies pattern properties and their values. A pattern property is used to specify dimensions and names for shapes and gauges. The following example sets the pattern property p1 value to 21.

<Properties>
	<Property name="p1" default="21"/>
</Properties>

If no pattern properties are needed, the following empty element tag can be specified:

<Properties/>
<Layers>

Specifies layout layer information and settings.

The following attributes are specified in the <Layer> tag:

  • Layernumber — Specifies the layer to place the pattern. By default, the layernumber value is “1” for the first layer and increments to the next integer as the user adds layers.

  • Name — Specifies the name of the layer. By default, the layer name is “Layern” where n is the layer number.

To specify layout properties for a pattern, use the “name” attribute in <Property> elements embedded in the <Layer> element. The following name values must be specified:

  • Color — Specifies the displayed color for the pattern. The color value must be a hexadecimal number preceded by “#”. By default, color values are randomly generated.

  • Visible — Specifies an optically visible data layer. “1” specifies an optically visible layer. “0” specifies a hidden layer. The default is “1” (visible).

The pattern library supports two types of gauges — centered and uncentered gauges. The following elements support the shape types:

<CenterGauge name="ctr_gauge_name">

An optional element specifying a centered gauge. The following properties must be specified as Property attributes in the <ShapeProperties> element:

  • CenterGaugeType — Specifies the gauge orientation. “0” specifies horizontal and “1” specifies vertical.

  • Length — Specifies the length of the center gauge in nanometers.

  • Loc — Specifies the location field. “0” specifies across polygon and “1” specifies across space.

  • Struct — Specifies the structure name for the gauge. The default value is “NA”.

The following example specifies a 20 nm horizontal center gauge:

<CenterGauge name="HorizontalCenterGauge">
	<ShapeProperties>
		<Property name="CenterGaugeType">0</Property>
		<Property name="Length">20</Property>
		<Property name="Loc">0</Property>
		<Property name="Struct">NA</Property>
	</ShapeProperties>
</CenterGauge>
<Gauge name="gauge_name">

An optional element specifying an uncentered gauge. The following properties must be specified as Property attributes in the <ShapeProperties> element:

  • Loc — Specifies the location field. “0” specifies across polygon and “1” specifies across space.

  • Struct — Specifies the structure name for the gauge. The default value is “NA”.

  • X1, Y1 — Specifies the start location of the gauge in nanometers.

  • X2, Y2 — Specifies the end location of the gauge in nanometers.

The following example specifies a 50 nm horizontal gauge:

<Gauge name="Gauge_0">
	<ShapeProperties>
		<Property name="Loc">0</Property>
		<Property name="Struct">NA</Property>
		<Property name="X1">-25</Property>
		<Property name="X2">25</Property>
		<Property name="Y1">0</Property>
		<Property name="Y2">0</Property>
	</ShapeProperties>
</Gauge>

The pattern library supports three types of shapes — lines, lines list, and lines matrix. At least one shape must be specified in a pattern library. The following tags support the shape types:

<Line name= "line_name">

An optional element specifying a line shape. The following properties must be specified as Property attributes in the <ShapeProperties> element:

  • x and y — Specifies the lower left coordinates of the line in nanometers.

  • height — Specifies the line height in nanometers.

  • width — Specifies the line width in nanometers.

  • Orientation — Specifies the orientation of the line to the layout. Only “0” and “90” are supported. “90” aligns the line height to the y‑axis; “0” aligns the line height to the x‑axis. The default value is “90”.

<List name="list_name" type="Line">

An optional element specifying a series of repeated shapes. The following properties must be specified as Property attributes in the <ShapeProperties> element:

  • Orientation — Specifies the orientation of the shape height to the layout. Only “0” and “90” are supported. “90” aligns the shape height to the y‑axis; “0” aligns the shape height to the x‑axis. The default value is “90”.

  • count — Specifies the number of the repeated shapes in the series.

  • direction — Specifies the repetition axis of the repeated shape. Only “0” and “90” are supported. “90” aligns the repetition axis to the y‑axis; “0” aligns the repetition axis to the x‑axis. The default value is “0”.

  • height — Specifies the height of the repeated shape in nanometers.

  • pitch — Specifies the pitch on the repetition axis of the repeated shape in nanometers.

  • width — Specifies the width of the repeated shape in nanometers.

  • x and y — Specifies the lower left coordinates of the first shape in nanometers.

Specify the repeated shape after the list <ShapeProperties> element and within the <List> element.

<Matrix name="matrix_name" type="Line">

An optional element specifying a 2D array of repeated shapes. The following properties must be specified as Property attributes in the <ShapeProperties> element:

  • HDirection — Specifies the horizontal axis of the repeated shape. Only “0” and “90” are supported. “90” specifies the horizontal axis as the y‑axis; “0” specifies the horizontal axis as the x‑axis. The default value is “0”.

  • Orientation — Specifies the orientation of the shape height to the layout. Only “0” and “90” are supported. “90” aligns the shape height to the y‑axis; “0” aligns the shape height to the x‑axis. The default value is “90”.

  • VDirection — Specifies the vertical axis of the repeated shape. Only “0” and “90” are supported. “90” specifies the vertical axis as the y‑axis; “0” specifies the vertical axis as the x‑axis. The default value is “90”.

  • countX and countY— Specifies the number of the repeated shapes in the x-axis and y-axis.

  • height — Specifies the height of the repeated shape in nanometers.

  • pitchX and pitchY— Specifies the pitch on the x-axis and y-axis of the repeated shape in nanometers.

  • width — Specifies the width of the repeated shape in nanometers.

  • x and y — Specifies the lower left coordinates of the first shape in nanometers.
