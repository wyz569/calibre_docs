sgd_add

CM1 and VEB Commands

Related Formats: Gauge Object Reference, Super Gauge Data File Format

Appends a new gaugedata set to an existing sgd object.

Usage

sgd_add object {‑gf filename | ‑gobj gauge_object } ‑dose value ‑defocus value
[‑type value [‑plane value]] ‑weight value [-opticaldir dir1] [-opticaldir2 dir2]

Arguments

object

A required argument that specifies the input sgd Tcl object.

-gf filename

A required argument that specifies the input gaugedata file name.

‑dose value

A required argument that specifies the dose value as a positive floating point value. This is a dimensionless unit.

‑defocus value

A required argument that specifies the input defocus value in microns (um).

‑type value

An optional argument that specifies a value for the type column. Do not use if object is a version 1 super gauge set. Legal values are CD and SRAF.

‑plane value

An optional argument that specifies the plane in microns at which the gauge should be measured. Use only with SRAF gauges (‑type SRAF).

‑weight value

A required argument that specifies the input weight value as a positive floating point value. This is a dimensionless unit.

-opticaldir dir1

An optional input that specifies the directory path for the first optical model to associate with the gauge data set.

-opticaldir2 dir2

An optional input that specifies the directory path for the second optical model to associate with the gauge data set.

Description

A gaugedata set consists of the gauge filename and values for the corresponding dose, defocus, and weight. This command returns the index of the newly appended gaugedata set.

Examples

set obj [sgd_create -in “initial.sgd”]
sgd_add $obj -gf “/tmp/sample.gg” -dose 1.04 -defocus 0.1 -weight 1.0
