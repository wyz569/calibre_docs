imagediffusion

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Defines Gaussian function(s) to convolve with the image.

Usage

imagediffusion { 0 | sigma | weight1 weight1 sigma1 sigma1 weight2 weight2 sigma2 sigma2 }

Arguments

0

An optional argument specifying no image diffusion. This is the default.

sigma

An optional argument specifying the sigma value of the Gaussian function convolved with the image.

The value for sigma is specified in microns and is restricted to be less than 0.2 um. Additionally, sigma cannot be greater than 0.25 of the optical diameter (hoodpix) setting.

weight1 weight1 sigma1 sigma1 weight2 weight2 sigma2 sigma2

An optional argument specifying the sigma values of the Gaussian functions convolved with the image and their respective weights.

The values for sigma are specified in microns and are restricted to be less than 0.2 um. Additionally, sigma cannot be greater than 0.25 of the optical diameter (hoodpix) setting. Negative weights are not permitted with double Gaussian image diffusion.

An optional miscellaneous parameter defining the Gaussian function(s) to convolve with the image.

Tip

Tuning imagediffusion with CM1 models is generally not recommended. In some cases, however, tuning imagediffusion from 0-20 nm (0 to 0.02) can significantly improve accuracy.

The benefit gained if you tune imagediffusion should be compared to the chosen CM1 model form, not only a CTR model. This is because the CTR fit error with imagediffusion in the optical model is almost always better than without imagediffusion, but only the calibrated CM1 model represents the fitness change.
