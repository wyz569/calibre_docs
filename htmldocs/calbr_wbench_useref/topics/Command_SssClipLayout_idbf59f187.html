sss_clip_layout

Test Pattern Commands

This command globally flattens the input layout, filters out all structures that do not overlap with search rectangles, and writes the result to the specified output file.

Usage

sss_clip_layout object {-l handle | -m inputfile} -extra extra_dist -out layout_name
[-mode {touch | cut}]

Arguments

object

A required argument that specifies an input super spreadsheet object, previously loaded into the Calibre WORKbench environment.

-l handle | -m inputfile

A required argument specifying either an input layout handle (-l argument) or database file (-m argument) for the test pattern to be clipped. The input design must be in a Calibre WORKbench-supported format (such as GDS or OASIS).

-extra extra_dist

A required argument, specifying the amount to size each search rectangle in database units. Search rectangles are drawn around a single coordinate data point (x,y) using the specified extra_dist value as the lower left (x-extra_dist, y-extra_dist) and upper right (x+extra_dist, y+extra_dist) coordinates.

-out layout_name

A required argument, specifying the output handle. As shown in the example below, this should be an empty layout handle.

-mode {touch | cut}

An optional argument specifying the clipping mode type. In the default “touch” mode, geometries that intersect clipping boxes have their geometries preserved. In “cut” mode, shapes that intersect clipping boxes have portions that are outside the clipping box cut off.

Examples

# create sss object
set obj [sss_create -in sample.sss]
# load source layout
set l_src[layout create “tp.oas”]
# create empty layout to be used as output
set l_clip [layout_create]
$l_clip create cell [lindex[$l_src topcell]0]
set dbunits [$l_src units database]
$l_clip units database $dbunits
# now clip the layout
sss_clip_layout $obj -l $l_src -extra 100 -out $l_clip
$l_clip oasisout “tp_clipped.oas”
