cdruler

Test Pattern Commands

Returns measurement data for each of the sample points in the specified spreadsheet file. Note that you will obtain the most meaningful data if the sample points are located close to the sites you are interested in.

Usage

cdruler -ss spreadsheet -f setup { -m gds | -l handle }

Arguments

-ss spreadsheet

A required argument and value pair defining the pathname for the spreadsheet file.

-f setup

A required argument and value pair defining the pathname for the setup file.

-m gds | -l handle

A required argument and value pair defining the layout to be measured.

  • If the layout data is not loaded into the application use -m to define the pathname for a valid test pattern file.

  • If the layout data is one of the layouts currently loaded into the application, use -l and specify the handle of that layout.

Return Values

The returned string contains a list of records of the following format:

size_in_dbu size_in_nm measureType x1 y1 x2 y2 measureInput

where:

  • size_in_dbu — The distance between polygon edges, expressed in database units. See measureType for more information.

  • size_in_nm — The distance between polygon edges, expressed in database nm. See measureType for more information.

  • measureType — The Location setting from the spreadsheet file:

    • 0 = measure the width of the polygon at this location

    • 1 = measure spacing to the nearest polygon

  • (x1, y1), (x2, y2) — The points between which the spacing is measured. (x1,y1) is the point on the left or bottom; (x2,y2) is the point on the right or top.

  • measureInput — The Measurement setting from the spreadsheet file.
