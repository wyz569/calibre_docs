Using Optical Models for Process Window Optimization

CM1 Center includes the ability to vary the optical models for dose and defocus (modeling a process window under different conditions).

You use the Gauges tab when you have a super gauge file loaded; each gauge object in the super gauge file can have its own optical model associated with it.

The following techniques are available for optimizing for varied dose and defocus:

  • Computed — Use a single optical model and have CM1 Center calibrate for varying doses and defocus values for the gauge files.

  • Linked — Link each gauge file to one or two separate optical model directories. You would use this method when you have previously optimized optical models corresponding to specific dose and defocus values.

Prerequisites

  • At least one optical model directory. An optical model definition file is also required for the Pre-build Optical Models option.

  • A super gauge object file, which can be the result of performing the Making a Super Gauge Object by Adding Gauge Data procedure.

Procedure

  1. In CM1 Center, switch to the Gauges tab, and load the super gauge object if you do not already have it loaded.
  2. Load the nominal optical model(s).
  3. Set where the optical model process window information will be generated from:

    • If you only have a single nominal optical model:

      1. Select Gauges > Link > 1st Optical Model and link it to the most appropriate gauge object (usually this will be the best dose and focus setting).

      2. If you have a second optical model, link it using the Gauges > Link 2nd Optical Model option.

      This instructs CM1 Center to vary the single loaded optical model by the linked optical model using the setting(s) for dose and defocus relative to the defocus start value(s) in the Optics tab. Essentially, CM1 Center associates the beamfocus value with a 0 defocus value.

      For example, if you have a first optical model with a beamfocus value of 0.075 nm in your loaded Optical model and a gauge tab with a defocus value of -0.05nm, CM1 Center treats the gauge measurements as if they were located at 0.025 nm.

      Tip

      You can use the Gauges > Pre-build and Link menu items to have CM1 Center generate separate optical models for the dose and defocus values of the gauge file.

    • If you have multiple optical models optimized for the gauge object:

      For each gauge tab, link each first optical model using the Gauges > Link > 1st Optical Model menu item. If your system is a double exposure system, also use the Gauges > Link > 2nd Optical Model option to link each second optical model. You must ensure that all gauge objects have an appropriate model linked to them.

      Similarly, if you have an implant model system, use the Gauges > Link > 3d Optical Model option to link the third optical model.

  4. Set the optical model mode to Gauges Link(s) in the Optics tab.
  5. Set the optical model mode to O (Optimize) or T (Try) in the button bar at the top of the CM1 Center window.
  6. When you have finished linking your optical model(s) to gauges, click the Run button to optimize your optical models.
    Tip

    The Optics > Save Model menu selections generate optical model files based on the current settings in the Optics tab.

Results

A log file is generated on a successful run. You can examine the log file for separate entries on each process condition. These log entries show CDerrRms values and how many points were within spec. (You can set this value in the Prec field in the gauge file.)
