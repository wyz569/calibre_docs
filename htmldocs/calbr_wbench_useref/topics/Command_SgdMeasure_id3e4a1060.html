sgd_measure

CM1 and VEB Commands

Measures mask CDs on the specified layer and puts the measurements into the specified gauge record field or column.

Usage

sgd_measure object {–l wblayout | –m layoutfile} –layer layerno [–col name]

Arguments

object

A required argument specifying an input super gauge data object.

–l wblayout | –m layoutfile

One of either -l or -m are required arguments specifying the input test pattern. Use -l to specify a handle of an already loaded layout. Use -m to specify the path to an input file in one of the supported formats (such as GDS or OASIS).

–layer layerno

A required argument specifying a layer number in the design to use as input.

–col name

An optional argument specifying a column name in the gauge data record to fill with measured CDs. Supported names are as follows:

drawn

Fills the “Drawn” column (Default).

other

Fills the “Other” column.

meas

Fills the “Meas” (resist measured CD) column.

sim

Fills the “Simul” (resist simulated CD) column.

meas_etch

Fills the “MEtch” (etch measured CD) column.

sim_etch

Fills the “SEtch” (etch simulated CD) column.

Examples

# measure CD on layer 8 and put measurements into column Drawn
sgd_measure $obj  –m “tp.oas” –layer 8
