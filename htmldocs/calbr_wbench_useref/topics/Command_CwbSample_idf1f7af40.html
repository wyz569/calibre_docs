cwb_sample

Optical Model Commands

Selects a set of representative gauges, sites, or clips from an input group.

Usage

cwb_sample { {‑insites sites_in} | {‑ingauges gg_in} | {‑insites sites_in ‑ingauges gg_in} } ‑layouthandle layout_handle ‑lithomodel lithomodel_dir ‑simlayer sim_layer ‑nout N
[‑outsites sites_out] [‑outgauges gg_out]

Arguments

‑insites sites_in

A required argument that specifies an input CSV Sites file (or sites file, as described in the Calibre nmModelflow User’s and Reference Manual’s “Sites CSV File Guidelines” section of Contour Analysis Tab) for site selection. Either one or both of ‑insites or ‑ingauges must be specified for cwb_sample input.

‑ingauges gg_in

A required argument that specifies an input gauge file for gauge selection. Either one or both of ‑ingauges or ‑insites must be specified for cwb_sample input. Both .gg and .sgd file formats are supported. Disabled input gauges are ignored. If an .sgd file is input, then it must contain a single process condition.

‑layouthandle layout_handle

Specifies a single layout that includes either the gauges in gg_in or the sites in sites_in. The layout must be flat.

‑lithomodel lithomodel_dir

A required argument that specifies a lithomodel directory that the optical match is calculated from.

‑simlayer sim_layer

A required argument specifying the layer number of the target or simulation layer in the layout.

‑nout N

A required argument specifying the number of enabled gauges and/or sites clips in the output file. In cases where both gauges and sites clips are specified, N specifies the total number of gauges and sites clips to output.

When applied to sites clips, the N value for cwb_sample can be significantly smaller than the N value using sites for gauge_reduce. A sites clip can contain multiple sites which can make the number of sites clips be significantly smaller than the number of sites.

‑outsites sites_out

An optional argument that specifies the sites file name where the reduced set of sites clips is saved. The file is identical to sites_in, with the chosen sites enabled and the remaining sites disabled by setting the corresponding site weight (SW) to zero. By default, the command outputs out_sites.csv for the output site file.

‑outgauges gg_out

A required argument that specifies an output gauge file name where the reduced set of gauges is saved. The file is identical to gg_in, with the chosen gauges enabled and the remaining gauges disabled. By default, the command outputs out_gauges.gg for the output gauge file.

Description

This command reduces a set of input sites clips and/or gauges to a subset of optically unique gauges and/or sites clips. Using the specified litho model, the command generates a feature vector characterizing each gauge or sites clip and the surrounding features. The input gauges or sites clips are compared for similarity and a representative subset is generated.

While performing a similar function, the cwb_sample command differs from gauge_reduce in the following:

  • The criteria used to characterize each gauge or sites clip.

  • The support of input gauges and sites clips. The cwb_sample command supports simultaneous input of both gauges and sites clips. The gauge_reduce command supports processing one file type in a single instance.
