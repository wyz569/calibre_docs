Custom Kernel File Format

CM1 and VEB Model Formats

Defines a custom kernel file.

Format

The custom kernel file format supports the following kernels:

  • Pattern specific kernel (PSK).

The custom kernel format requirements include the following:

  • A custom kernel file is composed of keyword statements and comments.

  • Each statement must start its own line.

  • Any line beginning with a hash symbol (#) is considered a comment line.

The custom kernel file format can be specified inline in a CM1 file with KERNELINLINE kernelfilename preceding the custom kernel parameters:

KERNELINLINE kernelfilename
version 1
type CustomKernel
...

Custom kernel file format syntax:

version number 
type CustomKernel
numx N
numy M
step gridstep_in_nm
data spatial
	V(x1,y1)... V(xN,y1)
	...
	V(x1,yM)... V(xN,yM)

Parameters

version number

Only one version statement is allowed in the custom kernel file. Starting with Calibre WORKbench 2020.3, all new custom kernels must use version 1 (the default).

type CustomKernel

Only the CustomKernel kernel type is supported at this time.

numx N numy M

A required argument set specifying the number of grid points in the x and y directions. A minimum of 3 points must be used in each axis.

step gridstep_in_nm

A required argument set specifying the stepsize between grid points in nm.

data spatial

A required argument set that must be specified as is.

V(x1,y1)... V(xN,y1)

A required argument set specifying the kernel values at each point in the grid. Kernel values must be space‑separated and preceded by the data spatial argument set on a separate line. Kernel values are in the arbitrary units of the kernel, though the values are normalized in many cases.

Examples

Example 1

A basic PSK custom kernel file.

version 1 
type CustomKernel
numx 200
numy 200
step gridstep_in_nm
data spatial
	... 
	# kernel value matrix
	...
