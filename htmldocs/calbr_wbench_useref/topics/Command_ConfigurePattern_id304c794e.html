configure_pattern

Scope: Calibre Pattern Generator API Commands

Configures a pattern template index.

Usage

$cpg configure_pattern ‑name pattern_name
‑properties  "{prop_name prop_val}... "
[‑patternLabel pattern_label] [‑toneInverted {false|true}]

Arguments

$cpg

A required argument specifying $cpg as a Tcl configuration object or variable.

‑name pattern_name

A required argument set specifying the pattern template name.

‑properties "{prop_name prop_val}... "

A required argument set specifying the pattern template properties and their values. The list of properties and values must be enclosed with quotation marks.

‑patternLabel pattern_label

An optional argument set specifying the pattern label.

  • To define a single line of label text, specify the text after the ‑patternLabel argument.

  • To define multi-line label, add two consecutive semicolons between each line of label text. The following example places “3Bar” and “25nm” on separate lines:

    3Bar;;25nm

  • To use a variable in the label text, a dollar sign “$” must precede the variable name and followed by one of the following delimiters:

    • a space

    • two consecutive semicolons (that is, the new-line delimiter)

    • a dollar sign

    The following example places “3Bar” and a width value on separate lines:

    3Bar;;width $w nm
‑toneInverted false|true

An optional argument set specifying inversion of the pattern tone. To invert the pattern tone, specify “true”. The default is “false” (that is, no pattern tone inversion.

Description

A required command set defining a pattern template index for the layout configuration file. The configuration statement must be enclosed with braces in an “lappend pattern_list” statement. The command returns a configured pattern template index that is used by place_pattern.

Examples

The following example configures a pattern named 3Bar without pattern labels or tone inversion:

lappend pattern_list [$cpg configure_pattern ‑name 3Bar \
	‑properties "EW 200 S 200 CW 100 EL 2000 CL 2000"]
