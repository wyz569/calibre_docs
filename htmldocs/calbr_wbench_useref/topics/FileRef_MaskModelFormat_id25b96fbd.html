Mask Model Format

CM1 and VEB Model Formats

Input for: Model-Layer File Format

Defines a mask model.

The nominal mask model is used in combination with the optical model(s) for calculating dense aerial image intensities. It is used during modelflow_v2 calibration.

Format

A mask model must conform to the following restrictions:

  • ASCII text file.

  • The mask model is not compatible with sparse AI simulations, which means that your optical model must include ddm_model approx.

  • It can used only if options stage1 dense and stage2 dense are on. If stage1 is not required, it can be skipped. If stage2 is not required, you still need to set ‘stage2 dense try’ explicitly.

  • The mask model is indicated from the Model-Layer File Format using the following syntax line:

    mask_model [maskNum dose] {file | ‘{’ inlined_model_text ‘}’}

Parameters

maskNum

Specifies the mask number and exposure index (0 | 1 | 2 | 3) for this model. Required for multiple exposures only. The default is 0, which means that maskNum and dose are not needed for a single exposure simulation.

dose

Specifies the optional dose value. The default is 1. This option is required only if maskNum is specified.

file

Specifies the name of a file containing a mask model information block.

inlined_model_text

Specifies an inlined mask model information block, which must be inside braces ({}).

Either file or inlined_model_text is required.

Mask Model Information Blocks

A mask model information block defines the mask model, and contains the background transmission information plus a list of mask_layer statements. Each mask_layer statement uses a template specifying information about that layer.

When the litho model is used in Calibre nmOPC or Calibre OPCverify, each mask_layer statement in the mask model file is translated into an equivalent layer definition statement.

Note:

Mask model simulation support is incorporated as part of the litho model in Calibre OPCverify and Calibre nmOPC.

The mask model information block is defined as follows:

version 2

Specifies the version of the mask model. This must always be 2.

background trans

Sets the background transmission information for the mask model file. This must be the second line of the file, appearing right after “version 2”. trans can be one of the following values: dark, clear, atten x, phase90, phase270, phase180, real imaginary.

mask_layer n TRANS trans [CX value] [CC value] [BIAS value] [XBIAS value]    [YBIAS value] [XSHIFT value] [YSHIFT value] [DDM value] [CATEGORY name]

Defines the mask layer. At least one mask_layer statement with n=0 must be defined. Additional mask_layer statements are optional, and must increment n to the next sequential value.

  • CX value

    CC value

    Convex and concave cornerchop values, respectively, in microns, to be applied to the mask layer before using it in simulation.

  • BIAS value

    XBIAS value

    YBIAS value

    XSHIFT value

    YSHIFT value

    Bias and shift values in microns, to be applied to the mask layer before using it in simulation. If bias is used, specify either BIAS or XBIAS and YBIAS, but not both.

    Geometry preprocessing before simulation is applied in the following order:

    1) BIAS

    2) XSHIFT/YSHIFT

    3) XBIAS/YBIAS

    4) Corner chopping

DDM modelname | ddm_groupname

The name of a DDM model or DDM group, without path information. The ddm model either must be in the litho model directory or a symbolic link from the directory.

CATEGORY name

A user-defined name to document the usage of this mask_layer template. This is intended for use as a comment and is never referenced by the setup file.

Examples

#load a model from a file
mask_model maskmodel_v2.mod 
# define a mask model inline, dose 1 
mask_model 1 0.96 {
	version 2 
	background clear 
	mask_layer 0 TRANS atten 0.06 CX 0.02 CC 0.01 XBIAS 0.002 YBIAS 0.001 
	DDM test_light.ddm CATEGORY dense
}
