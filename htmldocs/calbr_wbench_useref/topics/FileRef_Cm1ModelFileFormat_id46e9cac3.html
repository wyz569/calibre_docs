CM1 Model File Format

CM1 and VEB Model Formats

Defines a resist model file.

Format

  • A CM1 model file is composed of keyword statements and comments.

  • Each statement must start its own line.

  • Any line beginning with a pound sign (#) is considered a comment line. 

    version number
modelType CM1
[RESIST_POLARITY {1|-1}]
DEFINE expression
...
COEFMAX cmax
THRESHOLD tvalue
CONST_TERM factor
[SMOOTH_NEUTRALIZATION {2|1|0}]
[SMOOTH_GRADIENT {2|1|0}]
[SHRINK_DIFFUSION i length]
[SHRINK_RADIUS i radius]
[SHRINK_THRESHOLD i threshold_value]
[SHRINK_POLARITY i polarity]
[SHRINK_SIDO {0|1}]
MCOEF iterm coeff
...
[SCOEFF (n * i) value
 SCOEFF (n * i + 1) value
 SCOEFF (n * i + 2) value
 [SCOEFF (n * i + 3) value]] # (used only when SHRINK_SIDO is 1)
	# (n=3 or 4, based on SHRINK_SIDO setting)
...
[HORIZONTAL_BIAS a=aval b=bval s=sval d=dval c=cval]
[SEM_SHRINK a=aval b=bval r=rval s=sval c=cval d=dval e=eval]
[SIDEWALL_BIAS v=vval t=tval b=bval d=dval s=sval c=cval]
[VERTICAL_SHRINK u=uval v=vval l=lval t=tval d=dval s=sval c=cval]
[Z_NOMINAL value
[Z_SLOPE a=aval b=bval c=cval]]
[CURVATURE_TERM i t=tval d=dval s=sval c=cval]
[DISSOLUTION i type={Standard | Balanced} sym=sym_val
 {+|-}threshold=threshold_val span=sval radius=rval 
 orderp=pval orderq=qval coef=cval]
[CONSISTENCY_SETTING {0 | 1}]
[PATTERN_DENSITY i {+|-}threshold=threshold_val span=sval
 [radius=rval] coef=cval]
[ELECTRON_ABSORPTION i threshold=threshold_val fov=fov_val
 [radius=rval] coef=cval]
[CROSSTERM i term1=term_name1 term2=term_name2 coef=cval]
[MASK_LOADING i [type={AllMask | MainMask | SrafMask |  AsrafMask | 
 AllEdge | MainEdge | SrafEdge | AsrafEdge}]
 [maskconsistency {0 | 1}] [reversetone= 0 | 1] [srafsize=size_nm]
 [span=sval] [coef=cval]]
[PEB_SHRINK i {+|-}threshold=threshold_val span=sval
 [radius=rval] smax=shrink_max_val scale=scale_val [skew=skew_val]
 [poisson_ratio=poisson_val]  [coef=cval] ]
[RESIST_LAMBDA = lambda_val]
[RESIST_MU =  mu_val] 
[PSK_TERM i [{+|-}threshold=threshold_val] [span=sval]
 [kernelinline=psk_filename] [ coef=cval] ]
[KERNELINLINE kernel_filename
  kernel_block]

Parameters

version number

Only one version statement is allowed in the CM1 file. Starting with Calibre WORKbench 2018.4, all new calibrated models use version 10 (the default). Version 10 improves the normalization of symmetrization type -1 dissolution kernels.

modelType CM1

Only the CM1 model type is supported at this time.

RESIST_POLARITY {1| -1}

Optionally sets the resist polarity. This setting is only used by 3D-ready CM1 models, and is inactive for standard 2D CM1 models. Use 1 for PTD resists. Use -1 for NTD resists. The default is 1 (positive tone).

DEFINE expression

The DEFINE statement defines modeling terms. expression has the following expanded syntax:

DEFINE Miterm {+|-}b=bterm k=kterm n=nterm s=sterm p=pterm

where Table 1 defines the parameters:

Table 1. Modeling Term Definitions

Term

Definition

Default

iterm

A term number. Up to 40 terms (M0 through M39) are supported.

0

bterm

The value of the neutralization parameter b. An explicit plus or minus sign must be specified in front of b.

0

kterm

The value of the derivative order parameter k.

0

nterm

The value for the power of the term, parameter n.

1

sterm

The diffusion length in nanometers.

0

pterm

The Gauss-Laguerre base order.

0
COEFMAX cmax

Regularization constant. Should not be changed by the user.

Note:

Starting with the 2018.2 release, the default value of COEFMAX was updated to 1.2E-4. Models created and calibrated in the 2018.2 release will produce different results than models created in an earlier release and calibrated in the 2018.2 release.

THRESHOLD tvalue

Value of the constant term T.

CONST_TERM factor

Floating point value defining the aerial image to CM1 normalization factor. This can be left at zero or tuned.

SMOOTH_NEUTRALIZATION {2 | 1 | 0}

Toggles the use of the smooth neutralization algorithm. 2 = spline smoothed neutralization, default starting with 2012.2, 1 = square root smoothed neutralization, or off (0).

SMOOTH_GRADIENT {2 | 1| 0}

Toggles the use of gradient smoothing calculated as the square of the gradient (2, default), square root of the gradient (1) or off (0).

SHRINK_DIFFUSION i length

An optional value of resist shrink smoothing diffusion length for the i-th resist shrink term. The index i may be omitted, in which case it is assumed to be equal to 0. The shrink diffusion length must be non-negative. A resist shrink diffusion length of 0 means no smoothing for this resist shrink term.

SHRINK_RADIUS i radius

An optional value of the resist shrink radius in nanometers for the resist shrink term. The index i may be omitted, in which case it is assumed to be equal to 0. The shrink radius must be non-negative. A resist shrink radius of 0 means that the resist shrink term with index i is ignored. If there are k resist shrink terms (or, more precisely, triples of terms) in the model, then the resist shrink term indices must go from 0 to (k - 1), without any omissions.

If a resist shrink term is present, then its shrink radius, shrink threshold, shrink polarity, and three shrink coefficients (SCOEFF) must all be present.

SHRINK_THRESHOLD i threshold_value

An optional value of the resist shrink threshold for the resist shrink term i. The index i may be omitted, in which case it is assumed to be equal to 0.

SHRINK_POLARITY i polarity

An optional value of the resist shrink polarity for the resist shrink term i. Its value polarity must be 1 or -1. The index i may be omitted, in which case it is assumed to be equal to 0.

SHRINK_SIDO {0 | 1}

An optional parameter used to toggle the additional shrink induced development offset (SIDO) coefficient for shrink terms. By default, SHRINK_SIDO is off (0).

MCOEF iterm coeff

For each DEFINE statement, an MCOEF statement must also be defined with a matching iterm value (the number of coefficients must be exactly equal to the number of DEFINE statements). coeff is the floating point coefficient value of the term.

Tip

By default, the following SCOEFF (shrink coefficient) terms come in triplets (n=3), which means that three SCOEF coefficients are required for each shrink term. The notation (n *i + m) indicates one of the three coefficients. For example, resist shrink term 1 (i=1) uses SCOEF 3, 4, and 5.

However, if the SHRINK_SIDO parameter is set to 1, a fourth coefficient is added to handle the SIDO term, which means that four SCOEF coefficients (n=4) are required for each shrink term. Using the previous example, resist shrink term 1 (i=1) would use SCOEF 4, 5, 6, and 7.

SCOEF (n * i) value

An optional parameter used to represent the resist shrink term coefficient s0 for the resist shrink term i.

SCOEF (n * i + 1) value

An optional parameter used to represent the resist shrink term coefficient s1 for the resist shrink term i.

SCOEF (n * i + 2) value

An optional parameter used to represent the resist shrink term coefficient s2 for the resist shrink term i.

SCOEF (n * i + 3) value

An optional parameter used to represent the resist shrink term coefficient s3 for the resist shrink term i. This parameter can only be used when SHRINK_SIDO is set to 1.

HORIZONTAL_BIAS a=aval b=bval s=sval d=dval c=cval

An optional parameter specifying one or more values for the horizontal bias terms a, b, s, d, and the horizontal bias term coefficient c. The order of parameters is required.

  • Parameters a and b are dimensionless.

  • Parameter a must be non-negative.

  • Parameter b must be positive.

  • The smoothing length d is in nanometers and must be non-negative.

  • The diffusion length parameter s is in nanometers and must be non-negative.

  • Parameter c is a linear coefficient.

Note:

HORIZONTAL_BIAS is calibrated as hbias_a0, hbias_b0, hbias_s0, and hbias_d0 by default. If more than one HORIZONTAL_BIAS parameter is specified in the model file, each additional parameter is given the next higher index for calibration purposes. In other words, the second HORIZONTAL_BIAS statement found is calibrated with hbias_a1, hbias_b1, hbias_s1, and hbias_d1.

SEM_SHRINK a=aval b=bval r=rval s=sval c=cval d=dval e=eval

An optional parameter used to represent the resist shrinkage from CDSEM resist measurement.

  • Parameters a (the intensity scale) and b (the intensity threshold) are dimensionless, but must be positive.

  • Parameter r is the shrink radius in nanometers, and must be positive.

  • Parameter s is an optional parameter for smoothing diffusion length. Recommended.

  • Parameters c, d, and e are term coefficients.

SIDEWALL_BIAS v=vval t=tval b=bval d=dval s=sval c=cval

An optional parameter specifying the sidewall bias term parameters v, t, b, d, and s, and the sidewall bias term coefficient c, in that order.

  • Parameter v is a version. Versions 1 and 3 are supported. Modelforms that contain SIDEWALL_BIAS use version 3. Otherwise, the default is 1.

  • Parameters t and b (intensity thresholds) are dimensionless and must be positive.

  • Parameter d is used as a smoothing term to improve consistency. It must be set to be larger than 3*the simulation pixel size starting with the 2017.2 release, but can be set to 0 for backwards compatibility with pre-2017.2 releases.

  • Parameter s is the diffusion length parameter in nanometers. It must be non-negative.

VERTICAL_SHRINK u=uval v=vval l=lval t=tval d=dval s=sval c=cval

An optional parameter specifying vertical shrinkage for 3D resist models.

  • Parameters u and v are the uniform and variable portion of vertical shrinkage, normalized to the thickness of the resist film.

  • Parameters l and t are transition length and dose relative to the image intensity.

  • Parameter d is a positive vertical smooth parameter normalized to the thickness of the resist film.

  • Parameter c is a linear coefficient.

Z_NOMINAL value

An optional parameter specifying the value in microns of the nominal resist plane for 3D resist models. It follows the resist plane convention with the minimum value of 0 at the resist bottom interface, and the maximum value of the resist film thickness at the resist top interface. The value must be within this range.

Z_SLOPE a=aval b=bval c=cval

An optional parameter to model the z-varying slope of 3D resist. Z_SLOPE makes the regular slope term z-dependent. The equation is 1 + avalz + bvalz2 + cvalz3.

To use Z_SLOPE, the regular slope term (a DEFINE statement with k=1) must be present in the default CM1 model.

Z_SLOPE can only be specified with Z_NOMINAL.

CURVATURE_TERM i t=tval d=dval s=sval c=cval

An optional parameter specifying one or more values for the contour curvature terms t, d, s, and the curvature coefficient c. A maximum of 2 contour curvature terms can be used starting with index 0. The required index i may be 0 or 1 and cannot be duplicated.

  • Parameter t is the threshold for the aerial image contour and is dimensionless. The value must be positive.

  • Parameters d and s are diffusion and smoothing lengths in units of nm.

  • Parameter c is a linear coefficient.

DISSOLUTION i type= {Standard | Balanced} sym=sym_val {+|-}threshold=threshold_val span=sval radius=rval orderp=pval orderq=qval coef=cval

An optional parameter specifying one or more values for the dissolution terms type, sym, threshold, span, radius, orderp, orderq, and the dissolution coefficient coef. A maximum of 20 dissolution terms can be used. The required index i may be 0 up to 19 and cannot be duplicated. The indices must be sequential starting with 0 and have no breaks in the sequence.

  • Parameter type is the kernel type. The default value is Standard, and dissolution kernels prior to the 2020.2 release used the Standard kernel. The Balanced kernel may improve prediction results on product structures.

  • Parameter sym is the symmetrization type. The specified value must be 1, 2, or -1.

  • Parameter threshold specifies a neutralization threshold level. The parameter is dimensionless and must be positive, A ‘+’ specifies plus (acid) neutralization and a ‘-’ specifies minus (base) neutralization.

  • Parameter span is the dissolution diffusion length in nanometers, and must be positive.

  • Optional parameter radius is a dissolution smoothing length in nanometers, and must be positive. Default value is 0nm, and 40nm is recommended.

  • Parameters orderp and orderq are the dissolution kernel orders. Values of 0, 2, and 4 are supported.

  • Parameter coef is a linear coefficient.

CONSISTENCY_SETTING {0 | 1}

An optional model configuration setting. Specifying 1 for CONSISTENCY_SETTING provides improved model consistency, at a possible cost of increased runtime during the calibration step. It changes the way CM1 handles the following items:

  • Aerial images are modified before performing CM1 simulations so that boundary discontinuities of the aerial image are removed, but the aerial image in the central region of the grid is not changed. This modification reduces the high frequency Fourier components of the aerial image and makes the computation of aerial image derivatives (which are computed in many CM1 models) more stable.

  • For calibration purposes (but not for simulations), the resist interaction distance is increased, which makes computation of the convolutions with the Gauss-Laguerre kernels more exact. Resist interaction distance is not increased in the Calibre simulations because during simulations, the whole frame includes the optical interaction radius along with the resist interaction radius, while in the calibration stage only the resist interaction radius is used.

  • COEFMAX is increased for horizontal calibration.

  • The model version is upgraded to version 7 on output.

Starting with release 2015.1, CM1 uses a CONSISTENCY_SETTING of 1 by default. Use setting 0 to keep your CM1 model consistent with previous Calibre release behavior.

PATTERN_DENSITY i {+|-}threshold=threshold_val span=sval radius=rval coef=cval

An optional parameter used to represent resist shrinkage related to pattern density for NTD resists.

A maximum of 3 pattern density terms can be used. The required index i may be 0 up to 2 and cannot be duplicated.

  • Parameter threshold specifies a neutralization threshold level. The parameter is dimensionless and must be positive, A ‘+’ specifies plus (acid) neutralization and a ‘-’ specifies minus (base) neutralization.

  • Parameter span specifies a diffusion length in nanometers.

  • Optional parameter radius specifies a smoothing length in nanometers. Default is 40nm.

ELECTRON_ABSORPTION i threshold=threshold_val fov=fov_val radius=rval coef=cval

An optional parameter used to represent resist shrinkage related to pattern density during CDSEM measurement.

A maximum of 3 electron absorption terms can be used. The required index i may be 0 up to 2 and cannot be duplicated.

  • Parameter threshold specifies a neutralization threshold level.

  • Parameter fov specifies a diffusion length in nanometers.

  • Optional parameter radius specifies a smoothing length in nanometers. Default is 40nm.

  • Parameter coef is a linear coefficient.

CROSSTERM i term1=term_name1 term2=term_name2 coef=cval

An optional parameter that is the product of existing terms. The product can be either the combination of two separate terms or one term multiplied by itself (effectively squaring the term).

  • Parameter i is a required index.

  • Parameters term_name1 and term_name2 are modeling terms defined in the CM1 model file. The term name parameter supports the following:

    • The modeling terms M0 through M39 in the DEFINE statement.

    • CURVATURE0 and CURVATURE1 for the corresponding CURVATURE_TERM.

    • DISSOLUTION0 through DISSOLUTION19 for the corresponding DISSOLUTION term.

    • HORIZONTAL_BIAS for the corresponding HORIZONTAL_BIAS term.

    • MASK_LOADING0 through MASK_LOADING3 for the corresponding MASK_LOADING term.

    If both term_name1 and term_name2 specify the same modeling term, this effectively squares the modeling term.

  • Parameter coef is a linear coefficient.

MASK_LOADING i type={AllMask | MainMask | SrafMask | AsrafMask | AllEdge | MainEdge | SrafEdge | AsrafEdge} [maskconsistency {0 | 1}] [reversetone=0 | 1] [srafsize size_nm] span=sval coef=cval

An optional parameter specifying mask loading contributions to the resist model.

A maximum of four tunable mask loading terms can be used. The required index i may be 0, 1, 2, or 3 and cannot be duplicated. Mask loading terms with non-tunable parameters can also be added.

  • Parameter i is a required index.

  • Parameter type is the type of mask layer used as image input. The following mask layer types are supported:

    • AllMask — Image input is all patterning mask layers. This is the default.

    • MainMask — Image input is main features only.

    • SrafMask — Image input is SRAF features only.

    • AsrafMask — Image input is negative SRAF features only.

    • AllEdge — Image input is the edge mask for all patterning mask layers.

    • MainEdge — Image input is the edge mask for main features only.

    • SrafEdge — Image input is the edge mask for SRAF features only.

    • AsrafEdge — Image input is the edge mask for negative SRAF features only.

    The mask layer types AllMask, MainMask, SrafMask, and AsrafMask use whole polygons as input; the edge mask layer types AllEdge, MainEdge, SrafEdge, and AsrafEdge use the edges of the input mask polygons as input.

  • Parameter maskconsistency sets the mask rasterization method. The following settings are supported:

    • 0 — Type 0 rasterization. This is the default when type equals AllMask, MainMask, SrafMask, or AsrafMask.

    • 1 — Type 1 rasterization. This is the default when type equals AllEdge, MainEdge, SrafEdge, or AsrafEdge.

  • Parameter reversetone sets the mask tonality relative to the aerial image. The following settings are supported:

    • 0 — Sets the mask tone to be the same as the aerial image. This is the default.

    • 1 — Sets the mask tone to the reverse of the aerial image.

  • The srafsize parameter classifies all polygons in the input mask layer smaller than or equal to size_nm as SRAFs. This argument is required for mask types MainMask, SrafMask, and AsrafMask and ignored for mask type AllMask.

  • The diffusion length parameter span is in nanometers and must be non-negative.

  • Parameter coef is a linear coefficient.

PEB_SHRINK i {+|-}threshold=threshold_val span=sval [radius=rval] smax=shrink_max_val scale=scale_val skew=skew_val [poisson_ratio=poisson_val] coef=cval

An optional parameter used to represent resist shrinkage related to the post exposure bake (PEB) process.

For many applications, one PEB_SHRINK term is sufficient, but in some cases, two terms can be used. Only parameters in the first and second terms are tuned. The required index i must be 0 or greater and cannot be duplicated. RESIST_LAMBDA and RESIST_MU must be specified when using PEB_SHRINK.

  • Parameter threshold specifies a neutralization threshold level. A ‘+’ threshold is typically specified for PEB resist shrinkage.

  • Parameter span specifies a diffusion length in nanometers.

  • Optional parameter radius specifies a smoothing length in nanometers. Default is 40nm.

  • The parameters smax, scale, and skew represent a method of modeling the shrink-loss curve.



    Figure 1. PEB shrink-loss curve

    • The parameter smax represents a normalized maximum shrinkage where 0 represents no shrinkage during the PEB step. The smax value should be between 0 and 1, with typical values being between 0 and 0.3. The default smax value is 0.

    • The parameter scale represents the width of transition in dose from no resist thickness loss to maximum thickness loss. A good starting value is one fourth of the dose range over which PEB shrinkage occurs. So if PEB shrinkage occurs over a dose range of 0.8, then 0.2 is a good starting value for scale. The scale value is typically tuned between 0.01 and 0.2.

    • The parameter skew represents the skew order of the polynomial. The default skew value is 8.

  • Parameter poisson_ratio is the Poisson ratio value for the resist.The Poisson ratio represents the ratio that a material expands in the transverse direction when compressed or stretched along one axis. Typical values are between 0 and 0.49. The default value is 0.49.

  • Parameter coef is a linear coefficient.

RESIST_LAMBDA = lambda_val RESIST_MU = mu_val

Two parameters required when using the PEB_SHRINK parameter. The parameters are the Lamé first and second coefficients that model the stress-strain relationship for the resist.

Typical values are 5 GPa for resist_lambda and 2 GPa for resist_mu. Units are gigapascals (GPa).

PSK_TERM i {+|-}threshold=threshold_val span=sval kernelinline=psk_filename coef=cval

An optional parameter specifying pattern specific contributions to the resist model. Commonly used with NTD models.

The required index i must be positive integer and cannot be duplicated. No PSK_TERM parameters are tunable.

  • Parameter i is a required index.

  • The diffusion length parameter span is in nanometers and must be non-negative.

  • The kernelinline parameter specifies a PSK filename created by an “mdf optimize psk” command. The default psk_filename value is “toppatt_num.psk” where patt_num is the total number of patterns.

  • Parameter coef is a linear coefficient.

KERNELINLINE kernel_filename

kernel_block

An optional parameter set defining a custom kernel inside the CM1 file.

  • The kernel_filename specifies the name of the inline kernel.

  • The kernel_block specifies the custom kernel. See “Custom Kernel File Format” for the block contents and format.

Multiple KERNELINLINE statements are supported in a CM1 file. The KERNELINLINE statements are always appended to the CM1 model file text after the non-custom kernel block.

The KERNELINLINE parameter set currently supports inline pattern specific kernels (PSKs). The KERNELINLINE kernel_filename must match the PSK_TERM psk_filename.

Examples

Example 1

A basic CM1 model. 

#Comments are here
version 10
modelType CM1
DEFINE M0 +b=0.1 n=1 s=40 p=0
DEFINE M1 -b=0.2 k=1 n=2 s=33
DEFINE M2 s=10
COEFMAX 1.200000e-04
THRESHOLD 0.25
CONST_TERM 0.000000e+00
SMOOTH_NEUTRALIZATION 1
SMOOTH_GRADIENT 2
MCOEF 0 1
MCOEF 1 0.01
MCOEF 2 -0.01

Example 2

A CM1 model with shrink, bias, contour curvature, and dissolution terms.

#Comments are here
version 10
modelType CM1
DEFINE M0 +b=0.1 n=1 s=40 p=0
DEFINE M1 -b=0.2 k=1 n=2 s=33
DEFINE M2 s=10
DEFINE M3 k=1 s=80
ANCHOR 0
COEFMAX 1.200000e-04
THRESHOLD 0.25
SMOOTH_NEUTRALIZATION 2      # use smooth neutralization 2
SMOOTH_GRADIENT 2                  # use smooth gradient 2
MCOEF 0 1
MCOEF 1 0.01
MCOEF 2 -0.01
MCOEF 3 0.05
SHRINK_RADIUS 0 150.0 # resist shrink term 0, radius 150nm
SHRINK_THRESHOLD 0 0.2 # resist shrink term 0, threshold 0.2
SHRINK_POLARITY 0 -1 # resist shrink term 0, polarity -1
SHRINK_RADIUS 1 100.0 # resist shrink term 1, radius 100nm
SHRINK_THRESHOLD 1 0.3 # resist shrink term 1, threshold 0.3
SHRINK_POLARITY 1 1 # resist shrink term 1, polarity 1
SCOEF 0 0.5 # resist shrink term 0, coefficient s0 = 0.5
SCOEF 1 -0.2 # resist shrink term 0, coefficient s1 = -0.2
SCOEF 2 0.4 # resist shrink term 0, coefficient s2 = 0.4
SCOEF 3 0.3 # resist shrink term 1, coefficient s0 = 0.3
SCOEF 4 0.15 # resist shrink term 1, coefficient s1 = 0.15
SCOEF 5 -0.1 # resist shrink term 1, coefficient s2 = -0.1
HORIZONTAL_BIAS a=0.4 b=0.2 s=200 d=100 c=0.1  # horizontal bias term
SIDEWALL_BIAS v=3 z=-.06 t=0.3 b=0.5 s=200 c=0.1 # sidewall bias term
VERTICAL_SHRINK u=0.2 v=0.2 l=0.3 t=0.3 d=0.01 s=100 c=1 # vertical_shrink term
CURVATURE_TERM 0 t=0.2 d=20 s=20 c=1 # curvature term
DISSOLUTION 0 sym=1 +threshold=0.1 span=60 radius=40 orderp=2 orderq=0 coef=0.1 # dissolution term 0
DISSOLUTION 1 sym=2 -threshold=0.1 span=60 radius=40 orderp=2 orderq=0 coef=0.1 # dissolution term 1
