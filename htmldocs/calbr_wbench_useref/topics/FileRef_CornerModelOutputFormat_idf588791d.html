Corner Model Output Format

DDM Modeling Extension

The corner model modifies DDM modeling by changing the way that mask transmission signals are computed.

When a DDM corner model is used, corner signals are added to the mask transmission functions. The signals approximate the average electric field due to each corner’s 3D geometry.

Format

corner_model c2 
[small_corner_limit value]
corner_signal {bright_feature | dark_feature} 
	corner_type {NW | NE | SE | SW} 
	theta offaxis_angle phi azimuthal_angle 
	polarization {TE | TM | UNPOLARIZED} ‘{’
		corner_size signal_real signal_imag offsetx offsety
		[corner_size signal_real signal_imag offsetx offsety]...
	‘}’

Parameters

corner_model c2

A required command identifying which file format the rest of the corner keywords follow. Only c2 is supported.

small_corner_limit value

An optional command that limits where in the layout DDM corner signals are added. Extremely short edges will not behave electromagnetically as corners, so the signal should not be added.

The value is in mask-scale microns. If the minimum adjacent edge to a corner (before corner chop, if applicable) is less than value, then the corner signal will not be added. The default is 0.010.

corner_signal {bright_feature | dark_feature} corner_type {NW | NE | SE | SW} theta offaxis_angle phi azimuthal_angle polarization {TE | TM | UNPOLARIZED} ‘{’

A required argument for corner modeling that signifies the beginning of a corner signal. All settings in this line are required. Each occurrence of the keyword signifies a single impulse for a unique corner.

  • bright_feature or dark_feature — Specifies the tone of the corner as shown in Figure 4 in the section “DDM Library Input File Format”.

  • corner_type — Specifies the orientation of the corner.

  • theta — Specifies the off-axis angle in degrees.

  • phi — Specifies the azimuthal angle in degrees.

  • polarization — Specifies the polarization.

Note that the settings for theta, phi, and polarization are identical to the settings that correspond to the main DDM signals elsewhere in the file. Although it is not required to have the main signals and the corner signals specified at identical conditions for theta, phi, and polarization, there is no known reason to specify them for separate conditions.

  • { — An opening brace that indicates the start of the signal definition. The brace must be followed by a new line.

corner_size signal_real signal_imag offsetx offsety

A set of keywords specifying a signal for the corner described by corner_signal. This set must be used at least once, but can be specified any number of times. Calibre Masksim generates these signals when given a DDM Library Input File Format that contains the corner_model command.

  • corner_size — Specifies the length in mask-scale microns for the corner edge. Manhattan corners use 0.

  • signal_real signal_imag — Specifies the value of this particular corner signal.

  • offsetx offsety — Specifies the position of the signal relative to the corner.

}

A required closing brace that indicates the end of the corner signal block. This brace must be on a separate line.
