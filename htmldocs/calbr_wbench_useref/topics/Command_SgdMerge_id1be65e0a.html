sgd_merge

CM1 and VEB Commands

Merges two gauge or super gauge data objects or files.

Usage

sgd_merge {–obj1 obj | –sgf1 file} {–obj2 obj | –sgf2 file} [–out file]
[‑keepreplicas {all | 1 | 2 | none}]

Arguments

–obj1 obj | –sgf1 file

A required argument specifying the first input gauge data or super gauge data object or file.

–obj2 obj | –sgf2 file

A required argument specifying the second input gauge data or super gauge data object or file.

–out file

An optional argument specifying an output file name to write the merged object. The default is to return a Tcl object.

–keepreplicas {all | 1 | 2 | none}

An optional argument that tells the tool how to handle gauge replicas. If the second super gauge data file or object contains a gauge data record whose coordinates as well as defocus and dose values coincide with that of a gauge in the first super gauge data file or object, this gauge is considered to be a replica. Accepted options are:

  • all – Keep enabled replicas from both input sgd files/objects. This is the default behavior.

  • 1 – Keep enabled replicas from the first super gauge data file or object and disable the corresponding replicas in the second super gauge data file or object.

  • 2 – Keep enabled replicas from the second super gauge data file or object and disable the corresponding replicas in the first super gauge data file or object.

  • none – Disable replicas found in both input super gauge data files or objects.

Examples

# merge two sgd files and save the resulting dataset to a file
set obj [sgd_merge -sgf1  “res.sgd” -sgf2 “veb.sgd” -out “res_etch.sgd”]
