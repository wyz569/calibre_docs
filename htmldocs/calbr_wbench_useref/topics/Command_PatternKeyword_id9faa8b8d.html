PATTERN Keyword

Test Pattern Commands

Defines the header information inside a test pattern file.

Note:

The PATTERN keyword is used as the “header” information for a test pattern file. It must be accompanied by at least one FEATURE Keyword after all its subkeyword arguments have been defined. Each subkeyword must be on its own line.

Usage

PATTERN pattern_name
dbgrid db_grid
designgrid design_grid
topcellname top_cell_name
[gdsfile gds_file_name]
[oasisfile oasis_file_name]
ssfile ss_file_name
[ggfile gg_file_name]
markers {none | sem | oddeven | cross}
border {0 | 1}
[blockheight val]
[blockwidth val]
[blockclip {0 | 1}]
[insetheight val]
[insetwidth val]

Arguments

dbgrid db_grid

A required argument specifying the database grid, in user units (microns), in which to specify the test pattern features.

designgrid design_grid

A required argument specifying the design grid on which to snap data, specified in user units (microns). Must be a multiple of dbgrid.

topcellname top_cell_name

A required argument specifying the name of the top cell to create for this test pattern.

Note:

At least one output option (gdsfile or oasisfile) must be specified, but both can also be specified.

gdsfile gds_file_name

An optional argument that specifies the output file to write in GDS format, specified as either an absolute path or relative to the directory from which you invoke the Calibre WORKbench application.

oasisfile oasis_file_name

An optional argument that specifies the output file to write in OASIS format, specified as either an absolute path or relative to the directory from which you invoke the Calibre WORKbench application.

ssfile ss_file_name

A required argument that specifies the output sample spreadsheet file to write specified as either an absolute path or relative to the directory from which you invoked Calibre WORKbench.

ggfile gg_file_name

An optional argument that specifies an output gauge file to write specified as either an absolute path, or relative to the directory from which you invoked Calibre WORKbench. No gauge file is written if this option is not specified.

markers {none | sem | oddeven | cross}

A required argument that specifies the type of SEM alignment markers to use.

border {0 | 1}

A required argument that sets whether the generator outputs a border around the pattern.

blockheight val

blockwidth val

An optional argument set that specifies the height and width of each pattern block in microns. (default: 7.0 um)

Note:

Previous to 2017.2, only a single blocksize variable was available to set the block size. In the 2017.2 release, you must convert blocksize statements in your .tpf file to blockheight and blockwidth; tepCompile issues an error otherwise.

blockclip {0 | 1}

An optional argument that specifies the output to be clipped to the pattern block. Default is 1 (active).

insetheight val

insetwidth val

An optional argument set that specifies the inset height and width in microns. (default: 1.5 um)

Note:

Previous to 2017.2, only a single inset variable was available to set the inset size. In the 2017.2 release, you must convert inset statements in your .tpf file to insetheight and insetwidth; tepCompile issues an error otherwise.
