Using the HHA Library With Calibre OPCverify and Calibre nmOPC

A combined set of optical and DDM libraries can be loaded into Calibre nmOPC and Calibre OPCverify.

The commands used are as follows:

  • optical_model_load

  • ddm_model_load

  • image

    Tip

    These commands are described in the Calibre OPCverify User’s and Reference Manual and the Calibre nmOPC User’s and Reference Manual.

Prerequisites

  • You must have completed the “Creating a DDM Library With masksim Batch” task.

  • Re-calibrated the optical model using the task “Optimizing the HHA-Enhanced Optical Model.”

  • Licenses for Calibre nmOPC or Calibre OPCverify.

Procedure

  1. In a text editor, create a Calibre nmOPC or Calibre OPCverify command file.
  2. Use the optical_model_load command to load the new optical model from the task “Optimizing the HHA-Enhanced Optical Model”.
  3. Add one or more ddm_model_load calls:
    ddm_model_load ddm_label [ddm_input_filename | { inline_definition }]

    where ddm_label is the unique name that Calibre OPCverify will use to refer to the DDM library, followed by one of the following kinds of files:

    • A .ddm output file from the task “Creating a DDM Library With masksim Batch”

    • A filename to a masksim input file

    • An inline masksim input file

  4. Reference the model from Calibre OPCverify image commands. Each layer substatement can have a DDM model associated with it, such as in the following example fragment:
    setlayer c1 = image optical mo1 background clear \
        layer M1OPC -1.245 0.0 ddm ddmm1 \
        layer sbar -1.245 0.0 ddm ddmm2 \
        resist_model mr1
  5. Run Calibre OPCverify or Calibre nmOPC as normal.

Results

You should see normal results from your Calibre OPCverify or Calibre nmOPC script, with the accuracy changes that come from having a DDM and HHA model used in place of a normal optical model.
