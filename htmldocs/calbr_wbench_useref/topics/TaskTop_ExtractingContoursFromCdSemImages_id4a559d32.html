Extracting Contours from CD SEM Images

Use the Contour Data Flow tool to extract contours from CD SEM images.

Prerequisites

  • Input gauge file path.

  • Input layout file path.

  • Directory with CD SEM images.

Procedure

  1. In Calibre WORKbench, invoke the Calibre SEMSuite from the Litho > Calibre SEMSuite menu item.
  2. Select the Contour Data Flow (CDF) tool by clicking the Tools icon at the top right and selecting Contour Data Flow in the menu.


  3. Add a CDF session to the session list using one of the following:

    • Load a session saved to disk. For details, refer to Step 4.

    • Create a new session. For details, see Step 5.

  4. To load a session from disk, select Add Session > Load Session File.
  5. To create a new session:
    1. Select Add Session > Gauge File Session.
    2. In the Open Gauge File Session dialog box, configure the input gauge file path, SEM images base directory, and input layout file path. Specify the SEM images pixel size and layout input layer. Click Ok.
    3. (Optional) Change the session name from the “CDF Session x” default by clicking on the name.
  6. Click the Settings gear icon at the top right and configure the setting values.

    • Default Setup File — Imports settings from a specified setup file.

    • Results Directory — Specifies the file path to output results. The default file path is /CDF_Runs/Session_x where x is the session id.

    • Input Layout — Specifies the input layout file path and layer.

    • Output Precision and Analysis Precision — Siemens EDA recommends an initial value of 200000.

  7. Create a new run in the Runs Browser by clicking the “+” icon.

    The Runs Browser displays the parameters to use in the contour extraction run. For the first run, the default parameter values can typically be used. Optimize the parameter values in subsequent runs.

  8. (Optional) Categorize SEM images as accepted, rejected, or undecided by selecting the images and clicking the corresponding category icon. The categories allow you to selectively run contour extraction on specified groups of images. If no images are categorized, the tool applies the contour extraction run settings to all images.
  9. Start the contour extraction run by clicking Run in the Runs Browser.

    As each image is processed, the processed image with contours appears in the All Targets window, and the Log viewer displays a status message for contour extraction on the gauge.

  10. Inspect each processed image and the contours.

    Click on individual images to inspect individual images at higher resolution and click the arrow next to the site name at the top left to return to the All Targets view. For brightfield images, compare the inner contour to the feature edges; for darkfield images, compare the outer contour to the feature edges. For successful contour extraction, the inner or outer contour should cleanly capture the feature edges in the SEM image and introduce no extra contours.

    Figure 1. Clean Optimized Contour from a SEM Image

  11. To create a contour extraction run from a previous-executed run, perform one of the following:

    • Click the + icon to add a new run and configure all parameters for the contour extraction run.

    • Click the Clone Setup icon next to the setup name and adjust the run parameter values of interest. The following parameters are typically adjusted:

      • coarse contour tuning

        In the Contour Extraction Parameters table, adjust the coarse contour tuning parameter when the inner contour is undersized or oversized relative to the feature. If the inner contour is undersized, then increase the coarse contour tuning value. If oversized, then decrease the coarse contour tuning value.

        Figure 2. Undersized Inner Contour from a SEM Image

        Figure 3. Oversized Inner Contour from a SEM Image

      • roi in pixels

        In the Filtering Parameters table, this parameter creates a region of interest (roi) to exclude pixels from contour extraction. Specify the region of interest in the following format:

        lower_left_x:lower_left_y;upper_right_x: upper_right_y

        where the coordinates represent the lower left and upper right corners of a bounding box in pixels.

    • number of measurements

      space between measurements in pixels

      In the Contour Extraction Parameters table, number of measurements specifies the number of cutlines averaged for a measurement and space between measurements in pixels specifies the gap distance between each cutline. Adjust the cutline number and spacing to match the measurable region of the feature.

  12. Continue to create and test contour extraction run settings until you identify a best-performing set of parameters.
  13. Save the session settings, hover the cursor over the session name and click the disk icon.

Results

The tool automatically saves an output_layout.oas layout file for the most recent run. The WORKbench viewer displays the layout file from the most recent run. Contour results are stored in a site gauge file which can be used for model calibration.
