Topographical Modelform Parameters (topomf)

modelflow_v2 modelform

In modelflow_v2, the topo optimization settings and topo model can be specified either implicitly or explicitly. Use the topomf parameter to implicitly set a topographical modelform.

Arguments

topomf string

A required argument that specifies the topo model form to be calibrated. The name of the model form defines which topo signals should be used to generate model and optimization settings. Each signal is signified with a code number or letter (or two letters in the case of FinFET sidewall, “Fw”), and the available modelforms are specific combinations of the codes. For example, “topo3STGI” is “Post-PC, density underlying, sidewall, trench, gate, and interconnect signals.”

Note:

Previous versions of the topomf option (before Calibre 2017.2) listed additional signals, such as “T2” (trench2), “Sim”(sidewall imaginary), and “Tim” (trench imaginary). Modelforms that included those signals (such as “topo2SSim”) are no longer accepted, but the individual signals can still be added as custom signals.

For each of the following arguments, three types of argument configurations are permitted:

  • Single value (for example, topomf_dsigma 1.1) — Sets the term to be the specified value, and the term is not calibrated. “auto” may not be used in single value mode.

  • Double value (for example, topomf_dsigma 0.7 1.3) — Sets the search values explicitly to the specified range.

  • Mixed value and “auto”, or both “auto” (for example, topomf_disigma 0.7 auto) — The “auto” keyword tells the tool to use the default search parameters for the range value.

In addition, these parameters are not compatible with explicit optimization settings.

An optional argument that specifies the diffusion length of the vertical corner diffraction kernels in P1 units. The default value is 1.

An optional argument that specifies lower and upper bounds of search intervals for vterm arguments.

topomf_tsurface value

Use this option to specify the value of the tsurface parameter in the topo model. Possible values are “active” or “field”.

topomf_transit value1 value2 …

Use this option to specify the range of optical models used for the transit parameter in the topo model.

topomf_transit_dose value1 value2 …

Use this option to specify values for the transit_dose parameter in the topo model.

topomf_density_exposure value1 value2 …

Use this option to specify values for the density exposure parameter in the topo model.

For the topo modelform, a single density exposure case is assumed. However, you can specify multiple densities by passing the topomf_density_exposure parameter with multiple values.

Examples

In modelflow_v2, a topo model form requires a dummy topo model to be set with the -mlstring argument. You can use the following “dummy” topo model:

set mlstr {
.....
topo_model {
  transit 0 1
  from 0
  to 1 
  property stack_weight
  transit_dose 1 1 1
}
.....
}
modelflow_v2 -mlstring $mlstr \

When modelflow_v2 runs with the modelform argument, the dummy topo model is replaced with the generated one. A corresponding set of measurement gauges is introduced into the calibration process.

An example topo modelform usage in modelflow_v2 stage1:

modelflow_v2 \
       .....
        -mlstring $mlstr \
         .....
stage1 dense \
        threshold optimize \
        search frontzoom \
        explore $explore_s1 \
        topomf topo2ST \
        topomf_ttermnum 3 \
        topomf_tsigma 0.7 1.3 \
        topomf_squad_mult -0.3 0.3 \
stage2 \
       ...

The code instructs modelflow_v2 to use an implicit topo model of the form topo2ST (sterms and tterms) with additional topomf_* parameters. This is equivalent to explicitly setting the following topo model:

transit 0 1
from 0
to 1
property stack_weight
transit_dose 1 1
sterm 0 0.000000e+00
sterm 1 0.000000e+00
sterm 2 0.000000e+00
sterm 3 0.000000e+00
sterm 4 0.000000e+00
sterm 5 0.000000e+00
sterm 6 0.000000e+00
sterm 7 0.000000e+00
tsurface active
tterm 0 0.000000e+00
tterm 1 0.000000e+00
tterm 2 0.000000e+00
