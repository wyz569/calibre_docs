How to Read the modelflow Version 1 Log File

The primary output from VT5 Center is a text log file. It contains information about each stage of the optimization run, plus all the information you need to recreate the optimal model.

Separator comments in the log file, which are considered part of the format, simplify parsing of the log file for information extraction and analysis.

You use this log file for data mining and debugging of VT5 Center runs.

Overall Structure of a modelflow Version 1 Log File

The log file is divided into records, which are encapsulated by BEGIN and END separator strings. For example, the following encapsulates the input setup file echoed in the transcript:

-----  BEGIN SETUP FILE 
<data>
-----  END SETUP FILE

The stage 1, stage 2, and stage 3 log file records contain detailed information about the fitting statistics during those stages. In each stage record, recorded statistics are relative to the filtered spreadsheet file used in that stage (if used), not the global spreadsheet. The following incomplete list shows some of BEGIN record separators:

-----  BEGIN Model Flow
-----  BEGIN SETUP FILE 
-----  BEGIN INPUT OPTICAL MODEL 1 
-----  BEGIN POST STAGE 0 SPREADSHEET
-----  BEGIN STAGE 2
-----  BEGIN STAGE 3
-----  BEGIN STAGE 3 SAMDATA
-----  BEGIN VT5 FINAL MODEL

Formatting for VT5 Center GUI Plotting

When you load a log file into VT5 Center, the tool uses information in two sections of the file for plotting in the Analyze Model window. These are:

  • POST STAGE 0 SPREADSHEET — contains the global spreadsheet after stage 0 weights are applied.

  • STAGE 3 SAMDATA — contains the sample data for the final models at the end of stage 3 (not present if stage 3 is skipped).

These two sections are related to each other in the following way: If you supply the spreadsheet listed in the POST STAGE 0 SPREADSHEET section as input to samlinev2, then the samlinev2 output data file is the same as STAGE 3 SAMDATA. The number of lines in those two sections are also equal. This has at least two consequences:

  • Analysis plotting is not available if you skip stage 3.

  • Any text file formatted with these two sections can be plotted in the Analyze Model window. This opens up the possibility for users to write scripts which generate pseudo VT5 Center log files, which can be plotted for analysis.
