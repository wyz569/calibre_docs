Format of the SRAF Printing Table Data Output

The SRAF printing table data output is used for SRAF printing model simulation or calibration.

You generate the table by specifying the srafprinttable option in modelflow_v2 in the modelflow_v2: Output Arguments.

The table is generated in comma-separated value (csv) format, and can be loaded in most popular spreadsheet programs for further analysis. It consists of a headline with abbreviated names of the quantities involved, and multiple data lines equal to the total number of gauges+1.

Only SRAF gauge sets in the input super gauge file are shown in the table. For each data line, the SRAF printing prediction error counts and rates are simulated by the current model, but with a varying threshold on each line.

The list of records and abbreviations used are the following:

  • Threshold — Threshold for the CTR model or CM1 resist model

  • CI — Contiguous index varying between 1 and Ng where Ng is the total number of gauges

  • GI — Original gauge index within a given input gauge set

  • GT — SRAF gauge GT: or LT: type (printed gauge or non-printed gauge)

  • All_PEC — prediction error count for all SRAF gauge sets

  • All_FNC — false negative error count for all SRAF gauge sets

  • All_FPC — false positive error count for all SRAF gauge sets

  • All_PER — prediction error rate for all SRAF gauge sets

  • All_FNR — false negative error rate for all SRAF gauge sets

  • All_FPR — false positive error rate for all SRAF gauge sets

  • GSi_PEC — prediction error count for ith gauge set

  • GSi_FNC — false negative error count for ith gauge set

  • GSi_FPC — false positive error count for ith gauge set

  • GSi_PER — prediction error rate for ith gauge set

  • GSi_FNR — false negative error rate for ith gauge set

  • GSi_FPR — false positive error rate for ith gauge set
