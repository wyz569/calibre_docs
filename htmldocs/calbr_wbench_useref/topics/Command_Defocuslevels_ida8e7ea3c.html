defocuslevels

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Defines the defocus information.

Usage

defocuslevels {num start step} | {planesd1 d2 … dn [weights w1 w2 … wn]}

Arguments

num

A required argument specifying the number of planes.

start

A required argument specifying the starting optical plane in microns.

step

A required argument specifying the delta in microns between planes. Values should be between 0.002 and 0.5 microns, inclusive.

d1 d2 … dn

A required argument set specifying the defocus planes in microns to be used. Each plane is uniformly averaged together unless the weights argument is also specified.

weights w1 w2 … wn

A optional argument set specifying the weight for the corresponding dn plane when averaging is performed.

Description

A required Z-axis parameter specifying the defocus information.

For a single defocus level, use the num start step arguments. For multiple defocus levels, use the planes d1 d2 … dn and weights w1 w2 … wn arguments.

Note:

For a complete explanation of single defocus levels, refer to “Step 4 - Entering Film Stack Information”.

For a single defocus level, the coordinate system of start depends on the model, both for origin and positive/negative direction.

  • Aerial imagery — The origin is the best focus in air. Positive values represent planes above the best focal plane in air (closer to the lens). Negative values represent planes below the best focal plane in air (farther from the lens).

  • Thin-film imagery — The origin is the interface between the ambient and first film. Positive values represent planes below the film interface (farther from the lens). Negative values represent planes above the film interface (closer to the lens).
