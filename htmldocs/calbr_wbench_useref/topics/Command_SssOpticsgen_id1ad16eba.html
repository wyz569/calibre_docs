sss_opticsgen

Test Pattern Commands

Generates optical models for defocus values in a super spreadsheet file.

Usage

sss_opticsgen {–obj object | –sss file} {–str string | –opticalfile file}
[–str2 string | –opticalfile2 file]
[–str3 string | –opticalfile3 file]
–opticaldir modelpath
[–remotefile file | –remotestr string] [–out file]
[–modelname name] [–nolookuptables]

Arguments

–obj object

A required argument specifying an input sss Tcl object. Cannot be set if –sss is specified.

–sss file

A required argument specifying an input sss file name. Cannot be set if –obj is specified.

–str string

A required argument specifying an input string or Tcl variable representing the first optical model. Cannot be set if –opticalfile is specified.

–opticalfile file

An argument specifying the input first optical model file. Cannot be set if –str is specified.

–str2 string

An optional argument specifying an input string or Tcl variable representing the second optical model. Cannot be set if –opticalfile2 is specified.

–opticalfile2 file

An optional argument specifying a second optical model file. Cannot be set if –str2 is specified.

–str3 string

An optional argument specifying an input string or Tcl variable representing third optical model. Cannot be set if –opticalfile3 is specified.

–opticalfile3 file

An optional argument specifying a third optical model file. Cannot be set if –str3 is specified.

–opticaldir modelpath

A required argument specifying the output directory path to save optical models.

–remotefile file

An optional argument specifying an input remote configuration file. Cannot be used if the option –remotestr is set.

–remotestr string

An optional argument specifying an input remote configuration string. Cannot be used if the option –remotefile is set. The format of the remote configuration string is the same as that of the remote file.

Note:

If -remotefile or -remotestr is present, then parallel execution on the machines specified in the remote string will be used to create optical models. This option does not require additional licenses.

–out file

An optional argument specifying an output file where to write the new or updated sss object.

–modelname name

An optional argument specifying the input name of the subdirectories associated with different defocus values, namenum and secondOpticalnamenum or thirdOpticalDefocusnum, where the suffix num stands for the defocus value in microns in the format X.XXX or –X.XXX as described following (Default = defocus).

–nolookuptables

An optional argument that when specified instructs the tool to omit the creation of binary files with lookup tables for SOCS kernel-mask convolutions. In this case, the optical model involves just SOCS kernels (*.fsk files) and SOCS weights (*.omf files) and takes up significantly less disk space.

Description

Use the sss_opticsgen command to generate first (and optionally, second and third optical models) for defocus values specified in the input sss object or file.

If –obj is specified, this command resets the opticalDirectory fields in the input sss object, otherwise it returns a handle to the new sss tcl object with the opticalDirectory fields filled.

Optionally, sss_opticsgen writes the new or updated sss object to the specified file.

First optical models are saved to the specified directory under the name defocusnum where the suffix num stands for the defocus value in microns using format X.XXX or –X.XXX, for example, defocus0.000 or defocus-0.160. Similarly, second and third optical models, if present are saved to the specified directories under the names secondOpticalDefocusnum or thirdOpticalDefocusnum. The option –modelname allows you to specify the name of the subdirectories associated with different defocus values.

Examples

# this command loads sss file, generates optical models and saves these to 
# disk, creates sss object with opticalDirectory specified, dumps this 
# object to file, and returns the object handle to tcl
set obj [sss_opticsgen –sgf in.sss –opticalfile optical.txt 
	–opticaldir . –out out.sss]
# same as above but the models are generated using distributed processing
# on several machines specified in file configure.txt
set obj [sss_opticsgen –sgf  in.sss –opticalfile optical.txt 
	–opticaldir . –out out.sss –remotefile configure.txt]
