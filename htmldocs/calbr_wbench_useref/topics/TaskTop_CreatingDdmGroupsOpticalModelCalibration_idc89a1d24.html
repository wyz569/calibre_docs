Creating DDM Groups for Optical Model Calibration

Optical model calibration using modelflow_v2 also supports DDM groups, which allows fine calibration of a DDM model stackslope parameter. Because a single DDM model might not be effective for all layers (main features versus SRAF and fill layers might have different DDM characteristics) you can create DDM groups from multiple DDM models. Calibre WORKbench can then be used to optimize the DDM groups in stage1 of modelflow_v2.

Restrictions and Limitations

  • DDM groups are designed to be used as part of a mask model, and are intended for calibration purposes only. They should not be used inside litho models for simulations.

  • DDM groups are intended to help calibrate 1D features.

Prerequisites

  • At least one optical model

  • At least two DDM models with different values for stackslope

  • The corresponding super gauge data file containing measurements for the optical model combined with the DDM models

Procedure

  1. Create a DDM group file using the DDM Group Format. One group with two DDM models is the minimum requirement.
  2. Add the DDM group to a mask model file.
  3. Create a model-layer file that includes the DDM group and mask model file, associating the DDM group with the appropriate layers in the model-layer file.
  4. Modify a modelflow_v2 script as follows:
    1. Include the model-layer file using the -mlfile or -mlstring argument.
    2. Add the interpolated_ddm filename argument to output the optimized DDM library.
    3. In stage1, optimize the ddmgroupparamM parameter(s) with a recommended value of “auto auto 20”, which uses the top and bottom values in the DDM group as the start and end ranges. You optimize one ddmgroupparam per DDM group file; if you only have one DDM group file, optimize only ddmgroupparam0.
  5. Run modelflow_v2 with only stage1 active.

Results

After modelflow_v2 finishes, you will have an optimized DDM file. Use this DDM library file as input to other tools, such as Calibre nmOPC or Calibre OPCverify.

You can also examine the log file for the section “BEGIN FINAL DDM_GROUPM” where M is one of the DDM group numbers. Inside the DDM group definition, the ddmgroupparam parameter is replaced with the optimized value.

ddm_group0 ddm_grp {
ddmgroupparam 84.1144
paramtype slope
ddm 83 83.ddm
ddm 90 90.ddm
}

Examples

The following example shows a single DDM group used with three mask layers in a model-layer file.

modelpath /models
ddm_group0 ddm_grp {
ddmgroupparam 87
paramtype slope
ddm 83 83.ddm
ddm 90 90.ddm
}
mask_model {
version 2
background 1 0
mask_layer 0 TRANS -1.2503 -0.0200 DDM ddm_grp
mask_layer 1 TRANS -1.2503 -0.0200 DDM ddm_grp
mask_layer 2 TRANS -1.2503 -0.0200 DDM ddm_grp
}
