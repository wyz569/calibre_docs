Layout Configuration File Format

Calibre Pattern Generator File Formats

A text format for generating test pattern layouts using the Calibre Pattern Generator (CPG) tool.

The common file extension for layout configuration files is *.xml.

Format

Input for:

  • Calibre Layout Generator (LG)

Generated by:

  • Calibre Layout Generator (LG)

The following criteria apply:

  • The syntax must conform to the XML specification maintained here:

    https://www.w3.org/TR/REC-xml/

  • The first line must be the following XML declaration:

    <?xml version=“1.0” encoding=“UTF-8”?>

  • The file is case-sensitive. The parameter values themselves may not be treated as such when used within the Calibre Pattern Generator depending on the value and the command that references the value.

  • Elements begin with a start-tag of the form <Name> and end with an end-tag of the form </Name>. Each “Name” is pre-defined as part of the accepted syntax. User-defined elements are unsupported.

  • The root element is <Test_layout>.

  • The main sections are embedded in the following elements:

    • <Version>

    • <Pattern_libraries>

    • <Output_layout>

    • <Layout_Precision>

    • <UserUnits>

    • <LabelsFont>

    • <Gauges_options>

    • <SEM_Alignment_Markers>

    • <Layout_grid>

    • <Cell_extent>

    • <Layers>

    • <DRC_Check_configurations>

    • <Mapping>

    • <Pattern_configurations>

  • Attribute values must be in quotes. Example:

    <element attribute=”value”>

  • Attribute names are pre-defined as part of the accepted syntax. User-defined attribute names are unsupported.

The following is the basic schema:

<?xml version="1.0" encoding="UTF-8"?>
	<Test_layout>
		<Version>2.0</Version>
		<Pattern_libraries>
			<pattern_library>patt_library_name</pattern_library>
			...
		</Pattern_libraries>
		<Output_layout>output_layout_name</Output_layout>
		<Layout_Precision>precision_value</Layout_Precision>
		<UserUnits>units_value</UserUnits>
		<LabelsFont>font_size</LabelsFont>
		<Gauges_options>
			<Output_gauge>gauge_file</Output_gauge>
			<layer>gauge_layer</layer>
		</Gauges_options>
		<SEM_Alignment_Markers>
			<library_path>sem_align_library</library_path>
			<fineAlignment_library>finealign_library</fineAlignment_library>
			<autofocus_libraryPath>af_library</autofocus_libraryPath>
			<layer>align_layer</layer>
			<selection_mode>sel_mode</selection_mode>
			<placement_mode>place_mode</placement_mode>
			<auto_focus_marker>
				<enable> true|false </enable>
				<configuration> config </configuration>
			</auto_focus_marker>
		</SEM_Alignment_Markers>
		<Layout_grid>
			<rows>num_rows</rows>
			<cols>num_cols</cols>
			<grid_size>gr_size</grid_size>
			<grid_space>gr_space</grid_space>
			<label_space_y>label_sp_size</label_space_y>
			<clipping_allowed>true|false</clipping_allowed>
			<cells_connected>false|true</cells_connected>
			<connection_width>conn_width</connection_width>
			<connection_layer>conn_layer</connection_layer>
		</Layout_grid>		
		<Cell_extent>
			<draw_extent>false|true</draw_extent>
			<extent_layer>ext_layer</extent_layer>
		</Cell_extent>
		<Layers>
			<pattern_label_layer>pat_label_layer</pattern_label_layer>
			<layout_label_layer>lo_label_layer</layout_label_layer>
		</Layers>
		<DRC_Check_configurations>
			<executeDrcChecks>false|true</executeDrcChecks>
			<SVRF_path>DRC_filename</SVRF_path>
			<result_action>Highlight_Results</result_action>
		</DRC_Check_configurations>
		<Mapping>
			<Layer number="input_map_layer">output_map_layer</Layer>
			...
		</Mapping>
		<Pattern_configurations>
			<configuration>
				<Row>cell_row</Row>
				<Col>cell_col</Col>
				<shift_x>x_shift</shift_x>
				<shift_y>y_shift</shift_y>
				<AutoFocus>
					<Configuration>
						<x>x_val</x>
						<y>y_val</y>
						<marker_config marker="marker_name">
							<cd>cd_value</cd>
						</marker_config>
						<shift_y>af_y_shift</shift_y>
					</Configuration>
				</AutoFocus>
				<FineAlignmentMarker>
					<Configuration>
						<x>x_val</x>
						<y>y_val</y>
						<marker_config marker="marker_name">
							<cd>cd_value</cd>
						</marker_config>
					</Configuration>
				</FineAlignmentMarker>
				<pattern_configure pattern="pattern_name">
					<patternOrientation>0|90</patternOrientation>
					<Centred>true|false</Centred>
					<ToneInverted>false|true</ToneInverted>
					<Gauges_Configuration>
						<Gauge name="gauge_name">
							<Property name="prop_name">prop_val</Property>
							...
						</Gauge>
						
					</Gauges_Configuration>
				</pattern_configure>
			</configuration>
			...
		</Pattern_configuration>
	</Test_layout>

Parameters

<Test_layout>

A required element enclosing the layout configuration settings.

<Version>

A required element specifying the layout configuration file version. The default version is 2.0.

<Pattern_libraries>

A required element specifying the paths to all pattern libraries used in the layout. Each pattern library filepath must be enclosed with a <pattern_library> element. Multiple <pattern_library> elements may be specified.

<Output_layout>

A required element specifying the output layout file name to write in OASIS format.

<Layout_Precision>

A required element specifying the output layout precision value. The default precision value is 10000.

<UserUnits>

A required element specifying the dimension units to apply to parameters. The only supported value is “nm” (nanometers).

<LabelsFont>

A required element specifying the label font size in user units. The font size applies to row, column, and pattern labels. If no font size is specified, the tool calculates the pattern labels font size based on the count of characters to fit the grid space.

<Gauges_options>

An optional element set specifying output gauge file properties. Specify the following child elements in the <Gauges_options> element:

  • <Output_gauge>

    An optional child element specifying the output gauge file name. The default is “test.gg”.

  • <layer>

    An optional child element specifying the gauge layer in the output layout file. If not specified, no gauge layer is generated.

<SEM_Alignment_Markers>

An optional element set specifying properties for several types of alignment markers. Specify the following child elements in the <SEM_Alignment_Markers> element:

  • <library_path>

    An optional child element specifying an input template library of SEM alignment markers. The tool automatically selects and places marker shapes from the specified library.

  • <fineAlignment_libraryPath>

    An optional child element specifying an input template library of fine alignment markers. The tool automatically selects and places marker shapes from the specified library.

  • <autofocus_libraryPath>

    An optional child element specifying an input template library of autofocus markers. The tool automatically selects and places marker shapes from the specified library.

  • <layer>

    An optional child element specifying the alignment marker layer in the output layout file. If not specified, no alignment marker layer is generated.

  • <selection_mode>

    An optional child element specifying the method of marker selection from marker libraries.

    • 0 — Sequential selection method.

    • 1 — Random selection method.

  • <placement_mode>

    An optional child element specifying the method of alignment marker placement between patterns.

    • 0 — Places alignment markers at the corners of adjacent patterns.



    • 1 — Places markers above and below adjacent patterns.



  • <auto_focus_marker>

    An optional child element specifying the autofocus marker properties. Specify the following child elements in the <auto_focus_marker> element:

    • <enable>

      An optional child element that turns on autofocus marker placement. “true” turns on autofocus marker placement; “false” turns off autofocus marker placement.

    • <configuration>

      An optional child element specifying the placement and size of the autofocus marker. Specify the following child elements in the <configuration> element:

      • <x> and <y>

        Optional child elements specifying the location of the autofocus marker relative to the origin of the cell extent.

      • <marker_config>

        An optional child element specifying the autofocus marker properties. Specify the marker name with the “marker” attribute. Specify the marker size in a <cd> child element.

      The following <configuration> example places a 50 nm autofocus marker at (0,1000):

      <configuration>
	<x>0</x>
	<y>1000</y>
	<marker_config marker="X">
		<cd>50.0</cd>
	</marker_config>
</configuration>
<Layout_grid>

A required element set defining the structure of the output test layout. Specify the following child elements in the <Layout_grid> element:

  • <rows> and <cols>

    A required set of child elements specifying the number of rows and columns in the output test layout.

  • <grid_size>

    A required child element specifying the maximum width and height of the grid cell for placing a pattern in the layout. To specify a grid size with different width and height values, use the <grid_size_width> and <grid_size_height> child elements. CPG writes the <grid_size_width> and <grid_size_height> child elements. Placing a pattern larger than the grid size generates an error message.

  • <grid_space>

    A required child element specifying the horizontal and vertical distance between patterns placed in the layout. To specify placing distances with different horizontal and vertical values, use the <grid_space_width> and <grid_space_height> child elements. CPG writes the <grid_space_width> and <grid_space_height> child elements.

  • <label_space_y>

    A required child element specifying the vertical distance between the pattern label and the lower left corner of the cell extent.

  • <clipping_allowed>

    A required child element that enables clipping patterns larger than the grid cell. “true” enables pattern clipping; “false” disables pattern clipping.

  • <cells_connected>

    A required child element that enables the connecting patterns with shorting bars. “true” enables drawing cell connections; “false” disables drawing cell connections. When enabled, define the <connection_width> and <connection_layer> child element values.

  • <connection_width>

    A required child element specifying the width in user units of shorting bars between patterns.

  • <connection_layer>

    A required child element specifying the connection layer in the output layout file.

<Cell_extent>

A required element set defining cell extent properties. Specify the following child elements in the <Cell_extent> element:

  • <draw_extent>

    A required child element that enables the drawing of the cell extent layer. “true” or “1” enables cell extent layer drawing; “false” or “0” disables cell extent layer drawing. The default is “false”.

  • <extent_layer>

    A required child element specifying the cell extent layer in the output layout file. The default layer is 0.

<Layers>

A required element set defining miscellaneous layers. Specify the following child elements in the <Layers> element:

  • <pattern_label_layer>

    A required child element specifying the pattern label layer in the output layout file. The default layer is 8.

  • <layout_label_layer>

    A required child element specifying the layout label layer in the output layout file. The default layer is 0.

<DRC_Check_configurations>

A required element set defining design rule check (DRC) options. Specify the following child elements in the <DRC_Check_configurations> element:

  • <executeDrcChecks>

    A required child element that enables running a Calibre nmDRC SVRF rule file on the output layout. “true” enables a Calibre nmDRC run; “false” disables a Calibre nmDRC run. The default is “false”.

  • <SVRF_path>

    A required child element specifying the filepath of a Calibre nmDRC SVRF rule file to run on the output layout.

  • <result_action>

    A required child element specifying results output of the DRC run. The element supports the following value:

    • Highlight_Results — Highlights DRC violations in the layout.

<Mapping>

A required element set specifying the mapping of any layer to another layer. Mapping is typically used to map two or more patterns to the same layer. By default, the Pattern Creator creates each pattern on a unique layer. To place the patterns on the same layer in the Layout Generator, set the <Mapping> element to map each input pattern to one output layer. Specify the following child elements in the <Mapping> element:

  • <Layer number= “input_map_layer”>

    A required child element specifying the mapping of an input pattern layer to an output layout layer. Specify one <Layer> element for each input pattern. By default, each input pattern is mapped to output to the same layer number as the input. The input_map_layer must be enclosed in quotation marks. To map multiple patterns to the same output layer, specify multiple <Layer> elements that map to the same output_map_layer. By default, LG sets output_map_layer to equal input_map_layer.

<Pattern_configurations>

A required element set specifying all cell configurations in the layout. Specify the following child elements in the <Pattern_configurations> element:

  • <configuration>

    A required child element defining all properties of a layout cell. Specify one <configuration> child element for each cell in the layout. Specify the following child elements in the <configuration> element:

    • <Row> and <Col>

      Required child elements specifying the row and column of the cell in the output layout. Each <Row> and <Col> pair must be unique. Duplicating a <Row> and <Col> pair generates an error message.

    • <shift_x> and <shift_y>

      Required child elements specifying the horizontal and vertical shifts in user units of the cell in the output layout. The default values are 0.

    • <Autofocus> and <FineAlignmentMarker>

      Required child elements specifying the properties of the autofocus and fine alignment markers of the cell in the output layout. If the markers are not used in the cell, specify an empty child element. If markers are used in the cell, specify a <configuration> element and the marker properties. To specify marker properties, define the following child elements inside the <configuration> element:

      • <x> and <y>

        Optional child elements specifying the marker location in the cell in user units.

      • <marker_config marker=“marker_name”>

        Optional child elements specifying the marker id. The marker name must be enclosed in quotation marks. Specify the following child element inside the <marker_config> child element:

        • <cd>

          An optional child element specifying the height and width in user units of the marker.

    • <pattern_configure pattern=“pattern_name”>

      A required child element set specifying pattern and gauge placement properties in the cell in the output layout. Specify the following child elements inside the <pattern_configure> elements:

      • <patternOrientation>

        A required child element specifying a rotation angle in degrees of the pattern in the cell. Supported values are 0 and 90. The default is 0 (no rotation).

      • <Centred>

        A required child element enabling automatic gauge placement at the center of the pattern in the cell. “true” enables centered gauge placement; “false” disables centered gauge placement. The default is “true”.

      • <ToneInverted>

        A required child element enabling inversion of the pattern tone. “true” enables tone inversion; “false” disables tone inversion. The default is “false”.

      • <Gauges_Configuration>

        A required child element set specifying gauge properties for pattern placed in the cell. To define gauge file properties, specify a <Gauge> child element inside the <Gauges_Configuration> element using the following schema: 

        <Gauge name="gauge_name">

        The gauge name must be enclosed in quotation marks.

        Next, specify the <Property> attributes inside the <Gauge> element using the following schema:

        <Property name="property_name">property_value</Property>

        The property names represent columns of a gauge file. The following property_name values are supported:

        95%CI

        MEtch

        SAerial

        Bias

        MEtchConstraint

        SEtch

        Col

        Meas

        Sem

        Comment

        MeasConstraint

        Simul

        Drawn

        On

        Weight

        GID

        Other

        defocus

        Group

        PCWeight

        dose

        GroupName

        Prec

        vebBiasing
