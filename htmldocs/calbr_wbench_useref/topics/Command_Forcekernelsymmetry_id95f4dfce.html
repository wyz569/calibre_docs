forceKernelSymmetry

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Sets the kernel count to preserve symmetries in the TCC matrix.

Usage

forceKernelSymmetry { 0 | -1 | 1 }

Arguments

0

An optional argument that keeps the original kernel count. This is the default for optical version < 9.

-1

An optional argument that decrements the kernel count until it finds a symmetric setting. This results in an output optical model with reduced kernel counts, but TCC symmetries are preserved. If decrementing the kernel count does not preserve the original TCC symmetries, then the original kernel count is kept without modification (similar to the 0 setting).

1

An optional argument that increments the kernel count until it finds a symmetric setting. This results in an output optical model with increased kernel counts, but TCC symmetries are preserved. If an incremented kernel count cannot be found within 10 kernels that preserves the original TCC symmetries, then the original kernel count is kept without modification (similar to the 0 setting). (default for optical version 9 and greater)

Description

An optional parameter that examines the TCC matrix to discover what symmetries exist for the TCC, and sets the kernel count such that these symmetries are preserved.

This command does not guarantee a symmetric model, as the original TCC may not be symmetric. Internally, the command calls kernsym on the decomposed kernels to check for current model symmetry.
