SRAF Calibration Key Concepts

The following definitions may help you to understand how to perform the tasks in this appendix.

Differences between process modeling and SRAF modeling

Modeling SRAFs is similar to modeling printing of regular features. However, the optimum optical setting for SRAFs may be different than the ones used for OPC features. This is because SRAFs resolve at different heights in the resist profile and the binary nature of SRAF gauges.

Objective of SRAF modeling

During OPC, a design is never supposed to have SRAFs that print.

However, a test pattern may have SRAFs that print in order to calibrate the model.

An SRAF printing model is calibrated by using SRAFs on a test pattern. Some SRAFs are made to intentionally print; others are normal non-printing SRAFs. This is in order to properly capture the boundary between printing and non-printing. The model can make two types of mistakes:

  • False Negative — The SRAF actually prints on the wafer, but the model did not indicate this.

  • False Positive — The SRAF does not print on the wafer, but the model indicates that it does.

In general, a model cannot make both error types zero. False negative errors are considered more serious in general, because SRAF printing can lead to etching failures. The model should make the number of false negatives 0 or very small, and maintain a small false positive rate (< 10%).

The SRAF model can be as simple as a CTR model where only optical parameters and a threshold are calibrated. It could also be a more complex CM1 model that may help improve the accuracy. A CTR model can be calibrated with only SRAF gauges. A CM1 model generally requires both CD main feature gauges and SRAF gauges. It is good practice to calibrate a CTR model first, because it is a simpler model, and may provide an acceptable error rate. It also serves as a reference to see how much a CM1 model can improve accuracy.
