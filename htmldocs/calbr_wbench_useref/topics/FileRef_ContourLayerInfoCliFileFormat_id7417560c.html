Contour Layer Info (CLI) File Format

Input for: Contour calibration

The Contour Layer Info (CLI) file format describes a clip file contained in a design file.

Format

# this is a comment
version version_number
contourSetCount number_of_process_window_conditions
[simulationPixelSize pixelsize]
[contourLog log_sigma_multiplier log_max_number]
[contourAverage {off | on | {on bin_size_in_um} } ]
[contourSiteExtension length_in_um]
[contourComprehensiveDataTableFile filename]
[contourCostMethod { vector | dense | vector single}]
[contourClearance {off | {on size_um} } ]
contourSet contour_set_number_x ‘{‘
	contourlayer contour_layer_number_x
	[mresist|metch|sresist|setch][do_averaging]...
	[contourfragment length_in_um | 
	 contouruniformstep length_in_um]
	cliplayer clip_layer_number_x
	[contourbias bias_value_x]
	dose dose_value
	defocus defocus_value
	[weight weight_value]
	[weightlayer weightlayer_layer_number_x {layer_weight_value | auto}]
	[{contourshift {x_shift | auto} {y_shift | auto}} |
		{[contourshiftresist {x_shift | auto} {y_shift | auto}]
		[contourshiftetch {x_shift | auto} {y_shift | auto}]}
	]
	[outmeasuredlayer layer_number [mresist|metch]]
	[outstartlayer out_layer_number [1|2|3]]
	[outputs2m {on | off}]
	[outweightlayer layer_number weighting_unit_size [mresist|metch]]
	[outweightrange min max 
		[stdevthreshold value step | linear][mresist|metch]]
	[s2merrorweight value]
	[averagingmaxdeviation value_in_um]
‘}’ ...

Parameters

version version_number

A required parameter that specifies the version of the CLI file. This is currently always 1.

contourSetCount number_of_process_window_conditions

A required parameter that specifies the number of contour set definitions contained in the file. Each contour set is used for a different process window condition.

simulationPixelSize pixelsize

An optional parameter only used for dense (CM1) simulation. Specifies the pixelsize in um (default is 0.01).

contourLog log_sigma_multiplier log_max_number

An optional parameter that specifies the threshold for what errors are written to the log file; only errors that are worse than this sigma factor are output (default is 3). In addition, only log_max_number errors will be output (recommended value is 10000000).

contourAverage {off | {on [bin_size_in_nm]} }

An optional parameter that, when used with multiple contours that overlap the same test structure, averages the contours on an internally specified distance (“on” option) or for a user-specified distance (“on” option with an explicit bin_size_in_nm).

Contours used for this option must all be on the same layer.

  • When the CLI file is used with modelflow_v2, this parameter outputs smoothed versions of the input contours (the contour is drawn more smoothly).

  • This option is not meant for use with Calibre nmModelflow; while the mdf contour command uses CLI files, it uses a more robust averaging algorithm.

contourSiteExtension length_in_um

An optional parameter that sets an extension in microns that is added to the measured-to-simulated vector. Adding this extension allows for better correlation of IPSE parameters to gauges, because the IPSE measurements along the extended cutline are more similar to the corresponding gauges. The default value is 0 (off).

contourComprehensiveDataTableFile filename

An optional parameter that triggers the output of a comprehensive data table in comma-separated format (CSV) that can be opened in any spreadsheet program. The filename can contain relative or absolute paths; if only a name is given it is written to the current working directory. It follows the format described in “Contour Comprehensive Data Table File Format.”

This option is not supported in modelflow (Version 1).

contourCostMethod {vector | dense | vector single}

An optional parameter that specifies the method used in calculating the contour cost. Specifying dense or vector single will give you different erms results than the default due to the measurement technique:

  • vector (default) takes sites along each contour, and uses bidirectional measurements to calculate the cost.

  • dense uses a grid measurement system.

  • vector single is a version of vector that calculates costs in one direction (from the measured contour to the simulated contour).

    Note:

    The setting you specify for contourCostMethod has specific trade-offs and may give different results.

Table 1. contourCostMethod Options

Option

Notes

vector

Performs bidirectional vector calculations. Catches all errors, but may also report false errors if a SEM contour that corresponds to a simulation contour is off of a clip edge.

This is the default behavior.

vector single

Performs one-directional vector calculations. This is the fastest mode, but can miss errors when a simulated contour has no corresponding measured SEM contour.

dense

Uses a grid measurement system. Slowest mode.
contourClearance off | {on size_um}

An optional parameter that specifies a method to treat sites that are near clip edges. To use contourClearance, you must specify “contourCostMethod vector single”. The default behavior is off.

When enabling contourClearance, specify a value for size_um in microns. When size_um is negative, the tool excludes contour vertices and sites that are within size_um of the clip edge. When size_um is positive, the tool keeps all vertices and sites that are inside the clip edges and expands the clip edges by size_um to evaluate the difference between measurement and simulation contours.

contourSet contour_set_number_x ‘{’ ‘}’

A required parameter that specifies the start of a contour set definition. The contour_set_number_x parameter is a sequentially increasing number from 1 to the number of contour sets. Each contour set is enclosed in braces ({}) and contains the following sub-keywords:

  • contourlayer contour_layer_number_x [mresist | metch | sresist | setch] [do_averaging]

    A required parameter that specifies the layer number containing the contour layer wire or path used in this specific contour set. Multiple contour layers can be specified for the contour set.

    The following optional keywords identify the type of contour layer for the contourSet:

    • mresist — Contains a measurement resist layer. This maps to the “resist” keyword from versions previous to 2014.4.

    • metch — Contains a measurement etch layer. This maps to the “etch” keyword from versions previous to 2014.4

    • sresist — Contains a pre-simulated contour for the resist layer.

    • setch — Contains a pre-simulated contour for the etch layer.

    Both sresist and setch are required when the -align_only option is specified for the ccalFitness command.

    Note:

    Input VEB users must provide two contours: one for the resist contour and one for the etch contour.

    If the do_averaging keyword is specified, the Calibre ContourCal tool averages the contours on this layer and any measured contours that are generated from this layer. The do_averaging keyword requires the outmeasuredlayer parameter to be specified.

    If mresist or metch are specified with do_averaging, the outmeasuredlayer, outweightrange, and outweightlayer statements must also have a corresponding mresist or metch parameter specified.

  • contourfragment length_in_um

    An optional parameter that specifies the maximum possible edge length (in microns) in the measured contour for error calculation and in the resulting output. If this option is specified with a positive value, all measured contour edges with a length larger than the specified length are split until there are no edges in the measured contour that are larger than length_in_um.

    You would use this option to improve the accuracy of EPE calculation and to provide more precise output of the measured and simulated pairs of points. The default is -1, which instructs Calibre WORKbench to use an internal value for splitting long edges.

  • contouruniformstep length_in_um

    An optional parameter that works similarly to contourfragment in that it splits contour edges that are larger than length_in_um. However, contouruniformstep also filters out measured and simulated points located too close to each other (closer than length_in_um); those points are removed from the error cost calculation and do not have EPE values or text markers created for them.

    You would use this parameter to improve the uniform distribution of samples along contours, and may see some runtime improvement by doing so.

    Note:

    The contouruniformstep option is mutually exclusive with contourfragment.

  • cliplayer clip_layer_number_x

    A required parameter that specifies the layer number containing clipping rectangles around the simulation regions. These rectangles cover the calibration contours and restrict contour calibration only to the marked areas.

    This layer can be shared by different contourSet definitions.

  • contourbias bias_value

    An optional parameter that specifies a fixed bias value to apply to all contour cost calculations in order to adjust for a discrepancy between vendor contours and CD measurements.

  • dose dose_value

    A required parameter that specifies the dose of the corresponding contour_layer_number.

  • defocus defocus_value

    A required parameter that specifies the defocus of the corresponding contour_layer_number.

  • weight weight_value

    An optional parameter that specifies the weight of a contourSet. By default, it sets the weight of all contours within this contourSet definition.

weightlayer weightlayer_layer_number_x {layer_weight_value | auto}

An optional parameter that specifies a weighting layer. Each weight layer contains a clip that highlights different parts of the contour. It is a one weight per layer relation, so multiple weightlayer definitions inside a contourSet are acceptable. Each layer can be shared by different contourSet definitions (and possibly with different weight values). Polygons on this layer (and inside the clip region) determine the weight contour parts where they intersect.

  • Overlapping polygons are allowed.

  • The weight value can be negative. The final weight of a contour region is the sum of the contourSet’s weight and the weights of overlapping polygons from different weight layers. If the sum is negative, then the contour region will be ignored during cost calculation.

  • The “auto” keyword instructs ContourCal to instead derive the weight for a polygon from properties calculated by averaging multiple contours according to localized standard deviations of the contours.

contourshift {x_shift | auto} {y_shift | auto}

An optional parameter that specifies shifts of the contour by the specified amounts in um. Specifying “auto” causes Calibre WORKbench to attempt to shift the contour to the best alignment location.

The contourshift parameter shifts both the resist and etch contours by the same amount. You can use the contourshiftresist and contourshiftetch parameters to specify indepenent shift values for those contours instead of the contourshift parameter.

contourshiftresist {x_shift | auto} {y_shift | auto}

An optional parameter that specifies shifts of the resist measured contours only. This option can be specified with contourshiftetch, but not contourshift.

contourshiftetch {x_shift | auto} {y_shift | auto}

An optional parameter that specifies shifts for etch measured contours only. This option can be specified with contourshiftresist, but not contourshift.

Note:

If you only specify contourshiftresist or contourshiftetch, the parameter you do not specify is considered to have a shift value of zero.

outmeasuredlayer layer_number [mresist | metch]

An optional parameter that specifies the layer number for generated averaged contours, which is output with an “mprint” label. It is required if the do_averaging flag is specified with the contourlayer parameter.

If the optional mresist or metch keywords are specified, it designates that the output is for the contourlayer with the corresponding mresist or metch parameter. (These keywords are only used when separate contourlayer ... mresist and contourlayer ... metch statements are specified.)

outstartlayer out_layer_number [1 | 2 | 3]

An optional parameter that specifies the layer(s) where the output of the contour values will be written. Up to eight layers (simulation layer, weight layer, EPE layers for measurement to simulated and alignment to simulated layers, leader line guide layers for the EPE layers, and alignment layers) can be requested, starting with the specified layernumber. Output layers are written to the active layout file. If a layer written to already contained information, it will be lost; be sure to specify layers that will not conflict with your existing layers.

If calibration occurs for more than one stage, each stage applies the contour layer output, with the following stage adding 10 to layernumber. For example, outstartlayer 200 3 will return layers 200 to 207 for stage1, and 210 to 217 for stage2.

The outstartlayer parameter can have a verbosity level applied to results:

  • 1 — Outputs the simulation print layer.

  • 2 — Outputs a set of error value layers:

    • A weight value layer

    • Two pairs of EPE value layers with leader line guides:

      • measurement to simulated layer EPE (m2s_epe and m2s)

      • aligned measured layer to simulated layer EPE (a2s_epe and a2s)

        However, note that these alignment layers are only generated if the contourshift parameter is also specified, and either the X- or Y-shift is a nonzero value.

    Also includes the layers from setting 1.

  • 3 — Outputs alignment measurement results layers (oclip and aprint). Includes the layers from settings 1 and 2.

outputs2m {on | off}

An optional parameter that outputs layers showing simulation to measurement values to the design file. Three layers are output:

  • a layer with the error line

  • a text layer containing the weight

  • a text layer containing the EPE error value

This option is off by default. It is intended to display a bidirectional (measured to simulated versus simulated to measured) error cost calculation information for comparison purposes.

Note:

The outputs2m option is not compatible with the contourCostMethod dense option. Specifying “outputs2m on” results in Calibre ContourCal switching the contourCostMethod to vector, but only for the specific contour set containing an “outputs2m on” setting.

outweightlayer layer_number weighting_unit_size [mresist | metch]

An optional parameter that specifies a layer number to store weighting units. Weighting units are squares of the specified size in um, and are used for filtering the output.

If the optional mresist or metch keywords are specified, it designates that the output is for the contourlayer with the corresponding mresist or metch parameter. (These keywords are only used when separate contourlayer ... mresist and contourlayer ... metch statements are specified.)

outweightrange min max [stdevthreshold stdvalue step | linear] [mresist | metch]

An optional parameter that specifies the range of values given to weighting units based on their standard deviation. The largest standard deviation value gets the min value; the smallest standard deviation value gets the max value. All other weighting units have weights assigned between min and max.

Note:

The value of a weighting unit is added to the global weight value for a contour set during simulation. Because the global weight is 1 in most cases, setting min to -2 filters out bad weighting units from the total error cost.

Weights can be manually adjusted as properties on the weight unit. This allows you to individually filter out or add back bad weight units as needed.

If the optional stdevthreshold parameter is specified, it modifies the weighting algorithm using the specified stdvalue in microns:

  • step — Assigns the min value to all weighting cell units having a standard deviation > stdvalue, and max to all weighting cell units with a standard deviation < stdvalue.

  • linear — Assigns the min value to all weighting cell units with a large standard deviation > stdvalue, and uses the following formula for the weight:

    weight = min_weight + ((max_weight - min_weight) / (stdvalue - min)) * (stdvalue - std_dev))

If the optional mresist or metch keywords are specified, it designates that the output is for the contourlayer with the corresponding mresist or metch parameter. (These keywords are only used when separate contourlayer ... mresist and contourlayer ... metch statements are specified.)

s2merrorweight value

An optional parameter that changes the weighting factor of simulated to measured error weights and measured to simulated error weights as a function of the total error. When this parameter is set as a value between 0 and 1, the total error formula used is:

total_error = ((1-value) * m2s_error) + (value * s2m_error)

In other words, a value of 0 considers only measured to simulated errors, and a value of 1 considers only simulated to measured errors.

If this parameter is not specified, the default is to consider both error types equally:

total_error = 0.5 * (m2s_error + s2m_error)

averagingmaxdeviation value

An optional parameter used with multiple contours. It prunes any contours that have a maximum error that is outside the standard deviation in the specified value when compared to the average of all contours. If this option is not specified, no averaging is performed.

Examples

Example 1 (Standard Mode and modelflow)

Given the following layers specified in the file standard.oas:

#
MASK - 0 (green)
CLIP - 237 (purple)
CONTOUR - 200, 201, 202 (blue, yellow, red)
Figure 1. Standard Mode Example

Create the CLI file standard.cli:

#
version 1
contourSetCount 1
contourSet 1 {
contourlayer 200
cliplayer 237
dose 1.0
defocus 0.0
weight 1.0
}

The example code treats contourSet 1 as “any contours on layer 200 enclosed by polygons on layer 237” being used for contour calibration, for the nominal (1.0) dose and nominal (0.0) defocus. The other two contour layers (201 and 202) are ignored.

This example is run with the command: 

modelflow -f setup.in -m standard.oas -cli standard.cli -contourweight .33 (other modelflow options)...

Example 2 (Weighting Layers and modelflow_v2)

Given the following layers in the file contour_test.oas:

#
MASK - 0 
CONTOUR - 202 
CLIP - 10 
WEIGHTS - various layers (11,12,13,14,15)

Create the following contour_test.cli file:

#
version 1
contourSetCount 1
contourLog 1 100000
contourSet 1 {
	contourlayer 200
	cliplayer 10
	dose 1.0
	defocus 0.0
	weight 1.0
	weightlayer 11 -1.0
	weightlayer 12 -1.0
	weightlayer 13 -1.0
	weightlayer 14 -1.0
	weightlayer 15 1.0
}

The example code treats contourSet 1 as “any contour sections on layer 200 with clipping polygons on layer 10 AND layer 15” being used for contour calibration, for the nominal (1.0) dose and nominal (0.0) defocus. Any clipping polygons on layer 10 and layers 11 through 14 are ignored (the 1.0 weight and -1.0 weightlayer settings on layers 11 through 14 cancel each other out).

This example is run with the command:

modelflow_v2 -f setup.in -m contour_test.oas -cli contour_test.cli 
-ccalweight .33 ...

Example 3 (Contour Averaging)

Given the following layers in the file contour_average.oas, which assumes that layer 202 has several contours flattened onto the same layer:

#
MASK - 0 (green)
CONTOUR - 202 (red)
CLIP - 10 (blue) 
#

Create the following contour_test.cli file: 

#
version 1
contourSetCount 1
contourLog 1 10000
contourAverage on
contourSet 1 {
	contourlayer 202
	cliplayer 10
	dose 1.0
	defocus 0.0
	weight 1.0
}

The example code takes the average of all the contours on the flattened layer and creates its own custom contour based on the average of the multiple contours. This generated contour is used for the calculations.

This example is run with the command:

modelflow_v2 -f setup.in -m contour_test.oas -cli contour_test.cli 
-ccalweight .33 ...
