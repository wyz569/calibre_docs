Generating a Test Pattern Using Batch Commands

As an alternative to using the Test Pattern tool in Calibre WORKbench, you can create and use your own test pattern generation file. Calibre WORKbench supports the use of batch commands that take formatted files as input.

Prerequisites

  • Familiarize yourself with the Test Pattern File (TPF) formats:

    • “FEATURE Keyword”

    • “PATTERN Keyword”

Procedure

  1. Create a text file that follows the test pattern file format syntax and guidelines.

    • A TPF file consists of a series text lines of keywords (PATTERN or FEATURE) and associated subkeywords.

      • Each keyword defines the scope for any subsequent text lines until another keyword is encountered.

      • A TPF file contains one instance of a PATTERN keyword to define the design file, and one or more FEATURE keywords, each of which defines a single structure.

      • Each FEATURE type has associated subkeywords that must be defined to complete the definition. For example, the iso FEATURE requires the Widths and Heights fields to be defined, while the contact FEATURE only requires a Sizes field to be defined.

      • Subkeywords are valid within the scope of their preceding keyword.

    • Keywords and subkeywords are case-sensitive.

    • The first word in any text line must be a keyword or subkeyword, unless the text line contains a comment.

    • A line starting with a pound sign (#) is considered a comment, and all text on that line is ignored.

    • When you define a TPF file, the rows are numbered from the bottom up, so that a test pattern with five rows will have the highest numbered row (5) at the top of the pattern, just under the header lines.

  2. Save the file, and use it as input to either the tepCompile or tepVerification commands.

Results

Open the generated design file in Calibre WORKbench and the accompanying sample spreadsheet in VT5 Center or CM1 Center.
