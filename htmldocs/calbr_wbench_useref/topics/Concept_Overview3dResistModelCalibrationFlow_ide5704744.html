Overview of the 3D Resist Model Calibration Flow

3D Resist modeling is performed in Calibre WORKbench using CM1 Center.
Figure 1. 3D Resist Model Calibration Flow

3D Resist Model Calibration Requirements

Certain items are required to perform the 3D resist model calibration flow.

  • A set of resist height measurements after resist development has been performed in a gauge data file. The height gauge file must follow these guidelines:

    • At least 30 height gauges.

    • At least one large feature to provide a baseline (global) resist loss offset.

    • If possible, a process condition set of data (one nominal, one positive defocus, one overexposed).

    • The “Meas” field in the gauge file corresponds to the gauge height at the gauge coordinate. All Meas values must be less than the original resist height.

    • Gauges must be located on a resist line; space gauges cannot be measured vertically (because a space is a lack of resist height). If a height gauge is set on a resist space, the simulation output typically returns a value of 0.

    • The drawn length of a height gauge does not need to be very long, because only the height at the center of the gauge is reported.

    • Optional but recommended: CDs at multiple heights using a Super Gauge Data File Format at version 2 or higher. CDs at multiple heights can improve the calibration of the vertical diffusion length.

      Tip

      CM1 Center displays the version 2 Type and Plane values on the right side of the Gauges tab.



  • The corresponding set of resist width (critical dimension) measurements in a separate gauge data file.

    This should be measured at multiple process conditions if possible.

  • Ensure that the RESIST_POLARITY parameter is set to -1 if you are using an NTD resist. (The default is 1, which is used for PTD resist films.)

3D Resist Model Calibration Key Concepts

Certain terms and definitions may help you to understand how to perform the tasks for 3D resist modeling.

  • PTD — Positive Tone Developer. A bright field or dark field mask is used with a PTD to create chip structures. Darkfield is typically used for metal layers. The narrower the mask’s CD width is, the less likely you will get a resist height with a full height.

  • NTD — Negative Tone Developer. Unexposed areas get developed, and exposed areas do not. Metal layers in the 20 nm node and below may use NTD resists with a brightfield mask.

  • Imin — A variable representing the minimum intensity value recorded. The higher the Imin, the more likely 3D resist occurs. Imin can be used to help explain the resist loss process, but is not necessary for calibration.

  • beamfocus — An optical model setting. For 3D resist modeling, beamfocus is meant to correspond to the scanner defocus value, but is not known initially and must be calibrated.

  • def_start — An optical model setting, short for “defocus start.” For 3D resist modeling, def_start correlates with the resist CD measurement height. Changing def_start simulates CDs at different planes (the horizontal lines in the figure). The CD measurements at different planes should be placed in a separate Super Gauge Data File Format data file than the height gauges. The super gauge data file must be version 2.

    Figure 2. def_start in 3D Resist Modeling

  • Height gauge — A vertical measurement of the resist profile (the vertical line in the figure). Height gauges are specified in a separate file from the horizontal CD measurements. This file must be in version 1 Super Gauge Data File Format, with the Meas field containing the resist height for the gauge.

  • imageplanes — An optical model parameter that replaces the defocuslevels parameter for 3D resist modeling. When using the imageplanes parameter, the optical model for multiple planes is internally precomputed. It is used with height gauges to simulate resist profiles.
