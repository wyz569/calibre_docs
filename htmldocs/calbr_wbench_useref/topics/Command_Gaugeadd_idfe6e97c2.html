gaugeAdd

CM1 and VEB Commands

Related Formats: Gauge Object Reference

Adds one or more gauges to the specified gauge object.

Usage

gaugeAdd object_output {-g object_input | -f “gauge_filename” | -s gauge_string}

Arguments

object_output

A required argument specifying a gauge object to add to. This can be an existing gauge object or an empty gauge object created by the gaugeCreate command.

{-g object_input | -f “gauge_filename” | -s gauge_string}

A required argument set specifying the input form of the gauge object to add:

  • -g — A previously-defined gauge object (which can be an empty gauge object).

  • -f — A gauge file. The gauge_filename must be relative to the local directory and enclosed in double quotes.

  • -s — A string defining a single gauge, which must be enclosed in quotes and be one of 18, 20, or 22 fields in length. A gauge object’s fields are defined in order and are separated by spaces. See the section “Gauge Object Reference” for a description of the fields.

Examples

Creates a gauge object from a file:

% set aobj [gaugeCreate]
%gaugeAdd aobj -f “gauge.gg”

Creates a gauge object from a string:

% set bobj [gaugeCreate]
% gaugeAdd bobj -s “1 pitch 1 2 8200 12005 8315 12210 0 120 -1 95 94.12 \
 -1 -1 -1 1 0 0 0”

Creates a gauge object from an existing handle:

%set cobj [gaugeCreate]
%gaugeAdd cobj -g bobj
%gaugeGet cobj -row 0 

1 pitch 1 2 8200 12005 8315 12210 0 120 -1 95 94.12 -1 -1 -1 1 0 0 0 NA NA
