Calibrating the Optical Model With Height Gauges and CD Gauges

Follow this task to calibrate the optical model in modelflow_v2. You perform two calibrations; the first only calibrating beamfocus and defocus start, and the second using beamfocus, def_start, and vertical diffusion to see if it provides a more accurate model.

Note:

This task is the “modelflow_v2 stage 1” box in Figure 1 of “Overview of the 3D Resist Model Calibration Flow.”

Restrictions and Limitations

  • Height gauge files and standard (width) gauge files must be separate. Siemens EDA advises that you name the files appropriately, as the gauge files are not internally differentiated for height and width gauges.

  • Calibration of height gauges cannot be done in CM1 Center, but the display of the results can be loaded into CM1 Center.

Prerequisites

  • An optical model that includes HHA and DDM information. The model must also have the vertical_diffusion parameter specified.

  • A model-layer file that includes the optical model file and a CTR resist model. NTD resists must have the RESIST_POLARITY flag set to -1.

  • Recommended: Some CD gauges located at least two different heights around the middle and top of the resist profile. These gauges can be obtained from an AFM profile scan or XSEM image. The gauges are useful when calibrating vertical diffusion.

Procedure

  1. Edit the optical model, replacing the defocuslevels keyword (usually found under the opticalsystem keyword) out for the imageplanes keyword. 
    	opticalsystem 0.193 1.35
	imageplanes nominal 0 auto
  2. Create a modelflow_v2 script that includes the arguments designating the width gauge file and height gauge file.
    	array set retval [modelflow v2 \
	-l $L \ 
	-remotestring $rmstr \
	-sgf width_gauges_filename.sgd \
	-heightsgf height_gauge_filename.sgd \
    Tip

    The example code above includes the -remotestring argument, which specifies remote machines for use with Calibre MTflex.

  3. Add an output section for the height information and height gauges after the -heightsgf parameter.
    	output \ 
		opticalmod height_opt
		log beamfocus_with_height.log \
		heightsgfout beamfocus_with_height.sgd \
		...
  4. Add calibration settings for the first calibration test, which is for beamfocus and def_start only. This configuration quickly creates a set of reference models for further calibration.
    Tip

    If you are using a Super Gauge Data File Format version 2 file, and all gauge sets have a “plane” field specified, you can omit the def_start item from the calibration list, because modelflow_v2 then recognizes this as a 3D resist gauge file. It simulates all CDs at their given planes.

    If any gauge set does not have a “plane” field specified (such as for a SEM CD gauge set), it uses the def_start calibration setting. 

    	stage1 \	
		threshold_optimize \
		dense \
		search front \
		explore 200 \ 
		def_start 0 0.95 10 \
		beamfocus 0.0.095 10 \
	stage2 off]

    If you are using NTD resist with height gauges, you also need the stage1 parameter “vd_top_shrink_a” to model the large resist losses and possibly the parameter “vd_top_surfc_a” to model the resist sidewall angle. These parameters must be in the optical model of the setup. A good way to tell if surface contamination is needed is to simulate a profile using the RET Flow Tool GUI. If the profile is reentrant, but a cross-SEM image or AFM profile is not, then surface contamination calibration is also needed.

    Note:

    If you use vd_top_shrink_a or vd_top_surfc_a you must remove them from the input optical model in stage2 calibration. The stage2 calibration uses vertical shrink and zSlope instead.

  5. Run the calibration.
  6. Edit the model-layer file to use the optical and resist models from Step 5.
  7. Edit the modelflow_v2 script to set up the second calibration test, which adds the vertical diffusion parameter (vd_length) to stage1. If you know the inherent resist loss bias, add the vd_top_adiabatic parameter for adiabatic compensation.
    	stage1 \	
		threshold_optimize \
		dense \
		search front \
		explore 200 \ 
		def_start 0 0.95 10 \
		beamfocus 0.0.095 10 \
		vd_length 0 0.03 10 \
		vd_top_adiabatic 0 1 0 \
	stage2 off]
    Tip

    If you have the computing resources available, Step 7 can be run in parallel with step 4 in order to save time; however, the calibration run in Step 7 will take longer due to the vertical diffusion.

  8. Run the calibration. Examine the results.

Results

Examine the log file output (in the example code, beamfocus_with_height.log). This log file contains the log height error statistics reported. Search for the lines:

  • CTR Height RMS error

  • CTR Height error range

  • CTR Height max abs error

  • CTR Height mean error

  • CTR CD RMS error

  • CTR CD error range

  • CTR CD max abs error

  • CTR CD mean error

The calibration run also creates a super gauge file that shows simulated height gauge data. This file can be loaded into CM1 Center for viewing.

If you are satisfied with the results, use the optical directory height_opt as the optical model in the next task.

The stage1 calibration should have a low RMS error for CD and Height but does not have the lowest error because the CM1 resist model has not been calibrated. Generally, the results in stage1 should show the correct trends through focus and dose. For example, if the measured height data decreases through focus, so should the simulated height through focus.
