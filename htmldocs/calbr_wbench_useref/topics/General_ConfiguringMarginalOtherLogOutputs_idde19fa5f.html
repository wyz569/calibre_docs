Configuring Marginal and Other Log Outputs

Both CM1 Center and modelflow_v2 support adding additional information to the log and analysis files. This procedure details how to configure information to be added to the output. It uses CM1 Center as the basis of the procedure, but the modelflow_v2 options will be listed by each step.

None of the output options listed are required, and they can be set in any order or on later optimization trials.

Figure 1. CM1 Center, Misc Tab (Marginal Output Options)

  • Marginal log file options

    For use with CM1 model development. The following extra outputs can be added in the in the Misc tab by switching their values from ‘off’ to a number of errors of that type to be returned.

    Results appear in the log file after a successful run.

Output of aerial image to resist difference added to the log for each iteration.

  • Additional output options

    Use the Output field in the Misc tab to add any output options that are not configured as part of the GUI from the list shown in Table 2. Separate arguments with spaces.

    Results appear as named files or directories after a successful CM1 Center run.

Table 2. Misc Tab Output Field Accepted Values

modelflow_v2 Equivalent

Brief Description

log log

An alternate name to hold log file results.

optical file

A name for the finalized optical model parameter file.

opticalmod dir

A name for the optical model output directory.

gfout file

A name for an output file containing gauge data with simulated CD/space measurements.

stage1resist file

A name for an output file of the finalized CTR model used with the optical model.

finalresist file

A name for the output of the final CM1 resist model.

etch file

A name for the output of the final VEB etch model.

setup file

A name for the output of a setup file with the finalized optical and resist models inlined.

cache dir

A name for a directory containing the aerial image grids.
