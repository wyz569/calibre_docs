sgd_to_table

CM1 and VEB Commands

Related Formats: Gauge Object Reference, Super Gauge Data File Format

Outputs the supplied super gauge object to a table string, and if supplied, the specified file.

Usage

sgd_to_table { -obj super_gauge_object | -sgf sgdfile} [-out tablefile]

Arguments

-obj super_gauge_object

A required argument that specifies an existing super gauge object to write. Incompatible with the -sgf option.

-sgf sgdfile

A required argument that specifies an existing gauge or super gauge data file to write. Incompatible with the -obj option.

-out tablefile

An optional argument that specifies an output file to write to. If not specified, the output string is written to the terminal window.

Description

The format of the super gauge data in the output string is a contiguous table that is not split into separate gauge sets (each process condition has its own line):

gauge_Set_number dose defocus [type plane] gaugeSet_weight flag struct_name row column x1 y1 x2 y2 location drawn other measured_CD simulated_CD measured_etch_CD simulated_etch_CD precision weight stdev count 95%ci [#comment]

where gaugeSet_number, dose, defocus, and gaugeSet_weight are from the super gauge object and refer to the gauge object’s process condition. The columns “type” and “plane” are only output when the supergauge object is version 2. The remaining values are the same as found in “Gauge Data Objects Defined”.
