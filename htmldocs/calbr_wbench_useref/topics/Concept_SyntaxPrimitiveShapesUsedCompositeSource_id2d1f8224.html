Syntax for Primitive Shapes Used with Composite Source

Each primitive shape definition has a specific syntax for its parameters.

  • STANDARD profile WEIGHT w SIGMA sigma {DSIGMA Δ sigma}a {FILE map_file}b {INLINE id}c {X_LOCATION {none | loc}}d

  • ANNULAR profile WEIGHT w SIGMA sigma SIGMA_IN sigma_in {DSIGMA Δ sigma DSIGMA_IN Δ sigma_in}a {FILE map_file}b {INLINE id}c {X_LOCATION {none | loc}}d

  • QUASAR profile WEIGHT w SIGMA sigma SIGMA_IN sigma_in ILLUMANGLE angle {DSIGMA Δ sigma DSIGMA_IN Δ sigma_in DILLUMANGLE Δ angle}a {FILE map_file}b {INLINE id}c {X_LOCATION {none | loc}}d

  • CROSS_QUAD profile WEIGHT w SIGMA sigma SIGMA_IN sigma_in {FILE map_file}b{INLINE id}c {X_LOCATION {none | loc}}d

  • QUAD profile WEIGHT w SIGMA_CNTR sigma_center ROTANGLE rot_angle RADIUS radius {DRADIUS Δ radius}a {FILE map_file}b {INLINE id}c {X_LOCATION {none | loc}}d

  • DIPOLE profile WEIGHT w SIGMA_CNTR sigma_center ROTANGLE rot_angle RADIUS radius {DRADIUS Δ radius}a {FILE map_file}b {INLINE id}c {X_LOCATION {none | loc}}d

  • ASML_DIPOLE profile WEIGHT w SIGMA sigma SIGMA_IN sigma_in ILLUMANGLE angle ROTANGLE rot_angle {DSIGMA Δ sigma DSIGMA_IN Δ sigma_in DILLUMANGLE Δ angle}a {FILE map_file}b {INLINE id}c {X_LOCATION {none | loc}}d

  • OFFAXIS profile WEIGHT w SIGMA_CNTR sigma_cntr ROTANGLE rot_angle RADIUS radius {DRADIUS Δ radius}a {FILE map_file}b {INLINE id}c {X_LOCATION {none | loc}}d

  • ASML_POLE profile WEIGHT w SIGMA sigma SIGMA_IN sigma_in ILLUMANGLE angle ROTANGLE rot_angle {DSIGMA Δ sigma DSIGMA_IN Δ sigma_in DILLUMANGLE Δ angle}a {FILE map_file}b {INLINE id}c {X_LOCATION {none | loc}}d

  • ASML_MIRRORED_POLE profile WEIGHT w SIGMA sigma SIGMA_IN sigma_in ILLUMANGLE angle ROTANGLE rot_angle {DSIGMA Δsigma DSIGMA_IN Δ sigma_in DILLUMANGLE Δ angle}a {FILE map_file}b {INLINE id}c {X_LOCATION {none | loc}}d

  • SQUARE WEIGHT w CENTER_X x CENTER_Y y SIZE t GAUSS_SIGMA sigma

  • AUTOMAPe weight w {LIMIT v} {SLENGTH l} {SWIDTH d} FILE map_file | INLINE id {X_LOCATION {none | loc}}d

  • COHERENT WEIGHT weight COHERENT_X x COHERENT_Y y

profile = {SHARP | ERF | MAP}

loc = Scanner slit position in microns.

a Only valid for ERF profile.

b Valid for MAP profile and XML source files.

c Only valid for MAP profile, where id is the access identifier of the inlined source map.

d Valid for MAP profile when an XML source file is used. The loc value sets the scanner slit location used to extract the source map for the specified slit location. If the specified location is not in the XML file, then an error occurs. If an XML is used without X_LOCATION or “none” is specified, then all slit locations in the file are averaged to create the final source map. The default is “none”.

e See the section “Using the AUTOMAP Illumination Primitive” for more information.
