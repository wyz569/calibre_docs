modelflow_v2: Stage 1 Optical Model Tuning Options

Parent topic: modelflow_v2

This topic describes the build and optimize modes for optimizing optical models using the modelflow_v2 command.

stage1 is used for tuning the optical model to the input using an empirical data set. The stage1 optical tuning is decoupled from the resist model optimization implemented in stage2. Note the following:

  • This stage is off by default. The nominal optical model is used if stage1 is not specified.

  • The options in stage1 are incompatible with contour-based VEB calibration and cannot be used if stage3 inputlayer is specified.

An intermediate CTR model is used to calculate the CD DIFF RMS

(RMS of CDmeasured - CDsimulated) fitness for each trial optical model.

Two modes of use are available:

  • Run stage1 simulation in “Build” regime, without specifying optical parameters (threshold only, no search specified):

    stage1 threshold {auto | optimize | fixed_threshold}

    In this case, the nominal optical settings will be used to calculate the error RMS for a CTR model with an optimized (in the case of auto or optimize) or user specified (fixed_threshold) threshold value.

    or

  • Run stage1 simulation in the “Optimize” regime to optimize the CTR threshold and/or optical parameters (search mode followed by threshold specified):

    stage1 search full na 0.58 0.65 7 threshold auto sigma 0.4 0.6 3

    In this case, both a search method and threshold must be specified.
