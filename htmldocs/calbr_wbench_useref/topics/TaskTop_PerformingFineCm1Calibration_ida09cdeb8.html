Performing Fine CM1 Calibration

Follow this procedure to fine-tune the CM1 model.

Note:

You can alternatively run optimization in batch mode, using the command set specified in “modelflow_v2 Reference”.

Prerequisites

  • A log file from a completed coarse calibration run (see “Creating an Initial CM1 File”) in order to get the starting range values for this step.

  • The layout file used to create the test pattern loaded into Calibre WORKbench, and the corresponding setup file available.

  • The optimized optical model created in the procedure “Pre-CM1 Calibration Process”.

Note:

If you are continuing directly from the previous procedure “Creating an Initial CM1 File,” you may skip directly to Step 3.

Procedure

  1. In CM1 Center (Litho > CM1 Center), load the session file you created in “Creating an Initial CM1 File”.
  2. Switch to the Optimize tab.

    The Optimize tab displays both the search ranges for the coarse calibration and also the resulting best value found.

  3. For each parameter listed, adjust the Begin, End, and Levels columns to new values for the fine calibration run.

    • In general, set Begin = Best – stepsize/2 and End = Best + stepsize/2, where stepsize is the step size used in the coarse calibration.

    • If the Best value is close to either Begin or End from the coarse search, you may wish to extend the range of the fine search somewhat beyond the point in question to better guarantee the true optimal value is found.

    • Levels should be set to achieve the final resolution you want.

    • You can speed up optimization by specifying that some of the optimization parameters are dependent on others. This reduces the dimensionality of the optimization process, which is equal to the number of optimization parameters.

      • Variations of an independent parameter are determined by the search interval (Begin and End) and the number of parameter values (Levels) to explore.

      • Variations of a dependent parameter are determined by the search interval and a Tcl expression constraint, which specifies a relationship between the dependent parameter and other independent parameters involved in the optimization.

        Note:

        For an explanation of how independent and dependent parameters are implemented, see “Using the Independent, Constrained, and Dependent Optimization Parameters” in “modelflow_v2: Stage 1 Optimization Run Modes”.

        For more information how to configure independent and dependent (“constraint”) values, see the section “Hints for CM1 Calibration”.

  4. If you select Gradient or Newton search, then enter a value in the Explore field that corresponds to at least 10 percent of parameter combinations you would explore if you selected Full factorial.
  5. If you have a multi-threading capable machine, you should also select the Turbo checkbox.
    Note:

    The Turbo and Distributed run options are mutually exclusive. Selecting one automatically de-selects the other.

  6. Save your session file (File > Save Session).
  7. When all parameters are correctly defined for the optimization run, click the Run button.

    CM1 optimizes for the model, and adds a result log to the pane at the bottom of the window. It also returns the lower and upper bounds found during the search in the Optimize tab (for the Front, Full and Gradient search methods).

  8. Select and analyze the results by selecting the new result log entry and clicking Plot: Analyze. If you are satisfied with the results:

    • Click Accept: Gauges and Accept: CM1.

    • Save the session file (File > Save Session).

    • Save the log file (Save button at the bottom right corner of the CM1 Center window).

    • Save the CM1 model (CM1 > Save menu item).

    If you are not satisfied with the results:

    • Add additional terms (up to a maximum of 16) and see if fitting improves.

    • Remove terms with an extremely small (< 0.0001) coefficient.

    • Remove outliers.
