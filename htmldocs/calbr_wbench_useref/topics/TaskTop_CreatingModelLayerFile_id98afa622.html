Creating a Model-Layer File

If you do not have a sparse modeling setup file, you can create a simplified setup file that contains only model and layer information, referred to as a ‘model-layer’ file.

Tip

This task is optional, but recommended if you have not had a setup file supplied to you by your modeling engineer. Most of the settings in a setup file are not used for model calibration.

This file can be used in CM1 Center and modelflow_v2 in place of a standard sparse modeling setup file.

Prerequisites

  • Knowledge of your design file layer list and background information

  • At least one optical model

Procedure

  1. Using your preferred text editor, create the following entries:

    • modelpath path_to_model_files

    • optical_model mask_number optical_model_directory_name

    • resistpolyfile resist_model_filename

    These entries specify the models that will be used for calibration and simulation. For more information, see the section “Model-Layer File Format”.

  2. Optionally, if you have a DDM and HHA masksim output file (created using the task “Creating a DDM Library With masksim Batch”), load it with the command:

    • ddm_model ddm_alias masksim_output_file

    • You can also use one or more ddm_group commands in place of a DDM model statement:

      ddm_groupM ddm_alias {filename | ‘{’ inline_definition ‘}’}

      where M can be from 0 to 4, representing a DDM group.

  3. Add the following entries based on the design specifications:

    • background background_type

    • One or more layer statements describing the layer(s) with the test structures using one of the following formats:

      • Standard format:

        layer layernum1 {trans [attFactor] | real imag} [bias xbias ybias] [maskNum dose]

      • DDM and HHA format:

        layer layernum1 real imag ddm ddm_alias

        Only the layers that you specify in this file will be calibrated. All other design layers will be ignored.

    For more information, see the section “Layers Section” in “Model-Layer File Format”.

  4. Optionally, if you are using the Mask Model Format, add one or more mask model lines:

    mask_model [maskNum dose] {file | ‘{’ inlined_model_text‘}’}

    Note:

    If you calibrate CM1 using mask models, you must also output using the litho model directory format, which requires the setting the Output field in the Misc tab in CM1 Center to lithomodel directory_name.

  5. Save the file.

Results

You should have a file that looks similar to the following: 

	modelpath .:models:/home/taldin/.calibrewb_workspace/models 
	opticalmodel defaultopt 
	background dark 
	layer 0 0.9 0.02
