SRAF Calibration Requirements

The following items are required to perform the SRAF calibration flow.

  • An SRAF gauge data file with printing and non-printing SRAFs. Gauges for the SRAFs must be designated as:

    • non-printing — Set a value in the Meas field of LT:0

    • printing — Set a value in the Meas field of GT:0.

      Note:

      Gauges should be centered on a single SRAF, not the associated structure in the test pattern. The length of a gauge should just cover the SRAF as shown in the figure.

      Figure 1. Example Illustration of a Gauge File and Associated SRAF Gauge

    However, if you are modeling dual tone SRAFS, the gauge data file must be separated into two files, such that one file contains SRAFs as widths (using a Loc field value of 0) and the other file contains SRAFs as spaces (using a Loc field value of 1). You calibrate these gauge files separately, resulting in an optical model for width-based SRAFs, and a second optical model for space-based SRAFs.

  • A standard CD gauge data file, required for the CM1 model.

  • The corresponding test pattern design file. This ideally should have a wide variety of test patterns with variable-sized SRAFs.

    For a CTR model, the number of test patterns should be at minimum five per number of variables optimized, showing the transition from not printing to printing. Approximately one hundred test patterns are preferable.
