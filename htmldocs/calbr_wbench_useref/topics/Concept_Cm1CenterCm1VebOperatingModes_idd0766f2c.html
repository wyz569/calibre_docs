CM1 Center CM1 and VEB Operating Modes

CM1 Center’s CM1- and VEB-based action buttons (Optimize, Build, and Try, shown in CM1 Center as O, B, and T buttons and the X(off) button) execute the modelflow_v2 command in different modes.


All simulations are done on the currently-loaded layout.

Table 1. Optimize, Build, and Try Modes in CM1 Center CM1 Modes

Mode

Definition

O(ptimize)

Searches for the best values of both linear and non-linear coefficients entering the CM1 and VEB models, depending on whether these models are selected in the Optimization tab.

B(uild)

Executes exactly one optimization iteration. It tunes the linear coefficients of the model.

T(ry)

Uses the current settings in CM1 Center without optimization. You use this mode as a fitness check on the currently-loaded model in CM1 Center.

(e)X(cluded)

Deactivates CM1 and VEB generation for this run.

Tip

The Time button can be used to take a reading on how long one frame of simulation will take to run. This is mainly used to test for the run time of different modelforms in CM1 and VEB models.
