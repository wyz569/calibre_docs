Setting Optimizations for the Optical Model

In this procedure, you set options to tune the optical model, corresponding to modelflow Stage 1.

Configuring options on this page causes VT5 Center to explore small variations on a set of parameters in order to find the optical model that best matches the data using an EPE RMS testing algorithm.

Once the optical model has been optimized, you can switch the mode for this phase to Try mode. This is because VT5 Center will use the now-optimized optical model shown in the Optics, in tab, and it optimizes the reference threshold (Stage 2) and resist model (Stage 3) separately.

Prerequisites

  • Optical model loaded into the Optics, in tab from “Loading an Optical Model”

Procedure

  1. In VT5 Center, click the Optics, 1 tab.
    Figure 1. VT5 Center GUI (Optics, 1 Tab)

  2. Choose the search method VT5 Center will use to explore the parameter space. Use one of the options described in Table 1. Siemens EDA recommends selecting the Front search for the current release.
    Table 1. Optimization Algorithms in VT5 Center

    Option

    Description

    Results

    Full Search

    A full factorial search over all combinations of variables.

    Slowest, but more exact.

    Multithreaded Full

    A multi-threaded version of the full factorial search.

    Use this if you have additional computing power available; takes advantage of up to 32 CPUs.

    Gradient Descend

    A simple conjugate gradient descent method.

    Faster than Full search, but potentially less exact.

    Multithreaded Gradient

    A multi-threaded version of the gradient descent method.

    Use this if you have additional computing power available; takes advantage of up to 32 CPUs (only 2 during the gradient search phase, however).

    Front Search

    A global optimization search performed across the whole parameter space.

    Searches are independent of the local starting point (initial guess). Iterations limited by the Run Time or Number of Explorations GUI settings if specified; otherwise limited to 10000.

    Multithreaded Front

    A multi-threaded version of the Front search method.

    Uses up to 32 CPUs in ‑turbo mode or more if multi-threaded options are set.

    FrontZoom

    A combination of a global (front) and local (zoom) optimization search method.

    —

    Multithreaded FrontZoom

    A multi-threaded version of the frontzoom search method.

    —

    Newton Search

    A quasi-Newton search method, in which the second derivatives are evaluated in terms of the sequence of gradients from successive iterations using the Broyden-Fletcher-Goldfarb-Shanno update formula for the Hessian matrix.

    Faster and more exact than gradient search.

    Multithreaded Newton

    A multi-threaded version of the quasi-Newton search.

    Use this if you have additional computing power available; takes advantage of up to 32 CPUs.

    Note the following when selecting an optimization algorithm:

    • When using the gradient or quasi-Newton methods, use the Number of Exploration Runs field to limit the size of its Full or Front factorial search. See the modelflow (Version 1) batch command page for more information.

      Note:

      Using the Number of Exploration Runs field with gradient searches gives different results than using it without. To use the standard gradient search, set the Exploration Runs to 0.

      Note:

      You can limit the number of CPUs used by the Multithreaded Full and Multithreaded Newton algorithms by setting the following Linux® environment variable before invoking Calibre WORKbench to a value below the maximum CPU limit (32). The multi-threading options do not consume additional licenses:

      setenv CWB_MT_THREAD_COUNT value

    • Use 1D Optical Model to optimize only 1D optical models. 2D optical models are filtered from the spreadsheet.

    • Use force symmetry to force symmetry of the final optical model by increasing the number of kernels used. This option does nothing if no optical parameters are selected in the Optical Parameters to Optimize section.

  3. Select one or more parameters to optimize.

    By default, VT5 Center starts with no parameters to be optimized. Selecting a parameter’s check box specifies that parameter for optimization. Each parameter has a default Min, Max, and step range that must be inspected and modified if necessary. VT5 Center will test the within the specified range settings with combinations of other selected parameters.

    CAUTION:

    The more parameters you select, the longer optimization runs take. It is also possible to exceed your memory space, which will cause Calibre WORKbench to crash during an optimization run.

    • The largest sigma_in value may not exceed the smallest sigma_out value.

    • The Min value should be the smallest numerically and the Max value should be the largest numerically.

    Use the information in the section “Best Practices for Optical Modeling” to decide what to optimize.

  4. Optionally, use the Filter Option at the bottom of the tab to only optimize the optical model for a subset of the available test structures; specify the structures by clicking Wizard and using the dialog box that appears. Note that the structures used are selected from the set you optionally created with the Pass and Exclude Filter in “Setting Global Sample Filtering”.
  5. You are now ready to optimize the optical model alone and can proceed to:

    • “Inspecting Run Options and Running the Calibration”.

    or

    • Continue configuring the VT5 Center GUI to optimize reference threshold and VT5 models by continuing to “Setting Reference Threshold Optimization Filtering”.

    or

    • “Setting Optimization for the VT5 Model”.
