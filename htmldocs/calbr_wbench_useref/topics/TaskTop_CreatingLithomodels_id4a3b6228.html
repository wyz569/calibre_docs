Creating Litho Models

The Lithomodel Creation Wizard collects all inputs that are required to create a litho model file.

Restrictions and Limitations

  • Litho models only support a single resist model per litho model definition. Only CM1 resist models can be used in the Lithomodel Creation Wizard; VT5 resist models are not supported.

  • Optical models must use resolvable paths for included source, Jones pupil, apodization, and laser spectrum files. Both absolute paths and relative paths (from the optical directory or current working directory) are accepted.

  • The Calibre WORKbench Lithomodel Creation Wizard and Calibre nmModelflow GUIs cannot be open simultaneously. If using Calibre nmModelflow after creating a litho model in the Calibre WORKbench Lithomodel Creation Wizard, close the wizard first and then activate Calibre nmModelflow.

Prerequisites

  • Any model components needed to describe the lithographic exposure.

    Although the Lithomodel Creation Wizard can create new models, the typical OPC workflow requires at least one existing optical model and one existing resist model to supply to the litho model. For the MPC and CDSA workflows, no resist or optical models are used.

  • If using previously-computed optical model modes, you must provide the additional optical models.

  • Other models, such as DDM models, are optional. The additional models, however, may be required for simulation accuracy.

  • An empty directory.

    The Lithomodel Creation Wizard creates a new directory to store its files. It links the original source directory optical model within that directory. Do not use the source directory containing any models you want to keep as the wizard may overwrite the original files.

  • Calibre WORKbench invoked.

Procedure

  1. Choose the Litho > Litho Model menu item. The Lithomodel Creation Wizard appears. The wizard lists each step that requires input (Output Directory, Lithomodel Components, Exposure, and Finish).


  2. In the Output Directory page, enter or navigate to an empty working directory. The directory will contain the completed litho model. (You can create a new directory from the file chooser dialog.) Name the new litho model directory in the Enter Lithomodel Name field.

    In most cases, leave the Create Lithomodel For radio button set as OPC. However, users who want to create MPC, CDSA, or global litho models will need to select the appropriate option. The flows for each type of litho model are different.

  3. In the Lithomodel Components page, fill out the fields to match your configuration and click Next.
    Table 1. Lithomodel Creation Wizard, Lithomodel Components

    Field

    Notes

    No. of Exposures

    Required. Controls the number of exposure masks. Increasing the number of exposures adds more items to the list when you click Next.

    Image Threshold

    Sets the image threshold for the litho model. By default, 0.35 is used.

    Resist Model

    Required. Specifies the path to the resist model for the litho model.

    • You can alternatively select a preset modelform template using the dropdown menu.

      Tip

      The Preview Resist Modelform button brings up the View Modelform window, where you can see the model template definition for a selected modelform.

    • If you select the 3D Resist checkbox, additional modelform options related to 3D modeling become available. This option is unavailable when the modelform is an active lithomodel.

    Etch Model

    Use only if an etch model is part of your process.

    Specifies the path to the etch model for the litho model.

    • You can alternatively select a preset modelform template using the dropdown menu.

      Tip

      The Preview Etch Modelform button brings up the View Modelform window, where you can see the model template definition for a selected modelform.

    • You can alternatively create your own custom VEB etch model by clicking the Create button (see “Create Custom Etch Model Wizard”).

    Topo Model

    Use only if modeling resist topography.

    Specifies the path to the topo model for the litho model.

    You can alternatively select a preset modelform template using the dropdown menu.

    Tip

    The Preview Topo Modelform button brings up the View Modelform window, where you can see the model template definition for a selected modelform.
    Note:

    The following four items only appear if you select the EUV Model checkbox.

    Black Border Model

    This item is used only by EUV modeling processes.

    Specifies the path to a black border model to be used for the litho model.

    Note:

    Specifying a black border model also requires you to specify a flare map file.

    Flare Model

    This item is used only by EUV modeling processes.

    Specifies the path to a flare model to be used for the litho model.

    Stochastic Model

    This item is used only by EUV modeling processes.

    Specifies the path to a stochastic model to be used for the litho model.

    For more information on stochastic models and how to use them, see “Stochastic Model Support in Calibre EUV”.

    Shadow Bias Model

    This item is used only by EUV modeling processes.

    Specifies the path to the shadow bias model to be used for the litho model.

    Note:

    Shadow bias models are not compatible with global litho models.

    ZPlanes

    This item is used for SRAF printing and TOPLOSS models.

    Specifies the path to the ZPlanes model for the litho model.

    If you select the ZPlanes checkbox, additional model options become available. ZPlanes allows for additional model parameters to be adjusted for models that can have a different image plane, such as for SRAF printing or TOPLOSS models.
  4. For each exposure, you must fill out three screens:
    1. Nominal Optical Model — Specify the nominal optical model (dose 1.0, defocus 0) in this screen. You have a choice of adding additional defocus models.

      • Nominal optical models can be loaded from the disk or generated.

        • To load an optical model directory from the disk, choose Use Existing Nominal Optical Model.

        • To use an existing Optical file on the disk as a starting point, choose Generate Using Existing Optical File. Modify the loaded file and then continue.

        • If you want to create an Optical file directly in the Optical File Contents field (or copy and paste contents from another file), choose Generate Using New Optical File. You can modify the Optical file and then click Generate Optical Model.

        • If you do not have an Optical file, choose Generate Using Optical Model Tool. You then fill out the Optical Model Tool and its results are used in the text field.

        For more information on generating optical models, see “Generating the Optical Model”.

      • If you already have the defocus models generated, check “Add Defocus Model using existing model on disk”.

      • If you only have the nominal optical model and desire to add more defocus conditions, check “Add Defocus levels to Generate new Optical Models”. The wizard will prompt you to enter the defocus range when you click Next.

      Leaving both of the checkboxes unset uses only the nominal optical model.

    2. Background and Dose — Specify the background transmission type and initial dose value if it is different than 1.0 for the nominal optical model. For more information, see “Mask Layer Transmission” types in Table 2.
    3. Mask Layer Index — Specify characteristics for one or more mask layers. All items except Mask Layer Transmission are optional. Click Next.
      Figure 1. Lithomodel Creation Wizard, Mask Layer Screen

    4. Click Add New Mask Layer if you need to define multiple mask layers. Otherwise, click Next.
      Table 2. Lithomodel Creation Wizard, Mask Layer Index Fields

      Field

      Description

      Mask Layer Transmission

      Specifies the transmission type of the mask_layer template.

      The field allows the following types: dark, clear, atten x, phase90, phase270, phase180, or a real imaginary pair.

      Category

      Specifies a comment. The comment can only contain alphanumeric, dash, and underscore characters.

      CX (um)

      Specifies a convex cornerchop value in microns. Corner chop removes a right triangle polygon with isosceles legs equal to the CX value from convex corners. Only right-angle corners are chopped.

      CC (um)

      Specifies concave cornerchop value in microns. Corner chop adds a right triangle polygon with legs equal to the CC value to concave corners. Only right-angle corners are filled.

      X Shift (um)

      Shifts all polygons in the X direction by the specified amount.

      Y Shift (um)

      Shifts all polygons in the Y direction by the specified amount.

      DDM Model

      Specifies a DDM model to associate with the mask layer.

      Bias (um)

      Expands or contracts all edges by the specified amount in microns.

      Bias values are applied to both horizontal (X) and vertical (Y) directions.

      X Bias (um)

      Expands or contracts only horizontal edges.

      Y Bias (um)

      Expands or contracts only vertical edges.
  5. The Finish screen shows a summary of the commands that the Lithomodel Creation Wizard executes when you click Finish.
  6. Click the View Lithomodel button to examine the contents of generated litho model.
  7. Click the Close button.

Results

A litho model directory is created in the specified directory.
