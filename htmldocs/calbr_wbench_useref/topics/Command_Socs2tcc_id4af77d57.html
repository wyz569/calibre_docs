socs2tcc

Optical Model Commands

Converts a SOCS model to a TCC model.

Usage

socs2tcc -in {optical_model} -out {sparse_tcc_file} [-numkerns N]

Arguments

-in optical_model

A required argument specifying the optical model, which must be a SOCS model.

-out sparse_tcc_file

A required argument specifying the output TCC file.

-numkerns N

An optional argument specifying the number of kernels to truncate from the calculation. If this argument is not specified, no kernels are truncated.

Description

This command is useful when making decisions about how many SOCS kernels are needed for a good approximation of the TCC matrix. It loads the SOCS kernels and weights from the input optical model and computes the approximate TCC matrix that corresponds to the SOCS truncation when specified by the optional argument -numkerns.

By default, all available SOCS kernels and weights are used to compute the approximate TCC matrix. If specified, N cannot exceed the number of available SOCS kernels present in the model.

The output TCC matrix is written in sparse text form (as described in the reference page for “Sparse TCC Matrix File Format”).

Examples

First, an optical model is computed with a large number of kernels (200 in this example). Then, iterative use of socs2tcc and comparetccs is used to decide how many kernels to keep in the optical model. For a SOCS truncation with 40 kernels, the results look like this: 

% socs2tcc -in optmod -out approxtcc.txt -numkerns 40 
% comparetccs -tcc1 optmod -tcc2 approxtcc.txt 
Comparing TCC matrices: 
Size of first (35,35)^2 - Number of nonzero elements 377432 
Size of second (33,29)^2 - Number of nonzero elements 590729 
L1_norm = 0.00048739
