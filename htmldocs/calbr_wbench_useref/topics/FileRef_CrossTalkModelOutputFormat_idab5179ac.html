Cross Talk Model Output Format

DDM Modeling Extension

The cross talk model modifies DDM modeling by changing the way that mask transmission signals are computed.

When a cross talk model is used, the mask transmission functions are computed by adding constant signals at rectangular regions defined by opposing nearest edges. The value of these signals depend on the distance between the opposing edges, the type of opposing edges (horizontal, vertical, or diagonal) and the angle of incidence and polarization of the incident wave.

Figure 1. Cross Talk Edge Illustration

Consider polygons P1 and P2. Yellow signifies the mask layer and white is the mask background. Consider this to be a clear background mask.

  • There are two pairs of opposing edges with the space between them corresponding to the mask layer highlighted as green in Figure 1. These two pairs as defined by the vertices are: (v2,1v2,2, v2,5v2,3) and (v1,1v1,3, v1,5v1,4). The space between these two pairs corresponds to “dark” transmission.

  • There is one pair of opposing edges with the space between them corresponding to the mask background highlighted as red in the figure. This pair as defined by the vertices is: (v2,4v2,3, v1,1v1,2). The space between this pair corresponds to “bright” transmission.

Format

crosstalk_model {none | bright_gap | dark_gap | all}
crosstalk_range [bright_gap r1_um r2_um] [dark_gap r3_um r4_um]
small_edge_limit_crosstalk value 
crosstalk_euv_smoothing [{0 | 1} | {no | yes}]
crosstalk_signal {bright_gap | dark_gap} 
	edge_type {horizontal | vertical | diagonal45 | diagonal135} 
	theta offaxis_angle phi azimuthal_angle 
	polarization {TE | TM | UNPOLARIZED}‘{’
distance_in_um1 real1 imag1 [deltawidth1 shift1]
distance_in_um2 real2 imag2 [deltawidth2 shift2]
...
‘}’

Parameters

crosstalk_model {none | bright_gap | dark_gap | all}

An optional keyword that indicates the type of cross talk model. The following options are set on output:

  • none — No cross talk model is used. This is always the setting for EUV, and when DUV models do not have any cross talk information in the file.

For DUV models, the crosstalk_model is set depending on the contents of the optical model file (“bright_gap”, “dark_gap”, or “all” for both).

  • bright_gap — Specifies that cross talk signals are added only for opposing edges when the space between them corresponds to the “bright” mask transmission. The bright mask transmission value will correspond to the presence or absence of a polygon, depending on the transmission values of the background and the layer (“bright” is the largest transmission between the background and the layer).

  • dark_gap — Specifies that cross talk signals are added only for opposing edges when the space between them corresponds to the “dark” mask transmission (“dark” is the smallest transmission between the background and the layer).

  • all — Specifies that cross talk signals are added for all opposing edges (separate cross talk signals for opposing edges surrounding bright_gap space or dark_gap space).

crosstalk_range [bright_gap r1_um r2_um] [dark_gap r3_um r4_um]

An optional keyword when cross talk modeling is active that specifies the lower and upper bounds of distances between opposing edges for which cross talk signals will be applied. If crosstalk_range is not specified, then the range will be specified according to the optical model. For DUV models, the crosstalk range is set to the range of values in the library. For EUV models, the maximum range is set to 0.2 microns.

When crosstalk_range is specified, at least one range (for bright_gap or dark_gap) must be specified.

Range values are specified in microns in mask dimensions. Ranges must be specified such that 0 < r1_um < r2_um and 0 < r3_um < r4_um.

Signals from cross talk are added to the mask transmission function only for opposing edges whose distance is less than r2_um or less than r4_um in the bright_gap or the dark_gap category, respectively.

When the distance of opposing edges is less than r1_um or r3_um, the cross talk signal for distances equal to r1_um and r_3_um respectively is added.

Cross talk signals are zero for larger distances, but equal to the value of the low bound for very small distances. If the cross talk model is set up correctly, there will not be edges that are closer to one another than the low bound.

small_edge_limit_crosstalk value

An optional keyword that causes the model to ignore crosstalk for edges smaller than a threshold. Specify value in mask dimensions. The threshold is the value divided by the reduction factor. This option can be used independently of small_edge_limit. (Default is 0 microns — no edges are ignored.)

crosstalk_euv_smoothing [{0 | 1} | {no | yes}]

An optional keyword set that improves EUV crosstalk consistency. The keyword is valid for DDM version 6 and later libraries. Either the numerical or text option may be specified. Starting in 2019.2, the default output is “crosstalk_euv_smoothing 1”. The default behavior is no EUV crosstalk smoothing.

crosstalk_signal {bright_gap | dark_gap} edge_type{horizontal | vertical | diagonal45 | diagonal135} theta offaxis_angle phi azimuthal_angle polarization {TE | TM | UNPOLARIZED}

A required keyword set for cross talk modeling that signifies the beginning of a cross talk signal. All settings in this line are required.

One and only one of the choices “bright_gap” or “dark_gap” is required.

  • bright_gap — Specifies that the signal that follows corresponds to gaps when the transmission inside the gap is “bright.”

  • dark_gap — Specifies that the signal corresponds to gaps with “dark” transmission.

The “edge_type” must also be specified. It categorizes the cross talk signal depending on the orientation of the opposing edges that form the gap.

The settings for “theta” and “phi” correspond to the off-axis and the azimuthal angle and are specified in degrees. Note that the settings for “theta”, “phi” and “polarization” are identical to the settings that correspond to the main DDM signals elsewhere in the file. Although it is not required to have the main signals and the cross talk signals specified at identical conditions for theta, phi, and polarization, there is no known reason to specify them for separate conditions.

distance_in_um real imag [deltawidth shift]

Each line of the cross talk signal block consists of triplets of (distance, real, imag) numbers specifying the distance between two opposing edges and the corresponding complex value of the cross talk signal. One triplet is given per line, and there is no limit on the number of triplets.

The first triplet must follow immediately after the opening brace “{” of the respective crosstalk_signal line and the last one must be followed by a line with just a closing brace “}”.

One triplet is sufficient to specify a cross talk signal but in a typical case, there will be many lines corresponding to different distances.

Optionally, a deltawidth and shift value may be specified for any given line.

  • deltawidth is the change in width in microns at mask dimensions of the cross talk tophat signal. Positive deltawidth values make the width larger, while negative deltawidth values make the width smaller.

  • shift is the change in direction for the signal. Positive shift will move the tophat cross talk signal in the +x direction or +y direction for vertical and horizontal cross talk, respectively. Negative shift will move the tophat cross talk signal in the -x direction or ‑y direction for vertical and horizontal cross talk, respectively.

In other words, the two edges of the tophat cross talk signal will move as follows:

	e1’ = e1 + shift - deltawidth/2
	e2’ = e2 + shift + deltawidth/2
