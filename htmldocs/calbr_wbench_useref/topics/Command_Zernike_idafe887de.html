zernike

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Defines Zernike coefficients for lens aberrations.

Usage

zernike vals | {{file filename | inline inline_id} x_location loc}

Arguments

vals

A required argument specifying Zernike coefficients using the Born & Wolf numbering convention.

Up to 200 Zernike coefficients can be specified in order from z1 to z200 to model aberrations in the pupil of the stepper or scanner, where z1 = piston type aberration and z4 = defocus type aberration. All Zernike coefficients must be specified, up to the highest non-zero coefficient input. All Zernike coefficients after the highest non-zero coefficient are assumed to be zero.

The default is no aberrations.

file filename

A required argument set specifying an .xml input file from an ASML scanner. filename is the path to the file.

  • Each Zernike location is stored in the XML hierarchy at: Results::ZernikeTable::elt. The x-position, in millimeters, is stored in <x>. The y-position, in millimeters, is stored in <y>.

  • The actual Zernike values are stored in <Zn> entries, where n is 1 to 200. The values are in units of nanometers “peak”.

    See the Examples section of an example of <ZernikeTable>.

  • Z37 has special rules. If Z37 is the final value specified, it is assumed to actually be the value for Z49. Otherwise, all values are used directly.

  • If one of the first 5 Z -values is larger than 1000, this is assumed to be invalid and ignored.

  • The .xml file will usually contain several Y-positions. A weighted average of these locations is used as the Zernike values for simulation. The weighting is determined by the <MachineType> entry in the <Header> section of the .xml file.

    If the <MachineType> is NXE33* or NXE34* (where * can be any string) and there are exactly 5 Y-locations, the weighting follows Table 1 (field position 1 is the most negative y-value). If not, all Y-positions are weighted equally.

inline inline_id

A required argument set specifying an inline Zernike block by its identifier inline_id. The inline Zernike block must be specified using an inline_zernike command.

x_location loc

A required argument set specifying the position in microns in the slit from which the aberrations should be loaded from. The x_location value must match a location from the specified inline identifier or external file which contains Zernikes.

Description

This optional pupil parameter defines up to 200 Zernike coefficients when they are used to define lens aberrations. Parameters are defined using space-separated numbers; tab-separated values are ignored. The default is no aberrations.

Note:

To use the file argument, zernike_multiplier_flag must be set to 1 and zernike_rms_flag must be 0. If unset, they are set automatically.

Examples

The following example specifies a simple defocus aberration of 0.2 using val arguments:

zernike 0.0 0.0 0.0 0.2

An example of a <ZernikeTable> section is as follows:

<Results>
	<ZernikeTable>
		<elt>
			<x>-12.72</x>
			<y>1.47</y>
			<Z2>1000000.00</Z2>
			<Z3>1000000.00</Z3>
			<Z4>1000000.00</Z4>
			<Z5>-0.15</Z5
			<Z6>0.53</Z6>
			...
			<RMS>0.00</RMS>
		</elt>
	</ZernikeTable>
</Results>
