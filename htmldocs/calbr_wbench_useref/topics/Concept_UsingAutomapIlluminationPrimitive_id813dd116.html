Using the AUTOMAP Illumination Primitive

The AUTOMAP illumination primitive method is the preferred way to enter a source pupil (sourcemap) in the optical file when the simulation engine is TCCcalc. When the simulation engine is rapidTCC, using AUTOMAP is not needed.
Tip

With TCCcalc, the use of AUTOMAP is recommended in place of MAP for complex sources, such as SMO sources. With rapidTCC, the MAP function is still recommended; if used with simple sources, AUTOMAP may produce unnecessarily long runtimes unless the optional LIMIT, SLENGTH and SWIDTH parameters are also set.

The direct computation of the TCC integrals that is employed in TCCcalc results in significant computation inefficiencies when a sourcemap is specified without proper bounds, such as existing inside of the unit circle. In such a case, many integration points may be wasted in places where the source is dark. Also, if the sourcemap describes a source that has many small illumination spots, the simulation accuracy can suffer greatly unless the integration order is increased, because some spots may be missed altogether. Moreover, it is cumbersome and inefficient to introduce multiple composite elements in order to induce tighter bounds for the sourcemap.

Using AUTOMAP solves these problems by internally computing tight bounds for the integration. AUTOMAP divides the pupil into sectors of fixed length and angular width and only the sectors that contain bright source points are kept for the subsequent calculations. Automap uses the following syntax:

AUTOMAP weight w {LIMIT v} {SLENGTH l} {SWIDTH d} FILE map_file | INLINE id

where

  • LIMIT v

    Optionally Specifies a limit in units of intensity value below which the source is considered dark (I=0). Default is 1e-3.

  • SLENGTH l

    Optionally specifies the radial length of the sectors in fractions of the unit pupil circle. Default is 0.1.

  • SWIDTH d

    Optionally specifies the angular extent of the sectors in degrees. Default is 5.

  • FILE filename | INLINE id

    A required argument that specifies either a filename (FILE argument) for the optical sourcemap, or an inline ID of a text string block that defines the sourcemap found elsewhere in the Optical file.
