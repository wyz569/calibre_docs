How to Select a Modeling Flow

The process you use when you send a mask design to the mask shop dictates how you create the test pattern and use it with the VT5 Center tool. Two flows are presented here, differing in terms of the relationship between the target CD, design CD, and mask CD.

In Brief

Target CD = Design CD != Mask CD

Description

Referred to as “Simple” because the target, design, and mask CDs are all the same.

Typically used for a contact layer with an attenuated PSM that has global sizing applied. For example, the target and design CDs might be 100nm, and the mask CD needed to obtain a wafer CD of 100nm might be 120nm.

Step 1

Create the test pattern GDS file and sample spreadsheet at mask CD.

  • See “Generating a Test Pattern With the Calibre WORKbench GUI” for information on how to use this tool.

Step 2

—

Apply a bias (by sizing the pattern so that it represents the mask CD) to the test pattern GDS file that you will also apply to your data before generating the mask.

Step 3

Put the test pattern on the mask.

Put the modified test pattern on the mask.

Step 4

Get the CD measurements from the fab and enter the data in the sample spreadsheet.

Get the CD measurements from the fab and enter that data in the sample spreadsheet that was generated in Step 1.

Step 5

Build and optimize the optical and resist models.

Build and optimize the optical and resist models using the modified GDS file created in Step 2, plus the sample spreadsheet with the CD measurements from the fab. The VT5 Center tool has all the data it needs:

  • the target CD (in the spreadsheet)

  • the mask CD (from the GDS file)

  • the wafer CD (fab data in the spreadsheet)

Step 6

Use both models for performing simulation within Calibre OPCpro, Calibre ORC, and Calibre PRINTimage.

Use these optical and process models for performing simulation within the Calibre OPCpro, Calibre ORC, and Calibre PRINTimage batch tools.

Note: When the models are used with the OPCpro batch tool, you will not need to apply the bias to the data from the designer, as that biasing is embedded within the model.
