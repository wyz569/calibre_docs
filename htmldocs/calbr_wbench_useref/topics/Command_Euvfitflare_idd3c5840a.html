euv::fitflare

Optical Model Commands

Uses input from: Flare Intensity File Format

Creates a flare model file from a flare intensity file.

Usage

euv::fitflare –in input_FIF [–out output_FIF] [-kernels kernel_count]
[-rmin rmin] [-rmax rmax]
[-nmin nmin] [-nmax nmax]
[-n1 n1_value] [-n2 n2_value] [-nstep delta_n]
[-u aspect] [-umin umin] [-umax umax]
[-inmodel input_FMF_file] [-outmodel output_FMF_file] [-verbosity level]

Arguments

-in input_FIF

A required argument that specifies a file that contains the experimental levels of flare intensity, in flare intensity file (FIF) format.

-out output_FIF

An optional argument that specifies an output FIF file that on completion contains the predicted levels of flare in the Ipred column of the FIF file. All other columns are copied from the input_FIF.

-kernels kernel_count

An optional argument that specifies the number of fractal kernels to fit, 1 or 2. Default is 1.

-rmin rmin

An optional argument that specifies that the command is run only for those rows of the input FIF where r >= rmin. The default is no lower bound.

-rmax rmax

An optional argument that specifies that the command is run only for those rows of the input FIF where r <= rmax. The default is no upper bound.

-nmin nmin -nmax nmax

An optional argument that specifies the lower and upper search range for the n1 and n2 parameters during calibration. Default is 0.1 for -nmin and 3.0 for -nmax.

-n1 n1_value

An optional argument that specifies that the parameter n1 is excluded from the calibration procedure, and its value is fixed at n1_value. By default, n1 is optimized.

-n2 n2_value

An optional argument that specifies that the parameter n2 is excluded from the calibration procedure, and its value is fixed at n1_value. By default n2 is optimized.

-nstep delta_n

An optional argument that specifies the sampling step size for n1 and n2 parameters during calibration. Default is 0.01.

-u aspect

An optional argument that specifies the parameter u, signifying the aspect, is excluded from the calibration procedure, and is fixed at aspect. By default, u is optimized.

-umin umin -umax umax

An optional argument that specifies the lower and upper search range boundaries for u during calibration. Default for -umin is 0.9 and -umax is 1.1.

-inmodel input_FMF_file

An optional argument that specifies an input file in Flare Model File Format. The -kernels, ‑rmin, and -rmax values are read from this file. If -inmodel is used in conjunction with ‑kernels, -rmin, or -rmax, the parameter settings override the input_FMF_file values.

-outmodel output_FMF_file

An optional argument that specifies an optional default output FMF file, which is constructed with the optimized values calculated by euv::fitflare. No FMF file is created if this option is not specified.

-verbosity level

An optional argument that specifies the command output message level. Can be 0, 1, or 2.

  • 0 – Only user errors are reported.

  • 1 – In addition to 0-level messages, the output string is printed out. This is the default.

  • 2 – In addition to 1-level messages, all warnings and intermediate results are reported.

Return Values

The euv::fitflare command returns the TCL string: 

-A1 A1 -A2 A2 -n1 n1 -n2 n2 -u u -TIS TIS -rrms rrms -maxerr maxerr

where A1, A2, n1, n2, and u are optimal values of the model, TIS is the TIS of analytical approximation, and rrms and maxerr are statistics of the model fitness.

If the -verbosity setting is larger than or equal to 1, then this string is also printed to the shell window.

Description

Converts a Flare Intensity File Format input to a Flare Model File Format output. This command is designed to be run as part of a Calibre WORKbench batch command file or from the shell. It runs a fitting algorithm on the input data.
