Recommendations for modelflow_v2 Starting Models

Configure the modelflow command scripts with these suggested starter settings:
Table 1. modelflow_v2 Starter Settings

Setting

(modelflow v2 flag)

Notes

Setup file (-f) or Model‑layer file (‑mlfile)

Find its defined models and input layers.

CM1/VEB models do not use the fragmentation scheme included with a standard setup file, which is the reason that model-layer files were created.

Input test pattern

(‑m or ‑l option)

Load the test pattern for calibration.

None.

Gauge file (-gf) or super gauge file (-sgf)

Compute the measured data for calibration.

  • Special rules apply to etch (VEB) data entry.

  • Super gauge files are used for process window modeling.

  • The gauge length of main pattern structures does not impact accuracy once it is long enough, but longer gauges can impact calibration runtime.

Remote file (‑remotefile)

Specify a remote machine configuration file. Used for parallel execution.

Optional, recommended for Stage 1, except when Stage 2 is also specified; in these cases, use -turbo instead.

Search algorithm (search)

Specify the type of mathematical search for model coefficients.

Six types (full, gradient, newton, front, zoom, and frontzoom) are available.

Frontzoom is the preferred search type for CM1; front is the preferred search method for VEB.

Runtime (runtime)

Set a maximum run time in hours when the front search type is selected.

Best used when trying to do initial coarse model fitting.

Threshold Tolerance (thresholdtolerance)

Set a percentage threshold tolerance value for gauges to resolve within; any gauges that fail to resolve within the range are penalized.

Recommended value: 1.5

Note:

Previous to 2014.4, the recommended setting was for the Threshold Stability field with a setting of 4.

Max AI Diff (maxaidiff)

Constrain the calibration algorithm to limit the percentage difference between CM1 and aerial image surface to be less than the specified value.

Recommended value: 10 stable

Notes

Kernel selection

  • A maximum of six kernels can be used in the VEB model. Owing to runtime performance concerns, it is recommended that no more than one visible kernel in either direction (looking in or looking outwards) be used.

    Note that only 4 kernels are presently supported using VEB modelforms; using more than 4 kernels requires customization of the VEB model.

  • Use the least number of kernels required; 1 gauss and 1 visible kernel are usually sufficient for most 65 nm test cases.

Diffusion length

  • Use a value no greater than 600nm for visible kernels, since this will have an adverse effect on Calibre OPCverify and nmOPC operations.

  • Use a value no smaller than 35nm for DUV Gaussian kernels; specifically, the es parameter needs to be 35 or larger. For EUV kernels, the es parameter lower limit is 15nm or larger.
