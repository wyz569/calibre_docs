Calibrating CTR Models With SRAF Gauges

Use modelflow_v2 Stage 1 to create and calibrate a CTR model for use with SRAF test gauges.

An SRAF test gauge file contains a set of SRAFs, some of which are designed to print for testing purposes (true SRAFs are never intended to print).

Prerequisites

  • A gauge file containing gauges centered on the SRAF structures.

  • The corresponding test pattern design file loaded in Calibre WORKbench.

  • The relevant optical model file, which ideally should have an HHA and DDM model included, along with a mask model if available.

  • A generic CTR resist model.

  • A model-layer file for use with CM1 Center, with links to the CTR model and optical model.

Procedure

  1. In Calibre WORKbench, open CM1 Center.
  2. If you created an SRAF gauge file using the task “Creating an SRAF Gauge File Using SEM Images”, skip to step 6.
  3. Load the SRAF gauge file into CM1 Center. Disable any non-SRAF structures for this calibration run by selecting the structures and setting their Enable field to 0.
  4. Set the gauge file Meas field. Use LT:0 for not printing, and GT:0 for printing SRAFs. Note that this assumes a table has been prepared by other means that shows which SRAFs print or not print on the wafer.
  5. Save the SRAF gauge file with a different name (use the Gauges > Save as Super Gauge menu option) and close CM1 Center.
  6. Create a modelflow_v2 script, similar to the one listed in the Examples section, including the following arguments:
    Table 1. CTR Calibration for SRAFs Suggested Arguments

    Argument

    Description

    Input section

    -mlf
    Note:

    Ensure that the model-layer file has default values for the vertical_diffusion parameter in the optical model.

    -remotestring remotefile

    Executes MTflex during the modelflow_v2 run.

    -l layout_handle

    Specifies the design file layout handle.

    -sgf super_gauge_filename

    Loads the specified super gauge data file. Specify the file from step 4 (or step 2 if you ran createsrafgauges).

    -objective errrate

    Changes the calibration objective mode to an overall error rate criteria:

    overall error rate = (# of false positive + # of false negative) / (total # of SRAF gauges)

    -sraferrweight weight

    Modifies the -objective errrate argument to use weight multipliers for SRAF gauges:

    weighted error rate = (1-weight) * false positive error rate + weight * false negative error rate

    Note that if you set weight to 1.0, false positives will have a weight consideration of 0.

    -srafcalbyweight

    Changes the calibration weighting to use the gauge weight for each gauge data item in SRAF prediction printing error rate calculations.

    Output section

    log filename

    Saves the run log to the specified file.

    optical directory_name

    Saves the calibrated optical model to the specified directory for later use.

    stage1resist filename

    Saves the accompanying resist model to the specified filename for later use.

    lithomodel directory_name

    Outputs a litho model to the specified directory.

    stage1 section

    stage1 threshold optimize

    Selects optimization for the image threshold.

    search front

    Selects the front search algorithm.

    explore value

    Sets the number of explorations for the search. The value used depends on the number of variables to be optimized and the range; 100 per variable is reasonable. Use a value of 200 when optimizing def_start and beamfocus, and 300 if you also optimize vd_length.

    epsilon 0.01

    Sets the convergence criterion for the search.

    def_start 0.0 0.0105 10

    Sets optimization for the defocus start parameter1.

    beamfocus 0.0 0.105 10

    Sets optimization for the beamfocus parameter.1

    vd_length optim_set

    Selects optimization for the vertical diffusion length (optional; see the Tip following the table).

    dense

    Selects dense mode optimization.

    stage2 section

    off

    Turns off resist model optimization. The CTR model in the model-layer file is used for resist calculations.
    Tip

    As a best practice, Siemens EDA suggests calibrating as a two-stage process: first, calibrate def_start and beamfocus. (Note that through-focus gauges are needed to calibrate beamfocus. However, if beamfocus has been previously calibrated through the bossung_adjust procedure, it does not need to be calibrated again.)

    Second, try a calibration with def_start, beamfocus, and vd_length, and compare the models. This is because vertical diffusion takes longer, but does not guarantee better results.

  7. Run the modelflow_v2 script.
    	calibrewb script.tcl
  8. Examine the resulting log by loading it in CM1 Center and using the Analyze functionality. In the log file, look for the section titled “SRAF Analysis.”
    Figure 1. Analyze Logs From CM1 Center

    • You can visualize how far away the SRAFs are from the printing threshold by using the Analyze dialog.

    • You can navigate to a gauge of interest by highlighting the gauge and finding it in the CM1 Center Gauges tab.

    • You can visualize the trade-off between false positive and false negative error rates by varying the final threshold.

  9. If you are satisfied with the results, use the resulting litho model as the final model.

Results

The output log contains the SRAF error statistics for the gauges that contained inequalities. Acceptable results are a false negative error of zero and a false positive error rate of 10% or less. You can click the View button to see these results.

Figure 2. Viewing SRAF Error Statistics

Simulated CDs for SRAFs have a different meaning depending on the configuration of the line versus space designation (the Loc field), the background and mask values, and the designation in the Meas field (LT:0 for not printing, and GT:0 for printing SRAFs).

  • An LT:0 (not printing) SRAF and a Simul field value of -1 indicate that no false positive error exists for this SRAF.

  • A GT:0 (printing) SRAF and a Simul field value of less than 0 indicate a false negative error exists for this SRAF. The Simul field contains a value given by a penalty term that depends on how far the SRAF is from printing.

    This penalty value is not applied as part of the error rate calculation, but it is used when the objective selected for calibration is the CD RMS error type.

    Larger absolute values indicate an SRAF that is further away from printing.

  • A GT:0 (printing) SRAF and a Simul field value that is positive indicate a true positive error.

  • A GT:0 (printing SRAF and a Simul field of exactly 1 indicate a true positive error, but also indicates that the threshold exceeded the max intensity, and the SRAF may have merged with a main feature. Check the SRAFs by using the Cutline tool to understand the issue.

  • An LT:0 (not printing) SRAF and a positive Simul field value that is positive indicate a false positive error.

Examples

The following code is part of an example script for CTR calibration:

modelflow_v2    \
  -remotestring $rmstr \
  -l $L \
  -sgf gauges/sraf_all_ineq_pw5.sgd \
  -mlstring $mstring \
  -objective errrate \
  -sraferrweight 1.0 \
  output \
    log logs/stage1_ctr/cal_beamfocus_5.log \
  stage1 \
    threshold optimize \
    search front \
    explore 200 \
    epsilon 0.001 \
    def_start 0.0 0.105 10 \
    beamfocus 0.0 0.105 10 \
    dense \
  stage2off
