Generic Test Pattern Features

This section describes generic test structures.

These are structures that you can add to test patterns you create using the batch command tepCreate.

rect_h_array

User control over:

  • Ewidths

  • Eheights

  • Spaces

  • Counts

  • Titles

rect_v_array

User control over:

  • Ewidths

  • Eheights

  • Spaces

  • Counts

  • Titles

rect_matrix

User control over:

  • Rows

  • Cols

  • Eheights

  • Ewidths

  • Vspaces

  • Hspaces

rect_h_sandwich

User control over:

  • Layers

  • Spaces

  • Titles

  • Ethicks

  • Espaces

Notes on Using rect_h_sandwich and rect_v_sandwich

The rect_h_sandwich and rect_v_sandwich structures both use the same control arguments to create custom patterns in a sandwich style, using the following rules:

  • The E-type arguments are required.

  • Up to three different layer types can be defined:

    • P-type and Q-type arguments override the default E-type arguments for a sandwich layer.

    • P-type layers are above E-type layers. If no P-type arguments are specified, the matching E-type values are used instead.

    • Q-type layers are below E-type layers. If no Q-type arguments are specified, the matching P-type values are used instead (and therefore inherit E-type values if no corresponding P-type value was specified).

    • By default, the layers are in sets of three (PEQ), with extra layers stacking upwards starting with an extra E layer. Increasing the Counts arguments (Ecounts, Pcounts, and Qcounts) changes the number of columns (rect_h_sandwich) or rows (rect_v_sandwich) of that layer type.
