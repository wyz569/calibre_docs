sgd_resize

CM1 and VEB Commands

Related Formats: Gauge Object Reference, Super Gauge Data File Format

Resizes the length of a specified gauge or all gauges of the input super gauge data object.

Usage

sgd_resize object set_index gauge_index {–extra integer | –scale value}

Arguments

object

An optional argument that specifies an input gauge or super gauge data file.

set_index

A required argument specifying the required index of a gaugedata set within the sgd object. Accepted values for set_index are the following:

  • integer — Resizes gauges in the specified gauge set. The index specified must be between 1 and the number of gauge files in the super gauge object.

  • end — Resizes gauges in the last gaugedata set.

  • all — Resizes gauges in all gaugedata sets.

gauge_index

A required argument specifying the index of the gauge to be modified. Accepted values for gauge_index are:

  • integer — Resizes the gauge specified by this index number which must be between 0 and the size of the specified gaugedata set-1.

  • end — Resizes the last gauge of the specified gaugedata set.

  • all — Resizes all gauges of the specified gaugedata set.

One of the mutually exclusive -extra or -scale arguments is required:

–extra integer

Increases (integer > 0) or decreases (integer < 0) the gauge length by 2*integer in database units.

–scale value

Rescales the gauge length by the specified amount. The coordinates of the end gauge points are changed so that the new gauge length is equal to the old one multiplied by value.

Examples

# resize all gauges by 200 dbu
sgd_resize $obj all all –extra 200
# rescale gauge No 18 of gaugedata set 2 by a factor of 1.3
sgd_resize $obj 2 18 –scale 1.3
