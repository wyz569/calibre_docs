modelflow_v2: Stage 3 Etch Model Tuning Options

Parent topic: modelflow_v2

Note:

The “stage3” parameter starts the stage3 block and is followed by the stage3 arguments.

Arguments

The options related to the optimization method of stage3 as well as the syntax of expressions used to specify optimization parameters and search intervals are identical to those of stage1.

search {full | gradient | newton | front}

For descriptions of the search options, see search {full | gradient | newton | front | zoom | frontzoom} in “modelflow_v2: Stage 1 Optimization Run Modes”.

explore integer

For descriptions of the explore argument, see explore integer and search {full | gradient | newton | front | zoom | frontzoom} in “modelflow_v2: Stage 1 Optimization Run Modes”.

epsilon value

For descriptions of the epsilon argument, see epsilon value and search {full | gradient | newton | front | zoom | frontzoom} in “modelflow_v2: Stage 1 Optimization Run Modes”.

Also, the norm values of all etch optimization parameters (for norm definition, see comments on the epsilon option of stage1) are set to 100 nm.

try

An optional flag that uses the input VEB model without fitting it to empirical data. It cannot be used if any modelflow_v2: Stage 3 Optimization Parameters or 0 options are specified. It is designed for use with the inputlayer argument.

biasing_type 1{0 [cdtol]} | 1 | {auto [cdtol]}

An optional flag that specifies the way the VEB bias should be computed. This option replaces the force1D option.

  • 0 — Instructs modelflow_v2 to compute the VEB bias value at a particular point of the resist contour based on the densities at that point, without actually moving the resist contour into the final position of the etch contour.

    biasing_type 0 is the default starting with the 2016.4 release. In the 2016.1 to 2016.3 release, biasing_type 1 was the default.

    stage3 try and biasing_type 0 cannot be specified simultaneously. Also, if biasing_type 0 is set, the VEB model is tuned using fast approximate simulations, while the final CDs and error statistics are computed accurately without this approximation.

  • 1 — Instructs modelflow_v2 to perform contour movement from the resist to the final etch contour position before measuring the displacement.

  • auto — Instructs modelflow_v2 to automatically detect which gauges should receive biasing_type 1 and which biasing_type 0 treatment. The decision is made by comparing simulated etch CDs. If the CD difference is greater than cdtol, biasing_type 1 is used.

  • cdtol — An optional value in nanometers that can be specified with 0 or auto. If not specified, the smaller of 4 dbu or 0.5 nm is used.

Results obtained with biasing_type 0 and biasing_type 1 can differ. In the process of contour movement, the contour may intersect with itself and form a “swallow tail” or a loop. The latter can happen for 2D gauges only if the contour shrinks (moves inward) and the bias value is large enough.

Tip

For a suggested best practice for using the biasing_type 0 with the selective_biasing_type1 and structs_biasing_type1 arguments, see “VEB Calibration Runtime Improvements.”

selective_biasing_type1 filter_layer_number

An optional argument that can only be specified with biasing_type 0. It marks gauges that are to be treated more accurately than the approximation applied by the biasing_type 0 option. The input value is the layer number to be used as a filter. If a gauge is covered by the filter, it is treated accurately, as if the option biasing_type 1 were set. If a gauge is not covered by the filter, the approximation used by biasing_type 0 is performed.

(Default is to treat all gauges equally based on the input biasing_type value).

structs_biasing_type1 struct_list

An optional argument that can only be specified with biasing_type 0. It marks by structure name those gauges that are to be treated accurately without using the approximation underlying the biasing_type 0 option. The input argument is a comma-separated list without spaces of strings specifying the gauge struct name templates to be used as a filter. If a gauge structure name contains one of the strings from the list, it is treated accurately, as if the option biasing_type 1 were set. If a gauge structure name does not match the filter, the approximation underlying biasing_type 0 is used.

For example, specifying “structs_biasing_type1 2D,T2S” selects gauges that contain “2D” and “T2S” inside their struct name.

(Default is to treat all gauges equally based on the input biasing_type value).

etch_underlying veb_underlying_index layer_number

An optional argument used only for associating the underlying kernels in a multilayer etch model with corresponding layers in the input layout. The veb_underlying_index is a sequential index used to specify an underlying kernel (only 1 is allowed in this release) in the nominal etch model.

The total number of stage3 etch_underlying specifications must match the number of underlying kernels in the nominal etch model.

layer_number is an integer specifying a layer number in the input layout to associate with the veb_underlying_number. The layer should be defined in the model-layer file (described in “Model-Layer File Format”) using the etch_underlying keyword.

fit {relative | absolute}

An optional flag that specifies the objective function used to optimize the etch model (Default = absolute):

  • relative means that the etch model parameters are to be determined by minimizing the difference between simulated and measured etch bias. The latter is determined as the difference between etch and resist CD, namely: CDetch – CDresist.

    This option cannot be used if the inputlayer argument is also specified.

  • absolute means that the etch model is to be fit by minimizing the difference between simulated and measured etch CD data; in other words, the model is fit directly to CD data rather than to bias data.

inputlayer [layernumber] [cli]

An optional flag that includes supplied resist contours in its VEB calibration calculations. They can be found on:

  • the specified layernumber

  • using the specified contour layer information (CLI) file specified in the -clifile or ‑clistring input arguments

  • both the specified layer number and CLI file if layernumber cli is specified

    Neither a resist nor an optical model are used in this case, but dummy files must be supplied in order to prevent parsing errors if the -f (setup file) input option is specified. However, if the -mlfile or -mlstring (model-layer file) options are specified, the model-layer file does not require a specified resist or optical model, since it assumes a direct VEB model.

Resist contours must be polygons. The resist contour layer you specify has its information added to the gauge information for calculation purposes.

Resist contours should be available for all features crossed by gauges if you use this option.

Explicit Mode

esn optim_set

diffusion length in nm of n-th resist density, 0 esn, n = 1, 2, 3, or 4. (Default = use nominal model setting.)

un optim_set

shift in nm of n-th resist density (Dn term in the etch model), n = 1, 2, 3, or 4. (Default = use nominal model setting.)

Similar to stage 2, the stage3 option can also be used without specifying the optimization method and parameters. If no options are specified, only linear coefficients of the input etch model are optimized, while the rest of the model parameters (shifts and diffusion lengths) are kept fixed at their nominal values.

Description

Stage 3 is designed for fitting the etch model parameters to the input empirical data set. It can be set only if the setup file contains the keyword vetchbiasfile, which specifies the path to the input (nominal) etch model:

vetchbiasfile etch.mod

Etch model fitting is off by default and, accordingly, the nominal etch model is used to calculate the final CD DIFF RMS fitness if stage 3 is not specified. There is no weight or filter option available for stage 3; all gauge data present in the input gauge file are used as is without any changes. If stage1 (optical) or stage 2 (resist) are present, the best model(s) resulting from the tuning of those stages will be used to tune the etch model. Otherwise, the nominal optical or resist models specified by the setup file will be employed.
