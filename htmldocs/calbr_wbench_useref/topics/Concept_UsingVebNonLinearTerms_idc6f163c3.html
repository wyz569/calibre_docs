Using VEB Non-Linear Terms

Non-linear terms are helpful in improving the VEB model fitting power below 45 nm.

Using non-linear terms allows you to fit etch bias data with Gaussian kernels only. (Using linear terms requires visible kernels and Gaussian kernels to fit etch data.) Using only Gaussian kernels results in improved runtime when you use the resulting models with Calibre OPCverify.

BTERMs have one of the syntax forms described in Table 1:

Table 1. VEB Linear and Non-Linear Kernel Syntax Definitions

BTERM Format

Description

Example

ci Dn^1

Linear term.

A polynomial that only includes linear terms is considered a linear polynomial.

BTERM 50 D2^1

ci Dn^p

(where p is between 2 and 4 )

Non-linear term.

A polynomial that includes any non-linear terms is considered a non-linear polynomial.

BTERM 0 D1^3

ci Dn^p*Dm^q

(n and m cannot be equal; p and q can be any value between 2 and 4, and can be equal)

Non-linear cross term, which multiplies two kernels together. Each can be raised to an integer power.

A polynomial that includes any non-linear terms is considered a non-linear polynomial.

BTERM 30 D1^2*D2^3

ci Dn^p*Dm^q*Dl^r

(n, m and l cannot be equal; p, q and r can be any value between 1 and 4, and can be equal)

Non-linear cross term, which multiplies three kernels together. Each can be raised to an integer power.

A polynomial that includes any non-linear terms is considered a non-linear polynomial.

BTERM 5 D1^1*D2^1*D3^1

ci Dn^p*Dm^q*Dl^r*Dk^s

(n, m, l and k cannot be equal; p, q, r and s can be any value between 1 and 4, and can be equal)

Non-linear cross term, which multiplies four kernels together. Each can be raised to an integer power.

A polynomial that includes any non-linear terms is considered a non-linear polynomial.

BTERM 5 D1^1*D2^1*D3^1*D4^1

Bias polynomials with linear-only terms have the following format:

bias = c0 + c1 * D1 + c2 * D2

Bias polynomials with non-linear terms and no cross terms would look similar to:

bias = c0 + c1 * D1 + c2 * D2 + c3 * D1^2 + c4 * D2^2

Bias polynomials with non-linear terms and cross terms would look similar to:

bias = c0 + c1 * D1 + c2 * D2 + c3 * D1^2 + c4 * D2^2 + c5 * (D1^2)*(D2^2)

You define bias polynomials in the following ways:

  • As part of a modelflow_v2 command:

    1. Define the model in a separate file, as described in “Variable Etch Bias (VEB) Model File Format”.

    2. Designate the file as the argument to the vetchbiasfile keyword in the setup file or model-layer file.

    3. Use the setup or model-layer file as input to the modelflow_v2 command.

  • In CM1 Center, in the VEB tab. The Expression field shown in Figure 1 is an editable text field.
