modelflow_v2: Stage 2 Resist Model Tuning Parameters

Parent topic: modelflow_v2

This section specifies resist model optimization settings for the modelflow_v2 command.

Note:

The “stage2” parameter starts the stage2 block and is followed by the stage2 arguments.

Arguments

stage2off

Optimizes the optical model (stage1) without running stage2 calculations.

stage2

Fits the resist model parameters to the empirical data set.

try

Tells the tool to use the input resist model as is without fitting it to empirical data (also known as the “Try” method).

The options related to the optimization method of stage2 as well as the syntax of expressions used to specify optimization parameters and search interval are identical to those established in stage1:

search {full | gradient | newton | front | zoom | frontzoom}
explore integer
epsilon value
Note:

The epsilon argument in stage2 is different from the epsilon argument in stage1 in that the stage2 version can be used without specifying any other stage2 optimization parameters. This causes modelflow_v2 to optimize only threshold and linear coefficients, while the other model parameters are kept fixed at nominal values. Norm values are described in the epsilon argument in stage1. Stage2 uses the following norm values:

  • The norm values of all resist optimization parameters except for diffusion length parameters are set to 1.
  • Diffusion length parameters (such as sn) have their norm values set to 100nm.
brelative

An optional argument that specifies that the begin and end values of the CM1 b term search intervals are multiplied by the clearfield intensity. (Default is to use the input search intervals.)

dense

Optionally uses a dense optical routine to generate aerial image grids for stage 2 calibration. The dense engine uses a square coarse (at the Nyquist rate) aerial image grid over the area of interest and interpolates the grid intensity at the specified control points. It operates at a linear speed, and can be faster than the default sparse engine for large numbers of gauges (> 1000). As of version 2011.1, this option is recommended unless you have only 1D gauges in your gauge data file.

Note:

In CM1 Center only, selecting stage2 (with the CM1 “O”, “B”, or “T” buttons) automatically outputs “stage2 dense” to the output, which makes dense the default mode for CM1 Center.

Note:

If a flare model is involved, the options stage1 dense and stage2 dense should be specified. If stage1 is not required it can be skipped. If stage2 is not required, you will still need to specify stage2 dense try explicitly.

horizontal | horizontaloff

Specifying the optional argument horizontaloff deactivates horizontal calibration for linear coefficients and activates vertical calibration, which was the default behavior previous to 2013.4. The default from 2013.4 onwards is horizontal.

nolp

An optional argument that turns off linear programming during CM1 calibration. Linear programming is only used with ‑objective tolerance1. Turning off linear programming does improve performance, but reduces accuracy.

snap2fftoff

An optional argument that turns off the snapping of the size of AI grids to good FFT sizes. Grid snapping was added as the default behavior for modelflow_v2 in the 2016.2 Calibre WORKbench release to better match Calibre nmModelflow results. Use this option to revert to the pre-2016.2 behavior.

extentoff

An optional argument that turns off the forcing of tile sizes to be the same for modelflow_v2 and Calibre nmModelflow. The forcing was added as the default behavior in the 2016.2 Calibre WORKbench release; use the extentoff option to restore the pre-2016.2 behavior.

useevalcache

An optional argument that caches parameter and objective values during optimization. Caching was turned off by default in the 2016.2 Calibre WORKbench release in order to synchronize modelflow_v2 and Calibre nmModelflow results. Use the useevalcache option to restore the pre-2016.2 behavior.

airesistdifference
Note:

This option was named aicm1difference previous to 2009.1.

Optionally writes the difference between the CM1 and aerial image surfaces to the log file at each iteration. Cannot be used with the -turbo, -remotefile, and -remotestring options. (Default: write the difference only once after calibration completes.)

This option slows down the simulation.

maxlinear value

Use this option to run CM1 calibration with additional constraints. If maxlinear is set, then during the linear coefficient optimization, the aerial image coefficient is set to 1. The sum of absolute values of all other coefficients will not exceed value. This option cannot be used with models that do not have a pure aerial image term.

The value must be greater than or equal to zero. The anchor term linear coefficient is set to 1, and if the anchor term is not CTR it is internally converted.

Maxlinear cannot be used with maxaidiff or if the optimization set involves linear CM1 parameters.

maxaidiff value [loose | stable]

Use this option to run CM1 calibration with the additional constraint that the percentage difference between CM1 and aerial image surfaces must be less than the specified non-negative value. Do not specify a value of 0, as it will cause CM1 Center to fit for a CTR model. (Recommended value: 10 percent)

This option is designed to minimize the presence of SRAF printing in the final resist model, which should be achieved when it differs from the CTR by 10 percent or less.

Additional qualifiers can be specified for this argument:

  • loose — Filters out CM1 models that do not meet the maxaidiff constraint after calibration. This was the default setting previous to the 2014.3 release.

  • stable — Forces the tool to generate only those candidates that meet the maxaidiff constraint during CM1 model building. If none of the candidates satisfies the constraint, the model building function returns the anchor term. The tool internally switches the anchor term to a CTR term if it is not a CTR term. At least one CTR term (b=0, s=0, p=0, k=0, n=1, and so on, with b and s remaining constant during calibration) must be present in the model definition. In other words:

    • If the user-defined anchor term is a CTR term, then the user requirement for a CTR term is satisfied.

    • If the user-defined anchor term is not a CTR term, selecting “stable” automatically switches it to a CTR term, but the user-defined model must also include a CTR term as one of the non-anchor terms in order to meet the user-defined CTR requirement.

    The default setting is “stable”, starting with the 2014.3 release. However, “stable” is not compatible with modelforms 1, 2, 3, and 31 when you set them while also using the cm1mf argument. If your modelform_v2 run has this combination, modelflow_v2 issues a warning and changes the setting to “loose.”

thresholdstability value
Note:

Thresholdstability has been deprecated starting with the 2014.4 release.

Optionally sets a percentage threshold tolerance where all geometries crossed by gauges are supposed to remain resolved without printing extra features or holes. If set, it tells the tool to penalize, within the specified range of threshold values, any gauge that fails to resolve at all, or which resolves with printing extra features that were not intended for printing. A “penalized gauge” has its associated simulated CD set to zero so that the contribution of the penalized gauge to the total RMS error becomes proportional to the CDmeas squared.

If thresholdstability is set, the tool computes constrained and unconstrained error data. If the final sets of error statistics differ, the tool reports both the constrained and unconstrained error data in the log file. To reduce the error difference, rerun the calibration with larger thresholdstability values.

This option is designed for gauge-based model calibration and has no impact on contour-based calibration, which involves a small number of dummy gauges.

This option only penalizes the gauges that do not print precisely at the threshold. (Recommended value if thresholdtolerance is not used: 4 percent)

The thresholdstability option supports any search method, and directly verifies the model stability. It should be used in combination with maxaidiff of 10 percent. If thresholdstability is used by itself, it may result in a higher maxaidiff value.

This option cannot be specified if thresholdtolerance is specified.

Note:

Previous to version 2014.4, stage2 thresholdstability 4 was the default in the CM1 Center GUI (as the Threshold Stability field in the Optimize tab, CM1 subtab). Starting with version 2014.4, thresholdtolerance 1.5 is the default in the CM1 Center GUI (as the Threshold Tolerance field in the Optimize tab, CM1 subtab).

thresholdtolerance value

Optionally sets the percentage for threshold tolerance, a nonnegative quantity for the range of threshold values over which equality-constrained gauges are supposed to be resolved correctly. If set, modelflow_v2 computes simulated CDs not only at a specific threshold (th) but also at two more values, (th - delta) and (th + delta), where delta = 0.01 * value * th. The gauge is considered to be resolved correctly if it prints at all of the three threshold values, and if the simulated CD remains a linear function of the threshold within the specified tolerance. If the latter is not the case, the gauge is penalized by setting the associated simulated CD to zero, such that the contribution of this gauge to the total RMS error becomes proportional to CDmeas squared.

This option is a more robust alternative to thresholdstability. Both are designed to filter out unstable models during calibration, but the implementation of these options is different.

If thresholdtolerance is set, the tool computes constrained and unconstrained error data. If the final sets of error statistics differ, the tool reports both the constrained and unconstrained error data in the log file. To reduce the error difference, rerun the calibration with larger thresholdtolerance values.

This option cannot be set if the thresholdstability is specified. The recommended value starting with the 2014.4 release is 1.5.

Description

Stage 2 is used to fit the resist part of the model to the input empirical data set. It has the following usage notes:

  • This fitting is off by default and, accordingly, the nominal resist model is used to calculate the final CD DIFF RMS fitness if stage2 is not specified.

  • There is no weight or filter option available for stage2. All gauge data present in the input gauge file are used ‘as is’ without any changes.

  • If stage1 is present, the best optical model resulting from optical tuning will be used to tune the resist model, otherwise the nominal optical model will be employed.

  • stage2 cannot be set if contour-based VEB calibration is set using the stage3 inputlayer argument.

  • If stage2 is active, you should use the -turbo argument instead of the -remotefile argument.

Definitions

  • CTR — Constant Threshold Resist Model of type CM1.

  • Nominal optical, resist, and etch models are the initial models specified in the setup file.

  • optim_set — Defined as one of the optimization expressions as described in Using the Independent, Constrained, and Dependent Optimization Parameters of “modelflow_v2: Stage 1 Optimization Run Modes”.
