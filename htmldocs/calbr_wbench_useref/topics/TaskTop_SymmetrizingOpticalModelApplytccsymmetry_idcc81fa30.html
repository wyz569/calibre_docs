Symmetrizing an Optical Model With applyTCCsymmetry

As an alternative to using the Source View tool to explicitly symmetrize a source map, you can specify it during optical model generation using a setting in the optical model definition file.

By using the applyTCCsymmetry command, everything that could break the model symmetry is converted, including the source, Jones pupil data, and lens aberrations.

Prerequisites

  • An existing optical model definition file

Procedure

  1. Open the optical model in either the Optical Model Tool or in a text editor. If you are using the Optical Model Tool, switch to the Misc tab.
  2. Add one of the following lines to the optical model definition file:
    	applyTCCsymmetry xy 
    	applyTCCsymmetry xydiag

    • If xy is selected, the TCC values from positions (i,j,m,n), (-i,j,-m,n), (i,-j,m,-n) and (‑i,‑j,‑m,‑n) are averaged resulting in a model that exhibits identical symmetry with a model resulting from an x-dipole source or a y-dipole source.

    • If xydiag is selected, the TCC values from positions (i,j,m,n), (-i,j,-m,n), (i,-j,m,-n), (‑i,‑j,‑m,‑n), (j,i,n,m), (-j,i,-n,m), (j,-i,n,-m), and (-j,-i,-n,- m) are averaged resulting in a model that exhibits identical symmetry with a model from a standard source or an annular source or a QUASAR source or a QUAD source with 0 or 45 degree rotation.

  3. Save the file and generate the optical model as normal.

Results

When the optical model is generated, optical symmetry will be enforced based on the setting you specified.
