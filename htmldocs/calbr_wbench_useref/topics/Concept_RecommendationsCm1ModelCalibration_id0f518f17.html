Recommendations for CM1 Model Calibration

Consider the following best practices when optimizing a CM1 model:

  • Tuning imagediffusion with CM1 models is generally not recommended. In some cases, however, tuning imagediffusion from 0-20 nm (0 to 0.02) can significantly improve accuracy.

    The benefit gained from tuning imagediffusion should be compared to the chosen CM1 model form, not only the CTR model. This is because the CTR fit error with imagediffusion in the optical model will almost always be better than without imagediffusion and only the calibrated CM1 model represents the fitness change.

  • Using the cm1mf integer command in modelflow_v2 lets you implicitly define an initial CM1 modelform for calibration without having to explicitly set modelform parameters. Setting the cm1mf sinterval or binterval start or end arguments to “auto” uses default values.

  • Use modelform 21 or 22 for the best balance between data fitting and model approximation, except when negative tone develop (NTD) layers are part of your process; multiple modelforms for NTD are available.

  • Use modelform 58 in most cases for all layers employing NTD resist, because these layers are typically bright field metal and contact layers at 28nm and below.

  • Optimize “b” parameters with automatic limits and 10 levels (“auto auto 10”). This sets the lower limit of the -b parameter to be equal to the maximum Imin in the dataset plus 0.1 and the lower limit of the +b parameter equal to zero. The upper limit of all b parameters is set to be the lower limit of b plus 0.3.

  • If a model has non linear parameters (b or s) at the lower limit of the parameter space after calibration, this is considered a successful calibration result. The lower limits guarantee model consistency for OPC.

  • If a final model has non linear parameters (b or s) at the upper limit of the parameter space after calibration, Siemens EDA recommends a local optimization search (zoom) with extended parameter limits to identify the model with best objective value.

    Use the previously calibrated CM1 model as input to a new zoom search. Set the new parameter limits as listed following and use a zoom search with an explore setting of 0 (to start the search at the location of the previous best model).

    • Extend the local search upper limit of +/-b = max(Imax) according to the image parameter space (IPS).

      Note:

      The Calibre IPSE tool is helpful in viewing the image parameter space values for a model. See “Using Calibre IPSE” for more information.

      You can also view the IPS in the Calibre nmModelflow Calibration Job Record, under the GaugeSimStats Ctrlmax vs Ctrlmin graph.

    • Extend the local search upper limit of s = 300nm.

  • Optimize “s” parameters from “auto” (automatic lower limit) to 200 with 10 levels, except when k =1. Optimize “s” parameters from 25 to 200 when k=1. The “auto” setting indicates that the lower limit of the “s” parameter is defined using 3*simulation pixel dimensions.

  • Use the defaults for the CM1 parameters ANCHOR (0), SMOOTH_NEUTRALIZATION (2, which is the new default as of 2012.3), and SMOOTH_GRADIENT (2, which is the default as of 2012.2).

  • It is not recommended to change the value of COEFMAX.

    Note:

    Starting with the 2018.2 release, the default value of COEFMAX was updated to 1.2E-4. Models created and calibrated in the 2018.2 release will produce different results than models created in an earlier release and calibrated in the 2018.2 release. To produce a calibrated model that uses the previous value of COEFMAX, create the model in a previous release.

  • For stage2 calibration, use the Frontzoom algorithm as follows:

    • Simple modelform (21, 22): 20000 explorations.

    • Complex PTD modelform (120): 25000 explorations.

    • NTD (modelforms 38 to 48): 40000 explorations. If greater accuracy is required, create a predecessor job with the gagradient search algorithm. The gagradient search algorithm is available only as a batch command. See “Model Calibration With the Genetic Algorithm” in the Calibre nmModelflow User’s and Reference Manual for more information.

    • Sidewall bias (modelforms 58 and 78): 60000 explorations.

  • Do not use MTflex (-remotefile option) when you are calibrating CM1 models (stage2 is active); use MT (-turbo option) instead.

  • Using the -foptions parallel option improves scalability but may give slightly different calibration results depending on the number of remotes and CPUs used (a warning message to this effect is generated). This is normal behavior.

  • For CM1 calibration, use the stage2 horizontal (default setting in CM1 Center as of 2013.2) and maxaidiff 10 stable options to achieve the best RMS value in most cases.

  • Optimize the shrink threshold (“shrink_threshold0”,…,”shrink_threshold3”) parameters with a lower limit of 0.2 when using modelforms with a horizontal shrink term.

  • Optimize the horizontal bias d (“hbias_d0”,...,”hbias_d3”) parameter when using modelforms with a horizontal development bias term.
