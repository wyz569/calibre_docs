Setting Optimization for the VT5 Model

This procedure describes how to specify in VT5 Center what parts of the VT5 resist model calculations to optimize. An optimized resist model is the best-fit polynomial (shown as field coefficients in the VT5, 3 tab) describing the behavior of the measured process data.

Prerequisites

  • Completed the task “Defining a VT5 Input Model”

    The modelform you selected in Step 4 affects what you can optimize.

  • Sample data spreadsheet file

Procedure

  1. Click the VT5, in tab and record items that appear in the following tabs:

    • Kernels — Convolution kernels

    • TTerms — Threshold polynomials

    • BTerms — Bias polynomials

  2. In VT5 Center, click the VT5, 3 tab.
    Figure 1. VT5 Center GUI (VT5, 3 Tab)

  3. Select one or more Dn terms if you are optimizing a modelform that contains Gaussian density kernels. The Dn terms are shown in the Kernels, TTerms, or BTerms subtabs as one or more Dn terms. The following rules apply:

    • You can optimize up to four Gaussian kernels during any single run. If you need to optimize more than four, you can make multiple VT5 Center runs.

    • You cannot optimize non-Gaussian kernels.

      Note:

      The kernel polynomial experiments require either a threshold (step 4) or bias polynomial (step 5) to be evaluated with them. The reverse is not required; thresholds and bias polynomials can be evaluated with the default convolution kernel.

      Note:

      In the VT5 tab, the Dm, Dn, Dk, and Dj sigmas are shown as D1, D2, D3, and D4 by default, but these values are changeable to reference kernels other than 1, 2, 3, and 4, respectively.

    For each unique trial value of the Dm, Dn, Dk, and Dj sigmas specified, VT5 Center re-optimizes the threshold and bias coefficients. The tool then chooses best overall density kernel values based on EPE error RMS criteria.

    Regarding the search over the density kernel ranges:

    1. If no Dm range is given, then Dm is completely untouched from the nominal model.
    2. Otherwise, the Dm kernel is changed to a Gaussian model, overriding the previous kernel.

    Inside the loop that searches the density kernel parameters, VT5 Center re-optimizes the VT5 polynomial coefficients so that the best VT5 solution is found for each setting of (Dm, Dn, Dk, and Dj).

    • Select a kernel from the Density column to optimize that kernel in the model. Note that if you choose to optimize more than one Gaussian density kernel, the Density values cannot overlap (the first density kernel maximum must be smaller than the second density kernel minimum, and so on).

    • In the Type column, set the kernel type according to the Kernel tab in the VT5, in tab:

      • CKERNEL (convolution kernel) — choose gauss

      • DKERNEL (directional kernel) — choose gaussm for gauss-minus, gaussp for gauss-plus

    • Use the Min, Max, and Step columns to specify the variation range and step increments to test. Note that the first density kernel minimum must be greater than 0.

    • Select Check Reference Threshold to stop the evaluation if the reference threshold does not pass through the aerial image in this step. If not selected, VT5 Center passes sites that do not cross a reference threshold.

  4. If you are optimizing threshold polynomials (at least one coefficient was present in the TTerms tab), you can set the following options in the Threshold Polynomial section:

    • Selecting Copy of Reference Threshold forces the threshold polynomial to a constant threshold whose value is equal to the reference threshold. This corresponds to the CTR model.

    • Selecting Optimize Variable Threshold enables crosschecking (Cross Check / No Cross Check radio buttons) and calculates a constant offset (epeBias) that delivers the best threshold polynomial fit.

      Crosschecking involves splitting the data in the input spreadsheet into two data sets that can be used to check for a stable model. If both sets converge to the same eigenvalue the VT5 model is stable. If they do not converge, either some of the data in one of the groups is bad or the model is not stable.

      By default, VT5 Center performs crosschecking any time the number of threshold coefficients exceeds 16.

      Using Cross Check may change your minEigenval setting from your initial VT5 model if it exceeds the minimum default value.
