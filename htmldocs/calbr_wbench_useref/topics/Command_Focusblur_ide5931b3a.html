focusblur

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Accounts for focus errors from the laser source, mechanical vibrations, and wafer stage tilt.

Usage

focusblur [lorentz gamma n] [gauss sigma] [rectangle width]
[ {ssfile scanner_slit_file} | {ssinline ss_inline_id} zrange low_val high_val]
[{lsfile laser_spectrum-file } | {lsinline ls_inline_id} lca lca_per_pm]
[integorder num] [filterval fval]

Arguments

lorentz gamma n

An optional argument set that specifies chromatic aberration from the finite laser bandwidth using a modified Lorentzian function.

Both gamma (Γ) and n are real numbers. The allowable ranges are: 0 <= gamma <= 0.5 microns and 2.0 <= n <= 4.0. The default for n is 2.8.

gauss sigma

An optional argument set that specifies a Gaussian function that models focus errors that arise from vibrations of the wafer stage.

The allowable range for sigma (σ) is 0 <= σ <= 0.5 microns.

rectangle width

An optional argument set that models stage tilting during scan.

The parameter width (W) signifies the total focal range of each image point. The allowable range for W is 0 <= W <= 1.0 microns.

ssfile scanner_slit_file

An optional argument set that specifies the measured intensity distribution inside the scanner slit.

ssinline ss_inline_id

An optional argument set specifying an inline_focusblur block of scanner slit information in the Optical file.

The ss_inline_id identifier must be specified in an inline_focusblur block.

zrange low_val high_val

An optional argument set that specifies the range of relative defocus.

The low_val and high_val values that follow the zrange argument are the lower and higher limits of relative defocus in microns. The lower limit must be negative and is the relative defocus that corresponds to the back edge of the scanner slit. The higher limit must be positive and is the relative defocus that corresponds to the front edge of the scanner slit.

lsfile laser_spectrum-file

An optional argument set that specifies the laser spectrum information in an external file.

A laser spectrum file (.lsf) specifies the laser spectrum information. It is not recommended to use lsfile and lorentz simultaneously as both parameters model the same physical effect.

lsinline ls_inline_id

An optional argument set specifying an inline_focusblur block of laser spectrum information in the Optical file.

The ls_inline_id identifier must be specified in an inline_focusblur block. It is not recommended to use lsinline and lorentz simultaneously as both parameters model the same physical effect.

lca lca_per_pm

An optional argument set that specifies the scanner longitudinal chromatic aberration (LCA) information.

The LCA parameter lca_per_pm is specified in um of defocus per pm of laser offset.

integorder num

An optional argument set that specifies the number of focal planes for the numerical integration of the blurred image.

The allowable range for num is: 3 <= num <= 51. The default num value is 5.

filterval fval

An optional argument set that takes into account fblur values that are larger than the fval value.

Using a small value has the effect of preserving the tails of the total blur function, whereas using a larger value has the effect of truncating the tails of the total blur function. The default value for fval is 0.05.

Description

This optional miscellaneous parameter enables modeling of three types of focus errors in the optics:

  1. Focus errors because of chromatic errors of the laser source (finite source bandwidth).

    • Can be modeled with the lorentz argument.

    • Can also be modeled by supplying a laser spectrum measurement file (lsfile or lsinline argument) that contains the corresponding longitudinal chromatic aberration (lca) parameter in microns.

    The lorentz and lsfile/lsinline arguments can both be specified, but it is recommended that you use one or the other, since they model the same effect.

  2. Focus errors because of mechanical vibrations of the wafer stage along the optical axis.

    Can be modeled using the gauss argument, which specifies a Gaussian distribution function.

  3. Focus errors because of systematic (purposeful) vertical motion or tilt of the wafer stage during the scan.

    • Can be modeled using the rectangle argument, which is a uniform intensity distribution function for the slit position.

    • Can also be modeled by supplying a file (ssfile or ssinline argument) that contains measurements of the maximum source intensity within the slit and the corresponding range of focus in microns during the scan.

    The rectangle and ssfile/ssinline arguments can both be specified, but it is recommended that you use one or the other, since they model the same effect.

    Note:

    All of the above terms for the total blur distribution are optional, but at least one must be specified to perform the focusblur calculations.

Examples

The following are examples of focusblur:

  • The following example models focus errors caused by finite laser BW and stage vibrations. The default integration order is used.

    focusblur lorentz 0.1 2.8 gauss 0.05

  • The following example models a wafer that is tilted during the scan in a way that each image point goes through 0.2 um of focus around the center point. The integration order is dropped to three.

    focusblur rectangle 0.2 integorder 3

  • The following example uses a laser spectrum file. The corresponding LCA parameter is 0.3 um of defocus per pm of laser offset. The integration order is increased to 11 (from the default of five points) and the tails of the blur function are kept for laser intensity values larger than 0.001.

    focusblur lsfile laser1.lsf lca 0.3 integorder 11 filterval 0.001
