Format of the Comprehensive Data Table

The comprehensive data table described in this section is output to the modelflow_v2 log file.

It can also be output separately to a file if the modelflow_v2 run contains the comprehensivetable filename option. Each line of the comprehensive data table output includes the following groups of data:

  • Input gauge data that unambiguously specify the gauge

  • Simulation gauge data that are not related to a particular model or stage

  • Two sets of simulation data that are associated with CTR and resist models

  • Simulation data that are related to etch model

Input Gauge Data

These records are required for converting a comprehensive table to a super gauge file. The data includes all gauge records except the enabled flag (which is assumed to be 1) and simulated resist and etch CDs (which belong to groups of simulation data). The list of records that are written to the table is as follows:

  • CI — contiguous index

  • GI — original gauge index within a given input gaugeSet

  • Def — defocus in microns

  • Dose — dose value

  • PCW — process condition weight

  • Struct — structure name

  • Row — row

  • Col — column

  • X1, Y1, X2, Y2 — gauge coordinates in dbu

  • Loc — location

  • Drawn — drawn CD value in nm

  • Other — other column value in nm

  • Meas — resist measured CD in nm

  • MEtch — etch measured CD in nm

  • Prec — precision

  • Weight — weight

  • StDev — standard deviation in nm

  • Count — count

  • 95%CI — confidence interval in nm

  • Sem — SEM picture name

  • Comment — gauge comments, which should not include commas

Simulation Data That Are Not Related to a Particular Model or Stage

  • Dim — dimensionality that can be 1 or 2 for 1D and 2D features, respectively

  • NWeight — gauge weight normalized to 1

  • Flare — flare value used only for single exposure models with standard gauge data (version 1) and only if a PNG file is present

CTR and Resist Data

Depending on the modelflow_v2 run settings, there can be up to two different simulation data sets. (The abbreviation “NA” is used if the data are not available.) They are related to the following:

  • CTR model (prefix used in the headline is Ctr, always available)

  • stage2 model (no prefix used, available if stage2off is not set)

Each resist data set includes the following columns:

  • SimCD — simulated CD in nm

  • Err — error defined as the difference between simulated and measured CD in nm

  • Shift — image shift defined as the distance between mid points of the drawn and simulated geometries in nm

  • Print — print status, which is 0 for successfully resolved gauges and 1 otherwise

  • EmTh — empirical threshold

  • DiffTh — difference between empirical and model threshold

  • Imin — minimal intensity value attained along a given gauge

  • Imax — maximal intensity value

  • Contr — contrast defined as the difference (Imax - Imin) / (Imax + Imin)

    (previous to the 2013.4 release, this was formerly calculated as Imax - Imin)

  • NS — normalized slope

  • ILS — image log slope in 1/um

  • NILS — normalized image log slope

  • AIDiff — integrated difference between aerial image and resist cutlines (0 for CTR)

  • MinAIDiff — minimal difference between aerial image and resist intensity values (0 for CTR)

    This quantity is normalized/divided by ||I||/sqrt(h) where h is the integration step size used when computing the aerial image norm ||I||.

  • MaxAIDiff — maximal difference between aerial image and resist intensity values (0 for CTR)

    This quantity is normalized/divided by ||I||/sqrt(h) where h is the integration step size used when computing the aerial image norm ||I||.

Etch Data

The etch data set contains the following columns:

  • SimEtchCD — simulated etch CD in nm

  • EtchErr — difference between simulated and measured etch CD in nm
