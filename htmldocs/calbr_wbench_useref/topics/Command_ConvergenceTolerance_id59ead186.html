convergence_tolerance

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Sets a convergence value for the rapidTCC engine.

Usage

convergence_tolerance val

Arguments

val

An optional argument specifying a convergence value for the rapidTCC engine.

Recommended values for TCC convergence tolerance are between 0.005 and 0.02. Tolerance values smaller than approximately 0.001 are not practical. The default value is 0.008.

Description

An optional miscellaneous parameter specifying a convergence value for only the rapidTCC engine. This parameter has no effect if used with the TCCcalc engine.

When a convergence tolerance value is specified, an internal pre-calculation step is used to automatically compute the required TCC upsample factors along the x- and y-axes (see also the rapidTCC_uf parameter). The upsample factors for x- and y-axes will be identical, unless the model is computed for a simulation window that is rectangular (1D models are a special case of this).

During the pre-calculation step, the tool computes a reference 1D model at a very large upsample factor. It then computes 1D models with upsample factors starting from one up to a maximum number that is case-specific. The final upsample factor is the smallest upsample factor that results in maximum relative differences of the TCC matrix (relative to the baseline) that are smaller than val.

Note that it may not be possible to achieve the specified convergence tolerance. In that case, the maximum upsample factor is used. Recommended values for TCC convergence tolerance are between 0.005 and 0.02. Also, note that tolerance values smaller than approximately 0.001 are not practical, since such a convergence level for the TCC matrix cannot be achieved within the limits of upsample factors (these limits are imposed from memory and runtime considerations). Smaller tolerance values result in better model accuracy at the expense of larger runtimes for the model calculation.
