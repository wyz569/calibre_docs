Variable Etch Bias (VEB) Model File Format

CM1 and VEB Model Formats

Input for: modelflow_v2, CM1 Center

Defines a VEB model.

The VEB model takes as input resist contours and returns biased versions of these contours. The direction of the bias is perpendicular to the resist contour. The bias is positive for the outward direction of the polygons and is based on density calculations.

The following types of density kernels are available for use in VEB calculations:

  • Shifted Gaussian (specified with the parameter SKERNEL type=gauss)

    Convolves with a Gaussian kernel and then shifts using a gradient. This kernel characterizes the separation between polygons.

  • Direct visibility (specified with the parameter SKERNEL type=visible)

    Calculates density using a “Point of Interest” (POI) vertex using the diffusion length as a scanning radius.

    • By default, the scanning direction is external and perpendicular to the polygon edge from the point of interest; this is referred to as direct external visibility.

      Direct external visibility characterizes the separation between layout features.

    • Specifying the additional VEB field “looking” with a value of “in” changes the direct visibility scanning direction to be internal to the polygon from the point of interest; this is referred to as direct internal visibility. (Specifying “looking=out” uses external visibility.)

      Direct internal visibility characterizes the size of the polygon relative to the point of interest.

  • Flux density (specified with the parameter SKERNEL type=flux)

    The flux kernel is an extension of the direct visibility kernel that differs from the direct visibility kernel in two aspects:

    • An additional parameter to account for the three-dimensional effect of feature height.

    • An option to use a full circle to define visibility instead of the half circle used by the visibility kernel.

  • Curvature (specified with the parameter SKERNEL type=curvature)

    Calculates density using a “point of interest” vertex that handles the curved areas of resist contours. The calculated density value is scale*curvature.

    • When the point of interest is on a straight line area of the resist contour, the value of this kernel is zero. Curves with higher concave values have an increasingly negative curvature. Curves with higher convex values have an increasingly positive curvature value.

    • Because VEB density values must be between 0 and 1, negative kernel values are not permitted. Correctly setting the parameter “looking” filters the curvatures by type. Use “looking=out” for concave curves. Use “looking=in” for convex curves.

The use of density kernels requires the creation of a bias polynomial that contains a constant and one or more associated polynomial terms for modeling coefficients.

Format

A VEB file must conform to the following restrictions:

  • Variable Etch Bias modifications can be run on the dense results of CM1 modeling. VEB is only available for Calibre nmOPC and Calibre OPCverify simulations.

  • There are two ways to specify a VEB file:

    • With the setup file keyword vetchbiasfile:

      vetchbiasfile filename

      where filename must be an absolute path to the name of the file (modelpath directories are ignored).

    • Alternatively, you can specify a VEB etch model file inline:

      vetchbiasfile inline 
inline_etch { 
VEB_file_definition
}

The VEB file format is defined as follows:

#Comments are here
version 1
modelType VEB
SKERNEL D1 type=polytype es=sval \
 u=uval [looking={in|out|both}] [h=feat_ht] [underlying=n]
SKERNEL D2 type=polytype es=sval \
 u=uval [looking={in|out|both}] [h=feat_ht] [underlying=n]
...
btermCount n
BTERM c0 
BTERM c1 { Dn1^p | Dn1^p*Dm1^q | Dn1^p*Dm1^q*Dl1^r | 
				Dn1^p*Dm1^q*Dl1^r*Dk1^s}
BTERM c2 { Dn2^p | Dn2^p*Dm2^q | Dn2^p*Dm2^q*Dl2^r | 
				Dn2^p*Dm2^q*Dl2^r*Dk2^s}
...
xBTERM cx0 
xBTERM cx1 { Dnx1^px | Dnx1^px*Dmx1^qx | Dnx1^px*Dmx1^qx*Dlx1^rx | 
				Dnx1^px*Dmx1^qx*Dlx1^rx*Dkx1^sx}
xBTERM cx2 { Dnx2^px | Dnx2^px*Dmx2^qx | Dnx2^px*Dmx2^qx*Dlx2^rx | 
				Dnx2^px*Dmx2^qx*Dlx2^rx*Dkx2^sx}
...
yBTERM cy0 
yBTERM cy1 { Dny1^py | Dny1^py*Dmy1^qy | Dny1^py*Dmy1^qy*Dly1^ry | 
				Dny1^py*Dmy1^qy*Dly1^ry*Dky1^sy}
yBTERM cy2 { Dny2^py | Dny2^py*Dmy2^qy | Dny2^py*Dmy2^qy*Dly2^ry | 
				Dny2^py*Dmy2^qy*Dly2^ry*Dky2^sy}
...
[regularization type={{diffuse grid_size=size [diff_len=length]}|
							{curve [iterations=n]} | curve_old}
[average_bias {on | off} [consistency = 0 | 1 | 2 | 3] ]
[maxsegment distance]
[segmentation {line | circle} [consistency=0|1] ]
[resist_profile p=value]
[veb_kernel_group_limit limit]

Parameters

version 1

A required parameter specifying the version number. Currently, only a value of 1 is used.

modelType VEB

A required parameter specifying the model type, which must be VEB.

SKERNEL kernel_parameters

A required parameter specifying the type of the kernel. Each SKERNEL entry contains the following parameters:

  • Dn — Polynomial identifier. n can be from 1 to 6.

  • type — A required parameter specifying the polynomial type (gauss, visible, flux, curvature, or resist_slope).

    The flux kernel resembles the visible kernel with two key differences:

    • Adds the h parameter to account for the three-dimensional effect of feature height.

    • Adds a value to an existing keyword (looking) to use a full circle to define visibility instead of the half circle used by the direct visibility kernel. To use full circle visibility, set looking to both.

    The resist_slope kernel is available only for MPC flows and accounts for the slope of the resist image. A maximum of one resist_slope kernel can be used.

    The resist_slope kernel resembles the resist_profile parameter with a few key differences:

    • The resist_slope kernel allows linear, non-linear, and cross term BTERMs; the resist_profile parameter allows one linear BTERM.

    • To perform the functionality of the resist_profile parameter using the resist_slope kernel, specify resist_slope only in a linear BTERM.

    • The resist_slope BTERM value and resist_profile p value perform the same function, but their values will be different. In many cases, the BTERM value will be 20 times larger than the p value.

    The resist_slope kernel cannot be used with the resist_profile parameter.

  • es — For gauss, visible, and flux kernels, a required parameter specifying the diffusion length of the kernel in nm. For curvature kernels, es is a required scale parameter to compute the density value of the kernel.

  • u — A required parameter specifying the shift of the kernel, in nm. For visible kernels, the value must be between 0 and 30 nm and it is recommended to keep this value at 10 nm. For flux kernels, the value must be non-negative if looking is set to in or out; the value can be negative if looking is set to both and represents an inward shift.

  • looking — An optional parameter that toggles the direction of the visible and flux kernels. It is not valid for Gaussian or resist_slope kernels.

    • For visible kernels, the default value is looking=out, which scans externally to the polygon.

    • For flux kernels, the default value is looking=out. For looking=in and looking=out, the shift value must be non-negative. For looking=both, the shift value may be negative. The both setting improves accuracy by using a full circle to define visibility instead of a half-circle.

    • For curvature kernels, there is no default. Use the looking parameter to set the kernel’s intended curve type to scan: positive/convex curved zones (looking=in) or negative/concave curved zones (looking=out). Both looking=in and looking=out kernels resolve to zero on straight lines.

  • h — For flux kernels, a required parameter specifying feature height in nm. The value must be non-negative. It is not valid for other kernels.

  • underlying — Optional parameter that specifies an ID number corresponding to an indexed layer from a Calibre OPCverify veb_simulate command1. The only value supported in this release is 1, which references the second layer statement in the Calibre OPCverify command setup file.

btermCount n

A required parameter specifying the number of bias modeling terms in the list to follow. n can be from 0 to 4.

For models used by Calibre® nmMPC™ to capture asymmetric mask writer bias, include the xBTERM and yBTERM terms in the count.

BTERM c0 |

BTERM ci Dn^p |

BTERM ci Dn^p*Dm^q |

BTERM ci Dn^p*Dm^q*Dl^r |

BTERM ci Dn^p*Dm^q*Dl^r*Dk^s

Specifies the common symmetric bias modeling terms.

  • c0

    Constant portion of the bias polynomial, in nanometers (nm).

  • ci

    Model coefficients, in nanometers (nm).

  • Dn^p

xBTERM cx0 |

xBTERM cxi Dnx^px |

xBTERM cxi Dnx^px*Dmx^qx |

xBTERM cxi Dnx^px*Dmx^qx*Dlx^rx |

xBTERM cxi Dnx^px*Dmx^qx*Dlx^rx*Dkx^sx

yBTERM cy0 |

yBTERM cyi Dny^py |

yBTERM cyi Dny^py*Dmy^qy |

yBTERM cyi Dny^py*Dmy^qy*Dly^ry |

yBTERM cyi Dny^py*Dmy^qy*Dly^ry*Dky^sy

Specifies the horizontal and vertical bias modeling terms. Available only for Calibre nmMPC for modeling asymmetric mask writer biases.

Note:

If an asymmetric bias is specified, then the model cannot have a constant common BTERM (that is, no B0 term).

  • cx0 and cy0

    Constant portion of the horizontal and vertical bias polynomial, in nanometers (nm).

  • cxi and cyi

    Horizontal and vertical bias model coefficients, in nanometers (nm).

The coefficient is multiplied by one (or two to four for non-linear cross terms) of the previously-defined SKERNELs using one of the following syntaxes:

  • Dnx^px

    Dny^py

    The related polynomial identifier nx and ny from the SKERNEL keyword list and the exponent value px (usually 1, with a range between 1 and 4). A value of 1 specifies a linear kernel. Any value for px other than 1 indicates a non-linear kernel.

  • Dnx^px*Dmx^qx

    Dny^py*Dmy^qy

    If you are using a two-term cross term, non-linear bias polynomials, they are specified by using this extended syntax.

    No whitespace is permitted between the ^ and * characters. Dmx^qx represents a second polynomial with exponent identifier, with the horizontal bias identifiers mx and qx distinct from nx and px. Similar distinctions apply to the vertical bias identifiers my, qy, ny, and py.

    This polynomial format represents the product of two density kernels. For the horizontal bias, this is the product of Dnx and Dmx. The second kernel representation is analogous to the previously described common symmetric bias equation. The total horizontal bias is the sum of the common symmetric bias and the horizontal asymmetric component. The total vertical bias is a similar summation of the common and asymmetric components.

  • Dnx^px*Dmx^qx*Dlx^rx

    Dnx^px*Dmx^qx*Dlx^rx*Dkx^sx

    Dny^py*Dmy^qy*Dly^ry

    Dny^py*Dmy^qy*Dly^ry*Dky^sy

regularization type=curve [iterations=n]

Optionally sets the method of final contour regularization. Due to the evolution of a contour, it is possible for a sharp corner to appear as a legitimate mathematical solution. Applying a regularization type sets the method used to smooth out the corners:

curve — Attempts to fit a parametric curve as a sum of Chebyshev polynomials with coefficients determined by a least-squares fit. This is the default setting.

The option “iterations” is ignored for other types. It specifies the number of iterations when fitting. Larger numbers give a better fit, but run longer. Good values for n are in the range 2 to 6. The default is 3.

regularization type=diffuse grid_size=size [diff_len=length]

Optionally sets the method of final contour regularization. Due to the evolution of a contour, it is possible for a sharp corner to appear a legitimate mathematical solution. Applying a regularization type (one of curve, diffuse, or curve_old) sets the method used to smooth out the corners.

diffuse — Computes the characteristic function (1 inside, 0 outside) of the polygon. Once computed, it recontours and patches in the new contour. This is the equivalent of convolution with a Gaussian.

The grid_size and diff_len parameters are only used with the type=diffuse method.

  • grid_size — Required for type=diffuse and ignored for type=curve. Good values for grid_size are in the range of 2 to 5, specified in nanometers (nm).

  • diff_len — Optional for type=diffuse and ignored for type=curve. Good values are 0 to 2*grid_size.

regularization type=curve_old

Optionally sets the method of final contour regularization. Due to the evolution of a contour, it is possible for a sharp corner to appear a legitimate mathematical solution. Applying a regularization type (one of curve, diffuse, or curve_old) sets the method used to smooth out the corners.

curve_old — Does not perform regularization. (This was the default in versions earlier than Calibre 2010.3. Automatically converted to curve if found starting with 2011.1)

average_bias {on | off} [consistency={0 | 1 | 2 | 3}]

An optional parameter that provides additional contour regularization. It reduces the amount of spikes and jogs that may appear on the contour due to bias variation. The default is on.

Adding the consistency parameter with a value of 1, 2, or 3 activates an improved consistency function between Calibre nmModelflow and Calibre OPCverify. However, it should only be used for line ends of small 2D features, because it can slow down etch simulations significantly. Use a consistency value of 1 for less aggressive cases and a value of 2 for more aggressive cases. A consistency parameter with a value of 3 should be used only for the most aggressive cases where the final consistency is expected to be less than 0.5 nm.

maxsegment distance

An optional parameter that defines the maximum distance between points of interest on a contour. The distance is in nm and defaults to 2*pixsize.

The maxsegment option is useful when a long edge has a sharp deviation. You could reduce pixsize instead, but that has a more global effect. Setting maxsegment to less than 2*pixsize has the same effect on the contour with less of an increase in runtime.

segmentation {line | circle} [consistency=0|1]

An optional parameter that defines POI placement. Specifying circle enables advanced circle POI placement along the edge in highly curved 2D features, such as line-ends. The circle option is recommended for complex cases such as EUV to improve grid-shift consistency.

Adding the consistency parameter with a value of 1 improves the accuracy of POI placement. However, it should only be used for line ends of small 2D features, because it can slow down etch model calibration and simulations significantly.

resist_profile p=value

An optional parameter that activates resist profile awareness in the VEB model. value signifies the resist sidewall angle, which is calculated as bias=p*(1/g), where (1/g) is the approximation to the resist image slope. This is a linear parameter; setting p to any nonzero value in the initial model causes modelflow_v2 and Calibre nmModelflow to optimize it for the final model.

The resist_profile parameter cannot be used for MPC. Do not use resist_profile with the resist_slope kernel.

veb_kernel_group_limit limit

An optional parameter that groups VEB Gaussian kernels in the Calibre nmMPC simulation flow. Any kernel less than the specified limit in nanometers is considered a very short kernel and kernels of the specified limit or greater are considered short kernels. The default value is 100 nm, and the valid range is between 50 nm and 10000 nm.

Examples

version 1
modelType VEB

#SKERNEL D1 type=gauss es = 400 u=0 
SKERNEL D1 type=visible es = 400 u=10 looking=out
SKERNEL D2 type=gauss es = 350 u = 50
btermCount 3
BTERM -40
BTERM 80 D1^1 
BTERM 50.0 D2^1
