version

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Sets the version number for the optical model.

Usage

version num

Arguments

num

A required argument specifying one of the following version numbers:

  • 3 — The source is rotated by 90 degrees.

  • 4 — For this version, the source is not rotated by 90 degrees. This is the only difference between version 3 and version 4. (While the source rotation is not a problem for 90-degree rotationally symmetric illumination sources, such as STANDARD, ANNULAR, and QUAD, rotationally asymmetric sources such as DIPOLE, require attention when choosing a version < 4.)

  • 5 — For this version, the source is not rotated by 90 degrees. The differences between 4 and 5 are related to image normalization and where the image is calculated.

    • In Version 5, the image normalization is by the source area. The image can be calculated at any plane, regardless of whether or not it is inside the first film. This allows you to compute for models with TARC.

    • In Version 4, the image can ONLY be calculated inside the first film. You have no control over image normalization, and the default image normalization depends on whether the optical model is aerial or thin-film (in air vs. inside resist). For aerial models, the default image normalization is by the source area. For thin-film models the default image normalization forces the open frame intensity to one at the first film interface.

  • 6 — Starting with version 6, the decomposition of the TCC matrix into SOCS kernels utilizes a faster eigenvalue decomposition (EVD) routine, as opposed to a slower (but more general) singular value decomposition routine (SVD). The time required for EVD compared to SVD can be up to six times shorter, so the time for optical model creation can be improved significantly for large TCC matrices (more than ~100,000 elements). Note that there are a few examples where EVD fails to converge; in those cases, the algorithm reverts to SVD. The numerical differences of SOCS models generated with EVD or SVD are negligible.

  • 7 — An updated version of version 6, which must be used for optical models that have kernel diffusion or non-TCCS kernels. Otherwise, this version is identical to version 6.

  • 8 — Includes the focusblur primitives (and may include vector entrance pupil information in the 2007.1 release). It contains a faster SOCS decomposition routine that computes the SOCS kernels and respective weights up to the number you request (use approxorder to set the number of kernels). The resulting partial decomposition of the TCC matrix is faster than the previous versions.

  • 9 — Standardizes the following defaults:

    • forceKernelSymmetry =1

    • vectormodelflag = 1

    • reduction_factor = 4

    • forcePower2LUT can no longer be set, and is automatically forced to a value of zero.

    • kerngrid is required to be set such that hoodpix/kerngrid is an even integer.

      If kerngrid is not set, Calibre WORKbench sets it internally to approximately (lambda/NA)/(1+sigma)/12.

      If kerngrid is set but the OD is not an integer multiple of kerngrid, kerngrid is adjusted slightly so that hoodpix/kerngrid is an even integer.

      hoodpix is never changed due to kerngrid or forcePower2LUT settings.

    Version 9 also adjusts the way hoodpix is handled. The hoodpix parameter is now fixed once you set it. Previous versions would modify hoodpix such that hoodpix divided by the kerngrid value was an integer. It could also be modified by the forcePower2LUT flag based on the setting for kerngrid.

  • 10 — Adds file storing format changes to the optical model files inside the optical model directory. Additionally, it combines the lookup table files into one file. Optical file sizes are reduced to half their previous size or smaller. The model file also improves writing and reading performance speed, up to four times faster.

  • 11 — Adds additional information to the optical model directory to improve accuracy and consistency of some DDM models with no performance penalty if the DDM library version is greater than or equal to 3. Otherwise, the behavior of a version 11 model is identical to version 10.

  • 12 — Improves optical model accuracy over version 11 for EUV. Additionally, HHA models use the following default values for HHA sectorization:

    • -tight 1

    • -strict_center 1

  • 13 —Adds internal symmetry detection that recognizes sourcemaps which are only symmetric in the X or Y directions. This does not affect simulation, but the information is used by other flows such as RET CELLARRAY in Calibre® CellArray™ RET.

    This is the default version starting with Calibre WORKbench version 2020.4.

Description

A required general parameter setting the version number for the optical model.

The default is 13. Versions prior to 3 are not recommended due to minor interpolation inaccuracies within the model. Any versions prior to 4 can be read in and understood, and used exactly as they were used in past releases. Any new model you create will be the most recent version by default, and any features present in earlier versions can be used with full backwards compatibility.

Regenerating Older Optical Model Versions

To regenerate older optical model versions, the opticsgen command cannot generate models previous to version 5 unless one of these environment variables are set.

  • WB_OPTICAL_V2

  • WB_OPTICAL_V3

  • WB_OPTICAL_V4

The rules used to set the version are as follows:

  • The highest specified environment variable version number must be equal to the version number in the optical parameter file. A mismatch causes optical model generation to exit with an error.

  • When the most recent version number is specified in the optical parameters file, then no WB_OPTICAL_* variables should be set.

Note:

If you use these variables, check the file named Optical in the optical model directory to ensure that the generated version matches what was intended.
