Creating an Initial CM1 File

Follow this procedure to create a coarse CM1 calibration file. You will be building up parameters by using the modelforms in CM1 Center.

Prerequisites

  • The setup file used in your test pattern process

  • A clipped test pattern layout loaded into Calibre WORKbench (from the procedure “Creating a Clipped Layout File”)

Procedure

  1. Load the layout file in Calibre WORKbench (File > Open Layout).
  2. Invoke CM1 Center (Litho > CM1 Center).
  3. Switch to the CM1 tab (Figure 1). Click the Modelform button to select the modelform (accept the changes in the dialog box that appears).
    Tip

    See the section "Modelform Trade-off Analysis" for information on modelform runtime proportions.

    By default, only the relevant b and s parameters for each chosen modelform are listed and selected. The larger modelforms result in the smallest fitness value, but also longer run times.

    If you choose one of the negative tone development (NTD) modelforms (38, 40 through 49, 58, 59, 79, 79, 88, or 89) the shrink and bias fields are also filled in for you. In CM1 Center, shrink and bias are the following fields:

    Table 1. NTD Term Fields

    CM1 Model Equivalent

    Specifies a shrink term.

    Optimizable using the shrink_radius parameter.

    SHRINK_THRESHOLD

    Specifies a shrink term threshold.

    Optimizable using the shrink_threshold parameter.

    SHRINK_POLARITY

    Specifies the shrink term polarity.

    Polarity can only be 1 or -1, and cannot be optimized.

    SCOEF

    Figure 1. CM1 Center, CM1 Tab

  4. Optionally audit the other CM1 optimization controls:

    • ANCHOR should be left at the default value (0).

    • THRESHOLD is automatically derived by CM1 Center during calculations.

    • CONST_TERM is the aerial image to CM1 normalization and is also automatically derived during CM1 calculations.

    • SMOOTH_NEUTRALIZATION sets the type for the smooth neutralization algorithm. Use a setting of 2 (spline smoothed neutralization, default), 1 (square root smoothed neutralization) or 0 (off). The recommended setting is 2 for non-CPA users. However, CPA users must use 1.

    • SMOOTH_GRADIENT controls the use of gradient smoothing. Use 2, which is the default, 1, or off (0). The recommended setting is 2. However, CPA users must use setting 0.

      Tip

      Using settings of SMOOTH_NEUTRALIZATION 2 and SMOOTH_GRADIENT 2 is recommended due to their ability to reduce output inconsistency of the model when used with Calibre nmOPC.

    • CONSISTENCY_SETTING specifies the use of a more consistent model at the cost of increased runtime during calibration when set to 1.

    • RESIST_POLARITY sets the resist polarity. Use 1 for PTD resists. Use -1 for NTD resists. The default is 1 (positive tone).

  5. Switch to the Optimize tab.
  6. Choose a search method. (Siemens EDA recommends the Front method.)

    • Newton — is for quasi-Newton optimization performed within the specified search ranges. Requires epsilon, Begin, End, and Levels settings.

    • Gradient — is a straightforward implementation of the method of local variations performed on a regular grid; the grid is constructed around the starting point. Requires Begin, End, and Levels settings.

    • Full — is a full factorial search over all possible combinations of parameter values performed on a grid created by search intervals based on the Begin, End, and Levels settings. You must enter a value of at least ten percent of your full factorial in the “explore” field.

    The Newton, Gradient, and Full searches can start from either nominal settings (set explore = 0) or from the best settings found during the exploration stage (set explore > 0).

    • Front — is a global search using the start and end parameters as a limit to the search range. It does not require a nominal starting point, but requires Begin and End settings. Front uses the runtime field to limit the duration in hours that the simulation will run.

      The optional Max AI Diff % field can be used with the Front algorithm to include a constraint on the percentage difference between CM1 and the aerial image surface. If used, this field must be filled with a positive non-zero value. It attempts to constrain the percentage difference to that value or less, but returns the closest value if it fails to do so. Siemens EDA recommends a value of 10 and an accompanying setting of 1.5 in the Threshold Tolerance field. It is recommended that you use Threshold Tolerance and Max AI Diff % together as a pair of settings.

    • Zoom is a local optimization routine. Unlike Gradient, the zoom search is performed on a self-adaptive grid, and it does not involve numerical differentiation as is the case for the Newton search.

    • Frontzoom is a combination of a global (Front) and local (Zoom) optimization. The front optimization is used to explore the search space and find a good initial guess (starting point) for the zoom optimization. The number of front iterations is determined by the explore or runtime option.

      Note:

      Siemens EDA recommends the Frontzoom search method to calibrate CM1 resist. Use the -turbo option (also located on the Misc tab in CM1 Center) to speed up calibration. On average, no more than 4 CPUs are required for stage 2, and no more than 20 for stage 1. Do not use the -remote option, which slows down resist calibration.

      Figure 2. CM1 Center, Initial CM1 Model Generation

  7. Switch to the Misc tab, and enter the setup file in the appropriate field.
    Figure 3. CM1 Center, Misc Tab

  8. Optionally, set a prefix for the working files CM1 Center generates. Changing the prefix allows you to save the settings and output from a specific run of CM1 Center for later study. Use the radio options to control whether working files with the same prefix should be silently overwritten (Overwrite mode) or if CM1 Center issues a warning before overwriting the files (Warn mode, which is the default).
  9. To run CM1 Center as a background process from CM1 Center, select the Job Farm control.
  10. Activate one or more of the Marginal controls to include reporting of up to the ten worst (largest) marginal variations in the session log. More information can be found in the section “Configuring Marginal and Other Log Outputs”.
    Tip

    The Misc tab also allows you to specify other controls for modelflow_v2 (as described in the “modelflow_v2 Reference”). Of special note are the following controls:

    • CM1/AI Diff only works if the Run Turbo checkbox is disabled.
    • Resume Run only works if the CM1 Center settings match the previous run, used a front search, and only one stage (Optics, CM1, or VEB) was selected for optimization.
  11. If you have a set of verification gauges, enter it in the Verification Gauges field. You perform this task if you want to run a verification on the calibration output immediately after calibration completes.
  12. If you have a distributed hardware system, switch to the Remote tab and create a remote host file (see “Distributed Processing in Calibre WORKbench” for more information). You can also select the Remote > Template option to generate a helper template.
  13. If you have any mask_model entries in your model-layer file, you must also include a lithomodel directoryname entry in the Output field.
  14. Save the session file (File > Save Session), click the O button in the CM1 section, and then click Run.

    • When the run completes, the lower and upper bounds found during the search (for the Front, Full and Gradient search methods) are shown in the Optimize tab. A result record is added to the list in the bottom pane of CM1 Center, showing the values found for the run.

      Tip

      If you added verification gauges in step 11, two log results appear; one with the suffix _c (Calibration) and one with the suffix _v (Verification).

    • If you activated Job Farming, an activity meter appears in the result record indicating the run in progress. You may continue working with CM1 Center, including setting up a parallel calibration run (change the temp prefix before you start a new run to avoid overwriting the parallel run).

      Tip

      Each Job Farming run can use distributed processing and uses the current settings in the Remote tab. Up to two background primary processes can be run simultaneously. Additional distributed runs are placed in a queue and run once a currently-running job finishes.

      Figure 4. Results of a CM1 Model Run

    • Clicking the Analysis Log filter button (labeled …, shown in red in the figure above) raises a selector box to set which items you want to show in the log line:

      Figure 5. Analysis Log Results Picker

    You can also view the values that were searched by clicking the Analyze button, then switch to the Search Data > CM1 section.

  15. Click the Accept: Gauges button in the Results pane (Figure 4) to load the gauges from the run into CM1 Center.
  16. Click the Accept: Optics button to load the optics file from the run into CM1 Center, in the Optics and Optimize (Optics) tabs.
  17. Click the Accept: CM1 button to load the newly-created model into CM1 Center.

    The model is loaded into the CM1 tab; the search range is shown in the Optimize (CM1) tab.

  18. Click the Accept: VEB button to load the newly-created model into CM1 Center.

    The model is loaded into the VEB tab; the search range is shown in the Optimize (VEB) tab.

  19. Save the session file (File > Save Session menu item).
  20. Save the log file by making sure the new result line is selected, then click the Log: Save button.
  21. Optionally, save the CM1 model (CM1 > Save menu item).

    • Restrictions on the Parameters

      Mathematical considerations (like avoiding taking derivatives of discontinuous functions or taking square roots of negative numbers) impose the following restrictions on the parameters:

      • k cannot be 1 or 2 if b > 0

      • n cannot be 2 or 3 if p > 0

    • Modelform Trade-off Analysis

      Using various modelforms is a trade-off between accurate predictions and performance speed. Lower numbered modelforms use less terms to describe the data points; higher numbered modelforms are typically more accurate but take longer to compute.

      However, each additional term usually contributes less to the equation than the previous term and requires less computational time.
