selectKernels

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Specfies SOCS kernels contained in the optical model.

Usage

selectKernels list

Arguments

list

A required argument list of space-separated non-negative integer numbers representing the list of which SOCS kernels the optical model contains. List construction differs between non-HHA and HHA models.

For non-HHA models, the number of list elements must match with the number of kernels specified in the approxorder argument. By default, SOCS kernels are used sequentially from one to the number of kernels for non-HHA models.

For HHA models, the number of list elements does not need to match with the number of kernels specified in the approxorder argument. The list of elements is a global linear list of kernels sector-by-sector. If the first HHA sector has nk1 kernels, then the first kernel in the first sector is 1, the second sector is 2, and the final kernel is nk1. A minimum of one kernel must be selected in each sector. The first kernel of the second sector is then nk1+1, the second kernel is nk1+2, and so on.

The intended use case is to generate a model without selectKernels set, and then choose the kernels from that model to keep. The file aux_info contains the order of the sectors and the number of kernels in each sector for the model without selectKernels set.

Description

An optional SOCS/kernel parameter representing the list of SOCS kernels the optical model contains. This is an advanced control intended for advanced users.

Note:

Beginning with Calibre version 2020.3, the selectKernels parameter is compatible with the hha argument. In prior versions, the selectKernels parameter was incompatible with the hha argument and could not be used with an HHA optical model. If hha was also specified, then the SOCS/kernel list was not used.

If selectKernels is used with hha, HHA symmetry detection is always disabled.

Examples

This is a non-HHA example. This is an advanced control has only non-negative integers, with no duplicates, and the highest kernel number is smaller than the maximum allowable SOCS approximation. For example: 

approxorder 10
selectKernels 1 2 3 4 5 7 8 11 12 15
