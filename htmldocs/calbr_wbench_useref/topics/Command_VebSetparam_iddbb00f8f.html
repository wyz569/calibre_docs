veb_setparam

CM1 and VEB Commands

Related Formats: Variable Etch Bias (VEB) Model File Format

Sets a VEB parameter for a loaded or created VEB object.

Usage

veb_setparam vebobj parameterName [index] value

Arguments

vebobj

A required argument that specifies the input VEB Tcl object.

parameterName

A required argument that specifies the input parameter name.

All model parameters are divided in two groups:

  • object parameters that belong to the VEB object as a whole.

  • term parameters are associated with a particular kernel in the VEB model expression.

If an object parameter is specified, then index must not be present. If a term parameter is specified, then a valid term index must also be specified. A valid index is an integer between 1 and (btermcount – 1) inclusive, where btermcount is the total number of linear coefficients (BTERMs) in the input VEB object.

All parameter names are case-sensitive. The following keywords return the associated values from the vebobj:

  • btermCount — Sets the number of terms in the VEB object, dimensionless.

  • BTERM0 — Sets the constant term (BTERM0) coefficient in nanometers.

Accepted term parameter names are the following:

  • u — Sets the kernel shift in nanometers.

  • es — Sets the kernel diffusion length in nanometers.

  • looking — Sets a string “in” or “out”, only for visible kernels

  • type — Sets the kernel type, either “gauss” or “visible”

  • BTERM — Sets the kernel term coefficient in nanometers.

index

An argument that specifies the input term index. It is required only if a term parameter is specified for parameterName.

The valid index values are also the same as for the veb_getparam command:

1 <= index <= btermCount – 1

value

A required argument that specifies the input parameter value to set.

Depending on the parameter name, the input value must satisfy the following limitations:

  • 0 <= btermCount <= 4

  • b >= 0

  • s >= 0

  • type can be gauss or visible

Description

Use the veb_setparam command in the Calibre WORKbench Tcl shell to set the specified parameter of the given VEB object to a particular value. Values can be retrieved using the veb_getparam command.

Setting a new btermCount value greater than the one in the input VEB object is the same as adding default (type=gauss s=50 u=0) kernels with the associated linear coefficients set to zero (BTERM 0). For example, if the input object specified by $obj has btermCount 2, then running “veb_setparam $obj btermcount 4” results in adding two more VEB kernels, D2 and D3, as defined by the “Variable Etch Bias (VEB) Model File Format”): 

SKERNEL D2 type=gauss s=50 u=0 
SKERNEL D3 type=gauss s=50 u=0

with zero linear coefficients:

BTERM 0 D2^1
BTERM 0 D3^1

Similarly, setting a new btermCount value smaller than the one in the input VEB object is equivalent to removing from the input object all kernels with indexes greater than or equal to the new btermCount value.

Examples

In this example, the VEB object loaded from a file is modified such that the linear coefficient (BTERM 0) is set to a new value (16.3), and then the second kernel’s BTERM (BTERM 2) and u (U 2) parameters are set to new values (0.011 and 25 respectively).

set obj [veb_create –file “veb.mod”]
# reset one of the object parameters, e.g., BTERM0
veb_setparam $obj BTERM0 16.3
# reset some of the term parameters, e.g., BTERM 
# and u of a 2nd term/kernel
veb_setparam $obj BTERM 2 0.011
veb_setparam $obj U 2 25
