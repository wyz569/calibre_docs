Source Map File Format

Optical Model File Formats

Specifies the illumination intensity on a grid and can be used to create an optical model with the COMPOSITE source, using the MAP profile type.

Input for:

  • mergemap

  • opticsgen — when creating a model with a composite source using MAP profile

Generated by:

  • opticsgen

  • Calibre WORKbench Optical Model Tool

  • Third-party tools

An output source map (named “illummap.smf”) is also automatically generated in the optical model directory created by opticsgen whenever the TCCcalc or rapidTCC engine is used. Note that you use this output file only for viewing purposes; it is never used during model creation. Therefore, you may notice differences between an input source map used for model creation and the output source map used for viewing. In particular, the input and output source maps may be defined on a different grid; the grid step size of the output source map is by default 0.01 (you can change it by setting the environment variable SRC_MAP_STEP in .cshrc or .signaext) which can be different than the step size of the input source map.

Note:

When an optical model with an associated source map file is exported from the Calibre nmModelflow tool, the source map file is saved inside the associated optical model directory. You can access each optical model subdirectory from within the litho model directory.

The symbolic links access a local copy of the model files with an internally-generated structure, which may be subject to change in future releases.

Format

  • ASCII text file.

  • Extensions are .smf, .src.

  • Keywords are case-sensitive.

  • One keyword per line.

  • Comments are preceded by double slashes (“//”).

  • Source map files start with a header block as follows:

    [Version]
version
[Parameters]
[comments]
[format]
step value
[DATA]

and then one of the following two data line types:

unpolarized sources: 

X Y intensity
…

polarized sources:

X Y Emag DOP FvAngle Phase
…

Parameters

[Version]

version

A required block at the top of the file specifying the version of the source map file. This section must begin with the keyword [Version] (the brackets are part of the syntax), and contains only one item; the version of Calibre WORKbench used to create the source map file. It ends with the next keyword.

[Parameters]

[comments]

[format]

step

value

A block at or near the top of the file used to define any relevant parameters. This section must begin with the keyword [Parameters]. It ends at the [DATA] keyword.

  • comments — Specifies user defined comments. If this optional parameter is not specified, blank is used. The string “; User Defined Source Shape Name” is always appended to the comment.

  • format — Specifies the format used for the source map. 0 is grid; 1 is discrete. This optional parameter has a default of 1.

  • step value — Specifies the pixel step size of the source map. Required.

[DATA]

The main portion of the file, used to define the intensity of light at each pixel in the illumination source. This section is required, and it must begin with the keyword [DATA]. It ends at the end of the file.

This section contains a single line for each pixel for which the intensity is non-zero. (You need only specify non-zero pixels. Any pixel not defined is assumed to be zero.)

The (X,Y) values for the pixels are specified in units of (step*N), where N is an integer index of the 2D matrix of source map values.

The pupil coordinates X and Y are unitless, and represent normalized spatial frequencies.

X Y intensity …

Unpolarized light sources use this three-column format. The intensity values of the illumination are unit-less. They can have any non-negative value.

  • If multiplied by NA/λ, they represent spatial frequencies in units of 1/um (if λ is specified in microns).

  • If multiplied by NA, they represent direction cosines with respect to the x- and y-axes.

X Y Emag DOP FvAngle Phase …

Polarized light sources can be viewed in the Source View tool (see “Post-Generation - Optical Model Visualization Tools”). Polarized light sources use this six-column format instead of the three-column format above.

  • Emag — Field amplitude |E|

  • DOP — Degree of Polarization p

  • FvAngle — Field vector angle, azimuthal tangent atan (Ey/Ex) in degrees

  • Phase — Phase shift between Ex and Ey in degrees (Ø (Ey) - Ø (Ex))

    Note:

    Using opticsgen or the Optical Model Tool to create a source map file will always output a three-column intensity map, which already accounts for polarization information if defined as part of the optical model. The six-column format is transparently supported by the Source View tool.
