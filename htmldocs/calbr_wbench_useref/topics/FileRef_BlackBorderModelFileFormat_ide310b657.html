Black Border Model File Format

Input for: flare models in Calibre WORKbench or Calibre OPCverify

Describes a black border intensity model for Calibre WORKbench and Calibre OPCverify.

Black border files are separate from the Flare Model File Format file.

A black border file can only be loaded and referenced inside a litho model, using the syntax “black_border filename”. When a litho model references a black border model, Calibre WORKbench and Calibre OPCverify apply the black border parameters dose_ring and blade_shadow to the long range flare PNG file.

Note:

Litho models containing black border models automatically include the black border model’s effects by default. If a litho model contains a black border model, but the black border model effects are not wanted for a Calibre OPCverify, Calibre nmOPC, or Calibre pxOPC image or Calibre nmOPC pw_condition command, you must add the “no_black_border” argument to the instances of those commands.

The specific widths of the dose_ring and blade_shadow regions are calculated from the edges of the flare PNG file, which allows you to change black border parameters without needing to regenerate the long range flare map.

Note:

All flare PNG files generated by the RET FLARE_CONVOLVE command starting with version 2015.1 are version 2 files. Version 2 model files separate out black border intensities into a separate model file. The black border model described here cannot be used with version 1 flare model files.

Format

version 1 
dose_ring dose { left_and_right top_and_bottom | 
	left right top bottom } 
blade_shadow { 					left_and_right top_and_bottom | 
	left right top bottom } 
[field_size x_um y_um]

Parameters

version 1

A required parameter that specifies the version of the black border model. Currently the only accepted value is 1.

dose_ring dose {left_and_right top_and_bottom | left right top bottom}

A required argument that specifies the value of the dose overlap from neighboring dies. The dose value is in units of intensity.

Note:

Doses in the corners have overlap from three neighboring dies, and thus have three times the dose value.

The left_and_right, top_and_bottom, and left right top bottom values are specified in microns. They represent the widths of the dose overlap region starting from the edge of the flare map and working inwards.

blade_shadow {left_and_right top_and_bottom | left right top bottom}

A required argument that represents the widths of the linear shadow region due to the masking blades specified in microns. The widths start at the end of the dose_ring and work inwards by the specified width arguments (left_and_right and top_and_bottom or left right top bottom). In this region, overlapping doses ramp down linearly from the dose value in the dose_ring argument to zero by the end of the blade_shadow width.

field_size x_um y_um

An optional argument that specifies the field size around which the black border is placed. If field_size is specified in the black border model, the corresponding Calibre OPCverify and Calibre nmOPC setup files must also contain the keyword “euv_field_center x_um y_um”. When both field_size and euv_field_center are specified, the information is used to add a black border to the specified field size in microns centered around the euv_field_center coordinates.

If this argument is not specified, the black border is added to the edges of the flare map.
