Resuming Stopped Runs from the GUI

Stopped runs can be continued if you have the session file.

Prerequisites

  • Only one of the (O)ptimization types (Optical, CM1, or VEB) is set; the other two types must be set to non-Optimization modes

  • Front search is used for the type being optimized

  • No other optimization parameters or search intervals can be changed (you must have the original modelflow_v2 run file (.mfr file if you used CM1 Center))

  • Log file from the original run is available

Procedure

  1. Load the session file (File > Open Session)

    If you did not have a session file saved, reload and reset the following items:

    • Gauge or super gauge file

    • The Optimization tab settings associated with the log file

  2. Switch to the Misc tab and Import the original log file.
  3. Select the Resume Run check box.
  4. Click the Run button to resume the search.

Results

When you view the log from the second run, you will see lines similar to the following (assuming a run with 15 front explorations and a log file “original.log” with three records in it): 

--> FRONT RUNS = 15
--> PREVIOUSLY GENERATED FRONT OPTIMIZATION RECORDS READ FROM original.log
BEGIN STAGE 2 OPTIMIZATION RECORD 1
...
(3 optimization records from previous run here)
...
--> DONE WITH PREVIOUSLY GENERATED FRONT OPTIMIZATION RECORDS
BEGIN STAGE 2 OPTIMIZATION RECORD 4

The total number of explorations shown at the end of this log is the sum of both runs.
