Working With Input VEB Contours

Use this task when you only have a VEB model and corresponding etch model contours to work from. This task skips calibration of optical and resist models

Prerequisites

  • VEB model

  • Design file for the test chip with the following:

    • Clip layer with clip shapes over all gauges of interest.

    • Layer containing the resist contour.

    • Layer containing the etch contour.

  • SEM pictures of the etch contour.

Procedure

  1. If you do not have both resist and etch contours already available in the design, complete the task “Alternate Method to Create Contour Design Files”, including the step to generate the etch contour.
  2. Create a CLI file (described in the task “Creating a Contour Layer Information File”) that includes one contourlayer statement for each layer type (resist and etch).
  3. In CM1 Center, switch to the Contour tab. Select the VEB Input Layer box so that it is checked, and set the Contour Weight field to 1.0.
  4. Ensure that the CLI file includes a contourlayer statement for both the resist layer and etch layer. An example completed CLI file might look similar to the following:
    	# CLI Template
	version 1
	contourSetCount 1
	contourSet 1 {
		contourlayer 103 resist
		contourlayer 104 etch
		cliplayer 94
	 	dose 1.0
		defocus 0
 	 	outstartlayer 300 3
	}
  5. Switch to the Gauges tab and ensure that all gauges are disabled.
  6. Switch to the VEB tab and load your VEB model.
  7. Switch to the Misc tab and enter the same of the setup or model layer file in the appropriate field.
  8. Switch the CM1 Center mode controls to:

    • Optics: X - Deactivates the optical calibration step.

    • CM1: X - Deactivates the resist calibration step.

    • VEB: T - Instructs modelflow_v2 to use the loaded VEB model as is, and calibrates using the contour instead.

  9. Click the Run button to generate the data statistics for the input VEB model and contour. Data is output to the design window as additional layers.

Results

Data is output to the design window as additional layers. A log file appears at the bottom of the CM1 Center window, but it cannot currently be used to analyze the results.
