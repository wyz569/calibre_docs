Simulating EUV Including Stochastic Models

Stochastic models are not intended to replace existing resist models. Instead, they allow for calibration using an uncertainty factor defined in the stochastic model. Use this task to simulate the behavior of the stochastic model.

Prerequisites

  • The corresponding design file

  • A litho model containing calibrated optical, CM1, and stochastic models, similar to the following:

    version 1
resist resist.mod
image_threshold 0.201636
flare flare.mod
stochastic stcm.mod
mask 0 {
  optical nominal
  background dark
  mask_layer 0 TRANS dark CX 0.007000 CC 0.007000 BIAS -0.000750 DDM ddm_setup_noxtalk_intmx_sw_89_dark.ddm}
    Note:

    Calibre nmModelflow’s Lithomodel/Setup/MDF Configuration Creation Wizard supports the creation of litho models that include stochastic models starting with the 2017.4 version.

  • Calibre nmModelflow invoked

Procedure

  1. In a text editor, create a setup file (named setupfile.in) similar to the following: 
    modelpath .
layer m1
flare_longrange LITHOMODEL constant 0.02
processing_mode flat
tilemicrons 25
simulation_consistency 2
rasterizer_upsample_factor 2 1
image_options MDF {
 layer m1 visible
 litho_model LITHOMODEL
}
setlayer nom = image MDF
  2. In a text editor, create a Calibre nmModelflow script similar to the following code:

    • Include a section that loads the gauges file with stochastic data, and loads the setup file:

    mdf shutdown
mdf gauges close
mdf setupfile close
mdf gauges gauges.sgd -fixdrawn off input.oas 1
mdf setupfile setup.in

    • Run resist image and stochastic simulation using the litho model and gauges, outputting the results to a gauge file (gauges.txt):

    
mdf simulate
mdf stochastic simulate
mdf gauges export -format txt -pw all "gauges.txt"

    Save the file.

  3. Run the script from the Calibre nmModelflow Command window.

Results

The script computes stochastic variations at the feature edges of each gauge using the parameters defined in the stochastic compact model file (stcm.mod in this case). The stochastic variations are reported by adding two columns to the output gauges: Stcm1 and Stcm2, which represent the widths of the stochastic bands for two edges of each gauge.

Starting with stochastic model version 2, you can also check for stochastic failure by adding the following command after the mdf stochastic simulate line:

mdf stochastic failure_rate -N iterations [-seed random_seed]
[{-tol_percent percent | -tol_nm value}] [-min_cd value]

where

  • -N is the number of trials to perform. The result from adding this command is a new column, Stcm_Failure, that shows the fraction of the times a gauge failed to print for the specified number of trials.

  • -seed is a quasi-random seed to use as a random number generator. The default is 1.

  • -tol_percent and -tol_nm are mutually exclusive tolerance constraints (allowed percent and allowed nm variation, respectively) to use as a failure check. If neither tolerance value is specified, then failure to print is used as a criteria.

  • -min_cd is an optional argument that specifies the minimum allowed CD for the gauge in nm. If a gauge’s CD falls below the specified value, it is counted as a failure.
