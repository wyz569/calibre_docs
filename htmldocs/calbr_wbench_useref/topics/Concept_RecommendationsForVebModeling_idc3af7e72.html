VEB Model Kernels

Proper selection and configuration of VEB kernels affects the accuracy and computation speeds of a completed VEB model. Calibre WORKbench provides VEB modelform templates that cover most etch process technology configurations for wafer processes. Calibre WORKbench also supports manual creation of modelforms (as shown in the Select Model Form dialog, available from CM1 Center). Using a custom combination of kernels and ranges of parameters to support various processes is also supported for when no modelform matches your needs.



Modelforms are pre-defined by types and numbers of kernels, kernel combinations, and if the model uses linear or non-linear operations.

  • A maximum of six kernels can be used in the VEB model.

  • Due to runtime performance concerns, it is recommended that you use at most one visible-type kernel in either direction (choosing from “looking in” or “looking outwards”).

  • Use a curvature kernel for resist contours that contain areas with highly curved zones, such as line ends or concave zones.

    • Use a “looking in” curvature kernel for gauges placed on line-end patterns.

    • Use a “looking out” curvature kernel for gauges placed near concave curves.

  • Use the least number of kernels if there is a similar level of accuracy preserved from the model calibration. One gauss and one visible kernel (modelform #21 or #22) are usually sufficient for most cases, but some cases may need two gauss and one visible kernel (modelform #31 or #32).

  • Use the front search algorithm with a range depending on the number of kernels used, as shown. Using fewer kernels enables using exploration values closer to the lower end of the range. Using more kernels tends to require a higher number of explorations.

    For multiple Gaussian kernels:

VEB Model Parameter Range Recommendation

Note:

The VEB default search ranges for es and u are only suggestions, and should be determined with an analysis of etch bias response versus feature size for your specific process of interest.

  • Selecting the Upper Limit for the es Operator

    The diffusion length (es) modeling parameter range is based on the kernel type.

    • Gaussian kernels have a default range from 35 to 600 nm, but the upper search limit can be set as high as 800 nm.

    • Visibility kernels have a default range of 0 to 600 nm. If the upper limit is reached, do not increase the upper search range limit. Values greater than 800nm for visible kernels can have an adverse runtime effect on Calibre OPCverify and Calibre nmOPC operations.

    A best practice is to limit the es value to the largest line-space structure where the etch bias is still variable. You can safely exclude structures that show flat bias values. For example, in the following figure, you do not need to check an es range higher than 400.

    Figure 1. Finding the Upper es Limit

    As a sanity check, the BTERM coefficient (in nm) should be found to approximate the etch bias range when the fitting calibration completes. In Figure 1, the magnitude of BTERM coefficients should not be much greater than +/- 50 nm.

    In relative mode, you can use the VEB Analysis Graph, Fitting tab and look at the Relative CD (Simulated and Measured, Per Gauge) to see if the CD_VEB - CD_CM1 values are less than the BTERM coefficients. If they are not, then the model will need further adjustment.

  • Checking for Non-convergence

    After each calibration, you should check that the es parameters have not reached the upper or lower limits of their search ranges, as this could lead to poor model quality.

    Similarly, u parameters should be checked to see if the found parameters are at their upper or lower limits before using them as final values.

  • Keep Shift u Parameters Less than es Parameters

    If the absolute value of the shift u parameter greatly exceeds the es parameter, Calibre WORKbench may end up sampling neighboring features more than the feature of interest, which is not a recommended practice.

    For Gaussian kernels, use a default search value of -50 nm to 50 nm. Ensure that the u parameter does not exceed 1.3 times the es value.

    Visible kernel shift is fixed and cannot be optimized. It allows for a range of 5 to 35 nm for u. Siemens EDA recommends that the shift for u be kept at 10 nm, but it can be shifted within the allowed range as long as it remains static for that calibration run.

VEB Calibration Runtime Improvements

By default, modelflow_v2 computes the VEB bias density on the contour without moving the resist contour. This is known as “biasing_type 0” behavior. Selecting “biasing_type 1” causes modelflow_v2 to compute its etch bias by moving the resist contour to the etch contour position. Using “biasing_type 1” generates more consistent results, but can be slower, because it instructs modelflow_v2 to move edges while it computes the edge bias. The “biasing_type 0” argument is a faster, because it uses a simpler approximation method.

To avoid degrading model accuracy with fast approximation, selective biasing options for faster approximation are available.

Using the “biasing_type auto [cdtol]” argument automatically detects potentially “bad” gauges on which the “biasing_type 0” approximation can substantially fail, and marks these gauges as “biasing_type 1.” The marked gauges are then treated accurately, while the rest of the gauges are to be treated approximately.

To maintain calibration runtime, Siemens EDA suggests using the default “biasing_type 0” argument for the initial stage of model calibration, then carefully validating VEB model fitness between calibrations by using the “try” regime with the settings from the transcript log.

  1. Ensure that “biasing_type 1” is not active in stage3 of modelflow_v2.

  2. Run calibration, comparing gauge simulated results to the actual results in silicon.

  3. If the gauge accuracy is within tolerance, continue using the “biasing_type 0” argument in order to benefit from the improved calibration speed due to least computation.

    • If a gauge set only contains 1D features, or its 2D features have very limited etch bias volume (less than 30nm) from the analysis, then using “biasing_type 0” delivers an equivalent accuracy to the “biasing_type 1” argument.

    • If a gauge set with 2D structures has a large observed etch bias (bigger than 30nm) then you should carefully look at the VEBerms value in the transcript log to see the difference between using “biasing_type 0” and “biasing_type 1”.

    • If the gauge accuracy is not acceptable, you can use one of the following biasing_type enhancement flags on individual gauges.

      selective_biasing_type1 [filter_layer_number] 
      structs_biasing_type1 [struct_list]

      These enhanced features select gauges using either an additional filter layer (selective_biasing_type1) or substrings (structs_biasing_type1) in their names in gauge file. Then perform calibration as if “biasing_type 1” were active for those gauges. The option “biasing_type auto [cdtol]” provides automatic and objective decisions upon comparing simulated etch CDs from the initial model form.

    • If there is still a huge discrepancy found even when using the biasing_type algorithm, then you may consider using the “biasing_type 1” option for entire gauge sets.

      Note:

      Siemens EDA recommends using the “-foptions parallel” argument to preserve parallelization by adding more front candidates when you add more cpus through a distributed remotefile.

VEB Relative and Absolute Fitting Modes

Understanding when to use relative and absolute fitting modes can assist in VEB calibration.

  • When only measured etch CD values are available, use absolute fitting mode.

  • In absolute mode, the final etch CDs are fit to be closer to the target values,

  • In relative mode, the final etch CDs are fit to minimize the etch bias error (the relative difference between the measured and simulated values for both etch and resist).

Guidance for Non-Linear Terms for VEB Models

Correct use of non-linear terms during VEB calibration assists in calibrating models properly.

  • Non-linear terms can only have exponents in the 2 to 4 range, and can multiply at most two densities per term. The use of negative powers is not permitted.

  • Non-linear crossterms can be enabled by the end-user, but Siemens EDA recommends against the use of crossterms in order to avoid overfitting of the model.

  • You should always test the final model on verification data for model accuracy and stability.
