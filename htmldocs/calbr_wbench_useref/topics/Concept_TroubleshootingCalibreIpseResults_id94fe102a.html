Troubleshooting Calibre IPSE Results

Known issues that you may encounter while running Calibre IPSE are as follows:

  • “Can’t run Calibre” message

    The Calibre IPSE must be able to find the “calibre” executable in the environment path, since it runs the Calibre batch mode command from Calibre WORKbench.

    Workaround: Quit Calibre WORKbench, then re-invoke Calibre WORKbench from the $CALIBRE_HOME/bin directory, or add $CALIBRE_HOME/bin to your path.

  • Simulate button remains greyed out

    Calibre IPSE runs in parallel with Calibre WORKbench. You can perform other actions in Calibre WORKbench while a Calibre IPSE simulation is running. For large designs, the simulation may take some time to complete.

  • Other error messages

    Other errors can arise from one of the following conditions:

    • The setup file does not have the right layer number in its layer list for the OPC layer in the layout.

    • Layer names in the setup file cannot begin with a number. For example, “S1” is acceptable, while “11” is not.

    • Layer names in the setup file cannot be named MARKERLAYER and MARKERLAYERX.

    • The precision set for the design does not match the setting manually specified in the Layout Precision field.
