getflare

Calibre WORKbench batch command

Probes PNG flare map output from RET FLARE_CONVOLVE.

Usage

getflare mode [gaugefile filename] [verify] -pngin filein
[-lithomodel litho [clear | dark] [-field_center x_um y_um]
[-flare_map_shift x_um y_um] [-slit_center x_um]]

Arguments

mode

A required argument that sets the operating mode for the command. The options available are as follows:

  • difference -pngin2 filename2 -pngout out_filename

    A mode that can be used to generate a difference map between two flare maps or a slope map for a flare map.

    To generate a difference map, two PNG files must be specified and the second file (‑pngin2 argument) is subtracted from the first file (-pngin argument). The difference is written to the specified out_filename.

    To generate a slope map, one PNG file must be specified using the -pngin argument. The slope value for each pixel is the absolute value of the maximum difference between the flare value of the specified pixel and its four adjacent neighbors in the x- and y-axes. The slope is written to the specified out_filename.

    You can use the output file in a subsequent getflare command to examine its properties.

  • extent

    A mode that specifies that the flare map extent coverage (bounding box covered by the flare PNG file). It returns a value in microns using the following format:

    x_ll y_ll x_ur y_ur

    For example, the command:

    getflare extent -pngin mypng.png

    returns the value:

    -4248.000000 -7160.000000 4248.000000 7160.000000

    where x_ll is the lower-left corner in the x-direction, y_ll is the lower-left corner in the y-direction, x_ur is the upper-right corner in the x-direction, and y_ur is the upper-right corner in the y-direction.

  • range [x_ll y_ll x_ur y_ur]

    A mode that returns the flare range within the entire PNG file. The output is as follows:

    flare_min flare_average flare_max

    A set of coordinates can optionally be specified to return the minimum, average, and maximum flare values within the bounding box.

  • values2d x_ll y_ll x_ur y_ur [-gt value|-gte value] [-lt value|-lte value]

    A mode that returns flare values contained within an input bounding box given by lower left and upper right corner coordinates (x_1l,y_1l) and (x_ur,y_ur). Calibre takes the original values from the PNG file on the original PNG grid and outputs them. These values are not guaranteed to be fully enclosed in the input bounding box, but generally are within one pixel of the bounding box requested.

    The optional flags -gt, -gte, -lt and -lte can be used to filter flare values above, below or between specified thresholds. The flags -gt X or -gte X may be specified to filter flare values ‘greater than’ or ‘greater than or equal to’ the value Y. The flags -lt and -lte may not be specified at the same time. The flags -lt Y or -lte Y may be specified to filter flare values ‘less than’ or ‘less than or equal to’ the value Y. The flags -lt and -lte may not be specified at the same time. To create a combined boolean range, one of the ‘greater than’ flags can be combined with one of the ‘less than’ flags and both conditions must be satisfied for the pixel to be reported.

    The format of the output is:

    x1 y1 val2
x2 y2 val2
x3 y3 val3
...
xN yN valN

  • values1D x1 y1 x2 y2

    A mode describing a 1D cutline. Calibre interpolates values to the exact cutline location in the 1D direction, but maintains the original spacing of the flare PNG file along the cutline direction. This means that output flare values correspond to points exactly along the line requested, but the location of points along that line are dictated by the spacing in the original PNG file. 1D cutlines cannot be computed at the edges of the PNG file; you must be inside the extent of the PNG file enough to allow interpolation.

gaugefile filename

An optional argument that uses the gauges defined in the specified filename to output flare values around the gauges.

-pngin filename

A required argument specifying a reference PNG file as the input source for the command. It is assumed that the PNG file was created using the RET FLARE_CONVOLVE. command.

-lithomodel litho [clear | dark]

An optional argument that specifies a single litho model or global litho model. The model specified by the litho argument must include a flare and an optical model, possibly inside a lithomodel subdirectory.

You can specify the tone of the mask with either clear or dark settings. With these inputs, the convolution values of the input PNG file are converted to true flare values using the following equations:

  • dark: flare_value = Iclear * kf * png_value

  • clear: flare_value = Iclear * kf * (TIS - png_value)

When litho is a global litho model, getflare loads the optical and DDM models based on the flare map location, using the closest slit position.

-field_center x_um y_um

An optional argument that can only be specified if a black border model that includes the field_size keyword is used in the input litho model. It sets the field center in microns for the black border model to be centered about.

If a global litho model is specified for the -litho_model argument, then -field_center is required.

-flare_map_shift x_um y_um

An optional argument specifying a directional shift of the flare map origin in microns. This shift is performed before any other coordinates from the command are considered.

-slit_center x_um

An optional argument that must be specified when a global litho model is specified for the ‑lithomodel argument. It sets the center slit position in microns of the global litho model relative to the OASIS coordinates of the associated flare map.

verify

An optional parameter that is used to check for non-physical pixels in the flare map. If the flare map is clean, an empty string is reported. If bad pixels exist, their locations and flare values are reported in the format:

x1 y1 val1
x2 y2 val2
 ...
xN yN valN

Examples

The following example demonstrates using getflare to output the flare PNG extent:

getflare extent -pngin mypng.png 

-4248.000000 -7160.000000 4248.000000 7160.000000

In the following example, getflare is used to find the maximum and minimum PNG values:

getflare range -pngin mypng.png

0.02581 0.866760

In the following example, getflare is used to produce a vertical cutline in the middle:

getflare values 0.0 -7160 0.0 7160.0 -pngin mypng.png 

0.000000 -7156.000000 0.349736 
... 
0.000000 7156.000000 0.634736
