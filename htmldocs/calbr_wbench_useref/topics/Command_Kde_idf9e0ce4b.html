kde

Optical Model Commands

Computes coverage by estimating the Gaussian kernel density of gauge parameters.

Usage

kde -ing calibration_cvs_file -ine evaluation_csv_file -cols col1 [... col4]
[-threshold threshold_value] [-bandwidthmethod {Scott | Silverman | {Constant float_num}}]

Description

The kde command estimates how well evaluation gauges are covered by calibration gauges. It requires a calibration gauge file and a related evaluation gauge file. Based on one or more columns (with numerical values) from the first gauge file (such as Imin, Imax, or normalized slope), it estimates the coverage of each evaluation gauge by computing the density of the evaluation gauge in the space constructed by the existing calibration gauges.

The output of this command is the file risk_above_<threshold_setting>.gg; by default, this is risk_above_10.0.gg. This file contains the subset of gauges that have an estimated density that is small enough that its reciprocal is larger than the threshold. The last column of the gauge file contains the calculated risk value (calculated as 1/estimated density).

Arguments

-ing

A required argument that specifies the calibration gauge file. It must follow these rules:

  • The file must be in CSV format.

  • The first row must list all the column names.

  • It is an error for a column name to be empty. “xxx, , yyy” is an illegal construct.

  • Column names are case sensitive.

  • Columns named in the -cols argument must be present in both the calibration and evaluation gauge files.

-ine

A required argument that specifies the evaluation gauge file. The evaluation gauge file must use the rules specified for -ing.

-cols col1 [... col4]

An argument that specifies one or more string names of columns in the gauge file to use for the comparison. At least one argument is required, and up to four columns can be specified.

-threshold threshold_setting

An optional argument that sets the threshold for output as a floating point value. The default value is 10.0.

-bandwidthmethod {Scott | Silverman | {Constant float_num}}

An optional argument that selects the method to set the width of an individual Gaussian curve as the Scott or Silverman methods, or a constant (with a specified floating point value). The default is Scott.
