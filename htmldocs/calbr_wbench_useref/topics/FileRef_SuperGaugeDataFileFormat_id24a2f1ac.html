Super Gauge Data File Format

CM1 and VEB Model Formats

Defines a super gauge file, which is a group of multiple gauge objects in a single file.

The super gauge file is a concatenation of standard gauge files. Each gauge file is characterized by its unique combination of defocus and dose parameters, and each gauge file is given a weight value.

Input for: modelflow_v2, CM1 Center

Format

A super gauge data (.sgd) file must conform to the following restrictions:

  • The defocus and dose specify the process conditions at which the corresponding gauge file measurements were performed.

  • The weight specifies the priority of this data set in the cost function used for calibration.

  • Comments are signified by the use of the pound sign (#) in the first position of the line and can be inserted anywhere.

    # this is a comment
version 2
gaugeSetCount number_of_gaugefiles
    
# first gauge file 
gaugeSet 1 {
	dose dose_value1
	defocus defocus_value1
	[type {CD | SRAF | TOPLOSS | HEIGHT}]
	[plane TOP | MIDDLE | BOTTOM | value1 ]
	weight weight1
	gaugeFile gaugefilename1
	opticalDirectory modelpath1a
	secondOpticalDirectory modelpath1b
	thirdOpticalDirectory modelpath1c
		data {
			 gaugefile
		}
}
    
# second gauge file
gaugeSet 2 {
	dose dose_value2
	defocus defocus_value2
	[type {CD | SRAF | TOPLOSS | HEIGHT }]
	[plane TOP | MIDDLE |  BOTTOM | value2 ]
	weight weight2
	gaugeFile gaugefilename2
	opticalDirectory modelpath2a
	secondOpticalDirectory modelpath2b
	thirdOpticalDirectory modelpath2c
		data {
			gaugefile
		}
}…

Parameters

version {1 | 2}

Specifies the version number. Version 1 is used for releases prior to 2014.2. Version 2 was added in Calibre WORKbench 2014.3 in order to support SRAF calibration.

gaugeSetCount number_of_gaugefiles

Specifies the total number of gauge files included. Cannot be greater than 128.

gaugeSet numberN

Specifies a sequential gauge file number (index) ranging from 1 to number_of_gaugefiles.

dose dose_valueN

Specifies the dose value associated with a given gauge file. (dimensionless)

defocus defocus_valueN

Specifies the defocus value associated with a given gauge file, in microns.

Note:

Each dose and defocus pairing in the super gauge data object must be unique.

type { CD | SRAF | TOPLOSS | HEIGHT}

Specifies the type of gauge for the defined gaugeSet. This option was added in version 2 super gauge files.

  • CD specifies a standard gaugeSet.

  • SRAF specifies that the gaugeSet is an SRAF gaugeSet.

  • TOPLOSS specifies that the gaugeSet is a 3D resist profile when measurement is taken at multiple image planes.

  • HEIGHT specifies that the gaugeSet is a 3D resist profile where the measurement is taken at the tops of the resist (height gauges).

plane TOP | MIDDLE | BOTTOM | value

Specifies that the def_start for the plane is independently measured and calibrated at a set spot in the resist (bottom, middle, or top of the resist, or at the specified value in nm measured from the bottom of the resist).

The following combinations of the type and plane arguments are supported:

  • [type CD], plane not specified — Regular CD gauges (“type CD” is optional here) as the standard notation from SGDv1.

  • type CD, plane TOP, MIDDLE, BOTTOM — CD gauges measured at the top, middle, or bottom of the resist respectively.

  • type CD, plane value — CD gauges measured at the specified metrology plane (value in um measured from the bottom of resist). This supports the R3D model calibration flow using CD data measured at multiple metrology planes using a version 2 super gauge data file.

  • type SRAF, plane TOP — Designates SRAF gauges near the resist top (where the exact plane value is a calibration parameter, initialized by the -sraftopplane parameter).

  • type SRAF, plane MIDDLE — Not allowed.

  • type SRAF, plane BOTTOM — SRAF gauges near the resist bottom (where the exact plane value is a calibration parameter, initialized by the -srafbotplane parameter).

  • type TOPLOSS — Does not work with plane.

  • type HEIGHT — Does not work with plane.

weight weight_valueN

Specifies the weight of the gauge file data in the cost function. This is a dimensionless positive quantity.

gaugeFile gaugefilenameN

Specifies the name of the original gauge file the data came from. This information is maintained only for the CM1 Center to reference and has no effect on optimization results.

Note:

Each gaugeSet can have its own set of up to three optical model directories, each of which contains a relevant optical model. These directories are designated Na, Nb, and Nc in this parameter description, where N is the gaugeSet number.

opticalDirectory modelpathNa

Specifies an optional directory to read the optical model associated with the gauge object.

secondOpticalDirectory modelpathNb

Specifies an optional second optical model directory associated with the gauge object.

thirdOpticalDirectory modelpathNc

Specifies an optional third optical model directory associated with the gauge object.

data ‘{’ gaugefile ‘}’

Specifies the gauge file data associated with a gauge set.
