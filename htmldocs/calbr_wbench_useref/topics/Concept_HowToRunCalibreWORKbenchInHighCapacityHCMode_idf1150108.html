How to Run Calibre WORKbench in High Capacity (HC) Mode

Use the -HC option for faster loading of large layouts. 
calibrewb ‑HC  filename

This option invokes the layout viewer in High Capacity (HC) mode, which provides improved performance (over standard mode) when loading and working with large layouts. HC mode consumes less memory than standard mode.

When a layout file is opened in High Capacity mode, the name of the current layout appears in the following places in the Calibre WORKbench GUI:

  • At the top of the GUI next to the Calibre WORKbench release version.

  • At the bottom of the GUI in the Status Toolbar.

The string “(HC)” appears after the layout name.

Refer to the Calibre DESIGNrev Layout Viewer User’s Manual for information on how to use High Capacity mode.

There are differences between running Calibre WORKbench in standard mode and in HC mode. These include the following:

  • Some options in the Tools, Litho, and Optics menus are unavailable in HC mode. The RET Flow Tool, RET Flow Tool v2.0, and Load Flare Map options in the Litho menu are fully supported in HC mode.

  • Some layout viewer batch commands are not supported or are partially supported. Refer to “Unsupported Commands and Arguments in High Capacity (HC) Mode” in the Calibre DESIGNrev Reference Manual for information.

  • HC mode is configured to support the behavior of specific layout exception severity settings. When opening a layout in HC mode, the transcript identifies the exception severities defined in the readerprefs file, generates a warning for each severity that it ignored in HC mode, and identifies if a different severity setting is being used. Refer to “readerprefs File Format” in the Calibre DESIGNrev Layout Viewer User’s Manual for information on supported severities in HC mode.

  • License requirements are different. Calibre WORKbench High Capacity (HC) mode requires one Calibre WORKbench license and one Calibre DESIGNrev license. Refer to the Calibre Administrator’s Guide for more information on licensing these products.
