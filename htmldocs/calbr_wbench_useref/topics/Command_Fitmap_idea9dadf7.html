fitmap

Optical Model Commands

This command calculates the parameters needed to represent the specified source map as a composite source.

Usage

fitmap -in filein -type ‘{’
comptype parameter min max [parameter min max] …
[, comptype parameter min max [parameter min max] …]
‘}’
-profile profiletype [-out fileout] [-filter sigma_low sigma_high] [-limit {1 | 0}]
[-method {0 | 1 | 2}] [-num_iters iterations] [-memory {0 | 1}] [-turbo {0 | 1}]

Arguments

-in filein

A required argument and value pair specifying the source map file to be fit.

-type ‘{’ comptype parameter min max [parameter min max] …

[, comptype parameter min max [parameter min max] …] ‘}’

A required argument and value set defining the type(s) of the composite source that is used to fit the input map. Allowed values for comptype are:

  • annular — Optimizes sigma and sigma_in.

  • quasar — Optimizes sigma, sigma_in and illumangle.

  • standard — Optimizes only sigma.

  • dipole — Optimizes sigma_cntr and radius.

  • quad — Optimizes sigma_cntr and radius.

  • cross_quad — Optimizes sigma and sigma_in.

  • asml_pole — Optimizes sigma, sigma_in, and illumangle.

  • asml_dipole — Optimizes sigma, sigma_in, and illumangle.

Composite sources can be calculated using fitmap:

  • Each composite source type can have multiple parameters specified for it; each parameter must have its own related min and max value.

  • A composite source type must have min and max values specified for all parameters associated with that comptype as shown above. For example, a comptype of quasar requires three parameter min and max settings (sigma, sigma_in, and illumangle).

  • Although the dipole, asml_pole, asml_dipole, and quad comptypes accept a rotangle parameter, rotangle will not be optimized. If rotangle is provided, its max and min values must be set equal. The default value used for rotangle min max is 0 0.

  • Any parameter whose min and max values are equal will not be optimized.

  • The full list must be enclosed in curly braces ({ }).

  • Additional comptypes after the first must be separated with commas.

-profile profiletype

A required argument and value pair defining the profiletype for the source. Allowed values are as follows:

  • erf — Optimizes the “d” parameters as well as the base parameters. The “d” parameters (dsigma, dsigma_in, dillumangle) define the transition length. They are also referred to as the user-defined delta ranges, in which the intensity rises from 0.5% to 99.5%.

  • sharp — Optimizes only the base parameters as needed for the -type.

-out fileout

An optional argument and value pair instructing the application to write the sourcemap that results from fitting to the specified file. When not specified, the application does not save the sourcemap.

-filter sigma_low sigma_high

An optional argument that when specified filters the input sourcemap such that only content between sigma_low and sigma_high is kept. Content outside the boundaries is set to zero. This argument is useful when the input source map describes a composite sourcemap with more than one element, as the original source map can be filtered for each individual element over multiple executions to find the best values for each element.

-limit {1 | 0}

An optional argument and value pair defining the type of optimization, constrained or unconstrained, to perform. Allowed values are as follows:

  • 1 — Constrained optimization.

    • Limits sigma, dsigma, sigma_in, and dsigma_in, to values between 0 and 1.

    • Limits illumangle and dillumangle to values between 0 and 90 degrees (180 for asml_dipole and 360 for asml_pole sources).

  • 0 — Unconstrained optimization.

The default is 1.

-method {0 | 1 | 2}

An optional argument that sets the optimization engine to Quasi-Newton1(0), steepest descent (1), or front (2, the default).

-num_iters iterations

An optional argument that sets the maximum number of iterations to run each combined search and optimization pass. The default is 8000.

-memory 0 | 1

An optional argument. If set to 1, fitmap uses a limited memory model for calculations.

-turbo 0 | 1

If set to 1, allows multi-threading.

Return Values

fitmap returns a string containing the parameters of the composite source that best approximate the input source map (see Figure 1 for a comparison of original source map versus fitted equivalent).

The output of this command is a string that lists the fitting for each of the composite source types that you specify, relevant to the input source map, using the following syntax:

-compositetype { weight n param1 setting param2 setting param3 setting}

followed by a -fit result value for the source.

For a complete description of how these parameters are used to define a source map, refer to “Syntax for Specifying a Composite Source”.

Description

To calculate the source map, the command first performs a full-factorial search for the main (non-smoothing) parameters with a step value of 0.1 to find a preliminary solution. It then switches to a default front search optimization for as many as seven parameters of the composite source: sigma out, sigma in, delta sigma out, delta sigma in, illumination angle, delta illumination angle, and weight. The optimization objective F is the norm-1 difference between input source map S and composite source map C:

Examples

Example 1

The following example shows a single composite source type:

fitmap -in source_map_file.src -type {annular sigma 0.0 1.0 \ 
	sigma_in 0.0 1.0 dsigma 0.0 1.0} -profile erf

It outputs the following solution:

-annular {weight 0.708 sigma 0.895 sigma_in 0.555} -fit 0.0370

Example 2

The following example shows multiple composite types:

fitmap -in source_map_file2.src -type {standard sigma 0.0 1.0, \
	annular sigma 0.0 1.0 sigma_in 0.0 1.0 dsigma 0.0 1.0} -profile erf

It outputs a string similar to the following:

-standard {weight 1.0 sigma 0.55} -annular {weight 0.708 sigma 0.895  sigma_in 0.555} -fit 0.0370
