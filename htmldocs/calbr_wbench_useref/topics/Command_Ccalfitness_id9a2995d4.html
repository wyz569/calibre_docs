ccalFitness

VT5 Resist Model Commands

Runs a fitness check on a layout and assorted contour layers.

Usage

ccalFitness {-m filename | -l handle}
{ –f setup | -align_only [-outfile filename]}
{–cli clifile | -clistring clistring}
[-refmode {0 | 1 | 2 [-simpixsz pixelsizenum]}]

Description

The output from the ccalFitness command appears in the terminal window, returning an overall fitness value and sum of the weights. In addition, four layers are added to the open layout corresponding to the simulation results, weight values, EPE values, and leader lines.

This command can only be used for VT5 modeling.

Arguments

-m filename

Is a required argument specifying GDS / OASIS layout file containing contours.

-l handle

Is the required WORKbench layout handle if you are using an already-loaded layout file.

(One of -m or -l must be specified.)

–f setup

Is a required OPCpro setup file name, specifying models to use for simulation. This option is mutually exclusive with the -align_only option.

-align_only [‑outfile filename]

Is a required argument that changes the input requirement in the contour layer info file to be pre-simulated contours. This option skips the need for optical and resist models since no simulation is performed by the ccalFitness command.

The optional -outfile argument writes the alignment information file to the disk.

–cli clifile | -clistring clistring

Specifies a required input contour layer info .cli file or an equivalent string in Contour Layer Info (CLI) File Format. This file must specify the total number of contour layers (contour data sets) available, and for each contour data set it must specify the contour layer number, clip layer number, dose, defocus, and weight values of the process condition associated with this contour layer, as well as the weight of each contour point. If the process condition (dose and defocus) associated with a given contour layer is not present in the input super gauge file, this contour data will be neglected.

-refmode {0 | 1 | 2}

Selects simulation method options used to compare to SEM contours:

  • 0 — original layout (mask). This is the default.

  • 1 — aerial image contour (threshold selected to minimize error)

  • 2 — resist print image

-simpixsz pixelsizenum

Is the simulation pixel size (in um). This setting is only used when refmode 2 (dense print image) is specified.

Examples

ccalFitness { -m contour_layout.gds} –f contour_setup.in 
	-cli contour_example.cli –refmode 1
ccalFitness { -m contour_layout.gds} –f contour_setup.in
		-cli contour_example.cli 
		-refmode 2 -simpixsz 0.01
