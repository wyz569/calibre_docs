sgd_get_flare

CM1 and VEB Commands

Computes flare values from a specified super gauge data object.

Use the sgd_get_flare command to get flare values.

Usage

sgd_get_flare {–obj object | –sgf file} –flare file –png file –precision value
{‑bg clear | dark} [‑optstr string | –optfile file | –optdir dir] [–out file]

Arguments

–obj object | -sgf file

A required argument specifying the input single or super gauge data Tcl object or file.

–flare file

A required argument specifying an input flare model.

–png file

A required argument specifying an input PNG file.

–precision value

A required argument specifying the input precision for converting gauge coordinates from dbu to microns.

–bg clear | dark

A required argument specifying the input background type. The accepted types are clear and dark.

–optstr string | -optfile file | -optdir dir

An optional argument specifying the input optical model as an inline string, file, or directory.

–out file

An optional argument specifying a file name output. If not specified, results are returned in the input gauge object.

Description

Flare values are computed at the mid-gauge points and the results are written to the gauge object’s Comment column. The background intensities required for computing flare values are determined using one of the following methods:

  • From the input nominal optical model, in which case optical models for off-focus process conditions are generated at runtime,

  • Using sgd optical links, which must be present in all process conditions.

If neither input optical model nor sgd optical links are present, background intensities are determined using process condition dose values.

Examples

sgd_get_flare -sgf gauge.sgd -flare flare.mod -png flare.png \ 
	-precision 10000 -bg dark -optfile optical.txt -out out.sgd
