Calibrating the Resist Model for 3D Resist Modeling

Follow this task to calibrate a CM1 resist model for use with the 3D resist optical model.

Using the optical model from “Calibrating the Optical Model With Height Gauges and CD Gauges” to calibrate a resist model results in a model that is typically different than a regular (width-based) OPC model.

Note:

This task is the “modelflow_v2 stage 2” box in Figure 1 of “Overview of the 3D Resist Model Calibration Flow.”

Prerequisites

  • The modelflow_v2 script, model-layer file, and optical model output from “Calibrating the Optical Model With Height Gauges and CD Gauges”

Procedure

  1. Edit the model-layer file to use the output optical model from the height calibration.
  2. Edit the modelflow_v2 script, making the following changes:

    • Add the parameter to save the calibrated CM1 model to the output:

      	output \		
		finalresist cm1mf_cal.mod \
		...

    • Remove the calibration parameters from stage1. This leaves only the following code for the stage:

      	stage1 dense \
		threshold optimize
		...
      Tip

      If you are using a Super Gauge Data File Format version 2 file, any gauge set that does not have a “plane” field specified (such as for a SEM CD gauge set) uses the defocuslevels or imageplanes value in the optical model.

      If only CD gauges are used (none of the gauges have their “plane” setting specified), CM1 Center uses the defocuslevels setting in the optical model instead of imageplanes.

    • Modify stage2 to include a either a modelform 22 CM1 model (PTD) or 44 or 48 CM1 model (NTD). 

      	stage2 \
		search frontzoom \	
		explore 20000 \	
		epsilon 0.001 \
		maxaidiff 10 \
		thresholdstability 4 \
		cm1mf 22 \
		dense]
  3. If you are using an NTD model (modelform 44 or 48), also make these changes:

    • If the optical model used vd_top_shrink_a, vd_top_shrink_l, vd_top_surfc_a, or vd_top_surfc_l, remove the parameters.

    • Modify stage2 to include zslope and vshrink terms. The zslope calibration requires CDs at three planes when using all three coefficients. The vshrink calibration requires height gauges.

      	stage2\
		...
		zslope_a -10 10 10 \
		zslope_b -10 10 10 \
		zslope_c -10 10 10 \
		vshrink_v 0 0.4 10\
		vshrink_l 0.001 4 10 \
		vshrink_t 0.001 1 10 \
		vshrink_d 0.001 0.1 10 \
		vshrink_s 25 250 10 ]

    • To model vertical shrink, verify that the input CM1 model includes VERTICAL_SHRINK and Z_SLOPE parameters. See “CM1 Model File Format” for details on the keywords.

      	...
resistpolyfile inline
inline_resist {
	version 7
	modelType CM1
	DEFINE M0 k=0 ...
	...
	Z_NOMINAL 1.50e-02
	Z_SLOPE a=-6.40 b=-9.60 c=-8.00
	VERTICAL_SHRINK u=0.30 v=0.10 l=0.20 t=0.31 d=0.01 s=100 c=1
}

    • Modify the modelflow_v2 script to provide height gauges (‑heightsgf parameter) and optionally the number of planes for the height gauges (‑heightznum parameter).

  4. Save the file and run the calibration.

    If you are using height gauges, you should use at least 16 CPUs. If a large enough computer is not available, run the calibration in two steps:

    1. First without vshrink, and no height gauges.
    2. In the CM1 model produced by step a, add vshrink.
    3. Calibrate only vshrink with height gauges. Keep other CM1 parameters as set by step a.

Results

The modelflow_v2 run produces a calibrated CM1 resist model.

If your RMS error is too large (> 5 nm), you can adjust the calibration parameters as follows:

  • Manually change the optimized def_start and beamfocus values in the modelflow_v2 script. Note that increasing these values usually increases the 3D resist results.

    • If simulated 3D resist is greater than the measured 3D resist on average, decrease def_start by approximately the average difference.

    • If the simulated 3D resist is less than the measured 3D resist on average, increase def_start by approximately the average difference.

  • Inspect gauges for mistakes and outliers. Common mistakes include the following:

    • Height gauges not centered properly.

    • The AFM profile from which CDs were obtained was not properly decoupled.

    • CD inconsistencies on similar features when measured with AFM, SEM, or XSEM. You should estimate metrology offsets and account for them in the input gauges.

  • Selectively weight the gauge sets. Give the highest weight (1) to CDSEM, and decrease the weight the closer the gauge set is to the top. For example, give the middle CDs a weight of 0.5, and the top CDs a weight of 0.1.
