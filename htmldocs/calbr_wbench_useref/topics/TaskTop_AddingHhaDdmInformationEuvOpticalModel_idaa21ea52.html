Adding HHA and DDM Information to the EUV Optical Model

An optical model file cannot fully model the offsets required for mask edge movement. Calibre WORKbench uses a separate domain decomposition model (DDM) file to simulate 3D mask reflection. DDM models precompute rigorous edge responses to incident plane waves.

To add HHA and DDM information to your EUV optical model, you first edit the optical model file, and then run Calibre MaskSim to model the effects at the center of the slit.

Prerequisites

  • The optical model from “Creating an EUV Optical Model File”

Procedure

  1. Using a text editor, open the optical model file, Optical. This file is typically located in the optical model directory.
  2. If the opticalmodel keyword is 9 or less and if the forcePower2LUT setting is specified as non-zero, change it to 0. 
    forcePower2LUT 0
  3. At the end of the optical model file, add the following keywords:
    ddm_model approx

    The approx DDM keyword can be used for unpolarized sources, such as EUV.

    hha 1

    This keyword activates the HHA subroutines built into opticsgen. It allows for multiple TCC matrices.

    After the hha keyword, specify the hha_sectorization keyword with the energy argument:

    hha_sectorization energy 0.2
  4. Save the file and regenerate it using the following command: 
    calibrewb -a opticsgen -opticalfile filename -o new_directory_name
  5. Open a new text file and create a DDM library input file. Use the following format and keywords (as described in the section “DDM Library Input File Format”):

    • make_ddm_edge_library

    • wavelength val_in_um

    • film thickness n k “comment” [local_slope_in_degrees]

    • euv_multilayer {numbilayer | numbaseunits} num [bottom thickness n k] [top thickness n k]

    • capping_layer thickness n k

    • objectplane {val_in_um | imageshift}

      Note:

      For universal DDM libraries with version 4 or later, set the objectplane value to 0. Alternatively, remove the objectplane line for universal DDM library generation.

    • illumination path_to_optical_file | {universal [dsigma dsigma] [dphi dphi] [na NA]}

    • chief_ray nxe3400_throughslit | {theta theta_value phi {phi_value | {‘[’phi_list‘]’ center_slit_phi phi_list_value}}}

    • {bright_outputfile fname} | {dark_outputfile fname}

    • compute_diagonals {0 | 1}

      “compute_diagonals 1” is recommended for EUV.

  6. Optionally add crosstalk input parameters as appropriate to your DDM library input file.
    Note:

    Crosstalk modeling in EUV requires Calibre version 2016.1 UR2 or higher to function properly. DDM libraries generated with EUV crosstalk_model using the baseline Calibre 2016.1 version may be invalid due to incorrect crosstalk values of 0 for some sectors.

    • crosstalk_model {none | bright_gap | dark_gap | all}

    • crosstalk_range [auto |

      [bright_gap ‘[’ r1low[:r1step:r1high] [r2low[:r2step:r2high]] … ‘]’]

      [dark_gap ‘[’ t1low[:t1step:t1high] [t2low[:t2step:t2high]] … ‘]’]

  7. Save the file.
  8. In the console shell prompt, enter the following command:
    calibrewb -remotefile remote_file -a masksim -i input_file \
 -o sim_outfile

Results

You know the run is complete when the shell prompt returns. One or two DDM files are created in the working directory, depending if you specified bright_outputfile, dark_outputfile, or both.

Calibre WORKbench generates one or more DDM files that can be used as input to other Calibre tools.
