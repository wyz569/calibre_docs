forcePower2LUT

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Controls the allowed relationship between the optical diameter and the kernel grid size.

Usage

forcePower2LUT { 1 | 0 }

Arguments

1

An optional argument that forces the optical diameter to be a power of two multiple of the kernel grid units. This is the default previous to optical model version 9.

0

An optional argument that allows the optical diameter to be any integer multiple of the kernel grid. This is the default starting with optical model version 9.

Description

Note:

This option can no longer be set starting with optical model version 9; use the forceKernelSymmetry parameter instead. It is automatically set to 0. Previous versions of the optical model allow the use of this parameter.

An optional SOCS/Kernel parameter controlling the allowed relationship between the optical diameter and the kernel grid size. It has a direct effect on the hoodpix flag, as follows:

  • 1 (default previous to version 9) — A potential slowdown can occur in the TCC calculation phase when your optical diameter divided by the kernel grid is not a power-of-2 if it rounds the optical diameter upward significantly to meet the restriction.

  • 0 (default starting with version 9) — Since the optical diameter can be set to be any integer multiple of the kernel grid, non-power-of-2 values can slow down the FFT calculation time, since FFT calculation is optimized for power-of-2 multiples.
