Optical Model Generation Methods (GUI Versus Optical Parameters File)

While the general flow of optical model creation is the same, Calibre WORKbench makes two methods of optical model creation available:

  • Method 1 - Use the Optical Model GUI

  • Method 2 - Use the opticsgen Batch Command with an Optical Parameters File

Method 1 - Use the Optical Model GUI

Access the Optical Model GUI through the Optics menu. The Optical Model GUI provides the fastest method to create an optical model, since many complex optical model fields are reduced to single choice controls. It is recommended that new users start with the Optical Model GUI in order to generate optical models.

Creating an optical model requires you to fill in information in the first five tabs of the Optical Model Tool: Main, Kernels/Image, Illumination, Pupil, and Film Stack. The contents of the Optical parameters file created are visible in the Preview tab. Advanced features can be entered in the Misc tab. Once the first five tabs are filled with properties relevant to the optical system of interest, clicking the Generate button creates the optical model.

Details on this method are described starting with “Step 1 - Configuring the Main Optical System Parameters”.

Using the Preview Tab to Check Your Progress

When you make changes to the Optical Model Tool controls, Calibre WORKbench immediately adds or removes keywords in the Preview tab (Figure 1). You can also use this tab to learn which GUI elements correspond to keywords and settings.

Method 2 - Use the opticsgen Batch Command With an Optical Parameters File

Creating an optical model by entering multiple parameters in a text file (used as an argument to the opticsgen batch command) provides more comprehensive control over the optical model parameters, but can be more difficult to use due to the large number of choices available. Advanced users can study the Optical Parameters File Format to make the most effective use of the features available in the optical model.

Alternatively, an optical model definition text block can be inserted inside a Calibre WORKbench setup file instead of pointing to an optical model directory; this has the advantage of generating the model during runtime.
