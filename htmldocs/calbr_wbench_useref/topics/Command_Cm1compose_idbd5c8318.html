cm1compose

CM1 and VEB Commands

Related Formats: CM1 Model File Format

Creates a template CM1 object based on a modelform.

Usage

cm1compose {–m modelform_number [-polarity p -rm3d num] | -cm1 cmiobj} [–o filename]
[-optimvar vname] [‑optimtcl name][‑description var] [‑mfv 2 | 3]

Arguments

–m modelform_number

A required argument specifying the input modelform number (see also “List of Modelforms”). The accepted values are as follows:

  • 0 — CTR model.

  • 1 — diffused aerial image.

  • 2 — diffused aerial acid.

  • 3 — double Gaussian diffusion.

  • 10 — complex modelform with image intensity as anchoring term, acid, and bases.

  • 11 — complex modelform with image intensity as anchoring term, double Gaussian diffusion, acid, and bases.

  • 20 — modelform 11 plus slope.

  • 21 — modelform 11 plus slope, Imax, and Imin.

  • 22 — modelform 11 plus slope, Imax, Imin, and Laplacian.

  • 28 — modelform 22 with multi-range SEM shrink.

  • 29 — modelform 22 with multi-range SEM shrink and SEM electron absorption.

  • 31 — diffused acid and base with 8 basis functions.

    Note:

    Starting from the 2018.1 release, this modelform is no longer available from the CM1 Center and Calibre nmModelflow GUI, but can be specified using various batch commands.

  • 38 — modelform 21 with doubled b-terms plus shrink and horizontal bias, used for negative tone developer (NTD) configurations.

  • 40 — modelform 22 with sidewall bias (used for NTD).

  • 41 — modelform 22 with dual polarity shrink (used for NTD).

  • 42 — modelform 22 with shrink terms (used for NTD).

  • 43 — modelform 22 with horizontal bias (used for NTD).

  • 44 — modelform 22 with shrink and horizontal bias (used for NTD).

  • 45 — modelform 22 with shrink and sidewall bias (used for NTD).

  • 46 — modelform 22 with dual polarity shrink and sidewall bias (used for NTD).

  • 47 — modelform 22 with single polarity shrink, horizontal bias, and sidewall bias (used for NTD).

  • 48 — modelform 22 with doubled b-terms, plus dual polarity shrink and horizontal bias (used for NTD).

  • 49 — modelform 22 with fixed smoothing length, doubled b-terms, plus dual polarity shrink and horizontal bias (used for NTD).

  • 50 — modelform 10 with eight-fold dissolution, contour curvature, double pattern density, and triple crossterms (used for NTD).

  • 52 — modelform 22 with sidewall bias, contour curvature, six-fold dissolution, and tuning of the acid anchor term, M0. This model is available in nmModelflow only and cannot be calibrated in modelflow_v2 (used for NTD).

  • 58 — modelform 22 with doubled b-terms, dual polarity shrink, horizontal bias, and sidewall bias (used for NTD).

  • 59 — modelform 22 with fixed NTD smoothing terms, doubled b-terms, dual polarity shrink, horizontal bias, and sidewall bias (used for NTD).

  • 60 — modelform 22 with eight-fold dissolution, contour curvature, double pattern density, and eight crossterms (used for NTD).

  • 61 — modelform 22 with nine-fold dissolution, contour curvature, and 11 crossterms (used for NTD).

  • 78 — modelform 22 with doubled b-terms, quadruple polarity shrink, dual horizontal bias, and sidewall bias (used for NTD).

  • 79 — modelform 22 with fixed NTD smoothing terms, doubled b-terms, quadruple polarity shrink, dual horizontal bias, and sidewall bias (used for NTD).

Note:

Modelforms 88 and 89 require SHRINK_SIDO be turned on (i.e. set to 1). By default, SHRINK_SIDO is off (0).

  • 88 — modelform 22 with doubled b-terms, quadruple polarity shrink with SHRINK_SIDO, dual horizontal bias, and sidewall bias.

  • 89 — modelform 22 with fixed NTD smoothing terms, doubled b-terms, quadruple polarity shrink with SHRINK_SIDO, dual horizontal bias, and sidewall bias.

  • 90 — modelform 22 with fixed NTD smoothing terms, doubled b-terms, quadruple polarity shrink, horizontal bias, sidewall bias, contour curvature, and quintuple dissolution. This model is available in nmModelflow only and cannot be calibrated in modelflow_v2 (used for NTD).

  • 91 — modelform 22 with fixed NTD smoothing terms, doubled b-terms, quadruple polarity shrink, horizontal bias, sidewall bias, contour curvature, quintuple dissolution, and tuning of the acid anchor term, M0. This model is available in nmModelflow only and cannot be calibrated in modelflow_v2 (used for NTD).

  • 92 — modelform 22 with fixed NTD smoothing terms, doubled b-terms, quadruple polarity shrink, horizontal bias, sidewall bias, contour curvature, quintuple dissolution, pattern density, and 11 crossterms. This model is available in nmModelflow only and cannot be calibrated in modelflow_v2 (used for NTD).

  • 120 — combination of modelform 22 and 12.

    Note:

    Starting from the 2018.1 release, this modelform is no longer available from the CM1 Center and Calibre nmModelflow GUI, but can be specified using various batch commands.

  • 144 — modelform 120 with shrink and horizontal bias (used for NTD).

    Note:

    Starting from the 2018.1 release, this modelform is no longer available from the CM1 Center and Calibre nmModelflow GUI, but can be specified using various batch commands.

  • 130 — Deprecated in 2010.3. If you receive the error: “modelflow_v2: CM1 smooth maximum base deprecated due to consistency issues as of 2010.3”, you must recalibrate your CM1 model using a different modelform (21 or 22 are recommended).

-polarity p -rm3d num

An optional argument set used with 3D resist model configurations.

  • -polarity — Specifies the polarity (resist_polarity parameter) of the model. Use 1 for PTD resists, and -1 for NTD resists.

  • -rm3d — Specifies the 3D resist modelform to use. Only four modelforms are available for 3D resists:

    • 0 — none

    • 1 — z_slope

    • 10 — vertical_shrink (NTD)

    • 11 — vertical_shrink + z_slope (NTD)

    The 3D resist modelforms 1 and 11 use z_slope, and require a slope term in the CM1 modelform they are used with; therefore, -rm3d settings of 1 and 11 cannot be used with a CM1 modelform below 20, or modelform 31.

    The 3D resist modelforms 10 and 11 use the vertical_shrink term, and therefore require a -polarity setting of -1 and a modelform that uses NTD.

-cm1 cm1obj

A argument that can be used in place of the -m argument. This option uses the modeling coefficients in the CM1 model supplied in cm1obj.

Note:

Optimization settings generated with cm1compose for the same CM1 model using the ‑m and –cm1 options may differ from each other. This is because the ‑m optimization settings for a particular model form intentionally keep some of the nonlinear parameters fixed at their nominal values. When ‑cm1 is used, the optimization settings are generated under the assumption that each parameter should be optimized.

–o filename

An optional argument that specifies the output file name. Use this option to output the CM1 model to a file.

-optimvar vname

An optional argument that specifies a Tcl variable name. Outputs CM1 model optimization parameters to the specified Tcl variable.

‑optimtcl name

An optional argument that outputs the CM1 model optimization settings in Tcl format to a Tcl variable specified by name. The Tcl-format settings can be pasted into a modelflow_v2 script.

‑description var

An optional argument that outputs a short description of the specified CM1 modelform to a Tcl variable specified by var.

‑mfv 2 | 3

An optional argument specifying which version of modelflow to use to create the CM1 template. The default is 3 (Calibre nmModelflow). To use modelflow_v2, specify ‑mfv 2.

Return Values

If successful, cm1compose returns TCL_OK and any specified Tcl output.

If not successful, cm1compose returns TCL_ERROR.

Description

Use the cm1compose command in the Calibre WORKbench Tcl shell to create a template CM1 object and optionally output the template model to a file.

Examples

#create cm1 object of modelform 0
set cm1obj1 [cm1compose –m 0]
#create cm1 object of modelform 11 and dump this template to a file
set cm1obj2 [cm1compose –m 11 –o “cm1.mod”]
#output the optimization settings for an existing CM1 object
cm1compose -cm1 cm1obj3 -optimtcl tcl_str_out
