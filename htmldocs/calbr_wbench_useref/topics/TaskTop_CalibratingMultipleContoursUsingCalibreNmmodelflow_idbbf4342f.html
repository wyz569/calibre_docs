Calibrating Multiple Contours Using Calibre nmModelflow

Calibre nmModelflow can be used to calculate an average contour from a set of multiple measured contours taken at different dose and defocus settings. The averaging is a weighted calculation, where contours that deviate too far from the average are excluded from the final results.

Prerequisites

  • Calibre WORKbench invoked

  • Calibre nmModelflow invoked in Calibre WORKbench (Litho > nmModelflow)

    Tip

    For more information on the Calibre nmModelflow tool, see the Calibre nmModelflow User’s and Reference Manual.

  • A layout design file with the following characteristics:

    • A target layer

    • A single contour layer containing all averaging contours

    • A clip layer containing markers over areas of interest

  • A litho model containing at least an optical and resist model

  • A Calibre nmOPC litho setup file containing a link to the litho model and the basic aerial image generation commands, similar to the following:

    modelpath litho_models
layer L0
tilemicrons 100.0
image_options MDF {
  layer L0 visible mask_layer 0
  litho_model theLithoModel
}
setlayer im0 = image MDF mask 0
setlayer im0.ai = image MDF mask 0 aerial 0.20

Procedure

  1. In a text editor, create a Contour Library Information (CLI) file containing at least the following parameters (see “Contour Layer Info (CLI) File Format”):
    Table 1. CLI Suggested Parameters for Averaging Multiple Contours

    Parameter

    Suggested Setting

    Notes

    version

    1

    Must always be set to 1.

    contourSetCount

    1

    Only one contour set layer is required.

    The contourSet should contain the following items:

    contourlayer

    contour_layer_number do_averaging

    The do_averaging keyword activates contour averaging on the specified layer.

    cliplayer

    cliplayer_number

    A clip layer contains clipping rectangles that cover the calibration contours.

    dose

    1.0

    Specify the nominal dose.

    defocus

    0.00

    Specify the nominal defocus.

    weight

    1

    A weight value of 1 indicates that this is a contour-only calibration; no gauge file is needed.

    outmeasuredlayer

    a_new_layer_number

    A generated layer that is used by Calibre ContourCal to place its averaging contours.

    outweightlayer

    a_new_layer_number 0.04

    A generated layer that is used by Calibre ContourCal to place its weight blocks. The value of 0.04 is the size of a side of the weight block in nm.

    outweightrange

    -2 1

    Sets the range of the weight values used in the weight block assignments. The largest deviation value gets the first value (-2, meaning “ignore”), and the smallest deviation gets the second (1, meaning “full weight”).

    outstartlayer

    a_new_layer_number 3

    A layer number used by Calibre ContourCal as a starting layer to write multiple layers containing output contours, weight values, and EPE results.

    Save the file.

  2. In the Calibre WORKbench main window, ensure the Options > Preferences, Objects tab, Text section, Height option is set to Fixed (no text attributes).
  3. In the Calibre nmModelflow Command window, load the design file by entering the command:

    mdf contour layout filepath

  4. Identify the target layer in the design file to Calibre nmModelflow.

    mdf contour layers target_layer_number

  5. Load the CLI file into Calibre nmModelflow.

    mdf contour cli path_to_clifile

  6. Load the Calibre nmOPC setup file into Calibre nmModelflow.

    mdf setupfile load path_to_setup_file

  7. Run the Calibre nmModelflow simulate command using the design and setup.

    mdf simulate write "result.txt" -output_contours 1

Results

Calibre nmModelflow adds the following layers to your design file as shown in the figure:

  • A measured contour layer with the average of all input contours (s1_mprint layer)

  • A layer containing weight blocks (s1_weight layer)

    The weight block layer is provided for diagnostic purposes and can usually be hidden.

  • Six contour calibration output layers:

    • A simulation print layer (sprint)

    • A weight value layer (s2_weight)

    • A measured to simulated EPE value layer (m2s_epe)

    • A set of leader lines showing the point tested between the measured and simulated contours (m2s)

    • The clipping layer used for the contour (oclip)

      The oclip layer is typically a copy of the clip layer and can be hidden.

    • An alignment measurement layer (aprint)

Note:

You may have to zoom in on the contour to see the EPE and weight values.

Figure 1. Multiple Contours Averaging Results

You can save the resulting contours using File > Save As from the Calibre WORKbench main window, or the following command in the Calibre nmModelflow command window:

mdf contour output layout filename layout-type
