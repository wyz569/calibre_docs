applySourceSymmetry

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Applies symmetry to an optical source map.

Usage

applySourceSymmetry { none | xydiag | xy | x | y } [ avg | max | min ]

Arguments

none | xydiag | xy | x | y

An optional argument set specifying the symmetrization direction. Symmetrization options include the following:

  • x y

    Folds over the respective axis (2 symmetry points).

  • xy

    Folds over both the X- and Y-axes (4 symmetry points).

  • xydiag

    Folds along the diagonals (8 symmetry points).

The default behavior is no source symmetrization performed (none).

avg | max | min

An optional argument set specifying the method for computing the new reported source intensities. Computation methods include the following:

  • avg

    Adds the intensity from all symmetric points. If a point is not present in the original map, it is considered to have an intensity of 0 for the calculation. This is the default.

  • max

    Uses the maximum of all the points as the final intensity.

  • min

    Uses the minimum of all the points as the final intensity.

Description

An optional miscellaneous parameter symmetrizing sourcemaps.

When other options are specified, elements in a sourcemap are flipped to create a symmetric map immediately after a source file is read in.

It is considered an error if the maximum of the intensity difference between the new and original maps is larger than 0.01 * the intensity maximum in the original source map. This error is used to inform the user when the map changes considerably as a result of symmetrization.
