Calibrating Over a Process Window Using Super Spreadsheets

Build more robust optical and VT5 models by sampling a range of process conditions.

Sampling over a range of process conditions widens the IMAX, SLOPE, and other ranges and makes VT5 models more stable.

The general recommendation is to collect data from three to four process conditions, such as two different defocus and two different dose conditions.

Prerequisites

  • A spreadsheet of CD data collected over varying process conditions of defocus and dose.

    • Defocus — Is an offset from the best focus conditions in um. In simulations, it is added to the beamfocus value when beamfocus is present in the nominal model, or to the defocus start (def_start) parameter of the optical model otherwise.

    • Dose — Is a dimensionless parameter specifying the dose ratio to the best dose. It is mapped during simulations to the setup file dose for the first mask (in other words, the setup file dose is multiplied by this dose).

    Defocus and dose can be sampled in an arbitrary manner (shown as the circles in the figure), not necessarily using fixed deltas or covering all corners of the process window.

    Figure 1. Sampling Example

Procedure

  1. Collect all relevant spreadsheet (.ss) files, one per process window sampling. Each .ss file can sample CDs from different locations. The number of lines in each .ss file is independent of other .ss files. However, all locations have to be from the same GDS file.
  2. Concatenate the .ss files into one super spreadsheet file.
  3. Run the modelflow (Version 1) command, specifying the super spreadsheet file to the ‑sss option:
    modelflow … -sss superspreadsheet …
