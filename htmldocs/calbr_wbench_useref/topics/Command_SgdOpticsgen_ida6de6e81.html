sgd_opticsgen

CM1 and VEB Commands

Related Formats: Gauge Object Reference, Super Gauge Data File Format, Optical Parameters File Format

Generates optical models for a super gauge data object.

Usage

sgd_opticsgen {-obj object | -sgf file}
{ -str string | -opticalfile file} -opticaldir modelpath
[{-str2 string | -opticalfile2 file}]
[{-str3 string | -opticalfile3 file}]
[-out file] [-remotefile filename | -remotestr string] [-modelname name] [-nolookuptables]

Arguments

-obj object

A required argument specifying an input sgd tcl object. This option cannot be set if -sgf is specified.

-sgf file

A required argument that specifies an input super gauge data file. This option cannot be set if -obj is specified.

-str string

A required argument specifying an input string or Tcl variable representing the optical model. This option cannot be set if -opticalfile is specified.

-opticalfile file

A required argument specifying an input optical model file. This option cannot be set if -str is specified.

-opticaldir modelpath

A required argument specifying an output pathname to save optical models in.

-str2 string

An optional argument specifying an input string or Tcl variable representing a second optical model. This option cannot be set if -opticalfile2 is specified.

-opticalfile2 file

An optional argument specifying an input optical model file for a second optical model. This option cannot be set if -str2 is specified.

-str3 string

An optional argument specifying an input string or Tcl variable representing a third optical model. This option cannot be set if -opticalfile3 is specified.

-opticalfile3 file

An optional argument specifying an input optical model file for a third optical model. This option cannot be set if -str3 is specified.

-out file

An optional argument specifying a file to write the new or updated sgd object to.

{-remotefile filename | -remotestr string}

Optionally specifies either a remote configuration file (-remotefile) or inline configuration string (-remotestr) for parallel execution of this command. See “Distributed Processing in Calibre WORKbench” for more information.

-remotefile and -remotestr are mutually exclusive.

-modelname name

Optional argument that sets the name of the subdirectories of the defocus values, namenum and secondOpticalnamenum, where num stands for the defocus value in microns (X.XXX or -X.XXX). Default is the string “defocus.”

-nolookuptables

Optional argument that when specified causes sgd_opticsgen to omit the creation of the binary files with the lookup tables for SOCS kernel and mask convolution. In this case, the optical model is comprised of the SOCS kernels (an .fsk file) and the SOCS weights (an *_0_0_0.omf file) and takes up significantly less disk space. Models without SOCS lookup tables are appropriate for dense image simulations.

Description

Use the sgd_opticsgen command to generate optical models for the defocus values specified in the input sgd object or file (which can also be an LFD type optical model). If the -obj argument is specified, this command resets the opticalDirectory fields in the input sgd object. Otherwise, it returns a handle to the new sgd Tcl object with the opticalDirectory fields filled. Optionally, sgd_opticsgen dumps the new or updated sgd object to the specified file. Models are saved to the specified directory under the name defocusnum, where num is the defocus value in microns of format X.XXX or -X.XXX, such as defocus0.000 or defocus-0.160.

Examples

This command loads the sgd file, generates optical models, and saves them to the disk:

# creates sgd object with optical Directory specified, writes this object 
# to file,and returns the object handle to tcl
set obj [sgd_opticsgen -sgf in.sgd -opticalfile optical.txt -opticaldir . -out out.sgd]
