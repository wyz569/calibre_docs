Custom Test Pattern Recommendations

If you are going to create a custom test pattern, your pattern should follow certain requirements.

  • Must have an odd number of long features arranged in a line-and-space configuration.

  • The length of the features must be at least 5 times λ/NA in order to minimize any impact of 2D effects.

  • Features should also not be asymmetrical; Calibre generates its models expecting to find symmetrical features.

  • Do not use only post-OPC (corrected) test patterns for calibration purposes. You can only use an uncorrected test pattern, an uncorrected test pattern with a corrected test pattern (on an external-type layer) or a composite uncorrected and corrected layer (this latter method is used when you can only have one mask layer).

  • If you are making your own test pattern, the feature names should start with the same names as the nearest equivalent in the Calibre WORKbench-generated spreadsheet (line_end, iso, and so on), in order to take advantage of the full analysis capabilities of Calibre WORKbench.

  • All fields in the spreadsheet require proper values, as described in the section “Test Pattern File Format”; note that Calibre WORKbench uses the coordinates you supply to locate features.

A group of these patterns with different main pitches should be created (Figure 1). The more pitches you include in the test pattern, the more data you will have to verify and calibrate the model later on.
