Contour Comprehensive Data Table File Format

Generated using: Contour Layer Info (CLI) File Format

The Contour Comprehensive Data Table is created when a CLI file that contains the keyword contourComprehensiveDataTableFile is part of a model calibration run.

Format

The Contour Comprehensive Data Table File is a comma-separated ASCII spreadsheet. It can be opened in any text editor or spreadsheet program that handles .csv file format.

The table is divided into sections, one per stage. If a modeling stage is turned off, the section is not written to the file.

Each section begins with a label identifying the stage. This is followed by the field headings, and then the data for each point in the contour, as described in the Parameters section. An empty entry line separates the data from the next stage label.

Stage 1:
<Headings>
<Data for stage 1 point 1>
<Data for stage 1 point 2>
...

Stage 2:
<Headings>
<Data for stage 2 point 1>
<Data for stage 2 point 2>
...

Stage 3:
<Headings>
<Data for stage 3 point 1>
<Data for stage 3 point 2>
...

Within each row, commas separate each field. Empty entry rows and stage labels are padded with commas that use 19 fields.

Parameters

Table 1. Contour Comprehensive Data Table File Format Stages

Heading

Description

Layer

The contour layer of the data point. Identified in the CLI file by contourlayer.

Dose

Specifies the dose used. Identified in the CLI file by dose.

Defocus

The defocus used. Identified in the CLI file by defocus.

Block ID

A number identifying the polygon to which this point belongs. All rows with the same block ID are inside the same clipping shape.

X_measured (dbu)

The x coordinate of the SEM-measured point given in database units. In the Contour Information Log Output, this is EMPX.

Y_measured (dbu)

The y coordinate of the SEM-measured point given in database units. In the Contour Information Log Output, this is EMPY.

X_simulated (dbu)

The x coordinate of the simulated point given in database units. In the Contour Information Log Output, this is MODX.

Y_simulated (dbu)

The y coordinate of the simulated point given in database units. In the Contour Information Log Output, this is MODY.

Error (nm)

The difference between the measured and simulated coordinates, translated from dbus into nanometers.

Weight

The weight of the contour set to which this point belongs. Identified in the CLI file by weight.

Imax

The maximum image intensity along the contour measurement line.

Imin

The minimum image intensity along the contour measurement line.

Contrast

The difference Imax - Imin.

Slope_measured (1/um)

The slope of the intensity line as it passes through X_measured, Y_measured.

ILS_measured (1/um)

The image log slope (ILS) as it passes through X_measured, Y_measured.

Slope_simulated (1/um)

The slope of the intensity line as it passes through X_simulated, Y_simulated.

ILS_simulated (1/um)

The ILS as it passes through X_simulated, Y_simulated.

Slope_avg (1/um)

The average slope.

ILS_avg (1/um)

The average ILS.
