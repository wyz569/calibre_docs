Inspecting Run Options and Running the Calibration

This procedure describes how to perform optimization. You may have to run optimization multiple times to find the best fit.

Prerequisites

  • Test pattern layout loaded and active in Calibre WORKbench

  • Required tabs filled in:

    • “Setting VT5 Center Inputs and Outputs”

    • “Loading or Creating a Sample Data Set”

    • “Loading an Optical Model”

  • For optical model optimization: “Setting Optimizations for the Optical Model”

  • For resist model optimization: “Defining a VT5 Input Model” and “Setting Optimization for the VT5 Model”

  • If contour data is to be included: “Including Contour Calibration Information”

Procedure

  1. In VT5 Center, click the Run tab and inspect the contents for accuracy. Note that this tab cannot be edited.
    Figure 1. VT5 Center GUI (Run Tab)

  2. Select how to handle the optical model for this run:

    • To optimize the optical model, switch the Optics selector to (O)ptimize mode as shown in Figure 2.

    • To use the model presently showing in the Optics, in tab, switch the selector to (T)ry mode.

    • To use the optical model defined in the setup file, switch the selector to (X) mode.

      Figure 2. Where to Set Optimization Selectors

  3. Select how to handle resist model for this run:

    • To optimize the resist model, switch the VT5 selector to (O)ptimize mode as shown in Figure 2.

    • To use the model presently showing in the VT5, in tab, switch the selector to (T)ry mode.

    • To use the resist model defined in the setup file, switch the selector to (X) mode.

  4. Save the session file by selecting File > Save Session or File > Save Session As. This is useful in case the simulation crashes for any reason; you can restore the settings you made by using the File > Open Session option.
  5. Optionally, load a remote machine configuration file into the Remote, in tab by using the Remote > Open menu item. For information on remote files, see the section “Distributed Processing in Calibre WORKbench”.
  6. Optionally, export the session as a runset file with File > Export as Runset. This creates a script file that can be used as input to the modelflow (Version 1) batch command.
  7. Click the Run button.

Results

Assuming that you do not get any run-time errors, VT5 Center adds a new line to the Model Flow log section at the bottom of the window. These log files are used in the next task, “Analyzing the Results of a VT5 Center Run”.
