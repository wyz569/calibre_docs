Test Pattern Overview

Test patterns are predetermined groups of geometries that do not have a circuit function, but when broken up and recombined with other test pattern geometries can be made to resemble functional design elements.

The standard Siemens EDA test patterns include:

  • isolated, double, and dense lines

  • inverse lines

  • isolated, double, and inverse line ends

  • various types and configurations of contacts

  • various sizes and pitches for lines and contacts

Calibre WORKbench can generate patterns using the built-in Test Pattern tool, or you can define a test pattern file using a per-line command format, then send the file to the test pattern batch command generator.

You can also create your own test pattern geometries in the Calibre WORKbench GUI and export them to a spreadsheet.

For additional information on Topo-specific test patterns, see “Topo Test Pattern User’s Guide” in the Calibre® WORKbench Topography Modeling User’s and Reference Manual.

About Verification Test Patterns

Verification test patterns are a type of test pattern that contains additional and different test structures that you use to verify the accuracy of your model. The purpose of having a separate verification test pattern is to avoid overfitting the model to the structures represented on the test chip.

Create verification test patterns either through the Test Pattern V11/V12 dialog box, shown in “Generating a Test Pattern With the Calibre WORKbench GUI”, or in batch mode using the test pattern compiler language, which is explained in detail in “Test Pattern File Format”.
