sgd_from_sss

CM1 and VEB Commands

Related Formats: Gauge Object Reference, Super Gauge Data File Format

Converts a super spreadsheet file to a Tcl super gauge object and returns a Tcl handle to this object.

Usage

sgd_from_sss –sss sssfile {–l wblayout | –m layoutfile} –layer layerno [-etch] [–extra length]
[–force [horizontal | vertical] | -center] [-length factor] [-measstat prec stdev count]
[-search distance] [-skipped string]

Arguments

–sss sssfile

A required argument that specifies the input super spreadsheet file.

–l wblayout

A required argument that specifies the input layout handle with the test pattern. This option cannot be used if –m is set.

–m layoutfile

A required argument that specifies the input database file in one of the supported formats — GDS, OASIS, and so on. This option cannot be used if –l is set.

–layer layerno

A required argument that specifies the input layer number of the design layer where the measurements are to be done.

-etch

An optional argument that converts measured .sss data to the etch column instead of the resist column of the super gauge object.

–extra length

An optional argument that specifies lengthening all gauges by 2*length in nm (database units pre-2007.3) The default value is 200.

–force [horizontal | vertical]

An optional argument that forces Calibre WORKbench to produce possibly bad gauges (such as gauges not centered on the nearest edge). Additional qualifiers, horizontal and vertical, can be specified to convert the sample points to the specified type. This option cannot be used with ‑center.

-center

An optional argument that instructs Calibre WORKbench to center gauges if the input sites are not centered on the nearest edge. This option cannot be used with ‑force.

Note:

If neither ‑force nor ‑center is specified, Calibre WORKbench skips over sites that are not centered.

-length factor

An optional argument that filters out gauges whose net length in nanometers (before adding -extra) is the specified factor less than the measured CD. The default is to not exclude any gauges.

-measstat prec stdev count

An optional argument used to set measurement statistics for all gauges to the specified values. Default is to set prec, stdev, and count to -1 (inactive).

This option is active starting with version 2009.1, replacing the -mode option.

-skipped string

An optional argument that outputs to the specified string the list of skipped sample points that were not converted to gauges.

–mode mode count dist

This option was removed starting with version 2009.1. Use ‑measstat instead. The ‑mode option was used to create subgauges and set mode for doing measurements (Default = create one gauge per site).

  • mode — integer number which specifies how CD and space measurements are done if subgauges are present (count > 0); acceptable values are 0, 1, 2, or 3, which mean normal, average, min, or max measurements, respectively.

  • count — number of subgauges to be added to the left and right of the main gauge (the total number of subgauges is 2*count +1).

  • dist — distance between subgauges in database units.

-search distance

An optional argument specifying the search radius in nanometers (microns in versions previous to 2007.3) to find two edges to be crossed with a gauge. The default is 500 nanometers.

Description

This command converts a super spreadsheet file to a Tcl super gauge object. It requires the following conditions:

  • The input database must be flat; if you have a hierarchical database, flatten it using the layout copy or clipLayout commands.

  • If measurements were done on several layers, you must merge the layers before running sgd_from_sss.

The following sections provide guidance on how to use this command.

  • The loc value in the super spreadsheet object accepts the following values as a directive where to look for the nearest CD (interior polygon edge) or space (exterior polygon edge) to place the gauge:

    • 0 - nearest CD

    • 1 - nearest space

    • 2 - nearest horizontal CD

    • 3 - nearest horizontal space

    • 4 - nearest vertical CD

    • 5 - nearest vertical space

  • In some cases, it is possible to receive the error message “No edges found near ss point” even after merging the measurement layers and increasing the search distance. This can occur when using double-exposure masks (such as altPSM), where the second exposure is used partially to protect some area from the first exposure, resulting in overlapping features that are missed by sgd_from_sss. For these situations, split your input sample into separate files for each measurement layer, converting sample data to gauge files, and then merge the resulting gauge files together.

Examples

Example 1 (Flat Database)

# use -m option to specify layout
set aobj [sgd_from_sss -sss sample.sss -m tp.gds -layer 0 -extra 10]
# use -l option to specify layout
set l_obj [layout create tp.gds]
set aobj [sgd_from_sss -sss sample.sss -l $l_obj -layer 0 -extra 10]

Example 2 (Hierarchical Database)

set l_obj [layout create tp_hier.gds]
layout copy $l_obj l_obj_flat
set aobj [sgd_from_sss -sss sample.sss -l l_obj_flat -layer 0 -extra 10]
