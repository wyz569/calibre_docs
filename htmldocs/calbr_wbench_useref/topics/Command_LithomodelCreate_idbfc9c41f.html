lithomodel create

Litho Model Command Reference

Generated by: Lithomodel Creation Wizard

Creates a litho model directory using the specified source models.

Usage

lithomodel create -lmout lmout -resistmod resistmod
[-imagethreshold thresholdvalue]
[-etchmod etchmod]
[-topomod topomod]
[-sbmmod sbmmod]
[-flaremod flaremod [-bbmmod bbmmod]]
[-stochasticmod stochasticmod]
[-cdsamod cdsamod]
[-zplanesmod zplanesmod]
-masks ‘{’ mask_section1 [mask_section2 mask_section3 …] ‘}’

where each mask_section is a Tcl list enclosed in braces ({}) of the form:

  ‘{’
   -nomopt nomopt
   -background background_trans
   -mask_layers ‘{’ mask_layer1 [mask_layer2 mask_layer3 …] ‘}’
   [-impopt “impopt1 [impopt2 …]”]
   [-genopt “defocus1 [defocus2 defocus3 …]”]
   [-dose dose]
‘}’

and each mask_layer section is a Tcl list enclosed in braces ({}) of the form:

  ‘{’
   -trans trans
   [-cx cx]
   [-cc cc]
   [-bias bias]
   [-xbias xbias]
   [-ybias ybias]
   [-xshift xshift]
   [-yshift yshift]
   [{-impddm impddm} | {-ddmgroup ddmgroup}]
   [-category category]
‘}’

Arguments

-lmout lmout

A required argument that specifies the path of the output directory. If an existing directory is specified, its contents are erased.

-resistmod resistmod

A required argument that specifies a CM1 resist model. The filename should end in .mod if loaded using the GUI. If the resist model file is not in the home directory, an absolute or relative path may be specified.

-imagethreshold thresholdvalue

An optional argument that specifies a constant threshold value to use.

-etchmod etchmod

An optional argument that specifies a VEB etch model.

-topomod topomod

An optional argument that specifies a topographical (topo) model.

-sbmmod sbmmod

An optional argument that specifies an EUV shadow bias model file.

-flaremod flaremod

An optional argument that specifies an EUV flare model.

-bbmmod bbmmod

An optional argument that specifies an EUV black border model. This argument requires you also specify a flare model using -flaremod.

-stochasticmod stochasticmod

An optional argument that specifies an EUV stochastic model.

-cdsamod cdsamod

An optional argument that specifies a compact DSA model file.

-zplanesmod zplanesmod

An optional argument that specifies a zplanes model.

-masks ‘{’ ‘}’

A required argument that defines a list of one or more exposure mask definition blocks, referred to as a mask_section.

Note:

Each mask_section must have its own set of braces ({ }), and the -masks argument has its own required set of enclosing braces. In other words, a double exposure would have the argument: 

-masks { {mask_section1} [mask_section2]}
-nomopt nomopt

A required argument for each mask_section that specifies the nominal (on-focus) optical model for the exposure.

-background background_trans

A required argument for each mask_section that specifies the background transmission type for the exposure.

-mask_layers ‘{’ ‘}’

A required argument that defines a list of one or more mask layer definition blocks, referred to as a mask_layer.

Note:

Each mask_layer must have its own set of braces ({ }), and the -mask_layers argument has its own required set of enclosing braces. In other words, a single mask_section with three mask_layers would have the argument: 

-masks {{-nomopt nomopt -background -mask_layers { 
{mask_layer1} {mask_layer2} {mask_layer3} } }}

The contents of the mask_layer block are

-impopt “impopt1 [impopt2] …”

An optional argument that defines a list of paths to one or more additional optical models that have been pre-generated. This list must be enclosed in quotation marks (“ ”) and each list item is separated by spaces.

Each optical model included with this argument must be defocused relative to the nominal optical model; the defocus value is read from the file(s).

-genopt “defocus1 [defocus2] [defocus3] …”

An optional argument that defines a list of relative defocus values in microns to vary the nominal optical model by. The list must be enclosed in quotation marks (“ ”) and each list item is separated by spaces. When this option is specified, additional optical models using the defocus values in the list are generated.

-dose dose

An optional argument that sets a different dose value for the mask that is not 1.0.

-trans trans

A required argument specifying the transmission value for the mask_layer.

-cx cx

An optional argument specifying the convex cornerchop value in microns for the mask_layer.

-cc cc

An optional argument specifying the concave cornerchop value in microns for the mask_layer.

-bias val -xbias val -ybias val

Optional arguments in microns causing all the geometric data on the mask to be resized in both X and Y scales (bias) or X-only (xbias) or Y-only (ybias) scales. Positive values expand shapes; negative values reduce shapes.

-xshift val -yshift val

Optional arguments in microns causing all the geometric data on a given mask to be shifted by x,y. If specified, these options cause Calibre to clone all rotated or reflected cell transforms. The default is 0.

-impddm impddm | -ddmgroup ddmgroup

An optional argument specifying a DDM model file or DDM group file to include in the litho model. The previously-created DDM model file or DDM group file is specified using a relative or absolute path. DDM group files are supported by the Calibre nmModelflow Lithomodel Creation Wizard and unavailable in the Calibre WORKbench Lithomodel Creation Wizard.

For a DDM model file, the filename is associated with the mask_layer. For a DDM group file, the name of the first file in the group is associated with the mask_layer and is used as the DDM group name.

-category category

An optional argument that specifies a comment. The comment can only contain alphanumeric, hyphen, or underscore characters.

Description

Creates a litho model directory using the specified source models.

  • At least one mask_section must be specified.

  • At least one mask_layer must be specified for each mask_section. Each mask_layer will be created starting at mask_layer index 0 within its own mask_section.

Note:

The mask_sections and mask_layers are Tcl lists. This means that the entire list must be enclosed in braces (or built using the Tcl list command) and each list element has its own set of fully-enclosing braces. This leads to double braces around arguments to this command. The generated Lithomodel key file does not contain the braces.

Examples

This example creates a double exposure litho model directory with the following characteristics:

  • Uses the resist model cm1default.mod in the current working directory.

  • Exposure 0 uses the opt1 optical model and contains two mask layers.

  • Exposure 1 uses the opt2 optical model and contains one mask layer.

  • Exposure 1 also sets its dose at 0.95. 

    lithomodel create -lmout lm_demo -resistmod cm1default.mod 
-masks {
	 {-nomopt models/opt1
	 -background clear
	 -mask_layers { 
		{-trans dark -category poly}
		{-trans dark -category contact}
	 } 
	}
	 {-nomopt models/opt2
	 -background clear
	 -mask_layers { 
		{-trans dark -bias 0.1 -category test }
	 }
	-dose 0.95}
}
