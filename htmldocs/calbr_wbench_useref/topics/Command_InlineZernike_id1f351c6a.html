inline_zernike

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Defines Zernike coefficients in the Optical file for lens aberrations.

Usage

inline_zernike inline_id ‘{’
   zernike_block
‘}’

Arguments

inline_id

A required argument specifying an inline Zernike block by its identifier inline_id. The inline Zernike block identifier must be referenced by an zernike command.

zernike_block

A required argument set specifying the Zernike locations and values. The block must be enclosed with braces.

  • Each Zernike location is stored in the ZernikeTable. The x-position, in millimeters, is stored in <x>. The y position, in millimeters, is stored in <y>.

  • The actual Zernike values are stored in <Zn> entries, where n is 1 to 200. The values are in units of nanometers “peak”.

    See the Examples section of an example of <ZernikeTable>.

  • Z37 has special rules. If Z37 is the final value specified, it is assumed to actually be the value for Z49. Otherwise, all values are used directly.

  • If one of the first 5 Z-values is larger than 1000, this is assumed to be invalid and ignored.

  • The Zernike table will usually contain several Y-positions. A weighted average of these locations is used as the Zernike values for simulation.

Description

This optional parameter enables specifying lens aberrations in the Optical file instead of loading a separate file from disk. Up to 200 Zernike coefficients may be used to define the lens aberrations. Parameters are defined using Zernike coefficients at different locations. The default is no aberrations.

Examples

An example of a <ZernikeTable> section is as follows:

<Results>
	<ZernikeTable>
			<elt>
				<x>-12.72</x>
				<y>1.47</y>
				<Z2>1000000.00</Z2>
				<Z3>1000000.00</Z3>
				<Z4>1000000.00</Z4>
				<Z5>-0.15</Z5
				<Z6>0.53</Z6>
				...
				<RMS>0.00</RMS>
			</elt>
	</ZernikeTable>
</Results>
