Creating a Setup File in RET Flow Tool

Calibre LITHOview and Calibre WORKbench provide default values for simulation that can be used to create an initial setup file. There are several session types for various tools, but simulation tasks require only a Simulation setup.

Prerequisites

  • A design file with layers.

  • A file directory for which you have write permission.

  • The Calibre WORKbench or Calibre LITHOview GUI open.

  • A set of process models, which may or may not be contained in a litho model.

Procedure

  1. Select Litho > RET Flow Tool v2.0.
  2. In the RET Flow Tool v2.0 window, select LM as the session type if you have a litho model, or Sparse if you only have individual models. .
  3. Click the Add Session button, then select Litho Model (or New if you are using Sparse models).
  4. Click the Process Configuration arrow on the right and view or adjust the RET Flow Tool v2.0 simulation settings to match your layout file and model locations.
  5. Adjust the Masks and WORKbench Layer to show the layer you intend to simulate.

    Only layers in the WORKbench Layer list are visible for simulation. Use the Add button if no layers are displayed. All layers use the same background setting.

    Note:

    The background and layer trans should be set to complementary values. For example, if background is dark, trans should be clear.

  6. Verify the values for the nominal process window condition. You can use the + button to enter other process window conditions to include in the simulation. (The check box to the left of the PWC Name column toggles whether conditions are selected. Red indicates the condition will be used in the next simulation.)
  7. Save the setup file.

    The setup file contains only keywords suitable for use in a litho setup file such as you would use when running Calibre OPCverify from within an SVRF file. The session file contains the setup file and also the settings for the operation options such as layer mapping, cutline colors, and so on.

    By convention, setup files use the .in extension and session files the .lsf extension, but these are not automatically supplied. For simplicity when using the file browser later to open the files, it is recommended you use the extensions.
