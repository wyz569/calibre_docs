Arguments

Two forms of specifying CM1 optimization are accepted for stage 2 of modelflow_v2:

Implicit mode

Implemented in the 2013.1 release for convenience. Implicit mode allows you to specify a set of parameters by shorthand commands similar to choosing a modelform.

Explicit mode

The detailed, customizable format. Individual terms can be specified. This is the mode that is used in releases previous to 2012.4.

Implicit Mode

cm1mf {integer | nominal}

Use this option to specify a CM1 modelform to be calibrated. Accepted modelform numbers are the same as for the cm1compose Tcl command. Also, if sinterval and binterval are not specified, the search intervals for s and b parameters coincide with those generated by the cm1compose Tcl command.

Note:

The “maxaidiff stable” default argument is not compatible with modelforms 1, 2, 3, and 31 when you set them while also using the cm1mf argument. If your modelform_v2 has this combination, modelflow_v2 issues a warning and changes the setting to “maxaidiff loose.”

sinterval {auto | value} {auto | value}

Use this option to specify lower and upper bounds of search intervals for s parameters, which are the neutralization level of an n-th CM1 basis function. The keyword auto tells the tool to use the default values for s. (Default = use search intervals determined by cm1compose Tcl command.) Can be set only if cm1mf is present.

binterval {auto | value} {auto | value}

Use this option to specify lower and upper bounds of search intervals for b parameters, which are the diffusion length in nm of n-th CM1 basis function. The keyword auto tells the tool to use the default value. (Default = use search intervals determined by cm1compose Tcl command.) Can be set only if cm1mf is present.

For example, using an implicit optimization setting of the form:

cm1mf 20 \

is equivalent to the following explicit mode settings:

s1 0 200 10 \
s2 0 200 10 \
s3 0 200 10 \
s4 25 200 10 \
b1 0 0.3 10 \
b2 0 0.3 10 \
b3 0 0.3 10 \

Similarly, using

cm1mf 20 \
sinterval 10 auto \
binterval 0.1 0.2 \

is equivalent to the following explicit mode settings:

s1 10 200 10 \
s2 10 200 10 \
s3 10 200 10 \
s4 25 200 10 \
b1 0.1 0.2 10 \
b2 0.1 0.2 10 \
b3 0.1 0.2 10 \

Note that the lower limit for s4 remains 25, even though in sinterval it is set to 10. This is because the diffusion length of the gradient term cannot be smaller than 25 nm.

To get an idea of the available model forms and default optimization settings, use the cm1compose Tcl command and analyze the output associated with the options ‑o and ‑optimtcl, respectively.

Explicit Mode

Note:

There are three limitations on the CM1 parameters available for optimization:

  • b and s parameters of no more than 20 basis functions entering the cm1 nominal model can be optimized simultaneously. That is, bn and sn with n > 19 are not available for optimization even if the nominal model involves Mn terms with n > 19.
  • b parameters are available for optimization only for those M terms that have k = 0. Combinations of k = 1 or 2 and b > 0 are prohibited.
  • s parameters must be greater than a minimum value of 3*pixel size. Lower limits of search intervals for s parameters that do not meet this constraint are automatically reset to 3*pixel size. If any s parameters that are not specifically optimized do not meet the constraint, modeflow_v2 exits with an error.

Explicit mode for the bn and sn options uses the following syntax:

bn optim_set

Used for CM1 models. Specifies the neutralization level of nth CM1 basis function (Mn term) as a non-negative dimensionless quantity 0<= bn, n = 0, 1, …, 19. For b = 0 the image intensity remains unchanged.

Specifying “auto” for an upper or lower bound argument causes modelflow_v2 to calculate the upper and lower bounds for the term.

  • For b parameters associated with positive neutralization terms, the search interval is set to [0, 0.3].

  • For b parameters associated with negative neutralization terms, the search interval is computed as follows:

    1. Generate aerial image cutlines and find all minima smaller than the threshold.

    2. Find the maximum of these minima.

    3. Add 0.1 to this maximum. If auto is specified for the lower bound, use the new value as the lower limit of the search interval.

      If auto is used for the upper bound, the upper limit is determined as the lower limit plus 0.3. Otherwise it remains as 0.3 by default.

(Default = use nominal model setting.)

sn optim_set

Used for CM1 models. Specifies the diffusion length in nm of nth CM1 basis function (Mn term), 3*pixel size <= sn, n = 0, 1, …,19. (Default = use nominal model setting of 3*pixel size.)

Resist Shrink, Horizontal Bias, and Sidewall Bias Options

Note:

The terms in this section cannot be used if the cm1mf parameters are already present in the modelflow_v2 script.

Note:

Specifying “auto” for the lower bound of the shrink radius and diffusion length parameters (shrink_radiusj, swbias_d, swbias_s, hbias_d, hbias_s, and vshrink_s) uses a value of 3*pixel size.

If a value less than 3*pixel size is specified for any shrink radius or diffusion length parameter, modelflow_v2 exits with an error.

shrink_radiusj optim_set

Used for CM1 models. Specifies the CM1 shrink radius for the jth term in nm. shrink_radiusj must be positive, and its lower bound must be at least 3*pixel size. Up to two shrink terms can be specified (j= 0 or 1). For previous users who have the original shrink_radius option, modelflow_v2 converts it to shrink_radius0. (Default = use nominal model setting.)

shrink_thresholdj optim_set

Used for CM1 models. Specifies the CM1 shrink threshold in nm. shrink_thresholdj must be positive. Up to two shrink terms can be specified (j= 0 or 1). For previous users who have the original shrink_threshold option, modelflow_v2 converts it to shrink_threshold0. (Default = use nominal model setting.)

shrink_diffusionk optim_set

Used for CM1 models. Specifies the CM1 resist shrink diffusion length in nm. Up to three shrink terms can be specified (k=0, 1, or 2). ((Default = use nominal model setting.)

sem_shrink_ak optim_set

Used for CM1 models and SEM resist processes. Specifies the intensity scale parameter of the kth CM1 SEM_SHRINK term, where k=0, 1, or 2.

sem_shrink_bk optim_set

Used for CM1 models and SEM resist processes. Specifies the intensity threshold parameter of the kth CM1 SEM_SHRINK term, where k=0, 1, or 2.

sem_shrink_rk optim_set

Used for CM1 models and SEM resist processes. Specifies the radius parameter of the kth CM1 SEM_SHRINK term, where k=0, 1, or 2.

sem_shrink_sk optim_set

Used for CM1 models and SEM resist processes. Specifies the diffusion parameter of the kth CM1 SEM_SHRINK term, where k=0, 1, or 2.

hbias_aj optim_set

Used for CM1 models. Specifies the CM1 horizontal bias as a dimensionless indexed parameter (j is between 0 and 3). hbias_aj must be positive.

hbias_bj optim_set

Used for CM1 models. Specifies the CM1 neutralization level for horizontal bias as an indexed parameter (j is between 0 and 3). hbias_bj must be positive.

Specifying “auto” for an upper or lower bound causes modelflow_v2 to calculate the boundaries for the term automatically.

  • Positive horizontal bias terms use a range of [0, 2.5].

  • Negative horizontal bias terms are calculated as follows:

    1. Generate aerial image cutlines and find all minima smaller than the threshold.

    2. Find the maximum of the minima.

    3. Set this maximum is set as the calculated lower limit.

    4. Use the lower limit is used as shown in the table, depending on the arguments specified for hbias_bj.

      auto auto

hbias_dj optim_set

Used for CM1 models. Specifies a “smoothing length”, which is an additional diffusion length in nm applied to the whole horizontal bias term. hbias_dj is an indexed parameter (j is between 0 and 3) that must be greater than or equal to 3*pixel size.

hbias_sj optim_set

Used for CM1 models. Specifies the CM1 Gaussian diffusion length for horizontal bias as an indexed parameter (j is between 0 and 3) that must be greater than or equal to 3*pixel size.

swbias_t optim_set

Used for CM1 models. Specifies a dimensionless threshold level for the aerial image entering sidewall bias. swbias_t must be positive.

swbias_b optim_set

Used for CM1 models. Specifies a dimensionless neutralization level for the aerial image entering the sidewall bias. swbias_b must be positive.

swbias_d optim_set

Specifies a smoothing length as an additional diffusion length in nm applied to the entire sidewall bias term. swbias_d must be greater than or equal to 3*pixel size.

swbias_s optim_set

Specifies a Gaussian diffusion length in nm entering the sidewall bias. swbias_s must be greater than or equal to 3*pixel size.

vshrink_u optim_set

Used for 3D resist models with negative tone (NTD) modelforms. Specifies a normalized uniform shrinkage. Values must be greater than or equal to 0.

vshrink_v optim_set

Used for 3D resist models with negative tone (NTD) modelforms. Specifies a normalized variable shrinkage. Values must be greater than or equal to 0.

vshrink_l optim_set

Used for 3D resist models with negative tone (NTD) modelforms. Specifies a dimensionless transition length. Values must be greater than 0.

vshrink_t optim_set

Used for 3D resist models with negative tone (NTD) modelforms. Specifies a dimensionless transition threshold. Values must be greater than or equal to 0.

vshrink_d optim_set

Used for 3D resist models with negative tone (NTD) modelforms. Specifies a dimensionless smoothing length. Values must be greater than 0.

vshrink_s optim_set

Used for 3D resist models with negative tone (NTD) modelforms. Specifies a Gaussian diffusion length in nanometers. Values must be greater than or equal to 3*pixel size.

zslope_a optim_set

Specifies a dimensionless scaling parameter for the first order z-dependent slope term. Used for 3D resist models to optimize Z_SLOPE.

zslope_b optim_set

Specifies a dimensionless scaling parameter for the second order z-dependent slope term. Used for 3D resist models to optimize Z_SLOPE.

zslope_c optim_set

Specifies a dimensionless scaling parameter for the third order z-dependent slope term. Used for 3D resist models to optimize Z_SLOPE.

Description

Definitions

  • CTR — Constant Threshold Resist Model of type CM1.

  • Nominal optical, resist, and etch models are the initial models specified in the setup file.

  • optim_set — Defined as one of the optimization expressions as described in Using the Independent, Constrained, and Dependent Optimization Parameters of “modelflow_v2: Stage 1 Optimization Run Modes”.
