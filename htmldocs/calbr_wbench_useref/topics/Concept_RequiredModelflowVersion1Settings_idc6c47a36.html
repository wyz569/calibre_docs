Required modelflow Version 1 Settings

All new modelflow Version 1 scripts should include at least the following settings.
Table 1. modelflow_v1 Starter Settings

Setting (modelflow v1 flag)

Calibre WORKbench Uses it to…

Notes

Setup file (-f)

Load fragmentation and model locations.

The setup file must have a nominal optical model and nominal resist model.

Layout file (-d)

Load a test pattern file to calibrate a process model on.

None.

A sample spreadsheet file (‑ss option) or super spreadsheet file (-sss)

Associate spreadsheet(s) measurement and coordinate data with the loaded pattern file.

The spreadsheet must have measurements filled in by this point.

Recommendations for modelflow Version 1 Process Models

Configure the modelflow command scripts to optimize the process model using these additional settings after specifying the starter and optical settings:

Table 2. Suggested Settings for modelflow_v1 Process Model Parameters

Setting (modelflow v1 flag)

Calibre WORKbench Uses it to…

Notes

modelform

Set the default modelform coefficient.

See the section “Best Practices for Selecting a VT5 Modelform.”

check_ref_thresh

Check to see if the threshold passes through the aerial image.

If this option is active, modelflow stops with an error if a threshold does not pass through an aerial image on any structure.

D1, D2, D3, D4 Gaussian kernels

Optimize coefficients with different trial sigmas.

You can specify half-Gaussians using the gaussm and gaussp arguments (see Stage 3 Arguments in “modelflow (Version 1)” for more information).

crosscheck

Check for a stable model by splitting data into two sets. It finds the optimum eigenvalue.

Modelflow performs crosschecking by default whenever the number of threshold coefficients exceeds 16.

See also “Best Practices for Improving VT5 Model Stability.”

Information on modelflow can be found in the chapter “modelflow Version 1 Reference”.
