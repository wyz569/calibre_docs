image_normalization

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Normalizes the image created by the optical model.

Usage

image_normalization { source_area | integ_source_area | TCC0000 | first_interface | ambient_level | {z_position z_in_um} | {clearfield value} }

Arguments

source_area

An optional argument specifying normalization by the source area. This argument is used for both aerial (ambient) models and thin-film models.

  • Aerial image models — Normalization by the source area results in “open frame intensity” OFI = 1 (This is not true if you use a reduction factor different than 1 or if you use apodization_loss). The source area is calculated analytically. Note that there are cases where analytical calculation of the source area is not possible, such as COMPOSITE sources that contain COHERENT elements or elements with MAP or ERF profiles.

  • Thin-film image models — Normalization by the source area will, in general, result in OFI not equal to 1. Energy conservation is built-into the thin-film model, therefore variations in OFI with respect to the image plane location can be a result of energy loss from reflection in the air and resist interface or presence of standing waves inside the resist and other films.

This is the default image normalization whenever possible for aerial models when the optical model version is less than 5.

integ_source_area

An optional argument specifying calculating the source area through integration. Used for both aerial (ambient) models and thin-film models. This is the default image normalization for aerial models when the optical model version is less than 5 and source_area is not applied.

TCC0000

An optional argument specifying image normalization where OFI = 1 regardless of reduction factor, apodization_loss or image averaging over a number of planes.

Image normalization by z_position z_in_um makes OFI = 1 at a plane specified by z_in_um.

first_interface

An optional argument specifying image normalization that results in OFI = 1. Used for thin-film models only. This is the default image normalization for thin-film models when the optical model version is less than 5.

(In general, the open frame intensity will be different than 1 at other planes inside the resist film because of standing waves and/or attenuation).

ambient_level

An optional argument specifying image normalization that makes OFI = 1 for aerial image models (or models in a uniform ambient medium), regardless of magnification or apodization loss. Used for both aerial (ambient) and thin-film models.

Using ambient_level in thin-film models will, in general, result in OFI < 1 inside the resist, because of energy lost from reflection, refraction and lossy media in the film stack.

z_position z_in_um

An optional argument set specifying image normalization that makes OFI = 1 at the plane specified by z_in_um.

clearfield value

An optional argument set normalizing the clearfield of the TCC matrix to the specified value. Due to energy loss from SOCS decomposition, the actual clearfield from the SOCS model will be slightly different, unless all the kernels are kept.

Description

CAUTION:

Due to continuing improvements in the way optical model normalization is calculated, models generated in different versions are likely to be different. Major changes occurred in versions 2013.1 and 2014.1. In order to achieve consistent results, you must generate all optical models (nominal and off-nominal) with the same version of Calibre WORKbench; in other words, you cannot mix models created with different versions of Calibre WORKbench.

This optional miscellaneous parameter allows you to normalize the image created by the optical model. Modifying the image normalization can affect TCCcalc output results.

  • Defaults (for version < 5):

    • Aerial models — source_area whenever possible, otherwise the default is integ_source_area

      Also applies to any models in a uniform ambient medium (scalar or vector)

    • Thin-film models (scalar or vector) — first_interface

  • Defaults (for version >= 5)

    source_area whenever possible, otherwise the default is integ_source_area

Image normalization is an advanced modeling feature, used primarily in research situations. For most production models, the default normalization is appropriate.

This setting relates to the image intensity of a transparent mask area that contains no geometry, sometimes referred to as “open frame intensity (OFI)”.
