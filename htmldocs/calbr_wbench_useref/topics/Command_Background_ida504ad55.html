background

Optical Model Commands

A setup file configuration command describing the imaging background.

Note:

The background command is used in conjunction with the layer setup file command. However, the background command is ignored if DDM libraries are used with an optical model.

Usage

background {dark | clear | {attenuated factor} | {real imag}}…

Arguments

dark

A literal argument designating dark field imaging. Transmission added to mask will be {layertrans + 0} for this layer.

clear

A literal argument designating clear field imaging. Transmission added to mask will be {layertrans + 1} for this layer.

attenuated factor

An attenuated phase-shifting background is specified using either the attenuated command or its abbreviation, atten, with the attenuation factor. This factor is specified as the normalized intensity that should transfer to the wafer for a clear exposure.

  • factor — A required numerical argument to the attenuated option. It is the attenuation factor for attenuated backgrounds. The unit is a percentage. For an 8% attenuated phase shift mask, the factor should be 0.08. The maximum allowed value is 36%.

    When Calibre LITHO tools encounter attenuated factor, they translate the factor into a real imaginary pair such that the attenuated background has a transmission value

    RE(bg) = -sqrt(percent/100)

    IM(bg) = 0

Do not use an attenuated background with a phase180 layer, as this will cause an error.

Transmission added to mask will be {1.value - bg} for this layer.

real imag

An optional pair of real numbers specifying the layer transmission value and background value, respectively. Note that the attenuated syntax for the transmission is recommended over the (real, imaginary) pair syntax. The (real, imaginary) syntax supports any arbitrary phase and transmission combination.

If you specify background with the “real imag” pair for a specified layer transmission value, the resulting transmission value for the output layer with a “real imag” pair is resolved relative to the background pair. The layer transmission value is resolved according to the following equation:

background (real imag) + layer (real imag) = transmission_value

For example: 

background 1 0 
name			opt 				type 
chrome			visible	 			0 0

The background is specified with a (1, 0) pair, while the chrome layer is defined with a transmission value of (0,0). With clear background (1,0) and a layer specified with (0,0), the resulting transmission layer is (1,0) + (0,0) = clear (1,0). 

background 1 0 
name 		opt 				type 
p180 		visible 				-1 0

In this case, the layer p180 is defined as (-1 0). With clear background (1,0) and a layer specified with (0,0), the resulting transmission layer is (1,0) + (-1,0) = dark (0,0).

Description

You must specify one background type per mask exposure. When multiple masks are used, the arguments must be supplied in order, with the first argument referring to Mask 0, the second referring to Mask 1, and so on.

Figure 1 shows how to specify multiple-mask exposures using different arguments.

Figure 1. Multiple-Mask Exposure

The first background applies to the first mask exposure, “mask#0”; the second background applies to the second exposure, “mask#1”.

Note:

The layer transmission value is tied to the background value. For example, a clear field mask does not support any layers that are defined as clear.

Examples

Example 1

This example shows the combination of layer and background settings needed to define an attenuated phase shift mask. The background is attenuated and the layer type is clear because light is not obstructed when passing through the polygons in the VIA layer. 

background attenuated 0.06
layer VIA opc clear

Example 2

An alternative way to define the mask is to use the transmission options available in the layer instead of the background. This is performed where the background is clear and the layer is attenuated, using the following example code:

background clear
layer poly opc atten 0.06

This can also be expressed in {real imag} transmission notation:

background clear
layer poly opc -1.244949

With an attenuated phase shift mask intensity transmission of 6%, the transmission of the electric through the mask is -sqrt(0.06), where the - sign accounts for the 180 degree phase shift of the attenuating phase shifter layer. To obtain the transmission of [Re, Im] = [-sqrt(0.06), 0], the layer transmission setting must be set to [-sqrt(0.06) -1,0], since the layer statement transmission is added to the background transmission of [1,0]
