modelflow (Version 1)

VT5 Resist Model Commands List

This command creates an optimized VT5 model and an optimized optical model (or maskwriter model) using a fixed model optimization procedure that begins with a pair of “nominal” models.

Arguments

Input Arguments

Use the input arguments to define the process window conditions, gauges, layout, and other settings for model calibration. For syntax, see “modelflow (Version 1): Input Arguments”.

Output Arguments

Use the output arguments to define the calibrated models, log files, and setup files. For syntax, see “modelflow (Version 1): Output Arguments”.

Stage 0 Arguments

Use the Stage 0 arguments to define preprocessing operations, such as filtering input data and setting a template model form. For syntax, see “modelflow (Version 1): Stage 0 Arguments”.

Stage 1 Arguments

Use the Stage 1 arguments to define settings for the optical or maskwriter models. For syntax, see “modelflow (Version 1): Stage 1 Arguments”.

Stage 2 Arguments

Use the Stage 2 arguments to adjust the reference threshold. For syntax, see “modelflow (Version 1): Stage 2 Arguments”.

Stage 3 Arguments

Use the Stage 3 arguments to optimize the resist model. For syntax, see “modelflow (Version 1): Stage 3 Arguments”.

Gaussian Arguments

Use the Gaussian arguments to apply VT5 Gaussian density kernels. For syntax, see “modelflow (Version 1): Gaussian Arguments”.

Description

The modelflow operations perform three types of experiments, which you can run separately or in series. These experiments are referred to in modelflow as “stages.”

  • Stage 1 — Optical model or maskwriter model tuning

  • Stage 2 — Reference Threshold optimization

  • Stage 3 — VT5 Resist model optimization

modelflow reflects these stages in the Calibre WORKbench GUI as tabs, and as keyword stages in the modelflow batch command.

When you run modelflow, you decide which type or types of experiments to perform. However, some dependencies exist:

  • Tuning the optical model is dependent on having valid data.

  • Optimizing the reference threshold is dependent on having a good optical model plus valid data.

  • Optimizing the resist model is dependent on having the best reference threshold plus a good optical model plus valid data.

Run all of the stages at once for a comprehensive solution. Run each stage independently for quicker results.

VT5 Model Form Descriptions

Because of the large number of model forms you can design with VT5, Siemens EDA has developed a model naming convention and a set of basic model forms. When you use these model forms in VT5 Center as the basis for a new VT5 model, the tool fills in the data fields with appropriate default values. When you use a model form in modelflow, it defines the basic form of the VT5 models it evaluates.

A crossterm is defined as a term that is the product of two image parameters. An example crossterm is (Imax x slope). No model forms have crossterms between densities. Crossterms may or may not be present between VT5 image parameters. Quadratic models with no cross terms have linear and squared terms.

Note:

The following modelflow options are not changed by model forms, but inherit their values from the nominal VT5 model. You should ensure that the nominal VT5 model contains the desired settings for these options. You can examine the modelflow output log to check which options were inherited by the final VT5 model.

  • epeBias
  • referenceThreshold
  • useLogSlope
  • sampleSpacing
  • kerngrid
  • hoodpix
  • minEigenval

Model form numbers between 110 and 199 represent BIAS models. The only valid model form numbers for BIAS models are those that correspond to a valid THRESHOLD model.

For example, 145 is a valid model form:

  • Modelform 45 = slope only, quadratic, no cross terms, 10 tophat kernel threshold polynomial

  • Modelform 145 = slope only, quadratic, no cross terms, 10 tophat kernel bias polynomial

There are no BIAS model forms analogous to model form 0.
Note:

The model form you use for your process affects setup file contents. For example, you would not use visible layers for OPC with Printing Assist Features (PAFs) if your model form is one of the VT5 models that have density kernels (12, 13, 43, 63, 83, 112, 113, 143, 163, and 183).

OPC with PAFs performs OPC on assist features you want printed (such as dummy fill for density assistance). You must specify the PAF layers in your setup file. A longer discussion of how to perform these advanced OPC methods is discussed in the Calibre OPCpro User’s and Reference Manual, in the section “OPC Examples.”

Calibration Over a Process Window Using Super Spreadsheets

To build more robust optical and (especially) VT5 models over a process window (also referred to as “PW calibration”), see “Calibrating Over a Process Window Using Super Spreadsheets”.

Notes

  • Using the Stage 3 check_ref_threshold option picks the spreadsheet at the best process conditions (dose = 1, defocus = 0).

  • Similarly, if you use the -csf option, you must have a spreadsheet containing best process conditions, because SEM contours are assumed to be taken at dose = 1, defocus = 0.

  • Different spreadsheet files inside the super spreadsheet can refer to the same process conditions, but this is not recommended, because it may increase the run-time slightly. The spreadsheet with the best process conditions is not merged; only the first one containing dose = 1, defocus = 0 is used.

Examples

Example 1: Gradient Search Involving SEM Calibration File Example

In this example, several optical model parameters are tuned to fit a CTR model using data from a spreadsheet file and a SEM calibration file. The number of exploration runs is set to zero and therefore the gradient search starts with nominal optical settings. After 42 iterations, the optimization process has converged to a model whose fitness is better by a factor of 2 than that of the nominal model.

modelflow -sss weighted.sss -csf sem.csf -semweight 0.4 \
	-m tp.gds -f setup.in \
	output log out.log \
	stage0 modelform 0 \
	stage1 \
		filter pitch end \
		search gradient explore 0 \
		sigma_out 0.7 0.9 0.02  \
		sigma_in 0.3 0.54 0.02  \
		illumangle  25 35 1 \
		dsigma_out 0 0.1 0.01  \
		dsigma_in 0 0.1 0.01 \
		dillumangle 0 10 1 \
		imagediffusion 0 0.1 0.01

Example 2: Maskwriter Model Optimization

This example optimizes diffusion lengths of three Gaussian density kernels to simulate a mask layout, bypassing the optical model.

modelflow \
  -f setup.in \
  -m tp.gds \
  -ss sample_final.ss \
  output \
    log out.log \
  stage0 \
    modelform 0 \
    weight 1 filter all end \
    weight 4 filter pitch end \
    weight 2 filter iso end \
  stage1 \
    maskwriter_model g1 0.01 0.05 0.01 g2 0.04 0.2 0.04 g3 0.2 0.5 0.1 \
  stage2 off \
  stage3 off

Example 3: VT5 With Gaussian Half-densities

This example optimizes diffusion lengths of two regular and two directional (gaussm and gaussp) density kernels. Note that for the half-densities to be properly rotated, you must also set the environment variable LITHO_VT5_DIRECTIONAL_DENSITY to be 1.

modelflow \
  -ss test.ss  \
  -m tp.gds  \
  -f setup.in  \
  output \
    log out.log \
  stage2 \
    off  \
  stage3 \
    D1 0.05 0.15 0.1 \
    D2 gauss  0.38 0.48 0.1 \
    D6 gaussm 0.16 0.26 0.1 \
    D3 gaussp 0.27 0.37 0.1 \
  stage3a off
