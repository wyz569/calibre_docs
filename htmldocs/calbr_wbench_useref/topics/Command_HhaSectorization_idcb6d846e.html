hha_sectorization

Parent topic: Optical Parameters File Format

Input for: opticsgen, modelflow (Version 1), modelflow_v2

Breaks the input source into sectors.

Usage

hha_sectorization {{energy [sector_thresh] [-tight {0 | 1}] [-strict_center {0 | 1}] [‑max_sectors_per_pol max_sectors] } | composite | manual}

Arguments

The choice of “energy”, “composite”, and “manual” has different behaviors:

energy [sector_thresh] [-tight {0 | 1}] [-strict_center {0 | 1}] [‑max_sectors_per_pol max_sectors]

A value of “energy” automatically breaks up the source into sectors based on clumps of energy. The source is decomposed into a central sector of varying sigma that allows it to enclose energy near the center of the source (if present), and ASML poles of energy extending out to sigma=1.0. These sectors are placed to surround local regions of intensity. If any sector’s integrated intensity ratio (given by sector intensity divided by total source intensity) is less than sector_thresh, then the sector is merged into neighboring sectors until all sectors have intensity ratios above sector_thresh. sector_thresh defaults to 0.05 (5% of the total source intensity).

Note:

For EUV modeling, “energy” is the recommended value.

  • -tight {0 | 1}

    An optional argument specifying the production of HHA sectors with the tightest possible bounds around the source map for each sector. A value of 1 means tight bounds. Default is 1 only for version 12 optical models, and 0 for earlier optical models.

  • -strict_center {0 | 1}

    An optional argument specifying the strict creation of a central sector. A value of 1 creates a central sector only if there is a dip in the radial intensity. Default is 1 for version 12 optical models and 0 for earlier optical model versions.

  • ‑max_sectors_per_pol max_sectors

    An optional argument specifying that each available polarization in the model have a maximum of max_sectors hha sectors. In some cases, this argument produces unexpected sources such as sources with a central sector. The default behavior sets no maximum number of hha sectors for each available polarization.

composite

A value of “composite” allows you to break up the HHA sectors based on the incoming composite illumination breakup. You can specify a composite dipole primitive with adjusted bounds to ensure that continuous regions of source intensity all fall into the same HHA sector. The composite option still automatically breaks up the input source into separate polarizations for the defined geometrical sector.

manual

A value of “manual” allows you to manually break up the HHA sectors using the hha_sector sigma parameter. This gives you complete control over the sectorization, and separates the source definition from the source sectorization. The manual option still automatically breaks up the input source into separate polarizations for the defined geometrical sector.

Description

An optional miscellaneous parameter that breaks the input source into sectors.

hha_sectorization is used when hha is set to 1. Sector files are stored in the optical directory.

Note:

The hha_sectorization auto option is not available starting with the 2018.4 release. Use hha_sectorization energy instead for all DUV sources, continuous EUV sources, and all EUV sources generated by SMO. Use hha_sectorization composite for measured or rendered EUV sources. See the RET Glossary for a definition of rendered source.
