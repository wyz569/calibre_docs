interpolateddmsectors

Optical Model Commands

Creates a DDM output library from a universal DDM library.

Usage

interpolateddmsectors -ddmin  ddmlibraryin
-opticalfile optical_file [-chief_phi_out phi_out [... phi_out]]
[-objectplane_mode {through_slit | field_maximum | horizontal_edge | none}
   [-objectplane_cd feature_size] [-objectplane_background {dark | clear}]
   [-objectplane_threshold value]]
[-create_universal]
[-crosstalk {on | off}]
[-corners {on | off}]
[-background {clear | dark | unchanged}]
-ddmout ddmlibaryout [-help]

Description

The interpolateddmsectors function is used with universal DDM libraries generated by masksim. The masksim library must contain the “illumination universal” parameter. The output of this command is a custom DDM library with modeltype “ready”.

Arguments

-ddmin ddmlibraryin

A required argument that specifies the DDM input library, which must have type “universal.”

-opticalfile optical_file

A required argument specifying a path to the optical model defining the sectorization for the output library.

-chief_phi_out phi_out

An optional argument used with -opticalfile or -create_universal. If specified, interpolateddmsectors will select all of the signals from the input library which should be included in a standard universal library with the chief ray specified by -chief_phi_out.

-chief_phi_out can accept a single value, or multiple arguments in a list enclosed by quotes.

This option requires the DDM input library to have an illumination universal chief_phi_range value large enough to produce the signals needed to interpolate at phi_out.

If -chief_phi_out is not specified, the chief_ray value from the input universal library is used.

objectplane_mode {through_slit | field_maximum | horizontal_edge | none} [objectplane_cd feature_size] [objectplane_background {dark | clear}] [objectplane_threshold value]

An optional argument that calculates an object plane with the best minimized shift based on the specified mode. This option is only valid for version 4 or later DDM libraries. When run, interpolateddmsectors outputs the DDM library for the optimized object plane.

  • through_slit — Minimizes the slope of the shift for the edges of the approximate image of an isolated vertical line across the center slit. This is the default mode for EUV.

  • field_maximum — Minimizes the shift of the maximum location of the DDM edge signal simulation for a horizontal feature of the center slit chief ray field, filtered to three orders. This was the method used for releases previous to version 2017.4.

  • horizontal_edge — Minimizes the average shift of the edges of the approximate image of an isolated horizontal line at the center slit.

  • none — Performs no optimization. This is the default mode for DUV, but it is not allowed for EUV DDM version 4 or later libraries.

You can control the specified mode using the following optional keywords:

  • -objectplane_cd feature_size

    Controls the feature size, in microns, to be used for the through_slit and horizontal_edge modes. The default value is 0.2.

  • -objectplane_background dark | clear

    Controls the mask background tone for approximate image simulations. The default is dark.

  • -objectplane_threshold value

    Controls the threshold that defines the image edge for the through_slit and horizontal_edge modes. This value assumes an image with clearfield = 1.0. The default is 0.3.

-create_universal

An optional argument that instructs interpolateddmsectors to create a universal library containing a subset of signals from the input library. It selects only signals that were specified in the -chief_phi_out argument in order to generate universal libraries for specific slit positions.

For example, the following statement creates seven universal library files. (Note that if no signals are within range of the slit, a library cannot be generated; this is an error.)

interpolateddmsectors -ddmin universal.throughslit.ddm \
-chief_phi_out "-60 -70 -80 -90 -100 -110 -120" -create_universal \
-ddmout universal.ddm
-crosstalk on | off

An optional argument that controls if crosstalk signals are included in the output. By default, this argument is set to “on” when crosstalk signals are included in the input DDM library. Specifying “off” excludes crosstalk signals in the output, even if they are present in the input DDM library.

An error is issued if this argument is set to on when there are no crosstalk signals in the input DDM library.

-corners on | off

An optional argument that controls if corner signals are included in the output. By default, this argument is set to “on” when there are corner signals in the input DDM library. Specifying “off” excludes corner signals in the output, even if they are present in the input DDM library.

An error is issued if this argument is set to on when there are no corner signals in the input library.

-background clear | dark | unchanged

An optional argument that controls the background tone of the output DDM library. Specifying “clear” outputs a clearfield DDM library; specifying “dark” outputs a darkfield DDM library; specifying “unchanged” (the default) outputs a library with the same tone as the input universal DDM library.

-ddmout ddmlibaryout

A required argument that specifies the filename for the DDM output library. The library is created in the current working directory.

-help

An optional argument that displays help information.
