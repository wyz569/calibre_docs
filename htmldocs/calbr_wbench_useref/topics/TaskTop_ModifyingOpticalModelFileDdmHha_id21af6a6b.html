Modifying an Optical Model File for DDM and HHA

For low k1, high-NA systems, where mask illumination angles of incidence are approaching > 20 degrees, the standard assumption that mask scattering is independent of the angle of incidence (known as the Hopkins Approximation) is no longer valid.

3D mask transmission effects, which include a loss of image contrast and a best focus that is feature-dependent, begin to play an important role in image formation. The Hybrid Hopkins Abbe (HHA) method with DDM extends the optical model SOCS method to account for off-axis illumination.

An optical model by itself does not properly perform HHA modeling; the 3D mask model requires both an HHA optical model and a DDM model (only the HHA settings are part of the optical model). The task of preparing an optical model for HHA changes some of the existing optical model settings.

Note:

The primary difference between a normal optical model and an HHA-enabled optical model is the handling of the source; in an HHA model, each off-axis source is computed separately with its own set of kernels.

Prerequisites

  • An optical model file.

Procedure

  1. Using a text editor, open the optical model file. Typically named Optical, the file is located in the optical model directory.
  2. If the forcePower2LUT setting is specified as non-zero, change it to 0. 
    	forcePower2LUT 0
  3. Add the following setting:
    	hha 1

    This setting activates the HHA subroutines built into opticsgen for this optical model. It allows for multiple TCC matrices.

  4. Add the following setting:
    	ddm_model full

    This setting is required for DDM files. It instructs Calibre to build a 3D mask.

  5. Optionally, add the hha_approxorder keyword after the existing optical model approxorder keyword as shown:
    	approxorder kernels
    	hha_approxorder {all | scale value}

    where:

    • all

      Sets the number of kernels to use for each sector of the composite source equal to the approxorder setting.

      It is important to note that the specified number of kernels is multiplied by the number of sectors (with a approxorder setting of 20 and a hha_approxorder of all, a dipole would use 40 kernels and a quadrupole would use 80 kernels). Using a high kernels value may cause very long computation times.

    • scale value

      Multiplies the number of kernels set in ‘approxorder’ by the scale value, then divides the result among all HHA source sectors depending on its intensity contribution to the entire source. It uses the following formula:

      round(scale x approxorder x sector_weight) +1

      For example, an approxorder setting of 24 kernels with the default scale value of 1.5 and even weight for a quadrupole source would result in each source having (24x1.5x0.25)=9+1, rounding up to 10 x 4 = 40 kernels divided among all sources. (For a dipole, each pole would have 24x1.5x0.5=18+1=19 kernels, rounding up to 20 x 2 = 40 kernels.)

      The kernels may not be evenly divided between sectors, such as in situations where a sector receives a lower amount of light from the source. This will change the internally-computed sector_weight value and sectors with a lower weight will have fewer kernels allocated to them.

  6. Set the hha_sectorization parameter depending on exposure wavelength and source type:
    Note:

    The hha_sectorization auto mode is not available starting with the 2018.4 release. Use hha_sectorization energy instead for all DUV sources, continuous EUV sources, and all EUV sources generated by SMO. Use hha_sectorization composite for measured or rendered EUV sources. See the RET Glossary for a definition of rendered source.

    • hha_sectorization energy sector_thresh

      This setting is recommended for:

      • All DUV sources.

      • Continuous EUV sources.

      • All EUV sources generated by SMO.

      Use the default energy mode at the default sector threshold of 0.05 (or 5% of the total source intensity). To increase the number of sectors, lower the sector threshold value. To decrease the number of sectors, increase the sector threshold value. If greater control of the sectorization is required, select either the composite or manual modes as described in “hha_sectorization”.

    • hha_sectorization composite

      This setting is recommended for measured or rendered EUV sources.

      If you use the “composite” setting, you must define a composite source with the illumtype COMPOSITE keyword, The sectorization algorithm calculates the number of sectors based on the composite source definition. For example, a dipole defined as a COMPOSITE source would have two sectors, one for each dipole.

      When you define the composite source, follow these guidelines to avoid erroneous sources:

      • The sum of the illumangle settings must cover the entire circle (0 to 1) with no gaps or overlaps among elements.

      • Only ASML_DIPOLE, ASML_POLE, and STANDARD source elements can be used.

      • Only LINEARX, LINEARY, SPLIT_XY, TE, TM, and UNPOLARIZED polarization types are supported. The polarization type used must be correct for the source.

    Tip

    The sector map can be viewed with the Source View tool to verify sectorization. See “Post-Generation - Optical Model Visualization Tools” for more information.

  7. Save the file. You can now regenerate the optical model using the opticsgen command.
    	calibrewb -a opticsgen -opticalfile filename -o new_directory_name
  8. Go to one of the next tasks:

    • Creating a DDM Library With masksim Batch

    • Creating a DDM Library Using the MaskSim GUI

Results

The new optical model is generated with the HHA kernels included. Additional optical files may be added to the optical model directory. This model is dense-only and does not generate .omf files.

  • The LINEARX, LINEARY, SPLIT_XY, TE, TM, and UNPOLARIZED polarization types are supported.

  • The ASML_DIPOLE, ASML_POLE, STANDARD source types are supported.

  • To model a split XY polarization source, use a composite source:

    • For Rotangle 0, use LINEARY 1

    • For Rotangle 90, use LINEARX 1

  • If a source symmetry error is reported, use the mergemap command to combine multiple source files. 

    	mergemap –out output -in sourcefile sym all

  • The file aux_info contains the kernel assignment information per sector, where sectors are identified by their geometrical center at theta and phi values.

    Note:

    Theta values in the aux_info file created by opticsgen are measured in radians, which is different than the degrees returned by the sectorinfo command. (To convert a theta value from aux_info to its sectorinfo equivalent, multiply the aux_info value by 180/pi.) The aux_info file reports symmetries in group types; sectorinfo returns each sector individually.
