VT5 Resist Model Commands List

This command creates an optimized VT5 model and an optimized optical model (or maskwriter model) using a fixed model optimization procedure that begins with a pair of “nominal” models.

Usage

modelflow
{Input Arguments}
[output
    [log log]
    [optical file] [opticalmod dir
    [optical2 file] [opticalmod2 dir]
    [optical3 file] [opticalmod3 dir]
    [setup file]
    [vt5 file]
]

Arguments

log log

Optional argument supplying a name for the log file. The log file shows details of each stage of processing. Default = modelflow.log. If “stdout” is specified, the log will be written to the stdout instead of a file. The log file contains debugging information.

optical file [opticalmod dir]

Optional argument supplying a filename and path location for the output file to create for the final optical parameters text file, which modelflow creates. The optional opticalmod keyword supplies a pathname for a full optical model directory that will also be generated. Both optical and opticalmod can be specified separately. Default = none. This option is ignored if the input optical file is encrypted (contains the “skipModelHeader 1” keyword).

optical2 file [opticalmod2 dir]

Optional argument supplying a filename and path location for the final second optical parameters text file. If opticalmod2 is supplied, a full optical model directory is generated as well. Default = none. This option is ignored if the input second optical file is encrypted (contains the “skipModelHeader 1” keyword).

optical3 file [opticalmod3 dir]

Optional argument supplying a filename and path location for the final third optical parameters text file. If opticalmod3 is supplied, a full optical model directory is generated as well. Default = none. This option is ignored if the input third optical file is encrypted (contains the “skipModelHeader 1” keyword).

setup filename

Optional argument specifying a unique file name for the final setup file. The modelflow command writes the final setup file to the disk after the modelflow run completes, and includes the final VT5 and optical models inline. Default = none. This option is ignored if any optical model used as input is encrypted.

vt5 file

Optional argument supplying a separate pathname for the final optimized VT5 model, which modelflow creates. Default = none.

Description

Tip

Nominal models are the initial models, specified by the setup file. In this case, the models are the optical model and the VT5 model used as a starting point for creating the optimized models.

The nominal models provide default values for parameters that are not optimized. modelflow evaluates variations to the nominal models, as specified by the command line. It optimizes parameters in distinct optimization stages that are performed sequentially.
