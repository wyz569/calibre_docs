Tutorial for Sandwich Test Patterns

The sandwich test pattern templates allow you to create highly customized test patterns.

Getting the test pattern you want requires an understanding of how to use the parameters for the test pattern to create the intended behavior.

Basic Parameter Definitions

The following parameters are the minimum requirement for a sandwich test pattern.

  • Layers, EWidths, and EThicks — A test pattern is comprised of polygon lines with a defined width and thickness. In the case of test patterns, one line horizontally (for horizontal sandwiches) or vertically (for vertical sandwich patterns) is referred to as a layer, and the Layers keyword is used to set the total number of layer lines that appear in the test pattern.

    • For horizontal patterns, “Ewidths” defines the horizontal width, and “Ethicks” defines the vertical height, or layer thickness.

    • The first layer is centered on the gauge marker at the center of the pattern. Additional layers are added first above (or to the right of center for vertical patterns), then below (or to the left of center for vertical patterns), alternating between above/right and below/left for layers higher than three.

    The following figure shows one, four, and six horizontal layers:

    It was created with the following parameters:

    FEATURE rect_h_sandwich
Titles L1 L4 L6
Layers 1 4 6
Spaces 0.5 0.5 0.5
Ewidths 0.5 0.5 0.5
Ethicks 0.1 0.1 0.1
Espaces 0.2 0.2 0.2

  • Spaces — When you have multiple layers, the Spaces keyword defines the separation between layers.

  • Titles — Each test pattern entry must have a title associated with it; this is the entry that is used in the accompanying spreadsheet.

  • Ecounts and Espaces — Each layer has one polygon at the specified Ewidths and Ethicks width and height by default. Adding the optional Ecounts and Espaces parameters creates multiple polygons along the horizontal or vertical layer line, spaced the specified distance apart.

    Layers are always symmetrical about the specified axis. An odd value of Ecounts places the first polygon at the center of the pattern. An even value of Ecounts places a space at the center of the pattern.

    The following figure shows Ecounts of two, three, and four for horizontal and vertical layers:

    The vertical patterns were created with the following code fragment:

    FEATURE rect_v_sandwich
Titles C2 C3 C4
Layers 1 1 1
Spaces 0.5 0.5 0.5
Ewidths 0.5 0.5 0.5
Ethicks 0.1 0.1 0.1
Espaces 0.2 0.2 0.2
Ecounts  2 3 4

Adding P and Q Layers

Because not all test patterns are symmetrical across layers, the test pattern syntax allows you to define up to two additional separate layer definitions in the same test pattern, designated as P and Q. P and Q layers have corresponding parameters that match the basic E parameters:

  • Pwidths, Pthicks, Pcounts, Pspaces

  • Qwidths, Qthicks, Qcounts, Qspaces

If any Q parameter is not specified, the corresponding P parameter is used for it.

If any P parameter is not specified, the corresponding E parameter is used for it.

The Layers parameter now represents the total number of layers in the test pattern, and the Players and Qlayers parameters change how many layer lines comprise each Players or Qlayers set according to the following rules:

Additional P layers are placed above (for horizontal sandwiches) or to the right (for vertical sandwiches) of the E layers.

Similarly, additional Q layers are placed below or to the left of the E layers.

When Players and Qlayers are not defined or are set to one, the order of placed layers builds outwards from the center, alternating with E then P above/right, and Q and E below/left.

When Player is a value larger than one, that number of defined P layers are placed. The next layer after that is a Q layer, on the opposite side of the E layer. (If no Q layers have been defined, the test pattern generator starts placing more P layers below/left of the E layer, because Q defaults to P.)

Advanced P and Q Layer Controls

When you use P and Q layers, you can offset the layers from the center E layer using the following parameters:

Offsets — Sets the amount of the offset. Positive values shift additional layers to the right (for horizontal sandwiches) or upwards (for vertical sandwiches), while negative values shift the layers to the left and downwards, respectively.

Offtypes — Sets the type of offset pattern.

  • Shift — Shifts all additional P, E, and Q layers by the specified amount.

  • Shear — Shifts individual layers in a stair-step pattern.

  • Alter — Only P and Q layers are shifted. E layers remain unshifted.
