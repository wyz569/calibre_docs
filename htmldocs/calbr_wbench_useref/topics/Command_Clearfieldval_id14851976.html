clearfieldval

Optical Model Commands

Calculates the clearfield value for an image in an optical model. This command loads either an optical file or an optical model directory. It may optionally load a DDM library. It calculates two values of the clearfield value, based on the full TCC and on the kernel approximation.

Usage

clearfieldval {-file opticalfile | -dir modeldirectory} [-ddm ddmlibrary [-dark]]
[-opticsgen] [-help]

Arguments

-file opticalfile | -dir modeldirectory

A required argument specifying the optical model as either a filename or a model directory. The path can be relative or absolute. The optical model must include HHA model information.

-ddm ddmlibrary [-dark]

An optional argument specifying a DDM library. If this is specified, the layer transmission or background transmission (whichever is larger) corresponding to each source sector is used. The DDM library must match the sectorization in the optical file. The optional -dark flag computes the dark intensity region instead of the default clear field region.

-opticsgen

If this argument is present, the kernels will be generated if they are not included in the input model.

-help

An optional argument that displays the syntax of the command.

Return Values

The command prints the clearfield values to the terminal and also returns them as the following string in Tcl:

clearfield_value_tcc clearfield_value_kernels

For large approxorder values in the optical file, these two values are very close. For smaller approxorder values, there is a difference between them due to the kernel approximation. Only the clearfield_value_tcc value is generated if an optical file is supplied unless the ‑opticsgen argument is also included. If a value is not calculated or there is an error, the command returns the value as ‑1.
