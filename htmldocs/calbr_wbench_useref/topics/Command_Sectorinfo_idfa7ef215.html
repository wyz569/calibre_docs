sectorinfo

Optical Model Commands

Analyzes an HHA-enabled, sectorized optical source.

Usage

sectorinfo -in optical_model_file [-out [matlab]] [-outputDir directory] [-chief_ray theta phi] [‑mode {angles | sigmas | sectorization | {energydist [-range min max]}}]

Description

This command returns the pertinent illumination settings for the masksim command.

  • Theta and phi values are in degrees.

  • Each sector is returned individually.

Note:

Theta values in the aux_info file created by opticsgen are measured in radians, which is different than the degrees returned by the sectorinfo command. (To convert a theta value from aux_info to its sectorinfo equivalent, multiply the aux_info value by 180/pi.) The aux_info file reports symmetries in group types; sectorinfo returns each sector individually.

Arguments

-in optical_model_file

A required argument specifying the Optical file inside an optical model directory.

-out [matlab]

An optional argument that when specified exports the symmetry-reduced source sector files. If the optional “matlab” keyword is specified, the exported files are in the MATLAB data format. Otherwise, the command outputs standard sourcemap files.

-outputDir directory

An optional argument specifying a directory to export the output files to. The default is the current working directory.

-chief_ray theta phi

An optional argument that specifies the angle at the (0,0) point in the source. This is used for EUV sources, where the chief_ray can be non-zero. The settings for theta and phi correspond to the off-axis and the azimuthal angle and are specified in degrees.

-mode {angles | sigmas | sectorization | {energydist [-range min max]}}

An optional argument that specifies the command mode output.

  • angles — Returns information about the theta and phi pairs for each sector of the illumination. This is the default mode.

  • sigmas — Returns sigma information for each sector of the illumination source.

  • sectorization — Returns exact sectorization information in the form of “hha_sector” lines that may be submitted in an Optical file with the “hha_sectorization manual” parameter in order to exactly replicate the sectorization of the given opticalfile.

    Using this option allows you to find “hha_sectorization manual” equivalent entries for other sectorization options like auto, composite, and energy.

  • energydist [-range min max] — Returns information related to how an “hha_sectorization energy sector_thresh” command would break up the source by returning a series of text lines containing three numbers: “sector_thresh total_num_sectors num_unique_sectors”.

    This line can be read as:

    For a given sector_thresh, a total number of sectors would be generated. Only num_unique_sectors are unique, and the rest are inferred from symmetry. Use the output matrix to better figure out which sector_threshvalue to use to achieve a desired number of sectors.

    The -range argument is used to explicitly set the search range for sector_threshas percentage values. By default, it uses the values min=1 and max=50, which correspond to 0.01 and 0.5 values for hha_sectorization energy, respectively.

Examples

% sectorinfo -in Optical
Optical file read successfully ...

polarization = UNKNOWN

Illumination settings for masksim:
********************************************
illumination theta 0.000000 phi 0.000000
********************************************
