FEATURE Keyword

Test Pattern Commands

Defines a single test pattern structure as part of a test pattern file.

Note:

Must immediately follow a PATTERN Keyword, except for topo test patterns, in which case a FINFET keyword may precede it.

Usage

FEATURE feature_type [ROW number] [COL number] [NONEWLINE]
subkeyword1 units [units]…
subkeyword2 units [units]…
…
[COLUMNID {letter | number}]

Arguments

feature_type

A valid feature type listed in “Feature Types and Associated Subkeywords”.

Note:

See also “Test Pattern Geometry Descriptions” for a list of what structures appear in the various test pattern versions.

ROW number

An optional keyword that instructs the test pattern generator to place this feature on the specified row.

COL number

An optional keyword that instructs the test pattern generator to place this feature in the specified column.

NONEWLINE

An optional keyword that instructs the test pattern generator to append this feature to the current row.

COLUMNID {letter | number}

An optional keyword that is placed after the subkeywords. If specified, an identifier row of letters or numbers is placed above each column directly beneath the new feature row, which has the effect of writing two rows (one for the feature and one for the column IDs).

subkeywordn units…

A required (in most cases) set of one or more subkeywords that specify information about the feature type. See the following section, “Feature Types and Associated Subkeywords,” for a list of subkeywords. Each subkeyword has one or more units (or in the case of the Titles subkeyword, string) values following it, separated by spaces.

Description

Feature Types and Associated Subkeywords

The subkeywords vary according to feature_type.

You must specify one units value for each feature in the array. Values for units must be real numbers. Non-numerical value for units will result in a blank column.

The value for units (in any subkeyword) should never exceed:

            (blocksize - 2 * inset)

where blocksize and inset are defined through the PATTERN Keyword. A value of 4um is used by default.

  • If a non-inverse feature (such as iso or contact) geometry has a size greater than blocksize, it may be ignored by tepCompile, which will continue to the next block.

  • Output geometries may be clipped to the block’s size.

  • Use rect_v_array, rect_h_array, and rect_matrix structure types to generate structures in these situations.

Sets the offset type between slide (shift top and bottom), shear (shift top and bottom layers in opposite directions in a staircase pattern), and alter (only alternating P and Q layers are moved).

Qlayers n

Qspaces units

Sizes {units}+
