Creating a Global Litho Model Manually

A global litho model is used during EUV through-slit modeling. It is an extended form of a litho model that contains multiple DDM and optical models (referred to as sub-litho models) at multiple x-coordinates in the EUV field. Each litho model represents a slit location.

The use of global litho models is optional, but can improve the quality of your results if you have through-slit optical model information available.

Note:

You can use a wizard in Calibre nmModelflow to create a global litho model. See “Creating a Global Litho Model in Calibre nmModelflow”.

Prerequisites

At least two previously-created litho models to be contained inside the global litho model.

Procedure

  1. Create a text file named Lithomodel in an empty working directory. This file is used as the key file for the global litho model.
  2. Copy the litho models that form the global litho model into the working directory.
  3. Edit the Lithomodel file, using the following format:
    version 1 
lithotype EUV_RETICLE 
[dynamic_model_generation [all | none | ddm | optics]
litho_model sub_dir_name_1 x xcoord_um_1 
litho_model sub_dir_name_2 x xcoord_um_2 
[litho_model sub_dir_name_i x xcoord_um_i] 
...

    where

    • lithotype EUV_RETICLE — A required parameter declaring that this Lithomodel key file contains an EUV reticle definition. It must be typed exactly as shown.

    • dynamic_model_generation {all | none | ddm | optics} [slit_resolution val_um] — An optional parameter that controls which models are dynamically generated between slit positions. If the first argument is not specified, the default is “ddm”.

      • “all” means both ddm and optical models are dynamically generated.

      • “none” means that no models are dynamically generated. The model for the slit position closest to each tile center is used instead.

      • “ddm” means that the DDM model is dynamically generated based on a combination of nearby DDM libraries and the spatial location.

      • “optics” means that the optical model is dynamically generated using a combination of nearby optical models and the spatial location. When a tile is not located between two slit positions, the optical model for the nearest slit position is used.

      If “ddm” or “optics” is specified alone, then the model that was not specified is not dynamically generated.

    • litho_model sub_dir_name x xcoord_um — A parameter identifying a subdirectory containing a discrete litho model to be used at a slit position. There must be at least two litho_model lines, and no more than 128 litho_model lines. xcoord_um is the x-coordinate in microns relative to the slit center to which this submodel applies.

      All x-coordinates must be unique. They are not required to be at fixed intervals.

  4. Save the file.

Results

You can now call the global litho model as if it were a normal litho model. The following product commands support global litho models:

  • Calibre OPCverify

    • setlayer image command

    • setlayer shadow_bias command

  • Calibre nmOPC

    • setlayer image command

    • pw_condition command

    • DENSE_SIMULATE command

  • SVRF RET FLARE_CONVOLVE command
