Kernel and SOCS Modifications

Several functions are available to advanced users to modify the standard SOCS approximation. These include:

  • Selecting individual kernels.

  • Changing kernel type from the SOCS kernel to either Gaussian or Tophat.

    The replaced kernel can be used as is or convolved with the SOCS kernel.

  • Scaling individual kernels by values other than the SOCS eigenvalues.

  • Using either squared or unsquared summations of convolutions.

These special features are used in cases when the image is not formed optically, such as in electron beam lithography used in mask patterning. Figure 1 summarizes the mathematical model features that can be controlled.

Specifying Image Normalization

The following optional setting allows normalization of the image created by the optical model. Modifying the image normalization can affect TCCcalc and rapidTCC output results. This setting defines the image intensity of a transparent mask area that contains no geometry, sometimes referred to as “open frame intensity (OFI)”. In the simple aerial image models, normalization by the source area results in OFI of 1 in most cases. source_area normalization uses analytical integration of the source intensity. integ_source_area normalization uses numerical integration source intensity. For thin film models, OFI is generally not 1. first_interface normalization that sets the OFI to 1 at the first ambient/film interface is used.

For most production models, the default normalization is appropriate.

  • Defaults (for optical model versions < 5):

    • Aerial models or, in general, models in a uniform ambient medium (scalar or vector): source_area whenever possible, else integ_source_area

    • Thin-film models (scalar or vector): first_interface

  • Defaults (for optical model versions >= 5): source_area whenever possible, else integ_source_area

Specify image normalization in the Optical Model Tool Misc tab. In the Text field, enter the image_normalization keyword with settings according to the following table:

Accounting for Lens Aberrations

You can incorporate known measured lens aberration data, available from the scanner manufacturer, into the optical model.

Aberrations should be used in an optical model only if:

  • The aberrations do not change across the exposure field.

  • The aberrations are fully characterized, and symmetrical.

One should never use aberrations as extra parameters for fitting other physical effects such as mask effects.

Calibre WORKbench uses Zernike terms to model aberrations. You can enter these in the Pupil tab of the Optical Model Tool (Optics > Optical Model). Configure the controls in the lower half of the window so that they match the Zernike polynomial for your stepper. The settings appear in the Preview tab as zernike and zernikemultiplierflag.

Note:

Lens aberrations contain both symmetric and asymmetric terms. Application of an asymmetric Zernike coefficient in an OPC model causes the correction to asymmetric for different orientations of the same feature. Hence using asymmetric models requires careful consideration of cell placement in the design relative to the mask orientation in the exposure tool. For this reason, a safe best practice would be the application of only radial symmetric coefficients (spherical aberration and defocus).

In addition, the application of asymmetric models requires cloning of cells in a hierarchical run.
