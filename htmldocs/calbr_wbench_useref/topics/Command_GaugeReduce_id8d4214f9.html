gauge_reduce

Optical Model Commands

Reduces a set of input gauges to a subset based on optical similarity.

Usage

gauge_reduce {{-ggin gg_in} | {‑sitesin sites_in}}
{{-ggout gg_out} | {‑sitesout sites_out}}
-nout N
{ -lithomodel lithomodel_dir |
-model model_dir
-field_tone {clear | dark}
[-atten attenuation] }
{-layout layout_handle}
-layernums "layer1 [layer2 …]"
[-gauge_length {auto | input | val_in_um}]
[-approxorder {auto | input | numkern }]
[-enable_bad_gauges {1 | 0}]
[-group {0 | 1}]
[-matrix_out filename1]
[-matrix_in filename2]
[‑parameterization OMW | AM]
[‑method coverage | diverse | representative]
[-h | -help]

Arguments

-ggin gg_in

A required argument that specifies an input gauge file that will be reduced. Either -ggin or ‑sitesin must be specified for gauge_reduce input. Both .gg and .sgd file formats are supported. Disabled input gauges are ignored. If an .sgd file is input, then it must contain a single process condition.

‑sitesin sites_in

A required argument that specifies an input sites file that will be reduced. Either -ggin or ‑sitesin must be specified for gauge_reduce input. Using site files requires specifying ‑parameterization AM. The CSV Sites file format (or sites file) is described in the Calibre nmModelflow User’s and Reference Manual’s “Sites CSV File Guidelines” section of Contour Analysis Tab.

-ggout gg_out

A required argument that specifies an output gauge file name where the reduced set of gauges is saved. The file is identical to gg_in, with the chosen gauges enabled and the remaining gauges disabled.

‑sitesout sites_out

A required argument that specifies the sites file name where the reduced set of sites is saved. The file is identical to sites_in, with the chosen sites enabled and the remaining sites disabled by setting the corresponding site weight (SW) to zero.

-nout N

A required argument specifying the number of enabled gauges or sites in the output file.

-lithomodel lithomodel_dir

A required argument that specifies a lithomodel directory that the optical match is calculated from. Either -lithomodel or -model and associated arguments must be specified. Specifying -lithomodel replaces the -model, -field_tone, and -atten arguments.

-model model_dir

A required argument that specifies a generated model directory used to calculate the optical match. Either -model or -lithomodel and associated arguments must be specified.

-field_tone {clear | dark}

A required argument that specifies the tone of the background for the mask. Specify clear when the background is clear and polygons are dark. Specify dark when the polygons are clear and background is dark.

-atten attenuation

An optional argument that specifies the transmission value of the dark region with a 180-degree phase shift. This is identical to the atten command for setting the mask transmission in a setup file. The default is 0.

-layout layout_handle

Specifies a single layout that includes the gauges in gg_in. The layout must be flat.

-layernums “layer1 [layer2 …]”

A required argument that specifies the layer number and datatype from the layout(s) to be included in the simulations (for example, 15.1 for layer 15 datatype 1). The layer numbers and datatypes must be enclosed in double quotes with each layer separated by a space.

-gauge_length {auto | input | val_in_um}

An optional argument that specifies the gauge length simulated using the following options:

  • auto sets the gauge length simulated to the hoodpix value (that is, the optical diameter) of the optical model.

  • input sets the gauge length simulated to the length of the gauges in the input gauge file.

  • val_in_um is the value (in microns) of the gauge length simulated.

This argument affects only the simulation and the output gauge lengths are not affected. The default is auto.

-approxorder {auto | input | numkern}

An optional argument that specifies the number of kernels simulated using the following options:

  • auto sets the value to 30 if the number of kernels in the input optical model is greater than 30. Otherwise, the number of kernels in the input optical model is used.

  • input sets the value equal to the number of kernels in the input optical model.

  • numkern is an integer that specifies the number of kernels used from the input optical model. The value must be less than or equal to the number of kernels in the input optical model.

-enable_bad_gauges {1 | 0}

An optional argument that enables gauges that cannot be resolved in the output gauge file. If a gauge cannot be resolved, then it cannot be determined if it is an appropriate gauge. Specifying “1” enables all gauges that cannot be resolved. Specifying “0” disables all gauges that cannot be resolved.

The total number of enabled gauges in the output gauge file is always set by -nout. If the number of bad gauges is above -nout and ‑enable_bad_gauges 1 is specified, then an error will occur. The default is 1.

‑group {0 | 1}

An optional argument that runs gauge_reduce on each unique group specified in the group column of the input gauge file. The number of enabled gauges for each group in the output file is distributed in proportion to the number of enabled gauges for each group in the input file. The output gauge file contains a minimum of one enabled gauge for each unique group. Specifying “1” enables group gauge reduction. Specifying “0” disables group gauge reduction. In this case, the group id is ignored and gauge_reduce runs on all groups together. The default is 0.

-matrix_out filename1

An optional argument that saves the similarity matrix to the specified text file in a comma-separated value (CSV) format. Each row and column of the file corresponds to a calibration gauge. Each value is the “similarity” of each pair.

-matrix_in filename2

An optional argument specifying a text file containing a similarity matrix for the input gauge file. The file is generated using the matrix_out option of optmatchweight.

‑parameterization OMW | AM

An optional argument specifying the matrix type used to select gauges.

  • OMW selects gauges using optmatchweight to create a similarity matrix. The OMW matrix type cannot be selected with sites. This is the default matrix type.

  • AM selects gauges or sites using an attribute matrix for the set of gauges or sites based on the input Lithomodel. Each attribute is a combination of the contribution of a specific kernel or term to the image at a specific location and the slope of the contribution at that location. All simulations include the mask geometry information (for example, bias, corner chop, and so on) and DDM information from the Lithomodel. If sites files are used, the AM matrix type must be specified.

‑method coverage | diverse | representative

An optional argument specifying how gauges or sites are selected. Specify one of the following selection methods:

  • coverage selects gauges with minimum optical similarity so that any gauges or sites excluded from the output are covered by optically similar gauges or sites included in the output. The coverage method is the default.

  • diverse selects the most similar gauge to each eigenvector until N gauges are selected. The diverse method uses optmatchweight to create a similarity matrix, and singular value decomposition (SVD) is performed on the matrix. The diverse method selects gauges or sites using the calculated eigenvalues and eigenvectors.

  • representative selects the number of gauges or sites similar to each eigenvector in proportion to their eigenvalues. The representative method uses optmatchweight to create a similarity matrix, and singular value decomposition (SVD) is performed on the matrix. The representative method chooses more gauges or sites matching eigenvectors with larger eigenvalues and fewer gauges or sites from eigenvectors with smaller eigenvalues.

-h | -help

An optional argument that prints the command syntax.

Description

This command reduces a set of input gauges or sites clips to an optically similar subset using optmatchweight. The input gauges or sites clips are compared for similarity and a representative subset is generated.

While performing a similar function, the gauge_reduce command differs from cwb_sample in the following:

  • The criteria used to characterize each gauge or sites clip.

  • The support of input sites clips and gauges. The cwb_sample command supports simultaneous input of both gauges and sites clips. The gauge_reduce command supports processing one file type in a single instance.
