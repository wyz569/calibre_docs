ss2gauge

CM1 and VEB Commands

Related Formats: Gauge Object Reference

Creates a gauge object from a sample data spreadsheet file.

Usage

ss2gauge -ss spreadsheet {-l wblayout | -m layoutfile} -layer layerno [-extra extra]
[-force [horizontal | vertical] | center] [-length factor] [-mode mode count dist][-etch]
[-search distance] [-skipped string]

Arguments

-ss spreadsheet

A required argument that specifies a spreadsheet filename to convert.

The -l and -m arguments are mutually exclusive, and one of the two must be specified.

-l wblayout

A required argument that specifies a previously-loaded Calibre WORKbench layout name.

-m layoutfile

A required argument that specifies a layout file to load, in one of the supported formats (such as GDS or OASIS).

-layer layerno

A required argument specifying the layer number where the measurements are to be done.

-extra extra

An optional argument specifying that the gauge must be lengthened by 2*extra nanometers. Default is 200 nanometers.

-force [horizontal | vertical]

An optional argument that forces Calibre WORKbench to produce possibly bad gauges (such as gauges not centered on the nearest edge). Additional qualifiers, horizontal and vertical, can be specified to convert the sample points to the specified type.

-center

An optional argument that instructs Calibre WORKbench to center gauges if the input sites are not centered on the nearest edge.

Note:

-force and -center are mutually exclusive options. The default is for neither option to be selected (Calibre WORKbench skips over sites that are not centered).

-length factor

An optional argument that filters out gauges whose net length in nanometers (before adding -extra) is the specified factor less than the measured CD. The default is to not exclude any gauges.

-mode mode count dist

An optional argument that specifies the mode of operation, dependent on the following values:

  • mode — one of 0, 1, 2 or 3, corresponding to normal, average, min or max measurements. The mode specifies how CD and space measurements are done if subgauges are present.

  • count — number of subgauges added to the left and right of the main gauge (the total number of subgauges is 2*count +1).

  • dist — distance between subgauges, in database units.

The default value is to create one gauge and use normal mode for measurement.

-etch

An optional argument that switches the ss2gauge command to treat the sample spreadsheet file as a variable etch bias (VEB) file. This transfers the data into the appropriate etch fields in the gauge object instead of into the measured resist fields (writing to the measured resist fields is the default).

Information on variable etch bias files can be found in the section “Variable Etch Bias (VEB) Model File Format”.

-search distance

An optional argument that specifies the search distance radius in nanometers. This is the area around a spreadsheet measurement that are searched for the edges crossed by the gauge. The default is 500 nm.

-skipped string

An optional argument that outputs to the specified string the list of skipped sample points that were not converted to gauges.

Description

Produces a Tcl gauge object from the specified spreadsheet. The command requires the following conditions:

  • The input database must be flat; if you have a hierarchical database, flatten it using the layout copy1 or clipLayout commands.

  • If measurements were done on several layers, you must merge the layers before running ss2gauge.

  • The spreadsheet file must be in the following format:

    structure_name row column x y loc drawn other meas weight

  • The loc value accepts the following values as a directive where to look for the nearest CD (inside polygon edge) or space (outside polygon edge) to place the gauge:

    • 0 - nearest CD

    • 1 - nearest space

    • 2 - nearest horizontal CD

    • 3 - nearest horizontal space

    • 4 - nearest vertical CD

    • 5 - nearest vertical space

In some cases, it is possible to receive the error message “No edges found near ss point” even after merging the measurement layers and increasing the search distance. This can occur when using double-exposure masks (such as altPSM), where the second exposure is used partially to protect some area from the first exposure, resulting in overlapping features that are missed by ss2gauge. For these situations, split your input sample into separate files for each measurement layer, converting sample data to gauge files, and then merge the resulting gauge files together.

Examples

This example is for a flat database and creates the gauge object “aobj” from the specified GDS file and spreadsheet. It creates the gauges from the objects on layer 0 and lengthens the gauge by 20 dbu (2*10).

set aobj [ss2gauge -ss sram.ss -m “sram.gds” -layer 0 -extra 10]

This example is for a hierarchical database, which requires the layout to be flattened before gauge objects can be created:

set l_obj [layout create sram_hier.gds]
layout copy $l_obj l_obj_flat
set aobj [ss2gauge -ss sram.ss -l l_obj_flat -layer 0 -extra 10]
