Optimizing the HHA-Enhanced Optical Model

Because you have modified your optical model for HHA approximations, you must recompile and recalibrate the optical model. This procedure describes modifying a model layer file to load the DDM model file before optimizing the optical model.

Prerequisites

  • The optical and DDM models from “Modifying an Optical Model File for DDM and HHA” and “Creating a DDM Library With masksim Batch”

  • The corresponding resist model

  • A design file containing a test pattern

Procedure

  1. In a text editor, create a new model-layer file. It must include the following keywords: 
    modelpath path
opticalmodel {foldername | inline}
resistpolyfile {filename | inline} 
background bgtype [factor]
ddm_model ddm_id filename
layer layernum real imag ddm_id

    where:

    • modelpath path

      Defines the directory paths to search for models; separate multiple directories by colons (:). Required if any model is not inline.

    • opticalmodel {foldername | inline}

      Specifies the name of the optical model directory. If inline is specified instead of a filename, it indicates that the model is defined using the inline_optical1 keyword.

    • resistpolyfile {filename | inline}

      Specifies the CM1 resist model name. This should at least be a CTR model even if you are only using an aerial image model.

    • background bgtype [factor]

      Specifies the background transmission for the masks.

      • bgtype - Required. Must be either clear, dark, or attenuated.

      • factor - Optional. Specifies the factor for attenuated backgrounds.

    • ddm_model ddm_id filename

      A required argument for DDM/HHA. ddm_id specifies a string ID name for the specified filename, which must be set to either the bright or dark DDM library generated by the masksim statement. This ID name will be used in subsequent layer statements.

    • layer layernum real imag ddm_id

      At least one layer statement must be specified; each layer statement specifies a layer’s parameters.

      • layernum - An integer used to specify the layer on the layout.

      • real imag - A required pair of values that specifies the real and imaginary components of the transmission value. These values will be overwritten by the contents of the DDM model, and are replicated here for user reference.

      • ddm_id - An ID name for a ddm library defined using the ddm_model statement above.

  2. Save the file, then recalibrate the optical model (and resist model if you have a CM1 resist model enabled) in CM1 Center or in modelflow_v2.

    • Use the new model-layer file as the argument to the -mlfile argument.

    • Calibrate the following variables:

      • def_start

      • beamfocus

    • Be sure to specify stage1 dense and stage2 dense if you are using DDM optical models during those calibration stages.

Results

When optimized, the new optical model and resist model are usable in simulations.
