Flare Model File Format

Input for: flare models in Calibre OPCverify or Calibre nmOPC.

Describes a flare model.

The flare model file is created in one of the following ways:

  • Entered manually, and associated with a PSF file. The flare model reads information from the supplied PSF file, and only uses the longform grid, shortform grid, TIS auto, and psfgridfile file options.

  • Entered manually to model a double fractal kernel. The flare model uses the values in its kf, ka, aspect, balance, s1, s2, rmin, rmax, and rsl arguments to model the PSF behavior. The double fractal kernel equation is as follows:

    The n1, n2, A1, and A2 values are automatically derived from the input PSF data for PNG files. Any values smaller than the smallest defined rmin value are assigned a value of 0.

  • Generated automatically. Users who have a flare data file can enter the data in the Flare Intensity File Format and use that file as input for the euv::fitflare command. The output from the command is a flare model file.

Note:

When an optical model with an associated flare model file is exported from the Calibre nmModelflow tool, the flare model file is saved in the same directory as the Lithomodel key file.

Format

A flare model format (.fmf) file must conform to the following restrictions:

  • Comments begin with the pound sign (#).

  • Must be an ASCII text file.

  • Each parameter must have its own line.

[version 1] 
[TIS TIS | auto]  
[kf kf_value]  
[ka ka_value]  
[aspect u]  
[balance a]  
[s1 s1_value]  
[s2 s2_value]  
[rmin rmin]
[rmax rmax] 
[rsl rsl_dist] 
[assist value]
[flarelevel value] 
[psf_start  
	... 
 psf_end] 
[psfgridfile]

Parameters

version 1

Specifies the version number of the flare model file. The only available value is 1.

TIS {TIS | auto}

Specifies the total integrated scatter (TIS) value of the PSF. If it is not given, or “auto” is specified, then the TIS value is automatically calculated from the input PSF file. Default is auto.

kf kf_value

Specifies the flare coefficient. Default is 1.

ka ka_value

Specifies the double Gaussian (dgauss) coefficient. Using a value other than 0 activates fractal kernel calculations. Default is 0.

aspect u

Specifies the x and y scaling aspect. Default is 1. Using a value of 2 activates asymmetric flare modeling.

balance a

Specifies the proportion of the second Gaussian kernel. Varies from 0 to 0.5. Default is 0.

s1 s1_value

Specifies the diffusion length of the first Gaussian kernel in microns. Can be set only for the dgauss kernel type. Default is 0.4 microns.

s2 s2_value

Specifies the diffusion length of the second Gaussian kernel in microns. Can be set only for the dgauss kernel type. Default is 0.2 microns.

rmin rmin

Specifies the lower bound for the fractal kernel in microns. Default is 0.1 microns.

rmax rmax

Specifies the higher bound for the fractal kernel in microns. Default is 1.2 microns.

rsl rsl_dist

Specifies the interaction radius in microns of the division between short and long range flare data.

assist value

Specifies a scale multiplier to approximate the contribution of assist features to the intensity. This value should be proportional to the ratio of side lobes to the main peaks, typically around 0.15.

flarelevel value

Specifies the flare value for a layer, in order of the layer definitions. Supported only for OASIS format flare files.

psf_start

...

psf_end

Specifies headers which are placed at the beginning and end of a list of PSF pair values.

psfgridfile file

Specifies a file with PSF values. If this argument is present, then fractal kernel parameters are ignored, and PSF values are read from this file. Use of this argument is not recommended.
