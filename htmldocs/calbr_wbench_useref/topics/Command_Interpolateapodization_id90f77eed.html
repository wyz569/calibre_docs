interpolateapodization

Optical Model Commands

Interpolates two apodization files based on slit positions.

Usage

interpolateapodization -in1 apod_file1 -slit_x1 val1 -in2 apod_file2 -slit_x2 val2
-slit_xout valout -out filename [ -help ]

Arguments

-in1 apod_file1

A required argument specifying the first apodization input file.

-slit_x1 val1

A required argument specifying the slit position of the first apodization input file in microns.

-in2 apod_file2

A required argument specifying the second apodization input file.

-slit_x2 val2

A required argument specifying the slit position of the second apodization input file in microns.

-slit_xout valout

A required argument specifying the slit position for the output file in microns.

-out filename

A required argument that specifies an output apodization filename.

-help

An optional argument that displays help information.

Description

This command takes two apodization text files, interpolates them linearly based on the slit positions input, and outputs a new apodization text file. For the format of the text files, see the pupilApodization description in the Optical Parameters File Format. In addition, the following checks are also run on the input files:

  • The grids in each input file must be the same size.

  • Each point must be valid for both files or unknown for both files.

  • The slit values must be between -15000 and 15000 microns, because this is approximately the physical limit of an EUV scanner slit, but also allows for a DDM library at the slit edge.
