optmatchweight

Optical Model Commands

Modifies the weights in a gauge file. It is intended to be used to improve the accuracy of a calibrated model for predicting a set of verification patterns.

Usage

optmatchweight -model model_dir
{-layout layout_handle | -layout_cal layout_handle_cal -layout_ver layout_handle_ver}
-layernums "layer1 [layer2 …]"
-field_tone {clear | dark}
-ggcalibrate gg_cal
-ggverify gg_ver
-ggout gg_out
[-gauge_length {auto | input | val_in_um}
[-approxorder {auto | input | numkern }
[-atten attenuation]
[-matrix_out filename]
[-h | -help]

Arguments

-model model_dir

A required argument that specifies a generated model directory that the optical match is calculated from.

Note:

One of either -layout or -layout_cal with -layout_ver is required.

-layout layout_handle

Specifies a single layout that includes the gauges in gg_cal and gg_ver. This argument is not compatible with the -layout_ver and -layout_cal argument. The layout must be flat.

-layout_cal layout_handle_cal -layout_ver layout_handle_ver

An argument pair specifying two separate layouts that include the gauges in gg_cal and the gauges in gg_ver respectively. These arguments are incompatible with the -layout argument. The layouts must be flat.

-layernums “layer1 [layer2 …]”

A required argument that specifies the layer number and datatype from the layout(s) to be included in the simulations (for example, 15.1 for layer 15 datatype 1). The layer numbers and datatypes must be enclosed in double quotes with each layer separated by a space.

-field_tone {clear | dark}

A required argument that specifies the tone of the background for the mask. clear means that the background is clear and polygons are dark. dark means that the polygons are clear and background is dark.

-ggcalibrate gg_cal

A required argument that specifies an input gauge file that includes the calibration gauges, which can either be a gauge file or a super gauge file with a single process window condition. It does not need to include any measurements.

-ggverify gg_ver

A required argument that specifies an input gauge file that includes the verification gauges, which can either be a gauge file or a super gauge file with a single process window condition. It does not need to include any measurements.

-ggout gg_out

A required argument that specifies an output gauge file. The contents of the file are identical to gg_cal, but with updated weights.

-gauge_length {auto | input | val_in_um}

An optional argument that specifies the gauge length simulated using the following options:

  • auto — Sets the gauge length simulated to the hoodpix value (that is, the optical diameter) of the optical model.

  • input — Sets the gauge length simulated to the length of the gauges in the input gauge file.

  • val_in_um — Is the value (in microns) of the gauge length simulated.

This argument affects only the simulation and the output gauge lengths are not affected. The default is auto.

-approxorder {auto | input | numkern}

An optional argument that specifies the number of kernels simulated using the following options:

  • auto — Sets the value to 30 if the number of kernels in the input optical model is greater than 30. Otherwise, the number of kernels in the input optical model is used.

  • input — Sets the value equal to the number of kernels in the input optical model.

  • numkern — Is an integer that specifies the number of kernels used from the input optical model. The value must be less than or equal to the number of kernels in the input optical model.

-atten attenuation

An optional argument that specifies the transmission value of the dark region with 180 degree phase shift. This is identical to the atten command for setting the mask transmission in a setup file. The default is 0.

-matrix_out filename

An optional argument that saves the similarity matrix to the specified text file in a comma-separated value (CSV) format. Each row of the file corresponds to a calibration gauge, and each column corresponds to a verification gauge. Each value is the “similarity” of each pair.

-h | -help

An optional argument that prints the command syntax.

Description

The final weight in the output gauge file for each gauge depends on how similar each gauge is to a set of verification patterns. If a calibration gauge is very similar to several gauges, it is given a higher weight. This function is valuable because no measurements are needed. Calibration weights can be chosen to match any verification patterns, even if measurements for those patterns are not available.

The similarity of the gauges is determined by simulating every gauge with every kernel in the input model individually.
