Hints for CM1 Calibration

The following section suggests some additional hints for CM1 calibration settings.

  • Dependent and Independent Terms for Constraints

    In the Optimize tab, setting a Constraint column to a value $vn where v is one of the s parameters and n is a term number creates a dependency on that parameter.

    Figure 1. Calibration Constraints

    For example, setting a constraint of $s8 for terms 13, 14, and 15 means that terms 8, 13, 14, and 15 will be varied simultaneously, and also that variations of terms 13, 14, and 15 will vary precisely with the variations of term 8 (that is, s13=s14=s15=s8).

    • Setting the Constraint column to a value of NA makes that term an independent variable, or in other words, this term will be varied independently of the others.

    • Default dependencies will appear in the constraint column when you select a modelform. These may be edited, but note that removing a Constraint (such as replacing $s10 with NA) produces more independent variables and slows down the optimization runtime.

    • Do not use b term dependencies.

  • The main entries for calibration adjustment are the Begin, End, and Levels (# of steps) columns. Feel free to edit those as desired, keeping runtime consideration in mind.

  • For coarse calibration using full factorial search the total range explored can be reasonably large. The number of levels should be high enough such that the result of coarse calibration is close to the absolute minimum that the fine calibration would find the absolute minimum.

  • You can run optimization in batch mode, using the modelflow_v2 command.

  • Use the log file contents to examine the difference between the CM1 model and CTR model by finding the CTRtoCM1 value. The smaller this value is, the more the final model is CTR-like.
