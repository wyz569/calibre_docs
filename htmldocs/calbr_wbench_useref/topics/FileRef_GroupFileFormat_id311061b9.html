Group File Format

CM1 and VEB Model Formats

Input for: modelflow_v2, sgd_create_group_specs, sgd_get_error_stat

Defines an auxiliary input file that groups gauge objects together for more precise user control over gauge tolerances, shifts, skew factors, and weights.

The gauge format supports a Group column that sets a gauge’s group identifier. This file contains the information used for that group.

Format

A group file must conform to the following restrictions:

  • Use ASCII text.

  • Have the .gsf extension.

  • Start comments with the hash symbol (#).

    name
tolerance shift skew weight

Parameters

name

Specifies the name of the group. This must be a unique combination of at most 256 alphanumeric, +, -, or underscore characters without whitespace. Any enabled gauge with its Group field containing this string is considered part of the group.

tolerance

Specifies the error tolerance for the group, expressed in either nanometer (for example, “6.1nm”) or percentage (for example, “5.5%”) format. The use of “nm” or “%” is required and signifies which tolerance type to use.

shift

Specifies the error shift in nm. If no shift is needed, you must still specify 0.

skew

Specifies the skew factor as a positive dimensionless value.

weight

Specifies the weight factor as a positive dimensionless value.

Examples

# name tolerance shift skew weight
gn1 5% 0 1 2.1
group_two 1.1nm 0 2 1.3
inv_iso 3nm 0 1 2.2
