layer

Optical Model Commands

Defines layer characteristics such as name, type, transmission, and mask

Note:

The layer command is ignored if DDM libraries are used with an optical model.

Usage

layer layer_name layer_type transmission [mask_num dose]

Arguments

layer_name

A required argument specifying the name of the layer. Calibre nmOPC uses this name in the rest of the setup file to refer to this layer.

Layer names must be alphabetic and numeric strings less than 32 characters in length.

layer_type

A required parameter that specifies how the layer is used by Calibre nmOPC. The layer type is typically ignored by Calibre nmOPC, but in cases where the input layer is used for any non-opc simulation, the layer type should be defined. The following table. lists the valid layer_types and explains how each type is used in simulations.

Table 1. layer Command layer_types

Layer Type

Function

hidden (optional)

By default, any layer not specified in the setup file is marked as a hidden layer. Hidden layers are used for a number of functions in Calibre nmOPC, including serving as retargeting layers and marker layers, or for re-opc and “no opc” regions.

visible (required)

Visible layers are used in optical simulations. Visible layers should not interact in any way with opc layers. Any overlap or abutment between an opc layer and a visible layer will result in false edges on the opc layer. False edges are layer edges that do not have any corresponding mask edges. In cases where correction is not wanted on some polygons, those polygons should be put on an opc layer.
transmission

A required argument for visible layers that defines the optical transmission type for the mask layer.

The optical transmission type for the layer must be the complement of the background. That is, you cannot specify clear features if you specify a clear background. Refer to the background command in this chapter for more information about physical layer transmission in relation to background. The following table lists the valid transmission choices.

Table 2. layer Command transmission Argument Settings 

Transmission

Description

clear

Defines the optical transmission for the layer to be 100%. (clear features)

dark

Defines the optical transmission for the layer to be 0%. (dark features)

real imag

Defines the optical transmission for the layer to be as specified by the real and imaginary value pair. Note that the attenuated syntax for the transmission is recommended over the real imag pair syntax.

If you specify background with the real imag pair for a specified layer transmission value, the resulting transmission value for the output layer with a real imag pair is resolved relative to the background pair. The layer transmission value is resolved according to the following equation:

background (real imag) + layer (real imag) = transmission_value

For example: 

background 1 0 
name 		 opt 	    type 
chrome 		visible 	0 0

The background is specified with a (1, 0) pair, while the chrome layer is defined with a transmission value of (0,0). With a clear background (1,0) and a layer specified with (0,0), the resulting transmission layer is (1,0) + (0,0) = clear (1,0). 

background 1 0 
name 		opt 			type 
p180 		visible 			-1 0

In this case, the layer p180 is defined as (-1 0). With clear background (1,0) and a layer specified with (0,0), the resulting transmission layer is (1,0) + (-1,0) = dark (0,0).

attenuated factor

atten factor

Defines the layer to be an attenuated phase shifting layer and the optical transmission to be a percentage of incident light, as specified by factor.

When Calibre LITHO tools encounter the attenuated factor command, they translate the factor into a real imaginary pair such that the attenuated background has a transmission value:

RE(layer) = -sqrt(percent/100)

IM(layer) = 0

The maximum allowed attenuation factor is 36.

phase60 | phase90 | phase120 | phase180 | phase270

Defines the layer to be a phase-shifting layer transmitting light with the given phase relative to the incident light.

mask_num dose

An optional parameter pair used with phase shifting masks to allow the simulators to support multiple masks.

  • mask_num — The number of the mask to which the layer belongs.

    The default is mask 0. When multiple masks are used, they must be numbered sequentially, beginning with 0. The opc layer must appear on mask 0.

  • dose — The light energy dose for the layer, expressed as a percentage of the full energy dose for the mask.

    The dose is specified as a positive real number. For example; 0.5 is 50% of the full energy dose. Values greater than one represent over exposures. Values less than one represent under exposures.

Description

A required setup file command that names and describes optical properties of an input layer, used in conjunction with the background setup file command. Every layer to be used in OPC or simulation operations must be declared in the setup parameters using this command.

Note the following:

  • The transmission and mask_num values are only used with the visible layer type.

  • Calibre OPCpro does not simulate layers designated as the hidden layer type.

Examples

Example 1

Declares two layers, one for the OPC layer (SHAPES) and one for the original target polygons (ORIG):

layer SHAPES visible dark
layer ORIG hidden dark

Example 2

Declares an attenuated phase shifting layer:

layer OPC visible attenuated 0.06 0 1

Example 3

Declares two alternating phase shifting layers:

layer PH0 visible clear 1 1.0
layer PH180 visible phase180 1 1.0
