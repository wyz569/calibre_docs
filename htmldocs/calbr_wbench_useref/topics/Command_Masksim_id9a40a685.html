masksim

Optical Model Commands

Runs the Calibre MaskSim batch command on the supplied input file.

Usage

masksim [-report_only {bright|dark}] -i input_file -o output_file

Arguments

-report_only {bright|dark}

Optional argument that only writes a report to standard out and does not generate a DDM file. The report contains layer and background information and the number of unique theta/phi/polarization combinations. This option is only valid for input files that contain the “make_ddm_edge_library” keyword.

The bright and dark options specify the type of field intensity reported.

  • bright — Reports the bright field intensity. This is the default.

  • dark — Reports the dark field intensity.

-i input_file

Required argument that specifies the input file on which to run.

-o output_file

Specifies an output file for the MaskSim log. This argument is required even if the ‑report_only option is specified.

Description

The input file must be in the format described in the “DDM Library Input File Format”.

For pre-2013.2 releases, before running this command with an HHA-enabled library, ensure that the optical model file contains the following required lines: 

    forcePower2LUT 0
    hha 1
    ddm_model full

When run, masksim creates a Domain Decomposition Method (DDM) Model Output File.
