Step 2 - Illumination Source Definitions

An illumination source that matches the optical lithography exposure conditions must be defined before the optical model is created.

In Calibre WORKbench, the illumination properties are configured using the Illumination Tab of the Optical Model Tool, accessed using the Optics menu, or as a line in the Optical Model file. It is simpler to use the Optical Model tool GUI since the relevant parameters are automatically shown whenever you choose a source type.

Select an illumination source type and configure its parameters. In Calibre WORKbench, the source can be defined in two ways:

  • “Using Pre-defined Illumination Primitives”

  • “Using an Imported Source Map”

Using Pre-defined Illumination Primitives

A number of illumination primitives are pre-defined for you in Calibre WORKbench. These can be selected using a keyword as shown in Figure 1. These sources have either sharp or smooth edges. Sources with sharp edges have unit intensity inside each shape and zero intensity outside. The error function with adjustable parameters is used for sources with smooth edges.

Using an Optical Model File

Optical model file designers who choose not to use the GUI tool must add a sequence of lines to their optical model file to designate the same functionality:

  1. One illumtype COMPOSITE line to designate that a composite source is being defined:

    	illumtype COMPOSITE

  2. One or more composite source lines, using syntax such as:

    	STANDARD SHARP weight 1.0 sigma 0.6 UNPOLARIZED 1.0
    	STANDARD SHARP weight 1.0 sigma 0.6 TE 0.5 TM 0.5
    Note:

    Users of composite sources with two or more ASML_DIPOLEs should be aware that only the first two sources can be optimized with the VT5 Center and CM1 Center tools. The mapping of ASML_DIPOLE source to the modelflow batch parameters is determined by the order listed in the Optical model file (the first ASML_DIPOLE line is assigned to sigma_out1, rotangle1, and so on, while the second ASML_DIPOLE line is assigned to sigma_out2, rotangle2, and so on).

Using an Imported Source Map

Another method of defining a source in the optical model is through the use of a source map when one is available. The source map can be obtained from measurements from the lithography scanner. A symmetrized version of the measured source map is considered to be the most accurate representation of the illumination suitable for OPC optical models.

In some cases, measured source can be represented parametrically by fitting the measured source using the fitmap command. For more information, see the section “Generating a Source Map File Using fitmap”.

The source map data must be in the source map file format described in “Source Map File Format”.

Follow these steps:

  1. In the Illumination tab, click the Add button.

  2. Select the shape type as STANDARD in the Add Source Element window.

  3. Select the MAP profile and enter 1 for the sigma value. This ensures that the entire source map is used.

  4. Enter the weight and Polarization.

  5. Enter the name of the source map file that contains the source data.

  6. When you are finished adding the source map information, go to the next task, “Step 3 - Selecting Optical Kernel Parameters”.

Calibrating Optical Models for SMO Sources

Note:

You can now use Calibre® pxSMO™ (documented in the Calibre SMO RET Selection User’s and Reference Manual) to automatically calibrate source mask optimization models.

Source Mask Optimization (SMO) models use the same recommendations as conventional optical source models. However, the following guidelines should be observed:

  • The recommended choice of engine is rapidTCC for optical diameters greater than 1.2 microns.

  • With the rapidTCC engine, you should also use the default convergence_tolerance value of 0.008. The upsample factor may need to be increased using the rapidTCC_uf command (with an upsample value between 2 and 3) if the convergence tolerance is not reached.

  • For optical diameters less than 1.2 microns, you can also use the TCCcalc engine, which should be used in conjunction with the AUTOMAP command (see “Using the AUTOMAP Illumination Primitive”).

  • A source map representative of the scanner resolution map, not a raw SMO pixel map, should be used as the file input to the AUTOMAP command.
