Using Optical Models with Double Exposure Flows

CM1 Center and modelflow_v2 are designed to work with single exposure optical models by default. However, support for double exposure optical conditions can be implemented using the following procedure.

Prerequisites

  • A super gauge object, containing 20 to 30 focus-sensitive features measured at three or more focus conditions plus the corresponding design file with the double exposure mask in it. Geometries on the same mask must not overlap.

    To create a super gauge object, see “Making a Super Gauge Object by Adding Gauge Data”.

  • A model-layer file

    The model-layer file must specify both optical models using the optical_model 0 and 1 arguments. For syntax, see “Model-Layer File Format”.

Procedure

  1. In Calibre WORKbench, open CM1 Center (Litho > CM1 Center).
  2. Load the super gauge object (Gauges > Open Super Gauge).
  3. In the CM1 Center Misc tab, add the model-layer file.

    If you are using modelflow_v2, specify the ‑f option for the setup file or the ‑mlfile or ‑mlstring options for the model-layer file.

  4. Load both optical models using one of the following methods:

    • By linking gauges to pre-built optical models in the Gauges tab (using the Gauges > Link 1st Optical Model and Link 2nd Optical Model menu items).

    • By loading them directly into CM1 Center in the Optics tab (using the Optics > Open 1st Model and Open 2nd Model menu items).

  5. In the Optimize tab, Optics subtab, calibrate (optimize) the models iteratively, using the following sequence:
    1. beamfocus and def_start together.
    2. beamfocus_ex2, together with def_start_ex2 set to the results of def_start, since both models should have a common output plane.
    3. If desired, variables specific to the first exposure. This is not a common practice.
    4. If desired, variables specific to the second exposure. This is also not a common practice.
      Note:

      When calibrating variables specific to one exposure, it is recommended that you use features exposed primarily by that mask, while setting the other exposure mask to be dark.

      If the second exposure variables you want to tune are not one of the beamfocus_ex2, def_start_ex2, or imagediffusion_ex2 variables, they can only be tuned if you swap the first and second exposure models such that the second model is used as the first model, and the first model is used as the second model.

    5. If desired, variables common to both exposures. Note that this is recommended for optical model calibration only when the property is known to be common to both exposures, such as imagediffusion.
  6. Save the calibrated optical models using the Optics > Save 1st Model and Save 2nd Model menu items.
