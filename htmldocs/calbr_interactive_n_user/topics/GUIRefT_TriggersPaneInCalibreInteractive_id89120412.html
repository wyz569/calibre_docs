Triggers Page in Calibre Interactive

To access: Choose Settings > Show Pages > Triggers

The Triggers page defines trigger functions that execute before and after the batch Calibre run. Trigger functions can run in the shell environment or in the design tool environment. Trigger definitions are saved in the runset. You can also specify trigger functions in the custom configuration file.

Description

The transcript prints the following lines when a pre- or post-execution trigger is executed:

INFO: Running ... pre-trigger: <trigger_function> 
INFO: Running ... post-trigger: <trigger function>

These lines are displayed in the warnings and errors section of the Transcript page as type “Info.”

Tabs

Pre-execution triggers run before the Calibre application executes; post-execution triggers run after the Calibre application finishes. Each tab contains the list of trigger functions for that internal trigger.

Note:

Calibre Interactive PEX and xACT have additional pre- and post-trigger functions. See “Internal Trigger Execution in Calibre Interactive”.

Buttons

The buttons take the following actions:

  • Add trigger at end of list

  • Remove selected trigger

  • Remove all triggers

  • Move selected trigger up in list

  • Move selected trigger down in list

Use

Table column

Execute or not execute the trigger function.

Type

Table column

  • Process — Execute in the shell.

  • Layout — Execute in the layout design tool.

  • Schematic — Execute in the schematic design tool.

Trigger

Table column

The trigger function, including arguments. The arguments listed in the Parameter Substitution area can be used.

Parameter Substitution

GUI area

The listed parameters are used to pass various filenames and other values specified in the GUI to the trigger program. For example, %L is replaced with the name of the layout primary cell and passed to the trigger program. See “Trigger Parameters in Calibre Interactive”.
