Correspondence of Tcl Customization Commands to New Configuration Commands

This section provides a table listing Tcl customization commands and the corresponding command for the new configuration file format. Some typical Tcl scripting commands and the corresponding command in the new configuration file are also given.

See “Tool Properties” and “Option Properties” for definitions of the tool and option properties referred to in the following table.

There is not a special command to get a runset value. GUI settings are available with the option property “value”. 

var topCell = tool.layout.topCell.value

CUSTOM::setRunsetVarValue

CUSTOM::setRunsetVarValue
 drcStartRVE [set $wb_startRVE]

There is not a special command to set a runset value. GUI options are set by assigning the “value” option property.

tool.layout.topCell.value = TOP

The controls for existing GUI options can easily be moved to a new page with the tool.option.pages property. The value is set as usual using the GUI.

See “Adaptation Example: Move and Set GUI Options”.

CUSTOM::setOKCallback

Not directly applicable, as there is no OK button.

Use the runCallback tool property to specify a callback function. In GUI mode, this function is executed upon startup when a runset containing a reference to the customization file is loaded and when the Run button is pressed and a DEFINE or VARIABLE has changed. In GUI-batch mode, this function is executed at run time. See “Example: Attach Callback Function to Run Button With tool.runCallback”.

Use the connect() function to attach function calls to a button or to a change signal from a GUI control. See “Adaptation Example: Callback Function”.

Runset values defined with this function, such as the rules file, override runset changes made in the GUI.

Also see addAction and addChoicesAction.

CUSTOM::setOKCallbackError

If there is an error condition, set the tool property blockRun to true—this halts the run when the Run app button is clicked and displays the tool.blockReason string in a popup dialog box.

When using the runCallback tool property, you can use the runCallbackError property.

See the example with getCalibreVersion.
