Using Calibre Interactive Runsets

Calibre Interactive runsets save the settings in the GUI so you can easily return to the same setup. There are preference settings and environment variables that affect loading and saving runsets.
Table 1. Loading, Saving, and Using Calibre Interactive Runsets

Action

Method and Description

Load a runset from the GUI

File > Load Runset

File > Recent Runsets

Tip

You can pre-populate the recent runset list; see “Populating the Load Runset Files Dialog Box and Recent Files Lists”.

Automatic runset loading

You can load a runset from the command line, with an environment variable, or using a startup preference setting. See “Methods to Load a Calibre Interactive Runset”.

Save a runset

File > Save Runset

Only non-default options are saved to the runset. To save all options, set the environment variable MGC_CALIBRE_SAVE_ALL_RUNSET_VALUES to 1.

Set preferences for saving runsets

Choose Settings > Show Pages > Preferences to view the Preferences page and expand the Runset option area. The preferences behave as described.

  • On Close Runset:

    • Prompt to save changes before closing runset

    • Save changed settings before closing runset

    • Do not save changed settings

  • Save runset each time Calibre is run

Specify runset type and information

Choose Settings > Show Pages > Preferences to view the Preferences page, expand the Runset option area, and specify Type and Info.
