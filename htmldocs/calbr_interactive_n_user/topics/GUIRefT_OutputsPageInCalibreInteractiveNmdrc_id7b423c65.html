Outputs Page in Calibre Interactive nmDRC

To access: Click Outputs in the left panel

The Outputs page includes settings for the DRC results database, opening side RDBs, and for creating a CTO file, DRC HTML report, and DRC summary report.

Objects

Object

Description

DRC Results Database

Specifies the name, format, and related options for the DRC results database.

Filenames that end in .gz are compressed using gzip. Filenames that end in .bz2 are compressed using pbzip2. If the compression utility (gzip or pbzip2) is not in your path, you are prompted to supply the path.

Rule file statements: DRC Results Database, DRC Cell Name, and DRC Check Text

Side RDBs

Specify the side RDBs to open automatically when Calibre RVE is invoked after the run completes.

To populate the side RDB table, click the button on the Rules page next to the rule file entry. Select the checkbox in the Open column to have the RDB opened automatically.

The following RDB types are recognized:

  • Check Map — DRC Check Map

  • Density — Density operation with the RDB keyword

  • DFM — DFM RDB

  • LIER — Layout Input Exception RDB

  • NAR — Net Area Ratio

Tip

You can also specify side RDBs to open using Calibre RVE. See “Viewing Auxiliary Databases” in the Calibre RVE User’s Manual.

Create CTO File

(only displayed if the DRC Results Database format is ASCII)

Specifies whether to create a Check Text Override (CTO) file during the run. The CTO file includes RVE Show Layers statements based on the rule checks in the rule file. It can optionally include category information.

See “Creating a Check Text Override (CTO) File With Calibre Interactive nmDRC”.

Create HTML Report

Specifies whether to create a DRC HTML Report of the results.

Template:

  • Cell-Check — Group the results tree by Cell/Check.

  • Check-Cell-SC — Group the results tree by Check/Cell/#SC, where #SC is the Shape Class calculated property.

  • Check-Cell — Group the results tree by Check/Cell.

  • Check-SC — Group the results tree by Check/#SC.

  • Custom Template — Provide a custom DRC HTML Report configuration file. Specify the file in the Custom Template field which appears when this option is selected.

When #SC (Shape Class) is in the group hierarchy, only one result per shape class is shown for each check/cell combination. This is useful in order to reduce the size of the report.

Format:

  • HTML — Standard HTML format.

  • MHTML — Single file HTML format.

Output Directory: Specifies the directory in which to create the report. The directory is created if it does not exist.

View Report: Open the report after it is created.

See “DRC HTML Reporting” in the Calibre RVE User’s Manual.

Reports

Specifies the name and options for the DRC Summary Report.

Rule file statement: DRC Summary Report

Reusable Hierarchical Database

Specifies whether to save or restore an RHDB (Reusable Hierarchical Database) file. An RHDB is useful in some flows to save time by avoiding the overhead of repeatedly constructing a hierarchical database. Specify the RHDB file name and the Mode:

  • Save — Saves a proprietary image of the hierarchical database, generated from all input files in the rule file, as an RHDB file. The RHDB file is saved with an embedded checksum. This option disables CTO file generation, HTML report generation, Area DRC, and automatic RVE invocation.

  • Restore — Restores an RHDB file from the specified file. A checksum is generated from all input files and compared with the checksum in the specified file. A mismatch between the checksums generates an error.
