waiver_util Runtime Messages

The following tables document the error, warning, and note messages from the waiver_util tool.
Table 1. waiver_util Messages

Message

Description

ERROR: Cell suffix file -s <file> does not exist.

The specified file is not present.

ERROR: Could not open layout file <file>.

The -i input file is not accessible.

ERROR: Incorrect GDSII/OASIS format in file <file>.

There is a problem with the layout format.

ERROR: Invalid format while parsing waiver description file at line <N>.

The waiver description file has a syntax problem.

ERROR: Invalid OASIS/GDS character “<character>” found in the suffix “<suffix>” for cell “<cell>” at file <filename>:<line>

There is an invalid character in the -s suffix file. This character cannot be used for cell names.

ERROR: Invalid waiver type defined at file <description file> : <line number>.

The waiver type may only be SINGLE or MULTI.

ERROR: Output file <file> not generated.

The input file may not have any of the specified cells for the run, or no waiver criteria have been updated for the input design. Review the transcript for other WARNING messages.

ERROR, Result overlap entry must be an integer between 0 and 100 inclusive at file <name>:<line>.

The waiver criteria file uses an incorrect number for the result overlap parameter.

ERROR: Specify proper layer number with -l switch.

The -l option is improperly specified. It must have a valid layer number as an argument.

ERROR: Waiver description file -d <file> does not exist.

The specified file is not present.

ERROR: Waiver overlap entry must be an integer between 0 and 100 inclusive at file <name>:<line>.

The waiver criteria file uses an incorrect number for the waiver overlap parameter.

WARNING: Ignoring cell <cell> since input layer <layer> is not a waiver layer for this cell.

No waiver shape is present on the specified layer for this cell.

WARNING: No conflicting criteria found in the input file “<file>” at the waiver layer. No criteria have been updated.

These are the possible causes:

1. The criteria file waiver criteria match what is found in the input design.

2. The -l layer is not a waiver layer for any cell.

WARNING: No waiver cell in the input file “<file>” matched with the entries in the waiver description file(s) “<description file>”. No criteria have been updated.

These are the possible causes:

1. No waiver cell found in the input design.

2. No cell mentioned in the description file is found in the input design.

WARNING: Skipping dummy cell <cell>.

Empty cells with embedded waiver criteria that are generated by waiver_flow or Calibre RVE are not updated.

WARNING: Waiver description criteria found in the cell <cell> at layer <layer>. Using waiver description file(s) criteria instead.

Waiver criteria text objects exist in a waiver cell. The criteria from the waiver description file are written instead.

WARNING: -x switch has been deprecated. HEX checksum generation is default now.

The -x is no longer used because checksums are only formatted in hexadecimal.

NOTE messages:

NOTE: Cell suffix file is <file>.

The -s option is used and the file is read.

NOTE: Merging layout file

<input_layout_file> with waiver

description file <desc_file>.

Waiver criteria in the input design are being merged with information in the waiver description file.

NOTE: Output file <file> generated.

The -o file has been generated.

NOTE: Updated waiver criteria for cell <cell> at layer <layer>.

Waiver criteria from the waiver criteria file have been added to the specified cell.

NOTE: Ensure texttypes do not collide among all the text objects.

The texttypes of text objects should be different from each other.
