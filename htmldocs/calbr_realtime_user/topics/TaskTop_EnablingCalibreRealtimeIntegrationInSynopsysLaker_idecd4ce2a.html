Enabling the Calibre RealTime Integration in Synopsys Laker

You must add some code to your $HOME/.lakerenv file in order to enable Calibre RealTime. You must also set an environment variable to specify the location of the Calibre software tree.

Prerequisites

  • See “Calibre RealTime Custom Requirements” for system and licensing requirements.

  • You are using a supported version of Synopsys Laker3; see “Supported Versions of Synopsys Laker3 for Calibre RealTime”.

Procedure

  1. Set the environment variable MGC_HOME or CALIBRE_HOME to the location of the Calibre software tree. See “Setting the CALIBRE_HOME Environment Variable” in the Calibre Administrator’s Guide for details.
  2. Add the following code to your $HOME/.lakerenv file
    if { [info exists ::env(CALIBRE_HOME)] && \ 
	[file exists $::env(CALIBRE_HOME)] } { 
  		set ::env(MGC_HOME) $::env(CALIBRE_HOME) 
} 
# set directory for install script 
set tdir shared/pkgs/icv/tools/realtime/laker 
 
if { [info exists ::env(MGC_HOME)] } { 
if { [file exists $::env(MGC_HOME)/$tdir/laker_realtime_install.tcl] && \ 
       [info commands isLaker] != ""     && \ 
       [isLaker] } { 
    source $::env(MGC_HOME)/$tdir/laker_realtime_install.tcl 
  } 
}
    Tip

    You can also view a README file at $MGC_HOME⁄shared⁄pkgs⁄icv⁄tools⁄realtime⁄laker⁄lakerREADME
