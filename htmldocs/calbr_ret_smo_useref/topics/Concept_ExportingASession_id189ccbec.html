Exporting a Session

To save space, session directories in the database contain references to shared information such as the layout and gauges. To move the session to another location it must first be exported in order to include the referenced information.

There are different mechanisms to use if you are exporting the session to create a test case than if it is being moved to another database.

  • Use Export Session if you want to include the results. The session can be imported into other databases, allowing different sessions to be compared.

  • Use Create Testcase from Session if size is a concern or to create a fully relocatable set of data.

  • Use Create Archive for the most complete saved data. The archive has both relocatable data that can be run outside of RET Selection and also all generated files and results. It is compressed by default.

Testcases cannot be imported into other databases. If size is a concern and you want to be able to import, use Export Session and select the “Create archive” option. This does not save the same set of data as an archive, but does compress the exported directory into a single file.

Exporting and Importing a Session

Export a session by either right-clicking the session and selecting Export Session or from the main menu File > Export > Session. The Export Session dialog box allows you to export just the gauges, setup, and results, or the gauges, setup, results, and layout, and to compress the directory into a .tar.gz file.

You can later import the session to another database with File > Import > Session or File > Import > Layout for sessions exported with a layout. The wizard’s directory navigator indicates appropriate directories with special icons: a pale aqua folder with an L for layout directories, and a gold folder or rectangle with an S for session exports. Layout directories contain session directories. Archives (a rectangle with a red A), although they contain session data, can only be imported using File > Import > Archive.

Note:

When you import a session into another layout/gauge pair within the same database, if you specify a target session ID in the wizard, it moves — not copies — the session. The session no longer appears in the original layout/gauge pair. If you do not specify a target session ID, the wizard creates a new session in the destination layout/gauge pair.

Creating a Runnable Test Case

You can create a test case using either the session right-click menu Create Testcase from Session or from the main menu File > Export > Create Testcase.

The RET Selection GUI does not allow you to create a directory in sensitive locations such as the database. If Calibre WORKbench opens a dialog box with “No error message returned by Calibre Workbench,” check the Log Viewer window for an mdf smo error message. Most likely the intended parent directory for the test case does not allow writing.

The test case directory contains the following files and directories:

runscript.tcl

A Tcl script containing the mdf smo commands to execute the session.

dofile

A shell script that when executed cleans up any files from the previous run and then calls Calibre pxSMO with runscript.tcl. Results are written in the test case directory.

cleanup

A shell script that removes the output files created by dofile.

pxopc.setup, pxsmo.setup.<number>, smoo.setup, verify.setup, setup.in

The setup files from the original session. Any relative paths have been modified to point to a local version.

gauges, layout, models

Directories containing the gauge file, the layout, and the litho model with its collection of models, respectively.

To run a test case, execute dofile.

These files are also available in archives, but you must uncompress them first.
