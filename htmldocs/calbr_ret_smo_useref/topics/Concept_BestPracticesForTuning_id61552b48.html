Recommendations for Tuning Results

There are several things that can improve your optimized source or mask, but are too case-specific for inclusion in the procedures.

The following list is organized from most commonly applicable to highly-specific situations.

  • Tuning the smoothed curve to more closely match the achievable nominal contour usually improves results. The defaults are rarely adequate. Check the curve on a variety of feature types. A good initial approximation can often be had with the following:

    • Use curve_target instead of curve.

    • Set lineEndRatio between 1.2 and 1.5.

    • Set criticalDistance to approximately the minimum dimension. The default for curve_target is based on the opticalsystem setting in the nominal optical model.

  • Investigate how well the process window conditions cover PV band maxima and minima using Bossung plots and contours of verification conditions. Increasing the number of conditions can sometimes improve common DOF compared to the standard three process window conditions.

  • When tuning process window conditions, focus is typically the most sensitive parameter.

  • Ready-type DDM libraries are not adequate for SMO because they have already determined which sectors are bright. During SMO, any pixel in the illuminator can turn on. There needs to be sufficient DDM signals across the pupil to support accurate imaging calculations. Universal-type DDM libraries use a high-frequency sampling that makes this possible.

    A ready-type DDM library can be converted into a universal-type DDM library by adding the line “modeltype universal” and replacing illumination lines with “illumination universal na obj_na”to the masksim input file and re-running Calibre MaskSim.

  • If the NILS values are not good, check the litho model’s aerial image threshold. In the litho model, this is the image_threshold parameter. For runs using CTR modelforms or only aerial image, use the Calibre WORKbench RET Flow Tool image map.

  • For MEEF, check that the mask delta value is optimal for the MEEF calculation. By default, the mask delta is equal to the mask grid. Use the arrows for MEEF Mask Delta to increment the value. A small mask delta value from 0.25 to 1 nm is typically recommended. The Setup window shows the mask grid on the main Verify Setup pane and the mask delta when you select Verify > MEEF.   • For compound optical sources, simulations use the linear polarization of the first element. If this is not what you want, modify the optical file to put the desired polarization first. Frequently this is SPLIT_XY.
