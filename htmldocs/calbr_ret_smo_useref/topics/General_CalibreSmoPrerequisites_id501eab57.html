Calibre SMO Prerequisites

Before starting Calibre pxSMO and the RET Selection tool, you need to have certain required files and licenses.

The following items are required in order to run Calibre pxSMO:

  • Calibre pxSMO and Calibre WORKbench Licenses — For more information on licensing, refer to the Calibre Administrator’s Guide. You do not need separate licenses for Calibre pxOPC, Calibre nmOPC, or Calibre OPCverify, even though they are called by the RET Selection runs.

  • Layout — A design file containing clips (portions of layout) that are challenging to manufacture using existing processes. The clips should be separated by approximately the clip width.

  • Gauge File — A file in the .gg or .sgd format containing gauge objects. The gauges must be specific to the layout file as they contain coordinates. Refer to “Gauge Object Reference” in the Calibre WORKbench User’s and Reference Manual for different techniques for creating a gauge file.

  • Models — A litho model containing the typical models used for OPC. These include an optical model that will be used as the starting point for optimization and a CM1 resist model. The litho model may also contain other models such as DDM.

    The litho model only needs to have the nominal defocus condition. Other focus conditions are built as needed from the process window conditions.

  • Process Window Conditions — Details on which process window conditions are relevant. As the best solution is dependent on which conditions are common and what measures are most important, you must have decided what you are trying to optimize before beginning.

The following items are also recommended, but not required:

  • Configuration File for Distributed Processing — Calibre WORKbench performs distributed processing using the ‑remotefile switch. See “Distributed Processing in Calibre WORKbench” in the Calibre WORKbench User’s and Reference Manual for details.

  • Memory — If you are using distributed processing, the hardware should have at least 20 GB for the primary CPU, and at least 4 GB for each remote. If the SMO setup includes more than three process window conditions, more memory will improve performance.
