parallelize_by_pw_conditions

PxSMO Setup File Commands

Recommended. Enables distributed processing.

Usage

parallelize_by_pw_conditions {yes | no}

Arguments

yes | no

A required argument indicating whether parallelization by process window conditions is enabled (yes). The default is to parallelize only by clips (no).

Description

An optional setting that enables additional parallelization when run in Calibre MTflex mode.

By default, parallelization is done by clips only, with a remote host receiving the clip and all enabled process window conditions. (A clip is a portion of layout plus a gauge.) When this command is set to “yes”, the remote host receives a clip and single process window condition. To get the maximum benefit from this option, the remote setup file or script should request as many remote CPUs as (number of process window conditions) x (number of gauges).

The results may be slightly different between “yes” and “no” runs. This is expected because Calibre pxSMO optimization is sensitive to the order of calculations. The final sources, however, are not significantly different.

This command is added to pxsmo.in when the Parallel PW option in the SMO Setup Advanced tab is selected.

Examples

parallelize_by_pw_conditions yes
