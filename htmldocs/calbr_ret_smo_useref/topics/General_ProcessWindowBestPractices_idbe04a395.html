Process Window Best Practices

These process window recommendations have been developed from extensive testing. Settings are relative to those used in the Calibre OPCverify setup file unless otherwise noted.

Note:

More information is available on Support Center in KB article MG598348 “pxSMO Best Practice: PW Condition Selection.”

Siemens EDA strongly recommends the following practices:

  • Use more aggressive definitions for SMO and OPC stages than for the Verify stage.

    The standard recommendation is to make the process window conditions about 1- 10% worse (relative to nominal) for SMO and OPC. This is illustrated in Figure 1 and tabulated in Table 1 and Table 2.

  • Use at least five process window conditions: nominal, two defocus (for example, ±50 nm), and two off-nominal dose (for example, ±4 percent). If the initial process window conditions do not produce a sufficient common process window value, then the conditions can be adjusted. Adjustments depend on the requirements for the layer and the solutions from the initial SMO runs.

  • Investigate how well the SMO process window conditions cover PV band maxima and minima using Bossung plots and contours of verification conditions. The chosen process window conditions must represent the maxima and minima of PV band variation across all feature types. The SMO and OPC stage process window conditions should be slightly larger than the settings used by the Verify stage. Reasonable starting values are 10% defocus and 1-2% dose.

  • In the beginning, use zero for the mask size setting. The weights should be set to one in the PW table and can be adjusted in each point tool’s setup pane.

  • Values for focus-dose pairs within the dose and defocus ranges. The number of pairs must meet the polynomial convergence criteria.
