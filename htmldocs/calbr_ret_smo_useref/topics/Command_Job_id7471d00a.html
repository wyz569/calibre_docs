job

PxSMO Setup File Commands

Calls the various phases and allows local overrides of settings.

Usage

job number type

Arguments

number

A required integer identifying the job. Jobs are numbered sequentially starting at 1.

type

A required argument specifying the job type. The following values are supported:

  • start — Optimizes the source based on the slope of the image intensity, taking into consideration extra printing and EPE.

  • decorate — Optimizes the mask by adding SRAFs. The placement and sizes of the SRAFs are set to deliver maximum process latitude for the given process corners. This job is optional if your main focus is on source optimization.

  • improve — Optimizes the edge placement, as measured by reducing the range of values for the edge placement error across the selected process windows. SRAF printing is eliminated.

  • polish — Performs fine-grained optimization for the best match between simulated contours and target and optimizes for the slope of the aerial image or CM1 surface. Any extra printing is eliminated.

  • cdof — Optimizes source and mask for maximal depth of focus at a specified exposure latitude.

  • el — Optimizes source and mask for maximal exposure latitude at a specified depth of focus.

In general, your run should begin with start and end with polish. The other job types can be used at your discretion. If all job types are used, list them in the order in the list above.

For a run performing source-only optimization, use the following job types:

  • start_so

  • improve_so

  • polish_so

  • cdof_so

  • el_so

Each source-only job type performs similar functions as the full SMO version without changing mask features. If all source-only job types are used, list them in the order in the list as previously described.

For a run performing mask-only optimization, use the following job types:

  • start_mo

  • decorate_mo

  • improve_mo

  • polish_mo

  • cdof_mo

  • el_mo

Each mask-only job type performs similar functions as the full SMO version without changing the source. If all mask-only job types are used, list them in the order in the list as previously described.

Description

The Calibre pxSMO setup file begins with general options such as pupil_fill and then lists specific jobs. The job types are specific combinations of solvers and preset objectives that work well in sequence. You can tune the job results by specifying local overrides using other setup file commands, such as iteration.

Example 1

In this simple recipe generated by the RET Selection interface, each job uses a different combination of process window conditions. (The definitions of the process window conditions is not shown.)

job 3 improve
