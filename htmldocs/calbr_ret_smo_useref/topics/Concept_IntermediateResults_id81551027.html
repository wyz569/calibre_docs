Intermediate Results

Even fast runs take several hours and you may sometimes wonder if calculations are progressing. The RET Selection GUI offers several ways to check on sessions.

The longest stage is typically SMO. There are three indicators in the Result tab of interest: the current source view, the session log, and the objective plot.

The current source view and the objective plot do not display any information until after all pixels have been processed. The session’s execution status (shown in the Job Status tab) should have reached at least the SMO Run stage before checking them. The session log, on the other hand, records progress as soon as the job is submitted.

Current Source View

The Result tab shows the current source map before SMO is complete. Unlike the final view in the Result tab, the intermediate view provides information on the current iteration as well as a view of the source. (If Source Map Capture in the Advanced tab of the SMO setup is cleared, the Result tab shows a gray box instead of the source map.)

Logs

The Logs button menu lets you view the job log, transcript, and session log, which track the current information from the different processes executing the run. The job log is a transcript captured from each job execution. The run transcript captures standard output and standard error messages during runscript execution. The session log captures the entire transcript related to the session when it is active - the models being used, setup files, gauge creation, and so on. It includes the interactions in the command window and the dispatcher job transcripts. The session history log is an archive of session logs. It updates when a session is deactivated, so it does not show the progress of a run.

PxSMO Objective Plot

The objective plot, accessed through the PxSMO Plot icon, demonstrates how each job is converging on the objective. For runs with debug_level 1 set, a plot similar to the following figure is drawn. Generally a steep curve indicates significant movement and a flatter curve that optimization is approaching a final solution. It gives an idea of relative progress.

The following figure was captured after the run completed. The Objective Plot window includes the transcript, which you can search using the text entry field and red arrow buttons. If you set debug_level 0, the transcript may not include the same level of detail. Unlike the session log, it records only SMO activity.

The icon next to Plot sub_iter opens the Source Map Movie player. It runs through a sequence of snapshots of intermediate optical sources. Only an image of the source is saved; not the entire source map. If intermediate sources were turned off, the icon is dimmed.
