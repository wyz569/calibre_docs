mdf smo commands

Commands that begin “mdf smo” control the runs (for example, pre-SMO or Verify).

archive

Imports or exports a self-contained, runnable archive. Archives contain a session and its associated layout.

  • export session_id archive_file — Creates archive_file, which contains the layout, session, setup files, results, and a Tcl script to run the archive as a testcase.

  • import [‑layout_only] archive_file layout_directory — Imports the contents of archive_file into the RET Selection database as a new layout.

clips

Generates clips. There are several subcommands:

  • list — Also the default, if you see “mdf smo clips” with no other arguments. Lists information about the current clips.

  • draw — Draws clips in Calibre WORKbench. For example, the following command draws clip 3:

    mdf smo clips draw 3

  • gen or generate — Generates clips for SMO and OPC steps. It also generates the masking layer for the Verify step.

  • load — Loads previously generated clips.

  • select — Selects optically significant clips from a set based on threshold.

  • upd or update — Updates the gauge weight and anchor gauge for clips.

configopt

Configures the in-session optical models (that is, pre_so_model and post_so_model) before loading the model.

configopt [‑imageplane_mode {auto | only}] [‑planes {imageplane}...] [‑no_extra_planes] [‑specs {spec_name}...] [‑spec_config spec_config_file] [‑no_extra_specs]

  • ‑imageplane_mode auto | only — Specifies the mode of the imageplanes command. If ‑imageplane_mode is not specified, then the ‑planes and ‑no_extra_planes arguments are ignored.

    • auto — Instructs the tool to select image planes.

    • only — Turns interpolation off.

  • ‑planes imageplane — Specifies a list of additional image plane values in microns that are stored in the target optical models.

  • ‑no_extra_planes — Removes image plane values not specified in the ‑planes argument from the target optical models.

  • ‑specs spec_name — Specifies an active spec list. The list of spec names is used to select specs from the spec_config_file. The selected specs are either added to the target optical model or replace existing specs in the target optical model. If ‑specs is not specified, then all of the specs in spec_config_file are used in the target optical model.

  • ‑spec_config spec_config_file — An optional argument specifying a spec configuration file containing information in the following format:

    spec spec_name
{
optical_model_text_line
 ...
}
...

    Use the command in presmo, preopc, and preverify command blocks to configure the optical model for the corresponding stage (SMO, OPC, or Verify).

    A spec file can define one or more optical model specifications (specs); each spec block specifies one or more optical model text lines. For more information on specific optical model commands, refer to “Optical File Parameters” in the Calibre WORKbench User’s and Reference Manual.

    The following spec file example specifies a spec for a Z5 Zernike coefficient (vertical astigmatism) and another spec with multiple Zernike coefficients:

    spec Z5
{
zernike 0 0 0 0 0.02
zernike_multiplier_flag 0
zernike_rms_flag 1
}
spec multi_z
{
zernike 0 0 0 0 0.02 0 0 0.64 0 0.05
zernike_multiplier_flag 0
zernike_rms_flag 1
}

  • ‑no_extra_specs — Removes specs not specified in the ‑specs argument from the target optical models.

copyclip

copyclip source_gid copied_gid { anchor | anchor_noopc }

Copies the specified gauge source_gid to the specified gauge id copied_gid in the retsel_cliparray.oas layout file and retsel_cliparray.gg gauge file.

This command is issued in the presmo block before “mdf litho inter_copy” and “mdf litho clip_bias” to create an anchor gauge clip with CD bias on a separate layer.

You can use this command to copy the specified gauge from the target layer to the correction layer in the preSMO step, which increases the number of gauges. The keyword anchor indicates that the gauge is an anchor gauge; the keyword anchor_noopc causes the tool to skip OPC on that anchor gauge.

end

A required command when running in batch mode. It ends the RET Selection operations and removes the objects from memory.

flarelongrange

Configures flare_longrange statements in the pre_so_model and post_so_model setup.in files.

flarelongrange {default_flare | none | flare_value‘:’name}

  • default_flare — Adds a “flare_longrange lithomodel constant default_value” statement. lithomodel specifies the name of the lithomodel. default_value specifies a flare value between 0 and 1.

  • none — Removes all “flare_longrange lithomodel constant default_value” statements.

  • flare_value‘:’name — Adds “flare_longrange lithomodel constant flare_value name name” statement. The name argument specifies the name of a constant flare value in the lithomodel file.

genggl

Creates SMO-style gauges and writes the layout. Each gauge is on a separate layer. For example, the following command writes the gauges to a layout file named geng.oas:

mdf smo genggl geng.oas

layout

Operates on the layout/gauge pair. There are several subcommands:

  • No subcommand — That is, just “mdf smo layout”. Shows information about the current layout/gauge pair and session.

  • A number — For example, “mdf smo layout 1”. Sets the layout/gauge pair associated with that number to the active layout/gauge. This also has the subcommands loadspec, savespec, and delspec.

  • new — Creates a new layout/gauge pair in the current directory. You must specify the layout and gauge names, for example:

    mdf smo layout new {mdf gauges gauges.gg layout.oas 6 6}

  • clone — Clones a layout/gauge pair. The full syntax is “mdf smo layout clone id new_dir”, where id is the layout/gauge pair number and new_dir is the new directory to keep files associated with that layout/gauge pair.

  • name — Sets the name of a layout/gauge pair. The full syntax is “mdf smo layout id name {pair_name}” where id is the layout/gauge pair number and pair_name is a name or description for the layout/gauge pair.

list

Lists all existing layout/gauge pairs and their sessions.

mask

Performs operations on the mask. For example, the following command imports an external mask:

mdf smo mask import {maskfile}

This command runs Calibre pxOPC to generate optimized mask clips. 

mdf smo mask {
	mdf litho add loadclips \
	-inputs {./tmpworkdir/smo/retsel_smoothtarget.oas \
		./tmpworkdir/smo/retsel_squaretarget.oas \
		./tmpworkdir/smo/retsel_tolband_inner.oas \
		./tmpworkdir/smo/retsel_tolband_outer.oas} \
	-outputs {tgtsmth squaretgt tol_inner tol_outer} \
	-command multiload; \
	mdf litho add pxopc \
		-setup pxopc.setup
		-inputs {squaretgt tgtsmth tol_inner tol_outer} \
		-outputs {retsel_mask} \		
		-command pxopc; \
	mdf litho check off; \
	mdf litho run -distributed; \
	}

maxclipsize

Controls the maximum layout size used to generate a clip layout from a gauge.

mdf smo maxclipsize size_val

Limits the clip layouts created in presmo to be no larger than size_val in microns. The default size_val is 5.0 microns.

parametric

Operates on Parametric Explorer sessions to perform mask-only optimization. There are several subcommands:

  • new — Sets up a parametric source exploration run. The following example sets up the run using settings from a file named doeParam.setup:

    mdf smo parametric new doeParam.setup

  • run — Starts a parametric source exploration run. The following example runs using settings from a file named doescript.tcl and specified process window analysis settings:

    mdf smo parametric run 1 doescript.tcl max_parallel 2 pwoptions \
	{‑cdtol 10 ‑doselat 5 ‑type elliptical}

polarization

Sets the source polarization.

polarization { split_xy | te | tm | linearx | lineary | none } [weight]

  • split_xy | te | tm | linearx | lineary | none

    • split_xy — Splits any illumination source along four quadrant lines (y=x and y=-x). Anything in the east or west quadrants (abs(x) > abs(y)) is treated as lineary; anything in the north or south quadrants (abs(y) > abs(x)) is treated as linearx.

    • te — Transverse electric or S (S-polarized).

    • tm — Transverse magnetic or P (P-polarized).

    • linearx — X-axis polarization.

    • lineary — Y-axis polarization.

    • none — Unpolarized.

    For detailed information on individual polarization types, refer to “Source Polarization” in the Calibre WORKbench User’s and Reference Manual.

  • weight — An optional parameter specifying a weight value for the specified polarization type. The default value is 1.

postsmo

A required command when running in batch mode. This command should be used to run any remaining commands after the Verify stage.

mdf smo postsmo {}

preopc

This command should be used to run any mdf commands needed before “mdf smo mask”, such as selecting process window conditions.

presmo

A required command when running in batch mode. This command should be used to run any mdf commands needed before “mdf smo setup”, such as selecting process window conditions. The following command runs the target smoothing and saves it in targetSmooth.oas.

mdf smo presmo {mdf litho add smoo -setup smoo.setup -inputs {6} \
	-outputs {tgtsmth};mdf litho run;mdf litho savelayout \
	./tmpworkdir/smo/targetSmooth.oas {tgtsmth:7 *};mdf gauges layout \
	./tmpworkdir/smo/targetSmooth.oas 7 6 ;}

preverify

This command should be used to run any mdf commands needed before “mdf smo verify”, such as selecting process window conditions.

pwa

Creates or deletes process window objects.

  • mdf smo pwa new sgd [‑gid_from_struct] [[‑disable] ‘{’gauge_list‘}’]

    Creates a new process window object from the data in sgd, a super gauge data file. You can select which gauges to use with the gauge_list. The new process window object has default settings of 13-term polynomial with third order dose and fourth order focus; target CD based on drawn CD values; 5% dose latitude; and the inscribed ellipse should center on nominal dose of 1.0.

    If ‑gid_from_struct is specified, gauge IDs are based on the structure column of the file; for example, G015 becomes gauge ID 15.

  • mdf smo pwa delete $PWO

    Deletes the process window object, $PWO. There are no optional keywords.

For additional operations that can be performed on a process window object, see “Process Window Object Commands”.

pwanalyze

Computes the common depth of focus (DOF) for a super gauge data file. By default, it uses a CD tolerance of 10% and dose latitude of 5% on drawn CDs on the nominal process window. These can be overridden by additional arguments.

The optional subcommand set -load pwa_file loads process window analysis settings from an XML file.

pwoutput

Generates plots of exposure latitude (EL) versus DOF. This can be useful for determining the process window.

session

Operates on sessions. There are several subcommands:

  • A number — Activates the specified session id number. The following example changes the active session to session 2:

    mdf smo session 2

  • new — Creates a new session and adds it to the database with the next sequential session id number. The new session becomes the current active session. The following example creates a new session from the mdf_setup.in setup file:

    mdf smo session new {mdf_setup.in}

  • name — Sets the session name. Full syntax is “mdf smo session id name {session_name}” where id is the session id number and session_name is a name or description for the session.

  • files — Loads files into a session. The following example loads a gauge file and setup files into the active session:

    mdf smo session files {session_1_test_pattern.gg; \
	smoo.setup;pxsmo.setup;pxopc.setup;verify.setup}

  • pxsmo — Enables pxSMO. An “mdf smo session pxsmo yes” statement is required before “mdf smo source” in a Free-Form pxSMO session; the statement is not used in a Parametric Explorer session.

  • close — Makes the session inactive.

  • state — Reports properties of the active session. The property smoState can have the following values:

    smoState Number

    smoState Description

    Associated mdf Command

    0

    (blank)

    mdf smo start

    1

    MDFLITHOMODEL (1)

    mdf smo new {setup.in}

    2

    PRESMO (2)

    mdf smo presmo {}

    3

    Setup Done (3)

    mdf smo setup

    4

    Clips Done (4)

    mdf smo clips gen

    5

    SOCS Done (5)

    mdf smo source

    6

    SO Done (6)

    mdf smo source

    (covers both smoState 5 and 6 and does not stop at smoState 5)

    7

    MO Done (7)

    mdf smo mask {}

    8

    Verify Done (8)

    mdf smo verify {}

    9

    Complete (9)

    mdf smo postsmo {}

In the GUI, the smoState descriptions appear in the upper-right corner of the Session Browser.

setclips

Copies a custom clip file to the session directory and database. The custom clip file information is later used by the pre-SMO step and gencliparray command. For example,

mdf smo setclips customClips.dat

setgauges

Copies a custom gauge file to the session directory and database. The custom gauge file information is later used by the pre-SMO step and gencliparray command. For example,

mdf smo setgauges customGauges.gg

setup

Loads a setup file. For example, 

mdf smo setup smo_setup_file.in

source

Without any other options, runs Calibre pxSMO to generate an optimized source.

Some options work on components unique to Calibre pxSMO, such as this command to import an existing optical source:

mdf smo source import /user/opt_engr1/session_2/output/source_sopxopc.src

To execute pxSMO in the mask (that is, OPC) command block, add the command with the maskgen argument using the following syntax:

mdf smo source maskgen ‑opc_setup pxsmo_setup_file [‑opc_clip clip_file] [‑opc_control {[set_focus_range:0|1] [use_raw_src:0|1] } ]

For example, the following executes pxSMO in the OPC block using settings specified in pxsmo_mask.setup:

mdf smo mask {mdf smo source maskgen -opc_setup pxsmo_mask.setup}

  • ‑opc_setup pxsmo_setup_file — A required argument specifying a pxSMO setup file.

  • ‑opc_clip clip_file — An optional argument specifying a clip file to use for custom gauge weights.

  • ‑opc_control [set_focus_range:0|1] [use_raw_src:0|1] — An optional argument specifying inputs inherited by second-stage pxSMO.

    • set_focus_range — Replaces the second-stage pw_condition_group focus_range value with the output value from the first-pass pxSMO stage.

      • When set to 1, second-stage SMO uses the output first-pass focus_range value.

      • When set to 0, second-stage SMO uses the initial focus_range value from the first-pass stage.

      For example, if the initial first-pass focus_range value is 100 nm and the output focus_range value is 75 nm, setting set_focus_range to 1 uses 75 nm as the initial second-stage focus_range value whereas setting_focus_range to 0 uses 100 nm as the initial second-stage value. The default value is 1.

    • use_raw_src — Specifies the type of source file to use for the initial source in second-stage pxSMO.

      • When set to 1, second-stage pxSMO uses the raw pxSMO output source file from the first-pass pxSMO stage as the initial source for optimization.

      • When set to 0, second-stage pxSMO uses the standard Calibre output source file as the initial source.

      Using the standard Calibre output source file can introduce some changes from the raw pxSMO source file. The default value is 1.

start

This command is required in batch runs before any other MDF commands are issued. It initializes the process.

tgtextent

Sets the extents of the clips for the OPC and OPCverify stages. The following example sets the clip extents to the product of the optical diameter of the model (that is, the hoodpix value) and the multiplier 1.1:

mdf smo tgtextent 1.1

verify

Runs Calibre OPCverify with any specified commands. Output is stored in the ./output directory by default. For example, the following command runs the Verify stage on a clip named “squaretgt”:

mdf smo verify {mdf litho add loadclips1 \
	-inputs {./tmpworkdir/smo/retsel_squaretgt.oas} \
	-outputs {squaretgt} \
	-command load
