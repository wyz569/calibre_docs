Key Concepts

You need to be familiar with some organizing principles in order to use Calibre pxSMO and the RET Selection interface efficiently.

Object Hierarchy

To enable reuse of setups between experiments, all items (for example, layouts, gauge files, and optimized sources) are stored in a database. While the individual objects are the same as used in the rest of Calibre WORKbench, the database organizes them hierarchically.

  • Layout/Gauge Pair — This is the top level. It is formed by the layout and gauge specified when you start. When you change the layout or gauge file, it starts a new set of data. Note that you can change which gauges are used without starting a new layout/gauge pair.

  • Sessions — A layout/gauge pair contains multiple sessions. A session is a specific set of models, active gauges, and process window conditions that constitute a single experiment. A single session typically includes Calibre pxSMO, Calibre pxOPC, and Calibre OPCverify. The sessions within a layout/gauge pair may differ in tool settings, active gauges, or even models.

  • Runs — Within a session, each point tool constitutes a run. This is the same as in batch runs of the Calibre OPC tools.

Active Session

In the RET Selection window you can have multiple sessions and even layout/gauge pairs, but only one layout/gauge pair and session are active at a time. This may not be the same as the one being displayed.

The active session is the session that receives any changes you make to gauges, process window conditions, setup files, and so on. If you are editing these, the title of the various areas will indicate the session that is being edited. If you use the command window, however, there is no indicator.

Another detail to keep in mind is that if you make changes to a session after it has started executing, those changes are picked up by the tool. For example, if Run Control shows the active session is executing SMO and you change the process window conditions, the OPC and Verify runs will not use the same process window condition settings as SMO.

Optimal Results

Because source-mask optimization is a tool used to develop manufacturing processes, there is no single definition of “best results”, or even “good enough results.” There are more trade-offs than just accuracy versus run time. You might choose to emphasize process window or mask complexity, or the print quality of dense arrays more than the uniformity of via clusters. Because of this, it is important that you decide what factors are most important to optimize for.

The SMO solution can be set up to report many metrics, and provides tools to compare across experiments. Before you begin analysis, you must decide what are the limits of “acceptable” and also the relative importance of different factors.

In general, Calibre pxSMO works to minimize process variation (PV bands). It attempts to minimize the edge placement error (EPE) of all target shapes at all process window conditions.

Session Mode

For a session, a mode can be selected that enables full source-mask co-optimization or optimization of the mask for a batch of source shapes. After a session has been run, its mode can be changed.

  • Free-Form pxSMO — This session mode performs a co-optimization of both source and mask. Free-Form pxSMO outputs the co-optimized source shape and mask with results for evaluation criteria. When Free-Form pxSMO is selected as the session mode, pxSMO is displayed in the Setup items tree.

  • Parametric Explorer — This session mode performs a mask-only optimization for a batch of selected source shapes, outputs the optimized masks, and reports results for each tested source. When Parametric Explorer is selected as the session mode, Parametric Explorer is displayed in the Setup items tree.
