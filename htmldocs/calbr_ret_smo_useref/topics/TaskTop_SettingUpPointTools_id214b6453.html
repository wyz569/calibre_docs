Setting Up the Point Tools

The RET Selection GUI shows you which parts of your session have valid configurations. You can set up a basic run with just a few clicks, or customize nearly every aspect. The Session Browser pane is located to the right of the Run Control/Layout Session View pane in the default layout when you have a session selected.

Prerequisites

  • A layout/gauge pair and session.

  • A populated PW Table. See “Defining Process Window Conditions”.

Video

The video series “How to Use CDOF and EL/ILS Optimization” describes and demonstrates the main functionality and key workflow steps for using CDOF or EL optimization in a Free-Form pxSMO session.

  • The “Demonstration: Setting Up CDOF Optimization” video demonstrates how to set up the CDOF job in the pxSMO step.



  • The “Demonstration: Setting Up EL Optimization” video demonstrates how to set up the EL job in the pxSMO step.



Procedure

  1. In the Run Control/Layout Session View pane of the RET Selection window, select a session. If you see the Gauge Analysis pane, select the Session Browser tab at the bottom of the pane.
    Note:

    The session mode default is Free-Form pxSMO. To perform setup for Parametric Explorer, change the session mode to Parametric Explorer and refer to “Setting Up the Point Tools for Parametric Explorer”.

    Figure 1. Initial Setup Pane

    A green check means that the item is able to be run with no changes. (You may still want to check its settings, however.) A circled red X means the item requires additional information.

  2. Adjust the smoothed target. You can create several preview layers to compare, and use the best one.
    1. Click Pre-SMO in the Setup Items tree to open the Pre-SMO Setup pane.
    2. Adjust values. The goal is to match the achievable nominal contour.

      The default values for curve_target are critical distance of (λ/NA)/3 based on the nominal optical model, a line end ratio of 1.0, and maxJog of 0.010 microns. It is recommended to tune these values to achieve the expected curvature of the layer.

      • You can smooth the target with “setlayer curve” instead by changing the Curve Command setting.

      • You can tune the results by entering any keyword-value pairs for the command in the Curve Options field. The options for setlayer curve and setlayer curve_target are described in the Calibre nmOPC User’s and Reference Manual.

    3. If you are optimizing CDOF or EL/ILS, click the Tolerance Bands tab to make adjustments.

      To adjust the CDOF tolerance band settings, make changes in the Tolband-1 (cDOF) sub-tab. If you are optimizing EL/ILS, you can make adjustments to the EL/ILS tolerance band settings in the Tolband-2 (EL/ILS) sub-tab.

    4. Check the results by clicking Curve Target Preview.

      In the dialog box, specify a unique layer number so that you can compare the various experiments. Use the layer name to record the values you used. To inspect the gauges and tolerance bands for individual gauges, click Generate Preview Layout. If you receive any errors, adjust the keywords in the Curve Options field.



      Inspect the tolerance bands and markers for each gauge. If the tolerance bands appear unusual across a grating or array, reduce the maximum pitch value for the axis capturing a non-uniformity in Copy Tolband Marker Options.

      If the tolerance bands for an individual gauge appear unusual, the pre‑curve_target shapes can be inspected. To view the pre‑curve_target shapes, re-run the curve target preview steps with Show All Layers activated in the Curve Target Preview window. Inspect the inner and outer targets for the gauges with unusual tolerance bands.

      Unlike the other parts of the setup, previewing the curved target requires the session is the active session. Click Activate to make it the active session.

  3. Check the pxSMO options. By default, the RET Selection GUI generates the Calibre pxSMO setup. It uses all process window conditions selected for pxSMO in the PW Table at all stages. The defaults follow the best practices for good results.

    • If you have an existing Calibre pxSMO setup file you would prefer to use, toggle the Generate pxSMO Setup field to off and use the folder to load the setup file.

    • Check that the scanner type and pupil fill values are appropriate for your source and litho model. If the scanner wavelength does not match the litho model, an error message is displayed.

    • Set the Image Model to Aerial (the default) or CM1. This sets the image model for all point tools (pxSMO, OPC, and Verify). The CM1 option adds a Polish‑2 job. The first four jobs always use the aerial image.

      Restriction:

      The CM1 option is not available in the current release.

      Tip

      Even if a litho model that includes a CM1 model is loaded, Aerial is the recommended option. The Aerial option can search for solutions beyond the loaded CM1 model parameters.

    • To improve run time, turn off some process window conditions for some jobs.

      • The Start job only requires the nominal process window condition.

      • The Decorate, Improve, Polish, and Polish-2 jobs should use all process window conditions.

    • To anchor the litho model threshold to a gauge, perform “Anchoring to a Gauge”.

    • To assign custom objectives to a pxSMO job for a marker layer with specific process window conditions, go to the Objective Region Tab of the pxSMO job. Add the marker layer and custom objectives. Then select the process window conditions for the optimization.

    • To improve the nominal condition’s solution convergence and common process window, the Improve and Polish jobs should use a weight of 5 for the nominal process window condition. To set the weight, click the job in the Setup Items tree and change the weight value in the PW Table.

    • To add overrides for the jobs, click the job in the Setup Items tree. The pxSMO Job pane lets you enter any valid setup file commands, increase or decrease the iterations, and select process window conditions. The setup file commands are documented in “Calibre pxSMO Setup File Commands”.

    • To optimize CDOF or EL/ILS, click Job:cDOF or Job:EL, respectively, in the Setup Items tree.

      Verify that Optimize is set to the correct optimization. The nominal condition for the job is turned on by default. Add static process window conditions for SRAF print suppression. For the Polish job, the default number of iterations changes to 100. Specify the exposure latitude and DOF starting value.

      For CDOF optimization, set the exposure latitude higher than the target value. For example, if 5 percent is the target, set the exposure latitude value to 6 percent. For an initial run, set the Focus Maximization Factor value to 10.

      Tip

      Test different values of the Focus Maximization Factor and choose the one that maximizes common depth of focus.



      For EL/ILS optimization, specify the EL starting value and depth of focus. Set the Dose Maximization Factor to 100.

    • To specify a custom objective for a marker layer in one of the pxSMO jobs, click the Add button in the job’s Objective Region tab. Select a marker layer and specify “add” to add the custom objective to the current objective or “set” to set the custom objective as the only optimization objective. Define the custom objective by choosing the objectives and specifying their coefficients. Click OK.

      In the Objective Region Table, choose the PW conditions to use with the marker layer and custom objective.

    • To add constant flare across all process window conditions, an EUV lithomodel with a flare model is required. To change the constant flare value, click the Set button next to Constant Flare (%). Select the Set mode and adjust the flare value. To set constant flare values for unique process window conditions, double-click the flare value for the condition in the PW Table and adjust.

    • To save source, mask, or contour files for job-level debugging, click the job in the Setup Items tree and select the desired file types under File Save. For each job, you can choose the appropriate files to save and name each file.

  4. Configure the OPC Setup pane. At a minimum, you need to assign a process window condition to each job. You can use either Calibre pxOPC, Calibre nmOPC or mask-only pxSMO as the OPC engine. Advanced users can use Custom OPC to build a custom flow.

    • For pxOPC:

      • Assign all available process window conditions to all jobs by clicking Check All above the table. (You can reduce the number of conditions per job by clearing individual checkboxes. Reducing the number of conditions reduces run time.)

      • To assign custom objectives to a pxOPC job for a marker layer with specific process window conditions, go to the Objective Region Tab of the pxOPC job. Add the marker layer and custom objectives. Then select the process window conditions for the optimization.

      • If you have an existing Calibre pxOPC setup file you would prefer to use, toggle the Generate pxOPC Setup switch to off and click the folder to load the setup file.

    • For nmOPC, complete “Setting Up the nmOPC Engine”.

    • For mask-only pxSMO:

      • After selecting pxSMO, the Setup Items tree updates the pxOPC items for mask-only pxSMO jobs and the setup pane displays the pxSMO PW table and a reduced set of pxSMO options. Configure the mask-only job similar to the pxSMO job. If 2nd Stage Run is disabled in the pxSMO setup pane, the mask-only pxSMO settings are applied to all clips. If 2nd Stage Run is enabled, then the mask-only pxSMO settings are applied to only to SMO-enabled clips.

      • (Optional) To adjust clip weights and properties for the mask-only job, use Custom Clip Weight to configure the clip settings.

    • For custom OPC flows, complete “Setting Up a Custom OPC Flow”.

    • To add overrides to specific jobs, select a job from the tree and type the commands in the Job Options box at the bottom of the pane. For more information, see the Calibre pxOPC User’s and Reference Manual, especially “Calibre pxOPC Best Practices”.

      Note:

      Although pxSMO does optimize the mask, there are two reasons why it is beneficial to run OPC this second time. First, the second run allows SMO to search without constraints for the best possible source. Second, the independent mask optimization tests the source. Following best practices, more clips are used for the second mask optimization, as it is checking that the optimization is not overfitted to any one structure.

    • To make changes to the mdf smo mask command block, click Edit in the Run Control pane. Find the mdf mask command block in the Commands pane, make changes, and click Close. To run an error check on changes, click Syntax Check at the bottom of the OPC Setup pane. The logs viewer in the session browser displays error messages when errors are detected.

    • To save the mask and contours for each OPC job for debugging, select Enable Job Outputs.

  5. Check the Verify Setup pane.

    • If the Verify job does not have a green check, examine the Verify column in the PW Table to confirm that conditions have been selected.

      If the Verify job warns that the defocus count exceeds the internal limit of 21, reduce the number of unique defocus conditions selected in the Verify column of the PW Table. The defocus count will be greater than the number of unique defocus conditions in the PW Table because the defocus count includes values that are automatically generated for PVBands, Image Shift, and Depth of Focus calculations.

    • Select which contours to write to the results layout and which checks to run. The gauge-based checks run exactly on the gauge polygon. The contour outputs measure properties along the contours. The arrow buttons open additional boxes to configure the most commonly customized fields for each.

      Tip

      Include the Depth of Focus check to calculate common process window and Bossung plots, as well as exposure latitude versus DOF.

    • To save the mask and contours for each OPC job for debugging, activate pxOPC Job Contours.

    • To gather data such as ILS at additional process window conditions, click Property PW in the Setup Items tree. Select Custom and click SET to edit the list of simulated process window conditions.

    • To make changes to the mdf smo verify command block, click Edit in the Run Control pane. Find the mdf verify command block in the Commands pane, make changes, and click Close. To run an error check on changes, click Syntax Check at the bottom of the Verify Setup pane. The logs viewer in the session browser displays error messages when errors are detected.

Results

All top level entries in the Setup Items tree show green checks.
