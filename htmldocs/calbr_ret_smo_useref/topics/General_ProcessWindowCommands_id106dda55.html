Process Window Object Commands

The process window object commands act on a process window object created by mdf smo pwa new. Use them to customize common process window calculations.

mdf smo pwa new | delete — Creates or deletes process window objects ($PWO). The process window objects can be manipulated with the following commands:

  • $PWO cdtol set [-gid gauge | ‘{’gauge_list‘}’] pctcd | {mincd maxcd} — Sets per-gauge CD tolerance by either percent (pctcd) or absolute minimum and maximum. For minimum and maximum, you must specify ‑gid. The default CD tolerance for all gauges is 10%.

  • $PWO compute [‑gid gauge] [‑doselat pctdose] [‑type {rectangular | elliptical}] [‑center {none | dose | focus | both}] [‑nomdose dose] [‑nomfocus focus] — Computes the common process window or (if gauge is specified) the depth of focus for an individual gauge.

    The result is returned as type (ellipse or rectangle), center focus, center dose, maximum latitude, and depth of focus.

  • $PWO filter disable [‑gid gauge_list] — Disables two-pass filtering for the specified gauge list, or all gauges if ‑gid is not specified.

  • $PWO filter enable [‑gid gauge_list] [{‑rmse mult} | {‑maxcd cd}] — Enables two-pass filtering based on first-pass RMSE or maximum residual CD value. Only one of ‑rmse or ‑maxcd can be specified at a time; the default is an RMSE multiplier of 2.0.

  • $PWO gauge get ‑gid gauge {stats | model} — Returns a Tcl list with the requested statistics or model coefficients.

  • $PWO maxlatitude [‑gid gauge | all] [‑dof value] [‑type {rectangular | elliptical}] [‑center {none | dose | focus | both}] [‑nomdose dose] [‑nomfocus focus] — Computes the maximum exposure latitude (EL) as a percentage of the nominal dose for an individual gauge or over all gauges (the common process window). Use ‑dof to specify a fixed-width ellipse or rectangle. If ‑dof is not specified it defaults to 0.

    The result is returned as type (ellipse or rectangle), center focus, center dose, maximum latitude, and depth of focus.

  • $PWO model set {poly13 | poly8 | {dose_order focus_order}} — Sets the modelform to be used for process window analysis. The default is a 13-term polynomial. Both dose_order and focus_order cannot exceed 4.

    You can use $PWO model {enableterm | disableterm} dose_order focus_order to access specific terms.

  • $PWO prefilter disable [-gid gauge_list] — Disables checking the gauges before fitting the model. The pre-fitting filter removes gauges with CD outliers and is recommended.

  • $PWO prefilter enable [-gid gauge_list] [{‑maxcd cd ‑mincd cd} | ‑pctcd pctcd] — Enables checking the gauges before fitting the model. You can further limit the affected gauges with either ‑pctcd pctcd (percentage of the target CD) or ‑maxcd cd ‑mincd cd (explicit range). The default is ‑pctcd 50.

  • $PWO target set [‑gid ‘{’gauge_list‘}’] {nominal | drawn | cd} — Sets the target CD to the nominal or drawn value. To use a custom value (cd), you must specify a list of gauges.

By default, process windows models use two-pass filtering with a threshold of RMSE 2.0. Two pass filtering is a method of fitting the input data and computing the residual values, then removing data points that fail the filter criteria. The remaining data points are then re-fit. This process is done to remove outliers.

Examples

To create a process window object (PW) from the super gauge data file ldof.ellipse.sgd:

set PW [mdf smo pwa new ldof.ellipse.sgd]

Change the prefiltering value:

$PW prefilter enable -pctcd 75

Calculate the DOF on gauge 8 and save the value in a variable, dof:

set dof [$PW compute -gid 8]

Delete the process window object when done to release memory:

mdf smo pwa delete $PW
