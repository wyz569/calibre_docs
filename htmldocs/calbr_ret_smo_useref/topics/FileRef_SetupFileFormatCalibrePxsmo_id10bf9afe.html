Setup File Format for Calibre pxSMO

Input for: Calibre pxSMO

Generated by: RET Selection

Litho setup files are used by most Calibre post-tapeout tools to specify local settings and encapsulate product-specific commands. You can quickly generate a Calibre pxSMO or Calibre pxOPC setup file using the RET Selection window in Calibre WORKbench.

This file reference information is provided in case you want to modify a generated file by hand.

Unlike most litho setup files, Calibre pxSMO setup files are not used with the LITHO FILE command in SVRF rule files. This is because even in batch mode Calibre pxSMO is run by Calibre WORKbench, and not the Calibre hierarchical engine invoked by “calibre -drc”. Other than that, the Calibre pxSMO setup file conforms to the standard litho setup file rules.

Format

A Calibre pxSMO litho setup file must conform to the following formatting and syntax rules:

  • Each statement in the setup file must be on a line of its own. Statements can span multiple lines by ending the line with a backslash (\). The backslash character must not be followed by any other characters, even whitespace.

  • Statements can appear only once in the setup file unless otherwise specified.

  • Keywords are case insensitive.

  • Comments begin with a pound sign (#) and end at the end of the line.

  • All keywords are reserved words. Do not use them as layer or process window condition names.

  • Blank lines are ignored.

Examples

This is a generated setup file. It was created using the Preview PxSMO Setup button.

version 0
pixel_socs_dir <session_path>/tmpworkdir/smo/socs_dir_1.0
output_source_file ./output/source_sopxopc.src
clip_file ./tmpworkdir/smo/clips.dat
source_symmetry +
pupil_fill 0.05
scanner asml
parallelize_by_pw_conditions yes
# PW conditions (aerial image-based)
pw_condition nominal pc_lm_fast aerial model
layer targetSmooth target mask_layer 0
layer target correction
layer weight hidden
weight_layer weight set 5
job 1 start
job 2 decorate
job 3 improve
job 4 polish
