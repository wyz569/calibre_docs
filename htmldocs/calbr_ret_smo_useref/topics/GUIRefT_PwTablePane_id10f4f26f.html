PW Table Pane

To access: In the Session Browser, activate a session

The PW Table pane manages all the process window conditions used by the session.

Description

The PW Table shows the process window conditions and assigns them to different runs within the session. It also performs simple checks, such as for duplicate condition names or non-integer defocus values.

Figure 1. PW Table and Process Window Conditions

Objects

Object

Description

Create PW

Adds a row to the table.

Export

Opens the Export PWs browser. You can save your process window conditions in comma-separated version (.csv) format and use them in other runs.

Import

Opens the Import PWs browser. Rather than manually entering the process window conditions, you can import a previously saved spreadsheet in .csv format. The conditions can be appended to those conditions already in the table or replace them.

Spec

Opens the Optical Model Spec Manager. The Optical Spec Manager creates, edits, and deletes optical model specifications with Zernike coefficients.

Columns

Adds GIDs and Plane columns to the PW Table.

  • GIDs — Displays the gauge IDs selected for PW conditions in a column in the PW table. By clicking the GIDs column for a PW condition, gauges can be selected for the PW condition. Gauges selected for a PW condition are used by pxSMO only; not by OPC or Verify, or with marker layers.

  • Planes — Displays image planes values for PW conditions in the PW Table.

PW Options

Sets the following PW Table options:

  • Global PW (without GID) — Enables or disables process window conditions to be used globally with a GID.

  • Image Plane Interpolation — Specifies the mode of image plane interpolation. The options include Use Model Setting, Enabled (auto mode), Disabled (only mode), and No Extra Planes (only mode). The options reflect the mdf smo configopt command arguments.

  • No Extra Optical Specs — Ignores optical model specifications from the input optical model that are not listed in the Optical Model Spec Manager.

  • Check Duplicate PWs — Highlights duplicate process window conditions in yellow. Conditions are considered duplicates if they share the same name (case-insensitive) or if all numeric values are the same. You should not run with duplicate PW Names because the behavior is unpredictable.

Delete Selected PW Conditions

Click to delete the currently selected PW Table row. If the icon is not blue, you need to select a row in the table.

Table area

You can interact with the table in the following ways:

  • Reorder rows by clicking on column headings.

  • Enter names and values in the non-checkbox columns. Defocus must be an integer.

  • Assign conditions to point tools using the right three columns.

Usage Notes

Only process window conditions that have a run type (pxSMO, OPC, or Verify) selected appear in the setups for the point tools. The nominal condition must exist, and should be assigned to all three.

The Process Window Conditions matrix (Figure 1, right side) shows the dose and defocus coverage of the PW Table. Use the highlight options to quickly check that the Verify conditions are closer to nominal than the pxSMO and OPC conditions.
