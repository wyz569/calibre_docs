Clip File Format

Input for: RET SMO commands

The clip file contains information about clipping rectangles and their properties, relevant for setting boundary conditions, and taking advantage of possible periodicities.

Format

A clip file must conform to the following restrictions:

  • Use one line per clipping rectangle.

  • Use “#” or “//” to indicate a comment.

  • At least one clipping rectangle must be described.

rect x1 y1 x2 y2 option [option]... gid gid [gid...] cid cid \
 [smo 1|0] [opc 1|0] [cname "clipname"] [weight weight_val]
rect x1 y1 x2 y2 option [option]... gid gid [gid...] cid cid\
 [smo 1|0] [opc 1|0] [cname "clipname"] [weight weight_val]

Parameters

rect x1 y1 x2 y2

The required keyword rect indicates the line describes a clipping rectangle. The values x1, x2, y1, and y2 are coordinates in microns. (x1, y1) is the lower left and (x2, y2) is the upper right.

The coordinates are followed by one or more options.

option

An optional keyword and value applied to the clipping rectangle. Valid options are:

  • periodicity {XY | X | Y | N} — Defines if the clip is periodic in X, Y, or both, or not periodic (N). See “Table 4”. The default is not periodic (N).

    For the “half-periodic” clips (periodicity X or Y), the non-periodic direction is treated as aperiodic and the simulation region extends one optical radius beyond the clip.

  • symmetry {F | ‘|’ | - | + | ST | SL } — Defines the clip to have one of the symmetries described in “Table 5”.

    The symmetries can be thought of as being in two groups: reflection (|, -, and +) and flip with reflection (ST and SL).

gid gid [gid...]

A required parameter set specifying the gauge id. The gid value must be an integer.

If multiple gauges are in the same clipping rectangle, specify the additional gauge ids in the same line. Place a comma between each gauge id without spaces.

smo 1 | 0

An optional parameter enabling the clip for the SMO stage. The default value is 1 (clip enabled).

opc 1 | 0

An optional parameter enabling the clip for the OPC and Verify stages. The default value is 1 (clip enabled).

cid cid

A required parameter set specifying the clip id. The cid value must be an integer.

cname "clipname"

An optional parameter set specifying the clip name. The clipname value must be enclosed with quotation marks.

weight weight_val

An optional parameter set specifying a weight applied to a clip and its gauges. Clip weight values specified in the clips file overwrite gauge weight values. The default behavior applies gauge weights.

Examples

Example 1

The following clip file contains information for two clips with no periodicity or symmetry: 

# common clip file name is clips.dat

rect -0.640000 9.360000 0.640000 10.640000 periodicity N symmetry F gid 2 cid 1
rect -0.640000 -0.640000 0.640000 0.640000 periodicity N symmetry F gid 1 cid 2

Example 2

The following clip file contains information for one clip with 3 gauges (comma-separated gids) with no periodicity or symmetry:

rect -0.640000 9.360000 0.640000 10.640000 periodicity N symmetry F gid 1,2,3 cid 1

Example 3

The following clip file contains information for two named clips with the pitch_y clip disabled during the SMO stage:

rect 12.162500 2.197250 13.442500 3.477250 periodicity N symmetry F gid 2\
 cid 2 cname "pitch_x"
rect 22.174625 2.232500 23.454625 3.512500 periodicity N symmetry F gid 3\
 cid 3 cname "pitch_y" smo 0
