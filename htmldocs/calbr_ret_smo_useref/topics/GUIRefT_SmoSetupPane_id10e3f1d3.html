pxSMO Setup Pane

To access: In the Session Browser, activate a session and click pxSMO in the Setup Items tree

The pxSMO setup pane enables quick setup of the SMO run before execution, or viewing of settings when the run is complete.

Description

The pxSMO Setup pane creates a file called pxsmo.setup by default. The more commonly changed settings are available in the Basic tab.

Objects

Object

Description

On/Off Generate PxSMO Setup

Set to Off to use a prepared pxSMO setup file instead of the GUI for configuring pxSMO runs.

pxSMO Mode

Specifies the optimization type:

  • SMO — Source-mask optimization.

  • MO (Mask Only) — Mask-only optimization.

  • SO (Source Only) — Source-only optimization.

Changing the pxSMO Mode changes the job types in the pxSMO setup file. MO mode uses job types ending with _mo; SO mode uses job types ending with _so.

Preview PxSMO Setup

Shows the equivalent SMO setup file for the settings currently selected.

Basic Tab

PW Table

Shows the process window conditions that will be used for each pxSMO job. The nominal condition should always be used.

Only conditions with the pxSMO column selected in the session PW Table PW Table Pane are shown in the pxSMO Setup PW Table. To add a condition, use the session PW Table and click Update.

Scanner

ASML, Nikon, EUV3300, and EUV3400 are particular sets of defaults. They are described in detail in “scanner”. Use Custom to specify the pixel width.

Image Model

Aerial (the default) is recommended when the litho model uses a CTR resist model or aerial images. Setting CM1 adds a Polish‑2 job and can provide a better match with the OPC and Verify results, but requires that the litho model include a CM1 resist model.

Restriction:

The CM1 option is not available in the current release.

Source Symmetry

The symmetry to use for the optimized source. The types are described in “source_symmetry”.

Pupil Fill (%)

The minimum percentage of pixels to illuminate in the source. The scanner provider usually specifies this value.

Anchor/Thres/Dose

Opens the Anchor/Threshold/Dose Settings Dialog Box to set up anchoring gauge and threshold.

Debug Level

Sets the level of messages to be echoed to the transcript. Setting this value as low as possible improves run time and file size.

2nd Stage Run

Applies mask-only optimization (MO) to SMO-disabled clips. After enabling, click the gear icon to set up the MO recipe.

Constant Flare (%)

Appears only if an EUV litho model with a flare model is loaded. Sets the flare level applied across all process window conditions.

Weight Layer Weight

Appears only if at least one of the SMO Layer Type cells is set to Weight when creating the layout/gauge pair. Weight layers are unavailable for CDOF or EL/ILS optimization.

Advanced Tab

Sigma Min/Sigma Max

Specifies the inner (Min) and outer (Max) limits of the source that should be bright in normalized units.

Polarization

Specifies the polarization type used by the scanner and the weighting given to the polarization type. This allows you to compare results from different polarization types.

The default behavior is to use the source polarization type from the first element defined in the “illumtype COMPOSITE” section of the optical model file. Use the pulldown menu to select a different polarization type. The default polarization type is split_xy for DUV scanners and none for EUV scanners. The weighting range is 0.00 to 1.00.

Delete SOCS

Clear the checkbox if you do not want the SOCS directory removed from the session directory at the end of the run. By default Calibre pxSMO deletes the directory after reading it to preserve space.

Common SOCS

Creates a directory to cache the optical model kernels for reuse within the sessions of a single layout/gauge pair. This is not recommended with distributed processing.

Parallel PW

Increases the possible parallelization with distributed processing. If this is set, each combination of gauge and process window condition is sent to a remote CPU. If this is not set, each gauge with all associated process window conditions is sent.

Source Map Capture

Clear the checkbox if space is a concern. By default, Calibre pxSMO saves source map images to update the Result tab before optimization is complete.

Header Options

Type in Calibre pxSMO commands to use in the header level. (For per-job commands, select the Job: node in the tree to the right.) Manual customization is not recommended for the inexperienced.

Usage Notes

To override default job values such as iterations, select the pxSMO Job listing in the navigation tree on the right.
