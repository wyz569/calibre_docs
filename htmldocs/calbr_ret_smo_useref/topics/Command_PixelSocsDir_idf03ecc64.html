pixel_socs_dir

PxSMO Setup File Commands

Specifies a new directory in which to cache optical models, potentially improving runtime.

Note:

To reuse cached optical models, you must set Dispatcher Mode: Off in Run Control. Using one of the dispatcher modes generally improves run time more than pixel_socs_dir.

Usage

pixel_socs_dir cache_directory

Arguments

cache_directory

A required argument specifying a directory to use for caching optical models. The directory should either not exist, or should have been created by an earlier pixel_socs_dir command. It must not be shared with other content. Specifying an existing directory generates a warning.

Use a full path for cache_directory. When running in distributed mode, a local directory will not necessarily be visible to all remote CPUs.

Description

The pixel_socs_dir command uses the cache_directory directory to store all optical models that are needed during optimization for all clips, defocus conditions, and source pixels. Calibre pxSMO reuses appropriate optical models as determined by the combination of clip size, defocus condition, and source pixels. If no appropriate model is in the directory, it will generate a new model and store it as well.

Note:

The models in cache_directory all use the same mapping of the source. When something changes how the source is pixelated, such as changing the scanner type, the models in the cache_directory cannot be reused.

If you are running sessions with different sigmas, pixel widths, or scanner settings, specify a different cache_directory for each pixel type. When a session is set up with a populated cache it cannot use, it deletes the contents and rebuilds the models. If you have two incompatible sessions sharing a directory, they will interfere with each other.

This command improves runtime when there are multiple experiments on the same host that use the same optical file or clip settings, because optical models can be reused rather than regenerated each time. When using distributed processing (any of the Dispatcher Modes other than Off), the SOCS directory should not be shared.

While it is acceptable to delete the directory, the contents should never be modified by a user. Calibre pxSMO has an internal record of the contents and generates an error if the files are touched.
