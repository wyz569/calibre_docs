mdf litho commands

The commands that begin “mdf litho” configure sessions.

Typical commands interact with layout files, load setup files, and start runs.

add

Adds a litho operation to the current list of litho operations.

add op_name1 [‑at‑end | ‑at‑beginning | {‑before | ‑after op_name2}] [‑setup setup_filename] [‑inputs ‘{’input...‘}’ ] [‑outputs ‘{’output_layer...‘}’] [‑gauge_prop prop_name] [‑command {opcverify | denseopc | load | mbsraf | multiload | nmbias | opcbias | pxopc | retsbar }]

Adds a litho operation named op_name1 to the current list of litho operations. The new litho operation is added to the end of the list of litho operations unless another location is specified. If setup_filename is specified, the list of input layers must match the input layer definitions in the setup file.

The ‑command argument runs the specified tool on setup_filename. If the load command is specified, then no setup file is required; the input must be a single .oas file, and the output must be a single layer name. Similarly, if the multiload command is specified, the number of input files must equal the number of output layer names. The layout must correspond to the loaded gauge file. If prop_name is specified with the load command, then a property named prop_name is created and set to the gauge id for each gauge.

Common litho operation names and examples include:

  • loadclips — Transfers layout files into memory. The following example loads clips and assigns the “mdf_gid” property to the gauge id of each gauge:

    mdf litho add loadclips1 ‑inputs {retsel_gaugelayer.oas} \
	‑outputs {retsel_gauges} ‑gauge_prop mdf_gid ‑command load

  • nmopc — Performs the nmOPC step. Usage requires a presraf litho operation and either a pos_sraf or neg_sraf litho operation.

    mdf litho add nmopc ‑setup nmopc.setup \
	‑inputs {squaretgt retsel_pos_sraf tgtsmth retsel_no_mbfrags} \
	‑outputs {retsel_opc_output} ‑command denseopc

  • pos_mbsraf and neg_mbsraf — Performs the Pos-SRAF or Neg-Sraf step for the nmOPC engine. Usage requires presraf and nmopc litho operations.

    mdf litho add pos_mbsraf ‑setup nm_pos_sraf.setup \
	‑inputs {squaretgt tgtsmth} ‑outputs {retsel_pos_sraf} \
	‑command mbsraf

  • presraf — Performs the Pre-Sraf step for the nmOPC engine. Usage requires an nmopc litho operation and either a pos_sraf or neg_sraf litho operation.

    mdf litho add presraf -setup nmopc_pre_sraf.setup \
	‑inputs {squaretgt tgtsmth} \
	‑outputs {retsel_presraf retsel_no_mbfrags} -command denseopc

  • pxopc — Performs the pxOPC step.

    mdf litho add pxopc ‑setup pxopc.setup ‑inputs {squaretgt tgtsmth} \
	‑outputs {retsel_mask} -command pxopc

  • verify — Performs the OPCverify step.

    mdf litho add verify -setup verify.setup -inputs \
	{squaretgt retsel_mask retsel_gauges smo_clips smooth_target} \
	‑outputs {pv nominal f0_d1_b0 gauge_layer}

    The following example implies usage of the default opcverify command that smooths the target:

    mdf litho add smooth -setup smooth.setup -inputs {6} \
	-outputs {targetSmooth}

check

Examines all litho operations to verify that they use the same litho model. The check is on by default and performed at the beginning of an “mdf litho run” command. The following example disables the check:

mdf litho check off

clip_bias

clip_bias ‑input input_layer ‑output output_layer ‑clips gid_list [‑biaspct bias_percent] [‑biastype width|space|space_width|{fix fixed_bias}] [‑minwidth min_width] [‑maxwidth max_width] [‑minspace min_space] [‑maxspace max_space] [‑tablestep table_step]

Applies a bias to clips specified by gid_list.

Commonly used in the presmo block with “mdf smo copyclip” and “mdf litho inter_copy” to create an anchor gauge clip with CD bias on a separate layer.

The following options control edge biasing:

  • ‑biaspct bias_percent — Specifies a fixed percentage CD bias for moving target edges.

  • ‑biastype — Specifies the default edge bias type for moving target edges. Edge bias types include:

    • space — Moves edges based on space dimensions.

    • width — Moves edges based on width dimensions.

    • space_width — Moves edges based on the smaller value of the space or width dimensions. For this option, a table-based bias rule is used, and the ‑tablestep option specifies the bias table step size. The default step size is 0.005.

  • The following options control the biasing of lines and spaces:

    • ‑minwidth min_width ‑maxwidth max_width ‑minspace min_space ‑maxspace max_space

    • If the target CD is between the minimum and the maximum, then the CD bias is applied to the target CD.

    • If the target CD is below the minimum, the CD bias is applied to the minimum value.

    • If the target CD is above the maximum, then the CD bias is applied to the maximum value.

    • When min_space and max_space are set to zero, then they use the min_width and max_width values. All values are in microns.

    • ‑tablestep table_step — Specifies the granularity of the bias table in microns. The default value is 0.005 um steps.

gencliparray

Generates a version of the layout that has adequate spacing between clips to prevent any interaction between optimized masks. For global litho models (used for EUV sources), clips and gauges are written at each slit location. The new layout is written to retsel_cliparray.oas. The adjusted gauges are retsel_cliparray.gg.

The RET Selection GUI automatically includes this command in the pre-SMO stage and loads the new layout to Calibre WORKbench.

genggl

Creates the gauges as polygons. Each gauge goes on a separate layer.

mdf litho genggl layout_filename

inter_copy

inter_copy ‑input input_layer ‑copy copy_layer ‑output output_layer ‑clips gid_list

Copies clips specified by gid_list from copy_layer to output_layer. All clips from input_layer are also copied to output_layer.

This command is used in the presmo block with “mdf smo copyclip” and “mdf litho clip_bias” to create an anchor gauge clip with CD bias on a separate layer. The anchor gauge is copied from the target layer to the correction layer to ensure the correction layer has the target layer anchor clip.

list

Displays the current list or queue of litho operations.

measure

Measures properties based on gauges. For example,

mdf litho measure -input layout_filename -gauges gauges.gg \
	-output mdf_litho.dat -model lithomodel -props {nils meef ils eils epe}
	[‑compute_pw {0|1} ‑pw_focus ‘{’focus_list‘}’

computes NILS, MEEF, ILS, EILS, and EPE for the gauges in gauges.gg with the post-OPC mask in layout_filename. (Each gauge should have a separate layer.) The file specified by ‑output contains the measured property values for each gauge.

Use the ‑target target_layer option if gauges do not span post-OPC mask shapes. This can occur when the mask requires significant OPC adjustments, typically around line ends. The ‑target option can be used with ‑scale factor. This adjusts the scaling of the gauge length against the drawn CD. When used with ‑target, the default is 5. When ‑scale is used without ‑target, the default is 2.

Use the ‑compute_pw 1 option to compute NILS, MEEF, ILS, EILS, or EPE at custom process window conditions. Specify the process window conditions using the following additional syntax:

‑compute_pw 1 ‑pw_focus ‘{’focus_list‘}’ ‑pw_dose ‘{’dose_list‘}’

where focus_list and dose_list specify defocus and dose conditions for simulation. Each list must be enclosed with braces. By default, focus_list uses all defocus conditions specified by mdf smo preverify. The default dose_list is 0.90 to 1.10 step 0.02. By default, ‑compute_pw is set to 0 (disabled).

All clips are merged into one layout. By default, this is tmpworkdir/smo/retsel_maskmerge.oas but you can specify a different name with ‑merge_out filename.

merge_layouts_gauges

merge_layouts_gauges ‑layouts_in ‘{’ ‘{’ input_layout_filename input_layer_list ‘}’... ‘}’ ‑gauges_in ‘{’ input_gauge_filename ...‘}’ ‑layers_out output_layer_list ‑merged_layout merged_layout_filename ‑merged_gauges merged_gauges_filename

Combines multiple layout and gauge pairs into a single pair of files.

The number of input layout files must match the number of input gauge files. The input_layout_filename and input_layer_list must be enclosed with braces. Additionally, the number of layers in input_layer_list must match the number of layers in output_layer_list. If input_layer_list or output_layer_list has more than one layer in the list, the layers must be space separated and enclosed in braces.

run

Executes the current list or queue of litho operations built using “mdf litho add”. Use additional arguments to configure the distributed run. For example, the following command sets the execution to a mode like Calibre® MT™ parallel processing:

mdf litho run -parallel

saveclip and savelayout

Writes a file containing the clips or the entire layout to the specified directory. For example, the following command writes clips to an OASIS layout on the clipLayer layer:

mdf litho saveclip db_home/retsel_clip.oas clipLayer

tol_bands

Generates inner and outer tolerance bands around a target layer.

mdf litho tol_bands ‑input target_layer ‑outputs ‘{’inner_tol_band outer_tol_band‘}’ [‑gauges filename] [‑cdtolmap {layernum:cdtol}...] [‑tolpct pct_cd_tol] [‑minwidth minwidth] [‑maxwidth maxwidth] [‑maxspace maxspace] [‑minspace minspace] [‑biastype {width | space | space_width | {fix fix_value}}] [‑tablestep tablestep] [‑gauge_type_bias {0 | 1}] —

Generates Manhattan inner and outer tolerance bands around the target layer. In a later step, a user-specified pre-SMO curve_target recipe smooths the tolerance bands.

To control the tolerance band width at specific locations, use the -cdtolmap option to specify an input layout containing gauge markers. In the specified layout, the gauge markers must be placed on separate layers with one layer per tolerance value.

Tolerance band generation uses the NMBIAS operation to generate tolerance band edges by biasing target edges. For more information, see the Calibre Rule-Based OPC User’s and Reference Manual, especially the section “LITHO NMBIAS Command”. The following options control edge biasing:

  • ‑biastype — Specifies the default edge bias type for moving target edges. Edge bias types include:

    • space — Moves edges based on space dimensions.

    • width — Moves edges based on width dimensions.

    • space_width — Moves edges based on the smaller value of the space or width dimensions. For this option, a table-based bias rule is used, and the ‑tablestep option specifies the bias table step size. The default step size is 0.005.

  • ‑cdtolmap — Moves the gauge edges using the custom CD tolerance band value.

  • ‑gauge_type_bias — Applies edge bias based on the gauge type when set to 1 and ignores ‑biastype setting. A CD gauge receives a width-based bias and a space gauge receives a space-based bias. Default behavior ignores the gauge type during edge biasing. If a custom CD tolerance is specified for a gauge layer, the gauge-type bias is ignored.

The following example creates inner and outer tolerance bands using the smaller value of the space or width dimensions:

mdf litho tol_bands ‑input {6} ‑outputs {target_inner target_outer} \
	‑tolpct 10 ‑minwidth 0.02 ‑maxwidth 0.25 ‑minspace 0 ‑maxspace 0 \
	‑biastype space_width
