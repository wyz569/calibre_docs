Using the Platform LSF Runscript

The run_lsf.sh runscript executes sessions within the RET Selection environment using the Platform LSF distribution system.

Prerequisites

  • $PATH environment variable must have the LSF bsub command included

Procedure

  1. Create a script to set up your environment for RET Selection, similar to the following. 
    #!/bin/bash
export CALIBRE_MDF_WORKDIR_DEFAULT="./tmpworkdir"
export CALIBRE_MDF_DISPATCHER_DIR="./mdfdispatcher"
export CALIBRE_MDF_DB_HOME="./mdfdatabase"
$MGC_HOME/bin/calibrewb
  2. Run the script to invoke Calibre WORKbench in SMO mode, then select Litho > RET Selection to open the RET Selection GUI.
  3. Edit an existing session by clicking on the following items in succession:
    1. Run Control pane
    2. The session to edit
    3. The Run Config button
  4. Change the Dispatcher Configuration option to Run Script. In the Run Script field, specify the run_lsf.sh script and provide the arguments as follows:
    run_lsf.sh -nummastercpus NUM_PRIMARY_CPUS -numremotecpus NUM_REMOTE_CPUS -masterbsuboptions MASTER_BSUB_OPTIONS 
-remotebsuboptions REMOTE_BSUB_OPTIONS

    where:

    • NUM_PRIMARY_CPUS is the number of CPUs the primary should contain. LSF is instructed to allocate a single machine with that number of cores to be the primary.

    • NUM_REMOTE_CPUS is the number of remote cores to allocate to the job. LSF is instructed to launch Calibre MTflex remotes on that many cores on LSF-allocated machines.

    An example Run Script might look like the following:

    run_lsf.sh -nummastercpus 8 -numremotecpus 8 -masterbsuboptions 'type=Linux -q calcmq' -remotebsuboptions 'type=Linux -q calcmq'

    Save the command.

    Note:

    The example script is generic; you might need to add additional options for your configuration. Contact Siemens EDA for support.
