Assemble Jobdeck Options

To access: In the CMi window, select Tools > Assemble Jobdeck

The Assemble Jobdeck Options dialog box contains controls for the Assembly operation.

Figure 1. Assemble Jobdeck Options Dialog Box

Objects

Table 1. Assembly Options

Field

Description

Chip File Name

One OASIS or GDS-format chip layout may be overlaid onto the mask layout. This provides data for additional layers, and the chip hierarchy. Additional layers may be useful for visual reference when drawing markers, or for importing graphical marker locations from the designer. Drawn layers that are different from the mask data (for example, pre- OPC) may be useful in determining the target CD.

The chip hierarchy may be useful to add markers which repeat in many locations across the mask.

When this option is used, carefully check the overlay of the data to the mask file pattern layer. If the data or transformations are not correctly specified, then the chip data will not be aligned properly to the mask, and the CMi job can produce incorrect results.

Chip Top cell name

The name of the cell in the chip file. The top cell name has restrictions for characters allowed. The allowed and not allowed characters are described in “Allowed Characters in Cell Names”.

Mask Chip cell name

The corresponding name of the chip cell in the mask layout. After assembly, the number of instances of the given cell in the mask is indicated in the center gray box.

Fracture Log File

Loads the transform values from a specified Calibre log file.

The fracture log (the log_file_name specified in the Calibre FRACTURE command, not the Calibre log file) contains the values for Fracture Magnification, Fracture Mirror, Fracture Rotate, and Fracture Shift.

Fracture Magnification

After assembly, the approximate magnification is indicated in the center gray box for reference. This value is determined from the extents of the data, and may be less than the correct value due to the presence of other layers and data outside the fracture extent.

Fracture Mirror

This value allows only none and x transformations. For y, xy, or yx, the rotation and shift must be adjusted.

Fracture Rotate

The choices are 0, 90, 180, or 270 degrees counter-clockwise.

Fracture Shift X Y

After assembly, the approximate shift is indicated in the center gray box for reference. This value is determined from the extents of the data, and may be less than the correct value due to the presence of other layers and data outside or not touching the fracture extent. It may also

be incorrect due to chip rotation by 90 or 270 degrees. The correct value can be obtained from the lower-left coordinate of the fracture extent.

Mask Layer bump

The Bump value is added to the layer numbers in the job deck to avoid collisions with the chip layer numbers. It is common to use a value like 1000.
