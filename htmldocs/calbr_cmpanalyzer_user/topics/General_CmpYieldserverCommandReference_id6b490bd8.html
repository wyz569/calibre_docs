CMP YieldServer Command Reference

Calibre YieldServer includes commands specific to Calibre CMPAnalyzer.

See “Examples for CMP YieldServer Commands” and “Tcl Script for User-Defined Hotspots” for example scripts using these commands.

Refer to “Syntax Conventions” for the command syntax conventions used in the following commands and descriptions.

The following table contains CMP YieldServer commands. These commands can be used in Tcl procedures to calculate hotspots and write report data.

Table 1. YieldServer Commands

Command

Description

cmp::calculate_hotspots

[-threshold_values '{' '{'layer_name hotspot_name hotspot_value [hotspot_name hotspot_value …] '}' …'}']

[-no_defaults]

Runs hotspots calculation for the specified input layers and hotspot name and threshold value pairs.

Arguments:

  • -threshold_values '{' '{'layer_name hotspot_name hotspot_value [hotspot_name hotspot_value …] '}' … '}'

    Specifies one or more input layers and a list of hotspot name and threshold value pairs for each one. Supported hotspot rule names are: Dishing, Erosion, and Depth_Of_Focus.

    If this argument is not specified, all predefined rules are calculated using default threshold values, and all registered rules are calculated using threshold values specified in the user-defined hotspots rule file.

  • -no_defaults

    Specifies when to omit default hotspot calculations. Use this option when user-defined hotspot rules have been specified. Do not use this option with ‑threshold_values.

cmp::cleanup_clusters

‑layer layer_name

Removes hotspot cluster information from the DFM database for the specified layer after a cmp::run_clusterization command.

Arguments:

  • ‑layer layer_name

    Specifies the name of the layer that the clustering data is removed from in the DFM database.

cmp::export_clusters

‑file filename

‑layer layer_name

‑level level_number

[cluster_collection]

[-format {rdb | oasis| gds}]

[-properties property_list]

[-label label_name]

Exports hotspot cluster data from a cmp::run_clusterization command to the specified results database (RDB) file.

The command has two flows (See Step 7 in “Running CMP Hotspots Clustering From Calibre YieldServer” for flow examples):

  • Flow 1 — Dumps hotspot clusters data generated by the cmp::run_clusterization command (.rdb format is default).

  • Flow 2 — Dumps hotspot clusters from a DFM database that contains clusters saved by the cmp::save_clusters command.

Arguments:

  • ‑file filename

    File path to the results database file.

  • ‑layer layer_name

    Specifies the layer, from which the cluster data is exported. (required for Flow 2).

  • ‑level level_number

    Specifies the cluster level (required for both flows).

  • [cluster_collection]

    Specifies an object that is a collection of all hotspot cluster data from a cmp::run_clusterization command and saved by a cmp::save_clusters command. (Flow 1 and Flow 2) For example:

    cmp::export_clusters $clusterManager
  ‑layer M1 ‑level 1 -file ./clusters.rdb

  • [-format {rdb | oasis| gds}]

    Specifies format options for the output database (RDB format is the default), OASIS, or GDSII.

  • [-properties property_list]

    Specifies the cluster property name in the DFM database, from which the data are exported. (required for Flow 2)

  • [-label label_name]

    Defines a label that can be used in the RDB instead of the original layer name. For GDS and OASIS output, do not use existing layer names in the DFM database as a label.

cmp::register_hotspot_proc

-rule_name hotspot_rule_name

‑proc proc_name

[-output_properties list_of_property_names]

Registers the procedure as a hotspot metric to be calculated for the specified rule during the hotspot calculation phase.

Arguments:

  • -rule_name hotspot_rule_name

    Specifies the hotspot rule name. The rule name should be the same as the property name for the hotspot metric.

  • -proc proc_name

    Specifies the procedure name.

  • -output_properties list_of_property_names

    Specifies list of property names generated by the hotspot procedure.

Example:

cmp::register_hotspot_proc -rule_name hs1 ‑proc hs1_metric

See “Tcl Script for User-Defined Hotspots” for a complete example.

cmp::register_hotspot_rules

-file hotspot_rule_file

Registers hotspot rules defined in the specified hotspot_rule_file. The file must contain definitions of hotspot metric calculation and hotspot threshold checking Tcl procedures. You must specify this command before you specify cmp::run_cmp_analysis and cmp::calculate_hotspots.

The hotspot metric calculation procedure should accept a layer as its only input argument. For example:

proc hs1_metric { layer } { 
    ... 
}

The threshold procedure should take two arguments — the layer name and the threshold value — and should return the number of failed tiles. For example:

proc hs1_fails { layer value } { 
# calculate the number of failing tiles 
 
return $failCount 
 
}

cmp::register_hotspot_threshold_proc

-rule_name hotspot_rule_name

-proc proc_name

-values threshold_value

[-output_properties list_of_property_names ]

Registers a procedure that checks metric values against specified threshold values.

Arguments:

  • -rule_name hotspot_rule_name

    Specifies the hotspot rule name. The rule name should be the same as the property name for the hotspot metric.

  • -proc proc_name

    Specifies the procedure name.

  • -values threshold_value

    Specifies a threshold value.

  • -output_properties list_of_property_names

    Specifies list of property names generated by the hotspot procedure.

Example:

cmp::register_hotspot_threshold_proc
-rule_name hs1 -proc hs1_fails -values 100

See “Tcl Script for User-Defined Hotspots” for important information on writing the Tcl procedure.

cmp::rerun_extraction

-layers     sorted_list_of_layer_indexes

-runset modified_runset

[-resimulate]

Reruns extraction and multilayer simulation in the “Calibre CMPAnalyzer Rerun Flow”.

Arguments:

  • -layers sorted_list_of_layer_indexes

    Specifies a sorted list of layer index numbers.

  • -runset modified_runset

    Specifies the name of the updated runset used for the Calibre CMPAnalyzer rerun flow.

  • -resimulate

    Resimulates all the layers starting from the lowest layer index number specified in the ‑layers option.

cmp::run_clusterization

‑layer layer_name

‑property property_name

‑distance max_distance

[-level level_number]

[-area_threshold [1,0]]

[‑density_threshold [0,1]]

Runs hotspot clustering and generates cluster data. See “Running CMP Hotspots Clustering From Calibre YieldServer”

Arguments:

  • ‑layer layer_name

    Specifies the layer for clustering.

  • ‑property property_name

    Specifies the property name for clustering. If no hotspots are defined for the given property, an error message is generated.

  • ‑distance max_distance

    Specifies the maximum distance between clusters for merging to occur (microns).

  • [-level level_number]

    Specifies the maximum level of clustering. The clusterization stops at this level, even if merging is still possible. By default, the clusterization goes to the maximum possible clustering level and stops.

  • [-area_threshold [1,0]]

    Prevents cluster generation in areas greater than (Chip Area * area_threshold). This prevents the cases where a cluster can cover all or a large part of the chip. The default is 1.

  • [‑density_threshold [0,1]]

    Prevents cluster generation if the density of single‑pixel hotspots is less than the density_threshold. The density of single‑pixel hotspots in the cluster is defined as the ratio of the number of single‑pixel hotspots to the number of pixels. The default is 0.

cmp::run_cmp_analysis

[‑input_layers list]

[‑input_layers_types types]

[‑no_save]

[‑step {extraction | simulation | hotspot}]

[‑stop stop_layer_number]

[‑recipefile file]

Runs CMP analysis (the current revision must be the master revision).

Arguments:

  • -input_layers list

    The list of layers to use (the default is all layers).

  • -input_layers_types types

    A list of the types of the input layers to use (corresponding to Model Index values in the GUI table for the CMP simulator).

  • -no_save

    When specified, does not save and freeze the database after the analysis is complete. The default is to save and freeze the database.

  • -step {extraction | simulation | hotspots}

    Run only the specified step of the CMP analysis flow (extraction, simulation, or hotspot calculation). Note that the extraction step can only be performed once. If the simulation step is skipped (the extraction and hotspot calculation steps are performed, in that order), only user-defined hotspots are calculated after extraction. Extraction is done automatically if you specify the simulation step and extraction has not yet been performed.

  • -stop stop_layer_number

    Stop hotspot simulation at the specified number of metal layers in the stack. This option enables partial stack simulation. For example, if -stop 3 is specified, hotspot simulation will occur for the first three metal layers.

  • -recipefile file

    Runs the simulation step with the specified recipe file.

cmp::save_clusters

cluster_collection

Saves a collection of all hotspot cluster data to an object generated by the cmp::run_clusterization command.

Arguments:

  • cluster_collection

    Specifies an object that is a collection of all cluster data from a cmp::run_clusterization command. It can be referenced by a cmp::export_clusters command.

    cmp::save_clusters $clusterManager

cmp::save_hotspots_report

-file filename

[‑format {txt | rdb}]

Writes a file that for all hotspot rules, contains coordinates of windows that violate thresholds for the rules.

Arguments:

  • -file filename

    Specifies the path to the output file.

  • ‑format {txt | rdb}

    Specifies format options for the output file as plain text format (the default) or RDB file format.

cmp::save_xrc_report

-dir path

Saves thickness data to a set of files.

Arguments:

  • -dir path

    Path to the output directory.

Command

Description

cmp::abs_grid

grid

Returns the absolute grid for the specified grid. Each element of the result grid is an absolute (value) of the corresponding element of the input grid.

Arguments:

grid

Specifies the grid object.

cmp::add_grid

grid1 grid2

Returns the sum grid of two specified grids. Each element of the result grid is a sum of the corresponding elements of the input grids.

Arguments:

  • grid1

    Specifies the first grid object.

  • grid2

    Specifies the second grid object.

Grid objects must have the same dimensions.

One of the grid arguments can be a double‑precision floating‑point number.

grid1 grid2

Returns the AND grid of two specified grids. Each element of the result grid is a result of a logical AND operation on the corresponding elements of the input grids.

Arguments:

  • grid1

    Specifies the first grid object.

  • grid2

    Specifies the second grid object.

Grid objects must have the same dimensions.

One of the grid arguments can be a double‑precision floating‑point number.

cmp::average_area_grid

grid

‑radius {R | '{'Rx Ry'}'}

Returns a grid where each element is calculated as an average value of the given neighboring area in the input grid.

Arguments:

  • grid

    Specifies the grid object.

  • ‑radius {R | '{'Rx Ry'}'}

    Defines the radius of area around the element for which the average value is calculated. For border elements, the average value is calculated based on the existing neighboring elements.

For example, the average value for the first element of a grid (a00) with radius 1 is calculated as follows:

average(a00)=(a00 + a01 + a10 + a11)/4

cmp::create_grid

{‑layer layer_name

    ‑property property_name |

‑column column_count

     ‑row row_count}

[‑value initial_value]

Creates a new grid. The grid can be created either based on values of the specified property of the layer or based on the specified row and column dimensions and initial value.

Arguments:

  • ‑layer layer_name

    Specifies the layer name.

  • ‑property property_name

    Specifies the name of a property of the layer.

  • ‑column column_count

    Specifies the column count of the created grid.

  • ‑row row_count

    Specifies the row count for the created grid.

  • ‑value initial_value

    Specifies an initial value to be assigned to all elements of the grid (double‑precision floating‑point number). The default value is 0.

cmp::divide_grid

grid1 grid2

Returns a quotient grid of two specified input grids. Each element of the result grid is a quotient of the corresponding elements of the input grids.

Arguments:

  • grid1

    Specifies the first grid object.

  • grid2

    Specifies the second grid object.

Grid objects must have the same dimensions.

One of the grid arguments can be a double‑precision floating‑point number.

cmp::equal_grid

grid

{‑row | ‑column | ‑element}

‑limit limit_value

‑result_value {t_value |

     '{'t_value f_value'}'}

Returns a grid matrix with values t_value if the selected parameter for the corresponding element is equal to the limit and f_value otherwise. By default, f_value is the corresponding value of the input grid.

Arguments:

  • grid

    Specifies the grid object.

  • ‑row | ‑column | ‑element

    Defines the parameter of the grid used in the comparison operations.

  • ‑limit limit_value

    Defines the comparison value (double‑precision floating‑point number).

  • ‑result_value {t_value |

         '{'t_value f_value'}'}

    Defines the elements in the result grid matrix based on the outcome of the comparison operation. The values can be either a double‑precision floating‑point number or a pair of double‑precision floating‑point numbers.

cmp::export_grid

grid

‑file filename

[‑standard | ‑matx | ‑maty | ‑matz]

[-rdb]

[-coords]

[-layer layer_name]

[-property property_name]

[{-less | -greater | -equal |     ‑less_equal | ‑greater_equal} threshold_value]

[-append]

Dumps grid data into a specified file.

Arguments:

  • grid

    Specifies the grid object.

  • ‑file filename

    File used for saving grid data.

  • ‑standard

    Specifies to dump the grid sizes, coordinates, and values. This is the default.

  • ‑matx

    Specifies to dump the x‑coordinates of the grid.

  • ‑maty

    Specifies to dump the y‑coordinates of the grid.

  • ‑matz

    Specifies to dump a 2D matrix with property values.

  • -rdb

    Specifies to output a file with property values in RDB file format (.rdb). The default format is text.

  • -coords

    Specifies to dump a text file similar to -standard but with tile coordinates and property information.

  • -layer layer_name

    Specifies the name of the layer, mandatory for RDB (.rdb) format.

  • -property property_name

    Specifies the name of the property (can be a string), mandatory for RDB (.rdb) format.

  • {-less | -greater | -equal | ‑less_equal | ‑greater_equal}     threshold_value

    Specifies the condition type and threshold value. For example, “‑greater 0” means that the RDB file contains tiles with threshold value > 0. If none of these options are specified, the RDB file contains all tiles.

  • -append

    Specifies to append additional grid data to an existing RDB output file.

cmp::gradient_grid

grid

‑radius {R | '{'Rx Ry'}'}

[‑max | ‑min | ‑average]

Returns a grid where each element is a calculated gradient for a corresponding element of the input grid in the given neighboring area.

Arguments:

  • grid

    Specifies the grid object.

  • ‑radius {R | {'{'Rx Ry'}'}

    Defines the radius from the element for which the gradient is calculated.

  • ‑max | ‑min | ‑average

    Defines which difference from the element in the neighboring area should be used. By default, the command calculates the maximum difference.

Example:

cmp::gradient_grid grid1 ‑radius 1 -average

The average value for the first element (a00) of grid1 with radius 1 is calculated as follows:

average(a00)=(a00 + a01 + a10 + a11)/4
