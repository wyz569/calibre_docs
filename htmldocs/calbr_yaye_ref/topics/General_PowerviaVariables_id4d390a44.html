powervia Variables

The powervia utility uses certain variable specifications for running the Calibre YieldEnhancer PowerVia flow

The applicable environment variables and techlib.tcl file variables are shown in the following table.

Table 1. Variables for powervia Utility

Variable Name

Description

POWER_VIA_TECH
Note:

(Required.) You must specify this environment variable in the command line before running the powervia utility.

Set value to the path where the techlib.tcl file resides. Both full and relative (./) paths (to current directory) are supported.

RUN_DEF_BA

In techlib.tcl file:

Set value to “YES” (case insensitive) to enable the backannotation flow. See “Backannotation Flow Specifications”.

DEF_DESIGN

In techlib.tcl file:

Set value to the design cell name of the DEF. By default, the design name is set to primary, the cell name specified in LAYOUT_PRIMARY. In the case of multiple designs, the first design topcell name is used. When DEF_DESIGN is specified, the tool does not use Calibre DESIGNrev to retrieve the input database topcell.

See “Backannotation Flow Specifications”.

DEF_FILE

In techlib.tcl file:

Set value to the filename of the incremental DEF file in the backannotation flow. If not specified, the default file name is incremental_<primary>.def, where primary is a cell name specified in LAYOUT_PRIMARY. See “Backannotation Flow Specifications”.

DEF_VERSION

In techlib.tcl file:

Set value to the DEF version number to use in the backannotation flow. By default, the DEF version is 5.8.

See “Backannotation Flow Specifications”.

INSERTED_VIA_FILE_PREFIX

In techlib.tcl file:

Set value to the filename of the output database with the filled vias.

Note:

Do not include the file extension (.oas/.gds), because the tool automatically appends the extension to the end of the filename based on OUTPUT_GDS.

By default, the filename is via_punch_fill.(oas/gds). If the techlib.tcl OUTPUT_DIR variable is not specified, the file is generated in the current directory; otherwise, it generates in the output directory.

POWER_VIA_LAYOUT_PATH

Environment variable:

Specify in the command line where value is the full path or relative (./) path (to current directory) to the input layout database.

LAYOUT_PATH

In techlib.tcl file:

Set value to full path of the input layout database including filename (.gds or .oas) using one of two options:

  • Specify a full path enclosed in quotation marks (“  ”).

  • Specify a relative path to your current run directory.

LAYOUT_PRECISION

In techlib.tcl file:

Set value to the precision value of the input design. Valid values are positive integers or floating‑point numbers. If specified, the flow does not require a Calibre DESIGNrev license to retrieve the input database precision. Refer to “Calibre YieldEnhancer” in the Calibre Administrator’s Guide for complete licensing information.

LAYOUT_SYSTEM

(Required.) In techlib.tcl file:

Set value to “GDSII” or “OASIS” or “LEFDEF” for the format of the input layout database file.

MERGED_OUTPUT

This option is available only with LAYOUT_SYSTEM “GDSII” and “OASIS”.

In techlib.tcl file:

Set value to the full path and mode of the merged output.

  • file_name — Specify the full path, including file name, for the merged output of the original input and the via_punch_fill.(oas/gds) layout database files.

  • mode — Specify “append” or “rename”:

    • “append” — Concatenates the contents of the duplicate cells together.

    • “rename” — Renames all cells output from powervia to make them unique with added suffix “_powervia”.

    Note:

    This option requires a Calibre DESIGNrev license. Refer to “Calibre YieldEnhancer” in the Calibre Administrator’s Guide for complete licensing information.

OUTPUT_DIR

In techlib.tcl file:

Set value to control the path of the output files. If this environment variable is not specified, by default, the output files are written to your working directory. Both full and relative (./) paths (to current directory) are supported.

OUTPUT_GDS

(Required.) In techlib.tcl file:

Set value to “Yes” for GDSII output layout database file.

SELECT_LAYER

(Required.) In techlib.tcl file:

Set value to specify one of two modes of operation for via insertion:

  • “SINGLE” — Inserts vias on all layers defined in the layer stack.

  • “via1 via2 … viaK” — Inserts vias only on the specified layers.

TECH_SPECIFIC_FLAG

(Required.) In techlib.tcl file:

Set value to your foundry process technology keyword enclosed in quotation marks (“  ”).

ENC_TEMPLATE

(Required.) In techlib.tcl file:

Set value to either the full or relative (./) path (to the current directory) that includes the file name of the via enclosure template file.

This file is an editable template provided by your Siemens EDA representative and is required input for all foundry processes.

PERC_FLOW

In techlib.tcl file:

Set value to absolute or relative (to the current directory) path to an SVRF file that contains the layers used for the Calibre® PERC™ flow.

If this variable is set, the techlib.tcl flow uses the PERC flow layers defined in the specified file instead of generating these layers. See “Calibre PERC Flow File Specifications”.

Refer to the Calibre PERC User’s Manual for more information on the Calibre PERC flow.

POWER_VIA_NETS_FILE_NAME

Environment variable:

Specify in the command line where value is either the full path or relative (./) path (to the (current directory) to the file that contains net names for via insertion or to the name of the net names file in your working directory. This variable performs the same functionality as POWER_VIA_NETS_NAME.

NETS_FILE_NAME

In techlib.tcl file:

Set value to either the full path or relative (./) path (to the current directory) to the net names file for via insertion or to the name of the net names file in your working directory. If specified with NETS_NAME, then NETS_FILE_NAME is used as the path to the net names file.

POWER_VIA_NETS_NAME

Environment variable:

Specify in the command line where value is either the full path or relative (./) path (to the current directory) to the file that contains the nets for via insertion or to the name of the net names file in your working directory. This variable performs the same functionality as POWER_VIA_NETS_FILE_NAME.

NETS_NAME

In techlib.tcl file:

Set value to either the full path or relative (./) path (to the current directory) to the net names file for via insertion or to the name of the net names file in your working directory. If specified with NETS_FILE_NAME, then NETS_FILE_NAME is used as the path to the nets name file.

NETS

In techlib.tcl file:

Set value to net name(s) for via insertion enclosed in braces ({ }) and space separated.

POWER_VIA_BEOL_STACK

Environment variable:

Specify in the command line where value is the BEOL stack name for your design.

BEOL_STACK

In techlib.tcl file:

Set value to the BEOL stack name for your design.

LAYER_MAP_FILE

(Required.) In techlib.tcl file:

Set value to your layermap.svrf file enclosed in quotation marks (“  ”). Both full and relative  (./) paths (to the current directory) are supported.

LAYOUT_PRIMARY

In techlib.tcl file:

Set value enclosed in quotation marks (“  ”) to specify a topcell name for the utility to run on. If not specified, the value defaults to “*”, the topcell of the input design. See “Layout Primary” in the Standard Verification Rule Format (SVRF) Manual for more information on name conventions.

RDB_FILE

In techlib.tcl file:

Set value to the name of the output results database (RDB) file. Starting and trailing whitespaces are ignored in this file. By default, the file name is set to powervia.rdb.

RESOLUTION

In techlib.tcl file:

Set value to a positive integer that specifies the step size of the layout grid in database units. If specified, it appears once in the SVRF rule file. Specifying the step size causes the grid step to be square (equal in x‑ and y‑directions).

SVRF_VERSION

In techlib.tcl file:

Set value to the Calibre version specified as vDDDD.D_D*.D*, where “v” is a literal, “D” is a decimal digit, and D* is one or more decimal digits.

If specified, the SVRF VERSION statement with the Calibre version are written to the SVRF rule file. If a Calibre version earlier than specified is used, the tool exits with an error message. If this variable is not specified, no Calibre version checking is done.

TEXT_LAYERS

(Required.) In techlib.tcl file:

Set value to the text_layers file path. See your Siemens EDA representative to determine which one of these path specifications is used for your foundry flow:

  • Specify the full or relative (./) path (to the current directory) to the existing text_layers file.

  • Specify the path including the directory and filename for the text_layers file which needs to be generated.

FILTER_RECT

(Required.) In techlib.tcl file:

Set value for via insertion behavior on overlapping shapes with 45 degree (skew) angles:

  • “Yes” — Only the rectangular region of the overlapping shapes is considered for via insertion.

  • “No” — The entire region of the overlapping shapes is considered for via insertion.

VIA0i_blockage_layers

In techlib.tcl file:

Set value to a string that contains a derivation of layers to be printed as blockage layers in the SVRF file. For example:

set VIA0i_blockage_layers “layer_derivations”

If this variable is not specified, the flow generates the blockage layers by default.

The use of this variable is specific to the process flow. Contact your Siemens EDA representative for the variables specific to your process flow.

VIA_VARIABLES

(Required.) In techlib.tcl file:

Set value to the path of the input via variables file. See your Siemens EDA representative to determine which input via variables file is used for your flow. Both full and relative (./) paths (to the current directory) are supported.

INITIAL_PLACEMENT

In techlib.tcl file:

Set value to specify the starting origin of the placement grid with the corresponding via insertion region to one of the mutually‑exclusive options:

  • “LL” — Lower left.

  • “LR” — Lower right.

  • “UL” — Upper left.

  • “UR” — Upper right.

  • “CENTER” — Center. (This is the default origin if the INITIAL_PLACEMENT variable is not specified.)

DISABLE_ SPECIAL_FILTERS

In techlib.tcl file:

Set value to “Yes” to disable special empty‑area via filters. The default is “No”.

DEBUG_DIR

In techlib.tcl file:

Set value to control the content and location of the generated intermediate files (via_addition.sum, net_aware_PWR_GND.oas, calibre.log):

  • “NONE” — Specifies to delete all the intermediate files.

  • <debug_dir> — Creates a directory named <debug_dir>.time_stamp, which contains all the intermediate files.

    If the DEBUG_DIR variable is not specified, then the flow generates all the intermediate files in your current working directory.

DEBUG_SPACE

In techlib.tcl file:

Set value to specify a list of fill vias for which debugging information is written to an output file. This produces an OASIS file with fillable regions and spacing layers that represent empty areas for the specified fill vias (LRG, BAR, SQR…). For example:

set DEBUG_SPACE “VIA0i VIA1i”

The output filename is specified by the DEBUG_SPACE_FILE variable.

DEBUG_SPACE_FILE

In techlib.tcl file:

Set value to specify the output filename for DEBUG_SPACE. The file must be OASIS format or an error is generated. By default, the output filename is debug_space.oas.

If the techlib.tcl DEBUG_DIR variable is not specified, the output file is generated in the current directory; otherwise, it outputs to the specified debug directory.

VIA_TYPE

In techlib.tcl file:

Set value to define a list of via types (LRG, BAR, and SQUARE) to insert in a single run. The sequence of the via types in the list is not important. If this variable is not defined in the techlib.tcl file, the flow inserts square vias by default.

The VIA_TYPE variable can be used in combination with the techlib.tcl SELECT_LAYER variable. If a via is defined in SELECT_LAYER and is not defined with a via type in VIA_TYPE, by default the tool inserts square vias for that via. For example:

set SELECT_LAYER “V3 V4 V5”

set VIA_TYPE “V4BAR V5LRG V5”

The tool inserts V3 square (default), V4BAR, V5 square, and V5 LRG vias.

See “Insertion of Different Via Types (LRG, BAR, SQUARE)” for more usage information.
