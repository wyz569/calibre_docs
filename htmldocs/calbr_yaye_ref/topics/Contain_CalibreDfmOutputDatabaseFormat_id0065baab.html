Calibre DFM Output Database Format

The Calibre DFM products can output a Results Database (RDB) and a DFM database.

The following table summarizes the differences between the DFM output formats.

Table 1. Comparing Output Database Options for DFM

Format

Pros

Cons

DFM RDB File

  • Designers who are familiar with the Calibre DRC flow are used to using this format.

  • Stores object properties if the are created using DFM Property.

  • Larger file sizes.

  • Does not support storage of object attributes.

DFM Databases

  • Smallest file size.

  • Stores object attributes.

  • Stores object properties.

  • Supports interactive drill‑down.

  • Proprietary.

  • Saving DFM Results to an RDB
