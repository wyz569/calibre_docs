Concurrency for DFM Operations and Statements

Concurrency refers to running multiple layer operations simultaneously, thereby reducing processing time.

Whenever Calibre nmDRC performs layer operations, the tool locates all required layer operations within the set that can run concurrently and executes them as a single group. You can optimize your rule file by taking advantage of this feature. Table Table 1 lists the supported concurrency configurations.

Table 1. Supported Concurrency for DFM Operations and Statements

Operations

Operations Run Concurrently when they have…

DFM Analyze

Same input layers (in the same order), and same WINDOW, STEP, and INSIDE OF parameters

DFM Measure

N/A

DFM Critical Area

Same input layers, same particle size list, same particle density list

DFM Transition

Same input layer and file parameters

DFM Function

N/A

DFM Grow

N/A

DFM Expand Enclosure

N/A

DFM Property

Same input layers and same INSIDE OF parameters

DFM RDB

N/A

For more information on this topic, refer to “Concurrency” in the Calibre Verification User’s Manual.
