Save Dialog Box

To access: In the Recipe tab, select Add > save.

Use this dialog box to set up or edit a save command in a process recipe file. This dialog box is not used for saving sessions or process recipes.

Field

Description

+

–

Toggles the dialog box to show or hide additional fields.

Property options

Various properties that can also be chosen to appear. The full list is documented under “save”.

  • Dump saves the entire state in a binary format.

  • State saves the entire state in an ASCII format.

  • WL is the frame area divided by number of polygons (area per feature).

  • Z1 is the non-trench, or field, height.

  • Z2 is the trench height.

  • Z3 is the height of the trench bottom.

File

A base name for the saved grid files. You can control the grid file type by including the .txt, .text, or .bin suffix as part of the base filename. If you are saving “dump” or “state”, the filename automatically gets a .dump or .sta file extension.

InFile

A grid file that indicates the frames to save by weighting them with a value of 1. CMP Model Builder looks for file.txt, file, and cmp/file.txt in that order.

Other

Use for entering additional command parameters manually.

Usage Notes

See “save” for more details on the property definitions.

For simulations involving multiple models, it is recommended to save the state after each model’s simulation and re-load it with initialize. This reduces optimization time.
