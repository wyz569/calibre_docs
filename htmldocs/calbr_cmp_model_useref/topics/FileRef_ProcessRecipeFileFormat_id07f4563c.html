Process Recipe File Format

Generated by: Recipe tab or ASCII text file

The process recipe file contains commands describing the process steps your silicon goes through in the manufacturing process.

Format

  • Process steps (deposition, etch, and CMP) must be presented in the same order as they are executed in manufacturing.

    • The first non-comment line must be the initialize command.

    • This must be followed by one or more process commands (deposit, etch, cmp) that mirror your manufacturing flow.

    • All model (define_model) and mask definition (define_mask 2D) commands must be supplied at the end of the file.

  • Parameters are separated by whitespace. Do not use a space between a value and its unit.

  • Commands that extend over several lines must use a backslash (\) as the final character at the end of a line to indicate the command continues on the next line.

  • Comments are lines that begin with a pound sign (#).

  • When representing physical quantities, you can provide the value and units of measure. If units are not specified, it is assumed to be in the default units for that parameter.

    No spaces should separate the value and units.

  • Any command may appear multiple times in the recipe, including initialize and save.

    Tip

    To improve simulation time, after each simulation save the optimized process model and load it into the next simulation using the initialize command.

Parameters

Command

Description

cmp

Indicates a polishing step, typically for a copper or oxide layer. Each platen should have a cmp command.

define_mask 1D

Defines a series of trench arrays that can be used as input instead of an existing layout. (The 2D form is preferred.)

define_mask 2D

Defines a region of trench arrays that can be used as input instead of an existing layout.

define_model

Defines models (sets of parameters) that are called by the cmp, deposit, and etch command.

deposit

Adds a new layer of material to the simulated process.

etch

Indicates a trench etching step.

initialize

Required. Specifies the geometry at the start of the simulation. This command can also be used to load saved grids when modeling multiple layers.

save

Saves the simulation information to a file.
