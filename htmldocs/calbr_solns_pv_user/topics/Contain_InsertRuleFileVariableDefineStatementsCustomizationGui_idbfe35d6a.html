Insert Rule File Variable and #DEFINE Statements with the Customization GUI

You can use the Customization GUI in Calibre Interactive to place #DEFINE and #UNDEFINE pre-processor directives and variable definitions in the rule file. The customization GUI is a dialog box with user-coded controls which place statements in the Calibre Interactive control file.

The customization GUI is coded using the Tcl language, as described in “Customization Files” in the Calibre Interactive User’s Manual. The following figure shows a customization GUI based on the examples in the following section.

Figure 1. Customization GUI in Calibre Interactive

The Customization GUI can be used with both interactive and batch execution.

Since the customization file is written in Tcl, it can read environment variables set in the shell environment and use them in coding the customization controls.

Note:

When using the Customization GUI to insert #DEFINE or #UNDEFINE statements into the Calibre Interactive control file, be sure that the rule file does not contain any #DEFINE or #UNDEFINE statements for the same variable, since later statements take precedence over those in the control file.

See the topic “Methods to Specify a Customization File” in the Calibre Interactive User’s Manual for information on loading a customization file. See the other topics in the chapter “Customization Files” for details about the customization language and commands.
