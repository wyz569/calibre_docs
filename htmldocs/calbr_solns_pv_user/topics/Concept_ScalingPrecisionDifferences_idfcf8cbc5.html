Scaling and Precision Differences

Scaling is determined from the physical precision and the rule file precision.

Assuming the following definitions of terms:

L

Layout Precision, or the reciprocal of the physical precision.

R

Rule file Precision, or the reciprocal of the measurement precision.

S

Scaling factor, which is the factor by which coordinate data is multiplied due to differing precisions.

The following relation holds:

S = L / R

Example

Assume the following:

LAYOUT PRECISION 1000 // 0.001 user units
PRECISION 10000 // 0.0001 user units

Then we have this scaling factor:

1/10 = 1000/10000

Data is scaled by a factor of 1/10 upon read-in. Similar reasoning applies on output when comparing the Layout Precision to the DRC Results Database Precision.

We now introduce these terms:

M

Layout Magnify statement multiplication factor.

F

Final multiplication factor applied to the layout. It is the product of the scaling factor S and the Layout Magnify factor M.

The following relation holds:

F = M * S

Assume the following:

LAYOUT PRECISION 1000 // 0.001 user units
PRECISION 10000 // 0.0001 user units
LAYOUT MAGNIFY 10

then this follows:

S = 1/10

M = 10

F = 1/10 * 10 = 1

Scaling and magnification have important implications for where data is placed during a Calibre run. In particular, this affects where skew edges get placed, because they are subject to their endpoints being snapped to the new grid.
