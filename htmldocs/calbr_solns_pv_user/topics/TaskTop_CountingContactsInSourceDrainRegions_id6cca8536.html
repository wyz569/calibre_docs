Counting Contacts in Source-Drain and Gate Regions

You can use a multilayer DFM Property operation to return the number of contacts in each source-drain region and each gate region.

The process first counts the number of contacts in each source-drain region and saves this as the property sd_con attached to the source-drain shapes on the layer sd_count.

The source-drain regions abut the gate shapes, so a multilayer DFM Property operation with the ABUT ALSO keyword is used to sum the contacts in each source-drain region abutting a gate. The MULTI keyword is needed because a source-drain region can be associated with more than one gate geometry. If you do not specify MULTI, each source-drain region is associated with a single gate, which Calibre determines arbitrarily, and a warning is issued in the transcript. The PROPERTY() function is used to read the sd_con property value attached to each source-drain region.

Calibre Advanced DRC (eqDRC) Tutorial and Example Kit

This example is in the contact_via_checks/sd_count example directory.

Procedure

  1. Derive the layers for recognizing gate and source-drain regions:
    LAYER CON              // contact layer
gate = POLY AND AA 
imp = AA NOT POLY 
sd = imp INTERACT POLY
  2. Count the contacts in each source-drain region using a DFM Property operation:
    sd_count = DFM PROPERTY sd CON INTERSECTING 
               [sd_con = COUNT(CON)]
  3. Sum the contacts in the source-drain regions associated with each gate. 
    gate_count = DFM PROPERTY gate sd_count OVERLAP ABUT ALSO MULTI 
                 [gate_con = PROPERTY(sd_count,sd_con)]
  4. Output the results of the DFM Property operation using DFM RDB.
    contact_counts {
    DFM RDB sd_count eqdrc.rdb CHECKNAME "%_l_" 
        ALL CELLS CELL SPACE
    DFM RDB gate_count eqdrc.rdb CHECKNAME "%_l_" 
        ALL CELLS CELL SPACE
}
