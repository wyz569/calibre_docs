DRC HTML Report for Waiver Flow

The following example demonstrates using DRC HTML reporting to generate a report for a DRC waiver run. The report is used to document the errors that remain in the design and the waivers that were applied.

The DRC waiver run produces the following DRC results databases:

  • DIGBLK_drc_w_waivers.db — The DRC error results database. Several results are marked as fixed or waived in Calibre RVE.

  • waived.rdb — The waived results database.

  • unused_waiver.rdb — The results database containing waivers that were not used.

See the Calibre Auto-Waiver User’s and Reference Manual for complete information.

A view of the results in Calibre RVE is shown in the following figure, where the results databases are shown side by side.

Figure 1. Calibre RVE View of Waiver Flow Results

The error database (DIGBLK_drc_w_waivers.db) is shown on the left and the waived.rdb and unused_waiver.rdb databases are shown on the right.

The DRC HTML report described in this section is run on all databases so that the remaining errors, the applied waivers, and the unused waivers can be documented. The DRC HTML report includes the following sections:

  • Summary grouped by Cell/Check

  • Summary grouped by Cell/Check/#SC (shape class)

  • DRC results with highlights for check METAL1_SPACE. This report is grouped by Cell/Check/#SC and sets RowCount = 1, so that only one result per shape class is seen. This report only processes the main results database by default.

  • Table view of DRC results for METAL1_SPACE that are waived within Calibre RVE. This report only processes the main results database by default.

  • Highlights of the results in the Waiver RDBs. This report section includes the key “ApplyTo = Waiver” so that it only processes the two waiver RDBs (waived.rdb and unused_waiver.rdb) produced by the run.

    This section produces two reports in the output; for the example shown each report includes one highlight image.

A check text override file is used to specify the layers shown during highlighting, and a DESIGNrev preferences file is used to adjust display options.

The complete HTML report is shown in Figure 2; details about each section are given following the figure, including the relevant sections of the configuration file.

Figure 2. DRC HTML Report for Waiver Flow

Command Line

% calibre -rve -drc -input ./rdbSpec_all.txt 
	-report ./batch_waivers_all.ini -outputdir batch_waivers_all 
	-summaryfile ./DIGBLK_drc_w_waivers.summary

Input Files

rdbSpec_all.txt — A DRC HTML Report Database Map File. This file specifies the DRC Results Database, the layout database, and a Calibre DESIGNrev layer properties file, as shown here: 

; error database
DIGBLK_drc_w_waivers.db,,DIGBLK_W.calibre.gds,DIGBLK_W.calibre.gds.layerprops

batch_waivers_all.ini — The DRC HTML Report Configuration File. The text for this file is described in the section “Configuration File and Output Description”.

DIGBLK_drc_w_waivers.summary — A DRC Summary Report created by the Calibre run that created the results database.

Output Directory and Files

The HTML report is named index.htm and created in the output directory batch_waivers_all in this example. All supporting files are located in the output directory.

Configuration File and Output Description

Each section in the configuration file is described in the following section and the resulting report section is shown. The complete configuration file is included at the end of this example.

Common section

This example common section specifies the directory for a Calibre DESIGNrev preferences file, the layout image size, and other general settings. 

[Common]
UseFrames = 1
DrvOptionsFilePath = ../
LayoutWidth = 200
LayoutHeight = 250
WorldView = 1

For the views shown here, the preferences file contains one line: 

prefs_grid_style -1

This setting turns off grid markers in the layout images. This preferences file is merged with the defaults used by DRC HTML reporting, with settings in the specified file taking precedence if there is a conflict.

DRCResultsSummary report section

This example includes two sections that produce a DRCResultsSummary report. The first summary is grouped by Cell/Check, the second by Cell/Check/#SC (shape class): 

[Summary]
ReportType = DRCResultsSummary
GroupBy = Cell/Check

[Summary grouped by shape class]
ReportType = DRCResultsSummary
GroupBy = Cell/Check/#SC

The view for the second report is shown in the following image and corresponds to the tree view shown on the left side of Figure 1.



DRCResults report section

The first DRCResults report section is grouped by Cell/Check/#SC and only includes results for the METAL1_SPACE check. The intent of this report is to show one highlight for each of the three result shapes that are not waived or fixed.

; group by shape class, 
; set RowCount=1 to get just one highlight for each result shape
[METAL1_SPACE Results, errors only, highlight one result per shape]

ReportType = DRCResults
GroupBy = Cell/Check/#SC 
OrCheckFilter = METAL1_SPACE
; view errors only
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed
RowCount = 1
Highlight = 1
ZoomLayoutFactor = 0.6

The report section is shown in the following figure:



The second DRCResults report section does not include highlights and selects only waived results. By default, this section is only applied to the main results database, which is of RDB type “General”. 

[METAL1_SPACE Results, waived, report table only]
ReportType = DRCResults
GroupBy = Cell/Check/#SC 
OrCheckFilter = METAL1_SPACE
WaivedStateFilter = Waived

The report section is shown in the following figure:



The last DRCResults report section highlights the waivers found in the waived.rdb and unused_waivers.rdb databases produced during a DRC waiver run. The ApplyTo key is used to apply this section to only Waiver RDBs. 

; ; highlight applied and unused waivers using ApplyTo = Waiver
[View waiver RDBs]
ReportType = DRCResults
ApplyTo = Waiver
GroupBy = Cell/Check
Highlight = 1

Complete Configuration File

[Common]
UseFrames = 1
DrvOptionsFilePath = ../
LayoutWidth = 200
LayoutHeight = 250
WorldView = 1
[Summary]
ReportType = DRCResultsSummary
GroupBy = Cell/Check

[Summary grouped by shape class]
ReportType = DRCResultsSummary
GroupBy = Cell/Check/#SC 
; group by shape class, then set RowCount=1 to get just one highlight for
; each result shape
[METAL1_SPACE Results, errors only, highlight one result per shape]
ReportType = DRCResults
GroupBy = Cell/Check/#SC 
OrCheckFilter = METAL1_SPACE
WaivedStateFilter = NotWaived
FixedStateFilter = NotFixed
RowCount = 1
Highlight = 1
ZoomLayoutFactor = 0.6
[METAL1_SPACE Results, waived, report table only]
; ; applies to General RDBs by default
ReportType = DRCResults
GroupBy = Cell/Check/#SC 
OrCheckFilter = METAL1_SPACE
WaivedStateFilter = Waived
; ; highlight applied and unused waivers using ApplyTo = Waiver
[View waiver RDBs]
ReportType = DRCResults
ApplyTo = Waiver
GroupBy = Cell/Check
Highlight = 1
