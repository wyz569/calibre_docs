Converting DBCLASSIFY to DFM Pattern Classify for Result Classification

DBCLASSIFY is an obsolete layer operation that was used in some RET flows to classify errors (or patterns) by shape. DFM Pattern Classify can also classify error locations.

Note:

As of the 2019.3 release, DBCLASSIFY no longer functions and is not documented.

The following two rule checks show a basic classify check done with each operation, where hotspot is the seed or error marker layer, and target1 and target2 are the context or pattern layers.

db_all { 
  DBCLASSIFY FILE [
    HALO 0.4
    CENTERPT YES
  ] 
  hotspot target1 target2 
  MAP "ALL ERRORS"    
}
 
pmclass_all {
  DFM PATTERN CLASSIFY
     SEED hotspot HALO 0.4  
     // see note on ALLOWED_SHIFT
     ALLOWED_SHIFT 0.2 
     LAYER_TARGET target1 
     LAYER_TARGET target2
     CLASSIFY_PROPERTY CLASS_ID
     OUTPUT ALL 
     DUP_COUNT DUP_COUNT
}
Note:

ALLOWED_SHIFT in DFM Pattern Classify enables hotspots with slightly different center points relative to a pattern to be classified as the same pattern.

However, DFM Pattern Classify does not have an option similar to the DBCLASSIFY TOLERANCE option, so pattern geometries must be an exact match.

Use the following table to assist in converting existing DBCLASSIFY operations to DFM Pattern Classify. See DFM Pattern Classify in the SVRF Manual for the full syntax of the operation. A discussion of possible differences in output follows the table, then an example.

ALLOWED_SHIFT value [SEARCH_LEVELÂ EXACT]

Recommended. This option performs auto-centering of the pattern center, with an allowed shift of value. The recommended value is one half of the halo size.

The optional SEARCH_LEVEL EXACT keyword uses an improved search algorithm that attempts to optimize the number of duplicate patterns, and can increase runtime.

Possible Differences in Output

The number of unique, duplicate, and unclassified results from the two operations can vary for these reasons:

  • DBCLASSIFY sometimes does not merge error_layer shapes that are split across the hierarchy. This causes DBCLASSIFY to classify two different error markers, while DFM Pattern Classify classifies one merged shape.

  • For error_layer shapes with skew edges, small differences in the center point of the shapes can lead to different classification results. The use of ALLOWED_SHIFT value and SEARCH_LEVELÂ EXACT in DFM Pattern Classify usually prevents this.

  • DFM Pattern Classify classifies all locations without a target layer edge within the halo region with a class ID of â1 (indicating no valid pattern). For example, a location with no target layer geometry and a location with a target layer geometry that completely covers and extends beyond the halo region are both classified with a class ID of â1. DBCLASSIFY classifies such locations as unique results.

Example 1

The example uses DBCLASSIFY, and writes results to an RDB. It also writes two rule check outputs, one with all results and one with unique results. DFM Pattern Classify is used to generate similar rule check output and an RDB that is similar to that produced by DBCLASSIFY. 

LAYOUT SYSTEM OASIS
LAYOUT PATH 'layout.oas'
LAYOUT PRIMARY TOPCELL

LAYER target1  6
LAYER target2  7 
LAYER hotspot  5 

DRC RESULTS DATABASE drc_results
DRC SUMMARY REPORT   drc.summary
db_all { 
    DBCLASSIFY FILE [
        HALO 0.4
        CENTERPT YES
        RDB dbclass.rdb
    ] 
    hotspot target1 target2 
    MAP "ALL ERRORS"    
}
db_uniq { 
    DBCLASSIFY FILE [
        HALO 0.4
        CENTERPT YES
    ] 
    hotspot target1 target2
    // MAP "UNIQUE ERRORS"  // this is default
}
pmclass_all = DFM PATTERN CLASSIFY
  SEED hotspot HALO 0.4  
  ALLOWED_SHIFT 0.2 // SEARCH_LEVEL EXACT
  LAYER_TARGET target1 LAYER_TARGET target2
  CLASSIFY_PROPERTY CLASS_ID
  OUTPUT ALL  // ALL is default
  DUP_COUNT DUP_COUNT
pmclass_all { 
    copy pmclass_all
}
pmclass_uniq = DFM PATTERN CLASSIFY
  SEED hotspot HALO 0.4 
  ALLOWED_SHIFT 0.2 // SEARCH_LEVEL EXACT
  LAYER_TARGET target1 LAYER_TARGET target2
  CLASSIFY_PROPERTY CLASS_ID
  OUTPUT UNIQUE      
  DUP_COUNT DUP_COUNT
pmclass_uniq { 
    copy pmclass_uniq
}
// Use DFM Property to remove invalid locations (with CLASS_ID -1)
// from pmclass_all and pmclass_uniq layers. Get unclassified locations
// with CLASS_ID = -1.     
pmclass_rdb {
all_valid = DFM PROPERTY pmclass_all 
            [ CLASS_ID = PROPERTY(pmclass_all, CLASS_ID)] != -1
            [ DUP_COUNT = PROPERTY(pmclass_all, DUP_COUNT)]
uniq = DFM PROPERTY pmclass_uniq 
            [ CLASS_ID = PROPERTY(pmclass_uniq, CLASS_ID)] != -1
            [ DUP_COUNT = PROPERTY(pmclass_uniq, DUP_COUNT)]
// get locations of duplicates
dupl_loc = all_valid NOT uniq
dupl = DFM PROPERTY all_valid dupl_loc  
            [ CLASS_ID = PROPERTY(all_valid, CLASS_ID)] != -1
            [ DUP_COUNT = PROPERTY(all_valid, DUP_COUNT)]
            [ - = COUNT(dupl_loc)] > 0
uncl = DFM PROPERTY pmclass_all 
            [ CLASS_ID = PROPERTY(pmclass_all, CLASS_ID)] == -1
            [ DUP_COUNT = PROPERTY(pmclass_all, DUP_COUNT)]
DFM RDB uniq "pmclass.rdb"  ALL CELLS MAXIMUM ALL checkname "%_l_"
DFM RDB uncl "pmclass.rdb"  ALL CELLS MAXIMUM ALL checkname "%_l_"
DFM RDB dupl "pmclass.rdb"  ALL CELLS MAXIMUM ALL checkname "%_l_"
}
