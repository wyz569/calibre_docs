Verifying Contact Run Length for Current Crowding

You can quantify current crowding through a gate based on the run length of contacts on each side of the gate and identify gates that have the worst current crowding effects.

This rule uses the ratio of gate width to total contact run length. As this ratio increases, the current density around the contacts is likely to increase, causing an undesirable increase in the series resistance of the device. See Figure 1.

This example is in the contact_via_checks/current_crowding_con_run_length example directory.

Procedure

  1. Derive the contact to gate edge pairs:
    con_gate_spc = EXT CON gate < 0.15 OPPOSITE
  2. Derive the gate edges:
    gate_edges = COIN EDGE gate sd
  3. Use the LENGTH function to measure the length of the gate edge, and the EC function to measure the common run length of the contact-to-gate edge pairs. Divide these quantities to obtain a ratio: 
    current_crowding = DFM PROPERTY gate_edges con_gate_spc
                     OVERLAP ABUT ALSO
                     [ratio = LENGTH(gate_edges)/EC(con_gate_spc)]

    A high ratio suggests increased current crowding.

  4. Output the results using a DFM RDB operation.
    current_crowding {
  DFM RDB current_crowding eqdrc.rdb CHECKNAME "%_l_" 
       CELL SPACE ALL CELLS 
}
