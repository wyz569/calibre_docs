Selecting Checks in Calibre nmDRC Recon

By default, Calibre nmDRC Recon automatically enables a set of rule checks that run fast and are not context-dependent. You can customize the checks that are run, depending on where you are in the design cycle.

These specification statements assist in customizing the checks that are enabled:

Specification Statement

Description

DRC Recon Add Check

Executes only the specified checks. You can specify predefined groups of checks with keywords. For example, RECON_CONNECT selects all connectivity checks.

DRC Recon Remove Check

Removes the specified checks. You can specify predefined groups of checks with keywords.

DRC Recon Add Check By Layer

Selects checks to execute by layer.

DRC Recon Remove Check By Layer

Selects checks to omit by layer.

When using these statements, an explicitly named rule check must be selected to run in a standard Calibre nmDRC run. All of the add statements are processed first, then the remove statements, so it is possible for a remove statement to disable a check that is enabled by an add statement. See the command references for more detail.

The following table lists the keywords for the predefined rule check groups available with the DRC Recon Add Check and DRC Recon Remove Check statements. The Default column indicates whether the set of rule checks is executed in a default Calibre nmDRC Recon run.

 

Table 1. Keywords for Rule Check Groups in Calibre nmDRC Recon

Keyword

Definition

Default?

RECON_DEFAULT

All rule checks that would be executed by default in Calibre nmDRC Recon mode.

Yes

RECON_DENSITY

All selected rule checks that require a Density operation.

Yes

RECON_OFFGRID

All selected rule checks that require an Offgrid Directional or non-simple Offgrid operation. A non-simple Offgrid operation uses any of the following keywords: INSIDE OF LAYER, RELATIVE, ABSOLUTE, EDGE, HINT, or OFFSET.

No

RECON_CONNECT

All selected rule checks that require connectivity, including those with DFM DV operations.

No

All selected rule checks that require a DFM DV operation.

The following procedure gives an example of how to use these statements to customize the set the checks that are run by Calibre nmDRC Recon, depending on what stage in the design cycle you are in.

Procedure

  1. For the initial run, execute the default set of Calibre nmDRC Recon checks, but exclude all density checks and exclude all checks that depend on layers m3, m4, and m5 anywhere in the layer derivation tree of the rule check. Add this to the rule file:
    DRC RECON REMOVE CHECK BY LAYER m3 m4 m5
DRC RECON REMOVE CHECK RECON_DENSITY

    Start a run:

    calibre -drc -hier -turbo -recon rules |& tee log

    Debug the results and proceed to the next stage when the design is ready.

    Tip

    If you use DRC Recon Add Check By Layer, it adds all checks that require the specified layer, including checks not selected by default in a Calibre nmDRC Recon run. To remove checks not run by default, do the following:

    DRC RECON ADD CHECK BY LAYER m1
DRC RECON REMOVE CHECK RECON_CONNECT RECON_DV RECON_MP 
          RECON_PM RECON_OFFGRID
  2. Run with the default set of Calibre nmDRC Recon checks on all layers, but still exclude density checks. Comment out the first specification statement from the previous step:
    //DRC RECON REMOVE CHECK BY LAYER m3 m4 m5
DRC RECON REMOVE CHECK RECON_DENSITY

    Do a Calibre nmDRC Recon run and debug the results. Proceed to the next stage when the design is ready.

  3. Run with the default set of Calibre nmDRC Recon checks, include density checks and connectivity checks, but exclude DFM DV checks. Density checks are run by default, so it is not necessary to explicitly add them. 
    DRC RECON ADD CHECK RECON_DEFAULT RECON_CONNECT
DRC RECON REMOVE CHECK RECON_DV
    Note:

    If you use DRC Recon Add Check or DRC Recon Add Check By Layer, you must specify all checks or rule check groups that you want to execute. For example, the following statement executes only the connectivity checks:

    DRC RECON ADD CHECK RECON_CONNECT // does not include default checks

    Do a Calibre nmDRC Recon run and debug the results. Continue this process as necessary. The remaining checks to enable are DFM DV, Calibre Pattern Matching, and Calibre Multi-Patterning checks.
