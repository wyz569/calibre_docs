Empty Layer Checks

In certain situations, it is desirable to check if a particular layer is empty.

In a TVF FUNCTION, you can use the following:

if {[tvf::IS_LAYER_EMPTY $layer]}
  # layer is empty
  ...
} else {
  # layer is non-empty
  ...
}

The tvf::IS_LAYER_EMPTY command returns a 1 if the layer is empty and a 0 otherwise. This command has excellent performance and is probably the best solution in most cases. It could be used as follows:

TVF FUNCTION empty_layer [/*
    tvf::GET_LAYER_ARGS inputLayer myExtent
    if {[tvf::IS_LAYER_EMPTY $inputLayer]} {
         tvf::OUTLAYER AREA $myExtent == 0
    } else {
         tvf::OUTLAYER COPY $myExtent
    }
*/]

extent_ = EXTENT DRAWN ORIGINAL
// if in_layer is empty, empty1 is empty. otherwise, empty1 is the extent
// of the database.
empty1 = TVF empty_layer in_layer extent_

The following uses traditional SVRF coding:

// x is the layer whose emptiness is of interest
chip = EXTENT DRAWN ORIGINAL
chip_x = chip INTERACT x

M1.space {
// if x is not present, measure this
EXT (M1 NOT chip_x) < 0.5
// else measure this
EXT (M1 AND chip_x) < 0.3
}

The motivation here is to derive a layer covering the extent of a layer of interest (x in this case). Then, if that layer is present, derive a certain result; otherwise, derive another result. The EXT operation is arbitrary here. The NOT and AND operations are also arbitrary to a certain degree; yet the typical implementation is to use inverse layer operations to cover the cases of whether chip_x is present or not.

For global layer checking related to (possibly) empty layers, the following table shows some useful options.

Table 1. Global Empty Layer Checks

Rule File Statement

Description

Layout Input Exception Severity EMPTY_LAYER 1

Lists the empty simple layers in the transcript. This is the default setting.

Layout Input Exception Severity UNUSED_DATA 2

Issues a warning in the transcript for each layer and datatype pair that are not used in the run. The default is 1, which means to list all layers and datatypes in the transcript.

Layout Input Exception Severity DATA_MAP_SOURCE 1

Issues a warning in the transcript when a simple layer required for the run has only some of its data transferred to other layers by Layer Map statements. This is the default setting.

Layout Input Exception Severity DATA_MAP_TARGET 1

Issues a warning in the transcript when a simple layer required for the run has additional data transferred to it by a Layer Map statement. This is the default setting.
