Running Calibre nmLVS Recon Softchk

Calibre nmLVS Recon Softchk performs stand-alone Sconnect stamping conflict (soft connection) checks independent of an LVS run. This permits conducting these checks in parallel with things like stand-alone short isolation.

These verification runs are performed on a Mask SVDB Directory produced with the SI keyword. For complete details, see “-recon -softchk” in the “Calibre nmLVS Reconnaissance Command Line” section of the Calibre Verification User’s Manual.

Prerequisites

  • A Calibre nmLVS Reconnaissance license.

  • Calibre nmLVS/Calibre nmLVS-H license pairs.

    A Calibre Query Server license is also required if using -softchk=tcl_script.

  • A layout design.

  • An LVS circuit extraction rule file.

Procedure

  1. Specify the SI keyword in a Mask SVDB Directory statement in your rule file.
  2. Run hierarchical circuit extraction:
    calibre -spice layout.sp -turbo ... rules |&tee log
  3. Run stand-alone soft connection checks:
    calibre -recon -softchk -turbo -svdb svdb
  4. (Optional.) As an alternative to step 3, prepare a Query Server Tcl shell script called softchk.qs that contains either the qs::softchks or qs::softchk command. Then run stand-alone soft connection checks as follows:
    calibre -recon -softchk=softchk.qs -turbo -svdb svdb

Results

If step 3 is followed, soft connection checks are run on the layout data in the PHDB in a similar manner as LVS Softchk LOWER in a circuit extraction run. The checks are run on any Sconnect stamped layers (the second layer in the Sconnect syntax) for which there was a stamping conflict during circuit extraction. Such conflicts are indicated in the circuit extraction report like this:

WARNING:  Stamping conflict in SCONNECT - Multiple source nets stamp one target net.

If step 4 is followed, then soft connection checks are run on the layout data in the PHDB according to the qs::softchks or qs::softchk command configuration.

The output filename is softchk.rdb, and this can be opened as an ERC database in Calibre RVE.
