Reducing Layout Magnification

It is desirable to reduce layout magnification (shrinking the layout) without causing problems due to grid snapping. This is accomplished by increasing the precision.

Prerequisites

  • Physical layout and rule file.

  • Familiarity with the concepts discussed under “Considerations When Magnifying Data,” “Key Specification Statements for Magnification,” and “Magnification and Precision Management.”

Procedure

  1. Choose your magnification factor and multiply it by 10. For example, if your multiplication factor is 0.9, use 9 for the following step.
  2. Add the following statement to the rule file:
    LAYOUT MAGNIFY 9 PLACE // intending to use a 0.9 shrink factor
  3. Use a Precision that is 10 times the Layout Precision value:
    PRECISION 10000 // scaling layout by 1/10
LAYOUT PRECISION 1000 // database grid unit is 0.001 user units

    This set of statements in conjunction with Step 2 causes a final layout multiplication factor of 0.9, with coordinate values maintained to the fourth decimal place.

  4. [Optional.] You can use Size … UNDEROVER with the new measurement dbu value on drawn layers to fill in possible gaps due to grid snapping:
    layer_snap = SIZE layer BY 0.0001 UNDEROVER

    Be aware this can have a performance penalty if it causes much layer merging across the hierarchy. Another alternative is to adjust measurement constraints by 2 dbu for layers subject to grid snapping.

Results

The DRC Results Database Precision follows the Precision statement setting by default. So in this case, the physical precision of the output is 0.0001 user units. The scaled results are reduced by a factor of 0.9 and are on the output database unit grid.
