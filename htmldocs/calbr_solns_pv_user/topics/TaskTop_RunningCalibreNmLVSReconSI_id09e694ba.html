Running Calibre nmLVS Recon SI

Calibre nmLVS Recon SI performs stand-alone short isolation independent of an LVS run. Such runs are typically very fast and can be used early in the design verification process to avoid various problems in later LVS runs.

There are two flows:

  • Rule file flow — Performs short isolation using LVS circuit extraction rules and a layout as inputs.

  • Mask SVDB Directory flow — Loads an SVDB from a previous LVS circuit extraction run or previous Calibre nmLVS Recon SI run, and performs short isolation on that data.

Only connectivity extraction and short isolation are performed (no device extraction or anything dependent upon it). For complete details, see “-recon -si” in the “Calibre nmLVS Reconnaissance Command Line” section of the Calibre Verification User’s Manual.

Prerequisites

  • A Calibre nmLVS Reconnaissance license.

    If -si=tcl_script is specified, a Calibre Query Server license is also required.

Required inputs depend on the flow:

  • Rule file flow:

    • A layout design.

    • A rule file with LVS circuit extraction rules that include a Mask SVDB Directory statement that produces a persistent hierarchical database (PHDB). Some of the common keyword options produce the PHDB are include CCI, PHDB, QUERY, and SI.

    • A Calibre nmLVS/Calibre nmLVS-H license pair.

  • Mask SVDB Directory flow:

    • An SVDB from a previous LVS circuit extraction run in which the Mask SVDB Directory SI keyword was used or a previous Calibre nmLVS Recon SI run.

Procedure

  1. (Optional) Specify the Connect and Sconnect statements that are processed prior to short isolation. By default, all such statements that produce connectivity to the level of the device pins are processed. Reducing the number of Connect statements that are processed improves runtime at the potential cost of finding fewer shorts.

    The method you use depends on the flow:

    • Rule file flow: Include the LVS SI Select Connects statement in the rule file.

    • Either flow: Specify a Query Server Tcl shell script with the short_db::select_connects command. The Query Server script is specified on the command line in the next step.

  2. Run Calibre nmLVS Recon SI with one of the following command line syntaxes, where opts corresponds to run options such as ‑turbo:

    Rule file flow:

        calibre -recon -si['='{ALL | DB | IO | PG | tcl_script }] [opts] rule_file_name

    Mask SVDB Directory flow

        calibre -recon -si['='{ALL | IO | PG | tcl_script }] [opts] -svdb svdb_dir [ top_cell]

    By default, all nets at the primary level of the design are processed for short isolation. If the optional equals sign (=) is specified after ‑si, then the ALL, IO, and PG modal options specify to process nets at all levels, I/O signal nets at the primary level, or power and ground supply nets at the primary level. The tcl_script option specifies a Query Server Tcl shell script that runs commands from that interface on the PHDB data. The DB modal option is used for two-step processing where just the Mask SVDB Directory is produced in the first run, and short isolation is run separately on the SVDB in subsequent runs. This can have performance benefits in some cases.

    An LVS Isolate Shorts NO statement, specified either explicitly or by default, is ignored in this flow. LVS Isolation Shorts YES secondary keywords are observed if specified in the rules and are applied to the run if they do not conflict with the command line option behavior. See the “-recon -si” command-line argument descriptions in the Calibre Verification User’s Manual for further details.

Results

Connectivity text shorts, opens, and unattached label warnings are given in the transcript, which is the same as for short isolation when performed in a calibre -spice run. However, a circuit extraction report is not written, nor is a .shorts database.

The tail of the transcript contains a section like this when shorts are detected:

Current short serial number is 1
  Short circuit (index: 0) (cell: TOPCELL) (net_id: 2) (text: PWR).
    PWR at (1.3 82.4) (layer metal2)
    top_short at (3.4 78) (layer metal2)

Warning: 1 short paths from 1 out of 1 shorts were selected in primary 
cell TOPCELL for short isolation.

Shorts from non-primary cells may also be shown depending on rule file settings. If no shorts are detected, then this is shown instead:

Short Isolation results are clean for ALL cells.

The rule file flow creates a new Mask SVDB Directory with short isolation results, while the Mask SVDB Directory flow updates the existing SVDB directory. Short isolation results from either flow can be viewed and debugged as usual in Calibre RVE for LVS. The results presentation follows the LVS Isolate Short BY CELL BY LAYER options. You can also use the Calibre Query Server Tcl shell.
