Finding Pad Edges Closest to the Chip Edge

You can identify edges of pads that are closest to the chip edge using marker polygons. This check can be useful for determining which way pads should be oriented.

Figure 1. Pad Edges Closest to the Chip Edge

    Try It!

Calibre Advanced DRC (eqDRC) Tutorial and Example Kit

Go to this page on Support Center to download the complete eKit.

This example is in the miscellaneous_checks/pads_chip_edge example directory.

Procedure

  1. Using the Enclosure operation, derive edge pairs that measure the separation from the exterior-facing sides of the pad layer to the interior-facing sides of the chipedge layer.
    VARIABLE enc_dist 0.15 
enc_error = ENC pad chipedge <= enc_dist OPPOSITE

    Use an appropriate measurement constraint for your design.

  2. Expand by a small amount each pad edge that faces the chip edge to create marker polygons:
    enc_region = EXPAND EDGE (ENC [pad] chipedge <= enc_dist OPPOSITE)
               OUTSIDE BY 0.001

    The use of brackets ([]) surrounding the pad layer in the Enclosure operation causes edge-directed output, rather than the default error-directed output.

  3. Attach DFM properties to the marker polygons to indicate the distance between each marker polygon and the chip edge.
    enc_value = DFM PROPERTY enc_region enc_error OVERLAP ABUT ALSO 
              [ChipEdgeDistance = MIN(EW(enc_error))]
  4. Identify the pad edges closest to the chip edge. To do this, use the MIN() function to determine the smaller distances, then transfer the DFM properties to the pad layer.
    enc_min = DFM PROPERTY pad enc_value OVERLAP ABUT ALSO 
              [ChipEdgeDistanceMin = 
                 MIN(PROPERTY(enc_value,ChipEdgeDistance))]
  5. Identify the marker polygons that correspond to the closest edges. To do this, first, for each marker polygon, divide each distance by the minimum distance. Then, use a constraint of “==1” to select the marker polygons with the minimum distance. The non-persistent property name “-” is used because the value is not of interest.

    Output the chip to pad distance as the property ChipEdgeDistance.

    closer_edge = DFM PROPERTY enc_value enc_min OVERLAP ABUT ALSO MULTI 
                 [- = PROPERTY(enc_value,ChipEdgeDistance)/
                      PROPERTY(enc_min,ChipEdgeDistanceMin)] == 1 
                 [ChipEdgeDistance = 
                      PROPERTY(enc_value,ChipEdgeDistance)]

    If a pad at the chip corner has edges with equal distances to the chip edge, both marker polygons are output.

  6. Output the results of the DFM Property operation using DFM RDB.
    pad_edge {
    DFM RDB closer_edge advDRC.rdb CHECKNAME "%_l_" 
       CELL SPACE ALL CELLS
}
