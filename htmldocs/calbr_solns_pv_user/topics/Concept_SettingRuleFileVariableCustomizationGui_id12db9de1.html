Setting a Rule File Variable with the Customization GUI

You can use the customization GUI to insert a Variable statement in the Calibre Interactive control file. The following example reads the environment variable METAL_WIDTH from the shell and uses the value in a Variable statement.

Suppose the SVRF rule file has this code, where MW_var is an SVRF variable used to specify the width constraint. 

// width constraint set by svrf variable MW_var
min_poly_width {
 @ Minimum poly width = ^MW_var
 int poly < MW_var opposite
}

Then suppose that you want to set the value of MW_var using the environment variable METAL_WIDTH. The following customization file does that.

if {[info exists ::env(METAL_WIDTH)]} { 
  CUSTOM::LABEL  -tool DRC \
   -prompt "METAL_WIDTH set in shell: $::env(METAL_WIDTH)"
  CUSTOM::VARIABLE  -prompt "Metal width setting" \
    -name "MW_var" -initval $::env(METAL_WIDTH) -tool DRC
} else {
  CUSTOM::LABEL -tool DRC \
   -prompt "METAL_WIDTH not set in shell - define it here" 
  CUSTOM::VARIABLE -name "MW_var"  \
   -prompt "Metal width setting" -tool DRC
}

If you are using a TVF rule file, add the “‑tvf” option to the CUSTOM::VARIABLE statements, which places a Tcl variable in the Calibre Interactive control file.
