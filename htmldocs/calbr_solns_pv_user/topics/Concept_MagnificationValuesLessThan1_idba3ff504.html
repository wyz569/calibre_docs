Magnification Values Less Than 1

When generating mask data output and using magnification values less than 1, it is recommended to increase your measurement precision.

For example, consider these coordinates in user units:

(1.996, 1.654)

Assume the following are in the rule file:

PRECISION 1000 
LAYOUT PRECISION 1000
LAYOUT MAGNIFY 0.9

The product of the layout magnification with the coordinates is this:

(1.7964, 1.4886)

Because the rule file Precision is 1000, these values are rounded to the third decimal place and the vertex coordinates are snapped to these values:

(1.796, 1.489)

In one case the coordinate is rounded down, in the other it is rounded up. In either case, if this occurs at a cell boundary, gaps in geometry can result in placements of that cell after magnification.

Referring to the discussion of the final magnification factor F under “Scaling and Precision Differences”, you can eliminate the effects of rounding by choosing these rule file values:

PRECISION 10000 // increased by a factor of 10
LAYOUT PRECISION 1000
LAYOUT MAGNIFY 9 PLACE

You still have a final magnification factor of 0.9, but because of the increase in Precision to the fourth decimal place, you eliminate the rounding and grid snapping. The PLACE keyword for Layout Magnify is recommended when reducing magnification.
