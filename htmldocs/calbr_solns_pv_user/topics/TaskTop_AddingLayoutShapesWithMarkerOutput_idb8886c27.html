Adding Layout Shapes Using Pattern Marker Output

You can use pattern matching to identify a problem area in a layout and fix the problem by adding layout shapes.

Figure 1. Layout Correction with Pattern Marker Output

Pattern matching generates a polygon layer with the required layout corrections.

This procedure uses exact pattern matching for simplicity. In reality, some variation in the position of the pattern edges is usually allowed. The procedure “Adding Constraints to Pattern Edges and the Output Marker” discusses that case.

Prerequisites

  • A pattern library with a pattern that matches the area of interest. See “Capturing a Pattern From the Layout”.

  • A layout in GDS or OASIS format. The layout should be suitable for testing with your pattern library.

Procedure

  1. Open the pattern library in the Calibre Pattern Matching GUI:
    calibrepm lib.pmdb
  2. Edit the pattern to add a custom marker layer.
    1. Select the pattern in the pattern list in the lower left of the GUI.
    2. Click the Layers tab at the bottom of the panel to the right of the pattern canvas.
    3. On the Layers tab, click the button to add a marker layer.

      In the Add Marker dialog box, change the marker name to m_edit and choose a marker type of Custom.

  3. Add a custom marker shape to the pattern to correct the layout.
    1. In the Layers tab, select the m_edit marker layer.
    2. Above the pattern canvas, click the button to enter Create Rectangle drawing mode.

      Use the cursor to draw a rectangle in the pattern where the added layout shape should be placed. Make the marker shape extend 0.1 microns to the right of edge C (refer to Figure 1).



  4. Click in the top toolbar to save the library.
  5. Use the GUI to run pattern matching with the library. This step creates a basic rule file that is edited in a later step.
    1. Click the button in the top right of the GUI to start a pattern matching run with the open library.
    2. Select Layout File as the layout source and fill in the text field with the path to the layout.
      Tip

      You can also run on an open layout by selecting Active Layout as the layout source. See “Running Pattern Matching From the GUI” in the Calibre Pattern Matching User’s Manual.

    3. In the layers table, make sure the library layers correspond to the correct layer number in the layout. If not, click in the Layout Layers cell and select the correct layer number.

      Other options can remain at their default settings.

    4. Click Run.

    The transcript is displayed, along with a list of the files created by the run.

  6. Examine the results of the pattern matching run:
    1. Open the DRC results by clicking the folder icon in the DRC RDB row.


      The layout is opened in Calibre DESIGNrev and the DRC results database is opened in Calibre RVE.

    2. In Calibre RVE, select the pm_Marker_out check and type “H” on the keyboard to highlight all results.

      All matching layout locations should be highlighted. The highlight marker is the pattern extent.

    3. In Calibre RVE, select the pm_m_edit_out check and type “H” on the keyboard to highlight all results.

      The custom marker shapes are highlighted. Confirm that the shapes make the layout modifications that you want.

  7. Edit the pattern matching rule file to save an OASIS layout file with the pm_m_edit_out custom marker shapes.
    1. In the run directory, copy the file pm_test_lib.svrf to the new file edit_layout.svrf.
    2. Open the file edit_layout.svrf in a text editor.

      The file includes the following lines:

      INCLUDE <path>/lib.svrf
CMACRO pm:lib pm_layer1 " " pm_Marker_out pm_m_edit_out

      The CMACRO performs pattern matching with the compiled library (lib.svrf). The arguments to the CMACRO are the following:

      • pm:lib — References the compiled library

      • pm_layer1 — The layout layer to match

      • “ ” — CMACRO options (none in this case)

      • pm_Marker_out — Output layer from the default “Marker” output (the pattern extent)

      • pm_m_edit_out — Output layer from the custom marker “m_edit”

    3. Add the following code to the file edit_layout.svrf to write out the layers of interest to an OASIS file. 
      // optional, include other original design layers if 
// you want to write them out in the OASIS file
//LAYER layer2 2
//LAYER via    3

layout_out {
DFM RDB OASIS new_layout.oas
    pm_layer1     1 0  // the layer that is matched to
    pm_m_edit_out 1 10 // custom marker output
//    layer2        2 0  
//    via           3 0  
}
      // An OASIS file can also be written using DRC Check Map.
// pm_edit_out is both a layer and a rule check name
// DRC Check Map pm_m_edit_out OASIS 1 10 "map_layout.oas"
  8. Run Calibre with the new rule file:
    calibre -drc -hier edit_layout.svrf |& tee log

Results

For the layout shown in Figure 1 with the procedure “Capturing a Pattern From the Layout”, there are two matched locations, where one matched location is flipped across the y-axis. The file new_layout.oas looks like the following image, The two locations on the right are not exact matches to the pattern, so they are not corrected.



As part of the run, the pattern library is compiled and saved as the file lib.svrf, where lib.pmdb is the name of the pattern library. Other files are also created; see “Running Pattern Matching From the GUI” in the Calibre Pattern Matching User’s Manual.

See “Adding Constraints to Pattern Edges and the Output Marker” for instructions on adding constraints to this pattern so that the two layout locations on the right are matched.
