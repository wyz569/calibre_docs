Effects on Coordinates

When data is read into Calibre, layout database coordinates (dbu) are divided by the Precision statement value, and the resulting coordinate values are used in the Calibre database. If the Precision value and the Layout Precision value (reciprocal of the physical precision) are different, then scaling occurs.

When Layout Magnify is used, the coordinates are multiplied by the magnification factor. After all magnification effects are taken into account, the coordinates used in the Calibre database are rounded and truncated.

Assume the simple case where the reciprocal of the physical precision is the same as the rule file Precision and any magnification record within the layout database is 1.

For example, assume the following in the rule file:

PRECISION 1000      // no scaling
LAYOUT PRECISION 1000 
LAYOUT MAGNIFY 3    // M = 3

Assume the following layout coordinates:

(10345, 1412409) dbu

After magnification (multiplication by 3 in this case) there is this:

(31035,4237227) dbu

which is represented in the Calibre database.

Now, convert these coordinates back to the original magnification. Assume the rule file contains the following:

DRC RESULTS DATABASE PRECISION 1000
DRC MAGNIFY RESULTS 0.333

After magnification (multiplication by 0.333 in this case) truncation and rounding, the output coordinates are these:

(10335, 1410997) dbu

This implies grid coordinates can differ after application of certain magnification values. This can lead to undesirable outcomes.

A better choice for DRC Magnify Results in this case would be 0.33333333. The maximum number of significant figures for the coordinates is seven, so at least seven-place accuracy is needed to restore the original coordinates. In this case, the output coordinates are these after rounding and truncation:

(10345, 1412409)

which are the original values.

This line in the run transcript can assist in telling you how many significant figures you need for magnification values:

:--- DATABASE EXTENT = [ 0 , 0 ] -> [ x , y ]

The x and y values are given in user units, from which you can deduce the number of significant figures needed. In general, choosing at least one significant figure more than the maximum number of significant figures in the database extent is a good practice.

In the more complicated case where the physical precision and the rule file precision differ, and where there is a magnification record in the database for a given cell, then the final coordinates are multiplied by all these factors. Let C be the magnification record value in the layout, let S be the scaling factor as defined previously, and let M be the Layout Magnify value. Then this is what occurs for all coordinates:

Before read-in:

(x, y) user units (floating point)

After read-in:

(C*S*M*x, C*S*M*y) database units (integer, truncated and rounded)

Similar implications for grid snapping apply as discussed previously.
