Checking Contact Balance

You can check that the contacts in the source-drain regions on both sides of a gate are in balance.

Figure 1. Contact Balance Concept

The logic for this rule is implemented using conditional expressions and a function that is defined in a DFM Function statement.

    Try It!

Go to this page on Support Center to download the complete eKit.

This example is in the contact_via_checks/contact_balance example directory.

Procedure

  1. Define a DFM Function named “unbalanced” that accepts two numeric values and evaluates a condition. The function returns a 0 if the condition passes and a 1 if it fails. To do this, use a nested conditional expression:
    DFM FUNCTION [ unbalanced(NUMBER nmax, NUMBER nmin) 
   (nmin >= 2) ? (( nmax - nmin <= 1 ) ? 0 : 1 ) : 
   (( nmax == nmin ) ? 0 : 1 ) ]

    The two evaluations in the conditional expression read as follows:

    • If the minimum number of contacts for either source-drain region is at least 2, the difference in the number of contacts must be less than or equal to 1. If this statement evaluates true, return 0. If this statement evaluates false, return 1.

    • Else, if the minimum number of contacts for either source-drain region is less than 2, the number of contacts for both source-drain regions must be equal. If this statement evaluates true, return 0. If this statement evaluates false, return 1.

  2. Derive a layer that counts the number of contacts in each source-drain region using a DFM Property operation: 
    sdcont = DFM PROPERTY sd CON [N = COUNT(CON)]
  3. Derive a layer that identifies gates that do not pass the condition:
    contact_balance = DFM PROPERTY gate sdcont OVERLAP ABUT ALSO MULTI 
                  [ - = unbalanced(MAX(PROPERTY(sdcont, N)),
                        MIN(PROPERTY(sdcont, N)))] > 0

    This is how the rule check works:

    • A non-persistent DFM property is used to identify failing gates without attaching property values to them.

    • The source-drain regions are clustered with each gate that they overlap or abut. The MIN and MAX measurement functions and the PROPERTY access functions are used to pass the minimum and maximum number of contacts (the N property) to the DFM Function named “unbalanced” from Step 1.

    • A constraint value of 0 causes output when the unbalanced function returns 1.

  4. Output the results using the DFM RDB operation.
    eqdrc_out {
   DFM RDB contact_balance eqdrc.rdb CHECKNAME "%_l_" 
       CELL SPACE ALL CELLS  
}
