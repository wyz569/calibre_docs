Flat Device Counts From Rule Checks

You can obtain flat device counts from DRC-style rule checks. Flat counts include physical, non-reduced devices.

There are numerous layer operations that can be used for this purpose. This section shows commonly-used ones. To get device counts from rule checks, you need to know the seed layers from devices of interest.

Seed layers are most easily obtained from Device statements in the circuit extraction rule file, like this:

DEVICE mp (PMOS_VTL) pgate gate (G) psrcdrn (S) psrcdrn (D) nwell …
DEVICE mn (NMOS_VTL) ngate gate (G) nsrcdrn (S) nsrcdrn (D) psub …
DEVICE mp (PMOS_VTH) hvtpgate gate (G) psrcdrn (S) psrcdrn (D) nwell …
DEVICE mn (NMOS_VTH) hvtngate gate (G) nsrcdrn (S) nsrcdrn (D) psub …

The first layer shown in these statements is the seed layer that corresponds to the type and model (or subtype) of the device. If you do not have a circuit extraction rule file, then you will need to look at the layer derivation rules to determine what each seed layer is. With that information, you can proceed to write rule checks to determine the device counts.

As shown previously, MP devices of the PMOS_VTL model correspond to the pgate layer. The Copy operation can find devices per layer. DRC Check Map places the devices into their own results database.

// find the MP(PMOS_VTL) devices
MP_PMOS_VTL {COPY pgate}
DRC CHECK MAP MP_PMOS_VTL ASCII devices.db MAXIMUM RESULTS ALL

DRC SUMMARY REPORT drc.report HIER

The DFM Copy operation is also useful for this application and has some additional features.

For the preceding check, the devices.db results are hierarchical if the run is hierarchical. For a hierarchical run, you get the flat device counts from the DRC Summary Report as shown here:

--- RULECHECK RESULTS STATISTICS
---
...
RULECHECK MP_PMOS_VTL ............ TOTAL Result Count = 410       (125260)

The number in parentheses is the flat device count for the check.

The DFM Property and DFM RDB operations offer the ability to get the flat counts into a results database as properties; hence, you would not need to look at a report. This example groups MN and MP devices by type:

chip = EXTENT DRAWN

mn = DFM PROPERTY chip ngate hvtngate OVERLAP ABUT ALSO
      [mn_count=count(ngate)+count(hvtngate)] > 0
mp = DFM PROPERTY chip pgate hvtpgate OVERLAP ABUT ALSO
      [mp_count=count(pgate)+count(hvtpgate)] > 0

dfm_devices {DFM RDB mn devices.rdb mp NOPSEUDO ALL CELLS ABUT ALSO
 CHECKNAME "devices"}

The devices.rdb file will contain the mn_count and mp_count properties, which are the flat device counts for MN and MP devices, respectively.

If your run is comprised only of rule checks that find devices, you can get the total device count in the run transcript from this line:

--- TOTAL RESULTS GENERATED = 410 (125260)

This shows the hierarchical and flat counts. If the run is flat, only the flat count appears.
