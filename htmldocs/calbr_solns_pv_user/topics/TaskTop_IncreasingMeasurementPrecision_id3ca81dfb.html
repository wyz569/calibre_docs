Increasing Measurement Precision

Sometimes it is desirable to increase the measurement precision beyond the physical precision of the layout. This is used for DRC runs where a smaller measurement grid is desired than what the layout has inherently.

Prerequisites

  • A physical layout and DRC rule file.

  • Familiarity with the concepts discussed under “Considerations When Magnifying Data,” “Key Specification Statements for Magnification,” and “Magnification and Precision Management.”

Procedure

  1. Note:

    This procedure makes some assumptions about the physical precision and measurement precision. You can adjust the values in your rule file to be consistent with your own design.

    Place the following entries in your rule file to govern layout input:
    LAYOUT PRECISION 1000 // database unit is 1 nm.
PRECISION        4000 // measure to the nearest 0.25 nm;
                      // data is scaled by 1/4;
LAYOUT MAGNIFY   4    // place the data at original dimensions.
  2. The following statement is optional and governs output results. The rule file Precision is used by default.
    DRC RESULTS DATABASE PRECISION 4000 // matches PRECISION by default.
  3. [Optional.] Use Size … UNDEROVER with the new measurement dbu value on drawn layers to fill in possible gaps due to grid snapping:
    layer_snap = SIZE layer BY 0.00025 UNDEROVER

    Be aware this can have a performance penalty if it causes much layer merging across the hierarchy. Another alternative is to adjust measurement constraints by 2 dbu for layers subject to grid snapping.

Results

Results are presented in Calibre RVE using the same grid that you measured with, namely, 0.25 nm.

This method of output is not recommended:

DRC RESULTS DATABASE PRECISION 1000
DRC MAGNIFY RESULTS 0.25 // compensate for the magnification by 4.

It can lead to grid snapping on account of the loss of precision and affects the integrity of the results. As mentioned in the Procedure section, using Size … OVERUNDER is recommended.

When both Layout Magnify and DRC Magnify Results statements are used as shown in this procedure, the product of the two values should be as close to 1 as possible. If not 1 exactly, the product should give 9s to at least one decimal place beyond the maximum number of significant figures in the coordinates of your layout database. Suppose the maximum number of significant figures is 7. Suppose your Layout Magnify factor is 3. To restore the original coordinates, you should use DRC Magnify Results 0.33333333. The product gives 9s to the eighth significant figure, which gives better accuracy.
