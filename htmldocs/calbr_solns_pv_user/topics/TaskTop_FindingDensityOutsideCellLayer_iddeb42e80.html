Finding Density Outside of a Cell or Layer

The following procedure demonstrates how to perform density checking outside of a cell by using an exclusion layer.

You may want to perform this sort of Density operation in cases like these:

  • You have an IP block in your design, and it may be desirable not to perform density checking inside the placed block. Frequently, this is because the density specification for the block is different from the rest of the design.

  • You perform density checking within some cell and it is clean by itself. However, after placing the cell, density checking at a higher level may no longer be clean because of the placement location. Therefore, you do not want to check the placed cell to avoid false errors.

  • You may have a marker layer that is used to identify an exclusion area where density checking should not occur. This could apply in the case of a cell you want to exclude, as in the first two cases, or simply in some area of the design that is not cell-specific.

One advantage to writing the Density operations in the way shown in this section is you can have multiple Density operations using differing constraints (but otherwise using the same parameter values) and they all run concurrently.

Prerequisites

  • Thorough understanding of the Density AREA() function and math operators.

Procedure

  1. Write the following operation to derive an exclusion layer called keepout where density is not measured:
    keepout = EXTENT CELL "cell" ORIGINAL
  2. Write a density check using the keepout layer from Step 1, as follows:
    filldensity = DENSITY fill keepout < d 
              [(AREA(fill)/!AREA(keepout))/AREA()] 
              WINDOW w STEP s 
              RDB filldensity.rdb

filldensity { COPY filldensity }

    The d, w, and s parameters in this rule can be chosen for your process and set up as variables in the rule file.

    The expression (AREA(fill)/!AREA(keepout))/AREA() measures the density of the fill layer in the data capture window wherever the keepout layer is absent. The !AREA(keepout) function acts as a detector of whether the keepout layer is present. If keepout is present, !AREA(keepout) returns a 0. Division by 0 due to the !AREA(keepout) function does not satisfy the Density operation constraint, so any windows where this function is 0 are not output.
