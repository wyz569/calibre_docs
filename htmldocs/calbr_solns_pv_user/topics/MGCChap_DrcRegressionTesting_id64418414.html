DRC Regression Testing

Regression testing is employed to verify tool behaviors across software versions, to benchmark rule file performance, to compare layout versions, and so forth. This testing ensures consistency of mask generation across variable factors of the design process. Constructing a robust regression flow is therefore a common concern.

For DRC rule file regression testing, these things are assumed:

  • A set of “golden” rule checks is chosen as a baseline for regression comparison.

  • A “golden” DRC results database is generated using the chosen rule checks and saved for comparison against future results.

  • The DRC rule file can undergo revisions except for the baseline rule check names, as these names are used for layer comparison rather than original layer names.

  • Changes to rule checks (but not the names of the “golden” set of checks) can produce results that are flagged during layer comparison regression testing.

  • The design layout can undergo revisions. Any changes in the geometry that correspond to the reference results are flagged during layer comparison regression testing.

  • Edge layers or error cluster layers are converted and processed as polygon data to facilitate XOR comparison.

A primary goal of the rule file regression flow discussed in this chapter is to permit revision of the rule file (except baseline rule check names) while still producing useful results comparisons based upon a “golden” set of DRC results. To do this easily, the comparison must be based upon rule check names rather than layer numbers. Refer to the following figure.

In order to meet the aforementioned goal, ASCII DRC results databases are used to generate an XOR comparison rule file that contains layer names based upon rule check names. This mapping of rule check names to compared layer names is a key feature of the flow. The current layout database, golden DRC results database, and the regression DRC results database are combined into a single layout, and the rule check results are compared using the generated XOR rules.

Calibre DESIGNrev, Calibre nmDRC-H, and Calibre RVE are the primary tools for the flow.
