LVS Rule File Elements to Avoid

The following table shows rule file elements that should be avoided in most circumstances.
Table 1. LVS Rule File Elements to Avoid

Element to Avoid

Impact

Layer operations that cause device seed promotion.

Certain layer operations do not preserve layout hierarchy. When these operations are used to derive device body (seed) layers, this can cause seed promotion during connectivity extraction, which is undesirable. See “Hierarchy Preservation by Layer Operations” in the Calibre Verification User’s Manual for details.

Specifying LVS Push Devices SEPARATE PROPERTIES YES can help to mitigate seed promotion issues.

Shielded Connect or Sconnect statements such as these:

 

CONNECT a b c d BY e

SCONNECT f g h BY i

Connectivity statements that have more than two layers before the BY keyword cause shielding (see “Shielding” in the Calibre Verification User’s Manual). Shielded connectivity statements can have numerous negative side-effects. It is better to rewrite shielded connectivity statements like this:

CONNECT a b BY e

CONNECT a c BY e

CONNECT a d BY e

LVS Report Maximum ALL

The ALL setting can cause huge LVS reports that are unmanageable. The default setting is 50, which is fine for many situations. Try a setting closer to 300 if you want to see more discrepancies in the report.

Discrepancies in the LVS report are typically listed with the biggest problems first, with smaller problems later. Fixing the bigger problems often resolves the smaller ones.

LVS Report Option A C

Options A and C are best avoided unless you have good reasons to use them. They can cause huge LVS reports for little gain of information. Instead, use the DETAILED INSTANCE CONNECTIONS section of the report. If option FX is used, the DETAILED ERROR ANALYSIS section is useful to review.

enclosure_vector,

enclosure_parallel,

enclosure_perpendicular

These are Device property computation language functions used for calculating enclosure measurements. Use the “Device Annotation Built-In Language” functions instead.
