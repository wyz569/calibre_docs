Set Environment Variables with Calibre Interactive

You can use Calibre Interactive to set the environment variables that are in effect for the Calibre run and save those settings to a runset. Environment variables defined in the runset take precedence over environment variables set in the shell environment.

This method is useful for saving different run variations, since the runset saves the environment variables set within Calibre Interactive along with all the other Calibre Interactive settings. Once you have Calibre Interactive configured, you can save a runset and run Calibre Interactive in batch mode, as described in “Running Calibre Interactive in Batch Mode” in the Calibre Interactive User’s Manual.

The following example shows how to open and use the Set Environment Variables dialog box in Calibre Interactive. The rule file in this example has the following lines, where DRC_PROC is an environment variable expected to be defined in the shell:

// SVRF rule file
// de-reference shell environment variable with $
#ifdef $DRC_PROC 7b
   LAYER top_m metal1
#else 
    #ifdef $DRC_PROC 8a
        LAYER top_m metal3
    #else
          LAYER top_m metal2
    #endif
#endif

Prerequisites

  • Calibre Interactive is open. See “Running Calibre Interactive” in the Calibre Interactive User’s Manual.

Procedure

  1. Click Rules on the left panel and specify the rule file.
  2. (Optional) Click Load on the Rules pane to load settings from the rule file into the Calibre Interactive GUI fields.
  3. Select Setup > Set Environment to open the Set Environment Variables dialog box.


    The Set Env. tab displays environment variables used in the rule file or defined in the runset. Environment variables used in the rule file are indicated with a check mark in the “In Rules” column.

    The Current Env. tab displays the environment variables defined in the shell environment, subject to the filter defined in the Patterns field.

    In this example, the environment variable DRC_PROC is used in the rule file, but not defined in the environment; as shown in the following figure.



  4.  To define the DRC_PROC environment variable in the runset, do the following:
    1. Click in the Runset Value column and enter a value.
    2. Click the checkbox in the Runset column to select the runset value.
    3. Click OK to dismiss the dialog box.

  5. Set the rest of the Calibre Interactive options for the run.
  6. Select File > Save Runset to save the runset.
  7. Click Run DRC (or other Calibre application) to start the run.

    The runset value defined for DRC_PROC is used during the Calibre run.

    Tip

    Use File > Load Runset to load the runset, including the environment variable settings, into Calibre Interactive.

    The runset can also be used to run Calibre Interactive in batch mode, without opening the GUI.
