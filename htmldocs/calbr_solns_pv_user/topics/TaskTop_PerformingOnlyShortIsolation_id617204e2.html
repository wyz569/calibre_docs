Performing Only Short Isolation

In certain cases, it can be desirable to concentrate on performing layout short isolation over other circuit extraction tasks. With the appropriate rule file configuration and knowledge of Query Server commands, it can be faster to perform short isolation as discussed in this section.

Prerequisites

  • A physical layout.

  • An LVS circuit extraction rule file.

  • A Calibre nmLVS/Calibre nmLVS-H license pair, a Calibre RVE license, and a layout viewer supported by Calibre RVE.

Procedure

  1. Determine from your rule file the Device seed layer that defines the fewest devices in your design. Once that layer is found, declare it in an LVS Select Device By Seed Layer statement in your rules:
    LVS SELECT DEVICE BY SEED LAYER my_uncommon_device_layer

    This causes only the device instances to be extracted for the layer you have specified. (The entire Device set is not needed to perform short isolation using this method.) This usage saves runtime.

  2. Declare the following statements in your rules:
    LVS ISOLATE SHORTS NO // the default
    MASK SVDB DIRECTORY svdb SI // for interactive short isolation

    These settings provide enough information to isolate specific shorts using the Query Server without isolating all possible shorts. This saves time in getting to results.

  3. Run circuit extraction:
    % calibre -spice layout.sp -turbo [other options] rules |& tee log
  4. Review the circuit extraction report (.ext file extension) to determine shorts of primary interest for fixing.

    Shorts are listed by connectivity extraction text label name (or value), location in top-level space, and layer for a given net.

    WARNING:  Short circuit - Different names on one net:
          Net Id:  1
          (1)  name  "A2"  at location  (-25,30)  on layer  10
          (2)  name  "B"  at location  (-20,30)  on layer  10
          The name "A2" was assigned to the net.
  5. Start the Calibre Query Server Tcl shell:
    % calibre -qs -svdb svdb

...

-------------------------------------------------------------------
-------------------------------------------------------------------
---                       CALIBRE::LVS QUERY SERVER             ---
-------------------------------------------------------------------
-------------------------------------------------------------------
Opening database "svdb/TOP_CELL.phdb/db", revision "master"
Loading hierarchy and connectivity
Loading device info
>
  6. Execute the following command in the Tcl shell:
    > short_db::print_shorts

    The output gives similar information to the circuit extraction report, but is formatted differently. Here is one entry:

    Short circuit (serial number: 0) (index: 0) (cell: TOP_CELL) (net_id: 1) (text: A2)
    A2 at (-25,30)
    B at (-20,30)

    Notice the shorts are organized by serial number. Locate the serial numbers of shorts of interest from step 4.

  7. Using the serial number of a short from the previous step as an argument for the ‑sn option, execute the following command:
    > short_db::isolate_short -sn 0

    Successful short isolation returns a line in the transcript like this:

    FOUND_SHORT 1 REPORT_PATH svdb/TOP_CELL.phdb/isi/TOP_CELL-default-sn0.shorts

    Notice the path of the short isolation database, which is an ASCII DRC-style results database. (The FOUND_SHORT return string should be handled as a Tcl array in a script.)

  8. Open the short database in Calibre RVE from your layout viewer using whatever method you typically use (but do not simply open the Mask SVDB Directory, as this will show no shorts). Here is an example using Calibre DESIGNrev:
    % calibredrv layout.oas -rve -lvs svdb/TOP_CELL.phdb/isi/\
TOP_CELL-default-sn0.shorts

    The database opens in Calibre RVE in interactive short isolation mode.

  9. Proceed with resolving the short as discussed under “Viewing the Complete Short” and subsequent sections in the Calibre RVE User’s Manual.
