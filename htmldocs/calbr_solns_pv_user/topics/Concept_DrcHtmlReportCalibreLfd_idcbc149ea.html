DRC HTML Report for Calibre LFD

The following example shows a DRC HTML report created for Calibre LFD (Litho Friendly Design) results.

The report processes two results databases (RDBs), and selects only results involving active layers. The report creates a histogram of the PVI (Process Variability Index) property for the selected checks, and highlights one result per selected check in each results database. The number of highlights is limited for brevity. The report is shown in Figure 1.

Figure 1. DRC HTML Report for Calibre LFD

The following sections describe the command line and input files used to create the report.

Command Line

calibre -rve -drc -input lfd_output_map.txt -report rpt_lfd.ini 
	-outputdir batch_out

Input Files

lfd_output_map.txt — A DRC HTML Report Database Map File. This file specifies the results databases, the layout database, and a Calibre DESIGNrev layer properties file, as shown here: 

lfd_checks.rdb,,lfd_output.oas,lfd_output.oas.layerprops
lfd_pvbands.rdb,,lfd_output.oas,lfd_output.oas.layerprops

rpt_lfd.ini — The DRC HTML Report Configuration File. The text for this file is described in the section “Configuration File”.

Output Directory and Files

The HTML report is named index.htm and created in the output directory batch_out in this example. All supporting files are located in the output directory.

Configuration File

[Common]
DrvOptionsFilePath = ../
UseFrames = 1
LayoutHeight = 200
LayoutWidth = 250
ZoomLayoutFactor = .5
WorldView = 1
[Summary - active]
ReportType = DRCResultsSummary
GroupBy = Check
; only include *Active* checks
OrCheckFilter = *Active*
[histogram - active]
ReportType = DRCHistogram
GroupBy = [Summary - active]
Property = PVI
[Results]
ReportType = DRCResults
GroupBy = [Summary - active]
Highlight = 1
RowCount = 1
; turn off display of Coordinates column
ShowColumns = Coordinates 0
