Recommended LVS Rule File Statements

LVS in Calibre is performed in two primary phases: circuit extraction and circuit comparison. Performing circuit extraction depends upon your rule file containing the appropriate layer derivations, connectivity extraction statements, text recognition statements, and device recognition statements. Hence, considerable rule file setup is required to perform circuit extraction, some of which may be performed by your foundry.

The overhead for setting up the rule file for LVS comparison is small in comparison to circuit extraction. It is a good practice to begin running Calibre nmLVS as closely as possible to the default configuration. The default settings are well chosen for most comparison situations. However, there are numerous specification statements that cover less common situations, and you can make necessary adjustments to meet your needs.

The required LVS rule file statements are discussed under “Required Statements” in the Calibre Verification User’s Manual.

For a complete summary of LVS specification statements for use in the rule file, see “LVS Specification Statements” in the SVRF Manual.

This table shows recommended rule file statements that are not defaults and the circumstances for their use.

Table 1. Recommended LVS Rule File Statements

Statement

Usage

DEVice with device_annotation_program block.

The device annotation program block is especially useful when there are Device properties used for device characterization rather than LVS comparison. Annotation properties are calculated as DFM properties.

ERC Cell Name YES CELL SPACE XFORM

Specifies that ERC results have cell coordinates rather than global coordinates. A transformation between top-level space and cell space is provided so that you can shift back and forth in Calibre RVE.

ERC Summary Report HIER

Specifies that the ERC Summary report give results on a per-cell basis in addition to a per-rule-check basis.

Layout Base Layer

Specifies the names of device-forming layers as well as contact layers. This is a performance optimizing statement. It is important that well, substrate, via, interconnect metal, and marker or other special-purpose layers not be specified.

LVS Auto Expand Hcells PRESET [FILL]

Specifies that hcells are reported and expanded automatically during Calibre nmLVS-H circuit extraction. This is a performance optimizing statement.

For LVS runs after fill has been inserted, add the FILL keyword and specify Layout Dummy Layer with the fill layers. Specifying Exclude Cell for fill cells or LVS Execute ERC NO can also be helpful.

LVS Hcell Report

Specifies to produce a detailed hcell report. The report can be particularly useful if you are unsure of your hcell selection methodology, or if you are trying to optimize an hcell list.

LVS Cpoint

Specifies a correspondence point between layout and source. This command is especially useful for corresponding net names that differ in layout and source.

LVS Ground Name

Specifies the names of ground nets in the design. This statement is used for ERC, logic gate recognition, and other LVS modules.

LVS Power Name

Specifies the names of power nets in the design. This statement is used for ERC, logic gate recognition, and other LVS modules.

LVS Isolate Shorts {YES BY LAYER} | NO

If performing short isolation during circuit extraction, BY LAYER specifies to present the results for each layer. This allows you to highlight polygons in Calibre RVE on a per-layer basis, which is helpful in tracing shorts. There is also a BY CELL option that shows results per cell that is frequently useful.

For faster LVS turnaround time, specify NO and do short isolation as a separate step. See “Performing Only Short Isolation”.

LVS Push Devices SEPARATE PROPERTIES YES

Specifies that properties not used for LVS comparison be removed from devices to facilitate pushdown after promotion. This setting is generally recommended for all flows. It is required for the pushdown separated properties (PDSP) flow that uses the Calibre Connectivity Interface in a subsequent step to LVS.

LVS Report Option BX EB EC EO ES F FX NOK R S V W

Specifies the following LVS Report contents:

BX: List LVS Box cells.

EB: Bad devices in circuit extraction cause an INCORRECT LVS result.

EC: Sconnect conflicts in circuit extraction cause an INCORRECT LVS result.

EO: Open circuits in circuit extraction cause an INCORRECT LVS result.

ES: Short circuits in circuit extraction cause an INCORRECT LVS result.

F: Do not report smashed MOSFETS.

FX: Provide detailed fix suggestions for common problems.

NOK: Report contains comparison details and statistics only for incorrect cells. Particularly useful when larger hcell lists are used.

R: Disables reporting of ambiguity resolution points when the overall comparison result is CORRECT.

S: Sconnect conflict details appear in the run transcript and the circuit extraction report.

V: Reports virtual connections in the circuit extraction report.

W: Causes LVS-H to report an error when source and layout refer to the same file.

LVS Summary Report

Specifies that LVS generates a summary report file after a circuit comparison run. Used in conjunction with Mask SVDB Directory QUERY.

Mask SVDB Directory svdb QUERY [CCI]

The QUERY option specifies to write an LVS results database that can be loaded into Calibre RVE for debugging LVS discrepancies.

Use the CCI option in a PDSP flow or if you need the Calibre Connectivity Interface of the Query Server for post-processing.

For faster LVS turnaround time, you can omit Mask SVDB Directory from the rules. Specifying only the PHDB option can be useful in some cases where you need the Query Server for post-processing.

Virtual Connect Report YES

Specifies to report virtual connections when they apply.
