Insert a Boolean #DEFINE Based on an Environment Variable

This example demonstrates using the Calibre Interactive Customization GUI to place a #DEFINE pre-processor directive in the control file. A shell environment value is read to determine the initial value of the #DEFINE directive.

Suppose an SVRF rule file uses the following code with a pre-processor directive based on DENS_CHECKS:

// SVRF rule file
#ifdef DENS_CHECKS
metal1_density { 
    @ density metal1 greater than 50%, or 25%
    DENSITY metal1 < 0.5 WINDOW 50 RDB lab2_m1_density.rdb
}
#endif

In this example, DENS_CHECKS is not expected to be defined in the shell environment, since it is not preceded with a $ character. The following customization GUI code determines if the shell environment variable RUN_DENS_CHECKS is defined, then uses that information to place a #DEFINE or #UNDEFINE directive for DENS_CHECKS in the Calibre Interactive control file. 

# customization file

# RUN_DENS_CHECKS is a Boolean that may be set in the shell environment
if {[info exists ::env(RUN_DENS_CHECKS)]} { 
     CUSTOM::LABEL  -tool DRC \
       -prompt "RUN_DENS_CHECKS set in shell: $::env(RUN_DENS_CHECKS)"
   # create Boolean checkbox which defines or un-defines DENS_CHECKS
   # setting initialized to the RUN_DENS_CHECKS env var
   # command defines or un-defines DENS_CHECKS
  CUSTOM::DEFINE -name "DENS_CHECKS" -prompt "Run Density checks" \
    -tool DRC -select $::env(RUN_DENS_CHECKS) 
} else {
  CUSTOM::LABEL -tool DRC \
    -prompt "RUN_DENS_CHECKS not set in shell - define it here" 
  # create Boolean checkbox which defines or un-defines DENS_CHECKS
  CUSTOM::DEFINE -name "DENS_CHECKS" -select 0 \
    -prompt "Run Density checks" -tool DRC
}
