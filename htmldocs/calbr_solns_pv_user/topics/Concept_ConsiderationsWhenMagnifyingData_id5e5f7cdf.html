Considerations When Magnifying Data

Layout data uses a discrete coordinate system. There is a smallest unit of distance in a layout (called a database unit or dbu) that is indivisible. In other words, there is no infinitesimal approach (as in the real number system) to a given point in a layout.
Figure 1. Discrete Database Coordinates

All distances are a multiple of that database unit. For example, if the database unit is 1 nanometer, then the smallest measured distance in the coordinate system is 1 nm. There is no such thing as a 0.5 nm distance between vertices when the database grid unit is 1 nm.

This has important implications for layout magnification, because vertices must be snapped to the physical grid after magnification is performed. Magnification is also performed hierarchically, so cell placements themselves may snap and are also affected by any grid snapping that has occurred within cells they reference. Hence, polygon vertices and cell instances can end up in unexpected locations.
