Key Specification Statements for Magnification

Certain rule file specification statements govern how magnification is performed for input or output.

Before summarizing the specification statements, it is important to understand the following two terms:

  • Physical precision — This is the minimum fraction of a user unit represented in the layout database. Typically, the user unit is the micron (1E-06 meters) and the physical precision is 0.001 um. Hence, 1/1000 of a micron (or 1 nanometer) is the minimum fraction of a user unit represented in the layout and is the database unit. In the rule file, the Layout Precision statement (or the Layout Path PREC keyword) controls checking of the physical precision at runtime.

  • Rule file (measurement) precision — This is the number of database units per user unit. In Calibre, this is the reciprocal of the minimum fraction of a user unit for which measurements can be taken by layer operations in the rule file. For example, a precision specification of 1000 means measurements can be taken to the nearest 1/1000 of a user unit. In most cases, this means to the nearest 0.001 micron, or 1 nanometer. This is set with the Precision statement.

If the rule file precision and the physical precision agree, then the units are in a ratio of 1:1 and no scaling occurs. Suppose the rule file Precision is 10000 and the physical precision is 0.001. This implies the measured units versus physical units are in the ratio of 0.0001:0.001 or 1:10. In this case, the data is scaled by a factor of 1/10 (or shrunk by a factor of 10). Magnification effects such as these have to be taken into account in your flow.

The following table is a summary of statements that affect magnification.

Table 1. Specification Statements Affecting Magnification

Statement

Description

Input statements

Precision

Specifies the number of database units in a user unit. Default setting is 1000, which corresponds to a measurement precision of 0.001 user units. By default, having a rule file Precision that does not agree with the physical precision of the layout results in a fatal error. This can be overridden by various rule file settings.

Layout Precision

Specifies the reciprocal of the physical precision of the layout. This is used for verifying the layout precision only; it has no impact on measurements or output precision. Specifying this statement allows the rule file Precision to differ from the physical precision without causing an error. The same function is performed by the Layout Path PREC secondary keyword.

Layout Use Database Precision

Specifies that the physical precision of the layout be used for the rule file Precision value.

Layout Magnify

Specifies to multiply all coordinate values in the input database by a specified value. Used to expand or shrink input data. The same function is performed by the Layout Path MAG secondary keyword.

Layout Input Exception Severity

Specifies how certain exception conditions are handled (ignored, warning, fatal error, and so forth). The exceptions of special note in the context of magnification are these:

METRIC_INPUT_FILE — handles database unit consistency for multiple GDSII layout input files.

METRIC_RULE_FILE — handles agreement of the physical precision with the rule file Unit Length.

PRECISION_INPUT_FILE — handles physical precision consistency for multiple layout input files.

PRECISION_LAYOUT — handles agreement of the physical precision with the Layout Precision specified in the rule file.

PRECISION_RULE_FILE — handles agreement of the physical precision with the rule file Precision.

POLYGON_DEGENERATE — handles polygons that are not properly formed, possibly due to magnification or precision issues.

POLYGON_ZERO_AREA — handles the special class of improper polygons having zero area.

Output statements

DRC Results Database Precision

Sets the physical precision of the DRC results output. The reciprocal of the output physical precision is specified in the statement. By default, this setting is consistent with the rule file Precision. If the output physical precision is less than the input physical precision, data is shrunk. Conversely, if the output physical precision is greater than the input physical precision, data is expanded.

DRC Magnify Results

Specifies to multiply all coordinate values in the Calibre database by a specified value before output to the DRC Results Database. Can be used to expand or shrink mask output data.

DRC Magnify Density

Applies a magnification to Density RDB results upon output. This functions independently from DRC Magnify Results.

DRC Magnify NAR

Applies a magnification to Net Area Ratio RDB results upon output. This functions independently from DRC Magnify Results.
