Using Local Property Measurements

Local property measurements use seed layer declarations in Device property computation language function calls. A local measurement can use a seed layer, or a seed layer and a pin layer (the pin layer may be referenced by layer name or pin name), or a seed layer and an auxiliary layer. Using seed layers in function calls extracts the device properties in the immediate context of the device seed shapes; hence, the measurements are local.

Non-local measurements, in contrast to local measurements, do not use a seed layer at all. Such measurements are usually inefficient and should be avoided.

Procedure

  1. Identify the function calls that do not use local measurements. These measurements do not reference seed layers.
    gate = poly AND diff
pgate = gate AND pdiff

DEVICE MP pgate poly(G) psd(S) psd(D) nw(B) <aux>

[
  PROPERTY w, l, z
  z = perimeter_coincide(G, aux)
  ...
]

    Here, the G pin is associated with the poly layer. Poly can have large polygons that traverse the hierarchy. Using G in a function call like this can cause flattening of the poly layer. This should be avoided.

  2. Change the Device property computation function to use a device seed layer instead. 
    DEVICE MP pgate poly(G) psd(S) psd(D) nw(B) <aux>

[
  PROPERTY w, l, z
  z = perimeter_coincide(pgate, aux)
  ...
]

    In this case, pgate is associated with a gate seed shape.
