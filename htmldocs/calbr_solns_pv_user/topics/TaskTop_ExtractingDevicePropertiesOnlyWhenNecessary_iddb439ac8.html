Extracting Device Properties Only When Necessary

Many device properties are used in post-layout simulation but are not used in regular LVS debugging. Many of these properties are costly to extract in terms of LVS run time and memory consumption.

It is a good idea to avoid extracting these properties in regular LVS runs. In addition, these properties often require special auxiliary layers in Device operations. It may be important to ignore these auxiliary layers in regular LVS runs.

To achieve these goals, you can use pre-processor directives (also called conditionals).

Procedure

  1. Identify the post-layout simulation properties and auxiliary layers (if any) in a Device statement. In this case, the highlighted auxiliary layers and properties are used for PEX simulation.
    DEVICE MP pgate pgate(G) psd(S) psd(D) nw(B) <prop_layer_1>
  <prop_layer_2>

[
  PROPERTY w, l, as, ad, ps, pd
  ...
]
  2. Change the Device statement to use conditional compilation. In this case, if the variable $PEX is set in the shell environment, then the #IFDEF blocks are compiled.
    DEVICE MP pgate pgate(G) psd(S) psd(D) nw(B)
  #IFDEF $PEX
// these layers for parasitic extraction only
  <prop_layer_1> <prop_layer_2>
  #ENDIF
[
  PROPERTY w, l
  #IFDEF $PEX
// these properties for parasitic extraction only
  , as, ad, ps, pd
  #ENDIF
  ... 
]
