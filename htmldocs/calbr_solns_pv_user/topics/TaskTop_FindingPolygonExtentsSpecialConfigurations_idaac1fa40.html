Finding Polygon Extents for Special Configurations

In certain cases, polygons on a layout can be arranged in such a way that finding their extents cannot be done using a single Extents operation.

In the following figure, the extents of the two MET1 polygons overlap, but the polygons themselves do not. Since the output of the Extents operation is merged, the extents of the individual polygons cannot be determined.

Figure 1. Overlapping Polygon Extents

The DFM Property operation supports the application of the :EXTENT_EDGE layer derivation to the input layer of measurement functions. This can be used to calculate the x- and y- extent of each polygon on the input layer and attach the values as DFM properties on each polygon.

In addition, the DFM Property Select Secondary operation can use the :EXTENT layer derivation to output an unmerged polygon layer that contains the extent bounding box of each shape on the input layer. Unmerged polygons layers are supported by a limited number of operations.

The following procedure uses both approaches (DFM Property and DFM Property Select Secondary).

Procedure

  1. Use DFM Property to calculate the polygon extent in the x- and y-directions. The output layer is a copy of the input layer with DFM properties attached.

    The :EXTENT_EDGE layer derivation returns the edges of the bounding box of the object on the input layer. The edges are returned as an error layer. The ECX and ECY measurement functions calculate the total x and y lengths of the input edges. 

    MET1_extent_prop = DFM PROPERTY MET1 
                       [horizlength = ECX(MET1:EXTENT_EDGE)/2]
                       [vertlength  = ECY(MET1:EXTENT_EDGE)/2]
  2. Use DFM Property Select Secondary to output the extent bounding boxes as an unmerged polygon layer. The SELECT keyword specifies the MET1 layer with the :EXTENT layer derivation. For this example the horizontal and vertical lengths are calculated using the :EXTENT layer derivation and the PERIMX and PERIMY measurement functions. 
    MET1_bbox_unmerged = DFM PROPERTY SELECT SECONDARY MET1 
                         [horizlength = PERIMX(MET1:EXTENT)/2]
                         [vertlength  = PERIMY(MET1:EXTENT)/2]    
                         SELECT MET1:EXTENT
  3. Output the results with a DFM RDB operation.
    met1_extent {
      DFM RDB MET1_extent_prop advDRC.rdb CHECKNAME "%_l_" 
          CELL SPACE ALL CELLS 
      DFM RDB MET1_bbox_unmerged advDRC.rdb CHECKNAME "%_l_" 
          CELL SPACE ALL CELLS 
}
