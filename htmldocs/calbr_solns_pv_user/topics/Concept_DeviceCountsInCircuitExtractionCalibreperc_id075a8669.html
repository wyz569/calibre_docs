Device Counts in Circuit Extraction and Calibre PERC

Flat device counts can be obtained from a circuit extraction run through ERC checks, the circuit extraction report, the Query Server, or Calibre PERC rule checks.

ERC checks can be run as a part of circuit extraction (as in calibre -spice). The checks discussed under “Flat Device Counts From Rule Checks” can also be run as ERC checks. Additionally, the Device Layer operation can be used in an ERC check to find devices by type and subtype, like this:

// find the MP(PMOS_VTL) devices
MP_PMOS_VTL {DEVICE LAYER MP(PMOS_VTL)}

// or find all MP devices
// "MP" {DEVICE LAYER MP}

ERC SELECT CHECK MP_PMOS_VTL
ERC RESULTS DATABASE erc.db
ERC SUMMARY REPORT erc.report HIER

For the preceding check, the erc.db results are hierarchical if the run is hierarchical. For a hierarchical run, you can get the flat device count from an ERC Summary Report as shown here:

RULECHECK MP_PMOS_VTL ........... TOTAL Result Count = 410       (125260)

The number in parentheses is the flat device count for the check.

A circuit extraction run (which can also perform ERC) produces a layout SPICE netlist. In the top-level subcircuit definition, there is an FDC annotation in a comment, like this:

.SUBCKT top gnd vdd vddio vdda out_o clka_i vref_i in_i clk_i reset_i
** N=1164 EP=10 IP=2703 FDC=248921

This is the total flat device count of the design.

If you specify a Mask SVDB Directory QUERY statement in your circuit extraction rule file, you can use the Query Server qs::device_templates_used or DEVICE TEMPLATES USED commands to find the hierarchical and flat device counts by type and subtype. Here is an example session from the standard Query Server shell:

% calibre -query svdb
...
INITIATING HDB QUERY SERVER:
--------------
NOTE: Cross reference commands will be unavailable since a valid cross 
reference (XDB) database was not found.
   OK: Ready to serve.
DEVICE TEMPLATES USED
Device_Templates_Used 2000
Templates:
0 0 8 Oct 15 12:11:13 2013                    
mp(pmos_vtl) 0 410 (125260) <--final two numbers are hier. and flat counts
mn(nmos_vtl) 1 334 (121946)
mp(pmos_thkox) 4 91 (729)
mn(nmos_thkox) 5 147 (719)
c 6 2 (34)
r(rp) 7 3 (37)
r 8 2 (180)
END OF RESPONSE
0 0 0 Oct 15 12:11:13 2013

(You can ignore the note about the lack of an XDB database because running LVS comparison is not necessary to get these results.)

For another connectivity-based solution, you can get device counts by using Calibre PERC. See the DEVICE_COUNT high-level check or Example 2 under perc::check_data in the Calibre PERC User’s Manual for rule check code.
