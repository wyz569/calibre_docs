Verifying Contact Placement for Current Crowding

You can check for current crowding effects based on the number of contacts and their positions in the source and drain regions.

Note:

This rule check generates and processes non-rectangular (skew) polygons. In general, processing large numbers of skew polygons is not recommended, as it can degrade performance.

The rules set developed in this procedure mimics the actual flow of current between contacts across one or more gates by producing polygons on each gate. The current that flows through each gate is represented visually by the area of each of these polygons. See the following figure.

This example is in the contact_via_checks/current_crowding_con_placement example directory.

Procedure

  1. Derive the gate layer, source-drain (sd) layer, and the contacts in the source and drain regions.
    gate = POLY AND AA 
sd = (AA NOT POLY) INTERACT gate 
con_and_sd = CON AND sd
  2. Derive the source and drain regions that contain one or more contacts.
    sd_w_con = DFM PROPERTY sd con_and_sd 
                    [- = COUNT(con_and_sd)] > 0
  3. Derive an edge layer containing the contact edges that face each gate. Use a constraint based on the minimum gate spacing. 
    VARIABLE min_gate_spacing 0.11
a1 = EXT [con_and_sd] gate < 1.5*min_gate_spacing OPPOSITE
  4. Derive an error layer containing pairs of contact edges that span across one or more gates (that is, one or more gates are between the edges in an edge pair). The constraint distance should be at least twice min_gate_spacing plus the gate width. Use a larger constraint distance to include results such as the lower right and center results in Figure 1.
    a2 = EXT a1 < 8*min_gate_spacing
  5. Only consider a2 edge pairs that cross only one gate and do not span more than one active area region.
    a3 = DFM PROPERTY a2 sd_w_con gate AA REGION OVERLAP MULTI 
       [- = COUNT(sd_w_con)] == 2 // pairs that span only two sd 
                                     // regions with contacts, that is,
                                  // cross only one gate.
       [- = COUNT(gate)] > 0  // pairs that cross at least one gate
       [- = COUNT(AA)] == 1   // pairs that cross only one AA region
  6. Derive the gate overlap regions.
    a4 = DFM COPY a3 REGION 
a5 = a4 AND gate
  7. Expand gate regions to account for current spreading effects.
    a6 = SIZE a5 BY 0.08 INSIDE OF gate
  8. For each gate, use a DFM Property operation to calculate the percentage of the gate not covered by the overlap region.
    gate_current = DFM PROPERTY a6 gate OVERLAP MULTI
    [pct_w_loss = ROUND(100*(AREA(gate)-AREA(a6)/AREA(gate)),0.1)]
  9. Output the results of the DFM Property operation using DFM RDB.
    gate_current {
   DFM RDB gate_current eqdrc.rdb CHECKNAME "%_l_" 
       CELL SPACE ALL CELLS
}
