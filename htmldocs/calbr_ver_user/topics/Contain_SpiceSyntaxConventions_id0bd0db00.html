SPICE Syntax Conventions

Certain syntactical conventions are used for SPICE that are distinct from the syntax conventions used elsewhere in the book.

These conventions apply to the SPICE element syntax shown in this chapter.

Table 1. SPICE Netlist Notational Conventions

Syntax

Description

UPPERCASE

Uppercase letters indicate literal keywords.

lowercase

Lowercase letters indicate arguments to be substituted by other values.

< >

Indicates an optional argument.

+

SPICE continuation character used when the syntax description spans across several lines.

<ARG=val>

Unless indicated otherwise, optional parameters preceded by literal names can appear in any order.

<val>

Parameters not preceded by literal names must appear in exactly in the orders shown relative to other such parameters as defined under “Element Statements”.
