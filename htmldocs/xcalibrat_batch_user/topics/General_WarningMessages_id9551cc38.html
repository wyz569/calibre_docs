Warning Messages

Warning messages should be reviewed to determine if they indicate a real problem in your MIPT file.

Warning messages have the following format:

WARNING: warning_message
Table 1. xCalibrate Warning Messages

Message

Possible Causes

No SEED layers exist in the MIPT file

A poly layer is defined without an associated seed layer. Make sure the poly layer definition contains a seed_layers parameter. See seed_layers for details.

No SRC_DRN layers exist in the MIPT file

A src_drn layer is defined without an associated diffusion layer. Make sure the diffusion layer definition contains a src_drn_layers parameter. See src_drn_layers for details.

Layer layer_name. has no extension.

The extension parameter is missing from a diffusion, src_drn, poly, or seed layer, layer_name. See extension parameter for details.

Ignore the ignore_gateext_to_diff parameter. SVRF is generated based on ignore_endcap parameter setting.

The ignore_gateext_to_diff parameter is set to yes and ignore_endcap parameter is set to yes for the same seed or pcaux layer.

NO POLY CONTACT LAYERS IN MIPT FILE FOR layername.

The poly layer layername does not have an associated contact or via layer. Make sure the poly layer name has been specified as either the measured_from or measured_to value for a via or contact layer in your MIPT file.

“layername” ronly layer is referenced but not defined in MIPT file. It will inherit its properties from the parent layer “layername”.

A layername that does not exist was specified in the ronly_layers parameter in the layer definition. This ronly layer will use the properties from the layer in which is was defined.

Table (name,property) : (table_name, sidewall_k) for layer: layername has invalid values for spacing. The values must be in ascending order.

For SIDEWALL_K table type only: if the values for width or spacing are not in ascending order, these values are automatically put in ascending order during calibration and this warning is generated.

For all other table types, an error is generated. See “Error Messages”.

RSD of diffusion layer: “layername” does not touch device_li layer: “layername”.

The bottom of device_li layer layername does not touch or is not aligned with the top of the RSD layer. Make sure the zbottom of the device_li layer is the same as the ztop of the RSD layer.

layer “layername” overlaps with the Base layer “layername”.

The base layer is a reference layer for all layers above it. Make sure there are no layers overlapping with the base layer in the stack. See Base layer for details.

Placed Seed (layername) does not have dielectric conformal around it while its parent poly (layername) has dielectric conformal (layername) around it.

The placed seed layer specified by the ref_layer keyword in the parent poly layer does not have it’s own conformal dielectric layer. To use the same dielectric for the placed seed layer that is specified for the parent poly, specify another conformal dielectric layer with the same properties, except set the ref_layer keyword to the placed seed layer name. See dielectric layer for an example.

Devices keyword is used in derived seed layer “layername” definition. Please move the devices keyword definition to parent layer “layername”.

The derived layer type does not allow the devices parameter. Specify the devices parameter in the derived layer’s parent layer. See devices parameter for details.

No contact is defined for src_drn layer “layername”.

A corresponding contact layer must be defined for the src_drn layer. See src_drn layer syntax and contact layer syntax for details.

Seed layer “layername” is referenced in two parents: “layername” and “layername”. Please remove seed layer “layername” from one of the parent layers.

A layername specified in the seed_layers parameter appears in more than one layer definition.

The parameter order in table “tablename” is (index2, index1), which is unexpected. Table values are functions of (index1, index2). The parameter order for table “tablename” is switched to (index1, index2) in this calibration.

The parameters (index2, index1) specified for the table tablename are not in the correct order. The parameter order will be reversed during calibration to ensure the correct output values are calculated. To avoid this warning correct the order of the parameters in tablename.

The parameter order in table “tablename” is (index2, index1), which is unexpected. Table values are functions of (index1, index2). If this is not intended, please switch the table parameter order to (index1, index2).

The parameters (index2, index1) specified for the table tablename are not in the expected order. To avoid this warning correct the order of the parameters in tablename.

ignore_diff_intrinsic is set to “NO” for layer: layername.

This warning is generated when the parameter ignore_diff_intrinsic is not set to the default (YES).

ignore_diff_to_diff_under_poly is set to “NO” for layer: layername.

This warning is generated when the parameter ignore_diff_to_diff_under_poly is not set to the default (YES).

ignore_gate_to_diff is set to “NO” for layer: layername.

This warning is generated when the parameter ignore_gate_to_diff is not set to the default (YES).

ignore_gateext_to_diff_only is set to “YES” for layer: layername.

This warning is generated when the parameter ignore_gateext_to_diff_only is not set to the default (NO).

Duplicate Vias measured from layername to layername.

This warning is generated when more than one via layer definition has the same measured_from and measured_to layer names specified.
